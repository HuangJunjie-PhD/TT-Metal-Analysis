This page documents the runtime configuration system (`RunTimeOptions`) that controls TT-Metal behavior through environment variables and programmatic APIs. Runtime options affect compilation, debugging, profiling, hardware features, and system initialization without requiring code recompilation.

* * *

## Overview and Purpose

The `RunTimeOptions` class [[tt_metal/llrt/rtoptions.hpp 153-330](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/llrt/rtoptions.hpp#L153-L330)] provides a centralized configuration system that:

*   **Reads environment variables** during initialization to configure Metal's runtime behavior.
*   **Provides typed accessors** for all configuration options with validation.
*   **Controls subsystem initialization** including watcher, profiler, inspector, and fabric.
*   **Manages build artifacts** through cache and kernel path configuration.
*   **Enables/disables features** dynamically without recompiling Metal or user code.

The runtime options are initialized once during system startup and accessed throughout the Metal stack via `MetalContext::instance().rtoptions()`[[tt_metal/impl/context/metal_context.cpp 159-161](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/impl/context/metal_context.cpp#L159-L161)].

### Kernel Execution Control

Options that modify kernel behavior for testing and debugging:

*   `TT_METAL_NULL_KERNELS`: Disable kernel execution entirely (stubs) [[tt_metal/llrt/rtoptions.hpp 212](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/llrt/rtoptions.hpp#L212-L212)].
*   `TT_METAL_KERNELS_EARLY_RETURN`: Kernels return immediately after initialization [[tt_metal/llrt/rtoptions.hpp 213](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/llrt/rtoptions.hpp#L213-L213)].

**Implementation**: These flags affect JIT compilation by defining preprocessor symbols such as `WATCHER_ENABLED` or `PROFILE_KERNEL` in the device firmware/kernel build environment [[tt_metal/jit_build/build.cpp 160-210](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/jit_build/build.cpp#L160-L210)].

### Debug Print (DPRINT) Configuration

DPRINT allows selective printf-style debugging from device kernels with fine-grained targeting.

*   `TT_METAL_DPRINT_CORES`: Enable DPRINT on specific worker cores [[tt_metal/llrt/rtoptions.cpp 641](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/llrt/rtoptions.cpp#L641-L641)].
*   `TT_METAL_DPRINT_CHIPS`: Specify chips by physical ID [[tt_metal/llrt/rtoptions.cpp 641](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/[tt_metal/llrt/rtoptions.cpp#L641-L641)].

## Summary

The `RunTimeOptions` system provides a comprehensive configuration framework for TT-Metal that:

1.   **Centralizes configuration** through environment variables with typed accessors.
2.   **Controls subsystem initialization** for debugging, profiling, and hardware features.
3.   **Affects JIT compilation** by generating appropriate defines and flags.
4.   **Supports multi-device configurations** with fabric topology and telemetry settings.
5.   **Provides programmatic APIs** for runtime modification of select settings.
