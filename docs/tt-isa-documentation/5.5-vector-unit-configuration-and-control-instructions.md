This page documents the Vector Unit (SFPU) instructions that configure execution behavior, manipulate control state, and enable advanced instruction-level parallelism. These instructions do not perform arithmetic computation directly, but instead set up the execution environment and control how other Vector Unit instructions execute.

For arithmetic and logical compute instructions, see [Vector Unit Core Instructions](/tenstorrent/tt-isa-documentation/5.3-vector-unit-core-instructions). For data movement between `Dst` and `LReg`, see [Vector Unit Data Movement Instructions](/tenstorrent/tt-isa-documentation/5.4-vector-unit-data-movement-instructions).

## Overview

The Vector Unit provides three categories of configuration and control instructions:

1. **Configuration Manipulation**: `SFPCONFIG` for setting `LaneConfig`, `LoadMacroConfig`, and special `LReg` values
2. **Advanced Scheduling**: `SFPLOADMACRO` for loading data plus scheduling up to 4 additional instructions with precise timing control
3. **Vector Reorganization**: `SFPSHFT2` for cross-lane shuffles and bitwise shifts
4. **Conditional Control**: Instructions like `SFPIADD` and `SFPPOPC` that manipulate lane flags and conditional execution state

These instructions enable sophisticated control flow, instruction-level parallelism, and data reorganization patterns that maximize Vector Unit throughput.

## LaneConfig Register

`LaneConfig` is an 18-bit per-lane register that controls various aspects of Vector Unit instruction execution. `SFPCONFIG` with `VD=15` manipulates this register.

### LaneConfig Bit Fields

| First Bit | # Bits | Name | Purpose |
| --- | --- | --- | --- |
| 0 | 1 | `ENABLE_FP16A_INF` | Controls FP16 infinity interpretation in `SFPLOAD` |
| 1 | 1 | `DISABLE_BACKDOOR_LOAD` | When `false`, instructions with `VD ≥ 12` write to `LoadMacroConfig.InstructionTemplate[VD-12]`; set `true` to execute `SFPSTORE`/`SFPSWAP` with `VD ≥ 12` |
| 2 | 1 | `ENABLE_DEST_INDEX` | Causes `SFPSWAP` to perform argmin/argmax instead of min/max |
| 3 | 1 | `CAPTURE_DEFAULT_DEST_INDEX` | With `ENABLE_DEST_INDEX`, causes `SFPLOAD` to perform 2nd `LReg` write containing `Dst` index |
| 4 | 1 | `BLOCK_DEST_WR_FROM_SFPU` | When `true`, `SFPSTORE` will not write to `Dst` |
| 5 | 1 | `BLOCK_SFPU_RD_FROM_DEST` | When `true`, `SFPLOAD` will not write to `LReg` |
| 6 | 1 | `DEST_RD_COL_EXCHANGE` | When `true`, `SFPLOAD` always loads from odd columns of `Dst` |
| 7 | 1 | `DEST_WR_COL_EXCHANGE` | When `true`, `SFPSTORE` always stores to odd columns of `Dst` |
| 8 | 1 | `EXCHANGE_SRCB_SRCC` | Inverts `SFPSWAP` comparison result (computing max/min instead of min/max) |
| 9 | 2 | `BLOCK_DEST_MOV` | Disables individual columns during `MOVA2D`, `MOVB2D`, `MOVD2A`, `MOVD2B` |
| 11 | 1 | Reserved |
| 12 | 4 | `ROW_MASK` | Used in lane predication mask computation |
| 16 | 2 | Reserved |

## LoadMacroConfig Structure

`LoadMacroConfig` enables `SFPLOADMACRO` to schedule up to 4 instructions for future execution with precise timing control. Each of the 32 lanes has its own `LoadMacroConfig` (though software typically configures all lanes identically).

### Sequence Entry Format

Each `Sequence[i]` contains 32 bits encoding 4 sub-units (Simple, MAD, Round, Store), with 8 bits per sub-unit:

| Bits | Purpose |
| --- | --- |
| [2:0] | Template selector: 0=none, 1=undefined, 2=SFPNOP, 3=SFPSTORE with VD=0, 4-7=InstructionTemplate[0-3] |
| [5:3] | Delay: 0-6 cycles/instructions delay, 7=don't schedule |
| [6] | VD override: 0=use VD from SFPLOADMACRO, 1=use VD=16 |
| [7] | Input register override: 0=VC from SFPLOADMACRO, 1=VB from SFPLOADMACRO |

## SFPCONFIG Instruction

`SFPCONFIG` is a versatile instruction with multiple modes determined by the `VD` field. It always takes input from the first 8 lanes and broadcasts to all 32 lanes.

### SFPCONFIG Modes by VD

### SFPCONFIG Input Sources

| Mod1 Flag | Input Source |
| --- | --- |
| `MOD1_IMM16_IS_VALUE=0` | Value from `LReg[0][Lane & 7]` |
| `MOD1_IMM16_IS_VALUE=1` | For `VD=11-14`: hardcoded default constants For `VD=4-8,15`: value from `Imm16` field |

When manipulating `LoadMacroConfig.Misc` or `LaneConfig`:

| `MOD1_BITWISE_XOR` (6) | XOR: `Target ^= Value` |

## SFPLOADMACRO Instruction

`SFPLOADMACRO` combines data loading with instruction scheduling, enabling the Vector Unit to execute up to 5 instructions per cycle: one load instruction plus up to 4 scheduled instructions (one per sub-unit).

### Execution Model

### Sub-Unit Compatibility

`SFPLOADMACRO` can schedule instructions to 4 independent sub-units:

| Sub-Unit | Compatible Instructions |
| --- | --- |
| **Simple** (max 1) | `SFPABS`, `SFPAND`, `SFPCAST`, `SFPCOMPC`, `SFPCONFIG`, `SFPDIVP2`, `SFPENCC`, `SFPEXEXP`, `SFPEXMAN`, `SFPIADD`, `SFPLZ`, `SFPMOV`, `SFPNOP`, `SFPNOT`, `SFPOR`, `SFPPOPC`, `SFPPUSHC`, `SFPSETCC`, `SFPSETEXP`, `SFPSETMAN`, `SFPSETSGN`, `SFPSHFT`, `SFPSWAP`, `SFPTRANSP`, `SFPXOR` |
| **MAD** (max 1) | `SFPADD`, `SFPADDI`, `SFPLUT`, `SFPLUTFP32`, `SFPMAD`, `SFPMUL`, `SFPMULI`, `SFPNOP` |
| **Round** (max 1) | `SFPNOP`, `SFPSHFT2`, `SFPSTOCHRND` |
| **Store** (max 1) | `SFPSTORE` |

### Delay Timing Modes

Instructions can be scheduled with two different delay timing modes:

| Mode | Behavior |
| --- | --- |
| `WaitForElapsedCycles` | Delay counter decrements every cycle |
| `WaitForElapsedInstructions` | Delay counter decrements when thread issues Vector Unit instruction |

When **any** scheduled instruction uses `WaitForElapsedInstructions`, **all** scheduled instructions use that mode. This is useful for scheduling instructions that consume results from future instructions.

## SFPSHFT2 Instruction

`SFPSHFT2` performs vector shuffles and bitwise shifts, executing on the Round sub-unit. The behavior is determined by the `Mod1` field.

### SFPSHFT2 Operation Modes

| Mod1 Value | Mode Name | Operation |
| --- | --- | --- |
| 0 | `SFPSHFT2_MOD1_COPY4` | Within each lane: `L0→L1→L2→L3→0` |
| 1 | `SFPSHFT2_MOD1_SUBVEC_CHAINED_COPY4` | Copy4 + shift L0 left by 8 lanes into L3 |
| 2 | `SFPSHFT2_MOD1_SUBVEC_SHFLROR1_AND_COPY4` | Copy4 + rotate VC right by 1 lane (within 8-lane groups) into L3 |
| 3 | `SFPSHFT2_MOD1_SUBVEC_SHFLROR1` | Rotate VC right by 1 lane within 8-lane groups → VD |
| 4 | `SFPSHFT2_MOD1_SUBVEC_SHFLSHR1` | Shift VC right by 1 lane within 8-lane groups → VD |
| 5 | `SFPSHFT2_MOD1_SHFT_LREG` | Bitwise shift VB by amount in VC → VD |
| 6 | `SFPSHFT2_MOD1_SHFT_IMM` | Bitwise shift VB by immediate amount → VD |

### Wormhole Hardware Bug in SUBVEC\_SHFLSHR1

In Wormhole, `SFPSHFT2_MOD1_SUBVEC_SHFLSHR1` has a bug where lane 0 (and 8, 16, 24) should receive `0` but instead receives `vc0[Lane + 7]`, where `vc0` is the VC value from the most recent `SFPSHFT2_MOD1_SUBVEC_SHFLROR1_AND_COPY4` or `SFPSHFT2_MOD1_SUBVEC_SHFLROR1` instruction.

**Workaround**: Execute `SFPSHFT2_MOD1_SUBVEC_SHFLROR1` with both VD and VC set to 9 before using `SFPSHFT2_MOD1_SUBVEC_SHFLSHR1` to ensure `vc0` is zero.

This bug is **fixed in Blackhole**.

### Instruction Scheduling Constraints

**Wormhole**: After using modes 1, 2, 3, or 4, software must insert `SFPNOP` on the next cycle to avoid data hazards:

* Mode 2: Next instruction cannot read `LReg[0-3]` or write `LReg[1-3]`
* Modes 3-4: Next instruction cannot read `LReg[VD]`
* Modes 1-4: Next instruction cannot be from a specific list of simple sub-unit instructions

**Blackhole**: Hardware automatically stalls for one cycle after modes 1, 2, 3, or 4. Software can optionally insert `SFPNOP` to reduce the sequence from 3 cycles to 2 cycles.

**Caution**: Automatic stalling does **not** apply to `SFPSHFT2` instructions scheduled via `SFPLOADMACRO`.

### SFPPOPC for Stack Manipulation

`SFPPOPC` manipulates the 8-deep `FlagStack`:

| Mod1 | Operation |
| --- | --- |
| 0 | Pop stack, restore `LaneFlags` and `UseLaneFlagsForLaneEnable` |
| 1-12 | Peek stack top, perform boolean operation with current `LaneFlags` |
| 13 | Invert `LaneFlags` |
| 14 | Set `UseLaneFlagsForLaneEnable=true`, `LaneFlags=true` |
| 15 | Set `UseLaneFlagsForLaneEnable=true`, `LaneFlags=false` |

## Summary of Key Instructions

| Instruction | Primary Purpose | Execution Unit |
| --- | --- | --- |
| `SFPCONFIG` | Manipulate `LaneConfig`, `LoadMacroConfig`, and special LRegs | Simple sub-unit |
| `SFPLOADMACRO` | Load data + schedule up to 4 additional instructions | Load sub-unit |
| `SFPSHFT2` | Vector shuffle and bitwise shift operations | Round sub-unit |
| `SFPIADD` | Integer arithmetic with flag setting | Simple sub-unit |
| `SFPPOPC` | Flag stack manipulation for conditional execution | Simple sub-unit |
| `SFPPUSHC` | Push flags onto stack | Simple sub-unit |
| `SFPSETCC` | Directly set flag state | Simple sub-unit |

These instructions work together to provide:

* Fine-grained control over Vector Unit behavior (`SFPCONFIG`)
* High instruction throughput via parallel execution (`SFPLOADMACRO`)
* Data reorganization capabilities (`SFPSHFT2`)
* Conditional execution support (`SFPIADD`, `SFPPOPC`, etc.)

For arithmetic operations using these configured settings, see [Vector Unit Core Instructions](/tenstorrent/tt-isa-documentation/5.3-vector-unit-core-instructions). For data movement instructions that respect these configurations, see [Vector Unit Data Movement Instructions](/tenstorrent/tt-isa-documentation/5.4-vector-unit-data-movement-instructions).
