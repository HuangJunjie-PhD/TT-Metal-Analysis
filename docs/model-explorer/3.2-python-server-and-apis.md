This document covers the Python server implementation that provides the REST API layer for Model Explorer. The server acts as the middleware between model conversion adapters and the frontend web application, managing model loading, extension execution, and configuration data.

For information about the model conversion adapters that the server coordinates, see [Model Conversion Adapters](https://deepwiki.com/tenstorrent/model-explorer/3.1-model-conversion-adapters). For details about the frontend services that consume these APIs, see [Core Services](https://deepwiki.com/tenstorrent/model-explorer/4.1-core-services).

## Flask Server Architecture

The Model Explorer server is built using Flask and provides a comprehensive REST API for model visualization. The `server.py` module implements the core server functionality with route handlers, extension management, and static file serving.

**Flask Server Core Components**

Sources: [src/server/package/src/model_explorer/server.py 242-525](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/server/package/src/model_explorer/server.py#L242-L525)

The server initialization follows this pattern:

| Component | Purpose | Key Functions |
| --- | --- | --- |
| `Flask(__name__)` | Main application instance | Route handling, request processing |
| `ExtensionManager` | Adapter registry and execution | Loading builtin/custom adapters |
| `ServerDirectiveDispatcher` | Real-time communication | Server-sent events for UI updates |
| Port management | Dynamic port allocation | Automatic port finding (8080-8099) |

## Extension Management System

The `ExtensionManager` class serves as the central registry for all model conversion adapters and provides a unified interface for adapter execution.

**Extension Manager Operations**

| Method | Purpose | Return Type |
| --- | --- | --- |
| `load_extensions()` | Initialize all adapter modules | `None` |
| `get_extensions_metadata()` | Retrieve adapter metadata for frontend | `list` |
| `run_cmd(cmd)` | Execute adapter with given command | `Any` |
| `cleanup(cmd)` | Post-execution cleanup | `None` |

Sources: [src/server/package/src/model_explorer/extension_manager.py 32-152](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/server/package/src/model_explorer/extension_manager.py#L32-L152)

The extension system supports both builtin adapters (C++ implementations accessed via Python bindings) and custom Python adapters. The `BUILTIN_ADAPTER_MODULES` list defines the core adapters available by default.

Sources: [src/server/package/src/model_explorer/extension_manager.py 32-40](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/server/package/src/model_explorer/extension_manager.py#L32-L40)

## Configuration System

The `ModelExplorerConfig` class manages model sources, node data, and server settings. It provides methods for adding models and serializing configuration for URL parameters or server reuse.

**Configuration Data Flow**

The configuration system handles three main data types:

1.   **Model Sources**: File paths or special URIs (e.g., `graphs://model_name/index`)
2.   **Node Data**: Additional visualization data linked to specific models
3.   **Server Settings**: Host/port information for server reuse scenarios

Sources: [src/server/package/src/model_explorer/config.py 51-270](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/server/package/src/model_explorer/config.py#L51-L270)

## Package Structure and Dependencies

The Python server is distributed as the `ai-edge-model-explorer` package with specific dependency requirements and build configuration.

**Core Dependencies**

| Package | Purpose | Usage |
| --- | --- | --- |
| `flask` | Web server framework | HTTP request handling, routing |
| `portpicker` | Dynamic port allocation | Finding available ports |
| `requests` | HTTP client | Version checking, server communication |
| `termcolor` | Console output formatting | Colored terminal messages |
| `packaging` | Version parsing | Semantic version comparison |

**Optional Dependencies**

*   `ai-edge-model-explorer-adapter` (C++ adapters, excluded on Windows)
*   `torch` (PyTorch model support)

Sources: [src/server/package/pyproject.toml 17-28](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/server/package/pyproject.toml#L17-L28)

The package includes the compiled frontend web application in the `web_app` directory, served as static files by the Flask server.
