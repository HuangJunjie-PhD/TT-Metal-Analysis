## Purpose and Scope

This document describes the protocol objects and data types that define the interface contract between clients and vLLM's API servers. These types handle request validation, parameter configuration, and response formatting across all API endpoints.

**Related documentation:**

*   For information about the OpenAI-compatible API server implementation, see [OpenAI-Compatible Server](https://deepwiki.com/tenstorrent/vllm/4.2-openai-compatible-server)
*   For information about the Python API (LLM class), see [Python API](https://deepwiki.com/tenstorrent/vllm/4.1-python-api)
*   For streaming responses and tool calling features, see [Streaming and Tool Calling](https://deepwiki.com/tenstorrent/vllm/4.4-streaming-and-tool-calling)

The protocol types are primarily defined in [vllm/entrypoints/openai/protocol.py](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py) and include request objects, response objects, sampling parameters, and auxiliary data structures.

## Request Types

### ChatCompletionRequest

The `ChatCompletionRequest` object handles chat-based text generation with conversational context.

**Core fields:**

| Field | Type | Description |
| --- | --- | --- |
| `messages` | `list[ChatCompletionMessageParam]` | Conversation history |
| `model` | `str` | Model identifier |
| `temperature` | `float` | Sampling temperature (0.0-2.0) |
| `max_tokens` / `max_completion_tokens` | `int` | Maximum tokens to generate |
| `top_p` | `float` | Nucleus sampling threshold |
| `frequency_penalty` | `float` | Penalize frequent tokens |
| `presence_penalty` | `float` | Penalize any repeated tokens |
| `stop` | `Union[str, list[str]]` | Stop sequences |
| `stream` | `bool` | Enable streaming responses |
| `tools` | `list[ChatCompletionToolsParam]` | Function calling tools |
| `tool_choice` | `Union[str, ChatCompletionNamedToolChoiceParam]` | Tool selection strategy |
| `response_format` | `AnyResponseFormat` | Structured output format |

**vLLM-specific extensions:**

| Field | Type | Description |
| --- | --- | --- |
| `request_id` | `str` | Custom request identifier |
| `mm_processor_kwargs` | `dict` | Multi-modal processor arguments |
| `priority` | `int` | Request priority for scheduling |
| `best_of` | `int` | Generate N sequences, return best (V0 only) |
| `cache_salt` | `str` | Prefix cache salt for security (V1 only) |

**Conversion to SamplingParams:**

**Sources:**[vllm/entrypoints/openai/protocol.py 437-729](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L437-L729)

### CompletionRequest

The `CompletionRequest` object handles direct text completion without conversational structure.

**Core fields:**

| Field | Type | Description |
| --- | --- | --- |
| `prompt` | `Union[str, list[str], list[int], list[list[int]]]` | Input prompt(s) |
| `model` | `str` | Model identifier |
| `suffix` | `str` | Text to append after completion |
| `echo` | `bool` | Return prompt in response |
| `best_of` | `int` | Generate N, return best (V0 only) |

The `CompletionRequest` shares most sampling parameters with `ChatCompletionRequest`.

**Sources:**[vllm/entrypoints/openai/protocol.py 731-904](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L731-L904)

### EmbeddingRequest

The `EmbeddingRequest` handles vector embedding generation for text inputs.

**Core fields:**

| Field | Type | Description |
| --- | --- | --- |
| `input` | `Union[str, list[str], list[int], list[list[int]]]` | Text or tokens to embed |
| `model` | `str` | Embedding model identifier |
| `encoding_format` | `Literal["float", "base64"]` | Output encoding |
| `dimensions` | `int` | Truncate to N dimensions |
| `truncate_prompt_tokens` | `int` | Left-truncate prompts |

**Sources:**[vllm/entrypoints/openai/protocol.py 906-984](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L906-L984)

### TokenizeRequest and DetokenizeRequest

Tokenization utilities:

**TokenizeRequest:**

*   `prompt`: Text or conversation to tokenize
*   `model`: Model identifier
*   Returns: `tokens` (list of integers), `count` (token count)

**DetokenizeRequest:**

*   `tokens`: Token IDs to convert to text
*   `model`: Model identifier
*   Returns: `prompt` (decoded text)

**Sources:**[vllm/entrypoints/openai/protocol.py 1393-1466](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L1393-L1466)

### ScoreRequest and RerankRequest

Cross-encoder scoring and reranking:

**ScoreRequest:**

*   `text_1`: First text or list of texts
*   `text_2`: Second text or list of texts
*   Returns: Similarity scores

**RerankRequest:**

*   `query`: Query text
*   `documents`: List of documents to rank
*   Returns: Ranked documents with scores

**Sources:**[vllm/entrypoints/openai/protocol.py 1468-1584](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L1468-L1584)

## Sampling Parameters

### SamplingParams

The `SamplingParams` class controls text generation behavior. It is defined in [vllm/sampling_params.py](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py) and used across all generation endpoints.

**Core sampling fields:**

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `temperature` | `float` | 1.0 | Randomness (0=greedy, higher=more random) |
| `top_p` | `float` | 1.0 | Nucleus sampling threshold |
| `top_k` | `int` | 0 | Top-K sampling (0=disabled) |
| `min_p` | `float` | 0.0 | Minimum probability threshold |
| `frequency_penalty` | `float` | 0.0 | Penalize token frequency |
| `presence_penalty` | `float` | 0.0 | Penalize any repetition |
| `repetition_penalty` | `float` | 1.0 | Overall repetition penalty |

**Generation control:**

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `n` | `int` | 1 | Number of sequences to generate |
| `best_of` | `int` | None | Generate N, return best (V0 only) |
| `max_tokens` | `int` | 16 | Maximum tokens to generate |
| `min_tokens` | `int` | 0 | Minimum tokens before EOS |
| `seed` | `int` | None | Random seed |
| `stop` | `Union[str, list[str]]` | None | Stop strings |
| `stop_token_ids` | `list[int]` | None | Stop token IDs |
| `ignore_eos` | `bool` | False | Continue after EOS |

**Output control:**

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `logprobs` | `int` | None | Return N top logprobs per token |
| `prompt_logprobs` | `int` | None | Return prompt token logprobs |
| `detokenize` | `bool` | True | Convert tokens to text |
| `skip_special_tokens` | `bool` | True | Exclude special tokens from text |
| `output_kind` | `RequestOutputKind` | CUMULATIVE | Output format (cumulative/delta/final) |

**Advanced features:**

| Field | Type | Description |
| --- | --- | --- |
| `structured_outputs` | `StructuredOutputsParams` | Structured output configuration |
| `logits_processors` | `list[LogitsProcessor]` | Custom logit modifiers |
| `truncate_prompt_tokens` | `int` | Left-truncate prompt to N tokens |

**Sampling type determination:**

**Sources:**[vllm/sampling_params.py 85-323](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L85-L323)

### BeamSearchParams

Beam search is an alternative to sampling for text generation (V0 only):

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `beam_width` | `int` | 1 | Number of beams |
| `max_tokens` | `int` | 16 | Maximum tokens to generate |
| `temperature` | `float` | 0.0 | Sampling temperature |
| `length_penalty` | `float` | 1.0 | Length normalization factor |
| `early_stopping` | `bool` | False | Stop when first beam finishes |

**Sources:**[vllm/sampling_params.py 328-430](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L328-L430)

### StructuredOutputsParams

Configuration for constrained generation:

**Mutual exclusivity:** Only one constraint type can be specified at a time (json, regex, choice, grammar, or json_object).

**Backend selection:** The `_backend` field is set by the `Processor` during validation and determines which structured output backend is used (e.g., "xgrammar", "outlines", "lm-format-enforcer").

**Sources:**[vllm/sampling_params.py 32-61](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L32-L61)

## Request-to-Parameter Conversion Flow

This diagram shows how API requests are converted to internal parameter objects:

**Key conversion logic:**

1.   **ChatCompletionRequest → SamplingParams:**

    *   Extracts `temperature`, `top_p`, `max_tokens`, etc.
    *   Processes `response_format` into `StructuredOutputsParams`
    *   Handles tool calling configurations
    *   Sources: [vllm/entrypoints/openai/protocol.py 603-729](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L603-L729)

2.   **CompletionRequest → SamplingParams:**

    *   Similar to chat, but simpler
    *   Handles `echo` and `suffix` differently
    *   Sources: [vllm/entrypoints/openai/protocol.py 843-904](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L843-L904)

3.   **EmbeddingRequest → PoolingParams:**

    *   Extracts `truncate_prompt_tokens`
    *   Sources: [vllm/entrypoints/openai/protocol.py 952-984](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L952-L984)

4.   **ResponsesRequest → SamplingParams:**

    *   Extracts sampling params with defaults
    *   Processes `text.format` for structured outputs
    *   Sources: [vllm/entrypoints/openai/protocol.py 357-398](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L357-L398)

## Structured Output Configuration

Each `StructuralTag` contains:

*   `structural_tag_schema`: JSON schema
*   `end`: Closing tag

## Logits Processors

Custom logits processors can be specified in requests:

**Definition:**

*   `qualname`: Fully qualified name of processor class
*   `args`: Constructor positional arguments
*   `kwargs`: Constructor keyword arguments

**Security:** The `logits_processor_pattern` engine argument controls which processors are allowed via regex matching.

**Helper function:**`get_logits_processors()` validates and instantiates processors.

**Sources:**[vllm/entrypoints/openai/protocol.py 229-271](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/entrypoints/openai/protocol.py#L229-L271)
