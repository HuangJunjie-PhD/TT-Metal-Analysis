The `job_id` action is a composite GitHub Action designed to retrieve the unique numeric ID of a running job. While GitHub provides the `github.run_id` (workflow level) in the standard context, it does not natively expose the specific `job_id` for the currently executing job. This action bridges that gap by querying the GitHub REST API.

### Purpose and Implementation

The action identifies the correct job by matching a provided `job_name` against the list of jobs associated with the current `github.run_id`[.github/actions/job_id/action.yml 26-28](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L26-L28) Because large workflows may contain many jobs, the action implements a pagination loop to ensure all jobs are searched [.github/actions/job_id/action.yml 34-52](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L34-L52)

### Matrix Job Name Construction

A critical requirement for this action is providing the correct `job_name`. For standard jobs, this is typically `${{ github.job }}`. However, for matrix jobs, GitHub automatically appends matrix values to the job name in the API (e.g., `my-job (1)`). Users must manually construct this string to match the API's representation [.github/actions/job_id/Readme.md 4-5](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/Readme.md?plain=1#L4-L5)

### Implementation Details

#### Pagination and Filtering

The action uses a `while true` loop in Bash to iterate through the GitHub API results [.github/actions/job_id/action.yml 34](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L34-L34)

1.   **Request**: It fetches 90 jobs per page using `curl`[.github/actions/job_id/action.yml 35-36](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L35-L36)
2.   **Filtering**: It uses `jq` with the `contains` function to find the job matching the `JOB_NAME` variable [.github/actions/job_id/action.yml 38](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L38-L38)
3.   **Termination**: The loop breaks if a `job_id` is found or if the number of fetched jobs reaches `total_count`[.github/actions/job_id/action.yml 40-49](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L40-L49)

#### Usage Example

In a matrix workflow, the action is called as follows:

`- name: Fetch job id  id: fetch-job-id  uses: ./.github/actions/job_id  with:    job_name: "${{ github.job }} (${{ matrix.test_group_id }})"`
**Sources:**[.github/workflows/test_job_id.yml 20-24](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/test_job_id.yml#L20-L24)[.github/actions/job_id/Readme.md 11-15](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/Readme.md?plain=1#L11-L15)

### Error Handling

If the loop completes without finding a matching job name, the action prints an error message and exits with code `1`, failing the step [.github/actions/job_id/action.yml 54-57](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/job_id/action.yml#L54-L57)
