This document describes the convolution operations system in tt-metal, including Conv2D, Conv1D, Conv Transpose2D, and experimental Conv3D implementations. These operations are part of the TTNN (Tenstorrent Neural Network) high-level API and provide optimized convolution kernels for neural network inference and training.

## Overview

The convolution operations system implements standard 2D convolution (`ttnn::conv2d`), 1D convolution (`ttnn::conv1d`), transposed convolution (`ttnn::conv_transpose2d`), and experimental 3D convolution (`ttnn::experimental::conv3d`) with extensive optimization support. The system handles:

*   **Multiple data types**: Support for `bfloat16`, `bfloat8_b`, and `float32`[ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp 101-103](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp#L101-L103)
*   **Sharding strategies**: Support for Height, Width, and Block sharding [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp 169-215](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp#L169-L215)
*   **Memory management**: Automatic selection between L1 and DRAM execution paths and slicing for DRAM-bound operations [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp 125-138](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp#L125-L138)
*   **Grouped and depthwise convolutions**: Optimized transformations and kernels for grouped execution [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp 161-163](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp#L161-L163)
*   **Performance optimizations**: Kernel stride folding, activation reuse, and double buffering [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp 66-78](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp#L66-L78)[ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp 80-96](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp#L80-L96)

The implementation automatically selects between direct convolution and matmul-based approaches depending on operation parameters such as kernel size and stride [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp 49-55](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp#L49-L55)

### Conv2D

The primary convolution operation, `ttnn::conv2d`, supports standard 2D convolutions. It handles both sharded and interleaved inputs, with automatic resharding logic [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp 140-150](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.cpp#L140-L150)

*   **Input shape**: `[batch, height, width, channels]` in NHWC layout.
*   **Return types**: Configurable via `Conv2dResultWithOptions` to return output only, output dimensions, or preprocessed weights/bias [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.hpp 22-29](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d.hpp#L22-L29)

### Conv Transpose2D

Implements transposed convolution (deconvolution) using the Conv2D micro-op with flipped weights.

*   **Weight transformation**: Weights are prepared via `prepare_conv_weights` which handles the necessary flipping for transposed kernels [ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp 17-24](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp#L17-L24)
*   **Input transformation**: For stride > 1, the halo operation adds interleaved zeros to the input tensor to expand the spatial dimensions [ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp 95-100](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp#L95-L100)
*   **Execution**: The `SlidingWindowConfig` has an `is_transpose` flag set to true to signal specialized behavior in the Conv2D and Halo micro-ops [ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp 99-101](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp#L99-L101)
*   **Dimensions**: Output spatial dimensions are calculated using `compute_conv_transpose2d_dimensions`, which accounts for input padding and output padding [ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp 81-82](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv_transpose2d/conv_transpose2d.cpp#L81-L82)

### Conv3D (Experimental)

Experimental implementation of 3D convolution supporting volumetric data.

*   **Physical Layout**: Conv3D input is typically `(B, C, T, H, W)`. On device, it is permuted to `(B, T, H, W, C_aligned)` in `ROW_MAJOR` layout [tests/ttnn/nightly/unit_tests/operations/conv/test_conv3d.py 26-41](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tests/ttnn/nightly/unit_tests/operations/conv/test_conv3d.py#L26-L41)
*   **Implementation**: Uses a volume-to-column (`vol2col`) reader to convert volumetric activations into patches for matmul-based computation [ttnn/cpp/ttnn/operations/experimental/conv3d/device/conv3d_program_factory.cpp 84-87](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/experimental/conv3d/device/conv3d_program_factory.cpp#L84-L87)
*   **Subblock Optimization**: Benefitting from larger subblocks on Wormhole B0 (e.g., 2x4) for weight reuse, while Blackhole utilizes optimized blockings with smaller `out_subblock_h`[ttnn/cpp/ttnn/operations/experimental/conv3d/device/conv3d_program_factory.cpp 125-134](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/experimental/conv3d/device/conv3d_program_factory.cpp#L125-L134)

## Weight Preparation and Transformation

Weights must be transformed into optimized matrix layouts before execution on Tenstorrent hardware.

*   **WeightLayoutThreader**: A 2D threading utility that parallelizes weight transformation across input and output channel dimensions on the host CPU. It calculates an optimal thread count based on hardware concurrency (using `std::sqrt(hw_concurrency - 1)`) to avoid oversubscription while maintaining load balance [ttnn/cpp/ttnn/operations/conv/conv2d/prepare_conv2d_weights.cpp 45-73](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/prepare_conv2d_weights.cpp#L45-L73)
*   **Tiled Layouts**: Weights are converted to `Layout::TILE` and can be optimized for specific sharding schemes, such as `convert_conv_weight_tensor_to_tiled_layout_block_sharded`[ttnn/cpp/ttnn/operations/conv/conv2d/prepare_conv2d_weights.hpp 46-51](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/prepare_conv2d_weights.hpp#L46-L51)
*   **Alignment**: Input channels are padded to ensure 16-byte alignment in L1 memory. For `bfloat16` (2 bytes/element), this requires the channel dimension to be a multiple of 8 [ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp 114-120](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/conv2d_utils.cpp#L114-L120)

## Program Factories and Performance Optimizations

### Activation Reuse Optimization

In `Conv2dShardedProgramFactory`, the `ActivationReuseConfig` handles optimizations where readers process full image widths to skip `if` conditions in the kernel [ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_sharded_program_factory.cpp 59-78](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_sharded_program_factory.cpp#L59-L78)

*   **Reuse Window Offset**: Calculates the number of bytes to move the CB read pointer when passing to a new output image row, skipping `filter_w * conv_act_c_read_bytes` elements [ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_sharded_program_factory.cpp 95-101](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_sharded_program_factory.cpp#L95-L101)
*   **Partial Core Handling**: Manages cores that have less work than the full shard size to ensure compute kernels are not blocked by pushing the remaining required tiles [ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_sharded_program_factory.cpp 118-135](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_sharded_program_factory.cpp#L118-L135)

### Width Sharding Factory

`Conv2dWidthShardedProgramFactory` optimizes convolutions where activations are sharded across channels. It calculates the optimal matrix-multiply shape for the activation volume, rounding up the number of rows to be a multiple of `num_cores_nhw * act_block_h_datums`[ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_width_sharded_program_factory.cpp 30-49](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/conv/conv2d/device/conv2d_op_width_sharded_program_factory.cpp#L30-L49)
