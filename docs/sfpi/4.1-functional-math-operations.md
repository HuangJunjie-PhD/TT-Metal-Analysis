## Introduction

This page documents the functional math operations provided by `sfpi_lib.h`, which form the building blocks for implementing complex mathematical functions on the SFPU. These operations provide direct access to floating-point components, lookup table functionality, and specialized mathematical primitives that enable efficient implementation of transcendental functions, normalizations, and numerical algorithms.

The functional math operations include:

*   Lookup table (LUT) operations for function approximation
*   Exponent extraction and manipulation (`exexp`, `setexp`, `addexp`)
*   Mantissa extraction and manipulation (`exman8`, `exman9`, `setman`)
*   Sign manipulation (`setsgn`)
*   Leading zero counting (`lz`, `lz_nosgn`)
*   Absolute value (`abs`)
*   Bit shifting (`shft`)
*   Type reinterpretation (`reinterpret`)

These operations are implemented as inline functions in the `sfpi` namespace and compile directly to SFPU hardware intrinsics.

## Lookup Table (LUT) Operations

Lookup tables provide fast approximation of mathematical functions by interpolating between precomputed values. The SFPU implements specialized LUT instructions that are used extensively for transcendental functions like `tanh`, `sigmoid`, `exp`, and `log`.

## Exponent and Mantissa Manipulation

Direct manipulation of IEEE-754 floating-point components enables efficient implementation of mathematical operations. These functions operate on the binary representation of floating-point numbers.

**Diagram: IEEE-754 Component Access**

Sources: [include/wormhole/sfpi_lib.h 78-149](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L78-L149)[include/blackhole/sfpi_lib.h 78-149](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L78-L149)

### Exponent Operations

| Function | Description | Hardware Intrinsic |
| --- | --- | --- |
| `exexp(v)` | Extract exponent with bias removal (returns as `vInt`) | `__builtin_rvtt_sfpexexp` with `SFPEXEXP_MOD1_DEBIAS` |
| `exexp_nodebias(v)` | Extract raw exponent without bias removal | `__builtin_rvtt_sfpexexp` with `SFPEXEXP_MOD1_NODEBIAS` |
| `setexp(v, exp)` | Set exponent field (accepts `uint32_t` or `vInt`) | `__builtin_rvtt_sfpsetexp_i` or `_v` |
| `addexp(v, exp)` | Add to exponent (multiply by power of 2) | `__builtin_rvtt_sfpdivp2` with `SFPSDIVP2_MOD1_ADD` |

**Key Points:**

*   `exexp()` removes the IEEE-754 bias (127) and returns a signed integer
*   `setexp()` accepts both immediate values and vector operands
*   `addexp()` is equivalent to multiplying by 2^exp but operates at the bit level

Sources: [include/wormhole/sfpi_lib.h 78-125](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L78-L125)[include/blackhole/sfpi_lib.h 78-125](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L78-L125)

## Shift Operations

Bit shift operations provide both logical and arithmetic shifting with support for immediate and vector shift amounts.

### Shift Functions

| Function | Description | Hardware Intrinsic |
| --- | --- | --- |
| `shft(vUInt v, int amt)` | Logical shift by immediate (Wormhole) | `__builtin_rvtt_sfpshft_i` |
| `shft(vUInt v, vInt amt)` | Logical shift by vector (Wormhole) | `__builtin_rvtt_sfpshft_v` |
| `shft(vInt v, int amt)` | Arithmetic shift by immediate (Blackhole) | `__builtin_rvtt_sfpshft_i` with `SFPSHFT_MOD1_ARITHMETIC` |
| `shft(vInt v, vInt amt)` | Arithmetic shift by vector (Blackhole) | `__builtin_rvtt_sfpshft_v` with `SFPSHFT_MOD1_ARITHMETIC` |

**Key Differences:**

*   **Wormhole**: `shft()` performs logical shifts only
*   **Blackhole**: `shft()` supports both logical (for `vUInt`) and arithmetic (for `vInt`) shifts

Positive shift amounts shift left, negative amounts shift right.

Blackhole architecture provides additional hardware-accelerated math functions not available in Wormhole.

**Diagram: Blackhole-Specific Operations**

Sources: [include/blackhole/sfpi_lib.h 311-325](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L311-L325)

### Blackhole-Only Functions

| Function | Description | Hardware Intrinsic |
| --- | --- | --- |
| `rand()` | Generate random integer vector | `__builtin_rvtt_sfpmov_config` with `SFPCONFIG_SRC_RAND` |
| `approx_recip<uncond>(src)` | Fast reciprocal approximation | `__builtin_rvtt_sfparecip` with `SFPARECIP_MOD1_RECIP` |
| `approx_exp(src)` | Fast exponential approximation | `__builtin_rvtt_sfparecip` with `SFPARECIP_MOD1_EXP` |

**Key Points:**

*   These functions are only available when compiling for Blackhole (`__riscv_xtttensixbh`)
*   `approx_recip()` can be conditional (`uncond=false`) or unconditional (`uncond=true`)
*   These provide hardware-accelerated alternatives to software implementations

## Performance Considerations

When using special functions and operations, consider the following performance aspects:

1.   LUT operations can provide fast approximations but have precision limitations
2.   Exponent/mantissa manipulation is typically faster than full floating-point operations
3.   Chain operations carefully to minimize register pressure
4.   Vector operations can provide significant performance improvements for matrix calculations
5.   Architecture-specific functions may offer better performance but reduce portability

Sources: [tests/grayskull/ckernel.cc 246-265](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/grayskull/ckernel.cc#L246-L265)[tests/grayskull/ckernel.cc 596-614](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/grayskull/ckernel.cc#L596-L614)

## Summary

The SFPI special functions and operations provide low-level access to hardware capabilities that enable efficient implementation of complex mathematical functions. By directly manipulating floating-point components, using lookup tables, and leveraging specialized vector operations, developers can achieve significant performance improvements for numerical computations on Tenstorrent hardware.
