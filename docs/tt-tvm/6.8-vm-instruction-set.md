## Purpose and Scope

This page documents the bytecode instruction set used by the Relay Virtual Machine (VM). The VM uses a register-based architecture to execute compiled Relay programs with support for dynamic shapes, control flow, and heterogeneous device execution. This page focuses on the semantics and encoding of individual VM instructions.

For information about the VM compiler that generates this bytecode, see [Virtual Machine Backend](https://deepwiki.com/tenstorrent/tt-tvm/6.7-virtual-machine-backend). For details on the executable file format and serialization, see [VM Executable Format](https://deepwiki.com/tenstorrent/tt-tvm/6.9-vm-executable-format). For the runtime execution engine, see [Virtual Machine Executor](https://deepwiki.com/tenstorrent/tt-tvm/7.2-virtual-machine-executor).

## VM Architecture Overview

The Relay VM is a register-based virtual machine that executes bytecode generated from Relay IR. Unlike stack-based VMs, each function maintains a register file where intermediate values are stored during execution.

### Key Components

**Sources:**[include/tvm/runtime/vm/vm.h 82-110](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/include/tvm/runtime/vm/vm.h#L82-L110)[src/runtime/vm/vm.cc 399-413](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L399-L413)

### Register File Organization

Each `VMFunction` has a fixed-size register file determined at compile time by the `register_file_size` field. Registers store `ObjectRef` values, which can be:

*   `NDArray` (tensors)
*   `ADT` (algebraic data types / tuples)
*   `VMClosure` (closures)
*   `Storage` (memory buffers)
*   Scalar values wrapped as NDArray

**Sources:**[include/tvm/runtime/vm/vm.h 87-110](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/include/tvm/runtime/vm/vm.h#L87-L110)[src/runtime/vm/vm.cc 552-556](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L552-L556)

## Instruction Categories

The VM instruction set can be organized into eight functional categories:

| Category | Instructions | Purpose |
| --- | --- | --- |
| Data Movement | `Move`, `LoadConst`, `LoadConsti` | Transfer data between registers and load constants |
| Memory Allocation | `AllocTensor`, `AllocTensorReg`, `AllocStorage`, `AllocADT`, `AllocClosure` | Allocate memory for various object types |
| Data Access | `GetField`, `GetTag`, `ShapeOf` | Extract information from composite objects |
| Tensor Operations | `ReshapeTensor` | Manipulate tensor shapes |
| Control Flow | `If`, `Goto`, `Ret`, `Fatal` | Program flow control |
| Function Calls | `Invoke`, `InvokeClosure`, `InvokePacked` | Invoke different types of callable objects |
| Device Management | `DeviceCopy` | Transfer data between devices |
| Register Management | `KillRegister` | Memory management hints |

**Sources:**[src/relay/backend/vm/compiler.cc 306-332](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L306-L332)

## Complete Instruction Reference

### Data Movement Instructions

#### Move

```
Move(from: RegName, dst: RegName)
```

Copies the value from register `from` to register `dst`. Both registers must contain compatible `ObjectRef` values.

**Compilation:** Emitted when assigning variables or moving values between registers.

**Execution:** Performs shallow copy of `ObjectRef`.

**Sources:**[src/relay/backend/vm/compiler.cc 320](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L320-L320)[src/runtime/vm/vm.cc 639-645](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L639-L645)

#### LoadConst

```
LoadConst(const_index: Index, device_index: Index, dst: RegName)
```

Loads a constant from the constant pool at `const_index`, ensures it resides on device `device_index`, and stores it in register `dst`. Implements caching to avoid redundant allocations.

**Compilation:** Emitted for `ConstantNode` in Relay IR.

**Execution:** Uses `const_pool_` cache; allocates on specified device if not cached.

**Sources:**[src/relay/backend/vm/compiler.cc 368-378](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L368-L378)[src/runtime/vm/vm.cc 649-667](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L649-L667)

#### LoadConsti

```
LoadConsti(val: Index, dst: RegName)
```

Loads an immediate integer constant `val` into register `dst` as a 64-bit integer NDArray.

**Compilation:** Emitted for small integer constants and control flow parameters.

**Execution:** Creates scalar NDArray on host device.

**Sources:**[src/relay/backend/vm/compiler.cc 437](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L437-L437)[src/runtime/vm/vm.cc 668-678](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L668-L678)

### Memory Allocation Instructions

#### AllocTensor

```
AllocTensor(storage: RegName, offset: RegName, shape: int64_t[], dtype: DLDataType, dst: RegName)
```

Allocates a tensor with static shape from pre-allocated storage. The `storage` register contains a `Storage` object, `offset` contains the byte offset, and `shape` is a compile-time constant array.

**Compilation:** Emitted for `memory.alloc_tensor` with constant shape.

**Execution:** Creates NDArray view into storage buffer.

**Sources:**[src/relay/backend/vm/compiler.cc 557-591](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L557-L591)[src/runtime/vm/vm.cc 750-772](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L750-L772)

#### AllocTensorReg

```
AllocTensorReg(storage: RegName, offset: RegName, shape_register: RegName, dtype: DLDataType, dst: RegName)
```

Allocates a tensor with dynamic shape. Shape is read from `shape_register` at runtime.

**Compilation:** Emitted for `memory.alloc_tensor` with dynamic shape.

**Execution:** Reads shape from register, converts to `ShapeTuple`, creates NDArray.

**Sources:**[src/relay/backend/vm/compiler.cc 586-591](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L586-L591)[src/runtime/vm/vm.cc 821-843](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L821-L843)

#### AllocStorage

```
AllocStorage(allocation_size: RegName, alignment: Index, dtype_hint: DLDataType, device_index: Index, shape: int64_t[], dst: RegName)
```

Allocates raw storage buffer of size specified in `allocation_size` register. The `shape` array is used for debugging/profiling. Returns a `Storage` object.

**Compilation:** Emitted for `memory.alloc_storage`.

**Execution:** Uses device-specific allocator to allocate buffer.

**Sources:**[src/relay/backend/vm/compiler.cc 593-624](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L593-L624)[src/runtime/vm/vm.cc 844-917](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L844-L917)

#### AllocADT

```
AllocADT(tag: Index, num_fields: Index, datatype_fields: RegName[], dst: RegName)
```

Allocates an Algebraic Data Type (ADT) with specified tag and fields. Used for tuples and variant types.

**Compilation:** Emitted for `TupleNode` and `ConstructorNode` calls.

**Execution:** Creates `ADT` object containing field values.

**Sources:**[src/relay/backend/vm/compiler.cc 387-398](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L387-L398)[src/runtime/vm/vm.cc 918-947](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L918-L947)

#### AllocClosure

```
AllocClosure(func_index: Index, num_freevar: Index, free_vars: RegName[], dst: RegName)
```

Allocates a closure capturing free variables. The `func_index` identifies the function to be closed over.

**Compilation:** Emitted for `GlobalVarNode` and closure-producing calls.

**Execution:** Creates `VMClosure` object.

**Sources:**[src/relay/backend/vm/compiler.cc 429](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L429-L429)[src/runtime/vm/vm.cc 948-987](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L948-L987)

### Data Access Instructions

#### GetField

```
GetField(object: RegName, field_index: Index, dst: RegName)
```

Extracts field at `field_index` from an ADT/tuple stored in `object` register.

**Compilation:** Emitted for `TupleGetItemNode`.

**Execution:** Indexes into ADT and returns field value.

**Sources:**[src/relay/backend/vm/compiler.cc 416-421](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L416-L421)[src/runtime/vm/vm.cc 679-697](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L679-L697)

#### GetTag

```
GetTag(object: RegName, dst: RegName)
```

Extracts the tag from an ADT, used for pattern matching on variant types.

**Compilation:** Emitted during pattern match compilation for tag comparison.

**Execution:** Returns tag as integer NDArray.

**Sources:**[src/relay/backend/vm/compiler.cc 766](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L766-L766)[src/runtime/vm/vm.cc 698-712](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L698-L712)

#### ShapeOf

```
ShapeOf(tensor: RegName, dst: RegName)
```

Extracts the shape of a tensor as an int64 NDArray.

**Compilation:** Emitted for `vm.shape_of` operator.

**Execution:** Reads tensor shape, creates int64 NDArray.

**Sources:**[src/relay/backend/vm/compiler.cc 625-636](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L625-L636)[src/runtime/vm/vm.cc 1078-1108](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1078-L1108)

### Tensor Operations

#### ReshapeTensor

```
ReshapeTensor(tensor: RegName, newshape: RegName, dst: RegName)
```

Reshapes a tensor to a new shape specified in `newshape` register without copying data.

**Compilation:** Emitted for `vm.reshape_tensor` after memory planning determines reshape is safe.

**Execution:** Creates new NDArray view with different shape over same data.

**Sources:**[src/relay/backend/vm/compiler.cc 637-645](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L637-L645)[src/runtime/vm/vm.cc 1109-1143](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1109-L1143)

### Control Flow Instructions

#### If

```
If(test: RegName, target: RegName, true_offset: Index, false_offset: Index)
```

Conditional branch based on comparison of `test` and `target` registers. Jumps forward by `true_offset` if equal, `false_offset` if not equal.

**Compilation:** Emitted for `IfNode`. Offsets are patched after true/false branches are compiled.

**Execution:** Compares scalar values, updates program counter.

**Sources:**[src/relay/backend/vm/compiler.cc 432-476](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L432-L476)[src/runtime/vm/vm.cc 713-749](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L713-L749)

#### Goto

```
Goto(pc_offset: Index)
```

Unconditional jump forward by `pc_offset` instructions.

**Compilation:** Emitted at end of true branch in `IfNode` to skip false branch.

**Execution:** Updates program counter.

**Sources:**[src/relay/backend/vm/compiler.cc 446](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L446-L446)[src/runtime/vm/vm.cc 1159-1167](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1159-L1167)

#### Ret

```
Ret(result: RegName)
```

Returns from current function with value in `result` register. Pops call stack frame.

**Compilation:** Emitted at end of function body.

**Execution:** Sets `return_register_`, pops frame, returns to caller.

**Sources:**[src/relay/backend/vm/compiler.cc 735](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L735-L735)[src/runtime/vm/vm.cc 750-772](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L750-L772)

#### Fatal

```
Fatal()
```

Throws a fatal runtime error. Used as default case in pattern matching when no clause matches.

**Compilation:** Emitted for impossible/unreachable code paths.

**Execution:** Throws `std::runtime_error`.

**Sources:**[src/relay/backend/vm/compiler.cc 761](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L761-L761)[src/runtime/vm/vm.cc 646-648](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L646-L648)

### Function Call Instructions

#### Invoke

```
Invoke(func_index: Index, args: RegName[], dst: RegName)
```

Calls a VM function (another `VMFunction`) by its global index. Arguments are passed in registers.

**Compilation:** Emitted for `CallNode` with `GlobalVarNode` target.

**Execution:** Pushes new frame, copies arguments to new register file, sets PC to function start.

**Sources:**[src/relay/backend/vm/compiler.cc 686](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L686-L686)[src/runtime/vm/vm.cc 1004-1023](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1004-L1023)

#### InvokeClosure

```
InvokeClosure(closure: RegName, args: RegName[], dst: RegName)
```

Invokes a closure by first extracting its function index and free variables, then calling the function with combined arguments.

**Compilation:** Emitted for `CallNode` with `Var` target (higher-order function call).

**Execution:** Unpacks closure, prepares extended argument list, invokes function.

**Sources:**[src/relay/backend/vm/compiler.cc 698](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L698-L698)[src/runtime/vm/vm.cc 1052-1077](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1052-L1077)

#### InvokePacked

```
InvokePacked(packed_index: Index, arity: Index, output_size: Index, args: RegName[])
```

Invokes a primitive operator (TVM packed function) implementing a Relay operator. Arguments and outputs are both passed as registers.

**Compilation:** Emitted for `vm.invoke_tvm_op` after lowering primitives.

**Execution:** Unpacks arguments into `TVMArgs`, calls packed function.

**Sources:**[src/relay/backend/vm/compiler.cc 479-527](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L479-L527)[src/runtime/vm/vm.cc 1024-1051](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1024-L1051)

#### DeviceCopy

```
DeviceCopy(src: RegName, src_device_index: Index, dst_device_index: Index, dst: RegName)
```

Copies tensor from source device to destination device. Handles device-to-device transfers.

**Compilation:** Emitted for `device_copy` annotation nodes after device planning.

**Sources:**[src/relay/backend/vm/compiler.cc 529-544](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/backend/vm/compiler.cc#L529-L544)[src/runtime/vm/vm.cc 1144-1158](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/runtime/vm/vm.cc#L1144-L1158)

### Register Management Instructions

#### KillRegister

```
KillRegister(dst: RegName)
```

Marks a register as no longer needed, allowing garbage collection of its contents. This is a performance hint for memory management.

**Compilation:** Emitted for `memory.kill` operator after liveness analysis.

**Execution:** Clears register to allow object deallocation.

## Instruction Serialization

Instructions are serialized to bytecode using the `VMInstructionSerializer` structure. Each instruction is encoded as:

1.   **Opcode** (as integer)
2.   **Field array** (operands specific to the instruction type)

The serialization preserves instruction semantics while enabling compact storage and deserialization.

**Bytecode Format Example:**

```
VM Function[0]: main(x, y)
# reg file size = 10
# instruction count = 5
opcode, fields # inst(text):
 0: 49 1 0 1        # LoadConst const_idx=1 device_idx=0 dst=1
 1: 32 2 3          # GetField obj=2 field_idx=3 dst=3
 2: 17 0 1 2        # InvokePacked packed_idx=0 arity=1 output_size=2
 3: 5 4 5           # Move from=4 dst=5
 4: 6 5             # Ret result=5
```

Each line shows: instruction index, opcode, fields, and text representation for debugging.
