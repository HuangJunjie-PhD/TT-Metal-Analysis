## Purpose and Scope

This page documents the declarative test planning system used for operator-level sweep testing in `forge/test/operators/`. It covers how test parameter spaces are specified via `TestCollection`, how the full Cartesian product of test vectors is generated by `TestPlan`, how failure expectations are encoded as failing rules, how `TestQuery` provides a fluent filtering API over generated vectors, how `TestPlanScanner` auto-discovers test plans across the package, and how environment variables drive runtime test selection.

For information on how individual operator tests invoke the compilation and verification pipeline, see [Operator Testing](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.2-operator-testing). For failure classification logic (`FailingReasons`, `FailingReasonsFinder`), see [Failure Classification](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.5-failure-classification-system). For CI/CD test execution, see [CI/CD Pipeline](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.4-cicd-pipeline).

### InputSource

`InputSource` is an `Enum` that controls how the tensor enters the compiled graph:

| Value | Description |
| --- | --- |
| `FROM_HOST` | Tensor supplied directly from the host at runtime |
| `FROM_ANOTHER_OP` | Tensor is the output of a preceding op in the graph |
| `FROM_DRAM_QUEUE` | Tensor lives in DRAM and is read from a queue |
| `CONST_EVAL_PASS` | Tensor is a constant folded at compile time |

Sources: [forge/test/operators/utils/plan.py 38-43](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L38-L43)

### TestVector

`TestVector` is a dataclass representing a single test case. All six primary dimensions — `operator`, `input_source`, `input_shape`, `dev_data_format`, `math_fidelity`, `kwargs` — combine to form a unique test ID via `get_id()`. The `test_plan` back-reference enables the vector to call `test_plan.verify()` and `test_plan.check_test_failing()` on itself.

Key methods:

| Method | Description |
| --- | --- |
| `get_id(fields=None)` | Returns a dash-delimited string ID; `fields` allows partial IDs |
| `get_marks()` | Returns `pytest.mark.xfail` or `pytest.mark.skip` from `failing_result` |
| `to_param()` | Wraps the vector in a `pytest.param` with marks and ID |
| `verify(test_device)` | Delegates to `test_plan.verify(test_device, self)` |

Sources: [forge/test/operators/utils/plan.py 81-134](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L81-L134)

### TestCollection

`TestCollection` specifies a bounded parameter space. It is reused for two purposes:

1.   In `TestPlan.collections` — defines what vectors to **generate**
2.   In `TestPlan.failing_rules` — defines which vectors are **expected to fail**

Every field is `Optional[List[...]]`. A `None` field means "any value" (wildcard). The `criteria` field is a `Callable[[TestVector], bool]` for arbitrary predicate filtering. The `kwargs` field can also be a `Callable[[TestVector], List[dict]]` for dynamically generating kwargs based on the vector context.

The `__contains__` method on `TestCollection` delegates to `TestPlanUtils.test_vector_in_collection()`, enabling `test_vector in test_collection` membership testing.

Sources: [forge/test/operators/utils/plan.py 137-199](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L137-L199)

### TestPlan

`TestPlan` holds:

*   `collections`: a list of `TestCollection` objects defining the test space
*   `failing_rules`: a list of `TestCollection` objects with `failing_reason` or `skip_reason` set
*   `verify`: a `Callable[[TestVector, TestDevice], None]` that runs the actual test

Each `TestPlan` also tracks the set of operator names it covers (from its collections), which `TestSuite` uses for indexing.

Sources: [forge/test/operators/utils/plan.py 335-454](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L335-L454)

### TestSuite

`TestSuite` aggregates multiple `TestPlan` objects and builds an index mapping operator names to their owning plan. This enables fast lookup in `load_test_vectors_from_id_list()` — given a test ID, the suite resolves which plan owns it and assigns the back-reference.

Sources: [forge/test/operators/utils/plan.py 457-488](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L457-L488)

### TestResultFailing

`TestResultFailing` carries the failure expectation for a vector. It holds a `failing_reason` (produces `pytest.mark.xfail`) and/or a `skip_reason` (produces `pytest.mark.skip`). Both must be `FailingReasons` enum members — raw strings or `FailingReason` instances are rejected in `__post_init__`.

Sources: [forge/test/operators/utils/plan.py 45-78](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L45-L78)

## Test Vector Generation

`TestPlan.generate()` produces `TestVector` instances by iterating the Cartesian product of all dimensions in each `TestCollection`. The iteration order is:

```
operator → input_source → input_shape → dev_data_format → math_fidelity → kwargs
```

If a `criteria` callable is set, it is evaluated on a partial vector (without kwargs) before expanding kwargs. If `kwargs` is a function, it is called with the partial vector to produce the kwargs list dynamically.

## Failing Rules System

`TestPlan.check_test_failing()` determines the `TestResultFailing` for a given vector by scanning `failing_rules` in order:

1.   For each `TestCollection` in `failing_rules`, check `test_vector in test_collection`.
2.   If it matches and has a `failing_reason` or `skip_reason`, set `failing_result` to a new `TestResultFailing`.
3.   If it matches with neither reason set, **clear**`failing_result` (allows a broad rule to be overridden by a narrower passing rule).
4.   If the rule has `subcollections`, apply the same logic recursively — subcollections can refine the parent rule's outcome.

Rules are processed in list order, with later rules overriding earlier ones. This layering allows broad failure patterns with specific exceptions.

`TestPlanUtils.test_vector_in_collection()` checks all six fields plus `criteria`, `failing_reason`, and `skip_reason`. A `None` field in the collection matches any value in the vector.

Sources: [forge/test/operators/utils/plan.py 360-390](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L360-L390)[forge/test/operators/utils/plan.py 602-618](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L602-L618)

### FailingRulesConverter

`FailingRulesConverter` is a helper for defining failing rules as positional tuples rather than full keyword-argument `TestCollection` constructors. `build_rules()` accepts a list of tuples and a `params` list that names the positional slots; `build_rule()` handles single-value-to-list normalization before constructing `TestCollection`.

## TestQuery: Fluent Filtering Interface

`TestQuery` wraps a `Generator[TestVector, None, None]` and provides chainable transformation methods. All methods return a new `TestQuery` wrapping a new generator — evaluation is **lazy** until `to_params()` is called.

| Method | Description |
| --- | --- |
| `filter(*filters)` | Keep vectors where all `filters` return `True` |
| `skip(filters)` | Drop vectors where any filter returns `True` |
| `index(*args)` | Keep vectors at specific integer positions |
| `range(start, end)` | Keep vectors in a positional range |
| `index_skip(*args)` / `range_skip(start, end)` | Inverse of the above |
| `sample(percent, random_seed)` | Probabilistic sampling via `RateLimiter` |
| `group_limit(groups, limit)` | Cap number of vectors per group (identified by `get_id(fields=groups)`) |
| `calculate_failing_result()` | Sets `failing_result` on each vector via `TestPlan.check_test_failing()` |
| `reverse()` | Materialize and reverse the generator |
| `log()` | Print all vector IDs and count, then pass through |
| `to_params()` | Materialize as `Generator[pytest.param]` for `@pytest.mark.parametrize` |

Sources: [forge/test/operators/utils/plan.py 202-332](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L202-L332)

## TestPlanScanner: Auto-Discovery

`TestPlanScanner` is responsible for finding all `TestPlan` definitions across the operator test package without requiring manual registration.

**Discovery convention**: Any `.py` file in the scanned package that contains `def get_test_plans(` is imported and its `get_test_plans()` function is called. The function must return a list of `TestPlan` or `TestSuite` objects.

**Key methods:**

| Method | Description |
| --- | --- |
| `find_modules_in_directory(directory)` | Walks the directory tree, returns list of module dotted paths |
| `scan_and_invoke(scan_file, scan_package, ignore_paths, method_name)` | Imports each eligible module and yields results from `get_test_plans()` |
| `collect_test_plans(result)` | Flattens `TestSuite` into individual `TestPlan` objects |
| `build_test_suite(scan_file, scan_package, ignore_test_plan_paths)` | Top-level entry point; returns a `TestSuite` of all discovered plans |

The constant `METHOD_COLLECT_TEST_PLANS = "get_test_plans"` defines the discovery convention name.

At module import in `test_all.py`, `TestSuiteData.all` is populated by:

```
TestPlanScanner.build_test_suite(scan_file=__file__, scan_package=__package__, ...)
```

This scans all sibling and sub-package modules within `forge/test/operators/pytorch/`.

Sources: [forge/test/operators/utils/plan.py 809-900](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L809-L900)[forge/test/operators/pytorch/test_all.py 229-255](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/pytorch/test_all.py#L229-L255)

### RunQueryParams

`RunQueryParams` is a dataclass populated by `RunQueryParams.from_env()`. All fields are `Optional`; unset env vars result in `None` (no restriction). Key behavior:

*   `OPERATORS` prefixed with `-` sets `exclude_operators=True` — those operators are excluded instead of included.
*   `ID_FILES` and `TEST_ID` are mutually exclusive; both set simultaneously raises `ValueError`.
*   `FILTERS` names attributes from `VectorLambdas` or `QueryLambdas` by string.

Sources: [forge/test/operators/pytorch/test_all.py 32-140](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/pytorch/test_all.py#L32-L140)

### VectorLambdas

`VectorLambdas` contains predefined `Callable[[TestVector], bool]` lambdas for per-vector filtering. They are named and can be activated via the `FILTERS` environment variable.

| Lambda | Behavior |
| --- | --- |
| `QUICK` | Matches vectors in `TestCollectionData.quick` (reduced shapes/formats) |
| `FILTERED` | Matches vectors in `TestCollectionData.filtered` (env-var-driven) |
| `EXCLUDED` | Excludes vectors in `TestCollectionData.excluded` |
| `FAILING` | Has a `failing_reason` set |
| `NOT_FAILING` | No `failing_reason` |
| `SKIPED` | Has a `skip_reason` set |
| `SHAPES_2D` / `SHAPES_3D` / `SHAPES_4D` | Filter by rank |
| `MICROBATCH_SIZE_ONE` / `MICROBATCH_SIZE_MULTI` | Filter by batch size |
| `HAS_DATA_FORMAT` / `NO_DATA_FORMAT` | Filter by format presence |
| `FAILING_REASONS` / `SKIP_REASONS` | Match specific `FailingReasons` from env vars |
| `SKIP_IGNORE_LIST` | Excludes IDs in the loaded ignore list |

Sources: [forge/test/operators/pytorch/test_all.py 300-365](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/pytorch/test_all.py#L300-L365)

### QueryLambdas

`QueryLambdas` contains predefined `Callable[[TestQuery], TestQuery]` lambdas for query-level operations:

| Lambda | Behavior |
| --- | --- |
| `SAMPLE` | Random percentage sampling using `query_params.sample` |
| `RANGE` | Positional range filter using `query_params.range` |
| `UNIQUE_KWARGS` | One vector per `(operator, kwargs)` group |
| `CALCULATE_FAILING_RESULT` | Triggers `check_test_failing()` on all vectors |
| `REMOVE_FAILING_RESULT` | Clears all `failing_result` fields |
| `REMOVE_XFAIL_REASON` | Clears `failing_reason` but preserves `skip_reason` |
| `LOG` | Prints vector IDs and count |
| `SAMPLE5` / `SAMPLE10` / `SAMPLE20` | Fixed-percentage sampling shortcuts |

Sources: [forge/test/operators/pytorch/test_all.py 281-298](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/pytorch/test_all.py#L281-L298)

### TestQueries

`TestQueries.query_source()` determines the initial `TestQuery` from one of three sources:

1.   **ID file** (`ID_FILES` set): loads vectors by parsing test IDs from files
2.   **Single ID** (`TEST_ID` set): loads a single vector parsed from its ID string
3.   **Full generation** (default): calls `TestSuite.generate()`

`TestQueries.query_filter()` applies filters in this order:

1.   `FILTERED` / `EXCLUDED` vector lambdas (if env vars set)
2.   `CALCULATE_FAILING_RESULT`
3.   `FAILING_REASONS` / `SKIP_REASONS` filters (if set)
4.   Named filters from `FILTERS` env var (looked up in `VectorLambdas` and `QueryLambdas`)
5.   `SAMPLE` (if set)
6.   `SKIP_IGNORE_LIST` (if `ID_FILES_IGNORE` set)
7.   `RANGE` (if set)

Sources: [forge/test/operators/pytorch/test_all.py 368-478](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/pytorch/test_all.py#L368-L478)

## Test ID Format

Each `TestVector` has a unique string ID encoding all six parameters:

```
{operator}-{input_source}-{kwargs}-{input_shape}-{dev_data_format}-{math_fidelity}
```

For multi-operand operators (e.g., `concatenate`), a 7-part variant is used. Example:

```
ge-FROM_HOST-None-(1, 2, 3, 4)-forge.Float16_b-HiFi4
```

The parser in `TestPlanUtils.test_id_to_test_vector()` handles negative numbers in tuples (which would otherwise appear as extra `-` separators) by scanning for `-` (space before dash) and `(-` (open paren before dash) and preserving them.

ID files (for `ID_FILES`, `ID_FILES_IGNORE`) are plain text with one ID per line. Lines starting with `#` are comments. The `no_device-` prefix is stripped automatically.

Sources: [forge/test/operators/utils/plan.py 620-719](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/plan.py#L620-L719)
