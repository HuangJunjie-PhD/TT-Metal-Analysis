## Purpose and Scope

This document describes how the packer hardware calculates L1 memory addresses when writing tiles back to L1. The packer consists of four independent packer units that write different portions (faces) of a tile to L1 memory. Each packer requires a specific address offset to ensure data is written to the correct location without overlap. This page covers the offset calculation logic, tile header compensation, and destination address programming mechanisms.

For general packer configuration and operation, see [Core Packer Operations and Configuration](https://deepwiki.com/tenstorrent/tt-llk/6.1-core-packer-operations-and-configuration). For untilize-specific address calculations, see [Untilize and Stride Configuration](https://deepwiki.com/tenstorrent/tt-llk/6.3-untilize-and-stride-configuration).

## L1 Base Address Configuration

The packer base address in L1 is set through the `l1_dest_addr` field in the `pack_config_t` structure. This 32-bit address determines where the packer begins writing tile data.

### Address Configuration Fields

The `pack_config_t` structure contains several fields that control L1 addressing:

| Field | Bit Position | Purpose |
| --- | --- | --- |
| `l1_dest_addr` | word 1 [31:0] | Base L1 destination address |
| `add_l1_dest_addr_offset` | word 2 [1] | Enable automatic offset addition |
| `sub_l1_tile_header_size` | word 2 [14] | Subtract tile header size from address |
| `add_tile_header_size` | word 2 [15] | Add tile header size to address |
| `l1_src_addr` | word 2 [23:16] | L1 source address (for L1 accumulation) |

## Tile Header Offset Management

The packer hardware can automatically adjust addresses to account for tile headers through the `add_tile_header_size` and `sub_l1_tile_header_size` configuration flags. These flags control whether the hardware adds or subtracts the tile header size when computing the final write address.

### Tile Header Size Configuration

The tile header size is stored in GPR registers and used for address calculations:

**Diagram 6.4.3**: Tile header size storage in GPR registers

### Header Offset Application

The hardware applies tile header adjustments based on configuration flags:

| Flag | Effect | Use Case |
| --- | --- | --- |
| `add_tile_header_size = 1` | Final address += tile_size | Skip over pre-existing header |
| `sub_l1_tile_header_size = 1` | Final address -= tile_size | Account for header in offset calculation |
| Both flags = 0 | No adjustment | Direct addressing without header compensation |

## Programming Packer Destination Addresses

The `program_packer_destination` function sets the base L1 address where the packer will write tile data. This function programs the `THCON_SEC0_REG1_L1_Dest_addr_ADDR32` register with the destination address.

### Address Programming Sequence

**Diagram 6.4.5**: Packer destination address programming sequence

### Implementation Details

The `program_packer_destination` function performs the following steps:

| Step | Operation | Purpose |
| --- | --- | --- |
| 1 | `LLK_ASSERT(is_valid_L1_address(addr), ...)` | Validate address is in valid L1 memory region |
| 2 | `new_l1_addr = (1 << 31) | addr` | Set bit 31 to enable addressing mode |
| 3 | `TT_SETDMAREG(..., LOWER_HALFWORD(addr), ..., LO_16(OUTPUT_ADDR))` | Load lower 16 bits to GPR |
| 4 | `TT_SETDMAREG(..., UPPER_HALFWORD(new_l1_addr), ..., HI_16(OUTPUT_ADDR))` | Load upper 16 bits (with bit 31) to GPR |
| 5 | `TTI_STALLWAIT(p_stall::STALL_CFG, p_stall::THCON)` | Wait for configuration to be ready |
| 6 | `TTI_WRCFG(OUTPUT_ADDR, 0, THCON_SEC0_REG1_L1_Dest_addr_ADDR32)` | Write GPR to configuration register |
| 7 | `TT_SETDMAREG(..., UPPER_HALFWORD(addr), ..., HI_16(OUTPUT_ADDR))` | Restore original upper bits (clear bit 31) |
| 8 | `TTI_DMANOP` | Synchronization barrier |

**Sources**: [tt_llk_blackhole/common/inc/cpack_common.h 515-533](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L515-L533)

## Address Configuration Register Summary

The following configuration registers and GPRs control packer addressing in Blackhole:

### Configuration Registers

| Register | Purpose | Set By |
| --- | --- | --- |
| `THCON_SEC0_REG1_L1_Dest_addr_ADDR32` | Base L1 destination address (32-bit) | `program_packer_destination` |
| `PCK0_ADDR_BASE_REG_0_Base_ADDR32` | Per-tile address offset | `program_packer_dest_offset_registers` |
| `THCON_SEC0_REG1_Row_start_section_size_ADDR32` | Pack configuration (includes offset flags) | `set_packer_config`, `configure_pack` |

### Configuration Flags (pack_config_t)

| Flag Field | Bit Location | Purpose |
| --- | --- | --- |
| `l1_dest_addr` | word 1 [31:0] | Base destination address |
| `add_l1_dest_addr_offset` | word 2 [1] | Enable automatic offset from `PCK0_ADDR_BASE_REG_0` |
| `sub_l1_tile_header_size` | word 2 [14] | Subtract tile header from calculated address |
| `add_tile_header_size` | word 2 [15] | Add tile header to calculated address |
| `l1_src_addr` | word 2 [23:16] | Source address for L1 accumulation mode |

### GPR Usage

| GPR | Usage | Context |
| --- | --- | --- |
| `p_gpr_pack::OUTPUT_ADDR` | Base address storage during programming | `program_packer_destination` |
| `p_gpr_pack::TEMP_TILE_OFFSET` | Tile offset storage | `program_packer_dest_offset_registers` |
| `p_gpr_pack::TILE_HEADER` (4 regs) | Tile header size storage | `configure_pack` |
| `p_gpr_pack::TMP0`, `p_gpr_pack::TMP1` | Temporary calculation storage | Various configuration functions |

**Sources**: [tt_llk_blackhole/common/inc/cpack_common.h 27-62](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L27-L62)[tt_llk_blackhole/common/inc/cpack_common.h 515-533](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L515-L533)[tt_llk_blackhole/common/inc/cpack_common.h 563-571](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L563-L571)
