## Purpose and Scope

This document covers the fundamental operations and primitives available within the TT-Metal framework for low-level programming on Tenstorrent hardware. These operations provide the building blocks for data movement, matrix computation, testing, and other essential functions that form the foundation of kernel development.

For information about the overall TT-Metalium programming model and architecture concepts, see [TT-Metalium Programming Model](https://deepwiki.com/tenstorrent/tt-tutorial/4.1-tt-metalium-programming-model). For specific hands-on programming examples and kernel development tutorials, see [Programming Examples](https://deepwiki.com/tenstorrent/tt-tutorial/4.3-programming-examples). For higher-level neural network operations, see [TT-NN Neural Network Library](https://deepwiki.com/tenstorrent/tt-tutorial/3.2-tt-nn-neural-network-library).

## Core Operation Types

### Data Movement Operations

Data movement operations handle the transfer of data between different memory hierarchies and across the Network-on-Chip (NOC). These operations are essential for efficient data flow in distributed computations and enable optimal memory bandwidth utilization.

| Operation Category | Purpose | Hardware Target |
| --- | --- | --- |
| Memory Transfer | Move data between memory levels | Memory Hierarchy |
| NOC Communication | Inter-core data exchange | Network-on-Chip |
| DMA Operations | Direct memory access | Memory Controllers |

### Matrix Computation Operations

Matrix computation operations provide access to the specialized mathematical capabilities of Tensix cores, enabling efficient execution of linear algebra operations that form the foundation of neural network computations.

| Operation Type | Function | Optimization Focus |
| --- | --- | --- |
| Matrix Multiply | Core GEMM operations | Tensix Core Utilization |
| Element-wise Operations | Vector arithmetic | Parallel Processing |
| Reduction Operations | Sum, Max, Min operations | Cross-core Communication |

### Testing and Validation Operations

Testing operations provide mechanisms for validating kernel behavior, debugging, and ensuring correctness across different hardware configurations and operation modes.

## Integration Points

Operations and primitives serve as the foundation layer that higher-level frameworks build upon. The TT-NN neural network library utilizes these primitives to implement complex neural network operations, while the TT-Metalium programming model provides the execution environment and resource management capabilities.

This layered approach ensures that developers can work at the appropriate abstraction level for their use case, whether implementing custom low-level kernels using operations directly, or utilizing higher-level neural network functions that internally leverage these primitives.
