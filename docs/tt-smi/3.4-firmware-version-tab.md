## Purpose and Scope

The Firmware Version Tab displays version information for all firmware components running on detected Tenstorrent devices. This tab provides visibility into ARC firmware, Ethernet firmware, M3 bootloader/application firmware, and firmware bundles. The tab is read-only and presents a static snapshot of firmware versions collected during backend initialization.

For information about device hardware properties, see [Device Information Tab](https://deepwiki.com/tenstorrent/tt-smi/3.2-device-information-tab). For real-time telemetry monitoring, see [Telemetry Tab](https://deepwiki.com/tenstorrent/tt-smi/3.3-telemetry-tab).

* * *

## Tab Structure and Navigation

The firmware tab is the third tab in the TabbedContent widget and can be accessed via keyboard shortcut `3` or by clicking the "FW Version (3)" tab header.

**Display Structure**: The firmware table uses a `TTDataTable` widget with header defined by `FIRMWARES_TABLE_HEADER` and displays one row per device.

## Firmware Version Types

The firmware tab reports seven distinct firmware components for each device:

| Column Header | Field Name | Description | Source Field |
| --- | --- | --- | --- |
| FW Bundle Version | `fw_bundle_version` | Combined firmware bundle version | `FW_BUNDLE_VERSION` |
| TT-Flash Version | `tt_flash_version` | TT-Flash utility version | `TT_FLASH_VERSION` |
| CM FW Version | `cm_fw` | Chip Management (ARC) firmware | `ARC0_FW_VERSION` |
| CM FW Date | `cm_fw_date` | CM firmware build date | `WH_FW_DATE` |
| ETH FW Version | `eth_fw` | Ethernet firmware | `ETH_FW_VERSION` |
| BM BL Version | `bm_bl_fw` | Board Management bootloader | `M3_BL_FW_VERSION` |
| BM App Version | `bm_app_fw` | Board Management application | `M3_APP_FW_VERSION` |

## Data Collection Pipeline

**Collection Timing**: Firmware versions are collected once during backend initialization and are not updated in real-time. To see updated firmware versions after flashing new firmware, TT-SMI must be restarted.

Sources: [tt_smi/tt_smi_backend.py 83-96](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L83-L96)[tt_smi/tt_smi.py 148-160](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L148-L160)

## get_firmware_versions() Implementation

The `get_firmware_versions()` method in `TTSMIBackend` extracts firmware version data from the SMBUS telemetry structure and converts it to human-readable formats.

**Conversion Functions**: Firmware versions are stored as hex values in telemetry and converted using utility functions from `tt_tools_common.utils_common.tools_utils`:

*   `hex_to_semver_m3_fw()`: Converts M3/ARC firmware hex to semantic versioning (e.g., `0x01020304` → `"1.2.3.4"`)
*   `hex_to_semver_eth()`: Converts Ethernet firmware hex to semantic versioning
*   `hex_to_date()`: Converts hex timestamp to date string

Sources: [tt_smi/tt_smi_backend.py 588-649](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L588-L649)[tt_smi/tt_smi.py 36-42](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L36-L42)

## Display Formatting and Styling

The `format_firmware_rows()` method creates styled text entries for the firmware table:

**Styling Rules**:

*   **Device Index**: Yellow bold, center-justified (`Text(f"{i}", style="yellow_bold", justify="center")`)
*   **N/A Values**: Gray style to indicate unavailable firmware (`style="gray"`)
*   **Valid Versions**: Green style to indicate successfully detected firmware (`style="text_green"`)

## Platform-Specific Behavior

### Galaxy System Firmware Bug

Galaxy 6U systems (board type `wh_4u`) have a known firmware flashing bug documented in issue #72 where the `FW_BUNDLE_VERSION` field is not properly set during flashing.

**Implementation Details**:

*   Lines 634-644 in `get_firmware_versions()` implement the workaround
*   Galaxy systems manually set `tt_flash_version` to "N/A" (line 629-630)
*   The actual bundle version is read from the `TT_FLASH_VERSION` field instead
*   Comment at line 639-640 references GitHub issue #72

Sources: [tt_smi/tt_smi_backend.py 629-630](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L629-L630)[tt_smi/tt_smi_backend.py 634-648](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L634-L648)

## Firmware Version Constants

### FW_LIST Definition

The `FW_LIST` constant defines the iteration order for firmware fields:

`FW_LIST = [    "fw_bundle_version",    "tt_flash_version",    "cm_fw",    "cm_fw_date",    "eth_fw",    "bm_bl_fw",    "bm_app_fw",]`
This list drives both data collection (`get_firmware_versions()`) and display formatting (`format_firmware_rows()`).

Sources: [tt_smi/constants.py 123-131](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/constants.py#L123-L131)

### SMBUS Telemetry Fields

Firmware versions are read from specific fields in the SMBUS telemetry structure:

| Python Field | Telemetry Enum |
| --- | --- |
| `ARC0_FW_VERSION` | ARC (Chip Management) firmware |
| `ETH_FW_VERSION` | Ethernet firmware |
| `M3_BL_FW_VERSION` | M3 bootloader firmware |
| `M3_APP_FW_VERSION` | M3 application firmware |
| `TT_FLASH_VERSION` | TT-Flash utility version |
| `FW_BUNDLE_VERSION` | Firmware bundle version |
| `WH_FW_DATE` | Firmware build date |

Sources: [tt_smi/constants.py 8-64](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/constants.py#L8-L64)

## Table Header Configuration

The firmware table header is defined by `FIRMWARES_TABLE_HEADER`:

`FIRMWARES_TABLE_HEADER = [    "#",    "FW Bundle Version",    "TT-Flash Version",    "CM FW Version",    "CM FW Date",    "ETH FW Version",    "BM BL Version",    "BM App Version",]`
The table uses `header_height=2` to accommodate multi-line column headers.

## Integration with Backend

The firmware tab integrates with the backend through the following data flow:

**Backend Properties**:

*   `backend.firmware_infos`: List of dictionaries, one per device, containing firmware version data
*   `backend.devices`: List of `PciChip` objects representing detected devices
*   `backend.smbus_telem_info`: Raw telemetry data from SMBUS

## Key Differences from Other Tabs

| Aspect | Firmware Tab | Telemetry Tab | Device Info Tab |
| --- | --- | --- | --- |
| Update Frequency | Static (once at init) | 100ms real-time updates | Static (once at init) |
| Data Source | SMBUS firmware fields | SMBUS telemetry fields | SMBUS + PCI sysfs |
| Thread Usage | No background thread | Uses telemetry thread | No background thread |
| Conversion | Hex to semver | Hex to float/int | Minimal conversion |
| Platform Differences | Galaxy workaround | WH vs BH telemetry | PCIe properties |

**Note**: Unlike the Telemetry Tab which updates continuously via a background thread, the Firmware Tab displays a static snapshot. Firmware versions do not change during normal operation and require a device reset or power cycle to update.
