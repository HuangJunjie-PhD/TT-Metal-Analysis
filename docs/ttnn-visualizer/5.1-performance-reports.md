The Performance Reports feature in the TT-NN Visualizer provides detailed insights into the execution performance of Tenstorrent Neural Network operations. It allows you to analyze operation-level metrics such as execution time, memory usage, throughput, and mathematical fidelity in a searchable, sortable tabular format. For operation-specific memory analysis, see [Operation Details](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.2-operation-details).

## Overview

Performance Reports display detailed metrics about each operation executed by the TT-NN model, including timing information, resource utilization, and optimization opportunities. The data is generated during model execution and can be loaded from either local or remote files.

## Performance Data Structure

The performance report displays a comprehensive set of metrics for each operation in a tabular format. The table includes the following key columns:

| Column | Description |
| --- | --- |
| ID | Unique identifier for the operation |
| Total % | Percentage of total execution time |
| Bound | Performance bottleneck (DRAM, FLOP, etc.) |
| OP Code | Type of operation (Matmul, LayerNorm, etc.) |
| Device Time | Execution time on the device (µs) |
| Op-to-Op Gap | Latency between operations (µs) |
| Cores | Number of cores used for the operation |
| DRAM | DRAM bandwidth utilization (GB/s) |
| DRAM % | Percentage of peak DRAM bandwidth |
| FLOPs | Floating point operations (TFLOPs) |
| FLOPs % | Percentage of peak FLOPs |
| Math Fidelity | Numerical precision used (HiFi4, HiFi2, LoFi) |

## Backend Implementation

The backend implements several key classes to handle performance data:

### CSV Query Runners

The system provides two query runners for accessing performance data:

1.   `LocalCSVQueryRunner`: Handles CSV files stored locally
2.   `RemoteCSVQueryRunner`: Accesses CSV files stored on remote servers via SSH

Both implementations provide similar interfaces for executing queries against CSV data, with methods to:

*   Execute filtered queries on CSV data
*   Retrieve CSV headers
*   Apply formatting and transformations

*   [backend/ttnn_visualizer/csv_queries.py 19-92](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/backend/ttnn_visualizer/csv_queries.py#L19-L92)

### Performance Queries

The `OpsPerformanceQueries` class specifically handles operations performance data:

*   Locates and loads the appropriate performance CSV file
*   Provides methods to query by operation code or retrieve all entries
*   Handles both local and remote file access patterns

The CSV files follow a standard format with a prefix of `ops_perf_results` and contain detailed metrics about each operation's execution.

Sources:

*   [backend/ttnn_visualizer/csv_queries.py 389-561](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/backend/ttnn_visualizer/csv_queries.py#L389-L561)

### Performance Report Generation

The `OpsPerformanceReportQueries` class uses the external `tt-perf-report` library to generate formatted performance reports:

*   Processes raw CSV data into a structured report
*   Calculates additional metrics like total percentage and bottleneck identification
*   Generates optimization advice for operations like Matmul
*   Formats the data for frontend consumption

## Performance Metrics and Visualization

The performance report uses color-coding to highlight important performance characteristics:

### Operation Types

Operations are color-coded based on their type to help identify patterns:

*   Matmul operations: magenta
*   Attention operations: blue
*   Layer normalization: cyan
*   Convolution operations: orange
*   Torch operations: red

### Performance Indicators

Various performance metrics are highlighted to draw attention to potential issues:

*   Operations with less than the minimum percentage (0.5%) appear in grey
*   DRAM or FLOP bound operations appear in green
*   Slow operations appear in yellow
*   High dispatch latency (>6.5µs) appears in red
*   Suboptimal core utilization appears in red (full utilization in green)

### Math Fidelity Analysis

The system analyzes the appropriateness of the chosen math fidelity (HiFi4, HiFi2, LoFi) based on input and output datatypes:

*   Appropriate fidelity appears in green
*   Too high fidelity (wasting performance) appears in red
*   Too low fidelity (potential accuracy issues) appears in cyan

## High Dispatch Operation Analysis

The Performance Report includes specialized analysis for operations with high dispatch latency (>6.5µs):

1.   Identifies operations with abnormal dispatch latency
2.   Calculates the total time spent in high dispatch operations
3.   Estimates the percentage of overall execution time that could be saved
4.   Provides recommendations for reducing dispatch latency

This analysis is only displayed when the "Highlight high dispatch ops" toggle is enabled.

## Matmul Optimization Advice

When the "Show Matmul optimization analysis" toggle is enabled, the system displays specialized advice for matrix multiplication operations:

1.   For each Matmul operation, a list of potential optimizations is presented
2.   Advice is tailored based on the specific parameters of the operation
3.   Recommendations focus on improving performance without sacrificing accuracy

This advice is generated by the `tt-perf-report` library during report generation and displayed in expandable rows below each Matmul operation.

## Integration with Other Components

The Performance Report integrates with other components of the TT-NN Visualizer:

1.   **Operation List Integration**:

    *   Links from performance report entries to corresponding operations in the Operation List
    *   Tooltips showing operation details when hovering over operation IDs

2.   **Performance Range Selection**:

    *   Uses the global selected performance range atom to filter operations
    *   Integrates with the range slider in the application footer

3.   **Remote Data Management**:

    *   Works with both local and remote performance data
    *   Integrates with the remote sync configurator for accessing remote files
