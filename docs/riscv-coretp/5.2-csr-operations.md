## Purpose and Scope

This page documents the Control and Status Register (CSR) operations available in the coretp framework through the `CsrRead` and `CsrWrite` test steps. These primitives enable test scenarios to read and modify processor control and status registers during test execution, which is essential for configuring privilege modes, enabling/disabling features, and verifying processor state.

For information about other test step types, see:

*   Memory operations: [Memory Operations](https://deepwiki.com/tenstorrent/riscv-coretp/5.1-memory-operations)
*   Assertions and verification: [Assertions and Verification](https://deepwiki.com/tenstorrent/riscv-coretp/5.3-assertions-and-verification)
*   Control flow and multi-hart operations: [Control Flow and Multi-Hart Operations](https://deepwiki.com/tenstorrent/riscv-coretp/5.4-control-flow-and-multi-hart-operations)

## CsrWrite Operations

The `CsrWrite` class enables modification of CSR values through three mutually exclusive operation modes:

| Mode | Parameter | Description | Example Usage |
| --- | --- | --- | --- |
| **Set Mask** | `set_mask` | Set specific bits to 1, leave others unchanged | Enable specific feature bits |
| **Clear Mask** | `clear_mask` | Clear specific bits to 0, leave others unchanged | Disable specific feature bits |
| **Full Value** | `value` | Write complete value to CSR | Initialize CSR to known state |

### Write Operation Modes

**Sources:**[coretp/step/csr.py 13-42](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/step/csr.py#L13-L42)

### Set Mask Examples

The `set_mask` parameter enables specific bits without affecting others. Common in enabling features or interrupt bits:

`# Enable mcounteren.tm bit (bit 1)mcounteren_set = CsrWrite(csr_name="mcounteren", set_mask=0x2) # Enable henvcfg.STCE bit (bit 63)henvcfg_set = CsrWrite(csr_name="henvcfg", set_mask=(1 << 63)) # Enable mip.LCOFIP interrupt bit (bit 13)set_lcofip = CsrWrite(csr_name="mip", set_mask=(1 << 13))`
**Sources:**[coretp/plans/sstc/sstc_scenarios.py 112-122](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sstc/sstc_scenarios.py#L112-L122)[coretp/plans/sscofpmf/sscofpmf_scenarios.py 568-569](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L568-L569)

### Clear Mask Examples

The `clear_mask` parameter disables specific bits while preserving others:

`# Disable mcounteren.tm bitmcounteren_clear = CsrWrite(csr_name="mcounteren", clear_mask=0x2) # Disable menvcfg.STCE bitmenvcfg_clear = CsrWrite(csr_name="menvcfg", clear_mask=(1 << 63)) # Stop mcycle countingset_cy = CsrWrite(csr_name="mcountinhibit", set_mask=0x1)`
**Sources:**[coretp/plans/sstc/sstc_scenarios.py 19-34](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sstc/sstc_scenarios.py#L19-L34)[coretp/plans/sscofpmf/sscofpmf_scenarios.py 177-178](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L177-L178)

### Full Value Examples

The `value` parameter replaces the entire CSR contents:

`# Write specific pattern to test counterwrite_pattern = CsrWrite(csr_name="mhpmcounter3", value=0x123456789ABCDEF0) # Reset CSR to zeroclear_mcountinhibit = CsrWrite(csr_name="mcountinhibit", value=0) # Write all ones to test WARL behaviorwrite_all_ones = CsrWrite(csr_name="mcountinhibit", value=0xFFFFFFFFFFFFFFFF)`

## Direct vs. Indirect Access

Both `CsrRead` and `CsrWrite` support `direct_read` and `direct_write` boolean flags that control privilege mode handling:

| Mode | Behavior | Use Case |
| --- | --- | --- |
| **Indirect (default)** | Framework handles privilege switching | Testing cross-privilege access control |
| **Direct** | Execute instruction in current privilege mode | Testing same-privilege operations or exception generation |

### Direct Access Pattern

Direct access is used when testing access violations or when the current privilege mode is correct:

`# Test illegal instruction exception in non-M modetime_read = CsrRead(csr_name="time", direct_read=True)assert_exception = AssertException(    cause=ExceptionCause.ILLEGAL_INSTRUCTION,     code=[time_read])`
**Sources:**[coretp/plans/sstc/sstc_scenarios.py 22-23](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sstc/sstc_scenarios.py#L22-L23)

## Common Usage Patterns

### Pattern 1: Enable-Test-Disable

**Sources:**[coretp/plans/sstc/sstc_scenarios.py 112-143](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sstc/sstc_scenarios.py#L112-L143)

### Pattern 2: Before-After Counter Comparison

Used extensively in PMU tests to verify counter behavior:

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 109-126](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L109-L126)

Example implementation:

`clear_cy = CsrWrite(csr_name="mcounteren", clear_mask=0x1)read_before = CsrRead(csr_name="mcycle") # Fill with arithmetic operationsfor _ in range(NUM_FILL):    steps.append(Arithmetic()) read_after = CsrRead(csr_name="mcycle")delta = Arithmetic(op="sub", src1=read_after, src2=read_before)zero = LoadImmediateStep(imm=0)assert_incremented = AssertNotEqual(src1=delta, src2=zero)`
**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 111-122](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L111-L122)

### Pattern 3: Bit Isolation and Verification

Extract and verify specific bit fields within CSR values:

`# Read mip and check LCOFIP bit (bit 13)read_mip = CsrRead(csr_name="mip")lcofip_mask = LoadImmediateStep(imm=(1 << 13))lcofip_value = Arithmetic(op="and", src1=read_mip, src2=lcofip_mask)zero = LoadImmediateStep(imm=0)assert_lcofip_clear = AssertEqual(src1=lcofip_value, src2=zero)`
**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 333-335](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L333-L335)

### Pattern 4: WARL Behavior Testing

Write-Any-Read-Legal (WARL) testing verifies that reserved bits remain unchanged:

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 48-76](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L48-L76)

Example from SSCOFPMF testing reserved bits:

`# Write all ones to mhpmevent3write_all_ones = CsrWrite(csr_name="mhpmevent3", value=0xFFFFFFFFFFFFFFFF)read_back = CsrRead(csr_name="mhpmevent3") # Check reserved bits [57:56] are clearreserved_mask = LoadImmediateStep(imm=0x0300000000000000)reserved_bits = Arithmetic(op="and", src1=read_back, src2=reserved_mask)zero = LoadImmediateStep(imm=0)assert_reserved_clear = AssertEqual(src1=reserved_bits, src2=zero)`
**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 52-58](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L52-L58)

## StepOrInt Type

The `CsrWrite` class uses the `StepOrInt` type hint for its parameters, allowing values to be either:

*   **Direct integer values**: Immediate constants like `0x2` or `(1 << 63)`
*   **TestStep dependencies**: Results from previous steps like `LoadImmediateStep` or `Arithmetic`

This flexibility enables both simple operations and complex computed values:

`# Simple integer valuesimple_write = CsrWrite(csr_name="mcounteren", clear_mask=0x2) # Computed value from LoadImmediateStepmask = LoadImmediateStep(imm=(1 << 63))computed_write = CsrWrite(csr_name="menvcfg", set_mask=mask) # Result from arithmetic operationread_val = CsrRead(csr_name="mhpmcounter3")delta = Arithmetic(op="add", src1=read_val, src2=1)incremented_write = CsrWrite(csr_name="mhpmcounter3", value=delta)`
**Sources:**[coretp/step/csr.py 9-37](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/step/csr.py#L9-L37)

## Summary of CSR Operations in Test Scenarios

CSR operations form the foundation of processor state management in coretp test scenarios:

| Operation | Primary Use Cases | Common Patterns |
| --- | --- | --- |
| **CsrWrite with set_mask** | Enable features, set interrupt bits | Enable-test-disable sequences |
| **CsrWrite with clear_mask** | Disable features, clear interrupt bits | Access control testing |
| **CsrWrite with value** | Initialize state, test WARL behavior | WARL verification, counter resets |
| **CsrRead** | Capture state, verify values | Read-verify, before-after delta |
| **direct_read/write** | Test access violations | Exception testing with AssertException |

The combination of these operations with arithmetic steps and assertions enables comprehensive verification of RISC-V CSR behavior across privilege modes, virtual memory configurations, and extension features.
