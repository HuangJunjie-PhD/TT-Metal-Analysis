## Common Patterns

### PyTorch Model Test Flow

Most PyTorch text model tests follow a uniform structure. The model is loaded from `third_party.tt_forge_models`, optionally wrapped to adapt its forward signature, compiled with `forge.compile`, and verified with `verify`.

### `TextModelWrapper`

Many LLM tests wrap the loaded model with `TextModelWrapper` before calling `forge.compile`. This class is imported from `test.models.models_utils` and accepts an optional `text_embedding` parameter, which receives the model's token embedding layer (e.g., `model.model.embed_tokens`).

`# Pattern with embedding extraction (CausalLM tasks)framework_model = TextModelWrapper(model=model, text_embedding=model.model.embed_tokens) # Pattern without embedding extraction (classification tasks)framework_model = TextModelWrapper(model=model)`
Models using `TextModelWrapper`: Llama 3, Gemma, Phi2, Phi3, Phi4, Qwen2, Phi3.5.

Sources: [forge/test/models/pytorch/text/llama/test_llama3.py 110-113](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/text/llama/test_llama3.py#L110-L113)[forge/test/models/pytorch/text/gemma/test_gemma_2b.py 46-47](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/text/gemma/test_gemma_2b.py#L46-L47)[forge/test/models/pytorch/text/phi2/test_phi2.py 76](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/text/phi2/test_phi2.py#L76-L76)[forge/test/models/pytorch/text/phi3/test_phi3.py 62](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/text/phi3/test_phi3.py#L62-L62)

### Inline `Wrapper` Classes

Some tests define a local `Wrapper` class to adapt the model's forward signature. For example, Falcon 7B Instruct's test passes `None` for the KV-cache argument:

`class Wrapper(torch.nn.Module):    def forward(self, input_ids, attention_mask):        return self.model(input_ids, None, attention_mask)`
T5 and NanoGPT use analogous inline wrappers for ONNX export.

Sources: [forge/test/models/pytorch/text/falcon/test_falcon.py 44-51](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/text/falcon/test_falcon.py#L44-L51)[forge/test/models/onnx/text/t5/test_t5_onnx.py 20-28](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/onnx/text/t5/test_t5_onnx.py#L20-L28)

## xfail Patterns Summary

Most large LLMs in this suite are registered but explicitly failed because single-chip Tenstorrent hardware cannot run models of that parameter count.

| Model | Passing Variants | xfail Reason |
| --- | --- | --- |
| Falcon | `FALCON_1B` | Other variants: multi-chip |
| Llama 3 | `LLAMA_3_2_1B`, `LLAMA_3_2_1B_INSTRUCT` | Others: multi-chip / OOM |
| Qwen 2 | None | multi-chip |
| Gemma | None | multi-chip / issue #2844 |
| Mistral | None | multi-chip |
| Phi2 | None | multi-chip |
| Phi3 | None | multi-chip |
| Phi3.5 Mini | None | Segfault |
