This page documents the dataset infrastructure in TT-Blacksmith: the abstract base class, the `get_dataset()` factory, all concrete dataset subclasses, and shared collation utilities. For model loading and fine-tuning adaptation methods, see page 5.1.

## Architecture Overview

All PyTorch datasets extend `BaseDataset` and are retrieved via the `get_dataset()` factory in `dataset_utils.py`. The factory dispatches on `config.dataset_id` to select the appropriate concrete class.

## PyTorch Dataset Infrastructure

### `BaseDataset` Abstract Class

`BaseDataset` in [blacksmith/datasets/torch/torch_dataset.py 42-99](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/torch_dataset.py#L42-L99) is the abstract base for all PyTorch dataset implementations. It inherits from `torch.utils.data.Dataset` and `abc.ABC`.

| Method | Abstract | Purpose |
| --- | --- | --- |
| `__init__(config, split, collate_fn)` | No | Stores config/split/collate_fn, calls `_prepare_dataset()` |
| `_prepare_dataset()` | **Yes** | Load and preprocess the raw dataset |
| `_get_dataloader()` | **Yes** | Build and return a `DataLoader` |
| `get_dataloader()` | No | Calls `_get_dataloader()`, then wraps with `_prepare_test_dataloader()` |
| `__len__()` | No | Returns `len(self.dataset)` |
| `__getitem__(idx)` | No | Returns a single example dict |

The public entry point for callers is `get_dataloader()`, not `_get_dataloader()`. The `_prepare_test_dataloader()` helper checks whether `config.test_config.max_steps_per_epoch` is set and, if so, wraps the result in a `TestDataLoaderWrapper` before returning it.

Sources: [blacksmith/datasets/torch/torch_dataset.py 42-99](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/torch_dataset.py#L42-L99)

### `TestDataLoaderWrapper`

`TestDataLoaderWrapper` in [blacksmith/datasets/torch/torch_dataset.py 13-39](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/torch_dataset.py#L13-L39) wraps a `DataLoader` and limits the number of batches yielded per epoch using `itertools.islice`. It is used in CI test runs to bound training time without changing the training script logic.

| Attribute | Description |
| --- | --- |
| `dataloader` | The underlying `DataLoader` |
| `max_steps_per_epoch` | Maximum batches to yield per epoch |

`__len__` returns `min(len(dataloader), max_steps_per_epoch)`. `__getattr__` delegates all other attribute access to the wrapped dataloader, so callers see it as a transparent proxy.

Sources: [blacksmith/datasets/torch/torch_dataset.py 13-39](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/torch_dataset.py#L13-L39)

### `get_dataset()` Factory

`get_dataset()` in [blacksmith/datasets/torch/dataset_utils.py 30-52](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/dataset_utils.py#L30-L52) is the single entry point that experiment scripts call to obtain a dataset. It dispatches on `config.dataset_id` (case-insensitive) using the `AvailableDataset` enum.

### SST-2 Dataset Implementation

The `SSTDataset` class implements sentiment analysis on the Stanford Sentiment Treebank (SST-2) dataset from the GLUE benchmark. It formats movie review sentences with prompt templates for causal language model fine-tuning.

#### Tokenization and Prompt Formatting

[blacksmith/datasets/torch/sst2/sst2_dataset.py 32-54](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_dataset.py#L32-L54) implements tokenization with prompt formatting:

The dataset loads from HuggingFace using the `dataset_id` and `dataset_configuration` from the training config (typically `"glue"` and `"sst2"`). Each example is processed through `_tokenize_function()` which:

1.   Formats the input sentence using `PROMPT_TEMPLATE`
2.   Formats the sentiment label (0=negative, 1=positive) using `RESPONSE_TEMPLATE`
3.   Concatenates prompt and response into full text
4.   Tokenizes the full text
5.   Creates labels by cloning input_ids and masking the prompt portion with -100 (ignored during loss computation)
6.   Filters examples exceeding `max_length`

The prompt templates are defined in [blacksmith/datasets/torch/sst2/sst2_utils.py 7-23](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_utils.py#L7-L23):

*   `PROMPT_TEMPLATE`: Simple format `"Review: $input\nOutput:"`
*   `RESPONSE_TEMPLATE`: JSON format `'{"label": "$label"}'`
*   `LBL2VALUE`: Maps numeric labels to string values (0→"negative", 1→"positive")

#### DataLoader Creation

[blacksmith/datasets/torch/sst2/sst2_dataset.py 77-93](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_dataset.py#L77-L93) creates the DataLoader with:

*   `DataCollatorForSeq2Seq` for padding to `max_length`
*   Optional custom `collate_fn` (e.g., `collate_fn_for_causal_lm` for label shifting)
*   Shuffling enabled for training split
*   `drop_last=True` to ensure consistent batch sizes

**Sources:**[blacksmith/datasets/torch/sst2/sst2_dataset.py 1-94](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_dataset.py#L1-L94)[blacksmith/datasets/torch/sst2/sst2_utils.py 1-24](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_utils.py#L1-L24)

### Text-to-SQL Dataset Implementation

The `TextToSQLDataset` class implements instruction-following for SQL query generation tasks. It uses the `Clinton/Text-to-sql-v1` dataset with template-based prompt formatting.

#### Prompt Structure

[blacksmith/datasets/torch/text2sql/text2sql_dataset.py 15-22](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/text2sql/text2sql_dataset.py#L15-L22) defines the instruction prompt template:

`PROMPT_TEMPLATE = Template(    """### Instruction:\nGenerate an SQL query for the given question and database schema.\n\n### Input:\nQuestion: $prompt\nSchema: $context\n\n### Output:\n""")`
This three-section format (Instruction, Input, Output) follows common instruction-tuning patterns.

#### Tokenization Pipeline

[blacksmith/datasets/torch/text2sql/text2sql_dataset.py 41-67](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/text2sql/text2sql_dataset.py#L41-L67) implements `_tokenize_function()`:

1.   Extracts `sql_prompt`, `sql_context`, and `sql` from the example
2.   Substitutes values into `PROMPT_TEMPLATE`
3.   Appends target SQL as `target_text`
4.   Tokenizes the full sequence
5.   Creates labels with prompt portion masked (-100)
6.   Filters to only include examples with `sql_complexity == "basic SQL"`
7.   Filters examples exceeding `max_length`

The dataset uses the same `DataCollatorForSeq2Seq` pattern as SST-2, with optional custom collate functions for label shifting required by causal language models.

**Sources:**[blacksmith/datasets/torch/text2sql/text2sql_dataset.py 1-108](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/text2sql/text2sql_dataset.py#L1-L108)

## JAX Dataset Implementation

JAX experiments use a different pattern based on functional programming and NumPy arrays rather than PyTorch's Dataset/DataLoader abstractions.

### SST-2 for DistilBERT

[blacksmith/datasets/jax/distil_bert/sst2_dataset.py 8-24](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/jax/distil_bert/sst2_dataset.py#L8-L24) implements `load_sst2()`:

The function:

1.   Loads raw GLUE SST-2 dataset using HuggingFace `datasets.load_dataset()`
2.   Applies `tokenize_batch()` with truncation and padding to `max_length`
3.   Renames `"label"` column to `"labels"` for consistency
4.   Sets format to `"np"` for NumPy array output
5.   Returns train split, validation split, and column names

### Batch Iterator Pattern

[blacksmith/datasets/jax/distil_bert/sst2_dataset.py 28-44](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/jax/distil_bert/sst2_dataset.py#L28-L44) implements `numpy_batch_iter()`, a generator function that:

*   Takes dataset, batch size, columns, shuffle flag, and optional seed
*   Creates an infinite iterator over the dataset
*   Shuffles the order at the start of each epoch if enabled
*   Yields batches as dictionaries with NumPy arrays
*   Casts arrays to appropriate dtypes (`np.int32` for input_ids, attention_mask, labels)

This functional approach fits JAX's design philosophy and enables explicit control over shuffling and batching:

`def numpy_batch_iter(dataset, batch_size, columns, shuffle=True, seed=None):    n = len(dataset)    rng = np.random.default_rng(seed)    order = np.arange(n)        while True:        if shuffle:            rng.shuffle(order)                for start in range(0, n, batch_size):            idx = order[start : start + batch_size]            batch = {k: dataset[k][idx] for k in columns}            # Cast to int32            yield batch`
The iterator is used in training loops without explicit DataLoader objects, allowing direct integration with JAX's functional programming model. See [blacksmith/experiments/jax/distil_bert/test_distil_bert_flax.py 158](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/distil_bert/test_distil_bert_flax.py#L158-L158) for usage in training.

**Sources:**[blacksmith/datasets/jax/distil_bert/sst2_dataset.py 1-45](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/jax/distil_bert/sst2_dataset.py#L1-L45)[blacksmith/experiments/jax/distil_bert/test_distil_bert_flax.py 154-159](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/distil_bert/test_distil_bert_flax.py#L154-L159)

## Common Data Collation Patterns

### Label Shifting for Causal Language Models

Causal language models predict the next token, requiring labels to be shifted by one position relative to inputs. TT-Blacksmith implements this in two ways:

1.   **Pre-shifted in Dataset**: [blacksmith/datasets/torch/sst2/sst2_dataset.py 42-46](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_dataset.py#L42-L46) and [blacksmith/datasets/torch/text2sql/text2sql_dataset.py 55-59](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/text2sql/text2sql_dataset.py#L55-L59) create labels by masking the prompt portion with -100, so labels represent target outputs.

2.   **Shifted in Training Loop**: When using `collate_fn_for_causal_lm`, the training code performs explicit shifting:

    *   [blacksmith/experiments/torch/gemma/test_gemma_finetuning.py 128-130](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/gemma/test_gemma_finetuning.py#L128-L130) shifts logits: `shift_logits = logits[:, :-1, :].contiguous()`
    *   Labels from the dataset are already pre-shifted to align with these shifted logits

This pattern ensures that token at position `i` in the input predicts token at position `i+1` in the labels.

### Padding and Truncation

All datasets use `max_length` from `TrainingConfig` to control sequence lengths:

*   **PyTorch**: `DataCollatorForSeq2Seq` with `padding="max_length"` pads all sequences to `max_length`
*   **JAX**: Tokenization includes `padding="max_length"` parameter directly in the tokenizer call

Truncation is applied during tokenization with `truncation=True` (JAX) or by filtering examples exceeding `max_length` after tokenization (PyTorch).

**Sources:**[blacksmith/tools/torch_helpers.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/tools/torch_helpers.py) (referenced but not in provided files), [blacksmith/datasets/torch/sst2/sst2_dataset.py 78-80](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/sst2/sst2_dataset.py#L78-L80)[blacksmith/datasets/torch/text2sql/text2sql_dataset.py 92-94](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/text2sql/text2sql_dataset.py#L92-L94)[blacksmith/datasets/jax/distil_bert/sst2_dataset.py 11-17](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/jax/distil_bert/sst2_dataset.py#L11-L17)

## Adding New Datasets

To add a new dataset implementation:

1.   **For PyTorch**: Create a new class inheriting from `BaseDataset` in `blacksmith/datasets/torch/<dataset_name>/`

    *   Implement `_prepare_dataset()` to load and tokenize data
    *   Implement `get_dataloader()` to create DataLoader with appropriate collation
    *   Add prompt templates if using instruction formatting
    *   Add any preprocessing utilities in a separate utils file

2.   **For JAX**: Create a new module in `blacksmith/datasets/jax/<dataset_name>/`

    *   Implement a `load_<dataset_name>()` function returning tokenized data
    *   Implement a batch iterator function if custom batching logic is needed
    *   Ensure outputs are NumPy arrays with appropriate dtypes

3.   **Update Configuration**: Add dataset-specific configuration fields to the experiment's `TrainingConfig` class

4.   **Integration**: Import and instantiate the dataset class in the training script, passing the `TrainingConfig` object
