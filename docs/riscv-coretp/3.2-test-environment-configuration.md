## Purpose and Scope

This document explains the test environment configuration system in coretp, which defines and resolves the execution context for test scenarios. The system uses a template-instance pattern where **TestEnvCfg** templates define parameter spaces (privilege modes, paging modes, register widths, etc.), **TestEnvSolver** resolves these templates into valid **TestEnv** instances, and **predicates** filter out invalid configurations.

For information about how test scenarios use these environments, see [Test Step System](https://deepwiki.com/tenstorrent/riscv-coretp/3.3-test-step-system). For information about creating test scenarios that specify environment configurations, see [Writing Test Scenarios](https://deepwiki.com/tenstorrent/riscv-coretp/6.2-writing-test-scenarios).

## TestEnvCfg: Environment Templates

`TestEnvCfg` defines the parameter space for test execution. Each field contains a list of possible values, and the configuration can generate all combinations.

### Class Structure

The class is defined as a frozen dataclass at [coretp/env/cfg.py 11-27](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/cfg.py#L11-L27)

### Default Values

| Field | Default Value | Description |
| --- | --- | --- |
| `reg_widths` | `[64]` | Register width in bits (typically 32 or 64) |
| `priv_modes` | `[M, S, U]` | All privilege modes: Machine, Supervisor, User |
| `hypervisor` | `[True, False]` | Whether hypervisor extension is enabled |
| `paging_modes` | `[DISABLED, SV39, SV48, SV57]` | All paging modes |
| `page_sizes` | `[SIZE_4K, SIZE_2M, SIZE_1G]` | All page sizes |
| `min_num_harts` | `1` | Minimum number of hardware threads |
| `virtualized` | `[True, False]` | Whether execution is virtualized |
| `deleg_excp_to` | `[M, S, U]` | Privilege modes exceptions can be delegated to |

**Sources:**[coretp/env/cfg.py 19-26](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/cfg.py#L19-L26)

## TestEnv: Concrete Environments

`TestEnv` represents a specific, concrete test environment with single values for each parameter. These instances are used by test scenarios to determine execution context.

### Class Structure

The class is defined as a frozen dataclass at [coretp/env/env.py 9-23](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/env.py#L9-L23):

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `reg_width` | `int` | `64` | Register width in bits |
| `priv` | `PrivilegeMode` | `PrivilegeMode.M` | Privilege mode for execution |
| `paging_mode` | `PagingMode` | `PagingMode.DISABLED` | Virtual memory paging mode |
| `page_size` | `frozenset[PageSize]` | `{SIZE_4K}` | Set of supported page sizes |
| `hart_count` | `int` | `1` | Number of hardware threads |
| `hypervisor` | `bool` | `False` | Hypervisor extension enabled |
| `virtualized` | `bool` | `False` | Execution is virtualized |

## TestEnvSolver: Configuration Resolution

`TestEnvSolver` resolves one or more `TestEnvCfg` templates into a list of valid `TestEnv` instances by applying validation predicates.

### Resolution Process

**Sources:**[coretp/env/solver.py 35-70](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L35-L70)

### Solver Initialization

The solver is initialized with an optional list of predicates at [coretp/env/solver.py 21-25](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L21-L25):

`TestEnvSolver(predicates: Optional[list[Callable[[TestEnv], bool]]] = None, debug: bool = False)`
*   If `predicates` is `None`, uses `DEFAULT_PREDICATES()`
*   If `predicates` is an empty list `[]`, no validation is applied
*   `debug` enables verbose output of predicate evaluation

**Sources:**[coretp/env/solver.py 21-25](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L21-L25)

### Solver Methods

| Method | Purpose |
| --- | --- |
| `solve(cfgs)` | Main entry point - generates and filters `TestEnv` instances |
| `_generate_all_envs(cfgs)` | Generates all combinations from all `TestEnvCfg` objects |
| `remove_duplicates(seq)` | Removes duplicate `TestEnv` instances while preserving order |
| `add_predicate(predicate)` | Dynamically adds a validation predicate |

**Sources:**[coretp/env/solver.py 27-70](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L27-L70)

## Predicates: Environment Validation

Predicates are callable functions that validate `TestEnv` instances. They return `True` if the environment is valid, `False` otherwise.

### Predicate Function Signature

`Callable[[TestEnv], bool]`
A predicate takes a `TestEnv` instance and returns a boolean.

**Sources:**[coretp/env/predicates.py 4-13](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/predicates.py#L4-L13)

### Default Predicates

The framework provides default predicates that enforce architectural constraints:

The `machine_no_paging` predicate enforces that Machine mode must always have paging disabled, as per RISC-V specification. Implementation at [coretp/env/predicates.py 16-23](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/predicates.py#L16-L23)

**Sources:**[coretp/env/predicates.py 16-27](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/predicates.py#L16-L27)

### Custom Predicates

Test scenarios can define custom predicates for specialized validation:

`def custom_predicate(env: TestEnv) -> bool:    # Example: Require SV39 paging for supervisor mode    if env.priv == PrivilegeMode.S:        return env.paging_mode == PagingMode.SV39    return True solver = TestEnvSolver()solver.add_predicate(custom_predicate)`
Custom predicates are added via `TestEnvSolver.add_predicate()` at [coretp/env/solver.py 27-33](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L27-L33)

## Usage Patterns

### Pattern 1: Full Parameter Space

Generate all possible configurations by using default values:

`cfg = TestEnvCfg()  # Uses all defaultsenvs = cfg.generate_all_cfgs()  # Generates hundreds of combinations`
This produces the Cartesian product of all default parameter lists. Useful for comprehensive coverage but generates many configurations.

**Sources:**[coretp/env/cfg.py 28-38](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/cfg.py#L28-L38)

### Pattern 2: Constrained Configuration

Specify limited parameter ranges to reduce configuration count:

`cfg = TestEnvCfg(    priv_modes=[PrivilegeMode.S],    paging_modes=[PagingMode.SV39, PagingMode.SV48])envs = cfg.generate_all_cfgs()  # Only S-mode with SV39/SV48`
**Sources:**[coretp/env/cfg.py 11-27](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/cfg.py#L11-L27)

### Pattern 3: Multi-Configuration Resolution

Use `TestEnvSolver` to combine multiple configurations with validation:

`cfg1 = TestEnvCfg(priv_modes=[PrivilegeMode.M])cfg2 = TestEnvCfg(priv_modes=[PrivilegeMode.S], paging_modes=[PagingMode.SV39]) solver = TestEnvSolver()envs = solver.solve([cfg1, cfg2])  # Combines both, removes invalid`
The solver merges all configurations, removes duplicates, and applies predicates to filter invalid combinations.

**Sources:**[coretp/env/solver.py 35-70](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L35-L70)

### Pattern 4: Custom Validation

Add custom predicates for specialized validation requirements:

`def require_hypervisor(env: TestEnv) -> bool:    return env.hypervisor solver = TestEnvSolver()solver.add_predicate(require_hypervisor)envs = solver.solve([cfg])  # Only returns envs with hypervisor enabled`
**Sources:**[coretp/env/solver.py 27-33](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/solver.py#L27-L33)[coretp/env/predicates.py 9-13](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/env/predicates.py#L9-L13)

## Configuration Flow in Test Scenarios

Test scenarios define `TestEnvCfg` to specify their execution requirements. The framework uses `TestEnvSolver` to resolve these into concrete `TestEnv` instances, then executes the test steps in each environment.
