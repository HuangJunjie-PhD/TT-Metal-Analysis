For general information about neural network workloads and the TTSIM framework, see [Neural Network Workloads](https://deepwiki.com/tenstorrent/polaris/4.1-neural-network-workloads). ## YOLO v8 Object Detection Model

The YOLO v8 implementation demonstrates a complete object detection pipeline built using the TTSIM framework. The model follows the standard YOLO architecture with a backbone-neck-head structure for real-time object detection.

### YOLO v8 Architecture

### YOLO v8 Component Details

The YOLO v8 model consists of several key components implemented as `SimNN.Module` subclasses:

| Component | Purpose | Key Operations |
| --- | --- | --- |
| `Conv` | Basic convolution block | `F.Conv2d`, `F.BatchNorm2d`, `SiLU` activation |
| `C2f` | CSP Bottleneck | Split, multiple `Bottleneck` layers, concat |
| `SPPF` | Spatial pyramid pooling | Multiple `F.MaxPool2d` with different receptive fields |
| `DetectV8` | Detection head | Separate box and class prediction branches |
| `DFL` | Distribution Focal Loss | `F.Conv2d` for distribution-based bbox regression |

### YOLO v8 Model Configuration

The model supports different variants (yolov8n, yolov8s, yolov8m, yolov8l, yolov8x) through configuration parameters that scale depth and width multipliers.

## BEVDepth 3D Object Detection Model

BEVDepth is a complex computer vision model for 3D object detection that processes multi-camera inputs to generate bird's-eye-view representations. It demonstrates advanced techniques like camera-aware depth estimation and voxel pooling.

### BEVDepth Architecture Overview

### BEVDepth Backbone Components

The `BaseLSSFPN` backbone is the core component that transforms multi-camera images into a unified 3D representation:

| Component | Class | Purpose |
| --- | --- | --- |
| Image Backbone | `ResNet` | Extract features from camera images |
| Image Neck | `SecondFPN` | Multi-scale feature fusion |
| Depth Network | `DepthNet` | Camera-aware depth estimation |
| ASPP Module | `ASPP` | Atrous spatial pyramid pooling |
| Voxel Pooling | `F.VoxelPooling` | Transform to 3D voxel representation |

### BEVDepth Processing Pipeline

### BEVDepth Specialized Components

The model includes several specialized components for 3D perception:

**DepthNet Class**: Performs camera-aware depth estimation using intrinsic and extrinsic camera parameters.

**ASPP Module**: Implements atrous spatial pyramid pooling for multi-scale feature extraction with different dilation rates.

**SELayer**: Squeeze-and-excitation attention mechanism for adaptive feature recalibration.

**Mlp**: Multi-layer perceptron for camera parameter processing and feature transformation.

## Tensor Operations in Model Implementation

Both YOLO v8 and BEVDepth models rely extensively on tensor operations provided by the TTSIM framework. These operations enable PyTorch-like tensor manipulation within the simulation environment.

### Common Tensor Operations

### Tensor Operation Examples

The models demonstrate various tensor operations:

**View Operations**: Used in YOLO v8 for reshaping detection outputs and in BEVDepth for camera matrix transformations.

**Concatenation**: Essential for feature fusion in both models, implemented through `cat()` and `stack()` functions.

**Interpolation**: BEVDepth uses `interpolate()` for upsampling features to match spatial dimensions.

**Slicing**: Complex indexing operations for extracting specific camera parameters and feature regions.

Sources: [ttsim/front/functional/tensor_op.py 1-574](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/tensor_op.py#L1-L574)

## Model Integration with TTSIM Framework

Both example models demonstrate key patterns for integrating with the TTSIM simulation framework:

### Framework Integration Patterns

### Configuration and Parametrization

Both models support flexible configuration through YAML files:

**YOLO v8 Configuration**: Model variants defined through `depth_multiple` and `width_multiple` parameters, with layer specifications in backbone and head sections.

**BEVDepth Configuration**: Separate configuration files for backbone and head components, with detailed camera and 3D perception parameters.

### Parameter Counting and Analysis

Both models implement `analytical_param_count()` methods that recursively calculate the total number of trainable parameters, enabling performance analysis and comparison.
