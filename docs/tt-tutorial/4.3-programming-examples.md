These examples illustrate fundamental concepts including integer addition using compute kernels and RISC-V processors, showcasing different approaches to low-level hardware programming.

## Compute Kernel Programming Example

The compute kernel example demonstrates the full TT-Metalium programming pipeline using dedicated data movement and compute kernels.

### Core Program Setup

**Compute Example Device and Memory Setup**

The program initializes the device, creates buffers for source and destination data, and establishes the core coordinate system for kernel execution.

Sources: [tt-metalium/prog_examples/add_2_integers_in_compute.md 26-63](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-metalium/prog_examples/add_2_integers_in_compute.md?plain=1#L26-L63)

### Circular Buffer Architecture

**Circular Buffer Configuration for Compute Example**

The example creates three circular buffers: two for input data (`c_0`, `c_1`) and one for output data (`c_16`), all using `Float16_b` data format.

Sources: [tt-metalium/prog_examples/add_2_integers_in_compute.md 65-81](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-metalium/prog_examples/add_2_integers_in_compute.md?plain=1#L65-L81)

### Kernel Execution Pipeline

**Kernel Execution Pipeline for Compute Example**

The execution follows a three-stage pipeline: data movement in (reader), computation (compute kernel), and data movement out (writer).

Sources: [tt-metalium/prog_examples/add_2_integers_in_compute.md 85-118](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-metalium/prog_examples/add_2_integers_in_compute.md?plain=1#L85-L118)

### Compute Kernel Operations

The compute kernel performs tile-based addition using the following key operations:

*   `binary_op_init_common()` - Initialize binary operation
*   `add_tiles_init()` - Initialize tile addition
*   `cb_wait_front()` - Wait for input data availability
*   `add_tiles()` - Perform tile addition operation
*   `pack_tile()` - Pack result to output buffer

Sources: [tt-metalium/prog_examples/add_2_integers_in_compute.md 166-192](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-metalium/prog_examples/add_2_integers_in_compute.md?plain=1#L166-L192)

## RISC-V Programming Example

The RISC-V example demonstrates simplified computation using data movement processors for basic arithmetic operations.

### Simplified Architecture

**RISC-V Example Simplified Architecture**

The RISC-V approach uses a single kernel that handles data movement and computation, eliminating the need for separate reader, writer, and compute kernels.

Sources: [tt-metalium/prog_examples/add_2_integers_in_riscv.md 84-105](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-metalium/prog_examples/add_2_integers_in_riscv.md?plain=1#L84-L105)

### RISC-V Kernel Operations

The RISC-V kernel performs the following operations in sequence:

1.   `noc_async_read()` - Read data from DRAM to L1 circular buffers
2.   Direct pointer arithmetic - `dat0[0] = dat0[0] + dat1[0]`
3.   `noc_async_write()` - Write result from L1 to DRAM

This approach demonstrates how data movement processors can perform basic computation tasks without requiring dedicated compute kernels.

Sources: [tt-metalium/prog_examples/add_2_integers_in_riscv.md 128-146](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-metalium/prog_examples/add_2_integers_in_riscv.md?plain=1#L128-L146)

## Programming Approach Comparison

| Aspect | Compute Kernel Approach | RISC-V Approach |
| --- | --- | --- |
| **Kernel Count** | 3 (reader, writer, compute) | 1 (combined) |
| **Processors Used** | RISC-V 0, 1 + Compute Engine | RISC-V 0 only |
| **Computation Method** | `add_tiles()` tile operations | Direct pointer arithmetic |
| **Data Format** | Tile-based (`Float16_b`) | Raw integer values |
| **Complexity** | Higher (full pipeline) | Lower (direct computation) |
| **Use Case** | Complex mathematical operations | Simple data movement + computation |

The compute kernel approach provides the foundation for complex mathematical operations and leverages the full TT-Metalium compute pipeline, while the RISC-V approach offers a simpler path for basic computational tasks that don't require dedicated compute resources.
