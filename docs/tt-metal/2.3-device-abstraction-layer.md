This document describes the device abstraction layer in TT-Metal, which provides a unified interface for managing Tenstorrent hardware devices. The device abstraction handles device lifecycle, memory management, coordinate transformations, and workload execution across single and multi-chip configurations.

For information about cluster management and device discovery, see [Cluster Management and Device Discovery (2.2)](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/Cluster%20Management%20and%20Device%20Discovery%20(2.2)) For details on program execution and command queues, see [Fast Dispatch and Command Queue System (2.5)](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/Fast%20Dispatch%20and%20Command%20Queue%20System%20(2.5))

## Device Interface Hierarchy

The device abstraction layer is built on a three-tier interface hierarchy that supports single devices, multi-device meshes, and provides common functionality across both.

### Class Hierarchy and Core Entities

The `IDevice` abstract base class defines the contract for all device types, while `Device` and `MeshDevice` provide specific implementations for single-chip and multi-chip topologies respectively.

## Multi-Device Abstraction (MeshDevice)

`MeshDevice` provides a unified interface for multi-chip systems, abstracting collections of devices arranged in mesh topologies. It implements `IDevice`, allowing users to treat a mesh as a single logical entity for certain operations.

### MeshDevice Architecture

The `MeshDevice` uses a PIMPL pattern with `MeshDeviceImpl` to manage a `MeshDeviceView` (the logical topology) and `ScopedDevices` (the physical device ownership).

## Memory Management and Allocators

Devices integrate with `AllocatorImpl` to manage L1 and DRAM. The `L1BankingAllocator` is the primary implementation, providing a banked memory model that simplifies data movement for kernels.

### Allocator Initialization

The device initializes its allocator by querying the SOC descriptor for core types and generating a configuration that defines reserved vs. unreserved L1 regions. The `AllocatorMode` (Hybrid or Lockstep) is determined by `RunTimeOptions`.

## Sub-Device Management

TT-Metalium supports partitioning a device into multiple logical sub-devices. This is managed by the `SubDeviceManager` and tracked by `SubDeviceManagerTracker`.

### Sub-Device Tracking

The `SubDeviceManagerTracker` manages the lifetime and switching of `SubDeviceManager` instances on a device. It ensures that the global allocator is correctly resized when switching between sub-device configurations.
