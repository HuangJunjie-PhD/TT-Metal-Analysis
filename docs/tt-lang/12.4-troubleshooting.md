## Error Detection Pipeline

The tt-lang system validates kernels at multiple stages, catching errors early with precise source location information. The pipeline moves from high-level Python syntax checks to low-level hardware constraint validation.

## Common Validation Errors

### Tensor and Device Constraints

The compiler and runtime API enforce strict requirements for tensor locations and grid dimensions. These are often caught during `__call__` time or MLIR type construction.

| Error Message | Cause | Solution |
| --- | --- | --- |
| `ValueError: No device found` | Passing a host tensor (on CPU) to an operation requiring a device. | Move tensors to device using `ttnn.to_device(tensor, device)`. [test/python/test_missing_device.py 78-79](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/test_missing_device.py#L78-L79) |
| `ValueError: no ttnn tensor arguments were provided` | Calling a `grid='full'` operation with only torch tensors or non-tensor args. | Ensure at least one `ttnn.Tensor` is passed to infer the device. [test/python/test_missing_device.py 102-103](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/test_missing_device.py#L102-L103) |
| `ValueError: All ttnn tensor inputs are on host` | Multiple `ttnn.Tensor` inputs provided, but none are on a Tenstorrent device. | At least one input must be on-device for the kernel to execute. [test/python/test_missing_device.py 119-122](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/test_missing_device.py#L119-L122) |
| `ValueError: different devices` | Inputs are located on different physical devices or device IDs. | Ensure all input tensors reside on the same device. [test/python/test_missing_device.py 210-212](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/test_missing_device.py#L210-L212) |

### Grid and Core Constraints

| Error Message | Cause | Solution |
| --- | --- | --- |
| `ValueError: Only 2D grids supported` | Attempting to use a 3D grid in `@ttl.operation`. | Ensure the `grid` parameter is a 2D tuple (e.g., `(1, 1)`). |
| `core() currently only supports dims=2` | Calling `ttl.node(dims=3)` inside a kernel. | Core/node indexing currently only supports 2D coordinates. |

## Runtime Issues and Simulation Errors

When running kernels in the functional simulator, specific runtime violations are caught that might lead to hangs on real hardware.

### Deadlock Detection

A common issue in kernel development is a circular dependency between threads (MATH, BRISC, NCRISC). The simulator's `GreenletScheduler` detects when all threads are blocked waiting for buffer space or data.

### Buffer Locking (ROR/RAW)

The simulator tracks the state of every buffer block. Attempting to write to a block while it is a source for an active `ttl.copy` operation will trigger a "Read-Only" (ROR) violation.

## Diagnostic Utilities

The `ttl.diagnostics` module provides Rust-style error formatting. When the MLIR compiler fails, the `format_mlir_error` function renders the diagnostic stream with source context.

### Error Formatting Features

*   **Source Context**: Displays the exact line and column from the Python file. [python/ttl/diagnostics.py 44-55](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/diagnostics.py#L44-L55)
*   **Note Indentation**: Notes are indented 2 spaces to show subordination to the primary error. [python/ttl/diagnostics.py 197-200](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/diagnostics.py#L197-L200)
*   **Deduplication**: Repeated notes at the same location are suppressed. [python/ttl/diagnostics.py 191-195](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/diagnostics.py#L191-L195)
*   **Internal Filtering**: MLIR-internal notes like `see current operation` are stripped to focus on user code. [python/ttl/diagnostics.py 189-190](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/diagnostics.py#L189-L190)

## Environment Variables

| Variable | Purpose | Usage |
| --- | --- | --- |
| `TTLANG_COMPILE_ONLY=1` | Skip hardware execution; only run the compilation pipeline. | `os.environ["TTLANG_COMPILE_ONLY"] = "1"` |
| `TTLANG_VERBOSE_ERRORS=1` | Append raw MLIR diagnostics to the formatted error message. | `export TTLANG_VERBOSE_ERRORS=1`. [python/ttl/diagnostics.py 39-42](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/diagnostics.py#L39-L42) |
