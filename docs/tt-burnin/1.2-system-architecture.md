The tt-burnin system follows a layered architecture with clear separation between hardware abstraction, workload management, and user interface components. The main execution flow orchestrates chip detection, reset operations, workload loading, and real-time monitoring.

## Reset and Device Management Operations

The system implements comprehensive device reset capabilities supporting different board types and reset methodologies, including PCI-based resets and IPMI-based galaxy resets.

| Reset Type | Target Hardware | Implementation | Function |
| --- | --- | --- | --- |
| Grayskull Reset | GS chips | `GSTensixReset` | `pci_board_reset()` |
| Wormhole Reset | WH chips | `WHChipReset.full_lds_reset()` | `pci_board_reset()` |
| Blackhole Reset | BH chips | `BHChipReset.full_lds_reset()` | `pci_board_reset()` |
| Galaxy Reset | WH Galaxy | `run_wh_ubb_ipmi_reset()` | `reset_6u_glx()` |

The reset system supports JSON configuration files for complex multi-board setups and can selectively reset subsets of devices while maintaining proper re-initialization sequences.
