This document covers the `ModelTester` base class and its implementation pattern through `ThisTester` subclasses used across tt-torch's model testing suite. The framework provides a consistent interface for testing PyTorch models with various compilation configurations and execution modes.

For information about test configuration management and dynamic test discovery, see [Test Configuration and Management](https://deepwiki.com/tenstorrent/tt-torch/4.2.2-test-configuration-and-management). For details about hardware-specific test execution, see [Hardware Matrix Testing](https://deepwiki.com/tenstorrent/tt-torch/4.2.3-hardware-matrix-testing).

## Framework Architecture

The Model Testing Framework centers around the `ModelTester` base class, which provides a standardized interface for testing models across different backends, compilation modes, and hardware configurations. Each model test implements a `ThisTester` class that inherits from `ModelTester` and provides model-specific loading logic.

## ThisTester Implementation Pattern

Each model test implements a `ThisTester` class following a consistent pattern with three core methods that must be implemented:

### Required Method Implementations

**Model Loading Pattern**

`class ThisTester(ModelTester):    def _load_model(self):        # Load and configure the model        model = load_framework_model()        model = model.to(torch.bfloat16)        return model`
**Input Loading Pattern**

`def _load_inputs(self):    # Load and preprocess inputs    inputs = load_test_data()    inputs = inputs.to(torch.bfloat16)    return inputs`
**Output Extraction Pattern (Optional)**

`def _extract_outputs(self, output_object):    # Extract specific outputs from complex return types    return (output_object["pred_logits"], output_object["pred_boxes"])`

## Configuration Management

The framework uses `CompilerConfig` to control compilation behavior and execution modes:

### CompilerConfig Setup Pattern

### Common Configuration Patterns

| Configuration | Usage | Example |
| --- | --- | --- |
| Op-by-Op Execution | Debugging and incremental testing | `cc.compile_depth = CompileDepth.EXECUTE_OP_BY_OP` |
| Full Compilation | Production testing | `cc.compile_depth = CompileDepth.EXECUTE` |
| Constant Evaluation | Parameter optimization | `cc.enable_consteval = True` |
| Data Parallelism | Multi-device testing | `data_parallel_mode = True` |
| Mesh Configuration | N300 hardware | `cc.mesh_shape = [1, 2]` |
