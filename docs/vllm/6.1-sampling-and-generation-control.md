Sampling & Text Generation in vLLM converts the probability distributions (logits) from language models into concrete token sequences that form the generated text. This system handles the decision-making process of how to select the next token at each step of generation, applying various techniques to control the randomness, creativity, and constraints of the generated text.

This page documents the sampling parameters, algorithms, implementation details, and how text generation is controlled in the vLLM system. For information about the core architecture of vLLM, see [Core Architecture](https://deepwiki.com/tenstorrent/vllm/2-core-architecture).

## Sampling Parameters

The `SamplingParams` class encapsulates all parameters that control how tokens are selected during text generation. These parameters provide fine-grained control over the generation process.

### Core Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `temperature` | float | 1.0 | Controls randomness. Lower values make output more deterministic, higher values make output more random. Zero means greedy sampling. |
| `top_p` | float | 1.0 | Nucleus sampling parameter. Only consider tokens whose cumulative probability exceeds this value. |
| `top_k` | int | -1 | Only consider the top k tokens. Set to -1 to consider all tokens. |
| `min_p` | float | 0.0 | Only consider tokens with probability >= min_p * (probability of most likely token). |
| `repetition_penalty` | float | 1.0 | Penalty for repeating tokens. Values > 1 discourage repetition. |
| `presence_penalty` | float | 0.0 | Penalty for tokens that appear in the text so far. Values > 0 encourage new tokens. |
| `frequency_penalty` | float | 0.0 | Penalty based on frequency of tokens in the text so far. Values > 0 encourage rarer tokens. |
| `n` | int | 1 | Number of completions to generate. |
| `best_of` | int | None | Number of completions to generate and return the best n of them. |
| `seed` | int | None | Random seed for reproducible sampling. |
| `max_tokens` | int | 16 | Maximum number of tokens to generate. |
| `min_tokens` | int | 0 | Minimum number of tokens to generate before EOS token is allowed. |
| `stop` | list[str] | None | Stop generation when one of these strings is generated. |
| `stop_token_ids` | list[int] | None | Stop generation when one of these token IDs is generated. |
| `ignore_eos` | bool | False | Whether to ignore the EOS token and continue generating. |
| `bad_words` | list[str] | None | Words that are not allowed to be generated. |

Sources: [vllm/sampling_params.py 108-239](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L108-L239)

## Sampling Process

The sampling process is implemented in the `Sampler` class. This class transforms raw logits into token selections through a series of operations.

### Logit Transformation Pipeline

Sources: [vllm/model_executor/layers/sampler.py 216-323](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L216-L323)

### Key Sampling Functions

The key steps in the sampling process are:

1.   **Min Tokens Penalty**: Prevents stop tokens from being generated before `min_tokens` are produced

    *   Implemented in `_apply_min_tokens_penalty` which sets logits of stop tokens to -infinity
    *   Sources: [vllm/model_executor/layers/sampler.py 340-384](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L340-L384)

2.   **Apply Penalties**: Modifies logits based on tokens that have already appeared in the text

    *   Applies presence, frequency, and repetition penalties
    *   Sources: [vllm/model_executor/layers/sampler.py 261-266](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L261-L266)

3.   **Temperature Scaling**: Divides logits by temperature to control randomness

    *   Higher temperature = more randomness, lower = more deterministic
    *   Sources: [vllm/model_executor/layers/sampler.py 268-271](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L268-L271)

4.   **Top-K & Top-P Filtering**: Implements nucleus and top-k sampling

    *   `_apply_top_k_top_p`: Masks out tokens with low probabilities
    *   Sources: [vllm/model_executor/layers/sampler.py 387-413](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L387-L413)

5.   **Min-P Filtering**: Implements min-p sampling to ensure diversity

    *   Only considers tokens with probability >= min_p * (highest token probability)
    *   Sources: [vllm/model_executor/layers/sampler.py 416-430](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L416-L430)

6.   **Token Sampling**: Selects tokens based on the sampling type

    *   `_sample`: Coordinates the sampling process based on sampling type
    *   Sources: [vllm/model_executor/layers/sampler.py 755-783](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L755-L783)

### Greedy vs. Random Sampling

The system supports three primary sampling methods:

1.   **Greedy Sampling** (`_greedy_sample`): Always selects the token with the highest probability

    *   Used when temperature = 0
    *   Sources: [vllm/model_executor/layers/sampler.py 433-466](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L433-L466)

2.   **Random Sampling** (`_random_sample`): Samples tokens based on their probabilities

    *   Used when temperature > 0 and no seed is specified
    *   Sources: [vllm/model_executor/layers/sampler.py 467-509](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L467-L509)

3.   **Seeded Random Sampling**: Same as random sampling but with reproducible results

    *   Used when temperature > 0 and a seed is specified
    *   Sources: [vllm/model_executor/layers/sampler.py 467-509](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L467-L509)

## Text Generation Control

vLLM provides several mechanisms to control the text generation process, including constraints on token counts and stop conditions.

### Generation Constraints

1.   **Min Tokens**: Ensures a minimum number of tokens are generated before EOS or stop tokens are allowed

    *   Implemented in `_apply_min_tokens_penalty`
    *   Sources: [vllm/model_executor/layers/sampler.py 340-384](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L340-L384)

2.   **Max Tokens**: Limits the maximum number of tokens generated per sequence

    *   Tracked in the Sequence object and checked during generation
    *   Sources: [vllm/sampling_params.py 208](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L208-L208)

3.   **Stop Sequences/Tokens**: Stops generation when specific strings or tokens are generated

    *   Configured via `stop` and `stop_token_ids` parameters
    *   Sources: [vllm/sampling_params.py 148-152](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L148-L152)

4.   **Bad Words**: Prevents specific words from being generated

    *   Implemented via token ID masking
    *   Sources: [vllm/sampling_params.py 156-507](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L156-L507)

## Performance Optimizations

vLLM's sampling implementation includes several optimizations for performance:

1.   **Batched Sampling**: Processes multiple sequences at once

2.   **GPU Acceleration**: Sampling operations are performed on GPU

3.   **Efficient Multinomial Sampling**: Custom implementation to avoid GPU-CPU sync

 
Sources: [vllm/model_executor/layers/sampler.py 517-536](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L517-L536)

4.   **FlashInfer Integration**: Optional high-performance kernel for top-k top-p sampling

 
Sources: [vllm/model_executor/layers/sampler.py 539-574](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/sampler.py#L539-L574)

5.   **Pinned Memory**: Used for efficient host-device data transfer

    *   Reduces latency when transferring sampling tensors to GPU
    *   Sources: [vllm/model_executor/sampling_metadata.py 514-536](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/sampling_metadata.py#L514-L536)
