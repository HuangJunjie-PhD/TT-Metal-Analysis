## Purpose and Scope

This page documents the ELF and DWARF parsing subsystem in TTExaLens, covering how firmware ELF files are read, how DWARF debug information is traversed, and the data structures that make symbolic access to firmware memory possible.

The outputs of this subsystem — `ParsedElfFile`, `ElfDie`, and `ElfVariable` — are consumed by the callstack unwinding system (see [Callstack Unwinding](https://deepwiki.com/tenstorrent/tt-exalens/7.5-callstack-unwinding)) and the symbolic variable access system (see [Symbolic Variable System](https://deepwiki.com/tenstorrent/tt-exalens/7.4-symbolic-variable-system)). For the ELF _loading_ (writing firmware to device memory), see [ELF Loader Implementation](https://deepwiki.com/tenstorrent/tt-exalens/6.6-elf-loader-implementation). For higher-level library API functions like `parse_elf` and `load_elf`, see [ELF Management](https://deepwiki.com/tenstorrent/tt-exalens/3.5-elf-management).

## Entry Point: `read_elf`

`read_elf` in [ttexalens/elf/parsed.py 353-368](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L353-L368) is the single entry point for consuming an ELF file.

```
read_elf(file_ifc, elf_file_path, load_address=None, require_debug_symbols=True) -> ParsedElfFile
```

| Parameter | Type | Description |
| --- | --- | --- |
| `file_ifc` | `FileAccessApi` | Abstraction over local/remote file access (see [Remote Device Access](https://deepwiki.com/tenstorrent/tt-exalens/7.2-remote-device-access)) |
| `elf_file_path` | `str` | Path to the ELF file |
| `load_address` | `int | None` | If set, returns a `ParsedElfFileWithOffset` to handle relocated firmware |
| `require_debug_symbols` | `bool` | Raises `ValueError` if the ELF has no DWARF info and this is `True` |

When `load_address` is `None`, a plain `ParsedElfFile` is returned. When `load_address` is provided, a `ParsedElfFileWithOffset` wraps it. This is needed when firmware is loaded at a different address than its link-time address — the offset adjusts all PC-to-symbol lookups accordingly.

Sources: [ttexalens/elf/parsed.py 353-368](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L353-L368)

## `ParsedElfFile`

`ParsedElfFile`[ttexalens/elf/parsed.py 145-298](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L145-L298) is the central object. It wraps a `pyelftools``ELFFile` and exposes DWARF information through Python `cached_property` attributes — meaning each property is computed once on first access and then cached.

### Cached Properties

| Property | Type | Description |
| --- | --- | --- |
| `variables` | `dict[str, ElfDie]` | All global variables found in DWARF, keyed by fully-qualified name |
| `types` | `dict[str, ElfDie]` | All type definitions (structs, classes, typedefs, enums) |
| `members` | `dict[str, ElfDie]` | All struct/class members |
| `enumerators` | `dict[str, ElfDie]` | All enum values |
| `subprograms` | `dict[str, ElfDie]` | All functions/methods found in DWARF |
| `symbols` | `dict[str, ElfDwarfSymbol]` | Symbol table from `.symtab`, each with `value` (address) and `size` |
| `file_line` | `dict[int, tuple[str, int, int]]` | PC address → (filename, line, column) mapping |
| `frame_info` | `FrameInfoProvider` | DWARF Call Frame Information for stack unwinding |
| `sections` | `list[ParsedElfFileSection]` | All ELF sections with name, address, size, and lazy data |
| `code_load_address` | `int` | Address of the `.text` (or `.firmware_text`) section |

All of `variables`, `types`, `members`, `enumerators`, and `subprograms` are populated by a single traversal via `recurse_dwarf()`[ttexalens/elf/parsed.py 62-89](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L62-L89) which walks every DIE in every compile unit.

### Methods

| Method | Returns | Description |
| --- | --- | --- |
| `find_die_by_name(name)` | `ElfDie | None` | Fast hierarchical lookup by `"::"` separated name; cached with `@cache` |
| `get_global(name, mem_access)` | `ElfVariable` | Returns a live `ElfVariable` backed by `mem_access`; reads on demand |
| `read_global(name, mem_access)` | `ElfVariable` | Like `get_global`, but immediately reads all bytes into a `CachedReadMemoryAccess` |
| `get_constant(name)` | `int | float | bool` | Returns the compile-time constant value for a `constexpr` variable |
| `get_enum_value(name)` | `int | None` | Returns the integer value of a named enumerator |

Sources: [ttexalens/elf/parsed.py 145-298](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L145-L298)

### `ParsedElfFileWithOffset`

`ParsedElfFileWithOffset`[ttexalens/elf/parsed.py 301-350](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L301-L350) subclasses `ParsedElfFile` and adapts all properties to a different load address. The `loaded_offset` field is `code_load_address - load_address`. Internally, it delegates `variables`, `types`, `symbols`, and `file_line` to the underlying `ParsedElfFile` to avoid redundant work, while providing an `ElfDwarfWithOffset` and `FrameInfoProviderWithOffset` that apply the offset to PC-based lookups.

Sources: [ttexalens/elf/parsed.py 301-350](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L301-L350)

## DWARF Traversal

The function `recurse_dwarf`[ttexalens/elf/parsed.py 62-89](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L62-L89) iterates over all compile units provided by `ElfDwarf`, then recursively visits every DIE. Each DIE is categorized and stored in a flat `dict[category][path]` structure.

**DWARF traversal data flow:**

Sources: [ttexalens/elf/parsed.py 25-89](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L25-L89)[ttexalens/elf/die.py 76-114](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L76-L114)

### DIE Categories

`ElfDie.category`[ttexalens/elf/die.py 76-114](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L76-L114) maps DWARF tags to one of the following strings:

| Category | DWARF Tags | Stored in |
| --- | --- | --- |
| `"variable"` | `DW_TAG_variable` | `ParsedElfFile.variables` |
| `"type"` | `DW_TAG_*_type`, `DW_TAG_typedef`, `DW_TAG_namespace` | `ParsedElfFile.types` |
| `"member"` | `DW_TAG_member` | `ParsedElfFile.members` |
| `"enumerator"` | `DW_TAG_enumerator` | `ParsedElfFile.enumerators` |
| `"subprogram"` | `DW_TAG_subprogram` | `ParsedElfFile.subprograms` |
| `"inlined_function"` | `DW_TAG_inlined_subroutine` | traversed but not stored |
| `"lexical_block"` | `DW_TAG_lexical_block` | traversed but not stored |

Tags in `IGNORE_TAGS` (e.g. `DW_TAG_compile_unit`, `DW_TAG_formal_parameter`) halt recursion. Tags like `DW_TAG_inheritance` and `DW_TAG_imported_declaration` return `None` for category and are silently skipped.

Sources: [ttexalens/elf/die.py 27-114](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L27-L114)

## `ElfDie`

`ElfDie`[ttexalens/elf/die.py 44](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L44-LNaN) is a thin wrapper around pyelftools' `DIE`. It provides typed, cached access to the most commonly used DWARF attributes.

**Key `ElfDie` properties:**

### Key Property Semantics

**`name`** — Decoded from `DW_AT_name`. For pointers with no name, synthesized as `"PointedType*"`. Falls back to `DW_AT_specification` for out-of-class definitions.

**`path`** — Fully qualified name including parent namespaces and types, e.g. `"GlobalStruct::InnerStruct"`. Uses `"::"` as a separator. For subprograms with a `DW_AT_specification`, resolves the path from the declaration.

**`resolved_type`** — Follows `DW_TAG_typedef`, `DW_TAG_const_type`, and `DW_TAG_volatile_type` chains recursively until a concrete type is found. For non-type DIEs, resolves the `DW_AT_type` attribute. For enumerations, resolves to the underlying integer type.

**`address`** — For global variables, extracted from the `DW_AT_location` DWARF expression (typically `DW_OP_addr`). For struct members, from `DW_AT_data_member_location`. For union members, returns `0`. Falls back to the `.symtab` symbol table if no DWARF location is available.

**`size`** — From `DW_AT_byte_size`. For arrays, computed as element size × element count. For pointers, hardcoded to 4 (32-bit target assumption). Falls back to the symbol table size.

Sources: [ttexalens/elf/die.py 44-300](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L44-L300)

## `ElfVariable`

`ElfVariable`[ttexalens/elf/variable.py 20](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L20-LNaN) is the runtime interface for reading and writing typed firmware data. It holds three things: a type `ElfDie`, a memory address, and a `MemoryAccess` object. It does **not** read memory until a value is actually requested.

For full coverage of `ElfVariable`'s operator overloads, iteration support, and `MemoryAccess` interface, see [Symbolic Variable System](https://deepwiki.com/tenstorrent/tt-exalens/7.4-symbolic-variable-system). A brief summary of the most commonly used methods follows.

| Method / Operator | Description |
| --- | --- |
| `.member_name` / `get_member(name)` | Navigate struct/union members; handles unnamed unions and inheritance |
| `[int]` | Array element or pointer offset indexing |
| `[str]` | Equivalent to `get_member(str)` |
| `read_value()` | Read scalar value (`int`, `float`, or `bool`) from memory |
| `write_value(value)` | Write scalar value to memory |
| `dereference()` | Follow a pointer; reads address from memory, returns new `ElfVariable` |
| `read()` | Reads the entire object into a `CachedReadMemoryAccess`, returning a new `ElfVariable` that requires no further hardware reads |
| `get_address()` | Returns the raw memory address |
| `get_size()` | Returns `type_die.size` |
| `as_list()` / `as_value_list()` | Materializes array elements |

Sources: [ttexalens/elf/variable.py 20-618](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L20-L618)

## Symbol Table Decoding

`decode_symbols`[ttexalens/elf/parsed.py 99-109](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L99-L109) reads the `.symtab` ELF section and returns `dict[str, ElfDwarfSymbol]`. Only symbols with types `STT_NOTYPE`, `STT_FUNC`, or `STT_OBJECT` are included.

`ElfDwarfSymbol`[ttexalens/elf/parsed.py 93-96](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L93-L96) is a simple dataclass:

```
@dataclass
class ElfDwarfSymbol:
    value: int | None   # address
    size: int | None
```

The symbol table is used as a fallback in `ElfDie.size` and `ElfDie.address` when DWARF attributes are absent.

Sources: [ttexalens/elf/parsed.py 92-109](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L92-L109)

## PC-to-Source Mapping

`decode_file_line`[ttexalens/elf/parsed.py 112-130](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L112-L130) parses DWARF line programs to produce a `dict[int, tuple[str, int, int]]` mapping each program counter address to a `(filename, line, column)` tuple. This map is exposed as `ParsedElfFile.file_line` and is used during callstack formatting to annotate each frame with a source location.

## `FrameInfoProvider` and CFI

`FrameInfoProvider`[ttexalens/elf/frame.py 102-120](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/frame.py#L102-L120) parses DWARF Call Frame Information (CFI) from the `.debug_frame` or `.eh_frame` section. On construction, it iterates all CFI entries and retains only `FDE` (Frame Description Entry) objects, stored as `(start_address, end_address, fde)` tuples.

`get_frame_description(pc, risc_debug)` returns a `FrameDescription`[ttexalens/elf/frame.py 16-70](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/frame.py#L16-L70) for the given PC if one exists. `FrameDescription` decodes the FDE table and provides:

*   `read_register(register_index, cfa)` — applies CFI register restore rules (currently supports `OFFSET` type)
*   `read_previous_cfa(current_cfa)` — computes the Canonical Frame Address for the calling frame

`FrameInfoProviderWithOffset`[ttexalens/elf/frame.py 123-132](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/frame.py#L123-L132) adapts the PC lookup by adding `loaded_offset` before searching the FDE list, matching the address-adjustment behavior of `ParsedElfFileWithOffset`.

`FrameInspection`[ttexalens/elf/frame.py 73-99](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/frame.py#L73-L99) ties together a `RiscDebug` instance, a `loaded_offset`, an optional `FrameDescription`, and a CFA value. Its `pc` property reads program counter register 32 (the RISC-V PC). It is used by `ElfDie.read_value()` when resolving local variable locations that depend on register values or stack offsets.

For how these classes are used to unwind callstacks, see [Callstack Unwinding](https://deepwiki.com/tenstorrent/tt-exalens/7.5-callstack-unwinding).

## `find_die_by_name`

`find_die_by_name`[ttexalens/elf/parsed.py 206-231](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L206-L231) provides fast, cached lookup by splitting a name on `"::"` and walking the DIE tree hierarchically. It handles:

*   `DW_AT_abstract_origin` — resolves to the abstract origin die (for inlined functions)
*   `DW_AT_specification` — resolves to the declaration die (for out-of-class definitions)
*   `DW_AT_declaration` — stores as a fallback `declaration_die` if no concrete definition is found

Results are cached with `@cache`, making repeated lookups for the same name O(1) after the first call.

Sources: [ttexalens/elf/parsed.py 206-231](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L206-L231)

## Member Resolution in `ElfVariable`

When navigating struct members, three resolution strategies are tried in order [ttexalens/elf/variable.py 31-51](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L31-L51):

1.   **Direct child lookup** — `type_die.get_child_by_name(member_name)` finds a direct `DW_TAG_member`.
2.   **Unnamed struct/union members** — `_resolve_unnamed_struct_union_member()`[ttexalens/elf/variable.py 53-69](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L53-L69) searches through anonymous unions/structs that have no `DW_AT_name`.
3.   **Inheritance** — `_resolve_inheritance_member()`[ttexalens/elf/variable.py 71-90](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L71-L90) walks `DW_TAG_inheritance` children recursively, accumulating offsets.

If none of these succeed, an exception is raised with the full `type::member` path for diagnostic clarity.

For pointer-type variables, member access automatically dereferences before looking up the member [ttexalens/elf/variable.py 38-39](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L38-L39)

Sources: [ttexalens/elf/variable.py 31-90](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L31-L90)

## `ElfCompileUnit` and `ElfDwarf`

These are internal classes not intended for direct user consumption.

*   **`ElfDwarf`** (`ttexalens/elf/dwarf.py`) wraps `DWARFInfo` from pyelftools and provides `iter_CUs()`, `get_die()`, `location_parser`, `range_lists`, and a reference back to the `ParsedElfFile`. `ElfDwarfWithOffset` adapts it for relocated firmware.
*   **`ElfCompileUnit`** (`ttexalens/elf/cu.py`) wraps a single DWARF compile unit. It caches `ElfDie` instances by `DWARF_DIE` pointer to avoid duplicate wrapper objects. Methods include `find_DIE_at_local_offset()`, `find_DIE_that_specifies()`, `get_die()`, and exposes `top_DIE`, `line_program`, `expression_parser`, and `version`.

Sources: [ttexalens/elf/__init__.py 1-21](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/__init__.py#L1-L21)[ttexalens/elf/die.py 44-58](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L44-L58)
