## Purpose and Scope

This document details the TTNN (Tenstorrent Neural Network) Python API surface, operation registration, and configuration systems. TTNN provides a high-level Python interface for tensor abstractions, device/mesh management, and runtime configuration tailored for Tenstorrent hardware.

Key topics include:

*   The global `CONFIG` configuration object and its initialization sequence.
*   Operation registration, C++ nanobindings, and documentation generation.
*   Device management API functions and dispatch core configurations.
*   The type system including tensor specifications, memory models, and sharding strategies.
*   Distributed multi-device tensor handling and mesh topologies.

This page serves as a technical leaf node documenting the core Python API and configuration layers of TTNN.

* * *

## Configuration System (CONFIG)

TTNN exposes a global singleton configuration `ttnn.CONFIG` initialized in the Python package `ttnn` upon import. It sources settings from a JSON config file (path from `TTNN_CONFIG_PATH` environment variable) and environment variable overrides `TTNN_CONFIG_OVERRIDES`.

### Configuration Loading and Management

Upon module import, the configuration loading sequence is:

*   **Initialization:** The C++ core initializes a default config object exposed as `ttnn._ttnn.core.Config`[ttnn/ttnn/__init__.py 20-21](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/__init__.py#L20-L21)
*   **File Loading:** If `TTNN_CONFIG_PATH` is set and the file exists, `load_config_from_json_file` applies updates [ttnn/ttnn/__init__.py 44-52](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/__init__.py#L44-L52)
*   **Overrides:**`TTNN_CONFIG_OVERRIDES` allows applying JSON-formatted overrides on top of the current configuration [ttnn/ttnn/__init__.py 75-77](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/__init__.py#L75-L77)
*   **Context Management:** The `manage_config(name, value)` context manager allows temporary changes to config keys, restoring original values (or safe defaults like empty strings) on exit [ttnn/ttnn/__init__.py 82-103](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/__init__.py#L82-L103)

Sources: [ttnn/ttnn/__init__.py 20-103](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/__init__.py#L20-L103)

## Operation Registration and C++ Bindings

TTNN operations are primarily implemented in C++ and exposed to Python via `nanobind`. This system handles complex overloads, docstring injection, and type conversion.

### Nanobind Operation Dispatch

Operations like `ttnn.add`, `ttnn.clamp`, and `ttnn.matmul` are bound using specialized macros and templates to handle different input types (e.g., Tensor-Tensor vs. Tensor-Scalar).

*   **Unary Operations:** Bindings in `unary_nanobind.cpp` use `ttnn::bind_function` to map Python names to C++ implementations like `ttnn::clamp`[ttnn/cpp/ttnn/operations/eltwise/unary/unary_nanobind.cpp 57-66](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/unary_nanobind.cpp#L57-L66)
*   **Binary Operations:** Bindings in `binary_nanobind.cpp` provide detailed documentation on broadcasting support (up to 5 dimensions) and performance considerations (tile format preference) [ttnn/cpp/ttnn/operations/eltwise/binary/binary_nanobind.cpp 42-52](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/binary/binary_nanobind.cpp#L42-L52) It uses function pointers for overload disambiguation between Tensor-Tensor and Tensor-Scalar variants [ttnn/cpp/ttnn/operations/eltwise/binary/binary_nanobind.cpp 55-76](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/binary/binary_nanobind.cpp#L55-L76)
*   **Composite Operations:** Some operations are implemented as composite wrappers in C++ before being bound to Python, such as `hypot_composite_wrapper`[ttnn/cpp/ttnn/operations/eltwise/binary/binary_nanobind.cpp 37-39](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/binary/binary_nanobind.cpp#L37-L39)

### Examples and Documentation

The API surface is extensively documented and mapped to examples for validation:

## Device and Mesh Management

TTNN manages hardware through `Device` and `MeshDevice` abstractions.

### Device Lifecycle

*   **Opening Devices:**`ttnn.open_device` initializes hardware with specific `l1_small_size` and `dispatch_core_config`. In the C++ nanobindings, this calls `ttnn::open_mesh_device`[ttnn/cpp/ttnn-nanobind/device.cpp 61-93](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn-nanobind/device.cpp#L61-L93)
*   **Closing Devices:**`ttnn.close_device` ensures a `synchronize_device` call before releasing hardware handles [ttnn/ttnn/device.py 34-60](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/device.py#L34-L60)
*   **Dispatch Configuration:**`DispatchCoreConfig` allows users to specify if dispatching should occur over `WORKER` or `ETH` cores and the axis (`ROW` or `COL`) [ttnn/ttnn/device.py 110-156](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/ttnn/device.py#L110-L156)

*   **Termination:**`terminate_fast_dispatch` drains work, clears fast dispatch programs, and restores stashed slow dispatch queues [tt_metal/distributed/dispatch_context.cpp 112-160](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/distributed/dispatch_context.cpp#L112-L160)

## Tensor Serialization

TTNN implements a high-performance tensor serialization format using `FlatBuffers` for headers and raw binary storage for data.

*   **Dumping Tensors:**`ttnn.dump_tensor` (or C++ `dump_tensor_flatbuffer`) serializes a tensor to a `.tensorbin` file. It supports `DISTRIBUTED_GATHER` mode to collect shards from multiple host ranks before writing [ttnn/core/tensor/serialization.cpp 53-103](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/core/tensor/serialization.cpp#L53-L103)
*   **Loading Tensors:**`ttnn.load_tensor` (or C++ `load_tensor_flatbuffer`) uses `mmap` to lazily read tensor data from disk, ensuring memory efficiency [ttnn/core/tensor/serialization.cpp 111-161](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/core/tensor/serialization.cpp#L111-L161)
