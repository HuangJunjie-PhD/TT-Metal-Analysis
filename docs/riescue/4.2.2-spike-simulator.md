## Purpose and Scope

This page documents the Spike instruction set simulator (ISS) integration within RiESCUE. Spike is the official RISC-V reference simulator used for validating RISC-V test execution. RiESCUE supports two variants: the standard `spike` from the RISC-V community and `tt_spike`, a Tenstorrent-specific fork with additional features.

For information about the Whisper simulator, see [Whisper Simulator](/tenstorrent/riescue/4.2.1-whisper-simulator). For the overall toolchain abstraction layer, see [Toolchain Abstraction Layer](/tenstorrent/riescue/7.3-toolchain-abstraction-layer).

---

## Overview

RiESCUE integrates with Spike through the `Spike` tool class, which provides a Python interface for executing compiled RISC-V ELF binaries on the simulator. The framework supports two simulator variants:

| Variant | Executable | Source Repository | Use Case |
| --- | --- | --- | --- |
| **spike** | `spike` | riscv-software-src/riscv-isa-sim | Standard RISC-V reference simulator |
| **tt\_spike** | `tt_spike` | tenstorrent/spike | Enhanced with Tenstorrent-specific features |

The default configuration uses `tt_spike` unless `--third_party_spike` is specified.

**Sources:** [riescue/lib/toolchain/tool.py273-335](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L273-L335) [infra/build\_spike.sh9-14](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L9-L14)

## Installation and Build Process

### Build Configuration

Both Spike variants are built from source during container setup. The build process uses specific commit SHAs for reproducibility.

The build script [infra/build\_spike.sh16-32](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L16-L32) defines a `build_spike()` function that:

1. Initializes a git repository in a build directory
2. Fetches the specified commit SHA
3. Runs `./configure` with variant-specific flags
4. Compiles with `make -j 12`
5. Installs the binary to system paths

**Sources:** [infra/build\_spike.sh1-69](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L1-L69)

## Spike Tool Class Implementation

### Constructor Parameters

The `Spike` class constructor accepts the following parameters:

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `spike_path` | `Optional[Path]` | `None` | Explicit path to spike executable |
| `spike_args` | `Optional[list[str]]` | `[]` | Additional command-line arguments |
| `spike_isa` | `str` | variant-specific | ISA string passed to `--isa` |
| `third_party_spike` | `bool` | `False` | Use standard spike instead of tt\_spike |
| `spike_max_instr` | `int` | `2000000` | Maximum instruction count (tt\_spike only) |

The constructor [riescue/lib/toolchain/tool.py273-302](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L273-L302) performs the following initialization:

1. **Executable Discovery**: Searches for `spike` or `tt_spike` using the inherited `find_executable()` method, checking:

   * Provided `spike_path` if specified
   * `SPIKE_PATH` environment variable
   * System PATH for executable name
2. **Base Arguments**: Appends `-l` and `--log-commits` to enable instruction logging
3. **ISA String Selection**: Sets default ISA based on variant:

   * **spike**: `RV64IMAFDCVH_zba_zbb_zbc_zfh_zbs_zfbfmin_zvfh_zvbb_zvbc_zvfbfmin_zvfbfwma_zvkg_zvkned_zvl256b_zve64d_svpbmt`
   * **tt\_spike**: `RV64IMAFDCVH_zba_zbb_zbc_zfh_zbs_zfbfmin_zvfh_zvbb_zvbc_zvfbfmin_zvfbfwma_zvkg_zvkned_zvknhb_svpbmt_sstc_zicntr`
4. **Vector Architecture**: For tt\_spike, sets `varch = {"vlen": 256, "elen": 64}` and appends `--max-instrs` argument

**Sources:** [riescue/lib/toolchain/tool.py273-302](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L273-L302)

## ISA String Configuration

### Default ISA Strings

The ISA strings differ between variants to match their capabilities:

**Standard Spike:**

```
RV64IMAFDCVH_zba_zbb_zbc_zfh_zbs_zfbfmin_zvfh_zvbb_zvbc_zvfbfmin_zvfbfwma_zvkg_zvkned_zvl256b_zve64d_svpbmt 
```

**TT Spike:**

```
RV64IMAFDCVH_zba_zbb_zbc_zfh_zbs_zfbfmin_zvfh_zvbb_zvbc_zvfbfmin_zvfbfwma_zvkg_zvkned_zvknhb_svpbmt_sstc_zicntr 
```

Notable differences:

* tt\_spike includes `zvknhb`, `sstc`, and `zicntr` extensions
* Standard spike includes `zvl256b` and `zve64d` explicitly

The ISA string can be overridden using the `--spike_isa` command-line argument.

**Sources:** [riescue/lib/toolchain/tool.py288-294](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L288-L294)

## Command-Line Interface

### Argument Specification

The `add_arguments()` static method [riescue/lib/toolchain/tool.py304-311](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L304-L311) registers the following command-line options:

* `--spike_path`: Explicit path to spike executable (overrides automatic discovery)
* `--spike_args`: List of additional arguments passed directly to spike
* `--spike_isa`: Custom ISA string (overrides defaults)
* `--third_party_spike`: Boolean flag to use standard spike instead of tt\_spike
* `--spike_max_instr`: Maximum instruction count for tt\_spike (default: 2,000,000)

**Sources:** [riescue/lib/toolchain/tool.py304-311](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L304-L311)

## Execution Flow

### run\_iss() Method

The `run_iss()` method [riescue/lib/toolchain/tool.py324-334](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L324-L334) is the primary interface for executing tests. It differs from the base `Tool.run()` method by accepting an `elf_file` parameter.

The method constructs the command line by:

1. Appending `--isa={spike_isa}` if ISA string is configured
2. Appending `--varch=vlen:{vlen},elen:{elen}` for tt\_spike
3. Adding the ELF file path
4. Calling the parent `Tool.run()` method with constructed arguments

**Sources:** [riescue/lib/toolchain/tool.py324-334](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L324-L334) [riescue/lib/toolchain/tool.py74-108](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L74-L108)

## Integration with RiescueD

### Simulation Execution

RiescueD's `simulate()` method [riescue/riescued.py311-411](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L311-L411) orchestrates Spike execution. The integration adds RiescueD-specific arguments based on `FeatMgr` configuration:

**Key Arguments Added by RiescueD:**

| Argument | Condition | Description |
| --- | --- | --- |
| `--priv=msu` | Always | Enable Machine, Supervisor, User privilege modes |
| `--pc=0x{reset_pc:x}` | Always | Set program counter to reset address |
| `--misaligned` | `not featmgr.force_alignment` | Allow misaligned memory access |
| `-p{num_cpus}` | `featmgr.mp_mode_on()` | Enable multiprocessor simulation |
| `--tt-tohost-nonzero-terminate` | `featmgr.tohost_nonzero_terminate` or `featmgr.fe_tb` | Terminate on nonzero tohost write (tt\_spike only) |
| `--big-endian` | `featmgr.big_endian` | Enable big-endian mode |
| `--end-pc {pc}` | `featmgr.wysiwyg` and failed\_pc found | Stop simulation at specific PC |

**Sources:** [riescue/riescued.py344-364](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L344-L364)

## WYSIWYG Mode Support

When `featmgr.wysiwyg` is enabled, RiescueD uses a special end-of-test mechanism that looks for `x31=0xc001c0de` to indicate test completion. This requires:

1. **Failed PC Detection**: Parse the disassembly file to find the `:` label PC address [riescue/riescued.py329-342](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L329-L342)
2. **End PC Configuration**: Add 0x10 to failed\_pc and pass to `--end-pc` argument
3. **Log Verification**: After simulation, parse the log file to verify `x31` contains `c001c0de` [riescue/riescued.py399-408](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L399-L408)

If verification fails, a `RunnerError` is raised.

**Sources:** [riescue/riescued.py329-408](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L329-L408)

## TT Spike Extensions

The `tt_spike` variant includes Tenstorrent-specific enhancements enabled during build:

| Configure Flag | Feature |
| --- | --- |
| `--enable-tt-stop-if-tohost-nonzero` | Terminates simulation when tohost address receives non-zero value |
| `--enable-tt-table-walk-debug` | Enhanced page table walk debugging output |
| `--enable-tt-expanded-dram-address-range` | Support for expanded DRAM addressing |

These features are compiled into the binary and activated via runtime command-line arguments like `--tt-tohost-nonzero-terminate`.

**Sources:** [infra/build\_spike.sh49](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L49-L49)

## Error Handling

The `_classify()` method [riescue/lib/toolchain/tool.py317-319](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L317-L319) checks the simulation result:

Any non-zero return code from Spike raises a `ToolchainError` with type `NONZERO_EXIT`, which includes:

* Tool name (`Spike`)
* Full command that was executed
* Return code
* Error output from stderr/stdout

## Container Integration

The Spike build is integrated into the container definition [infra/container/Container.def](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/container/Container.def) and managed by the container build system. The build is triggered during container creation to ensure both `spike` and `tt_spike` are available in the containerized development environment at `/usr/local/bin/spike` and `/usr/local/bin/tt_spike` respectively.

Container versions are tracked via [infra/container/singularity-id1](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/container/singularity-id#L1-L1) which stores a hash-based identifier. Any changes to Spike build configuration trigger a container rebuild.

**Sources:** [infra/build\_spike.sh1-69](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L1-L69) [infra/container/singularity-id1](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/container/singularity-id#L1-L1)
