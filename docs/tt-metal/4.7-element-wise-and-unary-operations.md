This page documents the element-wise and unary operation system in TTNN, which provides tensor operations that process elements individually or apply mathematical functions to single or multiple tensors. These operations form the foundation of neural network primitives and computational graph building blocks.

## Program Factory Selection and Memory Layout

The system selects a program factory based on the tensor's memory layout (Interleaved vs. Sharded) and the specific operation requirements.

### BinaryNg (Next-Gen) Framework

The `BinaryNgDeviceOperation` and `BinaryNgProgramFactory` provide a unified path for element-wise operations, including binary operations and unary-with-scalar cases.

## Composite Operations

Composite operations are implemented by combining multiple primitive unary, binary, or ternary operations into a more complex mathematical function.
