## Purpose and Scope

The Visualization Engine is the core rendering system that transforms model graphs into interactive WebGL-based visualizations. It handles graph layout computation, WebGL rendering, user interactions, and visual effects for the Model Explorer application. This system takes processed model data and presents it as an interactive, zoomable, and pannable graph visualization.

For information about model data processing and conversion, see [Model Conversion Adapters](https://deepwiki.com/tenstorrent/model-explorer/3.1-model-conversion-adapters). For details about the user interface components that interact with the visualization, see [User Interface Components](https://deepwiki.com/tenstorrent/model-explorer/4.3-user-interface-components).

## Core Rendering Pipeline

The rendering pipeline consists of several key stages that transform graph data into visual elements.

### WebglRenderer Component

The `WebglRenderer` class serves as the primary orchestrator for the visualization system. It manages the rendering lifecycle, coordinates between services, and handles user interactions.

| Component | Purpose | Key Methods |
| --- | --- | --- |
| `WebglRenderer` | Main rendering controller | `renderGraph()`, `updateNodesStyles()`, `handleMouseMove()` |
| `WebglRendererThreejsService` | Three.js scene management | `setupThreeJs()`, `render()`, `zoomFitGraph()` |
| `ThreejsService` | Three.js dependency management | Font loading, texture management |

The renderer maintains several critical data structures:

*   `nodesToRender`: Array of nodes to be displayed [src/ui/src/components/visualizer/webgl_renderer.ts 352](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/ui/src/components/visualizer/webgl_renderer.ts#L352-L352)
*   `edgesToRender`: Array of edges to be displayed [src/ui/src/components/visualizer/webgl_renderer.ts 354](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/ui/src/components/visualizer/webgl_renderer.ts#L354-L354)
*   `curModelGraph`: Current graph being visualized [src/ui/src/components/visualizer/webgl_renderer.ts 318](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/ui/src/components/visualizer/webgl_renderer.ts#L318-L318)

The `WebglRendererEdgeTextsService` handles rendering of labels and metadata on graph edges.

| Feature | Implementation | Configuration |
| --- | --- | --- |
| Tensor shapes | `renderEdgeTexts()` | `ShowOnEdgeItemType.TENSOR_SHAPE` |
| Output metadata | `getEdgeTextFromOutputMetadata()` | `outputMetadataKey` filter |
| Input metadata | `getEdgeTextFromInputMetadata()` | `inputMetadataKey` filter |
| Node attributes | `getEdgeTextFromNodeAttr()` | `sourceNodeAttrKey`, `targetNodeAttrKey` |

### Snapshot Service

The `WebglRendererSnapshotService` provides functionality for capturing and restoring graph visualization states.

Snapshot data includes:

*   Current viewport rectangle and zoom level
*   Expanded group node states
*   Selected node information
*   Show-on-node configuration
*   Visual settings like flatten layers mode
