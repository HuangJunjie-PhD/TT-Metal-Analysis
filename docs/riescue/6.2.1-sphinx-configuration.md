### Version Extraction

The version is dynamically extracted from [pyproject.toml](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml) at documentation build time rather than being hardcoded. This ensures the documentation version stays synchronized with the package version.

The extraction logic at [docs/source/conf.py 72-83](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L72-L83) performs the following steps:

1.   Constructs path to `pyproject.toml` relative to the config file
2.   Opens and reads the file line by line
3.   Searches for lines starting with `"version ="` or `"version="`
4.   Extracts the version string, stripping quotes
5.   Raises `ValueError` if version is not found
6.   Sets both `version` and `release` variables to the extracted version

**Sources:**[docs/source/conf.py 72-83](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L72-L83)

## Custom Directives

The configuration implements custom reStructuredText directives to extend Sphinx's capabilities beyond standard functionality.

### ArgparseDirective Class

The `ArgparseDirective` class ([docs/source/conf.py 21-56](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L21-L56)) provides automatic documentation generation for command-line argument parsers. This directive extracts argument definitions from classes that implement an `add_arguments` method.

**Implementation Details:**

| Component | Description |
| --- | --- |
| `required_arguments` | Set to 1, expects module path (e.g., `riescue.dtest_framework.RiescueD`) |
| `run()` method | Main execution method that processes the directive |
| Module import | Uses `__import__` with `fromlist` to dynamically load the target class |
| Parser creation | Creates temporary `ArgumentParser` instance for introspection |
| Node generation | Builds docutils definition list nodes for each command-line option |

The directive iterates through `parser._actions` ([docs/source/conf.py 37-54](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L37-L54)) to extract option strings and help text, creating formatted definition lists that render as readable documentation.

**Sources:**[docs/source/conf.py 21-56](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L21-L56)

## Autodoc Customization

The configuration customizes Sphinx's autodoc extension behavior through a custom documenter class that modifies how class documentation is generated.

### MockedClassDocumenter

The `MockedClassDocumenter` class ([docs/source/conf.py 59-66](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L59-L66)) extends `autodoc.ClassDocumenter` to suppress the display of `"Bases: object"` in class documentation. Since most Python classes inherit from `object` by default, this information is redundant and clutters the documentation.

The implementation overrides the `add_line()` method to filter out unwanted inheritance information:

*   If the line exactly matches `"   Bases: :py:class:`object`"`, it returns without adding the line
*   Otherwise, it calls the parent class's `add_line()` method

## Path Configuration

The configuration performs path manipulation to enable proper Python module imports during documentation generation.

### Repository Path Discovery

The path configuration process ([docs/source/conf.py 121-126](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L121-L126)):

1.   Computes repository root by going up 2 parent directories from `conf.py`
2.   Validates the path by checking for `.git` directory
3.   Raises `FileNotFoundError` if validation fails
4.   Inserts repository root at the beginning of `sys.path`

This allows Sphinx to import RiESCUE modules (e.g., `riescue.dtest_framework`) during documentation generation for autodoc and custom directives.

**Sources:**[docs/source/conf.py 121-126](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L121-L126)

## Setup Function

The `setup()` function ([docs/source/conf.py 129-131](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/source/conf.py#L129-L131)) is Sphinx's extension point for registering custom functionality. It receives the Sphinx application object and performs initialization tasks.

### Registered Components

| Registration Call | Argument | Effect |
| --- | --- | --- |
| `app.add_css_file()` | `"tt_theme.css"` | Adds custom Tenstorrent theme stylesheet |
| `app.add_directive()` | `"argparse", ArgparseDirective` | Makes `argparse` directive available |

The custom CSS file `tt_theme.css` is loaded from the `html_static_path` and provides additional styling beyond the base Read the Docs theme.
