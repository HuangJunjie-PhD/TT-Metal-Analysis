## Purpose and Scope

This document covers TT-Metalium, Tenstorrent's low-level programming model for kernel development on Tenstorrent hardware. TT-Metalium provides the foundational layer for developing high-performance compute kernels, data movement operations, and multi-device coordination primitives.

For higher-level neural network operations built on top of TT-Metalium, see [TT-NN Neural Network Library](https://deepwiki.com/tenstorrent/tt-tutorial/3.2-tt-nn-neural-network-library). For practical programming examples using TT-Metalium primitives, see [Programming Examples](https://deepwiki.com/tenstorrent/tt-tutorial/4.3-programming-examples).

## Key Systems and Components

### Memory and Data Management

**Tensor Layouts**: TT-Metalium provides sophisticated tensor layout management for optimizing memory access patterns and computational efficiency.

**Memory Allocator**: Advanced memory allocation system designed for high-performance computing workloads with support for different memory hierarchies.

**Data Formats**: Comprehensive support for various numerical data formats with hardware reconfiguration capabilities for optimal performance.

### Hardware-Specific Optimizations

**Matrix Engine**: Specialized computational unit optimized for matrix operations commonly used in neural network workloads.

**FlashAttention/FlashDecode**: Optimized implementations of attention mechanisms specifically designed for Tenstorrent's Wormhole architecture.

**Special Value Handling**: Robust handling of infinity, NaN, and denormal numbers in Tensix compute operations.

### Multi-Device and Distributed Computing

**Collective Communication Library (CCL)**: High-level primitives for distributed computing operations across multiple devices.

**Ethernet Multichip**: Low-level networking primitives for chip-to-chip communication over Ethernet.

**TT-Fabric Architecture**: Scalable fabric for connecting multiple Tenstorrent devices in large-scale deployments.

**Sub-Devices**: Fine-grained device partitioning for improved resource utilization and workload isolation.
