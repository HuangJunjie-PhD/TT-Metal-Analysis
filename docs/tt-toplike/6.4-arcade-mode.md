# Arcade Mode

Relevant source files
*   [src/animation/arcade.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs)

Arcade Mode is a unified psychedelic visualization that serves as a compositor for three distinct visualization modes: Starfield, Memory Castle, and Memory Flow [src/animation/arcade.rs 5-8](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L5-L8) It presents a high-density "btop++" inspired layout that maps real-time telemetry data to a "Hero" character (`@`) and various particle systems [src/animation/arcade.rs 74-75](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L74-L75)

### Visual Layout and Compositing

The screen is partitioned into three vertical regions, plus a header and footer. The content height is calculated by reserving 3 lines for the header and 3 lines for the footer [src/animation/arcade.rs 78](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L78-L78)

| Region | Height Allocation | Description |
| --- | --- | --- |
| **Starfield** | 40% of content height | Represents Tensix cores as stars and memory hierarchy as planets [src/animation/arcade.rs 85](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L85-L85) |
| **Memory Castle** | 30% of content height | Displays DDR channels, L2 cache, and L1 SRAM with upward flowing particles [src/animation/arcade.rs 12-89](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L12-L89) |
| **Memory Flow** | 30% of content height | Visualizes NoC particles and DDR perimeters with a central heat map [src/animation/arcade.rs 13-90](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L13-L90) |

The boundaries for these regions are calculated during initialization and stored as `starfield_start`, `castle_start`, and `flow_start` to ensure the overlay logic (such as the Hero character) aligns with the rendered lines [src/animation/arcade.rs 99-131](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L99-L131)

### The Hero Character (@)

The `ArcadeVisualization` maintains a "Hero" character that moves dynamically across the entire screen based on aggregate device telemetry [src/animation/arcade.rs 51-55](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L51-L55)

*   **Vertical Position (Y):** Driven by total power consumption. High power moves the hero toward the top; low power moves it toward the bottom [src/animation/arcade.rs 16-17](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L16-L17)
*   **Horizontal Position (X):** Driven by current draw [src/animation/arcade.rs 17](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L17-L17)
*   **Color:** Derived from temperature using `temp_to_hue()`, transitioning from cyan (cool) to yellow and red (hot) [src/animation/arcade.rs 18](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L18-L18)
*   **Smoothing:** The hero uses a 10-frame linear interpolation (`lerp`) to smooth transitions between target positions [src/animation/arcade.rs 242-243](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L242-L243)
*   **Trail:** A `Vec<TrailPosition>` stores recent coordinates to render a fading trail effect [src/animation/arcade.rs 34-38](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L34-L38)[src/animation/arcade.rs 253-257](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L253-L257)

### Data Flow and Update Cycle

The `ArcadeVisualization` acts as a proxy for the underlying specialized visualizations. It propagates sensitivity settings and topology information to ensure visual consistency.

**Sources:**[src/animation/arcade.rs 222-260](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L222-L260)[src/animation/arcade.rs 280-340](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L280-L340)

### Key Functions and Implementation

#### Initialization and Topology

The `initialize_topology` function fetches SMBUS data from the backend to build a `BoardTopology`[src/animation/arcade.rs 158-164](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L158-L164) This topology is then injected into the sub-visualizations so that the header diagram, stream characters, and column separators remain synchronized [src/animation/arcade.rs 167-170](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L167-L170)

#### The Render Cycle

The `render()` function performs a multi-step composition:

1.   **Header:** Renders a one-line topology diagram showing chip status (e.g., `[BH0 ██░ 16W]`) [src/animation/arcade.rs 173-176](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L173-L176)
2.   **Sub-renders:** Calls `render()` on `starfield`, `memory_castle`, and `memory_flow`, collecting their `Vec<Line>` outputs [src/animation/arcade.rs 305-325](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L305-L325)
3.   **Separators:** Inserts horizontal separators with labels (e.g., "TENSIX CORE STARFIELD") between regions [src/animation/arcade.rs 309-322](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L309-L322)
4.   **Hero Overlay:** The `overlay_hero()` function is called last. It calculates the screen row and column for the hero and its trail, then uses `ratatui`'s `Span` and `Style` to overwrite characters in the buffer [src/animation/arcade.rs 343-380](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L343-L380)
