## Overview

tt-flash supports three primary installation methods targeting different user types:

| Installation Method | Target Users | Distribution Channel | Package Format |
| --- | --- | --- | --- |
| PyPI Installation | Standard users | [pypi.org](https://pypi.org/project/tt-flash/) | Python wheel (`.whl`) |
| Building from Source | Developers, contributors | Git repository | Editable Python package |
| Debian Package | Ubuntu/Debian system administrators | Debian repository | `.deb` package |

All installation methods require Python 3.10 or later, as specified in [pyproject.toml 6](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L6-L6) The current stable release is version 3.5.0 [pyproject.toml 3](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L3-L3)

## Prerequisites

### Rust Toolchain

tt-flash depends on `pyluwen` (~0.8.0) [pyproject.toml 26](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L26-L26) which includes Rust components. The Rust toolchain must be installed before installing tt-flash.

#### Installation Methods for Rust

**Distribution Package Managers (Recommended)**

`# Fedora / Red Hat Enterprise Linux 9sudo dnf install cargo # Ubuntu / Debiansudo apt install cargo`
**Rustup (Alternative Method)**

`curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | shsource "$HOME/.cargo/env"`
**Sources**: [README.md 16-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L16-L26)

### Python Version Requirements

Verify Python 3.10 or later is installed:

`python3 --version`
The version constraint is enforced in [pyproject.toml 6](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L6-L6) and [setup.py 19](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L19-L19)

## Installation Method 1: PyPI Installation

### Standard Installation

For standard users, tt-flash is distributed via the Python Package Index (PyPI):

`pip install tt-flash`
This installs:

*   The `tt_flash` Python package
*   The `tt-flash` CLI command [pyproject.toml 50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L50-L50)
*   All runtime dependencies specified in [pyproject.toml 23-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L35)

### Installation with Virtual Environment

For non-system-level installations, a virtual environment is recommended to isolate tt-flash dependencies:

`# Create virtual environmentpython -m venv .venv # Activate virtual environmentsource .venv/bin/activate # Install tt-flashpip install tt-flash`
### Dependency Resolution

The PyPI installation automatically resolves and installs the following dependencies:

| Dependency | Version Constraint | Purpose |
| --- | --- | --- |
| `pyyaml` | `== 6.0.2` | Configuration and metadata parsing |
| `tt_tools_common` | `~= 1.5` | Shared Tenstorrent utilities |
| `pyluwen` | `~= 0.8.0` | Hardware I/O abstraction layer |
| `tabulate` | `== 0.9.0` | Terminal table formatting |
| `tomli` | `== 2.0.1` (Python < 3.11) | TOML parsing for older Python |

**Sources**: [pyproject.toml 23-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L35)[README.md 28-43](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L28-L43)

## Installation Method 2: Building from Source

### Repository Cloning

`git clone https://github.com/tenstorrent/tt-flash.gitcd tt-flash`
### Standard Build Installation

For developers who want to install tt-flash as a standard package:

`pip install .`
This triggers the build system defined in [pyproject.toml 74-81](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L74-L81):

*   Uses `setuptools >= 43.0.0` as the build backend
*   Executes `setuptools.build_meta` to generate wheel
*   Installs package data from [pyproject.toml 55-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L55-L59)

### Editable Installation

For active development with automatic code updates:

`pip install --editable .`
Editable mode allows modifications to the source code without reinstalling. Changes to Python modules in `tt_flash/` are immediately reflected when running the `tt-flash` command.

### Build System Architecture

**Diagram**: Build system flow showing dual configuration files (pyproject.toml as authoritative, setup.py for Ubuntu 22.04 compatibility) and package generation.

**Sources**: [pyproject.toml 74-81](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L74-L81)[setup.py 1-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L1-L26)[README.md 46-62](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L46-L62)

### Ubuntu 22.04 Compatibility Note

The [setup.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py) file exists solely for compatibility with older packaging tools on Ubuntu 22.04:

`# From setup.py:3-5"""Compatibility setup.py for Ubuntu 22.04 packaging tools.Uses setuptools_scm for dynamic versioning from git tags."""`
The file uses `tomli`[setup.py 8](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L8-L8) to read version and metadata from `pyproject.toml`[setup.py 11-12](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L11-L12) ensuring a single source of truth. Modern Python build tools (pip 19+, setuptools 43+) can build directly from `pyproject.toml` without requiring `setup.py`.

**Sources**: [setup.py 1-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L1-L26)

## Installation Method 3: Debian Package Installation

### Package Availability

Debian packages (`.deb`) are built for multiple Ubuntu versions through GitHub Actions workflows. The build process is documented in [Debian Packaging](https://deepwiki.com/tenstorrent/tt-flash/9.2-debian-packaging).

### Supported Ubuntu Versions

| Ubuntu Version | Support Status | Build Workflow |
| --- | --- | --- |
| Ubuntu 22.04 LTS | Supported | `build-debian.yml` |
| Ubuntu 24.04 LTS | Supported | `build-debian.yml` |
| Ubuntu Latest | Supported | `build-debian.yml` |

### Installation from Debian Repository

`# Add Tenstorrent repository (if configured)sudo apt updatesudo apt install tt-flash`
### Installation from Downloaded Package

`# Download .deb package from GitHub Releaseswget https://github.com/tenstorrent/tt-flash/releases/download/v3.5.0/tt-flash_3.5.0_<ubuntu_version>.deb # Install packagesudo dpkg -i tt-flash_3.5.0_<ubuntu_version>.deb # Install dependencies if neededsudo apt-get install -f`
### Debian Package Structure

Debian packages include:

*   Python package installed to system site-packages
*   `tt-flash` command in `/usr/bin/`
*   Changelog in `debian/changelog`
*   GPG signature for package verification

## Post-Installation Verification

### Verify Installation

Confirm tt-flash is installed and accessible:

`tt-flash --version`
Expected output:

```
tt-flash version 3.5.0
```

### Display Help Text

View available commands and options:

`tt-flash -h`
This displays the CLI interface defined in [main.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/main.py)'s argument parser, showing:

*   Global options: `--sys-config`, `--no-color`, `--no-tty`
*   Subcommands: `flash`, `verify`

### Verify Entry Point Resolution

The `tt-flash` command is registered as a console script entry point in [pyproject.toml 49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L49-L50):

`[project.scripts]tt-flash = "tt_flash.main:main"`
This maps the `tt-flash` CLI command to the `main()` function in [tt_flash/main.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py)

**Sources**: [pyproject.toml 49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L49-L50)[README.md 64-86](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L64-L86)

## Troubleshooting Common Installation Issues

### Issue: `pyluwen` Installation Fails

**Cause**: Rust toolchain not installed or not in PATH

**Solution**:

`# Verify Rust is installedcargo --version # If not found, install Rust (see Prerequisites section)sudo apt install cargo  # Ubuntu/Debian# orsudo dnf install cargo  # Fedora/RHEL`
### Issue: Python Version Incompatibility

**Cause**: Python version < 3.10

**Solution**:

`# Check Python versionpython3 --version # Install Python 3.10+ using distribution packages or pyenv`
The version constraint is enforced in [pyproject.toml 6](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L6-L6)

### Issue: Permission Denied During Installation

**Cause**: Attempting system-level installation without privileges

**Solution**:

`# Use --user flag for user-level installationpip install --user tt-flash # Or use virtual environment (recommended)python -m venv .venvsource .venv/bin/activatepip install tt-flash`
**Sources**: [pyproject.toml 6](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L6-L6)[README.md 16-43](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L16-L43)
