## Purpose and Scope

This document describes the performance profiling and benchmarking infrastructure in tt-torch. It covers build configuration for performance tracing, the Tracy-based profiling tools, collection and analysis of performance data, and automated benchmark workflows. For information about general testing infrastructure, see [Testing Framework](https://deepwiki.com/tenstorrent/tt-torch/4.2-testing-framework). For hardware-specific testing, see [Hardware Matrix Testing](https://deepwiki.com/tenstorrent/tt-torch/4.2.3-hardware-matrix-testing).

## Build Configuration for Performance Profiling

Performance profiling requires building tt-torch with specific flags that enable performance tracing at the runtime level.

### Performance Build Flags

The `--build_perf` flag enables performance tracing during the build process:

`python setup.py bdist_wheel --build_perf`
This flag triggers two key configurations:

1.   **CMake Configuration**: Sets `-DTT_RUNTIME_ENABLE_PERF_TRACE=ON`, which enables Tracy profiling hooks in the tt-mlir runtime
2.   **Additional Dependencies**: Installs profiling analysis dependencies (`pyyaml`, `click`, `loguru`, `pandas`, `seaborn`)

**Build Configuration Flow**

## Using the tt_profile Tool

The `tt_profile` command-line tool provides a convenient interface for profiling model execution.

### Command Registration

The tool is registered as a console script entry point:

`entry_points={    "console_scripts": [        "tt_profile = tt_torch.tools.tt_profile:main",    ],}`
Sources: [setup.py 225-232](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/setup.py#L225-L232)

### Performance Data Collection

**Tracy Event Types**

The profiler captures multiple categories of performance data:

| Event Type | Source | Data Captured |
| --- | --- | --- |
| `TT_DNN` | Device operations | Op execution time, global call count |
| `MLIR_OP_LOCATION` | Compile-time metadata | Source location in MLIR IR |
| `MLIR_CONST_EVAL_OP` | Compile-time metadata | Whether op was constant-evaluated |
| `MLIR_PROGRAM_METADATA` | Compile-time metadata | Program-level attributes |
| `CompileProgram` | Runtime overhead | Compilation time (excluded) |
| `HWCommandQueue_write_buffer` | Runtime overhead | Buffer write time (excluded) |

The profiler explicitly excludes certain child calls that represent overhead rather than device execution:

`child_calls = ["CompileProgram", "HWCommandQueue_write_buffer"]child_calls_str = f"-x {','.join(child_calls)}"`
Sources: [tt_torch/tools/profile_util.py 200-209](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tt_torch/tools/profile_util.py#L200-L209)

### MLIR Location Data Integration

The post-processing phase enriches performance data with MLIR-level information by correlating global call counts:

**Correlation Logic**

1.   Parse `tracy_ops_data.csv` for `MLIR_OP_LOCATION`, `MLIR_CONST_EVAL_OP`, and `MLIR_PROGRAM_METADATA` events
2.   Build mapping from global call count to MLIR metadata
3.   Iterate through `ops_perf_results.csv` and add corresponding MLIR data

**Example CSV Output Structure**

| GLOBAL CALL COUNT | OP NAME | DEVICE TIME | LOC | CONST_EVAL_OP | PROGRAM_METADATA |
| --- | --- | --- | --- | --- | --- |
| 1234567890 | ttnn.conv2d | 1.234 ms | loc("model.mlir":17:14) | false | {"grid": [8,8]} |
| 1234567891 | ttnn.matmul | 0.567 ms | loc("model.mlir":23:8) | false | {"shard": "width"} |

The `perf-benchmark.yml` workflow provides automated performance benchmarking on dedicated hardware.

**Workflow Structure**

Sources: [.github/workflows/perf-benchmark.yml 1-77](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/.github/workflows/perf-benchmark.yml#L1-L77)

### Integration with tt-forge

The workflow delegates actual benchmark execution to tt-forge's benchmark infrastructure:

`perf-benchmark:  needs: set-inputs  uses: tenstorrent/tt-forge/.github/workflows/perf-benchmark.yml@main  with:    project: tt-torch    run_id: ${{ github.run_id }}    run_id_source: tt-torch    ref: ${{ needs.set-inputs.outputs.ref }}`
This allows centralized benchmark infrastructure while maintaining per-project customization.

Sources: [.github/workflows/perf-benchmark.yml 66-77](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/.github/workflows/perf-benchmark.yml#L66-L77)

## Environment Variables

### Runtime Performance Tracing

| Variable | Purpose | Values |
| --- | --- | --- |
| `TT_RUNTIME_ENABLE_PERF_TRACE` | Enable Tracy profiling at runtime | `0` (disabled), `1` (enabled) |
| `TT_METAL_LOGGER_LEVEL` | Control tt-metal logging verbosity | `ERROR`, `WARN`, `INFO`, `DEBUG` |
| `TT_METAL_HOME` | Location of tt-metal installation | Path to tt-metal directory |

These environment variables must be set when executing profiled workloads:

`export TT_RUNTIME_ENABLE_PERF_TRACE=1python your_model_script.py`
Sources: [env/activate 44-51](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/env/activate#L44-L51)[tt_torch/csrc/CMakeLists.txt 48-50](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tt_torch/csrc/CMakeLists.txt#L48-L50)

### File Locations

**Profiler Output Paths**

| File | Default Location | Purpose |

| Tracy trace | `tracy_profile_log_host.tracy` | Binary trace data |

## Performance Build Requirements

### Required Dependencies

Performance builds require additional Python packages for data analysis:

```
pyyaml      # YAML parsing for configuration
click       # CLI framework for tools
loguru      # Logging
pandas      # Data frame operations
seaborn     # Statistical visualization
```

These are automatically installed when building with `--build_perf`.

Sources: [setup.py 180-181](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/setup.py#L180-L181)[requirements.txt 17-29](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/requirements.txt#L17-L29)

### Tracy Tool Binaries

The profiler requires two Tracy executables:

1.   **capture-release**: Tracy server that captures profiling events in real-time
2.   **csvexport-release**: Utility to export Tracy binary format to CSV

These are compiled as part of the tt-metal build when `TT_RUNTIME_ENABLE_PERF_TRACE` is enabled, and are packaged in wheels under the `tt-metal/tools/profiler/bin/` directory.

Sources: [tt_torch/tools/profile_util.py 27-32](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tt_torch/tools/profile_util.py#L27-L32)[setup.py 83-106](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/setup.py#L83-L106)
