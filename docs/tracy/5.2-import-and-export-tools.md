This document covers Tracy's suite of import and export tools that enable data interchange between Tracy and external systems. These tools provide command-line and GUI-based utilities for capturing live profiling data, importing traces from other profiling formats, and exporting Tracy data for external analysis.

For information about the underlying Tracy file formats and compression schemes, see [File Formats](https://deepwiki.com/tenstorrent/tracy/5.1-file-formats). For details about other command-line utilities beyond the capture tool, see [Command-Line Utilities](https://deepwiki.com/tenstorrent/tracy/6.2-command-line-utilities).

## Overview

Tracy provides several categories of import and export tools:

*   **Live Capture Tools**: Command-line utilities for connecting to instrumented applications and saving trace data
*   **File Import/Export**: GUI and programmatic interfaces for loading and saving Tracy trace files
*   **Format Conversion**: Tools for importing traces from Chrome Tracing, Fuchsia, and other external formats
*   **Data Export**: Utilities for exporting Tracy data to CSV and other analysis-friendly formats
*   **Trace Comparison**: Tools for loading multiple traces for comparative analysis

## Live Data Capture

### Capture Utility

The primary command-line tool for capturing live Tracy data is the `capture` utility, implemented in [capture/src/capture.cpp](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp)

**Command-Line Interface and Connection Management**

The capture utility supports several command-line options for flexible data capture:

*   `-a address`: Target application address (default: 127.0.0.1)
*   `-p port`: Target application port (default: 8086)
*   `-o output`: Output Tracy file path
*   `-f`: Force overwrite existing files
*   `-s seconds`: Capture duration limit

**Real-time Data Processing**

The capture process performs several key operations during live data collection:

1.   **Connection Establishment**: Uses `tracy::Worker` to establish TCP connection with instrumented application [capture/src/capture.cpp 161](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp#L161-L161)
2.   **Handshake Validation**: Verifies protocol compatibility and handles connection errors [capture/src/capture.cpp 164-179](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp#L164-L179)
3.   **Live Monitoring**: Displays real-time statistics including network throughput, compression ratio, and memory usage [capture/src/capture.cpp 211-241](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp#L211-L241)
4.   **Signal Handling**: Implements graceful shutdown on SIGINT using atomic operations [capture/src/capture.cpp 36-44](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp#L36-L44)

## GUI File Operations

### File Selection Framework

Tracy's GUI applications use a cross-platform file selection system implemented in `TracyFileselector` for opening and saving trace files.

**Platform-Specific Implementation**

The file selector provides unified interfaces across different platforms:

*   **Native Platforms**: Uses NFD (Native File Dialog) library for Windows, Linux, and macOS [server/TracyFileselector.cpp 77-91](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyFileselector.cpp#L77-L91)
*   **Web Platform**: Implements HTML5 File API integration for Emscripten builds [server/TracyFileselector.cpp 56-76](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyFileselector.cpp#L56-L76)
*   **Error Handling**: Tracks operation failures through `HasFailed()` mechanism [server/TracyFileselector.cpp 30-41](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyFileselector.cpp#L30-L41)

**File Operation Workflows**

The GUI integrates file operations in several contexts:

1.   **Main Trace Loading**: Primary interface for opening Tracy dump files
2.   **Trace Comparison**: Secondary trace loading for comparative analysis [server/TracyView_Compare.cpp 206-236](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Compare.cpp#L206-L236)
3.   **Save Operations**: Exporting current live or loaded traces [server/TracyView_ConnectionState.cpp 122-143](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_ConnectionState.cpp#L122-L143)

## Trace Comparison and External Format Import

### Multi-Trace Loading System

Tracy supports loading multiple traces simultaneously for comparative analysis, with special handling for external trace formats.

**Version Compatibility Handling**

The system implements comprehensive version checking to handle traces from different Tracy versions:

*   **Unsupported Versions**: Displays upgrade recommendations with download links [server/TracyBadVersion.cpp 69-89](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyBadVersion.cpp#L69-L89)
*   **Legacy Versions**: Provides guidance for using update utilities [server/TracyBadVersion.cpp 90-103](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyBadVersion.cpp#L90-L103)
*   **Corrupted Files**: Handles file corruption and read errors gracefully [server/TracyBadVersion.cpp 104-120](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyBadVersion.cpp#L104-L120)

**Comparative Analysis Features**

The comparison system supports three analysis modes:

1.   **Zone Comparison**: Matches zones by name, source file, and line number [server/TracyView_Compare.cpp 56-135](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Compare.cpp#L56-L135)
2.   **Frame Comparison**: Aligns frame sets between traces for performance analysis [server/TracyView_Compare.cpp 576-670](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Compare.cpp#L576-L670)
3.   **Source Diff**: Uses DTL library for unified diff generation of source code changes [server/TracyView_Compare.cpp 334-371](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Compare.cpp#L334-L371)

The clipboard export system formats callstack data with detailed frame information:

*   **Frame Numbering**: Sequential frame indices for stack depth visualization [server/TracyView_Callstack.cpp 75-77](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Callstack.cpp#L75-L77)

*   **Source Locations**: File paths and line numbers in parenthetical format [server/TracyView_Callstack.cpp 84-92](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Callstack.cpp#L84-L92)

**Data Formatting Utilities**
