This page documents the environment variables available for debugging, profiling, and configuring the TT-Metalium runtime system. These variables control various aspects of the debugging infrastructure including the Watcher, DPRINT, profiler, inspector, and kernel execution behavior.

For information about using specific debugging tools, see:

*   Watcher system usage: [8.2 Watcher Debug System](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/8.2%20Watcher%20Debug%20System)

*   Performance profiling: [8.4 Profiling and Performance Analysis](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/8.4%20Profiling%20and%20Performance%20Analysis)

## Environment Variable Processing Architecture

Environment variables are processed during the initialization of `RunTimeOptions` and affect system behavior at compile-time (JIT builds), runtime (kernel execution), and debug-time (tooling configuration). ## Path Configuration Variables

These variables control where the runtime looks for source files, writes output, and caches compiled artifacts.

| Environment Variable | Type | Purpose | Default |
| --- | --- | --- | --- |
| `TT_METAL_CACHE` | Path | Cache directory for compiled kernels | `~/.cache/tt-metal-cache/` or `/tmp/tt-metal-cache/` |
| `TT_METAL_KERNEL_PATH` | Path | Path to kernel source files | Root directory |
| `TT_METAL_LOGS_PATH` | Path | Directory for generated logs and debug output | Current directory |
| `TT_METAL_SIMULATOR` | Path | Path to simulator executable | None |
| `TT_METAL_MOCK_CLUSTER_DESC_PATH` | Path | Mock cluster descriptor path for testing | None |
| `TT_METAL_VISIBLE_DEVICES` | List | Comma-separated list of visible device IDs | All detected |

**Path Resolution Logic:**

 The default root path is determined in `get_default_root_path()`[tt_metal/jit_build/build.cpp 90-97](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/jit_build/build.cpp#L90-L97) which checks the `HOME` environment variable before falling back to `/tmp/tt-metal-cache/`.

## Watcher System Variables

The Watcher system provides runtime monitoring of device cores, detecting hangs, crashes, and NoC transaction errors. Settings are encapsulated in the `WatcherSettings` struct [tt_metal/llrt/rtoptions.hpp 97-108](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/llrt/rtoptions.hpp#L97-L108)

| Environment Variable | Type | Purpose |
| --- | --- | --- |
| `TT_METAL_WATCHER` | Flag | Enable the watcher system |
| `TT_METAL_WATCHER_DUMP_ALL` | Flag | Dump all watcher data on exit |
| `TT_METAL_WATCHER_APPEND` | Flag | Append to existing watcher logs instead of overwriting |
| `TT_METAL_WATCHER_NOINLINE` | Flag | Disable watcher function inlining for better callstacks |
| `TT_METAL_WATCHER_INTERVAL_MS` | Int | Polling interval for the watcher server thread |

The Watcher activation flags are stored in `rtoptions` and used during kernel compilation to control code generation with `-DWATCHER_ENABLED` define [tt_metal/jit_build/build.cpp 204-206](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/jit_build/build.cpp#L204-L206) The runtime spawns the `WatcherServer` thread if enabled during device attachment [tt_metal/impl/context/metal_context.cpp 40](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/impl/context/metal_context.cpp#L40-L40)

## DPRINT (Debug Print) Variables

DPRINT provides `printf`-style debug output from device kernels. The DPRINT targets and output are represented by the `TargetSelection` struct [tt_metal/llrt/rtoptions.hpp 81-95](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/llrt/rtoptions.hpp#L81-L95)

| Environment Variable | Type | Purpose |
| --- | --- | --- |
| `TT_METAL_DPRINT_CORES` | List | Target cores for debug printing (e.g., `(0,0),(1,1)` or `all`) |
| `TT_METAL_DPRINT_CHIPS` | List | Target chip IDs |
| `TT_METAL_DPRINT_RISCVS` | Mask | RISC-V processors to enable (BR, NC, TR0, TR1, TR2, ER) |
| `TT_METAL_DPRINT_FILE` | Path | Output file for debug prints |
| `TT_METAL_DPRINT_ONE_FILE_PER_RISC` | Flag | Write each RISC's output to a separate file |

When DPRINT feature is enabled, the kernel JIT build receives specific compile-time definitions enabling device print buffers integration [tt_metal/jit_build/build.cpp 211-213](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/jit_build/build.cpp#L211-L213)

## Profiler and Performance Variables

The device profiler collects performance metrics for kernel execution and NoC transactions.

| Environment Variable | Type | Purpose |
| --- | --- | --- |
| `TT_METAL_DEVICE_PROFILER` | Flag | Enable device-side profiling |
| `TT_METAL_DEVICE_PROFILER_DISPATCH` | Flag | Enable profiling for dispatch cores |
| `TT_METAL_PROFILER_SYNC` | Flag | Enable synchronous profiling (flushes after operations) |
| `TT_METAL_DEVICE_PROFILER_NOC_EVENTS` | Flag | Enable NoC events profiling |
| `TT_METAL_PROFILE_PERF_COUNTERS` | Int | Performance Counter mode (1: basic, 2: detailed) |
| `TT_METAL_PROFILER_SUM` | Flag | Aggregate profiling data (sum mode) |

Profiling settings are converted into compiler defines `-DPROFILE_KERNEL`, buffer sizes, and flags enabling NoC event capture during kernel compilation [tt_metal/jit_build/build.cpp 172-202](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/jit_build/build.cpp#L172-L202)

## JIT Build System Variables

These settings control the kernel build and cache behaviors.

| Environment Variable | Type | Purpose |
| --- | --- | --- |
| `TT_METAL_FORCE_JIT_COMPILE` | Flag | Ignore build cache and force JIT kernel recompilation |
| `TT_METAL_CCACHE_KERNEL_SUPPORT` | Flag | Enable use of `ccache` for kernel compilation |
| `TT_METAL_LOG_KERNELS_COMPILE_COMMANDS` | Flag | Log exact g++ commands used for kernel builds |
| `TT_METAL_RISCV_DEBUG_INFO` | Flag | Add `-g` debug info option to kernel compilation |
