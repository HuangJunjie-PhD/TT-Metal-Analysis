This page documents the device interface layer that bridges the Python testing framework with Tenstorrent hardware, enabling test execution on physical Tensix cores. This covers the mechanisms for loading compiled ELF binaries onto hardware, writing input data to L1 memory, monitoring execution via mailbox polling, and collecting results.

For information about the compilation process that generates ELF binaries, see [9.1](https://deepwiki.com/tenstorrent/tt-llk/9.1-compilation-architecture-and-artifacts). For the TestConfig orchestration layer, see [8.2](https://deepwiki.com/tenstorrent/tt-llk/8.2-testconfig-and-test-orchestration). For data packing/unpacking utilities, see [8.6](https://deepwiki.com/tenstorrent/tt-llk/8.6-stimuli-generation-and-data-packing).

## Device Initialization and Reset

Before each test execution, the device must be properly initialized and reset. This involves setting up hardware accelerator units, initializing semaphores, and resetting mailboxes.

### Core Reset and Setup

`# Core types mapped to RISC core IDsclass RiscCore(IntEnum):    BRISC = 11   # Control processor (not present on Quasar)    TRISC0 = 12  # Unpack processor    TRISC1 = 13  # Math processor      TRISC2 = 14  # Pack processor    TRISC3 = 14  # Additional processor on Quasar only`

## ARC Messages and Device State Management

The device interface sends ARC (ARC Resource Controller) messages to manage device power states.

### ARC Message Protocol

**Sources:**[tests/python_tests/conftest.py 275-282](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/conftest.py#L275-L282)[tests/python_tests/conftest.py 316-323](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/conftest.py#L316-L323)[tests/python_tests/helpers/device.py 318-330](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/helpers/device.py#L318-L330)

### Message Implementation

`def _send_arc_message(message_type: str, device_id: int):    ARC_COMMON_PREFIX = 0xAA00    message_codes = {        "GO_BUSY": 0x52,  # 0xAA52        "GO_IDLE": 0x54,  # 0xAA54    }        arc_msg(        device_id=device_id,        msg_code=ARC_COMMON_PREFIX | message_codes[message_type],        wait_for_done=True,        args=[0, 0],        timeout=datetime.timedelta(seconds=10)    )`
**Sources:**[tests/python_tests/helpers/device.py 318-330](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/helpers/device.py#L318-L330)

This ensures the device remains active throughout the test session, preventing power-saving modes that could interfere with test execution.
