# Disassembler and Objcopy Tools

Relevant source files
*   [riescue/lib/toolchain/tool.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py)

## Purpose and Scope

This page documents the `Disassembler` and `Objcopy` tool abstractions in RiESCUE, which provide Python interfaces to the RISC-V GNU toolchain's `riscv64-unknown-elf-objdump` and `riscv64-unknown-elf-objcopy` utilities. These classes handle executable discovery, subprocess execution, and error classification for disassembly and binary manipulation operations.

For information about the compiler tool, see [Compiler Tool](https://deepwiki.com/tenstorrent/riescue/4.1.1-compiler-tool). For details on the base `Tool` abstraction and general toolchain concepts, see [Tool Base Class](https://deepwiki.com/tenstorrent/riescue/7.3.1-tool-base-class).

**Sources:**[riescue/lib/toolchain/tool.py 1-335](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L1-L335)

## Overview

The `Disassembler` and `Objcopy` classes are concrete implementations of the abstract `Tool` base class, providing specialized interfaces to RISC-V binary utilities:

| Tool Class | Executable | Environment Variable | Primary Purpose |
| --- | --- | --- | --- |
| `Disassembler` | `riscv64-unknown-elf-objdump` | `RV_OBJDUMP` | Generate human-readable disassembly from ELF binaries |
| `Objcopy` | `riscv64-unknown-elf-objcopy` | `RV_OBJCOPY` | Copy and transform binary object files |

Both classes follow the same architecture pattern: they inherit from the `Tool` base class, implement the abstract `_classify` method for error handling, and provide static methods for command-line argument integration.

**Sources:**[riescue/lib/toolchain/tool.py 206-271](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L206-L271)

## Disassembler Class

The `Disassembler` class wraps the `riscv64-unknown-elf-objdump` utility for generating disassembly listings from ELF executables.

### Initialization

`def __init__(    self,    disassembler_path: Optional[Path] = None,    disassembler_opts: Optional[list[str]] = None,):`
The constructor accepts an optional path to the disassembler executable and optional additional arguments. If no path is provided, the executable is discovered using the standard search mechanism (see [Executable Discovery](https://github.com/tenstorrent/riescue/blob/07cefde3/Executable%20Discovery)).

**Note:** There is currently a known limitation where `disassembler_opts` are not passed to the underlying tool. A warning is logged if these options are provided, as legacy flows may pass options like `--disassembler_opts=zvbb` that are currently ignored.

**Sources:**[riescue/lib/toolchain/tool.py 206-214](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L206-L214)[riescue/lib/toolchain/tool.py 226-229](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L226-L229)

### Command-Line Integration

The `Disassembler` class provides static methods for argparse integration:

`@staticmethoddef add_arguments(parser: argparse.ArgumentParser):    disassembler_parser = parser.add_argument_group("Disassembler", description="Arguments for disassembler")    disassembler_parser.add_argument("--disassembler_path", type=Path, default=None,                                      help="Path to disassembler executable. If not provided, will use RV_OBJDUMP environment variable")    disassembler_parser.add_argument("--disassembler_opts", nargs="*", default=[],                                      help="Additional args to pass to the disassembler")`
Construction from parsed command-line arguments:

`@classmethoddef from_clargs(cls, args: argparse.Namespace):    if args.disassembler_opts:        log.warning(f"--disassembler_opts={', --disassembler_opts='.join(args.disassembler_opts)} are ignored for disassembler")    return cls(disassembler_path=args.disassembler_path, disassembler_opts=args.disassembler_opts)`
**Sources:**[riescue/lib/toolchain/tool.py 216-230](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L216-L230)

### Error Classification

The `_classify` method examines the subprocess return code and raises appropriate `ToolchainError` exceptions:

`def _classify(self, process: subprocess.CompletedProcess, output_file: Optional[Path]):    if process.returncode != 0:        if process.stderr is None:            if output_file is None:                stdout = "No output file was produced"            else:                with open(output_file, "r") as f:                    stdout = f.read().strip()            self._raise_toolchain_error(process, ToolFailureType.COMPILE_FAILURE, stdout)        else:            self._raise_toolchain_error(process, ToolFailureType.COMPILE_FAILURE, process.stderr)`
Non-zero return codes are classified as `COMPILE_FAILURE` type errors, with error text extracted from either stderr or the output file.

**Sources:**[riescue/lib/toolchain/tool.py 232-242](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L232-L242)

## Objcopy Class

The `Objcopy` class wraps the `riscv64-unknown-elf-objcopy` utility for copying and transforming binary object files.

### Initialization

`def __init__(    self,    objcopy_path: Optional[Path] = None,    objcopy_opts: Optional[list[str]] = None,):    if objcopy_opts is None:        objcopy_opts = []    super().__init__(path=objcopy_path, env_name="RV_OBJCOPY",                      tool_name="riscv64-unknown-elf-objcopy", args=objcopy_opts)`
Unlike `Disassembler`, the `Objcopy` class properly passes `objcopy_opts` to the base `Tool` constructor, making them available for all invocations.

**Sources:**[riescue/lib/toolchain/tool.py 245-253](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L245-L253)

## Executable Discovery Mechanism

Both `Disassembler` and `Objcopy` rely on the `Tool` base class's `find_executable` method for locating the appropriate binary. The discovery follows a three-tier priority system:

**Discovery Priority:**

| Priority | Source | Disassembler Example | Objcopy Example |
| --- | --- | --- | --- |
| 1 | Explicit parameter | `disassembler_path` | `objcopy_path` |
| 2 | Environment variable | `$RV_OBJDUMP` | `$RV_OBJCOPY` |
| 3 | System PATH | `riscv64-unknown-elf-objdump` | `riscv64-unknown-elf-objcopy` |

If the executable cannot be found through any of these methods, a `FileNotFoundError` is raised with a descriptive message.

**Sources:**[riescue/lib/toolchain/tool.py 36-55](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L36-L55)

## Execution Workflow

The execution workflow is identical for both tools, inherited from the `Tool` base class:

**Typical Usage Pattern:**

`# Initialize disassemblerdisasm = Disassembler(disassembler_path=Path("/path/to/objdump")) # Run disassembly, piping output to fileprocess = disasm.run(    output_file=Path("output.dis"),    args=["-D", "binary.elf"]  # Additional objdump flags) # Initialize objcopyobjcopy = Objcopy(objcopy_opts=["-O", "binary"]) # Run objcopy transformationprocess = objcopy.run(    args=["input.elf", "output.bin"])`
**Sources:**[riescue/lib/toolchain/tool.py 74-108](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L74-L108)

## Integration with RiESCUE Workflows

Both tools are typically used in the following contexts within RiESCUE:

### Disassembler Usage

The `Disassembler` is used after compilation to generate `.dis` disassembly files for human inspection and debugging:

Common `objdump` flags used:

*   `-D`: Disassemble all sections
*   `-S`: Intermix source code with disassembly
*   `-l`: Include line numbers
*   `--disassemble-zeroes`: Show zero bytes

**Sources:**[riescue/lib/toolchain/tool.py 206-243](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L206-L243)

### Objcopy Usage

The `Objcopy` tool is used for binary format conversions and section extraction:

Common `objcopy` operations:

*   `-O binary`: Convert to raw binary format
*   `-O ihex`: Convert to Intel HEX format
*   `-R .section`: Remove specific sections
*   `--only-section=.text`: Extract only specific sections

**Sources:**[riescue/lib/toolchain/tool.py 245-271](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L245-L271)

## Error Handling

Both classes implement error classification to provide meaningful error messages to users:

| Error Scenario | Classification | Error Text Source |
| --- | --- | --- |
| Non-zero return code | `ToolFailureType.COMPILE_FAILURE` | stderr or output_file |
| Segmentation fault (RC 139) | `ToolFailureType.SEGFAULT` | stderr |
| Missing output file | `ToolFailureType.COMPILE_FAILURE` | "No output file was produced" |

All errors are raised as `ToolchainError` exceptions with the following attributes:

*   `tool_name`: Class name (`"Disassembler"` or `"Objcopy"`)
*   `cmd`: Full command that was executed
*   `kind`: `ToolFailureType` enum value
*   `returncode`: Process exit code
*   `error_text`: Detailed error message

**Sources:**[riescue/lib/toolchain/tool.py 100-114](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L100-L114)[riescue/lib/toolchain/tool.py 232-242](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L232-L242)[riescue/lib/toolchain/tool.py 267-270](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L267-L270)

## Configuration Best Practices

### Environment Variable Setup

For consistent tool discovery across the RiESCUE environment, set the following environment variables:

`export RV_OBJDUMP=/opt/riscv/bin/riscv64-unknown-elf-objdumpexport RV_OBJCOPY=/opt/riscv/bin/riscv64-unknown-elf-objcopy`
These are typically set in the container environment or user's shell configuration.

### Command-Line Override

When explicit control is needed, tools can be specified via command-line arguments:

`riescued my_test.rasm \    --disassembler_path /custom/path/to/objdump \    --objcopy_path /custom/path/to/objcopy`
### Programmatic Usage

For programmatic usage in Python scripts:

`from pathlib import Pathfrom riescue.lib.toolchain.tool import Disassembler, Objcopy # Explicit pathdisasm = Disassembler(disassembler_path=Path("/usr/local/bin/riscv64-unknown-elf-objdump")) # Use environment variable or PATHobjcopy = Objcopy() # Run with custom argumentsdisasm.run(output_file=Path("output.dis"), args=["-D", "-S", "binary.elf"])`
