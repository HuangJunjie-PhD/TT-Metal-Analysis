## Chip-Specific Burnin Functions

The application implements separate start and stop functions for each supported chip architecture, following a consistent pattern across all chip types.

### Function Mapping

### Common Burnin Operations

Each chip-specific function follows the same operational sequence:

1.   **Reset Phase**: Apply soft reset to BRISC, TRISC, and NCRISC cores
2.   **ARC Messaging**: Send reset deassertion and busy state commands
3.   **Workload Loading**: Load appropriate TTX file for the chip architecture
4.   **Core Activation**: Take cores out of reset with optional staggered start

The soft reset constants are consistent across all architectures:

*   `BRISC_SOFT_RESET = 1 << 11`
*   `TRISC_SOFT_RESETS = (1 << 12) | (1 << 13) | (1 << 14)`
*   `NCRISC_SOFT_RESET = 1 << 18`

## Device Reset Management

The `reset_all_devices()` function at [tt_burnin/main.py 38-75](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/main.py#L38-L75) handles device reset operations before and after burnin execution.

### Reset Decision Logic

The reset system supports three different reset strategies:

1.   **Galaxy Reset**: Full 6U Galaxy system reset for `tt-galaxy-wh` boards
2.   **Targeted Reset**: JSON-configured reset for specific boards
3.   **Universal Reset**: Reset all locally attached devices

## User Interface and Monitoring

The application provides a real-time monitoring interface using the Rich library for terminal-based display.

### Live Monitoring System

The monitoring loop at [tt_burnin/main.py 347-355](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/main.py#L347-L355) implements:

*   **Live Table Updates**: Real-time telemetry display via `generate_table(devices)`
*   **User Input Detection**: Non-blocking stdin monitoring for graceful shutdown
*   **Update Frequency**: 10Hz refresh rate for smooth display updates
*   **Exit Condition**: Any keystroke triggers burnin termination

The interface combines device telemetry with user interaction prompts in a single live-updating display.
