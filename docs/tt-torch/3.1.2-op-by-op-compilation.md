Op-by-op compilation enables operation-by-operation compilation and execution for debugging and analysis purposes. This mode allows developers to isolate compilation failures, measure per-operation performance, and verify intermediate results during the compilation pipeline.

For information about the overall compilation pipeline flow, see [PyTorch to TTNN Flow](https://deepwiki.com/tenstorrent/tt-torch/3.1.1-pytorch-to-ttnn-flow). For compiler configuration options, see [Configuration and Control](https://deepwiki.com/tenstorrent/tt-torch/3.1.3-configuration-and-control).

## Overview

Op-by-op compilation breaks down model execution into individual operations, compiling and optionally executing each operation in isolation. This approach provides fine-grained visibility into the compilation process and enables targeted debugging of specific operations that may fail during full model compilation.

The system supports two primary compilation depths for op-by-op processing:

*   **COMPILE_OP_BY_OP**: Compiles each operation individually without execution
*   **EXECUTE_OP_BY_OP**: Compiles and executes each operation, comparing results against golden outputs

## Operation Tracking and Profiling

The system tracks individual operations using the `Op` class and `OpCompilationStatus` enum:

| Status | Description |
| --- | --- |
| `NOT_STARTED` | Operation not yet processed |
| `CREATED_GRAPH` | FX graph created for operation |
| `CONVERTED_TO_TORCH_IR` | Converted to Torch MLIR IR |
| `CONVERTED_TO_TORCH_BACKEND_IR` | Lowered to Torch Backend IR |
| `CONVERTED_TO_STABLE_HLO` | Converted to StableHLO |
| `CONVERTED_TO_TTIR` | Converted to TTIR dialect |
| `CONVERTED_TO_TTNN` | Converted to TTNN dialect |
| `EXECUTED` | Successfully executed |

## Configuration and Environment Variables

Op-by-op compilation is controlled through `CompilerConfig` properties and environment variables:

| Environment Variable | Config Property | Description |
| --- | --- | --- |
| `TT_TORCH_COMPILE_DEPTH` | `compile_depth` | Set to `COMPILE_OP_BY_OP` or `EXECUTE_OP_BY_OP` |
| `TT_TORCH_VERIFY_OP_BY_OP` | `verify_op_by_op` | Enable output verification |
| `TT_TORCH_VERIFY_INTERMEDIATES` | `enable_intermediate_verification` | Enable intermediate result capture |
| `TT_TORCH_CHECK_ALL_OPS_EXECUTE` | `check_all_ops_execute` | Assert all operations execute successfully |
| `RUN_GLOBAL_OP_IDX` | - | Run only specific operation by global index |

The system generates a `unique_ops.json` file containing detailed information about each processed operation, including compilation status, performance metrics, and intermediate representations.
