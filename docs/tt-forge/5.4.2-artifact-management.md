## Purpose and Scope

Artifact Management encompasses the systems and actions that handle build artifacts throughout the TT-Forge CI/CD pipeline. This includes downloading artifacts from workflow runs, uplifting artifacts from dependent repositories, and installing project-specific wheels. These systems enable seamless integration of components built across different repositories within the TT-Forge ecosystem.

## Overview

TT-Forge's artifact management system consists of specialized GitHub Actions that handle different aspects of artifact operations. The core functionality revolves around the `download-artifact` action, which provides a resilient wrapper around the GitHub CLI for retrieving files.

## Download Artifact Action

The `download-artifact` action is a composite action that provides a robust mechanism for downloading artifacts with automatic retry logic and extraction support. It is defined in `.github/actions/download-artifact/action.yaml`.

### Download Process with Retry Logic

The action implements a loop to handle transient network failures or GitHub API rate limiting. It uses the GitHub CLI (`gh run download`) as the underlying engine.

Sources: [.github/actions/download-artifact/action.yaml 38-119](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/.github/actions/download-artifact/action.yaml#L38-L119)

### Configuration Parameters

| Parameter | Required | Default | Description |
| --- | --- | --- | --- |
| `name` | Yes | - | Name of the artifact to download |
| `path` | No | `$GITHUB_WORKSPACE` | Target directory for extraction |
| `repository` | No | `${{ github.repository }}` | Source repository (owner/repo) |
| `run_id` | No | `${{ github.run_id }}` | Workflow run ID containing the artifact |
| `retry_count` | No | `3` | Maximum download attempts |
| `retry_wait` | No | `10` | Seconds to wait between attempts |

Sources: [.github/actions/download-artifact/action.yaml 7-33](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/.github/actions/download-artifact/action.yaml#L7-L33)

### Implementation Details

1.   **Path Validation**: The action strictly enforces that the download path resides within `${GITHUB_WORKSPACE}` to prevent path traversal vulnerabilities [[.github/actions/download-artifact/action.yaml:58-61]](https://deepwiki.com/tenstorrent/tt-forge/5.4.2-artifact-management).
2.   **Directory Preparation**: Unless downloading to the root workspace, the action ensures the target directory is empty before starting [[.github/actions/download-artifact/action.yaml:64-68]](https://deepwiki.com/tenstorrent/tt-forge/5.4.2-artifact-management).
3.   **Extraction Logic**: 
    *   It automatically detects `.tar` and `.tar.zst` files within the downloaded artifact [[.github/actions/download-artifact/action.yaml:82-85]](https://deepwiki.com/tenstorrent/tt-forge/5.4.2-artifact-management).
    *   For `.tar.zst`, it uses `tar --use-compress-program=zstd`[[.github/actions/download-artifact/action.yaml:87]](https://deepwiki.com/tenstorrent/tt-forge/5.4.2-artifact-management).
    *   After extraction, the original archive files are removed [[.github/actions/download-artifact/action.yaml:92]](https://deepwiki.com/tenstorrent/tt-forge/5.4.2-artifact-management).

## Integration Patterns

### Usage in Release Pipelines

Artifact management is critical for "uplifting" components (like MLIR binaries or Python wheels) from sub-repositories into a unified release.

`# Example usage of download-artifact in a workflow- name: Retrieve Build Artifacts  uses: ./.github/actions/download-artifact  with:    name: "tt-mlir-linux-x86_64"    repository: "tenstorrent/tt-mlir"    run_id: "123456789"    path: "external/mlir"`
### Community Workflow Integration

The repository also integrates with central Tenstorrent actions for community management. The workflow `.github/workflows/community-issue-tagging.yml` utilizes `tenstorrent/tt-github-actions/.github/workflows/on-community-issue.yml@main` to handle incoming issues and PRs, ensuring standardized labeling across the organization [[.github/workflows/community-issue-tagging.yml:1-16]](https://deepwiki.com/tenstorrent/tt-forge/5.4.2-artifact-management).
