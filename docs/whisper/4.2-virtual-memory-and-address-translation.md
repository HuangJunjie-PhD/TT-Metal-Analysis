# Virtual Memory and Address Translation

Relevant source files
*   [Hart.cpp](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp)

## Purpose and Scope

This page documents the virtual memory subsystem in Whisper, which handles address translation from virtual addresses (VA) to physical addresses (PA) using page tables and Translation Lookaside Buffers (TLBs). The virtual memory system implements RISC-V Sv32/Sv39/Sv48/Sv57 translation modes and supports two-stage translation for the hypervisor extension.

For physical memory management and protection mechanisms, see [Physical Memory and Protection](https://deepwiki.com/tenstorrent/whisper/4.1-physical-memory-and-protection). For hypervisor-specific two-stage translation details, see [Hypervisor Virtual Memory](https://deepwiki.com/tenstorrent/whisper/4.4-hypervisor-virtual-memory). For memory consistency model operations, see [Memory Consistency Model](https://deepwiki.com/tenstorrent/whisper/4.3-memory-consistency-model).

Sources: [Hart.hpp 40](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.hpp#L40-L40)[Hart.cpp 98-100](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L98-L100)

## Virtual Memory Architecture

The `VirtMem` class provides virtual-to-physical address translation services to the Hart. It maintains separate TLBs for different translation contexts and implements RISC-V privileged architecture's address translation modes.

**Virtual Memory Translation Flow**: The Hart's memory access operations go through VirtMem for address translation, which consults TLBs and performs page table walks on misses.

Sources: [Hart.hpp 40](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.hpp#L40-L40)[Hart.cpp 98-100](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L98-L100)[Hart.cpp 313-398](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L313-L398)

## Translation Modes

The VirtMem subsystem supports multiple RISC-V address translation modes. The active mode is configured via the SATP CSR (or VSATP/HGATP for two-stage translation).

### Supported Modes

| Mode | RV32 | RV64 | Page Size | Levels | VA Bits |
| --- | --- | --- | --- | --- | --- |
| `Bare` | ✓ | ✓ | N/A | 0 | Physical addressing |
| `Sv32` | ✓ | - | 4 KiB | 2 | 32 |
| `Sv39` | - | ✓ | 4 KiB | 3 | 39 |
| `Sv48` | - | ✓ | 4 KiB | 4 | 48 |
| `Sv57` | - | ✓ | 4 KiB | 5 | 57 |

### Mode Configuration

Translation modes are configured through CSR writes:

**Translation Mode Configuration**: The `updateAddressTranslation()` method reads CSRs and configures VirtMem state.

Sources: [Hart.cpp 718-752](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L718-L752)[Hart.cpp 254-305](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L254-L305)

The mode is set during reset and updated when SATP-family CSRs are written:

`// RV32 supports Bare and Sv32virtMem_.setSupportedModes({VirtMem::Mode::Bare, VirtMem::Mode::Sv32}); // RV64 supports Bare, Sv39, Sv48, Sv57virtMem_.setSupportedModes({VirtMem::Mode::Bare, VirtMem::Mode::Sv39,                            VirtMem::Mode::Sv48, VirtMem::Mode::Sv57 });`
Sources: [Hart.cpp 254-289](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L254-L289)

## TLB Architecture

The VirtMem subsystem maintains four separate TLB structures to optimize different translation scenarios:

### TLB Organization

**TLB Structure**: Four TLBs handle different translation contexts with entries containing VPN, PA, permissions, and identifiers.

Sources: [Hart.hpp 40](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.hpp#L40-L40)[Hart.cpp 98](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L98-L98)

### TLB Initialization

The TLBs are initialized in the Hart constructor with configurable sizes:

`Hart<URV>::Hart(unsigned hartIx, URV hartId, unsigned numHarts, Memory& memory,                Syscall<URV>& syscall, uint64_t& time)  : hartIx_(hartIx), numHarts_(numHarts), memory_(memory),    virtMem_(hartIx, memory.pageSize(), 2048)  // 2048 TLB entries{  setupVirtMemCallbacks();  // ...}`
Sources: [Hart.cpp 88-99](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L88-L99)

### TLB Entry Matching

TLB lookups match against multiple fields:

*   **Virtual Page Number (VPN)**: The page-aligned portion of the virtual address
*   **ASID**: Address Space Identifier from SATP
*   **VMID**: Virtual Machine Identifier from HGATP (for stage-2 TLB)
*   **World ID**: Security world identifier (when STEE is enabled)

## Address Translation Process

### Single-Stage Translation

For privilege modes below Machine mode (when SATP.MODE ≠ Bare), address translation follows this flow:

**Translation Sequence**: TLB lookup followed by page table walk on miss, with permission checking at each stage.

Sources: [Hart.cpp 1795-1940](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L1795-L1940)

### Load Translation Example

The `determineLoadException()` method orchestrates translation for load operations:

`ExceptionCause determineLoadException(uint64_t& addr1, uint64_t& addr2,                                        uint64_t& gaddr1, uint64_t& gaddr2,                                        unsigned ldSize, bool hyper, unsigned elemIx){  uint64_t va1 = URV(addr1);  // Virtual address  addr1 = gaddr1 = va1;       // Initialize    auto [pm, virt] = effLdStMode(hyper);    // Apply pointer masking  va1 = applyPointerMask(va1, true, hyper);    // Check for misalignment  uint64_t alignMask = ldSize - 1;  bool misal = addr1 & alignMask;    // Translate if not in Machine mode  bool translate = isRvs() and pm != PrivilegeMode::Machine;  if (translate) {    if (auto cause = virtMem_.translateForLoad(va1, pm, virt, gaddr1, addr1);        cause != ExceptionCause::NONE) {      ldStFaultAddr_ = addr1;      return cause;    }  }    // Check PMP/PMA on physical address  // Handle page crossing for misaligned access  // ...    return ExceptionCause::NONE;}`
Sources: [Hart.cpp 1793-1940](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L1793-L1940)

## Memory Access Callbacks

VirtMem uses callbacks to access physical memory during page table walks. These callbacks are set up in `setupVirtMemCallbacks()`:

**Memory Callbacks**: VirtMem uses callbacks to access physical memory and check permissions during page table walks.

Sources: [Hart.cpp 313-398](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L313-L398)

The callback implementations handle:

*   **STEE (Secure Tagged-pointer Extension Environment)**: Validates secure bits and clears them
*   **MCM (Memory Consistency Model)**: Routes reads through cache model if enabled
*   **PMP**: Physical Memory Protection checks
*   **PMA**: Physical Memory Attributes checks
*   **Byte swapping**: For big-endian systems

Example callback for memory reads:

`virtMem_.setMemReadCallback(<FileRef file-url="https://github.com/tenstorrent/whisper/blob/733fd921/this" undefined  file-path="this">Hii</FileRef> -> bool {  if (steeEnabled_) {    if (!stee_.isValidAddress(addr))      return false;      addr = stee_.clearSecureBits(addr);  }    // Read through MCM cache or direct memory  if (not ((mcm_ and dataCache_)?        peekMemory(addr, data, false) :        memory_.read(addr, data)))    return false;      if (bigEndian)    data = util::byteswap(data);  return true;});`
Sources: [Hart.cpp 316-332](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L316-L332)

## Pointer Masking

Pointer masking allows masking high-order bits of addresses before translation, supporting features like tagged pointers. The `PmaskManager` class handles this functionality.

### Pointer Masking Modes

**Pointer Masking Flow**: Masking is applied per privilege mode before translation, disabled when MXR is set.

Sources: [Hart.hpp 599-636](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.hpp#L599-L636)[Hart.cpp 1816](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L1816-L1816)

### Application in Translation

Pointer masking is applied early in the translation process:

`uint64_t applyPointerMask(uint64_t addr, bool isLoad, bool hyper = false) const{  auto [pm, virt] = effLdStMode(hyper);  bool bare = virtMem_.mode() == VirtMem::Mode::Bare;  if (virt) {    bare = virtMem_.vsMode() == VirtMem::Mode::Bare;    if (virtMem_.stage1ExecReadable())      return addr;   // If MXR, pointer masking does not apply.  }  else if (virtMem_.execReadable())    return addr;  // If MXR, pointer masking does not apply.      return pmaskManager_.applyPointerMask(addr, pm, virt, isLoad, bare);}`
Sources: [Hart.hpp 599-613](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.hpp#L599-L613)

## TLB Management and Invalidation

TLB entries must be invalidated when page tables or translation configuration changes. RISC-V provides SFENCE.VMA and HFENCE instructions for this purpose.

### SFENCE.VMA

Standard fence for single-stage translation:

**SFENCE.VMA Variants**: Different operand combinations control granularity of TLB invalidation.

Sources: [Hart.cpp 28-82](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L28-L82) (via execSfence_vma pattern in hypervisor.cpp)

### HFENCE.VVMA (VS-Stage TLB Invalidation)

For hypervisor to invalidate guest virtual memory translations:

`void Hart<URV>::execHfence_vvma(const DecodedInst* di){  auto& vsTlb = virtMem_.vsTlb_;  auto& stage2Tlb = virtMem_.stage2Tlb_;  auto vmid = virtMem_.vmid();  uint32_t wid = steeEnabled_? stee_.secureWorld() : 0;    if (di->op0() == 0 and di->op1() == 0) {    // Invalidate all entries for this VMID    vsTlb.invalidateVmid(vmid, wid);    stage2Tlb.invalidateVmid(vmid, wid);  }  else if (di->op0() == 0 and di->op1() != 0) {    // Invalidate all entries matching ASID and VMID    URV asid = intRegs_.read(di->op1());    vsTlb.invalidateAsidVmid(asid, vmid, wid);    stage2Tlb.invalidateAsidVmid(asid, vmid, wid);  }  else if (di->op0() != 0 and di->op1() == 0) {    // Invalidate specific VA across all ASIDs for this VMID    URV addr = intRegs_.read(di->op0());    uint64_t vpn = virtMem_.pageNumber(addr);    vsTlb.invalidateVirtualPageVmid(vpn, vmid, wid);    stage2Tlb.invalidateVmid(vmid, wid);  // Invalidate all G-stage  }  else {    // Invalidate specific VA+ASID combination    URV addr = intRegs_.read(di->op0());    uint64_t vpn = virtMem_.pageNumber(addr);    URV asid = intRegs_.read(di->op1());    vsTlb.invalidateVirtualPageAsid(vpn, asid, wid);    stage2Tlb.invalidateAsidVmid(asid, vmid, wid);  }}`
Sources: [hypervisor.cpp 28-82](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L28-L82)

### HFENCE.GVMA (G-Stage TLB Invalidation)

For hypervisor to invalidate guest physical address translations:

`void Hart<URV>::execHfence_gvma(const DecodedInst* di){  auto& stage2Tlb = virtMem_.stage2Tlb_;  auto& vsTlb = virtMem_.vsTlb_;  auto vmid = virtMem_.vmid();  uint32_t wid = steeEnabled_? stee_.secureWorld() : 0;    bool useGpa = not hfenceGvmaIgnoresGpa_;  // Config option    if (di->op0() == 0 and di->op1() == 0) {    // Invalidate all    stage2Tlb.invalidateVmid(vmid, wid);    vsTlb.invalidateVmid(vmid, wid);  }  else if (di->op0() == 0 and di->op1() != 0) {    // Invalidate by VMID    URV vmid = intRegs_.read(di->op1());    stage2Tlb.invalidateVmid(vmid, wid);    vsTlb.invalidateVmid(vmid, wid);  }  else if (di->op0() != 0 and di->op1() == 0) {    // Invalidate by GPA    URV addr = intRegs_.read(di->op0()) << 2;    uint64_t vpn = virtMem_.pageNumber(addr);    if (useGpa)      stage2Tlb.invalidateVirtualPageVmid(vpn, vmid, wid);    else      stage2Tlb.invalidateVmid(vmid, wid);  // Over-invalidate    vsTlb.invalidateVmid(vmid, wid);  }  else {    // Invalidate by GPA+VMID    URV addr = intRegs_.read(di->op0()) << 2;    uint64_t vpn = virtMem_.pageNumber(addr);    URV vmid = intRegs_.read(di->op1());    if (useGpa)      stage2Tlb.invalidateVirtualPageVmid(vpn, vmid, wid);    else      stage2Tlb.invalidateVmid(vmid, wid);    vsTlb.invalidateVmid(vmid, wid);  }}`
Sources: [hypervisor.cpp 85-153](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L85-L153)

### TLB Invalidation Methods

The TLB classes provide these invalidation methods:

| Method | Scope |
| --- | --- |
| `invalidate()` | Invalidate all entries |
| `invalidateAsid(asid, wid)` | Invalidate by ASID |
| `invalidateVmid(vmid, wid)` | Invalidate by VMID |
| `invalidateVirtualPageAsid(vpn, asid, wid)` | Invalidate single page with ASID |
| `invalidateVirtualPageVmid(vpn, vmid, wid)` | Invalidate single page with VMID |
| `invalidateAsidVmid(asid, vmid, wid)` | Invalidate by ASID and VMID |

The hypervisor extension provides HLV/HSV instructions for explicit guest memory access with two-stage translation:

**HLV/HSV Operation**: Hypervisor explicit access instructions temporarily set VS-mode translation context.

The `HLVX.HU` and `HLVX.WU` instructions enable execute permission checking for read access:

## Integration with Memory Protection

Virtual memory translation integrates with physical memory protection mechanisms:

**Protection Integration**: Translation works with PMP, PMA, and STEE to enforce access control.

Sources: [Hart.cpp 1830-1863](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L1830-L1863)[Hart.cpp 361-397](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L361-L397)

The protection checks occur in this order:

1.   **PTE permissions**: During page table walk
2.   **PMP**: On physical address after translation
3.   **PMA**: Physical memory attributes
4.   **STEE**: Secure memory validation (if enabled)

Example check from `determineLoadException()`:

`auto checkPa = [this, pm, misal] (uint64_t va, uint64_t& pa, Pma& pma, bool lower) -> ExceptionCause {  ldStFaultAddr_ = va;    // Check PMP  if (pmpEnabled_) {    const Pmp& pmp = pmpManager_.accessPmp(pa);    if (not pmp.isRead(pm) or (virtMem_.isExecForRead() and not pmp.isExec(pm)))      return ExceptionCause::LOAD_ACC_FAULT;  }    // Check STEE  if (steeEnabled_) {    if (not stee_.isValidAddress(pa))      return ExceptionCause::LOAD_ACC_FAULT;    bool& steeInsec = lower? steeInsec1_ : steeInsec2_;    steeInsec = stee_.isInsecureAccess(pa);    if (steeTrapRead_ and steeInsec)      return ExceptionCause::LOAD_ACC_FAULT;    pa = stee_.clearSecureBits(pa);  }    // Check PMA  pma = accessPma(pa);  pma = overridePmaWithPbmt(pma, virtMem_.lastEffectivePbmt());  if (not pma.isRead() or (virtMem_.isExecForRead() and not pma.isExec()))    return ExceptionCause::LOAD_ACC_FAULT;  if (misal and not pma.isMisalignedOk())    return pma.misalOnMisal()? ExceptionCause::LOAD_ADDR_MISAL : ExceptionCause::LOAD_ACC_FAULT;      return ExceptionCause::NONE;};`
Sources: [Hart.cpp 1830-1863](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L1830-L1863)
