# Mock Backend

Relevant source files
*   [src/backend/mock.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs)

The Mock Backend is a specialized implementation of the `TelemetryBackend` trait designed for development, testing, and demonstration purposes. It simulates realistic hardware behavior—including power fluctuations, temperature shifts, and firmware heartbeats—without requiring physical Tenstorrent hardware or specialized drivers.

### Purpose and Scope

*   **Development:** Enables UI and animation work on machines without PCIe access to Grayskull, Wormhole, or Blackhole devices.
*   **CI/CD:** Provides a stable, predictable environment for integration tests and GitHub Actions workflows [src/backend/mock.rs 12](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L12-L12)
*   **Scalability Testing:** Simulates high-density configurations like a **Galaxy** (32 chips) or **Quad Galaxy** (128 chips) to test UI performance and layout logic [src/backend/mock.rs 33-36](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L33-L36)

* * *

### Implementation Details

The `MockBackend` struct manages a collection of virtual `Device` objects and uses internal counters to generate temporally varied data.

#### Core Structure

The backend maintains internal maps for telemetry and base values to simulate persistence and drift over time.

| Field | Type | Description |
| --- | --- | --- |
| `device_count` | `usize` | Total number of virtual devices to simulate [src/backend/mock.rs 64](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L64-L64) |
| `scenario` | `MockScenario` | Predefined hardware layout (Mixed, Galaxy, etc.) [src/backend/mock.rs 67](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L67-L67) |
| `update_count` | `usize` | Counter incremented on every `update()` call to drive oscillations [src/backend/mock.rs 82](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L82-L82) |
| `base_power` | `HashMap<usize, f32>` | The baseline wattage for each device, used as a seed for random variation [src/backend/mock.rs 85](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L85-L85) |

#### Data Flow: Initialization to Update

When `init()` is called, the backend populates its device list based on the chosen `MockScenario`. The `update()` method then calculates new telemetry values for every device.

### Hardware Scenarios

The backend supports several `MockScenario` variants to test different architectural layouts and system densities [src/backend/mock.rs 29-39](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L29-L39)

| Scenario | Device Count | Architecture Distribution | Use Case |
| --- | --- | --- | --- |
| `Mixed` | 6 | Cycles E150 (GS), N150 (WH), P150 (BH) | General UI testing across all supported generations. |

| `QuadGalaxy` | 128 | All P150A (Blackhole) | Stress testing the TUI/GUI rendering and memory usage. |

### Realistic Data Generation

To ensure the UI looks "alive," the mock backend does not return static values. It simulates hardware dynamics using trigonometric functions and random offsets.

#### Power and Temperature Correlation

Power consumption is modeled as a sine wave combined with random noise. Temperature is then derived from the power value to simulate thermal mass [src/backend/mock.rs 15-16](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L15-L16)

*   **Power Calculation:**`base_power + (update_count.sin() * amplitude) + random_jitter`
*   **Temperature Correlation:** As power increases, the generated `asic_temp` in `Telemetry` increases proportionally [src/backend/mock.rs 16](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L16-L16)

#### Firmware and Limits

Each mock device is populated with realistic `FirmwaresInfo` and `DeviceLimits` to ensure the "System Info" panels in the UI are fully populated [src/backend/mock.rs 183-194](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mock.rs#L183-L194)

`// Example of mock limit populationdevice.limits = Some(DeviceLimits {    tdp_limit: Some(300.0),    tdc_limit: Some(40.0),    asic_fmax: Some(800),    thm_limit: Some(105.0),});`

### Usage in Workflows

The Mock Backend is triggered via CLI arguments or environment variables.

#### CLI Configuration

The `BackendType` enum handles the dispatching to `MockBackend` when the `--mock` flag is used [src/backend/mod.rs 56](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/backend/mod.rs#L56-L56)

*   **Standard Mock:**`tt-toplike --mock` (Uses `MockScenario::Mixed`)
*   **Custom Count:**`tt-toplike --mock-devices 8` (Generates 8 devices)
*   **Scenario-based:** Handled via internal configuration during development or testing.

#### Testing

The mock backend is the primary tool for the `test-modes.sh` script, which cycles through different TUI visualization modes (Starfield, Arcade, etc.) to ensure no regressions in the rendering engine without needing a physical lab machine.
