## Purpose and Scope

This document describes the Remote Data Management system in the TT-NN Visualizer, which enables users to connect to remote servers, browse and synchronize memory and performance report folders, and analyze TT-NN model data without requiring local copies of large files. For information about managing local data, see [Local Data Management](https://deepwiki.com/tenstorrent/ttnn-visualizer/3.1-local-data-management).

Sources: [src/hooks/useRemote.tsx 1-175](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/hooks/useRemote.tsx#L1-L175)[src/components/report-selection/RemoteSyncConfigurator.tsx 1-463](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/report-selection/RemoteSyncConfigurator.tsx#L1-L463)

## Overview

The Remote Data Management system allows users to:

1.   Configure and manage SSH connections to remote servers
2.   Browse available profiler reports and performance reports on remote machines
3.   Synchronize remote folders with the local system
4.   Mount remote folders for visualization and analysis
5.   Query data directly from remote sources

The system supports two operational modes:

*   **Sync Mode**: Remote data is synchronized to local storage before analysis
*   **Remote Querying Mode**: Data is queried directly on the remote server (currently limited due to SQLite version compatibility issues)

## Remote Connection Management

### Connection Configuration

Remote connections are defined by the `RemoteConnection` interface, which includes:

`interface RemoteConnection {    name: string;             // Display name for the connection    username: string;         // SSH username    host: string;             // SSH hostname or IP    port: number;             // SSH port (default: 22)    profilerPath: string;     // Path to memory reports on remote server    performancePath?: string; // Optional path to performance reports    sqliteBinaryPath?: string; // Path to SQLite binary on remote system    useRemoteQuerying: boolean; // Whether to query data remotely}`
The UI provides a dialog for adding and editing remote connections, which allows users to test the connection before saving it.

Sources: [src/definitions/RemoteConnection.ts 7-17](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/definitions/RemoteConnection.ts#L7-L17)[src/components/report-selection/RemoteConnectionDialog.tsx 22-254](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/report-selection/RemoteConnectionDialog.tsx#L22-L254)[src/hooks/useRemote.tsx 99-151](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/hooks/useRemote.tsx#L99-L151)

### Persistent State Management

Remote connection details and folder information are stored using the application's configuration system, which persists data between sessions:

| Configuration Key | Description |
| --- | --- |
| `remoteConnections` | Array of all saved remote connections |
| `selectedConnection` | Currently selected remote connection |
| `[connection.name] - reportFolders` | Saved memory report folders for a connection |
| `[connection.name] - performanceFolders` | Saved performance report folders for a connection |

This persistence allows users to quickly resume working with previously configured remote connections without re-entering details.

## Remote Folder Operations

### Listing Remote Folders

The system can retrieve a list of available memory report folders and performance report folders from a remote server using SSH:

Each remote folder is represented by the `RemoteFolder` interface:

`interface RemoteFolder {    testName: string;         // Name of the test/report    remotePath: string;       // Full path on remote server    lastModified: number;     // Timestamp of last modification    lastSynced?: number;      // Timestamp of last synchronization}`

### Synchronizing Remote Folders

The system supports synchronizing remote folders to the local system for analysis:

When a folder is synchronized, the system:

1.   Copies all necessary files from the remote server to local storage
2.   Updates the `lastSynced` timestamp to match the `lastModified` timestamp
3.   Updates the UI to show the folder is now up-to-date

### Mounting Remote Folders

Once a folder is synchronized or selected for remote querying, it can be mounted for visualization:

When a folder is mounted:

1.   The backend creates a session with the remote connection details
2.   The active report is set to the selected folder
3.   The report location is set to 'remote'
4.   The query cache is cleared to ensure fresh data is loaded

## Remote Data Query System

### CSV Query Execution

The backend supports querying CSV files on both local and remote systems through two classes:

1.   `LocalCSVQueryRunner` - For querying synchronized local copies of CSV files
2.   `RemoteCSVQueryRunner` - For directly querying CSV files on remote systems

The query system automatically selects the appropriate query runner based on the session's remote connection state and whether remote querying is enabled.

Sources: [backend/ttnn_visualizer/csv_queries.py 19-544](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/backend/ttnn_visualizer/csv_queries.py#L19-L544)

### Performance Report Handling

Performance reports are accessed through specialized query classes that handle the CSV format and perform conversions:

*   `DeviceLogProfilerQueries` - For querying device log profiler data
*   `OpsPerformanceQueries` - For querying operations performance data
*   `OpsPerformanceReportQueries` - For generating analyzed performance reports

These classes support both local and remote data sources and provide methods to query specific aspects of performance data.

The `RemoteConnectionSelector` component allows users to:

*   Select from saved remote connections

### Remote Folder Selector

The `RemoteFolderSelector` component provides:

*   A filterable dropdown list of available remote folders
*   Visual indicators for folder sync status
*   Buttons to trigger synchronization

The component visually indicates folder status:

*   Yellow history icon - Folder is out of date and needs syncing
*   Green checkmark - Folder is up to date
*   Spinner - Folder status is being checked

The `RemoteSyncConfigurator` component is the main interface for remote data management, integrating:

*   Adding new remote connections
*   Selecting and managing existing connections
*   Browsing and selecting memory and performance report folders
*   Synchronizing remote data
*   Mounting folders for visualization

The component maintains its own state for:

*   Currently selected connection
*   Selected memory report folder
*   Selected performance report folder

## Best Practices

### Remote Connection Setup

1.   Ensure SSH access is properly configured with key-based authentication where possible
2.   Verify paths to profiler and performance reports exist on the remote system
3.   Test connections before saving them
4.   Use descriptive connection names to easily identify different remote systems

### Folder Synchronization

1.   Only synchronize folders that need to be analyzed to minimize data transfer
2.   Check if a folder is already synchronized (green checkmark) before attempting to sync again
3.   Allow synchronization to complete before attempting to analyze the data

### Performance Considerations

1.   Prefer local synchronization for large datasets to avoid network latency during analysis
2.   Remote querying (when available) can be more efficient for quick checks on small datasets
3.   Consider network bandwidth and latency when working with remote data sources
