# Data Processing Pipeline

Relevant source files
*   [src/tt_perf_report/perf_report.py](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py)

This document explains the data processing pipeline in the tt-perf-report tool, which transforms raw CSV performance traces from Tenstorrent hardware into actionable performance analysis and optimization recommendations. It covers the flow of data from input through analysis to final output.

For information about how specific operations like matmul, convolution, and halo operations are analyzed in detail, see [Operation Analysis Subsystem](https://deepwiki.com/tenstorrent/tt-perf-report/7.2-operation-analysis-subsystem).

## Operation Analysis

Each operation is analyzed to extract performance metrics using the `analyze_op()` function:

Sources: [src/tt_perf_report/perf_report.py 375-512](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L375-L512)

### Matmul Analysis

For matrix multiplication operations, the system calculates:

*   DRAM bandwidth and utilization percentage
*   FLOPs (floating-point operations per second) and utilization percentage
*   Operation dimensions (M×K×N)
*   Core count and adjustment for DRAM-sharded configurations
*   Math fidelity and datatypes

Sources: [src/tt_perf_report/perf_report.py 208-277](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L208-L277)

### Convolution Analysis

For convolution operations, the system calculates:

*   FLOPs and utilization percentage
*   Operation dimensions and configuration parameters
*   Math fidelity
*   Specialized parameters like activation block height, double buffering, etc.

Sources: [src/tt_perf_report/perf_report.py 315-372](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L315-L372)

### Halo Analysis

For halo operations, the system extracts configuration details:

*   Window dimensions
*   Stride settings
*   Padding configuration
*   Dilation settings
*   Memory layout

### Console Output Format

The console output includes:

1.   A performance table with columns:

    *   ID: Original row number
    *   Total %: Percentage of total execution time
    *   Bound: Performance classification (DRAM, FLOP, BOTH, SLOW, HOST)
    *   OP Code: Operation name and dimensions
    *   Device Time: Operation execution time (μs)
    *   Op-to-Op Gap: Time between operations (μs)
    *   Cores: Number of cores used
    *   DRAM: DRAM bandwidth (GB/s)
    *   DRAM %: DRAM utilization percentage
    *   FLOPs: Compute throughput (TFLOPs)
    *   FLOPs %: Compute utilization percentage
    *   Math Fidelity: Math precision configuration

2.   An advice section with optimization recommendations:

    *   Fallback Advice: For host operations
    *   Op-to-Op Gap Advice: For high latency between operations
    *   Matmul Advice: For matrix multiplication optimization

Sources: [src/tt_perf_report/perf_report.py 633-663](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L633-L663)[src/tt_perf_report/perf_report.py 666-730](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L666-L730)
