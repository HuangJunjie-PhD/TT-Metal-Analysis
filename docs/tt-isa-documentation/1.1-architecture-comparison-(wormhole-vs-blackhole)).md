## Purpose and Scope

This page documents the architectural differences between Wormhole B0 and Blackhole A0 ASICs, focusing on instruction set enhancements, computational model improvements, and performance characteristics. Both architectures share the same fundamental tile-based structure with Tensix compute tiles, NoC interconnect, and memory hierarchy, but Blackhole introduces targeted improvements primarily in the Vector Unit (SFPU), register addressing, and floating-point computation.

For detailed information about specific instruction sets, see the Instruction Set Reference sections ([#5](https://deepwiki.com/tenstorrent/tt-isa-documentation/5-instruction-set-reference)). For tile architecture details, see Tensix Tile Architecture ([#3](https://deepwiki.com/tenstorrent/tt-isa-documentation/3-tensix-tile-architecture)) and Tensix Coprocessor Architecture ([#4](https://deepwiki.com/tenstorrent/tt-isa-documentation/4-tensix-coprocessor-architecture)).

## Architectural Commonalities

Both Wormhole B0 and Blackhole A0 share the same high-level architecture:

*   80 Tensix Tiles with identical component structure (5 Baby RISC-V cores, Tensix Coprocessor)
*   Dual 2D torus NoC topology (NoC #0 and NoC #1)
*   18 DRAM tiles with GDDR6 (2 GiB each, 36 GiB total)
*   16 Ethernet tiles for inter-ASIC communication
*   Same L1 memory size per tile (1464 KiB, 16 banks)
*   Same Matrix Unit (FPU) and Vector Unit (SFPU) sub-unit organization
*   Same register file dimensions: `SrcA` (64×16, 19-bit), `SrcB` (64×16, 19-bit), `Dst` (1024×16 or 512×16)

The improvements in Blackhole are evolutionary rather than revolutionary, targeting specific computational bottlenecks and hardware bugs present in Wormhole.

**Sources:**[WormholeB0/README.md](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/README.md?plain=1)[BlackholeA0/README.md](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/README.md?plain=1)

## Clock Frequency and Peak Performance

Blackhole increases the clock frequency from 1.00 GHz to 1.35 GHz, providing a **35% performance increase** across all execution units when compute-bound. This translates to:

*   **Matrix Unit (FPU):** ~5.530 TFLOP/s peak (estimated from 1.35× scaling)
*   **Vector Unit (SFPU):** 0.0864 TFLOP/s peak for `SFPMAD` (32 lanes × 2 ops × 1.35 GHz)

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 90](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L90-L90)[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 67](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L67-L67)

## Floating-Point Computation Model Differences

Both architectures implement partially-fused FMA via `fma_model_wh()` and `fma_model_bh()` functions:

**FMA Model Code Entity Comparison**

**Key Differences:**

1.   **Operand Negation Modifiers** ([BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 30-31](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L30-L31)):

    *   `SFPMAD_MOD1_NEGATE_VA`: Negates the `VA` operand before multiplication
    *   `SFPMAD_MOD1_NEGATE_VC`: Negates the `VC` addend
    *   Enables subtraction operations without separate instructions
    *   Implemented as simple XOR with sign bit (`a ^= 0x80000000`)

2.   **Automatic Instruction Scheduling** ([BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 69-86](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L69-L86)):

    *   Wormhole: Software must manually insert `SFPNOP` after `SFPMAD` to avoid read-after-write hazards
    *   Blackhole: Hardware automatically stalls if next instruction would read the result too early
    *   **Exception:** Automatic stalling does not apply to `SFPMAD` within `SFPLOADMACRO` sequences
    *   **Known bugs:** Several instructions not detected by stalling logic (`SFPAND`, `SFPOR`, `SFPIADD`, `SFPSHFT`, `SFPCONFIG`, `SFPSWAP`, `SFPSHFT2` modes)

3.   **Improved NaN and Zero Handling** ([BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 60](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L60-L60)):

    *   Blackhole always emits canonical NaN (`0x7fc00000`) rather than non-deterministic mantissa bits
    *   Negative zero handling improved (see FMA model section below)

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 1-91](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L1-L91)[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 1-68](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L1-L68)

## Floating-Point Computation Model Differences

Both architectures implement partially-fused FMA (not fully IEEE754-compliant), but with different algorithmic approaches:

**Detailed Differences:**

| Aspect | Wormhole | Blackhole | IEEE754 |
| --- | --- | --- | --- |
| **Zero Output Sign** | Always `+0.0` | Sign-preserved | Sign-preserved |
| **NaN Bit Pattern** | `0x7f800001` + variable bits | `0x7fc00000` (canonical) | Implementation-defined |
| **Denormal Rounding** | Flushes before rounding (buggy) | Flushes after rounding | Full precision |
| **Sticky Bit** | Bug in 2-bit shifts | Correct | N/A (IEEE uses infinite precision) |
| **Negative Zero Input** | May produce `+0.0` output | Correctly handled | Sign rules apply |

## Backward Compatibility Considerations

**Instruction Binary Compatibility:**

*   All Wormhole instruction encodings remain valid in Blackhole
*   New Blackhole modifiers (`NEGATE_VA`, `NEGATE_VC`) use previously reserved bits
*   Software compiled for Wormhole will execute on Blackhole but may miss optimization opportunities

**Numerical Compatibility:**

*   FMA results differ between architectures for edge cases (NaN, negative zero, near-denormal values)
*   Bulk computation results are **not** bit-identical between architectures
*   Applications requiring bit-exact reproducibility must account for architecture differences

**Programming Model Compatibility:**

*   Blackhole's automatic stalling reduces mandatory `SFPNOP` usage but doesn't eliminate it
*   Code optimized for Blackhole (omitting `SFPNOP`) will be inefficient on Wormhole
*   `SFPLOADMACRO` sequences require same manual scheduling on both architectures

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 1-91](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L1-L91)[Miscellaneous/FMA/README.md 12-31](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/README.md?plain=1#L12-L31)

## Summary of Key Differences

Blackhole A0 represents an incremental but meaningful improvement over Wormhole B0, focusing on:

1.   **35% higher clock frequency** (1.35 GHz vs 1.00 GHz)
2.   **Enhanced vector operations** with operand negation and automatic scheduling
3.   **Improved floating-point semantics** for better numerical behavior
4.   **Flexible register addressing** for bank conflict mitigation
5.   **Bug fixes** in critical shuffle operations

These improvements maintain instruction-level compatibility while providing better performance, numerical accuracy, and ease of programming.
