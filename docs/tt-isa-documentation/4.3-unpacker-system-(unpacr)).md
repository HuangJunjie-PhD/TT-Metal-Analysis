## Purpose and Scope

The Unpacker System is a backend execution unit within the [Tensix Coprocessor](https://deepwiki.com/tenstorrent/tt-isa-documentation/4-tensix-coprocessor-architecture) responsible for moving data from [L1 scratchpad memory](https://deepwiki.com/tenstorrent/tt-isa-documentation/3.2-memory-hierarchy-and-l1-ram) into the internal register files ([`SrcA`, `SrcB`](https://deepwiki.com/tenstorrent/tt-isa-documentation/4.7-register-files:-srca-srcb-dst), or [`Dst`](https://deepwiki.com/tenstorrent/tt-isa-documentation/4.7-register-files:-srca-srcb-dst)) that feed the compute units. The system performs three primary functions during data movement: (1) 4D tensor address generation, (2) decompression of block floating-point (BFP) formats, and (3) format conversion between L1 data types and internal representations.

This document covers the `UNPACR` instruction variants, addressing mechanisms, format conversion capabilities, and performance characteristics. For Matrix Unit operations that consume unpacked data, see [Matrix Unit (FPU) Architecture](https://deepwiki.com/tenstorrent/tt-isa-documentation/4.4-matrix-unit-(fpu)-architecture). For Vector Unit operations, see [Vector Unit (SFPU) Architecture](https://deepwiki.com/tenstorrent/tt-isa-documentation/4.5-vector-unit-(sfpu)-architecture). For the inverse operation (writing data back to L1), see [Packer System (PACR)](https://deepwiki.com/tenstorrent/tt-isa-documentation/4.6-packer-system-(pacr)).

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 1-5](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L1-L5)

## Architecture Overview

The Unpacker System consists of two independent unpacker units (`Unpacker[0]` and `Unpacker[1]`) that can operate concurrently:

**Unpacker Routing:**

*   **Unpacker[0]**: Can write to either `SrcA` or `Dst` (controlled by `REG2_Unpack_If_Sel` / `REG2_Unpack_if_sel_cntx`)
*   **Unpacker[1]**: Always writes to `SrcB`

Each unpacker maintains per-thread state including:

*   `SrcBank`: Which bank (0 or 1) to write to in `SrcA`/`SrcB`
*   `SrcRow[CurrentThread]`: Base row offset for current thread
*   `ContextCounter[CurrentThread]`: Counter for multi-context mode (3 bits)

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 36-39](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L36-L39)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 219-232](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L219-L232)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 394-401](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L394-L401)

## UNPACR Instruction Variants

The `UNPACR` instruction has three operational modes distinguished by encoding bits:

### Regular Data Movement

The primary `UNPACR` operation moves a specified number of datums from L1 to an internal register file. Key parameters in the encoding:

| Field | Bits | Purpose |
| --- | --- | --- |
| `WhichUnpacker` | 1 | Select Unpacker[0] or Unpacker[1] |
| `Ch0YInc`, `Ch0ZInc` | 2×2 | Channel 0 Y/Z dimension increment after instruction |
| `Ch1YInc`, `Ch1ZInc` | 2×2 | Channel 1 Y/Z dimension increment after instruction |
| `ContextNumber` | 3 | Explicit context selection (if not using counter) |
| `ContextADC` | 2 | Which ADC to use for XY addressing in multi-context mode |
| `MultiContextMode` | 1 | Enable context-based configuration selection |
| `FlipSrc` | 1 | Swap banks and reset row counter after instruction |
| `AllDatumsAreZero` | 1 | Skip L1 reads, write zeroes to destination |
| `UseContextCounter` | 1 | Use counter instead of explicit `ContextNumber` |
| `RowSearch` | 1 | Alternative addressing mode for compressed data |

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 7-26](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L7-L26)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 34-57](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L34-L57)

### Increment Context Counter

When using `MultiContextMode` with `UseContextCounter`, this variant advances the per-thread context counter without moving data:

`ContextCounter[CurrentThread] = (ContextCounter[CurrentThread] + 1) % (1 << Context_count)`
This enables rapid switching between up to 8 distinct configuration sets without RISC-V intervention.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_IncrementContextCounter.md 28-41](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_IncrementContextCounter.md?plain=1#L28-L41)

### Flush Cache

Each unpacker maintains four one-entry caches for row start table lookups during decompression. This variant invalidates these caches, necessary when L1 memory containing compressed data metadata is being rapidly reused.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_FlushCache.md 29-32](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_FlushCache.md?plain=1#L29-L32)

## Address Generation System

Unpacker address generation uses a complex system of Auto-incrementing Addressing Counters (ADCs) and configuration state to support 4D tensor addressing:

### ADC Structure

Each thread (T0, T1, T2) has its own set of ADCs, indexed as `ADCs[CurrentThread].Unpacker[WhichUnpacker]`. Each ADC has two channels:

*   **Channel[0]**: Input address generation (XY positioning within tile)
*   **Channel[1]**: Output address generation and additional input bounds

The ADC counters are incremented by the `Ch0YInc`, `Ch0ZInc`, `Ch1YInc`, `Ch1ZInc` fields after each `UNPACR` instruction completes.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 130-179](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L130-L179)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 384-393](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L384-L393)

### Multi-Context Mode

When `MultiContextMode` is enabled, the unpacker can rapidly switch between up to 8 configuration contexts:

In multi-context mode:

*   Unpacker 0 supports contexts 0-7
*   Unpacker 1 supports contexts 0-1 only
*   The `ContextADC` field (0-2) selects which thread's ADC to use for XY addressing
*   Context 0 always uses the non-context-specific configuration fields

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 42-56](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L42-L56)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 63-87](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L63-L87)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 99-105](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L99-L105)

## Data Pipeline

The unpacker data pipeline consists of four stages:

### Stage 1: L1 Read

Data is fetched from L1 in one of three streams:

1.   **Data stream** (`InAddr_Datums`): The actual datum bits
2.   **Exponent stream** (`InAddr_Exponents`): For BFP formats, shared exponents (1 byte per 16 datums)
3.   **Delta stream** (`InAddr_Deltas`): For compressed data, run-length deltas (4 bits per datum)

The address calculation includes wraparound logic for circular buffer support, controlled by `Unpack_fifo_size` and `Unpack_limit_address` configuration.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 181-203](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L181-L203)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 281-311](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L281-L311)

### Stage 2: Decompression

For compressed data (`IsUncompressed == false`), the unpacker reads a row start table and delta values:

*   **Row Start Table**: Array of `uint16_t` values, one per row (or per blob), indicating the index of the first datum in each row
*   **Delta Values**: 4-bit values (0-15) indicating how many consecutive zero datums follow each non-zero datum

The decompression inserts `DecompressionZeroes` zero datums after each non-zero datum read from L1.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 108-118](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L108-L118)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 166-179](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L166-L179)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 302-311](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L302-L311)

### Stage 3: Format Conversion

The unpacker performs format conversion from L1 data types to internal representations. See the Format Conversion section below for details.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 312-313](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L312-L313)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 420-493](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L420-L493)

### Stage 4: Output

Converted datums are written to the destination register file with:

*   **Output addressing**: Based on `OutAddr` calculation using configured strides
*   **Upsampling**: Optional insertion of zeroes between datums (controlled by `Upsample_rate`)
*   **Bank/row management**: Automatic bank flipping and row offset updates

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 318-373](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L318-L373)

## Format Conversion

The unpacker supports a wide range of format conversions, varying by destination register file:

### Conversions to SrcA/SrcB

| L1 Format | → | Internal Format | Configuration |
| --- | --- | --- | --- |
| FP32 | → | TF32, BF16, FP16 | `InDataFormat=FP32`, `OutDataFormat=TF32/BF16/FP16` |
| TF32 | → | TF32, BF16, FP16 | `InDataFormat=FP32`, `OutDataFormat=TF32/BF16/FP16` |
| BF16 | → | TF32, BF16 | `InDataFormat=BF16`, `OutDataFormat=BF16` |
| BFP8/4/2 | → | TF32, BF16 | `InDataFormat=BFP8/4/2`, `OutDataFormat=BFP8/4/2` |
| BFP8a/4a/2a | → | FP16 | `InDataFormat=BFP8a/4a/2a` |
| FP16, FP8 | → | FP16 | `InDataFormat=FP16/FP8` |
| INT8 (signed) | → | TF32, BF16 | `InDataFormat=BFP8`, `REG2_Force_shared_exp=1` |
| INT8/UINT8 | → | Integer "8" | `InDataFormat=INT8`, `SrcAUnsigned/SrcBUnsigned` |
| INT16 | → | Integer "16" | `InDataFormat=INT16` |

| L1 Format | → | Internal Format | Configuration |
| --- | --- | --- | --- |

| INT8/UINT8 | → | Integer "8" | `InDataFormat=INT8`, `SrcAUnsigned` |
| INT16 | → | Integer "16" | `InDataFormat=INT16` |

### BFP Conversion Details

Block Floating-Point formats share a single exponent across 16 datums:

The conversion algorithms:

*   **BFP8/4/2 → BF16**: [WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 495-506](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L495-L506)
*   **BFP8a/4a/2a → FP16**: [WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 508-520](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L508-L520)

### Integer Conversion

INT8 conversion has two modes:

*   **Sign-magnitude**: When `SrcAUnsigned/SrcBUnsigned = 0`, bit 7 is sign, bits 6-0 are magnitude
*   **Unsigned**: When `SrcAUnsigned/SrcBUnsigned = 1`, all 8 bits are unsigned value

The converted value is overlaid onto FP16 format with a dummy exponent of 16 (corresponding to 2^1 scale).

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/FormatConversion.md 1-81](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/FormatConversion.md?plain=1#L1-L81)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 459-471](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L459-L471)

## Decompression

The unpacker supports two compression schemes: run-length encoding and blob-based sparse addressing.

### Row Start Table Structure

For compressed data, the unpacker expects a row start table at the beginning of the data buffer (after the digest):

Each row start entry contains the datum index (not byte offset) of the first datum in that row. The delta stream indicates how many zero datums follow each non-zero datum.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 108-118](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L108-L118)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 166-179](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L166-L179)

### Blob-Based Sparse Addressing

When `BlobsPerXYPlane != 0`, the unpacker uses an alternative addressing mode for sparse data:

*   The tile is divided into vertical "blobs"
*   `BlobsYStart[0..7]` specifies the X coordinate (in units of 16) where each blob starts
*   The row start table has one entry per blob (not per Y row)
*   `RowSearch` mode interprets ADC.X as blob index and ADC.Y as position within blob

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 144-159](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L144-L159)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 167-168](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L167-L168)

### Delta Encoding

The compressed data is organized in 32-datum blocks:

*   32 datums (at `DatumSizeBytes` each)
*   32 delta values (4 bits each = 16 bytes total)

After reading a non-zero datum, the unpacker reads its 4-bit delta and inserts that many zero datums before the next non-zero datum.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 187-192](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L187-L192)[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 302-311](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L302-L311)

## Performance Characteristics

Unpacker performance is primarily limited by L1 memory bandwidth, with three speed modes:

### Bandwidth Modes

Each unpacker can operate at one of three speeds:

| Mode | Bandwidth | L1 Ports Used | Controlled By |
| --- | --- | --- | --- |
| **x1** | 16 bytes/cycle | 1 port | `Throttle_mode = 0` |
| **x2** | 32 bytes/cycle | 2 ports | `Throttle_mode = 1` |
| **x4** | 64 bytes/cycle | 4 ports | `Throttle_mode = 2` |

Certain modes force specific speeds:

*   **Always x4**: `DiscontiguousInputRows` mode
*   **Always x1**: Compressed data, `Upsample_rate >= 3`, BFP2/BFP2a formats
*   **x1 or x2**: `Upsample_rate == 1`, BFP4/BFP4a with `UnpackToDst`

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 574-598](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L574-L598)

### Multi-Unpacker Interference

When both unpackers operate concurrently, they can interfere with each other's bandwidth:

| Unpacker 1 Wants | Unpacker 0 Wants x1 | x2 | x4 |
| --- | --- | --- | --- |
| **x1** | Both get requested | U0 gets x2, U1 gets x1 | U0 gets x4, U1 gets x1 |
| **x2** | U0 gets x1, U1 gets x2 | Both get x1 ⚠️ | U0 gets x2, U1 gets x1 ⚠️ |
| **x4** | U0 gets x1, U1 gets x4 | U0 gets x1, U1 gets x2 ⚠️ | Both get x2 ⚠️ |

The interference pattern shows that when both unpackers want high bandwidth (≥x2), neither gets their full requested bandwidth.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 581-587](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L581-L587)

### Address Calculation Latency

Each `UNPACR` instruction has initial overhead before data movement begins:

*   **Uncompressed data**: Exactly 2 cycles
*   **Compressed data**: 2+ cycles (depends on row start table lookup)

During these cycles, no thread can start another `UNPACR` instruction, creating a serialization point.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 572-574](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L572-L574)

## Configuration Reference

The unpacker behavior is controlled by numerous configuration fields in the `Config[StateID]` array:

### TileDescriptor Configuration

The `THCON_SEC[WhichUnpacker].REG0_TileDescriptor` structure (128 bits) contains tile geometry:

| Field | Bits | Range | Description |
| --- | --- | --- | --- |
| `InDataFormat` | 4 | 0-15 | L1 data format encoding |
| `IsUncompressed` | 1 | 0-1 | `1` = uncompressed, `0` = compressed |
| `NoBFPExpSection` | 1 | 0-1 | `0` for BFP2/4 formats |
| `BlobsPerXYPlane` | 3 | 0-7 | Sparse blob count (0 = disabled) |
| `XDim` | 16 | 1-65535 | Tile width in datums |
| `YDim` | 8 | 0-255 | Tile height in rows |
| `ZDim` | 8 | 0-255 | Z dimension (0 interpreted as 1) |
| `WDim` | 8 | 0-255 | W dimension (0 interpreted as 1) |
| `BlobsYStart` | 32 | — | 8×4-bit blob X coordinates |
| `DigestSize` | 8 | 0-255 | Header size = (1+DigestSize)×16 bytes |

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 602-622](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L602-L622)

The `THCON_SEC[WhichUnpacker]` structure contains unpacker-specific configuration:

| Field | Purpose |

| `REG2_Unpack_If_Sel` | Unpacker 0: route to Dst (1) or SrcA (0) |

| `REG5_Tile_x_dim_cntx[0..3]` | Context-specific X dimensions |
| `REG5_Dest_cntx[0..3].address` | Context-specific output addresses |
| `REG7_Offset_address` | Address offset (context 0) |
| `REG7_Offset_cntx[1..3].address` | Context-specific offsets |
| `REG7_Unpack_data_format_cntx[0..7]` | Context-specific input formats |
| `REG7_Unpack_out_data_format_cntx[0..7]` | Context-specific output formats |

| `Unpack_Src_Reg_Set_Upd` | Auto-increment SrcRow after unpack |
| `Upsample_rate` | Insert 2^N - 1 zeroes between datums |
| `Upsample_and_interleave` | Skip zeroes when upsampling |

| `Throttle_mode` | Bandwidth mode (0=x1, 1=x2, 2=x4) |

### UNP Configuration

The `UNP[WhichUnpacker]` structure controls output addressing:

| Field | Purpose |
| --- | --- |
| `ADDR_BASE_REG_1_Base` | Output base address |
| `ADDR_CTRL_XY_REG_1_Ystride` | Y dimension stride |
| `ADDR_CTRL_ZW_REG_1_Zstride` | Z dimension stride |
| `ADDR_CTRL_ZW_REG_1_Wstride` | W dimension stride |
| `ADD_DEST_ADDR_CNTR_add_dest_addr_cntr` | Combine with context-specific dest address |
| `FORCED_SHARED_EXP_shared_exp` | Fixed exponent when `REG2_Force_shared_exp=1` |
