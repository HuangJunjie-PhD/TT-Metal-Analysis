This page details the client-side architecture of the tt-awesome website. The application is built using vanilla JavaScript in `main.js`, managing state, filtering, and navigation without the use of external frameworks. It utilizes a three-pane layout and synchronization between the DOM and the URL to ensure a seamless user experience across desktop and mobile devices.

## Application State

The application state is managed via global variables in `src/assets/main.js`. These variables track the current view, active filters, and selected content.

| Variable | Type | Purpose |
| --- | --- | --- |
| `activeCategory` | `String` | The slug of the currently selected category (e.g., "ai-models"). [src/assets/main.js 5](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L5-L5) |
| `activeFilters` | `Set` | A set of active affiliation filters: "community", "affiliated", and "official". [src/assets/main.js 6](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L6-L6) |
| `activeEntryId` | `String` | The ID of the currently selected project entry. [src/assets/main.js 7](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L7-L7) |
| `activeAuthorFilter` | `String` | The current author being filtered for in the list view. [src/assets/main.js 8](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L8-L8) |
| `activeReleasesView` | `String` | The view mode for the releases pane (defaults to 'all'). [src/assets/main.js 9](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L9-L9) |

## URL Persistence & Routing

The application maintains state in the URL query parameters to support deep linking and browser history (back/forward buttons).

### Navigation Functions

*   **`pushUrl(params)`**: Updates the browser URL without a page reload. It constructs a `URLSearchParams` object from the provided `params` and uses `history.pushState`. [src/assets/main.js 1410-1425](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1410-L1425)
*   **`restoreFromUrl()`**: Invoked on initial load and `popstate` events. It parses the URL parameters (`cat`, `entry`, `author`, `releases`, `q`) and invokes the corresponding internal `_apply*` functions to restore the UI state. [src/assets/main.js 1427-1460](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1427-L1460)

## Filtering Logic

The core of the interactive list is the `applyFilters(query)` function. It executes a client-side filter across all entries based on the current global state and search input.

### `applyFilters()` Implementation

1.   **Input Handling**: It accepts an optional `query` string. If not provided, it attempts to read from the `#search` input. [src/assets/main.js 1255-1259](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1255-L1259)
2.   **Affiliation Filtering**: It iterates through all `.entry-row` elements and checks if their `data-affiliation` matches the values in the `activeFilters` set. [src/assets/main.js 1265-1268](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1265-L1268)
3.   **Category/Author Filtering**: It further filters based on whether an entry belongs to the `activeCategory` or matches the `activeAuthorFilter`. [src/assets/main.js 1270-1275](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1270-L1275)
4.   **Search Matching**: If a query exists, it performs a substring match against the entry's `data-search` attribute (which contains the name and description). [src/assets/main.js 1277-1279](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1277-L1279)
5.   **UI Updates**: It toggles the `hidden` class on rows and updates the `list-count` label. [src/assets/main.js 1282-1293](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1282-L1293)

## Mobile Viewport Transitions

The site transitions from a three-pane layout on desktop to a single-pane "stack" on mobile (max-width: 767px). This is managed by CSS classes on the `#panes` container.

### Mobile State Transitions

The application uses specific internal functions (prefixed with `_`) to manage these transitions:

*   **`_applyHome()`**: Shows the category grid (Home) and hides the list/detail panes. [src/assets/main.js 1302-1317](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1302-L1317)
*   **`_applyCategory(slug)`**: Switches to `list-active` mode. On mobile, it updates the topbar title to the category name. [src/assets/main.js 1319-1345](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1319-L1345)
*   **`_applyEntry(id, fromReleases)`**: Switches to `detail-active` mode. It stores the origin (list or releases) in `dataset.detailFrom` to enable the back button to return to the correct previous pane. [src/assets/main.js 1367-1400](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L1367-L1400)

### Mobile Helper Functions

*   **`mobilePaneBack()`**: The logic for the "←" button. It checks the current active class on `#panes` and either returns to the list or returns to the home view. [src/assets/main.js 81-111](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L81-L111)
*   **`updateMobileTopbar(title, showBack)`**: Controls the visibility of the logo versus the back button and title label in the mobile header. [src/assets/main.js 60-74](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L60-L74)

## Utility: Relative Time

The `relativeTime(iso)` utility converts ISO 8601 timestamps (provided by the GitHub metadata fetcher) into human-readable strings like "2 days ago" or "3 months ago".

### Implementation Detail

The function calculates the difference between `Date.now()` and the provided timestamp, then uses a series of thresholds to return the appropriate unit:

*   **Minutes**: `< 60 mins`
*   **Hours**: `< 24 hrs`
*   **Days**: `< 7 days`
*   **Weeks**: `< 5 weeks`
*   **Months**: `< 12 months`
*   **Years**: `> 12 months`[src/assets/main.js 127-143](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L127-L143)

On `DOMContentLoaded`, the script scans for all elements with a `data-ts` attribute and populates their text content using this utility. [src/assets/main.js 150-153](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/main.js#L150-L153)
