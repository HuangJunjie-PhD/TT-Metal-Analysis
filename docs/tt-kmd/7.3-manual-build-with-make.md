## Purpose and Scope

This document describes how to manually build and install the Tenstorrent kernel driver using `make` and the Linux kernel build system. This method provides direct control over the build process without relying on automated packaging systems. It also covers NixOS flake support and the CI mass-build testing infrastructure used to ensure compatibility across kernel versions.

For automated build and installation using distribution-specific tools, see [DKMS Configuration](https://deepwiki.com/tenstorrent/tt-kmd/7.1-dkms-configuration) and [AKMS Configuration](https://deepwiki.com/tenstorrent/tt-kmd/7.2-akms-configuration). For configuring module behavior after installation, see [Module Parameters](https://deepwiki.com/tenstorrent/tt-kmd/7.4-module-parameters).

## Prerequisites

Before building the driver manually, ensure the following packages are installed:

| Component | Debian/Ubuntu | Fedora/RHEL | Alpine |
| --- | --- | --- | --- |
| Kernel headers | `linux-headers-$(uname -r)` | `kernel-devel` | `linux-headers` |
| Build tools | `build-essential` | `gcc`, `make` | `build-base` |
| Module tools | `kmod` | `kmod` | `kmod` |

The build system requires a kernel build tree matching the target kernel version and a compiler compatible with the kernel (typically GCC or Clang). The driver is build-tested against mainline kernel versions from 5.4 through 6.18 [README.md 15](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/README.md?plain=1#L15-L15)

## Makefile Targets and Structure

### Module Definition

The module is defined using standard kbuild variables [Makefile 4-5](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L4-L5):

*   `obj-m += tenstorrent.o`: Specifies the final module name.
*   `tenstorrent-y`: Lists the object files that comprise the driver, including `module.o`, `chardev.o`, `enumerate.o`, `interrupt.o`, `wormhole.o`, `blackhole.o`, `msgqueue.o`, `pcie.o`, `sg_helpers.o`, `memory.o`, `tlb.o`, and `telemetry.o`.

### Key Targets

| Target | Command | Description |
| --- | --- | --- |
| `all` | `make` | Default target; builds the module by calling `modules`[Makefile 18](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L18-L18) |
| `modules` | `make modules` | Compiles the kernel module using the kernel build system [Makefile 20-21](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L20-L21) |
| `modules_install` | `make modules_install` | Installs the module to `/lib/modules/$(uname -r)/extra/`[Makefile 23-24](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L23-L24) |
| `clean` | `make clean` | Cleans build artifacts from the module directory [Makefile 26-27](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L26-L27) |
| `dkms` | `make dkms` | Adds and installs the module using DKMS [Makefile 35-38](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L35-L38) |
| `akms` | `make akms` | Installs the module using Alpine Kernel Module Support [Makefile 49-51](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L49-L51) |
| `dkms-remove` | `make dkms-remove` | Removes all installed versions of the tenstorrent module from DKMS [Makefile 40-47](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L40-L47) |

## Version Management

The version is tracked in multiple locations and must be synchronized:

1.   **`dkms.conf`**: The primary source for package versioning [VERSION_UPDATE.md 6-7](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/VERSION_UPDATE.md?plain=1#L6-L7)
2.   **`module.h`**: Defines `TENSTORRENT_DRIVER_VERSION_MAJOR`, `MINOR`, and `PATCH`[VERSION_UPDATE.md 11](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/VERSION_UPDATE.md?plain=1#L11-L11)
3.   **`tools/current-version`**: A script that extracts the version string from `module.h`[tools/current-version 1-28](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tools/current-version#L1-L28)

The `Makefile` uses the `tools/current-version` script to determine the `VERSION` variable for DKMS and display purposes [Makefile 13-14](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L13-L14)

## CI and Mass-Build Testing

The project maintains a rigorous testing pipeline to ensure the driver builds across a wide range of Linux kernels.

### Mass-Build Script (`test/mass-build-test`)

This Python script automates the process of building the driver against multiple kernel versions [test/mass-build-test 1-12](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/test/mass-build-test#L1-L12)

*   **Functionality**: It clones/uses a Linux kernel tree, checks out specific tags (e.g., `v5.4` to `v6.18`), prepares the kernel headers (`defconfig`, `modules_prepare`), and attempts to build the `tt-kmd` module [test/mass-build-test 33-44](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/test/mass-build-test#L33-L44)
*   **Version Range**: It can resolve all kernel versions between two git refs using `git tag`[test/mass-build-test 85-121](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/test/mass-build-test#L85-L121)

### GitHub Actions Workflows

*   **`test.yml`**: Performs standard builds using `make`, `dkms`, and Nix on Ubuntu runners [.github/workflows/test.yml 12-56](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/%20.github/workflows/test.yml#L12-L56)
*   **`mass-build-test.yml`**: Runs the `mass-build-test` script against the `gregkh/linux` repository for a version range from v5.4 to v6.18 [.github/workflows/mass-build-test.yml 9-17](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/%20.github/workflows/mass-build-test.yml#L9-L17)
