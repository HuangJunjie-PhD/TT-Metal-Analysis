This page documents the Sphinx configuration used in the Tenstorrent documentation system. It covers the structure, purpose, and key technical elements of the Sphinx configuration files that power the multi-project documentation site.

## Overview

Sphinx is the primary documentation engine for Tenstorrent, configured to handle a mix of reStructuredText and Markdown (via MyST) files. The system generates a versioned documentation site hosted at docs.tenstorrent.com.

The configuration architecture is decentralized, utilizing separate `conf.py` files for different sub-projects while sharing static assets and templates.

| Project | Config Path | Purpose |
| --- | --- | --- |
| **Home (Core)** | `core/conf.py` | Main landing page and global settings. |
| **TT-NN** | `ttnn/conf.py` | API documentation for the TT-NN library. |
| **TT-Metalium** | `tt-metalium/conf.py` | SDK documentation for low-level hardware programming. |

## Core Configuration (Home)

The `core/conf.py` file defines the root of the documentation site. It is responsible for global asset management and dynamic versioning for system-level components.

### Dynamic Version Substitution

The core configuration reads version strings directly from the `syseng` directory to populate `myst_substitutions`. This allows Markdown files to use variables like `{{ ver_kmd }}` which are replaced at build time.

`ver_kmd        = Path('../syseng/kmd.version').read_text().rstrip()ver_fw         = Path('../syseng/firmware.version').read_text().rstrip()ver_sys_tools  = Path('../syseng/sys_tools.version').read_text().rstrip() myst_substitutions = {    'ver_fw': ver_fw,    'ver_kmd': ver_kmd,    'ver_sys_tools': ver_sys_tools,}`
Sources: [core/conf.py 40-48](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/conf.py#L40-L48)

### Extension Suite

The following extensions are enabled in `core/conf.py`:

*   `myst_parser`: Enables Markdown support with `colon_fence` and `substitution`[core/conf.py 21-38](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/conf.py#L21-L38)
*   `sphinx_substitution_extensions`: Facilitates variable replacement in text.
*   `sphinx_copybutton`: Adds "copy" icons to code blocks, specifically targeting `div.highlight-bash pre`[core/conf.py 33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/conf.py#L33-L33)
*   `sphinx_togglebutton`: Allows collapsible sections.
*   `sphinx_sitemap`: Generates `sitemap_core.xml` for SEO [core/conf.py 25-31](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/conf.py#L25-L31)

## Project-Specific Configurations (TTNN & Metalium)

The `ttnn` and `tt-metalium` projects share a similar configuration pattern but focus on API generation.

### Version Context and Sidebar

Both projects load versioning data from a shared `../versions.yml` file. This data is passed into the `html_context`, which the `sphinx_rtd_theme` uses to render the version picker in the sidebar.

`with open("../versions.yml", "r") as yaml_file:    versions = yaml.safe_load(yaml_file)["ttnn"] # or "tt-metalium" html_context = {    "versions": list(versions["versions"].keys()),    "project_code": metal_sphinx_config.shortname,    "current_version": os.environ.get("current_version"),    "logo_link_url": os.environ.get("homepage")}`
Sources: [ttnn/conf.py 102-110](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/ttnn/conf.py#L102-L110)[tt-metalium/conf.py 96-104](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/tt-metalium/conf.py#L96-L104)

## Static Assets and Customization

Tenstorrent documentation uses a customized version of the Read the Docs theme (`sphinx_rtd_theme`).

### Custom CSS and JS

Static assets are shared across projects via the `../shared/_static` directory. The `setup(app)` function in each `conf.py` injects custom styling.
