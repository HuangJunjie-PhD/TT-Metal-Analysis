This page describes the automated backend processes that facilitate community contributions and provide a machine-readable API for the frontend. It covers the transformation of GitHub issue submissions into structured JSON entries and the subsequent aggregation of those entries into a unified `data.json` export.

## Data Flow: Issue to Export

The system uses a unidirectional pipeline where community input via GitHub issues is converted into persistent JSON files, which are then aggregated for the frontend.

## Issue-to-Entry Automation

The `scripts/issue_to_entry.py` script is the core of the automated contribution path. It parses GitHub issue form submissions, which are delivered as Markdown with specific `###` headings.

### Key Parsing Logic

*   **`parse_sections(body)`**: Splits the issue body based on `###` headings to create a dictionary of field labels and values [scripts/issue_to_entry.py 27-58](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L27-L58)
*   **`parse_checkboxes(text)`**: Extracts labels from Markdown checkboxes (e.g., `- [x] ai-models`) to identify selected categories and hardware [scripts/issue_to_entry.py 61-69](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L61-L69)
*   **`slugify(text)`**: Converts the project name into a kebab-case ID used for the filename (e.g., "TT Console" becomes `tt-console`) [scripts/issue_to_entry.py 77-84](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L77-L84)

### Validation and Field Mapping

The script enforces strict validation before writing any data:

1.   **Required Fields**: `Name`, `Description`, `Affiliation`, `Repository URL`, and at least one `Category` must be present [scripts/issue_to_entry.py 112-135](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L112-L135)
2.   **Affiliation Constraints**: Must be one of `community`, `affiliated`, or `official`[scripts/issue_to_entry.py 116-117](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L116-L117)
3.   **Link Parsing**: Handles the "Additional Links" field by splitting lines on the `|` pipe character to extract type, URL, and optional labels [scripts/issue_to_entry.py 151-166](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L151-L166)

### Workflow Integration

The automation is triggered by the `.github/workflows/entry-from-issue.yml` workflow.

*   **Trigger**: Fires when a maintainer adds the `approved` label to an issue that already has the `submission` label [entry-from-issue.yml 18-20](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entry-from-issue.yml#L18-L20)
*   **Output**: The script writes the entry to `entries/{primary_category}/{id}.json`[scripts/issue_to_entry.py 214-220](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/issue_to_entry.py#L214-L220)
*   **PR Creation**: The workflow then creates a new branch, commits the file, and opens a Pull Request automatically [entry-from-issue.yml 50-79](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entry-from-issue.yml#L50-L79)

## Data JSON Export

The `scripts/generate_data_json.py` script produces `data.json`, a machine-readable API export located at the repository root. This file is the primary data source for the client-side filtering and search logic in the frontend.

### Export Structure

The export is organized by category to allow efficient access by the frontend:

| Key | Description |
| --- | --- |
| `meta.generated` | ISO-8601 UTC timestamp of the build [scripts/generate_data_json.py 77](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L77-L77) |
| `meta.total_entries` | Total count of non-hidden projects [scripts/generate_data_json.py 79](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L79-L79) |
| `categories` | A map where keys are category slugs and values contain the category label, entry count, and the list of entry objects [scripts/generate_data_json.py 63-72](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L63-L72) |

### Sorting Algorithm

To maintain consistency with the generated README, `generate_data_json.py` implements the **Community-First** sorting logic:

1.   **Affiliation Tier**: `community` (0) >`affiliated` (1) >`official` (2) [scripts/generate_data_json.py 41](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L41-L41)
2.   **Featured Status**: Featured entries within a tier are promoted [scripts/generate_data_json.py 54](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L54-L54)
3.   **Popularity**: Within the same tier and featured status, entries are sorted by GitHub star count descending [scripts/generate_data_json.py 55](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L55-L55)

## Comparison of Export Formats

While both `generate_readme.py` and `generate_data_json.py` process the same `entries/*.json` files, they serve different consumers:

| Feature | README.md (Markdown) | data.json (JSON API) |
| --- | --- | --- |
| **Consumer** | GitHub UI / Human Readers | Frontend JS (`main.js`) |
| **Logic** | `render_entry()` formatting [scripts/generate_readme.py 108](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L108-L108) | Raw data serialization [scripts/generate_data_json.py 91](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L91-L91) |
| **Icons** | `LINK_ICONS` emojis [scripts/generate_readme.py 34](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L34-L34) | Not included (handled by UI) |
| **Hidden Entries** | Excluded [scripts/generate_readme.py 77](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L77-L77) | Excluded [scripts/generate_data_json.py 47](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L47-L47) |
