## Purpose and Scope

This document details the LLK Basic Implementation, a core component of the LLK (Low-Level Kernel) System within the Tensix ISA Simulator. The basic implementation provides a fundamental, streamlined version of the LLK functionality, serving as an accessible entry point for understanding the LLK architecture.

For information about other LLK implementations, see [LLK Reference Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.3-llk-reference-implementation) and [LLK Compute Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.4-llk-compute-implementation). For details on the LLK API that this implementation conforms to, see [LLK API](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.1-llk-api).

## Build Process

The LLK Basic Implementation is built using a dedicated build script that compiles all C++ files in the `src/llk/basic/` directory into object files, which are then archived into a static library.

The build process is straightforward:

1.   Compile all source files in `src/llk/basic/` with C++17 support and optimization level O3
2.   Create a static library `llk_basic.a` from the resulting object files
3.   Clean up intermediate object files

Build command:

```
g++ -c -std=c++17 -O3 -I $SRC $SRC/llk/basic/*.cpp
ar rsc $LIB/llk_basic.a *.o
```

Sources: [tensix/whb0/prj/build_llk_basic.sh 9-15](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_llk_basic.sh#L9-L15)

## Demo Application

A demonstration application is provided to showcase the functionality of the LLK Basic Implementation. This demo is built using another dedicated script and depends on the LLK Basic library, the Core library, and the Schedule library.

### Demo Build Process

The demo application is built with:

```
g++ -o $BIN/demo_llk_basic -std=c++17 -O3 \
    -I $SRC \
    $SRC/demo/llk_basic/*.cpp \
    $LIB/llk_basic.a \
    $LIB/core.a \
    $LIB/schedule.a
```

Sources: [tensix/whb0/prj/build_demo_llk_basic.sh 12-17](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_basic.sh#L12-L17)

## Usage Guidelines

The LLK Basic Implementation can be used by:

1.   Including the appropriate headers from the LLK Basic implementation
2.   Linking against the `llk_basic.a` library
3.   Following the LLK API interfaces as specified in [LLK API](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.1-llk-api)

A typical usage pattern would follow the demo application structure, which includes the necessary headers and links against the required libraries.

## Summary

The LLK Basic Implementation provides a fundamental, streamlined version of the LLK functionality within the Tensix ISA Simulator. It is built as a static library (`llk_basic.a`) and depends on the Core library. The implementation serves as an accessible entry point for understanding the LLK system and can be demonstrated using the provided demo application.

As a basic implementation, it prioritizes simplicity and clarity over advanced features or optimizations, which are available in the [LLK Reference Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.3-llk-reference-implementation) and [LLK Compute Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.4-llk-compute-implementation).
