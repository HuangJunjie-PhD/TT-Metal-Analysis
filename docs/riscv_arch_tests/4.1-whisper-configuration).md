This document describes the Whisper Instruction Set Simulator (ISS) configuration files used in the RISC-V architectural test framework. These JSON configuration files define the processor architecture, enabled features, and register behavior for Whisper during test execution.

For information about Spike ISS integration, see [Spike Integration](https://deepwiki.com/tenstorrent/riscv_arch_tests/4.2-spike-integration). For details about test execution infrastructure, see [Test Execution](https://deepwiki.com/tenstorrent/riscv_arch_tests/3.1-test-execution).

## Configuration File Overview

The repository contains three primary Whisper configuration files that define different processor configurations for testing various RISC-V features and vector length combinations:

| Configuration File | Vector Length | Primary Focus | ISA Extensions |
| --- | --- | --- | --- |
| `whisper_config.json` | 128-bit | Full feature set | Comprehensive extension set including Crypto, Vector, BitManip |
| `whisper_config_vlen_128.json` | 128-bit | Basic vector support | Reduced extension set for compatibility |
| `whisper_config_vlen_256.json` | 256-bit | Extended vector support | Intermediate extension set with vector focus |

## Core Architecture Configuration

### Architecture Parameters

All configuration files define the base architecture parameters:

| Parameter | Value | Description |
| --- | --- | --- |
| `xlen` | 64 | 64-bit RISC-V architecture |
| `enable_performance_counters` | true | Enable hardware performance counters |
| `perf_count_atomic_load_store` | true | Count atomic memory operations |
| `perf_count_fp_load_store` | true | Count floating-point memory operations |
| `print_load_store_address` | true | Print memory addresses in trace logs |
| `num_mmode_perf_regs` | 2 | Two performance counters implemented (mhpmcounter3 and 4) |

## Vector Configuration

### Vector Architecture Parameters

Each configuration defines different vector processing capabilities:

| Configuration | Vector Length | Element Size | Configuration Details |
| --- | --- | --- | --- |
| `whisper_config.json` | 128-bit | 1-8 bytes | `bytes_per_vec: 16` |
| `whisper_config_vlen_128.json` | 128-bit | 1-8 bytes | `bytes_per_vec: 16` |
| `whisper_config_vlen_256.json` | 256-bit | 1-8 bytes | `bytes_per_vec: 32` |

All configurations support:

*   `min_bytes_per_elem: 1` (8-bit elements)
*   `max_bytes_per_elem: 8` (64-bit elements)
*   Hardware supports 32-bit ELEN with widening operations to 64-bit

## Disabled Features

### Consistently Disabled Components

All configurations disable the same set of features for testing consistency:

*   **PMP Registers**: All Physical Memory Protection addresses (`pmpaddr0` through `pmpaddr63`) are disabled
*   **Counter CSRs**: `cycle`, `instret`, and `time` counters are disabled
*   **Debug CSRs**: All trigger and debug CSRs (`tselect`, `tdata1-3`, `dcsr`, `dpc`, `dscratch*`) are hardwired to zero
*   **Misaligned Data Access**: `enabled_misaligned_data` is set to `false`
