## Purpose and Scope

This document describes how JAX integrates with TT-XLA through the PJRT plugin interface. It covers the `jax_plugin_tt` wrapper module, device discovery, JIT compilation flow, and the path from JAX programs to StableHLO graph generation for TT-MLIR compilation.

## Integration Architecture

The JAX backend integrates with TT-XLA through a thin Python wrapper (`jax_plugin_tt`) that registers the PJRT plugin with JAX's plugin discovery mechanism. This enables JAX to use Tenstorrent hardware as a backend device for JIT-compiled computations.

**JAX Backend Integration Architecture**

The integration consists of three main layers:

1.   **JAX Framework Layer**: JAX's native JIT compilation and XLA interface
2.   **Plugin Wrapper Layer**: `jax_plugin_tt` module that registers the PJRT plugin
3.   **PJRT Implementation Layer**: `pjrt_plugin_tt.so` providing device management and compilation

```
### Compilation Stages

1.   **JAX Tracing**: JAX traces the Python function to extract computational graph
2.   **HLO Generation**: JAX converts traced operations to HLO (High-Level Operations) IR
3.   **PJRT Compilation**: PJRT plugin receives HLO and generates StableHLO
4.   **TT-MLIR Compilation**: StableHLO is lowered through TTIR → TTNN → device code
5.   **Execution**: Compiled binary is executed on TT-Metal runtime

### Caching and Recompilation

JAX automatically caches compiled functions based on input shapes and types. Recompilation occurs when:

*   Input shapes change
*   Input dtypes change
*   Different devices are targeted

**StableHLO Graph Structure and Conversion**
```
### Graph Properties

*   **Operations**: Functional operations from the StableHLO specification
*   **Types**: Strongly-typed tensors with shape and dtype information
*   **Control Flow**: Structured control flow constructs (while loops, conditionals)
*   **Metadata**: Additional information for optimization and layout decisions

The StableHLO graph serves as a stable, versioned interface between JAX and TT-MLIR, ensuring compatibility across updates to either component.

## Limitations and Known Issues

Current limitations of the JAX backend:

1.   **Dynamic Shapes**: Limited support for dynamic shapes; prefer static shapes where possible
2.   **Control Flow**: Some advanced control flow constructs may require manual decomposition
3.   **Custom Operations**: Custom JAX primitives require explicit lowering rules
4.   **Multi-Device**: Multi-device JAX features (pmap, shard_map) have specific requirements

For detailed status of model and operation support, see the test configuration system documented in [Test Configuration System](https://deepwiki.com/tenstorrent/tt-xla/6.1-test-configuration-system).
