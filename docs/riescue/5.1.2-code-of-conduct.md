This page documents the community standards and behavioral expectations for all contributors and participants in the RiESCUE project. The Code of Conduct establishes guidelines for professional interaction and defines the enforcement process for violations.

For information about technical contribution standards (code quality, testing requirements, etc.), see [Contributing Guidelines](https://deepwiki.com/tenstorrent/riescue/5.1-contributing-guidelines). For information about issue and PR submission procedures, see [Issue and PR Templates](https://deepwiki.com/tenstorrent/riescue/5.1.1-issue-and-pr-templates).

## Overview

The RiESCUE project adopts the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/) version 2.0, which is stored in [.github/CODE_OF_CONDUCT.md 1-128](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L1-L128) This standard code of conduct is widely used across open source projects and provides clear behavioral expectations and enforcement procedures.

The Code of Conduct is referenced by the contributing guidelines at [.github/CONTRIBUTING.md 6](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CONTRIBUTING.md?plain=1#L6-L6) which instructs all contributors to follow it before making any contributions.

## Pledge

The community pledges to provide a harassment-free experience for all participants, regardless of:

*   Age
*   Body size
*   Visible or invisible disability
*   Ethnicity
*   Sex characteristics
*   Gender identity and expression
*   Level of experience
*   Education
*   Socio-economic status
*   Nationality
*   Personal appearance
*   Race
*   Religion
*   Sexual identity and orientation

The pledge commits the community to act in ways that contribute to an open, welcoming, diverse, inclusive, and healthy environment.

**Sources:**[.github/CODE_OF_CONDUCT.md 3-13](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L3-L13)

## Standards

### Acceptable Behavior

The following behaviors contribute to a positive environment:

| Behavior | Description |
| --- | --- |
| **Empathy and Kindness** | Demonstrating understanding toward other people |
| **Respectfulness** | Being respectful of differing opinions, viewpoints, and experiences |
| **Constructive Feedback** | Giving and gracefully accepting constructive criticism |
| **Accountability** | Accepting responsibility, apologizing for mistakes, and learning from them |
| **Community Focus** | Prioritizing what is best for the overall community, not just individuals |

### Unacceptable Behavior

The following behaviors are explicitly prohibited:

| Behavior | Description |
| --- | --- |
| **Sexualized Content** | Use of sexualized language/imagery or unwanted sexual attention/advances |
| **Trolling** | Insulting or derogatory comments, personal or political attacks |
| **Harassment** | Public or private harassment of any kind |
| **Privacy Violations** | Publishing others' private information without explicit permission |
| **Professional Misconduct** | Other conduct reasonably considered inappropriate in a professional setting |

**Sources:**[.github/CODE_OF_CONDUCT.md 15-37](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L15-L37)

## Scope

The Code of Conduct applies in the following contexts:

1.   **All Community Spaces**: GitHub repository, issues, pull requests, discussions
2.   **Official Representation**: When representing the community in public spaces
3.   **Official Communications**: Using official email addresses or social media accounts
4.   **Events**: Acting as a representative at online or offline events

**Sources:**[.github/CODE_OF_CONDUCT.md 51-57](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L51-L57)

## Enforcement Guidelines

Community leaders follow a graduated response system with four levels of enforcement:

### Level 1: Correction

**Trigger**: Use of inappropriate language or unprofessional behavior

**Response**:

*   Private written warning from community leaders
*   Explanation of why the behavior was inappropriate
*   Possible request for public apology

**Sources:**[.github/CODE_OF_CONDUCT.md 74-81](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L74-L81)

### Level 2: Warning

**Trigger**: Single incident or series of violations

**Response**:

*   Warning with consequences for continued behavior
*   No interaction with involved parties for a specified period
*   Applies to community spaces and external channels (e.g., social media)
*   Violation of terms may lead to temporary or permanent ban

**Sources:**[.github/CODE_OF_CONDUCT.md 83-93](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L83-L93)

### Level 3: Temporary Ban

**Trigger**: Serious violation of community standards, including sustained inappropriate behavior

**Response**:

*   Temporary ban from all community interaction and communication
*   No public or private interaction with involved parties
*   No unsolicited interaction with enforcers
*   Violation of terms may lead to permanent ban

**Sources:**[.github/CODE_OF_CONDUCT.md 95-104](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L95-L104)

### Level 4: Permanent Ban

**Trigger**:

*   Pattern of community standard violations
*   Sustained inappropriate behavior
*   Harassment of individuals
*   Aggression toward or disparagement of classes of individuals

**Response**:

*   Permanent ban from all public interaction within the community

**Sources:**[.github/CODE_OF_CONDUCT.md 106-113](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CODE_OF_CONDUCT.md?plain=1#L106-L113)

## Summary

The Code of Conduct is a critical component of the RiESCUE development workflow that:

1.   **Sets Expectations**: Establishes clear behavioral standards for all community interactions
2.   **Protects Participants**: Creates a safe, inclusive environment for diverse contributors
3.   **Provides Accountability**: Defines enforcement mechanisms and consequences for violations
4.   **Integrates with Process**: Works alongside technical contribution standards defined in other sections

All contributors must acknowledge and follow the Code of Conduct before participating in any project activities, including filing issues, submitting pull requests, or engaging in discussions.
