The `server_tests` framework provides a comprehensive integration testing suite for the Tenstorrent inference server. It validates the full stack—from API endpoints down to hardware execution—across various model modalities (LLM, VLM, Image, Video, Audio, CNN, Embedding) and hardware configurations.

## Architecture and Test Expansion

The framework uses a data-driven approach where test suites are defined in JSON files and expanded into a full test matrix based on hardware and model compatibility.

### Configuration Hierarchy

1.   **`server_tests_config.json`**: The root configuration defining model categories, available pytest markers, and hardware-specific defaults [server_tests/server_tests_config.json 1-112](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/server_tests_config.json#L1-L112)
2.   **`test_suites/*.json`**: Category-specific files (e.g., `video.json`, `cnn.json`) that define `test_matrices`. These matrices map models to compatible devices and specify which test templates to execute [server_tests/test_suites/video.json 4-15](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/test_suites/video.json#L4-L15)[server_tests/test_suites/cnn.json 4-18](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/test_suites/cnn.json#L4-L18)
3.   **`TestFilter` Class**: Responsible for loading these JSON files, merging template defaults, and auto-deriving pytest markers based on the model category and hardware target [server_tests/test_categorization_system/test_filter.py 38-53](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/test_categorization_system/test_filter.py#L38-L53)

## Key Components

### 1. `run_tests.py` (Test Orchestrator)

The main entry point for executing integration tests. It resolves the `ModelSpec`, sets up environment variables (like `OPENAI_API_KEY` via JWT secrets), and templates the `pytest` command [server_tests/run_tests.py 126-160](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/run_tests.py#L126-L160)

*   **`build_test_command`**: Constructs the CLI arguments for `pytest`, including the model name, implementation, and output paths [server_tests/run_tests.py 34-75](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/run_tests.py#L34-L75)
*   **Result Handling**: It creates a timestamped output directory for reports and returns the status of all executed tasks [server_tests/run_tests.py 174-209](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/run_tests.py#L174-L209)

### 2. `conftest.py` (Fixtures and Reporting)

Provides global fixtures and hooks for the integration test suite.

*   **`api_client`**: A session-scoped fixture that returns a helper function for making authenticated API requests to the server [server_tests/conftest.py 99-138](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/conftest.py#L99-L138)
*   **`results_report`**: Manages a dictionary that collects test outcomes, start/end timestamps, and error tracebacks, eventually writing them to a JSON report [server_tests/conftest.py 79-96](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/conftest.py#L79-L96)
*   **`pytest_runtest_makereport`**: A hook that captures the result of the "call" phase of each test for reporting purposes [server_tests/conftest.py 141-155](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/conftest.py#L141-L155)

### 3. Test Categories

Tests are generally split into three functional templates:

| Category | Description | Implementation Example |
| --- | --- | --- |
| **Load** | Concurrent request testing to verify throughput and target latencies. | `VideoGenerationLoadTest`[server_tests/test_cases/video_generation_load_test.py 49-122](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/test_cases/video_generation_load_test.py#L49-L122) |
| **Param** | Validation of API parameters (e.g., `num_inference_steps`, `seed`, `negative_prompt`). | `CnnParamTest`[server_tests/test_suites/cnn.json 32-36](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/test_suites/cnn.json#L32-L36) |
| **Eval** | Accuracy or specific feature evaluation (e.g., FID/CLIP scoring). | `VideoGenerationEvalsTest`[server_tests/test_cases/video_generation_eval_test.py 49-122](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/test_cases/video_generation_eval_test.py#L49-L122) |

## Next-Generation Media Test Framework (v2)

The `tt-inference-server-v2/test_module` introduces a refined framework for media-specific testing, emphasizing modularity and automated accuracy measurement.

### MediaContext and Dispatch

The framework uses a `MediaContext` to manage the environment and server state during tests [tt-inference-server-v2/test_module/context.py](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/context.py) The `dispatch.py` module routes requests to the appropriate test implementation based on the model category [tt-inference-server-v2/test_module/dispatch.py](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/dispatch.py)

### Accuracy Evaluations (eval_tests)

The `eval_tests` sub-module implements rigorous accuracy checks for different modalities:

*   **`VisionEvalsTest`**: Downloads ImageNet subsets and compares CPU vs. Device classification accuracy [tt-inference-server-v2/test_module/eval_tests/vision_evals_test.py 57-132](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/eval_tests/vision_evals_test.py#L57-L132) It utilizes a `_normalize_label` function to ensure consistency [tt-inference-server-v2/test_module/eval_tests/vision_evals_test.py 189-206](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/eval_tests/vision_evals_test.py#L189-L206)
*   **`ImageGenerationEvalsTest`**: Calculates FID and CLIP scores for generated images, comparing them against references in `model_accuracy_reference.json`[tt-inference-server-v2/test_module/eval_tests/image_generation_eval_test.py 101-201](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/eval_tests/image_generation_eval_test.py#L101-L201)
*   **`VideoGenerationEvalsTest`**: Extends accuracy testing to video by sampling frames and calculating CLIP scores against COCO dataset captions [tt-inference-server-v2/test_module/eval_tests/video_generation_eval_test.py 49-122](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/eval_tests/video_generation_eval_test.py#L49-L122)

### Load and Parameter Testing (load_param_tests)

These tests focus on the stability and parameter handling of the media server:

*   **`server_helper.py`**: Provides utilities for spawning the media server (`_launch_server`) and waiting for readiness by monitoring logs for `READY_LOG_TEXT`[tt-inference-server-v2/test_module/load_param_tests/server_helper.py 24-85](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/load_param_tests/server_helper.py#L24-L85)
*   **`BaseTest`**: A base class that manages test retries, timeouts, and server health checks [tt-inference-server-v2/test_module/_test_common/base_test.py 34-146](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/_test_common/base_test.py#L34-L146) It implements `wait_for_server_ready` using the `/tt-liveness` endpoint [tt-inference-server-v2/test_module/_test_common/base_test.py 151-197](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-inference-server-v2/test_module/_test_common/base_test.py#L151-L197)

## CI Configuration and Device Mapping

The `models-ci-config.json` file defines the source of truth for which models are tested on which hardware during different CI cycles (`nightly`, `weekly`, `release`).

*   **Supported Devices**: `N150`, `N300`, `T3K`, `GALAXY`, `P150X8`, etc.
*   **Inference Engines**: Models are mapped to `vLLM`, `MEDIA` (Python runner), or `FORGE` engines.
