The IR Builder provides a programmatic Python API for constructing TensorIR (TIR) programs using context managers and builder patterns. It allows developers to build TIR without writing TVMScript syntax, making it useful for automated code generation, transformations, and testing.

For information about the TVMScript DSL syntax that provides a declarative alternative to IR Builder, see [TVMScript DSL](https://deepwiki.com/tenstorrent/tt-tvm/5.6-tvmscript-dsl). For information about the Schedule API that transforms existing TIR programs, see [Schedule API](https://deepwiki.com/tenstorrent/tt-tvm/5.3-schedule-api).

## Purpose and Architecture

The IR Builder system enables imperative construction of TIR programs through a frame-based architecture. Rather than parsing Python decorators and syntax (as TVMScript does), IR Builder directly constructs TIR nodes through function calls within context managers.

**Key Design Principles:**

| Principle | Description |
| --- | --- |
| Frame-Based | Each IR construct (function, block, loop) has a corresponding frame that manages state |
| Context Managers | Python `with` statements automatically handle frame entry/exit |
| Stack Management | IRBuilder maintains a stack of active frames for hierarchical construction |
| Type Safety | Builder functions perform validation and type checking |

## Core Components

### IRBuilder and Frame Stack

The `IRBuilder` class manages the frame stack that tracks the current construction context. Each frame represents a scope in the IR being built.

**Frame Lifecycle:**

1.   `__enter__()`: Frame is pushed onto the IRBuilder stack
2.   Builder functions add statements/expressions to the current frame
3.   `__exit__()`: Frame pops from stack, constructs the IR node, adds to parent frame

## Programmatic Construction Example

**Complete Matrix Multiplication using IR Builder:**

`from tvm.script.ir_builder import IRBuilderfrom tvm.script.ir_builder import tir as T with IRBuilder() as ib:    with T.prim_func():        # Function parameters        A_handle = T.arg("A", T.handle())        B_handle = T.arg("B", T.handle())         C_handle = T.arg("C", T.handle())                # Match buffers        A = T.match_buffer(A_handle, (128, 128), "float32")        B = T.match_buffer(B_handle, (128, 128), "float32")        C = T.match_buffer(C_handle, (128, 128), "float32")                # Function metadata        T.func_name("matmul")        T.func_attr({"tir.noalias": True})                # Loop nest        for i, j in T.grid(128, 128):            with T.block("init"):                vi = T.axis.spatial(128, i)                vj = T.axis.spatial(128, j)                T.reads()                T.writes(C[vi, vj])                C[vi, vj] = T.float32(0)                        for k in T.serial(128):                with T.block("update"):                    vi = T.axis.spatial(128, i)                    vj = T.axis.spatial(128, j)                    vk = T.axis.reduce(128, k)                    T.reads(C[vi, vj], A[vi, vk], B[vk, vj])                    T.writes(C[vi, vj])                    C[vi, vj] = C[vi, vj] + A[vi, vk] * B[vk, vj] # Extract the constructed PrimFuncprim_func = ib.get()`

## Comparison with TVMScript

**IR Builder vs TVMScript DSL:**

| Aspect | IR Builder | TVMScript |
| --- | --- | --- |
| **Construction Style** | Imperative (function calls) | Declarative (decorated syntax) |
| **Use Case** | Programmatic generation, testing | Hand-written kernels, readability |
| **Entry Point** | `IRBuilder()` context | `@T.prim_func` decorator |
| **Syntax** | `with T.block():` | `with T.block():` (same context managers) |
| **Round-trip** | Not applicable | Parses Python → prints back Python |
| **Type Checking** | Runtime (builder functions) | Parse-time (AST analysis) |
| **Flexibility** | Can use Python loops/conditionals to generate IR | Limited to Python syntax expressible in AST |

**Shared Components:**

Both IR Builder and TVMScript use the same underlying:

*   Frame classes (`PrimFuncFrame`, `BlockFrame`, etc.)
*   Builder functions (`T.axis.spatial()`, `T.reads()`, etc.)
*   TIR node construction logic

The key difference is that TVMScript parses Python AST and calls builder functions automatically, while IR Builder requires explicit function calls.

## Common Patterns and Best Practices

**Pattern 1: Conditional IR Generation**

`# Use Python control flow to conditionally build IRwith T.prim_func():    for i in T.grid(128):        if some_condition:            # Generate different IR based on condition            with T.block("variant_a"):                ...        else:            with T.block("variant_b"):                ...`
**Pattern 2: Parameterized Loop Nests**

`def build_matmul(M, N, K):    """Generate matmul for arbitrary sizes"""    with IRBuilder() as ib:        with T.prim_func():            # Use parameters for dimensions            for i, j in T.grid(M, N):                with T.block("init"):                    vi = T.axis.spatial(M, i)                    vj = T.axis.spatial(N, j)                    ...    return ib.get()`
**Pattern 3: Buffer Region Computation**

`# Compute buffer regions for match_bufferwith T.block():    # Match a subregion of a buffer    start_i = i * tile_size    end_i = (i + 1) * tile_size    sub_A = T.match_buffer(A[start_i:end_i, :], (tile_size, 128))`
**Best Practices:**

1.   **Use Frame Context Managers**: Always use `with` statements for frames to ensure proper cleanup
2.   **Set Block I/O Early**: Define `T.reads()` and `T.writes()` before block body for better analysis
3.   **Name Variables Clearly**: Use descriptive names for axes (`vi`, `vj`, `vk` for matmul)
4.   **Validate Frame Context**: Builder functions check they're called in the right frame type
5.   **Handle Errors**: Catch `TVMError` exceptions for invalid IR construction

## Error Handling and Validation

The IR Builder performs extensive validation during construction:

**Common Error Scenarios:**

| Error Type | Cause | Detection |
| --- | --- | --- |
| **Frame Context Error** | Calling function in wrong frame type | `FindPrimFuncFrame()`, `FindBlockFrame()` |
| **Duplicate Declaration** | Declaring same buffer/axis twice | Frame state checks on `reads`, `writes`, etc. |
| **Type Mismatch** | Incompatible buffer types | `BinaryOpMatchTypes()`, type inference |
| **Invalid I/O Region** | Buffer region doesn't match buffer shape | `BufferRegionFromLoad()` validation |
| **Missing Required Field** | Block without axes, function without name | Frame exit validation |

**Error Reporting:**

`# Example error when calling function in wrong contextwith T.prim_func():    with T.block("compute"):        T.func_name("invalid")  # ERROR: func_name must be at PrimFunc level        # Raises: ValueError: T.func_name must be called at the top of a PrimFunc`
