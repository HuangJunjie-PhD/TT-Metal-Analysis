The navigation system dynamically generates navigation items based on application state:

| Item Type | Purpose | Condition | Examples |
| --- | --- | --- | --- |
| `baseNavItems` | Core navigation always visible | Always present | Home, RAG Management |
| `deployedNavItems` | TT-Studio mode navigation | `!isDeployedEnabled` | Models Deployed, Logs |
| `modelNavItems` | AI Playground mode navigation | `isDeployedEnabled` | Chat UI, Image Generation |
| `actionButtons` | Utility functions | Context-dependent | Dark Mode Toggle, Reset Board |

## Layout Patterns

### MainLayout and Footer Integration

The `MainLayout` component establishes the top-level structure of the application, wrapping children with the `NavBar` and `Footer`. It manages vertical spacing using a CSS variable `--footer-height` to prevent content overlap with the sticky footer.

*   **MainLayout Structure**: Wraps the application in a grid-patterned background and provides padding for navigation [app/frontend/src/layouts/MainLayout.tsx 7-25](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/layouts/MainLayout.tsx#L7-L25)
*   **Dynamic Footer Height**: The `Footer` component measures its own height and updates the `--footer-height` CSS variable on the document root [app/frontend/src/components/Footer.tsx 151-170](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/Footer.tsx#L151-L170)

### Vertical Navigation Layout

Used primarily for Chat UI and Image Generation interfaces, providing a compact sidebar navigation:

*   **Location**: Fixed left sidebar, 64px width (`w-16`) [app/frontend/src/components/NavBar.tsx 563-565](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/NavBar.tsx#L563-L565)

### Horizontal Navigation Layout

## Layout Providers and Contexts

TT-Studio uses several providers to manage the visibility and state of layout elements:

### FooterVisibilityProvider

Manages the visibility state of the global footer. It persists the user's preference in local storage and defaults visibility based on the current path (e.g., hidden by default in Chat UI) [app/frontend/src/providers/FooterVisibilityContext.tsx 11-20](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/providers/FooterVisibilityContext.tsx#L11-L20)

### HeroSectionProvider

Manages the toggle state for the "Hero Section" (promotional/introductory banners) on the home page. This state is shared with the `NavBar` via the `HeroSectionToggleMenuItem` to allow users to toggle the section visibility [app/frontend/src/providers/HeroSectionContext.tsx 10-22](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/providers/HeroSectionContext.tsx#L10-L22)

### Footer Resource Monitoring

The `Footer` component is more than a layout element; it acts as a system monitor, fetching real-time data:

*   **System Resources**: Fetches CPU and memory usage from `/board-api/footer-data/`[app/frontend/src/components/Footer.tsx 82-107](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/Footer.tsx#L82-L107)
*   **Hardware State**: Displays board name, temperature, and hardware health via `useDeviceState`[app/frontend/src/components/Footer.tsx 185-197](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/Footer.tsx#L185-L197)

## Integration Points

### Theme Integration

Navigation and Layout components integrate with the theme system through the `useTheme()` hook:

| Component | Usage | Light Theme | Dark Theme |
| --- | --- | --- | --- |
| `NavBar` | Icon/Text colors | `text-black` | `text-zinc-200` |
| `Footer` | Background/Borders | `bg-white/95` | `bg-zinc-900/95` |
| `Sidebar` | Help Card colors | `bg-white` | `bg-TT-slate` |
| `MainLayout` | Grid pattern | `bg-grid-black/[0.2]` | `bg-grid-white/[0.2]` |
