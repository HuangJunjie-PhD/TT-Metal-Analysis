## Purpose and Scope

This document describes the r37 Compute Components, which consist of the Compute Kernel (CKernel) and Compute Library. These components provide computational functionality for the r37 architecture, building upon the r37 Instruction Set Simulator infrastructure. The Compute Kernel provides the core computational primitives, while the Compute Library offers higher-level computational operations implemented using these primitives.

For information about the r37 Instruction Set Simulator, see [r37 Instruction Set Simulator](https://deepwiki.com/tenstorrent/tensix-isa-simulator/4.1-r37-instruction-set-simulator). For details about the r37 LLK implementation, refer to [r37 LLK Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/4.2-r37-llk-implementation).

## r37 Compute Kernel

The Compute Kernel (CKernel) provides the fundamental computational building blocks for the r37 architecture. It serves as an abstraction layer over the instruction set simulator, offering primitives that can be used to implement more complex computational operations.

### Architecture

**Diagram: r37 Compute Kernel Components**

Sources: [tensix/whb0/prj/build_r37_ckernel.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_ckernel.sh)

### Key Features

The Compute Kernel consists of:

1.   **Common Include Files**: Located in the `/r37/common/inc` directory, these files define the interfaces and shared functionality used by the compute components.

2.   **Core Computational Primitives**: These primitives provide direct abstractions over the r37 instruction set, offering operations such as:

    *   Vector arithmetic operations
    *   Matrix operations
    *   Data transformation functions
    *   Memory management utilities

3.   **Utilities**: Helper functions that simplify the implementation of common operations and patterns.

### Build Process

The Compute Kernel is built into a static library (`r37_ckernel.a`) by compiling C++ source files from the `/r37/common/inc` directory. The build process includes dependencies from various r37 architectural components.

Sources: [tensix/whb0/prj/build_r37_ckernel.sh 9-20](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_ckernel.sh#L9-L20)

## r37 Compute Library

The Compute Library builds upon the Compute Kernel to provide higher-level computational operations. It offers a more user-friendly interface for common computational tasks and implements algorithms that utilize the primitives provided by the Compute Kernel.

### Architecture

**Diagram: r37 Compute Library Components**

Sources: [tensix/whb0/prj/build_r37_compute.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_compute.sh)

### Key Features

The Compute Library provides:

1.   **Compute Algorithms**: Implementation of common computational algorithms optimized for the r37 architecture.

2.   **High-Level Operations**: User-friendly interfaces for complex operations that internally utilize Compute Kernel primitives.

3.   **Optimized Implementations**: Performance-optimized implementations of frequently used computational patterns.

### Build Process

The Compute Library is built into a static library (`r37_compute.a`) by compiling C++ source files from the `/r37/compute` directory. The build process includes dependencies from various r37 architectural components, including the Compute Kernel.

Sources: [tensix/whb0/prj/build_r37_compute.sh 9-21](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_compute.sh#L9-L21)

## Usage Patterns

Typical usage of the r37 Compute Components follows these patterns:

1.   **Direct Compute Kernel Usage**: For specific, performance-critical operations, applications may directly use the Compute Kernel primitives.

2.   **Compute Library Usage**: For most general computational needs, applications should use the higher-level abstractions provided by the Compute Library.

3.   **Combined Usage**: Complex applications typically use a combination of both components, using the Compute Library for standard operations and the Compute Kernel for specialized requirements.

## Summary

The r37 Compute Components provide a comprehensive computational framework for the r37 architecture:

*   The **Compute Kernel** offers low-level computational primitives that abstract the instruction set.
*   The **Compute Library** builds upon these primitives to provide higher-level computational operations.

Together, these components enable efficient implementation of computational tasks on the r37 architecture, while providing different levels of abstraction to suit various application needs.
