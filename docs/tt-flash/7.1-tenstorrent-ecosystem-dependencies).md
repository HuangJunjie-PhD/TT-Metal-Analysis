## Purpose and Scope

This page documents the Tenstorrent-specific dependencies that integrate `tt-flash` with the broader Tenstorrent hardware ecosystem. These dependencies provide hardware abstraction, device communication, and shared utilities across Tenstorrent tools. For standard Python library dependencies (pyyaml, tabulate, tomli), see [Python Dependencies](https://deepwiki.com/tenstorrent/tt-flash/7.2-python-dependencies).

The ecosystem dependencies covered here are:

*   **pyluwen**: Hardware I/O abstraction layer for SPI, AXI, and ARC communication
*   **tt_tools_common**: Shared utilities and driver compatibility checks
*   **tt-kmd**: Kernel mode driver for PCI device enumeration and chip access
*   **tt-smi**: System management tool providing complementary monitoring capabilities

## pyluwen: Hardware I/O Abstraction Layer

### Overview

`pyluwen` is the critical hardware abstraction library that provides low-level communication interfaces for Tenstorrent chips. tt-flash depends on `pyluwen ~= 0.8.0`, using the compatible release specifier to allow patch-level updates while preventing breaking changes.

**Dependency Declaration**: [pyproject.toml 26](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L26-L26)

### Core Capabilities

The library provides four primary interface categories:

| Interface | Purpose | Usage in tt-flash |
| --- | --- | --- |
| **SPI Operations** | Read/write to SPI flash memory | Flashing firmware images, verification reads |
| **AXI Operations** | Access to chip memory via AXI bus | Reading board IDs, chip configuration |
| **ARC Messaging** | Communication with ARC processor | Firmware version queries, SPI copy triggers, flash validation |
| **Device Detection** | PCI device enumeration | Detecting available chips, chip initialization |

**Sources**: [pyproject.toml 26](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L26-L26)[CHANGELOG.md 15-21](https://github.com/tenstorrent/tt-flash/blob/60c06032/CHANGELOG.md?plain=1#L15-L21)

## tt-kmd: Kernel Mode Driver

### Overview

`tt-kmd` is the Tenstorrent kernel mode driver that provides the foundational interface between user-space tools and Tenstorrent hardware. tt-flash requires `tt-kmd >= 2.0.0` for proper operation.

### Architecture Role

**Key Responsibilities**:

| Function | Description |
| --- | --- |
| **PCI Device Enumeration** | Discovers and registers Tenstorrent chips on the PCI bus |
| **Memory Mapping** | Provides memory-mapped I/O access to chip registers |
| **Device Management** | Handles device power states, resets, and lifecycle |
| **Access Control** | Mediates concurrent access from multiple user-space tools |

### Version Requirements

tt-flash explicitly requires `tt-kmd >= 2.0.0` as validated by `tt_tools_common`. The driver version check was updated in version 3.3.3 to ensure compatibility:

**Driver Compatibility Update**[CHANGELOG.md 23-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/CHANGELOG.md?plain=1#L23-L26):

```

- Bumped tt-tools-common version to fix driver version check 
  for compatibility with tt-kmd 2.0.0
```

## tt-smi Integration

### Overview

While not a direct dependency, `tt-smi` (Tenstorrent System Management Interface) is a complementary tool in the Tenstorrent ecosystem. tt-flash maintains version alignment with tt-smi to ensure system stability.

### Stability Alignment

**Coordination Example**[CHANGELOG.md 31-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/CHANGELOG.md?plain=1#L31-L35):

```

- luwen version bump to bring inline with tt-smi; 
  provides stability fixes
```

This coordination ensures:

*   **Consistent Hardware State**: Both tools see the same device state through pyluwen
*   **Compatibility**: Firmware flashed by tt-flash is correctly reported by tt-smi
*   **Stability**: Shared dependency versions are tested together in production environments

## Dependency Version Management

### Version Constraint Syntax

tt-flash uses PEP 440 version specifiers to control dependency versioning:

**Version Specifier Reference**:

| Specifier | Meaning | Example | Allows |
| --- | --- | --- | --- |
| `~= X.Y.Z` | Compatible release | `~= 0.8.0` | `0.8.0`, `0.8.1`, `0.8.2` but not `0.9.0` |
| `== X.Y.Z` | Exact version | `== 6.0.2` | Only `6.0.2` |
| `>= X.Y.Z` | Minimum version | `>= 2.0.0` | `2.0.0`, `2.1.0`, `3.0.0`, etc. |

**Rationale**:

*   **Tenstorrent Ecosystem Libraries** (`pyluwen`, `tt_tools_common`): Use compatible release (`~=`) to allow patch updates while preventing breaking changes
*   **Python Standard Libraries** (`pyyaml`): Use exact versions (`==`) for reproducibility
*   **System Requirements** (`tt-kmd`): Use minimum version (`>=`) for driver compatibility

**Sources**: [pyproject.toml 23-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L35)
