This page documents the experimental and specialized compute headers within the `tt-emule` JIT kernel API. These headers provide specialized implementations of complex algorithms (like Bitonic Sort or MoE Gate logic) and experimental features that extend the base compute API. In emulation, these are implemented as scalar C++ loops operating on the thread-local `__emule_dst` register file.

## Overview of Specialized APIs

While standard operations (addition, multiplication, etc.) are handled by the core compute headers, certain high-level operations require specialized logic to match the "contract" between kernel authors and Tenstorrent hardware.

| Header | Primary Functionality | Key Features |
| --- | --- | --- |
| `topk_xl.h` | Bitonic Sort Emulation | Fused index encoding, sequence merging, and rebuilding. |
| `welford.h` | Online Mean/Variance | Per-column accumulators, group-norm state swapping. |
| `kimi_moe_gate.h` | DeepSeek/Kimi MoE Logic | Specialized top-K ranking with bias-score fusion. |
| `fill_arange.h` | Experimental Tensor Fill | Fills tiles with arithmetic sequences. |
| `semaphore.h` | Quasar Semaphores | Experimental synchronization for the Quasar architecture. |

## Bitonic Sort (topk_xl)

The `topk_xl.h` header emulates the silicon distributed top-K compute kernel. On real hardware, this is implemented using SFPU bitonic sort instructions. In `tt-emule`, it is implemented using `std::sort` over a flat view of the `__emule_dst` tiles.

### Fused Index Encoding

A critical part of the Top-K contract is the fusion of values and indices into a single 32-bit word:

*   **Bits 31:16**: bfloat16 value.
*   **Bits 15:0**: uint16 index.

The emulator provides `decode_fused` and `encode_fused` to handle this bit-packing, ensuring that `std::sort` results match the hardware's expected output layout [include/jit_hw/api/compute/topk_xl.h 50-71](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/topk_xl.h#L50-L71)

### Key Functions

*   `topk_xl_copy_tile`: Unpacks values from a Circular Buffer into DST and initializes indices to zero [include/jit_hw/api/compute/topk_xl.h 105-142](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/topk_xl.h#L105-L142)
*   `topk_xl_add_lsb_indices`: Injects the lane position and `core_id` into the fused words [include/jit_hw/api/compute/topk_xl.h 148-164](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/topk_xl.h#L148-L164)
*   `topk_xl_local_sort`: Performs the actual sort on the specified DST tiles using `std::sort`[include/jit_hw/api/compute/topk_xl.h 173-188](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/topk_xl.h#L173-L188)

**Sources:**[include/jit_hw/api/compute/topk_xl.h 5-188](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/topk_xl.h#L5-L188)

## Welford Online Algorithm

The `welford.h` header implements the online algorithm for computing mean and variance, primarily used in LayerNorm and GroupNorm kernels.

### State Management

The emulator maintains per-column (32 lanes) accumulators in thread-local storage:

*   `__emule_welford_mean[32]`: Running mean [include/jit_hw/api/compute/welford.h 33](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/welford.h#L33-L33)
*   `__emule_welford_m2[32]`: Running sum of squares of differences from the mean [include/jit_hw/api/compute/welford.h 34](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/welford.h#L34-L34)

For GroupNorm, the emulator provides `welford_save_state` and `welford_restore_state` which swap these accumulators with a group-indexed backing store `__emule_welford_grp_mean`[include/jit_hw/api/compute/welford.h 120-132](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/welford.h#L120-L132)

### Implementation Details

*   **Reciprocal Handling**: Silicon uses a LUT for $1/n$. The emulator derives this directly from the `start_idx` passed to `welford_update`[include/jit_hw/api/compute/welford.h 87-96](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/welford.h#L87-L96)
*   **Finalization**: `welford_finalize` writes the results into row 0 of two consecutive DST tiles (Mean and Variance) [include/jit_hw/api/compute/welford.h 137-153](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/welford.h#L137-L153)

**Sources:**[include/jit_hw/api/compute/welford.h 6-153](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/welford.h#L6-L153)

## MoE Gate Shadows (Kimi / GLM)

Specialized headers like `kimi_moe_gate.h` and `glm_moe_gate.h` provide "Layer-1 shadows" of Mixture-of-Experts (MoE) gating logic.

### Kimi MoE Logic

The `kimi_moe_gate` function performs a specialized Top-K across 384 active experts.

1.   **Bias Fusion**: Computes `bias_scores = act + bias`.
2.   **Top-K**: Performs a stable sort on `bias_scores`.
3.   **Normalization**: Applies a scaling factor: $scores \times \frac{scale}{sum(scores) + \epsilon}$ [include/jit_hw/api/compute/kimi_moe_gate.h 105-113](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/kimi_moe_gate.h#L105-L113)
4.   **DST Layout**: Writes normalized scores to `DST[0]` and bit-packed indices to `DST[1]`[include/jit_hw/api/compute/kimi_moe_gate.h 120-136](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/kimi_moe_gate.h#L120-L136)

**Sources:**[include/jit_hw/api/compute/kimi_moe_gate.h 7-136](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/kimi_moe_gate.h#L7-L136)[include/jit_hw/api/compute/glm_moe_gate.h 7-124](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/glm_moe_gate.h#L7-L124)

## Experimental & Utility APIs

### Fill Arange

The `fill_arange_tile` function (in `fill_arange.h`) allows a kernel to fill a DST tile with a sequence $start, start + step, start + 2 \times step, \dots$. This is used for generating coordinate grids or index tensors directly on the compute engine [include/jit_hw/api/compute/experimental/fill_arange.h](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/experimental/fill_arange.h)

### Fast Untilize

The `fast_untilize` API provides an experimental path for converting tiled data in L1 to row-major format, bypassing some of the standard LLK pack/unpack overhead for specific performance-critical patterns [include/jit_hw/api/compute/experimental/fast_untilize.h](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/experimental/fast_untilize.h)

### Quasar Semaphores

The `experimental/semaphore.h` header defines the semaphore API for the Quasar architecture. Unlike the standard NOC semaphores, these are designed for the high-concurrency 12-thread model of the Quasar core, providing experimental primitives for thread-to-thread synchronization within a single core.

**Sources:**[include/jit_hw/api/compute/experimental/fill_arange.h](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/experimental/fill_arange.h)[include/jit_hw/api/compute/experimental/semaphore.h](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/experimental/semaphore.h)

## Compute Macro System

All specialized compute headers rely on the macro system defined in `common.h` to ensure they execute correctly on the single-threaded x86 JIT host.

| Macro | Hardware Meaning | Emulation Behavior |
| --- | --- | --- |
| `PACK(x)` | Runs on TRISC0 | Executed immediately |
| `MATH(x)` | Runs on TRISC1 | Executed immediately |
| `UNPACK(x)` | Runs on TRISC2 | Executed immediately |
| `ALWI` | Hardware `inline` | `FORCE_INLINE` (always_inline) |
