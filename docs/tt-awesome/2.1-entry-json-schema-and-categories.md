This page provides a technical reference for the data structure and taxonomy used in the Awesome Tenstorrent ecosystem. The system relies on a single source of truth: individual JSON files located in the `entries/` directory. These files are validated against a strict schema to ensure consistency across the generated README and the Eleventy-based web frontend.

## Entry JSON Schema

Each project or resource in the directory is represented by a JSON file. The schema is enforced by `scripts/validate.py`[scripts/validate.py 24-93](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L24-L93) and verified during CI via the `validate_entry` function.

### Core Fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | `string` | **Yes** | Kebab-case identifier. Must match the filename [scripts/validate.py 26-28](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L26-L28) |
| `name` | `string` | **Yes** | The display name of the project. |
| `description` | `string` | **Yes** | A 1ŌĆō3 sentence plain-text description. |
| `affiliation` | `enum` | **Yes** | One of: `community`, `affiliated`, `official`[scripts/validate.py 11](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L11-L11) |
| `categories` | `string[]` | **Yes** | List of category slugs. The first item determines the file's subdirectory [CONTRIBUTING.md 39](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L39-L39) |
| `links` | `object[]` | **Yes** | Array of link objects. Must contain at least one entry [scripts/validate.py 42-43](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L42-L43) |

### Link Object Schema

Links are defined as objects within the `links` array [scripts/validate.py 45-50](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L45-L50):

*   `type`: **Required**. One of: `repo`, `article`, `talk`, `video`, `website`, `demo`, `lesson`, `paper`[scripts/validate.py 17](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L17-L17)
*   `url`: **Required**. Must be a valid `https://` URL [scripts/validate.py 20](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L20-L20)
*   `label`: _Optional_. Custom display text for the link.

### Validation Logic

Validation is split into two categories: **Fatal Errors** (which block the build) and **Warnings** (which suggest improvements).

*   **`validate_entry(path, data)`**: Checks for required fields, regex matches for URLs/Dates, and membership in `VALID_*` enums [scripts/validate.py 24-93](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L24-L93)
*   **`validate_entry_warnings(path, data)`**: Checks for missing non-essential metadata like `added_at`[scripts/validate.py 96-101](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L96-L101)
*   **`isSafeHttpsUrl(url)`**: A utility in the data loader that strips non-HTTPS or malformed URLs before they reach the UI [src/_data/entries.js 16-24](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/entries.js#L16-L24)

### Affiliation & Sorting

The system prioritizes community contributions by design. This is implemented via the `AFFILIATION_ORDER` constant:

1.   `community` (Rank 0)
2.   `affiliated` (Rank 1)
3.   `official` (Rank 2)

Within each tier, entries are further sorted by `featured` status and then by `stars` (fetched from GitHub metadata) [src/_data/entries.js 84-93](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/entries.js#L84-L93)
