## Environment Variables for Test Control

The following tables document all environment variables that control test selection and execution:

### Test Selection Variables

| Variable | Description | Example Values |
| --- | --- | --- |
| `OPERATORS` | Comma-separated list of operators to test. Prefix with `-` to exclude. | `add,mul` or `-conv2d` |
| `FILTERS` | Comma-separated list of filter lambdas from `VectorLambdas` or `QueryLambdas` | `QUICK,NOT_FAILING,SINGLE_SHAPE` |
| `INPUT_SOURCES` | Filter by input source type | `FROM_HOST,FROM_ANOTHER_OP,CONST_EVAL_PASS` |
| `INPUT_SHAPES` | Filter by specific tensor shapes (Python list syntax) | `[(1,4),(2,3,4)]` |
| `DEV_DATA_FORMATS` | Filter by data format | `Float16_b,Int8` |
| `MATH_FIDELITIES` | Filter by math fidelity | `HiFi4,HiFi3` |
| `KWARGS` | Filter by operator parameters (Python dict syntax) | `[{'dim': 0}, {'dim': 1}]` |

### Test Range and Sampling Variables

| Variable | Description | Example Values |
| --- | --- | --- |
| `SAMPLE` | Percentage of tests to randomly sample | `10` (for 10%) |
| `RANDOM_SEED` | Seed for random sampling | `42` |
| `RANGE` | Limit test indices (start,end) | `0,100` or `100` |
| `TEST_ID` | Run a single test by its ID string | `add-FROM_HOST-{}-...` |
| `ID_FILES` | Comma-separated paths to files containing test IDs | `ids/passing.txt,ids/new.txt` |
| `ID_FILES_IGNORE` | Files containing test IDs to skip | `ids/blacklist.txt` |

### Execution Control Variables

| Variable | Description | Example Values |
| --- | --- | --- |
| `SKIP_FORGE_VERIFICATION` | Skip Forge compilation and verification | `true` / `false` |
| `DRY_RUN` | Skip test execution, only list tests | `true` / `false` |
| `IGNORE_XFAIL_FILES` | Ignore expected failure files | `true` / `false` |
| `IGNORE_SKIP_FILES` | Ignore skip files | `true` / `false` |
| `HF_TOKEN` | HuggingFace token for model downloads | `hf_...` |

**Sources:**[forge/test/operators/pytorch/test_all.py 31-140](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/pytorch/test_all.py#L31-L140)[forge/test/operators/utils/features.py 11-58](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/operators/utils/features.py#L11-L58)

### Test Matrix Strategy

The CI system uses a matrix strategy to parallelize test execution across hardware and test groups:

| Parameter | Description | Example Values |
| --- | --- | --- |
| `runs-on` | Hardware targets | `n150`, `n300`, `p150` |
| `test_group_id` | Test group number | `1`, `2`, `3`, ..., `6` |
| `test_group_cnt` | Total number of groups | `6` (default) |
| `test_mark` | pytest marker to select tests | `push`, `pr_models_regression`, `nightly` |

**Example Matrix Expansion:**

*   `runs-on: [n150, n300]` and `test_group_cnt: 6` creates **12 parallel jobs**
*   Each job runs approximately 1/6 of the test suite on its assigned hardware

**Sources:**[.github/workflows/test-sub.yml 79-104](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/.github/workflows/test-sub.yml#L79-L104)[.github/workflows/on-pr.yml 108-130](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/.github/workflows/on-pr.yml#L108-L130)

## Test Runner and Crash Handling

### Test Runner Script

The CI uses a custom test runner (`test_runner.py`) that provides:

*   **Crash Recovery:** Continues after pytest crashes
*   **Memory Logging:** Tracks memory usage per test
*   **JUnit XML Generation:** Structured test reports

When a test crashes, the runner:

1.   Logs the crashed test to `crashed_pytest.log`
2.   Restarts pytest with remaining tests
3.   Appends crash information to the job summary

**Sources:**[.github/workflows/test-sub.yml 253-288](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/.github/workflows/test-sub.yml#L253-L288)

## Test Result Interpretation

### pytest Outcomes

Tests can have the following outcomes:

| Outcome | Symbol | Description |
| --- | --- | --- |
| **PASSED** | `.` | Test passed successfully |
| **FAILED** | `F` | Test failed with an exception |
| **XFAIL** | `x` | Expected failure occurred (test marked with `@pytest.mark.xfail`) |
| **XPASS** | `X` | Unexpected pass (expected to fail but passed) |
| **SKIPPED** | `s` | Test was skipped (marked with `@pytest.mark.skip`) |
| **ERROR** | `E` | Error during test setup or teardown |

## Summary

Running tests in tt-forge-onnx involves:

1.   **Local Execution:** Use pytest directly with environment variables for filtering
2.   **Test Selection:** Apply filters through `OPERATORS`, `FILTERS`, `INPUT_SHAPES`, etc.
3.   **CI/CD:** Tests run automatically on PRs and nightly through GitHub Actions
4.   **Parallelization:** Matrix strategy splits tests across hardware and groups
5.   **Result Interpretation:** JUnit reports, xfail validation, and crash recovery
6.   **Advanced Features:** ID files, blacklisting, duration tracking, and verbose output
