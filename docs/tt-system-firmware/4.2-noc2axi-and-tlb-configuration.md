## Overview

The NOC2AXI interface enables the ARC processor to access memory and registers across the entire Blackhole chip fabric via the Network-on-Chip (NOC). Since the ARC cannot directly address the full 64-bit NOC address space, it uses **Translation Lookaside Buffers (TLBs)** to map 16MB windows of its 32-bit address space to arbitrary NOC coordinates and addresses.

This page documents the TLB architecture, configuration API, and common access patterns. For information about NOC coordinate translation and harvesting support, see [NOC Translation and Harvesting](https://deepwiki.com/tenstorrent/tt-system-firmware/4.1-noc-translation-and-harvesting).

**Key Capabilities:**

*   16 TLBs per NOC ring (32 total across NOC0 and NOC1)
*   16MB window size per TLB
*   Support for unicast, multicast, and broadcast operations
*   Configurable transaction ordering modes
*   Used to access MRISC, ERISC, Tensix, PCIe, and other subsystems

## TLB Architecture

### Address Mapping Mechanism

Each TLB consists of 4 configuration registers that control how ARC addresses are translated to NOC addresses:

**Address Translation Example:**

*   ARC writes to `0xC1234567` (NOC0, TLB 1, offset 0x234567)
*   TLB1 configured: `TLB0=0x40` (addr[31:24]), `TLB1=0x20` (addr[63:32]), `TLB2={x_end=5, y_end=3}`
*   Resulting NOC address: `0x2040234567` sent to coordinate (5,3)

Sources: [lib/tenstorrent/bh_arc/noc2axi.c 11-91](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.c#L11-L91)[lib/tenstorrent/bh_arc/noc2axi.h 13-42](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.h#L13-L42)

### TLB Register Structure

| Register | Bits | Field | Description |
| --- | --- | --- | --- |
| **TLB0** | [31:8] | `passthrough_bits` | Reserved for future use |
|  | [7:0] | `lower_addr_bits` | NOC address bits [31:24] |
| **TLB1** | [31:0] | `middle_addr_bits` | NOC address bits [63:32] |
| **TLB2** | [31:30] | `reserved_1` | Reserved |
|  | [29:28] | `sidebands_reserved` | Reserved |
|  | [27] | `linked` | Link TLB to next entry |
|  | [26:25] | `ordering_mode` | Transaction ordering (see below) |
|  | [24] | `multicast_en` | Enable multicast mode |
|  | [23:18] | `y_start` | Starting Y coordinate (multicast) |
|  | [17:12] | `x_start` | Starting X coordinate (multicast) |
|  | [11:6] | `y_end` | Target/ending Y coordinate |
|  | [5:0] | `x_end` | Target/ending X coordinate |
| **TLB3** | [31:24] | `num_destinations` | Number of multicast destinations |
|  | [23:20] | `quad_exclude_ctrl` | Quadrant exclude control |
|  | [19:14] | `quad_exclude_y` | Y coordinates to exclude |
|  | [13:8] | `quad_exclude_x` | X coordinates to exclude |
|  | [7:4] | `stride_y` | Y stride for multicast |
|  | [3:0] | `stride_x` | X stride for multicast |

Sources: [lib/tenstorrent/bh_arc/noc2axi.c 11-60](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.c#L11-L60)

### TLB Register Layout

The 16 TLBs per NOC ring are organized in a contiguous register block:

| Offset Range | Contents |
| --- | --- |
| `0x1000 + 0x00..0x1F` | TLB0/TLB1 registers (2 DWORDs per TLB, 16 TLBs) |
| `0x1000 + 0x20..0x3F` | TLB2 registers (1 DWORD per TLB, 16 TLBs) |
| `0x1000 + 0x40..0x5F` | TLB3 registers (1 DWORD per TLB, 16 TLBs) |

Base addresses:

*   **NOC0 TLBs**: `0x80051000`
*   **NOC1 TLBs**: `0x80059000` (bit 15 set to select ring)

## TLB Configuration Functions

### Basic TLB Setup

**`NOC2AXITlbSetup()`** - Configure TLB for single-target access

`void NOC2AXITlbSetup(uint8_t ring, uint8_t tlb_num,                      uint8_t x, uint8_t y, uint64_t addr)`
**Parameters:**

*   `ring`: NOC ring (0 or 1)
*   `tlb_num`: TLB index (0-15)
*   `x`, `y`: Target NOC coordinates
*   `addr`: NOC address (upper 40 bits; lower 24 bits from window offset)

**Usage Example - MRISC L1 Access:**

`// Configure TLB 13 to access MRISC L1 at physical address 0x20_00000000uint8_t x, y;GetGddrNocCoords(gddr_inst, port, 0, &x, &y);NOC2AXITlbSetup(0, 13, x, y, 0x2000000000ULL); // Now ARC can access via 0xCD000000 + offsetvolatile uint32_t *l1_window = GetTlbWindowAddr(0, 13, 0x2000000000ULL);`
Sources: [lib/tenstorrent/bh_arc/noc2axi.c 93-112](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.c#L93-L112)[lib/tenstorrent/bh_arc/gddr.c 72-79](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/gddr.c#L72-L79)

### Multicast TLB Setup

**`NOC2AXIMulticastTlbSetup()`** - Configure TLB for multicast operations

`void NOC2AXIMulticastTlbSetup(uint8_t ring, uint8_t tlb_num,                              uint8_t x_start, uint8_t y_start,                              uint8_t x_end, uint8_t y_end,                              uint64_t addr, Noc2AxiOrdering ordering)`
**Parameters:**

*   `x_start`, `y_start`: Starting coordinates of multicast rectangle
*   `x_end`, `y_end`: Ending coordinates of multicast rectangle
*   `ordering`: Transaction ordering mode (see [Ordering Modes](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/Ordering%20Modes))

**Usage Example - Tensix Firmware Load:**

`// Load firmware to all Tensix cores (X: 2-1, Y: 2-11)NOC2AXIMulticastTlbSetup(0, 0,                          2, 2,    // x_start, y_start                         1, 11,   // x_end, y_end                         0x6000,  // Load address                         kNoc2AxiOrderingStrict);`
Sources: [lib/tenstorrent/bh_arc/noc2axi.c 114-138](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.c#L114-L138)[lib/tenstorrent/bh_arc/tensix_init.c 227-231](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/tensix_init.c#L227-L231)

### Broadcast TLB Setup

**`NOC2AXITensixBroadcastTlbSetup()`** - Broadcast to all unharvested Tensix cores

`void NOC2AXITensixBroadcastTlbSetup(uint8_t ring, uint8_t tlb_num,                                    uint64_t addr, Noc2AxiOrdering ordering)`
This function automatically configures multicast to skip the ARC's row and respect broadcast exclusion masks set during NOC initialization. The exact coordinates depend on NOC translation state:

| Mode | NOC0 Rectangle | NOC1 Rectangle |
| --- | --- | --- |
| Translation OFF | X: 2-1, Y: 2-11 | X: 0-16, Y: 0-10 (skip row 11) |
| Translation ON | X: 2-1, Y: 2-11 | X: 1-2, Y: 11-2 (reversed) |

**Usage Example - Clear Tensix DEST:**

`// Write 0 to all Tensix DEST registersNOC2AXITensixBroadcastTlbSetup(0, 0, 0xFFB12244, kNoc2AxiOrderingStrict);NOC2AXIWrite32(0, 0, 0xFFB12244, 0);`

## Ordering Modes

TLB transactions support four ordering modes that control write posting and completion semantics:

**Mode Selection Guidelines:**

| Mode | Use Case | Example |
| --- | --- | --- |
| **Relaxed (0)** | High-throughput bulk transfers where ordering doesn't matter | DMA transfers to independent buffers |
| **Strict (1)** | Control register writes that must complete before proceeding | Enable bits, reset signals, synchronization |
| **Posted (2)** | Sequential writes that can be pipelined | Filling contiguous memory |
| **PostedStrict (3)** | Balance of performance and ordering | Firmware loading with dependencies |

**Common Usage in Codebase:**

*   MRISC/ERISC firmware loading: `kNoc2AxiOrderingStrict` (must complete before boot)
*   Tensix broadcast operations: `kNoc2AxiOrderingStrict` (synchronization required)
*   PCIe TLB configuration: `kNoc2AxiOrderingStrict` (setup must complete)

Sources: [lib/tenstorrent/bh_arc/noc2axi.h 19-24](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.h#L19-L24)[lib/tenstorrent/bh_arc/noc2axi.c 108-109](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.c#L108-L109)

## Access Helper Functions

### Window Address Calculation

**`GetTlbWindowAddr()`** - Convert NOC address to ARC-visible pointer

`static inline void volatile *GetTlbWindowAddr(uint8_t noc_id,                                               uint8_t tlb_entry,                                              uint64_t addr)`
Returns an ARC pointer that, when dereferenced, generates a NOC transaction through the specified TLB.

**Address Calculation:**

```
arc_addr = noc_base + (tlb_entry << 24) + (addr & 0xFFFFFF)

Where:
  noc_base = 0xC0000000 (NOC0) or 0xE0000000 (NOC1)
  tlb_entry: 0-15
  addr & 0xFFFFFF: Window offset (0 - 16MB)
```

Sources: [lib/tenstorrent/bh_arc/noc2axi.h 34-42](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/noc2axi.h#L34-L42)

**`NOC2AXIWrite8()`** - 8-bit NOC write

`static inline void NOC2AXIWrite8(uint8_t noc_id, uint8_t tlb_entry,                                 uint64_t addr, uint8_t data)`

## Hardware Block Access Patterns

### MRISC (GDDR Controller) Access

The MRISC firmware runs on RISC cores within each GDDR controller. The ARC must access both L1 memory and control registers.

**TLB Allocation:**

*   **TLB 13** - Dynamically reconfigured for L1 or register access

**Key Addresses:**

*   **L1 Memory**: `0x20_00000000` (128KB window)
*   **Registers**: `0x100_FFB14000` range 
    *   `MRISC_INIT_STATUS`: `0x100_FFB14010`
    *   `MRISC_POST_CODE`: `0x100_FFB14014`
    *   `MRISC_MSG_REGISTER`: `0x100_FFB14018`

**Typical Access Sequence:**

1.   `SetupMriscL1Tlb(gddr_inst)` - Configure TLB for L1
2.   Load firmware via DMA or word-by-word writes
3.   `MriscRegWrite32(gddr_inst, MRISC_INIT_STATUS, value)` - Set control register
4.   `ReleaseMriscReset(gddr_inst)` - Start MRISC execution
5.   Poll `MriscRegRead32(gddr_inst, MRISC_INIT_STATUS)` for completion

Sources: [lib/tenstorrent/bh_arc/gddr.c 72-115](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/gddr.c#L72-L115)[lib/tenstorrent/bh_arc/gddr.h 19-28](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/gddr.h#L19-L28)

### ERISC (Ethernet Controller) Access

Similar to MRISC, ERISC firmware runs on each Ethernet instance.

**TLB Allocation:**

*   **TLB 0** - `ETH_SETUP_TLB` for Ethernet access

**Key Addresses:**

*   **Firmware Base**: `0x70000` in L1
*   **Parameter Table**: `0x7C000` in L1
*   **Reset PC Registers**: `0xFFB14000` range

**Access Functions:**

*   `SetupEthTlb(eth_inst, ring, addr)` - Configure TLB for instance
*   `LoadEthFw()` - Load ERISC firmware binary
*   `LoadEthFwCfg()` - Load configuration parameters
*   `ReleaseEthReset()` - Start ERISC execution

Sources: [lib/tenstorrent/bh_arc/eth.c 31-50](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/eth.c#L31-L50)[lib/tenstorrent/bh_arc/eth.c 77-84](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/eth.c#L77-L84)[lib/tenstorrent/bh_arc/eth.c 219-227](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/eth.c#L219-L227)

### Tensix Core Access

Tensix cores require both L1 memory access and control register configuration.

**TLB Usage:**

*   **TLB 0** - Reconfigured for different phases: 
    *   Multicast for firmware loading
    *   Point-to-point for counter reads
    *   Broadcast for register writes

**Key Operations:**

1.   **L1 Wiping**: Broadcast zero-filled buffer to all Tensix L1
2.   **DEST Wiping**: Load firmware via multicast, run on TRISC0, poll atomic counter
3.   **Clock Gating**: Broadcast CG control register writes

**Register Addresses:**

*   `TRISC0_RESET_PC`: `0xFFB12228`
*   `SOFT_RESET_0`: `0xFFB121B0`
*   `CG_CTRL_EN`: `0xFFB12244`

Sources: [lib/tenstorrent/bh_arc/tensix_init.c 40-56](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/tensix_init.c#L40-L56)[lib/tenstorrent/bh_arc/tensix_init.c 117-178](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/tensix_init.c#L117-L178)[lib/tenstorrent/bh_arc/tensix_init.c 241-315](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/bh_arc/tensix_init.c#L241-L315)

### PCIe SerDes Access

PCIe initialization requires access to multiple hardware blocks, each mapped through dedicated TLBs.

**TLB Allocation:**

| TLB | Purpose | Base Address |
| --- | --- | --- |
| 0 | SerDes 0 Alphacore registers | `0xFFFFFFFF_E1000000` |
| 1 | SerDes 1 Alphacore registers | `0xFFFFFFFF_E1000000 + 0x04000000` |
| 2 | SerDes 0 Control registers | `0xFFFFFFFF_E0000000 + 0x03000000` |
| 3 | SerDes 1 Control registers | `0xFFFFFFFF_E0000000 + 0x07000000` |
| 4 | SII (System Integration Interface) | `0xFFFFFFFF_F0000000` |
| 5 | TLB Configuration registers | `0x1FC00000` |
| 14 | DBI (DesignWare Bridge Interface) | `0x3E_00000000` (TLB 62 address space) |

**DBI Access Special Handling:** The DBI uses NOC TLB ID 62 as a flag, requiring SII register setup:

## TLB Constraints and Best Practices

### TLB Limitations

1.   **Window Size**: Fixed at 16MB (2^24 bytes) per TLB

    *   Accessing beyond window boundaries requires TLB reconfiguration
    *   Large transfers may span multiple TLB setups

2.   **Number of TLBs**: Only 16 per NOC ring

    *   Must reuse TLBs for different purposes
    *   Careful management needed during concurrent operations

3.   **Address Alignment**: NOC addresses must align to TLB boundaries

    *   Lower 24 bits come from window offset
    *   Upper 40 bits configured in TLB registers

### Best Practices

**TLB Allocation Strategy:**

```
TLB 0:   General purpose, frequently reconfigured
TLB 1-5: PCIe-specific during initialization
TLB 6-12: Available for runtime use
TLB 13:  MRISC/GDDR access (by convention)
TLB 14:  DBI access (PCIe)
TLB 15:  Available
```

**Avoiding TLB Conflicts:**

*   Use different TLBs for concurrent access patterns
*   Document TLB allocation in module comments
*   Reconfigure TLBs immediately before use in transient operations

**Performance Optimization:**

*   Use multicast for identical writes to multiple targets
*   Use broadcast for all-Tensix operations
*   Prefer posted ordering modes for bulk transfers
*   Use strict ordering only when synchronization is required
