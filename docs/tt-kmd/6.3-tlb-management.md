# TLB Management

This document describes the Translation Lookaside Buffer (TLB) management system in the Tenstorrent kernel driver. The TLB system provides efficient access to the Network-on-Chip (NOC) by mapping device memory regions into user-accessible windows of various sizes.

## Overview

The TLB management system allows user-space applications to access specific regions of device memory through memory-mapped windows. These windows provide direct access to the NOC without requiring expensive system calls for each memory operation. The system supports multiple window sizes optimized for different access patterns and device architectures.

## TLB Window Types and Allocation

Different device classes support various TLB window sizes (e.g., 1MB, 2MB, 16MB, 4GB) optimized for different use cases. The allocation system tracks available windows using bitmaps and enforces per-device limits based on the `tenstorrent_device_class` definition.

### TLB Allocation Process

The allocator in `tlb.c` uses a bitmap to manage TLB windows. Since different sizes (kinds) of TLBs are stored in a single flat bitmap, the driver calculates an offset based on the kind index to find the correct range.

Sources: [tlb.c 9-53](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.c#L9-L53)

## TLB Configuration and NOC Access

Once allocated, TLB windows must be configured to map specific NOC addresses. The configuration specifies the target memory region and coordinates within the NOC fabric via the `tenstorrent_noc_tlb_config` structure.

### Configuration Implementation

The driver provides a polymorphic interface for programming hardware-specific TLB registers through the device class.

*   **Function**: `tenstorrent_device_configure_tlb`[tlb.c 77-84](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.c#L77-L84)
*   **Hardware Callback**: `tt_dev->dev_class->configure_tlb(tt_dev, tlb, config)`[tlb.c 81](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.c#L81-L81)

## IOCTL Interface

The TLB management system exposes three primary IOCTL commands for user-space interaction, handled in `memory.c`.

| IOCTL | Kernel Handler | Purpose |
| --- | --- | --- |
| `ALLOCATE_TLB` | `ioctl_allocate_tlb` | Allocates a TLB of requested size and returns mmap offsets. |
| `FREE_TLB` | `ioctl_free_tlb` | Releases the TLB window back to the pool. |
| `CONFIGURE_TLB` | `ioctl_configure_tlb` | Programs the TLB to point to a specific NOC address. |

## Data Structures and Ownership

The driver maintains strict ownership of TLB resources. When a process allocates a TLB, it is marked in both the global device bitmap and a per-file-descriptor bitmap.

### Key Structures

*   **`tlb_descriptor`**: Describes the physical location of a TLB window within a PCI BAR. 
    *   `bar`: The BAR index (e.g., BAR0 or BAR4). [tlb.h 13](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.h#L13-L13)
    *   `size`: The size of the window. [tlb.h 14](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.h#L14-L14)
    *   `bar_offset`: The offset within the specified BAR. [tlb.h 15](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.h#L15-L15)

*   **`tenstorrent_device`**: 
    *   `tlbs`: Global bitmap of all allocated TLBs. [tlb.c 38-43](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.c#L38-L43)
    *   `tlb_counts`: Array of TLB counts per kind. [tlb.c 22](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/tlb.c#L22-L22)

*   **`chardev_private`**: 
    *   `tlbs`: Bitmap of TLBs owned by this specific file descriptor. [chardev_private.h 40](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/chardev_private.h#L40-L40)

### Resource Cleanup

When a character device is closed, `tenstorrent_memory_cleanup` is called. This function iterates through the `priv->tlbs` bitmap and releases any TLBs that were not explicitly freed by the user, ensuring no resource leaks occur.
