The Random Graph Generator (RGG) is a robust testing system in PyBuda for automatically generating, compiling, and verifying computational graphs with randomized operators and tensor shapes. It creates diverse neural network architectures to thoroughly test the PyBuda compilation pipeline and hardware execution, uncovering potential bugs and edge cases that might not be discovered with predefined test cases.

For information about testing individual operators, see [Operator Testing](https://deepwiki.com/tenstorrent/tt-buda/3.2-operator-testing).

The RGG system consists of several key components that work together to generate valid computational graphs, convert them to executable models, and verify their correctness across different hardware targets.

## Graph Generation Process

The RGG uses a backward graph construction approach, building from outputs to inputs, which ensures that the generated graphs are connected and solvable.

Sources:

*   [pybuda/test/random/rgg/algorithms.py 203-404](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/test/random/rgg/algorithms.py#L203-L404)

### Key Components

#### 1. RandomizerNode

The fundamental building block representing an operation in the graph. Each node contains:

*   An operator definition from the framework's operator repository
*   Input and output shapes
*   Constructor and forward parameters
*   References to input nodes

Sources:

*   [pybuda/test/random/rgg/datatypes.py 18-73](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/test/random/rgg/datatypes.py#L18-L73)

#### 2. RandomGraphAlgorithm

The primary algorithm that builds random graphs by selecting operators, determining shapes, and connecting nodes. The algorithm:

1.   Chooses random operators from the framework repository
2.   Sets output shapes for each node
3.   Calculates appropriate input shapes
4.   Creates fork-join patterns when nodes connect to multiple outputs
5.   Adds constant inputs randomly

Sources:

## Generator Algorithm Details

The heart of RGG is its graph generation algorithm. Let's explore how it works in detail.

### Backward Graph Building

Unlike traditional forward model building, RGG builds graphs backward from outputs to inputs:

1.   Start with an empty graph
2.   Choose a random operator and create the first node with a random output shape
3.   For subsequent nodes: 
    *   Select another random operator
    *   Connect its output to one or more previous nodes' inputs
    *   Calculate appropriate input shapes based on output shape

4.   Continue until reaching the desired graph size
5.   Add input nodes for any remaining open inputs

This backward approach ensures that all nodes have valid connections and shapes, avoiding disconnected subgraphs.

Sources:

*   [pybuda/test/random/rgg/algorithms.py 247-400](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/test/random/rgg/algorithms.py#L247-L400)

### Shape Calculation

A key challenge in random graph generation is ensuring that tensor shapes are compatible between connected operations:

1.   The `calc_input_shapes` method determines valid input shapes for each operation based on its output shape
2.   Special shape handling is implemented for operations like matmul, interleave, stack, etc.
3.   When incompatible shapes are detected, the algorithm either adjusts parameters or skips the node

`# Example: Calculating input shapes for matmul@staticmethoddef matmul_inputs(calculation_context: ShapeCalculationContext) -> List[TensorShape]:    output_shape = calculation_context.output_shape    batch_shape = output_shape[:-2]    m = output_shape[-2]    n = output_shape[-1]    q = randomize_size(len(batch_shape) + 1, test_context)    input_shapes = [batch_shape + (m,q), batch_shape + (q,n)]    return input_shapes`

### Fork-Join Pattern Generation

The algorithm naturally generates fork-join patterns when a node's output is connected to multiple subsequent nodes:

1.   When building a new node, it can connect to multiple existing nodes with compatible input shapes
2.   The algorithm controls fork-join density through configuration parameters
3.   This creates realistic network architectures with data flowing through multiple paths

## Using the Random Graph Generator

### Running RGG Tests

RGG tests can be run as standard pytest tests with configuration via environment variables:

`RANDOM_TEST_COUNT=5 MIN_DIM=4 MAX_OP_SIZE_PER_DIM=64 pytest -svv pybuda/test/random/test_graphs.py::test_random_graph_algorithm_pybuda`
When executed, the test will:

1.   Generate `RANDOM_TEST_COUNT` random graphs with the specified configuration
2.   Convert each graph to a PyBuda model
3.   Verify the model execution with `verify_module`
4.   Report test results and optionally save failing tests

## Advanced Features

### Rate Limiting

The RGG uses rate limiting to control certain aspects of graph generation:

*   **Constant Input Rate**: Controls how frequently constant tensors are used as inputs
*   **Same Input Reuse**: Limits how often the same input is connected to multiple inputs of a node

`constant_input_rate_limiter = RateLimiter(rng_shape, 100, test_context.randomizer_config.constant_input_rate)same_inputs_rate_limiter = RateLimiter(rng_shape, 100, test_context.randomizer_config.same_inputs_percent_limit)`

### Parameter Adjustment

When specific operations require special parameters, the system automatically adjusts them:

*   `AdjustParameters` class contains specialized methods for different operations
*   These methods modify parameters to ensure operations like stack, interleave, and concatenate work correctly
*   When adjustment isn't possible, the node is skipped
