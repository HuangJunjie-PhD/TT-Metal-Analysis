This page covers Tracy's user interface framework and rendering system, which powers the profiler GUI application. Tracy's UI is built on top of Dear ImGui with extensive custom extensions for profiler-specific visualization needs.

For information about the overall profiler application architecture, see [Application Overview](https://deepwiki.com/tenstorrent/tracy/4.1-application-overview). For details about specific UI views like statistics and analysis windows, see [Statistics & Analysis Views](https://deepwiki.com/tenstorrent/tracy/4.6-statistics-and-analysis-views).

## Core ImGui Integration

Tracy uses Dear ImGui as its foundational UI framework, providing immediate-mode GUI capabilities for the profiler application. The integration centers around Tracy's custom ImGui wrapper that extends the base functionality.

**Core ImGui Components Used by Tracy:**

| Component | Purpose | Key Files |
| --- | --- | --- |
| Core Framework | Basic windowing, input handling | `imgui/imgui.cpp`, `imgui/imgui.h` |
| Drawing System | Primitive rendering, draw lists | `imgui/imgui_draw.cpp` |
| Widget System | UI controls, interactions | `imgui/imgui_widgets.cpp` |
| Table System | Complex data tables | `imgui/imgui_tables.cpp` |

## Tracy's ImGui Extensions

Tracy significantly extends ImGui's capabilities through `TracyImGui.hpp`, which provides specialized utilities for profiler visualization and enhanced UI components.

**Key Extension Categories:**

**Color and Syntax Highlighting:**

*   `SyntaxColors[]` - Color palette for code syntax highlighting
*   `AsmOpTypeColors[]` - Assembly instruction type coloring
*   `AsmSyntaxColors[]` - Assembly syntax element colors

**Custom UI Components:**

*   `ToggleButton()` / `SmallToggleButton()` - Stateful toggle buttons with visual feedback
*   `ButtonDisablable()` / `SmallButtonDisablable()` - Buttons with disabled state handling
*   `SmallCheckbox()` - Compact checkbox variant
*   `ClipboardButton()` - Copy-to-clipboard functionality

**Enhanced Text Rendering:**

*   `TextCentered()` - Horizontally centered text
*   `TextColoredUnformatted()` - Colored text without formatting
*   `DrawTextContrast()` / `DrawTextSuperContrast()` - Text with outline for readability

## Custom Drawing System

Tracy implements specialized drawing functions for profiler-specific visualizations that go beyond standard ImGui primitives.

**Specialized Drawing Functions:**

*   **`DrawZigZag()`** - Renders zigzag patterns using texture mapping for timeline elements
*   **`DrawStripedRect()`** - Creates hatched/striped rectangles with screen-space or object-space alignment
*   **`DrawLine()`** - Multi-point line drawing with consistent styling
*   **`DrawWaitingDots()`** - Animated loading indicators with time-based animation

**Advanced Features:**

*   Screen-space stripe alignment for consistent visual patterns
*   Texture-based pattern rendering for performance
*   Clipping integration for complex UI layouts
*   Animation timing for dynamic elements

## UI Component Architecture

Tracy's UI is organized into specialized view classes that handle different aspects of profiler data visualization, all built on the ImGui foundation.

**View Class Responsibilities:**

| View Class | Purpose | Key UI Elements |
| --- | --- | --- |
| `TracyView_Statistics` | Performance metrics and zone statistics | Tables, charts, filtering controls |
| `TracyView_FindZone` | Zone search and analysis tools | Search interface, result lists, grouping |
| `TracyView_Memory` | Memory allocation visualization | Memory maps, allocation tables |
| `TracyView_Samples` | Sampling profiler data display | Sample lists, call stacks, statistics |
| `TracyView_ContextSwitch` | Thread context switch visualization | Timeline drawing, thread state info |
| `TracyView_ZoneInfo` | Detailed zone information windows | Property panels, call trees |

**Common UI Patterns:**

*   **Table-based data presentation** using `ImGui::BeginTable()`
*   **Filtering and search interfaces** with `ImGuiTextFilter`
*   **Custom drawing overlays** for timeline and visualization elements
*   **Modal dialogs and popup windows** for detailed information

## Rendering Pipeline

The Tracy UI follows ImGui's immediate-mode rendering model, where the entire interface is reconstructed each frame through function calls.

**Rendering Flow Details:**

1.   **Frame Initialization** - ImGui prepares for a new frame with input processing and state updates
2.   **View Processing** - Each Tracy view processes its data and constructs UI elements
3.   **Custom Rendering** - Tracy's specialized drawing functions add profiler-specific visualizations
4.   **Command Generation** - ImGui builds optimized draw command lists
5.   **Platform Execution** - Graphics backend renders the final frame to screen

**Performance Optimizations:**

*   **Activity Tracking** - `s_wasActive` flag minimizes unnecessary redraws
*   **Clipping and Culling** - UI elements outside view are skipped
*   **Texture Caching** - Pattern textures reused across frames
*   **Draw Call Batching** - ImGui optimizes rendering commands

**Key Integration Points:**

*   Font atlas management for text rendering
*   Texture loading for custom drawing patterns
*   Input event routing for interaction handling
*   Memory management for UI state
