This document describes the register access system in TTExaLens, which provides read and write operations for configuration and debug registers on NOC blocks. Register access enables low-level control and inspection of hardware state on Tensix cores, Ethernet cores, and other functional blocks.

For memory access operations on L1 and private memory regions, see [Memory Access Operations](https://deepwiki.com/tenstorrent/tt-exalens/3.3-memory-access-operations). For symbolic access to ELF variables, see [Symbolic Variable Access](https://deepwiki.com/tenstorrent/tt-exalens/3.8-symbolic-variable-access). For RISC-V core control via debug registers, see [RISC-V Core Control](https://deepwiki.com/tenstorrent/tt-exalens/3.6-risc-v-core-control).

## Overview

TTExaLens provides two types of register access:

| Register Type | Description | Addressing Method | Typical Use Cases |
| --- | --- | --- | --- |
| **Configuration Registers** | Hardware configuration registers with bit-field access | Index + mask + shift | Setting data formats, operand addresses, performance counters |
| **Debug Registers** | Debug and control registers with direct addressing | Offset address | Controlling debug features, instruction buffers, signal monitoring |

Registers are accessed through the public API functions `read_register()` and `write_register()`, which abstract the underlying NOC operations and provide coordinate conversion, safe mode validation, and NOC failover capabilities.

## Register Types and Descriptions

### Configuration Registers

Configuration registers are hardware configuration registers that often use bit-field masking. They are described by the `ConfigurationRegisterDescription` class with three parameters:

*   **index**: Register index (0 to max_config_register_index)
*   **mask**: Bit mask to isolate the field (e.g., `0x1E000000`)
*   **shift**: Number of bits to shift right after masking (e.g., `25`)

Configuration registers are typically accessed by name (e.g., `"ALU_FORMAT_SPEC_REG2_Dstacc"`, `"UNPACK_CONFIG0_out_data_format"`).

**Example configuration register access:**

`# Read ALU data format configurationdf = read_register("0,0", "ALU_FORMAT_SPEC_REG2_Dstacc") # Write using RegisterDescriptionfrom ttexalens.register_store import ConfigurationRegisterDescriptionreg = ConfigurationRegisterDescription(index=1, mask=0x1E000000, shift=25)write_register("0,0", reg, value=2)`
Sources: [test/ttexalens/unit_tests/test_lib.py 361-396](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_lib.py#L361-L396)[test/ttexalens/unit_tests/test_lib.py 397-437](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_lib.py#L397-L437)

### Debug Registers

Debug registers are accessed via direct offset addresses. They are described by the `DebugRegisterDescription` class with a single parameter:

*   **offset**: Memory offset address (e.g., `0x54`, `0xA0`)

Debug registers control debugging infrastructure such as instruction buffers, debug bus control, and array read operations. Common debug registers include `"RISCV_DEBUG_REG_DBG_BUS_CNTL_REG"`, `"RISCV_DEBUG_REG_DBG_ARRAY_RD_EN"`, and `"RISCV_DEBUG_REG_DBG_INSTRN_BUF_CTRL0"`.

**Example debug register access:**

`# Read debug bus control register by namebus_ctrl = read_register("0,0", "RISCV_DEBUG_REG_DBG_BUS_CNTL_REG") # Write using RegisterDescriptionfrom ttexalens.register_store import DebugRegisterDescriptionreg = DebugRegisterDescription(offset=0x54)write_register("0,0", reg, value=18)`

## API Functions

### read_register()

`def read_register(    location: str | OnChipCoordinate,    register: str | RegisterDescription,    noc_id: int = 0,    neo_id: int | None = None,    device_id: int = 0,    context: Context | None = None,    safe_mode: bool | None = None,) -> int`
Reads the value of a register from the specified NOC location.

**Parameters:**

*   `location`: Location on chip in string format (e.g., `"0,0"`, `"1-2"`) or `OnChipCoordinate` object
*   `register`: Register name (string) or `RegisterDescription` instance
*   `noc_id`: NOC ID to use (0 or 1, default: 0)
*   `neo_id`: NEO ID for multi-NEO systems (optional)
*   `device_id`: Device ID (default: 0)
*   `context`: TTExaLens context (default: global context)
*   `safe_mode`: Enable safe mode validation (default: from context)

**Returns:** Integer value of the register (with mask and shift applied for configuration registers).

**Example:**

`# Read configuration register by namedf = read_register("0,0", "ALU_FORMAT_SPEC_REG2_Dstacc") # Read debug register with explicit NOCstatus = read_register("1,1", "RISCV_DEBUG_REG_DBG_INSTRN_BUF_STATUS", noc_id=0)`
Sources: [ttexalens/tt_exalens_lib.py 497-533](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L497-L533)

### write_register()

`def write_register(    location: str | OnChipCoordinate,    register: str | RegisterDescription,    value: int,    noc_id: int = 0,    neo_id: int | None = None,    device_id: int = 0,    context: Context | None = None,    safe_mode: bool | None = None,) -> None`
Writes a value to a register at the specified NOC location.

**Parameters:**

*   `location`: Location on chip in string format or `OnChipCoordinate` object
*   `register`: Register name (string) or `RegisterDescription` instance
*   `value`: Integer value to write (will be masked and shifted for configuration registers)
*   `noc_id`: NOC ID to use (0 or 1, default: 0)
*   `neo_id`: NEO ID for multi-NEO systems (optional)
*   `device_id`: Device ID (default: 0)
*   `context`: TTExaLens context (default: global context)
*   `safe_mode`: Enable safe mode validation (default: from context)

**Example:**

`# Write configuration registerwrite_register("0,0", "ALU_FORMAT_SPEC_REG2_Dstacc", value=2) # Write debug register to enable array readswrite_register("0,0", "RISCV_DEBUG_REG_DBG_ARRAY_RD_EN", value=1)`

## Register Access Architecture

#### Register Access Flow

**Flow description:**

1.   User calls `read_register()` or `write_register()` with location string and register identifier
2.   Location is converted to `OnChipCoordinate` via `convert_coordinate()`
3.   `RegisterStore` is obtained from the NOC block at that coordinate
4.   For reads: `RegisterStore.read_register()` performs the read operation
5.   For writes: `RegisterStore.write_register()` performs the write operation
6.   Register operations translate to `noc_read32()` or `noc_write32()` calls
7.   Device layer handles NOC operations with safe mode validation and failover
8.   Hardware access occurs via NOC0 or NOC1

Sources: [ttexalens/tt_exalens_lib.py 497-572](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L497-L572)[ttexalens/device.py 329-372](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/device.py#L329-L372)

#### Register Description Classes

The `RegisterStore` class maintains a dictionary mapping register names to their descriptions. When a string name is provided, it is looked up to obtain the corresponding `RegisterDescription` object.

Sources: [ttexalens/tt_exalens_lib.py 523-533](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L523-L533)[test/ttexalens/unit_tests/test_lib.py 397-437](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_lib.py#L397-L437)

## Implementation Details

### RegisterStore Class

The `RegisterStore` class is the core abstraction for register access. It maintains:

*   **register_descriptions**: Dictionary mapping register names to `RegisterDescription` objects
*   **_max_config_register_index**: Maximum valid configuration register index
*   NOC ID and NEO ID for multi-NOC/multi-NEO systems

Key methods:

*   `read_register(register, safe_mode)`: Reads register value with optional validation
*   `write_register(register, value, safe_mode)`: Writes register value with optional validation

Each NOC block provides its own `RegisterStore` instance via the `get_register_store(noc_id, neo_id)` method.

Sources: [ttexalens/tt_exalens_lib.py 523-572](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L523-L572)

### Configuration Register Access

Configuration registers use index-based addressing with bit-field masking:

1.   Read operation: `value = (raw_data & mask) >> shift`
2.   Write operation: `raw_data = (raw_data & ~mask) | ((value << shift) & mask)`

The index determines which configuration register to access, while mask and shift extract or insert the specific bit field.

**Index bounds checking:**

`# Valid index range: 0 to _max_config_register_indexif index < 0 or index > max_index:    raise ValueError("Configuration register index out of bounds")`
Sources: [test/ttexalens/unit_tests/test_lib.py 523-550](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_lib.py#L523-L550)

### Debug Register Access

Debug registers use direct offset addressing without bit-field operations:

1.   Read operation: `value = noc_read32(base_address + offset)`
2.   Write operation: `noc_write32(base_address + offset, value)`

The offset is added to the debug register base address for the NOC block.

Sources: [ttexalens/debug_tensix.py 74-116](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/debug_tensix.py#L74-L116)

### Safe Mode Validation

When safe mode is enabled, register access operations validate against the memory map to ensure the register address is within known and accessible regions. This prevents accidental writes to undefined or dangerous memory locations.

If safe mode is not explicitly specified, the context's safe mode setting is used.

Sources: [ttexalens/tt_exalens_lib.py 497-572](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L497-L572)[ttexalens/device.py 248-300](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/device.py#L248-L300)

### NOC Failover

Register access operations support automatic NOC failover:

1.   Register operations attempt to use the active NOC (NOC0 or NOC1 based on context)
2.   If a timeout occurs, the operation automatically retries on the alternate NOC
3.   If both NOCs fail, a `TimeoutDeviceRegisterError` is raised
4.   The active NOC can be switched via `device.switch_noc(noc_id)`

Sources: [ttexalens/device.py 186-216](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/device.py#L186-L216)[ttexalens/device.py 329-372](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/device.py#L329-L372)

### Multi-NOC and Multi-NEO Support

The register access system supports:

*   **Multiple NOCs**: Register operations can target NOC0 or NOC1 explicitly via `noc_id` parameter
*   **Multiple NEOs**: Some blocks have multiple NEO instances, accessed via `neo_id` parameter
*   **Per-NOC RegisterStores**: Each NOC block can provide separate `RegisterStore` instances for different NOC/NEO combinations

### Invalid Register Types

`# Raises TTException: register must be str or RegisterDescriptionread_register("0,0", 0xFFB12345)  # Invalid: address alone is insufficient`

`# Raises TTException: register name not foundread_register("0,0", "invalid_register_name")`
### Index Out of Bounds

`from ttexalens.register_store import ConfigurationRegisterDescription # Raises ValueError: index beyond valid rangeinvalid_reg = ConfigurationRegisterDescription(index=2**14)read_register("0,0", invalid_reg)`
