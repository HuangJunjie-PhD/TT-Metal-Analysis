## Purpose and Scope

The Programmable Interrupt Controller (PIC) provides interrupt management and clock gating capabilities for RISC-V cores in Tensix and Ethernet tiles. This document covers:

*   IRQ tracking and polling interface
*   Clock gating control for RISC-V NC (Tensix tiles) and RISC-V E (Ethernet tiles)
*   Memory-mapped register specifications

For details on the RISC-V cores themselves, see [Baby RISC-V Cores](https://deepwiki.com/tenstorrent/tt-isa-documentation/3.1-baby-risc-v-cores). For NoC Overlay stream configuration that generates IRQs, see the NoC documentation.

**Sources:**[WormholeB0/TensixTile/PIC.md 1-101](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L1-L101)

## Overview

The PIC provides a polling-based interrupt system rather than a hardware interrupt mechanism. RISC-V cores must explicitly poll the PIC to determine if any IRQs are pending. The PIC's key capabilities are:

*   **IRQ Management:** Track up to 11 different IRQ sources, with atomic hardware-side raising and software-side clearing
*   **Clock Gating:** Optionally stop the RISC-V NC (Tensix) or RISC-V E (Ethernet) clock when no IRQs are pending, reducing power consumption

The PIC is located in the memory map at base address `0xFFB1_3000` and is accessible from all RISC-V cores within the tile.

**Sources:**[WormholeB0/TensixTile/PIC.md 1-8](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L1-L8)[WormholeB0/TensixTile/BabyRISCV/README.md 86](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/BabyRISCV/README.md?plain=1#L86-L86)

## IRQ Sources and Indices

The PIC recognizes 11 active IRQ sources, mapped to specific bit positions in the 32-bit IRQ bitmasks:

| IRQ Index | Source | Enable Condition |
| --- | --- | --- |
| 0-3 | NoC Overlay streams 0-3 | `PIC_ENABLE.Bit[i]` AND `PIC_STREAM_IRQ_TYPE[i] != 0` |
| 4-7 | NoC Overlay streams 8-11 | `PIC_ENABLE.Bit[i]` AND `PIC_STREAM_IRQ_TYPE[i] != 0` |
| 8-15 | Unused | `PIC_ENABLE.Bit[i]` |
| 16 | NoC Overlay auto-config done | `PIC_ENABLE.Bit[16]` |
| 17 | Debug timestamper `bufs0_full` | `PIC_ENABLE.Bit[17]` (Tensix only) |
| 18 | Debug timestamper `bufs1_full` | `PIC_ENABLE.Bit[18]` (Tensix only) |
| 19-31 | Unused | Never enabled |

### Edge-Triggered Semantics

All IRQs are **edge-triggered**: when a condition transitions from `false` to `true`, the corresponding bit is atomically set in the pending set (if the IRQ is enabled). Once software clears the bit by writing to `PIC_STATUS`, the IRQ will only be set again when the condition transitions from `false` to `true` again.

This means that if an IRQ condition remains `true` while software clears it, the IRQ will **not** immediately re-assert. The condition must go `false` and then `true` again to raise a new IRQ.

**Sources:**[WormholeB0/TensixTile/PIC.md 10-24](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L10-L24)

### NoC Overlay Stream IRQs (0-7)

**Diagram: NoC Overlay Stream IRQ Configuration**

NoC Overlay streams 0-3 and 8-11 can raise IRQs when they start or end a phase. The `PIC_STREAM_IRQ_TYPE[i]` register controls which transitions raise IRQs, and `PIC_ENABLE.Bit[i]` must also be set for the IRQ to be added to the pending set.

**Sources:**[WormholeB0/TensixTile/PIC.md 14-18](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L14-L18)[WormholeB0/TensixTile/PIC.md 90-100](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L90-L100)

## Clock Gating

The PIC can automatically gate (stop) the clock to RISC-V NC (in Tensix tiles) or RISC-V E (in Ethernet tiles) when no IRQs are pending and no recent activity has occurred. This reduces power consumption during idle periods.

### Clock Running Conditions

The RISC-V NC/E clock will run if **any** of the following occurred in the previous `PIC_CLKGT.HYST` cycles:

*   `PIC_CLKGT.GATING_ENABLED` was `false` (default state - clock always runs)
*   `PIC_CLKGT.FORCE_ACTIVE` was `true`
*   Any IRQ was pending (i.e., `PIC_STATUS != 0`)
*   Any agent read from or wrote to any PIC register
*   Other miscellaneous tile activity (non-exhaustive list)

The hysteresis counter (`PIC_CLKGT.HYST`) prevents rapid clock toggling by keeping the clock running for a configurable number of cycles after the last activity.

## Register Reference

All PIC registers are located at base address `0xFFB1_3000`. The table below summarizes the registers:

| Register Name | Offset | Access | Purpose |
| --- | --- | --- | --- |
| `PIC_STATUS` | `0x0000` | RW | Read: query pending IRQs Write: clear pending IRQs |
| `PIC_NEXT_IRQ` | `0x0004` | RO | Read: iterate through pending IRQ indices |
| `PIC_ENABLE` | `0x0008` | RW | Bitmask of enabled IRQs |
| `PIC_CLKGT_EN` | `0x000C` | RW | Clock gating enable and control |
| `PIC_CLKGT_HYST` | `0x0010` | RW | Clock gating hysteresis (cycles) |
| `PIC_STREAM_IRQ_TYPE[0-7]` | `0x0014 + i*4` | RW | Stream phase start/end IRQ configuration |

**Sources:**[WormholeB0/TensixTile/PIC.md 43-52](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L43-L52)

### PIC_STATUS (0xFFB1_3000)

**Read Access:** Returns the 32-bit bitmask of pending IRQs. Does not modify any state.

**Write Access:** Atomically clears the specified bits from the pending IRQ bitmask. Writing a `1` to a bit position clears that IRQ; writing a `0` has no effect. Clearing a non-pending IRQ has no effect.

**Sources:**[WormholeB0/TensixTile/PIC.md 54-60](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L54-L60)

### PIC_NEXT_IRQ (0xFFB1_3004)

**Read Access:** Returns the **index** (0-31) of one pending IRQ if `PIC_STATUS != 0`, or returns `0` if no IRQs are pending. Reading this register does not modify `PIC_STATUS`, but **does** modify internal iteration state: repeated reads will cycle through all pending IRQ indices.

**Ambiguity:** When IRQ #0 is enabled, a read result of `0` is ambiguous - it could mean either no IRQs are pending, or IRQ #0 is pending. Software should check `PIC_STATUS` first to resolve this ambiguity.

**Typical Usage Pattern:**

**Sources:**[WormholeB0/TensixTile/PIC.md 62-66](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L62-L66)

### PIC_ENABLE (0xFFB1_3008)

A 32-bit bitmask of enabled IRQs. Initially zero at reset. Software sets bits corresponding to the IRQs it wants to monitor. An IRQ will only be added to the pending set if its corresponding enable bit is set.

**Format:**

| Bits | Description |
| --- | --- |
| [31:0] | Enable mask (1 = IRQ enabled, 0 = IRQ disabled) |

**Sources:**[WormholeB0/TensixTile/PIC.md 68-70](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L68-L70)

### PIC_CLKGT_EN (0xFFB1_300C)

Clock gating enable and control register.

| Bits | Field | Access | Description |
| --- | --- | --- | --- |
| [0] | `GATING_ENABLED` | RW | `1` = enable clock gating, `0` = clock always runs |
| [7:1] | Reserved | RO | Reads as `0`, writes ignored |
| [8] | `FORCE_ACTIVE` | RW | `1` = force clock to run (when gating enabled), `0` = allow automatic gating |
| [15:9] | Reserved | RO | Reads as `0`, writes ignored |
| [16] | `ACTIVE` | RO | `1` = clock is running, `0` = clock is gated (Local RISC-V always reads `1`; only remote agents can read `0`) |
| [31:17] | Reserved | RO | Reads as `0`, writes ignored |

**Sources:**[WormholeB0/TensixTile/PIC.md 72-81](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L72-L81)

### PIC_CLKGT_HYST (0xFFB1_3010)

Clock gating hysteresis register. Specifies the number of cycles the clock should remain running after the last activity.

| Bits | Field | Access | Description |
| --- | --- | --- | --- |
| [6:0] | `HYST` | RW | Hysteresis cycle count (0-127) |
| [31:7] | Reserved | RO | Reads as `0`, writes ignored |

Default value is `3` cycles, which may be aggressive depending on the IRQ service pattern.

**Sources:**[WormholeB0/TensixTile/PIC.md 83-88](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L83-L88)

### PIC_STREAM_IRQ_TYPE[0-7] (0xFFB1_3014 + i*4)

Eight registers controlling NoC Overlay stream IRQ behavior. `PIC_STREAM_IRQ_TYPE[0-3]` correspond to streams 0-3, while `PIC_STREAM_IRQ_TYPE[4-7]` correspond to streams 8-11.

| Bits | Field | Access | Description |
| --- | --- | --- | --- |
| [0] | Phase start | RW | `1` = raise IRQ when stream starts a phase |
| [1] | Phase end | RW | `1` = raise IRQ when stream ends a phase |
| [31:2] | Reserved | RO | Reads as `0`, writes ignored |

**Note:** The corresponding bit in `PIC_ENABLE` must also be set for the IRQ to be added to the pending set.

**Sources:**[WormholeB0/TensixTile/PIC.md 90-100](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/PIC.md?plain=1#L90-L100)

## Memory Map Integration

The PIC is accessible from all RISC-V cores within the tile (B, T0, T1, T2, NC in Tensix tiles; E in Ethernet tiles). The address range `0xFFB1_3000` to `0xFFB1_3FFF` is reserved for PIC registers.
