# Bounty Program

Relevant source files
*   [core/bounty_terms.md](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1)

This page provides a comprehensive overview of the Tenstorrent Bounty Program, which allows community contributors to help improve Tenstorrent's open-source software and earn rewards for their contributions. The document outlines eligibility requirements, contribution workflow, scope of eligible repositories, reward categories, and payment terms.

Sources: [core/bounty_terms.md 9-11](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L9-L11)

## Program Overview

The Tenstorrent Bounty Program is a community initiative designed to incentivize external contributors to help improve Tenstorrent's open-source projects. Unlike a competition, this program rewards qualifying individuals for addressing issues, implementing new features, or resolving performance challenges in designated repositories.

Contributors who submit "Accepted Contributions" (as defined below) can earn financial rewards ranging from $1 to $3,000 based on the difficulty level of the issue they solve.

## Qualifying Contributions

An **"Accepted Contribution"** refers to a merged pull request that meets the following criteria:

1.   **Repository Match**: Must be submitted to one of the 16 eligible repositories listed above [core/bounty_terms.md 33-50](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L33-L50)
2.   **Issue Tagging**: The PR must address an open GitHub issue tagged with: 
    *   `bounty`[core/bounty_terms.md 51](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L51-L51)
    *   One of the difficulty categories (e.g., `bounty_difficulty/warmup`) [core/bounty_terms.md 51](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L51-L51)

3.   **Assignment Requirement**: The contributor must be assigned to the issue on GitHub. Tenstorrent may re-assign the issue if the contributor is unresponsive for over two (2) weeks [core/bounty_terms.md 59-60](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L59-L60)
4.   **Licensing**: Contributions must be released under the specific license of the target repository [core/bounty_terms.md 61](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L61-L61)

## Reward Categories

The bounty rewards are categorized into four difficulty levels. The exact payment amount within each range is determined at Tenstorrent’s sole discretion.

| Category Tag | Definition | Examples | Payout Range (USD) |
| --- | --- | --- | --- |
| `bounty_difficulty/warmup` | Tasks for first-time contributors; low complexity. | Minor bug fixes, documentation, test cases, README updates. | $1 - $200 |
| `bounty_difficulty/easy` | Requires basic repo familiarity and architecture understanding. | Extending features, API updates, simple refactoring, new test suites. | $201 - $500 |
| `bounty_difficulty/medium` | Requires significant code base or domain knowledge. | Implementing features, new model support, performance debugging, library integration. | $501 - $1,999 |
| `bounty_difficulty/hard` | Complex tasks demanding deep architectural expertise. | Major feature implementation, core system redesign, new low-level kernels. | $2,000 - $3,000 |

Sources: [core/bounty_terms.md 131-138](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L131-L138)

## Participation Eligibility

To be eligible for a Bounty, a Participant must:

*   Be of legal age of majority in their country [core/bounty_terms.md 20](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L20-L20)
*   Not be a sanctioned person or resident of a sanctioned country (e.g., under U.S. embargo) [core/bounty_terms.md 23](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L23-L23)
*   Not be a current employee, vendor, contractor, or agent for Tenstorrent [core/bounty_terms.md 26](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L26-L26)
*   Comply with all applicable laws and regulations [core/bounty_terms.md 24](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L24-L24)

Sources: [core/bounty_terms.md 18-28](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L18-L28)

## Receiving Payment

The process for receiving a Payout involves:

1.   **PR Merger**: The contribution must be merged into the main branch of the repository [core/bounty_terms.md 51](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L51-L51)
2.   **Verification**: Participant must not be in breach of any Terms [core/bounty_terms.md 59](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L59-L59)
3.   **Information Submission**: Provide requested payment and tax information as required by law [core/bounty_terms.md 62](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L62-L62)
4.   **No Assignments**: Bounties cannot be designated to third parties; they are paid directly to the contributor [core/bounty_terms.md 63](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L63-L63)

*   **Governing Law**: Disputes are adjudicated solely in the courts of the State of California [core/bounty_terms.md 5](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L5-L5)
*   **Discretionary Nature**: Tenstorrent may modify, suspend, or terminate the program or specific payouts at any time [core/bounty_terms.md 13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/bounty_terms.md?plain=1#L13-L13)
