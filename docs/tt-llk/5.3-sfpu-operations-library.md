## Purpose and Scope

The SFPU (Special Function Processing Unit) Operations Library provides a comprehensive collection of mathematical and specialized computational functions that execute on the SFPU hardware unit within the Tensix core. This library includes over 50 operations ranging from basic mathematical functions (exp, log, sqrt) to advanced neural network activations (GELU, SILU) and specialized operations (quantization, dropout, topk).

The SFPU operates on data stored in destination registers after the FPU has completed its computation. For basic element-wise operations that do not require SFPU hardware, see [Element-wise Operations](https://deepwiki.com/tenstorrent/tt-llk/5.2-element-wise-operations). For matrix multiplication operations, see [Matrix Multiplication Operations](https://deepwiki.com/tenstorrent/tt-llk/5.4-matrix-multiplication-operations).

## SFPU in the Hardware Pipeline

The SFPU is a dedicated hardware unit within the math processing pipeline that provides specialized computational capabilities beyond standard FPU operations. It operates on data stored in destination registers and uses dedicated SFPU local registers (LREG0-7) for intermediate computations.

## SFPU Programming Patterns

### LREG Register Usage

The SFPU has 8 local registers (LREG0-7) used for intermediate computations. A common pattern divides them into input and output register groups.

**Sources:**[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 59-60](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L59-L60)

### Face-Based Tile Processing

Tiles are organized as 32x32 elements divided into four 16x16 faces. SFPU operations typically process data in 4-row chunks using face-aware addressing.

**Sources:**[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 64-74](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L64-L74)

### SFPLOAD/SFPSTORE Pattern

SFPU operations use `SFPLOAD` to read from destination registers into LREG registers and `SFPSTORE` to write results back. These instructions operate on 4-row granularity.

**Sources:**[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 87-94](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L87-L94)[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 109-113](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L109-L113)

### Transpose Operations

The `SFPTRANSP` instruction is used to rearrange data when working with 4-row SFPLOAD/SFPSTORE granularity but needing to access individual rows.

**Sources:**[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 95-98](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L95-L98)[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 105-108](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L105-L108)

## Advanced Example: Reshuffle Rows Operation

The reshuffle rows operation demonstrates advanced SFPU programming techniques including mask-based row selection, scatter-add accumulation, and complex addressing patterns.

### Operation Overview

### Implementation Details

The implementation at [tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 40-115](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L40-L115) shows:

1.   **Mask Access**: Reads destination row indices from L1 memory at `idx_addr + 16` (skipping tile header)
2.   **Row-by-Row Processing**: Iterates through all 32 rows, checking mask validity
3.   **Address Calculation**: Computes face-aware addresses for both input and output rows
4.   **Parallel Load**: Loads 4 rows at once for both input and output, processing even/odd columns
5.   **Transpose-Compute-Transpose**: Uses transpose operations to isolate target rows within 4-row blocks
6.   **Accumulation**: Performs `output[dst_row] += input[row]` using `SFPADD` with `LCONST_1` multiplier
7.   **Conditional Skip**: Skips rows where mask value is 255 or >= 32

**Sources:**[tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 18-115](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L18-L115)[tt_llk_blackhole/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h 18-115](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/sfpu/ckernel_sfpu_reshuffle_rows.h#L18-L115)

## SFPU Instruction Modifiers

SFPU load and store instructions support modifiers that control data format interpretation. These are defined in the `InstrModLoadStore` enumeration.

### Instruction Modifier Enumeration

| Modifier | Value | Description | Usage |
| --- | --- | --- | --- |
| `DEFAULT` | 0 | Default format (BFP8, BFP4, BFP2, Lf8) | Block floating-point formats |
| `FP16A` | 1 | FP16A format | Float16 data |
| `FP16B` | 2 | FP16B (bfloat16) format | Float16_b data |
| `FP32` | 3 | FP32 format | Float32, Tf32 data |
| `INT32` | 4 | INT32 format | Int32, UInt32 data |

## Synchronization and Control Flow

### SFPU Stall/Wait Pattern

SFPU operations require careful synchronization with the rest of the math pipeline using stall and wait instructions:

1.   **Before SFPU**: `TTI_STALLWAIT(p_stall::STALL_SFPU, p_stall::MATH)` - Stall SFPU operations until math unit completes
2.   **After SFPU**: `TTI_STALLWAIT(p_stall::STALL_CFG, p_stall::WAIT_SFPU)` - Stall configuration changes until SFPU completes

## Architecture-Specific Considerations

### Wormhole B0 vs Blackhole

Both architectures share the same SFPU operation library structure, with minor differences:

| Aspect | Wormhole B0 | Blackhole |
| --- | --- | --- |
| Core header | `tt_llk_wormhole_b0/common/inc/ckernel_sfpu.h` | `tt_llk_blackhole/common/inc/ckernel_sfpu.h` |
| Operation count | 50+ operations | 49 operations (missing `welfords`) |
| Address modifiers | ADDR_MOD_3/6/7 | ADDR_MOD_7 (different pattern) |
| SFPNOP usage | Required after some operations | Less frequent requirement |
