The `tt-sglang-plugin` enables Tenstorrent hardware acceleration within the SGLang runtime environment. It implements a patching mechanism to inject Tenstorrent-optimized LLM models into the SGLang `ModelRegistry`, allowing users to leverage SGLang's advanced features like RadixAttention and structured output while executing compute on Tenstorrent N150, N300, and T3K devices.

## Architecture Overview

The plugin operates by intercepting SGLang's model loading process and substituting standard implementations with `ttnn`-based model classes. It manages complex device orchestration, including multi-worker data parallelism and mesh device configurations for multi-chip systems.

## Environment Setup and Installation

The plugin requires a specific installation order to ensure that CPU-based PyTorch and SGLang kernels do not conflict with Tenstorrent's hardware requirements.

### `setup_sglang_tt.sh` Workflow

The setup script automates the creation of a specialized environment:

1.   **SGLang Cloning**: Clones the SGLang repository [tt-sglang-plugin/setup_sglang_tt.sh 43-48](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/setup_sglang_tt.sh#L43-L48)
2.   **Venv Creation**: Initializes a Python virtual environment (`sglang-venv`) [tt-sglang-plugin/setup_sglang_tt.sh 53-60](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/setup_sglang_tt.sh#L53-L60)
3.   **CPU PyTorch**: Installs `torch==2.5.0+cpu` specifically to avoid GPU-specific bloat and conflicts [tt-sglang-plugin/setup_sglang_tt.sh 74-76](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/setup_sglang_tt.sh#L74-L76)
4.   **SGLang CPU Build**: Installs SGLang using the `pyproject_cpu.toml` configuration [tt-sglang-plugin/setup_sglang_tt.sh 81-92](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/setup_sglang_tt.sh#L81-L92)
5.   **Plugin Integration**: Installs the `sglang-tt-plugin` in editable mode [tt-sglang-plugin/setup_sglang_tt.sh 131-138](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/setup_sglang_tt.sh#L131-L138)

The script generates an `activate_sglang_tt.sh` script that sets critical environment variables like `LD_LIBRARY_PATH` and `VIRTUAL_ENV`[tt-sglang-plugin/setup_sglang_tt.sh 143-172](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/setup_sglang_tt.sh#L143-L172)

## Server Launch and Model Registration

The `sglang-tt-server` script is the primary entry point. It configures the SGLang engine to use the CPU-based scheduler while offloading tensor computations to the TT device.

### Launch Sequence

1.   **Multiprocessing Setup**: Forces the `fork` start method to ensure child processes inherit the patched `ModelRegistry`[tt-sglang-plugin/sglang_tt_plugin/scripts/launch_tt_server.py 31-38](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/scripts/launch_tt_server.py#L31-L38)
2.   **CPU Engine Configuration**: Sets `SGLANG_USE_CPU_ENGINE="1"` and `VLLM_DEVICE_TYPE="cpu"`[tt-sglang-plugin/sglang_tt_plugin/scripts/launch_tt_server.py 16-25](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/scripts/launch_tt_server.py#L16-L25)
3.   **Model Patching**: Upon import, `register_tt_models` updates SGLang's `ModelRegistry` mapping architectures (e.g., `LlamaForCausalLM`) to TT-specific classes (e.g., `TTLlamaForCausalLM`) [tt-sglang-plugin/sglang_tt_plugin/patching/patching_model_registry.py 30-38](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/patching/patching_model_registry.py#L30-L38)
4.   **Hardware Discovery**: Parses `--tt-visible-devices` and `--mesh-shape` to determine the Data Parallel (DP) size and chip layout [tt-sglang-plugin/sglang_tt_plugin/scripts/launch_tt_server.py 113-120](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/scripts/launch_tt_server.py#L113-L120)

### Key Functions

*   `_build_page_table`: Extracts block IDs from SGLang's `ForwardBatch` to map logical sequences to physical KV cache blocks on the TT device [tt-sglang-plugin/sglang_tt_plugin/models/tt_llm.py 90-92](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/models/tt_llm.py#L90-L92)
*   `allocate_on_device`: Dynamically calculates the maximum token capacity based on the hardware architecture (e.g., Wormhole B0) and model path, then allocates the `ttnn` KV cache [tt-sglang-plugin/sglang_tt_plugin/models/tt_llm.py 152-182](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/models/tt_llm.py#L152-L182)
*   `setup_worker_from_process_title`: Critical for multi-worker DP; it parses the process title (e.g., `_DP1_TP0`) via `setproctitle` to isolate each worker's `TT_METAL_CACHE` and `TT_VISIBLE_DEVICES`[tt-sglang-plugin/sglang_tt_plugin/worker_setup/worker_setup.py 11-34](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/worker_setup/worker_setup.py#L11-L34)

### Device Opening Logic

1.   **Mesh Shape Resolution**: Checks `DEVICE_MESH_SHAPE` environment variable (passed via `--mesh-shape`) or defaults to a 1xN grid [tt-sglang-plugin/sglang_tt_plugin/utils/tt_utils.py 118-132](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/utils/tt_utils.py#L118-L132)
2.   **Fabric Configuration**: For multi-device setups, `_configure_fabric` is called before opening the mesh to enable inter-chip communication (e.g., `FABRIC_1D` or `FABRIC_2D`) [tt-sglang-plugin/sglang_tt_plugin/utils/tt_utils.py 156-180](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/utils/tt_utils.py#L156-L180)
3.   **Trace Support**: Allocates a default `trace_region_size` of ~50MB to support hardware command tracing [tt-sglang-plugin/sglang_tt_plugin/utils/tt_utils.py 31-35](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-sglang-plugin/sglang_tt_plugin/utils/tt_utils.py#L31-L35)
