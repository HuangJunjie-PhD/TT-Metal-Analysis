This page documents the tensor analysis features in the TT-NN Visualizer, which allow users to examine tensor properties, memory allocations, data layouts, and relationships between tensors in a neural network model. Tensor analysis provides insights into how data flows through the model and how memory is utilized.

For memory visualization features including L1 and DRAM plots, see [Memory Visualization](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.3-memory-visualization).

## 2. Tensor Details View

The Tensor Details View provides comprehensive information about individual tensors, including:

*   Tensor identifier, shape, data type, and layout
*   Memory allocation address and buffer type (L1, DRAM)
*   Memory layout configuration and sharding specification
*   Producer and consumer operations
*   Lifecycle information (creation, last use, deallocation)

### 2.1 Accessing Tensor Details

Tensor details can be accessed through:

1.   The Operation Details view, which lists all tensors involved in an operation
2.   The Tensor List view, which provides a filterable list of all tensors in the model
3.   Clicking on tensor representations in memory plots

Sources: [src/components/operation-details/OperationDetailsComponent.tsx 348-356](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/OperationDetailsComponent.tsx#L348-L356)[src/components/operation-details/TensorDetailsComponent.tsx 33-177](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/TensorDetailsComponent.tsx#L33-L177)

### 2.2 Tensor Property Visualization

The TensorDetailsComponent displays key properties of a tensor:

| Property | Description |
| --- | --- |
| Tensor ID | Unique identifier for the tensor |
| Address | Memory address where tensor is allocated |
| Buffer Type | Type of memory buffer (L1, DRAM, etc.) |
| Shape | Dimensions of the tensor |
| Dtype | Data type of tensor elements |
| Layout | Memory layout pattern (e.g., rowmajor, colmajor) |
| Memory Config | Additional memory configuration details |
| Sharding | How the tensor is distributed across cores |

## 3. Tensor Memory Layout Analysis

Tensor memory layout analysis helps visualize how tensors are allocated in memory and distributed across processing cores.

### 3.1 Memory Allocation Visualization

The tensor memory allocation visualization shows:

1.   How tensors are laid out in L1 or DRAM memory
2.   Memory address ranges for each tensor
3.   Allocation and deallocation patterns

Sources: [src/components/operation-details/OperationDetailsComponent.tsx 177-246](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/OperationDetailsComponent.tsx#L177-L246)[src/components/operation-details/OperationDetailsComponent.tsx 264-332](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/OperationDetailsComponent.tsx#L264-L332)

### 3.2 Tensor Sharding Visualization

The TT-NN Visualizer provides specialized visualization for how tensors are sharded or distributed across cores:

1.   **All allocations per core**: Shows how all tensor data is distributed across processing cores
2.   **Input/Output allocations per core**: Focuses on I/O tensor distribution

The sharding visualization is accessed through dedicated buttons in the Operation Details view and provides a detailed view of tensor memory allocation patterns.

## 4. Tensor Relationships and Data Flow

The TT-NN Visualizer provides tools to analyze the relationships between tensors and operations, helping users understand data flow through the network.

### 4.1 Producer-Consumer Analysis

For each tensor, the visualizer tracks and displays:

*   Producer operations that create or modify the tensor
*   Consumer operations that use the tensor
*   Unusually high numbers of consumers (flagged as potential issues)

Sources: [src/components/BufferDetails.tsx 24-153](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/BufferDetails.tsx#L24-L153)[src/components/TensorList.tsx 308-320](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/TensorList.tsx#L308-L320)

### 4.2 Tensor Buffer Lifecycle

The visualizer tracks the complete lifecycle of tensor buffers:

1.   Initial allocation by a producer operation
2.   Usage by consumer operations
3.   Last use before deallocation
4.   Deallocation
5.   Next reallocation of the same memory address

This helps users understand memory reuse patterns and identify potential memory issues.

## 5. Tensor List View

The Tensor List view provides a comprehensive list of all tensors in the model with powerful filtering and sorting capabilities.

### 5.1 Filtering and Navigation

The TensorList component offers several filtering options:

*   Text-based filtering by tensor properties
*   Buffer type filtering (L1, DRAM, etc.)
*   Focus on tensors with high numbers of consumers
*   Range selection to show tensors within specific operation ranges

## 6. Golden Tensor Comparison

The visualizer supports comparison of tensors against golden or reference values, which is useful for validation and debugging.

### 6.1 Comparison Metrics

Tensor comparisons include:

*   Actual PCC (Pearson Correlation Coefficient)
*   Desired PCC threshold
*   Both local and global matching results

Sources: [src/components/operation-details/TensorDetailsComponent.tsx 152-163](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/TensorDetailsComponent.tsx#L152-L163)[src/components/BufferDetails.tsx 104-134](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/BufferDetails.tsx#L104-L134)

### 6.2 Visualization of Comparison Results

Comparison results are displayed through the GoldenTensorComparisonIndicator component, which shows:

*   Whether the tensor matches the golden reference
*   The actual correlation value
*   The desired threshold for matching

This helps users quickly identify tensors that don't match expected values.

## 7. Integration with Operation Details

The tensor analysis features are tightly integrated with operation analysis, allowing users to navigate seamlessly between operations and their associated tensors.

### 7.1 Operation-Tensor Navigation

Users can:

1.   View all tensors used by a specific operation
2.   From a tensor, navigate to its producer and consumer operations
3.   Visualize how tensors flow between operations

### 7.2 Device Operation Integration

For operations that involve device-specific operations, the visualizer shows:

*   Device operations graph that includes tensor data flow
*   Per-device tensor allocations
*   Core-specific tensor sharding details

This integration provides a complete picture of how tensors are processed within the neural network model.
