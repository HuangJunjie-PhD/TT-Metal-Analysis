The `tt-awesome` repository includes a Claude Code plugin integration designed to streamline the contribution process. This integration allows users to interact with the repository using natural language commands, specifically for adding new projects to the curated Tenstorrent ecosystem list.

## Plugin Manifests

The plugin is defined by two primary configuration files located in the `.claude-plugin/` directory. These files establish the plugin's identity, authorship, and source location.

### plugin.json

The `.claude-plugin/plugin.json` file serves as the local manifest for the Claude Code environment. It defines the human-readable name and description that appear when the plugin is loaded in a workspace.

| Field | Value |
| --- | --- |
| `name` | `tt-awesome` |
| `description` | Claude Code plugin for the Awesome Tenstorrent list. Provides `/tt-awesome:add-project` to guide contributors through submitting new entries. |
| `author` | `Tenstorrent` |

**Sources:**

*   [.claude-plugin/plugin.json 1-7](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.claude-plugin/plugin.json#L1-L7)

### ID Generation Logic

To maintain parity with the automated `scripts/issue_to_entry.py` utility, the plugin implements a specific `slugify` logic for generating the `id` field from the project name:

1.   Convert to lowercase.
2.   Strip non-word characters (except hyphens).
3.   Replace whitespace/underscores with `-`.
4.   Collapse consecutive hyphens.
5.   Strip leading/trailing hyphens.

## Contribution Path Execution

The plugin facilitates three distinct exit paths depending on the user's environment and permissions.

**Title: Plugin Execution Paths and Tool Usage**

### Path Specifications

*   **Path A (Issue):** Uses `gh issue create` with a body formatted in `### Heading` sections. This format is required for compatibility with the repository's `issue_to_entry.py` automation which parses GitHub issues into JSON files. [[skills/add-project/SKILL.md:153-212]](https://deepwiki.com/tenstorrent/tt-awesome/6.1-plugin-configuration)
*   **Path B (Local):** Writes the JSON file to the local file system and executes the `scripts/validate.py` script to ensure schema compliance before providing git instructions to the user. [[skills/add-project/SKILL.md:214-230]](https://deepwiki.com/tenstorrent/tt-awesome/6.1-plugin-configuration)
*   **Path C (PR):** Automates the local write, validation, and branch creation, concluding with `gh pr create`. [[skills/add-project/SKILL.md:232-236]](https://deepwiki.com/tenstorrent/tt-awesome/6.1-plugin-configuration)
