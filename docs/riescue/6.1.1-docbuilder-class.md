# DocBuilder Class

Relevant source files
*   [docs/README.md](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/README.md?plain=1)
*   [docs/build.py](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py)

## Purpose and Scope

The `DocBuilder` class is the core implementation of the RiESCUE documentation build system, providing a Python wrapper around Sphinx's `sphinx-build` command with enhanced functionality for local development and CI/CD integration. This page documents the internal architecture, methods, and execution flow of the `DocBuilder` class.

For information on using the documentation build system from the command line, see [Building Documentation](https://deepwiki.com/tenstorrent/riescue/6.1-building-documentation). For details on local server features, see [Local Documentation Server](https://deepwiki.com/tenstorrent/riescue/6.1.2-local-documentation-server). For Sphinx configuration and extensions, see [Sphinx Configuration](https://deepwiki.com/tenstorrent/riescue/6.2.1-sphinx-configuration).

* * *

## Class Overview

The `DocBuilder` class is defined in [docs/build.py 21-219](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L21-L219) and serves as both a programmatic API and CLI wrapper for building Sphinx documentation. It encapsulates the entire build process, from cleaning directories to serving documentation locally.

### Class Attributes

| Attribute | Type | Purpose |
| --- | --- | --- |
| `docs_dir` | `Path` | Root directory for all documentation (`docs/`) |
| `defualt_src` | `Path` | Default source directory, selected based on `BUILD_TT_DOCS` environment variable |
| `repo_dir` | `Path` | Repository root directory |

The `defualt_src` attribute implements the dual documentation tree system: `docs/source` for public documentation and `docs/tt_source` for internal Tenstorrent documentation.

### Instance Attributes

The constructor [docs/build.py 29-52](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L29-L52) initializes the following instance attributes:

| Attribute | Type | Purpose |
| --- | --- | --- |
| `clean` | `bool` | Whether to remove build directory before building |
| `source_dir` | `Path` | Source directory containing `.rst` files |
| `build_dir` | `Path` | Output directory for generated HTML |
| `check` | `bool` | Treat Sphinx warnings as errors (CI mode) |
| `local_host` | `bool` | Start HTTP server after building |
| `port` | `int` | HTTP server port (default: 8888) |
| `host` | `str` | HTTP server host (default: "0.0.0.0") |
| `interactive` | `bool` | Enable interactive mode with reload capability |

Sources: [docs/build.py 21-52](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L21-L52)

## Command-Line Integration

### Argument Parser Configuration

The `add_args()` class method [docs/build.py 55-73](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L55-L73) configures the argument parser with three groups:

**Core Build Options:**

*   `--clean`: Clean build directory before building
*   `--check`: Enable strict mode (warnings as errors)

**Build Path Options:**

*   `--source_dir`: Source directory (default: `docs/source` or `docs/tt_source`)
*   `--build_dir`: Build output directory (default: `docs/_build`)

**Local Host Options:**

*   `--local_host`: Launch HTTP server after building
*   `--interactive` / `-i`: Enable interactive mode with reload
*   `--port`: HTTP server port (default: 8888)
*   `--host`: HTTP server bind address (default: "0.0.0.0")

### CLI Entry Point

The `run_cli()` class method [docs/build.py 76-81](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L76-L81) provides the CLI entry point:

1.   Creates argument parser
2.   Calls `add_args()` to configure arguments
3.   Parses command-line arguments
4.   Instantiates `DocBuilder` with parsed arguments as keyword arguments
5.   Calls `run()` method
6.   Returns the `DocBuilder` instance

This pattern allows the script to be used both as a command-line tool and as an importable module.

Sources: [docs/build.py 55-81](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L55-L81)

## Build Directory Management

### Cleaning Algorithm

The `clean_build_dir()` method [docs/build.py 95-100](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L95-L100) implements a safe directory cleaning algorithm:

1.   Print notification message
2.   Check if build directory exists
3.   If exists, remove entire directory tree using `shutil.rmtree()`
4.   Create fresh directory with `mkdir(parents=True, exist_ok=True)`

**Automatic Cleaning**: When `--local_host` is specified, the constructor [docs/build.py 51-52](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L51-L52) automatically sets `self.clean = True` to ensure fresh builds during local development.

## Sphinx Build Integration

### Build Process

The `build_docs()` method [docs/build.py 102-122](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L102-L122) wraps the Sphinx build process:

**Sphinx Command Construction**: The method builds the command as:

```
sphinx-build [-W -E] <source_dir> <build_dir>
```

Where:

*   `-W`: Treat warnings as errors (CI mode)
*   `-E`: Don't use saved environment, rebuild everything

**Locale Configuration**: The method sets `LC_ALL=C` and `LANG=C`[docs/build.py 118-120](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L118-L120) to avoid locale-related errors in containerized environments, particularly with character encoding.

Sources: [docs/build.py 102-122](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L102-L122)

## HTTP Server Implementation

### Server Architecture

Both `start_local_host()` and `start_interactive_host()` use a custom `ReusableTCPServer` class [docs/build.py 193-202](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L193-L202) that extends `socketserver.TCPServer`:

**Address Reuse**: The `allow_reuse_address = True` attribute prevents "Address already in use" errors when rapidly restarting the server, particularly important for interactive mode.

### Simple HTTP Server Mode

The `start_local_host()` method [docs/build.py 179-218](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L179-L218) implements a basic HTTP server:

1.   Changes working directory to `build_dir`
2.   Creates `ReusableTCPServer` with `SimpleHTTPRequestHandler`
3.   Serves files until `KeyboardInterrupt` (Ctrl+C)
4.   Restores original working directory in `finally` block

**Host Display Logic**: When `host` is "0.0.0.0", the display message shows "localhost" for user convenience [docs/build.py 182-183](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L182-L183) while the server actually binds to all interfaces.

### Interactive Server Mode

The `start_interactive_host()` method [docs/build.py 124-177](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L124-L177) adds reload capability:

**Threading Architecture**:

*   Server runs in daemon thread
*   Main thread handles user input
*   `threading.Event` coordinates shutdown between threads
*   Server thread checks `stop_event` during request handling [docs/build.py 198-200](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L198-L200)

**Interactive Commands**:

| Command | Aliases | Action |
| --- | --- | --- |
| STOP | s, stop | Terminate server and exit |
| RELOAD | r, reload | Rebuild docs and restart server |
| Ctrl+C | - | Terminate server and exit |

Sources: [docs/build.py 124-218](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L124-L218)

## Configuration and Environment

### Environment Variables

The class respects the following environment variable:

| Variable | Purpose | Impact |
| --- | --- | --- |
| `BUILD_TT_DOCS` | Select documentation tree | If set: use `docs/tt_source`, else: use `docs/source` |

This is evaluated at class definition time [docs/build.py 23-26](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L23-L26) to set the `defualt_src` class attribute.

### Path Resolution

The class uses `Path(__file__).parent`[docs/build.py 22](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L22-L22) to establish the `docs_dir`, making it resilient to being invoked from different working directories.

Sources: [docs/build.py 22-27](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L22-L27)

## Error Handling

### Build Errors

The `build_docs()` method uses `subprocess.run(check=True)`[docs/build.py 122](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L122-L122) which raises `CalledProcessError` if Sphinx returns a non-zero exit code. This occurs when:

*   Sphinx encounters errors
*   `--check` mode is enabled and warnings are present
*   Source directory doesn't exist (checked explicitly at [docs/build.py 105-106](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L105-L106))

### Server Errors

The `start_local_host()` method handles exceptions differently based on mode:

*   **Simple mode**: Propagates exceptions except `KeyboardInterrupt`[docs/build.py 209-216](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L209-L216)
*   **Interactive mode**: Catches exceptions and prints error messages without propagating [docs/build.py 213-214](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L213-L214)

This prevents server errors from terminating the interactive mode's outer loop.

Sources: [docs/build.py 102-122](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L102-L122)[docs/build.py 209-216](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L209-L216)

## Integration Points

### Sphinx Integration

The class integrates with Sphinx through:

*   Command-line invocation of `sphinx-build`[docs/build.py 110-114](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L110-L114)
*   Source directory containing `conf.py` and `.rst` files
*   Build directory receiving HTML output

### CI/CD Integration

CI systems use the class with specific options:

*   `--check`: Enable strict checking (example: [.github/workflows/smoke.yml](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/workflows/smoke.yml))
*   `--build_dir public`: Custom output location for GitHub Pages
*   No server options: Build only, no serving

### Container Integration

The class handles containerized execution by:

*   Setting locale environment variables [docs/build.py 118-120](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L118-L120)
*   Supporting `0.0.0.0` bind address for container port forwarding [docs/build.py 73](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L73-L73)
*   Using relative paths that work across mount points

Sources: [docs/build.py 102-122](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L102-L122)[docs/build.py 55-73](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L55-L73)
