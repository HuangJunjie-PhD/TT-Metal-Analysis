## Overview

The Tenstorrent T7000 is a high-density 4U rack-mount server designed for intensive AI and machine learning workloads. It features 8 Tenstorrent Wormhole™ n150s Tensix Processors, providing substantial compute capacity for deep learning training and inference tasks. The system is built on an AMD EPYC host configuration to ensure high-bandwidth data movement between the host and the accelerator mesh.

For information about other Tenstorrent systems, see [TT-QuietBox](https://deepwiki.com/tenstorrent/tenstorrent.github.io/2.1.1-tt-quietbox), [TT-LoudBox](https://deepwiki.com/tenstorrent/tenstorrent.github.io/2.1.2-tt-loudbox), or [T1000](https://deepwiki.com/tenstorrent/tenstorrent.github.io/2.1.4-t1000).

Sources: [core/systems/t7000/README.md 1-9](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/README.md?plain=1#L1-L9)[core/systems/t7000/specifications.md 1-2](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L1-L2)

## System Specifications

The T7000 is built on the SuperMicro A+ Server 4124GS-TNR+ platform [core/systems/t7000/specifications.md 23](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L23-L23) and features the following hardware specifications:

| Component | Specification |
| --- | --- |
| **CPU** | 2× AMD EPYC™ Rome 7352 (24C/48T ea., up to 3.2 GHz, 155W TDP) |
| **Memory** | 1TB (32×32 GB) DDR4-3200 ECC RDIMM (No Slots Free) |
| **Storage** | 2× 1TB U.2 NVMe PCIe 4.0 x4 |
| **Tensix Processors** | 8× Tenstorrent Wormhole™ n150s Tensix Processor |
| **Cables** | 8× QSFP-DD 400GbE (0.6m) + 4× QSFP-DD 400GbE (1m) |
| **Host System Connectivity** | 2× RJ45 1000Base-T via CPU 2× 100GbE QSFP56 via NVIDIA ConnectX-6 VPI 2× USB 3.0 (5 Gbps) Type-A 1× VGA |
| **Tensix Processor Connectivity** | 16× QSFP-DD Active 200G (2 per card) _Connects to other Wormhole Tensix Processors only_ |
| **Power Supply** | 2+2 Titanium Level PSUs (1000W @ 100V, 1800W @ 200V, 2000W @ 220V) |
| **Operating System** | None (Recommended: Ubuntu 22.04) |

## System Architecture

### T7000 Hardware Topology

The T7000 utilizes a dual-socket AMD EPYC architecture to manage 8x Wormhole n150s processors. The host-to-device communication is facilitated via PCIe 4.0, while device-to-device communication is handled via the QSFP-DD mesh.

Sources: [core/systems/t7000/specifications.md 13-23](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L13-L23)

### QSFP-DD External Mesh Connectivity

The T7000 includes 12 external QSFP-DD cables to establish a multi-card mesh topology. Each Wormhole n150s card provides 2x QSFP-DD ports [core/systems/t7000/specifications.md 21](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L21-L21) This high-speed Ethernet-based interconnect allows for collective operations (All-Reduce, etc.) across the 8-card cluster without host intervention.

Sources: [core/systems/t7000/specifications.md 18-21](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L18-L21)

## Installation and Setup

### Power Requirements

The T7000 requires a high-voltage input (200V+) for optimal performance [core/systems/t7000/specifications.md 26](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L26-L26) The 2+2 redundant Titanium Level PSUs provide power based on the input voltage:

*   **100-127Vac**: 1000W
*   **200-240Vac**: 1800W
*   **220-240Vac**: 2000W

Sources: [core/systems/t7000/specifications.md 22-26](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L22-L26)

### Operating System Requirements

The T7000 system is shipped without a pre-installed OS. Tenstorrent recommends **Ubuntu 22.04 (Jammy Jellyfish)** to ensure compatibility with the `TT-KMD` kernel driver and the wider Tenstorrent software stack [core/systems/t7000/specifications.md 30](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L30-L30)

### Software Deployment

Software setup involves installing the Tenstorrent drivers, firmware updates via `tt-flash`, and the `TT-Metalium` or `TT-Forge` SDKs. Detailed instructions are maintained in the global [Software Setup guide](https://docs.tenstorrent.com/getting-started/README.html).

Sources: [core/systems/t7000/specifications.md 38-40](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/specifications.md?plain=1#L38-L40)[core/systems/t7000/README.md 6](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/t7000/README.md?plain=1#L6-L6)
