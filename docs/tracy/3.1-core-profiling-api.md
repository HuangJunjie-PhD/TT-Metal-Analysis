This document covers Tracy's fundamental profiling APIs for instrumenting application code. It focuses on the basic macros and functions used to create profiling zones, mark frames, send messages, create plots, and track memory allocations. For GPU profiling capabilities, see [GPU Profiling](https://deepwiki.com/tenstorrent/tracy/3.2-gpu-profiling). For advanced features like callstack capture and lock profiling, see [Advanced Client Features](https://deepwiki.com/tenstorrent/tracy/3.4-advanced-client-features).

## Overview

Tracy's core profiling API provides the essential instrumentation macros that applications use to generate profiling data. The API is built around several key concepts:

*   **Zones**: Timed regions of code execution that form the backbone of Tracy's profiling
*   **Frames**: Markers that segment execution into discrete units (typically rendering frames)
*   **Messages**: Text annotations associated with specific timestamps
*   **Plots**: Time-series data visualization for numeric values
*   **Memory tracking**: Allocation and deallocation event logging

The API is available in both C++ and C variants, with the C++ version providing additional RAII convenience through scoped objects.

## Zone Profiling API

### Basic Zone Macros

The zone profiling system is built around the `ScopedZone` class and a set of convenience macros that create zones with RAII semantics.

**Zone Creation Macros**

The primary zone creation macros defined in [public/tracy/Tracy.hpp 152-155](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L152-L155):

*   `ZoneScoped` - Creates a zone for the current scope
*   `ZoneScopedN(name)` - Creates a named zone
*   `ZoneScopedC(color)` - Creates a colored zone
*   `ZoneScopedNC(name, color)` - Creates a named, colored zone

**Zone Manipulation Macros**

Once a zone is created, these macros can modify its properties [public/tracy/Tracy.hpp 157-170](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L157-L170):

*   `ZoneText(txt, size)` - Adds text annotation to the current zone
*   `ZoneTextF(fmt, ...)` - Adds formatted text using printf-style formatting
*   `ZoneName(txt, size)` - Sets the zone name
*   `ZoneNameF(fmt, ...)` - Sets formatted zone name
*   `ZoneColor(color)` - Sets zone color
*   `ZoneValue(value)` - Associates a numeric value with the zone

### ScopedZone Implementation

The `ScopedZone` class provides the core zone functionality with automatic cleanup:

The constructor [public/client/TracyScoped.hpp 25-40](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyScoped.hpp#L25-L40) handles zone initialization by:

1.   Checking if profiling is active and connected (in on-demand mode)
2.   Storing the connection ID for consistency checking
3.   Queuing a `ZoneBegin` event with current timestamp and source location

The destructor [public/client/TracyScoped.hpp 99-108](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyScoped.hpp#L99-L108) ensures proper zone closure by:

1.   Verifying the zone is still active and connection hasn't changed
2.   Queuing a `ZoneEnd` event with current timestamp

## Frame Marking API

Frame marking provides temporal segmentation of profiling data, typically used to mark rendering frames or other cyclical operations.

**Frame Marking Macros**

Defined in [public/tracy/Tracy.hpp 172-175](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L172-L175):

*   `FrameMark` - Marks a standard frame boundary
*   `FrameMarkNamed(name)` - Marks a named frame set
*   `FrameMarkStart(name)` - Begins a discontinuous frame
*   `FrameMarkEnd(name)` - Ends a discontinuous frame

The implementation uses the serialized queue to ensure frame marks are processed in order [public/client/TracyProfiler.hpp 282-306](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyProfiler.hpp#L282-L306)

## Message and Plot APIs

### Message API

Messages provide timestamped text annotations in the profiling timeline.

**Message Macros**

From [public/tracy/Tracy.hpp 206-216](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L206-L216):

*   `TracyMessage(txt, size)` - Sends a message with copied text
*   `TracyMessageL(txt)` - Sends a message with literal text (no copy)
*   `TracyMessageC(txt, size, color)` - Colored message with copied text
*   `TracyMessageLC(txt, color)` - Colored message with literal text

### Plot API

Plots enable visualization of time-series numeric data.

**Plot Macros**

The plot system [public/tracy/Tracy.hpp 217-218](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L217-L218) provides:

*   `TracyPlot(name, value)` - Adds a data point to a plot
*   `TracyPlotConfig(name, type, step, fill, color)` - Configures plot appearance

Plot data types are automatically detected and sent as `PlotDataInt`, `PlotDataFloat`, or `PlotDataDouble` queue events [public/client/TracyProfiler.hpp 338-388](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyProfiler.hpp#L338-L388)

**Sources**: [public/tracy/Tracy.hpp 206-218](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L206-L218)[public/client/TracyProfiler.hpp 338-449](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyProfiler.hpp#L338-L449)

## Memory Tracking API

Tracy provides comprehensive memory allocation tracking with optional callstack capture.

**Memory Tracking Macros**

Basic allocation tracking [public/tracy/Tracy.hpp 219-227](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L219-L227):

*   `TracyAlloc(ptr, size)` - Track allocation
*   `TracyFree(ptr)` - Track deallocation
*   `TracyAllocN(ptr, size, name)` - Track named allocation
*   `TracyFreeN(ptr, name)` - Track named deallocation

Secure variants that check profiler availability [public/tracy/Tracy.hpp 228-235](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L228-L235):

*   `TracySecureAlloc(ptr, size)` - Safe allocation tracking
*   `TracySecureFree(ptr)` - Safe deallocation tracking

Callstack variants [public/tracy/Tracy.hpp 236-243](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/Tracy.hpp#L236-L243):

*   `TracyAllocS(ptr, size, depth)` - Allocation with callstack
*   `TracyFreeS(ptr, depth)` - Deallocation with callstack

The implementation ensures thread-safe operation through serialized queues [public/client/TracyProfiler.hpp 487-560](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyProfiler.hpp#L487-L560)

Tracy provides a complete C API for environments where C++ is not available or desired.

`struct ___tracy_c_zone_context {    uint32_t id;    int active;};`
**C Zone Macros**

Key C zone macros [public/tracy/TracyC.h 249-258](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/tracy/TracyC.h#L249-L258):

*   `TracyCZone(ctx, active)` - Creates a basic zone
*   `TracyCZoneN(ctx, name, active)` - Creates a named zone
*   `TracyCZoneEnd(ctx)` - Ends a zone

## Queue Architecture

All profiling APIs funnel through Tracy's lock-free queue system for efficient data collection.

**Queue Types**

The queue system uses strongly-typed events defined in [public/common/TracyQueue.hpp 10-125](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/common/TracyQueue.hpp#L10-L125):

*   `ZoneBegin`/`ZoneEnd` - Zone timing events
*   `FrameMarkMsg` - Frame boundary events
*   `Message`/`MessageColor` - Text message events
*   `PlotDataInt`/`PlotDataFloat`/`PlotDataDouble` - Plot data events
*   `MemAlloc`/`MemFree` - Memory tracking events

**Queue Selection**

The API uses different queue strategies [public/client/TracyProfiler.hpp 137-155](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyProfiler.hpp#L137-L155):

*   Lock-free queue (`TracyLfqPrepare`/`TracyLfqCommit`) for most events
*   Serial queue (`QueueSerial`/`QueueSerialFinish`) for memory events and frames
*   Fiber-aware queuing when `TRACY_FIBERS` is enabled
