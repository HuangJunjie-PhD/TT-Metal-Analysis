This page provides a technical deep dive into the Eleventy configuration for `tt-awesome`. The site's frontend logic is primarily driven by custom Nunjucks filters defined in `.eleventy.js`. these filters handle complex data transformations, such as the showcase diversification algorithm, feed generation, and category-based entry aggregation.

## Configuration Overview

The Eleventy configuration file [.eleventy.js 7-87](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L7-L87) defines the build-time behavior of the static site generator. It manages asset passthrough, custom filter registration, and shortcodes.

### Passthrough Copies

The configuration ensures that static assets and generated data files are available in the final `_site/` directory:

*   `src/assets`: Contains CSS and client-side JavaScript [.eleventy.js 8](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L8-L8)
*   `.nojekyll`: Prevents GitHub Pages from running Jekyll on the output [.eleventy.js 9](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L9-L9)
*   `data.json`: The API export generated by `scripts/generate_data_json.py` is copied to the root to be accessible at `/data.json`[.eleventy.js 13](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L13-L13)

## Custom Nunjucks Filters

Filters are used within Nunjucks templates to process the global data objects (`entries`, `categories`, `planetFeeds`).

### Utility Filters

*   `truncate`: Slices a string to a specified length and appends an ellipsis [.eleventy.js 15-17](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L15-L17)
*   `prettyDate`: Formats ISO `YYYY-MM-DD` strings into `Mon DD, YYYY` (e.g., "May 8, 2026") [.eleventy.js 20-27](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L20-L27)
*   `monthKey` / `monthLabel`: Used in the Planet sub-site to group items by month [.eleventy.js 144-150](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L144-L150) (implied in usage).

### Category & Showcase Filters

*   `countForCategory`: Returns the integer count of entries belonging to a specific category slug [.eleventy.js 38-42](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L38-L42)
*   `featuredForCategory`: Finds the "best" entry for a category by looking for the `featured: true` flag, falling back to the first entry in the list [.eleventy.js 30-35](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L30-L35)
*   `diversifiedFeatured`: Implements the homepage showcase algorithm. It ensures each category displays a unique entry [.eleventy.js 48-87](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L48-L87)

## Showcase Diversification Algorithm

The `diversifiedFeatured` filter [.eleventy.js 48-87](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L48-L87) is responsible for selecting the "hero" project for each of the 12 categories on the landing page.

### Selection Logic

1.   **Ranking (`showcaseSort`)**: Candidates within a category are ranked: 
    *   Entries with `featured: true` always come first [.eleventy.js 65-66](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L65-L66)
    *   Ties are broken by GitHub `stars` count in descending order [.eleventy.js 67](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L67-L67)

2.   **Deprioritization (`isDeprioritized`)**: To ensure variety, certain entries are marked as "last resort": 
    *   Entries containing "BUDA" in the ID or Name [.eleventy.js 56](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L56-L56)
    *   Entries that only support "Grayskull" hardware [.eleventy.js 58-60](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L58-L60)

3.   **Deduplication**: As the algorithm iterates through categories, it tracks `used` entry IDs. An entry can only be the featured hero for one category [.eleventy.js 49-82](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L49-L82)
4.   **Preference Loop**: For each category, it tries to pick a candidate that is: 
    *   Not already used AND not deprioritized [.eleventy.js 77](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L77-L77)
    *   Fallback: Not already used (even if deprioritized) [.eleventy.js 79](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L79-L79)

## Feed & Planet Filters

These filters aggregate data from multiple sources (GitHub releases, project links, and RSS feeds) into unified streams.

### `articleFeedItems`

Generates a list of resources (articles, papers, videos) extracted from the `links` array of all entries [.eleventy.js 98-130](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L98-L130)

*   **Filter**: Only includes link types: `article`, `lesson`, `paper`, `talk`, `video`, `demo`[.eleventy.js 99](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L99-L99)
*   **Sorting**: Newest first based on the entry's `added_at` field [.eleventy.js 100-105](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L100-L105)
*   **Deduplication**: Ensures the same URL isn't featured twice [.eleventy.js 113-114](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L113-L114)

### `jsonFeedItems`

Constructs the items array for the site's JSON Feed 1.1 export [.eleventy.js 144-180](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L144-L180) It merges:

1.   **Releases**: One item per entry in `recentReleases`[.eleventy.js 149-158](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L149-L158)
2.   **New Entries**: The 50 most recently added projects [.eleventy.js 169-170](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L169-L170)
3.   **Resources**: Article-type links from entries [.eleventy.js 141](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L141-L141)

### `planetItems`

The most complex aggregator, used for the `/planet/` sub-site. It combines three distinct data sources into a single chronological stream:

*   **Planet Feeds**: Items from `planet_feeds.json` (YouTube, arXiv, etc.) [.eleventy.js 200](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L200-L200)
*   **Recent Releases**: Stable releases from `recentReleases.js`[.eleventy.js 205](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L205-L205)
*   **Entry Articles**: Article-type links extracted via `articleFeedItems` logic [.eleventy.js 210](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/%20.eleventy.js#L210-L210)

## Shortcodes

### `inlineFile`

This shortcode allows the build system to read a file from the disk and inject its content directly into a template. It is primarily used for inlining CSS or critical JS to improve performance and ensure the "three-pane" layout renders immediately without FOUC (Flash of Unstyled Content).
