## Purpose and Scope

This document details the different Tenstorrent board types supported by the tt-firmware repository. It covers the specific features, firmware components, and capabilities available for each board type. For information about firmware bundles and installation procedures, see [Firmware Bundles](https://deepwiki.com/tenstorrent/tt-firmware/3-firmware-bundles).

## Overview of Supported Boards

The tt-firmware repository provides firmware for three main Tenstorrent device families:

1.   **Grayskull (GS)** - First generation Tenstorrent AI processors
2.   **Wormhole (WH)** - Second generation Tenstorrent AI processors
3.   **Blackhole (BH)** - Latest generation Tenstorrent AI processors with enhanced networking capabilities

Each board type utilizes the system management processor firmware for hardware initialization, I/O connectivity management, and power/thermal regulation.

## Board Specifications and Features

### Grayskull (GS)

Grayskull represents the first generation of Tenstorrent AI processors. The firmware provides:

*   Hardware initialization sequence
*   I/O connectivity management for PCIe and DRAM
*   Power and thermal regulation
*   System management processor control

### Wormhole (WH)

Wormhole is the second generation of Tenstorrent AI processors, with the firmware providing similar core functionality to Grayskull with adjustments for the Wormhole hardware architecture:

*   Hardware initialization sequence
*   Enhanced I/O connectivity management
*   Power and thermal regulation
*   System management processor control

### Blackhole (BH)

Blackhole is the latest generation of Tenstorrent AI processors with significant enhancements, particularly in networking capabilities. The firmware includes several Blackhole-specific features:

*   **ERISC Firmware v1.4.0**: Embedded RISC processor firmware
*   **ETH Mailbox**: Communication interface with two message types: 
    *   **LINK_STATUS_CHECK**: Monitors Ethernet link status
    *   **RELEASE_CORE**: Releases control of RISC0 to execute functions at specified L1 addresses

*   **Virtual UART**: Enabled by default for firmware observability and debugging 
    *   Creates an in-memory virtual UART
    *   Accessible via `tt-console` to view `printk()` and `LOG_*()` messages from the host

*   **Enhanced Link Training**: Improved link training sequence for greater success rates on loopback cases
*   **Improved SMC I2C Recovery**: Better reset and re-enumeration success rate (99.6%)
*   **PCIe MPS Control**: Maximum Payload Size now set by TT-KMD, improving virtual machine stability

Sources: [README.md 31-49](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L31-L49)
