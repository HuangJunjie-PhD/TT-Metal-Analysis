## Purpose and Scope

This document provides a technical overview of the Tenstorrent documentation structure and organization. The documentation is built as a multi-project ecosystem using Sphinx, providing a unified portal for hardware specifications, software frameworks, and system utilities. It covers the layout of the home page, shared resources, URL schemes, and the integration of interactive components like the feedback widget.

## Multi-Project Layout

The documentation is organized into several distinct projects that are built independently but share a common visual identity and resource base.

## Shared Resources and Templates

To maintain consistency across projects, the repository uses a `shared/` directory containing common Sphinx templates and static assets.

### Layout Customization

The `shared/_templates/layout.html` file extends the base Sphinx `!layout.html`. It handles:

1.   **Sidebar Titles and Logos**: Dynamically generates the logo link and project title [shared/_templates/layout.html 3-18](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/layout.html#L3-L18)
2.   **Version Display**: Injects the version string into the navigation sidebar if enabled [shared/_templates/layout.html 20-30](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/layout.html#L20-L30)
3.   **Feedback Widget**: Conditionally includes the `feedback_widget.html` unless specifically disabled via metadata [shared/_templates/layout.html 36-42](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/layout.html#L36-L42)

### Feedback Widget

The feedback widget (`feedback_widget.html`) is a custom interactive component that allows users to rate pages.

Sources: [shared/_templates/feedback_widget.html 1-171](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/feedback_widget.html#L1-L171)[shared/_templates/layout.html 38-41](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/layout.html#L38-L41)

### Version Selection

The `shared/_templates/versions.html` template provides a dropdown menu for switching between different versions of the documentation (e.g., `latest`, `v0.51.0`). It constructs URLs based on the `project_code` and `version` variables [shared/_templates/versions.html 1-20](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/versions.html#L1-L20)

## URL Scheme and Sitemap

The documentation is hosted on GitHub Pages at `tenstorrent.github.io`. The URL structure follows a consistent pattern to support multi-project and multi-version hosting.

### URL Pattern

`https://tenstorrent.github.io/{project_code}/{version}/index.html`

*   **project_code**: The identifier for the sub-project (e.g., `ttnn`, `tt-metalium`).
*   **version**: The specific release version or `latest`.

Sources: [shared/_templates/versions.html 12](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/shared/_templates/versions.html#L12-L12)[core/index.rst 31-33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/index.rst#L31-L33)
