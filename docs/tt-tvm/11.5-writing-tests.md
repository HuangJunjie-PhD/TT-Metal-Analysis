## Purpose and Scope

This guide covers how to write tests in the TVM codebase, with emphasis on frontend converter tests and Relay operator tests. For information about setting up your development environment before writing tests, see [Setting Up Development Environment](https://deepwiki.com/tenstorrent/tt-tvm/11.1-setting-up-development-environment). For information about the CI/CD infrastructure that runs these tests, see [CI/CD Infrastructure](https://deepwiki.com/tenstorrent/tt-tvm/10-cicd-infrastructure).

Tests in TVM are primarily written using pytest and are organized in the `tests/python/` directory. The testing framework provides utilities for comparing TVM outputs against reference implementations from source frameworks (PyTorch, TensorFlow, ONNX, etc.).

## Test Organization

Tests are organized hierarchically in the `tests/python/` directory:

```
tests/python/
├── frontend/                   # Frontend converter tests
│   ├── onnx/
│   │   └── test_forward.py    # ONNX operator tests
│   ├── pytorch/
│   │   ├── test_forward.py    # PyTorch operator tests
│   │   ├── qnn_test.py        # Quantized model tests
│   │   ├── test_rnns.py       # RNN/LSTM tests
│   │   └── test_object_detection.py
│   ├── tflite/
│   │   └── test_forward.py    # TFLite operator tests
│   ├── tensorflow/
│   │   └── test_forward.py    # TensorFlow operator tests
│   ├── tensorflow2/
│   │   ├── test_functional_models.py
│   │   └── test_sequential_models.py
│   └── mxnet/
│       └── test_forward.py    # MXNet operator tests
└── relay/                     # Relay IR and operator tests
```

## Best Practices

### Test Naming

*   Use descriptive names: `test_forward_<op_name>` for operator tests
*   Use `test_<model_name>` for end-to-end model tests
*   Group related tests in same file

### Tolerance Settings

Sources: [tests/python/frontend/pytorch/test_forward.py 216](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/python/frontend/pytorch/test_forward.py#L216-L216)[tests/python/frontend/onnx/test_forward.py 246](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/python/frontend/onnx/test_forward.py#L246-L246)

### Test Isolation

*   Each test should be independent
*   Use `torch.set_grad_enabled(False)` for PyTorch tests
*   Clear CUDA cache: `torch.cuda.empty_cache()`
*   Use fixtures for shared setup
