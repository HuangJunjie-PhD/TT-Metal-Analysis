## Purpose and Scope

This document describes the architectural tests for RISC-V Atomic Memory Operations (RV64A extension). These tests verify the correct implementation of atomic memory operations, which are essential for synchronization between multiple processes or threads in multi-core systems. This page focuses on the bare metal machine mode tests for RV64A instructions, including atomic memory operations (AMO) and load-reserved/store-conditional (LR/SC) instruction pairs.

The tests validate both the functional correctness of individual atomic instructions and their memory ordering semantics as specified in the RISC-V architecture.

## RISC-V Atomic Extension Overview

The RISC-V A extension (Atomic Instructions) provides hardware support for atomic memory operations, which are crucial for implementing synchronization primitives such as mutexes, semaphores, and lock-free data structures in multi-core systems.

## Test Organization

The atomic memory operation tests are structured as individual test cases, each verifying a specific atomic instruction. The tests are executed in machine mode with paging disabled and run in a bare metal environment.

## Test Implementation Details

Each atomic memory operation test follows a similar implementation pattern. Below is a table summarizing the key tests and their implementation details:

| Test Case | Instruction | Memory Address | Verification Method |
| --- | --- | --- | --- |
| test1 | AMOADD.W.AQ | 0xeb529000 | Result value and memory content verification |
| test2 | LR.W.RL | 0xefd48000 | Result value verification |
| test3 | AMOXOR.D.AQ | 0x8dcf6000 | Result value and memory content verification |
| test4 | AMOMIN.D.RL | 0xc5ee6000 | Result value and memory content verification |
| test5 | AMOSWAP.D.AQRL | 0xacf8d000 | Result value and memory content verification |

Each test:

1. Loads specific test values into registers
2. Executes the atomic instruction with the appropriate memory ordering option
3. Verifies the returned result matches the expected value
4. For most instructions, also verifies memory content after operation

## Memory Ordering Semantics

A key aspect of the atomic memory operations tests is validating the memory ordering semantics specified by the `.aq` (acquire), `.rl` (release), and `.aqrl` (acquire-release) suffixes. These semantics are crucial for correct synchronization in weak memory model architectures like RISC-V.

| Ordering Suffix | Meaning | FENCE Equivalent | Usage in Tests |
| --- | --- | --- | --- |
| (none) | Unordered | None | Rarely used in tests |
| .aq | Acquire | FENCE r, rw | AMOADD.W.AQ, AMOXOR.D.AQ |
| .rl | Release | FENCE rw, w | LR.W.RL, AMOMIN.D.RL |
| .aqrl | Acquire-Release | FENCE rw, rw | AMOSWAP.D.AQRL |

The tests verify that these memory ordering semantics are correctly implemented, which is essential for proper operation of synchronization primitives.

The Atomic Memory Operations Tests provide comprehensive validation of the RISC-V RV64A extension instruction set.
