## Code Style and Quality Enforcement

The Model Explorer project uses a multi-tool approach for code quality enforcement, centered around the Pigweed presubmit framework and specialized Python formatting tools.

### Code Style Check Workflow

**Sources:**[ci/code_style_check.sh 1-95](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/code_style_check.sh#L1-L95)

The main code style checking is orchestrated by [ci/code_style_check.sh](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/code_style_check.sh) which performs two primary functions:

1.   **License Header Enforcement**: Uses the Pigweed presubmit framework to verify that all source files contain proper Apache 2.0 license headers
2.   **Python Code Formatting**: Uses `pyink` (a Python formatter) to ensure consistent code style across Python files

### Pigweed Integration

The project uses a customized version of the Pigweed presubmit framework for license and formatting checks. The integration involves downloading and patching Pigweed:

**Sources:**[ci/pigweed_download.sh 1-43](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/pigweed_download.sh#L1-L43)[ci/pigweed.patch 1-129](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/pigweed.patch#L1-L129)

The Pigweed framework is downloaded via [ci/pigweed_download.sh](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/pigweed_download.sh) and customized with [ci/pigweed.patch](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/pigweed.patch) Key customizations include:

*   Modified copyright notice format for "AI Edge Model Explorer Authors"
*   Adjusted license header template to match project requirements
*   Custom exclusion patterns for specific file types and directories

### Python Formatting Configuration

Python code formatting is handled by `pyink` with specific configuration options:

| Option | Value | Purpose |
| --- | --- | --- |
| `--pyink-use-majority-quotes` | Enabled | Consistent quote style |
| `--pyink-indentation` | 2 | Two-space indentation |
| `--line-length` | 80 | Maximum line length |
| `--extend-exclude` | `.downloads`, `.pyi` | Skip generated/external files |

**Sources:**[ci/code_style_check.sh 75-85](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/code_style_check.sh#L75-L85)

## Visual Regression Testing

Model Explorer implements comprehensive visual regression testing using Playwright to ensure UI consistency across changes.

### Playwright Testing Architecture

**Sources:**[ci/playwright_test.py 1-283](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/playwright_test.py#L1-L283)

### Test Coverage and Scenarios

The visual regression testing covers multiple adapter types and user interaction scenarios:

| Test Function | Model Type | Adapter | UI Interaction |
| --- | --- | --- | --- |
| `test_litert_direct_adapter` | TFLite | Flatbuffer | Canvas click at (469, 340) |
| `test_litert_mlir_adapter` | TFLite | MLIR | Canvas click at (514, 332) |
| `test_tf_mlir_adapter` | TensorFlow | MLIR | Canvas click at (444, 281) |
| `test_tf_direct_adapter` | TensorFlow | Direct | Graph expansion and navigation |
| `test_pytorch` | PyTorch | Export | Dynamic model serialization |

**Sources:**[ci/playwright_test.py 95-194](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/playwright_test.py#L95-L194)

### Image Comparison Algorithm

The visual regression system uses a pixel-based comparison algorithm with configurable tolerance:

**Sources:**[ci/playwright_test.py 35-64](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/playwright_test.py#L35-L64)

Key features of the comparison system:

*   Converts images to grayscale for comparison
*   Calculates mismatch ratio based on differing pixels
*   Generates diff images for debugging when tests fail
*   Configurable threshold for acceptable differences

### Test Environment Configuration

The testing infrastructure uses specific directory structures and constants:

**Sources:**[ci/playwright_test.py 27-32](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/ci/playwright_test.py#L27-L32)

The test environment is configured with these key paths:

*   `LOCAL_SERVER = "http://127.0.0.1:8080//"` - Test server endpoint
*   `TEST_FILES_DIR = ROOT_DIR / "test/test_models"` - Model files for testing
*   `EXPECTED_SCREENSHOT_DIR = ROOT_DIR / "test/screenshots_golden/chrome-linux"` - Reference images

This infrastructure ensures consistent test environments and reliable comparison of visual regression results across different development and CI environments.
