## Purpose and Scope

This page documents the wheel building and distribution system for the `tt-forge` metapackage. The system manages dependency resolution, dynamic wheel generation, and publication to Tenstorrent's private PyPI repository. The `tt-forge` package is a metapackage that bundles dependencies from multiple repositories (`tt-forge-fe`, `tt-xla`, `tt-tvm`) into a single installable package with consistent versioning.

## The tt-forge Metapackage Architecture

The `tt-forge` wheel is not a traditional Python package with source code. Instead, it is a metapackage that declares dependencies on other wheels published from separate repositories. This architecture allows users to install the complete TT-Forge stack with a single `pip install tt-forge` command.

## Dependency Resolution and Wait Logic

Before building the metapackage, the system must ensure that all dependent wheels are available on the Tenstorrent PyPI server. The `wait-on-tt-pypi-wheels.sh` script implements a polling mechanism.

### Polling Configuration

| Parameter | Value | Description |
| --- | --- | --- |
| `wait_time` | 60 seconds | Time between polling attempts |
| `attempts` | 120 | Maximum number of attempts |
| **Total Timeout** | **2 hours** | Maximum wait time before failure |

Sources: [.github/scripts/wait-on-tt-pypi-wheels.sh 11-13](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/.github/scripts/wait-on-tt-pypi-wheels.sh#L11-L13)

### Dependency Check Algorithm

The script iterates through `PIP_WHEEL_NAMES` and uses `pip index versions` to verify the presence of the specific `NEW_VERSION_TAG`.

Sources: [.github/scripts/wait-on-tt-pypi-wheels.sh 15-41](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/.github/scripts/wait-on-tt-pypi-wheels.sh#L15-L41)

## Version Resolution Strategy

The `tt-forge-wheel` action determines which versions of upstream packages to bundle based on the `release_type` input.

| Release Type | `PJRT_PLUGIN_TT_TAG` | `TT_FORGE_FE_TAG` | Implementation Detail |
| --- | --- | --- | --- |
| **Draft** | Last nightly tag | Last nightly tag | Queries `gh release list` for `.dev` tags |
| **Nightly** | `new_version_tag` | `new_version_tag` | Direct mapping of current build tag |
| **RC** | Latest RC tag | Latest RC tag | Queries `tt-xla` and `tt-forge-fe` for `rc` tags |
| **Stable** | Latest stable tag | Latest stable tag | Queries for non-prerelease tags matching MAJOR.MINOR |

Sources: [.github/actions/tt-forge-wheel/action.yml 34-75](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/.github/actions/tt-forge-wheel/action.yml#L34-L75)

## Template-Based Wheel Generation

The process uses `template-setup.py` to dynamically generate a valid `setup.py` at build time.

### Template Logic

The template uses environment variable placeholders that are substituted by `envsubst` in the `tt-forge-wheel` action.

`# .github/scripts/template-setup.pysetup(    name="tt-forge",    version="${NEW_VERSION_TAG}",    install_requires=[        "tt-forge-fe @https://pypi.eng.aws.tenstorrent.com/tt-forge-fe/tt_forge_fe-${TT_FORGE_FE_TAG}-cp311-cp311-linux_x86_64.whl",        "tt-tvm @https://pypi.eng.aws.tenstorrent.com/tt-tvm/tt_tvm-${TT_FORGE_FE_TAG}-cp311-cp311-linux_x86_64.whl",        "pjrt-plugin-tt @https://pypi.eng.aws.tenstorrent.com/pjrt-plugin-tt/pjrt_plugin_tt-${PJRT_PLUGIN_TT_TAG}-cp311-cp311-linux_x86_64.whl",    ],    python_requires=">=3.11",    has_ext_modules=lambda: True,  # Force platform-specific wheel)`

## Artifact Download and Verification

The `download-artifact` composite action is used to retrieve build artifacts (including wheels) across different workflow runs and repositories.

### Key Functionality: `download_artifact()`

This function uses the GitHub CLI (`gh run download`) to fetch artifacts and includes logic to automatically handle compressed archives.

*   **Retries**: Configurable via `retry_count` (default: 3) and `retry_wait` (default: 10s). [[.github/actions/download-artifact/action.yaml 22-29](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/[.github/actions/download-artifact/action.yaml#L22-L29)]
*   **Extraction**: Automatically detects and extracts `.tar` and `.tar.zst` files using `tar --use-compress-program=zstd`. [[.github/actions/download-artifact/action.yaml 82-94](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/[.github/actions/download-artifact/action.yaml#L82-L94)]
*   **Security**: Validates that the `DOWNLOAD_PATH` is within the `${GITHUB_WORKSPACE}` to prevent path traversal. [[.github/actions/download-artifact/action.yaml 58-61](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/[.github/actions/download-artifact/action.yaml#L58-L61)]

## Docker Image Integration

Built wheels are installed into environment-specific Docker images. The Dockerfile uses `ARG` variables to specify which wheel version to install during the build process.

*   **Base Image**: Ubuntu 22.04 [[.github/Dockerfile.ubuntu-22-04-py3-11 1](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/[.github/Dockerfile.ubuntu-22-04-py3-11#L1-L1)]
*   **Python Version**: 3.11 [[.github/Dockerfile.ubuntu-22-04-py3-11 39](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/[.github/Dockerfile.ubuntu-22-04-py3-11#L39-L39)]
*   **Build Arguments**: `WHL_VERSION`, `REPO_SHORT`, and `WHEEL_FILES_PATH` are used to locate and install the generated wheels. [[.github/Dockerfile.ubuntu-22-04-py3-11 5-7](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/[.github/Dockerfile.ubuntu-22-04-py3-11#L5-L7)]
