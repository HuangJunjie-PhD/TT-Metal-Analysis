This page documents the type conversion and casting operations provided by the SFPI library. These operations enable conversion between different numeric formats including FP32, FP16 (both fp16a and fp16b variants), and various integer types (int8, uint8, int16, uint16, int32). The conversion system supports multiple rounding modes and handles the dual FP16 format system specific to Tenstorrent architectures.

For information about the FP16 type system and constructors, see [FP16 Conversions and Type System](https://deepwiki.com/tenstorrent/sfpi/2.2-fp16-conversions-and-type-system). For vector manipulation operations that change data layout rather than numeric format, see [Vector Manipulation Primitives](https://deepwiki.com/tenstorrent/sfpi/4.3-vector-manipulation-primitives).

### FP16b Conversion Algorithm

The `fp32_to_fp16b()` method performs simple truncation, extracting the upper 16 bits of the FP32 value:

| Operation | Description |
| --- | --- |
| Input | 32-bit float value |
| Reinterpret | Cast to uint32_t |
| Shift | Right shift by 16 bits (`>> 16`) |
| Output | Upper 16 bits as FP16b format |

This fast conversion is suitable when denormal handling is not required and slight precision loss is acceptable.

**Sources:**[include/sfpi_fp16.h 120-130](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L120-L130)

## Runtime Type Conversion Functions

The SFPI library provides hardware-accelerated type conversion functions that operate on vector types. These conversions are performed by the SFPU hardware and support configurable rounding modes.

### Integer to Float Conversion

**Function signature:**

`sfpi_inline vFloat int32_to_float(vInt in, int round_mode = 1)`
**Sources:**[include/wormhole/sfpi_lib.h 189-192](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L189-L192)[include/blackhole/sfpi_lib.h 199-202](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L199-L202)

### Float to FP16 Conversions

The library provides separate functions for converting FP32 vectors to FP16a and FP16b formats:

Both functions use the `__builtin_rvtt_sfpstochrnd_i` intrinsic with different mode flags:

*   `SFPSTOCHRND_MOD1_FP32_TO_FP16A` for fp16a format
*   `SFPSTOCHRND_MOD1_FP32_TO_FP16B` for fp16b format

**Sources:**[include/wormhole/sfpi_lib.h 194-208](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L194-L208)[include/blackhole/sfpi_lib.h 204-218](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L204-L218)

### Float to Integer Conversions

The library supports conversion from FP32 to various integer widths:

| Function | Input Type | Output Type | Hardware Mode |
| --- | --- | --- | --- |
| `float_to_uint8()` | `vFloat` | `vUInt` | `SFPSTOCHRND_MOD1_FP32_TO_UINT8` |
| `float_to_int8()` | `vFloat` | `vUInt` | `SFPSTOCHRND_MOD1_FP32_TO_INT8` |
| `float_to_uint16()` | `vFloat` | `vUInt` | `SFPSTOCHRND_MOD1_FP32_TO_UINT16` |
| `float_to_int16()` | `vFloat` | `vUInt` | `SFPSTOCHRND_MOD1_FP32_TO_INT16` |

All functions accept a `round_mode` parameter (default: 1 for stochastic rounding).

**Sources:**[include/wormhole/sfpi_lib.h 210-272](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L210-L272)[include/blackhole/sfpi_lib.h 220-282](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L220-L282)

## Rounding Modes

All conversion functions that may lose precision accept a `round_mode` parameter. The SFPI library supports two rounding modes:

| Mode | Value | Hardware Constant | Description |
| --- | --- | --- | --- |
| Stochastic | 1 (default) | `SFPSTOCHRND_RND_STOCH` | Probabilistic rounding based on fractional part |
| Round-to-even | 0 | `SFPSTOCHRND_RND_EVEN` | IEEE 754 round-to-nearest-even (banker's rounding) |

### Rounding Mode Selection

Stochastic rounding is preferred for training workflows as it provides unbiased rounding over many operations, while round-to-even is suitable for inference and deterministic results.

**Sources:**[include/wormhole/sfpi_lib.h 196-223](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L196-L223)[include/blackhole/sfpi_lib.h 206-233](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L206-L233)

## Type Reinterpretation

The `reinterpret()` template function performs bitwise reinterpretation between SFPI vector types without modifying the underlying bit pattern:

**Function signature:**

`template <typename vType, typename std::enable_if_t<std::is_base_of<__vBase, vType>::value>* = nullptr>sfpi_inline vType reinterpret(const __vBase v)`
This is a zero-cost abstraction that only changes the C++ type, allowing the same bit pattern to be interpreted differently (e.g., treating float bits as integer for bitwise operations).

**Sources:**[include/wormhole/sfpi_lib.h 183-187](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L183-L187)[include/blackhole/sfpi_lib.h 193-197](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L193-L197)

## Architecture-Specific Differences

While the conversion API is consistent across Wormhole and Blackhole architectures, there is one notable difference in the `shft()` function implementation that affects type casting workflows:

### Wormhole Implementation

[include/wormhole/sfpi_lib.h 173-181](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L173-L181) provides:

*   `shft(vUInt, vInt)` - Assumes logical shift
*   `shft(vUInt, int)` - Assumes logical shift
*   No explicit mode parameter

### Blackhole Implementation

[include/blackhole/sfpi_lib.h 173-191](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L173-L191) provides:

*   `shft(vUInt, vInt)` - Explicitly uses `SFPSHFT_MOD1_LOGICAL`
*   `shft(vInt, vInt)` - Explicitly uses `SFPSHFT_MOD1_ARITHMETIC`
*   `shft(vUInt, int)` - Explicitly uses `SFPSHFT_MOD1_LOGICAL`
*   `shft(vInt, int)` - Explicitly uses `SFPSHFT_MOD1_ARITHMETIC`

Blackhole provides type-safe shift operations that automatically select logical or arithmetic shift based on the input vector type, which is important when reinterpreting between signed and unsigned types.

**Sources:**[include/wormhole/sfpi_lib.h 173-181](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L173-L181)[include/blackhole/sfpi_lib.h 173-191](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L173-L191)
