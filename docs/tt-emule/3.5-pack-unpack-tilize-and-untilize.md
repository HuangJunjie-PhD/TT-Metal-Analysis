The pack and unpack pipeline in `tt-emule` models the transition of data between Circular Buffers (CBs) in L1 memory and the DST register file. On Tenstorrent hardware, this pipeline handles format conversion, tile layout transformation (N-Faces vs. Row-Major), and fused math operations like ReLU. `tt-emule` emulates these hardware engines using C++ scalar loops and software codecs for block-float formats.

## The N-Faces Tile Layout

Silicon hardware typically stores 32x32 tiles in **N-Faces** format to optimize for parallel processing. A standard 1024-element tile is divided into four 16x16 "faces." `tt-emule` uses Look-Up Tables (LUTs) to translate between the hardware's N-Faces layout and the emulator's internal row-major `__emule_dst` representation.

## Pack Pipeline (`llk_pack`)

The `llk_pack` shim emulates the hardware's Packer engine. It reads from the `__emule_dst` register file, applies fused operations (like ReLU), performs format encoding, and writes to a destination CB.

### Fused ReLU and Formats

The packer honors a thread-global `__emule_pack_relu_mode` which mirrors the silicon `STACC_RELU` configuration [include/jit_hw/llk_pack.h 10-13](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/llk_pack.h#L10-L13)

Supported output paths include:

1.   **Tiled Path (`__llk_pack_tiled`)**: Writes RM DST to N-Faces CB [include/jit_hw/llk_pack.h 14](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/llk_pack.h#L14-L14)
2.   **Untilized Path (`__llk_pack_untilize`)**: Scatters RM DST into a row-major contiguous output buffer [include/jit_hw/llk_pack.h 96](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/llk_pack.h#L96-L96)

## Block-Float Encoding (BFP8 & BFP4)

`tt-emule` provides scalar implementations of the hardware's block-float codecs. Unlike standard floating point, these formats share a single exponent across a "face-row" (16 elements).

### BFP8_b Algorithm

*   **Encoding**: Finds the maximum exponent in a 16-element row, then aligns and rounds each element to a 7-bit mantissa [include/jit_hw/api/compute/bfp8.h 32-38](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp8.h#L32-L38)
*   **Rounding**: Uses round-to-nearest-even logic to match silicon's `convert_u32_to_bfp`[include/jit_hw/api/compute/bfp8.h 107-130](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp8.h#L107-L130)
*   **Layout**: A 1088-byte tile consists of 64 exponent bytes followed by 1024 mantissa bytes [include/jit_hw/api/compute/bfp8.h 12-16](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp8.h#L12-L16)

### BFP4_b Algorithm

*   **Encoding**: Similar to BFP8 but uses a 3-bit mantissa [include/jit_hw/api/compute/bfp4.h 8-9](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp4.h#L8-L9)
*   **Packing**: Two 4-bit elements (1-bit sign, 3-bit mantissa) are packed into a single byte [include/jit_hw/api/compute/bfp4.h 13-15](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp4.h#L13-L15)

**Sources:**[include/jit_hw/api/compute/bfp8.h 5-135](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp8.h#L5-L135)[include/jit_hw/api/compute/bfp4.h 5-111](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/bfp4.h#L5-L111)

## Tilize and Untilize Shims

These operations transform data between row-major "sticks" and tiled formats.

### `tilize_block`

This function reads a horizontal strip of data from an input CB and writes it as sequential tiles in N-Faces format [include/jit_hw/api/compute/tilize.h 58-65](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/tilize.h#L58-L65) It handles data-type re-encoding (e.g., FP32 input to BFP8 output) during the move [include/jit_hw/api/compute/tilize.h 90-116](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/tilize.h#L90-L116)

### `pack_untilize_dest`

This is the emulator for the silicon `pack_untilize_dest` LLK. It reads tiles from the DST register file and writes them to the output CB in row-major layout [include/jit_hw/api/compute/pack_untilize.h 80-88](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/pack_untilize.h#L80-L88)

*   **Logic**: It calculates the target row and column in the output buffer based on the `block_c_index` and `block_ct_dim`[include/jit_hw/api/compute/pack_untilize.h 117-121](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/pack_untilize.h#L117-L121)
*   **Granularity**: The `cb_reserve_back` page size is used for synchronization, but the physical L1 layout remains a contiguous row-major stick [include/jit_hw/api/compute/pack_untilize.h 89-93](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/pack_untilize.h#L89-L93)
