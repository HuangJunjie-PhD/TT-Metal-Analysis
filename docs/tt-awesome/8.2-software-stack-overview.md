The Tenstorrent software stack is a multi-layered ecosystem designed to bridge high-level machine learning frameworks with low-level hardware primitives. This stack ranges from kernel-level drivers to sophisticated MLIR-based compilers and user-facing inference servers.

## System and Hardware Layer

This layer interacts directly with the silicon, managing memory, PCIe communication, and device telemetry.

### Kernel and User Drivers

*   **tt-kmd**: The Linux kernel module driver. It is the fundamental requirement for interfacing with Tenstorrent PCIe accelerator cards [entries/hw-system/tt-kmd.json 3-14](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-kmd.json#L3-L14)
*   **tt-umd**: The user-mode driver that provides a high-level API for applications to communicate with the hardware without staying in kernel space.
*   **tt-system-firmware**: Low-level system initialization and control firmware that runs on-device [entries/hw-system/tt-system-firmware.json 3-13](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-system-firmware.json#L3-L13) Recent releases have integrated Zephyr 4.4.0 for Blackhole architectures [src/_data/planet_feeds.json 76-85](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/planet_feeds.json#L76-L85)

### Hardware Management Tools

*   **tt-smi**: The System Management Interface. It is the `nvidia-smi` equivalent for Tenstorrent, used to monitor device telemetry, health, and perform board-level resets [entries/hw-system/tt-smi.json 3-15](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-smi.json#L3-L15)
*   **tt-flash**: A utility for flashing new firmware onto accelerator cards from the command line [entries/hw-system/tt-flash.json 3-13](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-flash.json#L3-L13)
*   **tt-topology**: Used for configuring Ethernet routing on multi-card systems, specifically for scale-out deployments [entries/hw-system/tt-topology.json 3-15](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-topology.json#L3-L15)

## Programming and Compiler Layer

This layer provides the programming models used to write kernels and the compilers that lower high-level models to hardware-specific instructions.

### Programming Models

*   **TT-Metalium (tt-metal)**: The low-level programming model. It allows developers to write custom kernels that run on Tensix cores and manage data movement across the NoC (Network-on-Chip).
*   **TT-NN**: A high-level tensor library built on top of TT-Metalium, providing optimized operations for deep learning.

### Compilers and Frontends

*   **TT-Forge**: An MLIR-based compiler that serves as the primary path for lowering models to Tenstorrent hardware.
*   **tt-xla**: A PJRT device plugin that enables JAX and PyTorch/XLA to target Tenstorrent accelerators [entries/compilers/tt-xla.json 3-14](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/compilers/tt-xla.json#L3-L14)
*   **tt-torch**: A frontend integration for PyTorch using `torch.compile` to target the `tt-mlir` pipeline [entries/compilers/tt-torch.json 3-13](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/compilers/tt-torch.json#L3-L13)

## Developer Productivity & Simulation

Tools in this layer facilitate debugging, visualization, and hardware-free development.

### Simulation and Visualization

*   **ttsim**: A fast full-system simulator for Wormhole and Blackhole hardware. It allows running TT-Metalium workloads on x86_64 systems with bit-exact results relative to physical silicon [entries/riscv-arch/ttsim.json 3-16](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/riscv-arch/ttsim.json#L3-L16)
*   **ttnn-visualizer**: A tool for inspecting tensor operations and memory layouts.
*   **tt-exalens**: A developer tool for deep hardware inspection and debugging [entries/dev-tools/tt-exalens.json 1-30](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/dev-tools/tt-exalens.json#L1-L30) Recent updates include the ability to read program counters when a core is halted [src/_data/planet_feeds.json 90-98](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/planet_feeds.json#L90-L98)

## Application and Inference

The top of the stack focuses on deploying models at scale.

*   **tt-inference-server**: A high-performance server for deploying models. It supports large-scale models like Llama-3.3-70B on Blackhole architectures [src/_data/planet_feeds.json 20-29](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/planet_feeds.json#L20-L29)
*   **tt-buda**: A legacy compilation and execution framework (now largely succeeded by the TT-Metalium/TT-Forge path).
