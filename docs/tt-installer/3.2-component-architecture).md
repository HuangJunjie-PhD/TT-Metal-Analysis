## Purpose and Scope

This document describes the components that make up the Tenstorrent software stack as installed by the tt-installer. It focuses on how these components relate to each other and interact with the Tenstorrent hardware. For information about the installation process itself, see [Installation Process Flow](https://deepwiki.com/tenstorrent/tt-installer/3.1-installation-process-flow).

The Tenstorrent software stack consists of several interdependent components that together provide a complete environment for working with Tenstorrent AI hardware.

## Core Components

### Hardware Interface Components

These components provide direct interfaces to the Tenstorrent hardware.

#### 1. Kernel-Mode Driver (KMD)

The Kernel-Mode Driver serves as the foundation of the software stack, providing low-level access to Tenstorrent hardware.

*   **Repository**: `https://github.com/tenstorrent/tt-kmd.git`
*   **Purpose**: Enables communication between the operating system and Tenstorrent hardware
*   **Installation**: Installed via DKMS to ensure kernel module persistence across kernel updates
*   **Device Access**: Creates `/dev/tenstorrent` device for hardware communication

Sources: [install.sh 20-27](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L20-L27)[install.sh 533-553](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L533-L553)

#### 2. HugePages Configuration

HugePages is a memory management feature that improves performance when working with Tenstorrent hardware.

*   **Purpose**: Allocates large memory pages for efficient data transfer to Tenstorrent devices
*   **Configuration**: Sets up and enables the `tenstorrent-hugepages.service` and mounts the hugepages filesystem
*   **Access Point**: Creates `/dev/hugepages-1G` for large memory page allocation

Sources: [install.sh 574-603](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L574-L603)

### System Management Components

These tools provide monitoring and management capabilities for Tenstorrent hardware.

Sources: [install.sh 555-571](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L555-L571)[install.sh 607](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L607-L607)

#### 1. System Management Interface (tt-smi)

The System Management Interface provides monitoring and control capabilities for Tenstorrent hardware.

*   **Repository**: `https://github.com/tenstorrent/tt-smi`
*   **Purpose**: Allows users to view and manage Tenstorrent device status
*   **Dependencies**: Requires Kernel-Mode Driver
*   **Usage**: Command-line tool for checking hardware status and performance

Sources: [install.sh 607](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L607-L607)

#### 2. Firmware Management (tt-flash)

The tt-flash tool manages device firmware updates for Tenstorrent hardware.

*   **Repository**: `https://github.com/tenstorrent/tt-flash.git`
*   **Purpose**: Updates firmware on Tenstorrent devices
*   **Dependencies**: Requires Kernel-Mode Driver
*   **Firmware Source**: Firmware packages from `https://github.com/tenstorrent/tt-firmware.git`

Sources: [install.sh 29-34](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L29-L34)[install.sh 562](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L562-L562)[install.sh 555-571](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L555-L571)

### Development Environment Components

These components provide a containerized development environment for working with Tenstorrent hardware.

Sources: [install.sh 276-307](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L276-L307)[install.sh 309-354](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L309-L354)[install.sh 47-48](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L47-L48)

#### 1. Podman Container Runtime

Podman provides the containerization layer for running the Metalium development environment.

*   **Purpose**: Runs containerized applications without requiring root privileges
*   **Installation**: Installed via system package manager
*   **Dependencies**: None (independent component)

Sources: [install.sh 276-307](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L276-L307)

#### 2. tt-metalium Container

The tt-metalium container provides a preconfigured development environment for working with Tenstorrent hardware.

*   **Image Source**: `ghcr.io/tenstorrent/tt-metal/tt-metalium-ubuntu-22.04-release-amd64`
*   **Purpose**: Provides a complete development environment with pre-installed libraries and tools
*   **Dependencies**: Podman, Kernel-Mode Driver, HugePages
*   **Access Method**: Via the `tt-metalium` wrapper script located at `$HOME/.local/bin/tt-metalium`

Sources: [install.sh 47-48](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L47-L48)[install.sh 309-354](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L309-L354)

## Component Mapping to Operating System

The following table shows how the components map to operating system resources:

| Component | Operating System Resource | Purpose |
| --- | --- | --- |
| Kernel-Mode Driver | Kernel module (`tenstorrent.ko`) Device file (`/dev/tenstorrent`) | Hardware communication |
| HugePages | System service (`tenstorrent-hugepages.service`) Mount point (`/dev/hugepages-1G`) | Large memory page allocation |
| tt-smi | Python package (`tt-smi`) Command-line utility | Hardware monitoring |
| tt-flash | Python package (`tt-flash`) Command-line utility | Firmware updates |
| Podman | System package (`podman`) | Container management |
| tt-metalium | Container image Wrapper script (`$HOME/.local/bin/tt-metalium`) | Development environment |
