This document covers the installation process, configuration, and initial setup of the Tenstorrent System Management Interface (TT-SMI) tool. It provides step-by-step instructions for both standard users and developers, system requirements, and verification procedures to ensure proper installation. For information about using the command-line interface after installation, see [Command-line Interface](https://deepwiki.com/tenstorrent/tt-smi/2.1-basic-cli-operations).

## System Requirements

TT-SMI requires the following software components to be installed and properly configured on your system:

*   **Python**: Version 3.7 or newer (supports up to Python 3.11)
*   **Rust**: Required for the Luwen library dependency
*   **Operating Systems**: Linux distributions (tested on Ubuntu 20.04+)
*   **Hardware**: Access to Tenstorrent hardware devices (Grayskull, Wormhole, or Blackhole)

## Installation Methods

TT-SMI can be installed using different methods depending on your use case. This section covers both standard installation for end users and development installation for contributors.

### Standard Installation

#### 1. Install Rust (Required for Luwen Library)

`curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | shsource "$HOME/.cargo/env"`
#### 2. Optional: Create Python Virtual Environment

Creating a virtual environment is recommended to isolate TT-SMI dependencies from your system Python packages:

`python3 -m venv .venvsource .venv/bin/activatepip3 install --upgrade pip`
#### 3. Install TT-SMI

Once your environment is prepared, install TT-SMI:

`pip3 install .`
This will install the `tt-smi` command-line tool and all its dependencies.

Sources: [README.md 21-42](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/README.md?plain=1#L21-L42)

### Development Installation

For developers who want to modify TT-SMI code without rebuilding:

#### 1. Set Up a Development Environment

`python3 -m venv .venvsource .venv/bin/activatepip install --upgrade pippip install pre-commit`
#### 2. Install in Editable Mode

`pip install --editable .`
#### 3. Install Pre-commit Hooks (Recommended)

`pre-commit install`
This configures automatic code formatting on commit.

## Installation Verification

After installing TT-SMI, you can verify that the installation was successful:

### Version Check

`tt-smi --version`
This should display the current version of TT-SMI (e.g., 3.0.15).

### List Available Boards

`tt-smi --list`
This command lists all Tenstorrent devices available on your system.

### Launch GUI

`tt-smi`
Running `tt-smi` without any arguments should launch the text-based user interface (TUI) if installation was successful.

Sources: [README.md 65-105](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/README.md?plain=1#L65-L105)

## Configuration

TT-SMI uses configuration files for specialized functionality such as device resets. These files are typically stored in the user's home directory.

### Configuration File Locations

| File | Path | Purpose |
| --- | --- | --- |
| Reset Configuration | `~/.config/tenstorrent/reset_config.json` | Defines reset behavior for Tenstorrent devices |
| Snapshot Data | `~/tt_smi/<timestamp>_snapshot.json` | Default location for system snapshot data |

### Generating Default Configuration

To generate a default reset configuration file:

`tt-smi --generate_reset_json`
This creates a default configuration file at `~/.config/tenstorrent/reset_config.json` that can be customized for your specific needs.

## Troubleshooting Installation Issues

### Common Problems and Solutions

1.   **Missing Rust Environment**

    *   Symptom: Installation fails with errors about Rust or cargo
    *   Solution: Ensure Rust is installed and properly sourced with `source "$HOME/.cargo/env"`

2.   **Dependency Conflicts**

    *   Symptom: Installation reports package conflicts
    *   Solution: Use a clean virtual environment or check for conflicting packages with `pip list`

3.   **Permission Issues**

    *   Symptom: Installation fails with permission errors
    *   Solution: Use `pip install --user` or set up a virtual environment

4.   **Hardware Access Issues**

    *   Symptom: TT-SMI runs but cannot detect hardware
    *   Solution: Ensure proper driver installation and permissions for Tenstorrent devices

## Next Steps

After successful installation, you can:

1.   Run `tt-smi` to launch the interactive TUI
2.   Learn about CLI options with `tt-smi --help`
3.   Generate system snapshots with `tt-smi --snapshot`
4.   Reset devices with `tt-smi --reset`

For detailed information about using the command-line interface, see [Command-line Interface](https://deepwiki.com/tenstorrent/tt-smi/2.1-basic-cli-operations). For information about the TUI, see [Text User Interface (TUI)](https://deepwiki.com/tenstorrent/tt-smi/2.2-reset-commands).
