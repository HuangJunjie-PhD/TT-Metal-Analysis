## Purpose and Scope

This page documents the system-level dependencies required by RiESCUE, as installed by the [infra/build\_base\_deps.sh](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh) script. These are foundational packages for the Rocky Linux 9.1 environment that must be present before building or running any RiESCUE components.

For information about building Whisper and Spike simulators using these dependencies, see [Building Whisper](/tenstorrent/riescue/3.2.2-building-whisper) and [Building Spike](/tenstorrent/riescue/3.2.3-building-spike). For the container environment that orchestrates this installation, see [Container Definition and Build](/tenstorrent/riescue/3.1.1-container-definition-and-build). For Python package dependencies, see [Python Package Configuration](/tenstorrent/riescue/3.3-python-package-configuration).

## Overview

The base dependencies provide the compilation toolchains, system libraries, and development tools required by RiESCUE's external components. The installation is performed via the [infra/build\_base\_deps.sh1-39](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L1-L39) script, which uses the `yum` package manager to install packages from Rocky Linux 9.1 repositories.

**Sources:** [infra/build\_base\_deps.sh1-39](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L1-L39) [infra/Container.def29-30](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/Container.def#L29-L30) [.github/workflows/smoke.yml43-45](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/workflows/smoke.yml#L43-L45)

## Installation Script Architecture

The [infra/build\_base\_deps.sh](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh) script is a simple bash script that performs a single `yum` installation command with the CRB (Code Ready Builder) repository enabled.

### Script Structure

| Component | Description | Location |
| --- | --- | --- |
| Shebang | Bash interpreter directive | [Line 1](https://github.com/tenstorrent/riescue/blob/07cefde3/Line 1) |
| SPDX Headers | License and copyright information | [Lines 2-3](https://github.com/tenstorrent/riescue/blob/07cefde3/Lines 2-3) |
| yum Command | Package installation with CRB repo | [Lines 5-37](https://github.com/tenstorrent/riescue/blob/07cefde3/Lines 5-37) |

The script uses `yum --enablerepo=crb -y install` to enable the CRB repository, which provides additional development packages not available in the base Rocky Linux repositories. The `-y` flag automatically confirms installation prompts for non-interactive execution.

**Sources:** [infra/build\_base\_deps.sh1-39](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L1-L39)

## Integration with Build System

### Container Build Integration

The base dependencies are installed as the first step in the container build process:

The container build sequence ([infra/Container.def28-56](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/Container.def#L28-L56)):

1. Enter `%post` section for container customization
2. Execute `./build_base_deps.sh` to install system packages
3. Install Python dependencies using `pip`
4. Build Whisper using `./build_whisper.sh`
5. Build Spike using `./build_spike.sh`

**Sources:** [infra/Container.def28-56](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/Container.def#L28-L56)

## CRB Repository Requirement

The `--enablerepo=crb` flag in the `yum` command ([infra/build\_base\_deps.sh5](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L5-L5)) enables the Code Ready Builder repository. This repository is essential because it provides development packages not included in Rocky Linux's base repositories.

| Package | Why CRB is Required |
| --- | --- |
| `gcc-toolset-12` | Modern compiler toolchain beyond base GCC |
| Development headers | Many `-devel` packages are in CRB |
| Boost libraries | Complete Boost suite requires CRB |

Without CRB enabled, the installation would fail with package not found errors for several critical dependencies.

**Sources:** [infra/build\_base\_deps.sh5](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L5-L5)

## Non-Interactive Installation

The script uses the `-y` flag ([infra/build\_base\_deps.sh5](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L5-L5)) to automatically answer "yes" to all prompts. This is critical for:

1. **Container Builds**: Singularity container builds cannot handle interactive prompts
2. **CI/CD Execution**: GitHub Actions workflows require non-interactive execution
3. **Automation**: Script can be called programmatically without user intervention

Without this flag, the script would hang waiting for user confirmation in automated environments.

**Sources:** [infra/build\_base\_deps.sh5](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L5-L5)

## Troubleshooting

### Common Installation Issues

| Issue | Cause | Solution |
| --- | --- | --- |
| Package not found errors | CRB repository not enabled | Verify `--enablerepo=crb` flag is present |
| Connection timeouts | Network connectivity to Rocky repos | Check network access and retry |
| Dependency conflicts | Incompatible package versions | Use Rocky Linux 9.1 as specified |
| Permission denied | Insufficient privileges | Run with root or sudo privileges |
