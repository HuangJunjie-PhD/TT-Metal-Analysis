## Purpose and Scope

This document provides a detailed reference for the Whisper instruction set simulator (ISS) configuration files used by RiESCUE. These JSON files define the hardware architecture, ISA extensions, CSR behavior, memory maps, and other parameters that configure Whisper's simulation environment. For information about using the Whisper simulator itself, see [Whisper Simulator](https://deepwiki.com/tenstorrent/riescue/4.2.1-whisper-simulator). For CPU configuration files used by test generation, see [CPU Configuration](https://deepwiki.com/tenstorrent/riescue/7.2.1-cpu-configuration).

RiESCUE includes three pre-configured Whisper configuration variants located at [riescue/dtest_framework/lib/](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/):

*   `whisper_config.json` - Standard configuration with custom physical memory attributes
*   `whisper_basic_config.json` - Basic configuration with physical memory protection
*   `whisper_secure_config.json` - Secure configuration with STEE and extended PMP support

Sources: [riescue/dtest_framework/lib/whisper_config.json](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json)[riescue/dtest_framework/lib/whisper_basic_config.json](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json)[riescue/dtest_framework/lib/whisper_secure_config.json](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json)

## Configuration File Variants

### Configuration Comparison

The three Whisper configuration files share a common structure but differ in specific security and memory protection features:

| Configuration | File | Primary Use Case | Key Distinguishing Features |
| --- | --- | --- | --- |
| Standard | `whisper_config.json` | General testing | Custom PMACFG (Physical Memory Attribute Configuration), no PMP, larger performance counter count (8) |
| Basic | `whisper_basic_config.json` | Basic compliance | Physical Memory Protection (PMP) enabled with 16 regions, standard PMACFG, no cycle/instret counters |
| Secure | `whisper_secure_config.json` | Security testing | STEE (Secure Trusted Execution Environment), full PMP (64 regions), extensive PMA definitions, smaller memory (52-bit) |

## Top-Level Parameters

### Basic Architecture Parameters

These parameters define the fundamental processor architecture:

| Parameter | Type | Standard | Basic | Secure | Description |
| --- | --- | --- | --- | --- | --- |

### ISA String

All three configurations define a comprehensive ISA string at [whisper_config.json 3](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_config.json#L3-L3)[whisper_basic_config.json 3](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_basic_config.json#L3-L3) and [whisper_secure_config.json 3](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_secure_config.json#L3-L3):

```
rv64imafdchvus_svinval_zba_zbb_zbc_zbs_zfa_zfh_zfbfmin_zvfbfmin_zvfbfwma_
zvkg_zvkned_zvknhb_zvfh_zvbb_zvbc_zbkb_zbkx_zknd_zkne_zksed_zksh_zicbom_

```

This includes:

*   **Base ISAs**: `i` (integer), `m` (multiply/divide), `a` (atomic), `f` (single-precision float), `d` (double-precision float), `c` (compressed), `h` (hypervisor), `v` (vector), `u` (user mode), `s` (supervisor mode)
*   **Bit manipulation**: `zba`, `zbb`, `zbc`, `zbs`, `zbkb`, `zbkx`
*   **Floating-point**: `zfa`, `zfh`, `zfbfmin`
*   **Vector**: `zvfbfmin`, `zvfbfwma`, `zvkg`, `zvkned`, `zvknhb`, `zvfh`, `zvbb`, `zvbc`, `zvqdot`
*   **Cryptography**: `zknd`, `zkne`, `zksed`, `zksh`
*   **Cache management**: `zicbom`, `zicboz`
*   **Miscellaneous**: `zicond`, `zihintpause`, `zifencei`, `zimop`, `zcmop`, `zawrs`, `zjpm`
*   **Supervisor extensions**: `svinval`, `svpbmt`, `svadu`, `sstc`

## CSR Configuration

### CSR Configuration Structure

Control and Status Registers (CSRs) are configured in the `csr` object. Each CSR can specify:

*   `exists`: Boolean indicating if the CSR is implemented
*   `reset`: Initial value at reset (hex string)
*   `mask`: Write mask (which bits are writable)
*   `poke_mask`: Additional mask for direct CSR modification (used in testing)
*   `number`: CSR number (for custom CSRs)

### Key CSR Configurations

#### MISA Register

All configurations set MISA to indicate supported extensions at [whisper_config.json 23-28](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_config.json#L23-L28):

`"misa" : {    "reset-comment" : "imabfcvsu",    "reset" : "0x403411af",    "mask-comment" : "Misa is not writable by CSR instructions",    "mask" : "0x0"}`
The reset value `0x403411af` encodes the enabled ISA extensions. The mask `0x0` makes MISA read-only.

#### MSTATUS Register

All configurations initialize MSTATUS with floating-point and vector state enabled:

`"mstatus" : {    "mstatus-comment" : "FIXME : Hardwired to zero except for FS, VS, and SD.",    "reset" : "0x47804",    "mask" : "0xffffffffffffffff"}`
The reset value `0x47804` sets:

*   FS (bits 13-14): Floating-point state = Dirty
*   VS (bits 9-10): Vector state = Dirty
*   SD (bit 63): State dirty = 1

#### Counter Access Configuration

**Standard and Secure** ([whisper_config.json 29-37](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_config.json#L29-L37)[whisper_secure_config.json](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_secure_config.json#LNaN-LNaN)):

`"mcounteren": {    "mcounteren-comment" : "FIXME: Hardwired to 2 to enable time csr access.",    "reset" : "0x2"},"scounteren": {    "scounteren-comment" : "Hardwired to 2 to enable time csr access.",    "reset" : "0x2"}`
These settings enable the TIME CSR (bit 1) for lower privilege levels.

**Basic Configuration** differs in counter existence ([whisper_basic_config.json 130-140](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_basic_config.json#L130-L140)):

`"cycle" : { "exists" : "false" },"instret" : { "exists" : "false" },"time" : { "exists" : "true" }`
### Debug CSRs

All configurations disable debug CSRs by hardwiring them to zero. Example for TSELECT at [whisper_config.json 69-73](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_config.json#L69-L73):

`"tselect" : {    "Comment" : "Hardwired to 0",    "reset" : "0x0",    "mask" : "0x0"}`
Similarly hardwired to 0: `tdata1`, `tdata2`, `tdata3`, `dcsr`, `dpc`, `dscratch`, `dscratch1`.

Sources: [riescue/dtest_framework/lib/whisper_config.json 22-116](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L22-L116)[riescue/dtest_framework/lib/whisper_basic_config.json 22-124](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json#L22-L124)[riescue/dtest_framework/lib/whisper_secure_config.json 64-156](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L64-L156)

## Physical Memory Attribute Configuration (PMACFG)

### PMACFG Registers

PMACFG is a custom (non-standard RISC-V) CSR extension for physical memory attribute configuration. All three configurations implement 16 PMACFG registers (`pmacfg0` through `pmacfg15`) at CSR addresses `0x7e0-0x7ef`.

### PMACFG Register Structure

Each PMACFG register is defined with:

*   `number`: CSR address (hex)
*   `reset`: Initial value
*   `mask`: Write-enable mask
*   `poke_mask`: Testing mask

### Configuration Differences

**Standard Configuration** ([whisper_config.json 205-302](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_config.json#L205-L302)):

*   `pmacfg14` (0x7ee): `0x7c00000000000007` - Covers 0-2GB for I/O
*   `pmacfg15` (0x7ef): `0xd0000000000001e7` - Covers 2GB-56bit for DRAM

**Basic Configuration** ([whisper_basic_config.json 218-313](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_basic_config.json#L218-L313)):

*   `pmacfg0` (0x7e0): `0xd000000000000017` - Special initial configuration
*   `pmacfg1-15`: `0x0` - All other registers start at zero

**Secure Configuration** ([whisper_secure_config.json 249-346](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_secure_config.json#L249-L346)):

*   `pmacfg0` (0x7e0): `0xd0000000800001e7` - Secure memory configuration
*   `pmacfg14` (0x7ee): `0x7c00000000000007` - I/O region (0-2GB)
*   `pmacfg15` (0x7ef): `0xd0000000000001e7` - DRAM region (2GB-52bit)

Sources: [riescue/dtest_framework/lib/whisper_config.json 205-302](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L205-L302)[riescue/dtest_framework/lib/whisper_basic_config.json 218-313](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json#L218-L313)[riescue/dtest_framework/lib/whisper_secure_config.json 249-346](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L249-L346)

## Vector Configuration

All three configurations include identical vector extension configuration at [whisper_config.json 332-349](https://github.com/tenstorrent/riescue/blob/07cefde3/whisper_config.json#L332-L349):

`"vector" : {    "bytes_per_vec":        32,    "min_bytes_per_elem":   1,    "Comment":              "Hardware supports 32-bit ELEN plus some widening operations from 32 to 64-bit",    "max_bytes_per_elem":   8,    "mask_agnostic_policy": "ones",    "tail_agnostic_policy": "ones",    "update_whole_mask":    true,    "trap_invalid_vtype":   false,    "trap_non_zero_vstart": true,    "legalize_vsetvl_avl":  true,    "legalize_vsetvli_avl": true,    "legalize_for_egs":     true,    "max_bytes_per_lmul":   { "mf8": 1, "mf4": 2, "mf2": 4 },    "tt_fp_usum_tree_reduction": ["e16", "e32", "e64"],    "fp_usum_nan_canonicalize": ["e16", "e32", "e64"],    "partial_segment_update": false}`
### Vector Parameters

| Parameter | Value | Description |
| --- | --- | --- |
| `bytes_per_vec` | 32 | Vector register size (VLEN = 256 bits) |
| `min_bytes_per_elem` | 1 | Minimum element width (8 bits) |
| `max_bytes_per_elem` | 8 | Maximum element width (64 bits, with widening) |
| `mask_agnostic_policy` | "ones" | Write 1s to inactive mask elements |
| `tail_agnostic_policy` | "ones" | Write 1s to tail elements |
| `update_whole_mask` | true | Always update full mask register |
| `trap_invalid_vtype` | false | Don't trap on invalid vtype |
| `trap_non_zero_vstart` | true | Trap if vstart != 0 (not supported) |
| `legalize_vsetvl_avl` | true | Validate AVL in vsetvl |
| `legalize_vsetvli_avl` | true | Validate AVL in vsetvli |
| `legalize_for_egs` | true | Enforce element group constraints |
| `partial_segment_update` | false | Don't allow partial segment updates |

### LMUL Constraints

The `max_bytes_per_lmul` object defines maximum vector length for fractional LMUL values:

| LMUL | Max Bytes | Max Elements (8-bit) |
| --- | --- | --- |
| mf8 | 1 | 1 |
| mf4 | 2 | 2 |
| mf2 | 4 | 4 |

### Custom Vector Behaviors

*   `tt_fp_usum_tree_reduction`: Tenstorrent-specific FP unordered sum uses tree reduction for e16, e32, e64
*   `fp_usum_nan_canonicalize`: Canonicalize NaNs in FP unordered sum for e16, e32, e64

## Usage in RiESCUE

### Configuration File Selection

The RiESCUE framework automatically selects the appropriate Whisper configuration based on command-line flags or test requirements. The `Whisper` tool class in the toolchain abstraction layer handles configuration file selection and passing to the Whisper executable.

### Typical Usage Flow

The Whisper tool class constructs the appropriate command line:

```
whisper --configfile <config_path> <elf_file>
```

Where `<config_path>` is one of:

*   `riescue/dtest_framework/lib/whisper_config.json` (default)
*   `riescue/dtest_framework/lib/whisper_basic_config.json` (basic mode)
*   `riescue/dtest_framework/lib/whisper_secure_config.json` (secure mode)

## Configuration Customization

### Creating Custom Configurations

To create a custom Whisper configuration:

1.   Copy one of the existing configuration files as a template
2.   Modify the relevant sections (ISA string, CSR values, memory map, etc.)
3.   Save with a descriptive name (e.g., `whisper_custom_config.json`)
4.   Reference the custom file when invoking Whisper

### Common Customizations

| Customization | Sections to Modify | Use Case |
| --- | --- | --- |
| ISA extensions | `isa` string | Test specific RISC-V extensions |
| Memory size | `memmap.size` | Simulate different memory capacities |
| CSR reset values | `csr.<register>.reset` | Test boot behavior |
| PMP/PMACFG | `csr.pmpcfg*`, `csr.pmpaddr*`, `csr.pmacfg*` | Test memory protection schemes |
| Vector parameters | `vector.*` | Simulate different vector implementations |
| Performance counters | `num_mmode_perf_regs` | Adjust available counters |

### Validation

After creating a custom configuration, validate it by:

1.   Running Whisper with `--configfile <custom_config>`
2.   Checking for configuration parsing errors
3.   Verifying CSR values match expectations using test programs
4.   Confirming memory map is correctly interpreted
