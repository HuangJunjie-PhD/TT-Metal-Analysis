## System Requirements

### Operating System

tt-npe is developed and tested on Ubuntu Linux. The CI pipeline validates builds on Ubuntu (latest stable release).

Sources: [.github/workflows/build_and_test_ubuntu.yml 21](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/.github/workflows/build_and_test_ubuntu.yml#L21-L21)

### Compiler Requirements

tt-npe requires a C++20-compliant compiler. The following compilers are supported:

| Compiler | Version | Status |
| --- | --- | --- |
| Clang | 17 | Default and fully tested |
| GCC | 12+ | Supported with warnings for versions > 12 |

**Default Behavior**: If no compiler is explicitly specified, the build system automatically searches for and uses `clang-17` and `clang++-17`.

**Custom Compiler**: Set environment variables before running the build:

`export CMAKE_C_COMPILER=/path/to/compilerexport CMAKE_CXX_COMPILER=/path/to/compiler++`
Sources: [CMakeLists.txt 15-27](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L15-L27)[cmake/compilers.cmake 5-15](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/cmake/compilers.cmake#L5-L15)[cmake/compilers.cmake 31-38](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/cmake/compilers.cmake#L31-L38)

### Build Tools

*   **CMake**: Version 3.20 or higher required
*   **Ninja**: Used as the build system generator (automatically invoked by `build-npe.sh`)
*   **Python**: Python 3.x for running Python bindings and scripts

Sources: [CMakeLists.txt 1](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L1-L1)[build-npe.sh 26](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/build-npe.sh#L26-L26)

## Building from Source

### Clone Repository

`git clone git@github.com:tenstorrent/tt-npe.gitcd tt-npe/`
Sources: [README.md 56-57](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/README.md?plain=1#L56-L57)

### Run Build Script

The `build-npe.sh` script handles the entire build process:

`# Release build (default)./build-npe.sh # Debug build (with symbols and no optimization)./build-npe.sh debug`
**Build Script Behavior**:

1.   Configures CMake with Ninja generator
2.   Builds all targets in parallel (16 jobs by default)
3.   Removes previous installation
4.   Installs binaries, libraries, and headers to `./install`

The build type can be specified as an argument:

*   **Release**: Optimized build (`-O3`)
*   **Debug**: Debug symbols, no optimization (`-O0 -g`)
*   **RelWithDebInfo**: Optimized with debug symbols (`-O3 -g`)
*   **CI**: Optimized for continuous integration (`-O3`)

Sources: [build-npe.sh 1-34](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/build-npe.sh#L1-L34)[CMakeLists.txt 87-96](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L87-L96)[CMakeLists.txt 116-119](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L116-L119)

### Build Customization

The CMake configuration can be customized through environment variables or CMake options:

`# Use specific compilerexport CMAKE_C_COMPILER=gcc-12export CMAKE_CXX_COMPILER=g++-12./build-npe.sh # Enable sanitizers (for development)cmake -B build -DENABLE_ASAN=ONcmake --build build`
Supported sanitizer options:

*   `ENABLE_ASAN`: AddressSanitizer (memory errors)
*   `ENABLE_MSAN`: MemorySanitizer (uninitialized reads)
*   `ENABLE_TSAN`: ThreadSanitizer (data races)
*   `ENABLE_UBSAN`: UndefinedBehaviorSanitizer

Note: Only one sanitizer can be enabled at a time.

Sources: [CMakeLists.txt 131-148](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L131-L148)[CMakeLists.txt 185-195](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L185-L195)

## Environment Configuration

### ENV_SETUP Script

After building, configure your shell environment by sourcing the `ENV_SETUP` script:

`source ENV_SETUP`
**Important**: If you are using tt-npe with tt-metal profiler integration, source the tt-metal Python virtualenv setup **first**, then source `ENV_SETUP`:

`# Correct order for tt-metal integrationsource /path/to/tt-metal/python_env/bin/activatesource ENV_SETUP`
Sources: [README.md 68-73](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/README.md?plain=1#L68-L73)[docs/src/getting_started.md 17-24](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/docs/src/getting_started.md?plain=1#L17-L24)

### ENV_SETUP Functionality

The script performs the following actions:

1.   **Locates Repository Root**: Uses the `get_repo_location()` function to find the tt-npe directory
2.   **Checks for Updates**: Compares local branch with remote using `check_for_updates()`
3.   **Installs Python Requirements**: Runs `uv pip install -r requirements.txt`
4.   **Validates Installation**: Checks that `install/bin/` and `install/lib/` exist
5.   **Configures Environment**: 
    *   Adds `install/bin/` to `PATH` (for CLI tools)
    *   Adds `install/lib/` to `PYTHONPATH` (for Python bindings)
    *   Adds `install/bin/` to `PYTHONPATH` (for Python scripts)

Sources: [ENV_SETUP 2-69](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/ENV_SETUP#L2-L69)

### Virtual Environment Creation (Optional)

Two scripts are provided for creating isolated Python virtual environments if desired:

`# Using uv (faster)./create_venv_uv.sh # Using standard venv./create_venv.sh`
These are typically not required if you're using tt-metal's existing Python environment.

Sources: [create_venv_uv.sh 1-30](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/create_venv_uv.sh#L1-L30)[create_venv.sh 1-44](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/create_venv.sh#L1-L44)

## Installation Directory Structure

After a successful build and installation, the `install/` directory contains:

### Directory Contents

| Directory | Contents | Purpose |
| --- | --- | --- |
| `install/lib/` | `libtt_npe.so`, Python bindings | Shared libraries loaded at runtime |
| `install/bin/` | `tt_npe.py`, `tt_npe_run`, Python scripts | Executables and user-facing tools |
| `install/include/` | `npeAPI.hpp`, device headers | C++ API for external integration |

**Relocation Support**: The shared library `libtt_npe.so` is built with RPATH set to `$ORIGIN/../lib`, allowing the entire `install/` directory to be moved without breaking library paths.

Sources: [README.md 110-115](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/README.md?plain=1#L110-L115)[docs/src/api.md 3-8](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/docs/src/api.md?plain=1#L3-L8)[CMakeLists.txt 162](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L162-L162)

### Running Unit Tests

tt-npe includes comprehensive unit test suites for both C++ and Python code. Verify your installation by running:

`# Must be run after sourcing ENV_SETUPtt_npe/scripts/run_ut.sh`
This script executes:

1.   **C++ Unit Tests**: Built with GoogleTest (`tt_npe_ut` executable)
2.   **Python Unit Tests**: Run with pytest

Sources: [README.md 119-123](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/README.md?plain=1#L119-L123)[docs/src/api.md 13-17](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/docs/src/api.md?plain=1#L13-L17)[.github/workflows/build_and_test_ubuntu.yml 40-43](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/.github/workflows/build_and_test_ubuntu.yml#L40-L43)

Test the installation with a simple workload simulation:

## Troubleshooting

### Build Errors

**Problem**: CMake cannot find Clang-17

```
Clang-17 not found. Make sure you have clang-17 and clang++-17 installed
```

**Solution**: Install Clang-17 or specify a different compiler:

`export CMAKE_CXX_COMPILER=g++-12export CMAKE_C_COMPILER=gcc-12./build-npe.sh`
Sources: [cmake/compilers.cmake 9-11](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/cmake/compilers.cmake#L9-L11)

**Problem**: CMake version too old

```
CMake 3.20 or higher required
```

**Solution**: Upgrade CMake or build from source.

Sources: [CMakeLists.txt 1](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L1-L1)

### Environment Setup Issues

**Problem**: `ENV_SETUP` reports missing install directory

```
E: tt-npe install directory not found at './install'
```

**Solution**: Build the project first:

`./build-npe.shsource ENV_SETUP`
Sources: [ENV_SETUP 67-68](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/ENV_SETUP#L67-L68)

**Problem**: Python import errors after sourcing ENV_SETUP

```
ModuleNotFoundError: No module named 'tt_npe_pybind'
```

**Solution**: Check that PYTHONPATH includes `install/lib/`:

`echo $PYTHONPATH | grep "install/lib"`
If not present, re-source `ENV_SETUP`. If using tt-metal, ensure tt-metal's environment was sourced **before**`ENV_SETUP`.

Sources: [ENV_SETUP 64-65](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/ENV_SETUP#L64-L65)[README.md 69](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/README.md?plain=1#L69-L69)

### Linker Errors

**Problem**: Runtime linker cannot find `libtt_npe.so`

```
error while loading shared libraries: libtt_npe.so: cannot open shared object file
```

**Solution**: Ensure `LD_LIBRARY_PATH` includes `install/lib/` or rely on RPATH (which should work automatically for installed binaries):

`export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$(pwd)/install/lib`
Sources: [CMakeLists.txt 162](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/CMakeLists.txt#L162-L162)

### Update and Rebuild

To update tt-npe to the latest version:

`cd tt-npe/git pull./build-npe.shsource ENV_SETUP`
The `ENV_SETUP` script automatically checks if your local branch is behind the remote and warns you to rebuild.

Sources: [README.md 62-64](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/README.md?plain=1#L62-L64)[ENV_SETUP 22-47](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/ENV_SETUP#L22-L47)
