## Overview and Purpose

The tt-metalium container is a containerized development environment that provides Tenstorrent's library for building and running AI models on Tenstorrent hardware. This page documents the tt-metalium container's architecture, installation process, and usage. The container encapsulates all necessary dependencies, libraries, and tools required for developing with Tenstorrent hardware, providing a consistent environment regardless of the host system configuration.

For information about installing the Kernel Mode Driver (KMD), which is a prerequisite for using tt-metalium, see [Kernel-Mode Driver (KMD)](https://deepwiki.com/tenstorrent/tt-installer/5.1-kernel-mode-driver-(kmd)). For HugePages configuration, which is also required, see [HugePages Configuration](https://deepwiki.com/tenstorrent/tt-installer/5.3-hugepages-configuration).

## Installation Process

The tt-metalium container is installed via Podman during the `tt-installer` process. The installation involves:

1.   Checking if Podman is installed
2.   Installing Podman if needed
3.   Creating a wrapper script for easy usage
4.   Pulling the container image

## Container Configuration

The tt-metalium container is configured with the following mappings and settings:

| Resource | Host Location | Container Location | Purpose |
| --- | --- | --- | --- |
| HugePages | `/dev/hugepages-1G` | `/dev/hugepages-1G` | High-performance memory access |
| Home Directory | `${HOME}` | `/home/user` | Access to user files |
| Tenstorrent Device | `/dev/tenstorrent` | `/dev/tenstorrent` | Access to hardware |
| Working Directory | Current directory | `/home/user` | Preserve working context |
| Display | `${DISPLAY}` | `${DISPLAY}` | X11 forwarding for GUI apps |
| Network | Host network | Host network | Direct network access |

## Using the Container

The tt-metalium container can be used in several ways:

1.   **Interactive Shell**: Run `tt-metalium` to start an interactive shell inside the container
2.   **Run Commands**: Execute `tt-metalium [command]` to run specific commands
3.   **Run Python Scripts**: Use `tt-metalium python script.py` to execute Python scripts

The container mounts your home directory, so you can access your files from within the container without copying them.

## Container Image Details

By default, the tt-installer pulls the following container image:

*   **Image URL**: `ghcr.io/tenstorrent/tt-metal/tt-metalium-ubuntu-22.04-release-amd64`
*   **Tag**: `latest-rc`

These values can be customized through environment variables during installation:

*   `TT_METALIUM_IMAGE_URL`: To specify a different image source
*   `TT_METALIUM_IMAGE_TAG`: To specify a different image tag

## Installation Options

During the tt-installer process, users can choose whether to install the tt-metalium container. This choice can be controlled through:

1.   **Interactive Selection**: The installer will prompt the user unless in non-interactive mode
2.   **Environment Variable**: Setting `TT_SKIP_INSTALL_METALIUM_CONTAINER=0` will skip installation
3.   **Operating System Compatibility**: On Ubuntu 20.04, Metalium installation is automatically skipped as it's not supported

The installation can be customized through various environment variables:

| Environment Variable | Purpose | Default |
| --- | --- | --- |
| `TT_SKIP_INSTALL_METALIUM_CONTAINER` | Skip Metalium installation | `1` (install) |
| `TT_SKIP_INSTALL_PODMAN` | Skip Podman installation | `1` (install) |
| `TT_METALIUM_IMAGE_URL` | Specify container image URL | `ghcr.io/tenstorrent/tt-metal/tt-metalium-ubuntu-22.04-release-amd64` |
| `TT_METALIUM_IMAGE_TAG` | Specify container image tag | `latest-rc` |

## Container Dependencies

The tt-metalium container depends on:

1.   **Podman**: Container runtime used to run the tt-metalium image
2.   **HugePages**: Memory management configuration required for high-performance operation
3.   **Kernel Mode Driver (KMD)**: Driver that provides access to Tenstorrent hardware

If any of these components are missing, the container may not function correctly.

## Troubleshooting

Common issues when using the tt-metalium container:

1.   **PATH Issues**: If `~/.local/bin` is not in your PATH, you may need to update your shell RC file or restart your session to use the `tt-metalium` command.

2.   **Missing Device Access**: If the container can't access the Tenstorrent hardware, verify that:

    *   The KMD is properly installed
    *   You have permission to access `/dev/tenstorrent`
    *   The device is properly recognized by the system (`tt-smi` can help verify this)

3.   **HugePages Issues**: If the container can't access HugePages, verify that HugePages are properly configured using the instructions in [HugePages Configuration](https://deepwiki.com/tenstorrent/tt-installer/5.3-hugepages-configuration).
