This page covers the tt-mlir development environment: the `ttmlir-toolchain` directory layout, the `env/activate` scripts, the CMake-based toolchain build that compiles LLVM, Flatbuffers, StableHLO, and Shardy as `ExternalProject` dependencies, the Python virtual environment, the Docker images used in CI, and the key development workflow.

* * *

## Overview

The development environment is structured into two independent layers to separate infrastructure from the compiler logic:

1.   **Toolchain** (`TTMLIR_TOOLCHAIN_DIR`, default `/opt/ttmlir-toolchain`) — This contains the pre-built compiler infrastructure: LLVM/MLIR, Flatbuffers, StableHLO sources, Shardy sources, and a Python virtual environment. It is built using `env/CMakeLists.txt`[env/CMakeLists.txt 33-107](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/CMakeLists.txt#L33-L107)
2.   **Project Build** (`build/`) — This is the `tt-mlir` compiler itself, built by the top-level `CMakeLists.txt` and linked against the toolchain.

The `env/activate` script is the entry point for development; it must be sourced to wire together system paths, `PYTHONPATH`, and Tenstorrent-specific runtime roots [env/activate 1-32](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/activate#L1-L32)

### Key Environment Variables

| Variable | Value | Purpose |
| --- | --- | --- |
| `TTMLIR_TOOLCHAIN_DIR` | `/opt/ttmlir-toolchain` (default) | Root for all toolchain artifacts [env/activate 2-4](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/activate#L2-L4) |
| `TTMLIR_VENV_DIR` | `$TTMLIR_TOOLCHAIN_DIR/venv` | Python virtual environment location [env/activate 12-14](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/activate#L12-L14) |
| `TT_METAL_RUNTIME_ROOT` | `$(pwd)/third_party/tt-metal/src/tt-metal` | tt-metal source root for runtime [env/activate 26](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/activate#L26-L26) |
| `TTMLIR_ENV_ACTIVATED` | `1` | Flag to verify environment is ready [env/activate 24](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/activate#L24-L24) |

The toolchain build system uses `execute_process` to call the `activate` script and retrieve the directory path [env/CMakeLists.txt 13-16](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/env/CMakeLists.txt#L13-L16)

## Toolchain and Dependency Management

The toolchain is built as a separate project within the `env/` directory. It manages external dependencies that change less frequently than the core compiler code.

## Docker Infrastructure

The project uses a multi-stage Docker strategy to provide reproducible build and test environments.

### `Dockerfile.base`

The base image installs essential system dependencies and Clang.

*   **System Tools**: Includes `build-essential`, `ninja-build`, `cmake`, `ccache`, `doxygen`, and `pkg-config`[.github/Dockerfile.base 14-38](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/.github/Dockerfile.base#L14-L38)
*   **Hardware Dependencies**: Executes `install_dependencies.sh` from the `tt-metal` repository using a pinned commit (`TT_METAL_DEPENDENCIES_COMMIT`) [.github/Dockerfile.base 7-46](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/.github/Dockerfile.base#L7-L46)
*   **LLVM 20**: Specifically installs Clang 20 and sets up symlinks for `clang`, `clang++`, and `clangd`[.github/Dockerfile.base 53-64](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/.github/Dockerfile.base#L53-L64)

## Development Workflow

### Building the Project

Once the environment is activated, the standard workflow involves CMake and Ninja. Users can enable specific features like the runtime or the OpModel analysis system.

| Feature / Build Flag | Condition | Description |
| --- | --- | --- |
| `TTMLIR_ENABLE_RUNTIME` | Optional | Enables TTNN/Metal runtime execution [CMakeLists.txt 33](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/CMakeLists.txt#L33-L33) |
| `TTMLIR_ENABLE_OPMODEL` | Optional | Enables optimizer passes using OpModel [CMakeLists.txt 40](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/CMakeLists.txt#L40-L40) |
| `CMAKE_BUILD_TYPE` | Variable | Supports "Debug", "Release", "Asan" [CMakeLists.txt 64-67](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/CMakeLists.txt#L64-L67) |

### ttnn-standalone

The `ttnn-standalone` tool is used to compile and run TTNN operations independently. It links against `tt_metal`, `tt_stl`, and `_ttnncpp.so`[tools/ttnn-standalone/CMakeLists.txt 120-125](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttnn-standalone/CMakeLists.txt#L120-L125) It searches for headers in the `tt-metal` source tree and the CPM source cache [tools/ttnn-standalone/CMakeLists.txt 80-110](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttnn-standalone/CMakeLists.txt#L80-L110)
