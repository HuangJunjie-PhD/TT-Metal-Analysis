The `tt-inference-server` provides a robust benchmarking pipeline designed to measure the performance of various model types (LLM, VLM, CNN, Image, Audio, etc.) on Tenstorrent hardware. The system automates environment setup, request generation, and result reporting, allowing for sweeps across input/output sequence lengths and concurrency levels.

## Benchmarking Pipeline Overview

The primary entry point for performance measurement is `run_benchmarks.py`[[benchmarking/run_benchmarks.py:1-53]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking). This script orchestrates the benchmarking process by:

1.   **Configuration Resolution**: Loading model specifications and runtime configurations [[benchmarking/run_benchmarks.py:205-207]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
2.   **Environment Management**: Utilizing `WorkflowVenvType` to ensure benchmarks run in isolated, optimized Python environments via `uv`[[benchmarking/run_benchmarks.py:51]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking), [[workflows/workflow_venvs.py:22-50]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
3.   **Task Routing**: Dispatching requests to either standard vLLM benchmarking tools or specialized media clients via the `MediaClientFactory`[[benchmarking/run_benchmarks.py:21-22]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
4.   **Warmup and Trace Capture**: Managing hardware-specific trace captures to ensure steady-state performance measurement [[benchmarking/run_benchmarks.py:215-223]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

## Benchmark Configurations (`BENCHMARK_CONFIGS`)

Benchmarks are driven by `BenchmarkConfig` objects which map specific `DeviceTypes` to a list of `BenchmarkTaskParams`[[benchmarking/benchmark_config.py:59-62]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

### Parameter Sweeps

The system supports sweeping across multiple dimensions to find the saturation point of the hardware:

*   **ISL/OSL Pairs**: Default pairs for text models range from (128, 128) up to (65536, 128) [[benchmarking/benchmark_config.py:64-74]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
*   **Multimodal Resolutions**: For VLMs, pairs include image dimensions such as 512x512 and 1024x1024 [[benchmarking/benchmark_config.py:80-85]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
*   **Concurrency Sweeps**: When `--concurrency-sweeps` is enabled, the system expands benchmarks to powers-of-two up to the model's maximum supported concurrency [[benchmarking/run_benchmarks.py:126-129]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking). The `powers_of_two_up_to` helper is used to generate these values [[benchmarking/run_benchmarks.py:32]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

### Parameter Capping

To prevent out-of-memory (OOM) errors or invalid requests, the system calculates maximum concurrency based on memory constraints. `calculate_vision_tokens` is used for VLMs to determine the token overhead of images [[benchmarking/benchmark_config.py:184-225]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking). The logic ensures that `isl + osl + vision_tokens` does not exceed the hardware's `max_context`[[benchmarking/benchmark_config.py:133-142]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

## Trace Capture and Warmup

Tenstorrent hardware often requires a "warmup" phase to capture execution traces (graphs) for optimal performance.

*   **Built-in Warmup**: If `ModelSpec.has_builtin_warmup` is True, the client-side trace capture is automatically disabled to avoid redundant operations [[benchmarking/run_benchmarks.py:215-222]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
*   **Manual Trace Capture**: For models without built-in warmup, the benchmarking script triggers a preliminary request to ensure the device is ready before timers start, unless `disable_trace_capture` is set in the `RuntimeConfig`[[benchmarking/run_benchmarks.py:211-213]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

Sources: [[benchmarking/run_benchmarks.py:211-223]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking), [[workflows/model_spec.py:11-26]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking)

## Integration with External Tools

### genai-perf

The pipeline integrates with NVIDIA's `genai-perf` tool for standardized LLM performance metrics [[benchmarking/run_benchmarks.py:35]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking). This is triggered when `runtime_config.tools` specifies `genai-perf` and utilizes the `BENCHMARKS_GENAI_PERF` venv [[benchmarking/run_benchmarks.py:227]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking), [[workflows/workflow_types.py:44]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

### C++ Server Benchmarks

The C++ backend includes a dedicated shell script `run_benchmarks.sh` for high-performance LLM benchmarking [[tt-media-server/cpp_server/benchmarks/run_benchmarks.sh:1-15]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking). It performs:

1.   **Phase 1**: Increasing ISL (128 to 8192) with fixed OSL [[tt-media-server/cpp_server/benchmarks/run_benchmarks.sh:43-51]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
2.   **Phase 2**: Increasing OSL (128 to 8192) with fixed ISL [[tt-media-server/cpp_server/benchmarks/run_benchmarks.sh:53-61]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
3.   **Phase 3**: Concurrency sweeps (1 to 64) [[tt-media-server/cpp_server/benchmarks/run_benchmarks.sh:63-72]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

Sources: [[benchmarking/run_benchmarks.py:35]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking), [[tt-media-server/cpp_server/benchmarks/run_benchmarks.sh:19-72]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking)

## Performance Reference and Acceptance Criteria

Performance results are compared against a reference file: `model_performance_reference.json`[[workflows/model_spec.py:61-72]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).

| Metric | Description | Source |
| --- | --- | --- |
| `ttft_ms` | Time to First Token in milliseconds. | [[workflows/model_spec.py:99-105]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking) |
| `tput_user` | Tokens per second per user. | [[workflows/model_spec.py:106-108]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking) |
| `tput` | Total system throughput (tokens/sec). | [[workflows/model_spec.py:109-111]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking) |

The `run_reports.py` script performs an `acceptance_criteria_check` to determine if the current run meets the defined "theoretical" targets for the specific hardware/model combination [[workflows/run_reports.py:33-36]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking). This includes scaling targets for data parallel (DP) configurations [[workflows/model_spec.py:132-160]](https://deepwiki.com/tenstorrent/tt-inference-server/6.1-performance-benchmarking).
