## Overview

Model Testing validates end-to-end compilation and execution of complete neural network models from various frameworks (PyTorch, ONNX, TensorFlow, JAX, PaddlePaddle). This system tests 100+ models across vision, natural language processing (NLP), and multimodal domains to ensure the Forge compilation pipeline produces correct results on Tenstorrent hardware.

For information about individual operator testing, see [Operator Testing](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.2-operator-testing). For test planning and configuration generation, see [Test Planning Framework](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.1-test-planning-framework). For CI/CD automation, see [CI/CD Pipeline](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.4-cicd-pipeline).

## Test Structure and Decorators

Model tests use pytest decorators to control execution and parameterization:

### Common Pytest Markers

| Marker | Purpose | Usage |
| --- | --- | --- |
| `@pytest.mark.nightly` | Runs in nightly regression suite | All model tests |

## Verification System

The `verify()` function compares framework model outputs against compiled model outputs using configurable value checkers:

**Verification Process**: Executes both framework and compiled models on the same inputs, then compares outputs using PCC (Pearson Correlation Coefficient) and absolute/relative tolerance checks.

### Verification Configuration

From [forge/test/models/pytorch/vision/resnet/test_resnet.py 66-71](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/vision/resnet/test_resnet.py#L66-L71):

`verify(    input_sample,    framework_model,    compiled_model,    VerifyConfig(value_checker=AutomaticValueChecker(pcc=0.95)),)`
### PCC Thresholds by Model

Different models require different PCC thresholds based on precision sensitivity:

| Model | PCC Threshold | Reason |
| --- | --- | --- |
| ResNet-34 | 0.98 | Standard vision model |
| ResNet-50, ResNet-152 | 0.95 | Larger models with more accumulation |
| YOLO-v5L | 0.95 | Complex detection architecture |
| Default | 0.99 | High precision requirement |

Sources: [forge/test/models/pytorch/vision/resnet/test_resnet.py 158-167](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/vision/resnet/test_resnet.py#L158-L167)[forge/test/models/pytorch/vision/yolo/test_yolo_v5.py 35-79](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/pytorch/vision/yolo/test_yolo_v5.py#L35-L79)

### PyTorch Model Testing

### ONNX Model Testing

ONNX tests require model export and wrapper creation:

**ONNX Testing**: Wraps PyTorch models for named inputs, exports to ONNX format, validates with ONNX checker, then wraps in `forge.OnnxModule` for compilation.

#### ONNX Wrapper Pattern

From [forge/test/models/onnx/text/t5/test_t5_onnx.py 20-28](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/onnx/text/t5/test_t5_onnx.py#L20-L28):

`class Wrapper(torch.nn.Module):    def __init__(self, model):        super().__init__()        self.model = model     def forward(self, input_ids, decoder_input_ids):        inputs = {"input_ids": input_ids, "decoder_input_ids": decoder_input_ids}        output = self.model(**inputs)        return output`
#### ONNX Export and Loading

From [forge/test/models/onnx/text/t5/test_t5_onnx.py 57-64](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/test/models/onnx/text/t5/test_t5_onnx.py#L57-L64):

## Test Organization by Domain

Model tests are organized by domain and framework:

**Test Directory Structure**: Tests are organized first by framework (pytorch, onnx, tensorflow, paddlepaddle, jax), then by domain (vision, text, multimodal, timeseries).

### Vision Models

*   **ResNet**: Image classification, variants 18/34/50/101/152
*   **YOLO**: Object detection, variants N/S/M/L/X, multiple input sizes
*   **UNet**: Image segmentation, variants from different sources
*   **MobileNet**: Efficient image classification
*   **ViT**: Vision transformers

### Text/LLM Models

*   **BERT**: Masked language modeling
*   **GPT/NanoGPT**: Causal language modeling
*   **Llama**: Large language models
*   **T5**: Sequence-to-sequence generation
*   **Falcon**: Large language models

### Multimodal Models

*   **CLIP**: Vision-language contrastive learning
*   **LLaVA**: Visual question answering
*   **Phi3.5 Vision**: Multimodal reasoning
*   **PaddleOCR**: Text recognition from images

### Timeseries Models

*   **N-BEATS**: Time series forecasting with different basis functions
