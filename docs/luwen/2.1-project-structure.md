## Workspace Organization

Luwen is organized as a Rust workspace consisting of multiple crates, each with specific responsibilities. The workspace is structured to separate interface definitions from implementations, enabling flexibility and modularity.

Sources: [Cargo.toml 28-32](https://github.com/tenstorrent/luwen/blob/55e35616/Cargo.toml#L28-L32)

## Component Roles

Each crate in the Luwen workspace has a specific role in the overall system architecture:

| Crate | Version | Description | Primary Responsibility |
| --- | --- | --- | --- |
| luwen | 0.6.4 | Main package | High-level interface to Tenstorrent AI accelerators |
| luwen-core | 0.2.0 | Core types | Provides common types and utilities used across the codebase |
| luwen-if | 0.6.0 | Interface definitions | Defines abstract interfaces for chip communication |
| luwen-ref | 0.5.1 | Reference implementation | Implements the interfaces defined in luwen-if |
| ttkmd-if | 0.2.1 | Kernel driver interface | Interfaces with the Tenstorrent kernel-mode driver |
| pyluwen | 0.7.2 | Python bindings | Exposes Luwen functionality to Python |

## Layered Architecture

Luwen implements a layered architecture that provides increasing levels of abstraction as you move up the stack. This design enables separation of concerns and allows different implementations at various layers.

Sources: [Cargo.toml 11-15](https://github.com/tenstorrent/luwen/blob/55e35616/Cargo.toml#L11-L15)[crates/luwen-if/Cargo.toml 8-9](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/Cargo.toml#L8-L9)[crates/luwen-ref/Cargo.toml 8-11](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-ref/Cargo.toml#L8-L11)

## External Dependencies

Luwen relies on several external dependencies for functionality:

### Core Dependencies

*   **serde**&**serde_json** - For serialization and deserialization
*   **bincode** - For binary serialization
*   **thiserror** - For error handling
*   **prost**&**prost-build** - For Protocol Buffers support

### Language Bindings

*   **pyo3** - For Python bindings

### Development Tools

*   **clap** - For command-line parsing in utilities
*   **tracing** - For logging and diagnostics
*   **indicatif** - For progress indicators

## Build System

The Luwen project uses Cargo, Rust's package manager and build system. The root workspace includes all crates in the `crates/` and `app/` directories. The project is configured to allow independent versioning of each crate.

Sources: [Cargo.toml 25-32](https://github.com/tenstorrent/luwen/blob/55e35616/Cargo.toml#L25-L32)
