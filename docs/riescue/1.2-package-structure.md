## Overview

The RiESCUE repository is organized as a single Python package named `riescue` with three primary subpackages that implement the core testing framework layers. The package structure is defined in [pyproject.toml 31-35](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L31-L35) using setuptools, with the package root at `riescue/`:

| Subpackage | Primary Entry Point | Purpose |
| --- | --- | --- |
| `riescue.dtest_framework` | `RiescueD` class | Directed test framework with assembly generation, memory management, and OS simulation |
| `riescue.compliance` | `RiescueC` class | Compliance test generator with bringup and test plan modes |
| `riescue.lib` | Various utilities | Shared toolchain abstractions, CSR management, logging, and helper functions |

Three command-line entry points are registered in [pyproject.toml 46-49](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L46-L49) that map to main functions in the top-level `riescue` package:

*   `riescued` → `riescue.riescued:main`
*   `riescuec` → `riescue.riescuec:main`
*   `ctk` → `riescue.ctk:main`

Sources: [pyproject.toml 11-49](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L11-L49)[README.md 14-73](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L14-L73)

## Package Data and Test Assets

The package includes non-Python data files that are installed alongside the Python modules, defined in [pyproject.toml 37-42](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L37-L42):

| Package Path | File Pattern | Purpose |
| --- | --- | --- |
| `riescue.dtest_framework.tests` | `*.s` | Assembly test files used in unit tests and examples |
| `riescue.dtest_framework.lib` | `*.json` | Default CPU configuration and memory maps |
| `riescue.compliance.tests` | `*.json` | Compliance test definitions |
| `riescue.compliance.tests.configs` | `*.json` | Whisper and simulator configurations |
| `riescue.lib.csr_manager` | `*.json` | CSR (Control and Status Register) definitions |

These files are accessible at runtime through standard Python package resource mechanisms and are referenced in the code using path resolution relative to the package installation directory (see [riescued.py 413-432](https://github.com/tenstorrent/riescue/blob/07cefde3/riescued.py#L413-L432)).

Sources: [pyproject.toml 37-42](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L37-L42)

## Dependencies and External Packages

The package defines Python dependencies in [pyproject.toml 17-29](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L17-L29) which can be categorized by function:

**Special Dependency:** The `coretp` package at [pyproject.toml 28](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L28-L28) is installed directly from a GitHub repository at a pinned commit (`5da80d20`). This provides test plan definitions used by `RiescueC` in TP mode. See [External Libraries](https://deepwiki.com/tenstorrent/riescue/4.3-external-libraries) for more details.

**Non-Python Dependencies:** The package also requires external tools that must be installed separately:

*   RISC-V GNU Toolchain (compiler and binutils)
*   Whisper ISS (instruction set simulator)
*   Spike ISS (reference simulator)

These are documented in [README.md 88-92](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L88-L92) and are typically installed via the container system (see [Container System](https://deepwiki.com/tenstorrent/riescue/3.1-container-system)).

Sources: [pyproject.toml 17-29](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L17-L29)[README.md 75-92](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L75-L92)

## Type Checking Configuration

The package uses Pyright for static type checking, with configuration in [pyproject.toml 51-79](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L51-L79) The type-checked modules are explicitly listed:

**Type-Checked Modules:**

*   `riescue/lib` - All shared library code
*   `riescue/*.py` - Top-level CLI modules
*   `riescue/dtest_framework` - RiescueD framework
*   `tests/dtest_framework` - RiescueD tests
*   `riescue/compliance/*.py` - RiescueC top-level modules
*   `riescue/compliance/config` - Configuration modules
*   `riescue/compliance/src` - Generation source
*   `riescue/compliance/test_plan` - Test plan handlers

**Excluded Modules:**

*   `tests/compliance` - Compliance tests (excluded at [pyproject.toml 75](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L75-L75))
*   `riescue/dtest_framework/generator/generator.py` - Temporarily excluded pending refactoring (see comment at [pyproject.toml 76](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L76-L76))

The type checking mode is set to `"standard"` with a note that `"strict"` mode may be added incrementally in the future (see [pyproject.toml 67-70](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L67-L70)).

Sources: [pyproject.toml 51-79](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L51-L79)

## Build System and Metadata

The package uses modern Python packaging standards:

**Build Backend:** Setuptools with PEP 517 support ([pyproject.toml 7-9](https://github.com/tenstorrent/riescue/blob/07cefde3/pyproject.toml#L7-L9))

**Project Metadata:**

*   Package name: `riescue`
*   Version: `1.2.0`
*   Description: "RISC-V Directed Test Framework and Compliance Suite"
*   Minimum Python version: 3.9

**Installation Methods:**

1.   **From source:**`pip install -e .` (editable install for development)
2.   **From GitHub:**`pip install git+https://github.com/tenstorrent/riescue.git` (as documented in [README.md 82-83](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L82-L83))
3.   **Within container:** Package is pre-installed in the Singularity container (see [Container System](https://deepwiki.com/tenstorrent/riescue/3.1-container-system))

The package follows Python packaging best practices with declared dependencies, console script entry points, and proper package data inclusion.
