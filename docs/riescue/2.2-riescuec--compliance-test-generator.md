## Purpose and Scope

RiescueC is a compliance test generation system built on top of the RiescueD framework. This page provides an overview of RiescueC's architecture, its two operating modes (bringup and TP), and how it leverages RiescueD to generate RISC-V compliance tests. For detailed information about the underlying directed test framework, see [RiescueD: Directed Test Framework](https://deepwiki.com/tenstorrent/riescue/2.1-riescued:-directed-test-framework). For high-level ISA-based test generation, see [CTK: Compliance Test Kit](https://deepwiki.com/tenstorrent/riescue/2.3-ctk:-compliance-test-kit).

## Overview

RiescueC (`riescuec` command-line tool) is a specialized test generator for RISC-V architectural compliance testing. It automates the generation of directed tests that verify processor implementations against the RISC-V specification. Unlike RiescueD, which requires manual assembly test authoring, RiescueC automatically generates test sequences based on ISA extensions and test plans.

**Key capabilities:**

*   Automatic test generation for RISC-V extensions (I, M, A, F, D, C, V, B, and many more)
*   Self-checking test generation with automated pass/fail determination
*   Two distinct generation modes: datapath-focused (bringup) and privilege-scenario-focused (TP)
*   Support for RVA23U64 mandatory extensions
*   Integration with external test plan library (riscv-coretp)

## Two Operating Modes

RiescueC provides two distinct test generation modes, each targeting different aspects of RISC-V compliance verification.

### Mode Selection

| Mode | CLI Invocation | Primary Focus | Test Source |
| --- | --- | --- | --- |
| **Bringup** | `riescuec --mode bringup` | Datapath instruction compliance | ISA extension specifications |
| **TP (Test Plan)** | `riescuec --mode tp` | Privileged architecture scenarios | riscv-coretp test plans |

Sources: [README.md 40-64](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L40-L64)

### Bringup Mode

Bringup mode generates tests for instruction datapath validation. It focuses on:

*   Instruction encoding/decoding correctness
*   Basic functional behavior of instructions
*   Edge case handling (boundary values, special operands)
*   Extension-specific functionality (vector operations, floating-point, etc.)

For detailed information, see [Bringup Mode](https://deepwiki.com/tenstorrent/riescue/2.2.1-bringup-mode).

### Test Plan (TP) Mode

TP mode generates tests from structured test scenarios defined in the riscv-coretp library. It focuses on:

*   Privilege mode transitions and permissions
*   Virtual memory and paging configurations
*   Exception and interrupt handling
*   CSR access patterns
*   Hypervisor extension scenarios

## Command-Line Interface

RiescueC extends RiescueD's CLI with mode-specific arguments. The general invocation pattern is:

### Common Arguments

Arguments inherited from RiescueD:

*   `--run_dir`: Output directory for generated files
*   `--seed`: Random seed for test generation
*   `--run_iss`: Execute test on ISS after generation
*   `--priv`: Privilege mode (machine, super, user)
*   `--paging`: Paging mode (sv39, sv48, sv57, disabled)
*   `--test_env`: Test environment (bare, virtualized)

### Mode-Specific Arguments

**Bringup Mode:**

*   `--isa`: ISA string specifying extensions to test

**TP Mode:**

*   `--test_plan`: Name of test plan from riscv-coretp library
*   `--isa`: ISA string for feature configuration

## Supported Extensions

RiescueC aims to support all mandatory extensions in RVA23U64. The following table summarizes current support:

### Standard Extensions

| Extension | Description | Bringup Mode | TP Mode |
| --- | --- | --- | --- |
| RV64I | 64-bit base integer ISA | ✓ | ✓ |
| M | Integer multiplication/division | ✓ | ✓ |
| A | Atomic instructions | ✓ | ✓ |
| F | Single-precision floating-point | ✓ | ✓ |
| D | Double-precision floating-point | ✓ | ✓ |
| C | Compressed instructions | ✓ | ✓ |
| V | Vector extension | ✓ | ✓ |

### Bit Manipulation Extensions

| Extension | Description | Bringup Mode | TP Mode |
| --- | --- | --- | --- |
| B | Bit manipulation | ✓ | - |
| Zba | Address generation | ✓ | - |
| Zbb | Basic bit manipulation | ✓ | - |
| Zbc | Carryless multiplication | ✓ | - |
| Zbs | Single-bit instructions | ✓ | - |

### Privileged Extensions

| Extension | Description | Bringup Mode | TP Mode |
| --- | --- | --- | --- |
| Zicsr | CSR instructions | ✓ | ✓ |
| Zifencei | Instruction-fetch fence | - | ✓ |
| Zicond | Conditional operations | - | ✓ |
| Zimop | May-be-operations | - | ✓ |
| Zcmop | Compressed may-be-operations | - | ✓ |

### Supervisor Extensions

| Extension | Description | Bringup Mode | TP Mode |
| --- | --- | --- | --- |
| Svbare | Bare mode translation | ✓ | ✓ |
| Sv39 | 39-bit virtual memory | - | ✓ |
| Sv48 | 48-bit virtual memory | - | ✓ |
| Sv57 | 57-bit virtual memory | - | ✓ |
| Svadu | Hardware A/D updates | - | ✓ |
| Svinval | Fine-grained TLB invalidation | - | ✓ |

### Hypervisor Extensions

## Environment Solving in TP Mode

TP mode uses a constraint solving mechanism to find valid test environments that satisfy both test plan requirements and user-specified constraints.

### Predicate System

The `get_predicates()` method in `TpMode` creates filter functions that eliminate incompatible environments:

1.   **Privilege Check**: Ensures test runs in the specified privilege mode (M/S/U)
2.   **Paging Check**: Filters by paging mode (sv39/sv48/sv57/disabled)
3.   **Virtualization Check**: Matches virtualized vs. bare metal requirements
4.   **Delegation Check**: Ensures exception delegation matches configuration

If no valid environment exists after applying predicates, test generation fails with an error.

Sources: [riescue/compliance/tp.py 139-182](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/compliance/tp.py#L139-L182)
