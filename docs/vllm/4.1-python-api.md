## Purpose and Scope

The Python API provides a high-level synchronous interface for offline inference with vLLM through the `LLM` class. This API is designed for batch processing scenarios where you need to generate text, run chat completions, or compute embeddings from prompts without the overhead of an HTTP server.

For online serving with HTTP endpoints, see [OpenAI-Compatible Server](https://deepwiki.com/tenstorrent/vllm/4.2-openai-compatible-server). For asynchronous offline inference, the `LLM` class can be used with async methods. For low-level protocol objects and request/response types, see [Protocol and Data Types](https://deepwiki.com/tenstorrent/vllm/4.3-protocol-and-data-types).

This page covers:

*   Initializing the `LLM` class with configuration options
*   Text generation with `generate()`
*   Chat completions with `chat()`
*   Embeddings and pooling with `encode()`, `embed()`, `score()`, and `classify()`
*   Multimodal input handling
*   Sampling and pooling parameters

## LLM Class Architecture

The `LLM` class serves as the main entry point for Python-based inference. It encapsulates the engine, tokenizer, and processor components and provides high-level methods for various inference tasks.

**Key Components:**

*   **LLM**: Primary user-facing class that manages the engine lifecycle and provides convenience methods
*   **EngineArgs**: Configuration dataclass for specifying model, hardware, and execution parameters
*   **V1LLMEngine**: Backend engine that handles request scheduling, KV cache, and model execution
*   **Processor**: Validates and preprocesses inputs (tokenization, multimodal processing)
*   **Tokenizer**: Handles text tokenization using HuggingFace or Mistral tokenizers
*   **IOProcessor**: Processes multimodal inputs (images, audio, video)

## Initialization and Configuration

### Basic Initialization

The `LLM` class is initialized with a model identifier and configuration parameters:

`from vllm import LLM # Basic initializationllm = LLM(    model="meta-llama/Llama-3.2-1B-Instruct",    tensor_parallel_size=1,    dtype="auto")`
### Configuration Flow

### Key Initialization Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `model` | str | Required | HuggingFace model name or local path |
| `tokenizer` | str | None | Override tokenizer (uses model by default) |
| `tokenizer_mode` | TokenizerMode | "auto" | "auto", "slow", or "mistral" |
| `skip_tokenizer_init` | bool | False | Skip tokenizer initialization |
| `trust_remote_code` | bool | False | Allow remote code execution |
| `tensor_parallel_size` | int | 1 | Number of GPUs for tensor parallelism |
| `dtype` | ModelDType | "auto" | Data type ("float32", "float16", "bfloat16", "auto") |
| `quantization` | str | None | Quantization method ("awq", "gptq", "fp8", etc.) |
| `gpu_memory_utilization` | float | 0.9 | Fraction of GPU memory for KV cache |
| `enforce_eager` | bool | False | Disable CUDA graphs |
| `seed` | int | None | Random seed for sampling |

## Text Generation

### generate() Method

The `generate()` method is the primary interface for text generation tasks:

`outputs = llm.generate(    prompts=["Hello, my name is"],    sampling_params=SamplingParams(        temperature=0.8,        top_p=0.95,        max_tokens=100    ))`
### Input Formats

The `generate()` method accepts flexible input types:

| Input Type | Description | Example |
| --- | --- | --- |
| `str` | Single text prompt | `"What is AI?"` |
| `list[str]` | Batch of text prompts | `["Question 1", "Question 2"]` |
| `TextPrompt` | Prompt with explicit token IDs | `TextPrompt(prompt="Hello", prompt_token_ids=[...])` |
| `TokensPrompt` | Direct token ID input | `TokensPrompt(prompt_token_ids=[1, 2, 3])` |
| Multimodal | Text with images/audio/video | See Multimodal section |

### Output Format

The `generate()` method returns `list[RequestOutput]`:

`class RequestOutput:    request_id: str    prompt: str    prompt_token_ids: list[int]    prompt_logprobs: Optional[PromptLogprobs]    outputs: list[CompletionOutput]    finished: bool    metrics: Optional[RequestMetrics] class CompletionOutput:    index: int    text: str    token_ids: list[int]    cumulative_logprob: Optional[float]    logprobs: Optional[SampleLogprobs]    finish_reason: Optional[str]    stop_reason: Union[int, str, None]`
**Example:**

`outputs = llm.generate(["What is 1+1?"])for output in outputs:    prompt = output.prompt    generated_text = output.outputs[0].text    print(f"Prompt: {prompt}")    print(f"Generated: {generated_text}")`

## Embedding and Pooling Methods

### encode() Method

For embedding models, `encode()` extracts hidden states:

`llm = LLM(model="BAAI/bge-base-en-v1.5", task="embed") outputs = llm.encode(["Hello world", "Goodbye world"]) for output in outputs:    embedding = output.outputs.data  # torch.Tensor    print(f"Embedding shape: {embedding.shape}")`
### embed() Method

The `embed()` method is an alias for `encode()` that specifically targets embedding tasks:

`embeddings = llm.embed(    prompts=["Query text"],    pooling_params=PoolingParams(normalize=True))`
### score() Method

For cross-encoder scoring between text pairs:

`llm = LLM(model="BAAI/bge-reranker-base", task="score") text_1 = ["What is the capital of France?"]text_2 = ["Paris is the capital of France."] outputs = llm.score(text_1, text_2) for output in outputs:    score = output.outputs.data  # Similarity score    print(f"Score: {score}")`
### classify() Method

For sequence classification tasks:

`llm = LLM(model="bert-base-uncased", task="classify") outputs = llm.classify(["This is a positive review"]) for output in outputs:    logits = output.outputs.data    predicted_class = logits.argmax(dim=-1)`
### Task-Specific Output Types

| Method | Output Type | Data Format |
| --- | --- | --- |
| `encode()` | `EmbeddingRequestOutput` | `torch.Tensor` (embeddings) |
| `embed()` | `EmbeddingRequestOutput` | `torch.Tensor` (embeddings) |
| `score()` | `ScoringRequestOutput` | `torch.Tensor` (scores) |
| `classify()` | `ClassificationRequestOutput` | `torch.Tensor` (logits) |

## Sampling Parameters

### SamplingParams Configuration

The `SamplingParams` class controls text generation behavior:

`from vllm import SamplingParams params = SamplingParams(    # Temperature controls randomness    temperature=0.8,        # Top-p (nucleus) sampling    top_p=0.95,        # Top-k sampling    top_k=50,        # Min-p sampling    min_p=0.05,        # Maximum tokens to generate    max_tokens=100,        # Stop sequences    stop=["</s>", "\n\n"],    stop_token_ids=[2],        # Logprobs    logprobs=5,    prompt_logprobs=5,        # Repetition penalties    presence_penalty=0.1,    frequency_penalty=0.1,    repetition_penalty=1.1,        # Random seed for reproducibility    seed=42,        # Number of parallel outputs    n=1,        # Ignore EOS token    ignore_eos=False,        # Output control    skip_special_tokens=True,    spaces_between_special_tokens=True,        # Truncation    truncate_prompt_tokens=None)`
### Sampling Algorithm Selection

### Key Sampling Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `temperature` | float | 1.0 | Sampling temperature (0 = greedy, higher = more random) |
| `top_p` | float | 1.0 | Nucleus sampling threshold |
| `top_k` | int | 0 | Top-k sampling (0 = disabled) |
| `min_p` | float | 0.0 | Minimum probability threshold |
| `max_tokens` | int | 16 | Maximum tokens to generate |
| `stop` | str | list[str] | None | Stop generation at these strings |
| `presence_penalty` | float | 0.0 | Penalize tokens based on presence |
| `frequency_penalty` | float | 0.0 | Penalize tokens based on frequency |
| `repetition_penalty` | float | 1.0 | Repetition penalty (> 1 = penalize) |
| `seed` | int | None | Random seed for reproducibility |
| `n` | int | 1 | Number of parallel sequences |
| `logprobs` | int | None | Number of top logprobs to return |

Sources: [vllm/sampling_params.py 85-285](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/sampling_params.py#L85-L285)

## Pooling Parameters

### PoolingParams Configuration

For embedding and classification tasks, use `PoolingParams`:

`from vllm import PoolingParams params = PoolingParams(    # Pooling type    pooling_type="mean",  # "mean", "cls", "last", etc.        # Normalization    normalize=True,        # Softmax for classification    softmax=False,        # Additional data to return    additional_data={})`
### Pooling Types

| Pooling Type | Description | Use Case |
| --- | --- | --- |
| `"mean"` | Average of all token embeddings | General embeddings |
| `"cls"` | Use [CLS] token embedding | BERT-style models |
| `"last"` | Use last token embedding | Decoder-only models |
| `"max"` | Max pooling over tokens | Some classification tasks |
| `"sum"` | Sum of token embeddings | Rare, specific tasks |

### Example Usage

`# For embeddings with normalizationllm = LLM(model="BAAI/bge-base-en-v1.5") outputs = llm.encode(    ["Query text"],    pooling_params=PoolingParams(        pooling_type="mean",        normalize=True    )) # For classification without normalizationllm = LLM(model="bert-base-uncased", task="classify") outputs = llm.classify(    ["Text to classify"],    pooling_params=PoolingParams(        pooling_type="cls",        normalize=False,        softmax=True    ))`

## Advanced Features

### LoRA Adapters

The `LLM` class supports LoRA (Low-Rank Adaptation) for efficient fine-tuning:

`from vllm.lora.request import LoRARequest llm = LLM(    model="meta-llama/Llama-3.2-1B",    enable_lora=True,    max_loras=2,    max_lora_rank=16) lora_request = LoRARequest(    lora_name="my_adapter",    lora_int_id=1,    lora_local_path="/path/to/adapter") outputs = llm.generate(    "Prompt text",    sampling_params=params,    lora_request=lora_request)`
### Logits Processors

Custom logits processors can be passed during initialization:

`from vllm.v1.sample.logits_processor import LogitsProcessor class CustomProcessor(LogitsProcessor):    def __call__(self, logits, tokens):        # Custom logic        return logits llm = LLM(    model="meta-llama/Llama-3.2-1B",    logits_processors=[CustomProcessor])`
### Request Metrics

Enable detailed metrics collection:

`outputs = llm.generate(prompts=["Test"]) for output in outputs:    if output.metrics:        print(f"Time in queue: {output.metrics.time_in_queue}")        print(f"Time to first token: {output.metrics.first_token_time}")        print(f"Model forward time: {output.metrics.model_forward_time}")`
