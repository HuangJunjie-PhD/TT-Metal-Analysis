## Purpose and Scope

The RISC-V Base Backend provides the foundational infrastructure for RISC-V code generation in this GCC fork. It implements the core builtin function management system, register constraint definitions, target-specific feature flags, and standard RISC-V instruction patterns. This backend serves as the base layer upon which the Tenstorrent SFPU extensions are built.

For information about the Tenstorrent-specific SFPU extensions (Wormhole and Blackhole architectures), see [Tenstorrent SFPU Extensions](https://deepwiki.com/tenstorrent/sfpi-gcc/2.2-tenstorrent-sfpu-extensions). For the broader compilation pass infrastructure, see [Pass Management and Optimization Pipeline](https://deepwiki.com/tenstorrent/sfpi-gcc/3.3-pass-management-and-optimization-pipeline).

## Builtin Function Management System

The builtin function system provides a structured approach to registering, validating, and expanding RISC-V-specific builtin functions.

### Builtin Description Structure

Each builtin is defined by a `riscv_builtin_description` structure that encapsulates all necessary metadata:

**Sources:**[gcc/config/riscv/riscv-builtins.cc 88-105](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L88-L105)

### Builtin Types

The backend supports two primary builtin expansion strategies:

| Builtin Type | Description | Has Return Value | Example |
| --- | --- | --- | --- |
| `RISCV_BUILTIN_DIRECT` | Maps directly to an .md pattern with a return value | Yes | `__builtin_riscv_frflags()` |
| `RISCV_BUILTIN_DIRECT_NO_TARGET` | Maps directly to an .md pattern without a return value | No | `__builtin_riscv_fsflags()` |

**Sources:**[gcc/config/riscv/riscv-builtins.cc 72-78](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L72-L78)

### Availability Predicate System

Builtins are conditionally available based on target features. The `AVAIL` macro generates predicate functions:

**Sources:**[gcc/config/riscv/riscv-builtins.cc 81-152](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L81-L152)

Key availability predicates defined:

*   **Floating-Point:**`hard_float` - Available when `TARGET_HARD_FLOAT || TARGET_ZFINX`
*   **Cache Management:**`clean32/64`, `flush32/64`, `inval32/64`, `zero32/64` - Require `TARGET_ZICBOM` or `TARGET_ZICBOZ`
*   **Bit Manipulation:**`zbb`, `zbkb`, `crypto_zbkb` - Various bit manipulation extensions
*   **Cryptography:**`crypto_zknd`, `crypto_zkne`, `crypto_zknh`, `crypto_zksh`, `crypto_zksed`
*   **Vector:** Constraints in [gcc/config/riscv/constraints.md 174-181](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/constraints.md?plain=1#L174-L181)
*   **SFPU:**`wormhole`, `blackhole`, `sfpu` - Tenstorrent extensions

## Function Type System

The function type system defines the signatures of all builtin functions through a combination of enumeration and macro expansion.

### Type Code Enumeration

Function types are enumerated using macros from `riscv-ftypes.def`:

**Sources:**[gcc/config/riscv/riscv-ftypes.def 1-158](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-ftypes.def#L1-L158)[gcc/config/riscv/riscv-builtins.cc 48-69](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L48-L69)

### Argument Type Mapping

The `RISCV_ATYPE_*` macros map abstract type codes to concrete GCC tree types:

| Type Code | GCC Type | Description |
| --- | --- | --- |
| `RISCV_ATYPE_VOID` | `void_type_node` | Void type |
| `RISCV_ATYPE_UQI` | `unsigned_intQI_type_node` | Unsigned 8-bit integer |
| `RISCV_ATYPE_UHI` | `unsigned_intHI_type_node` | Unsigned 16-bit integer |
| `RISCV_ATYPE_USI` | `unsigned_intSI_type_node` | Unsigned 32-bit integer |
| `RISCV_ATYPE_UDI` | `unsigned_intDI_type_node` | Unsigned 64-bit integer |
| `RISCV_ATYPE_SI` | `intSI_type_node` | Signed 32-bit integer |
| `RISCV_ATYPE_V64SF` | `v64SF_type_node` | 64-element float vector |
| `RISCV_ATYPE_IPTR` | `instrn_ptr_type_node` | Volatile pointer for instructions |

**Sources:**[gcc/config/riscv/riscv-builtins.cc 212-223](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L212-L223)

### Function Type Construction

The `riscv_build_function_type()` function lazily constructs function types:

**Sources:**[gcc/config/riscv/riscv-builtins.cc 290-311](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L290-L311)

### Special Type Registration

The backend registers special floating-point types if not already provided by the language frontend:

*   **Float16 (`_Float16`)**: 16-bit IEEE 754 floating-point type in `HFmode`
*   **BFloat16 (`__bf16`)**: 16-bit brain floating-point type in `BFmode`
*   **V64SF**: 64-element single-precision vector type for SFPU operations

**Sources:**[gcc/config/riscv/riscv-builtins.cc 314-352](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L314-L352)

## Vector Extension Support

The base RISC-V backend includes preliminary support for the RISC-V Vector Extension (RVV), which is extended and specialized by the SFPU system.

### Vector Register Classes

**Sources:**[gcc/config/riscv/constraints.md 174-181](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/constraints.md?plain=1#L174-L181)

### Vector Immediate Constraints

Vector operations support several immediate value types:

*   **`vi`**: 5-bit signed immediate in range [-16, 15]
*   **`vj`**: Negated 5-bit signed immediate in range [-15, 16]
*   **`vk`**: 5-bit unsigned immediate in range [0, 31]
*   **`vl`**: CSR-compatible immediate (0 for XTheadVector, uimm5 for standard)

**Sources:**[gcc/config/riscv/constraints.md 197-216](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/constraints.md?plain=1#L197-L216)

### Vector Constant Constraints

Special constraints for vector constant patterns:

| Constraint | Pattern | Description |
| --- | --- | --- |
| `Wc0` | All zeros | `CONST0_RTX` |
| `Wc1` | All ones | `CONSTM1_RTX` |
| `Wb1` | Scalar mask | `{...,0,...,0,1}` pattern |
| `vp` | VLENB poly | Poly int equal to `BYTES_PER_RISCV_VECTOR` |
| `vu` | Undefined | `UNSPEC_VUNDEF` |

**Sources:**[gcc/config/riscv/constraints.md 187-195](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/constraints.md?plain=1#L187-L195)[gcc/config/riscv/constraints.md 218-234](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/constraints.md?plain=1#L218-L234)

### Builtin Indexing Strategy

The backend maintains two parallel indexing schemes for efficient builtin lookup:

1.   **Sequential Index (`riscv_builtin_decls[]`)**: Array indexed by builtin declaration order
2.   **Instruction Code Index (`riscv_builtin_decl_index[]`)**: Array indexed by `enum insn_code` for reverse lookup

**Sources:**[gcc/config/riscv/riscv-builtins.cc 277-284](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L277-L284)

## Key Data Structures

### riscv_builtin_description

Encapsulates complete metadata for a single builtin function.

| Field | Type | Purpose |
| --- | --- | --- |
| `icode` | `enum insn_code` | Instruction code for .md pattern lookup |
| `name` | `const char*` | Builtin name (e.g., `"__builtin_riscv_frflags"`) |
| `builtin_type` | `enum riscv_builtin_type` | Expansion strategy (DIRECT/DIRECT_NO_TARGET) |
| `prototype` | `enum riscv_function_type` | Function signature identifier |
| `avail` | Function pointer | Availability predicate function |

**Sources:**[gcc/config/riscv/riscv-builtins.cc 88-105](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L88-L105)

### riscv_builtin_type Enumeration

| Value | Expansion Behavior |
| --- | --- |
| `RISCV_BUILTIN_DIRECT` | Pattern has return value; create output operand |
| `RISCV_BUILTIN_DIRECT_NO_TARGET` | Pattern has no return value; void function |

**Sources:**[gcc/config/riscv/riscv-builtins.cc 72-78](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L72-L78)

### riscv_function_type Enumeration

Generated from `riscv-ftypes.def` via macro expansion. Contains entries like:

*   `RISCV_USI_FTYPE` - Returns unsigned int, no arguments
*   `RISCV_VOID_FTYPE_USI` - Returns void, takes unsigned int
*   `RISCV_V64SF_FTYPE_V64SF_V64SF` - Returns V64SF, takes two V64SF arguments

**Sources:**[gcc/config/riscv/riscv-builtins.cc 64-69](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-builtins.cc#L64-L69)[gcc/config/riscv/riscv-ftypes.def 22-158](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/riscv-ftypes.def#L22-L158)
