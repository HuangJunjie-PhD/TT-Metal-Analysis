The `tt-github-actions` repository employs a comprehensive testing strategy for its data collection and processing engine. This suite ensures that CI/CD telemetry and performance benchmark data are correctly parsed, transformed into Pydantic models, and formatted for downstream consumption. The testing infrastructure relies on **DeepDiff** for golden-file comparisons and rigorous constraint validation to prevent data duplication in the final outputs.

## End-to-End Pipeline Validation

The core of the testing suite resides in `test_generate_data.py`. These tests validate the full lifecycle of data transformation from raw GitHub API responses (JSON) to the final structured outputs used by the Tenstorrent data warehouse.

### Pipeline JSON Generation

The `test_create_pipeline_json` function performs end-to-end validation of the `create_pipeline_json` entry point. It utilizes historical GitHub run data stored in `test/data/` to verify:

*   **Job and Test Counts**: Ensures the number of jobs and individual test cases extracted match expected values for specific `run_id` fixtures [.github/actions/collect_data/test/test_generate_data.py 13-23](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_generate_data.py#L13-L23)
*   **Card Type Detection**: Validates that hardware card types (e.g., N300, N150, E150) are correctly identified and assigned to jobs [.github/actions/collect_data/test/test_generate_data.py 40-44](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_generate_data.py#L40-L44)
*   **Error Reporting**: Confirms that skipped tests have appropriate error messages attached [.github/actions/collect_data/test/test_generate_data.py 53-57](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_generate_data.py#L53-L57)

### Benchmark JSONL Validation

Benchmark data processing is verified via `test_create_benchmark_json`. This test compares generated JSONL (JSON Lines) files against "golden" fixtures using the `DeepDiff` library to ensure exact matches while ignoring non-deterministic fields like `step_start_ts`[.github/actions/collect_data/test/test_generate_data.py 63-94](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_generate_data.py#L63-L94)

### Uniqueness Constraints

To maintain data integrity in the database, the suite enforces strict uniqueness constraints through the `check_constraint` helper function.

| Entity | Uniqueness Key (Constraint) |
| --- | --- |
| **Test** | `github_job_id` + `full_test_name` + `test_start_ts` |
| **Job** | `github_pipeline_id` + `name` + `job_submission_ts` + `job_start_ts` + `job_end_ts` |

## Benchmark Mapper Unit Tests

The `test_benchmark_mapper.py` file focuses on the logic within specialized mapper classes that handle disparate benchmark formats from various Tenstorrent projects.

### Shield & Vllm Mappers

Tests for `ShieldBenchmarkDataMapper` and `VllmBenchmarkDataMapper` verify the transformation of raw report dictionaries into `CompleteBenchmarkRun` objects. Key areas of focus include:

*   **Metadata Integration**: Ensuring `model_id`, `inference_engine`, and `device` information are correctly mapped from the report's metadata block [.github/actions/collect_data/test/test_benchmark_mapper.py 59-86](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_benchmark_mapper.py#L59-L86)
*   **Model Spec Data**: Validating that environment variables (e.g., `MESH_DEVICE`, `ARCH_NAME`) and device specifications are correctly injected into the `config_params` field [.github/actions/collect_data/test/test_benchmark_mapper.py 88-118](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_benchmark_mapper.py#L88-L118)
*   **Evaluation Scores**: Testing the extraction of accuracy/evaluation metrics (score, published_score, gpu_reference_score) into the measurements list [.github/actions/collect_data/test/test_benchmark_mapper.py 120-137](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_benchmark_mapper.py#L120-L137)

### Model Name Formatting

Mappers include logic to normalize model names, such as stripping repository prefixes (e.g., converting `meta-llama/Llama-3.2-1B` to `Llama-3.2-1B`). This is verified through specific unit tests [.github/actions/collect_data/test/test_benchmark_mapper.py 201-210](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/test_benchmark_mapper.py#L201-L210)

## Test Data Directory Structure

The `test/data/` directory is organized by GitHub `run_id`, providing a self-contained environment for each test case.

### Directory Layout

Each `run_id` folder typically contains:

*   `workflow.json`: Mock response for the GitHub Workflow Run API [.github/actions/collect_data/test/data/14492364249/workflow.json 1-10](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/data/14492364249/workflow.json#L1-L10)
*   `workflow_jobs.json`: Mock response for the GitHub Jobs API, containing job steps and status [.github/actions/collect_data/test/data/12083382635/workflow_jobs.json 1-20](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/data/12083382635/workflow_jobs.json#L1-L20)
*   `artifacts/`: Directory containing raw benchmark reports (JSON) or test results (XML) [.github/actions/collect_data/test/data/14492364249/artifacts/benchmark_forge-fe_e2e_mnist_linear_32_32_40651588679.json 1-10](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/data/14492364249/artifacts/benchmark_forge-fe_e2e_mnist_linear_32_32_40651588679.json#L1-L10)
*   `expected/`: Contains the "golden" `benchmark.json` or `benchmark.jsonl` files for regression testing [.github/actions/collect_data/test/data/12890516473/expected/benchmark.json 1](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/actions/collect_data/test/data/12890516473/expected/benchmark.json#L1-L1)

### Sample Data Entities

| File | Role | Key Fields |
| --- | --- | --- |
| `workflow_jobs.json` | Job Metadata | `id`, `status`, `conclusion`, `started_at`, `completed_at` |
| `benchmark_...json` | Raw Perf Data | `measurements`, `device_info`, `ml_model_name` |
| `expected/benchmark.jsonl` | Golden Output | `run_type`, `git_commit_hash`, `measurements` (with values/targets) |
