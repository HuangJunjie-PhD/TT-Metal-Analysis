## Purpose and Scope

This section documents models that fall outside standard computer vision classification and NLP tasks. These models include:

*   **Generative models**: Audio generation (MusicGen), image generation (Stable Diffusion)
*   **Object detection and tracking**: YOLOv5, MobileNet SSD, Hand Landmark
*   **Models with unique integration requirements**: MediaPipe models, external dependencies, special preprocessing
*   **Experimental deployments**: Models under active debugging or with known limitations

For standard image classification models (ResNet, ViT, EfficientNet), see [Computer Vision Models](https://deepwiki.com/tenstorrent/tt-torch/5.1-computer-vision-models). For standard NLP models (BERT, GPT variants, Mistral), see [Natural Language Processing Models](https://deepwiki.com/tenstorrent/tt-torch/5.2-natural-language-processing-models).

## Overview

The specialized and experimental models in tt-torch represent edge cases in the testing matrix, often requiring custom configurations, unique preprocessing, or special handling. These models serve as validation targets for less common operation patterns and help identify compiler limitations.

### Model Categories

| Category | Models | Key Characteristics |
| --- | --- | --- |
| **Generative Models** | MusicGen Small, Stable Diffusion | Audio/image generation, autoregressive decoding |
| **Object Detection** | YOLOv5, MobileNet SSD | Bounding box prediction, multi-scale detection |
| **Hand Tracking** | Hand Landmark (MediaPipe) | Keypoint detection, third-party integration |
| **Experimental Status** | Various | Models marked as xfail or under active debugging |

## Generative Models

### MusicGen Small

MusicGen is a controllable music generation model from Facebook/Meta that produces audio waveforms from text descriptions. The model uses an autoregressive transformer architecture.

#### Model Characteristics

*   **Task**: Text-to-music generation
*   **Source**: `facebook/musicgen-small` (Hugging Face)
*   **Architecture**: Autoregressive transformer with audio codec
*   **Loader**: `third_party.tt_forge_models.musicgen_small.pytorch.ModelLoader`

#### Test Implementation

**MusicGen Test Architecture**

#### Configuration

The test uses `ThisTester` class inheriting from `ModelTester`:

## Hand Landmark (MediaPipe)

The Hand Landmark model detects hand keypoints using Google's MediaPipe framework. This model has unique integration requirements due to its MediaPipe dependency and custom model format.

### MediaPipe Integration

**Hand Landmark MediaPipe Integration**

Sources: [tests/models/hand_landmark/test_hand_landmark.py 18-38](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/hand_landmark/test_hand_landmark.py#L18-L38)

### Dependency Management

The Hand Landmark test implements a unique workaround for OpenCV dependencies:

1.   **Problem**: MediaPipe installs `opencv-python` which requires GPU support
2.   **Solution**: Uninstall GPU version, install `opencv-python-headless` (CPU-only)
3.   **Implementation**: Direct subprocess calls to pip within the test

This approach is necessary because the standard `dependencies` variable cannot control the installation order to prevent the GPU-dependent package.

Sources: [tests/models/hand_landmark/test_hand_landmark.py 60-106](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/hand_landmark/test_hand_landmark.py#L60-L106)

### Model Asset Download

The test requires downloading the MediaPipe model asset (`.task` file) from Google's cloud storage:

Files are downloaded to the test directory using `wget` and the `get_file()` utility from `tt-forge-models`.

### Standard Test Structure

### Parameterized Test Execution

All models support multiple execution modes:

| Parameter | Options | Purpose |
| --- | --- | --- |
| `mode` | `"train"`, `"eval"` | Testing mode (most skip train) |
| `op_by_op` | `STABLEHLO`, `TORCH`, `None` | Op-by-op backend selection |
| `data_parallel_mode` | `True`, `False` | Multi-device execution |

Op-by-op modes are mutually exclusive with data parallel mode.

## Model Status and Limitations

### Current Test Status

| Model | Status | PCC | ATOL | Notes |
| --- | --- | --- | --- | --- |
| **MusicGen Small** | ✓ Passing | ✓ | ✗ | ATOL disabled |
| **YOLOv5** | ⚠ XFAIL | ✓ | ✗ | Expected fail, high tolerance |
| **MobileNet SSD** | ⚠ Issues | ✗ | ✗ | NMS bfloat16 issue, checking disabled |
| **Hand Landmark** | ⚠ XFAIL | - | - | Under debug |

### Common Limitations

1.   **Training Mode**: Most specialized models test only evaluation mode
2.   **Validation Metrics**: ATOL checking frequently disabled due to numerical precision issues
3.   **Data Parallel**: Not all models support data parallel execution, especially with op-by-op modes
4.   **External Dependencies**: MediaPipe models require special dependency management
5.   **BFloat16 Support**: Some operations (e.g., NMS) lack bfloat16 kernels
6.   **Debugging Required**: Several models marked as xfail with open issues

Sources: [tests/models/musicgen_small/test_musicgen_small.py 38-39](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/musicgen_small/test_musicgen_small.py#L38-L39)[tests/models/yolov5/test_yolov5.py 118](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/yolov5/test_yolov5.py#L118-L118)[tests/models/hand_landmark/test_hand_landmark.py 44](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/hand_landmark/test_hand_landmark.py#L44-L44)[tests/models/mobilenet_ssd/test_mobilenet_ssd.py 17-18](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/mobilenet_ssd/test_mobilenet_ssd.py#L17-L18)
