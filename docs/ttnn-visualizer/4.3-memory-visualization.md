This page documents the memory visualization components and features within the TT-NN Visualizer. Memory visualization provides graphical representations of memory allocations for tensor operations, allowing users to inspect how L1 and DRAM memory is used across different operations, identify memory fragmentation issues, analyze peak memory usage, and compare memory usage between operations.

For tensor-specific analysis information, see [Tensor Analysis](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.4-tensor-analysis). For general operation details, see [Operation Details](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.2-operation-details).

## Memory Data Model

The memory visualization subsystem relies on the `OperationDetails` model to process and prepare memory data for visualization. This model extracts and organizes memory allocation information from operation data.

Sources: [src/model/OperationDetails.ts 267-455](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/OperationDetails.ts#L267-L455)

The `memoryData` method is central to memory visualization, producing a comprehensive data object that includes:

*   Chart data for memory plots
*   Memory chunk information (address and size)
*   Fragmentation information
*   Circular buffer (CB) data
*   Buffer allocation data
*   Condensed views of memory usage

Different memory types are identified using the `BufferType` enum:

*   `L1`: On-chip memory (faster, smaller)
*   `DRAM`: Off-chip memory (slower, larger)
*   `L1_SMALL`: Special L1 memory region
*   `L1_START`: Starting marker for L1 memory

Sources: [src/model/OperationDetails.ts 25-86](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/OperationDetails.ts#L25-L86)[src/model/OperationDetails.ts 287-455](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/OperationDetails.ts#L287-L455)

## Memory Visualization Components

The memory visualization system consists of several key components that work together to provide a comprehensive view of memory usage.

### Memory Plot Renderer

The `MemoryPlotRenderer` component is the core visualization element that renders memory plots using Plotly.js. It provides a consistent interface for visualizing different types of memory allocations.

Key features:

*   Renders rectangular blocks representing memory allocations
*   Supports zooming into specific memory regions
*   Highlights selected memory addresses
*   Displays hover information with memory details
*   Supports markers to indicate special memory regions

Sources: [src/components/operation-details/MemoryPlotRenderer.tsx 13-187](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/MemoryPlotRenderer.tsx#L13-L187)[src/definitions/PlotConfigurations.ts 8-134](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/definitions/PlotConfigurations.ts#L8-L134)

### L1 Memory Visualization

The `L1Plots` component visualizes L1 memory allocations and provides features for analyzing on-chip memory usage.

Key visualizations:

*   Previous operation's L1 usage
*   Current operation's L1 usage
*   Device operation tensors
*   Circular buffers
*   L1 Small memory region

The component supports toggling between different views and zooming into specific regions of interest.

Sources: [src/components/operation-details/L1Plots.tsx 30-341](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/L1Plots.tsx#L30-L341)[src/definitions/L1MemorySize.ts](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/definitions/L1MemorySize.ts)

### DRAM Memory Visualization

The `DRAMPlots` component visualizes DRAM memory allocations, providing insight into off-chip memory usage.

Key visualizations:

*   Previous operation's DRAM usage
*   Current operation's DRAM usage
*   DRAM delta showing changes between operations

The component handles large memory ranges by splitting the visualization into manageable segments.

Sources: [src/components/operation-details/DRAMPlots.tsx 28-393](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/DRAMPlots.tsx#L28-L393)

### Memory Legend Components

The memory legend components provide detailed information about memory allocations.

The `MemoryLegendElement` displays:

*   Color-coded memory blocks matching the visualization
*   Memory address (decimal and hex)
*   Memory size
*   Tensor information (ID, shape, data type)
*   Buffer type and layout

For addresses with multiple allocations, the `MemoryLegendGroup` component groups and collapses related entries.

Sources: [src/components/operation-details/MemoryLegendElement.tsx 16-141](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/MemoryLegendElement.tsx#L16-L141)[src/components/operation-details/MemoryLegendGroup.tsx 13-76](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/MemoryLegendGroup.tsx#L13-L76)[src/scss/components/MemoryLegendElement.scss](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/scss/components/MemoryLegendElement.scss)

## Memory Analysis Features

The memory visualization system provides several analysis features to help understand memory usage patterns.

### Peak Memory Analysis

Peak memory analysis calculates and displays the maximum memory usage across operations, highlighting potential bottlenecks.

The peak memory calculation tracks memory allocations and deallocations throughout device operations and identifies the maximum total memory used at any point.

Sources: [src/functions/processMemoryAllocations.ts 18-100](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/functions/processMemoryAllocations.ts#L18-L100)[src/components/operation-details/DeviceOperationsFullRender.tsx 366-382](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/DeviceOperationsFullRender.tsx#L366-L382)

### Delta Analysis

Delta analysis compares memory usage between consecutive operations, highlighting new allocations and deallocations. This helps understand how memory usage evolves throughout the model execution.

The `getMemoryDelta` method in `OperationDetails` calculates:

*   New memory allocations in the current operation
*   Memory deallocations from the previous operation
*   Visualization-ready data for the delta

Sources: [src/model/OperationDetails.ts 457-468](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/OperationDetails.ts#L457-L468)[src/components/operation-details/DRAMPlots.tsx 52-58](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/DRAMPlots.tsx#L52-L58)[src/components/operation-details/DRAMPlots.tsx 198-247](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/DRAMPlots.tsx#L198-L247)

### Fragmentation Analysis

Fragmentation analysis identifies empty spaces between memory allocations that could indicate inefficient memory usage.

The system:

*   Detects gaps between memory allocations
*   Calculates the size of each gap
*   Identifies the largest empty space
*   Visualizes fragmentation in memory plots and legends

Sources: [src/model/OperationDetails.ts 344-380](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/OperationDetails.ts#L344-L380)

## Device Operations Memory View

The `DeviceOperationsFullRender` component provides a detailed view of memory usage by device operation, showing how memory is allocated and deallocated during execution.

Key features:

*   Hierarchical view of device operations
*   Memory allocation tracking per operation
*   Peak memory load display
*   Circular buffer visualization
*   Buffer allocation details

The device operations view shows the progression of memory usage through the execution flow, highlighting operations that contribute to peak memory usage.

Sources: [src/components/operation-details/DeviceOperationsFullRender.tsx 25-416](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/operation-details/DeviceOperationsFullRender.tsx#L25-L416)[src/scss/components/DeviceOperationFullRender.scss](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/scss/components/DeviceOperationFullRender.scss)

## Memory Visualization Configurations

The memory visualization system uses predefined configurations for consistent rendering across different memory types.

| Configuration | Height | Purpose |
| --- | --- | --- |
| L1RenderConfiguration | 110px | Main L1 memory plots |
| L1SmallRenderConfiguration | 90px | L1 Small memory region |
| DRAMRenderConfiguration | 90px | DRAM memory plots |
| CBRenderConfiguration | 80px | Circular buffer plots |
| BufferRenderConfiguration | 82px | Buffer allocation plots |

These configurations define the visual appearance of memory plots, including height, margins, titles, and axis settings.

Sources: [src/definitions/PlotConfigurations.ts 8-62](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/definitions/PlotConfigurations.ts#L8-L62)[src/definitions/PlotConfigurations.ts 81-122](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/definitions/PlotConfigurations.ts#L81-L122)

## Summary

The memory visualization system provides a comprehensive set of tools for analyzing memory usage in TT-NN models. Through interactive visualizations, users can:

1.   Monitor memory allocations across different memory types (L1, DRAM)
2.   Identify peak memory usage and potential bottlenecks
3.   Analyze memory changes between operations
4.   Detect memory fragmentation
5.   Understand memory usage patterns in device operations

These capabilities help users optimize memory usage and diagnose memory-related issues in their models.
