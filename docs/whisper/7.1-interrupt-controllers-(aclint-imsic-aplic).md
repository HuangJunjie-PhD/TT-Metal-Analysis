## Purpose and Scope

This page documents the three interrupt controller mechanisms implemented in Whisper for RISC-V interrupt handling: ACLINT (Advanced Core Local Interruptor), IMSIC (Incoming Message Signaled Interrupt Controller), and APLIC (Advanced Platform-Level Interrupt Controller). These controllers implement the RISC-V Advanced Interrupt Architecture (AIA) specification and provide timer, software, and external interrupt delivery to harts.

For information about CSR management and privilege checking, see [Control and Status Registers (CSRs)](https://deepwiki.com/tenstorrent/whisper/3.1-control-and-status-registers-(csrs)). For details on how interrupts trigger exceptions, see [Hart Execution Unit](https://deepwiki.com/tenstorrent/whisper/2.2-hart-execution-unit).

## ACLINT: Core Local Interrupts

ACLINT provides core-local timer and software interrupts. Unlike external interrupts that come from platform devices, ACLINT interrupts are generated locally by timer comparisons or by software writes to memory-mapped registers.

### Timer Interrupts

Timer interrupts are generated when the `time` CSR exceeds a comparison value. Whisper implements both machine-mode (`MTIMECMP`) and supervisor-mode (`STIMECMP`) timer comparators:

The system maintains a single monotonic `time` value shared across all harts. Each hart has its own alarm register that is compared against `time`:

**Key Implementation Details:**

*   Machine timer alarm: stored per-hart in the `Hart` object via `getAclintAlarm()` / `setAclintAlarm()`
*   Supervisor timer (`STIMECMP`): CSR accessed via `CsrNumber::STIMECMP` (0x14D)
*   Timer comparison occurs on every instruction fetch
*   Virtual timer: When in VS-mode, `time` appears as `time + htimedelta`

**Sources:**[System.cpp 643-648](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L643-L648)[CsRegs.cpp 783-855](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L783-L855)[CsRegs.hpp 247-248](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.hpp#L247-L248)

### Software Interrupts

Software interrupts are triggered by writing to specific bits in the `MIP` CSR:

*   Machine software interrupt: `MIP[MSIP]` (bit 3)
*   Supervisor software interrupt: `MIP[SSIP]` (bit 1)

These are typically implemented as memory-mapped registers in the ACLINT MSWI (Machine Software Interrupt) device, though Whisper models them as CSR bits.

**Sources:**[CsRegs.cpp 736-754](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L736-L754)

## IMSIC: Message-Signaled Interrupt Controller

IMSIC implements the message-signaled interrupt (MSI) architecture defined in the RISC-V AIA specification. It uses memory-mapped files to receive interrupt messages and delivers them to hart privilege levels.

### IMSIC Architecture

**Sources:**[System.cpp 751-886](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L751-L886)[System.hpp 492](https://github.com/tenstorrent/whisper/blob/733fd921/System.hpp#L492-L492)

### IMSIC Memory Map

Each hart has separate memory-mapped files for machine, supervisor, and optionally multiple guest contexts:

| File Type | Base Address | Stride Formula | Address Range |
| --- | --- | --- | --- |
| Machine | `mbase` | `mbase + hartIx * mstride` | 1 page (4KB typically) |
| Supervisor | `sbase` | `sbase + hartIx * sstride` | 1 page |
| Guest [i] | `sbase + pageSize` | `sbase + sstride * hartIx + pageSize * (1+i)` | 1 page per guest |

**Configuration Requirements:**

*   `mbase`, `sbase` must be page-aligned
*   `mstride`, `sstride` must be multiples of page size
*   `sstride` must be large enough for guests: `sstride >= (guests + 1) * pageSize`
*   Machine and supervisor address ranges must not overlap

**Sources:**[System.cpp 762-817](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L762-L817)

### IMSIC CSR Interface

The IMSIC provides both memory-mapped and CSR-based access. The CSR interface uses indirect addressing:

**Reading IMSIC Registers via CSR:**

The indirect CSR mechanism allows software to access IMSIC interrupt files without memory-mapped I/O:

1.   Write the desired IMSIC register index to `MISELECT`/`SISELECT`/`VSISELECT`
2.   Read/write `MIREG`/`SIREG`/`VSIREG` to access the selected IMSIC register

**Implementation:** The `CsRegs::readMireg()` and related functions handle this indirection by calling `imsic_->readMireg()` with the select value.

**Sources:**[CsRegs.cpp 528-583](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L528-L583)[CsRegs.hpp 389-436](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.hpp#L389-L436)

### IMSIC Top Interrupt Registers

The `xTOPEI` and `xTOPI` registers provide efficient interrupt claiming:

**MTOPEI/STOPEI/VSTOPEI** (0x35C/0x15C/0x25C):

*   Returns the highest priority enabled and pending interrupt ID
*   Reading automatically claims the interrupt (clears pending bit)
*   Returns 0 if no interrupt is pending
*   Bits [26:16] duplicate bits [10:0] (per AIA spec)

**MTOPI/STOPI/VSTOPI** (0xFB0/0xDB0/0xEB0):

*   Returns interrupt ID and priority information
*   Read-only, does not claim interrupt
*   Provides priority for software interrupt arbitration

**Sources:**[CsRegs.cpp 626-661](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L626-L661)

### Virtual Supervisor IMSIC

In virtualized environments with the hypervisor extension, each guest has its own IMSIC file:

*   The `HSTATUS.VGEIN` field selects which guest file is active (1-indexed)
*   Guest 0 is reserved; valid guests are 1 to N
*   Each guest sees its own `VSIREG`/`VSTOPEI`/`VSTOPI` when in VS-mode
*   The hypervisor can access any guest's IMSIC by setting `VGEIN`

**Sources:**[CsRegs.cpp 553-583](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L553-L583)[System.cpp 800-805](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L800-L805)

APLIC routes wired interrupt sources from platform devices to harts. It supports hierarchical interrupt domains and can deliver interrupts either directly (setting bits in `MIP`/`SIP`) or via MSI messages to IMSIC.

### APLIC Direct Delivery

In direct delivery mode, APLIC sets interrupt pending bits directly in hart CSRs:

**Implementation Details:**

*   Callback defined in `System<URV>::configAplic()` at [System.cpp 895-920](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L895-L920)

## Interrupt Priority and Arbitration

When multiple interrupts are pending, the hart must determine which to service first. The priority mechanism varies by controller:

### IMSIC Priority

Within IMSIC:

*   Each interrupt ID can have an associated priority (via `eiprio[]` registers)
*   The `eithreshold` register masks interrupts below a certain priority
*   `xTOPEI` returns the highest priority pending interrupt
*   Priority 0 means disabled; higher numbers = higher priority

### Global Interrupt Priority

Across all interrupt sources, RISC-V defines a fixed priority order:

1.   Machine external (MEI)
2.   Machine software (MSI)
3.   Machine timer (MTI)
4.   Supervisor external (SEI)
5.   Supervisor software (SSI)
6.   Supervisor timer (STI)

The hart checks these in order during exception handling.

## Integration Summary

The three interrupt controllers work together to provide comprehensive interrupt support:

| Controller | Purpose | Delivery Mechanism | Configuration |
| --- | --- | --- | --- |
| **ACLINT** | Core-local timer and software interrupts | Direct to `MIP` bits | Per-hart alarm registers |
| **IMSIC** | Message-signaled interrupts (AIA) | Memory-mapped files → CSR bits | Memory map with machine/supervisor/guest files |
| **APLIC** | Platform-level wired interrupts | Direct to `MIP` or MSI to IMSIC | Domains with source routing |

**Typical Interrupt Flow:**

1.   External device (e.g., UART) asserts interrupt to APLIC source
2.   APLIC routes through domains and either: 
    *   Sets `MIP[xEIP]` directly (direct delivery), or
    *   Writes MSI message to IMSIC file (MSI delivery)

3.   IMSIC manages interrupt pending/enable bits and priority
4.   Hart checks `MIP & MIE` during instruction fetch
5.   If interrupt enabled and higher priority than current code, hart takes interrupt exception
