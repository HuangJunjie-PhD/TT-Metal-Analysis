This page documents the essential system utilities and tools for managing, monitoring, and deploying on Tenstorrent hardware. These utilities provide the foundation for hardware communication, device management, and system configuration required before using higher-level software frameworks.

## System Utilities Overview

The Tenstorrent system utilities include several components that enable communication with and management of Tensix processors. The primary low-level interface is provided by `luwen`, which serves as the backbone for higher-level tools like `tt-smi` and `tt-flash`.

## Installation & Configuration Tools

### tt-installer

The recommended method for setting up the environment is the `tt-installer` script. It automates the installation of drivers, firmware, and system tools [core/getting-started/README.md 43-45](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L43-L45)

**Key Automated Tasks:**

1.   **Driver Installation**: Deploys `TT-KMD` via DKMS [core/getting-started/README.md 136](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L136-L136)
2.   **Firmware Update**: Uses `TT-Flash` to apply `TT-System-Firmware`[core/getting-started/README.md 137-138](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L137-L138)
3.   **Memory Optimization**: Configures `HugePages` for high-performance memory access [core/getting-started/README.md 139](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L139-L139)
4.   **Monitoring**: Installs `TT-SMI`[core/getting-started/README.md 140](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L140-L140)

### TT-Flash & TT-System-Firmware

`TT-Flash` is the utility used to flash firmware bundles (`.fwbundle`) onto Tenstorrent devices [syseng/softwaresetup.md 34-36](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L34-L36)

*   **Installation**: `pip install git+https://github.com/tenstorrent/tt-flash.git`[syseng/softwaresetup.md 39](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L39-L39)
*   **Usage**: `tt-flash --fw-tar <bundle_name>.fwbundle`[syseng/softwaresetup.md 50](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L50-L50)
*   **Force Update**: If firmware is significantly outdated, use `--force`[syseng/softwaresetup.md 58](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L58-L58)

## Monitoring & Analysis Tools

### TT-SMI (System Management Interface)

`TT-SMI` is the primary command-line tool for telemetry and device management [core/tools/index.rst 7-9](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L7-L9)

*   **BIOS Prerequisite**: Requires PCIe AER Reporting Mechanism set to `OS First`[core/getting-started/README.md 31-33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L31-L33)
*   **Capabilities**: Reads power, temperature, voltage, and clock frequencies. It also provides device information and firmware versions [core/getting-started/manual-software-install.md 114](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L114-L114)

### TT-Toplike

A real-time terminal hardware monitor that provides visual feedback of chip health [core/tools/index.rst 11-13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L11-L13)

*   **Visualizations**: Includes "Starfield", "Memory Castle", and "Memory Flow" modes [core/tools/index.rst 13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L13-L13)
*   **Telemetry**: Reads power, temperature, and ARC firmware health [core/tools/index.rst 13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L13-L13)

### TT-NN Visualizer

An interactive tool for analyzing model execution [core/tools/index.rst 40-42](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L40-L42)

*   **Features**: Memory plots, tensor/buffer views, and operation flow diagrams [core/tools/index.rst 42](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L42-L42)
*   **Connectivity**: Supports loading reports via file or SSH [core/tools/index.rst 42](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L42-L42)

The canonical reference for deploying validated models on Tenstorrent hardware [core/tools/index.rst 18-20](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L18-L20)

*   **Management**: Handles Docker containers, model downloads, and serving configurations [core/tools/index.rst 20](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L20-L20)

A high-throughput LLM serving engine optimized for Tenstorrent silicon [core/tools/index.rst 22-24](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L22-L24) It serves as the backend for `TT-Inference-Server` during production LLM inference [core/tools/index.rst 24](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L24-L24)

A web-based interface that wraps `TT-Inference-Server` to provide a "point-and-click" deployment flow for users who prefer a GUI over the command line [core/tools/index.rst 26-28](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L26-L28)

## Hardware Compatibility & Requirements

### System Requirements (Wormhole/Blackhole)

*   **Operating System**: Ubuntu 22.04 LTS (Recommended) [core/getting-started/README.md 27-29](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L27-L29)
*   **Disk Space**: Minimum 360 GB free for recommended LLM deployments [core/getting-started/vLLM-servers.md 22](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L22-L22)
*   **BIOS**: PCIe AER Reporting must be `OS First`[core/getting-started/README.md 33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L33-L33)

### Gated Model Access

For tools like `vLLM` and `TT-Inference-Server`, access to specific models (e.g., Llama-3.3-70B) requires:

1.   **Hugging Face Account**: To request access to gated repositories [core/getting-started/vLLM-servers.md 60](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L60-L60)
2.   **HF_TOKEN**: An environment variable with read permissions to download weights [core/getting-started/vLLM-servers.md 82-86](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L82-L86)
