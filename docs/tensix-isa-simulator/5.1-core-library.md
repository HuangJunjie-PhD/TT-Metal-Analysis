# Core Library

Relevant source files
*   [tensix/whb0/prj/build_core.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_core.sh)

The Core Library provides fundamental functionality that serves as a foundation for the Tensix ISA Simulator. It offers common utilities, data structures, and abstractions that are used extensively by both the LLK System and r37 System components.

## Purpose and Scope

The Core Library is a essential supporting component that implements basic functionality required by multiple subsystems throughout the Tensix ISA Simulator codebase. By centralizing common code elements, it promotes consistency, reduces duplication, and simplifies maintenance across the system.

For information about other supporting libraries within the system architecture, see:

*   [Schedule Library](https://deepwiki.com/tenstorrent/tensix-isa-simulator/5.2-schedule-library)
*   [Serial Flash Programming Interface](https://deepwiki.com/tenstorrent/tensix-isa-simulator/5.3-serial-flash-programming-interface)
*   [Minicoro Coroutine Library](https://deepwiki.com/tenstorrent/tensix-isa-simulator/5.4-minicoro-coroutine-library)

## Build Process

The Core Library is built as a static library using a standardized build script:

Sources: [tensix/whb0/prj/build_core.sh 1-20](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_core.sh#L1-L20)

The build process defined in `build_core.sh` consists of:

1.   Compiling all C++ source files in the `../src/core/` directory using g++ with C++17 standard and optimization level O3
2.   Creating the static library archive `core.a` in the `../lib/` directory
3.   Cleaning up temporary object files

The Core Library is designed to be used as a foundation for higher-level components within the Tensix ISA Simulator. Common usage patterns include:

1.   **Direct Inclusion**: Components directly incorporating Core Library elements
2.   **Inheritance**: Higher-level classes extending base classes defined in the Core Library
3.   **Utility Usage**: Components utilizing helper functions provided by the Core Library

## Build Configuration

The build settings for the Core Library indicate it is compiled with:

| Setting | Value | Purpose |
| --- | --- | --- |
| C++ Standard | C++17 | Ensures modern language features are available |
| Optimization Level | O3 | Maximizes runtime performance |
| Include Path | ../src | Provides access to header files |
| Output | ../lib/core.a | Creates a static library for linking |

Sources: [tensix/whb0/prj/build_core.sh 11-15](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_core.sh#L11-L15)

## Summary

The Core Library serves as the foundation of the Tensix ISA Simulator, providing essential functionality used by both the LLK System and the r37 System. It centralizes common code elements to promote consistency and reduce duplication throughout the codebase. The library is built as a static component (core.a) that other system elements link against during their own build processes.
