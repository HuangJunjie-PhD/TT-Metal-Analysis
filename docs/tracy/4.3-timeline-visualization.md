This document covers the timeline visualization system in Tracy's profiler GUI, which renders the main interactive timeline view showing threads, GPU contexts, plots, and other profiling data. The timeline visualization handles real-time and offline trace data display, user interactions like zooming and panning, and hierarchical display of profiling events.

For information about the underlying data processing and storage, see [Data Processing Engine](https://deepwiki.com/tenstorrent/tracy/4.2-data-processing-engine). For details about UI framework integration, see [UI Framework & Rendering](https://deepwiki.com/tenstorrent/tracy/4.5-ui-framework-and-rendering).

## Timeline Rendering Pipeline

The timeline visualization follows a two-phase rendering approach: preprocessing and drawing. This separation allows for parallel data preparation while maintaining smooth interactive performance.

### Rendering Pipeline Flow

**Sources:**[server/TracyTimelineController.cpp 37-175](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineController.cpp#L37-L175)[server/TracyTimelineItem.cpp 23-132](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineItem.cpp#L23-L132)

The `TimelineContext` structure carries essential rendering parameters between phases:

| Field | Purpose |
| --- | --- |
| `w`, `ty`, `sty` | Width and text heights |
| `yMin`, `yMax` | Visible Y coordinate range |
| `pxns`, `nspx` | Pixels per nanosecond conversion |
| `vStart`, `vEnd` | Visible time range |
| `wpos` | Window position |
| `hover` | Mouse hover state |

**Sources:**[server/TracyTimelineContext.hpp 11-19](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineContext.hpp#L11-L19)

## Timeline Item Types and Responsibilities

### Thread Timeline Items

`TimelineItemThread` handles the most complex timeline rendering, displaying thread execution with multiple data types layered vertically.

**Sources:**[server/TracyTimelineItemThread.cpp 262-284](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineItemThread.cpp#L262-L284)[server/TracyTimelineItemThread.hpp 36-61](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineItemThread.hpp#L36-L61)

### Plot Timeline Items

`TimelineItemPlot` renders time-series data with automatic scaling and aggregation for dense datasets.

Plot rendering uses different strategies based on data density:

*   Individual points for sparse data
*   Min/max lines for dense regions
*   Sampling with up to 256 representative points for very dense data

**Sources:**[server/TracyTimelineItemPlot.cpp 122-236](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineItemPlot.cpp#L122-L236)[server/TracyView_Plots.cpp 17-232](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Plots.cpp#L17-L232)

## Data Preprocessing and Rendering

### Preprocessing Phase

Timeline items use parallel preprocessing to prepare drawing data without blocking the UI thread. The `Preprocess` method queues work using `TaskDispatch`:

**Sources:**[server/TracyTimelineItemThread.cpp 286-344](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineItemThread.cpp#L286-L344)[server/TracyTimelineItemThread.cpp 402-469](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineItemThread.cpp#L402-L469)

### Zone Folding Algorithm

Small zones that would be invisible at the current zoom level are automatically folded together to maintain performance:

`const auto MinVisNs = int64_t( round( GetScale() * MinVisSize * nspx ) );`
Zones smaller than `MinVisNs` nanoseconds are collapsed into folded representations showing the time span and count of contained zones.

## Context and Coordinate Systems

### Time-to-Pixel Conversion

The timeline visualization uses consistent coordinate transformations between time domain and pixel space:

| Conversion | Formula | Usage |
| --- | --- | --- |
| Time to Pixel | `(time - vStart) * pxns` | Positioning elements |
| Pixel to Time | `vStart + (pixel * nspx)` | Mouse position mapping |
| Zoom Calculation | `timespan / width` | Pixels per nanosecond |

**Sources:**[server/TracyView_Timeline.cpp 23-33](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyView_Timeline.cpp#L23-L33)[server/TracyTimelineContext.hpp 13-16](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyTimelineContext.hpp#L13-L16)

### Viewport Management

The timeline maintains viewport state in `ViewData`:

*   `zvStart`, `zvEnd`: Current visible time range
*   Drawing flags for different data types (`drawZones`, `drawGpuZones`, etc.)
*   Dynamic color mode and other visual preferences
