This page documents the Convolutional Neural Network (CNN) models supported by the PyBuda framework, detailing integration methods, configuration patterns, and hardware-specific optimizations. It covers how PyBuda efficiently runs CNN models on Tenstorrent hardware for computer vision tasks like image classification, object detection, and semantic segmentation.

For information about NLP models, see [NLP Models](https://deepwiki.com/tenstorrent/tt-buda/4.2-nlp-models).

CNN models benefit from specific optimizations in PyBuda:

### Key Optimization Techniques

1.   **Convolution Optimizations**:

    *   Pre-stride optimization: `compiler_cfg.enable_conv_prestride = True`

    *   Extra blob size configuration: `os.environ["TT_BACKEND_OVERLAY_MAX_EXTRA_BLOB_SIZE"] = "65536"`
    *   L1 margin control: `os.environ["PYBUDA_EXTRA_L1_MARGIN"] = "65536"`

## Conclusion

PyBuda provides comprehensive support for CNN models across different architectures and frameworks. Key considerations when working with CNNs in PyBuda include:

1.   **Model Selection**: Choose appropriate variant based on accuracy/performance requirements
2.   **Framework Integration**: Use `PyTorchModule` or `OnnxModule` to wrap models
3.   **Configuration Optimization**: Apply hardware-specific and model-specific configurations
4.   **Hardware Targeting**: Consider target hardware (Grayskull, Wormhole, Blackhole) when setting configurations

For detailed information about the PyBuda balancing and placement system used with CNN models, see [Balancer and Placement System](https://deepwiki.com/tenstorrent/tt-buda/2.2-balancer-and-placement-system).
