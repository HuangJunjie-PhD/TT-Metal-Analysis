This guide provides detailed instructions for installing vLLM, a high-performance inference and serving engine for Large Language Models. It covers various installation methods including pre-built packages, building from source, and using Docker containers. For information about supported models, see [Supported Models](https://deepwiki.com/tenstorrent/vllm/1.2-supported-models).

## System Requirements

vLLM supports different hardware platforms with the following requirements:

*   **CUDA (NVIDIA GPUs)**:

    *   Compute capability 7.0 or higher (V100, T4, RTX20xx, A100, L4, H100, etc.)
    *   CUDA Toolkit 11.8+
    *   Python 3.9-3.12

*   **CPU-only**:

    *   x86_64 architecture
    *   Python 3.9-3.12

*   **ROCm (AMD GPUs)**:

    *   Supported AMD GPUs (MI100, MI200, etc.)
    *   ROCm 5.6+
    *   Python 3.9-3.12

## Environment Variables

The following environment variables can be used to customize the vLLM installation:

| Variable | Description | Default |
| --- | --- | --- |
| `VLLM_TARGET_DEVICE` | Target device for building vLLM | `cuda` (if available) |
| `MAX_JOBS` | Maximum number of parallel compilation jobs | CPU count |
| `NVCC_THREADS` | Number of threads for NVCC compilation | 1 |
| `VLLM_USE_PRECOMPILED` | Use pre-compiled binaries | Unset |
| `VLLM_PRECOMPILED_WHEEL_LOCATION` | URL to pre-compiled wheel | Auto-detected |
| `VLLM_CUTLASS_SRC_DIR` | Path to local CUTLASS source | Downloaded automatically |
| `CMAKE_BUILD_TYPE` | CMake build type | `Debug` or `RelWithDebInfo` |
| `VERBOSE` | Enable verbose build output | Unset |
