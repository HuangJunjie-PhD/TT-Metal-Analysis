## Purpose and Scope

This page provides practical guidance for writing SFPU (Special Function Processing Unit) kernels using the SFPI API. It covers common coding patterns, idiomatic API usage, and real-world kernel examples. For information about the underlying SFPI types and operators, see [Vector Types and Operations](https://deepwiki.com/tenstorrent/sfpi/2.1-vector-types-and-operations). ## Kernel Function Structure

SFPU kernels are C++ functions that process data stored in destination registers. The SFPI API provides high-level abstractions over low-level SFPU hardware instructions.

### Basic Kernel Template

**Sources:**[include/sfpi.h 1-117](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L1-L117)[tests/wormhole/sfpi/gold/coverage.S 10-30](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/coverage.S#L10-L30)

### Minimal Kernel Example

A basic SFPU kernel follows this pattern:

1.   Load input values from destination registers using `dst_reg[index]`
2.   Perform computations using `vFloat`, `vInt`, or `vUInt` types
3.   Store results back to destination registers
4.   Increment the destination register pointer using `dst_reg++` for processing tiles

**Sources:**[include/sfpi.h 34-45](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L34-L45)[include/sfpi.h 241-251](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L241-L251)

## Register Access Patterns

### Destination Register Array

The `dst_reg` global provides indexed access to the 512-entry destination register file. Each entry holds a 64-element vector.

| Access Pattern | Code | Description |
| --- | --- | --- |
| Load from DReg | `vFloat a = dst_reg[0];` | Load vector from destination register 0 |
| Store to DReg | `dst_reg[2] = a * b;` | Store result to destination register 2 |
| Increment pointer | `dst_reg++;` | Advance by `SFP_DESTREG_STRIDE` |
| Offset increment | `dst_reg += 4;` | Advance by 4× stride |

**Sources:**[include/sfpi.h 194-251](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L194-L251)[include/sfpi.h 743-746](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L743-L746)

### Conditional Assignment Pattern

```
vFloat result = dst_reg[0];
v_if(result < 0.0f) {
    result = vConst0;  // Clamp negative values to zero
} v_endif;
dst_reg[0] = result;
```

This pattern compiles to `SFPSETCC` followed by conditional `SFPSTORE` instructions.

**Sources:**[tests/wormhole/sfpi/gold/coverage.S 572-651](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/coverage.S#L572-L651)[tests/wormhole/sfpi/gold/ckernel.S 693-733](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/ckernel.S#L693-L733)

### Exponential Kernel (Fast Approximation)

Fast exponential using bit manipulation:

**Key steps:**

1.   Scale input by 1/ln(2) to convert to base-2
2.   Extract integer part as exponent
3.   Compute fractional part approximation using LUT
4.   Reconstruct result with `setexp()`
5.   Handle overflow with conditional clamping

**Sources:**[tests/wormhole/sfpi/gold/ckernel.S 42-83](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/ckernel.S#L42-L83)

### GELU Activation

The Gaussian Error Linear Unit demonstrates complex function composition:

**Key steps:**

1.   Compute `x³` term using multiply operations
2.   Scale by constant (0.044715)
3.   Add linear term: `x + 0.044715 * x³`
4.   Multiply by √(2/π) constant
5.   Apply tanh LUT approximation
6.   Final scaling: `0.5 * x * (1 + tanh(...))`

**Sources:**[tests/wormhole/sfpi/gold/ckernel.S 924-984](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/ckernel.S#L924-L984)

### Conditional Comparison (Greater-Than)

Demonstrates comparison with immediate values:

**Key steps:**

1.   Load input as integer vector: `vInt val`
2.   Subtract comparison threshold
3.   Set condition code based on result sign
4.   Write 0.0 or 1.0 based on condition

### Absolute Value

Simple unary operation pattern:

```
vFloat val = dst_reg[0];
val = sfpi_abs(val);  // Uses SFPABS instruction
dst_reg[0] = val;
dst_reg++;
```

The `sfpi_abs()` function maps to the `SFPABS` hardware instruction which clears the sign bit.

**Sources:**[tests/wormhole/sfpi/gold/ckernel.S 673-686](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/ckernel.S#L673-L686)[tests/wormhole/sfpi/gold/coverage.S 1098-1113](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/coverage.S#L1098-L1113)

## Instruction Replay for Tiles

Processing 32×32 tiles efficiently using the `TTREPLAY` mechanism:

The pattern is:

1.   Write kernel code for processing one face (4 rows × 64 elements)
2.   Compiler automatically inserts `TTREPLAY` to repeat for remaining faces
3.   `dst_reg++` in the loop automatically advances pointers

**Sources:**[tests/wormhole/sfpi/gold/coverage.S 37-40](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/coverage.S#L37-L40)[tests/wormhole/sfpi/gold/ckernel.S 10-39](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/ckernel.S#L10-L39)

## Best Practices Summary

| Practice | Rationale |
| --- | --- |
| Use `vFloat` instead of raw intrinsics | Type safety, operator overloading, compiler optimization |
| Minimize local variable count | Limited to 8 LRegs; spills degrade performance |
| Reuse variables when possible | Reduces register pressure |
| Use constants from `vConst*` | Faster than load-immediate |
| Keep conditionals shallow | Deep nesting stresses condition code stack |
| Batch similar operations | Enables `TTREPLAY` instruction optimization |
| Load once, store once per face | Minimize memory operations |
