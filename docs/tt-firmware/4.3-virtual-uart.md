# Virtual UART

Relevant source files
*   [README.md](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1)

## Purpose and Scope

This document describes the Virtual UART system in the tt-firmware, which provides observability and debugging capabilities for Blackhole devices. The Virtual UART creates an in-memory interface that allows firmware messages to be streamed to and viewed from the host system.

This page focuses specifically on the Virtual UART implementation for Blackhole devices. For information about the broader ERISC Firmware, see [ERISC Firmware](https://deepwiki.com/tenstorrent/tt-firmware/4.1-erisc-firmware), and for ETH Mailbox functionality, see [ETH Mailbox](https://deepwiki.com/tenstorrent/tt-firmware/4.2-eth-mailbox).

## Overview

The Virtual UART is a debugging mechanism introduced in firmware version 18.2.0.0 that enables developers and users to access internal firmware log messages in real-time. It creates a virtual communication channel between the Blackhole's firmware and the host machine, without requiring a physical UART connection.

Sources: [README.md 39-41](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L39-L41)

## Architecture

The Virtual UART system consists of two main components:

1.   An in-memory circular buffer on the device that stores log messages
2.   A software interface that allows the host system to access these messages via the `tt-console` utility

The firmware components use standard logging functions like `printk()` and `LOG_*()` to write messages to the buffer, which are then made available to the host through the virtual UART interface.

Sources: [README.md 40-41](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L40-L41)

## Integration with Blackhole Firmware

The Virtual UART is enabled by default in Blackhole firmware bundles since version 18.2.0.0. It's an integral part of the Blackhole ERISC Firmware Architecture and serves as the primary debugging interface.

Sources: [README.md 35-41](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L35-L41)

## Usage

To view firmware debug messages, users can use the `tt-console` utility from the host system. This tool connects to the Virtual UART and displays messages in real-time.

The Virtual UART captures two primary types of messages:

1.   `printk()` messages - General debug output
2.   `LOG_*()` messages - Structured logging with different severity levels

## Configuration

The Virtual UART is enabled by default in Blackhole firmware bundles as of version 18.2.0.0. No additional configuration is required to use this feature with Blackhole devices.

| Feature | Status | Version Introduced |
| --- | --- | --- |
| Virtual UART | Enabled by default | 18.2.0.0 |
| Support for printk() | Available | 18.2.0.0 |
| Support for LOG_*() | Available | 18.2.0.0 |
| tt-console support | Available | 18.2.0.0 |

## Benefits and Use Cases

The Virtual UART provides several key benefits for firmware development and debugging:

1.   **Real-time observability**: Monitor firmware execution and state in real-time
2.   **Non-invasive debugging**: Observe system behavior without disrupting normal operation
3.   **Host-based access**: Access debug information directly from the host system
4.   **Persistent logging**: Capture firmware messages throughout the device lifecycle

Common use cases include:

*   Debugging firmware initialization issues
*   Monitoring system state during operation
*   Troubleshooting connectivity problems
*   Verifying firmware update processes

Sources: [README.md 39-41](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L39-L41)

## Technical Details

The Virtual UART leverages an in-memory buffer to store log messages, which are then made available to the host system. This approach eliminates the need for physical UART hardware while still providing comparable functionality.

Sources: [README.md 40-41](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L40-L41)

## Summary

The Virtual UART is a key debugging and observability feature for Blackhole firmware, providing a virtual communication channel between the device firmware and host system. Enabled by default since firmware version 18.2.0.0, it allows users to view firmware messages using the `tt-console` utility. The system captures both general debug output via `printk()` and structured logs via `LOG_*()` functions, making it an essential tool for firmware development and troubleshooting.
