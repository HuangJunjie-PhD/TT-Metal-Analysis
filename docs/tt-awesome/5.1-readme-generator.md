The `scripts/generate_readme.py` script is a core component of the `tt-awesome` automation pipeline. It is responsible for transforming the collection of JSON files in the `entries/` directory into a human-readable, formatted `README.md` file. This ensures that the repository's landing page remains a synchronized, "community-first" view of the Tenstorrent ecosystem without requiring manual Markdown edits.

### Purpose and Scope

The generator enforces a specific project philosophy through its sorting algorithm and visual presentation. It handles:

*   **Data Loading**: Aggregating all non-hidden project entries from the filesystem.
*   **Sorting**: Implementing a "community-first" order where community projects are prioritized over official ones.
*   **Formatting**: Converting JSON fields into Markdown blocks with icons, shields.io badges, and standardized link formats.
*   **Idempotency**: Producing consistent output every time it runs to avoid unnecessary git noise.

* * *

### Implementation Details

#### Data Loading and Filtering

The process begins in `load_entries()`, which recursively searches the `entries/` directory for `.json` files [scripts/generate_readme.py 70-75](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L70-L75) It performs a critical filtering step by excluding any entry where `"hidden": true` is set, allowing the maintainers to keep draft or historical data in the JSON database without cluttering the public README [scripts/generate_readme.py 76-77](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L76-L77)

#### Community-First Sorting Algorithm

The `sort_entries()` function implements the project's core ranking logic [scripts/generate_readme.py 80-91](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L80-L91) It uses a multi-level sort key to determine the order of projects within each category:

| Priority | Key | Logic |
| --- | --- | --- |
| 1 | **Affiliation** | Uses `AFFILIATION_ORDER`: `community` (0) <`affiliated` (1) <`official` (2). Lower values appear first [scripts/generate_readme.py 31-88](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L31-L88) |
| 2 | **Featured** | Featured entries (boolean `true`) are boosted to the top of their affiliation tier [scripts/generate_readme.py 89](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L89-L89) |
| 3 | **Popularity** | Entries are sorted by star count in descending order [scripts/generate_readme.py 90](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L90-L90) |

#### Markdown Rendering

The `render_entry(e)` function converts a single JSON object into a Markdown list item [scripts/generate_readme.py 108-147](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L108-L147)

*   **Badges**: It assigns a colored Shields.io badge based on the `affiliation` field [scripts/generate_readme.py 63-118](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L63-L118)
*   **Authors**: GitHub handles are automatically detected via regex and converted into profile links [scripts/generate_readme.py 94-105](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L94-L105)
*   **Link Icons**: Typed links (e.g., `repo`, `paper`, `video`) are prefixed with specific emojis defined in `LINK_ICONS`[scripts/generate_readme.py 34-130](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L34-L130)
*   **Packages**: If an entry contains package registry information (PyPI, APT, Cargo), the script generates install command snippets with registry-specific icons [scripts/generate_readme.py 40-138](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L40-L138)

#### Categorization and Table of Contents

The script iterates through the `CATEGORIES` constant to group projects [scripts/generate_readme.py 14-175](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L14-L175) It dynamically generates a Table of Contents, only including categories that contain at least one visible project [scripts/generate_readme.py 165-172](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L165-L172)
