This page describes how Python control flow constructs (for loops, if statements) are used within tt-lang kernels to enable iterative computation, streaming patterns, and conditional execution. Control flow in tt-lang operates at the **Python AST level** during kernel compilation, meaning standard Python syntax is transformed into MLIR Structured Control Flow (SCF) dialect operations.

* * *

## Supported Control Flow Constructs

tt-lang supports a subset of Python control flow within kernel functions decorated with `@ttl.compute()` and `@ttl.datamovement()`. The `TTCompilerBase` class defines the supported AST nodes for these transformations [[python/pykernel/_src/kernel_ast.py:69-127]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

| Construct | Supported | MLIR Lowering | Notes |
| --- | --- | --- | --- |
| `for i in range(...)` | ✅ Yes | `scf.for` | 1, 2, or 3 argument forms [[python/pykernel/_src/kernel_ast.py:210-247]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops) |
| `if condition:` | ✅ Yes | `scf.if` | Includes `else`. `elif` is handled as nested `if`[[python/pykernel/_src/kernel_ast.py:163-208]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops) |
| `while condition:` | ❌ No | N/A | Not in `supported_nodes`[[python/pykernel/_src/kernel_ast.py:69-127]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops) |
| `break` / `continue` | ❌ No | N/A | Not supported in SCF dialect |
| Nested loops | ✅ Yes | Nested `scf.for` | Used for multi-tile indexing [[python/pykernel/_src/kernel_ast.py:210-247]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops) |
| Ternary expressions | ✅ Yes | `scf.if` | `x if cond else y` mapped via `ast.IfExp`[[python/pykernel/_src/kernel_ast.py:82]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops) |

## For Loops with range()

### Basic Syntax and Lowering

For loops in tt-lang kernels must use the `range()` function [[python/pykernel/_src/kernel_ast.py:211]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops). The `visit_For` method in the compiler extracts the lower bound, upper bound, and step to create an `scf.for` operation.

The compiler automatically performs type casting to the MLIR `IndexType` to satisfy `scf.for` requirements [[python/pykernel/_src/kernel_ast.py:241-246]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

## If Statements and Conditionals

### Conditional Logic

The `visit_If` method handles branching. It validates that the condition is an integer type (or a boolean resulting from a comparison) [[python/pykernel/_src/kernel_ast.py:182-183]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops). If the condition width is not 1-bit, the compiler inserts an implicit comparison against zero (`ne 0`) to treat it as a boolean [[python/pykernel/_src/kernel_ast.py:185-189]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

### Core-Specific Execution

A common pattern is using core coordinates to guard execution. This is often achieved by querying the grid size or node ID:

`grid_cols, grid_rows = ttl.grid_size(dims=2) # Returns grid dimensionsnode_col, node_row = ttl.node(dims=2)       # Returns current node coordinates`
The `ttl.grid_size` function returns the size of the grid in the specified dimensionality [[docs/sphinx/specs/TTLangSpecification.md:113]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops). `ttl.node()` provides the zero-based coordinates used to offset into global tensors [[examples/elementwise-tutorial/step_3_multinode.py:96-109]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

## Loop Patterns for Streaming

### Pattern: Streaming and Double Buffering

The most critical use of loops is streaming data through `DataflowBuffers` (DFBs). The DFB lifecycle (reserve/wait/push/pop) is typically wrapped in a loop to process large tensors in smaller L1-resident blocks [[docs/sphinx/specs/TTLangSpecification.md:54-56]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

**Critical Constraint:** Loop iteration counts **must match** across compute and datamovement threads. Every `reserve()` or `push()` in a producer thread must be balanced by a `wait()` or `pop()` in a consumer thread [[examples/elementwise-tutorial/step_3_multinode.py:80-88]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops). Mismatches lead to permanent hardware deadlocks.

## Intermediate DFB Materialization

During compilation, if a loop or conditional branch produces a value that is required as a DFB-attached input by a subsequent operation, the compiler may insert intermediate DFBs.

The `TTLInsertIntermediateDFBsPass` materializes these values to L1 [[lib/Dialect/TTL/Transforms/TTLInsertIntermediateDFBs.cpp:9-15]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops). It hoists `BindCBOp` to the function entry to ensure deterministic ordering while keeping `reserve`, `store`, and `wait` operations at the definition site to preserve per-invocation accounting inside loops [[lib/Dialect/TTL/Transforms/DFBMaterialization.cpp:24-36]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

## Generated Code Structure

### Variable Scoping

The compiler manages variable visibility using a stack of symbol tables. Entering an `if` or `for` block pushes a new scope, which is popped upon exit [[python/pykernel/_src/kernel_ast.py:194-208]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).

### Error Handling in Control Flow

The compiler performs strict validation during AST visiting:

1.   **Non-Integer Conditionals**: Using non-integer types for `if` conditions results in a `ValueError` during compilation [[python/pykernel/_src/kernel_ast.py:183]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).
2.   **Undefined Names**: Referencing an undefined variable inside a loop or compute block produces a `TTLangCompileError`[[python/ttl/ttl_api.py:79-84]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).
3.   **Range Usage**: Only the `range()` function is supported for loop iteration [[python/pykernel/_src/kernel_ast.py:211]](https://deepwiki.com/tenstorrent/tt-lang/4.5-control-flow-and-loops).
