The Operation List provides a comprehensive view of all operations in a TT-NN model, allowing users to browse, search, filter, and sort operations to efficiently analyze model execution. This page documents the Operation List view, its features, and how to use it effectively.

For detailed information about analyzing a specific operation's memory usage and execution details, see [Operation Details](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.2-operation-details).

## Overview

The Operation List displays operations from the loaded TT-NN model profile in a virtualized scrollable list. Each operation entry shows its ID, name, source file reference, and provides expandable sections to view arguments, tensor information, and performance metrics when available.

Sources: [src/components/OperationList.tsx 1-374](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/OperationList.tsx#L1-L374)

## Data Structure

The Operation List displays information defined by the `OperationDescription` interface, which includes:

| Field | Description |
| --- | --- |
| `id` | Unique identifier for the operation |
| `name` | Name of the operation |
| `operationFileIdentifier` | Source file and line reference |
| `duration` | Execution time in seconds |
| `arguments` | Configuration parameters for the operation |
| `inputs` | Input tensors consumed by the operation |
| `outputs` | Output tensors produced by the operation |
| `device_operations` | Device-level operations executed |
| `stack_trace` | Python stack trace for debugging |

Sources: [src/model/APIData.ts 176-183](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/APIData.ts#L176-L183)

## Key Features

### Filtering and Searching

The Operation List includes a search field that filters operations by matching text in the operation ID, name, or source file identifier. This allows for quickly finding specific operations of interest.

### Sorting Operations

Operations can be sorted by:

*   ID (ascending/descending)
*   Duration (ascending/descending)

## Usage Guide

### Basic Navigation

1.   Use the scroll bar or scroll wheel to navigate through the list of operations
2.   Click on an operation header to expand/collapse details
3.   Use the "Memory details" button to navigate to detailed memory visualization for that operation

### Advanced Filtering and Sorting

1.   Enter text in the search field to filter operations by ID, name, or source file
2.   Click the sort buttons to sort operations by ID or duration
3.   Toggle between ascending and descending sort order
4.   Use the expand/collapse all button to show or hide all operation details

### Performance Tips

*   The Operation List renders only visible elements, allowing efficient browsing of very large models
*   When working with large models, use the search functionality to find specific operations
*   The "Scroll to top" and "Scroll to bottom" buttons provide quick navigation to model endpoints

## Related Components

The Operation List works in conjunction with several other components in the visualizer:

*   **Operation Details**: Provides detailed memory and execution analysis for a specific operation
*   **Range Slider**: Allows selection of a subset of operations in the model execution timeline
*   **Performance Report**: Displays performance metrics for operations when a performance report is loaded

For more information about these components, refer to [Operation Details](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.2-operation-details), [Performance Analysis](https://deepwiki.com/tenstorrent/ttnn-visualizer/5-performance-analysis), and [Buffer Summary](https://deepwiki.com/tenstorrent/ttnn-visualizer/6-buffer-summary).
