For information about the pytest fixtures that orchestrate test execution (including `compile_and_run`), see [pytest Infrastructure and Fixtures](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/4.1-pytest-infrastructure-and-fixtures). ## Comparison Utilities Overview

The testing framework provides three primary comparison methods for validating numerical accuracy between PyTorch native execution and ttnn-compiled execution:

| Comparison Method | Function | Use Case | Threshold Parameter |
| --- | --- | --- | --- |
| Pearson Correlation Coefficient (PCC) | `comp_pcc()`, `assert_with_pcc()` | Default accuracy metric, handles scale/offset invariance | `pcc` (default 0.999) |
| Units of Least Precision (ULP) | `comp_ulp()`, `assert_with_ulp()` | Strict floating-point accuracy, measures numerical precision | `ulp_threshold` (default 10) |
| Relative/Absolute Tolerance | `comp_allclose()`, `assert_allclose()` | PyTorch standard allclose semantics | `rtol`, `atol` |

## PCC-Based Comparison

Pearson Correlation Coefficient (PCC) measures the linear correlation between two tensors, ranging from -1 (perfect negative correlation) to +1 (perfect positive correlation). The implementation handles edge cases including NaN values, infinite values, and all-zero tensors.

### comp_pcc Function

[tests/utils.py 200-252](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L200-L252)

The core computation function:

**Key behaviors:**

*   Converts inputs to `torch.Tensor` if needed
*   Type-casts `calculated` to match `golden.dtype`
*   Returns `(True, 1.0)` if both tensors are all NaN
*   Returns `(False, 0.0)` if only one tensor is all NaN or all zero
*   Masks out NaN, inf, and -inf values before computing correlation
*   Returns `(True, 1.0)` if tensors are exactly equal after masking
*   Converts bfloat16 to float32 for computation
*   Uses `np.ma.corrcoef()` with masked invalid values

### assert_with_pcc Function

[tests/utils.py 266-272](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L266-L272)

## ULP-Based Comparison

Units of Least Precision (ULP) provides a strict measure of floating-point accuracy by counting how many representable floating-point numbers exist between two values.

### ulp Function

[tests/utils.py 307-339](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L307-L339)

Computes the ULP value for each element of a tensor:

**Implementation details:**

*   Uses `torch.nextafter()` to find the next representable floating-point number
*   Handles special case where `x == dtype_max` (would overflow to inf)
*   Ensures symmetry: `ULP(-x) == ULP(x)`
*   Supports ttnn.Tensor by converting to/from torch.Tensor

### comp_ulp Function

[tests/utils.py 342-379](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L342-L379)

Computes the maximum ULP error between two tensors:

**Key behaviors:**

*   Returns `(True, "Both tensors are empty")` for empty tensors
*   Checks for non-finite values unless `allow_nonfinite=True`
*   Uses `_comp_nonfinite()` to verify matching NaN/inf patterns
*   Masks out non-finite elements before computing ULP error
*   Measures ULP according to the `calculated` tensor's dtype
*   Handles higher-precision golden tensors for sub-1 ULP measurements

### assert_with_ulp Function

[tests/utils.py 395-440](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L395-L440)

The primary ULP assertion function with ttnn.Tensor support:

**Special handling:**

*   Converts ttnn.Tensor to torch.Tensor with appropriate dtype
*   For `ttnn.bfloat8_b`, converts to `torch.bfloat16` (not float32) for ULP measurement
*   Validates ULP threshold against maximum meaningful value per dtype: 
    *   float64: 2^52
    *   float32: 2^23
    *   float16: 2^10
    *   bfloat16: 2^7

**Usage example:**[tests/lowering/eltwise/binary/test_pow_scalar.py 44](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/lowering/eltwise/binary/test_pow_scalar.py#L44-L44)

## Metric String Rendering System

The metric string rendering system converts string representations of operation signatures (captured during metrics collection) back into executable Python values for automated test generation.

### Metric String Format

Metric strings follow this pattern:

```
<type><shape> <name> = <raw_val>
```

Examples:

*   `Tensor<[12, 1]> self = None`
*   `int dim = 0`
*   `Optional[float] alpha = 1.0`
*   `List[Tensor] indices = [None, None, <[12, 1]>, <[16]>]`

### MetricStrRenderer Class

[tests/utils.py 513-645](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L513-L645)

Parses a single metric string and renders it to a Python value:

**Supported types:**

*   Basic types: `Tensor`, `bool`, `int`, `float`, `number`, `Device`
*   Optional types: `Optional[Tensor]`, `Optional[bool]`, etc.
*   List types: `List[Tensor]`, `List[int]`, etc.
*   Nested types: `List[Optional[Tensor]]`, `Optional[List[int]]`, etc.

**Rendering methods:**[tests/utils.py 517-618](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L517-L618)

| Method | Type | Behavior |
| --- | --- | --- |
| `_by_tensor()` | `Tensor` | Parses shape, generates `torch.randn(shape).type(torch.bfloat16)` |
| `_by_basic()` | `bool`, `int`, `float` | Evaluates raw_val with `eval()` |
| `_by_listTensor()` | `List[Tensor]` | Parses list format, generates tensor for each shape |
| `_by_device()` | `Device` | Wraps raw_val in quotes for eval |

### MetricStringListHandler Class

[tests/utils.py 648-780](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L648-L780)

Handles lists of metric strings and performs post-processing adjustments for specific operations:

**Post-adjustment operations:**[tests/utils.py 652-748](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L652-L748)

| Operation | Adjustment | Reason |
| --- | --- | --- |
| `aten.bitwise_not.default` | Convert `self` to `torch.int` | Bitwise ops require integer types |
| `aten.masked_fill.Scalar` | Generate bool `mask` | Mask must be boolean tensor |
| `aten.where.self` | Generate bool `condition` | Condition must be boolean |
| `aten.embedding.default` | Clamp `indices` to `[0, weight.shape[0])` | Indices must be valid |
| `aten.index_select.default` | Clamp `index` to `[0, self.shape[dim])` | Indices must be in bounds |
| `aten.index.Tensor` | Clamp each element in `indices` list | Multi-dim indexing |
| `aten._native_batch_norm_legit_no_training.default` | Square `running_var` to ensure ≥0 | Variance must be non-negative |

**Args/kwargs construction:**[tests/utils.py 750-767](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L750-L767)

The `build_input_args_kwargs()` method separates positional and keyword arguments:

*   If no parameter named `self`, all inputs become kwargs
*   If `self` exists, inputs before `self` become positional args, inputs after become kwargs
*   This matches PyTorch's calling convention for methods vs functions

## Type Conversion Dictionary

[tests/utils.py 383-392](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L383-L392)

The `tt_dtype_to_torch_dtype` dictionary maps ttnn dtypes to torch dtypes for comparison operations:

| ttnn dtype | torch dtype | Notes |
| --- | --- | --- |
| `ttnn.uint8` | `torch.uint8` | Direct mapping |
| `ttnn.uint16` | `torch.int16` | Unsigned → signed (limitation) |
| `ttnn.uint32` | `torch.int32` | Unsigned → signed (limitation) |
| `ttnn.int32` | `torch.int32` | Direct mapping |
| `ttnn.float32` | `torch.float` | Direct mapping |
| `ttnn.bfloat16` | `torch.bfloat16` | Direct mapping |
| `ttnn.bfloat8_b` | `torch.float` | Upcast for general comparison |
| `ttnn.bfloat4_b` | `torch.float` | Upcast for general comparison |

**Special case for ULP:**[tests/utils.py 401-410](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L401-L410)

For ULP measurement, `ttnn.bfloat8_b` is converted to `torch.bfloat16` instead of `torch.float` because the precision of bfloat8_b matches bfloat16 (when all elements in a block share the same exponent).

**Sources:**[tests/utils.py 383-410](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/utils.py#L383-L410)
