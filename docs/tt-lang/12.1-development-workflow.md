This page describes the standard workflow for developing `tt-lang`, including environment setup, building from source, and running the test suite. `tt-lang` positions itself as an ergonomic middle ground between high-level TT-NN and low-level TT-Metalium, leveraging an MLIR-based compilation pipeline.

* * *

## Environment Architecture

The development environment integrates several distinct components: the Python-based DSL, the MLIR compiler infrastructure (built on LLVM and `tt-mlir`), and the `tt-metal` runtime.

## Installation and Setup

### Prerequisites

*   **CMake 3.28+**: Core build system [docs/sphinx/build.md 13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L13-L13)
*   **Ninja**: Recommended build generator [docs/sphinx/build.md 14](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L14-L14)
*   **Python 3.10+**: (3.12 recommended) [docs/sphinx/build.md 16](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L16-L16)
*   **Git**: Submodules must be initialized via `git submodule update --init --recursive`[docs/sphinx/build.md 17-18](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L17-L18)
*   **Tenstorrent Device**: Detected at configure time by checking `/dev/tenstorrent*`[cmake/modules/TTLangUtils.cmake 68-71](https://github.com/tenstorrent/tt-lang/blob/d76e6233/cmake/modules/TTLangUtils.cmake#L68-L71)

### Build Modes

`tt-lang` supports several build configurations depending on whether you want to build dependencies from source or use a pre-built toolchain.

| Mode | CMake Flags | Description |
| --- | --- | --- |
| **Default** | `cmake -G Ninja -B build` | Builds LLVM/MLIR and `tt-metal` from submodules [docs/sphinx/build.md 22-28](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L22-L28) |
| **Build Toolchain** | `-DTTLANG_BUILD_TOOLCHAIN=ON` | Builds and installs LLVM and `tt-metal` into a reusable prefix [docs/sphinx/build.md 38-42](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L38-L42) |
| **Use Toolchain** | `-DTTLANG_USE_TOOLCHAIN=ON` | Skips LLVM and `tt-metal` builds; uses pre-built artifacts from `/opt/ttlang-toolchain`[docs/sphinx/build.md 111-115](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/build.md?plain=1#L111-L115) |
| **Simulator Only** | `-DTTLANG_SIM_ONLY=ON` | Sets up Python venv for simulation; skips all C++ compiler builds [CMakeLists.txt 79-81](https://github.com/tenstorrent/tt-lang/blob/d76e6233/CMakeLists.txt#L79-L81) |

### Versioning

The build system automatically detects the version from git tags using `GetVersionFromGit.cmake`[CMakeLists.txt 16-18](https://github.com/tenstorrent/tt-lang/blob/d76e6233/CMakeLists.txt#L16-L18) If no tag is reachable, it falls back to `0.0.0+unknown`[cmake/modules/GetVersionFromGit.cmake 8-10](https://github.com/tenstorrent/tt-lang/blob/d76e6233/cmake/modules/GetVersionFromGit.cmake#L8-L10) Versions follow PEP 440 format: `MAJOR.MINOR.PATCH[.devN][+LOCAL]`[cmake/modules/TTLangUtils.cmake 12-13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/cmake/modules/TTLangUtils.cmake#L12-L13)

## Environment Activation

The `source build/env/activate` script is generated by CMake and is essential for development [env/activate.in 5](https://github.com/tenstorrent/tt-lang/blob/d76e6233/env/activate.in#L5-L5) It performs the following:

*   **Python Venv**: Activates the internal virtual environment containing `torch`, `ttnn`, and `mlir` bindings [env/activate.in 21-26](https://github.com/tenstorrent/tt-lang/blob/d76e6233/env/activate.in#L21-L26)
*   **PATH**: Adds built binaries (`ttlang-opt`), LLVM tools, and the venv bin to the system path [env/activate.in 42](https://github.com/tenstorrent/tt-lang/blob/d76e6233/env/activate.in#L42-L42)
*   **PYTHONPATH**: Includes the built `python_packages`, the source `python/` tree, and the `test/` directory [env/activate.in 45-49](https://github.com/tenstorrent/tt-lang/blob/d76e6233/env/activate.in#L45-L49)
*   **Device Detection**: Exports `TTLANG_HAS_DEVICE` based on hardware availability detected during configuration [env/activate.in 30](https://github.com/tenstorrent/tt-lang/blob/d76e6233/env/activate.in#L30-L30)

## Testing Infrastructure

`tt-lang` uses a multi-tiered testing strategy involving `llvm-lit` for MLIR/unit tests and `pytest` for end-to-end and simulation tests.

**Diagram: Test Suite Organization**

**Sources:**[test/TESTING.md 16-27](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/TESTING.md?plain=1#L16-L27)[.github/workflows/call-build.yml 81-100](https://github.com/tenstorrent/tt-lang/blob/d76e6233/.github/workflows/call-build.yml#L81-L100)

### Test Targets

| Target | Command | Scope |
| --- | --- | --- |
| `check-ttlang-mlir` | `cmake --build build --target check-ttlang-mlir` | MLIR dialect and pass tests using `FileCheck`[test/TESTING.md 20](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/TESTING.md?plain=1#L20-L20) |
| `check-ttlang-python-lit` | `cmake --build build --target check-ttlang-python-lit` | Verifies generated MLIR and C++ from Python DSL [test/TESTING.md 21](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/TESTING.md?plain=1#L21-L21) |
| `check-ttlang-pytest` | `cmake --build build --target check-ttlang-pytest` | Parametric Python end-to-end tests [test/TESTING.md 24](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/TESTING.md?plain=1#L24-L24) |
| `Simulator Tests` | `pytest test/sim` | Functional simulation and deadlock detection [test/TESTING.md 30-31](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/TESTING.md?plain=1#L30-L31) |

### Continuous Integration

The GitHub Actions workflow `call-build.yml` automates the build and test process in a Docker container [.github/workflows/call-build.yml 32-38](https://github.com/tenstorrent/tt-lang/blob/d76e6233/%20.github/workflows/call-build.yml#L32-L38) It runs smoketests, lit tests, and packaging tests before optionally triggering hardware tests on `n150` devices [.github/workflows/call-build.yml 76-114](https://github.com/tenstorrent/tt-lang/blob/d76e6233/%20.github/workflows/call-build.yml#L76-L114)
