## Purpose and Scope

This page documents the Collective Communication Library (CCL) operations in TTNN, which enable coordinated tensor operations across multiple Tenstorrent devices. CCL operations move and reduce data across chips connected via Ethernet links, relying on TT-Fabric for inter-chip packet routing.

This page covers the TTNN-level API, operation semantics, configuration parameters, and internal implementation structure. For TT-Fabric internals and the ERISC Data Mover (EDM), see [Control Plane and Fabric System (2.11)](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/Control%20Plane%20and%20Fabric%20System%20(2.11)) For `MeshDevice` topology setup, see [Device Abstraction Layer (2.3)](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/Device%20Abstraction%20Layer%20(2.3)) For multi-device programming patterns including tensor sharding, see [Performance Optimization Techniques (7.4)](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/Performance%20Optimization%20Techniques%20(7.4))

## Available Operations

### Core CCL Operations

| Python API | Description |
| --- | --- |
| `ttnn.all_gather` | Each device gathers a tensor shard from all other devices along a specified dimension |
| `ttnn.reduce_scatter` | All devices reduce (sum) tensors from all peers; each device receives a unique scatter shard of the result |
| `ttnn.all_reduce` | All devices reduce tensors; each device receives the full reduced result |
| `ttnn.all_broadcast` | Broadcasts a tensor from one device to all others |
| `ttnn.broadcast` | Broadcasts from a specific source device |
| `ttnn.mesh_partition` | Partitions a tensor across the mesh |
| `ttnn.point_to_point` | Sends a tensor from one device to a specific other device |
| `ttnn.all_to_all_combine` | Combines data from all devices |
| `ttnn.all_to_all_dispatch` | Dispatches data to all devices |
| `ttnn.reduce_to_root` | Reduces data to a root device |

These are declared in `ttnn/cpp/ttnn/operations/ccl/all_gather/all_gather.hpp`, `ttnn/cpp/ttnn/operations/ccl/reduce_scatter/reduce_scatter.hpp`, and other CCL headers [ttnn/cpp/ttnn/operations/ccl/reduce_scatter/reduce_scatter.cpp 1-25](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/ccl/reduce_scatter/reduce_scatter.cpp#L1-L25)

### Experimental Async Variants

High-performance variants with explicit fabric lifecycle management live under `ttnn.experimental`:
