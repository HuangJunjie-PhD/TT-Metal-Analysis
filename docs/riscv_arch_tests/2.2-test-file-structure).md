This document details the internal structure of RISC-V assembly test files and their associated linker scripts, explaining how the mini-OS, test logic, and memory layout components are organized within individual test executables.

For information about how tests are organized across the repository hierarchy, see [Test Organization](/tenstorrent/riscv_arch_tests/2.1-test-organization). For details about atomic memory operation testing implementation, see [Atomic Memory Operations Tests](/tenstorrent/riscv_arch_tests/2.3-atomic-memory-operations-tests).

## Assembly Test File Architecture

Each RISC-V test consists of two primary files: an assembly source file (`.S`) and a linker script (`.ld`) that work together to create a complete test executable with embedded mini-OS functionality.

## Test Logic Organization

### Synchronization Macros

The test framework provides sophisticated synchronization primitives for multi-hart testing:

| Macro Name | Purpose | Key Operations |
| --- | --- | --- |
| `MUTEX_ACQUIRE_AMO` | Atomic mutex acquisition | `amoswap.w.aq` for lock acquisition |
| `MUTEX_RELEASE_AMO` | Atomic mutex release | `amoswap.w.rl` for lock release |
| `SEMAPHORE_ACQUIRE_TICKET` | Semaphore-based synchronization | Decrement semaphore atomically |
| `OS_SYNC_HARTS` | Hart barrier synchronization | Multi-phase barrier implementation |
| `CRITICAL_SECTION_AMO` | Protected critical sections | Combine lock acquisition with function calls |

### Test Execution Control

The mini-OS provides function dispatch through the exception mechanism using special function codes in register `x31`:

| Function Code | Purpose | Implementation |
| --- | --- | --- |
| `0xf0000001` | Schedule next test | `enter_scheduler` |
| `0xf0000002` | Fail current test | `test_failed` |
| `0xf0000003` | End test successfully | `os_end_test` |
| `0xf0001001` | Switch to machine mode | `os_fn_f0001001` |
| `0xf0001002` | Switch to supervisor mode | `os_fn_f0001002` |
| `0xf0001003` | Switch to user mode | `os_fn_f0001003` |
| `0xf0002001` | Get hart ID | `os_fn_f0002001` |
