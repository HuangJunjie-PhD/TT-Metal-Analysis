# Environment Variables

Relevant source files
*   [install.sh](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh)

This page documents the environment variables that can be used to customize the Tenstorrent software stack installation process. Environment variables provide a way to control various aspects of the installation without modifying the installation script directly. They can be used to skip certain installation steps, specify component versions, configure Python environments, and determine installation modes.

For information about installation modes beyond environment variables, see [Installation Modes](https://deepwiki.com/tenstorrent/tt-installer/4.2-installation-modes).

## Skip Installation Flags

These variables allow you to skip specific installation steps. Set the value to `0` to skip the associated installation step, or `1` (default) to perform it.

| Variable | Description | Default |
| --- | --- | --- |
| `TT_SKIP_INSTALL_KMD` | Skip Kernel Mode Driver installation | `1` (install) |
| `TT_SKIP_INSTALL_HUGEPAGES` | Skip HugePages configuration | `1` (install) |
| `TT_SKIP_UPDATE_FIRMWARE` | Skip tt-flash and firmware update | `1` (install) |
| `TT_SKIP_INSTALL_PODMAN` | Skip Podman installation | `1` (install) |
| `TT_SKIP_INSTALL_METALIUM_CONTAINER` | Skip Metalium container installation | `1` (install) |

Example usage:

`TT_SKIP_INSTALL_KMD=0 ./install.sh`
This would run the installation process but skip the Kernel Mode Driver installation.

Sources: [install.sh 52-67](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L52-L67)[install.sh 415-430](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L415-L430)

## Version Specification Variables

These variables allow you to specify exact versions of components to install. If not specified, the installation script will fetch and install the latest available versions.

| Variable | Description | Default |
| --- | --- | --- |
| `TT_KMD_VERSION` | Kernel Mode Driver version | Latest from GitHub |
| `TT_FW_VERSION` | Firmware version | Latest from GitHub |
| `TT_SYSTOOLS_VERSION` | System tools version | Latest from GitHub |

The default behavior is to fetch the latest version tags from the respective GitHub repositories:

*   KMD: [https://github.com/tenstorrent/tt-kmd.git](https://github.com/tenstorrent/tt-kmd.git)
*   Firmware: [https://github.com/tenstorrent/tt-firmware.git](https://github.com/tenstorrent/tt-firmware.git)
*   System Tools: [https://github.com/tenstorrent/tt-system-tools.git](https://github.com/tenstorrent/tt-system-tools.git)

Example usage:

`TT_KMD_VERSION=2.0.0 TT_FW_VERSION=1.5.0 ./install.sh`
Sources: [install.sh 18-42](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L18-L42)[install.sh 70-74](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L70-L74)

## Installation Mode Variables

These variables control the behavior of the installation process itself.

| Variable | Description | Default Value | Effect |
| --- | --- | --- | --- |
| `TT_MODE_CONTAINER` | Container mode | `1` (disabled) | Set to `0` to enable container mode, which automatically skips KMD, HugePages, Podman, and disables reboot |
| `TT_MODE_NON_INTERACTIVE` | Non-interactive mode | `1` (disabled) | Set to `0` to enable non-interactive mode, which uses default values for all prompts |

When container mode is enabled, the following settings are automatically applied:

*   `SKIP_INSTALL_KMD=0`
*   `SKIP_INSTALL_HUGEPAGES=0`
*   `SKIP_INSTALL_PODMAN=0`
*   `REBOOT_OPTION=2` (never reboot)

Example usage:

`TT_MODE_NON_INTERACTIVE=0 ./install.sh`
Sources: [install.sh 95-113](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L95-L113)

## Python Environment Configuration

These variables control how Python packages are installed during the installation process.

| Variable | Description | Default | Values |
| --- | --- | --- | --- |
| `TT_PYTHON_CHOICE` | Python installation method | `2` | `1` = Use active venv `2` = Create new venv `3` = Use system Python `4` = Use pipx |
| `TT_NEW_VENV_LOCATION` | Location for new Python virtual environment | `${HOME}/.tenstorrent-venv` or `${XDG_DATA_HOME}/tenstorrent-venv` | Valid filesystem path |

## Container Configuration Variables

These variables control the configuration of the Metalium container when installed through Podman.

| Variable | Description | Default Value |
| --- | --- | --- |
| `TT_METALIUM_IMAGE_URL` | URL for the Metalium container image | `ghcr.io/tenstorrent/tt-metal/tt-metalium-ubuntu-22.04-release-amd64` |
| `TT_METALIUM_IMAGE_TAG` | Tag for the Metalium container image | `latest-rc` |

These settings are used to create the wrapper script for running the Metalium container. The wrapper script is installed at `${HOME}/.local/bin/tt-metalium`.

Example usage:

`TT_METALIUM_IMAGE_TAG=stable ./install.sh`
Sources: [install.sh 44-50](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L44-L50)[install.sh 308-354](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L308-L354)

## Reboot Behavior Variables

These variables control how the system handles reboot after installation.

| Variable | Description | Default | Values |
| --- | --- | --- | --- |
| `TT_REBOOT_OPTION` | Reboot behavior after installation | `1` | `1` = Ask the user `2` = Never reboot `3` = Always reboot |

In non-interactive mode (`TT_MODE_NON_INTERACTIVE=0`), if `TT_REBOOT_OPTION` is set to `1`, it will be automatically changed to `2` (never reboot).

Sources: [install.sh 84-91](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L84-L91)[install.sh 105-113](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L105-L113)[install.sh 644-656](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L644-L656)

## Example Usage Scenarios

Here are some common examples of using environment variables to customize the installation:

### Basic Installation with Custom Versions

`TT_KMD_VERSION=2.0.0 TT_FW_VERSION=1.5.0 ./install.sh`
### Skip Specific Components

`TT_SKIP_INSTALL_PODMAN=0 TT_SKIP_INSTALL_METALIUM_CONTAINER=0 ./install.sh`
### Non-Interactive Installation

`TT_MODE_NON_INTERACTIVE=0 ./install.sh`
### Customize Python Environment

`TT_PYTHON_CHOICE=1 ./install.sh  # Use existing active virtual environment``TT_PYTHON_CHOICE=2 TT_NEW_VENV_LOCATION=/opt/tenstorrent-venv ./install.sh  # Create new venv at custom location`
### Always Reboot After Installation

`TT_REBOOT_OPTION=3 ./install.sh`
### Container Mode Installation

`TT_MODE_CONTAINER=0 ./install.sh`
Sources: [install.sh 52-113](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L52-L113)[install.sh 390-656](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L390-L656)

## Default Values and Precedence

All environment variables have default values that are used if the variable is not explicitly set. The precedence order for determining configuration is:

1.   Explicitly set environment variables
2.   Mode-specific defaults (e.g., when container mode is enabled)
3.   User input (in interactive mode)
4.   Built-in default values
