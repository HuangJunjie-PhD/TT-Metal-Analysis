For information about the overall CI/CD architecture and workflow orchestration, see [6.1 CI/CD Architecture Overview](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/6.1%20CI/CD%20Architecture%20Overview) For details on hardware platform selection and runner configuration, see [6.5 Hardware Test Matrix and Runners](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/6.5%20Hardware%20Test%20Matrix%20and%20Runners) For nightly and performance-specific testing strategies, see [6.8 Performance and Profiler Testing](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/6.8%20Performance%20and%20Profiler%20Testing)

## Test Organization Overview

Tests in tt-metal are organized along two primary dimensions: **technology layer** (which components they test) and **test category** (what type of testing they perform). This organization is reflected in both the directory structure and the CI workflow definitions.

### Test Organization by Layer

**Sources:**[.github/workflows/cpp-post-commit.yaml 1-48](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/.github/workflows/cpp-post-commit.yaml#L1-L48)[.github/workflows/ttnn-post-commit.yaml 1-44](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/.github/workflows/ttnn-post-commit.yaml#L1-L44)[.github/workflows/blackhole-post-commit.yaml 1-64](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/.github/workflows/blackhole-post-commit.yaml#L1-L64)

### Test Categories

Tests are further categorized by their purpose and execution context, ranging from low-level UMD driver tests to high-level model demonstrations.

| Category | Description | Workflow Files | Typical Duration |
| --- | --- | --- | --- |
| **Unit Tests (C++)** | Low-level Metal, dispatch, and device tests using Googletest | `cpp-post-commit.yaml` | 15-45 min |
| **Unit Tests (Python)** | TTNN operation and functionality tests using pytest | `ttnn-post-commit.yaml` | 40 min |
| **Model Tests** | Individual model component and integration tests | `models-post-commit.yaml` | 20-35 min |
| **Demo Tests** | End-to-end model demonstrations with accuracy verification | `blackhole-demo-tests-impl.yaml` | 30-60 min per model |
| **Performance Tests** | Throughput and latency regression testing | `perf-device-models-impl.yaml` | 30-120 min |
| **Nightly L2 Tests** | Long-running stress and comprehensive op coverage | `tt-metal-l2-nightly.yaml` | 150+ min |
| **Upstream Tests** | Production-grade validation for release-candidate images | `upstream-tests.yaml` | Variable |
