This page documents the development build process for tt-flash, covering local compilation, virtual environment setup, and build target configuration. It is intended for developers who want to modify tt-flash or contribute to its development.

For information about installing pre-built packages, see [Installation](https://deepwiki.com/tenstorrent/tt-flash/2.1-installation). For the release and distribution packaging process, see [Debian Packaging](https://deepwiki.com/tenstorrent/tt-flash/9.2-debian-packaging) and [PyPI Distribution](https://deepwiki.com/tenstorrent/tt-flash/9.3-pypi-distribution).

## Overview

tt-flash uses a standard Python packaging workflow with PEP 517/518 compliance via `pyproject.toml` and a compatibility layer (`setup.py`) for older Ubuntu packaging tools. The build process supports both quick installation for testing changes and full development environments with code quality tooling.

**Sources:**[README.md 46-63](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L46-L63)[setup.py 1-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L1-L26)

## Prerequisites

### Rust Toolchain

tt-flash depends on `pyluwen`, which includes Rust components. Rust must be installed before building:

**Distribution Package Installation (Preferred):**

*   Fedora/EL9: `sudo dnf install cargo`
*   Ubuntu/Debian: `sudo apt install cargo`

**Rustup Installation:**

### Python Requirements

*   Python 3.10 or higher
*   `pip` package manager (upgrade recommended)
*   `venv` module for virtual environment creation

## Configuration Files

### pyproject.toml Structure

The authoritative build configuration following PEP 517/518 standards:

| Section | Purpose | Key Elements |
| --- | --- | --- |
| `[project]` | Package metadata | name, version, Python requirements |
| `[project.dependencies]` | Runtime dependencies | `pyluwen~=0.8.0`, `tt_tools_common~=1.5`, `pyyaml==6.0.2`, `tabulate==0.9.0`, `tomli==2.0.1` |
| `[project.optional-dependencies]` | Development tools | `[dev]`: `black`, `basedpyright` |
| `[project.scripts]` | Console entry points | `tt-flash = tt_flash:main` |
| `[build-system]` | Build backend | `setuptools >= 43.0.0`, `wheel` |

**Version Constraints:**

*   `~=0.8.0`: Compatible release (0.8.x), allows patch updates
*   `==6.0.2`: Exact version pinning for reproducibility

**Sources:** Referenced in diagrams, [setup.py 11-12](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L11-L12)

### setup.py: Compatibility Layer

Provides backward compatibility for Ubuntu 22.04's older packaging tools that do not fully support PEP 517:

**Key functions:**

1.   Reads `pyproject.toml` using `tomli`[setup.py 11-12](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L11-L12)
2.   Extracts version dynamically [setup.py 17](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L17-L17)
3.   Uses `find_packages()` for automatic package discovery [setup.py 18](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L18-L18)
4.   Defines console script entry point [setup.py 20-24](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L20-L24)

**Sources:**[setup.py 1-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L1-L26)

## Dependency Resolution

### Runtime Dependencies Installation Flow

**Installation order:**

1.   `setuptools` and `wheel` (build backend)
2.   `tomli` (required by setup.py)
3.   Tenstorrent ecosystem packages (`pyluwen`, `tt_tools_common`)
4.   Python utility libraries (`pyyaml`, `tabulate`)
5.   Development tools (if `[dev]` extras specified)

## Troubleshooting

### Common Build Issues

| Issue | Cause | Solution |
| --- | --- | --- |
| `cargo not found` | Rust not installed | Install Rust using distribution packages or rustup (see Prerequisites) |
| `Python version unsupported` | Python < 3.10 | Upgrade to Python 3.10+ |
| `pyluwen build failure` | Missing Rust components | Ensure `cargo` is in PATH and functioning |
| `tomli import error` | Missing in Python 3.11+ | `tomli` only needed for Python < 3.11; check Python version |
| `module 'tt_flash' not found` | Not installed or venv not activated | Activate virtual environment and verify installation |

### Verifying Rust Installation

Both commands should return version information. If not, ensure Rust is installed and `$HOME/.cargo/bin` is in your PATH.

**Sources:**[README.md 16-26](https://github.com/tenstorrent/tt-flash/blob/60c06032/README.md?plain=1#L16-L26)

## Package Structure After Build

Post-installation, the package structure in `site-packages/` (or source directory for editable installs):

```
tt_flash/
├── __init__.py
├── main.py          # CLI entry point
├── flash.py         # Flashing engine
├── chip.py          # Hardware abstraction
├── utility.py       # Helper functions
└── ...
```

The `tt-flash` console script is installed to the virtual environment's `bin/` directory (or system `bin/` for system-wide installs) and invokes `tt_flash:main`.
