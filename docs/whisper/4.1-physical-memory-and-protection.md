## Purpose and Scope

This document describes Whisper's physical memory model and Physical Memory Attributes (PMA) system. It covers the `Memory` class which provides the backing storage for the simulated system, the `PmaManager` which controls access attributes for memory regions, memory-mapped register support, and file loading capabilities.

For virtual memory and address translation, see [Virtual Memory and Address Translation](https://deepwiki.com/tenstorrent/whisper/4.2-virtual-memory-and-address-translation). For Physical Memory Protection (PMP) which provides per-hart runtime protection, see [Control and Status Registers (CSRs)](https://deepwiki.com/tenstorrent/whisper/3.1-control-and-status-registers-(csrs)) as PMP is managed through CSRs at the hart level.

## Physical Memory Attributes (PMA)

Physical Memory Attributes define the access characteristics of memory regions. Unlike Physical Memory Protection (PMP) which is per-hart and runtime-configurable, PMAs are system-wide and typically static.

### PMA Attribute Types

The `Pma` class ([PmaManager.hpp 28-148](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.hpp#L28-L148)) defines attributes as bit flags:

| Attribute | Value | Description |
| --- | --- | --- |
| `None` | 0x0 | No access allowed |
| `Read` | 0x1 | Load instructions allowed |
| `Write` | 0x2 | Store instructions allowed |
| `Exec` | 0x4 | Instruction fetch allowed |
| `Idempotent` | 0x8 | Non-I/O, reads have no side effects |
| `AmoSwap` | 0x20 | AMO swap operations allowed |
| `AmoLogical` | 0x40 | AMO logical operations allowed |
| `AmoOther` | 0x10 | AMO arithmetic operations allowed |
| `MemMapped` | 0x200 | Contains memory-mapped registers |
| `Rsrv` | 0x400 | LR/SC instructions allowed |
| `Io` | 0x800 | I/O region (non-idempotent) |
| `Cacheable` | 0x1000 | Cacheable region |
| `MisalOk` | 0x2000 | Misaligned access supported |
| `MisalAccFault` | 0x4000 | Misaligned access generates access fault |

**Default PMA:**`Read | Write | Exec | Idempotent | Amo | Rsrv | MisalOk`

**Sources:**[PmaManager.hpp 34-47](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.hpp#L34-L47)[PmaManager.cpp 24-77](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.cpp#L24-L77)

## Memory-Mapped Registers

Memory-mapped registers (MMRs) are special locations that appear at fixed addresses but have restricted access patterns and apply bit masks on writes.

### MMR Architecture

**Sources:**[PmaManager.hpp 263-280](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.hpp#L263-L280)[PmaManager.cpp 89-100](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.cpp#L89-L100)[Memory.hpp 112-114](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.hpp#L112-L114)[Memory.hpp 242-243](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.hpp#L242-L243)

### MMR Definition and Access

**Definition:**

Requirements:

*   `size` must be 4 or 8 bytes
*   `addr` must be aligned to `size`
*   Writes are masked: `value = (value & ~mask) | (new_value & mask)`

**Read Operations** ([PmaManager.cpp 136-231](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.cpp#L136-L231)):

*   Support 1, 2, 4, and 8-byte reads
*   Return current register value
*   For 8-byte reads from 4-byte registers, can combine two adjacent registers

**Write Operations** ([PmaManager.cpp 235-354](https://github.com/tenstorrent/whisper/blob/733fd921/PmaManager.cpp#L235-L354)):

*   Support 1, 2, 4, and 8-byte writes
*   Apply mask: `mmrv = (mmrv & ~byte_mask & ~mmrm) | (shifted_value & mmrm)`
*   Read-only bits (mask=0) are preserved
*   For 8-byte writes to 4-byte registers, writes to both if present

### ELF File Loading

The `loadElfFile()` method ([Memory.cpp 603-690](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L603-L690)) loads ELF executables:

**Process:**

1.   Parse ELF file using ELFIO library
2.   Validate ELF class (32/64-bit) matches register width
3.   Validate machine type is `EM_RISCV`
4.   Load all `PT_LOAD` segments into memory
5.   Collect symbol table entries
6.   Collect section information
7.   Return entry point and end address

**Key Features:**

*   Supports ELF32 and ELF64
*   Collects symbols for debugging ([Memory.cpp 555-586](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L555-L586))
*   Collects section addresses ([Memory.cpp 590-599](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L590-L599))
*   Can extract RISC-V architecture tags ([Memory.cpp 437-551](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L437-L551))
*   Validates against unmapped memory if `checkUnmappedElf_` enabled

**ELF Segment Loading** ([Memory.cpp 366-406](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L366-L406)):

### Hex File Format

The hex file format ([Memory.cpp 93-199](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L93-L199)) uses:

*   `@XXXXXXXX` to set current address (hexadecimal)
*   Space-separated hex bytes (00-FF)
*   Example: 
```
@80000000
37 05 00 00 13 05 85 00
@80001000
93 08 00 00
```

### Binary File Loading

Raw binary files ([Memory.cpp 203-256](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L203-L256)) are loaded at a specified offset:

*   No format parsing required
*   Loaded byte-by-byte at sequential addresses
*   Used for simple payloads or specific memory regions

### LZ4 Compressed Loading

For large images, LZ4 compression support ([Memory.cpp 281-363](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L281-L363)) provides:

*   Streaming decompression using LZ4 frame API
*   Page-aligned optimization: full pages written directly if all non-zero
*   Significantly faster loading for sparse memory images

**Sources:**[Memory.cpp 281-363](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L281-L363)

## Configuration via JSON

PMA regions and memory-mapped registers can be configured through JSON configuration files:

### Memory Region Configuration

**JSON Schema:**

**Attribute Names:**

*   `read`, `write`, `exec`: Basic access permissions
*   `idempotent`: Non-I/O memory
*   `amo`: Atomic operations (swap, logical, arithmetic)
*   `rsrv`: LR/SC reservation support
*   `io`: I/O region (non-idempotent)
*   `cacheable`: Cacheable memory
*   `misal_ok`: Misaligned access allowed
*   `misal_acc_fault`: Misaligned generates access fault

**Processing** ([HartConfig.cpp 1403-1639](https://github.com/tenstorrent/whisper/blob/733fd921/HartConfig.cpp#L1403-L1639)):

1.   Parse `memmap` array from configuration
2.   For each entry: 
    *   Extract address and size
    *   Parse attribute strings into `Pma` object
    *   Call `hart.defineMemoryRegion()`
    *   Process optional `masks` array for memory-mapped registers

3.   Update PMA region's `MemMapped` attribute if it overlaps any MMR

**Constraints:**

*   I/O regions: Cannot have write without read, cannot have AMO operations

## Snapshot Support

Memory state can be saved and restored for checkpointing:

### Snapshot Operations

**Compression Formats:**

*   **gzip**: `saveSnapshot_gzip()` / `loadSnapshot_gzip()` ([Memory.cpp 1039-1118](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L1039-L1118)[Memory.cpp 1121-1188](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L1121-L1188))
*   **lz4**: `saveSnapshot_lz4()` / `loadSnapshot_lz4()` (if `LZ4_COMPRESS` enabled)

**Used Blocks:** Only memory that has been accessed is saved, tracked via:

*   `SparseMem::getUsedBlocks()` (sparse mode)
*   `Syscall::getUsedMemBlocks()` (flat mode, estimates from stack pointer)

**System Integration** ([System.cpp 558-698](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L558-L698)): The `System::saveSnapshot()` method coordinates saving:

1.   Disable I/O devices
2.   Save registers for each hart
3.   Save used memory blocks list
4.   Save memory contents (compressed)
5.   Save timer/interrupt state
6.   Save file descriptors and mmap state
7.   Save address traces
8.   Re-enable I/O devices

**Sources:**[Memory.cpp 1039-1188](https://github.com/tenstorrent/whisper/blob/733fd921/Memory.cpp#L1039-L1188)[System.cpp 558-698](https://github.com/tenstorrent/whisper/blob/733fd921/System.cpp#L558-L698)
