The `tt-inference-server` provides a comprehensive ecosystem for deploying diverse model architectures across the Tenstorrent hardware lineup. This page catalogs the supported model types, the hardware configurations they run on, and the technical implementation details that enable cross-platform compatibility.

## Supported Model Modalities

The server supports a wide range of modalities, categorized by `ModelType`[scripts/release/generate_model_support_docs.py 131-140](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/release/generate_model_support_docs.py#L131-L140) Each type is associated with specific inference engines, primarily `vLLM` for text-based models, `MEDIA` (tt-media-server) for vision and audio tasks, and `FORGE` for CNN-based workloads [scripts/release/generate_model_support_docs.py 67-71](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/release/generate_model_support_docs.py#L67-L71)

## Implementation Architecture

The relationship between models and hardware is defined in the `MODEL_SPECS` registry and expanded into the `release_model_spec.json` file [release_model_spec.json 1-4](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/release_model_spec.json#L1-L4) This registry maps a model implementation (`ImplSpec`) to specific hardware constraints (`DeviceModelSpec`), environment variables, and inference engine arguments.

### Hardware-Specific Configurations

Each hardware target requires specific environment variables and mesh descriptors to function correctly. These are defined within the `device_model_spec` block of a model [release_model_spec.json 21-87](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/release_model_spec.json#L21-L87)

*   **T3K / Galaxy:** Often requires `WH_ARCH_YAML` set to `wormhole_b0_80_arch_eth_dispatch.yaml`[release_model_spec.json 95](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/release_model_spec.json#L95-L95) and `MESH_DEVICE` set to `T3K`[release_model_spec.json 96](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/release_model_spec.json#L96-L96)
*   **Galaxy T3K:** Additionally requires `TT_MESH_GRAPH_DESC_PATH` to point to the mesh graph descriptor [release_model_spec.json 160](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/release_model_spec.json#L160-L160)
*   **Blackhole (P-Series):** Uses specific Docker images and device mappings for `p150x4` or `p150x8` configurations [docs/model_support/llm/Llama-3.1-8B_p150x4.md 31-32](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/docs/model_support/llm/Llama-3.1-8B_p150x4.md?plain=1#L31-L32)

## Model Readiness and Status

Models in the registry are assigned a `ModelStatusTypes`[scripts/release/generate_model_support_docs.py 44](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/release/generate_model_support_docs.py#L44-L44) which indicates their stability and performance level:

1.   **TOP_PERF (🟢):** Highly optimized, meets all performance targets.
2.   **COMPLETE (🟢):** Fully functional and validated for accuracy [docs/model_support/cnn/vovnet_n150.md 27](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/docs/model_support/cnn/vovnet_n150.md?plain=1#L27-L27)
3.   **FUNCTIONAL (🟡):** Runs correctly but may lack performance optimizations [docs/model_support/cnn/segformer_n150.md 27](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/docs/model_support/cnn/segformer_n150.md?plain=1#L27-L27)
4.   **EXPERIMENTAL (🛠️):** Work-in-progress, may have stability or accuracy issues [docs/model_support/llm/DeepSeek-R1-0528_galaxy.md 47](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/docs/model_support/llm/DeepSeek-R1-0528_galaxy.md?plain=1#L47-L47)
