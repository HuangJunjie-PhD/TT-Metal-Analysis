## Purpose and Scope

The `pyluwen` module provides Python bindings for the Luwen library, enabling Python applications to interact with Tenstorrent AI accelerator chips. It exposes a comprehensive Python API that wraps the core functionality of Luwen, allowing for chip detection, initialization, and communication through various interfaces like NOC, AXI, and ARC messaging.

For information about the C++ bindings, see [C++ (luwencpp)](https://deepwiki.com/tenstorrent/luwen/5.2-c++-(luwencpp)).

## Architecture Overview

The `pyluwen` module bridges Python applications with the Luwen core functionality, providing wrapped access to chip operations and data structures. It utilizes PyO3 to create Python bindings for the Rust implementation.

## Core Functionality

### Chip Detection and Initialization

The module provides two main functions for chip detection:

1.   `detect_chips()` - Detects and automatically initializes all available chips
2.   `detect_chips_fallible()` - Detects chips but returns them in an uninitialized state

Sources: [crates/pyluwen/src/lib.rs 1405-1549](https://github.com/tenstorrent/luwen/blob/55e35616/crates/pyluwen/src/lib.rs#L1405-L1549)[crates/pyluwen/src/lib.rs 578-614](https://github.com/tenstorrent/luwen/blob/55e35616/crates/pyluwen/src/lib.rs#L578-L614)

### Communication Operations

The module provides various communication mechanisms for interacting with the chips:

#### NOC Operations

*   `noc_read()`/`noc_write()` - Read/write data through Network on Chip
*   `noc_read32()`/`noc_write32()` - Convenience methods for 32-bit data
*   `noc_broadcast()`/`noc_broadcast32()` - Broadcast to multiple cores

#### AXI Operations

*   `axi_read()`/`axi_write()` - Read/write data through AXI bus
*   `axi_read32()`/`axi_write32()` - Convenience methods for 32-bit data
*   `axi_translate()` - Translate named address to physical address

#### ARC Messaging

*   `arc_msg()` - Send message to ARC processors on the chip

Sources: [crates/pyluwen/src/lib.rs 322-475](https://github.com/tenstorrent/luwen/blob/55e35616/crates/pyluwen/src/lib.rs#L322-L475)

### Memory and DMA Operations

For efficient data transfer, the module provides DMA capabilities:

*   `allocate_dma_buffer()` - Allocate DMA buffer
*   `config_dma()` - Configure DMA settings
*   `dma_transfer_turbo()` - Perform high-speed DMA transfer
*   `setup_tlb()` - Set up Translation Lookaside Buffer entries

Sources: [crates/pyluwen/src/lib.rs 90-102](https://github.com/tenstorrent/luwen/blob/55e35616/crates/pyluwen/src/lib.rs#L90-L102)[crates/pyluwen/src/lib.rs 993-1055](https://github.com/tenstorrent/luwen/blob/55e35616/crates/pyluwen/src/lib.rs#L993-L1055)

### SPI Flash Operations

The module provides direct access to the SPI flash memory on the chips:

*   `spi_read()`/`spi_write()` - Read/write data from/to SPI flash
*   `decode_boot_fs_table()` - Decode boot file system table
*   `get_spirom_table_spi_addr()` - Get SPI address for a specified table

## Implementation Details

The `pyluwen` module is implemented using the PyO3 Rust crate, which provides the interface between Python and Rust. Each Python class is a thin wrapper around the corresponding Rust type, using the `Deref` and `DerefMut` traits to delegate functionality.

The implementation follows a pattern where common functionality for chip communication is implemented once in a macro (`common_chip_comms_impls!`) and then applied to each chip class.
