## Purpose and Scope

This document covers vLLM's Mixture of Experts (MoE) implementation, which enables efficient execution of sparse MoE models where only a subset of experts are activated per token. The implementation focuses on the `FusedMoE` layer and its associated quantization methods, expert routing strategies, and platform-specific backend optimizations.

For general quantization concepts beyond MoE, see [Quantization Strategies](https://deepwiki.com/tenstorrent/vllm/3.4-quantization-strategies). For attention mechanisms, see [Attention Mechanisms](https://deepwiki.com/tenstorrent/vllm/2.6-attention-mechanisms). For parallel execution strategies, see [Distributed Execution](https://deepwiki.com/tenstorrent/vllm/6.3-distributed-execution).

## Architecture Overview

vLLM's MoE system is built around the `FusedMoE` layer, which combines expert selection, weight computation, and result aggregation into optimized kernels. The architecture supports multiple quantization schemes, platform-specific backends, and various forms of parallelism.

## Core Components

### FusedMoE Layer

The `FusedMoE` class is the main entry point for MoE computation. It manages expert weights, routing logic, and delegates to quantization-specific methods.

**Key Responsibilities:**

*   **Weight Storage**: Maintains `w13_weight` (gate/up projection, fused) and `w2_weight` (down projection)
*   **Expert Selection**: Calls `select_experts()` to determine top-K experts per token
*   **Quantization Dispatch**: Delegates computation to quantization-specific methods

## Expert Routing and Selection

### Top-K Expert Selection

The `select_experts()` method implements expert routing with multiple strategies:

| Parameter | Description | Options |
| --- | --- | --- |
| `top_k` | Number of experts per token | 1-8 (typically 1-2) |
| `use_grouped_topk` | Use grouped expert selection | `True`/`False` |
| `renormalize` | Renormalize routing weights | `True`/`False` |
| `scoring_func` | Routing score function | `"softmax"`, `"sigmoid"` |
| `custom_routing_function` | Custom routing logic | `Callable` or `None` |

**Expert Parallelism Load Balancing (EPLB):** When `enable_eplb=True`, the system maintains dynamic load tracking and maps logical expert IDs to physical replicas to balance load across ranks.

**Sources:**[vllm/model_executor/layers/fused_moe/layer.py 911-1035](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/fused_moe/layer.py#L911-L1035)[vllm/model_executor/layers/fused_moe/fused_moe.py 78-156](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/fused_moe/fused_moe.py#L78-L156)

### Quantization Method Hierarchy

**Sources:**[vllm/model_executor/layers/fused_moe/layer.py 94-288](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/fused_moe/layer.py#L94-L288)[vllm/model_executor/layers/quantization/fp8.py 433-803](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/quantization/fp8.py#L433-L803)

### FP8 Quantization Details

FP8 MoE uses `float8_e4m3fn` for weights and activations with per-tensor or block-wise scaling:

| Scale Type | Weight Scales | Activation Scales | Block Shape |
| --- | --- | --- | --- |
| Per-tensor | `(num_experts,)` | `(num_experts,)` | `None` |
| Per-channel | `(num_experts, N)` | `(num_tokens,)` | `None` |
| Block-wise | `(num_experts, N/bn, K/bk)` | Per-token or per-block | `[bn, bk]` |

**Weight Storage:**

*   `w13_weight`: Shape `(num_experts, 2*intermediate_size, hidden_size)` - fused gate and up projections
*   `w2_weight`: Shape `(num_experts, hidden_size, intermediate_size)` - down projection
*   `w13_weight_scale`, `w2_weight_scale`: Per-tensor or block scales
*   `w13_input_scale`, `w2_input_scale`: Activation scales (static only)

**Sources:**[vllm/model_executor/layers/quantization/fp8.py 505-632](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/quantization/fp8.py#L505-L632)[vllm/model_executor/layers/fused_moe/config.py 1-180](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/fused_moe/config.py#L1-L180)

## Weight Loading and Initialization

### Weight Creation

Each quantization method implements `create_weights()` to register parameters:

`def create_weights(self, layer: torch.nn.Module, num_experts: int,                   hidden_size: int, intermediate_size_per_partition: int,                   params_dtype: torch.dtype, **extra_weight_attrs):    # Register w13_weight (fused gate_up_proj)    w13_weight = torch.nn.Parameter(torch.empty(        num_experts,        2 * intermediate_size_per_partition,        hidden_size,        dtype=params_dtype), requires_grad=False)    layer.register_parameter("w13_weight", w13_weight)    set_weight_attrs(w13_weight, extra_weight_attrs)        # Register w2_weight (down_proj)    w2_weight = torch.nn.Parameter(torch.empty(        num_experts,        hidden_size,        intermediate_size_per_partition,        dtype=params_dtype), requires_grad=False)    layer.register_parameter("w2_weight", w2_weight)    set_weight_attrs(w2_weight, extra_weight_attrs)`
**Weight Scale Attributes:** For proper loading, scales must have a `quant_method` attribute:

`extra_weight_attrs.update({    "quant_method": FusedMoeWeightScaleSupported.TENSOR.value    # or CHANNEL, GROUP, BLOCK})`
**Sources:**[vllm/model_executor/layers/fused_moe/layer.py 362-398](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/fused_moe/layer.py#L362-L398)[vllm/model_executor/layers/quantization/fp8.py 505-632](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/quantization/fp8.py#L505-L632)

### Post-Loading Processing

After weights are loaded, `process_weights_after_loading()` performs transformations:

**FP8 Example:**

`def process_weights_after_loading(self, layer: torch.nn.Module) -> None:    # Combine w1 and w3 scales to single per-expert scale    if layer.w13_weight_scale.dim() == 2:        max_w13_scales = layer.w13_weight_scale.max(dim=1).values                # Dequantize and requantize with unified scale        for expert_id in range(layer.num_experts):            # Dequantize            w1 = per_tensor_dequantize(                layer.w13_weight[expert_id, :intermediate_size],                layer.w13_weight_scale[expert_id, 0])            w3 = per_tensor_dequantize(                layer.w13_weight[expert_id, intermediate_size:],                layer.w13_weight_scale[expert_id, 1])                        # Requantize with max scale            qweight, _ = scaled_fp8_quant(                torch.cat([w1, w3]), scale=max_w13_scales[expert_id])            layer.w13_weight[expert_id] = qweight`
**Platform-Specific Processing:**

*   **ROCm**: Shuffle weights for AIter backend
*   **FlashInfer**: Swap w1/w3 halves to [w3, w1] layout
*   **Marlin**: Permute weights for Marlin kernel layout
*   **DeepGemm**: Transpose and align scales for TMA

**Sources:**[vllm/model_executor/layers/quantization/fp8.py 633-772](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/quantization/fp8.py#L633-L772)[vllm/model_executor/layers/fused_moe/layer.py 411-467](https://github.com/tenstorrent/vllm/blob/cd9e5b83/vllm/model_executor/layers/fused_moe/layer.py#L411-L467)

## Example: MoE Model Integration

### Model Definition

`class DecoderLayer(nn.Module):    def __init__(self, config: ModelConfig, vllm_config: VllmConfig):        self.self_attn = Attention(...)                # MoE Block        self.block_sparse_moe = FusedMoE(            num_experts=config.num_local_experts,            top_k=config.num_experts_per_tok,            hidden_size=config.hidden_size,            intermediate_size=config.intermediate_size,            reduce_results=True,            renormalize=True,            quant_config=vllm_config.quant_config,            prefix=f"{prefix}.block_sparse_moe",        )                # Router        self.router = torch.nn.Linear(            config.hidden_size,            config.num_local_experts,            bias=False        )        def forward(self, hidden_states, positions):        # Self-attention        hidden_states = self.self_attn(hidden_states, positions)                # MoE forward        router_logits = self.router(hidden_states)        hidden_states = self.block_sparse_moe(            hidden_states=hidden_states,            router_logits=router_logits,        )                return hidden_states`
