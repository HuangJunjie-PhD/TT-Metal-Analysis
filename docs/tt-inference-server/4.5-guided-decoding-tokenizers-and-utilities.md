This page describes the auxiliary C++ subsystems of the `tt-media-server` backend. These components handle structured output generation, text-to-token conversion, reasoning-model parsing, and system-wide infrastructure such as telemetry and thread management.

## Guided Decoding with xgrammar

The `GuidedDecoderManager` provides structured generation capabilities (e.g., JSON mode, JSON Schema enforcement) by integrating with the `xgrammar` library. It constrains the LLM's output by generating bitmasks that filter out invalid tokens during the sampling process.

### Implementation and Data Flow

The manager operates as a lifecycle handler for guided requests, maintaining a `GrammarMatcher` for each active task [tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp 19-23](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp#L19-L23)

1.   **Initialization**: When a request with a `ResponseFormatType` of `JSON_OBJECT` or `JSON_SCHEMA` arrives, `initRequest` compiles the grammar using `xgrammar::GrammarCompiler` and initializes a `GrammarMatcher`[tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp 38-60](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp#L38-L60)
2.   **Bitmask Generation**: Before each sampling step, the LLM engine calls `fillNextBitmask`. The manager uses the `GrammarMatcher` to populate a bitmask where each bit represents whether a token in the vocabulary is valid according to the grammar [tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp 62-80](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp#L62-L80)
3.   **Token Acceptance**: After the model selects a token, `acceptToken` updates the internal state of the matcher. It also detects if the grammar has reached a terminal state (completion) [tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp 86-95](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/guided_decoder_manager.cpp#L86-L95)

## Reasoning Parser

The `ReasoningParser` is a specialized utility for handling "Chain of Thought" models (like DeepSeek-R1) that emit reasoning content inside `<think>` markers [tt-media-server/cpp_server/src/services/reasoning_parser.cpp 101-142](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/reasoning_parser.cpp#L101-L142)

*   **Streaming Support**: In `processToken`, it tracks state across multiple calls to ensure that the start/end markers themselves are not emitted to the final user output while correctly tagging the interior text [tt-media-server/cpp_server/src/services/reasoning_parser.cpp 116-132](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/reasoning_parser.cpp#L116-L132)
*   **Post-processing**: `parseComplete` can take a full block of text and split it into a `ReasoningParseResult` containing the separated reasoning and answer strings [tt-media-server/cpp_server/src/services/reasoning_parser.cpp 14-88](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/reasoning_parser.cpp#L14-L88)
*   **DeepSeek Formatting**: The server supports a specific markdown format for DeepSeek reasoning via the `USE_DEEPSEEK_MD_FORMAT` setting [tt-media-server/cpp_server/src/config/settings.cpp 183-186](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/config/settings.cpp#L183-L186)

## Tokenizers

The server provides a unified `Tokenizer` interface with specialized implementations for different model architectures.

### Implementation Hierarchy

*   **`Tokenizer` (Base)**: Abstract interface defining `encode`, `decode`, and `isLoaded`[tt-media-server/cpp_server/include/utils/tokenizers/tokenizer.hpp 12-20](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/utils/tokenizers/tokenizer.hpp#L12-L20)
*   **`LlamaTokenizer`**: Implementation for Llama-based models. It supports complex chat templates including tool calling environment headers and system preambles [tt-media-server/cpp_server/src/utils/tokenizers/llama_tokenizer.cpp 18-127](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/utils/tokenizers/llama_tokenizer.cpp#L18-L127)
*   **`DeepSeekTokenizer`**: Specialized implementation for DeepSeek models. It handles specific special tags like `<｜User｜>`, `<｜Assistant｜>`, and tool call markers [tt-media-server/cpp_server/src/utils/tokenizers/deepseek_tokenizer.cpp 11-163](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/utils/tokenizers/deepseek_tokenizer.cpp#L11-L163)

### Tokenizer Configuration

The server resolves tokenizer paths dynamically relative to the executable path [tt-media-server/cpp_server/src/config/settings.cpp 111-134](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/config/settings.cpp#L111-L134)

*   **`tokenizerPath()`**: Returns the absolute path to `tokenizer.json` for the active model [tt-media-server/cpp_server/src/config/settings.cpp 125-134](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/config/settings.cpp#L125-L134)
*   **`tokenizerConfigPath()`**: Returns the path to `tokenizer_config.json`[tt-media-server/cpp_server/src/config/settings.cpp 138-147](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/config/settings.cpp#L138-L147)

## Prometheus Metrics and Telemetry

The `ServerMetrics` singleton manages real-time performance tracking using the Prometheus C++ library.

### Architecture

To minimize impact on the inference critical path, `ServerMetrics` uses a background thread and an event-based queue:

1.   **Caller Thread**: LLM service threads push lightweight event structs (e.g., `EventToken`, `EventRequestSubmitted`) into a `MetricsEvent` queue [tt-media-server/cpp_server/include/metrics/metrics.hpp 116-132](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/metrics/metrics.hpp#L116-L132)
2.   **Background Thread**: The `metricsLoop` consumes these events, updates internal `RequestContext` timers, and modifies Prometheus objects [tt-media-server/cpp_server/src/metrics/metrics.cpp 187-210](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/metrics/metrics.cpp#L187-L210)

## Infrastructure Utilities

### ThreadPool

The `tt::utils::ThreadPool` provides a fixed-size pool of worker threads to execute asynchronous tasks. It uses a `std::queue` for task management and is instrumented for performance profiling [tt-media-server/cpp_server/include/utils/thread_pool.hpp 17-65](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/utils/thread_pool.hpp#L17-L65)

### Logger

The server utilizes a centralized logging system (defined in `utils/logger.hpp`) supporting multiple levels: `TT_LOG_DEBUG`, `TT_LOG_INFO`, `TT_LOG_WARN`, and `TT_LOG_ERROR`[tt-media-server/cpp_server/include/utils/logger.hpp 15-30](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/utils/logger.hpp#L15-L30)

### Concurrent Collections and IPC

The backend utilizes thread-safe primitives for inter-thread and inter-process communication:

*   **Bounded Event Queues**: Used in the metrics system to prevent memory exhaustion if the background thread lags, with a limit of 65,536 events [tt-media-server/cpp_server/include/metrics/metrics.hpp 137](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/metrics/metrics.hpp#L137-L137)
*   **Shared Memory Metrics**: POSIX shared-memory segment name is resolved via `workerMetricsShmName()` to allow the main process to monitor worker health [tt-media-server/cpp_server/src/config/settings.cpp 212-214](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/config/settings.cpp#L212-L214)
*   **Queue Naming**: Environment-based configuration for `TT_TASK_QUEUE`, `TT_RESULT_QUEUE`, and memory management queues used by the scheduler [tt-media-server/cpp_server/src/config/settings.cpp 189-210](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/config/settings.cpp#L189-L210)
