This page provides a comprehensive technical catalog of the top-level GitHub workflow files located in `.github/workflows/`. These workflows serve as the entry points for the repository's CI/CD operations, data collection pipelines, utility testing, and community management.

## Data Collection & Testing Workflows

These workflows orchestrate the extraction of telemetry and test results from across the Tenstorrent organization.

### `produce_data.yml`

This is the primary production entry point for the `collect_data` action. It is typically triggered by other repositories to process their CI results. It executes the logic to fetch workflow metadata, download test artifacts, and upload processed JSON/JSONL data to SFTP servers.

### `test_collect_data_action.yml`

This workflow provides continuous integration for the `collect_data` action itself. It ensures that any changes to the data processing logic, Pydantic models, or parsers do not break existing functionality.

*   **Triggers:**`pull_request` affecting `.github/actions/collect_data/**`, `workflow_dispatch`, and `workflow_call`[.github/workflows/test_collect_data_action.yml 2-8](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_collect_data_action.yml#L2-L8)
*   **Implementation:**
    1.   Sets up a Python 3.10 environment [.github/workflows/test_collect_data_action.yml 17-19](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_collect_data_action.yml#L17-L19)
    2.   Installs dependencies from `requirements.txt`, including `pydantic`, `deepdiff`, and `xmltodict`[.github/actions/collect_data/requirements.txt 1-9](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/collect_data/requirements.txt#L1-L9)
    3.   Executes `pytest` with coverage reporting [.github/workflows/test_collect_data_action.yml 27-31](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_collect_data_action.yml#L27-L31)
    4.   Uses `MishaKav/pytest-coverage-comment` to post results to PRs [.github/workflows/test_collect_data_action.yml 34-38](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_collect_data_action.yml#L34-L38)

**Sources:**[.github/workflows/test_collect_data_action.yml 1-39](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_collect_data_action.yml#L1-L39)[.github/actions/collect_data/requirements.txt 1-9](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/collect_data/requirements.txt#L1-L9)

## Utility & System Workflows

These workflows validate and test the repository's standalone utility actions.

### `test_maximize_space.yml`

Validates the `maximize_space` action, which reclaims disk space on GitHub-hosted runners by purging pre-installed software like PyPy, CodeQL, and Android SDKs [.github/actions/maximize_space/action.yml 21-37](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/action.yml#L21-L37)

*   **Triggers:** PRs targeting `main` with changes in the action's directory [.github/workflows/test_maximize_space.yml 5-8](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_maximize_space.yml#L5-L8)
*   **Execution Modes:**
    1.   **Containerized:** Runs inside a `tt-mlir-ci` Ubuntu image to verify behavior in restricted environments [.github/workflows/test_maximize_space.yml 12-15](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_maximize_space.yml#L12-L15)
    2.   **Host-based:** Runs directly on `ubuntu-latest`[.github/workflows/test_maximize_space.yml 26-27](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_maximize_space.yml#L26-L27)

**Sources:**[.github/workflows/test_maximize_space.yml 1-37](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_maximize_space.yml#L1-L37)[.github/actions/maximize_space/action.yml 1-41](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/action.yml#L1-L41)

### `test_show_telemetry.yml`

Tests the `show_telemetry` action by generating dummy load and verifying that resource usage (CPU, RAM, Disk) is correctly captured and displayed in the GitHub Step Summary.

### `pre-commit.yml`

Runs `pre-commit` hooks across the repository to enforce linting, formatting, and `spdx-checker` compliance.

| `bounty-stale.yml` | `workflow_call` | Marks assigned bounty issues as stale after 14 days of inactivity [.github/workflows/bounty-stale.yml 20-29](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/bounty-stale.yml#L20-L29) |
