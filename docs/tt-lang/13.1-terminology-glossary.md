## Purpose and Scope

This page provides comprehensive definitions of terminology used throughout the `tt-lang` codebase, covering Python DSL constructs, MLIR infrastructure, hardware-specific architecture, and compilation pipeline components. It serves as a technical reference for developers working on the compiler or writing high-performance kernels.

## Programming Model Terms

### Threads and Processors

*   **BRISC / NCRISC:** The two primary data movement RISC processors on a Tensix core. In `tt-lang`, these are targeted by the `@ttl.datamovement` decorator.
*   **TRISC:** The three compute RISC processors (TRISC0, TRISC1, TRISC2) that manage the MATH engine. These execute code defined under the `@ttl.compute` decorator.
*   **SPMD (Single Program Multiple Data):** An execution mode where the same kernel code runs on every core in the grid, but operates on different data partitions based on its logical coordinates.
*   **SFPU (Special Floating Point Unit):** Hardware units used for non-linear operations (e.g., `exp`, `log`, `sqrt`). Lowering logic for these operations is often defined via templates in `TTLElementwiseOps.def`[lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp 14-15](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp#L14-L15)
*   **FPU (Floating Point Unit):** The primary execution unit for standard matrix and element-wise math. Certain ops like `add`/`sub`/`mul` are eligible to lower to FPU specific kernels [lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp 10-12](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp#L10-L12)

### Data Structures

*   **DFB (Dataflow Buffer):** The high-level abstraction for a hardware Circular Buffer (CB). It provides producer-consumer synchronization between data movement and compute threads. In simulation, it is implemented by `DataflowBuffer`[python/sim/dfb.py 28](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/sim/dfb.py#L28-L28)
*   **CB (Circular Buffer):** The hardware implementation of the DFB. It consists of a reserved memory region in L1 and a set of pointers managed by the hardware. Bindings occur at kernel scope [include/ttlang/Dialect/TTL/IR/TTLOps.td 25-50](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L25-L50)
*   **Tile:** The fundamental unit of data in Tenstorrent hardware, typically a 32x32 matrix. The default tile size is defined as 32x32 in the `ttcore` dialect [include/ttlang/Dialect/TTL/IR/TTLOps.td 109-110](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L109-L110)
*   **Block:** A collection of tiles treated as a single unit. In `tt-lang`, users pick block shapes to achieve best performance.
*   **TensorAccessor:** A lower-level representation of a tensor that includes physical addressing and layout metadata. Tensors with TTL layout encoding are converted to `TensorAccessor` for the TTKernel dialect [lib/Dialect/TTL/Transforms/ConvertTTLToTTKernel.cpp 75-80](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLToTTKernel.cpp#L75-L80)
*   **buffer_factor / block_count:** The number of blocks allocated within a single Circular Buffer to enable pipelining (e.g., `block_count=2` for double buffering) [include/ttlang/Dialect/TTL/IR/TTLOps.td 35-45](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L35-L45)

## Compilation and MLIR Terms

### Dialects and Ops

*   **TTL Dialect:** The primary MLIR dialect for `tt-lang`. It includes operations like `ttl.copy`, `ttl.bind_cb`, and `ttl.compute`[include/ttlang/Dialect/TTL/IR/TTLOps.td 25-154](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L25-L154)
*   **TTKernel Dialect:** A lower-level dialect mapping directly to hardware operations. TTL tile-level operations are lowered to this dialect using `DialectConversion`[lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp 9-17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp#L9-L17)
*   **EmitC:** An MLIR dialect used to generate C++ code. `tt-lang` uses it to produce the final kernel source mapped to C++ APIs [lib/Dialect/TTL/Transforms/LowerDPrintToEmitC.cpp 9-13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerDPrintToEmitC.cpp#L9-L13)
*   **SCF (Structured Control Flow):** An MLIR dialect for loops (`scf.for`). `tt-lang` lowers compute blocks to loops using these constructs [lib/Dialect/TTL/Transforms/ConvertTTLToCompute.cpp 17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLToCompute.cpp#L17-L17)

### Optimization Passes

*   **Subblocking:** An optimization partitioning a large block computation into smaller "subblocks" to fit within the limited DST register file capacity [lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp 30](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L30-L30)
*   **In-place Merging:** A DST register allocation optimization where an operation's result overwrites one of its inputs to conserve register space [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 26](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L26-L26)
*   **FPU Accumulation Protection:** Mechanism to ensure that L1 packer accumulation is guarded by proper reconfiguration guards (`pack_reconfig_l1_acc`) so that packs accumulate across iterations instead of overwriting [include/ttlang/Dialect/TTL/Passes.td 143-165](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/Passes.td#L143-L165)
*   **Affine Map / Projected-Permutation Map:** Mathematical representations used to describe indexing and tiling strategies. Indexing maps define how iteration space indices map to data indices [include/ttlang/Dialect/TTL/IR/TTLOps.td 16-17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L16-L17)
*   **unroll_factor:** Degree of loop unrolling applied during lowering to improve throughput.
*   **TilingInterface:** An MLIR interface used to implement tiling and subblocking strategies [include/ttlang/Dialect/TTL/IR/TTLOps.td 18](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L18-L18)

## Hardware and Memory Terms

### Memory Hierarchy

*   **L1:** Per-core SRAM holding Circular Buffers. It is one of the supported memory spaces [include/ttlang/Dialect/TTL/IR/TTLOps.td 143-145](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/IR/TTLOps.td#L143-L145)
*   **DST:** The destination register file in the MATH engine used for accumulating results. Its capacity is a primary constraint for kernel tiling [lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp 11-12](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp#L11-L12)
*   **NOC (Network on Chip):** The interconnect used for moving data between cores and DRAM.
*   **TT-Fabric:** Interconnect for chip-to-chip communication in multi-chip grids.

### Data Movement Patterns

*   **Pipe / Pipe Net:** Abstractions for point-to-point or multicast communication between nodes [python/ttl/pipe.py 77](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/pipe.py#L77-L77)
*   **if_src / if_dst:** Conditional execution guards used within kernels to specify code that should only run on specific ends of a pipe.

## Profiling and Debugging

*   **Signpost:** A lightweight profiling marker inserted into kernel code, captured by hardware profilers [python/ttl/ttl_api.py 63](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L63-L63)
*   **Roofline Analysis:** Performance model identifying if a kernel is memory-bound or compute-bound.
*   **CB Flow Graph:** Visualization of data movement through Circular Buffers, loaded from device profiler CSVs [python/ttl/ttl_api.py 55-58](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L55-L58)
*   **TTLANG_AUTO_PROFILE:** Enables automatic line-level profiling and report generation [python/ttl/ttl_api.py 53-61](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L53-L61)
*   **TTLANG_COMPILE_ONLY:** Stops the pipeline after C++ generation, skipping hardware execution [python/ttl/ttl_api.py 162-163](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L162-L163)
*   **TTLANG_INITIAL_MLIR / TTLANG_FINAL_MLIR:** Environment variables used to dump the MLIR at the start and end of the transformation pipeline.
