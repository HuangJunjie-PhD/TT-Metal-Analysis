## Purpose and Scope

This page documents the Low-Level Kernel (LLK) implementation specific to the r37 architecture. The r37 LLK serves as a hardware abstraction layer for the r37 processor, providing essential interfaces and functionality for higher-level applications to interact with the r37 hardware. For information about the general LLK system and other implementations, see [LLK System](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3-llk-system). For details about the r37 Instruction Set Simulator that this implementation relies on, see [r37 Instruction Set Simulator](https://deepwiki.com/tenstorrent/tensix-isa-simulator/4.1-r37-instruction-set-simulator).

## Overview

The r37 LLK implementation provides architecture-specific functions and interfaces that allow applications to leverage the capabilities of the r37 processor. It sits between the hardware (or hardware simulator) and higher-level applications, abstracting away hardware details while providing optimized access to r37-specific features.

Sources: [tensix/whb0/prj/build_r37_llk.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_llk.sh)[tensix/whb0/prj/build_demo_llk_r37.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_r37.sh)

## Architecture

The r37 LLK implementation consists of a set of C++ source files that provide the necessary functionality to interact with the r37 hardware through the instruction set simulator.

Sources: [tensix/whb0/prj/build_r37_llk.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_llk.sh)

## Key Components

The r37 LLK implementation provides several key components:

1.   **Hardware Abstraction Layer**: Abstracts the r37 hardware details, providing a clean interface for applications
2.   **Memory Management**: Handles memory allocation, access, and manipulation specific to the r37 architecture
3.   **Execution Control**: Provides functions to control program execution on r37 cores
4.   **I/O Operations**: Manages input/output operations for the r37 processor
5.   **Interrupt Handling**: Implements r37-specific interrupt handling mechanisms

## Build System

The r37 LLK implementation is built as a static library (`r37_llk.a`) using a dedicated build script. The build process compiles all C++ source files in the `r37/llk_lib` directory.

Sources: [tensix/whb0/prj/build_r37_llk.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_r37_llk.sh)

## Dependencies

The r37 LLK implementation has several dependencies:

| Dependency | Description | Usage |
| --- | --- | --- |
| r37_iss.a | r37 Instruction Set Simulator | Provides simulation of r37 hardware |
| core.a | Core utilities | Provides fundamental functionality |
| schedule.a | Scheduling library | Manages task scheduling |
| sfpi.a | Serial Flash Programming Interface | Handles interaction with flash memory |

Sources: [tensix/whb0/prj/build_demo_llk_r37.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_r37.sh)

## Usage

Applications that need to interact with the r37 hardware can link against the r37_llk.a library. The demo_llk_r37 application serves as an example of how to use the r37 LLK implementation.

To build an application that uses the r37 LLK:

1.   Include the necessary header files from the r37 directories
2.   Link against r37_llk.a and its dependencies (r37_iss.a, core.a, schedule.a, sfpi.a)
3.   Use the provided LLK interfaces to interact with the r37 hardware

Sources: [tensix/whb0/prj/build_demo_llk_r37.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_r37.sh)

## Demo Application

The repository includes a demo application specifically designed to showcase the r37 LLK implementation. This demo is built using the `build_demo_llk_r37.sh` script and demonstrates the usage of the r37 LLK functionality.

The demo application source files are located in `$SRC/demo/llk_r37/*.cpp` and it links against the following libraries:

*   r37_iss.a
*   core.a
*   schedule.a
*   sfpi.a

The demo provides practical examples of how to use the r37 LLK functions to interact with the r37 architecture.

Sources: [tensix/whb0/prj/build_demo_llk_r37.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_r37.sh)
