## Overview

This document describes the CMake build configuration system in tt-tvm. It covers the main `CMakeLists.txt` structure, the user customization workflow via `config.cmake`, and the relationship between configuration options, CMake modules, and build outputs. The focus is on how the build system is organized and how developers configure builds for different targets and feature sets.

For a comprehensive reference of all available build options, see [Build Options Reference](https://deepwiki.com/tenstorrent/tt-tvm/9.2-build-options-reference). For target-specific configuration details, see [Target-Specific Configuration](https://deepwiki.com/tenstorrent/tt-tvm/9.3-target-specific-configuration). For information about the library outputs produced by the build, see [Library Outputs](https://deepwiki.com/tenstorrent/tt-tvm/9.4-library-outputs). For runtime introspection of build configuration, see [Runtime Introspection](https://deepwiki.com/tenstorrent/tt-tvm/9.5-runtime-introspection).

## Configuration Workflow

The tt-tvm build system uses a two-stage configuration approach where users copy a template configuration file and customize it before running CMake.

**Sources:**[CMakeLists.txt 17-23](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L23)[cmake/config.cmake 18-37](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L18-L37)

### Configuration File Resolution

The CMake build looks for configuration files in the following order:

1.   `${CMAKE_BINARY_DIR}/config.cmake` (build directory - highest priority)
2.   `${CMAKE_SOURCE_DIR}/config.cmake` (source directory - fallback)

This allows users to maintain separate build configurations for different targets without modifying the source tree.

**Sources:**[CMakeLists.txt 17-23](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L23)

## User Customization via config.cmake

The `cmake/config.cmake` file serves as a template for users to create their own build configurations. It contains all available options with their default values and documentation.

### Configuration Workflow Example

**Sources:**[cmake/config.cmake 18-37](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L18-L37)

### Option Override Methods

Build options can be specified in three ways, in order of precedence:

1.   **Command-line override**: `cmake -DUSE_CUDA=ON ..`
2.   **User config.cmake**: `set(USE_CUDA ON)` in `build/config.cmake`
3.   **Default value**: Value in `tvm_option` declaration in `CMakeLists.txt`

**Sources:**[CMakeLists.txt 25-28](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L25-L28)

## Option Declaration System

Build options are declared using the `tvm_option` macro, which creates CMake cache variables with documentation:

The `tvm_option` macro allows three types of values:

| Type | Example | Meaning |
| --- | --- | --- |
| Boolean | `ON` / `OFF` | Enable/disable feature with auto-detection |
| Path | `/path/to/cuda` | Enable with specific installation path |
| String | `llvm-config --link-static` | Custom configuration command |

## CMake Module System

After option declaration, the build includes numerous CMake modules that configure specific features:

**Sources:**[CMakeLists.txt 518-560](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L518-L560)

### Module Include Order

The order of module inclusion is significant due to dependencies:

1.   **VTA and StandaloneCrt** - Independent infrastructure
2.   **Hexagon** - Must come before `Logging.cmake`
3.   **CLML** - Must come before `OpenCL.cmake`
4.   **Core targets** - CUDA, LLVM, ROCM, etc.
5.   **Contrib libraries** - DNNL, CUTLASS, TensorRT, etc.
6.   **Support modules** - Git, LibInfo, RustExt

**Sources:**[CMakeLists.txt 518-560](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L518-L560)
