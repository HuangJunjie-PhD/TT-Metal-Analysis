## Purpose and Scope

This document describes how ONNX (Open Neural Network Exchange) models are integrated into the Polaris simulation framework and converted to the internal graph representation. The ONNX integration system provides a bridge between external pre-trained models and the TTSIM simulation engine, enabling performance analysis of existing neural network models without requiring manual reconstruction.

For information about the broader workload system including TTSIM-native workloads, see [Neural Network Workloads](https://deepwiki.com/tenstorrent/polaris/4.1-neural-network-workloads). For details about the internal graph representation and operations, see [Operations and Graph Construction](https://deepwiki.com/tenstorrent/polaris/2.2.1-operations-and-graph-construction).

## ONNX Model Loading and Conversion

The ONNX integration system centers around the `onnx2graph` function, which handles the conversion of ONNX model files into the internal graph representation used by the simulation framework.

### Core Conversion Function

The primary entry point for ONNX model conversion is implemented in the `ttsim.front.onnx.onnx2nx` module:

`def onnx2graph(name: str, onnx_file_path: str) -> WorkloadGraph`
This function takes an ONNX model file and converts it into a `WorkloadGraph` object that can be processed by the simulation engine.

The ONNX integration system includes comprehensive testing to ensure reliable model conversion and compatibility with the simulation framework.

## Data Type and Format Integration

The ONNX integration system must handle data type conversions between ONNX format specifications and the internal simulation type system.

### Type Conversion Support

The system leverages the unified type system to handle conversions:

| ONNX Type | Internal Type | Conversion Function |
| --- | --- | --- |
| ONNX Data Types | `SimDataType` | `get_sim_dtype()` |
| ONNX Tensors | `SimTensor` | Tensor factory methods |
| ONNX Operations | `SimOp` | Operation mapping |

The type conversion system ensures that ONNX model semantics are preserved during the conversion to internal representations.
