# Configuration

Relevant source files
*   [README.md](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1)

This document details all configuration requirements for running ttsim, including mandatory and optional environment variables, SOC descriptor file placement, architecture-specific settings, and troubleshooting common configuration errors. For initial setup instructions, see [Getting Started](https://deepwiki.com/tenstorrent/ttsim/2-getting-started). For a complete reference of all environment variables, see [Environment Variables](https://deepwiki.com/tenstorrent/ttsim/7.2-environment-variables).

## Configuration Overview

ttsim requires three mandatory configuration items to function correctly:

| Configuration Item | Type | Required | Description |
| --- | --- | --- | --- |
| `TT_METAL_HOME` | Environment Variable | Yes | Path to TT-Metalium installation root |
| `TT_METAL_SIMULATOR` | Environment Variable | Yes | Absolute path to `libttsim_*.so` file |
| `TT_METAL_SLOW_DISPATCH_MODE` | Environment Variable | Yes | Must be set to `1` (fast dispatch not supported) |
| `soc_descriptor.yaml` | File | Yes | SOC descriptor co-located with simulator binary |

**Sources:**[README.md 22-61](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L22-L61)

## Environment Variables

### TT_METAL_HOME

**Purpose:** Specifies the root directory of the TT-Metalium installation.

**Requirement:** Mandatory. Must be set before running any TT-Metalium workload.

**Valid Values:** Absolute path to a valid TT-Metalium installation directory.

**Example:**

`export TT_METAL_HOME=~/tt-metal`
**Used By:** TT-Metalium runtime to locate:

*   Built executables in `build/`
*   Programming examples
*   Test suites
*   SOC descriptor source files in `tt_metal/soc_descriptors/`

**Validation:** The path must exist and contain a valid TT-Metalium installation structure. The runtime will fail during initialization if this variable is not set or points to an invalid location.

**Sources:**[README.md 25](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L25-L25)[README.md 48-50](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L48-L50)[README.md 57-59](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L57-L59)

### TT_METAL_SIMULATOR

**Purpose:** Points TT-Metalium to the simulator library instead of physical hardware.

**Requirement:** Mandatory when running on ttsim. Leave unset to use physical silicon.

**Valid Values:** Absolute path to a `libttsim_*.so` file. The file must exist and be readable.

**Architecture-Specific Values:**

*   Wormhole: Path to `libttsim_wh.so`
*   Blackhole: Path to `libttsim_bh.so`

**Examples:**

`# Wormholeexport TT_METAL_SIMULATOR=~/sim/libttsim_wh.so # Blackholeexport TT_METAL_SIMULATOR=~/sim/libttsim_bh.so`
**Used By:** TT-Metalium device initialization code performs a runtime check for this variable. When set, the runtime uses `dlopen()` to load the specified simulator library instead of initializing physical PCIe devices.

**Co-Location Requirement:** The directory containing the `.so` file must also contain a file named `soc_descriptor.yaml`. See [SOC Descriptor Placement](https://github.com/tenstorrent/ttsim/blob/67b1733e/SOC%20Descriptor%20Placement)

**Validation:** The runtime will fail with a configuration error if:

*   The file does not exist
*   The file is not a valid shared library
*   The file lacks required exported symbols
*   The `soc_descriptor.yaml` file is missing from the same directory

**Sources:**[README.md 41-42](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L41-L42)[README.md 47](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L47-L47)[README.md 56](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L56-L56)

### TT_METAL_SLOW_DISPATCH_MODE

**Purpose:** Enables slow dispatch mode, the only dispatch mode currently supported by ttsim.

**Requirement:** Mandatory. Must be set to `1` when running on ttsim.

**Valid Values:**`1` (enabled)

**Example:**

`TT_METAL_SLOW_DISPATCH_MODE=1 ./build/programming_examples/metal_example_add_2_integers_in_riscv`
**Used By:** TT-Metalium dispatch layer to select command queue implementation. Fast dispatch mode involves hardware features not yet implemented in ttsim.

**Known Limitation:** Fast dispatch is not working [README.md 64](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L64-L64) All workloads must run with slow dispatch mode enabled.

**Typical Usage:** This variable is usually set inline when invoking executables rather than being exported globally, to avoid accidentally forcing slow dispatch when running on physical hardware.

**Validation:** Attempting to run with fast dispatch (variable unset or set to `0`) will fail with an error message indicating that fast dispatch is not supported.

**Sources:**[README.md 51](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L51-L51)[README.md 60](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L60-L60)[README.md 64](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L64-L64)

## SOC Descriptor Placement

### File Location Requirement

The simulator requires a file named `soc_descriptor.yaml` in the same directory as the `libttsim_*.so` file. This co-location requirement is enforced by the simulator initialization code.

**Directory Structure:**

```
~/sim/
├── libttsim_wh.so
└── soc_descriptor.yaml
```

**Sources:**[README.md 42-43](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L42-L43)

### SOC Descriptor Sources

SOC descriptors are architecture-specific YAML files that describe the chip topology. TT-Metalium provides these descriptors in its source tree:

| Architecture | Source File | Copy Destination |
| --- | --- | --- |
| Wormhole | `$TT_METAL_HOME/tt_metal/soc_descriptors/wormhole_b0_80_arch.yaml` | `~/sim/soc_descriptor.yaml` |
| Blackhole | `$TT_METAL_HOME/tt_metal/soc_descriptors/blackhole_140_arch.yaml` | `~/sim/soc_descriptor.yaml` |

**Sources:**[README.md 48](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L48-L48)[README.md 57](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L57-L57)

### SOC Descriptor Content

SOC descriptor YAML files specify:

*   Number and location of Tensix cores
*   DRAM configuration
*   Network-on-chip (NoC) topology
*   Memory map
*   Architecture-specific parameters

The descriptor must match the architecture of the simulator binary (`libttsim_wh.so` requires a Wormhole descriptor, `libttsim_bh.so` requires a Blackhole descriptor).

**Sources:**[README.md 48](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L48-L48)[README.md 57](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L57-L57)

### Configuration Setup Command

`# Wormholecp $TT_METAL_HOME/tt_metal/soc_descriptors/wormhole_b0_80_arch.yaml ~/sim/soc_descriptor.yaml # Blackholecp $TT_METAL_HOME/tt_metal/soc_descriptors/blackhole_140_arch.yaml ~/sim/soc_descriptor.yaml`
**Sources:**[README.md 48](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L48-L48)[README.md 57](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L57-L57)

## Configuration Validation

### Pre-Flight Checklist

Before running workloads, verify the following:

| Check | Command | Expected Result |
| --- | --- | --- |
| `TT_METAL_HOME` set | `echo $TT_METAL_HOME` | Non-empty path |
| `TT_METAL_HOME` exists | `ls $TT_METAL_HOME` | Directory contents listed |
| `TT_METAL_SIMULATOR` set | `echo $TT_METAL_SIMULATOR` | Non-empty path to `.so` |
| Simulator file exists | `ls $TT_METAL_SIMULATOR` | File found |
| SOC descriptor exists | `ls $(dirname $TT_METAL_SIMULATOR)/soc_descriptor.yaml` | File found |
| SOC descriptor valid | `head $(dirname $TT_METAL_SIMULATOR)/soc_descriptor.yaml` | YAML content visible |

**Sources:**[README.md 22-61](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L22-L61)

### Validation Script

`#!/bin/bash# validate_ttsim_config.sh echo "Validating ttsim configuration..." # Check TT_METAL_HOMEif [ -z "$TT_METAL_HOME" ]; then    echo "ERROR: TT_METAL_HOME not set"    exit 1fi if [ ! -d "$TT_METAL_HOME" ]; then    echo "ERROR: TT_METAL_HOME directory does not exist: $TT_METAL_HOME"    exit 1fiecho "✓ TT_METAL_HOME: $TT_METAL_HOME" # Check TT_METAL_SIMULATORif [ -z "$TT_METAL_SIMULATOR" ]; then    echo "ERROR: TT_METAL_SIMULATOR not set"    exit 1fi if [ ! -f "$TT_METAL_SIMULATOR" ]; then    echo "ERROR: Simulator file does not exist: $TT_METAL_SIMULATOR"    exit 1fiecho "✓ TT_METAL_SIMULATOR: $TT_METAL_SIMULATOR" # Check SOC descriptorSIM_DIR=$(dirname "$TT_METAL_SIMULATOR")SOC_DESC="$SIM_DIR/soc_descriptor.yaml"if [ ! -f "$SOC_DESC" ]; then    echo "ERROR: soc_descriptor.yaml not found in simulator directory: $SIM_DIR"    exit 1fiecho "✓ SOC descriptor: $SOC_DESC" echo "Configuration valid!"`
**Sources:**[README.md 22-61](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L22-L61)

## Common Configuration Errors

### ConfigurationError: TT_METAL_HOME Not Set

**Error Message:**

```
ConfigurationError: TT_METAL_HOME environment variable not set
```

**Cause:** The `TT_METAL_HOME` environment variable is not defined.

**Resolution:**

`export TT_METAL_HOME=/path/to/tt-metal`
**Sources:**[README.md 25](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L25-L25)[README.md 74-75](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L74-L75)

### ConfigurationError: Simulator Library Not Found

**Error Message:**

```
ConfigurationError: Cannot open simulator library: /path/to/libttsim_wh.so
```

**Possible Causes:**

1.   `TT_METAL_SIMULATOR` points to a non-existent file
2.   File permissions prevent reading the `.so` file
3.   Path contains typos

**Resolution:**

`# Verify file existsls -l $TT_METAL_SIMULATOR # Check permissions (should be readable)chmod +r $TT_METAL_SIMULATOR # Verify correct pathexport TT_METAL_SIMULATOR=~/sim/libttsim_wh.so  # Use absolute path`
**Sources:**[README.md 47](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L47-L47)[README.md 74-75](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L74-L75)

### ConfigurationError: SOC Descriptor Not Found

**Error Message:**

```
ConfigurationError: soc_descriptor.yaml not found in simulator directory
```

**Cause:** The `soc_descriptor.yaml` file is missing from the directory containing the simulator `.so` file.

**Resolution:**

`# Determine simulator directorySIM_DIR=$(dirname $TT_METAL_SIMULATOR) # Copy appropriate descriptorcp $TT_METAL_HOME/tt_metal/soc_descriptors/wormhole_b0_80_arch.yaml $SIM_DIR/soc_descriptor.yaml# orcp $TT_METAL_HOME/tt_metal/soc_descriptors/blackhole_140_arch.yaml $SIM_DIR/soc_descriptor.yaml`
**Sources:**[README.md 42-43](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L42-L43)[README.md 48](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L48-L48)[README.md 57](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L57-L57)[README.md 74-75](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L74-L75)

### ConfigurationError: SOC Descriptor Mismatch

**Error Message:**

```
ConfigurationError: SOC descriptor architecture mismatch
```

**Cause:** The SOC descriptor file does not match the simulator architecture (e.g., using a Blackhole descriptor with `libttsim_wh.so`).

**Resolution:** Ensure the descriptor matches the simulator:

*   `libttsim_wh.so` → `wormhole_b0_80_arch.yaml`
*   `libttsim_bh.so` → `blackhole_140_arch.yaml`

`# For Wormholecp $TT_METAL_HOME/tt_metal/soc_descriptors/wormhole_b0_80_arch.yaml \   $(dirname $TT_METAL_SIMULATOR)/soc_descriptor.yaml # For Blackholecp $TT_METAL_HOME/tt_metal/soc_descriptors/blackhole_140_arch.yaml \   $(dirname $TT_METAL_SIMULATOR)/soc_descriptor.yaml`
**Sources:**[README.md 48](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L48-L48)[README.md 57](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L57-L57)[README.md 74-75](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L74-L75)

### UnsupportedFunctionality: Fast Dispatch Not Supported

**Error Message:**

```
UnsupportedFunctionality: Fast dispatch mode is not supported by the simulator
```

**Cause:** Attempting to run without `TT_METAL_SLOW_DISPATCH_MODE=1`.

**Resolution:** Always set slow dispatch mode:

`TT_METAL_SLOW_DISPATCH_MODE=1 ./your_executable`
**Sources:**[README.md 64](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L64-L64)[README.md 71](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L71-L71)

### SystemError: Invalid Shared Library

**Error Message:**

```
SystemError: Cannot load simulator library: undefined symbol
```

**Possible Causes:**

1.   Corrupted download
2.   Incompatible simulator version
3.   Wrong architecture (downloaded ARM binary on x86_64)

**Resolution:**

1.   Re-download the simulator binary
2.   Verify checksum if provided
3.   Ensure downloading the Linux/x86_64 version

`# Verify file typefile $TT_METAL_SIMULATOR# Should output: ELF 64-bit LSB shared object, x86-64 # Re-download if necessarycd ~/simrm libttsim_wh.sowget https://github.com/tenstorrent/ttsim/releases/download/vX.Y/libttsim_wh.so`
**Sources:**[README.md 3](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L3-L3)[README.md 13](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L13-L13)[README.md 74-75](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L74-L75)

## Environment Configuration Examples

### Persistent Configuration

For regular use, add configuration to your shell profile:

`# ~/.bashrc or ~/.zshrc # TT-Metalium configurationexport TT_METAL_HOME=~/tt-metal # ttsim configuration (Wormhole)export TT_METAL_SIMULATOR=~/sim/libttsim_wh.so # Helper function to run with slow dispatchttsim() {    TT_METAL_SLOW_DISPATCH_MODE=1 "$@"}`
**Usage:**

`source ~/.bashrcttsim ./build/programming_examples/metal_example_add_2_integers_in_riscv`
**Sources:**[README.md 22-61](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L22-L61)

### Per-Session Configuration

For temporary use:

`# Set up in current shell onlyexport TT_METAL_HOME=~/tt-metalexport TT_METAL_SIMULATOR=~/sim/libttsim_wh.so # Run workloadTT_METAL_SLOW_DISPATCH_MODE=1 ./build/test_executable`
**Sources:**[README.md 47-52](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L47-L52)

### Architecture Switching Script

`#!/bin/bash# switch_arch.sh - Switch between Wormhole and Blackhole ARCH=$1 case $ARCH in    wormhole|wh)        export TT_METAL_SIMULATOR=~/sim/wormhole/libttsim_wh.so        echo "Switched to Wormhole"        ;;    blackhole|bh)        export TT_METAL_SIMULATOR=~/sim/blackhole/libttsim_bh.so        echo "Switched to Blackhole"        ;;    *)        echo "Usage: $0 {wormhole|blackhole}"        exit 1        ;;esac`
**Usage:**

`source switch_arch.sh wormholeTT_METAL_SLOW_DISPATCH_MODE=1 ./your_executable source switch_arch.sh blackholeTT_METAL_SLOW_DISPATCH_MODE=1 ./your_executable`
