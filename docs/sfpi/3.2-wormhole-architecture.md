## Purpose and Scope

This document describes the Wormhole-specific implementation of the SFPI hardware abstraction layer. Wormhole is one of three Tenstorrent chip architectures supported by SFPI, featuring 51 hardware operations and specific register file characteristics.

For general hardware intrinsics concepts that apply across architectures, see [Hardware Intrinsics Overview](https://deepwiki.com/tenstorrent/sfpi/3.1-hardware-intrinsics-overview). For architecture-specific details of other platforms, see [Blackhole Architecture](https://deepwiki.com/tenstorrent/sfpi/3.3-blackhole-architecture) and [Grayskull Architecture](https://deepwiki.com/tenstorrent/sfpi/3.4-grayskull-architecture).

## Architecture Overview

The Wormhole implementation consists of three primary layers:

| Layer | File | Purpose |
| --- | --- | --- |
| Hardware Intrinsics | `include/wormhole/sfpi_hw.h` | Defines 51 hardware operations as compiler built-ins |
| Implementation | `include/wormhole/sfpi_imp.h` | Implements C++ type system and operators |
| Math Library | `include/wormhole/sfpi_lib.h` | High-level mathematical functions |

The architecture is conditionally compiled using the `__riscv_xtttensixwh` preprocessor macro, allowing code to target Wormhole-specific features while maintaining portability.

## Hardware Characteristics

### Register File Architecture

Wormhole provides three types of registers:

| Register Type | Count | Size | Purpose |
| --- | --- | --- | --- |
| LREG (Local) | 8 | 32-bit | Temporary scalar/vector storage |
| DREG (Destination) | 512 | 32-bit × 16 cols | Main vector storage (1024 rows × 16 bits or 512 rows × 32 bits) |
| CREG (Constant) | 16 | 32-bit | Read-only constant values |

The destination register file uses a stride-based addressing model:

*   `SFP_DESTREG_STRIDE = 2` - Each 32-bit row occupies 2 address units
*   `SFP_DESTREG_MAX_SIZE = 1024` - Maximum addressable rows
*   `SFP_DESTREG_MAX_ADDR = 512` - Maximum address value (1024 / 2)

Sources: [include/wormhole/sfpi_hw.h 92-97](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_hw.h#L92-L97)

### Vector Type Definition

The fundamental vector type is defined as:

`typedef float __rvtt_vec_t __attribute__((vector_size(64*4)));`
This represents a 64-element vector of 32-bit floats (256 bytes total).

Sources: [include/wormhole/sfpi_hw.h 14](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_hw.h#L14-L14)

### Address Modes

Wormhole uses `ADDR_MODE_NOINC = 3` for both load and store operations:

*   `SFPLOAD_ADDR_MODE_NOINC = 3` - Load without incrementing address pointer
*   `SFPSTORE_ADDR_MODE_NOINC = 3` - Store without incrementing address pointer

This differs from Blackhole which uses `ADDR_MODE_NOINC = 7`.

Sources: [include/wormhole/sfpi_hw.h 104-112](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_hw.h#L104-L112)[include/blackhole/sfpi_hw.h 108-116](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L108-L116)

## Key Architectural Differences

### Wormhole vs Blackhole

| Feature | Wormhole | Blackhole |
| --- | --- | --- |
| Total Operations | 51 | 54 |
| ADDR_MODE_NOINC | 3 | 7 |
| Unique Operations | None | `sfparecip`, `sfpmul24`, `sfpmov_config` |
| SFPCAST modes | 2 (RNE, RNS) | 4 (adds SM32_TO_INT32, INT32_TO_SM32) |
| SFPSHFT modifiers | None | Adds LOGICAL, ARITHMETIC, LREG7 modes |
| Random number support | Via math library | Native `SFPCONFIG_SRC_RAND` |

## Summary

Wormhole provides a comprehensive 51-operation instruction set for SFPU programming, with:

*   8 local registers (LREGs) for temporary storage
*   512 destination registers (DREGs) for vector data
*   16 constant registers (CREGs) including programmable constants
*   Stride-based addressing with ADDR_MODE_NOINC = 3
*   Full support for float, signed integer, and unsigned integer operations
*   Advanced features including LUT operations, stochastic rounding, and vector transposition
*   Strong type system implementation with operator overloading in C++

The architecture serves as the foundation for the Wormhole chip generation while maintaining compatibility with the common SFPI interface defined in [Core SFPI Programming Interface](https://deepwiki.com/tenstorrent/sfpi/2-core-sfpi-programming-interface).
