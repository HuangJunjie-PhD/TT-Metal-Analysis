This page documents the physical organization of the tt-flash codebase: directory layout, package structure, module responsibilities, and resource file locations. This covers the source code organization only; for information about building the project, see [Building from Source](https://deepwiki.com/tenstorrent/tt-flash/8.1-building-from-source), and for packaging and distribution artifacts, see [Release and Distribution](https://deepwiki.com/tenstorrent/tt-flash/9-release-and-distribution).

## Directory Layout

The tt-flash repository follows a standard Python project structure with PEP 517/518 compliance:

```
tt-flash/
├── tt_flash/              # Main Python package
│   ├── __init__.py        # Package initialization & versioning
│   ├── main.py            # CLI entry point
│   ├── flash.py           # Flashing engine
│   ├── chip.py            # Hardware abstraction layer
│   ├── utility.py         # Board identification & utilities
│   └── data/              # Data resources
│       └── fw_defines.yaml
├── pyproject.toml         # PEP 517 package configuration
├── setup.py               # Setuptools compatibility shim (Ubuntu 22.04)
├── Makefile               # Development build commands
├── README.md              # User documentation
├── LICENSE                # Apache 2.0 license
├── CHANGELOG.md           # Release history
├── debian/                # Debian packaging files
│   ├── changelog          # Debian version history
│   ├── control            # Package metadata
│   ├── rules              # Build rules
│   └── ...
└── .github/
    └── workflows/         # CI/CD automation
        ├── build-debian.yml
        ├── build-pypi.yml
        └── release.yml
```

**Sources:**[pyproject.toml 1-84](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L1-L84) repository structure inferred from high-level diagrams

## Package Structure

The `tt_flash` package is organized as a single-level Python package with the following module layout:

**Entry Point Configuration:** The `tt-flash` command-line tool is registered in [pyproject.toml 49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L49-L50):

`[project.scripts]tt-flash = "tt_flash.main:main"`
This creates the `tt-flash` executable that invokes the `main()` function from [tt_flash/main.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py)

**Sources:**[pyproject.toml 49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L49-L50) high-level system architecture diagrams

## Module Responsibilities

| Module | Primary Responsibility | Key Classes/Functions | Dependencies |
| --- | --- | --- | --- |
| `main.py` | CLI argument parsing and orchestration | `main()`, `flash` subcommand, `verify` subcommand | `flash.py`, `chip.py`, `utility.py` |
| `flash.py` | Firmware flashing engine and verification | `flash_chips()`, `FlashWrite` class, version comparison logic | `chip.py`, `utility.py`, `pyluwen` |
| `chip.py` | Hardware abstraction layer | `TTChip` (abstract base), `WhChip`, `BhChip`, `detect_local_chips()`, `detect_chips()` | `pyluwen`, `utility.py` |
| `utility.py` | Board identification and helpers | `get_board_type()`, `change_to_public_name()`, version utilities | None (leaf module) |
| `__init__.py` | Package initialization and version resolution | `__get_package_version()`, `__version__` attribute | `importlib.metadata`, `tomli` |

## Configuration Files

### pyproject.toml

The authoritative package configuration file following PEP 517/518 standards. Located at [pyproject.toml 1-84](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L1-L84)

**Key Sections:**

| Section | Purpose | Lines |
| --- | --- | --- |
| `[project]` | Package metadata (name, version, dependencies) | [1-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/1-35) |
| `[project.scripts]` | Entry point registration | [49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/49-50) |
| `[tool.setuptools]` | Setuptools-specific configuration | [52-73](https://github.com/tenstorrent/tt-flash/blob/60c06032/52-73) |
| `[tool.setuptools.package-data]` | Data file inclusion patterns | [55-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/55-59) |
| `[build-system]` | Build backend specification | [74-81](https://github.com/tenstorrent/tt-flash/blob/60c06032/74-81) |

**Package Data Configuration:**

The package includes non-Python resources specified in [pyproject.toml 55-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L55-L59):

`[tool.setuptools.package-data]"*" = [    ".ignored/version.txt",    "data/*/*.yaml"]`
This ensures `fw_defines.yaml` and version files are bundled with distributions.

**Sources:**[pyproject.toml 1-84](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L1-L84)

### setup.py

A compatibility shim for Ubuntu 22.04's older packaging tools. This file exists because `gbp buildpackage` on Ubuntu 22.04 does not fully support PEP 517. Modern tools should prefer `pyproject.toml`.

**Sources:** High-level build and distribution pipeline diagram, setup.py description

## Package Initialization and Versioning

The [tt_flash/__init__.py 1-60](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/__init__.py#L1-L60) module implements dynamic version resolution using a multi-tiered fallback strategy:

**Implementation Details:**

1.   **Distribution Mode**[tt_flash/__init__.py 28-31](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/__init__.py#L28-L31): When installed via `pip install`, version is retrieved from package metadata using `importlib.metadata.version("tt-flash")`

2.   **Development Mode**[tt_flash/__init__.py 32-49](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/__init__.py#L32-L49): When running from source, version is parsed from `pyproject.toml` using `tomli` (Python <3.11) or `tomllib` (Python ≥3.11), with a `+` suffix to indicate local/unreleased builds

3.   **Version Access**[tt_flash/__init__.py 54-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/__init__.py#L54-L59): The version is exposed as `tt_flash.__version__` via the `__getattr__` hook, implementing lazy evaluation

**Sources:**[tt_flash/__init__.py 1-60](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/__init__.py#L1-L60)

## Data Files and Resources

### fw_defines.yaml

Located at `tt_flash/data/fw_defines.yaml`, this YAML file defines firmware message types and constants used for hardware communication. Referenced in the package data configuration at [pyproject.toml 58](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L58-L58):

`"data/*/*.yaml"`
The file contains definitions for:

*   `MSG_TYPE_ARC_STATE` - ARC processor state queries
*   `MSG_TYPE_FW_VERSION` - Firmware version requests
*   `MSG_TRIGGER_SPI_COPY_LtoR` - Local-to-remote SPI copy (NEBULA_X2)
*   `MSG_CONFIRM_FLASHED_SPI` - Post-flash validation messages (BH firmware v19+)

**Sources:** High-level firmware message types description, [pyproject.toml 55-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L55-L59)

### Version Tracking

The `.ignored/version.txt` file (specified in [pyproject.toml 57](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L57-L57)) is used for build-time version tracking. This file is generated during the release process and excluded from version control, but included in distribution packages.

**Sources:**[pyproject.toml 55-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L55-L59)

## Build and Distribution Artifacts

Build artifacts are explicitly excluded from the package via [pyproject.toml 61-65](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L61-L65):

`[tool.setuptools.exclude-package-data]"*" = [  'build',  'debian',]`
**Artifact Locations:**

| Artifact Type | Location | Purpose |
| --- | --- | --- |
| Python wheels (`.whl`) | `dist/` | PyPI distribution |
| Debian packages (`.deb`) | `../` (parent directory) | Debian repository distribution |
| Build intermediates | `build/` | Setuptools temporary files |
| Debian packaging | `debian/` | Source-only, not in wheels |

The `build/` and `debian/` directories are development/build-time-only and excluded from packaged distributions to keep them lightweight.

**Sources:**[pyproject.toml 61-65](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L61-L65) high-level build and distribution pipeline diagram

## Python Version Compatibility

The project maintains compatibility across Python 3.10+ as specified in [pyproject.toml 6](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L6-L6):

`requires-python = ">=3.10"`
However, conditional dependencies handle older Python versions for development tools:

*   **tomli**[pyproject.toml 28](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L28-L28): Required for Python <3.11 (backport of `tomllib`)
*   **importlib-metadata**[pyproject.toml 31](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L31-L31): Required for Python <3.8
*   **importlib-resources**[pyproject.toml 34](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L34-L34): Required for Python <3.9

Type checking is configured for Python 3.7 compatibility in [pyproject.toml 67-69](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L67-L69) to ensure the codebase remains portable:

`[tool.basedpyright]pythonVersion = "3.7"typeCheckingMode = "standard"`
