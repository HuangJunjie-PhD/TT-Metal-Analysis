# Performance Comparison Tools

Relevant source files
*   [tools/compare_projections.py](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/tools/compare_projections.py)

This document describes the performance comparison utilities used to analyze differences between simulation runs in the Polaris system. The primary tool is `compare_projections.py`, which provides comprehensive comparison capabilities for projection run results, including configuration analysis, performance metrics comparison, and detailed layer-wise operator analysis.

For information about performance metrics extraction from simulation results, see [Performance Metrics Extraction](https://deepwiki.com/tenstorrent/polaris/3.1-performance-metrics-extraction). For details about correlation analysis against reference benchmarks, see [Correlation Analysis](https://deepwiki.com/tenstorrent/polaris/3.2-correlation-analysis).

## Overview

The performance comparison tools enable users to:

*   Compare simulation results between different projection runs
*   Analyze configuration differences that may affect performance
*   Perform layer-wise operator comparisons with configurable tolerance
*   Generate comprehensive HTML reports with interactive visualizations
*   Export comparison results to CSV and JSON formats

### Comparison Status System

The comparison system uses a comprehensive status enumeration to categorize differences:

| Status | Description |
| --- | --- |
| `Match` | Values are identical |
| `ApproxMatch` | Values are within epsilon tolerance |
| `Mismatch` | Values differ beyond tolerance |
| `TypeMismatch` | Values have different types |
| `Only_in_1` | Key exists only in first dataset |
| `Only_in_2` | Key exists only in second dataset |

## Command Line Interface

The `compare_projections.py` tool provides a comprehensive command-line interface:

`python tools/compare_projections.py \    --dir1 /path/to/projection_run_1 \    --dir2 /path/to/projection_run_2 \    --output /path/to/comparison_results \    --epsilon 0.05 \    --generate-html`
### Command Line Arguments

| Argument | Description | Default |
| --- | --- | --- |
| `--dir1`, `-d1` | First projection run directory | Required |
| `--dir2`, `-d2` | Second projection run directory | Required |
| `--output`, `-o` | Output directory for results | Required |
| `--epsilon`, `-e` | Tolerance for approximate matches | 0.05 (5%) |
| `--generate-html`, `--html` | Generate HTML reports | True |

Sources: [tools/compare_projections.py 650-665](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/tools/compare_projections.py#L650-L665)

The comparison system is designed to handle large datasets efficiently:

*   Uses generator expressions for memory-efficient processing
*   Implements lazy loading of statistical data
*   Provides configurable HTML generation for faster processing
*   Supports parallel processing through study-level separation
