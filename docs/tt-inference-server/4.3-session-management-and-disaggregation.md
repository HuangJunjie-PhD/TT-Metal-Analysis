Session management in the `tt-inference-server` C++ backend handles the lifecycle of user sessions, KV cache slot allocation, and the orchestration of distributed inference across split prefill and decode instances. This system enables "Disaggregation," where computationally heavy prefill operations are separated from latency-sensitive decode operations to optimize hardware utilization.

## Session Management and Slot Allocation

The `SessionManager` class is the central authority for managing user sessions and their associated hardware resources. It maintains a mapping between logical session IDs and physical KV cache slots on the Tenstorrent hardware.

### Key Components

*   **`SessionManager`**: Manages the `sessions` map and handles asynchronous memory allocation requests via IPC queues `memoryRequestQueue` and `memoryResultQueue``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/session_manager.cpp#L52-L68" min=52 max=68 file-path="tt-media-server/cpp_server/src/services/session_manager.cpp">Hii</FileRef>`.
*   **`domain::Session`**: Represents a unique user session, tracking its `session_id`, assigned `slot_id`, and `state_` (`IDLE`, `PREPARED`, or `IN_FLIGHT`) to prevent concurrent requests on the same KV cache state `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/domain/session.hpp#L19-L23" min=19 max=23 file-path="tt-media-server/cpp_server/include/domain/session.hpp">Hii</FileRef>`, `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/domain/session.hpp#L25-L93" min=25 max=93 file-path="tt-media-server/cpp_server/include/domain/session.hpp">Hii</FileRef>`.
*   **Memory IPC Queues**: `MemoryRequestQueue` and `MemoryResultQueue` (using Boost Interprocess) are used to communicate with the low-level memory manager for hardware slot allocation `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/session_manager.cpp#L54-L57" min=54 max=57 file-path="tt-media-server/cpp_server/src/services/session_manager.cpp">Hii</FileRef>`.

### Session Lifecycle and Prefix Caching

The manager supports advanced session routing through prefix caching, allowing requests with shared history to reuse existing KV cache slots.

1.   **Creation**: Sessions are created via `createSession`, which triggers an asynchronous `ALLOCATE` task `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/session_manager.cpp#L38-L41" min=38 max=41 file-path="tt-media-server/cpp_server/src/services/session_manager.cpp">Hii</FileRef>`.
2.   **Prefix Indexing**: `SessionManager` maintains a `prefixIndex` mapping content hashes to session IDs. This allows `tryAcquireByPrefixHash` to find a session whose registered prefix hash matches a new request `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/services/session_manager.hpp#L145-L151" min=145 max=151 file-path="tt-media-server/cpp_server/include/services/session_manager.hpp">Hii</FileRef>`, `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/services/session_manager.hpp#L98-L109" min=98 max=109 file-path="tt-media-server/cpp_server/include/services/session_manager.hpp">Hii</FileRef>`.
3.   **In-Flight Tracking**: To maintain consistency, `acquireInFlight` marks a session as busy and registers an optional `cancelFn`. If a second request arrives for the same session, it throws a `SessionInFlightException``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/session_manager.cpp#L160-L193" min=160 max=193 file-path="tt-media-server/cpp_server/src/services/session_manager.cpp">Hii</FileRef>`.
4.   **Closure**: `closeSession` removes the session from the map and index, and triggers a `DEALLOCATE` task to the hardware worker `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/session_manager.cpp#L107-L128" min=107 max=128 file-path="tt-media-server/cpp_server/src/services/session_manager.cpp">Hii</FileRef>`.

## Disaggregation and Prefill/Decode Split

Disaggregation splits the LLM inference pipeline into two distinct server roles to optimize hardware utilization.

### Inter-Server Communication

The `InterServerService` manages socket-based communication between instances using a custom messaging protocol `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/sockets/inter_server_service.hpp#L25-L44" min=25 max=44 file-path="tt-media-server/cpp_server/include/sockets/inter_server_service.hpp">Hii</FileRef>`.

*   **`SocketManager`**: Handles low-level TCP connections, supporting both `SERVER` (typically Decode) and `CLIENT` (typically Prefill) modes `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/sockets/socket_manager.cpp#L50-L101" min=50 max=101 file-path="tt-media-server/cpp_server/src/sockets/socket_manager.cpp">Hii</FileRef>`.
*   **`PrefillRequestMessage`**: Sent from Decode to Prefill. Includes `prompt`, `token_ids`, and the target `slot_id``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/sockets/socket_messages.hpp#L20-L53" min=20 max=53 file-path="tt-media-server/cpp_server/include/sockets/socket_messages.hpp">Hii</FileRef>`.
*   **`PrefillResultMessage`**: Sent from Prefill to Decode. Includes generated text, `token_ids`, and processing metadata `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/sockets/socket_messages.hpp#L62-L108" min=62 max=108 file-path="tt-media-server/cpp_server/include/sockets/socket_messages.hpp">Hii</FileRef>`.

### Disaggregation Logic

The `DisaggregationService` orchestrates the handoff between servers based on the configured `tt::config::LLMMode``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/disaggregation_service.cpp#L13-L20" min=13 max=20 file-path="tt-media-server/cpp_server/src/services/disaggregation_service.cpp">Hii</FileRef>`.

| Mode | Role in `DisaggregationService` |
| --- | --- |
| **`DECODE_ONLY`** | Receives client requests, sends `PrefillRequestMessage` to Prefill server via `socketService`, and waits for the result to resume local decoding via `llmService->submitStreamingRequest``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/disaggregation_service.cpp#L31-L84" min=31 max=84 file-path="tt-media-server/cpp_server/src/services/disaggregation_service.cpp">Hii</FileRef>`. |
| **`PREFILL_ONLY`** | Listens for `PrefillRequestMessage`, submits it to the local `LLMService` for a single-token prefill, and returns the result via `sendPrefillResult``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/disaggregation_service.cpp#L98-L147" min=98 max=147 file-path="tt-media-server/cpp_server/src/services/disaggregation_service.cpp">Hii</FileRef>`. |

## Socket-Based Communication Details

The `SocketManager` provides a robust, non-blocking communication layer.

*   **Initialization**: Configured via `tt::config::llmMode()`. `DECODE_ONLY` servers initialize as a socket server on `socketPort`, while `PREFILL_ONLY` servers connect as clients to `socketHost``<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/sockets/inter_server_service.cpp#L17-L40" min=17 max=40 file-path="tt-media-server/cpp_server/src/sockets/inter_server_service.cpp">Hii</FileRef>`.
*   **Reliability**: Implements TCP Keep-Alive probes (`TCP_KEEPIDLE`, `TCP_KEEPINTVL`, `TCP_KEEPCNT`) to detect silent connection drops `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/sockets/socket_manager.cpp#L17-L45" min=17 max=45 file-path="tt-media-server/cpp_server/src/sockets/socket_manager.cpp">Hii</FileRef>`.
*   **Health Monitoring**: Servers periodically exchange `HealthCheckMessage` containing CPU usage, memory usage, and active task counts to assist in orchestration `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/sockets/inter_server_service.cpp#L100-L114" min=100 max=114 file-path="tt-media-server/cpp_server/src/sockets/inter_server_service.cpp">Hii</FileRef>`, `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/sockets/socket_messages.hpp#L113-L130" min=113 max=130 file-path="tt-media-server/cpp_server/include/sockets/socket_messages.hpp">Hii</FileRef>`.

## Kafka-Based Session Offloading

For large-scale deployments, the server supports Kafka for session persistence and offloading. This allows session state (including KV cache metadata) to be persisted outside the immediate memory of a single server instance.

*   **Dependencies**: Requires `librdkafka` and is enabled via the `-DKAFKA_ENABLED=ON` CMake flag `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/install_dependencies.sh#L10-L19" min=10 max=19 file-path="tt-media-server/cpp_server/install_dependencies.sh">Hii</FileRef>`.
*   **Components**: 
    *   **`KafkaProducer`**: Publishes session state updates to a Kafka topic `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/messaging/kafka_producer.hpp#L1-L15" min=1 max=15 file-path="tt-media-server/cpp_server/include/messaging/kafka_producer.hpp">Hii</FileRef>`.
    *   **`KafkaConsumer`**: Subscribes to session updates to synchronize state across a cluster `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/messaging/kafka_consumer.hpp#L1-L15" min=1 max=15 file-path="tt-media-server/cpp_server/include/messaging/kafka_consumer.hpp">Hii</FileRef>`.
    *   **`MigrationWorker`**: Orchestrates the movement of session data between different hardware nodes or clusters `<FileRef file-url="https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/worker/migration_worker.hpp#L1-L15" min=1 max=15 file-path="tt-media-server/cpp_server/include/worker/migration_worker.hpp">Hii</FileRef>`.
