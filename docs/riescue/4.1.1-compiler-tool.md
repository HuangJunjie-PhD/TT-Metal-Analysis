# Compiler Tool

 Relevant source files

* [riescue/lib/toolchain/tool.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py)

The `Compiler` class provides a Python abstraction for the RISC-V GCC compiler (`riscv64-unknown-elf-gcc`). It handles executable discovery, march/mabi configuration, compilation options, and error classification for compiling RISC-V assembly and C source files into ELF executables. This tool is a fundamental component of the test generation pipeline, enabling RiESCUE to compile generated tests into binary executables that can be simulated.

For information about other toolchain components, see [Disassembler and Objcopy Tools](/tenstorrent/riescue/4.1.2-disassembler-and-objcopy-tools). For information about the base `Tool` abstraction layer, see [Tool Base Class](/tenstorrent/riescue/7.3.1-tool-base-class).

**Sources:** [riescue/lib/toolchain/tool.py128-204](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L128-L204)

### Default Compiler March

The `Compiler` class defines a comprehensive default march string that includes a wide range of RISC-V extensions:

```
rv64imafdcvh_svinval_zfh_zba_zbb_zbc_zbs_zifencei_zicsr_zvkned_zicbom_zicbop_zicboz_zawrs_zihintpause_zvbb1_zicond_zvkg_zvkn_zvbc_zfa_zk 
```

This default covers:

* Base ISA: `rv64imafdcvh` (RV64 with integer, multiply, atomic, float, double, compressed, vector, hypervisor)
* Standard extensions: Zifencei, Zicsr, Zba/Zbb/Zbc/Zbs (bit manipulation), Zfh (half-precision float)
* Vector crypto: Zvkned, Zvkg, Zvkn, Zvbc, Zvbb
* Cache management: Zicbom, Zicbop, Zicboz
* Additional: Svinval, Zawrs, Zihintpause, Zicond, Zfa, Zk

**Sources:** [riescue/lib/toolchain/tool.py133](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L133-L133)

### Standard Compilation Arguments

The compiler automatically includes these standard arguments for all compilations:

| Argument | Purpose |
| --- | --- |
| `-static` | Generate statically linked executables |
| `-mcmodel=medany` | Use medium any code model for addressing |
| `-fvisibility=hidden` | Default symbol visibility to hidden |
| `-nostdlib` | Don't link standard library |
| `-nostartfiles` | Don't use standard startup files |
| `-march=` | Specify target architecture |
| `-mabi=` | Specify ABI (if provided) |
| `-D=` | Define preprocessor macros (for each test\_equate) |

## Command-Line Interface

The `Compiler` class provides static methods for integrating with argument parsers and constructing instances from parsed arguments.

### Argument Parser Integration

The `add_arguments()` static method adds compiler-related arguments to an argument parser:

| Argument | Type | Description |
| --- | --- | --- |
| `--compiler_path` | `Path` | Path to compiler executable (falls back to `RV_GCC` env var) |
| `--compiler_opts` | `nargs="*"` | Additional arguments to pass to compiler |
| `--compiler_march` | `str` | Architecture string for `-march=` flag |
| `--compiler_mabi` | `str` | ABI specification for `-mabi=` flag |
| `--test_equates` / `-teq` | `action="append"` | Define variables with `-Dkey=value` (can be used multiple times) |
| `--config_json` | `Path` | Path to config.json for feature management |

## Error Handling and Classification

The `Compiler` implements the abstract `_classify()` method to provide meaningful error reporting for compilation failures.

### Error Classification Logic

The error classification extracts diagnostic information from either the output file (if stdout was redirected) or stderr (if not redirected), providing context about compilation failures.

**Sources:** [riescue/lib/toolchain/tool.py192-203](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L192-L203)

### ToolchainError Structure

When compilation fails, the `Compiler` raises a `ToolchainError` with the following information:

| Field | Description |
| --- | --- |
| `tool_name` | Class name: `"Compiler"` |
| `cmd` | Full command that was executed |
| `kind` | `ToolFailureType.COMPILE_FAILURE` or `ToolFailureType.SEGFAULT` |
| `returncode` | Process return code |
| `error_text` | Compiler output or stderr content |
