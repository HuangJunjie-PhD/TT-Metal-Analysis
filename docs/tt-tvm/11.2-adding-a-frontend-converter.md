## Purpose and Scope

This guide provides step-by-step instructions for implementing a frontend converter that translates ML models from a new framework into TVM's Relay IR. A frontend converter is responsible for parsing a framework's model representation (e.g., PyTorch ScriptModule, ONNX ModelProto, TensorFlow GraphDef) and converting it into a Relay `IRModule` with associated parameters.

For information about the overall compilation pipeline, see [Architecture and Compilation Pipeline](https://deepwiki.com/tenstorrent/tt-tvm/2-architecture-and-compilation-pipeline). For details about Relay IR operators, see [Neural Network Operators](https://deepwiki.com/tenstorrent/tt-tvm/4.1-neural-network-operators) and [Tensor Transformation Operators](https://deepwiki.com/tenstorrent/tt-tvm/4.2-tensor-transformation-operators). For testing guidelines, see [Writing Tests](https://deepwiki.com/tenstorrent/tt-tvm/11.5-writing-tests).

## Core Components and File Structure

### Required Files

When adding a new frontend converter, create the following files in `python/tvm/relay/frontend/`:

| File | Purpose | Key Components |
| --- | --- | --- |
| `<framework>.py` | Main converter implementation | Converter class, `from_<framework>()` function |
| `<framework>_ops.py` (optional) | Operator conversion implementations | Specific operator converters if numerous |
| `<framework>_utils.py` (optional) | Helper utilities specific to framework | Preprocessing, pattern matching, etc. |

### Common Infrastructure Usage

All converters should leverage the shared utilities in `common.py`:

| Utility | Purpose | Example Usage |
| --- | --- | --- |
| `AttrCvt` | Converts framework attributes to Relay attributes | Map "kernel_size" to "kernel_shape" |
| `infer_type` | Gets type annotation for an expression | Determine dtype of intermediate results |
| `infer_shape` | Gets shape of an expression | Validate dimensions, insert reshapes |
| `infer_value` | Evaluates constant expressions | Fold constants at conversion time |
| `set_span` | Adds source location information | Enable debugging and error messages |
| `get_relay_op` | Gets Relay operator by name | Look up "nn.conv2d", "add", etc. |

**Sources:**[python/tvm/relay/frontend/common.py 1-50](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/common.py#L1-L50)[python/tvm/relay/frontend/common.py 150-350](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/common.py#L150-L350)

### Step 4: Handle Parameters and Weights

Parameters must be converted from framework tensors to TVM NDArray format:

**Parameter Layout Considerations:**

Different frameworks use different memory layouts:

*   PyTorch Conv2D weights: `[out_channels, in_channels, height, width]`
*   TensorFlow Conv2D weights: `[height, width, in_channels, out_channels]`
*   ONNX Conv2D weights: `[out_channels, in_channels, height, width]`

Transpose weights if necessary using `np.transpose()` before creating NDArray.

**Sources:**[python/tvm/relay/frontend/pytorch.py 4100-4300](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/pytorch.py#L4100-L4300)[python/tvm/relay/frontend/onnx.py 4500-4700](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/onnx.py#L4500-L4700)

### Common Pitfalls

1.   **Forgetting Type Inference**: Always call `infer_type()` or `infer_shape()` when you need to know the properties of intermediate results
2.   **Missing Span Annotations**: Add `set_span()` calls for all converted operators to enable debugging
3.   **Incorrect Parameter Layout**: Remember to transpose weights if framework uses different layout than Relay
4.   **Ignoring Dynamic Shapes**: Handle `None` or negative dimensions by using `relay.Any()`
5.   **Not Testing Edge Cases**: Test with empty tensors, scalar inputs, and unusual dimension combinations

**Sources:**[python/tvm/relay/frontend/pytorch.py 156-290](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/pytorch.py#L156-L290)[python/tvm/relay/frontend/onnx.py 562-597](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/onnx.py#L562-L597)
