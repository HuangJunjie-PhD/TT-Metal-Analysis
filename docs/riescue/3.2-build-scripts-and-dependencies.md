## Purpose and Scope

This page documents the build scripts located in the `infra/` directory that install system-level dependencies and compile external tools required by RiESCUE. These scripts handle the installation of:

1.   System packages on Rocky Linux (`build_base_deps.sh`)
2.   Whisper instruction set simulator (`build_whisper.sh`)
3.   Spike and tt_spike instruction set simulators (`build_spike.sh`)

These scripts are invoked both during container builds (see [Container Definition and Build](https://deepwiki.com/tenstorrent/riescue/3.1.1-container-definition-and-build)) and in CI/CD pipelines (see [Automated Testing](https://deepwiki.com/tenstorrent/riescue/5.3.1-automated-testing)). For information about the RISC-V GNU toolchain, which is downloaded separately, see [RISC-V Toolchain Integration](https://deepwiki.com/tenstorrent/riescue/4.1-risc-v-toolchain-integration). For detailed usage of the simulators after they are built, see [Instruction Set Simulators](https://deepwiki.com/tenstorrent/riescue/4.2-instruction-set-simulators).

## Build Script Overview

The RiESCUE repository contains three main build scripts that establish the foundation for the development environment:

| Script | Purpose | Install Location | Execution Context |
| --- | --- | --- | --- |
| `build_base_deps.sh` | Installs Rocky Linux system packages | System-wide via `yum` | Container build, CI |
| `build_whisper.sh` | Builds Whisper ISS from source | `/usr/bin/whisper` | Container build, CI, development |
| `build_spike.sh` | Builds Spike and tt_spike from source | `/usr/local/bin/spike` `/usr/local/bin/tt_spike` | Container build, CI |

These scripts are designed to work in two distinct contexts:

*   **Inside a Singularity container**: Invoked via `container-run` wrapper
*   **Direct execution**: With `NO_SINGULARITY=1` environment variable set

**Sources:**[infra/build_base_deps.sh 1-39](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L1-L39)[infra/build_whisper.sh 1-73](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_whisper.sh#L1-L73)[infra/build_spike.sh 1-69](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L1-L69)

## System Package Dependencies

The `build_base_deps.sh` script installs Rocky Linux packages using `yum` with the CRB (CodeReady Builder) repository enabled:

### Package Categories

| Category | Packages | Purpose |
| --- | --- | --- |
| **Build Toolchain** | `gcc-toolset-12`, `make`, `perl`, `bc` | C/C++ compilation infrastructure |
| **Development Headers** | `python3-devel`, `openssl-devel`, `zlib-devel`, `boost-devel`, `libffi-devel`, `bzip2-devel`, `sqlite-devel`, `gdbm-devel`, `readline-devel`, `uuid-devel`, `xz-devel` | Header files for building Python and C extensions |
| **Boost Libraries** | `boost`, `boost-static`, `boost-devel`, `boost-regex`, `boost-program-options` | Required by Whisper ISS build |
| **Static Linking** | `libstdc++-static` | Static linking support for simulators |
| **System Utilities** | `wget`, `git`, `sudo`, `zstd`, `which` | Download and version control tools |
| **Python Environment** | `python3-pip`, `python-sphinx` | Python package management and documentation |
| **Device Trees** | `dtc` | Device tree compiler (appears twice in script) |
| **NUMA Support** | `numactl-libs` | NUMA-aware memory allocation |

The complete package list is defined in [infra/build_base_deps.sh 5-37](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_base_deps.sh#L5-L37)

## Spike Build Process

The `build_spike.sh` script builds two versions of the Spike ISS using a common build function:

### Build Function Architecture

**build_spike() Function Flow**

The function definition is at [infra/build_spike.sh 16-32](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L16-L32)

### Spike Variants

The script builds two separate Spike executables with different configurations:

#### Standard Spike

| Configuration Flag | Purpose |
| --- | --- |
| `--enable-dual-endian` | Supports both big-endian and little-endian modes |
| `--with-isa=RV64IMAFDCV_ZBA_ZBB_ZBC_ZBS` | Enables RISC-V extensions: I, M, A, F, D, C, V, and Zba/Zbb/Zbc/Zbs bit manipulation |
| `--with-priv=MSU` | Supports Machine, Supervisor, and User privilege levels |

Installed to: `/usr/local/bin/spike`

#### Tenstorrent Spike (tt_spike)

Additional Tenstorrent-specific flags:

| Configuration Flag | Purpose |
| --- | --- |
| `--enable-tt-stop-if-tohost-nonzero` | Halts simulation when `tohost` register is non-zero |
| `--enable-tt-table-walk-debug` | Enables page table walk debugging output |
| `--enable-tt-expanded-dram-address-range` | Supports expanded DRAM address space |

Installed to: `/usr/local/bin/tt_spike`

### Version Pinning

Both Spike variants are pinned to specific commits defined at [infra/build_spike.sh 9-14](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L9-L14):

## Installation Locations Summary

After successful execution of all build scripts, the following executables are installed:

| Executable | Source | Location | Description |
| --- | --- | --- | --- |
| `whisper` | `tenstorrent/whisper@b24e30f2` | `/usr/bin/whisper` | Tenstorrent Whisper ISS |
| `spike` | `riscv-isa-sim@4703ad98` | `/usr/local/bin/spike` | Standard RISC-V Spike ISS |
| `tt_spike` | `tenstorrent/spike@344b9ef3` | `/usr/local/bin/tt_spike` | Tenstorrent-modified Spike ISS |

These executables are then discovered and wrapped by the RiESCUE toolchain abstraction layer (see [Toolchain Abstraction Layer](https://deepwiki.com/tenstorrent/riescue/7.3-toolchain-abstraction-layer)).
