# TTX Workload Files

Relevant source files
*   [tt_burnin/ttx/bhpv.ttx](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx)

This document covers the TTX workload files used in the tt-burnin system for hardware stress testing. TTX files contain pre-compiled power virus workloads that are loaded onto Tenstorrent AI accelerator chips during burn-in testing to exercise hardware components and verify system stability.

For information about the TTX loading mechanisms and completion checks, see [TTX Workload System](https://deepwiki.com/tenstorrent/tt-burnin/2.3-ttx-workload-system). For details about the supported hardware architectures, see [Supported Chip Architectures](https://deepwiki.com/tenstorrent/tt-burnin/4.1-supported-chip-architectures).

## Overview

TTX workload files are binary packages containing compiled kernels, configuration data, and test specifications designed to stress-test specific Tenstorrent chip architectures. Each TTX file is architecture-specific and contains a power virus workload optimized for that particular chip design.

The tt-burnin system includes three main TTX workload files:

*   `gspv.ttx` - Grayskull Power Virus workload
*   `whpv.ttx` - Wormhole Power Virus workload
*   `bhpv.ttx` - Blackhole Power Virus workload

Sources: [tt_burnin/ttx/bhpv.ttx 1-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L1-L174)

## TTX File Structure

TTX files use a compressed archive format containing multiple components required for workload execution. The internal structure includes compiled kernels, binary images, and YAML configuration files.

### TTX File Format

Sources: [tt_burnin/ttx/bhpv.ttx 1-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L1-L174)

### File Components

| Component | Purpose | Description |
| --- | --- | --- |
| `ckernels.bin` | Compiled Kernels | Pre-compiled kernel code for hardware execution |
| `image.bin` | Binary Image | Hardware-specific binary data and initialization code |
| `image.yaml` | Image Configuration | Metadata and configuration for the binary image |
| `test.yaml` | Test Specification | Test parameters and execution configuration |

The archive structure follows a standardized format with a directory prefix (`0-0/`) containing all workload components.

Sources: [tt_burnin/ttx/bhpv.ttx 27-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L27-L174)

## Hardware-Specific Workloads

Each TTX file targets a specific Tenstorrent architecture with optimized power virus workloads designed to stress the unique characteristics of that chip design.

### Architecture Mapping

Sources: [tt_burnin/ttx/bhpv.ttx 1-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L1-L174)

### Workload Characteristics

Each power virus workload is tailored to exercise architecture-specific features:

*   **Grayskull Power Virus** (`gspv.ttx`): Optimized for first-generation Tenstorrent architecture
*   **Wormhole Power Virus** (`whpv.ttx`): Designed for enhanced compute and memory subsystems
*   **Blackhole Power Virus** (`bhpv.ttx`): Targets advanced features and higher performance capabilities

The workloads are designed to maximize power consumption and thermal stress while maintaining safe operating parameters.

Sources: [tt_burnin/ttx/bhpv.ttx 1-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L1-L174)

## Configuration and Metadata

TTX files include YAML configuration files that specify test parameters and execution settings for the workload.

### Test Configuration Format

The `test.yaml` component contains structured test specifications:

`# Example structure from test.yamltest_parameters:  duration: execution_time  iterations: repeat_count  monitoring: telemetry_settings`
### Image Configuration

The `image.yaml` component provides metadata about the binary image and hardware requirements:

`# Example structure from image.yaml  image_metadata:  architecture: target_chip_type  version: workload_version  checksum: verification_data`
Sources: [tt_burnin/ttx/bhpv.ttx 168-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L168-L174)

## Integration with Burn-in System

TTX workload files are loaded and executed through the tt-burnin system's workload management infrastructure.

### Loading and Execution Flow

The workload loading process involves extracting the TTX archive, parsing configuration files, and deploying the compiled kernels and binary images to the target hardware through the appropriate chip abstraction layer.

Sources: [tt_burnin/ttx/bhpv.ttx 1-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L1-L174)

## File Location and Management

TTX workload files are stored in the `tt_burnin/ttx/` directory as part of the package distribution. The files are included as binary assets and loaded at runtime based on the detected hardware configuration.

### Directory Structure

```
tt_burnin/
└── ttx/
    ├── gspv.ttx    # Grayskull Power Virus
    ├── whpv.ttx    # Wormhole Power Virus
    └── bhpv.ttx    # Blackhole Power Virus
```

The TTX files are platform-specific and cannot be used interchangeably between different chip architectures. The workload selection is handled automatically by the burn-in system based on hardware detection results.

Sources: [tt_burnin/ttx/bhpv.ttx 1-174](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/ttx/bhpv.ttx#L1-L174)
