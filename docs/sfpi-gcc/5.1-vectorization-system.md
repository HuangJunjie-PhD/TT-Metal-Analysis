## Purpose and Scope

The Vectorization System is GCC's framework for automatically transforming scalar code into vector code that can utilize SIMD (Single Instruction, Multiple Data) instructions available on target architectures. This system analyzes loops and basic blocks to identify opportunities for vectorization, performs dependence analysis, constructs vectorized code representations, and interfaces with target-specific instruction capabilities.

This document covers the core vectorization infrastructure including SLP (Superword Level Parallelism), loop vectorization, basic block vectorization, and target interaction mechanisms. For information about target-specific backend implementations, see [Target Backends](https://deepwiki.com/tenstorrent/sfpi-gcc/6-target-backends). For details about range analysis used in vectorization, see [Range Analysis and Value Propagation](https://deepwiki.com/tenstorrent/sfpi-gcc/5.2-middle-end-transformation-passes).

## SLP (Superword Level Parallelism) System

SLP vectorization identifies groups of scalar operations that can be combined into vector operations, even when they don't follow traditional loop-based patterns.

### SLP Tree Structure and Operations

Sources: [gcc/tree-vectorizer.h 160-233](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/tree-vectorizer.h#L160-L233)[gcc/tree-vectorizer.h 235-266](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/tree-vectorizer.h#L235-L266)

### SLP Permutation Handling

The SLP system handles complex data reordering through permutation vectors:

*   **Load Permutation**: Reorders loaded elements relative to stores
*   **Lane Permutation**: Maps input lanes to output lanes using `{operand_number, lane_number}` pairs
*   **Vector Permutation**: Generated `VEC_PERM_EXPR` operations for target-specific reordering

## Loop Vectorization

Loop vectorization transforms scalar loops into vector loops that process multiple data elements per iteration.

### Loop Vectorization Components

Sources: [gcc/tree-vectorizer.h 621-860](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/tree-vectorizer.h#L621-L860)

### Partial Vector Support

Modern vectorization supports partial vectors for handling loop iteration counts that don't divide evenly by the vector factor:

*   **Masks (`vec_loop_masks`)**: Boolean masks for conditional operations
*   **Lengths (`vec_loop_lens`)**: Length-based predication for architectures like SVE
*   **Bias Handling**: Adjustment for targets requiring non-zero lengths

## Vectorization Patterns and Recognition

The vectorization system includes pattern recognition to identify vectorizable code sequences that may not be immediately obvious:

### Common Vectorization Patterns

| Pattern Type | Description | Example Operations |
| --- | --- | --- |
| Reduction | Accumulation operations | Sum, min/max, logical operations |
| Widening | Type promotion sequences | Integer to wider integer/float |
| Narrowing | Type demotion sequences | Wide to narrow type conversion |
| Conditional | If-converted operations | Conditional assignments |
| Complex | Complex arithmetic | Real/imaginary operations |

The pattern recognition system (`vect_recog_*` functions) transforms these patterns into forms suitable for vectorization.

Sources: [gcc/tree-vectorizer.h 42-92](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/tree-vectorizer.h#L42-L92)

## Cost Model and Profitability Analysis

The vectorization system includes sophisticated cost modeling to ensure vectorization improves performance:

### Cost Analysis Components

*   **Statement Costs**: Individual operation costs per vector element
*   **Memory Costs**: Load/store operation costs including alignment penalties
*   **Permutation Costs**: Vector shuffle and permutation operation costs
*   **Scalar Costs**: Cost of remaining scalar operations
*   **Threshold Calculation**: Minimum iteration count for profitability

The cost model interfaces with target-specific cost functions to make accurate profitability decisions.
