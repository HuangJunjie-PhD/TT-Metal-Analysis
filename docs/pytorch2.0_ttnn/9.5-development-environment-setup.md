## System Requirements

The pytorch2.0_ttnn backend requires a specific environment to function correctly. These requirements are validated at import time.

### Platform and Python Version

The system enforces strict platform requirements:

*   **Operating System**: Linux only (Ubuntu 22.04 recommended)
*   **Python Version**: 3.10 exactly
*   **Architecture**: x86_64

These requirements are checked in [torch_ttnn/__init__.py 24-34](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/__init__.py#L24-L34) which raises an `ImportError` if the environment does not meet specifications. The check ensures compatibility with the `ttnn` hardware backend, which is only available on Linux with Python 3.10.

**Sources**: [torch_ttnn/__init__.py 24-51](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/__init__.py#L24-L51)

### Hardware Requirements

The system requires access to Tenstorrent hardware devices. In CI/CD environments, this is provided through:

*   Device mounting: `/dev/tenstorrent`
*   Hugepages: `/dev/hugepages-1G`

These are configured in Docker container options throughout the workflow files.

**Sources**: [.github/workflows/run-tests.yaml 61-62](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/.github/workflows/run-tests.yaml#L61-L62)[.github/workflows/run-tests.yaml 79-80](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/.github/workflows/run-tests.yaml#L79-L80)

### Volume Mounts and Device Access

Container configurations consistently mount:

1.   **Workspace**: `-v ${{ github.workspace }}:${{ github.workspace }} -w ${{ github.workspace }}`

    *   Provides access to checked-out code
    *   Sets working directory

2.   **Hugepages**: `-v /dev/hugepages-1G:/dev/hugepages-1G`

    *   Required for efficient memory allocation

3.   **Tenstorrent Device**: `--device /dev/tenstorrent`

    *   Provides hardware access

4.   **Model Cache** (model tests only): `-v /mnt/tt-metal-pytorch-cache/.cache:/root/.cache`

    *   Persistent cache for downloaded models
    *   Reduces network traffic and test time

**Sources**: [.github/workflows/run-tests.yaml 60-62](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/.github/workflows/run-tests.yaml#L60-L62)[.github/workflows/run-tests.yaml 118-121](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/.github/workflows/run-tests.yaml#L118-L121)

*   Version scheme: `python-simplified-semver`[pyproject.toml 45](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/pyproject.toml#L45-L45)

The version is derived from Git tags and imported in [torch_ttnn/__init__.py 10-12](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/__init__.py#L10-L12) falling back to "unknown" if not available.
