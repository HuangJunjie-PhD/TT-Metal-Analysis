It details the complete request lifecycle including RAG context retrieval, multimodal file processing, streaming message rendering with "thinking" blocks, and high-fidelity metrics tracking.

## RAG Context Integration

When RAG (Retrieval Augmented Generation) is enabled, the system retrieves relevant context before generating the prompt. The `getRagContext` function handles communication with the vector database.

### Retrieval Logic

*   **Collection-based RAG**: Queries specific ChromaDB collections via `/collections-api/{name}/query`.
*   **Global Search**: If `ragDatasource.id` is `"special-all"`, it queries across all collections using `/collections-api/query-all`.
*   **File-based RAG**: If a user uploads a text file during the session, its content is extracted and merged into the `ragContext.documents` array in `runInference`.

## Metrics Tracking

The system employs an `InferenceMetricsTracker` class to capture high-fidelity performance data, following methodologies used in the vLLM project.

### Key Metrics Tracked
| Metric | Description | Implementation |
| :--- | :--- | :--- |
| **TTFT** | Time to First Token | Measured from request start to first content token. |
| **ITL** | Inter-Token Latency | A list of millisecond intervals between every consecutive token. |
| **TPOT** | Time Per Output Token | The mean of the ITL list. |
| **Network Latency** | Overhead | Derived by subtracting backend-reported TTFT from client-measured TTFT. |
| **Reasoning Tokens** | Thinking Count | Count of tokens received inside `<think>` blocks. |

## Multimodal and File Processing

Requests can include various media types, which are transformed into OpenAI-compatible message structures.

- **Images**: If an image is uploaded or a URL is detected, `runInference` constructs a message with `type: "image_url"`.
- **Text Files**: Handled by treating the content as ephemeral RAG context.
- **Data Structure**: Files are represented as `FileData` objects containing the name, type, and either a `blob`, `url`, or raw `text`.

## API Routing Logic

The frontend dynamically routes requests based on the environment and the selected model type:

1.  **Agent Route**: If `isAgentSelected` is true, the request goes to `/models-api/agent/`.
2.  **Cloud Route**: If `VITE_ENABLE_DEPLOYED` is true, it routes to `/models-api/inference_cloud/`.
3.  **Local Route**: Default behavior routes to `/models-api/inference/`, passing the specific `deploy_id`.
