The Planet Tenstorrent sub-site (`/planet/`) serves as a unified, chronological aggregator for the Tenstorrent ecosystem. It centralizes diverse content streams—including academic papers, technical articles, video tutorials, conference talks, software demos, and GitHub releases—into a single, filterable interface.

## 1. Architectural Overview

The Planet sub-site is built as a standalone experience within the `tt-awesome` project. While it shares the core data layer, it utilizes a dedicated layout and CSS architecture to support a long-form scrolling feed rather than the three-pane application layout of the main site.

### Data Flow and Aggregation

The content stream is generated by merging three distinct data sources during the Eleventy build process:

1.   **Project Entries**: Articles, papers, and videos defined in the `entries/` JSON files.
2.   **Recent Releases**: Automated summaries of GitHub releases processed via the LLM pipeline.
3.   **Planet Feeds**: External RSS/Atom feeds (YouTube, arXiv, etc.) curated in `planet_feeds.json`.

## 2. Content Rendering (`index.njk`)

The core logic for the Planet page resides in `src/planet/index.njk`. It uses the `planetItems` custom Nunjucks filter to combine and sort all content by date [src/planet/index.njk 12](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/planet/index.njk#L12-L12)

### Item Grouping and Chronology

The feed is organized chronologically, with items grouped by month. The template tracks the month of the current item and compares it to the previous one to insert month headers:

*   **Filter `monthKey`**: Generates a unique string (e.g., "2023-10") for comparison [src/planet/index.njk 131](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/planet/index.njk#L131-L131)
*   **Filter `monthLabel`**: Formats the date into a human-readable header (e.g., "October 2023") [src/planet/index.njk 134](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/planet/index.njk#L134-L134)

### YouTube Privacy-Enhanced Embeds

For video content, the template includes logic to render privacy-enhanced YouTube embeds. If an item source is identified as "youtube", the template extracts the video ID and renders an `<iframe>` pointing to `youtube-nocookie.com`.

## 3. UI Components and Interactivity

The Planet UI consists of a two-column body: a sticky filter sidebar and a scrollable content area.

### Filter Mechanisms

The site provides two ways to filter content by type (e.g., `paper`, `video`, `release`):

1.   **Desktop Sidebar**: A vertical list of buttons (`.planet-filter-btn`) showing item counts for each category [src/planet/index.njk 63-115](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/planet/index.njk#L63-L115)
2.   **Mobile Chip Bar**: A horizontal scrolling bar of chips (`.planet-chip`) visible only on narrow viewports [src/planet/index.njk 34-57](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/planet/index.njk#L34-L57)

### CSS-Driven Filtering

Filtering is implemented efficiently using CSS attribute selectors. The interactivity script sets a `data-active-type` attribute on the `#planet-feed` container [src/_includes/base-planet.njk 68-71](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base-planet.njk#L68-L71) CSS then hides items that do not match the active type:

`/* Conceptual implementation in planet.css */.planet-feed[data-active-type] .planet-item {  display: none;}.planet-feed[data-active-type="video"] .planet-item[data-type="video"] {  display: block;}`
### Layout State Management

The sidebar can be collapsed to provide more horizontal space for the content feed. This state is managed by the `sidebar-collapsed` class on the `.planet-body` element, toggled by the `planet-collapse-btn` and `planet-expand-btn`[src/_includes/base-planet.njk 114-130](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base-planet.njk#L114-L130)

## 4. Layout and Styling (`base-planet.njk`&`planet.css`)

The `base-planet.njk` layout provides the HTML shell and includes the necessary assets.

### Styling Strategy

Unlike the main site, which uses a viewport-locked "app-style" layout, the Planet page allows for standard document scrolling. `planet.css` explicitly overrides the main site's overflow settings to enable this behavior [src/assets/planet.css 13-16](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/planet.css#L13-L16)

| CSS Variable | Purpose |
| --- | --- |
| `--planet-sidebar-w` | Width of the filter sidebar (default 200px) |
| `--planet-gap` | Standard spacing between elements |
| `--purple` | Accent color for specific badges |

## 5. Automated Feed Updates

The content in Planet Tenstorrent is kept fresh via GitHub Actions that run periodically to fetch external data and update the source JSON files.

| Workflow | Frequency | Script | Purpose |
| --- | --- | --- | --- |
| `nightly.yml` | 2x Daily | `summarize_releases.py` | Generates LLM summaries for new GitHub releases [github/workflows/nightly.yml 3-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/github/workflows/nightly.yml#L3-L4) |
| `planet-feeds.yml` | 8x Daily | `fetch_planet_feeds.py` | Polls RSS/Atom feeds for new community content [github/workflows/planet-feeds.yml 4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/github/workflows/planet-feeds.yml#L4-L4) |

When these workflows detect new content, they create a Pull Request to update `src/_data/planet_feeds.json` or `src/_data/github_meta.json`. New items from external feeds default to `approved: false` and require a manual flip to `true` by a maintainer before they appear on the Planet page [github/workflows/planet-feeds.yml 29](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/github/workflows/planet-feeds.yml#L29-L29)
