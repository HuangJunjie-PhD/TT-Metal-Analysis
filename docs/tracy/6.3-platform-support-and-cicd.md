This page documents Tracy's cross-platform support infrastructure and continuous integration/continuous deployment (CI/CD) systems. It covers the platform abstraction layer, backend implementations, automated testing pipelines, and deployment processes that ensure Tracy works consistently across different operating systems and hardware configurations.

For information about the build system configuration and compilation, see [Build System](https://deepwiki.com/tenstorrent/tracy/2.1-build-system). For integration guidance specific to target applications, see [Integration Guide](https://deepwiki.com/tenstorrent/tracy/2.2-integration-guide).

## Platform-Specific Implementations

### GLFW Backend (Cross-Platform)

The GLFW backend provides broad platform compatibility using the GLFW library for window management and OpenGL for rendering. It supports Windows, Linux, macOS, and web platforms through Emscripten.

Key features:

*   Cross-platform window management via GLFW
*   OpenGL 3.2 Core Profile rendering
*   DPI scaling support with `glfwGetMonitorContentScale`
*   Conditional compilation for platform-specific features
*   Emscripten support for web deployment

**Sources:**[profiler/src/BackendGlfw.cpp 63-221](https://github.com/tenstorrent/tracy/blob/68c5cbd6/profiler/src/BackendGlfw.cpp#L63-L221)

### Wayland Backend (Linux Native)

The Wayland backend provides native Linux desktop integration using Wayland protocols, offering better performance and tighter system integration compared to the GLFW X11 fallback.

The Wayland backend implements comprehensive protocol support:

*   **XDG Shell Protocol**: Window management, decorations, and state handling
*   **Input Handling**: Pointer, keyboard, and touch events with proper keymap support
*   **DPI Scaling**: Automatic scaling based on compositor preferences and output characteristics
*   **EGL Integration**: Native OpenGL context creation for efficient rendering

**Sources:**[profiler/src/BackendWayland.cpp 166-896](https://github.com/tenstorrent/tracy/blob/68c5cbd6/profiler/src/BackendWayland.cpp#L166-L896)[profiler/src/wayland/xdg-shell.h 1-2245](https://github.com/tenstorrent/tracy/blob/68c5cbd6/profiler/src/wayland/xdg-shell.h#L1-L2245)[profiler/src/wayland/xdg-shell.c 1-184](https://github.com/tenstorrent/tracy/blob/68c5cbd6/profiler/src/wayland/xdg-shell.c#L1-L184)

### Thread Management and Asynchronous Operations

The `RunQueue` class provides thread-safe task execution for cross-thread operations:

`class RunQueue {public:    void Queue(const std::function<void()>& cb, bool forceDelay = false);    void Run();private:    std::vector<std::function<void()>> m_queue;    std::mutex m_lock;    std::thread::id m_mainThread;};`
This enables safe communication between the networking threads (receiving profiling data) and the UI thread (rendering the interface).

## Build System Integration

### Visual Studio Integration

Tracy provides enhanced debugging support for Visual Studio through custom visualizers:

*   **Custom Data Structures**: Visualizers for `tracy::Vector`, `tracy::VarArray`, and `tracy::FastVector`
*   **Profiling Data Types**: Specialized views for `ContextSwitchData`, `ContextSwitchCpu`, and compressed integer types
*   **Hash Table Visualization**: Complex visualizer for `tracy::detail::Table` with load factor and element browsing

**Sources:**[profiler/build/win32/DebugVis.natvis 1-111](https://github.com/tenstorrent/tracy/blob/68c5cbd6/profiler/build/win32/DebugVis.natvis#L1-L111)

## Quality Assurance and Testing

### Multi-Configuration Testing

The CI pipeline builds and tests multiple configurations to ensure robustness:

*   **Debug and Release builds** for all major components
*   **Feature flag testing** with `TRACY_ON_DEMAND`, `TRACY_DELAYED_INIT`, and `TRACY_MANUAL_LIFETIME`
*   **Demangle testing** with `TRACY_DEMANGLE` for symbol processing
*   **Meson build system validation** alongside traditional Makefiles

### Artifact Generation

The Windows CI workflow packages release binaries for distribution:

*   Tracy Profiler GUI executable
*   Command-line utilities (capture, update, csvexport, import tools)
*   Client library DLL and import library for development
