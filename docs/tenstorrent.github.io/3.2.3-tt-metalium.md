TT-Metalium™ is Tenstorrent's low-level hardware programming SDK and hardware abstraction layer (HAL). It provides a bare-metal C++ environment for programming Tensix cores, offering direct access to RISC-V processors, the Network-on-Chip (NoC), and local SRAM. It sits between high-level libraries like [TT-NN](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/TT-NN) and the low-level system drivers, abstracting hardware-specific details while providing a consistent programming model across Grayskull, Wormhole, and Blackhole architectures.

## Overview

TT-Metalium enables developers to write custom kernels that run directly on the Tensix cores. It handles the complexities of kernel dispatch, memory management, and host-device synchronization.

## Core Components

### Bare-Metal Kernel Dispatch

TT-Metalium allows users to write kernels in C++ that are compiled for the specific RISC-V processors within each Tensix core. The SDK provides the `tt-metalium-models` container which includes a full build of the project and demo source code for various model architectures.

### Tensix Core Architecture Access

Each Tensix core contains multiple RISC-V processors (Briseis, NCRISC, TRISC). TT-Metalium provides the interfaces to:

*   **RISC-V Programming**: Write custom compute and data movement kernels.
*   **L1 SRAM Management**: Direct control over the 1MB+ local memory per core.
*   **Network-on-Chip (NoC)**: Program the NoC to move data between cores or between cores and DRAM/PCIe.

### Data Flow and Programming Model

The programming model revolves around "Command Queues" and "Kernels". Data is moved from the host to the device using `tt-smi` and `HugePages` for optimized memory performance.

| Component | Description | Code Entity / Tool |
| --- | --- | --- |
| **Driver** | Kernel-mode driver for hardware communication | `TT-KMD` |
| **Firmware** | On-device firmware for Tensix management | `TT-System-Firmware` |
| **Flash** | Utility to update device firmware | `TT-Flash` |
| **SMI** | System Management Interface for monitoring | `TT-SMI` |

## Performance Optimization

TT-Metalium leverages several system-level configurations to maximize throughput:

1.   **HugePages**: Used to improve memory performance by reducing TLB misses during large data transfers between host and device. [core/getting-started/README.md 139](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L139-L139)
2.   **PCIe AER Reporting**: Requires `OS First` configuration in BIOS for proper `TT-SMI` telemetry and error reporting. [core/getting-started/README.md 31-33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L31-L33)
3.   **Mesh Topology**: For Wormhole systems (QuietBox/LoudBox), `tt-topology -l mesh` is used to establish inter-chip communication links. [core/getting-started/vLLM-servers.md 33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L33-L33)
