The Chat UI provides the complete interactive chat interface for TT-Studio, including message handling, inference execution, and user interactions with AI models. This encompasses text and voice input, file upload capabilities, RAG context integration, AI agent selection, and real-time streaming responses with specialized "thinking" block support.

For model deployment and configuration, see [Model Deployment](https://deepwiki.com/tenstorrent/tt-studio/6-model-deployment). For backend inference processing details, see [Inference System](https://deepwiki.com/tenstorrent/tt-studio/7-inference-system).

## Architecture Overview

The Chat UI follows a component-based React architecture with centralized state management and streaming inference capabilities. The system integrates multiple input modalities, RAG context, and AI agents while maintaining responsive design for both desktop and mobile interfaces.

## Core Components

The main orchestrating component manages chat threads, model selection, and coordinates between all child components. It maintains chat history using `usePersistentState` and handles inference orchestration.

**Key State Management:**

*   `chatThreads`: Array of `ChatThread` objects containing messages and titles [app/frontend/src/components/chatui/ChatComponent.tsx 77-80](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/ChatComponent.tsx#L77-L80)

## Message Flow and Inference

### Inference Execution (`runInference.ts`)

The `runInference` function is the primary entry point for executing model requests.

1.   **RAG Context**: If a datasource is selected, it calls `getRagContext` to retrieve relevant documents [app/frontend/src/components/chatui/runInference.ts 44-50](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/runInference.ts#L44-L50)
2.   **File Processing**: Processes uploaded files using `processUploadedFiles`. Text files are treated as temporary RAG context [app/frontend/src/components/chatui/runInference.ts 53-76](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/runInference.ts#L53-L76)
3.   **Prompt Generation**: Uses `generatePrompt` to construct the final message list, incorporating chat history, system prompts, and hardware context [app/frontend/src/components/chatui/runInference.ts 143-150](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/runInference.ts#L143-L150)
4.   **API Selection**: Dynamically selects between `/models-api/agent/`, `/models-api/inference_cloud/`, or `/models-api/inference/` based on the selected agent and environment variables [app/frontend/src/components/chatui/runInference.ts 181-185](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/runInference.ts#L181-L185)
5.   **Metrics Tracking**: Uses `InferenceMetricsTracker` to calculate performance data like tokens per second and Time to First Token (TTFT) [app/frontend/src/components/chatui/runInference.ts 16](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/runInference.ts#L16-L16)

Sources: [app/frontend/src/components/chatui/runInference.ts 18-210](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/runInference.ts#L18-L210)

### Message Structure

Messages are represented by the `ChatMessage` type, which includes:

*   `sender`: "user" or "assistant".
*   `text`: The message content.
*   `thinking`: Optional reasoning content for models that support it.
*   `inferenceStats`: Performance metrics for the specific response.
