This document provides a comprehensive reference for all API endpoints available in TT-Studio, including their routing, request/response patterns, and integration points. The API system serves as the communication layer between the React frontend and the Django/FastAPI backend services.

For information about the backend services that implement these endpoints, see [Backend Services](https://deepwiki.com/tenstorrent/tt-studio/5-backend-services). For details about frontend components that consume these APIs, see [Frontend Components](https://deepwiki.com/tenstorrent/tt-studio/4-frontend-components).

TT-Studio uses a Vite-based development proxy to route frontend API calls to the appropriate backend services. The proxy configuration maps frontend API paths to backend service endpoints.

## Docker Management API (Django Backend)

The Docker API provides container lifecycle management for AI models. These endpoints are defined in the `docker_control` Django app and are proxied via `/docker-api/`.

### Docker API Endpoints

| Endpoint | Method | Purpose | Source |
| --- | --- | --- | --- |
| `/docker/get_containers/` | GET | List model implementations available for deployment | [app/backend/docker_control/urls.py 34](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/docker_control/urls.py#L34-L34) |
| `/docker/status/` | GET | Retrieve list of running containers and their health | [app/backend/docker_control/urls.py 41](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/docker_control/urls.py#L41-L41) |
| `/docker/deploy/` | POST | Deploy a new model container | [app/backend/docker_control/urls.py 35](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/docker_control/urls.py#L35-L35) |
| `/docker/stop/` | POST | Stop a running container and perform hardware reset | [app/backend/docker_control/urls.py 39](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/docker_control/urls.py#L39-L39) |
| `/docker/reset_board/` | POST | Perform a full system reset of Tenstorrent hardware | [app/backend/docker_control/urls.py 44](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/docker_control/urls.py#L44-L44) |
| `/docker/chip-status/` | GET | Retrieve real-time telemetry from TT-SMI | [app/backend/docker_control/urls.py 42](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/docker_control/urls.py#L42-L42) |

### Container Stop and Reset Flow

When a user stops a model, the backend ensures the container is removed and the hardware is reset to a clean state.

**Sources:**[app/backend/README.md 41-69](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/README.md?plain=1#L41-L69)

### Model Deployment Request

The `/docker/deploy/` endpoint expects configuration for the model and hardware allocation.

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `model_id` | string | Yes | The ID from the model catalog |
| `weights_path` | string | No | Custom path to model weights |
| `device_id` | integer | No | Specific Tenstorrent chip ID |

**Sources:**[app/backend/README.md 74-80](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/backend/README.md?plain=1#L74-L80)
