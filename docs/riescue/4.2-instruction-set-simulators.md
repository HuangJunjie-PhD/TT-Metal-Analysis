## Purpose and Scope

This page documents the Instruction Set Simulators (ISS) supported by RiESCUE and their integration with the test framework. It covers the two primary simulators—Whisper and Spike—their build processes, configuration, and usage within RiescueD. For information on the broader toolchain integration, see [RISC-V Toolchain Integration](/tenstorrent/riescue/4.1-risc-v-toolchain-integration). For details on the RiescueD framework that invokes these simulators, see [RiescueD: Directed Test Framework](/tenstorrent/riescue/2.1-riescued:-directed-test-framework).

---

## Overview

RiESCUE supports two RISC-V instruction set simulators for test execution and validation:

| Simulator | Source | Purpose | Key Features |
| --- | --- | --- | --- |
| **Whisper** | [tenstorrent/whisper](https://github.com/tenstorrent/riescue/blob/07cefde3/tenstorrent/whisper) | Tenstorrent's high-performance ISS | Multi-hart support, deterministic execution, JSON configuration |
| **Spike** | [riscv-software-src/riscv-isa-sim](https://github.com/tenstorrent/riescue/blob/07cefde3/riscv-software-src/riscv-isa-sim) | RISC-V reference model | Standard reference implementation |
| **tt\_spike** | [tenstorrent/spike](https://github.com/tenstorrent/riescue/blob/07cefde3/tenstorrent/spike) | Tenstorrent's Spike fork | Extended features for TT-specific testing |

Both simulators are built from source during container construction and are accessible as command-line tools and Python abstractions through the `Toolchain` class.

**Sources:** [README.md88-92](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L88-L92) [riescue/riescued.py311-396](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L311-L396)

## Whisper Simulator

### Overview

Whisper is Tenstorrent's RISC-V instruction set simulator, optimized for performance and feature completeness. It provides deterministic multi-hart simulation and extensive configuration through JSON files.

### Build Process

Whisper is built from source during container setup:

The build script [infra/build\_whisper.sh](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_whisper.sh) clones Whisper at commit `b24e30f238d462d3930744cb084d74129d0873a8` and builds with:

* Boost library from system paths
* Dynamic linking (`STATIC_LINK=0`)
* Soft float support (`SOFT_FLOAT=1`)
* Parallel build (`-j 20`)

**Installation Locations:**

* **Container environment:** `/usr/bin/whisper`
* **Development environment:** Built in `whisper/` submodule directory

**Sources:** [infra/build\_whisper.sh1-73](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_whisper.sh#L1-L73) [README.md90](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L90-L90)

### Configuration Files

Whisper uses JSON configuration files for system-level settings:

| Configuration File | Location | Purpose |
| --- | --- | --- |
| `whisper_config.json` | `dtest_framework/lib/` | Default configuration |
| `whisper_secure_config.json` | `dtest_framework/lib/` | Secure mode configuration |
| User-provided | Via `--whisper_config_json` | Override default configuration |

The configuration file is selected in `RiescueD.simulate()` [riescue/riescued.py367-373](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L367-L373):

**Sources:** [riescue/riescued.py367-373](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L367-L373)

### Command-Line Arguments

Whisper-specific arguments are constructed in the `simulate()` method [riescue/riescued.py365-386](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L365-L386):

| Argument | Condition | Purpose |
| --- | --- | --- |
| `--quitany` | Multi-processor mode | Exit simulation when any hart completes |
| `--harts` | Multi-processor mode | Number of hardware threads |
| `--deterministic 16` | Multi-processor mode | Deterministic scheduling window |
| `--seed` | Multi-processor mode | Random seed for deterministic execution |
| `--endpc` | WYSIWYG mode | End simulation at specific PC |

**Multi-processor mode detection:** [riescue/riescued.py375-384](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L375-L384)

## Spike Simulator

### Overview

Spike is the official RISC-V ISA simulator maintained by RISC-V International. RiESCUE supports both the standard Spike implementation and tt\_spike, Tenstorrent's fork with additional features.

### Build Process

Both Spike variants are built during container setup:

The build script [infra/build\_spike.sh](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh) uses a common `build_spike()` function to build both variants with specific configure options:

**Standard Spike configure options:** [infra/build\_spike.sh43](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L43-L43)

* `--enable-dual-endian`: Support both big and little endian
* `--with-isa=RV64IMAFDCV_ZBA_ZBB_ZBC_ZBS`: Supported ISA extensions
* `--with-priv=MSU`: Machine, Supervisor, and User privilege modes

**tt\_spike additional configure options:** [infra/build\_spike.sh49](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L49-L49)

* `--enable-tt-stop-if-tohost-nonzero`: Exit when tohost register is non-zero
* `--enable-tt-table-walk-debug`: Page table walk debugging
* `--enable-tt-expanded-dram-address-range`: Extended DRAM addressing

**Sources:** [infra/build\_spike.sh1-69](https://github.com/tenstorrent/riescue/blob/07cefde3/infra/build_spike.sh#L1-L69) [README.md91](https://github.com/tenstorrent/riescue/blob/07cefde3/README.md?plain=1#L91-L91)

Spike arguments are constructed in the `simulate()` method [riescue/riescued.py346-364](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L346-L364):

## Integration with RiescueD

### Simulation Workflow

The `simulate()` method [riescue/riescued.py311-411](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L311-L411) orchestrates ISS execution with configuration from `FeatMgr`.

**Sources:** [riescue/riescued.py311-411](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L311-L411)

### WYSIWYG Mode Handling

"What You See Is What You Get" (WYSIWYG) mode uses a special end-of-test mechanism where the test writes `0xc001c0de` to register `x31`. Since Whisper does not natively support this, RiescueD:

1. **Parses the disassembly** [riescue/riescued.py329-342](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L329-L342) to find the  label PC
2. **Adds offset** of `0x10` to get the end-of-test PC
3. **Passes `--endpc`** (Whisper) or `--end-pc`\*\* (Spike) argument
4. **Post-processes the log** [riescue/riescued.py399-408](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L399-L408) to verify `x31` contains `c001c0de`

**Sources:** [riescue/riescued.py329-342](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L329-L342) [riescue/riescued.py385-386](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L385-L386) [riescue/riescued.py399-408](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L399-L408)
