This page is reference documentation for `CompilerConfig` and `MLIRConfig`, the two primary configuration types that control how a model is compiled in tt-forge-onnx. It covers all fields, their types, defaults, and the environment variable override mechanism.

For the broader compilation pipeline that consumes these configurations, see [Compilation Pipeline](https://deepwiki.com/tenstorrent/tt-forge-onnx/3.1-compilation-pipeline-overview). For how verification behavior is configured, see [Verification System](https://deepwiki.com/tenstorrent/tt-forge-onnx/3.4-verification-system).

* * *

## Overview

When calling `forge.compile(model, sample_inputs=inputs, compiler_cfg=...)`, the `CompilerConfig` object governs every major decision in the pipeline: how deep compilation runs, what data formats are used, and what MLIR-level optimizations are applied. An optional nested `MLIRConfig` further controls passes in the TTNN MLIR pipeline.

## CompileDepth

`CompileDepth` is an enum defined in [forge/forge/config.py 21-46](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/config.py#L21-L46) that controls how far through the compilation pipeline execution proceeds. When set to a value less than `FULL`, compilation halts after the corresponding stage. This is primarily used in tests to validate partial pipeline output.

### CompileDepth Stage Reference

| Value | Integer | Description |
| --- | --- | --- |
| `INIT_COMPILE` | 0 | Entry point; initializes compile state |
| `GENERATE_INITIAL_GRAPH` | 1 | TVM relay graph has been generated |
| `POST_INITIAL_GRAPH_PASS` | 2 | Initial C++ graph passes applied |
| `CONSTEVAL_GRAPH` | 3 | Constant evaluation pass applied |
| `POST_PATTERN_MATCHER` | 4 | Pattern matcher pass applied |
| `OPTIMIZED_GRAPH` | 5 | Optimization passes complete |
| `AUTOGRAD` | 6 | Autograd graph generated (training only) |
| `POST_AUTOGRAD_PASS` | 7 | Post-autograd passes applied |
| `PRE_LOWERING_PASS` | 8 | Pre-lowering passes applied |
| `SPLIT_GRAPH` | 9 | Graph split into subgraphs |
| `RUN_MLIR_COMPILER` | 10 | MLIR lowering and TTNN pipeline run |
| `FINISH_COMPILE` | 11 | Binary finalized |
| `FULL` | 12 | Complete; model ready for execution |

## CompilerConfig

`CompilerConfig` is a `@dataclass_json` / `@dataclass` defined in [forge/forge/config.py 49-139](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/config.py#L49-L139) importable as `forge.config.CompilerConfig` or `forge.CompilerConfig`.

### Field Reference

#### Core Compilation

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `compile_depth` | `CompileDepth` | `FULL` | Halt compilation at this stage |
| `enable_training` | `bool` | `False` | Enable autograd and training passes |
| `enable_recompute` | `bool` | `False` | Enable gradient recomputation in autograd |
| `enable_consteval` | `bool` | `True` | Constant-evaluate eligible graph nodes |
| `compile_subgraphs` | `bool` | `False` | Compile each disjoint subgraph separately |
| `enable_pt2_fx_graph_link` | `bool` | `False` | Link past key-value pairs in the graph |
| `enable_optimization_passes` | `bool` | `False` | Enable C++ graph optimization passes |
| `match_subgraph_patterns` | `Optional[int]` | `None` | Compact isomorphic subgraphs via pattern matcher |

#### Data Format

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `default_df_override` | `Optional[DataFormat]` | `None` | Override data format for all nodes; `None` means auto-inferred |
| `default_accumulate_df` | `Optional[DataFormat]` | `None` | Accumulation data format for supported hardware |
| `default_math_fidelity` | `MathFidelity` | `HiFi3` | Math fidelity for all ops |
| `amp_level` | `Optional[int]` | `None` | Automatic Mixed Precision level (see AMP section) |
| `amp_properties` | `List[AMPNodeProperties]` | `[]` | Per-node AMP overrides |

#### TVM / Framework Ingestion

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enable_tvm_cpu_fallback` | `bool` | `False` | Create CPU device for unsupported Forge ops |
| `cpu_fallback_ops` | `Set[str]` | `{"embedding"}` | Op types to route to CPU fallback |
| `enable_tm_cpu_fallback` | `bool` | `False` | Extend CPU fallback to TM ops |
| `tm_cpu_fallback_max_depth` | `int` | `10` | Max depth for TM CPU fallback search |
| `enable_tvm_dropout` | `bool` | `False` | Emit dropout ops through TVM |
| `enable_tvm_unsupported_ops` | `bool` | `False` | Emit unsupported ops to Python for manual editing |
| `enable_tvm_constant_prop` | `bool` | `False` | Constant-propagate in TVM Relay |
| `convert_framework_params_to_tvm` | `bool` | `False` | Convert framework parameters to Relay params |
| `enable_xla_jax_convert` | `bool` | `False` | Convert JAX model to TensorFlow via XLA |
| `enable_tvm_jax_freeze_large_model` | `bool` | `True` | Enable large JAX model tracing (>2 GB) |
| `compile_tvm_to_python` | `bool` | `True` | Generate Forge Python class from TVM graph |
| `retain_tvm_python_files` | `bool` | `False` | Keep generated Python intermediates on disk |
| `tvm_graph_store_path` | `str` | `""` | Path to serialize TVM graph |
| `tvm_graph_load_path` | `str` | `""` | Path to deserialize a previously stored TVM graph |
| `extract_tvm_unique_ops_config` | `bool` | `False` | Extract unique op configurations only |
| `export_tvm_unique_ops_config_details` | `bool` | `False` | Export unique op info to Excel |

#### MLIR

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `mlir_config` | `Optional[MLIRConfig]` | `None` | Fine-grained MLIR pass configuration |

## MLIRConfig

`MLIRConfig` is a C++ struct defined in [forge/csrc/passes/mlir_compiler.hpp 27-80](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/passes/mlir_compiler.hpp#L27-L80) and exposed to Python via pybind11 in [forge/csrc/forge_bindings.cpp 172-206](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/forge_bindings.cpp#L172-L206) It is also importable as `forge.config.MLIRConfig` or `forge._C.MLIRConfig`.

Each field is `std::optional<bool>` in C++, meaning it is only applied to the MLIR pipeline if explicitly set. Unset fields leave the MLIR pipeline's built-in defaults in effect.

### Field Reference

| Field | Python setter | MLIR pipeline option | Description |
| --- | --- | --- | --- |
| `enable_consteval` | `set_enable_consteval(bool)` | `enable-const-eval` | Enable constant evaluation pass in MLIR |
| `enable_optimizer` | `set_enable_optimizer(bool)` | `enable-optimizer` | Enable TTNN optimizer pass |
| `enable_memory_layout_analysis` | `set_enable_memory_layout_analysis(bool)` | `memory-layout-analysis-enabled` | Enable memory layout analysis pass |
| `enable_fusing` | `set_enable_fusing(bool)` | `enable-fusing-pass` | Enable op fusion pass |
| `enable_fusing_conv2d_with_multiply_pattern` | `set_enable_fusing_conv2d_with_multiply_pattern(bool)` | `enable-fusing-conv2d-with-multiply-pattern` | Enable Conv2d+multiply pattern fusion |
| `custom_config` | `set_custom_config(str)` | appended verbatim | Raw option string passed directly to the pipeline |

The `config_to_pipeline_options` function in [forge/csrc/passes/mlir_passes.cpp 43-78](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/passes/mlir_passes.cpp#L43-L78) assembles these fields into the string that is passed to the `ttir-to-ttnn-backend-pipeline` MLIR pass manager.

## MathFidelity

`MathFidelity` is defined in C++ and bound to Python via [forge/csrc/forge_bindings.cpp 141-168](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/forge_bindings.cpp#L141-L168) The default in `CompilerConfig` is `HiFi3`.

| Value | Description |
| --- | --- |
| `LoFi` | Lowest precision; fastest compute |
| `HiFi2` | Medium-low precision |
| `HiFi3` | Medium-high precision (default) |
| `HiFi4` | Highest precision; slowest compute |

## AMP Levels

When `amp_level` is set on `CompilerConfig`, the following automatic data format assignments are applied across the graph:

| `amp_level` | Matmul inputs/outputs | Fused ops / Softmax / LayerNorm | GELU |
| --- | --- | --- | --- |
| `None` (default) | No change | No change | No change |
| `1` | `BFP8_b` | `FP16_b` | Unchanged |
| `2` | `BFP8` | `FP16` | `BFP8` |

Non-mentioned op types retain their default data format (typically set by `default_df_override` or `Float32`).

Sources: [forge/forge/config.py 123-129](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/config.py#L123-L129)

## Environment Variable Overrides

The function `apply_env_config_overrides` in `forge/forge/config.py` reads `FORGE_*` environment variables at runtime and applies them as overrides on top of a given `CompilerConfig` instance. This allows CI and test scripts to adjust compilation behavior without modifying source code.

The `compile_depth` field can be set via environment variable to halt the pipeline at a specific stage, which is used extensively in operator-level and model-level tests (see [Running Tests](https://deepwiki.com/tenstorrent/tt-forge-onnx/8.1-running-tests)).

Sources: [forge/forge/config.py 49-139](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/config.py#L49-L139)
