# Test Organization

Relevant source files
*   [README.md](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1)
*   [riscv_tests/bare_metal/machine/paging_bare/rv_a.list](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/riscv_tests/bare_metal/machine/paging_bare/rv_a.list)

This document describes how RISC-V architectural tests are systematically organized within the repository structure. The tests are hierarchically categorized by virtualization modes, privilege levels, paging mechanisms, and ISA extensions to enable comprehensive coverage of RISC-V architectural features.

For information about the internal structure of individual test files, see [Test File Structure](https://deepwiki.com/tenstorrent/riscv_arch_tests/2.2-test-file-structure). For details on atomic memory operation test implementations, see [Atomic Memory Operations Tests](https://deepwiki.com/tenstorrent/riscv_arch_tests/2.3-atomic-memory-operations-tests).

## Overview

The test organization follows a multi-dimensional categorization system that reflects the various operational modes and extensions of the RISC-V architecture. This structure enables targeted testing of specific architectural configurations while maintaining comprehensive coverage across all supported features.

**Test Organization Hierarchy**

Sources: [README.md 32-185](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1#L32-L185)

## Virtualization Mode Classification

Tests are first categorized by their virtualization execution environment:

| Mode | Directory | Description |
| --- | --- | --- |
| Bare Metal | `bare_metal/` | Tests executed directly on hardware without hypervisor |
| Hypervisor Extension | `h_ext/` | Tests executed within virtualized environment using H-extension |

The `bare_metal` mode contains the majority of tests and represents standard RISC-V execution environments, while `h_ext` contains tests specifically designed for hypervisor functionality validation.

Sources: [README.md 34-35](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1#L34-L35)

## Privilege Mode Organization

Within each virtualization mode, tests are organized by RISC-V privilege levels:

**Privilege Mode Structure**

| Privilege Mode | Directory | RISC-V Modes | Description |
| --- | --- | --- | --- |
| Machine | `machine/` | M | Highest privilege, full hardware access |
| Supervisor | `supervisor/` | MS | Operating system level, limited hardware access |
| User | `user/` | MSU | Application level, most restricted access |

Sources: [README.md 37-41](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1#L37-L41)

## Paging Mode Classification

Each privilege level (except machine mode with only `paging_bare`) supports multiple virtual memory configurations:

| Paging Mode | Directory | Description |
| --- | --- | --- |
| Bare | `paging_bare/` | No virtual memory translation |
| SV39 | `paging_sv39/` | 39-bit virtual addressing |
| SV48 | `paging_sv48/` | 48-bit virtual addressing |
| SV57 | `paging_sv57/` | 57-bit virtual addressing |

Machine mode tests only use `paging_bare` since machine mode typically operates without virtual memory translation.

Sources: [README.md 43-48](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1#L43-L48)

## ISA Extension Categorization

At the leaf level, tests are organized by RISC-V ISA extensions:

**ISA Extension Test Distribution**

### Supported ISA Extensions

| Extension | Directory | Description |
| --- | --- | --- |
| RV64-I | `rv_i/` | 64-bit integer base instruction set |
| RV-M | `rv_m/` | Integer multiplication and division |
| RV-A | `rv_a/` | Atomic memory operations |
| RV-F | `rv_f/` | Single-precision floating-point |
| RV-D | `rv_d/` | Double-precision floating-point |
| RV-C | `rv_c/` | Compressed instructions |
| RV-V | `rv_v/` | Vector operations |
| Zfh | `rv_zfh/` | Half-precision floating-point |
| Zba | `rv_zba/` | Address generation instructions |
| Zbb | `rv_zbb/` | Basic bit manipulation |
| Zbc | `rv_zbc/` | Carry-less multiplication |
| Zbs | `rv_zbs/` | Single-bit instructions |

Sources: [README.md 21-29](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1#L21-L29)

## Vector Extension Specialization

The vector extension (`rv_v/`) contains additional sub-categorization by vector configuration parameters:

| VLMUL Setting | Directory | Description |
| --- | --- | --- |
| M1 | `vlmul_m1/` | Vector length multiplier = 1 |
| M2 | `vlmul_m2/` | Vector length multiplier = 2 |
| M4 | `vlmul_m4/` | Vector length multiplier = 4 |
| MF2 | `vlmul_mf2/` | Vector length multiplier = 1/2 |
| MF4 | `vlmul_mf4/` | Vector length multiplier = 1/4 |
| MF8 | `vlmul_mf8/` | Vector length multiplier = 1/8 |

Sources: [README.md 148-155](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/README.md?plain=1#L148-L155)

## Test List File Organization

Each ISA extension directory contains a corresponding `.list` file that defines the test manifest:

**Test List File Structure**

### Test List Entry Format

Each test entry in a `.list` file follows this format:

```
name=<test_name> tool=<iss_tool> test=<test_path> opts=<options>
```

| Field | Description | Example Values |
| --- | --- | --- |
| `name` | Unique test identifier | `rv64a_9`, `rv64a_17` |
| `tool` | Target ISS for execution | `whisper`, `spike` |
| `test` | Full path to test binary/source | `riscv_tests/bare_metal/machine/paging_bare/rv_a/rv64a_9` |
| `opts` | Compilation/execution options | `default`, `cflags=""` |

Sources: [riscv_tests/bare_metal/machine/paging_bare/rv_a.list 1-200](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/riscv_tests/bare_metal/machine/paging_bare/rv_a.list#L1-L200)

## Directory Path Examples

The following table shows concrete examples of the complete directory paths for different test configurations:

| Configuration | Complete Path |
| --- | --- |
| Machine mode, bare paging, atomic extension | `riscv_tests/bare_metal/machine/paging_bare/rv_a/` |
| Supervisor mode, SV39 paging, integer extension | `riscv_tests/bare_metal/supervisor/paging_sv39/rv_i/` |
| User mode, SV48 paging, floating-point extension | `riscv_tests/bare_metal/user/paging_sv48/rv_f/` |
| Hypervisor supervisor mode, bit manipulation | `riscv_tests/h_ext/supervisor/rv_zba/` |
| Vector extension with VLMUL=M2 | `riscv_tests/bare_metal/machine/paging_bare/rv_v/xlen_256/vlmul_m2/` |

This hierarchical organization enables precise targeting of specific RISC-V architectural configurations while maintaining systematic coverage across all supported features and modes.
