The Data Processing Engine is the core system responsible for ingesting, processing, storing, and providing access to profiling data in Tracy. This system consists primarily of two major components: `TracyWorker` for data processing and storage, and `TracyView` for data visualization coordination.

For information about the timeline visualization and rendering system, see [Timeline Visualization](https://deepwiki.com/tenstorrent/tracy/4.3-timeline-visualization). For details about the UI framework and rendering components, see [UI Framework & Rendering](https://deepwiki.com/tenstorrent/tracy/4.5-ui-framework-and-rendering).

The Data Processing Engine operates as a multi-threaded system that handles real-time profiling data streams from instrumented applications and provides efficient access to both live and historical profiling data.

### Event Processing System

The worker processes different types of profiling events through specialized handlers:

| Event Type | Handler Method | Data Structure | Purpose |
| --- | --- | --- | --- |
| Zone Events | `ProcessZoneBegin/End` | `ZoneEvent` | CPU profiling zones |
| GPU Events | `ProcessGpuZoneBegin/End` | `GpuEvent` | GPU command profiling |
| Memory Events | `ProcessMemAlloc/Free` | `MemEvent` | Memory allocation tracking |
| Lock Events | `ProcessLockAnnounce/Wait/Obtain/Release` | `LockEvent` | Lock contention profiling |
| Messages | `ProcessMessage` | `MessageData` | Debug messages and annotations |
| Callstacks | `ProcessCallstackMemory/Sample` | `CallstackFrameData` | Call stack capture |

### Memory-Efficient Field Packing

Tracy uses sophisticated bit-packing techniques to minimize memory usage:

| Type | Size | Purpose | Bit Layout |
| --- | --- | --- | --- |
| `Int48` | 6 bytes | Timestamps | 48-bit signed integer |
| `Int24` | 3 bytes | Indices/IDs | 24-bit unsigned integer |
| `StringIdx` | 3 bytes | String references | 24-bit string index |
| `CallstackFrameId` | 8 bytes | Frame pointers | 62-bit ptr + 1-bit sel + 1-bit custom |

## Data Processing Flow

### Live Data Processing Pipeline

Sources: [server/TracyWorker.cpp 302-304](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyWorker.cpp#L302-L304)[server/TracyWorker.cpp 7084-7438](https://github.com/tenstorrent/tracy/blob/68c5cbd6/server/TracyWorker.cpp#L7084-L7438)

### File Loading Pipeline

When loading trace files, the worker processes data in a structured sequence optimized for loading performance:

1.   **Header Processing**: Version checks, basic metadata
2.   **String Tables**: Load all string data and build lookup maps
3.   **Source Locations**: Static and dynamic source location tables
4.   **Event Data**: Process events by type (zones, GPU, memory, locks)
5.   **Auxiliary Data**: Callstacks, symbols, frame images
6.   **Statistics Generation**: Build derived data structures for fast queries
