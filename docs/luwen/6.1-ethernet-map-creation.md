## Overview

The Ethernet Map Creation utility is a tool within the Luwen framework that generates a comprehensive mapping of Tenstorrent AI accelerator chips and their interconnections. The map provides critical information needed for multi-chip operations, routing decisions, and system topology understanding. This document explains how the utility operates and the structure of the generated map file.

Sources: [app/create-ethernet-map/src/lib.rs 28-276](https://github.com/tenstorrent/luwen/blob/55e35616/app/create-ethernet-map/src/lib.rs#L28-L276)

## Purpose and Function

The primary function of this utility is to:

1.   Discover all accessible chips in the system
2.   Identify each chip's architecture, coordinates, and capabilities
3.   Map the Ethernet connections between chips
4.   Document chip-specific attributes like harvesting patterns and board types
5.   Output this information in a structured map file

The resulting map file serves as a reference for applications that need to understand the system topology, enabling efficient communication and workload distribution across multiple chips.

## Ethernet Connectivity

The Ethernet map creation process relies on each chip's ability to discover its neighbors. For Wormhole chips, this involves:

1.   Getting the local chip's Ethernet coordinates
2.   Identifying all active Ethernet ports (cores) on the chip
3.   Querying each port for connection status and remote chip information
4.   Establishing a remote connection to each neighboring chip
5.   Recording the bi-directional connection information

## Ethernet Address Structure

Ethernet addresses uniquely identify each chip in the fabric and are crucial for the map creation. The address consists of:

| Component | Description | Usage |
| --- | --- | --- |
| `shelf_x` | X coordinate within a shelf | Local positioning |
| `shelf_y` | Y coordinate within a shelf | Local positioning |
| `rack_x` | X coordinate of the rack | System-wide positioning |
| `rack_y` | Y coordinate of the rack | System-wide positioning |

These coordinates form a hierarchical addressing scheme that enables chips to locate and communicate with each other across the fabric.

## Map File Format

The generated map file contains several sections, each providing important information about the system topology:

```
arch: {
   0: Wormhole,
   1: Wormhole,
   ...
}

chips: {
   0: [1,2,3,4],
   1: [1,2,3,5],
   ...
}

ethernet_connections: [
   [{chip: 0, chan: 3}, {chip: 1, chan: 7}, {routing_enabled: true}],
   ...
]

chips_with_mmio: [
   0: 1,
   1: 2,
   ...
]

harvesting: {
   0: {noc_translation: true, harvest_mask: 16},
   ...
}

boardtype: {
   0: GALAXY_E,
   1: NEBULA_X,
   ...
}
```

Each section serves a specific purpose:

*   `arch`: Maps chip IDs to their architecture type
*   `chips`: Maps chip IDs to their Ethernet coordinates
*   `ethernet_connections`: Lists all connections between chips
*   `chips_with_mmio`: Identifies chips accessible via MMIO and their interface IDs
*   `harvesting`: Contains chip-specific hardware configuration
*   `boardtype`: Identifies the board type for each chip

Sources: [app/create-ethernet-map/src/lib.rs 208-268](https://github.com/tenstorrent/luwen/blob/55e35616/app/create-ethernet-map/src/lib.rs#L208-L268)

## Ethernet Hardware Components

The Wormhole chip contains 16 Ethernet cores arranged along the edges of the chip, which enable inter-chip communication:

Each Ethernet core has:

*   Specific (x,y) coordinates on the chip
*   A training status that determines if it's operational
*   Capability to connect to a corresponding Ethernet core on another chip

## Usage Example

The Ethernet Map Creation utility is typically used during system initialization to discover the topology of connected chips:

1.   Call the utility to generate the map file
2.   Applications then load the map file to understand the system topology
3.   The map information guides communication and workload distribution decisions

For advanced routing configurations or dynamic topology management, see [Core Functionality](https://deepwiki.com/tenstorrent/luwen/4-core-functionality) for more details on chip detection and runtime chip management.
