This page documents the fundamental packer configuration structures, initialization routines, and hardware setup for the LLK packer system. The packer is the final stage in the three-stage Unpack-Math-Pack pipeline, responsible for reading processed data from destination registers and writing it back to L1 memory with format conversion and optional transformations.

For information about destination register management and double-buffering, see [Destination Register Management](https://deepwiki.com/tenstorrent/tt-llk/6.2-destination-register-management). For untilize operations and strided access patterns, see [Untilize and Stride Configuration](https://deepwiki.com/tenstorrent/tt-llk/6.3-untilize-and-stride-configuration). For L1 address calculation and tile header handling, see [Packer Address Calculation and L1 Offset](https://deepwiki.com/tenstorrent/tt-llk/6.4-packer-address-calculation-and-l1-offset). For ReLU threshold configuration, see [ReLU Configuration and Application](https://deepwiki.com/tenstorrent/tt-llk/6.5-relu-configuration-and-application).

## Configuration Structures

### pack_config_t Structure

The `pack_config_t` structure defines hardware configuration for data format conversion and packing behavior:

| Field | Bits | Description |
| --- | --- | --- |
| `row_ptr_section_size` | 16 | Row pointer section size for BFP formats |
| `exp_section_size` | 16 | Exponential section size (BFP formats) |
| `l1_dest_addr` | 32 | L1 destination address (set per pack) |
| `uncompress` | 1 | Enable uncompressed output (always 1) |
| `add_l1_dest_addr_offset` | 1 | Add L1 address offset |
| `out_data_format` | 4 | Output DataFormat enumeration |
| `in_data_format` | 4 | Input DataFormat from dest registers |
| `src_if_sel` | 1 | Source interface selection |
| `pack_per_xy_plane` | 7 | Packs per XY plane (typically 1) |
| `l1_src_addr` | 8 | L1 source address (unused in normal flow) |

**Wormhole-specific fields (word 3):**

*   `exp_threshold_en` (1 bit): Enable exponential thresholding for FP32→BFP_A conversion
*   `exp_threshold` (8 bits): Threshold value (typically 113)

The structure is 4 words (16 bytes) on Wormhole and 3 words (12 bytes) on Blackhole.

**Sources:**[tt_llk_wormhole_b0/common/inc/cpack_common.h 34-60](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L34-L60)[tt_llk_blackhole/common/inc/cpack_common.h 27-62](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L27-L62)

## Initialization Functions

### _llk\_pack\_hw\_configure_

Primary hardware configuration function called during packer initialization:

The function performs the following operations:

1.   Synchronizes if format changes between source and destination
2.   Sets up stride patterns for address counters
3.   Configures ALU destination format
4.   Programs ReLU configuration
5.   Writes complete `pack_config_t` to hardware registers
6.   Calculates L1 address offsets for each packer
7.   Initializes pack counters for edge masking
8.   Sets up address counter limits

**Sources:**[tt_llk_wormhole_b0/llk_lib/llk_pack.h 125-138](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/llk_lib/llk_pack.h#L125-L138)[tt_llk_wormhole_b0/common/inc/cpack_common.h 516-592](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L516-L592)

### set_packer_config

Core configuration function that writes `pack_config_t` to hardware:

For Wormhole, the function writes configuration to all 4 packers. For BFP formats, each packer gets a different `exp_section_size` value to account for the tile header and face offsets.

**Sources:**[tt_llk_wormhole_b0/common/inc/cpack_common.h 224-352](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L224-L352)[tt_llk_blackhole/common/inc/cpack_common.h 201-280](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L201-L280)

## Address Counter Initialization

### packer_addr_counter_init

Resets packer address counters to initial state:

The initialization clears all address counters except CH1 X, which is set to the tile X dimension elsewhere.

**Sources:**[tt_llk_wormhole_b0/common/inc/cpack_common.h 158-162](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L158-L162)

### set_packer_strides

Configures address counter stride patterns based on data format:

**Wormhole stride calculation:**

*   X stride: datum size (1, 2, or 4 bytes)
*   Y stride: `FACE_R_DIM` (16) × X stride
*   Z stride: `FACE_C_DIM` (16) × Y stride
*   W stride: `TILE_NUM_FACES` (4) × Z stride

**Blackhole adds tilize support:**

*   Z stride adjusted for 2-face increments when tilizing
*   CH1 Z stride separate for tilize operations

**Sources:**[tt_llk_wormhole_b0/common/inc/cpack_common.h 203-222](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L203-L222)[tt_llk_blackhole/common/inc/cpack_common.h 166-199](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L166-L199)

## Block Floating-Point Configuration

### Exponential Section Size Calculation

BFP formats require complex `exp_section_size` calculations for each packer to account for tile headers and face offsets:

The calculation accounts for:

1.   **Tile header**: 1 byte (always present)
2.   **Offset**: 2 bytes if `num_faces > 2`, else 0
3.   **Datum bytes**: Format-dependent (16/8/4 for Bfp8/4/2)

For partial faces with BFP formats, `exp_section_size` is set to 1 regardless of `num_faces`.

**Sources:**[tt_llk_wormhole_b0/common/inc/cpack_common.h 318-349](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L318-L349)[tt_llk_wormhole_b0/common/inc/cpack_common.h 168-201](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L168-L201)

### cache_exponential_section_sizes_in_gprs

This function pre-computes BFP exponential section sizes for all format variants and stores them in GPRs for fast reconfiguration:

The cached values enable fast format reconfiguration without recalculating exponential section sizes. The template parameter `<reconfiguring>` determines whether all variants are cached (false) or only face-0 variants (true).

**Sources:**[tt_llk_wormhole_b0/common/inc/cpack_common.h 168-201](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/common/inc/cpack_common.h#L168-L201)

### Register Map

The following table shows the hardware register layout for packer configuration:

| Register Name | Address | Fields | Purpose |
| --- | --- | --- | --- |
| `THCON_SEC0_REG1` | Base+offset | `pack_config_t` words 0-3 | Packer 0 configuration |
| `THCON_SEC0_REG8` | Base+offset | `pack_config_t` words 0-3 | Packer 1 configuration (WH only) |
| `THCON_SEC1_REG1` | Base+offset | `pack_config_t` words 0-3 | Packer 2 configuration (WH only) |
| `THCON_SEC1_REG8` | Base+offset | `pack_config_t` words 0-3 | Packer 3 configuration (WH only) |
| `PCK_DEST_RD_CTRL` | - | `dest_rd_ctrl_t` | Dest register read control |
| `STACC_RELU_ApplyRelu` | - | `relu_config_t` | ReLU configuration |
| `PCK0_ADDR_CTRL_XY_REG_0` | - | X/Y strides | Address counter X/Y |
| `PCK0_ADDR_CTRL_ZW_REG_0` | - | Z/W strides | Address counter Z/W |
| `PACK_COUNTERS_SEC0` | - | `pack_counters_t` | Pack counting control |
| `PCK_EDGE_OFFSET_SEC0` | - | `pck_edge_offset_t` | Edge masking |
