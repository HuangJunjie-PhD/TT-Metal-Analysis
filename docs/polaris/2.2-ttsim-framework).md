The TTSIM Framework provides the core simulation infrastructure for AI workload performance modeling. It implements a graph-based computational model where operations (`SimOp`) are connected through tensors (`SimTensor`) to form workload graphs (`WorkloadGraph`) that can be executed on device models to predict performance characteristics.

This document covers the fundamental components and architecture of the TTSIM simulation engine. For detailed information about specific operation types and graph construction, see [Operations and Graph Construction](https://deepwiki.com/tenstorrent/polaris/2.2.1-operations-and-graph-construction). For tensor manipulation and functional interfaces, see [Tensor Operations](https://deepwiki.com/tenstorrent/polaris/2.2.2-tensor-operations). For hardware specification and device modeling, see [Hardware Modeling](https://deepwiki.com/tenstorrent/polaris/2.2.3-hardware-modeling).

## Core Operation System

The `SimOp` class hierarchy provides the foundation for all computational operations in the framework:

| Class | Purpose | Key Methods |
| --- | --- | --- |
| `SimOp` | Base operation class | `get_perf_counts()`, `execute()`, `set_precision()` |
| `ConstantOp` | Constant value operations | Handles tensor constants and initializers |
| `EltwiseUnaryOp` | Element-wise unary operations | Identity, activation functions |
| `EltwiseBinaryOp` | Element-wise binary operations | Add, multiply, with broadcasting |
| `ConvOp` | Convolution operations | 2D convolution with various parameters |
| `MatMulOp` | Matrix multiplication | General matrix multiplication |
| `LayerNormalizationOp` | Layer normalization | Normalization with scale and bias |

Each `SimOp` instance maintains performance statistics and execution metadata:

`# Key SimOp attributes from ttsim/ops/op.pyself.perf_stats          # Performance statistics dictself.precision           # Data type precisionself.compute_cycles      # Compute execution cyclesself.mem_rd_cycles       # Memory read cycles  self.mem_wr_cycles       # Memory write cyclesself.uses_compute_pipe   # Assigned compute pipeline`

The framework provides several optimization passes that can be applied to the graph:

| Optimization | Method | Purpose |
| --- | --- | --- |
| Precision Setting | `set_precision(o2dt)` | Assign data types to operations |
| Node Removal | `remove_nodes(op_type_list)` | Remove unnecessary operations |
| Node Fusion | `fuse_nodes(fusion_patterns)` | Combine operations for efficiency |
| Resource Mapping | `map_resources(op2rsrc_map)` | Assign operations to compute resources |
