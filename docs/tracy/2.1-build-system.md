This document covers Tracy's build system architecture, including the supported build systems, configuration options, and platform-specific implementations. For information about integrating Tracy into applications after building, see [Integration Guide](https://deepwiki.com/tenstorrent/tracy/2.2-integration-guide).

## Overview

Tracy supports multiple build systems to accommodate different development environments and platforms. The primary build systems are CMake for cross-platform compatibility, Meson for modern build workflows, and traditional Unix Makefiles for Unix-like systems. Each build system can produce the same core components but with different configuration approaches.

## CMake Build System

The CMake build system provides the most comprehensive cross-platform support and is defined in the root `CMakeLists.txt`. It creates the `TracyClient` target as either a static or shared library based on configuration.

### Core CMake Configuration

Sources: [CMakeLists.txt 31-53](https://github.com/tenstorrent/tracy/blob/68c5cbd6/CMakeLists.txt#L31-L53)[CMakeLists.txt 101-148](https://github.com/tenstorrent/tracy/blob/68c5cbd6/CMakeLists.txt#L101-L148)

### CMake Build Options

The CMake system provides numerous configuration options through the `set_option` macro:

| Option | Default | Description |
| --- | --- | --- |
| `TRACY_ENABLE` | ON | Enable profiling |
| `TRACY_ON_DEMAND` | OFF | On-demand profiling |
| `TRACY_CALLSTACK` | OFF | Enforce callstack collection |
| `TRACY_NO_CALLSTACK` | OFF | Disable callstack functionality |
| `TRACY_ONLY_LOCALHOST` | OFF | Listen only on localhost |
| `TRACY_NO_BROADCAST` | OFF | Disable network discovery |
| `TRACY_STATIC` | Based on `BUILD_SHARED_LIBS` | Build as static library |

Sources: [CMakeLists.txt 55-89](https://github.com/tenstorrent/tracy/blob/68c5cbd6/CMakeLists.txt#L55-L89)[CMakeLists.txt 19-29](https://github.com/tenstorrent/tracy/blob/68c5cbd6/CMakeLists.txt#L19-L29)

### Component-Specific Configurations

Each Tracy component has its own build configuration:

| Component | Project Name | Special Flags | Dependencies |
| --- | --- | --- | --- |
| Profiler GUI | `Tracy` | `IMGUI_ENABLE_FREETYPE` | freetype2, capstone, wayland libs |
| Capture Utility | `capture` | `TRACY_NO_STATISTICS` | capstone |
| Update Utility | `update` | `TRACY_NO_STATISTICS` | capstone |
| Library | `libtracy` | `TRACY_ENABLE`, `-fpic` | pthread, dl |

| Platform | Special Dependencies | Notes |
| --- | --- | --- |
| Linux | TBB, debuginfod, wayland | Full feature support |
| macOS | TBB via brew | Clang compiler |
| FreeBSD | execinfo library | Required for backtrace |
| Android | NDK | Limited features |
