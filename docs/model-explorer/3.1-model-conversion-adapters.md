## Purpose and Scope

The Model Conversion Adapters system provides the core backend functionality for converting various machine learning model formats into standardized JSON graph representations that can be visualized by Model Explorer. This system supports multiple model formats including TensorFlow SavedModels, TensorFlow Lite flatbuffers, MLIR files, and HLO computations, offering both MLIR-based and direct conversion pathways.

For information about the Python server that exposes these adapters through REST APIs, see [Python Server and APIs](https://deepwiki.com/tenstorrent/model-explorer/3.2-python-server-and-apis). For frontend visualization of the converted graphs, see [Visualization Engine](https://deepwiki.com/tenstorrent/model-explorer/4.2-visualization-engine).

## Configuration and Customization

The system supports various configuration options through `VisualizeConfig`:

| Option | Purpose | Default |
| --- | --- | --- |
| `const_element_count_limit` | Limit constant element display | 16 |
| `disable_mlir` | Force direct conversion | false |

Command-line flags are processed in `models_to_json_main.cc`:

*   `-i`: Input file path
*   `-o`: Output file path
*   `--const_element_count_limit`: Constant display limit
*   `--disable_mlir`: Disable MLIR conversion path
