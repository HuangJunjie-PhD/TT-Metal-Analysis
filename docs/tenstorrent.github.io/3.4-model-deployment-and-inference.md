This guide provides technical instructions for deploying, serving, and running inference with AI models on Tenstorrent hardware. It covers the setup of the `tt-inference-server`, configuration of vLLM for production-grade serving, and the execution of model demos within containerized environments.

## Deployment Architecture

Tenstorrent supports model deployment through a tiered software stack. Users can serve models via an OpenAI-compatible API using the `tt-inference-server` (built on vLLM) or run local demos for validation using the `tt-metalium` models container.

## TT-Inference-Server & vLLM

The `tt-inference-server` project provides a production-ready wrapper around **vLLM**, optimized for Tenstorrent's Wormhole and Blackhole architectures.

### Prerequisites

*   **Docker:** Required for containerized deployment [core/getting-started/vLLM-servers.md 37-45](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L37-L45)
*   **Hugging Face Token:** Gated models (e.g., Llama 3.3) require a `read` permission token exported as `HF_TOKEN`[core/getting-started/vLLM-servers.md 76-86](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L76-L86)
*   **Mesh Topology (Wormhole Only):** Multi-card systems like the T3000 (QuietBox/LoudBox) must have the mesh configured via `tt-topology -l mesh`[core/getting-started/vLLM-servers.md 27-35](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L27-L35)

### Hardware Configuration

Deployment requires selecting a specific device profile which sets the `DEVICE` and `MODEL` environment variables.

| Hardware Target | `DEVICE` String | Recommended Model |
| --- | --- | --- |
| **TT-QuietBox (Blackhole)** | `p150x4` | Llama-3.3-70B-Instruct |
| **TT-QuietBox (Wormhole)** | `t3k` | Llama-3.3-70B-Instruct |
| **n150s / n150d** | `n150` | Llama-3.1-8B-Instruct |
| **p150a / p150b** | `p150` | Llama-3.1-8B-Instruct |

## Model Demos

For developers validating hardware or testing specific model architectures without a full API server, Tenstorrent provides a containerized **Model Demos** environment.

### Setup and Execution

The demos are managed via the `tt-installer` script and run in an ephemeral container.

1.   **Install Demo Container:** Run the installer with the `--install-metalium-models-container` flag [core/getting-started/model-demos.md 30-32](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/model-demos.md?plain=1#L30-L32)
2.   **Enter Environment:** Use the `tt-metalium-models` command to launch the interactive shell [core/getting-started/model-demos.md 36-38](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/model-demos.md?plain=1#L36-L38)
3.   **Verify Hardware Access:** Execute a basic matrix multiplication/exponentiation test: `python ttnn/ttnn/examples/usage/run_op_on_device.py`[core/getting-started/model-demos.md 50-52](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/model-demos.md?plain=1#L50-L52)

### Supported Model Catalog

The demos container includes source code and execution scripts for:

*   **LLMs:** Llama 2/3, Falcon.
*   **Vision:** ViT, ResNet, Object Detection (YOLO).
*   **Generative:** Stable Diffusion.
*   **Speech:** Whisper (Speech-to-Text).

Sources: [core/getting-started/model-demos.md 57-68](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/model-demos.md?plain=1#L57-L68)[core/systems/quietbox/quietbox-bh-2/support-bh-2.md 10-13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/quietbox/quietbox-bh-2/support-bh-2.md?plain=1#L10-L13)

## Implementation Detail: Blackhole Inference

In the **TT-QuietBox 2**, inference is powered by two `p300c` cards, totaling four Blackhole ASICs [core/systems/quietbox/quietbox-bh-2/specifications.md 35](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/quietbox/quietbox-bh-2/specifications.md?plain=1#L35-L35) These chips are interconnected via **Warp400** links using Samtec ARP6 cables, creating a unified memory pool of 128GB GDDR6 [core/systems/quietbox/quietbox-bh-2/specifications.md 49-67](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/quietbox/quietbox-bh-2/specifications.md?plain=1#L49-L67)

### Key Classes and Functions

*   **`ttnn.Device`**: Represents the physical Blackhole or Wormhole accelerator in Python.
*   **`tt-smi`**: Used to verify device recognition (e.g., seeing 4 accelerators in a QuietBox 2) [core/systems/quietbox/quietbox-bh-2/setup.md 154-160](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/systems/quietbox/quietbox-bh-2/setup.md?plain=1#L154-L160)
*   **`tt-topology`**: Configures the `mesh` for multi-device communication [core/getting-started/vLLM-servers.md 33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/vLLM-servers.md?plain=1#L33-L33)
