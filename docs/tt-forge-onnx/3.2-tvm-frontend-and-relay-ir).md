This page documents the two groups of graph optimization passes applied in the Forge compilation pipeline: C++ passes operating on the `ForgeGraphModule` graph representation, and Python-level TVM Relay passes operating on the Relay IR. Both groups run **before** MLIR lowering.

For the MLIR lowering step that consumes the optimized graph, see [MLIR Lowering](https://deepwiki.com/tenstorrent/tt-forge-onnx/3.3-mlir-backend-and-code-generation). For the broader compilation pipeline that invokes these passes, see [Compilation Pipeline](https://deepwiki.com/tenstorrent/tt-forge-onnx/3.1-compilation-pipeline-overview). For information about TVM integration in general, see [TVM Integration](https://deepwiki.com/tenstorrent/tt-forge-onnx/4.1-tvm-integration-and-relay-ir).

### `erase_inverse_ops`

**File:**`forge/csrc/passes/erase_inverse_ops.hpp` / `erase_inverse_ops.cpp`

This pass finds pairs of consecutive tensor manipulation operations (TM ops) that are inverses of each other, and removes both. The most common case is a `Transpose` immediately followed by another `Transpose` with the same axes, or a `Reshape` followed by a `Reshape` back to the original shape.

The pass uses `are_compatible_ops` from `commute_utils.hpp` to determine if two ops are inverses. The function checks:

1.   Both nodes are TM ops (e.g., `Reshape`, `Transpose`, `Unsqueeze`, `Squeeze`).
2.   The shape entering `a` matches the shape produced by `b`, accounting for any broadcast volumes on edges.
3.   The specific attribute matching function for that op type confirms the operations are inverse (see **`match_fns`** below).

The return value is `true` if any change was made. The calling loop in the pipeline re-runs this pass until it returns `false`.

**Example: Transpose cancellation**

In a graph of the form:

```
matmul0 → transpose0 → add → post_transpose → output
matmul1 → transpose1 ↗
```

All three `Transpose` nodes cancel, leaving:

```
matmul0 → add → output
matmul1 ↗
```

Sources: [forge/csrc/test/passes/test_erase_inverse_ops.cpp 50-63](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/test/passes/test_erase_inverse_ops.cpp#L50-L63)[forge/csrc/passes/commute_utils.cpp 191-238](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/passes/commute_utils.cpp#L191-L238)

### `replace_incommutable_patterns`

**File:**`forge/csrc/passes/replace_incommutable_patterns.cpp`

Some graph patterns cannot be commuted through directly by `erase_inverse_ops`. This pass identifies these patterns and restructures the graph by inserting inverse operations around the blocking node, enabling `erase_inverse_ops` to succeed in a subsequent iteration.

The primary pattern handled is a `Concatenate` along dimension `-2` (the y-dim) whose operands are `Reshape` nodes whose x-dim (last dimension) differs from the commute shape. This is detected by `is_y_dim_concat_with_changed_x_dim`.

**What the pass does (via `attempt_replace_downward_pattern`):**

1.   Inserts inverse `Reshape` nodes on each input edge of the blocking `Concatenate`.
2.   Updates the shape of the `Concatenate` node itself.
3.   Adds a `golden_transform` attribute (a `Reshape`) to track the intended shape for verification.
4.   Inserts `Reshape` clone nodes on each output edge of the `Concatenate` to restore the expected shape downstream.

After this restructuring, `erase_inverse_ops` can eliminate the newly inserted inverse pairs.

Sources: [forge/csrc/passes/replace_incommutable_patterns.cpp 21-120](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/passes/replace_incommutable_patterns.cpp#L21-L120)[forge/csrc/test/passes/test_erase_inverse_ops.cpp 218-248](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/test/passes/test_erase_inverse_ops.cpp#L218-L248)

### `insert_inverse_on_inputs` / `insert_inverse_on_outputs`

**Files:**`forge/csrc/passes/insert_inverse_on_io.hpp` / `insert_inverse_on_io.cpp`

When `erase_inverse_ops` cannot cancel an inverse pair because the intermediate value is also consumed by a graph output or a fork edge, these passes handle the boundary cases:

*   **`insert_inverse_on_outputs`**: Inserts the necessary inverse operation on output edges so that the consumer receives correctly-shaped data after the main inverse pair is erased.
*   **`insert_inverse_on_inputs`**: Inserts inverse operations on input edges where a TM op needs to be pushed upstream past a module input boundary.

Both functions return `true` if any modification was made. In the `erase_transpose_fork` test case, when `add` has both a `post_transpose` consumer and a direct output consumer, `insert_inverse_on_outputs` places a single `Transpose` on `out0` after `erase_inverse_ops` removes the main transpose pairs.

Sources: [forge/csrc/test/passes/test_erase_inverse_ops.cpp 65-101](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/csrc/test/passes/test_erase_inverse_ops.cpp#L65-L101)

### `commute_utils` — Helper Functions

**Files:**`forge/csrc/passes/commute_utils.hpp`, `forge/csrc/passes/commute_utils.cpp`

These utilities are shared across all C++ optimization passes.

**Key functions:**

| Function | Purpose |
| --- | --- |
| `can_commute_through_dim` | Checks whether `initial_op` (a `Reshape` or `Transpose`) can commute through a given dimension index, returning the new dimension index after commutation |
| `can_commute_reshape_through_dim` | Validates reshape commutation by comparing volumes above and below the target dimension in both the input and output shapes |
| `commute_through_concat` | Determines if a TM op can be pushed through a `Concatenate` node, optionally inserting clone nodes on inputs |
| `commute_through_select` | Determines if a TM op can be pushed through a `Select` (index/slice) node |
| `are_compatible_ops` | Returns `true` if op `a` followed by op `b` forms an inverse pair; accounts for broadcast volumes via `are_inverse_with_broadcast` |
| `are_inverse_with_broadcast` | Checks whether two shapes are inverse given a known broadcast volume on one edge |
| `total_broadcast_volume` | Computes the product of all broadcast sizes on a given graph edge |
| `shape_of_only_operand` | Returns the shape of the single operand of a unary op, used for inverse checking |

## TVM Relay Passes

These passes are defined as `DFPatternCallback` subclasses in `forge/forge/tvm_calls/relay/op/forge_passes.py` and are applied to the TVM Relay graph during the TVM compilation phase, before `ForgeGraphModule` is constructed.

Each subclass defines:

*   A **pattern** built from `wildcard()`, `is_op()`, `is_constant()`, etc.
*   A **`callback(pre, post, node_map)`** method that rewrites matched subgraphs.

### Pass Summary Table

| Class | Pattern Matched | Transformation |
| --- | --- | --- |
| `ConvertLayout` | `nn.conv2d`, `nn.max_pool2d`, `nn.avg_pool2d`, `nn.conv2d_transpose`, global/adaptive pooling, `image.resize2d` in NHWC layout | Converts HWIO/HWOI kernel layout to OIHW for NHWC convolutions |
| `ResolveConvChannels` | `transpose → transpose → conv/pool → transpose → transpose` | Removes four transposes and flips `data_layout` between NCHW and NHWC |
| `FuseConvAndPoolPadding` | `nn.pad → nn.conv2d` or `nn.pad → nn.max_pool2d` with zero constant pad | Absorbs `nn.pad` into conv/pool padding attributes; skips asymmetric maxpool or non-constant padding |
| `DecomposeStack` | `expand_dims → expand_dims → tuple → concatenate → reshape` | Replaces expand_dims+concat pattern with `relay.stack` + `relay.reshape` |
| `DecomposeRoll` | `gather` with constant indices forming a roll permutation | Decomposes into two `strided_slice` calls + `concatenate` |
| `DecomposeReverse` | `reverse` | Decomposes into `adv_index` (with reversed arange indices) + optional transposes |
| `DecomposeDynamicResize2d` | `dyn.image.resize2d` consumed by `add`, `subtract`, or `multiply` | Replaces dynamic resize with static `image.resize2d` using the consumer's inferred shape |
| `DecomposeMultiAxisMax` | `max` with more than one reduction axis | Replaces multi-axis max with sequential single-axis `max` calls |
| `RemoveCast` | Any `cast` op | Removes the cast and passes through the operand directly |

### Key Pass Details

#### `ConvertLayout`

Handles the case where a framework model uses `NHWC` data layout with `HWIO` or `HWOI` kernel layouts, which TVM may not canonicalize automatically. The `callback` detects this by inspecting `data_layout` and `kernel_layout` attributes, transposes the weight tensor to `OIHW`, and re-creates the `nn.conv2d` call with the corrected kernel layout.

Sources: [forge/forge/tvm_calls/relay/op/forge_passes.py 19-97](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/tvm_calls/relay/op/forge_passes.py#L19-L97)

#### `ResolveConvChannels`

Matches a double-transpose fence around a conv or pool: `T → T → op → T → T`. Since two identical transposes cancel, this pattern reduces to running the op with the opposite `data_layout`. The pass re-creates the op with flipped layout and removes all four transposes.

Sources: [forge/forge/tvm_calls/relay/op/forge_passes.py 99-158](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/tvm_calls/relay/op/forge_passes.py#L99-L158)

#### `FuseConvAndPoolPadding`

When a `nn.pad` with zero fill immediately precedes a `nn.conv2d` or `nn.max_pool2d` that has no existing padding, the pad values are merged into the op's `padding` attribute. Asymmetric padding for `nn.max_pool2d` and `edge`/`reflect` pad modes are excluded from fusion.

Sources: [forge/forge/tvm_calls/relay/op/forge_passes.py 160-215](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/tvm_calls/relay/op/forge_passes.py#L160-L215)

#### `DecomposeStack`

TensorFlow's `tf.stack` can appear in Relay as a sequence of `expand_dims → concatenate → reshape`. This pass detects that pattern and replaces it with the canonical `relay.stack` call followed by a reshape, which maps more cleanly to Forge's operation set.

Sources: [forge/forge/tvm_calls/relay/op/forge_passes.py 383-414](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/forge/forge/tvm_calls/relay/op/forge_passes.py#L383-L414)
