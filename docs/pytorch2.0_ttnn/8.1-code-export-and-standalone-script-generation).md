## Purpose and Scope

The code export system generates standalone Python scripts from compiled PyTorch graphs, enabling accuracy debugging and performance profiling outside the main compilation pipeline. This system is primarily used by developers to isolate and reproduce compilation issues, compare ATen and TTNN operation outputs, and profile TTNN execution without the full PyTorch compilation overhead.

For information about the core compilation passes that produce the graphs used by this system, see [Operation Lowering (ToTtPass)](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/2.2-operation-lowering-(tottpass)) and [Data Movement and Alignment (AddDataMovePass)](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/2.3-data-movement-and-alignment-(adddatamovepass)). For debugging guidance on using exported code, see [Debugging and Problem Solving](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/8.4-debugging-and-problem-solving).

**Sources**: [tools/export_code.py 1-932](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L1-L932)

## System Overview

The code export system is invoked through the `TorchTtnnOption.export_code` configuration parameter, which accepts two modes:

| Mode | Purpose | Output |
| --- | --- | --- |
| `"accuracy"` | Compare ATen and TTNN operation outputs | Script with accuracy checks between operation pairs |
| `"profiling"` | Generate tracy-compatible profiling code | Script with profiling instrumentation for TTNN operations |

The system processes the compiled graphs stored in `TorchTtnnOption._aten_fx_graphs` and `TorchTtnnOption._ttnn_fx_graphs`, along with serialized inputs from `TorchTtnnOption._all_inputs`, to produce two files:

1.   `{model_name}_code.py` - Standalone executable Python script
2.   `{model_name}_inputs.pickle` - Compressed input data (weights, biases, arguments)

**Sources**: [tools/export_code.py 32-35](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L32-L35)[tools/export_code.py 889-932](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L889-L932)

## Operation Mapping: ATen to TTNN

The system establishes a 1-to-many mapping between ATen operations and their corresponding TTNN operation sequences. This mapping is crucial for accuracy validation.

The mapping process uses `_compute_key()` to create unique identifiers from node metadata:

`# Simplified from tools/export_code.py:200-214key = str(node.meta["seq_nr"]) + node.meta["original_aten"]._name + str(tensor_meta)`
**Mapping Strategy**:

1.   **Key Generation**: Each ATen node receives a unique key based on sequence number, operation name, and tensor metadata [tools/export_code.py 200-214](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L200-L214)
2.   **TTNN Collection**: All TTNN nodes with matching keys are collected into lists [tools/export_code.py 217-249](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L217-L249)
3.   **Last Node Comparison**: Only the final TTNN node in each sequence is compared against the ATen output [tools/export_code.py 252-279](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L252-L279)

**Sources**: [tools/export_code.py 200-249](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L200-L249)[tools/export_code.py 252-279](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L252-L279)

## Code Generation Process

### Forward Function Assembly

Each graph pair (ATen and TTNN) generates a forward function with signature `def forward_{chunk_idx}_{graph_idx}(args..., device):`. The code generation interleaves ATen and TTNN operations for accuracy comparison.

**Code Generation Rules**:

1.   **Accuracy Mode**: Emit both ATen and TTNN code with accuracy checks [tools/export_code.py 438-468](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L438-L468)
2.   **Profiling Mode**: Emit only TTNN code with periodic profiler dumps [tools/export_code.py 451-453](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L451-L453)
3.   **Special Nodes**: Handle `get_attr` by embedding tensor constants [tools/export_code.py 306-318](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L306-L318)
4.   **Wrapper Functions**: Extract and include wrapper function source code [tools/export_code.py 326-335](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L326-L335)

**Node to Python Conversion**:

The `_node_to_python_code()` function converts FX nodes to executable Python statements [tools/export_code.py 295-359](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L295-L359):

| Node Type | Conversion |
| --- | --- |
| `placeholder` | Skipped (handled in function signature) |
| `get_attr` | Embed tensor data as `torch.tensor(...)` |
| `call_function` | Convert to `result = func(args, kwargs)` |
| `output` | Convert to `return args` |

**Sources**: [tools/export_code.py 362-469](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L362-L469)[tools/export_code.py 295-359](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L295-L359)

*   Detects `SymInt` instances

## Special Case Handling

### Tangents and Gradient Flow

Training graphs include tangent nodes that represent gradients. The system tracks tangent metadata to correctly wire outputs to subsequent forward functions.

**Tangent Collection**: `_collect_tangents()` identifies output nodes marked as gradients using metadata from `OutputType`[tools/export_code.py 812-850](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L812-L850):

*   Locates first primal node in outputs
*   Extracts nodes before primals
*   Checks `tangents` metadata flag
*   Stores in `TangentsInfo(chunk_idx, graph_idx, tangents)`

**Renaming Strategy**:

1.   **Outputs**: Append `_{chunk_idx}_{graph_idx}` suffix to outputs with tangents [tools/export_code.py 164-197](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L164-L197)
2.   **Arguments**: Match tangent arguments to previous outputs by length and position [tools/export_code.py 90-161](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L90-L161)

**Sources**: [tools/export_code.py 805-850](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L805-L850)[tools/export_code.py 90-197](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L90-L197)

### Clone Node and Input Aliasing

Graph breakage can result in clone nodes that alias previous outputs. The system tracks these relationships to avoid duplicate placeholder arguments.

**Clone-Primal Mapping**[tools/export_code.py 853-886](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L853-L886):

`# Simplified logicfor idx, clone in enumerate(clones):    if clone is not None:        for primal in reversed(primals[0:idx]):            if primal is not None:                primals_from_clones[clone] = primal                break`
**Replacement During Processing**:

When a clone placeholder is encountered in the TTNN graph, it is replaced with its corresponding primal node [tools/export_code.py 227-235](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L227-L235):

`if node == first_clone and first_clone in clone_nodes:    node.replace_all_uses_with(        clone_nodes[first_clone],        delete_user_cb=lambda node: node != clone_nodes[first_clone]    )`
**Sources**: [tools/export_code.py 853-886](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L853-L886)[tools/export_code.py 227-235](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L227-L235)

## Usage Example

### Enabling Code Export

Configure `TorchTtnnOption.export_code` before compilation:

`# From test examplesoption = torch_ttnn.TorchTtnnOption(device=device)option.export_code = "accuracy"  # or "profiling" model = torch.compile(model, backend=torch_ttnn.backend, options=option)result = model(inputs)`
### Generated File Locations

Files are saved to subdirectories based on mode [tools/export_code.py 674-676](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L674-L676):

*   Accuracy: `tests/export_code/accuracy/{model_name}_code.py`
*   Profiling: `tests/export_code/profiling/{model_name}_code.py`

### Running Exported Scripts

`# Accuracy debuggingpython tests/export_code/accuracy/model_code.py # Profiling with tracypython tests/export_code/profiling/model_code.py`
**Sources**: [tools/export_code.py 674-716](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L674-L716)

## Implementation Details

### Wrapper Function Extraction

The system extracts source code for wrapper functions to include in the generated script [tools/export_code.py 326-335](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L326-L335):

`if not opname.startswith("aten.") and not opname.startswith("ttnn."):    lines = inspect.getsource(node.target)    wrapper_funcs.add(lines)`
Decorators are removed during extraction [tools/export_code.py 580-586](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L580-L586):

`for i, line in enumerate(func_lines):    if line.lstrip().startswith("def "):        break    if line.lstrip().startswith("@"):        func_lines[i] = ""`
### TTNN Constant Replacement

Custom FX builtins representing TTNN constants are replaced with actual TTNN references [tools/export_code.py 344-358](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L344-L358):

| Placeholder String | Replacement |
| --- | --- |
| `ttnn_Specified_Device` | `device` |
| `ttnn_TILE_LAYOUT` | `ttnn.TILE_LAYOUT` |
| `ttnn_ROW_MAJOR_LAYOUT` | `ttnn.ROW_MAJOR_LAYOUT` |
| `ttnn_L1_MEMORY_CONFIG` | `ttnn.L1_MEMORY_CONFIG` |
| `ttnn_DRAM_MEMORY_CONFIG` | `ttnn.DRAM_MEMORY_CONFIG` |
| `ttnn_bfloat16` | `ttnn.bfloat16` |
| `ttnn_uint32` | `ttnn.uint32` |

### Multi-Device Support

For data parallel execution, the generated script includes mesh device initialization [tools/export_code.py 659-670](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L659-L670):

`if torch_ttnn_option.data_parallel:    device = ttnn.open_mesh_device(        ttnn.MeshShape(1, 2),        dispatch_core_config=dispatch_core_config,        l1_small_size=l1_small_size    )else:    device = ttnn.open_device(        device_id=0,        dispatch_core_config=dispatch_core_config,        l1_small_size=l1_small_size    )`
