# Grayskull

Relevant source files
*   [crates/luwen-if/src/chip/grayskull.rs](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs)

This page documents the Grayskull chip implementation in the Luwen framework. Grayskull is one of three chip architectures supported by Luwen, alongside Wormhole ([Wormhole](https://deepwiki.com/tenstorrent/luwen/3.2-wormhole)) and Blackhole ([Blackhole](https://deepwiki.com/tenstorrent/luwen/3.4-blackhole)). It provides specific interfaces and implementations for interacting with Tenstorrent's Grayskull AI accelerator chips.

## Overview

Grayskull is implemented as a concrete implementation of the `ChipImpl` trait ([Common Interface (ChipImpl)](https://deepwiki.com/tenstorrent/luwen/3.1-common-interface-(chipimpl))), providing Grayskull-specific functionality while maintaining a consistent interface with other chip architectures in the Luwen system.

Grayskull chips follow a specific initialization sequence:

1.   ARC (Accelerator Resource Controller) initialization
2.   DRAM initialization
3.   Telemetry setup

### Telemetry

Grayskull implements comprehensive telemetry collection, providing detailed information about the chip's state:

1.   Retrieves telemetry offset via `get_telemetry_offset`
2.   Reads telemetry values from AXI registers
3.   Returns a structured `Telemetry` object

The telemetry data includes:

| Category | Telemetry Data |
| --- | --- |
| Identifiers | Device ID, Board ID, ASIC RO/IDD |
| Firmware | ARC firmware versions, SPI bootrom version, TT flash version, FW bundle version |
| Memory | DDR speed, DDR status |
| Status | PCIE status, ARC health, Fault information |
| Performance | Clock frequencies (AI, AXI, ARC), Throttler status |
| Temperature | ASIC temperature, VREG temperature, ASIC thermal monitors |
| Power | Vcore, TDP, TDC, ASIC power, VDD limits |
| Miscellaneous | Fan speed, Boot date, Runtime seconds |

## Implementation Details

### Key Trait Implementations

Grayskull implements the following traits:

1.   `ChipImpl` - Core chip implementation interface

    *   `update_init_state` - Updates chip initialization state
    *   `get_arch` - Returns `Arch::Grayskull`
    *   `arc_msg` - Sends messages to the ARC
    *   `get_neighbouring_chips` - Returns an empty vector (Grayskull doesn't support this)
    *   `get_telemetry` - Collects chip telemetry

2.   `HlComms` - High-level communications interface

    *   `comms_obj` - Returns references to communication objects

Sources: [crates/luwen-if/src/chip/grayskull.rs 199-209](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs#L199-L209)[crates/luwen-if/src/chip/grayskull.rs 235-807](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs#L235-L807)

### Grayskull-Specific Features

Compared to other chips in the Luwen ecosystem, Grayskull has several distinctive characteristics:

1.   No Ethernet components (unlike Wormhole and Blackhole)
2.   Simplified SPI operations without ARC-based support
3.   No support for neighboring chips
4.   Specific initialization sequence focused on ARC and DRAM

Sources: [crates/luwen-if/src/chip/grayskull.rs 142-160](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs#L142-L160)[crates/luwen-if/src/chip/grayskull.rs 611](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs#L611-L611)

## Error Handling

Grayskull implements robust error handling throughout its operations:

1.   ARC messaging safety checks report specific error conditions
2.   Initialization process handles and categorizes different error types
3.   Telemetry collection includes version checks and fallbacks

Examples of handled error conditions:

*   ARC not ready (various states)
*   Unsupported firmware versions
*   AXI communication errors
*   DRAM training failures

Sources: [crates/luwen-if/src/chip/grayskull.rs 49-140](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs#L49-L140)[crates/luwen-if/src/chip/grayskull.rs 236-584](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/grayskull.rs#L236-L584)
