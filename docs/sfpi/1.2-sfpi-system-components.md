## Purpose and Scope

This document provides an architectural overview of the SFPI (Special Function Processing Unit Interface) system components and their interactions. It describes the layered structure from the high-level C++ programming interface down to hardware intrinsics, the compiler toolchain infrastructure, and the build/test systems that support the complete SFPI ecosystem.

For detailed information about the programming interface and vector types, see [Core SFPI Programming Interface](https://deepwiki.com/tenstorrent/sfpi/2-core-sfpi-programming-interface). For toolchain build procedures, see [Building the SFPI Toolchain](https://deepwiki.com/tenstorrent/sfpi/5-building-the-sfpi-toolchain). For testing infrastructure details, see [Testing and Validation](https://deepwiki.com/tenstorrent/sfpi/6-testing-and-validation).

## Core Programming Interface Layer

The core interface consists of three primary header files that user code directly interacts with:

### Main Interface: sfpi.h

The `sfpi.h` header provides the fundamental C++ wrapper around SFPU operations. Key components include:

| --- | --- | --- |

| **Register Files** | `dst_reg` (__vDReg), `l_reg` (__vLReg) | [include/sfpi.h 36-269](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L36-L269) |

| **Control Flow** | `v_if`, `v_elseif`, `v_else`, `v_endif` | [include/sfpi.h 54-70](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L54-L70) |
| **Conditional Execution** | `__vCond`, `__vCCCtrl` classes | [include/sfpi.h 639-715](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L639-L715) |

The interface uses conditional compilation to select platform-specific implementations:

**Sources:**[include/sfpi.h 127-131](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L127-L131)[include/sfpi.h 122-125](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L122-L125)

### FP16 Conversion Interface: sfpi_fp16.h

Provides conversion between FP32 and two FP16 formats:

| Class | Format | Use Case |
| --- | --- | --- |
| `s2vFloat16` | Base class | Generic FP16 conversion |
| `s2vFloat16a` | fp16a format | High-precision FP16 |
| `s2vFloat16b` | fp16b format | Standard FP16 |

The conversion is performed at compile-time using inline functions:

*   `fp32_to_fp16a()` - [include/sfpi_fp16.h 90-118](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L90-L118) handles denorms and rounding
*   `fp32_to_fp16b()` - [include/sfpi_fp16.h 120-130](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L120-L130) simple 16-bit truncation

**Sources:**[include/sfpi_fp16.h 24-48](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L24-L48)[include/sfpi_fp16.h 50-69](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L50-L69)

### Instruction Replay Interface: lltt.h

The Low-Level Tenstorrent instruction interface provides direct control over instruction recording and replay:

| Function | Purpose | Implementation |
| --- | --- | --- |
| `record<E>(start, length)` | Record instruction sequence | [include/lltt.h 27-29](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/lltt.h#L27-L29) |
| `replay(start, length)` | Replay recorded sequence | [include/lltt.h 31-33](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/lltt.h#L31-L33) |

## Hardware Abstraction Layer Structure

Each supported architecture (Wormhole, Blackhole, Grayskull) implements a three-tier abstraction:

### Hardware Intrinsics Layer (sfpi_hw.h)

Platform-specific builtin functions that map directly to SFPU instructions:

| Architecture | Operations | Key Differences |
| --- | --- | --- |
| Wormhole | 51 intrinsics | `ADDR_MODE_NOINC = 3` |
| Blackhole | 54 intrinsics | `ADDR_MODE_NOINC = 7`, adds `sfparecip`, `sfpmul24` |
| Grayskull | Base set | Basic hardware definitions |

Common intrinsic categories (referenced in [include/sfpi.h 727-732](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L727-L732)):

*   **Arithmetic**: `__builtin_rvtt_sfpadd`, `__builtin_rvtt_sfpmul`, `__builtin_rvtt_sfpmad`
*   **Memory**: `__builtin_rvtt_sfpload`, `__builtin_rvtt_sfpstore`, `__builtin_rvtt_sfpxloadi`
*   **Bitwise**: `__builtin_rvtt_sfpand`, `__builtin_rvtt_sfpor`, `__builtin_rvtt_sfpxor`
*   **Conditional**: `__builtin_rvtt_sfpxfcmps`, `__builtin_rvtt_sfpxicmps`, `__builtin_rvtt_sfpxvif`

**Sources:**[include/sfpi.h 127-131](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L127-L131)[include/sfpi_fp16.h 15-19](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L15-L19)

### Implementation Layer (sfpi_imp.h)

Implements the C++ semantics for vector classes using hardware intrinsics:

Class hierarchy from [include/sfpi.h 326-636](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L326-L636):

*   `__vBase` - Base class with initialization tracking [include/sfpi.h 326-341](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L326-L341)
*   `vFloat` - Floating-point vector operations [include/sfpi.h 344-397](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L344-L397)
*   `__vIntBase` - Integer base with bitwise operations [include/sfpi.h 400-458](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L400-L458)
*   `vInt` - Signed integer vectors [include/sfpi.h 460-548](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L460-L548)
*   `vUInt` - Unsigned integer vectors [include/sfpi.h 551-636](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L551-L636)

**Sources:**[include/sfpi.h 326-636](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L326-L636)[include/sfpi.h 145-151](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L145-L151)

### Math Library Layer (sfpi_lib.h)

Provides high-level mathematical operations built on intrinsics. Common functions include:

| Function Category | Examples | Purpose |
| --- | --- | --- |
| **Lookup Tables** | `lut()`, `lut2()`, `lut_sign()` | Table-based approximations |
| **Float Manipulation** | `setexp()`, `setman()`, `exexp()`, `exman()` | IEEE-754 field access |
| **Type Conversions** | `int32_to_float()`, `float_to_fp16a()`, `float_to_uint8()` | Format conversions |
| **Vector Operations** | `subvec_transp()`, `vec_swap()`, `vec_min_max()` | Data movement |

Blackhole extends the library with additional functions:

*   `rand()` - Random number generation
*   `approx_recip()` - Fast reciprocal approximation
*   `approx_exp()` - Fast exponential approximation

**Sources:**[README.md 6-13](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L6-L13)

## GCC Toolchain Components

The SFPI system includes a complete GCC-based RISC-V toolchain with Tenstorrent-specific enhancements:

### Toolchain Compilation Pipeline

The build process follows a multi-stage bootstrap approach:

1.   **Binutils** - Assembler, linker, and binary utilities
2.   **GCC Stage 1** - Minimal compiler without standard libraries
3.   **Newlib** - Standard C library compiled with Stage 1 GCC
4.   **GCC Stage 2** - Full compiler with complete standard library support

**Sources:**[README.md 56-84](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L56-L84)[README.md 4-13](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L4-L13)

### Build System Scripts

| Script | Purpose | Key Operations |
| --- | --- | --- |
| `scripts/build.sh` | Main build orchestration | Configure toolchain, compile, run tests |
| `scripts/release.sh` | Package creation | Create .txz, .deb, .rpm packages |
| `scripts/sfpi-info.sh` | Version management | Generate version information |

The build system generates a `src-hashes` file to track source tree state for reproducible builds.

**Sources:**[README.md 62-84](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L62-L84)[README.md 102-127](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L102-L127)

## Testing Infrastructure

The SFPI testing system consists of multiple validation layers:

### SFPI-Specific Tests

Located in `tests/{wormhole,blackhole,grayskull}/sfpi/`:

*   **coverage.cc** - Tests each hardware intrinsic individually
*   **ckernel.cc** - Tests complete kernel functions (exp, log, activations)
*   **regmov.cc** - Validates compiler's register move optimization pass
*   **kernels.cc** - Integration tests with numerical validation

Each test has corresponding gold standard files in the `gold/` subdirectory for assembly comparison.

**Sources:**[README.md 86-101](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L86-L101)[README.md 12](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L12-L12)

### Test Execution and Validation

Test execution uses QEMU simulator via wrapper script:

*   `scripts/riscv32-unknown-elf-run` - QEMU wrapper [scripts/riscv32-unknown-elf-run 1-15](https://github.com/tenstorrent/sfpi/blob/ed989b9d/scripts/riscv32-unknown-elf-run#L1-L15)
*   Passes arguments to `qemu-riscv32` for RISC-V user-mode emulation

**Sources:**[scripts/riscv32-unknown-elf-run 1-15](https://github.com/tenstorrent/sfpi/blob/ed989b9d/scripts/riscv32-unknown-elf-run#L1-L15)[README.md 162-212](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L162-L212)

The `xfails/` directory contains expected failure lists:

| File | Purpose |
| --- | --- |
| `gcc.xfail` | Known C compiler test failures |

## Component Interaction Flow

The following diagram illustrates how components interact during kernel compilation and execution:

## CI/CD Integration

The `.github/workflows/build-sfpi.yaml` workflow automates:

1.   **Build Matrix** - Multiple architectures (x86_64, aarch64) and distributions
2.   **Toolchain Compilation** - Executes `scripts/build.sh` with testing flags
3.   **Test Execution** - Runs `--test-tt` for SFPI-specific validation
4.   **Artifact Creation** - Executes `scripts/release.sh` to generate packages
5.   **Artifact Publishing** - Uploads .txz, .deb, .rpm to GitHub releases

Build artifacts include:

*   Compiled toolchain binaries (stripped)
*   SFPI header files (`include/`)
*   Version metadata and checksums
*   Test results and configuration logs

**Sources:**[README.md 129-160](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L129-L160)[README.md 22-36](https://github.com/tenstorrent/sfpi/blob/ed989b9d/README.md?plain=1#L22-L36)

## Release and Distribution

The version management system:

*   Base version from `version` file at repository root
*   Git branch name appended (except for `main` branch)
*   Generated version stored in build directory
*   Used by `tt-metal/sfpi-version.sh` for dependency resolution

Package formats:

*   **.txz** - Compressed tarball with complete toolchain
*   **.deb** - Debian/Ubuntu package (via FPM)
*   **.rpm** - Red Hat/Fedora package (via FPM)
*   **.sha256** - Checksum file for verification
