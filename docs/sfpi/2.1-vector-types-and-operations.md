# Vector Types and Operations

Relevant source files
*   [include/sfpi.h](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h)
*   [include/wormhole/sfpi_imp.h](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_imp.h)

The SFPI system provides three primary vector types for programming the SFPU: `vFloat`, `vInt`, and `vUInt`. These classes represent 64-element vectors and provide C++ operator overloading that maps directly to SFPU hardware instructions at compile time with zero runtime overhead. This page documents the vector type system, their constructors, operators, and how they translate to hardware operations.

For information on FP16 conversions, see page 2.2. For control flow and predicated execution, see page 2.4.

## Overview

The vector types are the workhorse classes of SFPI. They enable natural C++ syntax like `vFloat d = a * b + c;` which compiles directly to efficient SFPU instructions. The type system is designed around a class hierarchy that provides:

*   Type-safe vector operations for floating-point and integer types
*   Automatic mapping to hardware intrinsics during compilation
*   Support for immediate values, register loads, and inter-type conversions
*   Liveness tracking to correctly handle assignments within predicated blocks

## Base Vector Class: `__vBase`

All vector types inherit from `__vBase`, which provides the common infrastructure for vector operations. The class maintains two key fields:

| Field | Type | Purpose |
| --- | --- | --- |
| `v` | `__rvtt_vec_t` | The underlying hardware vector register representation |
| `initialized` | `bool` | Tracks whether the vector has been assigned a value |

The `initialized` flag is critical for liveness tracking in predicated execution. When a vector is assigned in a predicated block (e.g., inside a `v_if`), the compiler must preserve previous values for elements where the predicate is false. The `assign()` method uses `__builtin_rvtt_sfpassign_lv` when `initialized` is true to merge the previous and new values appropriately.

## vFloat: Floating-Point Vectors

The `vFloat` class represents 64-element vectors of 32-bit floating-point values. It is the primary type for mathematical operations in SFPI.

### Constructors

`vFloat` provides multiple constructors for different initialization patterns:

| Constructor | Purpose | Maps To |
| --- | --- | --- |
| `vFloat()` | Default constructor, uninitialized | N/A |
| `vFloat(float f)` | Load immediate float value | `sfpxloadi(SFPLOADI_MOD0_FLOAT, ...)` |
| `vFloat(const __vDReg dreg)` | Load from destination register | `sfpload(SFPLOAD_MOD0_FMT_SRCB, ...)` |
| `vFloat(const __vConstFloat creg)` | Copy from constant register | `sfpassignlreg(...)` |
| `vFloat(const s2vFloat16 f)` | Convert from FP16 | `sfpxloadi(SFPLOADI_MOD0_FLOATA/B, ...)` |
| `vFloat(const __rvtt_vec_t& t)` | Construct from raw vector | Direct assignment |

## Interaction with Registers

While vector types represent values in local registers during computation, they interact with other register types in the SFPU architecture:

### Loading from Destination Registers

The `__vDReg` type represents destination registers, accessed via the global `dst_reg` array. Vector constructors can load from these registers:

`vFloat a(dst_reg[0]);  // Load from destination register 0vInt b(dst_reg[5]);    // Load from destination register 5`
These constructors use `sfpload` intrinsics to move data from destination registers into local registers for computation.

## Type Conversions

Vector types support conversions between different numeric types and between vector and scalar representations.

### Explicit Type Casts

The `__vIntBase` class provides explicit conversion operators to other vector types:

`vInt a = 42;vUInt b = static_cast<vUInt>(a);  // Convert vInt to vUIntvFloat c = static_cast<vFloat>(a); // Convert vInt to vFloat`
These conversions reinterpret the bit pattern without changing the underlying `__rvtt_vec_t` value.

Conditional results (`__vCond`) can be converted to integer vectors, creating a mask where each element is 0 or -1:

`vFloat a = dst_reg[0];vInt mask = vInt(a > 0.0f);  // -1 for positive elements, 0 otherwise`
This uses the `sfpxcondi` intrinsic to convert the condition code result to an integer vector.

## Operator Overloading and Hardware Mapping

The vector types leverage C++ operator overloading to provide natural syntax that compiles to efficient hardware instructions. The key principle is zero-cost abstraction: the C++ operators should compile to the same instructions a developer would write by hand using intrinsics.

### Commutative Operations

Many operators are defined both as member functions and as free functions to support commutative operations:

`vFloat a = 5.0f;vFloat b = dst_reg[0];vFloat c = a + b;  // Uses vFloat::operator+(vFloat)vFloat d = 2.0f + b;  // Uses free function operator+(float, vFloat)`
The free functions (defined at [include/sfpi.h 968-1006](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L968-L1006)) enable natural ordering of operands without performance penalty.

Sources: [include/sfpi.h 968-1006](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L968-L1006)

### Assignment Operators

Assignment operators (`=`, `+=`, `-=`, `*=`, etc.) return the assigned value rather than a reference to enable chaining, but return void in some contexts to prevent misuse in conditionals where predication order matters.

Sources: [include/sfpi.h 360-361](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L360-L361)[include/sfpi.h 479-480](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L479-L480)[include/sfpi.h 572-573](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L572-L573)

### Template Metaprogramming

The integer base class uses template metaprogramming with SFINAE (Substitution Failure Is Not An Error) to provide type-safe operations that work with both `vInt` and `vUInt`:

`template <typename vType, typename std::enable_if_t<std::is_base_of<__vIntBase, vType>::value>* = nullptr>sfpi_inline vType operator&(const vType b) const{    return __builtin_rvtt_sfpand(v, b.get());}`
This ensures bitwise operations return the correct type while sharing implementation.

Sources: [include/sfpi.h 415-428](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L415-L428)[include/sfpi.h 431-457](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L431-L457)

## Liveness Tracking and Predicated Assignment

A critical implementation detail is how vector assignments interact with predicated execution. When a vector variable is assigned inside a `v_if` block, the SFPU hardware must preserve the previous value for elements where the condition is false.

**Assignment Behavior**

The `initialized` flag in `__vBase` tracks whether a vector has been assigned. On the first assignment, the value is directly stored. On subsequent assignments, `sfpassign_lv` is used to merge the previous and new values based on active condition codes.

Sources: [include/sfpi.h 89-101](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L89-L101)[include/sfpi.h 785-789](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi.h#L785-L789)

## Performance Considerations

### Minimize Register Pressure

The SFPU has limited local registers (8 LREGs). Avoid creating excessive temporary vector variables in tight loops:

`// Inefficient: creates many temporariesvFloat temp1 = dst_reg[0] + dst_reg[1];vFloat temp2 = temp1 * dst_reg[2];vFloat temp3 = temp2 + dst_reg[3];dst_reg[4] = temp3;Â // Better: chain operationsdst_reg[4] = (dst_reg[0] + dst_reg[1]) * dst_reg[2] + dst_reg[3];`
### Use In-Place Operators

In-place operators (`+=`, `-=`, `*=`) can be more efficient as they avoid creating temporaries:

`vFloat accumulator = vConst0;for (int i = 0; i < N; i++) {    accumulator += dst_reg[i];  // More efficient than accumulator = accumulator + dst_reg[i]}`
### Leverage Constant Registers

Loading constants from registers is faster than loading immediate values:

## Summary

The Core Interface Layer of SFPI provides an intuitive C++ interface to the SFPU hardware, enabling vector operations with a syntax similar to scalar C++ code. It maps directly to hardware instructions with no runtime overhead, and provides facilities for predicated execution and register management.
