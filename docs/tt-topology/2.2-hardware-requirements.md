## Supported Hardware Platforms

tt-topology supports a specific set of Tenstorrent hardware platforms. The tool performs hardware detection and validation during startup to ensure compatibility.

### Supported Board Types

The following table shows the currently supported hardware platforms:

| Board Type | Hardware Description | Supported Configurations |
| --- | --- | --- |
| `n300` | Wormhole N300 PCIe Cards | Standard topologies (linear, mesh, torus, isolated) |
| `n150` | Wormhole N150 PCIe Cards | Standard topologies + Octopus mode with Galaxy |
| `GALAXY` | Wormhole 4U Galaxy Systems | Octopus mode configurations only |

## Driver Requirements

### Tenstorrent Kernel Module Driver

tt-topology requires the Tenstorrent kernel module driver (`tt-kmd`) to communicate with hardware:

*   **Minimum Version**: tt-kmd 2.0.0+
*   **Detection**: Performed via `get_driver_version()` function
*   **Failure Behavior**: Tool exits with error if no driver detected

Sources: [tt_topology/tt_topology.py 411-418](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/tt_topology.py#L411-L418)[CHANGELOG.md 20](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/CHANGELOG.md?plain=1#L20-L20)

## System Requirements

### Python Environment

*   **Python Version**: Python 3.7+
*   **Package Manager**: pip3
*   **Rust Toolchain**: Required for pyluwen dependency compilation

### Memory and Access Requirements

Different operational modes require different levels of hardware access:

| Operation Mode | Ethernet Access Required | Local Only | Description |
| --- | --- | --- | --- |
| `--list` | Yes (`ignore_ethernet=False`) | No | Full topology detection |
| `--octopus` | Yes (`ignore_ethernet=False`) | No | Galaxy configuration |
| Standard Flash | No (`ignore_ethernet=True`) | Yes | PCIe chip configuration |

Sources: [tt_topology/tt_topology.py 428-436](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/tt_topology.py#L428-L436)

## Unsupported Hardware

The following hardware platforms are explicitly **not supported** and will cause the tool to exit with warnings:

### Blackhole (BH) Hardware

*   BH PCIe cards
*   BH 6U Galaxy systems

### Wormhole 6U Systems

*   WH 6U Galaxy systems

Sources: [README.md 13-17](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/README.md?plain=1#L13-L17)[tt_topology/tt_topology.py 463-470](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/tt_topology.py#L463-L470)

## Hardware Configuration Requirements

### Standard Topology Configurations

For `n300` and `n150` boards in standard mode:

*   **Minimum Devices**: 1 PCIe card
*   **Connection Types**: Ethernet cables between cards
*   **Physical Verification**: Tool detects and validates physical connections
*   **Reset Capability**: Requires board-level reset functionality

### Octopus Mode Requirements

For Galaxy configurations:

*   **Local Devices**: 4-8 `n150` boards
*   **Remote Devices**: 1-2 `GALAXY` systems
*   **Configuration File**: Valid reset JSON with mobo/credo mappings
*   **Training Time**: Extended periods for QSFP link training

Sources: [tt_topology/tt_topology.py 288-402](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/tt_topology.py#L288-L402)[README.md 158-232](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/README.md?plain=1#L158-L232)
