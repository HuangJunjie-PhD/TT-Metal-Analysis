The tt-awesome website utilizes a hierarchical Nunjucks template system driven by Eleventy to render a three-pane interactive interface. This system transforms static JSON entry data into a dynamic single-page application (SPA) experience, managing state through CSS classes and vanilla JavaScript while maintaining SEO-friendly static markup.

## Template Hierarchy & Data Flow

The layout is structured around a primary shell that includes specific panes based on the application state. Data flows from the `entries/*.json` files into the Eleventy data cascade, where it is enriched with GitHub metadata and categorized before reaching the templates.

### Data to Template Mapping

| Template | Purpose | Key Data Sources |
| --- | --- | --- |
| `base.njk` | HTML boilerplate, analytics, and global assets. | `entries`, `categories`, `embedTypes` |
| `home.njk` | Landing page featuring a category grid and stats. | `entries`, `categories` |
| `sidebar.njk` | Primary navigation and category selection. | `categories` |
| `entry-list.njk` | Middle pane showing filtered project summaries. | `entries` |
| `releases-list.njk` | Middle pane showing chronological release feed. | `recentReleases` |
| `entry-detail.njk` | Right pane showing full project details and links. | `entries` |

## Core Layout Components

### 1. Base Shell (`base.njk`)

The `base.njk` file serves as the global wrapper. It handles:

*   **Metadata:** Generates Open Graph and Twitter cards using the project count from `entries | length`[src/_includes/base.njk 14-23](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base.njk#L14-L23)
*   **Analytics:** Integrates PostHog using the project's specific API key [src/_includes/base.njk 53-75](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base.njk#L53-L75)
*   **Asset Injection:** Uses the `inlineFile` shortcode to embed critical CSS and JS directly into the HTML for performance [src/_includes/base.njk 45-79](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base.njk#L45-L79)
*   **Conditional Embeds:** Dynamically loads external stylesheets (e.g., `tensix-viz.css`) only if a project requiring that embed type exists in the current build [src/_includes/base.njk 48-50](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base.njk#L48-L50)

### 2. Home View (`home.njk`)

The landing page provides a high-level entry point.

*   **Category Grid:** Iterates through `categories` to create cards [src/_includes/home.njk 25-44](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/home.njk#L25-L44)
*   **Featured Projects:** Uses the `diversifiedFeatured` filter to select a representative project for each category card's "feature" slot [src/_includes/home.njk 23-38](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/home.njk#L23-L38)
*   **Statistics:** Displays live counts of total projects and categories [src/_includes/home.njk 6-19](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/home.njk#L6-L19)

### 3. Entry List Pane (`entry-list.njk`)

This template renders the middle column when a category is active.

*   **Search Attributes:** Each `.entry-row` includes a `data-search` attribute containing a lowercased concatenation of name, description, and tags for client-side filtering [src/_includes/entry-list.njk 21](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/entry-list.njk#L21-L21)
*   **Affiliation Badges:** Projects are visually tagged as `official`, `affiliated`, or `community`[src/_includes/entry-list.njk 25](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/entry-list.njk#L25-L25)

### 4. Recent Releases Pane (`releases-list.njk`)

Introduced in the Recent Releases feature, this pane provides a chronological feed of project updates.

*   **Data Source:** Consumes `recentReleases.js`, which aggregates the `latestStableRelease` from all entries [src/_data/recentReleases.js 27-52](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/recentReleases.js#L27-L52)
*   **Toggle Logic:** Includes buttons to filter the feed by affiliation (All/Official/Community) [src/_includes/releases-list.njk 7-14](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/releases-list.njk#L7-L14)

## Project Detail & Visual Components

The `entry-detail.njk` template is the most complex component, responsible for rendering the full project metadata and interactive embeds.

### Detail Pane Features

*   **Fork Detection:** If an entry is marked as a fork in `github_meta.json`, a `⑂` badge is rendered linking to the upstream repository [src/_includes/entry-detail.njk 11-14](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/entry-detail.njk#L11-L14)
*   **Release History:** Displays a "LATEST" callout for the most recent stable release, followed by a compact list of previous versions [src/_includes/entry-detail.njk 56-82](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/entry-detail.njk#L56-L82)
*   **Package Installation:** Renders copy-pasteable install commands for `pypi`, `cargo`, and `apt` based on the `packages` array in the entry JSON [src/_includes/entry-detail.njk 91-106](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/entry-detail.njk#L91-L106)

### Editorial Embeds (tensix-viz)

The system supports editorial embeds defined in entry JSON files (e.g., `entries/dev-tools/tensix-viz.json`).

*   **Declaration:** An entry specifies an `embed.type`[entries/dev-tools/tensix-viz.json 32-34](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/dev-tools/tensix-viz.json#L32-L34)
*   **Discovery:**`src/_data/embedTypes.js` scans all entries at build time to determine which embeds are active [src/_data/embedTypes.js 22-35](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_data/embedTypes.js#L22-L35)
*   **Rendering:**`entry-detail.njk` uses a whitelist to include the corresponding Nunjucks partial from `src/_includes/embeds/`[src/_includes/entry-detail.njk 122-126](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/entry-detail.njk#L122-L126)

## Analytics & Tracking

PostHog integration is baked into `base.njk` to track community engagement across categories and entries.

*   **Initialization:** The script is loaded asynchronously and initialized with the Tenstorrent documentation suite's project key [src/_includes/base.njk 53-75](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base.njk#L53-L75)
*   **Configuration:**
    *   `api_host`: `https://us.i.posthog.com`
    *   `person_profiles`: `identified_only` (to respect privacy for anonymous browsers) [src/_includes/base.njk 70-74](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/_includes/base.njk#L70-L74)
