## Purpose and Scope

The configuration parsers are responsible for reading and validating YAML-based configuration files that define workloads, hardware architectures, and their mappings within the Polaris simulation framework. These parsers convert raw configuration data into validated Python objects that can be consumed by the simulation engine.

For information about the overall configuration management system, see [Configuration Management](https://deepwiki.com/tenstorrent/polaris/2.3-configuration-management). For details about workload to architecture mappings, see [Workload to Architecture Mapping](https://deepwiki.com/tenstorrent/polaris/2.3.1-workload-to-architecture-mapping).

## Parser System Overview

The configuration parsing system consists of three primary parsers that handle different types of configuration data:

*   **Architecture Specification Parser** (`get_arspec_from_yaml`) - Processes hardware architecture definitions
*   **Workload Specification Parser** (`get_wlspec_from_yaml`) - Processes workload definitions
*   **Workload Mapping Specification Parser** (`get_wlmapspec_from_yaml`) - Processes operation-to-resource mappings

## Workload Specification Parser

The `get_wlspec_from_yaml` function processes workload definitions, creating validated workload objects organized by name.

### Workload Parser Structure

### Workload Database Structure

The parser returns a dictionary mapping workload names to lists of `TypeWorkload` objects:

`wldb: dict[str, list[TypeWorkload]]`
Each workload entry supports multiple instances with different configurations, allowing for parameterized workload definitions.

## Workload Mapping Specification Parser

The `get_wlmapspec_from_yaml` function processes operation-to-resource mappings, data type specifications, and operation fusion rules.

### Mapping Parser Output

### Required Configuration Fields

| Field | Purpose | Output |
| --- | --- | --- |
| `op_data_type_spec` | Operation data type mappings | `op2dt` list |
| `op_rsrc_spec` | Operation resource assignments | `op2rsrc` dict |
| `op_removal_spec` | Operations to remove | `null_ops` list |
| `op_fusion_spec` | Operation fusion rules | `op_fusion_list` |

### Operation Processing Details

The parser processes operations with specific transformations:

*   **Case Normalization**: Operations are converted to uppercase, resources to lowercase
*   **Global Types**: The `global_type` field creates a wildcard mapping (`*`)
*   **Resource Mapping**: Each operation is mapped to a specific compute pipe
*   **Fusion Rules**: Operations are grouped into fusion sequences

## Excel Configuration Parser

The `parse_xlsx_config` function processes Excel worksheets containing configuration data in tabular format.

### Excel Parser Structure

### Excel Configuration Features

*   **Column Filtering**: Ignores columns named `comments` or `remarks`
*   **Duplicate Detection**: Validates that architecture configurations are not duplicated
*   **Dynamic Configuration**: Creates `XlsxConfig` objects for each column
*   **Parameter Mapping**: Maps row parameters to column values
