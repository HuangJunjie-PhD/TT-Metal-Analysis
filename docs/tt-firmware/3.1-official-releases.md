## Purpose and Scope

This page documents the officially supported firmware releases for Tenstorrent hardware. It covers available firmware bundles, versioning scheme, and migration procedures between versions. For information about experimental variants with modifications for specific issues, see [Experimental Bundles](https://deepwiki.com/tenstorrent/tt-firmware/3.2-experimental-bundles).

## Available Firmware

The tt-firmware repository provides official firmware bundles that support all Tenstorrent hardware platforms. Each bundle contains firmware images for Grayskull, Wormhole, and Blackhole boards.

| Firmware | Description |
| --- | --- |
| `fw_pack-18.2.0.0.fwbundle` | Current official firmware package (supports Grayskull, Wormhole, and Blackhole boards) |
| `fw_pack-80.18.1.0.fwbundle` | Previous firmware package (superseded by 18.2.0.0) |
| `latest.fwbundle` | Symbolic link to the most recent firmware bundle |

## Installation Considerations

The firmware bundles are distributed in binary form due to third-party IP restrictions. The README explains:

> Tenstorrent products contain IP from third parties, we have agreements with these third parties which keeps us from distributing source code related to their property.

While specific installation procedures are not detailed in the repository, the firmware is designed to work with the Tenstorrent User Mode Driver (UMD). For more information about interacting with the firmware, refer to the UMD repository mentioned in the README.

Sources: [README.md 15-21](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L15-L21)

## Release Features and Changelog

### Current Release (18.2.0.0)

Key features and improvements in the current official release:

1.   **Blackhole ERISC Firmware Update to v1.4.0**

    *   Added ETH mailbox with two message types: 
        *   `LINK_STATUS_CHECK`: Checks Ethernet link status
        *   `RELEASE_CORE`: Releases control of RISC0 to run function at specified L1 address

2.   **Virtual UART Enhancements**

    *   Now enabled by default for Blackhole firmware bundles
    *   Creates in-memory virtual UART for firmware observability
    *   Compatible with `tt-console` for viewing debug messages (`printk()` and `LOG_*()`)

3.   **Stability Improvements**

    *   Improved link training sequence for better loopback success rates
    *   Fixed synchronization issue in BMFW that could cause deadlocks/enumeration failures
    *   Enhanced SMC I2C recovery (99.6% success rate for reset and re-enumeration)
    *   PCIe Maximum Payload Size (MPS) now set by TT-KMD, improving VM stability

Sources: [README.md 31-49](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L31-L49)

## Release Distribution Model

Due to the binary-only nature of the releases, the firmware packages are distributed as complete bundles rather than as source code. This model ensures compatibility across all supported hardware while respecting third-party IP restrictions.
