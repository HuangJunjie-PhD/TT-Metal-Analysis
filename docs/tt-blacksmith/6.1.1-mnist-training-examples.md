## Purpose and Scope

This page documents the MNIST training experiments in the TT-Blacksmith repository, which serve as practical examples demonstrating single-chip and multi-chip training strategies on Tenstorrent hardware. These examples showcase the training infrastructure's flexibility by implementing the same MNIST classification task with different parallelization strategies and model architectures.

For information about the underlying training infrastructure and manager classes, see [Training Framework](https://deepwiki.com/tenstorrent/tt-blacksmith/5-training-framework). For details on distributed training strategies, see [Distributed Training](https://deepwiki.com/tenstorrent/tt-blacksmith/8-distributed-training).

The MNIST experiments are located in [blacksmith/experiments/torch/mnist/](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/) and provide reference implementations for:

*   Single-chip training with linear and convolutional models
*   Data parallel training across multiple chips
*   Tensor parallel training with model sharding
*   Integration with the configuration system and manager classes

Data parallel training distributes batches across multiple chips, with each chip maintaining a full copy of the model. The `DeviceManager` handles batch sharding automatically based on the mesh configuration.

**Key Implementation Details:**

The data parallel implementation [blacksmith/experiments/torch/mnist/data_parallel/test_mnist_training.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/data_parallel/test_mnist_training.py) differs from single-chip training in these areas:

1.   **Batch Preparation:** Uses `device_manager.prepare_batch()` to shard inputs along the data axis [test_mnist_training.py 108](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/test_mnist_training.py#L108-L108)

## Tensor Parallel Training

Tensor parallel training shards the model weights across multiple chips, requiring each chip to perform only a portion of the computation. The MNIST tensor parallel example demonstrates the column-row-column sharding pattern.

### Model Sharding Configuration

**Sharding Pattern Explanation:**

The configuration [blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training_tp.yaml 58-61](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training_tp.yaml#L58-L61) defines regex patterns that match module names:

| Pattern | Module | Weight Shape | Sharding Spec | Effect |
| --- | --- | --- | --- | --- |
| `linear_relu_stack\.0$` | First layer | `[512, 784]` | `[null, "model"]` | Columns sharded: each chip gets `[256, 784]` |
| `linear_relu_stack\.2$` | Second layer | `[512, 512]` | `["model", null]` | Rows sharded: each chip gets `[512, 256]` |
| `linear_relu_stack\.4$` | Output layer | `[10, 512]` | `[null, "model"]` | Columns sharded: each chip gets `[5, 512]` |

**DeviceManager Tensor Parallelism Application:**

The `DeviceManager._apply_tensor_parallelism()` method [blacksmith/tools/device_manager.py 89-110](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/tools/device_manager.py#L89-L110) applies these patterns:

1.   Retrieves `model_sharding_patterns` from config [device_manager.py 93](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/device_manager.py#L93-L93)
2.   Iterates over `model.named_modules()`[device_manager.py 97](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/device_manager.py#L97-L97)
3.   For each module with weights, checks regex patterns [device_manager.py 101-106](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/device_manager.py#L101-L106)
4.   Marks matching module weights with `xs.mark_sharding()`[device_manager.py 106](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/device_manager.py#L106-L106)
5.   Synchronizes with `torch_xla.sync()`[device_manager.py 109](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/device_manager.py#L109-L109)

**Training Loop Modifications:**

The tensor parallel training script [blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training.py) includes:

1.   **Model Sharding per Batch:** Calls `device_manager.shard_model(model)` at each training step [test_mnist_training.py 114](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/test_mnist_training.py#L114-L114)
2.   **Batch Preparation:** Uses `prepare_batch()` even though data axis is 1 (no data sharding) [test_mnist_training.py 116](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/test_mnist_training.py#L116-L116)
3.   **Custom Loss Function:**`cross_entropy_loss()` maintains `[1, 1]` shape for tensor parallel operations [tensor_parallel/utils.py 8-17](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tensor_parallel/utils.py#L8-L17)
4.   **Compile Options:** Sets XLA compile options for export [test_mnist_training.py 196-201](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/test_mnist_training.py#L196-L201)

**Sources:**[blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training.py)[blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training_tp.yaml](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/tensor_parallel/test_mnist_training_tp.yaml)[blacksmith/experiments/torch/mnist/tensor_parallel/utils.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/tensor_parallel/utils.py)[blacksmith/tools/device_manager.py 75-110](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/tools/device_manager.py#L75-L110)

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `mesh_shape` | `Optional[list[int]]` | `None` | Device mesh dimensions `[x, y]` |

**Tensor Parallel:** Set `mesh_shape: [1, 2]` and `mesh_axis_names: ["data", "model"]` with second dimension >1 for model axis, plus define `model_sharding_patterns`.

## Training Workflow

The standardized training workflow is consistent across all MNIST variants:

**Validation Function:**

The `validate()` function [blacksmith/experiments/torch/mnist/test_mnist_training.py 24-58](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/test_mnist_training.py#L24-L58) computes metrics without gradient updates:

1.   Sets model to eval mode (caller responsibility)
2.   Iterates through validation DataLoader
3.   Performs forward pass on each batch
4.   Accumulates loss and computes accuracy (argmax predictions)
5.   Returns average loss and accuracy

For multi-chip variants, the validation function also handles appropriate sharding through `device_manager.prepare_batch()` or `device_manager.shard_model()`.

## Model Architectures

### MNISTLinear

The `MNISTLinear` model is a simple feedforward network defined in [blacksmith/models/torch/mnist/mnist_linear.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/models/torch/mnist/mnist_linear.py):

```
Input (784) → Linear(784, 512) → ReLU 
            → Linear(512, 512) → ReLU 
            → Linear(512, 10)  → Output (10)
```

The architecture is configured through `TrainingConfig`:

*   `input_size`: Flattened image dimensions (default 784 for 28×28)
*   `hidden_size`: Intermediate layer width (default 512)
*   `output_size`: Number of classes (default 10 for digits 0-9)
*   `bias`: Whether to include bias terms (default False)

The model is instantiated in training scripts with: [blacksmith/experiments/torch/mnist/test_mnist_training.py 70](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/test_mnist_training.py#L70-L70)

### MNISTCNN

The `MNISTCNN` model [blacksmith/models/torch/mnist/mnist_cnn.py 11-37](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/models/torch/mnist/mnist_cnn.py#L11-L37) implements a convolutional architecture:

```
Input (1, 28, 28) 
    → Conv2d(1, 32, kernel=3) → ReLU
    → Conv2d(32, 64, kernel=3) → ReLU
    → MaxPool2d(2)
    → Dropout(0.25)
    → Flatten
    → Linear(12*12*64, 128) → ReLU
    → Dropout(0.5)
    → Linear(128, 10) → Output (10)
```

Configuration parameters:

*   `conv1_channels`: First conv output channels (default 32)
*   `conv2_channels`: Second conv output channels (default 64)
*   `kernel_size`: Convolution kernel size (default 3)
*   `fc1_size`: First FC layer size (default 128)
*   `dropout1_rate`: Dropout after pooling (default 0.25)
*   `dropout2_rate`: Dropout before output (default 0.5)

The CNN model processes images in their 2D spatial format rather than flattened vectors.

**Sources:**[blacksmith/models/torch/mnist/mnist_linear.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/models/torch/mnist/mnist_linear.py)[blacksmith/models/torch/mnist/mnist_cnn.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/models/torch/mnist/mnist_cnn.py)
