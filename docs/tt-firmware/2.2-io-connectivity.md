# I/O Connectivity

Relevant source files
*   [README.md](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1)

## Purpose and Scope

This document describes how the Tenstorrent firmware manages external I/O connections, including PCIe, Ethernet, and DRAM interfaces. The firmware's System Management Processor ensures these connections are established and maintained throughout the device's operation. This page focuses specifically on the connectivity aspects of the firmware; for information about the System Management Processor itself, see [System Management Processor](https://deepwiki.com/tenstorrent/tt-firmware/2.1-system-management-processor), and for power and thermal management, see [Power and Thermal Management](https://deepwiki.com/tenstorrent/tt-firmware/2.3-power-and-thermal-management).

Sources: [README.md 8-13](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L8-L13)

## I/O Connectivity Overview

The Tenstorrent firmware is responsible for managing multiple types of external connections that allow the AI accelerator chips to communicate with host systems and other devices. These connections must be properly initialized during boot and maintained during operation to ensure stable performance.

## Supported I/O Interfaces

The firmware supports three primary I/O interfaces across all board types (Grayskull, Wormhole, and Blackhole):

| Interface | Purpose | Key Features |
| --- | --- | --- |
| PCIe | Host connectivity | Device enumeration, payload size management |
| Ethernet | Network connectivity | Link status monitoring, training sequence |
| DRAM | Local memory access | Memory initialization, access control |

## PCIe Connectivity

The PCIe interface provides the primary communication channel between the Tenstorrent device and the host system. The firmware manages PCIe link training, device enumeration, and performance configuration.

### PCIe Configuration Management

The firmware handles critical PCIe configuration parameters, including Maximum Payload Size (MPS). Recent updates have shifted MPS configuration responsibility to the TT-KMD (Kernel Mode Driver), improving virtual machine stability.

## Ethernet Connectivity

Ethernet connectivity is managed by the firmware to provide network communication capabilities for Tenstorrent devices. The firmware handles Ethernet link establishment, monitoring, and recovery.

### Link Training and Monitoring

The firmware includes sophisticated link training sequences to establish reliable Ethernet connections. Recent updates have improved the success rate of link training, particularly in loopback testing scenarios.

## DRAM Connectivity

The firmware manages DRAM initialization and access, ensuring the accelerator chips can efficiently use local memory resources.

### Board-Specific Memory Management

Each board type (Grayskull, Wormhole, and Blackhole) has specific DRAM configurations that the firmware must properly initialize and manage. The firmware handles memory timing, refresh rates, and access controls appropriate to each board type.

Sources: [README.md 11](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L11-L11)

## Blackhole-Specific I/O Features

The Blackhole board has several specialized I/O features managed by the firmware:

### ERISC Firmware and ETH Mailbox

The Blackhole ERISC Firmware (v1.4.0) includes an ETH mailbox system with two primary message types:

1.   **LINK_STATUS_CHECK**: A message type that checks for Ethernet link status
2.   **RELEASE_CORE**: A message type that releases control of RISC0 to run functions at specified L1 addresses

_Diagram: Blackhole ETH Mailbox architecture_

Sources: [README.md 35-38](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L35-L38)

### Virtual UART for Debugging

Blackhole firmware includes a Virtual UART feature that creates an in-memory UART for firmware observability and debugging. This allows developers to view `printk()` and `LOG_*()` messages from the host using the `tt-console` utility.

_Diagram: Virtual UART debugging flow_

## I/O Connectivity Reliability Features

### SMC I2C Recovery

The firmware includes an improved SMC I2C recovery function that enhances reset and re-enumeration success rates to 99.6%. This feature is critical for maintaining stable I/O connections, particularly after system resets or power cycles.

### Synchronization Improvements

Recent firmware updates have addressed synchronization issues that could potentially lead to deadlocks or failures during device enumeration. These improvements ensure more reliable I/O connectivity across all supported interfaces.

Sources: [README.md 47-48](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L47-L48)
