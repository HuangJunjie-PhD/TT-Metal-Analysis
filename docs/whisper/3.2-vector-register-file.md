## Purpose and Scope

This document describes the **Vector Register File** implementation in Whisper, which provides storage and access mechanisms for the RISC-V Vector Extension (V). The vector register file is implemented by the `VecRegs` class and manages 32 vector registers (v0-v31), each with configurable width (VLEN). This page covers register organization, element width (SEW), group multipliers (LMUL), configuration validation, and element access patterns.

For information about vector instruction execution and operations, see [Vector Extension (V)](https://deepwiki.com/tenstorrent/whisper/5.1-vector-extension-(v)). For CSR-level configuration (VL, VTYPE, VSTART), see [Control and Status Registers (CSRs)](https://deepwiki.com/tenstorrent/whisper/3.1-control-and-status-registers-(csrs)).

## Vector Register Organization

### Register File Structure

The vector register file consists of 32 vector registers (v0-v31), where each register has a configurable width called VLEN (Vector Length). VLEN must be a power of 2 and typically ranges from 128 to 4096 bits (16 to 512 bytes).

**Sources:**[VecRegs.hpp 124-146](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L124-L146)[VecRegs.cpp 140-144](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.cpp#L140-L144)

The `VecRegs` class stores all vector register data in a contiguous byte array:

| Field | Type | Description |
| --- | --- | --- |
| `regCount_` | `uint32_t` | Number of vector registers (always 32) |
| `bytesPerReg_` | `uint32_t` | VLEN / 8 (bytes per register) |
| `bytesInRegFile_` | `uint32_t` | Total bytes in register file (regCount × bytesPerReg) |
| `data_` | `std::vector<uint8_t>` | Contiguous storage for all registers |

**Sources:**[VecRegs.hpp 901-906](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L901-L906)

## Group Multiplier (LMUL)

### GroupMultiplier Enumeration

The group multiplier (LMUL) allows instructions to operate on register groups, effectively increasing or decreasing the number of elements processed. The `GroupMultiplier` enum represents LMUL values:

**Sources:**[VecRegs.hpp 37-47](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L37-L47)

### Scaled Group Representation

To avoid fractional arithmetic, group multipliers are stored as `groupX8_` (scaled by 8):

*   LMUL = 8 → `groupX8_` = 64
*   LMUL = 1 → `groupX8_` = 8
*   LMUL = 1/8 → `groupX8_` = 1

The static method `groupMultiplierX8(GroupMultiplier)` performs this conversion:

**Sources:**[VecRegs.hpp 354-361](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L354-L361)

## VLMAX and Element Count

### VLMAX Calculation

VLMAX is the maximum number of elements that can be processed with a given SEW/LMUL configuration:

**VLMAX = (LMUL × VLEN) / SEW**

In code terms:

```
vlmax = groupX8_ × bytesPerReg_ / sewInBits_
```

Where:

*   `groupX8_` = LMUL scaled by 8
*   `bytesPerReg_` = VLEN / 8
*   `sewInBits_` = SEW in bits (multiply by 8 to get result in terms of 8-bit units)

**Sources:**[VecRegs.hpp 365-377](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L365-L377)

### Element Count (VL)

The actual number of active elements is determined by VL (Vector Length), which is set by `vsetvl`, `vsetvli`, or `vsetivli` instructions and cached in `elems_`:

*   VL ≤ VLMAX
*   VL is read from the `VL` CSR and cached in `VecRegs::elems_`

**Sources:**[VecRegs.hpp 847-852](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L847-L852)[vector.cpp 949-1046](https://github.com/tenstorrent/whisper/blob/733fd921/vector.cpp#L949-L1046)

## Mask Registers and Masking

### Mask Register v0

Register v0 serves a special purpose as the **mask register**. Each bit in v0 controls whether the corresponding element in a vector operation is active (bit=1) or inactive (bit=0).

**Sources:**[VecRegs.hpp 686-698](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L686-L698)

### Mask Read/Write Operations

| Method | Description |
| --- | --- |
| `isActive(maskReg, ix)` | Returns true if bit `ix` of mask register is 1 |
| `writeMaskRegister(maskReg, i, value)` | Sets bit `i` to `value` (0 or 1) |
| `readMaskRegister(maskReg, i, value)` | Reads bit `i` into `value` |

**Sources:**[VecRegs.hpp 686-817](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L686-L817)

Bit operations use byte-level indexing:

```
byteIx = i >> 3        // i / 8
bitIx = i & 7          // i % 8
byte = data_[maskReg * bytesPerReg_ + byteIx]
bit = (byte >> bitIx) & 1
```

## Register Group Overlap Rules

### Destination-Source Overlap

Vector instructions with different element widths in source and destination operands have specific overlap constraints. The helper functions in `vector.cpp` enforce these rules:

**Sources:**[vector.cpp 503-525](https://github.com/tenstorrent/whisper/blob/733fd921/vector.cpp#L503-L525)

Example overlap scenarios:

*   **Widening operations** (vwadd.vv): `vd` has 2×SEW, `vs1`/`vs2` have SEW 
    *   Overlap allowed only at the highest destination registers

*   **Narrowing operations** (vnsrl.wv): `vd` has SEW, `vs1` has 2×SEW 
    *   Overlap allowed only if `vs1 == vd`

**Sources:**[vector.cpp 659-809](https://github.com/tenstorrent/whisper/blob/733fd921/vector.cpp#L659-L809)

## Additional Configuration Options

### Configuration Flags

The `VecRegs` class supports various configuration flags for behavioral customization:

| Flag | Description |
| --- | --- |
| `trapVtype_` | If true, trap on illegal VTYPE configurations; otherwise set VTYPE.VILL |
| `alwaysMarkDirty_` | If true, always mark VS dirty when instruction would write vector register |
| `updateWholeMask_` | If true, mask-producing instructions update entire destination register |
| `maskAgnOnes_` / `tailAgnOnes_` | If true, write all-1s to agnostic elements |
| `legalizeVsetvlAvl_` / `legalizeVsetvliAvl_` | If true, reduce VL to VLMAX when VL > VLMAX |
| `legalizeForEgs_` | If true, legalize VL/VSTART to multiples of EGS for crypto instructions |
| `partialSegUpdate_` | If true, partially update segment loads on exception |
| `vmvrIgnoreVill_` | If true, allow vmv*r.v instructions when VTYPE.VILL is set |

**Sources:**[VecRegs.hpp 421-478](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L421-L478)[VecRegs.hpp 917-925](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L917-L925)

### Floating-Point Reduction Configuration

Per-SEW configuration for unordered floating-point sum reductions:

*   `fpUnorderedSumTreeRed_[sew]`: Use tree reduction vs ordered reduction
*   `fpUnorderedSumCanonical_[sew]`: Apply NaN canonicalization

**Sources:**[VecRegs.hpp 448-469](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.hpp#L448-L469)[VecRegs.cpp 45-47](https://github.com/tenstorrent/whisper/blob/733fd921/VecRegs.cpp#L45-L47)
