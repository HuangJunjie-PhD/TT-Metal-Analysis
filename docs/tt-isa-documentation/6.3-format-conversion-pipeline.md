This page documents the format conversion functions implemented in the Unpacker and Packer execution units. These functions convert data between L1 memory formats and internal computation formats, handle BFP (Block Floating-Point) expansion/compression, and perform bit rearrangement to match internal register layouts.

For information about the data types themselves, see [Data Type Catalog](https://deepwiki.com/tenstorrent/tt-isa-documentation/6.2-data-type-catalog). For details about BFP compression formats and algorithms, see [Compression and BFP Formats](https://deepwiki.com/tenstorrent/tt-isa-documentation/6.4-compression-and-bfp-formats). For information about floating-point computation models, see [Floating-Point Models and FMA](https://deepwiki.com/tenstorrent/tt-isa-documentation/6.1-floating-point-models-and-fma).

## Unpacker Format Conversion

The Unpackers perform format conversion as data moves from L1 memory to internal computation registers (`SrcA`, `SrcB`, or `Dst`). The available conversions depend on the destination register.

### Supported Conversion Paths

When unpacking to **SrcA or SrcB** (19-bit internal format):

| L1 Format | → | SrcA/SrcB Format | Configuration |
| --- | --- | --- | --- |
| FP32 or TF32 | → | TF32, BF16, or FP16 | `InDataFormat=FP32`, `OutDataFormat=TF32/BF16/FP16` |
| BF16 | → | TF32 or BF16 | `InDataFormat=OutDataFormat=BF16` |
| BFP8/4/2 | → | TF32 or BF16 | `InDataFormat=OutDataFormat=BFP8/4/2` |
| BFP8a/4a/2a | → | FP16 | `InDataFormat=OutDataFormat=BFP8a/4a/2a` |
| FP16 or FP8 | → | FP16 | `InDataFormat=OutDataFormat=FP16/FP8` |
| INT8 (sign-magnitude) | → | "Integer 8" overlay | `InDataFormat=BFP8`, requires `Force_shared_exp` |
| UINT8 | → | "Integer 8" overlay | `InDataFormat=INT8`, configure `SrcAUnsigned`/`SrcBUnsigned` |
| INT16 | → | "Integer 16" opaque | `InDataFormat=OutDataFormat=INT16` |

When unpacking to **Dst** (16/32-bit rearranged format):

| L1 Format | → | Dst Format | Configuration |
| --- | --- | --- | --- |
| FP32 or TF32 | → | FP32, BF16, or FP16 | `InDataFormat=FP32`, `OutDataFormat=FP32/BF16/FP16` |
| BF16 | → | BF16 | `InDataFormat=OutDataFormat=BF16` |
| BFP8/4/2 | → | BF16 | `InDataFormat=OutDataFormat=BFP8/4/2` |
| BFP8a/4a/2a | → | FP16 | `InDataFormat=OutDataFormat=BFP8a/4a/2a` |
| FP16 or FP8 | → | FP16 | `InDataFormat=OutDataFormat=FP16/FP8` |
| INT32 | → | "Integer 32" | `InDataFormat=OutDataFormat=INT32` |
| INT16 | → | "Integer 16" opaque | `InDataFormat=OutDataFormat=INT16` |
| INT8/UINT8 | → | "Integer 8" overlay | `InDataFormat=OutDataFormat=INT8`, configure `SrcAUnsigned` |

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/Unpackers/FormatConversion.md 1-81](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/Unpackers/FormatConversion.md?plain=1#L1-L81)

Block Floating-Point formats store a shared exponent for multiple mantissas, requiring expansion during unpacking.

### BFP8ToBF16 Expansion

Converts an 8-bit BFP mantissa to BF16 by normalizing and applying the shared exponent:

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 495-506](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L495-L506)

### BFP8aToFP16 Expansion

Similar to BFP8ToBF16, but produces FP16 output with different exponent placement:

The key differences from `BFP8ToBF16`:

*   Exponent positioned at bits [14:10] instead of [14:7]
*   Mantissa positioned at bits [9:3] instead of [6:1]
*   Returns `0xFC00` for negative zero instead of `0xFF80`
*   Validates that exponent fits in 5 bits (bits [7:5] must be 0)

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 508-520](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L508-L520)

## Bit Rearrangement for Internal Formats

The internal register formats use non-standard bit layouts that require rearrangement.

### SrcA/SrcB Format (19-bit)

SrcA and SrcB use a 19-bit format with fields arranged as: `Sign, Mantissa, Exponent`

The `WriteSrcTF32()` function performs the rearrangement:

```
Input:  Sign[18] Exp[17:8] Man[7:0]
Output: Sign[18] Man[17:8] Exp[7:0]

Fields are swapped: (Sign << 18) | (Man << 8) | Exp
```

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 550-565](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L550-L565)

### Dst Format (16/32-bit)

Dst also uses rearranged layouts:

The rearrangement functions swap mantissa and exponent fields:

**WriteDstFP16():**

```
Input:  Sign[15] Exp[14:10] Man[9:0]
Output: Sign[15] Man[14:5]  Exp[4:0]
```

**WriteDstBF16():**

```
Input:  Sign[15] Exp[14:7] Man[6:0]
Output: Sign[15] Man[14:8] Exp[7:0]
```

**WriteDstFP32():**

```
Input:  Sign[31] Exp[30:23] Man[22:0]
Output: (WriteDstBF16 on hi 16 bits) | (lo 16 bits unchanged)
Result: Sign[31] ManHi[30:24] Exp[23:16] ManLo[15:0]
```

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 522-548](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L522-L548)

## Integer Format Handling

Integer formats receive special treatment to overlay them onto floating-point register formats.

### INT8 Conversion

INT8 can be either sign-magnitude or unsigned. The conversion produces an "Integer 8" value overlaid onto the FP16 format:

The configuration fields `ALU_FORMAT_SPEC_REG0_SrcAUnsigned` and `ALU_FORMAT_SPEC_REG0_SrcBUnsigned` determine whether INT8 is treated as unsigned.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 459-471](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L459-L471)

### INT16 and INT32

*   **INT16:** Treated as opaque 16-bit data. When unpacking to SrcA/SrcB, bits are rearranged: `((DatumBits & 0xFF00) << 3) | (DatumBits & 0xFF)`. When unpacking to Dst, passed through unchanged.

*   **INT32:** Only supported when unpacking to Dst. Uses the same bit rearrangement as FP32 (`WriteDstFP32`).

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md 487-489](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/UNPACR_Regular.md?plain=1#L487-L489)

## Configuration

Format conversion behavior is controlled by two primary configuration fields:

### Configuration Fields

| Field | Location | Purpose |
| --- | --- | --- |
| `InDataFormat` | `THCON_SEC[unpacker].REG0_TileDescriptor.InDataFormat` or `THCON_SEC[unpacker].REG7_Unpack_data_format_cntx[context]` | Specifies L1 memory format |
| `OutDataFormat` | `THCON_SEC[unpacker].REG2_Out_data_format` or `THCON_SEC[unpacker].REG7_Unpack_out_data_format_cntx[context]` | Specifies target internal format |

### Format Encoding

Both fields use the same 4-bit encoding:

| Bits [3:2] | Bits [1:0] |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
|  |  | `11` | `10` | `01` | `00` |
| `00` |  | BFP4a | BFP8a | FP16 | FP32 |
| `01` |  | BFP4 | BFP8 | BF16 | TF32 |
| `10` |  | BFP2a | FP8 | INT16 | INT32 |
| `11` |  | BFP2 | INT8 | — | — |

### Special Configuration

**For BFP formats without per-tile exponents:**

*   Set `REG2_Force_shared_exp = true`
*   Provide fixed exponent via `FORCED_SHARED_EXP_shared_exp`

**For unsigned INT8:**

*   Set `ALU_FORMAT_SPEC_REG0_SrcAUnsigned = true` (for unpacker 0)
*   Set `ALU_FORMAT_SPEC_REG0_SrcBUnsigned = true` (for unpacker 1)

## Packer Format Conversion

The Packers perform the reverse operations, converting from internal formats back to L1 memory formats. While the unpacker conversion details are provided in the functional model, the packer's format conversion pipeline includes additional features:

*   **Early format conversion:** Initial bit rearrangement from internal to IEEE754-like layout
*   **Late format conversion:** Final adjustments including: 
    *   Edge masking (zeroing out-of-bounds elements)
    *   ReLU activation (zeroing negative values)
    *   Exponent threshold checks
    *   BFP compression (exponent calculation, zero detection)

*   **Compression:** RSI (Row Start Indices) and RLE (Run-Length Encoding) for sparse data

The exact packer conversion functions mirror the unpacker functions in reverse, though implementation details are provided in the packer instruction documentation (see [Packer System](https://deepwiki.com/tenstorrent/tt-isa-documentation/4.6-packer-system-(pacr))).
