## Purpose and Scope

The ERISC (Embedded RISC) Firmware is a specialized component of the Blackhole board firmware stack in Tenstorrent devices. This document details the ERISC Firmware's functionality, features, and integration within the broader Blackhole architecture. The current version as of firmware bundle 18.2.0.0 is v1.4.0.

For information about the ETH Mailbox implementation specifically, see [ETH Mailbox](https://deepwiki.com/tenstorrent/tt-firmware/4.2-eth-mailbox). For details on Virtual UART integration, see [Virtual UART](https://deepwiki.com/tenstorrent/tt-firmware/4.3-virtual-uart).

## Overview

ERISC Firmware runs on Tenstorrent's Blackhole AI accelerator boards, providing specialized functionality for Ethernet communication and processor control. The firmware enables critical communications pathways and hardware control mechanisms specific to the Blackhole architecture.

## Key Features

The ERISC Firmware provides several key capabilities that enhance the functionality and reliability of Blackhole boards:

### ETH Mailbox

Version 1.4.0 introduced the ETH (Ethernet) mailbox, a communication mechanism that supports message-based interactions. This mailbox system processes two primary message types:

1.   **LINK_STATUS_CHECK**: Monitors and verifies the status of Ethernet links
2.   **RELEASE_CORE**: Controls RISC0 processor execution at specified L1 memory addresses

### Link Training Improvements

The v1.4.0 firmware includes significant enhancements to the link training sequence, resulting in higher success rates for loopback test cases. This improvement contributes to overall system stability and communication reliability.

## RISC0 Control Mechanism

The RELEASE_CORE message type provides a sophisticated mechanism for controlling the RISC0 processor on Blackhole boards. This capability enables:

1.   Dynamic release of RISC0 processor control
2.   Execution of functions at specifically designated L1 memory addresses
3.   Runtime reconfiguration of processor behavior without requiring firmware updates

## Integration with Virtual UART

The ERISC Firmware is designed to work seamlessly with the Virtual UART feature, which is now enabled by default in Blackhole firmware bundles. This integration provides:

1.   An in-memory virtual UART for firmware observability
2.   Capture and transmission of debug messages
3.   Access to `printk()` and `LOG_*()` message output via the `tt-console` utility from the host system

## Debugging Capabilities

With the integration of Virtual UART, debugging the ERISC Firmware has become more accessible. Developers can now:

1.   View real-time debug messages via `tt-console`
2.   Monitor Ethernet link status
3.   Trace RISC0 execution control

This improved observability significantly enhances the development and troubleshooting experience when working with Blackhole boards.
