This page documents the JAX-based experiments in TT-Blacksmith, their organization, configuration system, and implementation patterns. JAX experiments demonstrate training neural networks on Tenstorrent hardware using JAX's functional programming paradigm and XLA compilation.

For PyTorch-based experiments, see [PyTorch Experiments](https://deepwiki.com/tenstorrent/tt-blacksmith/6.1-pytorch-experiments). For detailed implementation of MNIST variants, see [JAX MNIST Implementations](https://deepwiki.com/tenstorrent/tt-blacksmith/6.2.1-jax-mnist-implementations). For advanced JAX examples including NeRF, see [JAX NeRF and Advanced Examples](https://deepwiki.com/tenstorrent/tt-blacksmith/6.2.2-jax-nerf-and-advanced-examples).

## Experiment Organization

JAX experiments are located under `blacksmith/experiments/jax/` and organized by model type and parallelization strategy:

```
experiments/jax/
├── mnist/
│   ├── configs.py                    # Configuration classes
│   ├── test_mnist.yaml               # Default configuration
│   ├── single_chip/
│   │   ├── test_pure_jax_mnist.py   # Pure JAX API
│   │   ├── test_flax_mnist.py       # Flax API
│   │   └── test_nnx_mnist.py        # NNX API
│   ├── multi_chip/
│   │   ├── data_parallel/
│   │   │   └── test_pure_jax_mnist.py
│   │   └── tensor_parallel/
│   │       └── test_pure_jax_mnist.py
│   └── logging/
│       ├── wandb_utils.py           # JAX-specific logging
│       └── shlo_ops_logging.py
└── nerf/
    └── ...
```

Each experiment script is self-contained and executable independently, following the pattern:

1.   Load configuration from YAML and CLI
2.   Initialize JAX devices and sharding
3.   Define model and training functions
4.   Execute training loop
5.   Log metrics and checkpoints

**Sources:**[blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py 240-244](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py#L240-L244)[blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py 248-252](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py#L248-L252)

## JAX API Styles

TT-Blacksmith provides three JAX API implementations, each demonstrating different levels of abstraction:

### Pure JAX

The pure JAX implementation uses low-level JAX primitives without any neural network library abstractions.

**Key Characteristics:**

*   Manual parameter initialization using `jax.random.normal`
*   Explicit forward pass function
*   Manual gradient computation with `jax.value_and_grad`
*   Manual parameter updates

**Example Model Definition:**

The MLP model is defined as a pure function [blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py 25-30](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py#L25-L30):

Parameters are tuples of arrays, updated manually [blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py 73-85](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py#L73-L85):

**Sources:**[blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py 1-245](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_pure_jax_mnist.py#L1-L245)

### Flax

Flax provides a higher-level neural network API with structured parameter management through `TrainState`.

**Key Characteristics:**

*   Module-based model definition
*   `TrainState` encapsulates parameters, optimizer state, and apply function
*   Integration with Optax for optimizers
*   Checkpoint serialization via `msgpack_serialize`

**Example Usage:**

Model initialization [blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py 60-80](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py#L60-L80):

Training step [blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py 146-148](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py#L146-L148):

**Sources:**[blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py 1-253](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_flax_mnist.py#L1-L253)[blacksmith/models/jax/mnist/model.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/models/jax/mnist/model.py)

### NNX

NNX (Neural Network eXtensions) is Flax's next-generation API with a more Pythonic, stateful interface.

**Key Characteristics:**

*   Object-oriented model definition with `nnx.Module`
*   Mutable state management with `nnx.Rngs`
*   Built-in metrics via `nnx.MultiMetric`
*   GraphDef/State split for device transfers
*   JIT compilation with `@nnx.jit` decorator

**Example Model Definition:**

NNX uses class-based modules [blacksmith/experiments/jax/mnist/single_chip/test_nnx_mnist.py 59-72](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_nnx_mnist.py#L59-L72):

**State Management for Device Transfers:**

NNX separates immutable graph structure from mutable state [blacksmith/experiments/jax/mnist/single_chip/test_nnx_mnist.py 129-131](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/single_chip/test_nnx_mnist.py#L129-L131):

### Training Step Execution

The training step varies by API style but follows a consistent pattern:

| Step | Pure JAX | Flax | NNX |
| --- | --- | --- | --- |
| **Forward + Grad** | `compute_loss_grads_logits(params, x, y)` | `compute_loss_grads_and_logits(state.params, x, y)` | `training_step(model, metrics, batch)` |
| **Update** | `update(params, grads, lr)` | `optimizer_step(state, grads)` | `optimizer.update(grads)` |
| **Return** | `updated_params` | `updated_state` | `grads, loss, logits` |

## Hardware Considerations

JAX experiments contain several hardware-specific workarounds documented in code comments:

### Known Limitations

| Issue | Workaround | Tracking Link |
| --- | --- | --- |
| Random number generation not supported on TT device | Initialize all parameters on CPU, then transfer | [tt-xla/issues/420](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-xla/issues/420) |
| Batch indexing not supported on TT device | Create batches on CPU, then transfer | [tt-mlir/issues/2309](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-mlir/issues/2309) |
| Argmax not supported in multi-chip context | Compute accuracy on CPU | [tt-mlir/issues/3963](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-mlir/issues/3963) |
| Dataset loading requires CPU | Load entire dataset on CPU | [tt-mlir/issues/2768](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-mlir/issues/2768) |
| Empty mesh in Shardy context | Cannot do on-device batching | [tt-mlir/issues/4636](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-mlir/issues/4636) |
| All-reduce on non-32-divisible dimensions | Use Column-Row-Column strategy | [tt-metal/issues/21987](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-metal/issues/21987) |
| Optimizer state not supported on TT device | Run optimizer.update() on CPU | [tt-xla/issues/342](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/tt-xla/issues/342) |
