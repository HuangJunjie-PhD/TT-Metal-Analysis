This page documents the preprocessing utilities that handle **input aliasing** and **tangent marking** in the PyTorch to TTNN compilation pipeline. These preprocessing steps occur before the graph is lowered to ATEN operations and are essential for correctness when transferring data between host and device.

For information about the overall compilation pipeline, see [Core Compilation System](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/2-core-compilation-system). For information about the AOT Autograd integration, see [Backend Entry Points and Pass Pipeline](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/2.1-backend-entry-points-and-pass-pipeline).

* * *

## Purpose and Scope

Input aliasing handling prevents correctness issues that arise when AOT Autograd optimizes by reusing input storage for outputs. Tangent marking enables proper gradient flow tracking during training by identifying which outputs participate in the backward pass.

These preprocessing utilities operate on high-level torch operations before ATEN lowering:

| Preprocessing Step | Purpose | Entry Point |
| --- | --- | --- |
| Clone Insertion | Prevent input aliasing issues | `insert_clones_for_input_aliasing()` |
| Tangent Marking | Track gradient-participating outputs | `mark_output_as_tangents()` |
| Clone Removal | Clean up temporary clones | `remove_clones_for_input_aliasing()` |

* * *

## Input Aliasing Problem

### The AOT Autograd Optimization Issue

AOT Autograd includes an optimization where if the output storage is determined to be the same as the input storage (i.e., the output is an alias of the input), it returns data directly from the input. This becomes problematic when transferring data between host and device, as the aliasing metadata can cause incorrect data movement operations.

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 8-27](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L8-L27)

### The Clone Insertion Solution

The workaround inserts `torch.ops.aten.clone.default` operations after input placeholder nodes before the graph is fully lowered to ATEN operations. This breaks the aliasing relationship while preserving correctness.

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 8-27](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L8-L27)

* * *

## Clone Insertion Implementation

### Function: `insert_clones_for_input_aliasing()`

This function inserts clone operations after each input placeholder node that:

*   Is a tensor (not a scalar or other type)
*   Is not subject to in-place mutation

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 31-98](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L31-L98)

#### Graph Transformation Example

Before clone insertion:

```
placeholder    l_input_    L_input_    ()
call_function  op          some_op     (l_input_,)
```

After clone insertion:

```
placeholder    l_input_       L_input_             ()
call_function  clone_default  aten.clone.default   (l_input_,)
call_function  op             some_op              (clone_default,)
```

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 101-108](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L101-L108)

#### Key Implementation Details

| Aspect | Implementation | Reason |
| --- | --- | --- |
| Insertion point | After all input nodes | Workaround for argument list truncation issue |
| Mutation check | `copy_` and `slice_scatter` ops | These ops mutate the first argument |
| Code elimination | Disabled before AOT | Prevents incorrect removal of un-functionalized mutations |

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 55-94](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L55-L94)

## Clone Removal Implementation

### Function: `remove_clones_for_input_aliasing()`

After AOT Autograd has computed aliasing metadata, the temporary clone operations are removed to restore the original graph structure.

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 101-125](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L101-L125)

#### Removal Criteria

A clone node is removed if:

1.   Its input is a placeholder node
2.   The placeholder has exactly one user (the clone node)
3.   The user is `torch.ops.aten.clone.default`

After removal, `GraphCleanup()` is called to eliminate dead code, lint the graph, and recompile.

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 101-125](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L101-L125)

## Tangent Marking for Gradient Flow

### Purpose of Tangent Marking

In training graphs with graph breakages, outputs are classified as either:

*   **Tangents**: Outputs that participate in the backward pass (gradients)
*   **Primals**: Model parameters, buffers, and intermediate activations

Tangent marking identifies which outputs require gradient computation by examining PyTorch's AOT metadata.

**Sources:**[torch_ttnn/preprocessing/handle_tangents.py 9-61](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_tangents.py#L9-L61)

### Function: `mark_output_as_tangents()`

This function adds `tangents` metadata to output nodes based on AOT Autograd's forward metadata.

**Sources:**[torch_ttnn/preprocessing/handle_tangents.py 9-61](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_tangents.py#L9-L61)

#### Output Type Filtering

Only outputs that meet all criteria are marked as tangents:

| Criterion | Description |
| --- | --- |
| OutputType | Must be `non_alias`, `unsafe_view_alias`, or `custom_function_view` |
| Type | Must be a subclass of `torch.Tensor` (not SymInt) |
| Gradient | Must have `requires_grad == True` |

**Sources:**[torch_ttnn/preprocessing/handle_tangents.py 41-59](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_tangents.py#L41-L59)

## Integration with Export Code Generation

The clone and tangent handling mechanisms are essential for the `export_code` functionality, which generates standalone Python scripts from compiled graphs.

### Clone-Primal Mapping

When graphs are broken across multiple forward functions, the last primal output from one graph becomes the clone input to the next graph.

**Sources:**[tools/export_code.py 53-88](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L53-L88)

### Functions for Clone-Primal Handling

| Function | Purpose | Key Logic |
| --- | --- | --- |
| `_get_first_clone_node()` | Find first clone in argument list | Searches for node starting with "clone" |
| `_match_last_primal_with_clone_node()` | Map clone to primal | Searches backwards for last primal |
| `_collect_matching_primals_from_clones()` | Build complete mapping | Creates `primals_from_clones` dictionary |

**Sources:**[tools/export_code.py 68-886](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L68-L886)

### Tangent Information Collection

The `TangentsInfo` named tuple organizes tangent data for graph reconstruction:

`class TangentsInfo(NamedTuple):    chunk_idx: int    # Index of graph chunk    graph_idx: int    # Index within chunk    tangents: list    # List of tangent node names`
**Sources:**[tools/export_code.py 805-810](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L805-L810)

### Tangent Renaming and Matching

When assembling forward functions, tangents from previous graphs must be matched with tangent arguments in subsequent graphs.

**Sources:**[tools/export_code.py 90-161](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L90-L161)

#### Renaming Logic

The `_rename_arguments_and_tangents()` function:

1.   Identifies input tangent arguments (nodes starting with "tangents")
2.   Matches them with the most recent set of output tangents
3.   Appends `_{chunk_idx}_{graph_idx}` suffix to other arguments from the same output list
4.   Creates assignment statements like `tangents_1 = mul_1_0_0`

**Sources:**[tools/export_code.py 90-161](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L90-L161)

## Pipeline Integration

### Preprocessing Flow in Compilation

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 8-27](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L8-L27)[torch_ttnn/preprocessing/handle_tangents.py 9-61](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_tangents.py#L9-L61)

### Integration Points

| Stage | Handler | Graph State |
| --- | --- | --- |
| Pre-AOT | `insert_clones_for_input_aliasing` | High-level torch operations |
| During AOT | `mark_output_as_tangents` | ATEN operations being functionalized |
| Post-AOT | `remove_clones_for_input_aliasing` | ATEN operations with metadata |
| Lowering | `aten_backend` | ATEN → TTNN transformation |

## Example: Complete Transformation

### Input Graph (Pre-processing)

```
placeholder    input_1         input_1         ()
call_function  add_1           torch.add       (input_1, 2.0)
output         output          output          (add_1,)
```

### After Clone Insertion

```
placeholder    input_1         input_1             ()
call_function  clone_default   aten.clone.default  (input_1,)
call_function  add_1           torch.add           (clone_default, 2.0)
output         output          output              (add_1,)
```

### After AOT Autograd (with metadata)

```
placeholder    input_1         input_1             ()
call_function  clone_default   aten.clone.default  (input_1,)
call_function  add_1           aten.add.Tensor     (clone_default, 2.0)
output         output          output              (add_1,)
# Aliasing metadata computed at this point
```

### After Clone Removal

```
placeholder    input_1         input_1         ()
call_function  add_1           aten.add.Tensor (input_1, 2.0)
output         output          output          (add_1,)
# Clean graph ready for lowering
```

**Sources:**[torch_ttnn/preprocessing/handle_input_aliasing.py 101-125](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/torch_ttnn/preprocessing/handle_input_aliasing.py#L101-L125)

## Utility Functions in Export Code

### Helper Functions

| Function | Input | Output | Purpose |
| --- | --- | --- | --- |
| `_get_first_clone_node()` | List of nodes/strings | First clone node or None | Find clone placeholder |
| `_match_last_primal_with_clone_node()` | Outputs, clone node | Last primal or None | Map clone to primal |
| `_rename_outputs()` | Output nodes, chunk/graph idx | List of renamed strings | Suffix outputs with tangents |
| `_collect_tangents()` | ATEN graphs | List[TangentsInfo] | Organize tangent metadata |
| `_rename_arguments_and_tangents()` | Arg names, tangent info | Renamed args, assignments | Map tangents across graphs |

**Sources:**[tools/export_code.py 68-886](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L68-L886)

### Graph Export Pattern

When exporting code with `--export_code accuracy`, the system:

1.   Collects tangent information using `_collect_tangents()`
2.   Maps clones to primals using `_collect_matching_primals_from_clones()`
3.   Renames outputs based on tangent presence
4.   Generates assignment statements to link tangents across graphs
5.   Produces standalone Python with correct data flow

**Sources:**[tools/export_code.py 889-931](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tools/export_code.py#L889-L931)
