## Purpose and Scope

The TensorFlow frontend converts TensorFlow models (both 1.x and 2.x) into Relay IR. It ingests TensorFlow `GraphDef` protocol buffers and produces a Relay `IRModule` with associated parameters. This page documents the conversion architecture, control flow detection, operator mapping, and special handling for TensorFlow-specific constructs.

For information about common frontend utilities shared across all frameworks, see [Common Frontend Infrastructure](https://deepwiki.com/tenstorrent/tt-tvm/3.1-common-frontend-infrastructure). For TensorFlow Lite model conversion, see [TFLite and Other Frontends](https://deepwiki.com/tenstorrent/tt-tvm/3.6-tflite-and-other-frontends). For Keras model conversion (which uses a different path), see [Keras Frontend](https://deepwiki.com/tenstorrent/tt-tvm/3.5-keras-frontend).

## Entry Point and Configuration

### Main Entry Function

The primary entry point is `from_tensorflow()` which accepts a TensorFlow GraphDef and produces Relay IR:

| Configuration Option | Default | Purpose |
| --- | --- | --- |
| `use_dense` | `True` | Convert `tf.matmul` to `transpose(weight) + nn.dense` instead of experimental `nn.matmul` |
| `use_nt_batch_matmul` | `True` | Convert `tf.batch_matmul` to `transpose(weight) + nn.batch_matmul_NT` format |

## Graph Conversion Process

### GraphProto Class

The `GraphProto` class is the core converter that manages the entire conversion process:

**Key Internal State:**

| Attribute | Type | Purpose |
| --- | --- | --- |
| `_nodes` | `Dict[str, TFNode]` | Maps node names to TensorFlow graph nodes |
| `_params` | `Dict[str, NDArray]` | Stores constant tensors and weights |
| `_output_shapes` | `Dict[str, List]` | Cached output shapes for each node |
| `_loops` | `Dict[str, Loop]` | Control flow loop constructs |
| `_branches` | `Dict[str, Branch]` | Control flow branch constructs |
| `_backtrack` | `Dict[str, Expr]` | Caches converted Relay expressions |
| `_control_flow_node_map` | `Dict[str, Set[str]]` | Maps execution frames to control flow primitives |

## Operator Conversion System

### Operator Map Structure

The `_convert_map` dictionary in `tensorflow_ops.py` maps TensorFlow operator names to converter functions:

### Converter Function Signature

All operator converters follow a standard signature:

`def converter(inputs, attr, params, mod):    """    inputs: List[relay.Expr] - Input expressions    attr: Dict - TensorFlow node attributes      params: Dict[str, NDArray] - Graph parameters    mod: IRModule - Current module for type inference        Returns: relay.Expr or Tuple[relay.Expr, ...]    """`
### Example Converters

| TensorFlow Op | Converter Function | Relay Output |
| --- | --- | --- |
| `Rsqrt` | `_rsqrt()` | `power(x, -0.5)` |
| `ArgMax` | `_argx(argmax, "argmax")` | `argmax(x, axis, keepdims=False)` |
| `MaxPool3D` | `_pool3d("max_pool")` | `nn.max_pool3d(...)` |
| `MatMul` | `_matmul()` | `nn.dense(...)` or `nn.matmul(...)` based on config |

## Special Features and Handling

### Variable Operations

TensorFlow variables require special handling during conversion:

| Variable Op | Handling Strategy |
| --- | --- |
| `Variable`, `VariableV2` | Pruned from frozen graphs |
| `ReadVariableOp` | Replaced with constant values |
| `VarHandleOp` | Converted to parameters |
| `AssignVariableOp` | Ignored in frozen graphs |

**Frozen Graph Pruned Ops:**

`_freezed_graph_pruned_op_list = [    "ReadVariableOp",    "ResourceGather",     "Variable",    "VariableV2",    "VarHandleOp",    "Assign",    "AssignVariableOp",]`

### Dynamic Shape Support

The frontend handles dynamic shapes through several mechanisms:

1.   **Shape Inference**: Uses `_infer_shape()` and `_infer_type()` from common infrastructure
2.   **Any Type**: Uses `relay.ty.Any()` for unknown dimensions
3.   **Runtime Shape Functions**: Inserts `shape_of()` operations for dynamic reshaping

**Helper Functions:**

*   `list_shape_of(tensor, ndim)`: Splits shape tensor into per-dimension slices
*   `check_symbolic_shape(shape)`: Detects if shape contains symbolic dimensions
*   `_get_more_static_shape(shape0, shape1)`: Chooses shape with fewer symbolic dimensions

**Sources:**

*   [python/tvm/relay/frontend/tensorflow_ops.py 43-51](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/tensorflow_ops.py#L43-L51)
