# Starfield and Topology Visualizations

Relevant source files
*   [src/animation/starfield.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs)
*   [src/animation/topology.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs)

The Starfield and Topology system provides a hardware-responsive visualization where physical components of Tenstorrent chips are mapped to celestial and structural metaphors. This module ensures that every visual element—motion, color, and brightness—is directly driven by real-time telemetry from the `TelemetryBackend`.

## HardwareStarfield System

The `HardwareStarfield` class is the primary engine for the starfield visualization. It transforms Tensix cores into stars, memory channels into orbiting planets, and inter-device communication into data streams.

### Core Entities and Mapping

The visualization maps hardware state to visual properties using the following logic:

*   **Tensix Cores (Stars):** Positioned according to chip architecture (e.g., Grayskull 10x12, Wormhole 8x10). 
    *   **Brightness:** Proportional to power consumption [src/animation/starfield.rs 12](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L12-L12)
    *   **Twinkle Rate:** Proportional to current draw [src/animation/starfield.rs 13](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L13-L13)
    *   **Color:** Mapped to temperature (Cyan → Green → Yellow → Orange → Red) [src/animation/starfield.rs 11](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L11-L11)

*   **Memory (Planets):** Represent L1, L2, and DDR hierarchy as orbiting bodies [src/animation/starfield.rs 87-89](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L87-L89)
    *   **L1/L2:** Rendered as diamonds (`◆`/`◇`) [src/animation/starfield.rs 125-126](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L125-L126)
    *   **DDR:** Rendered as intensity-based blocks (`░` to `█`) [src/animation/starfield.rs 129-130](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L129-L130)

### Star Rendering Logic

The `Star` struct uses a character progression to represent brightness and depth. Foreground stars (higher `depth`) appear larger and brighter [src/animation/starfield.rs 79-84](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L79-L84)

| Character | Meaning |
| --- | --- |
| `·` | Dim / Background |
| `∘` | Low Intensity |
| `○` | Medium Intensity |
| `◉` | High Intensity |
| `●` | Maximum Intensity |
| `✦` | Sparkle (Activity Spike) [src/animation/starfield.rs 75-77](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L75-L77) |

### Data Flow and Update Cycle

The starfield maintains an `AdaptiveBaseline` to detect relative activity changes rather than absolute values [src/animation/starfield.rs 199-200](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L199-L200)

## Board Topology and Sync Score

The `BoardTopology` module groups chips into physical boards to drive topology-aware rendering.

### Board Detection Logic

The system attempts to group devices using `SmbusTelemetry.board_id`[src/animation/topology.rs 19](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs#L19-L19) If IDs are missing, it falls back to architecture-aware grouping:

*   **Single-chip cards (p150, n150, e75):** 1 chip per board [src/animation/topology.rs 138](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs#L138-L138)
*   **Multi-chip carriers (p300, n300):** 2 chips per board [src/animation/topology.rs 139](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs#L139-L139)

### The Sync Score

The `sync_score` function computes a scalar (0.0–1.0) representing the synchronization of activity between two chips using a geometric mean: `(act_a * act_b).sqrt()`[src/animation/topology.rs 33-36](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs#L33-L36)

*   **Intra-board Links:** Connect chips on the same board. They have a minimum floor of `0.2` (`INTRA_BOARD_FLOOR`) to remain faintly visible even when idle [src/animation/topology.rs 58](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs#L58-L58)
*   **Inter-board Links:** Connect different boards. These only appear when both endpoints are active [src/animation/topology.rs 11](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/topology.rs#L11-L11)

## Implementation Detail: Fleet Mode

When the number of devices exceeds the capacity for individual star grids (usually > 8 devices), the `HardwareStarfield` switches to **Fleet Mode**[src/animation/starfield.rs 228-232](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L228-L232)

In Fleet Mode:

1.   Individual Tensix stars are replaced by a **Heat Map** of the entire fleet [src/animation/starfield.rs 221-223](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L221-L223)
2.   Each device is represented as a block in a grid.
3.   The color of the block represents the average device temperature, and the brightness represents power [src/animation/starfield.rs 11-13](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L11-L13)
