# Code of Conduct

Relevant source files
*   [CODE_OF_CONDUCT.md](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1)

This document outlines the behavioral expectations, standards, and enforcement procedures for all participants in the ttsim community. The Code of Conduct applies to all interactions within project spaces, including GitHub issues, discussions, and communications with project maintainers.

For information about contributing code and reporting technical issues, see [Contribution Process](https://deepwiki.com/tenstorrent/ttsim/5.1-contribution-process) and [Reporting Issues](https://deepwiki.com/tenstorrent/ttsim/5.2-reporting-issues). For security vulnerability reporting, see [Security Vulnerabilities](https://deepwiki.com/tenstorrent/ttsim/5.3-security-vulnerabilities).

* * *

## Overview

The ttsim project adopts the **Contributor Covenant Code of Conduct version 2.1** as its community standard. This establishes expectations for inclusive, respectful behavior and provides clear enforcement mechanisms for violations.

**Sources:**[CODE_OF_CONDUCT.md 1-134](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L1-L134)

## Community Pledge

The ttsim community pledges to provide a harassment-free experience for all participants, regardless of:

| Protected Characteristic | Examples |
| --- | --- |
| Personal attributes | Age, body size, visible or invisible disability |
| Identity | Gender identity and expression, sexual identity and orientation, ethnicity, race, caste, color |
| Background | Nationality, socio-economic status, education, level of experience |
| Beliefs | Religion, personal appearance |

The community commits to interactions that foster an open, welcoming, diverse, inclusive, and healthy environment.

**Sources:**[CODE_OF_CONDUCT.md 5-13](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L5-L13)

## Behavioral Standards

### Positive Behaviors

The following behaviors contribute to a healthy community environment:

*   **Empathy and kindness**: Demonstrating understanding toward other community members
*   **Respectful disagreement**: Being respectful of differing opinions, viewpoints, and experiences
*   **Constructive feedback**: Giving and gracefully accepting constructive criticism
*   **Accountability**: Accepting responsibility, apologizing for mistakes, and learning from the experience
*   **Community focus**: Prioritizing what is best for the overall community, not just individuals

**Sources:**[CODE_OF_CONDUCT.md 17-26](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L17-L26)

### Unacceptable Behaviors

The following behaviors are prohibited in all community spaces:

| Violation Category | Examples |
| --- | --- |
| **Sexualized content** | Use of sexualized language or imagery, sexual attention or advances |
| **Harassment** | Trolling, insulting or derogatory comments, personal or political attacks |
| **Privacy violations** | Publishing others' private information (physical/email addresses) without permission |
| **Professional misconduct** | Other conduct reasonably considered inappropriate in a professional setting |

Both **public and private harassment** are prohibited under this policy.

**Sources:**[CODE_OF_CONDUCT.md 28-37](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L28-L37)

## Scope of Application

### Code of Conduct Application Flow

**Scope: Code of Conduct Application**

The Code of Conduct applies to:

1.   **All community spaces**: GitHub repository, issues, discussions, pull requests, email communications
2.   **Official representation**: When individuals officially represent the ttsim community in public spaces
3.   **Events**: Both online and offline events where ttsim is represented

## Reporting Violations

### Reporting Process

Violations of the Code of Conduct should be reported to the enforcement team at:

**Email**: `opensource@tenstorrent.com`

All reports are:

*   **Reviewed promptly**: Community leaders investigate all complaints fairly
*   **Kept confidential**: Privacy and security of the reporter are protected
*   **Handled professionally**: Leaders respect the sensitivity of incidents

**Sources:**[CODE_OF_CONDUCT.md 61-68](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L61-L68)

## Enforcement Responsibilities

Community leaders are responsible for:

| Responsibility | Description |
| --- | --- |
| **Clarifying standards** | Explaining acceptable behavior standards |
| **Enforcing standards** | Taking appropriate corrective action for violations |
| **Content moderation** | Right to remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions not aligned with the Code of Conduct |

## Enforcement Guidelines

Community leaders follow a **4-tier Community Impact Guidelines** system to determine consequences for violations. Each tier corresponds to the severity and pattern of the violation.

### Enforcement Tier System

**Enforcement Tiers: Escalating Consequences**

**Sources:**[CODE_OF_CONDUCT.md 72-114](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L72-L114)

### Tier 1: Correction

| Aspect | Details |
| --- | --- |
| **Community Impact** | Use of inappropriate language or other behavior deemed unprofessional or unwelcome |
| **Consequence** | Private, written warning from community leaders providing clarity around the violation and explanation of why the behavior was inappropriate |
| **Additional Action** | A public apology may be requested |

**Sources:**[CODE_OF_CONDUCT.md 75-82](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L75-L82)

### Tier 2: Warning

| Aspect | Details |
| --- | --- |
| **Community Impact** | A violation through a single incident or series of actions |
| **Consequence** | Warning with consequences for continued behavior; no interaction with people involved, including those enforcing the Code of Conduct, for a specified period |
| **Scope** | Includes avoiding interactions in community spaces and external channels (e.g., social media) |
| **Escalation** | Violating these terms may lead to temporary or permanent ban |

**Sources:**[CODE_OF_CONDUCT.md 84-94](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L84-L94)

### Tier 3: Temporary Ban

| Aspect | Details |
| --- | --- |
| **Community Impact** | Serious violation of community standards, including sustained inappropriate behavior |
| **Consequence** | Temporary ban from any interaction or public communication with the community for a specified period |
| **Restrictions** | No public or private interaction with people involved, including unsolicited interaction with those enforcing the Code of Conduct |
| **Escalation** | Violating these terms may lead to permanent ban |

**Sources:**[CODE_OF_CONDUCT.md 96-105](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L96-L105)

### Tier 4: Permanent Ban

| Aspect | Details |
| --- | --- |
| **Community Impact** | Demonstrating a pattern of violation of community standards, including sustained inappropriate behavior, harassment of an individual, or aggression toward or disparagement of classes of individuals |
| **Consequence** | Permanent ban from any sort of public interaction within the community |
| **Finality** | No path for restoration of community participation |

**Sources:**[CODE_OF_CONDUCT.md 107-114](https://github.com/tenstorrent/ttsim/blob/67b1733e/CODE_OF_CONDUCT.md?plain=1#L107-L114)

## Summary

The ttsim Code of Conduct establishes:

1.   **Clear behavioral standards**: Specific examples of positive and unacceptable behaviors
2.   **Broad application scope**: Applies to all community spaces and official representation
3.   **Private reporting mechanism**: Violations reported to `opensource@tenstorrent.com`
4.   **Graduated enforcement**: 4-tier system (Correction → Warning → Temporary Ban → Permanent Ban)
5.   **Leader responsibilities**: Community leaders clarify, enforce, and communicate standards
6.   **Privacy protection**: Reporter privacy and security are protected throughout the process

This governance framework ensures the ttsim community remains welcoming, inclusive, and professional while providing clear recourse for addressing violations.
