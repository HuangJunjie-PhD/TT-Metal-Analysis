The testing infrastructure provides comprehensive end-to-end testing capabilities for Model Explorer, including visual regression testing to ensure UI consistency across different model formats and server configurations. This system validates the complete user workflow from model loading through visualization rendering.

For information about code quality tools and CI/CD pipelines, see [Code Quality and CI/CD](https://deepwiki.com/tenstorrent/model-explorer/5.2-code-quality-and-cicd).

## Test Architecture Overview

The testing infrastructure is built around Playwright for browser automation and visual regression testing. The system validates both functional behavior and visual consistency across different model types and usage patterns.

**Test Architecture**: The testing system uses Playwright as the primary test runner, with visual regression testing as a key component. Golden screenshots serve as reference images for validating UI consistency across different test scenarios.

Sources: [test/screenshots_golden/chrome-linux/](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/test/screenshots_golden/chrome-linux/)

## Visual Regression Testing

The visual regression testing system captures and compares screenshots to detect unintended UI changes. Golden reference images are stored for different browser environments and test scenarios.

**Golden Screenshot Structure**: Screenshots are organized by browser and platform (`chrome-linux`), with separate images for different model formats and server usage patterns.

The golden screenshots cover several categories:

*   **Model Format Tests**: Different input formats like MLIR, direct TFLite, PyTorch, and GraphDef
*   **Adapter Tests**: Testing different model conversion pathways
*   **Server Reuse Tests**: Validating server behavior across multiple model loads

## Test Data Management

The test data management system organizes golden reference images and handles browser-specific variations. The current structure supports Chrome on Linux with organized screenshot storage.

**Test Data Structure**: Golden screenshots are organized hierarchically by browser platform, with descriptive naming conventions that indicate the test scenario and model format being validated.

The naming conventions provide clear identification:

*   **Format-Adapter Pattern**: `litert_mlir.png`, `tf_mlir.png` indicate model format and adapter type
*   **Server Scenario Pattern**: `reuse_server_pytorch.png` indicates server reuse testing with specific model types
*   **Browser Platform**: `chrome-linux/` directory indicates the browser and OS combination
