This document covers the central Angular services that form the backbone of Model Explorer's frontend architecture. These services manage model loading, backend communication, application settings, and shared state coordination across the application.

For information about the visualization engine and rendering components, see [Visualization Engine](https://deepwiki.com/tenstorrent/model-explorer/4.2-visualization-engine). For backend server and API documentation, see [Python Server and APIs](https://deepwiki.com/tenstorrent/model-explorer/3.2-python-server-and-apis).

## Model Loader Service

The `ModelLoaderService` is the primary service responsible for managing model loading operations, graph collections, and communication with backend adapters for model conversion.

### Core Responsibilities

The service implements the `ModelLoaderServiceInterface` and manages several key areas:

| Responsibility | Methods | Description |
| --- | --- | --- |
| Model Loading | `loadModels()`, `loadModel()` | Orchestrates loading of multiple model files and individual models |
| Graph Management | `updateGraphCollections()`, `loadedGraphCollections` | Manages loaded graph data and state |
| Backend Communication | `sendExtensionRequest()`, `sendConvertRequest()` | Communicates with backend adapters for model conversion |
| Override Management | `updateOverrides()`, `overrides` | Handles model parameter overrides and modifications |
| File Operations | `uploadModelFile()`, `readTextFile()` | Manages file uploads and local file reading |
