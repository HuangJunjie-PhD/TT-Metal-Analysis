### Hardware Requirements

While `tt-tools-common` can be installed on any compatible system, hardware interaction features require:

*   **Tenstorrent Hardware**: Blackhole (BH), Wormhole (WH), or Grayskull (GS) accelerators
*   **Kernel Driver**: tt-kmd version >= 1.26.0 (see [Driver Version Management](https://deepwiki.com/tenstorrent/tt-tools-common/3.1-driver-version-management))
*   **PCIe Interface**: Hardware must be accessible via `/dev/tenstorrent/*` device nodes

## Installation Methods

**Diagram: Installation Flow and Package Sources**

**Sources:**[pyproject.toml 23-43](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/pyproject.toml#L23-L43)[debian/control 22-25](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/debian/control#L22-L25)

### Method 1: Install from PyPI (Recommended)

Install the latest stable release from PyPI using pip:

`pip install tt-tools-common`
For a specific version:

`pip install tt-tools-common==1.6.0`
To include hardware interaction capabilities, `pyluwen` must be installed separately:

`pip install tt-tools-common pyluwen~=0.8.0`
**Important:** The PyPI package does **not** include `pyluwen` as a core dependency ([pyproject.toml 23-34](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/pyproject.toml#L23-L34)), but it is required in [dev dependencies 40](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/dev%20dependencies#L40-L40) Consumer applications requiring hardware reset or chip detection features must install `pyluwen` explicitly.

**Sources:**[pyproject.toml 1-43](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/pyproject.toml#L1-L43)

### Method 2: Install Debian Package

For Ubuntu systems, install the pre-built Debian package:

`# Add Tenstorrent package repository (if not already configured)# Repository setup instructions provided by Tenstorrent # Install the packagesudo apt updatesudo apt install python3-tt-tools-common`
The Debian package automatically includes `python3-pyluwen` as a dependency ([debian/control 25](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/debian/control#L25-L25)), ensuring all hardware interaction features work out-of-the-box.

**Package Details:**

*   **Package Name**: `python3-tt-tools-common`
*   **Architecture**: `all` (architecture-independent Python code)
*   **Maintainer**: Sam Bansal [sbansal@tenstorrent.com](mailto:sbansal@tenstorrent.com)
*   **Build System**: Uses `gbp` (git-buildpackage) with GPG signing

**Sources:**[debian/control 19-35](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/debian/control#L19-L35)[.github/workflows/build-debian.yml 129-133](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/.github/workflows/build-debian.yml#L129-L133)

### Method 3: Development Installation

For development or contributing to `tt-tools-common`, clone the repository and install in editable mode:

`git clone https://github.com/tenstorrent/tt-tools-common.gitcd tt-tools-commonpip install -e ".[dev]"`
This installs the package with all development dependencies including:

*   `black>=23.11.0` - Code formatting
*   `pre-commit>=3.5.0` - Git hooks for code quality
*   `pyluwen~=0.8.0` - Hardware abstraction layer
*   `pytest>=7.0.0` - Testing framework
*   `pytest-asyncio>=0.21.0` - Async test support

See [Development Environment Setup](https://deepwiki.com/tenstorrent/tt-tools-common/6.4-development-environment-setup) for comprehensive development setup instructions.

**Sources:**[pyproject.toml 36-43](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/pyproject.toml#L36-L43)

After installation, verify the package is correctly installed and importable:

`# Check package versionpython -c "import importlib.metadata; print(importlib.metadata.version('tt-tools-common'))" # Verify core modules are importablepython -c "from tt_tools_common.reset_common import ChipReset"python -c "from tt_tools_common.ui_common.widgets import TTHeader"python -c "from tt_tools_common.utils_common import system_utils" # Check if pyluwen is available (for hardware features)python -c "import pyluwen; print('pyluwen available')" 2>/dev/null || echo "pyluwen not installed"`
### Testing Hardware Interaction

If `pyluwen` is installed and Tenstorrent hardware is available:

`# Attempt to detect chips (requires hardware)python -c "from tt_tools_common.utils_common.tools_utils import detect_chips_with_callback; \           detect_chips_with_callback(lambda x: print(x))"`
This command will fail gracefully if no hardware is present or if the kernel driver is not installed.

### Package Signing

All Debian packages are cryptographically signed using GPG:

1.   GPG private key imported from GitHub secrets ([build-debian.yml 68-71](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/build-debian.yml#L68-L71))
2.   Key fingerprint extracted and passed to `gbp buildpackage` ([build-debian.yml 133](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/build-debian.yml#L133-L133))
3.   `DEBSIGN_KEYID` environment variable configures signing

**Sources:**[.github/workflows/build-debian.yml 67-133](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/.github/workflows/build-debian.yml#L67-L133)

### Virtualization Support

`tt-tools-common` includes special handling for Xen HVM environments. When running under Xen, the reset system coordinates with xenstore for PCIe device management. No special installation steps are required; the library auto-detects virtualization at runtime.

See [Platform Support and Virtualization](https://deepwiki.com/tenstorrent/tt-tools-common/2.7-platform-support-and-virtualization) for implementation details.

## Troubleshooting Common Installation Issues

### Issue: Python Version Too Old

**Symptom:** Installation fails with "Python 3.10 or higher is required"

**Solution:** Upgrade Python or use a virtual environment with Python 3.10+:

`python3.10 -m venv venvsource venv/bin/activatepip install tt-tools-common`
### Issue: Missing pyluwen for Hardware Operations

**Symptom:**`ImportError: No module named 'pyluwen'` when attempting chip reset

**Solution:** Install pyluwen explicitly (PyPI installations only):

`pip install pyluwen~=0.8.0`
Debian package installations include this automatically.

### Issue: Package Data Files Not Found

**Symptom:**`FileNotFoundError` when loading CSS or YAML files

**Solution:** Verify package data was installed correctly:

`python -c "import tt_tools_common; print(tt_tools_common.__file__)"# Check if ui_common/*.css files exist in the printed directory`
If files are missing, reinstall the package:

`pip install --force-reinstall tt-tools-common`
