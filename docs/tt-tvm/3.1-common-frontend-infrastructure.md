## Purpose and Scope

The Common Frontend Infrastructure provides a unified set of utilities and abstractions used by all frontend converters (PyTorch, ONNX, TensorFlow, TFLite, MXNet, etc.) when translating framework-specific models into Relay IR. This infrastructure eliminates code duplication and ensures consistent behavior across different frontend implementations.

This page covers the shared utilities in [python/tvm/relay/frontend/common.py](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/common.py) For information about specific frontend converters that use this infrastructure, see:

*   PyTorch Frontend: [#3.2](https://deepwiki.com/tenstorrent/tt-tvm/3.2-pytorch-frontend)
*   ONNX Frontend: [#3.3](https://deepwiki.com/tenstorrent/tt-tvm/3.3-onnx-frontend)
*   TensorFlow Frontend: [#3.4](https://deepwiki.com/tenstorrent/tt-tvm/3.4-tensorflow-frontend)
*   Other frontends: [#3.5](https://deepwiki.com/tenstorrent/tt-tvm/3.5-keras-frontend), [#3.6](https://deepwiki.com/tenstorrent/tt-tvm/3.6-tflite-and-other-frontends)

## Type and Shape Inference System

The inference system provides functions to extract type, shape, and value information from Relay expressions during frontend conversion. This is critical for operators that need to know their input characteristics to determine output characteristics.

### Core Inference Functions

| Function | Purpose | Return Type |
| --- | --- | --- |
| `infer_type(node, mod)` | Infer the complete type of an expression | `relay.Type` |
| `infer_shape(node, mod)` | Extract shape tuple from type inference | `Tuple[int/Any]` |
| `infer_value(node, params, mod)` | Evaluate expression at compile-time | `NDArray` |
| `infer_value_simulated(node, params)` | Simulate value inference | `NDArray` |
| `infer_channels(inputs, transpose)` | Infer number of channels from layout | `int` |
| `try_infer_value(node, callback, default)` | Safe value inference with fallback | `Any` |

## Attribute Conversion (AttrCvt)

The `AttrCvt` class provides a systematic way to convert framework-specific operator attributes to Relay operator attributes. It handles attribute renaming, default values, and custom transformations.

### AttrCvt Class Structure

### Attribute Transformation Process

| Step | Description | Example |
| --- | --- | --- |
| **Rename** | Map framework attr name to Relay name | `"kernel_shape"` → `"kernel_size"` |
| **Transform** | Apply transformation function | `(3, 3)` with `(0, 0)` default for padding |
| **Exclude** | Remove attributes not needed | Exclude `"storage_order"` |
| **Validate** | Check required attributes present | Ensure `custom_check` passes |
| **Extras** | Add computed attributes | Add `"channels"` from weights |
