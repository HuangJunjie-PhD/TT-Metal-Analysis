## Purpose and Scope

This page documents the symbolic variable access system in TTExaLens, which enables reading and writing firmware variables by name using DWARF debug information. The `ElfVariable` class provides a Python interface that mirrors C/C++ syntax for accessing struct members, array elements, and pointer dereferences. This system bridges the gap between high-level variable names in source code and low-level memory addresses on the device.

For information about parsing ELF files and DWARF symbols, see [ELF and DWARF Parsing](https://deepwiki.com/tenstorrent/tt-exalens/7.3-elf-and-dwarf-parsing). For details about the underlying memory access mechanisms, see [Memory Access Operations](https://deepwiki.com/tenstorrent/tt-exalens/3.3-memory-access-operations).

**Sources:**[ttexalens/elf/variable.py 1-618](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L1-L618)[ttexalens/elf/parsed.py 1-369](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L1-L369)

## Core Architecture

The symbolic variable system consists of three primary components: DWARF parsing to extract type and location information, the `ElfVariable` wrapper that provides Python-native access patterns, and the memory access abstraction that reads/writes device memory.

**Diagram: Symbolic Variable Access Architecture**

The `ElfVariable` class maintains three essential attributes: the type DIE (`ElfDie`) describing the variable's type structure, the memory address where the variable resides, and a `MemoryAccess` object for reading/writing bytes. All symbolic operations (member access, indexing, dereferencing) return new `ElfVariable` instances with updated type and address information.

**Sources:**[ttexalens/elf/variable.py 20-24](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L20-L24)[ttexalens/elf/parsed.py 243-260](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L243-L260)

## Variable Construction

### Obtaining Global Variables

The `ParsedElfFile` class provides two methods for accessing global variables by name:

| Method | Description | Memory Reads |
| --- | --- | --- |
| `get_global(name, mem_access)` | Returns `ElfVariable` for lazy access | On-demand per field |
| `read_global(name, mem_access)` | Returns `ElfVariable` with cached data | Single bulk read |

**Diagram: Global Variable Access Paths**

The `get_global()` method [ttexalens/elf/parsed.py 243-260](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L243-L260) first attempts fast lookup using `find_die_by_name()`, falling back to the full `variables` dictionary if needed. Once the DIE is located, it extracts the address and resolved type to construct the `ElfVariable`. The `read_global()` method [ttexalens/elf/parsed.py 262-266](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L262-L266) wraps this with a single bulk memory read via `ElfVariable.read()`.

**Sources:**[ttexalens/elf/parsed.py 243-266](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L243-L266)[ttexalens/elf/variable.py 612-617](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L612-L617)

### Direct Construction

Advanced users can construct `ElfVariable` objects directly by providing the type DIE, address, and memory access interface:

`# Example pattern (from tests)variable_die = parsed_elf.variables["g_global_struct"]address = variable_die.addresself_var = ElfVariable(variable_die.resolved_type, address, mem_access)`
This pattern is used internally by all higher-level access methods and is useful when working with local variables or computed addresses.

## Array and Pointer Operations

### Array Indexing

The `__getitem__()` method [ttexalens/elf/variable.py 92-119](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L92-L119) handles integer indices for arrays and pointers:

`# Array accesselement = array_var[5]  # Returns ElfVariable for element # Pointer access (automatically dereferences)element = pointer_var[3]  # Equivalent to pointer_var.dereference()[3]`
For array types, the element address is calculated as:

```
new_address = base_address + index * element_type.size
```

For pointer types, the pointer is first dereferenced to obtain the pointed-to address, then the same offset calculation applies.

**Sources:**[ttexalens/elf/variable.py 92-112](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L92-L112)

## Value Reading and Writing

### Reading Values

The `read_value()` method [ttexalens/elf/variable.py 169-193](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L169-L193) extracts the actual value for base types, pointers, and enumerations:

| Type Category | Handling |
| --- | --- |
| `base_type` | Converts bytes to `int`, `float`, `bool` per type name |
| `pointer_type` | Returns dereferenced address |
| `enumeration_type` | Resolves to underlying base type |
`# Reading valuesint_val = int_var.read_value()      # Returns intfloat_val = float_var.read_value()  # Returns floatbool_val = bool_var.read_value()    # Returns boolptr_val = ptr_var.read_value()      # Returns address as int`
The method reads `type.size` bytes from memory and converts based on the type name: `float` uses `struct.unpack("f")`, `double` uses `struct.unpack("d")`, and integer types use `int.from_bytes()`.

**Sources:**[ttexalens/elf/variable.py 169-193](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L169-L193)

### Writing Values

The `write_value()` method [ttexalens/elf/variable.py 195-236](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L195-L236) converts Python values to bytes and writes to device memory:

`# Writing valuesvar.write_value(0xDEADBEEF)           # Write integerfloat_var.write_value(3.14159)        # Write floatbool_var.write_value(True)            # Write booleanenum_var.write_value(enum_value)      # Write enum value`
The `check_data_loss` parameter (default `True`) verifies that the written value can be round-tripped without precision loss. This catches errors like writing a float that exceeds the type's range or writing a non-integer value to an integer field.

**Sources:**[ttexalens/elf/variable.py 195-236](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L195-L236)[test/ttexalens/unit_tests/test_debug_symbols.py 412-430](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_debug_symbols.py#L412-L430)

### Cached Reading

The `read()` method [ttexalens/elf/variable.py 612-617](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L612-L617) performs a bulk read of the entire variable into a cached memory buffer:

`# Read entire struct once, then access members without additional device readscached_var = var.read()val1 = cached_var.field1  # No device accessval2 = cached_var.field2  # No device access`
This uses `CachedReadMemoryAccess` internally, which satisfies all subsequent reads from the cached buffer. This is particularly efficient for structs with many fields.

**Sources:**[ttexalens/elf/variable.py 612-617](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L612-L617)[test/ttexalens/unit_tests/test_debug_symbols.py 235-239](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_debug_symbols.py#L235-L239)

## Constant Access

The `ParsedElfFile.get_constant()` method [ttexalens/elf/parsed.py 268-298](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L268-L298) accesses compile-time constants without device memory access:

`# Access constexpr values from C++ codeuint_const = parsed_elf.get_constant("c_uint32_t")    # 0x11223344float_const = parsed_elf.get_constant("c_float")      # 0.5double_const = parsed_elf.get_constant("c_double")    # 2.718281828459`
This reads the `DW_AT_const_value` attribute from the variable's DIE and converts it to the appropriate Python type based on the variable's base type.
