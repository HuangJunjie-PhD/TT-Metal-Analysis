# SSH and Color Support

Relevant source files
*   [docs/SSH_COLOR_FIX.md](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1)

The `tt-toplike` TUI is designed to provide high-fidelity visual feedback of hardware telemetry. To achieve this across diverse environments—ranging from local high-end terminal emulators to restricted SSH sessions—the system implements a multi-tier color detection and fallback mechanism.

## Color Depth Architecture

The TUI supports two primary color modes based on the terminal's capabilities:

1.   **True-color (24-bit RGB):** Provides smooth gradients for temperature and power visualizations. This mode is activated when the `COLORTERM` environment variable is set to `truecolor` or `24bit`[docs/SSH_COLOR_FIX.md 48](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L48-L48)
2.   **256-color Fallback:** An indexed palette mode that approximates the hardware state using a subset of the Xterm 256-color map. This ensures that visualizations remain functional and informative even when 24-bit color is unavailable [docs/SSH_COLOR_FIX.md 49-51](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L49-L51)

### Detection Logic Flow

The application automatically determines the color depth during the initialization of the UI layer.

| Detection Step | Logic | Result |
| --- | --- | --- |
| **Primary Check** | Inspect `COLORTERM` environment variable | If `truecolor`, use 24-bit RGB [docs/SSH_COLOR_FIX.md 48](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L48-L48) |
| **Secondary Check** | Inspect `TERM` and `tput colors` | If 256 supported, use indexed fallback [docs/SSH_COLOR_FIX.md 58-60](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L58-L60) |
| **Fallback** | Default to basic 16-color or monochrome | Minimal visual representation |

## SSH Configuration and Forwarding

A common issue when monitoring remote Tenstorrent hardware is the loss of color fidelity over SSH. This occurs because the `COLORTERM` variable, which signals 24-bit support to the TUI, is not forwarded by the SSH daemon by default [docs/SSH_COLOR_FIX.md 7-9](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L7-L9)

### Client-Side Configuration

To enable high-fidelity visuals, the client must explicitly send the environment variable. This can be configured in `~/.ssh/config`[docs/SSH_COLOR_FIX.md 15-20](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L15-L20):

`Host *    SendEnv COLORTERM`
Alternatively, it can be passed per-session: `COLORTERM=truecolor ssh user@host`[docs/SSH_COLOR_FIX.md 28-31](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L28-L31)

### Server-Side Configuration

The remote server must be configured to accept the variable in `/etc/ssh/sshd_config`[docs/SSH_COLOR_FIX.md 33-42](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L33-L42):

`AcceptEnv COLORTERM`
## Fallback Mapping Implementation

When `tt-toplike` detects that true-color is unsupported, it switches the mapping functions in `src/ui/colors.rs` to use specific ANSI 256-color indices that best approximate the intended heatmap [docs/SSH_COLOR_FIX.md 77-79](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L77-L79)

### Temperature Color Mapping

The `temp_color()` function maps hardware temperature to the following indexed palette [docs/SSH_COLOR_FIX.md 73-74](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L73-L74):

| State | Color | Index |
| --- | --- | --- |
| Cool | Cyan | 51 |
| Warm | Yellow | 226 |
| Hot | Orange | 214 |
| Critical | Red | 196 |

### Power Color Mapping

The `power_color()` function uses a distinct gradient to differentiate power consumption from thermal state [docs/SSH_COLOR_FIX.md 75](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L75-L75):

| State | Color | Index |
| --- | --- | --- |
| Idle | Cyan | 51 |
| Active | Blue | 75 |
| High Load | Orange | 214 |
| Peak | Red | 196 |

## Testing Color Support

The TUI provides a mechanism to test these detection paths manually using the mock backend [docs/SSH_COLOR_FIX.md 62-65](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L62-L65):

1.   **Force 24-bit:**`COLORTERM=truecolor ./tt-toplike-tui --mock --mock-devices 2`
2.   **Force 256-color:**`unset COLORTERM && ./tt-toplike-tui --mock --mock-devices 2`

Users can verify their terminal environment using standard utilities before launching the TUI to troubleshoot display issues [docs/SSH_COLOR_FIX.md 55-60](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/SSH_COLOR_FIX.md?plain=1#L55-L60):

*   `echo $COLORTERM`
*   `tput colors`
