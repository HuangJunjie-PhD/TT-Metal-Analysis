The `entries/` directory serves as the single source of truth for the entire Tenstorrent ecosystem directory. It is organized into 11 category-based subdirectories, each containing JSON files that define specific projects, papers, tools, or guides.

## Directory Structure and Path Logic

The location of an entry file is strictly determined by its primary category (the first element in the `categories` array). The build system and validation scripts enforce this hierarchy to ensure the repository remains navigable.

## Category Subdirectories Reference

### 1. `ai-models/`

Contains machine learning models ported to or optimized for Tenstorrent hardware. These range from biomolecular models to large language models.

*   **Notable Entry:**`tt-boltz.json` - A biomolecular model for drug discovery supporting Blackhole Galaxy (32-chip) configurations [entries/ai-models/tt-boltz.json 1-32](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/ai-models/tt-boltz.json#L1-L32)
*   **Other Examples:**`tt-animatediff.json`, `open-oasis.json`, `gemma4-tt.json`.

### 2. `agents/`

Focuses on autonomous agents and LLM-driven tools specifically designed to interact with Tenstorrent environments.

*   **Notable Entry:**`tt-claw.json` - Agentic workflows for Tenstorrent development.

### 3. `kernels/`

Low-level compute kernels, SFPU implementations, and math libraries.

*   **Notable Entry:**`jasondavies-tt-series.json` - A deep technical series on optimal SFPU kernels for Wormhole and Blackhole [entries/kernels/jasondavies-tt-series.json 1-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/kernels/jasondavies-tt-series.json#L1-L4)
*   **Notable Entry:**`tt-llk.json` - The official Low-Level Kernels library that programs the TRISC RISC-V cores [entries/kernels/tt-llk.json 1-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/kernels/tt-llk.json#L1-L4)

### 4. `compilers/`

Software that translates high-level frameworks (PyTorch, JAX, XLA) into Tenstorrent-executable instructions.

*   **Notable Entries:**`tt-xla.json`, `tt-torch.json`.

### 5. `dev-tools/`

Utilities, debuggers, and visualizers for the development lifecycle.

*   **Notable Entry:**`tensix-viz.json` - A hardware topology visualizer that can be embedded directly into the frontend [entries/dev-tools/tensix-viz.json 1-34](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/dev-tools/tensix-viz.json#L1-L34)

### 6. `hw-system/`

Low-level system software, drivers, firmware, and hardware management tools.

*   **Notable Entries:**`tt-kmd.json` (Kernel Mode Driver), `tt-smi.json` (System Management Interface), `tt-flash.json`[entries/hw-system/tt-kmd.json 1-5](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-kmd.json#L1-L5)[entries/hw-system/tt-smi.json 1-5](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/hw-system/tt-smi.json#L1-L5)

### 7. `cloud-infra/`

Tools and configurations for deploying Tenstorrent hardware in cloud or data center environments.

### 8. `riscv-arch/`

Deep dives and references for the RISC-V architecture used within the Tensix cores.

*   **Notable Entry:**`blog-anuraagw-blackhole-arch.json` - Detailed architectural analysis of the Blackhole chip.

### 9. `research/`

Academic papers, workshop presentations, and experimental studies involving Tenstorrent technology.

*   **Notable Entry:**`paper-nbody-wormhole.json` - Accelerating astrophysical simulations on Wormhole [entries/research/paper-nbody-wormhole.json 1-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/research/paper-nbody-wormhole.json#L1-L4)
*   **Notable Entry:**`paper-swiftnpu.json` - Scalable allocation algorithms for multi-tenant NPUs [entries/research/paper-swiftnpu.json 1-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/research/paper-swiftnpu.json#L1-L4)

### 10. `games-demos/`

Creative applications, games, and interactive demonstrations.

*   **Notable Entry:**`tt-zork-and-more.json` - Text adventures running on Tenstorrent hardware.

### 11. `guides/`

Tutorials, blog posts, and educational resources.

*   **Notable Entry:**`tt-vscode-toolkit.json` - 48 interactive lessons for Tenstorrent developers [entries/guides/tt-vscode-toolkit.json 1-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/guides/tt-vscode-toolkit.json#L1-L4)
*   **Notable Entry:**`blog-martin-chang-programming.json` - A "honest guide" to circular buffers and NoC routing [entries/guides/blog-martin-chang-programming.json 1-4](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/guides/blog-martin-chang-programming.json#L1-L4)

## Data Flow and Implementation

The `entries/` directory is processed by a series of scripts to generate the final website and README.

**Entry Data Pipeline**

**Sources:**`scripts/validate.py`(), `src/_data/entries.js`(), `scripts/generate_readme.py`()

### Implementation Details

1.   **Validation:**`scripts/validate.py` ensures that every file in `entries/` matches the schema. It specifically checks that the `id` field matches the filename and that the file is located in the directory corresponding to its first category [scripts/validate.py 1-50](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/validate.py#L1-L50)
2.   **Affiliation:** Entries are marked as `official`, `affiliated`, or `community`. This determines their sorting order in the generated README and website [entries/guides/tt-vscode-toolkit.json 5](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/guides/tt-vscode-toolkit.json#L5-L5)[entries/kernels/jasondavies-tt-series.json 5](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/kernels/jasondavies-tt-series.json#L5-L5)
3.   **Links:** Each entry supports multiple link types (`repo`, `article`, `video`, `paper`, `lesson`, etc.), which are rendered with specific icons in the UI [entries/kernels/jasondavies-tt-series.json 19-50](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/kernels/jasondavies-tt-series.json#L19-L50)
4.   **Hardware:** The `hardware` array (e.g., `grayskull`, `wormhole`, `blackhole`) allows users to filter projects by the generation of silicon they support [entries/ai-models/tt-boltz.json 28-32](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/ai-models/tt-boltz.json#L28-L32)

### Feature Flags

Entries can include special flags:

*   `featured: true`: Promotes the entry to the "Featured" section of its category [entries/kernels/jasondavies-tt-series.json 55](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/kernels/jasondavies-tt-series.json#L55-L55)
*   `embed`: Allows interactive components (like `tensix-viz`) to be rendered directly in the entry detail view [entries/dev-tools/tensix-viz.json 32-34](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/entries/dev-tools/tensix-viz.json#L32-L34)
