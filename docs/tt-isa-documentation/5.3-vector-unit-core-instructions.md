This page documents the fundamental computational instructions executed by the Vector Unit (SFPU) within the Tensix coprocessor. These instructions operate on the Vector Unit's `LReg[0-16]` register file, with each register containing 32 lanes of 32-bit data.

**Scope:** Core computational instructions including `SFPMAD` (fused multiply-add), `SFPLUT` / `SFPLUTFP32` (piecewise linear function evaluation), `SFPTRANSP` (transpose), and `SFPIADD` (integer add/subtract). For data movement instructions including `SFPLOAD` and `SFPSTORE`, see page 5.4. For configuration and control flow instructions, see page 5.5. For complete Vector Unit architecture details, see page 4.5.

## SFPMAD - Fused Multiply-Add

`SFPMAD` performs lanewise FP32 fused multiply-add operations with partial fusion semantics (between IEEE754 fully-fused FMA and separate multiply-add).

### Instruction Format

`TT_SFPMAD(/* u4 */ VA, /* u4 */ VB, /* u4 */ VC, /* u4 */ VD, /* u4 */ Mod1)`
**Field encoding:**

*   `VA`: Source register A index (4 bits, 0-15)
*   `VB`: Source register B index (4 bits, 0-15)
*   `VC`: Source register C index (4 bits, 0-15)
*   `VD`: Destination register index (4 bits, 0-16)
*   `Mod1`: Modifier flags (4 bits)

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 10-18](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L10-L18)[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 7-15](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L7-L15)

### Functional Behavior

**Pseudocode:**

`lanewise {  if (VD < 12 || LaneConfig[Lane].DISABLE_BACKDOOR_LOAD) {    if (LaneEnabled) {      unsigned va = Mod1 & SFPMAD_MOD1_INDIRECT_VA ? LReg[7].u32 & 15 : VA;      uint32_t a = LReg[va].u32;      uint32_t b = LReg[VB].u32;      uint32_t c = LReg[VC].u32;            // Blackhole only:      if (Mod1 & SFPMAD_MOD1_NEGATE_VA) a ^= 0x80000000;      if (Mod1 & SFPMAD_MOD1_NEGATE_VC) c ^= 0x80000000;            uint32_t d = fma_model(a, b, c); // compute a * b + c            unsigned vd;      if ((Mod1 & SFPMAD_MOD1_INDIRECT_VD) && VD != 16) {        vd = LReg[7].u32 & 15;      } else {        vd = VD;      }            if (vd < 8 || vd == 16) {        LReg[vd].u32 = d;      }    }  }}`
**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 20-54](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L20-L54)[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 17-47](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L17-L47)

### Modifier Flags

| Modifier (Blackhole) | Bit | Effect |
| --- | --- | --- |
| `SFPMAD_MOD1_NEGATE_VA` | 0 | XOR source A with `0x80000000` (negate sign) |
| `SFPMAD_MOD1_NEGATE_VC` | 1 | XOR source C with `0x80000000` (negate sign) |
| `SFPMAD_MOD1_INDIRECT_VA` | 2 | Use `LReg[7] & 15` as VA index instead of instruction VA field |
| `SFPMAD_MOD1_INDIRECT_VD` | 3 | Use `LReg[7] & 15` as VD index instead of instruction VD field |

**Wormhole differences:** Only `INDIRECT_VA` (bit 2) and `INDIRECT_VD` (bit 3) are available. Bits 0-1 have no effect.

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 47-54](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L47-L54)[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 42-47](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L42-L47)

### IEEE754 Conformance and Divergence

**Key divergences from IEEE754 fully-fused FMA:**

1.   **Denormal handling:** Input denormals treated as signed zero, output denormals flushed to zero
2.   **Partial fusion:** Product `x * y` kept in higher precision than FP32 but not infinite precision
3.   **Overflow/underflow:** If `x * y` alone would overflow/underflow, result differs from fully-fused FMA
4.   **NaN bit patterns:** Blackhole uses canonical NaN `0x7fc00000`, Wormhole uses non-canonical patterns

**Bit-perfect software models:**

*   Blackhole: `fma_model_bh()` in [Miscellaneous/FMA/fma.c 102-189](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/fma.c#L102-L189)
*   Wormhole: `fma_model_wh()` in [Miscellaneous/FMA/fma.c 191-287](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/fma.c#L191-L287)
*   IEEE754 reference: `fma_model_ieee()` in [Miscellaneous/FMA/fma.c 14-100](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/fma.c#L14-L100)

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 56-66](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L56-L66)[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 49-59](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L49-L59)[Miscellaneous/FMA/README.md 1-31](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/README.md?plain=1#L1-L31)

### Instruction Scheduling and Hazards

**Latency:**`SFPMAD` requires 2 cycles to compute its result.

#### Blackhole: Automatic Hazard Detection

Hardware automatically stalls threads when detecting read-after-write (RAW) hazards. If `SFPMAD` writes to a register, and the next instruction reads from it, hardware inserts a 1-cycle stall.

**Limitations of automatic stalling:**

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 68-87](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L68-L87)

#### Wormhole: Manual Hazard Management

**No automatic stalling.** Software must insert `SFPNOP` instructions to avoid RAW hazards:

`TT_SFPMAD(0, 1, 2, 3, 0);  // Writes to LReg[3]TT_SFPNOP(0, 0);            // Mandatory 1-cycle gapTT_SFPADD(3, 4, 5, 0);      // Reads from LReg[3]`
**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md 61-63](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L61-L63)

### Performance Characteristics

**Theoretical peak:** Requires `SFPLOADMACRO` to overlap data movement with computation (see [Vector Unit Utility Instructions](https://deepwiki.com/tenstorrent/tt-isa-documentation/5.4-vector-unit-data-movement-instructions)).

## SFPIADD - Integer Add/Subtract

`SFPIADD` performs lanewise 32-bit integer addition or subtraction. It can operate on register operands or add a sign-extended 12-bit immediate value. The instruction can also set lane flags based on the sign of the result.

**Backend execution unit:** Vector Unit (SFPU), simple sub-unit

### Instruction Format

`TT_SFPIADD(/* i12 */ (Imm12 & 0xfff), /* u4 */ VC, /* u4 */ VD, /* u4 */ Mod1)`
**Field encoding:**

*   `Imm12`: 12-bit signed immediate value (sign-extended to 32 bits)
*   `VC`: Source register C index (4 bits, 0-15)
*   `VD`: Destination register index (4 bits, 0-16)
*   `Mod1`: Modifier flags (4 bits)

### Functional Behavior

`unsigned VB = VD;if (VD < 8 || VD == 16) {  lanewise {    if (LaneEnabled) {      if (Mod1 & SFPIADD_MOD1_ARG_IMM) {        LReg[VD].u32 = LReg[VC].u32 + SignExtend(Imm12);      } else if (Mod1 & SFPIADD_MOD1_ARG_2SCOMP_LREG_DST) {        LReg[VD].u32 = LReg[VC].u32 - LReg[VB].u32;      } else {        LReg[VD].u32 = LReg[VC].u32 + LReg[VB].u32;      }      if (VD < 8) {        if (Mod1 & SFPIADD_MOD1_CC_NONE) {          // Leave LaneFlags as-is.        } else {          LaneFlags = LReg[VD].i32 < 0;        }        if (Mod1 & SFPIADD_MOD1_CC_GTE0) {          LaneFlags = !LaneFlags;        }      }    }  }}`
**Note:** The instruction implicitly uses `VD` as source register `VB` when `SFPIADD_MOD1_ARG_IMM` is not set.

### Modifier Flags

| Modifier | Bit Pattern | Effect |
| --- | --- | --- |
| `SFPIADD_MOD1_ARG_LREG_DST` | 0 | Add: `VD = VC + VB` (where `VB = VD`) |
| `SFPIADD_MOD1_ARG_IMM` | 1 | Add immediate: `VD = VC + SignExtend(Imm12)` |
| `SFPIADD_MOD1_ARG_2SCOMP_LREG_DST` | 2 | Subtract: `VD = VC - VB` (where `VB = VD`) |
| `SFPIADD_MOD1_CC_LT0` | 0 | Set `LaneFlags = (VD < 0)` (signed comparison) |
| `SFPIADD_MOD1_CC_NONE` | 4 | Do not modify `LaneFlags` |
| `SFPIADD_MOD1_CC_GTE0` | 8 | Set `LaneFlags = (VD >= 0)` (signed comparison) |

### Hazard Detection Bug (Blackhole)

The automatic stalling logic in Blackhole does not detect that `SFPIADD` reads from `VD` (which is used as `VB`). If a prior instruction writes to the register that `SFPIADD` uses as `VD`/`VB`, software must manually insert `SFPNOP`:

`TT_SFPMAD(0, 1, 2, 3, 0);  // Writes to LReg[3]TT_SFPNOP(0, 0);            // Required: stalling logic doesn't detect SFPIADD reads VDTT_SFPIADD(0, 4, 3, 0);     // Reads from LReg[3] as VB (VB = VD = 3)`
**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPIADD.md 1-58](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPIADD.md?plain=1#L1-L58)[BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md 79](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPMAD.md?plain=1#L79-L79)

## SFPLUT - Piecewise Linear Function Evaluation (Wormhole)

`SFPLUT` evaluates a piecewise linear approximation to a function by selecting coefficients from a lookup table based on the magnitude of the input value. The instruction performs a multiply-add operation using 8-bit encoded FP coefficients stored in `LReg[0-2]`.

**Backend execution unit:** Vector Unit (SFPU), MAD sub-unit

**Architecture:** Wormhole B0 only. For Blackhole, see `SFPLUTFP32` below.

### Instruction Format

`TT_SFPLUT(/* u4 */ VD, /* u4 */ Mod0, 0)`
**Field encoding:**

*   `VD`: Destination register index (4 bits, 0-16)
*   `Mod0`: Modifier flags (4 bits)

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md 1-18](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md?plain=1#L1-L18)

### Functional Model

Title: SFPLUT Computation Flow

**Pseudocode:**

`lanewise {  if (VD < 12 || LaneConfig[Lane].DISABLE_BACKDOOR_LOAD) {    if (LaneEnabled) {      uint32_t l3 = LReg[3].u32;      uint32_t b = l3 & 0x7FFFFFFF; // absolute value      uint32_t coeffs = (b < 0x3F800000) ? LReg[0].u32 : // b < 1.0f                        (b < 0x40000000) ? LReg[1].u32 : // b < 2.0f                        LReg[2].u32;      uint32_t a = Lut8ToFp32((coeffs >> 8) & 0xFF);      uint32_t c = Lut8ToFp32( coeffs       & 0xFF);      uint32_t d = fma_model_wh(a, b, c); // compute a * b + c      if (Mod0 & SFPLUT_MOD0_SGN_RETAIN) {        d = (d & 0x7FFFFFFF) | (l3 & 0x80000000); // copy sign bit from l3      }      unsigned vd;      if ((Mod0 & SFPLUT_MOD0_INDIRECT_VD) && VD != 16) {        vd = LReg[7].u32 & 15;      } else {        vd = VD;      }      if (vd < 8 || vd == 16) {        LReg[vd].u32 = d;      }    }  }}`
**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md 26-54](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md?plain=1#L26-L54)

### Lut8ToFp32 Encoding

The `Lut8ToFp32` function converts an 8-bit encoded value to FP32:

`uint32_t Lut8ToFp32(uint8_t x) {  if (x == 0xff) return 0;  uint32_t Sign = x >> 7;  uint32_t Exp = (x >> 4) & 7;  uint32_t Man = x & 0xf;  return (Sign << 31) | ((127 - Exp) << 23) | (Man << 19);}`
The 256 possible values represent a logarithmically-distributed set of FP32 values ranging from approximately ±0.0078125 to ±1.9375. The encoding dedicates 3 bits to exponent and 4 bits to mantissa, with the exponent providing coverage across multiple orders of magnitude.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md 56-68](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md?plain=1#L56-L68)

### Modifier Flags

| Modifier | Bit | Effect |
| --- | --- | --- |
| `SFPLUT_MOD0_SGN_RETAIN` | 2 | Copy sign bit from `LReg[3]` to result |
| `SFPLUT_MOD0_INDIRECT_VD` | 3 | Use `LReg[7] & 15` as VD index instead of instruction VD field |

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md 56-59](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md?plain=1#L56-L59)

### IEEE754 Conformance

The FMA computation follows the same model as `SFPMAD` for Wormhole (see `fma_model_wh` in [Miscellaneous/FMA/fma.c 191-287](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/fma.c#L191-L287)). Key characteristics:

*   Denormal inputs treated as signed zero
*   Output denormals flushed to positive zero (sign discarded)
*   Non-canonical NaN outputs (≥ `0x7f800001`)
*   Partial fusion (not fully IEEE754-compliant FMA)

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md 83-85](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md?plain=1#L83-L85)

### Instruction Scheduling

`SFPLUT` requires 2 cycles to compute its result. Software must insert `SFPNOP` after `SFPLUT` if the next instruction reads from the destination register (Wormhole has no automatic hazard detection).

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md 87-89](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPLUT.md?plain=1#L87-L89)

## SFPLUTFP32 - Piecewise Linear Function Evaluation (Blackhole)

`SFPLUTFP32` provides enhanced piecewise linear function evaluation with multiple modes: 3-entry FP32 tables, 3-entry FP16 tables, and 6-entry FP16 tables with finer granularity. This instruction supersedes Wormhole's `SFPLUT`.

**Backend execution unit:** Vector Unit (SFPU), MAD sub-unit

**Architecture:** Blackhole A0 only.

### Instruction Format

`TT_SFPLUTFP32(/* u4 */ VD, /* u4 */ Mod1)`
**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 48-52](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L48-L52)

### Table Modes

Title: SFPLUTFP32 Table Selection and Range Mapping

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 1-46](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L1-L46)

### Mode Descriptions

#### SFPLUTFP32_MOD1_FP32_3ENTRY_TABLE (Mode 0)

Three ranges using full FP32 coefficient storage:

| Input Range | Computation |
| --- | --- |
| 0.0 ≤ `Abs(LReg[3])`< 1.0 | `VD = LReg[0] * Abs(LReg[3]) + LReg[4]` |
| 1.0 ≤ `Abs(LReg[3])`< 2.0 | `VD = LReg[1] * Abs(LReg[3]) + LReg[5]` |
| 2.0 ≤ `Abs(LReg[3])` | `VD = LReg[2] * Abs(LReg[3]) + LReg[6]` |

**Coefficient storage:** Requires 6 `LReg` registers for coefficients (slope in `LReg[0-2]`, intercept in `LReg[4-6]`).

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 6-11](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L6-L11)

#### SFPLUTFP32_MOD1_FP16_3ENTRY_TABLE (Mode 10)

Three ranges using FP16-encoded coefficients packed into 32-bit values:

| Input Range | Computation |
| --- | --- |
| 0.0 ≤ `Abs(LReg[3])`< 1.0 | `LReg[LReg[7] & 15] = Lut16ToFp32(LReg[0] >> 16) * Abs(LReg[3]) + Lut16ToFp32(LReg[0])` |
| 1.0 ≤ `Abs(LReg[3])`< 2.0 | `LReg[LReg[7] & 15] = Lut16ToFp32(LReg[1] >> 16) * Abs(LReg[3]) + Lut16ToFp32(LReg[1])` |
| 2.0 ≤ `Abs(LReg[3])` | `LReg[LReg[7] & 15] = Lut16ToFp32(LReg[2] >> 16) * Abs(LReg[3]) + Lut16ToFp32(LReg[2])` |

**Hardware bug:** Destination is `LReg[LReg[7] & 15]` instead of `LReg[VD]`. Software must load the desired destination index into the low 4 bits of `LReg[7]`.

**Coefficient storage:** Requires 3 `LReg` registers, each containing two 16-bit encoded coefficients.

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 13-21](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L13-L21)

#### SFPLUTFP32_MOD1_FP16_6ENTRY_TABLE1 (Mode 2)

Six ranges providing finer granularity using FP16-encoded coefficients:

| Input Range | Computation |
| --- | --- |
| 0.0 ≤ `Abs(LReg[3])`< 0.5 | `VD = Lut16ToFp32(LReg[0]) * Abs(LReg[3]) + Lut16ToFp32(LReg[4])` |
| 0.5 ≤ `Abs(LReg[3])`< 1.0 | `VD = Lut16ToFp32(LReg[0] >> 16) * Abs(LReg[3]) + Lut16ToFp32(LReg[4] >> 16)` |
| 1.0 ≤ `Abs(LReg[3])`< 1.5 | `VD = Lut16ToFp32(LReg[1]) * Abs(LReg[3]) + Lut16ToFp32(LReg[5])` |
| 1.5 ≤ `Abs(LReg[3])`< 2.0 | `VD = Lut16ToFp32(LReg[1] >> 16) * Abs(LReg[3]) + Lut16ToFp32(LReg[5] >> 16)` |
| 2.0 ≤ `Abs(LReg[3])`< 3.0 | `VD = Lut16ToFp32(LReg[2]) * Abs(LReg[3]) + Lut16ToFp32(LReg[6])` |
| 3.0 ≤ `Abs(LReg[3])` | `VD = Lut16ToFp32(LReg[2] >> 16) * Abs(LReg[3]) + Lut16ToFp32(LReg[6] >> 16)` |

**Coefficient storage:** Requires 6 `LReg` registers, each containing two 16-bit encoded coefficients (12 coefficient pairs total).

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 23-32](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L23-L32)

#### SFPLUTFP32_MOD1_FP16_6ENTRY_TABLE2 (Mode 3)

Identical to TABLE1 except the final range split occurs at 4.0 instead of 3.0:

| Input Range | Final Two Ranges |
| --- | --- |
| 2.0 ≤ `Abs(LReg[3])`< 4.0 | Use `LReg[2]` coefficients |
| 4.0 ≤ `Abs(LReg[3])` | Use `LReg[2] >> 16` coefficients |

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 34-45](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L34-L45)

### Lut16ToFp32 Encoding

Converts 16-bit encoded values to FP32 with special handling for denormals and infinities:

`float Lut16ToFp32(uint16_t x) {  uint32_t Sign = x >> 15;  uint32_t Exp  = (x >> 10) & 0x1f;  uint32_t Man  = x & 0x3ff;  return std::bit_cast<float>((Sign << 31) | ((Exp == 0x1f ? 0 : 112 + Exp) << 23) | (Man << 13));}`
**Key differences from IEEE754 FP16:**

*   Exponent value `0x1f` (infinity/NaN in IEEE754) maps to zero
*   Exponent value `0x00` (denormal in IEEE754) maps to normalized FP32 with exponent `112`

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 119-139](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L119-L139)

### Modifier Flags

| Modifier | Bit Pattern | Effect |
| --- | --- | --- |
| `SFPLUTFP32_MOD1_FP32_3ENTRY_TABLE` | 0 | Use FP32 3-entry mode |
| `SFPLUTFP32_MOD1_FP16_3ENTRY_TABLE` | 10 | Use FP16 3-entry mode (overlaps with `INDIRECT_VD`) |
| `SFPLUTFP32_MOD1_FP16_6ENTRY_TABLE1` | 2 | Use FP16 6-entry mode (split at 3.0) |
| `SFPLUTFP32_MOD1_FP16_6ENTRY_TABLE2` | 3 | Use FP16 6-entry mode (split at 4.0) |
| `SFPLUTFP32_MOD1_SGN_RETAIN` | 4 | Copy sign bit from `LReg[3]` to result |
| `SFPLUTFP32_MOD1_INDIRECT_VD` | 8 | Use `LReg[7] & 15` as VD index (overlaps with 3ENTRY_TABLE) |

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 108-117](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L108-L117)

### IEEE754 Conformance

The FMA computation follows the Blackhole model (`fma_model_bh` in [Miscellaneous/FMA/fma.c 102-189](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Miscellaneous/FMA/fma.c#L102-L189)):

*   Denormal inputs treated as signed zero
*   Output denormals flushed to signed zero (sign preserved)
*   Canonical NaN output (`0x7fc00000`)
*   Partial fusion semantics

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 141-143](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L141-L143)

### Instruction Scheduling

`SFPLUTFP32` has 2-cycle latency. Automatic stalling logic applies but has limitations:

*   When `INDIRECT_VD` is set in `Mod1Mirror`: assumes all `LReg` registers are read/written
*   When `INDIRECT_VD` is clear: assumes `LReg[0-6]` read, `VD` written
*   When using `FP16_3ENTRY_TABLE`: conservatively assumes `LReg[4-6]` are read even though they aren't used (performance penalty)

Software can disable automatic stalling if needed, but must then manage hazards manually.

**Sources:**[BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md 145-153](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/BlackholeA0/TensixTile/TensixCoprocessor/SFPLUTFP32.md?plain=1#L145-L153)

## SFPTRANSP - Transpose Rows Between Vector Registers

`SFPTRANSP` performs a specialized transpose operation on groups of four vector registers, treating each register as a 4×8 grid and transposing within each column. This operation is useful for data reorganization in SIMD algorithms.

**Backend execution unit:** Vector Unit (SFPU), simple sub-unit

### Instruction Format

`TT_SFPTRANSP(0, 0, /* u4 */ VD, 0)`
The `VD` field is not used for addressing but controls the `DISABLE_BACKDOOR_LOAD` check.

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md 10-14](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md?plain=1#L10-L14)

### Transpose Pattern

Title: SFPTRANSP Data Movement (4×4 Transpose Within Each Column)

**Important:** This is NOT a standard matrix transpose. The instruction:

1.   Views `LReg[0-3]` as a vertically-stacked 16×8 grid (32 lanes split across 4 registers)
2.   For each of the 8 columns independently: 
    *   Extracts the 16 values in that column
    *   Reshapes them to a 4×4 matrix
    *   Transposes the 4×4 matrix
    *   Writes the result back to the same column

3.   Repeats the process for `LReg[4-7]`

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md 1-7](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md?plain=1#L1-L7)

### Cross-Lane Data Movement

Title: SFPTRANSP Lane Mapping for a Single Column

The full cross-lane data movement pattern (assuming all 32 lanes active) is shown in [WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md 20-24](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md?plain=1#L20-L24)

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md 20-24](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md?plain=1#L20-L24)

### Functional Model

`if (VD < 12 || LaneConfig.DISABLE_BACKDOOR_LOAD) {  Transpose4(0);  Transpose4(4);} void Transpose4(unsigned Base) {  for (unsigned Column = 0; Column < 8; ++Column) {    for (unsigned i = 0; i < 4; ++i) {      for (unsigned j = 0; j < i; ++j) {        auto ij = LReg[Base + i][j * 8 + Column];        auto ji = LReg[Base + j][i * 8 + Column];        if (LaneEnabled[j * 8 + Column]) LReg[Base + i][j * 8 + Column] = ji;        if (LaneEnabled[i * 8 + Column]) LReg[Base + j][i * 8 + Column] = ij;      }    }  }}`
The operation processes two independent groups:

*   `Transpose4(0)`: Transposes `LReg[0-3]`
*   `Transpose4(4)`: Transposes `LReg[4-7]`

**Sources:**[WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md 28-50](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/TensixCoprocessor/SFPTRANSP.md?plain=1#L28-L50)

### Typical Usage Pattern

`SFPTRANSP` is commonly used in algorithms that need to reorganize data for efficient SIMD processing:

`// Load data into LReg[0-3] in row-major orderTT_SFPLOAD(0, 0, 0, addr0);TT_SFPLOAD(1, 0, 0, addr1);TT_SFPLOAD(2, 0, 0, addr2);TT_SFPLOAD(3, 0, 0, addr3); // Transpose to column-major order for different access patternTT_SFPTRANSP(0, 0, 0, 0); // Process transposed dataTT_SFPMAD(0, 4, 5, 0, 0);TT_SFPMAD(1, 4, 5, 1, 0);TT_SFPMAD(2, 4, 5, 2, 0);TT_SFPMAD(3, 4, 5, 3, 0); // Transpose back if neededTT_SFPTRANSP(0, 0, 0, 0);`

## Architecture Comparison: Wormhole vs Blackhole

| Feature | WormholeB0 | BlackholeA0 |
| --- | --- | --- |
| **SFPMAD Modifiers** | `INDIRECT_VA`, `INDIRECT_VD` | `INDIRECT_VA`, `INDIRECT_VD`, `NEGATE_VA`, `NEGATE_VC` |

| **Clock Rate** | 1.0 GHz | 1.35 GHz |
| **Peak FLOP/s** | 0.064 TFLOP/s per Vector Unit | 0.0864 TFLOP/s per Vector Unit |
| **FMA Model** | `fma_model_wh()` | `fma_model_bh()` |

## Programming Guidelines

### Hazard Avoidance

**Blackhole:**

`// Safe: Automatic stalling handles common casesTT_SFPMAD(0, 1, 2, 3, 0);TT_SFPADD(3, 4, 5, 0);      // Auto-stall inserted // Unsafe: SFPIADD hazard not detectedTT_SFPMAD(0, 1, 2, 3, 0);TT_SFPIADD(3, 4, 5, 0);     // BUG: No auto-stall // Safe: Manual SFPNOP insertionTT_SFPMAD(0, 1, 2, 3, 0);TT_SFPNOP(0, 0);TT_SFPIADD(3, 4, 5, 0);     // Correct`
**Wormhole:**

`// Always insert SFPNOP after SFPMADTT_SFPMAD(0, 1, 2, 3, 0);TT_SFPNOP(0, 0);            // MandatoryTT_SFPADD(3, 4, 5, 0);`
### Register Allocation

*   `LReg[0-7]`: General-purpose, read/write from all instructions
*   `LReg[8-11]`: Reserved for special purposes
*   `LReg[12-15]`: Protected by `DISABLE_BACKDOOR_LOAD` flag
*   `LReg[16]`: Special register for certain operations
*   `LReg[7]`: Used for indirect addressing in `INDIRECT_VA`/`INDIRECT_VD` modes

### Performance Optimization

1.   **Use SFPLOADMACRO** for peak throughput (see [Vector Unit Utility Instructions](https://deepwiki.com/tenstorrent/tt-isa-documentation/5.4-vector-unit-data-movement-instructions))
2.   **Minimize indirect addressing** (triggers conservative hazard detection in Blackhole)
3.   **Batch operations** to amortize instruction fetch overhead
4.   **Align data movement** with computation to hide latency
