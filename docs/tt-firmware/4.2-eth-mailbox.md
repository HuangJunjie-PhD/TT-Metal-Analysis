# ETH Mailbox

Relevant source files
*   [README.md](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1)

## Purpose and Scope

The ETH Mailbox is a communication mechanism introduced in Blackhole ERISC Firmware v1.4.0. It provides a structured way to interact with the Ethernet subsystem and control execution on the RISC0 processor. This document covers the architecture, message types, and functionality of the ETH Mailbox system specifically for Blackhole boards. For information about the broader ERISC Firmware functionality, see [ERISC Firmware](https://deepwiki.com/tenstorrent/tt-firmware/4.1-erisc-firmware). For details about debugging capabilities through Virtual UART, see [Virtual UART](https://deepwiki.com/tenstorrent/tt-firmware/4.3-virtual-uart).

Sources: [README.md 35-38](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L35-L38)

## Architecture Overview

The ETH Mailbox serves as a communication interface between the host system and the Blackhole board's firmware, specifically for Ethernet-related operations and execution control. It enables structured message passing for diagnostics and runtime control of the hardware.

## Message Types

The ETH Mailbox implements two distinct message types, each serving a specific purpose in the Blackhole firmware ecosystem:

### LINK_STATUS_CHECK

The LINK_STATUS_CHECK message type is designed to query and report the status of Ethernet links on the Blackhole board. When this message is processed, the firmware checks the current state of the Ethernet connection and returns appropriate status information.

Key capabilities:

*   Queries the current status of Ethernet links
*   Reports connectivity state
*   Facilitates network troubleshooting

### RELEASE_CORE

The RELEASE_CORE message type performs a critical control function. It releases control of RISC0 (one of the processors on the Blackhole board) to execute a function at a specified L1 memory address. This capability enables dynamic programming of the processor's execution flow.

Key capabilities:

*   Takes a parameter specifying the L1 address to execute
*   Transfers control to RISC0
*   Enables flexible reprogramming of processor behavior

## Integration with Virtual UART

The ETH Mailbox system works in conjunction with the Virtual UART feature, which was enabled by default in the same firmware update (Blackhole ERISC FW v1.4.0). This integration provides enhanced debugging capabilities:

*   Debug messages generated during ETH Mailbox operations can be viewed through the Virtual UART
*   The `tt-console` utility can be used to view `printk()` and `LOG_*()` messages from the host
*   This facilitates troubleshooting of both LINK_STATUS_CHECK and RELEASE_CORE operations

## Usage and Integration

The ETH Mailbox provides critical functionality for Blackhole boards, particularly for:

1.   **Network Diagnostics**: Using LINK_STATUS_CHECK to verify and troubleshoot Ethernet connectivity
2.   **Dynamic Execution Control**: Using RELEASE_CORE to execute specific functions at runtime

Applications typically interact with the ETH Mailbox through the TT User Mode Driver (tt-umd), which provides the necessary interface to send messages and receive responses.

### Debugging ETH Mailbox Operations

For debugging ETH Mailbox operations:

1.   Ensure Virtual UART is enabled (default in Blackhole firmware bundles since v1.4.0)
2.   Use `tt-console` to capture and display debug messages
3.   Look for specific printk() and LOG_*() messages related to ETH Mailbox operations

## Summary

The ETH Mailbox introduced in Blackhole ERISC FW v1.4.0 provides two critical message types:

| Message Type | Purpose | Key Functionality |
| --- | --- | --- |
| LINK_STATUS_CHECK | Network diagnostics | Checks and reports Ethernet link status |
| RELEASE_CORE | Execution control | Releases RISC0 to execute functions at specified L1 addresses |

This feature, combined with the Virtual UART capability, enhances both the functionality and observability of Blackhole boards, enabling more flexible control and easier troubleshooting.
