This document provides a technical overview of the Vector Processing Unit (VPU) in the RISC-V Ocelot processor. The VPU implements the RISC-V Vector Extension (V), enabling efficient parallel data processing. For information about the interface between the scalar core and the VPU, see [OviWrapper Interface](https://deepwiki.com/tenstorrent/riscv-ocelot/5.1-oviwrapper-interface).

## VPU Architecture Overview

The Vector Processing Unit is implemented as a modular pipeline that processes vector instructions issued from the scalar core. It consists of several key components that handle instruction decoding, execution, memory operations, and register file access.

Sources: [src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 1-184](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L1-L184)[src/main/resources/vsrc/vpu/tt_vec.sv 1-66](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vec.sv#L1-L66)

## Instruction Flow and Pipeline Stages

The VPU uses a multi-stage pipeline to process vector instructions with support for out-of-order execution within the vector domain.

Sources: [src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 296-328](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L296-L328)[src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 357-650](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L357-L650)

### Instruction Buffer (FIFO)

The instruction FIFO (`tt_fifo`) serves as the entry point for vector instructions from the scalar core. It performs several critical functions:

1.   Buffers instructions until they can be processed
2.   Handles instruction dispatch and tracking
3.   Manages load data buffer allocation for vector loads
4.   Initiates memory synchronization operations

The FIFO maintains a queue of instructions with their associated metadata, including:

```
- instruction opcode
- scoreboard ID
- scalar operand
- vector CSR state (VTYPE, VL)
- dispatch status flags
```

Sources: [src/main/resources/vsrc/vpu/tt_fifo.sv 1-38](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_fifo.sv#L1-L38)[src/main/resources/vsrc/vpu/tt_fifo.sv 40-65](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_fifo.sv#L40-L65)[src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 296-320](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L296-L320)

### Instruction Decode (ID) Stage

The ID stage (`tt_id`) decodes vector instructions and prepares them for execution. It extracts key instruction fields, determines operation types, identifies register dependencies, and manages register addressing.

Key functions of the ID stage include:

1.   Instruction field extraction and operation classification
2.   Register operand identification and forwarding
3.   Vector instruction replay logic for multi-cycle operations
4.   Handling of vector element width (SEW) and register group length (LMUL)
5.   Managing load/store instructions with different addressing modes

The ID stage also supports instruction replay mechanisms for complex vector operations that need to be broken down into multiple micro-operations.

Sources: [src/main/resources/vsrc/vpu/tt_id.sv 1-65](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_id.sv#L1-L65)[src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 357-452](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L357-L452)

### Execution Units

The VPU includes two primary execution paths:

1.   **EX Stage (`tt_ex`)**: Handles vector memory operations and some scalar-vector operations. It computes effective addresses for vector loads/stores and supports various addressing modes (unit-stride, strided, indexed).

2.   **Vector Execution (`tt_vec`)**: Processes vector arithmetic, logical, and reduction operations. It supports operations between:

    *   Vector and vector operands
    *   Vector and scalar operands
    *   Vector operations with immediate values

The execution units work with different element widths (8, 16, 32, 64 bits) and handle mask operations for predicated execution.

Sources: [src/main/resources/vsrc/vpu/tt_ex.sv 1-90](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_ex.sv#L1-L90)[src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 479-565](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L479-L565)[src/main/resources/vsrc/vpu/tt_vec.sv 1-66](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vec.sv#L1-L66)

## Vector Register File

The Vector Register File (`tt_vec_regfile`) provides storage for vector operands and results. It's implemented in [src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 585-600](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L585-L600) and has the following characteristics:

*   Supports multiple read ports for parallel operand access
*   Has dedicated write ports for results
*   Stores mask registers for predicated execution
*   Handles different vector register widths based on VLEN parameter

The register file interfaces with the ID, EX, and VEX stages, providing operand data and storing execution results.

## Memory Operation Handling

Vector memory operations (loads and stores) require specialized handling due to their complexity. The VPU includes several components dedicated to memory operations:

### Load Queue (LQ)

The Load Queue (`tt_lq`) manages vector load operations, including:

1.   Tracking pending load requests
2.   Buffering returned load data
3.   Coalescing related load operations
4.   Forwarding load data to dependent instructions

The LQ interfaces with the memory system and execution units, ensuring proper synchronization of memory operations.

Sources: [src/main/resources/vsrc/vpu/tt_lq.sv 1-106](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_lq.sv#L1-L106)[src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 667-769](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L667-L769)

### Mask and Store FSMs

The VPU includes specialized finite state machines for handling masked and store operations:

1.   **Mask FSM (`tt_mask_fsm`)**: Manages masked memory operations where only selected elements are accessed.

    *   Generates memory access patterns based on mask bits
    *   Handles different addressing modes with masked elements
    *   Coordinates with the Load Queue for proper synchronization

2.   **Store FSM (`tt_store_fsm`)**: Manages vector store operations.

    *   Buffers store data before sending to memory
    *   Handles store data gathering for different element widths
    *   Manages store credits to control memory bandwidth usage

3.   **Memory Operation FSM (`tt_memop_fsm`)**: Coordinates overall memory operation flow.

    *   Manages synchronization between memory operations
    *   Controls the start and end of memory operations
    *   Handles completion signaling

Sources: [src/main/resources/vsrc/vpu/tt_mask_fsm.sv 1-23](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_mask_fsm.sv#L1-L23)[src/main/resources/vsrc/vpu/tt_store_fsm.sv 1-24](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_store_fsm.sv#L1-L24)[src/main/resources/vsrc/vpu/tt_memop_fsm.sv 1-14](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_memop_fsm.sv#L1-L14)

## Scoreboarding and Instruction Tracking

The scoreboard (`tt_scoreboard_ovi`) is a critical component that tracks vector instruction status and manages dependencies. Its primary functions include:

1.   Tracking instruction state from issue to completion
2.   Managing dependencies between vector instructions
3.   Coordinating memory operation handling
4.   Signaling instruction completion back to the scalar core

The scoreboard maintains entries for all in-flight vector instructions, with each entry containing:

```
- Destination register information
- Execution status flags
- Exception information
- Load/store operation tracking
- Completion status
```

The scoreboard determines when instructions can be considered complete and signals completion to the scalar core, including any exception information.

Sources: [src/main/resources/vsrc/vpu/tt_scoreboard_ovi.sv 1-62](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_scoreboard_ovi.sv#L1-L62)[src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 912-970](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L912-L970)

## Vector Instruction Support

The VPU supports the RISC-V Vector Extension (V) instruction set, including:

| Instruction Category | Examples | Description |
| --- | --- | --- |
| Vector Arithmetic | VADD, VMUL, VSUB | Integer and floating-point arithmetic operations |
| Vector Logical | VAND, VOR, VXOR | Bitwise logical operations |
| Vector Comparison | VSEQ, VSLT, VMSEQ | Element-wise comparison, producing mask or vector results |
| Vector Reduction | VREDSUM, VREDAND | Reduction operations that produce scalar results |
| Vector Load/Store | VLE, VSE | Memory access with unit-stride, strided, or indexed addressing |
| Vector Mask | VMAND, VMOR | Operations on mask registers |
| Vector Permutation | VSLIDEUP, VSLIDEDOWN | Element permutation within vectors |

Sources: [src/main/resources/vsrc/vpu/tt_id.sv 615-637](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_id.sv#L615-L637)[src/main/resources/vsrc/vpu/autogen_riscv_imabfv.v 1000-1250](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/autogen_riscv_imabfv.v#L1000-L1250)

## Implementation Details

### Vector Length and Element Width

The VPU supports configurable vector length (VLEN) and element widths:

*   Vector registers are VLEN bits wide (default 256 bits)
*   Supported element widths (SEW): 8, 16, 32, and 64 bits
*   LMUL (register group multiplier) values: 1, 2, 4, and 8
*   Fractional LMUL values: 1/2, 1/4, and 1/8

The vector length register (vl) determines how many elements are processed in each operation, up to the maximum vector length (VLEN/SEW).

Sources: [src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 52-61](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L52-L61)[src/main/resources/vsrc/vpu/tt_id.sv 129-134](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_id.sv#L129-L134)

### Pipeline Control and Stalling

The VPU implements control mechanisms to handle pipeline hazards and stalls:

The pipeline control logic ensures proper handling of:

*   Resource conflicts (register file ports, execution units)
*   Data hazards (RAW, WAW, WAR)
*   Structural hazards (queue full, load/store unit busy)

Sources: [src/main/resources/vsrc/vpu/tt_vpu_ovi.sv 568](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_vpu_ovi.sv#L568-L568)[src/main/resources/vsrc/vpu/tt_id.sv 184-193](https://github.com/tenstorrent/riscv-ocelot/blob/920fd904/src/main/resources/vsrc/vpu/tt_id.sv#L184-L193)

## Interface with Ocelot Core

The VPU interfaces with the Ocelot core through several key interfaces:

1.   **Instruction Issue Interface**: Receives vector instructions from the scalar core
2.   **Dispatch Control Interface**: Handles instruction dispatch and kill signals
3.   **Completion Interface**: Signals instruction completion and exception information
4.   **Memory Interface**: Connects to the memory system for loads and stores

These interfaces ensure proper coordination between the scalar and vector domains, allowing for efficient execution of mixed scalar-vector code.
