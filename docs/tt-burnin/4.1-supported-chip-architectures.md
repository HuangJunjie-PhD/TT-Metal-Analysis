# Supported Chip Architectures

Relevant source files
*   [CHANGELOG.md](https://github.com/tenstorrent/tt-burnin/blob/809f293d/CHANGELOG.md?plain=1)
*   [tt_burnin/chip.py](https://github.com/tenstorrent/tt-burnin/blob/809f293d/tt_burnin/chip.py)

This document details the chip architectures supported by the tt-burnin utility, including their specific implementations, characteristics, and detection mechanisms. The tt-burnin tool currently supports three major Tenstorrent chip architectures: Grayskull, Wormhole, and Blackhole. Each architecture has distinct grid layouts, coordinate systems, and hardware-specific behaviors that are abstracted through a unified interface.

For information about the TTX workload files specific to each architecture, see [TTX Workload Files](https://deepwiki.com/tenstorrent/tt-burnin/4.2-ttx-workload-files). For details about the hardware abstraction layer implementation, see [Hardware Abstraction Layer](https://deepwiki.com/tenstorrent/tt-burnin/2.2-hardware-abstraction-layer).

## Grayskull Architecture (GsChip)

Grayskull represents the first generation of Tenstorrent AI acceleration chips, featuring a 13x12 grid layout with specific coordinate mapping and harvesting mechanisms.

### Tensix Core Management

The `get_tensix_locations()` method handles harvesting (disabling faulty cores) by:

1.   Retrieving harvest bits via ARC message `0x57`
2.   Converting physical row bits to NOC coordinates
3.   Filtering out disabled rows from the valid tensix grid

The `RemoteWhChip` class extends `WhChip` for multi-chip configurations where broadcast operations must be implemented as individual core writes:

## Common Interface Methods

All chip architectures implement the abstract `TTChip` interface, providing consistent access to core functionality:

### Hardware Communication

*   `noc_read32(noc, x, y, addr)` - Read 32-bit value from NOC coordinate
*   `noc_write32(noc, x, y, addr, data)` - Write 32-bit value to NOC coordinate
*   `noc_broadcast32(noc, addr, data)` - Broadcast 32-bit value to all cores
*   `axi_read32(addr)` / `axi_write32(addr, value)` - AXI bus operations
*   `spi_read(addr, size)` / `spi_write(addr, data)` - SPI interface operations

### Chip Information and Control

*   `arch()` - Return architecture string ("Grayskull", "Wormhole", "Blackhole")
*   `get_telemetry()` - Retrieve current telemetry data
*   `arc_msg()` - Send ARC processor messages
*   `get_harvest_bits()` - Get harvesting information for faulty core detection
