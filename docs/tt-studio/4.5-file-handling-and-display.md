This document covers TT-Studio's file handling and display system, including file upload components, file preview functionality, and display interfaces for various file types. The system provides comprehensive file management capabilities across different model interfaces including chat UI, object detection, and speech-to-text, as well as a dedicated RAG (Retrieval-Augmented Generation) management interface for document ingestion.

For information about navigation and layout components that integrate with file handling, see [Navigation and Layout](https://deepwiki.com/tenstorrent/tt-studio/4.1-navigation-and-layout). For model-specific file handling workflows, see [Model-Specific Interfaces](https://deepwiki.com/tenstorrent/tt-studio/4.4-model-specific-interfaces).

## File Upload Components

TT-Studio provides specialized file upload components with drag-and-drop functionality and animated user interfaces.

### FileUpload Component

The primary file upload interface is implemented in the `FileUpload` component, which provides a comprehensive drag-and-drop experience with visual feedback and file information display.

The `FileUpload` component supports multiple file selection and displays detailed information about uploaded files including file size, MIME type, and modification date. It uses `react-dropzone` for drag-and-drop functionality and `framer-motion` for smooth animations.

**Sources:**[app/frontend/src/components/ui/file-upload.tsx 30-171](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/file-upload.tsx#L30-L171)

## File Display System

The file display system handles the presentation of uploaded files with type-specific icons, preview capabilities, and responsive layout design.

### FileDisplay Component Architecture

The `FileDisplay` component categorizes files into specific types and renders appropriate icons and colors for each category. It supports both desktop and mobile layouts with responsive design patterns.

**Sources:**[app/frontend/src/components/chatui/FileDisplay.tsx 188-200](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/FileDisplay.tsx#L188-L200)[app/frontend/src/components/chatui/FileDisplay.tsx 40-151](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/FileDisplay.tsx#L40-L151)

### File Type Classification

The system classifies files into several categories with distinct visual representations:

| Category | Extensions | Icon Color | Badge Color |
| --- | --- | --- | --- |
| Documents | pdf, doc, docx, xls, xlsx, csv | Various (red, blue, green) | Corresponding background |
| Code Files | js, jsx, ts, tsx, py, java, cpp, c, h, rs | Language-specific colors | Yellow, blue, orange, pink |
| Data Files | json, xml, yaml, yml | Yellow, blue, green | Data format colors |
| Archives | zip, rar, 7z, tar, gz | Gray | Gray background |
| Media | mp4, mov, avi, mp3, wav, ogg | Purple, green | Media type colors |
| Text Files | txt, md, log | Gray | Gray background |

**Sources:**[app/frontend/src/components/chatui/FileDisplay.tsx 40-186](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/FileDisplay.tsx#L40-L186)

## RAG Management Interface

The `RagManagement` component provides a dedicated administrative interface for document ingestion and collection management.

### Document Ingestion Flow

The interface allows users to:

1.   **Create Collections:** Using `RagDataSourceForm`, which validates names for length and whitespace [app/frontend/src/components/rag/RagDataSourceForm.tsx 16-25](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/rag/RagDataSourceForm.tsx#L16-L25)
2.   **Upload Documents:** Files are uploaded to specific collections via the `uploadDocument` API [app/frontend/src/components/rag/RagManagement.tsx 17](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/rag/RagManagement.tsx#L17-L17)
3.   **Monitor Status:** The UI tracks which collections are currently uploading files using the `collectionsUploading` state [app/frontend/src/components/rag/RagManagement.tsx 134](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/rag/RagManagement.tsx#L134-L134)
4.   **Inspect Metadata:** Users can expand rows to see detailed `DocumentInfo` including `chunks_count` and `upload_date`[app/frontend/src/components/rag/RagManagement.tsx 107-115](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/rag/RagManagement.tsx#L107-L115)

**Sources:**[app/frontend/src/components/rag/RagManagement.tsx 125-234](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/rag/RagManagement.tsx#L125-L234)[app/frontend/src/components/rag/RagDataSourceForm.tsx 11-50](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/rag/RagDataSourceForm.tsx#L11-L50)

## Image Preview System

The image preview system provides hover-based preview functionality for image URLs and uploaded image files.

### ImagePreview Component

The `ImagePreview` component detects image URLs using pattern matching for common image file extensions (`jpg`, `jpeg`, `png`, `gif`, `webp`) and provides hover-based preview functionality [app/frontend/src/components/chatui/ImagePreview.tsx 15](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/ImagePreview.tsx#L15-L15)

## Responsive Design and Mobile Support

The file display system provides specialized mobile interfaces with touch-friendly interactions and space-efficient layouts. Mobile layouts use compact file icons with abbreviated file type badges, while desktop layouts provide expanded displays with full file names and hover tooltips.

**Sources:**[app/frontend/src/components/chatui/FileDisplay.tsx 274-382](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/chatui/FileDisplay.tsx#L274-L382)

## Integration Points

File handling components integrate with various parts of the TT-Studio interface:

*   **Chat UI:** Uses `FileDisplay` and `fileUtils` to handle attachments in the conversation thread.
*   **Navigation:** The `NavBar` provides access to specialized model interfaces (e.g., Image, Audio) that utilize these file components [app/frontend/src/components/NavBar.tsx 11-17](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/NavBar.tsx#L11-L17)
