## SPI Register Interface

The SPI (Serial Peripheral Interface) flash provides persistent storage for chip configuration parameters. The `TopoBackend` class implements the primary interface for reading and writing these parameters.

### Configuration State Management

The system maintains chip configuration state through the `get_eth_config_state()` method, which reads critical parameters from SPI flash:

| Parameter | Address | Purpose |
| --- | --- | --- |
| `ETH_FW_VERSION_ADDR` | 0x23210 | Firmware version verification |
| `ETH_PARAM_CHIP_COORD` | 0x21100 | X/Y coordinate storage |
| `ETH_PARAM_PORT_DISABLE` | 0x21108 | Ethernet port enable/disable |
| `ETH_PARAM_RACK_SHELF` | 0x211D4 | Rack and shelf identification |

Sources: [tt_topology/backend.py 142-216](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/backend.py#L142-L216)
