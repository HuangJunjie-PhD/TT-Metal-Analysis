# Firmware and tt-flash

## Purpose and Scope

This document describes Tenstorrent's firmware update system and the tt-flash utility, which is responsible for updating firmware on Tenstorrent hardware. Firmware is essential low-level software that runs directly on Tenstorrent devices, providing the foundation for hardware functionality and communication with higher-level software. The tt-flash utility is the official tool for managing firmware versions on your devices.

For information about the Kernel Mode Driver (KMD), see [Kernel-Mode Driver (KMD)](https://deepwiki.com/tenstorrent/tt-installer/5.1-kernel-mode-driver-(kmd)). For details about using tt-smi to monitor your devices, see [System Management Interface (tt-smi)](https://deepwiki.com/tenstorrent/tt-installer/5.4-system-management-interface-(tt-smi)).

## Firmware Overview

Firmware is essential software that runs directly on Tenstorrent hardware devices. It provides low-level functionality including:

1.   Hardware initialization and configuration
2.   Communication protocol implementation
3.   Hardware resource management
4.   Performance optimization features
5.   Basic hardware diagnostics

Tenstorrent firmware is packaged in `.fwbundle` files, which follow the naming convention `fw_pack-<VERSION>.fwbundle`.

Sources: [install.sh 28-34](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L28-L34)[install.sh 118](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L118-L118)

## The tt-flash Utility

The tt-flash utility is a Python-based tool that downloads and manages firmware on Tenstorrent hardware. It provides a straightforward interface for updating firmware to ensure optimal hardware performance and compatibility with the latest software stack.

### Installation

The tt-installer script installs tt-flash automatically during the installation process, unless firmware updates are explicitly skipped. You can also install tt-flash manually using pip:

`pip install git+https://github.com/tenstorrent/tt-flash.git`
Sources: [install.sh 562](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L562-L562)

### Firmware Update Process

During a firmware update:

1.   tt-flash downloads the specified firmware bundle from the tt-firmware repository
2.   The firmware bundle is verified for integrity
3.   tt-flash attempts to update the firmware on all detected Tenstorrent devices
4.   If the initial update fails, tt-flash can perform a force update

Sources: [install.sh 564-570](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L564-L570)[README.md 18-19](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/README.md?plain=1#L18-L19)

## Integration with tt-installer

The tt-installer script automatically handles firmware updates as part of its installation process. This integration ensures that your hardware is running the most appropriate firmware version for your installed software stack.

The installer:

1.   Installs tt-flash via pip from GitHub
2.   Downloads the firmware bundle from the tt-firmware repository
3.   Uses tt-flash to update the firmware on all detected devices
4.   Falls back to force update if the initial update fails

Sources: [install.sh 555-571](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L555-L571)[README.md 22-25](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/README.md?plain=1#L22-L25)

## Usage

### Basic Usage

To update firmware on all detected Tenstorrent devices:

`tt-flash --fw-tar <firmware_bundle_file>`
Where `<firmware_bundle_file>` is the path to a `.fwbundle` file.

### Force Update

If a normal update fails, you can force the update with:

`tt-flash --fw-tar <firmware_bundle_file> --force`
This is useful when the current firmware state prevents a standard update.

Sources: [install.sh 567-570](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L567-L570)

## Architecture

The tt-flash utility:

1.   Communicates with Tenstorrent hardware through the Kernel Mode Driver (KMD)
2.   Reads firmware bundles from the filesystem
3.   Writes firmware directly to the hardware
4.   Works alongside other tools like tt-smi which can display current firmware versions

Sources: [install.sh 22-34](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L22-L34)[install.sh 562-570](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L562-L570)

## Firmware Versioning

Firmware versions are managed through the tt-firmware repository. The tt-installer script automatically selects the latest available firmware version unless specified otherwise.

The latest firmware version is determined by:

1.   Checking the latest tags in the tt-firmware repository
2.   Using the version specified in environment variables (if provided)

Sources: [install.sh 28-34](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L28-L34)[install.sh 72-74](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L72-L74)

## Environment Variables

When using tt-installer, you can control firmware-related behavior with environment variables:

| Variable | Purpose | Default |
| --- | --- | --- |
| TT_FW_VERSION | Specify firmware version to install | Latest available version |
| TT_SKIP_UPDATE_FIRMWARE | Skip firmware update (set to 0) | 1 (perform update) |

Sources: [install.sh 61-62](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L61-L62)[install.sh 72-74](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L72-L74)

## Common Issues and Troubleshooting

### Failed Firmware Updates

If a firmware update fails, tt-flash will automatically attempt a force update. This is handled by the installer script:

`if ! tt-flash --fw-tar "${FW_FILE}"; then    warn "Initial firmware update failed, attempting force update"    tt-flash --fw-tar "${FW_FILE}" --forcefi`
Common causes for update failures include:

*   Current firmware is in an inconsistent state
*   Hardware is in use by another process
*   Insufficient permissions to access the hardware

Sources: [install.sh 567-570](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L567-L570)

### Hardware Detection Issues

If tt-flash cannot detect your hardware:

1.   Ensure the Kernel Mode Driver is properly installed and loaded
2.   Check that the user has permissions to access the device
3.   Verify the hardware is properly seated and powered

## Conclusion

The tt-flash utility provides a simple yet powerful interface for managing firmware on Tenstorrent hardware. Used correctly, it ensures your hardware is running the optimal firmware version for your software stack, maximizing compatibility and performance.
