## Purpose and Scope

The projection comparison system provides tools for analyzing differences between simulation runs. It enables users to compare the performance characteristics, configurations, and detailed statistics of two projection runs to identify changes, regressions, or improvements. This system is essential for validation, debugging, and performance analysis workflows.

For information about generating the projection runs that are compared, see [Project Orchestration](https://deepwiki.com/tenstorrent/polaris/2.4-project-orchestration). For details about the performance metrics that are compared, see [Performance Metrics Extraction](https://deepwiki.com/tenstorrent/polaris/3.1-performance-metrics-extraction).

### Comparison Status Types

The system uses a comprehensive status classification system to categorize differences:

| Status | Description | Use Case |
| --- | --- | --- |
| `Match` | Exact match between values | Identical results |
| `ApproxMatch` | Values within epsilon tolerance | Acceptable differences |
| `Mismatch` | Values differ beyond tolerance | Significant changes |
| `TypeMismatch` | Different data types | Structural changes |
| `Only_in_1` | Key exists only in first run | Removed features |
| `Only_in_2` | Key exists only in second run | Added features |

### Configuration Comparison

The configuration comparison analyzes differences in the configuration files used for each projection run:

`def compare_config_dir(self) -> dict[str, Any]:    configs_common = self.run1.confignames & self.run2.confignames    configs_only_in_1 = self.run1.confignames - self.run2.confignames    configs_only_in_2 = self.run2.confignames - self.run1.confignames`
This comparison:

*   Identifies configurations present in only one run
*   Compares common configuration files using `compare_dicts()`
*   Flattens nested configuration structures for detailed analysis
*   Generates rollup status for overall configuration comparison

Sources: [tools/compare_projections.py 283-298](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/tools/compare_projections.py#L283-L298)

### Return Code Interpretation

The tool returns different exit codes based on comparison results:

*   `0`: All comparisons match (within epsilon tolerance)
*   `1`: Differences found between the runs

This integration ensures that the comparison tool is validated against known scenarios and maintains compatibility with the simulation pipeline.
