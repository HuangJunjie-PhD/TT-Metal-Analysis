# PMU Overflow Tests (SSCOFPMF)

Relevant source files
*   [coretp/plans/sscofpmf/__init__.py](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/__init__.py)

## Purpose and Scope

This document describes the test scenarios for the **Sscofpmf** (Supervisor-mode Counter Overflow and Privilege Mode Filtering) extension. The test plan validates performance monitoring unit (PMU) overflow behavior, interrupt generation, shadow copy mechanisms, and WARL (Write-Any Read-Legal) properties of counter-related CSRs.

For general counter enable and access control testing, see [Counter Enable Tests](https://deepwiki.com/tenstorrent/riscv-coretp/4.3.2-counter-enable-tests). For timer counter testing, see [SSTC Timer Counter Tests](https://deepwiki.com/tenstorrent/riscv-coretp/4.3.1-sstc-timer-counter-tests). For details on the TestStep primitives used to compose these scenarios, see [Test Steps Reference](https://deepwiki.com/tenstorrent/riscv-coretp/5-test-steps-reference).

**Sources:**[coretp/plans/sscofpmf/__init__.py 1-12](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/__init__.py#L1-L12)[coretp/plans/sscofpmf/sscofpmf_scenarios.py 1-13](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L1-L13)

## Extension Overview

The Sscofpmf extension provides mechanisms for:

*   **Counter Overflow Detection**: Hardware sets the overflow (OF) bit in `mhpmeventX` CSRs when performance counters overflow
*   **LCOFIP Interrupt**: Local Counter Overflow Interrupt Pending bit (bit 13 in `mip`/`sip`) signals overflow events
*   **Shadow Copies**: The `scountovf` CSR provides read-only shadow copies of overflow bits filtered by counter enable CSRs
*   **Counter Inhibition**: The `mcountinhibit` CSR controls whether counters increment
*   **WARL Behavior**: Validation of Write-Any Read-Legal semantics for reserved and special bits

The test plan is registered with tags `["sscofpmf", "counters", "warl"]` and executes primarily in M-mode and S-mode privilege levels.

**Sources:**[coretp/plans/sscofpmf/__init__.py 6-10](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/__init__.py#L6-L10)

## WARL Behavior Tests

### Test 2a: mcountinhibit WARL Mask

**Function:**`SID_SSCOFPMF_02A_MCOUNTINHIBIT_WARL`

Validates that `mcountinhibit` bit[1] (time counter inhibit) is hardwired to zero and cannot be set, reflecting the WARL mask `0xFFFFFFFD`.

| Step | Operation | Purpose |
| --- | --- | --- |
| 1 | `CsrWrite(mcountinhibit, 0xFFFFFFFFFFFFFFFF)` | Write all ones |
| 2 | `CsrRead(mcountinhibit)` | Read back value |
| 3 | `Arithmetic(op="and", bit[1] mask)` | Extract bit[1] |
| 4 | `AssertEqual(bit[1], 0)` | Verify bit[1] is zero |

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 15-44](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L15-L44)

### Test 2b: mhpmevent3 Reserved Bits

**Function:**`SID_SSCOFPMF_02B_MHPMEVENT3_RESERVED_BITS`

Ensures that `mhpmevent3` bits[57:56] are WPRI (Write Preserve, Read Ignore) and remain zero after writing all ones.

| Step | Operation | Purpose |
| --- | --- | --- |
| 1 | `CsrWrite(mhpmevent3, 0xFFFFFFFFFFFFFFFF)` | Write all ones |
| 2 | `CsrRead(mhpmevent3)` | Read back value |
| 3 | `Arithmetic(op="and", mask=0x0300000000000000)` | Extract bits[57:56] |
| 4 | `AssertEqual(reserved_bits, 0)` | Verify reserved bits clear |

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 47-76](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L47-L76)

### Test 2c: mhpmcounter3 Write Persistence

**Function:**`SID_SSCOFPMF_02C_MHPMCOUNTER3_WRITE_STICKS`

Validates that `mhpmcounter3` is implemented as WARL and preserves written patterns (assumes counter is implemented).

**Test Pattern:**`0x123456789ABCDEF0`

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 79-101](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L79-L101)

## Counter Enable Tests

### Test 3a: mcounteren.CY Does Not Gate M-mode

**Function:**`SID_SSCOFPMF_03A_MCOUNTEREN_MCYCLE`

Verifies that clearing `mcounteren.CY` (bit[0]) does not inhibit `mcycle` counting in M-mode. The `mcounteren` CSR only controls access from lower privilege modes.

The test uses `NUM_FILL = 32` arithmetic operations to ensure measurable counter increments.

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 11-134](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L11-L134)

### Test 3b: mcounteren.IR Does Not Gate M-mode

**Function:**`SID_SSCOFPMF_03B_MCOUNTEREN_MINSTRET`

Similar to Test 3a, but verifies that clearing `mcounteren.IR` (bit[2]) does not inhibit `minstret` counting in M-mode.

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 137-167](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L137-L167)

## Counter Inhibit Tests

### Test 4a: mcountinhibit Stops mcycle

**Function:**`SID_SSCOFPMF_04A_MCOUNTINHIBIT_MCYCLE_STOPS`

Validates that setting `mcountinhibit.CY` (bit[0]) stops `mcycle` from incrementing.

The test includes a settling period (`NUM_SETTLE = 8`) for out-of-order processors to drain in-flight instructions before the main test sequence.

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 11-203](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L11-L203)

### Test 4b: mcountinhibit Resumes mcycle

**Function:**`SID_SSCOFPMF_04B_MCOUNTINHIBIT_MCYCLE_RESUMES`

Verifies that clearing `mcountinhibit.CY` resumes `mcycle` counting after inhibition.

**Test Sequence:**

1.   Set `mcountinhibit.CY` (inhibit)
2.   Clear `mcountinhibit.CY` (resume)
3.   Read `mcycle` before filler
4.   Execute 32 arithmetic operations
5.   Read `mcycle` after filler
6.   Assert delta ≠ 0

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 206-238](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L206-L238)

### Tests 4c and 4d: minstret Inhibit and Resume

**Functions:**`SID_SSCOFPMF_04C_MCOUNTINHIBIT_MINSTRET_STOPS` and `SID_SSCOFPMF_04D_MCOUNTINHIBIT_MINSTRET_RESUMES`

Parallel tests for `minstret` using `mcountinhibit.IR` (bit[2]).

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 241-309](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L241-L309)

## Overflow Behavior Tests

### Test 9a: mhpmevent3 Writes Do Not Set OF

**Function:**`SID_SSCOFPMF_09A_MHPMEVENT3_WRITES_NO_OVERFLOW`

Validates that software writes to `mhpmevent3` (with all bits set except OF) do not trigger the overflow bit or the LCOFIP interrupt.

| Value Written | OF Bit (bit 63) | LCOFIP Expected |
| --- | --- | --- |
| `0xFFFFFFFFFFFFFFFF ^ (1<<63)` | Must be 0 | Must be 0 |
| `0x0` | Must be 0 | Must be 0 |

The test clears `mip.LCOFIP` before writing, then verifies both `mhpmevent3.OF` and `mip.LCOFIP` remain clear after each write.

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 312-380](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L312-L380)

### Test 9b: mhpmcounter3 Writes Do Not Set OF

**Function:**`SID_SSCOFPMF_09B_MHPMCOUNTER3_WRITES_NO_OVERFLOW`

Verifies that writing boundary values to `mhpmcounter3` does not set the overflow bit or LCOFIP interrupt.

**Test Sequence:**

1.   Write all ones (`2**64 - 1`) to `mhpmcounter3`
2.   Check counter is implemented (`AssertNotEqual(read, 0)`)
3.   Verify `mhpmevent3.OF` is clear
4.   Verify `mip.LCOFIP` is clear
5.   Write zero to `mhpmcounter3`
6.   Verify `mhpmevent3.OF` remains clear
7.   Verify `mip.LCOFIP` remains clear

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 383-457](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L383-L457)

### Test 9c: Reverse Order Writes

**Function:**`SID_SSCOFPMF_09C_MHPMCOUNTER3_REVERSE_WRITES_NO_OVERFLOW`

Tests the reverse sequence (zero then all ones) to ensure overflow is only triggered by hardware counter increment, not software writes.

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 460-542](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L460-L542)

## Interrupt Management Tests

### Test 12a: mip.LCOFIP Isolation

**Function:**`SID_SSCOFPMF_12A_MIP_LCOFIP_READ_WRITE`

Validates that the LCOFIP bit (bit 13 in `mip`) can be independently cleared and set without affecting other interrupt pending bits.

**Bit Masks:**

*   `lcofip_mask = 1 << 13` (LCOFIP bit)
*   `all_bits_mask = (2**64 - 1) ^ (1 << 13)` (all other bits)

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 545-604](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L545-L604)

### Test 12b: sip.LCOFIP Delegation

**Function:**`SID_SSCOFPMF_12B_SIP_LCOFIP_READ_WRITE`

Tests LCOFIP isolation in `sip`. Since `sip` writes from M-mode only affect delegated interrupts, the test first sets `mideleg[13]` to enable `sip[13]` writes.

**Privilege Modes:**`[PrivilegeMode.M, PrivilegeMode.S]`

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 607-674](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L607-L674)

## Shadow Copy Tests

### Test 8a: scountovf Shadow Copy When Enabled

**Function:**`SID_SSCOFPMF_08A_SCOUNTOVF_SHADOW_COPY_ENABLED`

Validates that `scountovf[x]` reflects `mhpmeventx.OF` when the corresponding counter is enabled in `mcounteren` or `scounteren`.

**Test Flow:**

| Step | Action | scountovf[3] Expected | mhpmevent3.OF |
| --- | --- | --- | --- |
| 1 | Enable counter 3 in mcounteren/scounteren | - | - |
| 2 | Clear `mhpmevent3` | 0 | 0 |
| 3 | Set `mhpmevent3.OF` (bit 63) | 1 | 1 |
| 4 | Verify shadow reflection | Assert bit[3] = 1 | 1 |

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 677-746](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L677-L746)

### Test 8b: scountovf Reads Zero When Disabled

**Function:**`SID_SSCOFPMF_08B_SCOUNTOVF_READ_ONLY_ZERO_DISABLED`

Ensures that `scountovf[x]` reads zero when `mcounteren[x]` and `scounteren[x]` are disabled, even if `mhpmeventx.OF` is set.

**Test Configuration:**

*   `mcounteren[3] = 0` (disabled)
*   `scounteren[3] = 0` (disabled)
*   `mhpmevent3.OF = 1` (overflow bit set)

**Expected Result:**`scountovf[3] = 0` (filtered out)

**Privilege Mode:**`PrivilegeMode.S` (supervisor mode can read `scountovf`)

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 749-804](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L749-L804)

## Test Step Composition Pattern

All scenarios follow a common composition pattern using TestStep primitives:

**Common Patterns:**

1.   **Setup Phase:** Clear relevant CSRs and configure initial state
2.   **Filler Phase:** Execute `NUM_FILL` (32) or `NUM_SETTLE` (8) arithmetic operations
3.   **Verification Phase:** Read CSRs, compute masks/deltas, assert conditions
4.   **Cleanup Phase:** Restore CSRs to default state

**Sources:**[coretp/plans/sscofpmf/sscofpmf_scenarios.py 1-804](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/sscofpmf/sscofpmf_scenarios.py#L1-L804)[coretp/step/__init__.py](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/step/__init__.py)

## Summary

The Sscofpmf test plan provides comprehensive validation of performance monitoring overflow mechanisms through 14 test scenarios covering:

*   **WARL Compliance:** Reserved bits and hardwired values (3 scenarios)
*   **Counter Control:** Enable and inhibit mechanisms (6 scenarios)
*   **Overflow Detection:** Software write isolation (3 scenarios)
*   **Interrupt Handling:** LCOFIP generation and isolation (2 scenarios)
*   **Shadow Registers:** Filtered overflow visibility (2 scenarios)

All tests execute in M-mode or combined M/S-mode environments and utilize standard TestStep primitives for CSR manipulation, arithmetic computation, and assertion-based verification.
