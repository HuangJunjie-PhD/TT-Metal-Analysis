| Argument | Description |
| --- | --- |

The CSV file is the only required argument for the tool. This file contains the performance trace data generated by the Tracy profiler during the execution of a TT-Metal operation.

## Filtering Options

The following options allow filtering the trace data to focus analysis on specific operations or sections of the trace:

### Signpost Filtering

| Option | Description | Default |
| --- | --- | --- |
| `--signpost NAME` | Analyze operations after the specified signpost | Last signpost in file |
| `--ignore-signposts` | Ignore all signposts and analyze the entire trace | False |

Tracy signposts mark specific sections of code for analysis. By default, the tool analyzes operations after the last signpost in the trace, which typically represents the performance-critical section after initialization and warm-up.

Sources: [src/tt_perf_report/perf_report.py 910-912](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L910-L912)[src/tt_perf_report/perf_report.py 97-117](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L97-L117)

### Operation Filtering

| Option | Description | Default |
| --- | --- | --- |
| `--id-range RANGE` | Show only operations with IDs in the specified range (e.g., '5-10', '31-', or '-12') | All operations |
| `--min-percentage VALUE` | Hide operations that take less than the specified percentage of total time | 0.5 |
| `--no-host-ops` | Do not include host operations in the output | False |

The `--id-range` option allows focusing on specific sections of the trace, using the ID column values. This is particularly useful for isolating specific model components or phases of execution.

## Output Formatting Options

These options control the format and content of the performance report output:

| Option | Description | Default |
| --- | --- | --- |
| `--csv FILENAME` | Output results to a CSV file instead of terminal | Terminal output |
| `--no-advice` | Show only the performance table, skip optimization advice | Show advice |
| `--raw-op-codes` | Include raw operation codes in the output | False |

The CSV output option is particularly useful for automated analysis or for importing the results into spreadsheets or other tools for further processing.

Sources: [src/tt_perf_report/perf_report.py 923-926](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L923-L926)

## Visualization Options

These options control the visual formatting of the terminal output:

| Option | Description | Default |
| --- | --- | --- |
| `--color` | Force colored output even when output is redirected | Auto-detect |
| `--no-color` | Force output without color | Auto-detect |

By default, the tool automatically detects whether the terminal supports colors based on whether stdout is a TTY. These options override the auto-detection.

Sources: [src/tt_perf_report/perf_report.py 920-921](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L920-L921)[src/tt_perf_report/perf_report.py 17-48](https://github.com/tenstorrent/tt-perf-report/blob/076a1b39/src/tt_perf_report/perf_report.py#L17-L48)

## Advanced Options

These options provide additional control for advanced use cases:

| Option | Description | Default |
| --- | --- | --- |
| `--tracing-mode` | Do not sort operations by timestamp when in tracing mode | Sort by timestamp |
| `--raw-op-codes` | Include raw operation codes in output | False |

The tracing mode option is useful when analyzing traces where the timestamps may not accurately reflect the execution order of operations.

## Option Interactions

Some options have specific interactions or dependencies:

1.   `--signpost` and `--ignore-signposts` are mutually exclusive. If both are provided, `--ignore-signposts` takes precedence.

2.   When using `--id-range` with `--no-host-ops`, host operations are filtered out after applying the ID range filter.

3.   `--min-percentage` affects visual display by graying out less significant operations but doesn't remove them from the output unless they're filtered by other options.

4.   `--color` and `--no-color` are mutually exclusive. If both are provided, `--no-color` takes precedence.
