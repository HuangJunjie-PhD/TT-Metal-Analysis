## Test Categories and Organization

The project categorizes tests into several suites, each mapped to specific CMake targets and runners. This separation ensures that hardware-independent tests can run in parallel while hardware-dependent tests are managed to avoid resource contention.

## Writing Tests

### Python Lit Tests

Lit tests verify DSL functionality by checking generated MLIR and C++ output. A typical test uses `FileCheck` to validate the IR at different stages of the pipeline [[test/TESTING.md:5-9]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).

**Key Requirements:**

*   **Single Operation**: Each test file should contain only one `@ttl.operation` decorated function to avoid overwriting temporary files [[test/TESTING.md:40-42]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).
*   **Compile-Only Mode**: Most lit tests use `TTLANG_COMPILE_ONLY=1` to run without hardware [[test/TESTING.md:10-12]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).
*   **Feature Gating**: Tests requiring hardware use `# REQUIRES: tt-device`[[test/TESTING.md:49-51]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).

### Pytest Parametric Tests

Files named `test_*.py` are collected by `pytest` instead of `lit`[[test/TESTING.md:29-31]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). These are used for end-to-end validation across different tensor shapes, data types, and grid sizes.

## Environment Variables

The behavior of the test suite and the compilation pipeline is controlled via several environment variables.

| Variable | Role |
| --- | --- |
| `TTLANG_COMPILE_ONLY` | If `1`, skips hardware execution; only performs MLIR lowering and EmitC [[test/lit.cfg.py:140]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). |
| `TT_METAL_SIMULATOR` | Signals the hardware abstraction to use the TT-Metalium simulator [[test/lit.cfg.py:134]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). |
| `TTLANG_INITIAL_MLIR` | Path to save the MLIR immediately after Python AST conversion [[test/TESTING.md:36]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). |
| `TTLANG_FINAL_MLIR` | Path to save the MLIR after all transformation passes [[test/lit.cfg.py:107]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). |
| `TTLANG_SIM_ONLY` | Sets up the environment for simulation only, skipping compiler builds [[env/activate.in:32-34]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). |
| `TTLANG_TEST_SEED` | Sets the random seed for parametric tests (default: 42) [[.github/workflows/call-test-hardware.yml:63]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). |

## CI/CD and Hardware Integration

`tt-lang` utilizes GitHub Actions to ensure code quality across different environments, including simulation and physical hardware.

### Hardware Test Workflow

Hardware tests are orchestrated via `call-test-hardware.yml`, which targets specific runners like `n150`[[.github/workflows/call-test-hardware.yml:117]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). The environment is set up with specialized Docker options to access Tenstorrent devices:

*   **Device Mapping**: `--device /dev/tenstorrent`[[.github/workflows/call-test-hardware.yml:49]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).
*   **Hugepages**: Mapping `/dev/hugepages` for DMA transfers [[.github/workflows/call-test-hardware.yml:51-52]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).

### Simulation Workflow

The simulation suite (`test-sim`) runs on standard GitHub-hosted runners (`ubuntu-latest`, `macos-latest`) because it does not require specialized hardware [[.github/workflows/call-test-sim.yml:12-14]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests). It uses `pytest-xdist` (`-n auto`) for parallel execution [[.github/workflows/call-test-sim.yml:41]](https://deepwiki.com/tenstorrent/tt-lang/8.5-writing-and-running-tests).
