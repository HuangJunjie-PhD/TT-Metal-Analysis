This section covers the automated systems and developer configurations used to maintain code quality, license compliance, and consistent development environments across the `tt-inference-server` codebase. These tools ensure that every contribution adheres to Tenstorrent's legal requirements and technical standards.

## License Compliance (SPDX)

The project uses SPDX (Software Package Data Exchange) headers to manage license compliance. This is enforced through a combination of a GitHub Action and a Python utility script.

### Header Enforcement Mechanism

The `scripts/add_spdx_header.py` script is the primary utility for ensuring license headers are present in source files [scripts/add_spdx_header.py 20-26](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/add_spdx_header.py#L20-L26) It targets specific file extensions (`.py`, `.sh`) and `Dockerfile` instances across key directories including `vllm-tt-metal`, `tt-media-server`, and `workflows`[scripts/add_spdx_header.py 31-48](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/add_spdx_header.py#L31-L48)

The header follows the Apache-2.0 license format:

*   **Identifier:**`SPDX-License-Identifier: Apache-2.0`[scripts/add_spdx_header.py 13](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/add_spdx_header.py#L13-L13)
*   **Copyright:**`SPDX-FileCopyrightText: © [Year] Tenstorrent USA, Inc.`[scripts/add_spdx_header.py 15-17](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/add_spdx_header.py#L15-L17)

### Automated Workflow

The `SPDX Header Checker` workflow (`spdx-checker.yml`) runs on pull requests. If missing headers are detected, the workflow automatically runs the injection script and commits the changes back to the branch using an automated bot.

## Developer Tooling and Linting

The repository leverages `ruff` for Python linting/formatting and standardizes C++ code consistency for high-performance components.

### Python Linting (Ruff)

Ruff is configured via `pyproject.toml` with a target version of Python 3.8 [pyproject.toml 1-5](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/pyproject.toml#L1-L5)

*   **Configuration:** The project ignores specific import order rules (E402) to accommodate complex dependency paths often found in machine learning scripts [pyproject.toml 7-8](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/pyproject.toml#L7-L8)
*   **Version:** The project pins `ruff==0.15.0` in the development requirements to ensure consistency across CI and local environments [requirements-dev.txt 2](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/requirements-dev.txt#L2-L2)

### Pre-commit Hooks

The project includes a custom pre-commit hook script located at `scripts/hooks/pre-commit`[scripts/hooks/pre-commit 1-107](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/hooks/pre-commit#L1-L107) It performs the following sequence:

1.   **Tool Discovery:** The `find_tool_in_env_bins` function searches for `ruff` and `python` in various virtual environments (`.pre-commit`, `.venv`, or system paths) [scripts/hooks/pre-commit 9-19](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/hooks/pre-commit#L9-L19)
2.   **Schema Validation:** Validates `models-ci-config.json` against its JSON schema using `jsonschema` to prevent CI failures due to malformed configurations [scripts/hooks/pre-commit 46-66](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/hooks/pre-commit#L46-L66)
3.   **Linting & Formatting:** Runs `ruff format` and `ruff check --fix` on staged `.py` files to automatically resolve style issues [scripts/hooks/pre-commit 76-78](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/hooks/pre-commit#L76-L78)
4.   **Unit Testing:** Executes `pytest -p no:warnings` to ensure no regressions are introduced before the commit is finalized [scripts/hooks/pre-commit 98-103](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/hooks/pre-commit#L98-L103)

## Environment and Test Configuration

### Pytest Infrastructure

The project uses `pytest` for its testing framework, with configurations defined in `pyproject.toml`[pyproject.toml 10-23](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/pyproject.toml#L10-L23)

*   **Discovery:** Tests are discovered in the `tests` directory [pyproject.toml 14](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/pyproject.toml#L14-L14)
*   **Markers:** Extensive markers are defined to allow developers to target specific hardware or model types (e.g., `@pytest.mark.n150`, `@pytest.mark.llm`) [pyproject.toml 26-64](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/pyproject.toml#L26-L64)
*   **Coverage:** Code coverage instrumentation omits specific configuration files like `settings.py` to maintain focus on logic-heavy modules [pyproject.toml 66-78](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/pyproject.toml#L66-L78)

### Project Dependencies and Setup

Developers can set up their environment using `uv` or standard `venv`[docs/development.md 96-115](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/docs/development.md?plain=1#L96-L115)

*   **Development Requirements:**`requirements-dev.txt` defines the standard toolset: `pre-commit==3.5.0`, `ruff==0.15.0`, and `pytest==8.3.5`[requirements-dev.txt 1-3](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/requirements-dev.txt#L1-L3)
*   **IDE Rules:** The `.cursor/rules/` directory contains AI-assistance rules for `coding-standards.mdc`, `naming-conventions.mdc`, and `project-files.mdc` to guide development within the Cursor IDE.

### Release Automation Tooling

Beyond linting, the `scripts/` directory contains tools for maintaining the `MODEL_SPECS` registry:

*   **`update_model_spec.py`**: This script parses `ModelSpecTemplate` blocks and updates `tt_metal_commit` and `vllm_commit` based on the latest successful CI results [scripts/release/update_model_spec.py 6-29](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/release/update_model_spec.py#L6-L29)
*   **`build_docker_images.py`**: A resource-aware script that calculates `get_max_concurrent_builds` based on available system memory, disk, and CPU to prevent host crashes during parallel Docker builds [scripts/build_docker_images.py 100-155](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/scripts/build_docker_images.py#L100-L155)

### C++ Tooling and Formatting

The C++ backend components in `tt-media-server/cpp_server` are standardized using:

*   **Clang-Format**: Enforced via `.clang-format` to maintain a consistent coding style across all high-performance C++ source files.
*   **Clang-Tidy**: Static analysis configuration provided in `.clang-tidy` for detecting potential bugs and enforcing modern C++ standards.
*   **VS Code/Cursor Integration**: The `.vscode/settings.json` file configures `llvm-vs-code-extensions.vscode-clangd` as the default formatter for C/C++ files [tt-media-server/.vscode/settings.json 11-18](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/.vscode/settings.json#L11-L18)
