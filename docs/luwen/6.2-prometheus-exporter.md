## Purpose and Scope

The Prometheus Exporter in the Luwen repository collects telemetry data from Tenstorrent AI accelerator chips and exposes this information via a Prometheus-compatible HTTP endpoint. This enables integration with Prometheus monitoring systems for tracking chip health, performance metrics, and configuration details.

This document covers the architecture, functionality, and usage of the Prometheus Exporter application. For information about other utilities built on Luwen, see [Ethernet Map Creation](https://deepwiki.com/tenstorrent/luwen/6.1-ethernet-map-creation) and [Performance Testing](https://deepwiki.com/tenstorrent/luwen/6.3-performance-testing).

Sources: [app/prometheus-exporter/src/main.rs 1-18](https://github.com/tenstorrent/luwen/blob/55e35616/app/prometheus-exporter/src/main.rs#L1-L18)

## Overview

The Prometheus Exporter application leverages Luwen's chip abstraction layer to collect telemetry data from detected Tenstorrent chips (both local PCI-connected and remote). It periodically queries telemetry information from all available chips and translates this data into Prometheus metrics that can be scraped by a Prometheus server.

## Components

### Metrics Collection

The exporter collects a comprehensive set of metrics from each detected chip:

1.   **Software Information**:

    *   Firmware versions
    *   Firmware date
    *   Board type

2.   **Performance Metrics**:

    *   Clock speeds (AI, AXI, ARC)
    *   Power consumption
    *   Voltage and current

3.   **Temperature Data**:

    *   ASIC temperature
    *   VREG temperature
    *   Board temperatures (inlet/outlet)

4.   **PCIe Information** (for PCI-connected chips):

    *   PCI bus/device/function
    *   Vendor/device IDs
    *   Link configuration (width and generation)

Each metric is labeled with a board identifier, enabling monitoring of multiple chips simultaneously.

## Implementation Details

### Configuration Options

The exporter supports several command-line arguments for customization:

| Argument | Description | Default |
| --- | --- | --- |
| `--interval`, `-i` | Milliseconds between hardware queries | 1000 |
| `--port`, `-p` | HTTP port for the Prometheus endpoint | 8080 |
| `--no-grayskull`, `-n` | Flag to ignore Grayskull cards | false |

### Metrics Implementation

The exporter creates Prometheus gauge vectors for each metric, with labels to identify the source chip. The `Metrics` structure encapsulates all these gauges and provides methods to update them with fresh telemetry data.

Key implementation patterns:

1.   **Metric Registration**:

    *   Each metric is registered with the Prometheus registry
    *   Metrics are organized with namespace "tt" and subsystem "smi"
    *   All metrics include a "board_id" label

2.   **Telemetry Mapping**:

    *   The `update()` method maps data from Telemetry objects to Prometheus metrics
    *   Different handling for PCIe-connected and remote chips

3.   **Board Identification**:

    *   PCIe-connected chips are identified with their serial number and "_pcie" suffix
    *   Remote chips use a serial number with "_remote" suffix

## Usage

### Starting the Exporter

To run the Prometheus Exporter:

```
$ prometheus-exporter [OPTIONS]
```

Example with custom settings:

```
$ prometheus-exporter --port 9090 --interval 500
```

### Integrating with Prometheus

Add the following to your Prometheus configuration:

Replace 'hostname' with the host running the exporter and '8080' with the configured port.

### Available Metrics

The exporter exposes the following metrics under the "tt_smi" namespace:

| Metric | Description | Type |
| --- | --- | --- |
| tt_smi_sw_info | Software version information | Gauge with version labels |
| tt_smi_aiclk | AI clock speed in MHz | Gauge |
| tt_smi_aixclk | AXI clock speed in MHz | Gauge |
| tt_smi_arcclk | ARC clock speed in MHz | Gauge |
| tt_smi_voltage | Core voltage in V | Gauge |
| tt_smi_asic_temperature | Core temperature in °C | Gauge |
| tt_smi_vreg_temperature | VREG temperature in °C | Gauge |
| tt_smi_board_temperature_* | Board temperatures in °C | Gauge |
| tt_smi_power | Core power in W | Gauge |
| tt_smi_current | Core current in A | Gauge |
| tt_smi_pci_* | PCIe-related information | Gauge |

## Limitations

*   Has not been specifically tested on Galaxy systems
*   Requires firmware that supports telemetry gathering
*   Some metrics may not be available on all chip types
