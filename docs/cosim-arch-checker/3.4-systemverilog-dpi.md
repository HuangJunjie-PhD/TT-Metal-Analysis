## Purpose and Scope

This document describes the SystemVerilog Direct Programming Interface (DPI) implementation used in the cosim-arch-checker framework. The DPI provides a standardized mechanism for bi-directional communication between SystemVerilog testbenches and C/C++ verification code.

The DPI interface serves as the primary bridge for extracting architectural state from RISC-V DUT simulations and feeding it to the co-simulation verification framework. For information about the overall bridge orchestration system, see [Bridge System](https://deepwiki.com/tenstorrent/cosim-arch-checker/3.1-bridge-system). For details about Whisper ISS integration, see [Whisper Integration](https://deepwiki.com/tenstorrent/cosim-arch-checker/3.3-whisper-integration).

## Core DPI Data Types

The DPI interface defines several fundamental data types for representing SystemVerilog values in C/C++:

### Scalar Types

| DPI Type | SystemVerilog Equivalent | Description |
| --- | --- | --- |
| `svBit` | `bit` | 2-state scalar (0, 1) |
| `svLogic` | `logic` | 4-state scalar (0, 1, x, z) |
| `svScalar` | Generic scalar | Base type for bit/logic |

## DPI Value Representation

The DPI uses specific encoding schemes for representing multi-state logic values:

### 4-State Logic Encoding

| SystemVerilog Value | aval | bval | Description |
| --- | --- | --- | --- |
| 0 | 0 | 0 | Logic 0 |
| 1 | 1 | 0 | Logic 1 |
| X | 1 | 1 | Unknown |
| Z | 0 | 1 | High impedance |
