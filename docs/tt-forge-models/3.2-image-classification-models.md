This document covers the image classification models implemented in the tt-forge-models repository. These models follow the standardized ForgeModel interface and support a wide variety of architectures from classic CNNs to modern Vision Transformers. For object detection models, see [Object Detection Models](https://deepwiki.com/tenstorrent/tt-forge-models/3.1-object-detection-models). For multimodal models that combine vision with other modalities, see [Multimodal Models](https://deepwiki.com/tenstorrent/tt-forge-models/5-multimodal-models).

## Architecture Overview

All image classification models in the repository follow a consistent architectural pattern built around the ForgeModel base class. The system provides a unified interface while supporting diverse model architectures and sources.

## Model Sources and Integration

Different model families integrate with various external sources, each requiring specific loading mechanisms.

| Source | Models | Integration Pattern | Key Components |
| --- | --- | --- | --- |
| Hugging Face | ViT, Segformer, RegNet | `transformers` library | `AutoImageProcessor`, `*ForImageClassification` |
| TIMM | MLP-Mixer, GhostNet, HRNet | `timm.create_model()` | `resolve_data_config`, `create_transform` |
| PyTorch Hub | DenseNet, ResNeXt, HardNet | `torch.hub.load()` | Manual transforms |
| TorchVision | EfficientNet | `torchvision.models` | Weights enums, manual transforms |

Sources: [vit/pytorch/loader.py 8-22](https://github.com/tenstorrent/tt-forge-models/blob/0a7d2042/vit/pytorch/loader.py#L8-L22)[mlp_mixer/pytorch/loader.py 8-26](https://github.com/tenstorrent/tt-forge-models/blob/0a7d2042/mlp_mixer/pytorch/loader.py#L8-L26)[densenet/pytorch/loader.py 8-24](https://github.com/tenstorrent/tt-forge-models/blob/0a7d2042/densenet/pytorch/loader.py#L8-L24)[efficientnet/pytorch/loader.py 23-26](https://github.com/tenstorrent/tt-forge-models/blob/0a7d2042/efficientnet/pytorch/loader.py#L23-L26)
