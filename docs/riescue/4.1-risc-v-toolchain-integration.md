## Purpose and Scope

This page documents how RiESCUE integrates with the RISC-V GNU toolchain (`riscv-gnu-toolchain`) for compilation and disassembly of RISC-V assembly and C code into ELF executables. The toolchain integration provides a Python abstraction layer that discovers, configures, and invokes external RISC-V tools.

For detailed documentation of individual tool classes, see:

*   [Compiler Tool](https://deepwiki.com/tenstorrent/riescue/4.1.1-compiler-tool) - Detailed compiler configuration and usage
*   [Disassembler and Objcopy Tools](https://deepwiki.com/tenstorrent/riescue/4.1.2-disassembler-and-objcopy-tools) - Disassembly and object file manipulation
*   [Instruction Set Simulators](https://deepwiki.com/tenstorrent/riescue/4.2-instruction-set-simulators) - Whisper and Spike simulator integration

## RISC-V GNU Toolchain Overview

The RISC-V GNU Toolchain is a collection of GNU-based development tools for RISC-V, including:

| Tool | Executable | Purpose |
| --- | --- | --- |
| GCC Compiler | `riscv64-unknown-elf-gcc` | Compiles assembly and C into ELF executables |
| Objdump | `riscv64-unknown-elf-objdump` | Disassembles ELF files |
| Objcopy | `riscv64-unknown-elf-objcopy` | Manipulates object files |
| Other binutils | Various | Linking, archiving, symbol manipulation |

RiESCUE requires a RISC-V GNU Toolchain installation to compile test cases. The toolchain is typically installed from source (see [riscv-gnu-toolchain GitHub](https://github.com/tenstorrent/riescue/blob/07cefde3/riscv-gnu-toolchain%20GitHub)) or obtained via the RiESCUE container environment.

## Tool Discovery Mechanism

Each `Tool` subclass implements a three-tier discovery mechanism to locate executables:

The search order is:

1.   **Explicit Path**: If a `Path` object is provided to the tool constructor
2.   **Environment Variable**: Checks for tool-specific environment variable (e.g., `RV_GCC`, `RV_OBJDUMP`)
3.   **System PATH**: Uses `shutil.which()` to find the tool in the system PATH

This mechanism is implemented in [riescue/lib/toolchain/tool.py 36-55](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L36-L55):

`def find_executable(self, tool_path: Optional[Path], env_name: str, tool_name: str) -> Path:    """    Searches for executable. Checks for Path tool_path if provided    if no tool_path, checks for env_name environment variable    if no env_name, checks for tool_name in PATH    """`
### Environment Variables

| Tool | Environment Variable | Default Tool Name |
| --- | --- | --- |
| Compiler | `RV_GCC` | `riscv64-unknown-elf-gcc` |
| Disassembler | `RV_OBJDUMP` | `riscv64-unknown-elf-objdump` |
| Objcopy | `RV_OBJCOPY` | `riscv64-unknown-elf-objcopy` |

## The Toolchain Class

The `Toolchain` class is a container that aggregates individual tool instances and provides unified configuration:

`# Typical usage in RiescueDtoolchain = Toolchain()  # Creates with defaults# ortoolchain = Toolchain.from_clargs(cl_args)  # Creates from CLI arguments # Access individual toolscompiler = toolchain.compilerdisassembler = toolchain.disassemblersimulator = toolchain.simulator  # Optional: Whisper or Spike`
The `Toolchain` class provides:

*   Unified construction from command-line arguments via `from_clargs()`
*   Aggregation of `Compiler`, `Disassembler`, `Objcopy`, and optionally `Spike`/`Whisper`
*   Centralized argument registration via `add_arguments()`

## The Tool Base Class

The `Tool` abstract base class provides common functionality for all external tools:

Key methods:

*   **`__init__(path, env_name, tool_name, args)`**: Initializes tool with discovery parameters
*   **`find_executable()`**: Discovers the executable location
*   **`check_filepath()`**: Resolves data file paths relative to package install
*   **`run(output_file, cwd, timeout, args)`**: Executes the tool via subprocess
*   **`_classify(process, output_file)`**: Abstract method for error classification
*   **`_raise_toolchain_error()`**: Raises `ToolchainError` with standardized parameters

**Sources:**[riescue/lib/toolchain/tool.py 18-126](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L18-L126)

## Compiler Integration

### Compiler Class Structure

The `Compiler` class extends `Tool` to provide RISC-V GCC compilation:

### Default Compiler March String

The compiler uses a comprehensive default `-march` string supporting common extensions:

```
rv64imafdcvh_svinval_zfh_zba_zbb_zbc_zbs_zifencei_zicsr_zvkned_zicbom_zicbop_zicboz_zawrs_zihintpause_zvbb1_zicond_zvkg_zvkn_zvbc_zfa_zk
```

This can be overridden via `--compiler_march` argument or by setting `compiler_march` parameter.

**Sources:**[riescue/lib/toolchain/tool.py 128-166](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L128-L166)

### Compilation Workflow in RiescueD

The compilation process in [riescue/riescued.py 269-309](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L269-L309):

1.   **Setup Arguments**: Build base compiler arguments including include paths, linker script, and output file
2.   **Conditional Arguments**: Add `-mbig-endian` if big-endian mode is enabled
3.   **Additional C Files**: Add any C files specified in `featmgr.cfiles`
4.   **Execute Compiler**: Run GCC with all arguments to produce ELF file
5.   **Disassemble**: Run objdump to generate disassembly listing

**Sources:**[riescue/riescued.py 269-309](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L269-L309)

## Disassembler Integration

### Disassembler Class

The `Disassembler` class wraps `riscv64-unknown-elf-objdump`:

`# Constructiondisassembler = Disassembler(    disassembler_path=None,  # Optional path    disassembler_opts=[]     # Additional options) # Usage in RiescueDdisassembler_args = ["-D", str(elf_file), "-M", "numeric"]disassembler.run(    output_file=dis_file,    cwd=run_dir,    args=disassembler_args)`
The disassembler is invoked with:

*   **`-D`**: Disassemble all sections
*   **`-M numeric`**: Use numeric register names (e.g., `x1` instead of `ra`)

The output is written to a `.dis` file containing the full disassembly of the ELF executable.

## Command-Line Configuration

### Toolchain Arguments

The `Toolchain` class registers command-line arguments through `add_arguments()`:

`# Compiler configurationriescued \  --testfile test.s \  --compiler_path /opt/riscv/bin/riscv64-unknown-elf-gcc \  --compiler_march rv64imafdc \  --compiler_mabi lp64d \  --compiler_opts -O2 -g \  --test_equates DEFINE1=0x1000 \  --test_equates DEFINE2=0x2000 # Disassembler configurationriescued \  --testfile test.s \  --disassembler_path /opt/riscv/bin/riscv64-unknown-elf-objdump \  --disassembler_opts -S # Objcopy configurationriescued \  --testfile test.s \  --objcopy_path /opt/riscv/bin/riscv64-unknown-elf-objcopy \  --objcopy_opts --output-target=binary`
### Compiler-Specific Arguments

| Argument | Type | Description |
| --- | --- | --- |
| `--compiler_path` | `Path` | Explicit path to GCC executable |
| `--compiler_opts` | `list[str]` | Additional GCC flags (e.g., `-O2`, `-g`) |
| `--compiler_march` | `str` | ISA string for `-march` flag |
| `--compiler_mabi` | `str` | ABI string for `-mabi` flag (e.g., `lp64d`) |
| `--test_equates` | `list[str]` | Preprocessor defines in format `KEY=VALUE` |
| `--config_json` | `Path` | Configuration file for feature management |

### Test Equates

Test equates allow passing preprocessor definitions to the compiler:

`riescued --testfile test.s \  --test_equates MEMORY_BASE=0x80000000 \  --test_equates NUM_ITERATIONS=100`
These are converted to GCC `-D` flags: `-DMEMORY_BASE=0x80000000 -DNUM_ITERATIONS=100`

## Error Handling

The toolchain integration includes comprehensive error handling through the `ToolchainError` exception:

`class ToolchainError(Exception):    """    Raised when a tool execution fails.        Attributes:        tool_name: Name of the tool that failed        cmd: Command that was executed        kind: Type of failure (ToolFailureType enum)        returncode: Process return code        error_text: Error message or stderr output    """`
Error types defined in `ToolFailureType`:

*   **`SEGFAULT`**: Tool segmentation fault (return code 139)
*   **`COMPILE_FAILURE`**: Compilation or assembly error
*   **`NONZERO_EXIT`**: Non-zero return code from tool

**Sources:**[riescue/lib/toolchain/tool.py 100-114](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L100-L114)[riescue/lib/toolchain/tool.py 192-203](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L192-L203)

## Summary

The RISC-V toolchain integration in RiESCUE provides:

1.   **Flexible Tool Discovery**: Three-tier search (explicit path → env var → PATH)
2.   **Unified Configuration**: Single `Toolchain` class aggregating all tools
3.   **Comprehensive Abstraction**: `Tool` base class with common subprocess handling
4.   **Specialized Tool Classes**: `Compiler`, `Disassembler`, `Objcopy` with tool-specific logic
5.   **Robust Error Handling**: Classified errors with detailed diagnostics
6.   **CLI Integration**: Complete command-line argument support
7.   **Container Integration**: Seamless operation in containerized environments

This architecture enables RiESCUE to work with various RISC-V toolchain installations while providing a consistent, Pythonic interface for test generation and compilation.
