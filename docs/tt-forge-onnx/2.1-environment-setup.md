## Purpose and Scope

This page documents the installation process, dependency requirements, and environment configuration for the tt-forge-onnx compiler system. It covers Python package dependencies, system requirements, and the organization of requirement files.

For information about using the compiler after setup, see [Quick Start Guide](https://deepwiki.com/tenstorrent/tt-forge-onnx/2.2-quick-start-guide). For advanced configuration options, see [Compilation Configuration](https://deepwiki.com/tenstorrent/tt-forge-onnx/8.2-compilation-configuration).

## Core Dependencies

The `core_requirements.txt` file defines the foundational dependencies required by all tt-forge-onnx installations. These dependencies are organized into several functional categories:

### Multi-Framework Support

The compiler supports five major deep learning frameworks as inputs (see [Multi-Framework Support](https://deepwiki.com/tenstorrent/tt-forge-onnx/4.2-multi-framework-support) for architectural details):

| Framework | Package | Version | Purpose |
| --- | --- | --- | --- |
| **JAX** | `jax` | 0.7.1 | JAX model compilation (via jax2tf) |
| **JAX Ecosystem** | `flax` | >=0.10.6 | JAX neural network library support |
| **JAX Ecosystem** | `chex` | >=0.1.7 | JAX testing utilities |
| **TensorFlow** | `tensorflow` | 2.19.0 | TensorFlow model frontend |
| **TensorFlow Tools** | `tensorboard` | 2.19.0 | TensorFlow visualization and logging |
| **TensorFlow Tools** | `tf2onnx` | 1.15.1 | TensorFlow to ONNX conversion |
| **ONNX** | `onnx` | >=1.15.0 | ONNX model loading and manipulation |
| **ONNX Runtime** | `onnxruntime` | >=1.16.3 | ONNX model execution and verification |
| **PaddlePaddle** | `paddlepaddle` | 2.6.2 | PaddlePaddle model frontend |
| **PaddlePaddle NLP** | `paddlenlp` | 2.8.1 | PaddlePaddle NLP model support |

Note that PyTorch (`torch`) is not listed in `core_requirements.txt` because it is specified with platform-specific download URLs in the downstream requirement files.

### PyTorch CPU Distribution

The Linux requirements specify CPU-only PyTorch distributions to reduce artifact size in CI/CD pipelines:

```
torch @ https://download.pytorch.org/whl/cpu/torch-2.7.0%2Bcpu-cp312-cp312-manylinux_2_28_x86_64.whl
torchvision @ https://download.pytorch.org/whl/cpu/torchvision-0.22.0%2Bcpu-cp312-cp312-manylinux_2_28_x86_64.whl
```

This reduces Docker image size by several GB, improving CI/CD build and caching performance (see [CI/CD Pipeline](https://deepwiki.com/tenstorrent/tt-forge-onnx/6.4-cicd-pipeline)).

**Sources:**[env/linux_requirements.txt 15-19](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/env/linux_requirements.txt#L15-L19)

## Version Compatibility Notes

### Python 3.12 Compatibility

Several packages were updated to support Python 3.12:

| Package | Old Version | New Version | Reason |
| --- | --- | --- | --- |
| `pandas` | 1.5.3 | >=2.2.0 | Python 3.11 maximum → 3.12 support |
| `pybind11` | 2.6.2 | >=2.13.6 | Python 3.11 maximum → 3.12 support |
| `torch` | N/A | 2.7.0 | Python 3.12 wheel availability |

**Sources:**[env/core_requirements.txt 20-23](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/env/core_requirements.txt#L20-L23)

### JAX 0.7 Migration

The JAX ecosystem was upgraded from 0.3.16 to 0.7.1, requiring compatible dependency versions:

*   `jax==0.7.1` (previously 0.3.16)
*   `flax>=0.10.6` (for `ml_dtypes.float8_e4m3b11` support)
*   `chex>=0.1.7` (JAX 0.7 compatibility)

The `chex` package is explicitly pinned to avoid dependency conflicts between JAX versions and the `optax` transitive dependency.

**Sources:**[env/core_requirements.txt 3-7](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/env/core_requirements.txt#L3-L7)[env/core_requirements.txt 11-13](https://github.com/tenstorrent/tt-forge-onnx/blob/e519819c/env/core_requirements.txt#L11-L13)
