The API layer of the `tt-inference-server` (specifically the Python-based `tt-media-server`) is built on FastAPI and designed to provide OpenAI-compatible interfaces for a diverse range of AI tasks. It employs a dynamic routing mechanism that activates specific endpoints based on the configured model service, ensuring the server only exposes relevant APIs for the loaded hardware and model.

## FastAPI Application Structure

The application entry point is managed through a modular routing system. The `api_router` is the central hub where all service-specific and maintenance routes are registered [tt-media-server/open_ai_api/__init__.py 28](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/__init__.py#L28-L28)

### Service Routing Logic

The server uses a `SERVICE_ROUTER_MAP` to define which endpoints are available for a given `ModelServices` type [tt-media-server/open_ai_api/__init__.py 43-72](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/__init__.py#L43-L72) When the application starts, `register_service_routes()` inspects the `settings.model_service` and mounts the corresponding `APIRouter` instances [tt-media-server/open_ai_api/__init__.py 75-103](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/__init__.py#L75-L103)

| Service Type | Primary v1 Prefix | Legacy Prefix | Description |
| --- | --- | --- | --- |
| `LLM` | `/v1/chat/completions`, `/v1/completions` | N/A | Text generation and Chat |
| `IMAGE` | `/v1/images` | `/image` | Image generation (SDXL, Flux) |
| `AUDIO` | `/v1/audio` | `/audio` | Speech-to-text (Whisper) |
| `VIDEO` | `/v1/videos` | `/video` | Video generation (DiT, Wan2.2) |
| `CNN` | `/v1/cnn` | `/cnn` | Image search/classification |
| `TEXT_TO_SPEECH` | `/v1/audio` | `/audio` | Text-to-speech (SpeechT5) |
| `EMBEDDING` | `/v1` | N/A | Text embeddings |
| `TRAINING` | `/v1` | N/A | Fine-tuning and LoRA training |

## Service Resolver Singleton Pattern

To ensure hardware resources (Tenstorrent devices) are managed efficiently, the server utilizes a singleton pattern for model services. The `service_resolver` is a dependency-injection-friendly function that returns the appropriate `BaseService` subclass based on the runtime configuration.

### Implementation

The `service_resolver` uses a thread-safe lock and a dictionary of factory functions to lazily instantiate services [tt-media-server/resolver/service_resolver.py 13-61](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/resolver/service_resolver.py#L13-L61)

1.   **Request Arrival**: A FastAPI endpoint (e.g., `/v1/chat/completions`) is called.
2.   **Dependency Resolution**: The endpoint calls `Depends(service_resolver)`[tt-media-server/open_ai_api/chat.py 71](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/chat.py#L71-L71)[tt-media-server/open_ai_api/llm.py 27](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/llm.py#L27-L27)
3.   **Singleton Check**: `service_resolver` checks `_service_holders`. If the service (e.g., `LLMService`) doesn't exist, it uses the `_SUPPORTED_MODEL_SERVICES` map to import and create it [tt-media-server/resolver/service_resolver.py 52-60](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/resolver/service_resolver.py#L52-L60)
4.   **Service Return**: The same service instance is returned for all subsequent requests until the process restarts.

**Sources:**[tt-media-server/resolver/service_resolver.py 13-61](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/resolver/service_resolver.py#L13-L61)[tt-media-server/open_ai_api/chat.py 71](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/chat.py#L71-L71)[tt-media-server/open_ai_api/llm.py 27](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/llm.py#L27-L27)

## OpenAI-Compatible Endpoints

### Chat and Text Completions

The LLM endpoints support both legacy text completions and modern chat completions.

*   **Chat Template**: `chat_completions` uses `AutoTokenizer` to apply model-specific chat templates to the list of messages before passing the resulting prompt to the inference engine [tt-media-server/open_ai_api/chat.py 89-91](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/chat.py#L89-L91)
*   **Token Validation**: Requests are validated against `settings.vllm.max_model_length` to prevent context window overflows [tt-media-server/open_ai_api/llm.py 51-59](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/llm.py#L51-L59)
*   **Streaming**: Both `/completions` and `/chat/completions` support Server-Sent Events (SSE) streaming [tt-media-server/open_ai_api/llm.py 94-121](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/llm.py#L94-L121)[tt-media-server/open_ai_api/chat.py 138-180](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/chat.py#L138-L180)

### Image and Video Generation

*   **Image**: Supports `ImageGenerateRequest`, `ImageToImageRequest`, and `ImageEditRequest`. It includes validation for `num_inference_steps` which varies by model (e.g., Flux requires at least 4 steps) [tt-media-server/domain/image_generate_request.py 12-49](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/domain/image_generate_request.py#L12-L49) The `image.py` router dynamically selects sub-routers based on the `model_runner` setting [tt-media-server/open_ai_api/image.py 99-104](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/image.py#L99-L104)
*   **Video**: Implemented as an asynchronous job system via `BaseJobService` or synchronous direct download. It supports Text-to-Video (T2V) and Image-to-Video (I2V) via `Wan2.2`[tt-media-server/open_ai_api/video.py 100-137](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/video.py#L100-L137) Video files are processed for "faststart" (moving MP4 moov atom to the front) for better web streaming [tt-media-server/open_ai_api/video.py 68-80](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/video.py#L68-L80)

### Audio and CNN

*   **Audio (STT)**: Accepts base64 or raw bytes. It supports features like VAD (Voice Activity Detection) and speaker diarization [tt-media-server/domain/audio_processing_request.py 14-26](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/domain/audio_processing_request.py#L14-L26)
*   **CNN (Image Search)**: Supports both `multipart/form-data` for file uploads and `application/json` for base64-encoded prompts [tt-media-server/open_ai_api/cnn.py 34-61](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/open_ai_api/cnn.py#L34-L61)
