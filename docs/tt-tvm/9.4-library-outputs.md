This page documents the various library artifacts produced by the TVM build system. It covers the primary shared/static libraries (`libtvm.so`, `libtvm_runtime.so`), their constituent object libraries, build configuration options that affect library outputs, and the runtime introspection system that embeds build information into the libraries.

For information about configuring which features are included in these libraries, see [Build Options Reference](/tenstorrent/tt-tvm/9.2-build-options-reference). For details on the overall build system, see [CMake Configuration](/tenstorrent/tt-tvm/9.1-cmake-configuration).

---

## Overview of Library Artifacts

The TVM build system produces several library outputs, each serving different deployment scenarios:

| Library | Description | Primary Use Case | Size |
| --- | --- | --- | --- |
| `libtvm.so` | Full compiler and runtime library | Development, model compilation, training | Large (~100+ MB) |
| `libtvm_runtime.so` | Runtime-only library | Production deployment, inference | Small (~5-10 MB) |
| `libtvm.a` (optional) | Static version of full library | Embedded systems, custom linking | Large |
| `libtvm_runtime.a` (optional) | Static runtime library | Minimal embedded deployments | Small |
| `tvm_allvisible` (optional) | Full library with all symbols visible | Testing with HIDE\_PRIVATE\_SYMBOLS | Large |

**Sources:** [CMakeLists.txt586-608](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L586-L608)

## Primary Library Targets

### Full Compiler Library (`libtvm`)

The `libtvm` library contains the complete TVM compiler infrastructure including all frontends, Relay IR, TensorIR, optimization passes, code generators, and the runtime system.

**Build Target Definition:**

**Content Includes:**

* All compiler source files (auto\_scheduler, meta\_schedule, IR, TIR, Relay, etc.)
* Code generation backends (LLVM, VM compiler)
* Runtime system (execution, memory management, device APIs)
* Build configuration information (libinfo)

**Typical Size:** 100-200 MB depending on enabled backends

**Build Configuration:**

**Sources:** [CMakeLists.txt586-592](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L586-L592) [CMakeLists.txt682-686](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L682-L686)

---

### Runtime-Only Library (`libtvm_runtime`)

The `libtvm_runtime` library contains only the components necessary to execute pre-compiled TVM models. It excludes all compiler infrastructure, making it significantly smaller and suitable for production deployment.

**Build Target Definition:**

**Content Includes:**

* Graph executor, VM executor, AOT executor
* Device API implementations (CUDA, OpenCL, Vulkan, etc.)
* Memory management
* RPC infrastructure (if enabled)
* Profiling support (if enabled)
* Build configuration information (libinfo)

**Excludes:**

* Relay IR and optimization passes
* TensorIR scheduling
* Auto-tuning infrastructure
* Code generation backends
* Frontend converters

**Typical Size:** 5-15 MB depending on enabled runtimes

**Sources:** [CMakeLists.txt597-612](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L597-L612) [CMakeLists.txt343-349](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L343-L349)

## Intermediate Object Libraries

The build system uses CMake object libraries as intermediate compilation targets. These are not installed but are linked into the final libraries.

### Object Library Hierarchy

### tvm\_objs

Contains all compiler infrastructure source files compiled as position-independent code.

**Source File Categories:**

* `src/auto_scheduler/*.cc` - Auto-scheduling infrastructure
* `src/meta_schedule/*.cc` - Meta-scheduling system
* `src/node/*.cc` - IR node definitions
* `src/ir/*.cc` - High-level IR infrastructure
* `src/tir/*.cc` - TensorIR implementation
* `src/relay/**/*.cc` - Relay IR, operators, passes, backends
* `src/target/*.cc` - Code generation backends

**Compile Definitions Applied:**

* `DMLC_USE_LOGGING_LIBRARY=`
* `USE_RELAY_DEBUG` (if debug build)
* `USE_FALLBACK_STL_MAP` (configurable)
* `NDEBUG` (if release build)

### tvm\_runtime\_objs

Contains runtime-only source files for execution and device management.

**Source File Categories:**

* `src/runtime/*.cc` - Core runtime infrastructure
* `src/runtime/vm/*.cc` - Virtual machine runtime
* `src/runtime/memory/*.cc` - Memory management
* `src/runtime/rpc/*.cc` - RPC infrastructure (if USE\_RPC=ON)
* `src/runtime/graph_executor/*.cc` - Graph executor (if USE\_GRAPH\_EXECUTOR=ON)
* `src/runtime/cuda/*.cc` - CUDA runtime (if USE\_CUDA=ON)

## Build Configuration Options

### Static vs. Shared Libraries

**Shared Libraries (Default):**

By default, both `libtvm` and `libtvm_runtime` are built as shared libraries (`.so` on Linux, `.dylib` on macOS, `.dll` on Windows).

**Static Runtime Library:**

The runtime can be built as a static library using `BUILD_STATIC_RUNTIME`:

When enabled, `libtvm_runtime.a` is created instead of `libtvm_runtime.so`.

**Important Note for Static Linking:**

Static runtime libraries must be linked with the `--whole-archive` linker flag (or equivalent) because TVM relies on global constructors for its registry mechanism. Without this flag, some object files may be excluded from the final executable, making their runtime functions unavailable.

**Sources:** [CMakeLists.txt597-608](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L597-L608) [cmake/config.cmake366-376](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L366-L376)

**Version Information:**

* Git commit hash
* Git commit timestamp
* LLVM version
* MLIR version (if enabled)
* CUDA version (if enabled)

**Dependencies:**

* Third-party library paths (DLPACK, DMLC, etc.)
