## Purpose and Scope

This document describes the GitHub Actions-based CI/CD infrastructure for `tt-umd`. The pipeline handles automated building, testing across multiple hardware configurations, and artifact publishing. For build system configuration and dependency management, see [Build Configuration](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/Build%20Configuration) For test organization and utilities, see [Testing Infrastructure](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/Testing%20Infrastructure)

The CI/CD system supports:

*   Multi-OS/multi-architecture build matrices.
*   Hardware testing on Wormhole and Blackhole devices.
*   Multiple build types (Release, Debug, ASan, TSan).
*   Python wheel building and distribution via `cibuildwheel`.
*   Downstream client integration testing (e.g., `tt-metal`).
*   Automated release lifecycle based on versioning.

## Build Matrix Strategy

### Build Type Matrix

The CI system supports multiple build configurations with different optimization and instrumentation levels.

| Build Type | Optimization | Symbols | Sanitizers | CI Use Case |
| --- | --- | --- | --- | --- |
| `Release` | High | None | None | Production, PRs, Merge Queue |
| `RelWithDebInfo` | High | Yes | None | Performance debugging |
| `Debug` | None | Yes | None | Development |
| `ASan` | Moderate | Yes | Address | Memory safety (Merge Queue) |
| `TSan` | Moderate | Yes | Thread | Concurrency safety (Merge Queue) |
