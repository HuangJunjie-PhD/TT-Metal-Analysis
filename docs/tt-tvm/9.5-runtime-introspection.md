## Purpose and Scope

The Runtime Introspection system provides a mechanism to query TVM's build configuration at runtime. This allows applications and developers to programmatically determine which features, backends, and options were enabled when TVM was compiled, without needing to examine build files or recompile the library.

This page documents the introspection system that exposes build configuration through the `tvm.support.libinfo()` Python API and the `support.GetLibInfo` C++ function. For information about build configuration itself, see [CMake Configuration](https://deepwiki.com/tenstorrent/tt-tvm/9.1-cmake-configuration) and [Build Options Reference](https://deepwiki.com/tenstorrent/tt-tvm/9.2-build-options-reference).

## Build-Time Code Generation

### CMake Option Processing

The `add_lib_info()` function in `LibInfo.cmake` captures all CMake build options and converts them into C preprocessor definitions that are added to the `libinfo.cc` compilation unit:

The function converts each CMake variable into a corresponding preprocessor macro with the `TVM_INFO_` prefix. For example:

*   `USE_CUDA=ON` becomes `-DTVM_INFO_USE_CUDA="ON"`
*   `USE_LLVM="/usr/bin/llvm-config"` becomes `-DTVM_INFO_USE_LLVM="/usr/bin/llvm-config"`
*   `HIDE_PRIVATE_SYMBOLS=ON` becomes `-DTVM_INFO_HIDE_PRIVATE_SYMBOLS="ON"`

**Sources:**[cmake/modules/LibInfo.cmake 21-141](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake#L21-L141)[CMakeLists.txt 577-580](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L577-L580)

### Compile Definition Set

The `add_lib_info()` function sets compile definitions for approximately 80+ configuration options. Here is the pattern used:

| CMake Variable | Preprocessor Definition | Example Value |
| --- | --- | --- |
| `USE_CUDA` | `TVM_INFO_USE_CUDA` | `"ON"` or `"/path/to/cuda"` |
| `USE_LLVM` | `TVM_INFO_USE_LLVM` | `"OFF"` or `"/usr/bin/llvm-config"` |
| `USE_ROCM` | `TVM_INFO_USE_ROCM` | `"ON"` or `"/opt/rocm"` |
| `BUILD_STATIC_RUNTIME` | `TVM_INFO_BUILD_STATIC_RUNTIME` | `"ON"` or `"OFF"` |
| `HIDE_PRIVATE_SYMBOLS` | `TVM_INFO_HIDE_PRIVATE_SYMBOLS` | `"ON"` or `"OFF"` |

**Sources:**[cmake/modules/LibInfo.cmake 38-139](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake#L38-L139)

## Build Integration

### CMakeLists.txt Integration

The main `CMakeLists.txt` creates a separate object library `tvm_libinfo_objs` specifically for the libinfo compilation unit with all the definitions applied:

This ensures that `libinfo.cc` is compiled with all the `TVM_INFO_*` definitions, while other source files are compiled normally.

**Sources:**[CMakeLists.txt 577-606](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L577-L606)

### Validation

The codebase includes a lint script `check_cmake_options.py` that validates consistency between three files:

1.   Option definitions in `CMakeLists.txt` (via `tvm_option()`)
2.   Compile definitions in `LibInfo.cmake` (via `add_lib_info()`)
3.   Map entries in `libinfo.cc` (in `GetLibInfo()`)

This ensures that all CMake options are properly exposed through the introspection system.

**Sources:**[tests/lint/check_cmake_options.py 32-83](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/lint/check_cmake_options.py#L32-L83)
