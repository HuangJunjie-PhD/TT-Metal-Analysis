This page covers the development environment setup and configuration for TT-Studio, focusing on the dev mode with live reloading, pre-commit hooks, SPDX header tooling, VSCode configuration, and development utilities.

## Overview

TT-Studio provides a dedicated development mode designed for rapid iteration. It leverages Docker volume mounting and hot-reloading servers to allow developers to see code changes in real-time without rebuilding containers.

## Development Mode Setup

The primary entry point for development is the `run.py` script (which replaces the deprecated `startup.sh`). Running with the `--dev` flag configures the environment for active development.

### Launching Dev Mode

`python3 run.py --dev`
When this command is executed, the system performs the following:

1.   **Submodule Initialization**: Ensures `tt-inference-server` and other dependencies are present [[README.md 122-127](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[README.md?plain=1#L122-L127)[startup.sh 115-119](https://github.com/tenstorrent/tt-studio/blob/a6d347af/startup.sh#L115-L119)].
2.   **Container Orchestration**: Launches services using `docker-compose.yml` combined with `docker-compose.dev-mode.yml`[[app/README.md 43-45](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/README.md?plain=1#L43-L45)].
3.   **Live Mounting**: Local directories (`./app/backend` and `./app/frontend`) are mounted as volumes into the containers [[app/docker-compose.dev-mode.yml 10-12](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/docker-compose.dev-mode.yml#L10-L12)[app/docker-compose.dev-mode.yml 40-42](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/docker-compose.dev-mode.yml#L40-L42)].
4.   **Hot Reloading**: 
    *   **Backend**: The Django server runs via `uvicorn` with the `--reload` flag [[app/docker-compose.dev-mode.yml 13-14](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/docker-compose.dev-mode.yml#L13-L14)].
    *   **Frontend**: The Vite development server runs on port 3000 with Hot Module Replacement (HMR) [[app/docker-compose.dev-mode.yml 29-31](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/docker-compose.dev-mode.yml#L29-L31)[app/docker-compose.dev-mode.yml 45-46](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/docker-compose.dev-mode.yml#L45-L46)].

## Tooling and Utilities

### SPDX Header Tool

The `add_spdx_header.py` utility ensures all source files contain the required Apache-2.0 license and Tenstorrent copyright headers [[dev-tools/README.md 16-18](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[dev-tools/README.md?plain=1#L16-L18)].

**Supported Comment Styles:**

| Extension | Comment Style |
| --- | --- |
| `.py`, `.sh`, `Dockerfile` | `#` |
| `.ts`, `.tsx`, `.js`, `.jsx` | `//` |
| `.css` | `/* */` |
| `.html` | `<!-- -->` |

Sources: [dev-tools/README.md 25-32](https://github.com/tenstorrent/tt-studio/blob/a6d347af/dev-tools/README.md?plain=1#L25-L32)[app/frontend/src/components/ResetIcon.tsx 1-2](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ResetIcon.tsx#L1-L2)

### Pre-commit Hooks

Contributors are required to run pre-commit hooks to maintain code quality [[CONTRIBUTING.md 30-31](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[CONTRIBUTING.md?plain=1#L30-L31)]. The SPDX tool can be integrated into the `.pre-commit-config.yaml` to automate license header insertion [[dev-tools/README.md 110-122](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[dev-tools/README.md?plain=1#L110-L122)].

*   **Settings**: Enables `formatOnSave` and configures ESLint to run on the frontend directory [[.vscode/settings.json 1-16](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[.vscode/settings.json#L1-L16)].

## Frontend Development Detail

The frontend uses Vite for high-performance builds and development.

### Core Scripts

| Script | Command | Description |
| --- | --- | --- |
| `dev` | `vite` | Starts the HMR dev server. |
| `lint` | `eslint .` | Runs static analysis and type checks [[app/frontend/package.json 11-12](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/frontend/package.json#L11-L12)]. |
| `type-check` | `tsc --noEmit` | Validates TypeScript types [[app/frontend/package.json 9](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/frontend/package.json#L9-L9)]. |

### Proxy Configuration

To avoid CORS issues during development, the Vite server proxies API requests to the Django backend (`tt-studio-backend-api:8000`) [[app/docker-compose.dev-mode.yml 13-19](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/docker-compose.dev-mode.yml#L13-L19)].

## Hardware-Specific Development

When developing on Tenstorrent hardware, an additional compose file is used to expose the `/dev/tenstorrent` devices to the containers [[app/README.md 49-51](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/README.md?plain=1#L49-L51)]. The backend Dockerfile conditionally installs `tt-smi` and Rust tools if not in "AI Playground" (remote) mode to facilitate hardware monitoring [[app/backend/Dockerfile 28-45](https://github.com/tenstorrent/tt-studio/blob/a6d347af/[app/backend/Dockerfile#L28-L45)].
