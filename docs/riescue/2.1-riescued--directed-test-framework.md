RiescueD is the foundational directed test framework for RISC-V assembly test generation within the RiESCUE project. It provides a Python library and command-line tool (`riescued`) for authoring assembly tests with preprocessor-like directives that enable random data generation, memory management, page table generation, and multi-privilege mode testing.

**Scope**: This page covers the RiescueD framework architecture, workflow, command-line interface, and core components. For information about:

* Higher-level compliance test generation using RiescueD, see [RiescueC: Compliance Test Generator](/tenstorrent/riescue/2.2-riescuec:-compliance-test-generator)
* RiescueD command-line options and usage, see [RiescueD Command-Line Interface](/tenstorrent/riescue/2.1.1-riescued-command-line-interface)
* Directive syntax and examples, see [Test Directives and Syntax](/tenstorrent/riescue/2.1.2-test-directives-and-syntax)
* Internal component implementation details, see [RiescueD Internal Architecture](/tenstorrent/riescue/2.1.3-riescued-internal-architecture)

## Architecture Overview

RiescueD implements a pipeline architecture that transforms assembly test files with custom directives into executable ELF binaries. The framework is implemented in the `riescue.dtest_framework` package.

## Test Generation Workflow

The RiescueD workflow consists of four primary phases: configuration, generation, build, and simulation. Each phase is implemented as a method in the `RiescueD` class.

**Detailed Workflow Steps**:

| Phase | Method | Input | Output | Key Operations |
| --- | --- | --- | --- | --- |
| **Parse** | `Parser.parse()` | `.rasm` file | `ParsedTestHeader`, `Pool` | Parse directives into dataclasses |
| **Configure** | `RiescueD.configure()` | Test header, CPU config, CLI args | `FeatMgr` | Consolidate configuration from multiple sources |
| **Generate** | `RiescueD.generate()` | `FeatMgr`, `Pool` | `.S`, `.ld` files | Expand directives, generate OS code, memory maps |
| **Build** | `RiescueD.build()` | `.S`, `.ld` files | `.elf`, `.dis` files | Compile and disassemble |
| **Simulate** | `RiescueD.simulate()` | `.elf` file, `FeatMgr` | ISS log | Run on Whisper or Spike |

## RiescueD Class Interface

The `RiescueD` class at [riescue/riescued.py31-433](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L31-L433) serves as the main entry point for the directed test framework. It coordinates all phases of test generation.

### Constructor Parameters

| Parameter | Type | Description | Default |
| --- | --- | --- | --- |
| `testfile` | `Path` | Assembly test file with RiescueD directives | Required |
| `cpuconfig` | `Path` | CPU configuration JSON (memory map, features) | `dtest_framework/lib/config.json` |
| `run_dir` | `Path` | Directory for generated files | `.` (current directory) |
| `seed` | `Optional[int]` | Random seed for reproducibility | Random value from `initial_random_seed()` |
| `toolchain` | `Optional[Toolchain]` | Toolchain with compiler, disassembler, ISS | Default `Toolchain()` |

### Key Methods

| Method | Signature | Purpose |
| --- | --- | --- |
| `configure()` | `(args: Optional[argparse.Namespace]) -> FeatMgr` | Build `FeatMgr` from test header, CPU config, CLI args |
| `generate()` | `(featmgr: FeatMgr) -> GeneratedFiles` | Generate assembly and linker script |
| `build()` | `(featmgr: FeatMgr) -> GeneratedFiles` | Compile to ELF and disassemble |
| `simulate()` | `(featmgr: FeatMgr, iss: Union[Spike, Whisper]) -> Path` | Run ELF on ISS |
| `run()` | `(cl_args, elaborate_only, run_iss) -> GeneratedFiles` | Execute full workflow |

## Pool Component

The `Pool` class serves as a repository for all parsed test data. It acts as a data structure that bridges the `Parser` and `Generator` components, storing parsed directives in a structured format for later processing.

**Key Responsibilities**:

* Store parsed directives from `Parser`
* Provide access methods for `Generator` to retrieve test data
* Maintain test metadata (test name, sections, etc.)

**Common Pool Methods**:

* `add_parsed_data(data: ParsedRandomData)` - Store random data directive
* `add_parsed_addr(addr: ParsedRandomAddress)` - Store random address directive
* `add_parsed_page_mapping(mapping: ParsedPageMapping)` - Store page mapping
* `add_test_header(header: ParsedTestHeader)` - Store test header
* `get_parsed_addrs() -> dict` - Retrieve all parsed addresses
* `get_parsed_data() -> dict` - Retrieve all parsed data

## FeatMgr Component

The `FeatMgr` (Feature Manager) class consolidates configuration from three sources: test headers, CPU configuration JSON, and command-line arguments. It is built using the builder pattern via `FeatMgrBuilder`.

### Configuration Priority

The configuration sources have the following priority (highest to lowest):

1. **Command-line arguments** - Override all other settings
2. **Test header directives** - `;#test.` directives in test file
3. **CPU configuration JSON** - Default system configuration

### Key FeatMgr Attributes

| Attribute | Type | Description |
| --- | --- | --- |
| `priv_mode` | `RiscvPrivileges` | Privilege mode (machine/supervisor/user) |
| `paging_mode` | `RiscvPagingModes` | Paging mode (sv39/sv48/sv57/disable) |
| `env` | `RiscvTestEnv` | Test environment (bare\_metal/virtualized) |
| `secure_mode` | `bool` | Enable secure mode features |
| `num_cpus` | `int` | Number of CPUs for multiprocessor tests |
| `reset_pc` | `int` | Initial program counter value |
| `big_endian` | `bool` | Enable big-endian mode |

## Generator Component

The `Generator` class transforms parsed directives and configuration into concrete assembly code and linker scripts. It resolves random addresses, generates page tables, and produces the final `.S` and `.ld` files.

### Generator Responsibilities

1. **Address Resolution** - Generate random addresses constrained by directives
2. **Page Table Generation** - Create page tables for specified paging modes
3. **Assembly Expansion** - Replace directives with actual values/code
4. **Linker Script Generation** - Create memory layout script
5. **OS Code Injection** - Add exception handlers, scheduling code

### Generated Files

The `GeneratedFiles` dataclass at [riescue/dtest\_framework/artifacts.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/artifacts.py) tracks all output files:

| Attribute | File | Description |
| --- | --- | --- |
| `assembly` | `.S` | Preprocessed assembly with expanded directives |
| `linker_script` | `.ld` | Linker script with memory layout |
| `elf` | `.elf` | Compiled ELF binary |
| `dis` | `.dis` | Disassembled binary |

## Integration with RiescueC and CTK

RiescueD serves as the foundation for higher-level test generation tools:

* **RiescueC** (see [RiescueC: Compliance Test Generator](/tenstorrent/riescue/2.2-riescuec:-compliance-test-generator)) uses RiescueD's `generate()` and `build()` methods to create compliance tests from test plans
* **CTK** (see [CTK: Compliance Test Kit](/tenstorrent/riescue/2.3-ctk:-compliance-test-kit)) wraps RiescueC to generate full test suites from ISA strings
* Both tools leverage RiescueD's infrastructure for OS code, memory management, and ISS integration
