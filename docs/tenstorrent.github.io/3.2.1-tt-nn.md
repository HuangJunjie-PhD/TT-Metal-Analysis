## Purpose and Scope

TT-NN is Tenstorrent's high-level neural network operation library designed for developing and deploying AI models on Tenstorrent hardware. It provides a user-friendly Python and C++ API for working with tensors and neural network operations specifically optimized for Tensix processors.

TT-NN serves as the primary entry point for developers who want to run AI models without manual hardware management. It abstracts the complexities of the underlying hardware while providing performance-optimized implementations of standard operators. TT-NN sits above **TT-Metalium** in the software stack, leveraging its low-level kernel dispatch and memory management capabilities.

## Key Functions and Classes

### Device Management

TT-NN provides functions to manage the lifecycle of Tenstorrent hardware devices. This includes opening devices, managing memory, and ensuring execution completion.

*   `ttnn.open_device()`: Initializes a connection to a Tensix-based processor.
*   `ttnn.synchronize_device()`: A critical synchronization primitive that blocks host execution until all pending device operations are completed. [ttnn/conf.py 125](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/ttnn/conf.py#L125-L125)
*   `ttnn.close_device()`: Releases hardware resources.

### Tensor Operations

The `ttnn.Tensor` class is designed to be compatible with common frameworks like PyTorch. Key capabilities include:

*   **Memory Placement**: Moving tensors between Host (CPU) and Device (L1/DRAM).
*   **Layout Transformation**: Handling Tiled vs. Row-Major layouts required by hardware kernels.
*   **Shape Manipulation**: Optimized padding and reshaping for Tensix core alignment.

### Neural Network Operations

TT-NN includes a library of pre-compiled, highly optimized kernels for:

*   **Linear Algebra**: Matrix multiplication (`matmul`), additions, and broadcasts.
*   **Convolutions**: Optimized `conv2d` implementations for various strides and paddings.
*   **Activations**: ReLU, Softmax, Silu, and others.
*   **Transformer Blocks**: Specialized support for attention mechanisms and embeddings.
