## Purpose and Scope

This page documents RiescueC's **Bringup Mode**, the default mode for generating datapath instruction compliance tests. Bringup Mode focuses on testing individual instruction functionality across multiple RISC-V extensions through self-checking test generation.

For test plan-based privileged testing, see [Test Plan (TP) Mode](https://deepwiki.com/tenstorrent/riescue/2.2.2-test-plan-(tp)-mode). For the overall RiescueC architecture and both modes, see [RiescueC: Compliance Test Generator](https://deepwiki.com/tenstorrent/riescue/2.2-riescuec:-compliance-test-generator).

* * *

## Overview

Bringup Mode (`riescuec --mode bringup`) is the default RiescueC operational mode designed specifically for **datapath instruction compliance testing**. Unlike TP mode which generates tests from test plans in the `riscv-coretp` library, Bringup Mode focuses on systematic instruction-level validation across RISC-V extensions.

**Key Characteristics:**

*   **Target**: Datapath instruction functionality
*   **Approach**: Automated self-checking test generation
*   **Coverage**: Standard and extended RISC-V instruction sets
*   **Output**: Directed assembly tests that verify instruction behavior
*   **Self-contained**: Does not require external test plan definitions

Bringup Mode generates tests that validate correct instruction encoding, operand handling, and result computation for each supported extension. The tests are self-checking, meaning they include expected results and verification logic embedded in the generated assembly.

## Supported Extensions

### Standard Extensions

| Extension | Type | Description |
| --- | --- | --- |
| RV64I | Mandatory | 64-bit base integer instruction set |
| M | Mandatory | Integer multiplication and division |
| A | Mandatory | Atomic instructions |
| F | Mandatory | Single-precision floating-point |
| D | Mandatory | Double-precision floating-point |
| C | Mandatory | Compressed instructions |
| B | Mandatory | Bit-manipulation instructions |
| V | Mandatory | Vector extension |

| Extension | Status | Description |
| --- | --- | --- |
| Zba | Implicit (B) | Address generation instructions |

## Test Generation Strategy

Bringup Mode employs a systematic approach to generate comprehensive datapath tests:

### Test Characteristics

1.   **Self-Checking Tests**: Each generated test includes:

    *   Input operand generation
    *   Instruction execution
    *   Expected result computation
    *   Result verification logic
    *   Pass/fail indicators

2.   **Configurable Constraints**: Tests can be configured with:

    *   Specific instruction subsets
    *   Operand value ranges
    *   Exception scenarios
    *   Memory access patterns

3.   **Extension-Specific Generation**: Each extension receives tailored test generation:

    *   Integer instructions: boundary values, signed/unsigned edge cases
    *   Floating-point: NaN handling, rounding modes, special values
    *   Vector: element width variations, masking, register grouping
    *   Atomic: memory ordering, load-reserved/store-conditional sequences

## Architecture and Implementation

### Location in Codebase

Bringup Mode implementation resides in the compliance module:

*   **Source Directory**: `riescue/compliance/`
*   **Class Name**: `BringupMode`
*   **Command-Line Entry**: `riescuec` with `--mode bringup` (default mode)

### Relationship to RiescueD

Bringup Mode builds upon the RiescueD framework, leveraging its core test generation capabilities:

**Key Integration Points:**

1.   **Generator Reuse**: BringupMode uses the `Generator` class from `dtest_framework` to produce assembly files [riescue/riescued.py 265-266](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L265-L266)

2.   **FeatMgr Configuration**: Extension-specific features are configured through the `FeatMgr` system [riescue/riescued.py 222-237](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L222-L237)

3.   **Pool System**: Test data and random values are managed through the `Pool` abstraction [riescue/riescued.py 74](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/riescued.py#L74-L74)

4.   **Common Pipeline**: Both modes ultimately produce the same artifact types (assembly, linker scripts, ELF files)

## Comparison with TP Mode

| Aspect | Bringup Mode | TP Mode |
| --- | --- | --- |
| **Purpose** | Datapath instruction testing | Privileged architecture testing |
| **Input** | ISA string | Test plan name |
| **Source** | Internal instruction database | `riscv-coretp` library |
| **Focus** | Instruction functionality | Architectural scenarios |
| **Test Style** | Instruction-centric | Scenario-centric |
| **Extensions** | Datapath extensions (M, F, D, V, B, etc.) | Privileged extensions (S, H, Sv39, etc.) |
| **Complexity** | Individual instruction behavior | Multi-instruction sequences |
| **Environment** | Minimal privilege requirements | Configurable privilege modes |

For TP Mode documentation, see [Test Plan (TP) Mode](https://deepwiki.com/tenstorrent/riescue/2.2.2-test-plan-(tp)-mode).

## Usage Examples

### Example 1: Generate Tests for RV64GC

`riescuec --mode bringup --isa rv64gc`
This generates tests for:

*   RV64I (base integer)
*   M (multiply/divide)
*   A (atomics)
*   F (single-precision FP)
*   D (double-precision FP)
*   Zicsr (CSR access)
*   Zifencei (instruction fence)
*   C (compressed)

### Example 2: Generate Vector Tests

`riescuec --mode bringup --isa rv64gcv`
Adds vector extension tests including:

*   Base vector operations
*   Vector floating-point
*   Vector load/store
*   Vector mask operations

### Example 3: Generate with Extended Bit Manipulation

`riescuec --mode bringup --isa rv64gc_zba_zbb_zbc_zbs`
Tests all bit manipulation instructions across Zba, Zbb, Zbc, and Zbs extensions.
