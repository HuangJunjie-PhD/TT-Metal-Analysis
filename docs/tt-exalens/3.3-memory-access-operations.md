This page documents the Python API functions for reading and writing memory on Tenstorrent devices. These operations form the foundation for interacting with device memory through the Network-on-Chip (NOC) and debug interfaces.

For register-specific access operations, see [Register Access](https://deepwiki.com/tenstorrent/tt-exalens/3.4-register-access). For RISC-V private memory access through the debug interface, see [RISC-V Core Control](https://deepwiki.com/tenstorrent/tt-exalens/3.6-risc-v-core-control). For context initialization and device selection, see [Context and Initialization](https://deepwiki.com/tenstorrent/tt-exalens/3.1-context-and-initialization) and [Device and Coordinate Management](https://deepwiki.com/tenstorrent/tt-exalens/3.2-device-and-coordinate-management).

## Memory Read Functions

TTExaLens provides three primary functions for reading device memory, each optimized for different use cases.

### Function Overview

| Function | Purpose | Return Type | Typical Use Case |
| --- | --- | --- | --- |
| `read_word_from_device` | Read single 4-byte word | `int` | Reading individual register values or small data |
| `read_words_from_device` | Read multiple 4-byte words | `list[int]` | Reading arrays of 32-bit values |
| `read_from_device` | Read arbitrary byte count | `bytes` | Reading buffers, structures, or unaligned data |

### read_word_from_device

Reads a single 4-byte word from the specified location and address. This is the most efficient function for reading individual values.

**Key Parameters:**

*   `location`: Target core location (e.g., `"0,0"` for logical, `"0-0"` for NOC0, `"ch0"` for DRAM)
*   `addr`: Memory address to read from
*   `safe_mode`: When `True`, validates the access against the memory map before reading

**Implementation:**

*   [ttexalens/tt_exalens_lib.py 175-206](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L175-L206)

### read_words_from_device

Reads multiple consecutive 4-byte words starting from the specified address. Efficient for reading arrays of 32-bit integers.

**Key Parameters:**

*   `word_count`: Number of 4-byte words to read
*   `use_4B_mode`: Controls TLB access mode (when `None`, uses context default)

**Implementation:**

*   [ttexalens/tt_exalens_lib.py 210-249](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L210-L249)

### read_from_device

Reads an arbitrary number of bytes from the specified address. Supports unaligned reads and is the most flexible read function.

**Key Parameters:**

*   `num_bytes`: Number of bytes to read (can be any positive integer)

**Implementation:**

*   [ttexalens/tt_exalens_lib.py 253-289](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L253-L289)

## Safe Mode and Memory Validation

Safe mode protects against invalid memory accesses by validating each operation against the device's memory map.

### validate_noc_access_is_safe

Validates that a memory access falls within known, accessible memory blocks. This function:

1.   Queries the NOC memory map for the target block
2.   Verifies the block exists and is accessible
3.   For multi-block spans, validates each segment
4.   Checks read/write permissions based on block configuration

**Implementation:**

*   [ttexalens/tt_exalens_lib.py 131-172](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L131-L172)

### UnsafeAccessException

Custom exception raised when safe mode detects an invalid access:

The exception message includes:

*   Target location coordinates
*   Address range of the attempted access
*   Specific violating address
*   Whether it was a read or write operation

## NOC Configuration and Parameters

Memory operations can be configured with several NOC-related parameters that affect performance and routing.

### NOC Selection

The `noc_id` parameter controls which Network-on-Chip to use for the operation:

*   `noc_id=0`: Use NOC0 routing
*   `noc_id=1`: Use NOC1 routing
*   `noc_id=None`: Use context default (typically NOC0, or NOC1 if `context.use_noc1` is set)

**Implementation:**

*   [ttexalens/tt_exalens_lib.py 63-67](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L63-L67)

### 4B Mode

The `use_4B_mode` parameter controls TLB access granularity:

*   `use_4B_mode=True`: Use 4-byte TLB access mode (more efficient for word-aligned operations)
*   `use_4B_mode=False`: Use 1-byte TLB access mode (required for unaligned access)
*   `use_4B_mode=None`: Use context default

**Implementation:**

*   [ttexalens/tt_exalens_lib.py 70-73](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L70-L73)

### DMA Thresholds

The context contains DMA threshold settings that automatically switch between TLB and DMA access based on transfer size:

| Threshold | Default Value | Purpose |
| --- | --- | --- |
| `dma_read_threshold` | 256 bytes | Switch to DMA for reads larger than this |
| `dma_write_threshold` | 256 bytes | Switch to DMA for writes larger than this |
| `remote_dma_read_threshold` | 4096 bytes | DMA threshold for remote devices |
| `remote_dma_write_threshold` | 4096 bytes | DMA threshold for remote devices |

DMA operations are chunked to avoid timeouts on large transfers. The chunking happens at the UMD level.

**Sources:**[ttexalens/tt_exalens_lib.py 63-73](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L63-L73)[ttexalens/context.py](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/context.py) (implied from context references)

## Unaligned Memory Access

TTExaLens fully supports unaligned memory access for both reads and writes. The library handles alignment automatically at multiple levels.

### Unaligned Read Support

The `read_from_device` function can read any number of bytes starting at any address:

**Implementation Strategy:**

1.   Determine word-aligned boundaries that span the requested range
2.   Read complete words via NOC
3.   Extract the requested bytes from the word-aligned buffer
4.   Return precisely the bytes requested

**Test Coverage:**

*   [test/ttexalens/unit_tests/test_lib.py 272-306](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_lib.py#L272-L306)

### Unaligned Write Support

The `write_to_device` function supports writing any number of bytes to any address:

**Implementation Strategy:**

1.   For sub-word writes, read-modify-write the affected words
2.   Preserve surrounding bytes in the same word
3.   Only modify the exact bytes specified

## MemoryAccess Abstraction Layer

TTExaLens provides a `MemoryAccess` abstraction that decouples memory operations from the underlying access mechanism. This allows the same code to work with different memory types and access paths.

### MemoryAccess Class Hierarchy

**Sources:**[ttexalens/memory_access.py 56-237](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/memory_access.py#L56-L237)

### L1MemoryAccess

Direct NOC-based access to L1 memory. Used when the address is known to be in the L1 region.

**Key Implementation Details:**

**Characteristics:**

*   Direct hardware access via `OnChipCoordinate.noc_read()`/`noc_write()`
*   Validates accesses are within L1 boundaries
*   Most efficient for L1-resident data
*   No debug interface overhead

**Implementation:**

*   [ttexalens/memory_access.py 102-135](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/memory_access.py#L102-L135)

### RiscDebugMemoryAccess

Access through the RISC-V debug interface, used for private memory and when the core must be halted.

**Key Implementation Details:**

**Characteristics:**

*   Access via `RiscDebug.read_memory_bytes()`/`write_memory_bytes()`
*   Can automatically halt core for access (`ensure_halted_access`)
*   Can restrict to L1 + private memory only (`restricted_access`)
*   Required for private memory regions (0xFFB00000+, 0xFFC00000+)
*   Uses debug register protocol under the hood

**Restricted Access Validation:**

When `restricted_access=True`, validates that addresses fall within:

*   L1 memory block (from private address perspective)
*   Data private memory block (if available)

Raises `RestrictedMemoryAccessError` for addresses outside these regions.

**Implementation:**

*   [ttexalens/memory_access.py 157-215](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/memory_access.py#L157-L215)

### FixedMemoryAccess

Read-only access to an in-memory byte buffer. Used for non-addressable values.

**Characteristics:**

*   Backed by Python `bytes` object
*   No hardware access
*   Used when DWARF evaluation produces temporary values
*   Write operations raise an exception

**Implementation:**

*   [ttexalens/memory_access.py 137-155](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/memory_access.py#L137-L155)

### CachedReadMemoryAccess

Wrapper that caches a range of bytes and serves reads from cache when possible.

**Key Implementation Details:**

**Characteristics:**

*   Delegates to base `MemoryAccess` for cache misses
*   Useful for reading multiple fields from same struct
*   Writes always go to base implementation

**Implementation:**

*   [ttexalens/memory_access.py 217-237](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/memory_access.py#L217-L237)

### Factory Methods

The `MemoryAccess` class provides factory methods for common use cases:

**MemoryAccess.create(risc_debug, ...)**

Creates a `RiscDebugMemoryAccess` instance for accessing memory through a RISC core:

**MemoryAccess.create_l1(location)**

Creates an `L1MemoryAccess` instance for direct NOC access to L1:

**Implementation:**

*   [ttexalens/memory_access.py 89-100](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/memory_access.py#L89-L100)

### RestrictedMemoryAccessError

Exception raised when `RiscDebugMemoryAccess` with `restricted_access=True` detects an access outside L1 or private memory:

**Error Message Format:**

```
RISC 'brisc' at 0-0 (0,0), Restricted access: Address range [0xDEADBEEF, 0xDEADBEF3] is outside allowed memory regions.
```

**Implementation:**

### Memory Type Comparison

| Memory Type | Address Range | Access Abstraction | Hardware Path | Use Case |
| --- | --- | --- | --- | --- |
| L1 Memory | 0x00000000-0x00100000 | `L1MemoryAccess` or High-level API | Direct NOC | General data, multi-core accessible |
| DRAM | Device-specific | High-level API | Direct NOC | Large data storage |
| Data Private | 0xFFB00000-0xFFBFFFFF | `RiscDebugMemoryAccess` | Debug registers | RISC-V private data |
| Code Private | 0xFFC00000-0xFFCFFFFF | `RiscDebugMemoryAccess` | Debug registers | RISC-V private code |
| Tensix DEST | Blackhole-specific | `RiscDebugMemoryAccess` (unrestricted) | Debug registers | Tensix destination register file |
| In-Memory Values | N/A | `FixedMemoryAccess` | None | DWARF-evaluated temporaries |
