## Package Metadata and Dependencies

The `debian/control` file defines the source package metadata and binary package specifications.

### Source Package Configuration

| Field | Value | Purpose |
| --- | --- | --- |
| Source | `tt-burnin` | Source package name |
| Section | `utils` | Debian archive section |
| Priority | `optional` | Installation priority |
| Maintainer | `Sam Bansal <sbansal@tenstorrent.com>` | Package maintainer |
| Homepage | `https://github.com/tenstorrent/tt-burnin` | Project homepage |

### Build Dependencies

The package requires several build-time dependencies specified in [debian/control 5-13](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/control#L5-L13):

*   `debhelper (>= 10)` - Debian packaging helper tools
*   `dh-python` - Python-specific debhelper extensions
*   `pybuild-plugin-pyproject` - Support for PEP 517/518 builds
*   `python3-setuptools` - Python package building tools
*   `python3-wheel` - Wheel format support
*   `python3-pip` - Package installer

### Runtime Dependencies

The binary package depends on:

*   `python3-pyluwen` - Hardware interface library for Tenstorrent devices
*   Standard Python 3 dependencies resolved automatically

Sources: [debian/control 1-34](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/control#L1-L34)

## Build Process and Rules

The build process uses debhelper with the pybuild system to create Debian packages from the Python project.

### Build Rule Configuration

**Build Rules and Compatibility System**

The `debian/rules` file [debian/rules 1-12](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/rules#L1-L12) uses:

*   `dh $@` - Standard debhelper sequence
*   `--buildsystem=pybuild` - Python build system integration
*   `--with=python3` - Python 3 support
*   `override_dh_auto_test` - Skips tests during package build

The `setup.py` file [setup.py 1-16](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L1-L16) provides compatibility for Ubuntu 22.04 packaging tools using `setuptools_scm` for dynamic versioning from git tags.

Sources: [debian/rules 1-12](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/rules#L1-L12)[setup.py 1-16](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L1-L16)

## Package Integration with Build System

The Debian packaging integrates with the project's Python packaging and CI/CD system.

### Integration Points

| Component | Integration Method | Purpose |
| --- | --- | --- |
| `pyproject.toml` | Referenced by pybuild | Project configuration and dependencies |
| `setuptools_scm` | Git tag versioning | Dynamic version from git tags |
| CI/CD workflows | Automated building | Package generation in multiple Ubuntu versions |
| GPG signing | Package authentication | Ensures package integrity |

The packaging system supports multiple Ubuntu distributions and integrates with the broader release pipeline for automated package generation and distribution.
