# Python Environment Options

Relevant source files
*   [install.sh](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh)

## Purpose and Scope

This document details the different Python environment options available in the Tenstorrent software stack installation system. These options determine how Python packages like `tt-flash` and `tt-smi` are installed and configured during the installation process. For information about environment variables that can be used during installation, see [Environment Variables](https://deepwiki.com/tenstorrent/tt-installer/4.1-environment-variables).

Sources: [install.sh 76-83](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L76-L83)

## Overview

The tt-installer provides four different methods for installing Python packages, giving users flexibility in how Python tools are integrated into their system. Each option has different implications for environment isolation, path management, and user workflow.

Sources: [install.sh 76-83](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L76-L83)[install.sh 226-251](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L226-L251)[install.sh 489-525](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L489-L525)

## Python Environment Selection Process

During installation, the Python environment choice is determined in the following order:

1.   If the `TT_PYTHON_CHOICE` environment variable is set, that value is used
2.   If running in non-interactive mode, the default option (new venv) is used
3.   In interactive mode, the user is prompted to select an option
4.   Special case: On Ubuntu 20.04, pipx (option 4) is not available and defaults to new venv if selected

Sources: [install.sh 226-251](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L226-L251)[install.sh 483-486](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L483-L486)

## Python Environment Options in Detail

### Option 1: Existing Virtual Environment

This option uses an already activated Python virtual environment for package installation.

*   **When to use**: When you already have a virtual environment set up and activated
*   **Implementation**: Checks for `VIRTUAL_ENV` environment variable to ensure a venv is active
*   **Command used**: `pip install`
*   **User workflow**: User must activate the virtual environment before running the installer

Sources: [install.sh 490-499](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L490-L499)

### Option 2: New Virtual Environment (Default)

This option creates a new Python virtual environment specifically for Tenstorrent tools.

*   **When to use**: Default and recommended option for most users
*   **Implementation**: Creates a new venv at the determined location and activates it
*   **Command used**: `pip install`
*   **Location determination**: 
    *   Uses `TT_NEW_VENV_LOCATION` if set
    *   Otherwise, uses `${XDG_DATA_HOME}/tenstorrent-venv` if `XDG_DATA_HOME` is defined
    *   Otherwise, uses `${HOME}/.tenstorrent-venv`

Sources: [install.sh 253-264](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L253-L264)[install.sh 518-525](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L518-L525)

### Option 3: System Python

This option installs packages to the system-wide Python environment.

*   **When to use**: Not recommended unless you have specific requirements
*   **Implementation**: Installs directly to system Python paths
*   **Command used**: 
    *   Modern OS: `pip install --break-system-packages`
    *   Ubuntu 20: `pip install`

*   **Warning**: May interfere with system packages or cause conflicts

Sources: [install.sh 500-509](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L500-L509)

### Option 4: pipx

This option uses pipx for isolated package installation.

*   **When to use**: When you want isolated tool installation with dependencies
*   **Implementation**: Ensures pipx is installed and in PATH
*   **Command used**: `pipx install`
*   **Limitation**: Not available on Ubuntu 20.04

Sources: [install.sh 510-517](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L510-L517)

## Distribution-Specific Considerations

### Ubuntu 20.04 LTS

*   pipx installation (option 4) is not supported
*   If option 4 is selected, it will fall back to option 2 (new venv)
*   No `--break-system-packages` flag used for system installs, as this is a newer pip option

### Modern Distributions (Ubuntu 22.04+, Debian 12+, Fedora, etc.)

*   All four options are supported
*   System Python installation uses the `--break-system-packages` flag with pip

Sources: [install.sh 483-486](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L483-L486)[install.sh 504-508](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L504-L508)

## Python Packages Installed

Regardless of the environment option selected, the following Python packages are installed:

| Package | Purpose | Installation Method |
| --- | --- | --- |
| tt-flash | Firmware update tool | `${PYTHON_INSTALL_CMD} git+https://github.com/tenstorrent/tt-flash.git` |
| tt-smi | System Management Interface | `${PYTHON_INSTALL_CMD} git+https://github.com/tenstorrent/tt-smi` |

Sources: [install.sh 562](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L562-L562)[install.sh 607](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L607-L607)

## Environment Configuration Table

| Option | Description | Installation Command | Notes |
| --- | --- | --- | --- |
| 1 | Existing venv | `pip install` | Requires active venv before installation |
| 2 | New venv | `pip install` | Default option, creates venv at determined location |

## Post-Installation Considerations

After installation, users should be aware of the following:

*   If options 1 or 2 were used (venv), users need to activate the virtual environment to use the installed tools
*   A warning is displayed after installation if a venv was used: `You'll need to run "source ${VIRTUAL_ENV}/bin/activate" to use tenstorrent's Python tools.`
*   For pipx installations, tools are available in PATH without needing to activate any environment

Sources: [install.sh 632-633](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L632-L633)

## Configuring Python Environment via Environment Variables

Users can pre-configure their Python environment preference using environment variables:

*   `TT_PYTHON_CHOICE`: Set to 1, 2, 3, or 4 to select the corresponding option
*   `TT_NEW_VENV_LOCATION`: Specify a custom location for the new virtual environment if option 2 is selected

Example:

`# Use system Python installationexport TT_PYTHON_CHOICE=3 # Create a new venv at a custom locationexport TT_PYTHON_CHOICE=2export TT_NEW_VENV_LOCATION="/opt/tenstorrent-venv"`
