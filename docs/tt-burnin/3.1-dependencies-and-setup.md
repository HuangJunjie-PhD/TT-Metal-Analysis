## Python Version Support

The tt-burnin project supports a wide range of Python versions to ensure compatibility across different deployment environments. The minimum supported Python version is 3.7, with explicit support declared through Python 3.11.

| Python Version | Support Status | Notes |
| --- | --- | --- |
| 3.7 | Supported | Minimum version |
| 3.8 | Supported | Production target |
| 3.9 | Supported | Full feature support |
| 3.10 | Supported | Full feature support |
| 3.11 | Supported | Latest supported |

The project includes version-specific dependency handling for backward compatibility, particularly for modules that were added to the standard library in later Python versions.

## Core Dependencies

The tt-burnin utility relies on several external Python packages to provide hardware interface capabilities, configuration parsing, and shared utilities.

### Dependency Architecture

**Sources:**[pyproject.toml 26-35](https://github.com/tenstorrent/tt-burnin/blob/809f293d/pyproject.toml#L26-L35)

### Core Dependency Details

| Package | Version Constraint | Purpose |
| --- | --- | --- |
| `tt_tools_common` | `>=1.4.19` | Shared utilities and common functionality across Tenstorrent tools |
| `pyluwen` | `>=0.7.11` | Low-level hardware interface library for Tenstorrent devices |
| `tomli` | `==2.0.1` | TOML file parsing (Python <3.11 only) |
| `jsons` | `==1.6.3` | JSON serialization and deserialization |
| `importlib-metadata` | `==4.11.3` | Package metadata access (Python <3.8 only) |

The version constraints use different strategies:

*   **Minimum versions** (`>=`) for core Tenstorrent libraries to ensure compatibility
*   **Exact versions** (`==`) for serialization libraries to ensure reproducible behavior
*   **Conditional dependencies** for Python version compatibility

**Sources:**[pyproject.toml 27-34](https://github.com/tenstorrent/tt-burnin/blob/809f293d/pyproject.toml#L27-L34)

## Entry Points and CLI Commands

The project defines a single console script entry point that maps the `tt-burnin` command to the main application function.

### Entry Point Mapping

The entry point configuration creates the `tt-burnin` console command:

```
[project.scripts]
tt-burnin = "tt_burnin.main:main"
```

This maps the command-line invocation `tt-burnin` to the `main()` function in the `tt_burnin.main` module, providing the primary interface for the burn-in utility.

**Sources:**[pyproject.toml 48-49](https://github.com/tenstorrent/tt-burnin/blob/809f293d/pyproject.toml#L48-L49)

## Package Configuration

The project uses setuptools for package building and data file inclusion, with specific configuration for TTX workload files.

### Package Data and Structure

Key package configuration elements:

| Configuration | Value | Purpose |
| --- | --- | --- |
| `include-package-data` | `true` | Include non-Python files in package |
| `package-data` | `"*": ["ttx/*.ttx"]` | Include all TTX workload files |
| `packages.find.where` | `["."]` | Search current directory for packages |

The package data configuration ensures that TTX workload files are included in the distributed package, making them available at runtime for hardware burn-in testing.

**Sources:**[pyproject.toml 51-61](https://github.com/tenstorrent/tt-burnin/blob/809f293d/pyproject.toml#L51-L61)

## Build System Requirements

The project uses a modern Python packaging approach with setuptools as the build backend.

### Build System Configuration

Build system specification:

`[build-system]requires = [  "setuptools>=43.0.0",  "wheel"]build-backend = "setuptools.build_meta"`
This configuration:

*   Requires `setuptools` version 43.0.0 or later for modern packaging features
*   Includes `wheel` for building wheel distributions
*   Uses `setuptools.build_meta` as the PEP 517 build backend

The build system supports creating both source distributions and wheel packages for distribution through PyPI and other package repositories.
