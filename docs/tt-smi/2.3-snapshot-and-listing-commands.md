# Snapshot and Listing Commands

Relevant source files
*   [tt_smi/log.py](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py)
*   [tt_smi/tt_smi.py](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py)

This page documents TT-SMI's snapshot generation and device listing functionality. Snapshot commands capture device state and telemetry data in JSON format, while listing commands provide quick device enumeration without launching the full TUI. For reset operation documentation, see [Reset Commands](https://deepwiki.com/tenstorrent/tt-smi/2.2-reset-commands). For TUI operation, see [Interactive Text User Interface](https://deepwiki.com/tenstorrent/tt-smi/3-interactive-text-user-interface-(tui)).

## Overview

TT-SMI provides two categories of output-focused commands:

| Category | Purpose | Output Format | Exit Behavior |
| --- | --- | --- | --- |
| **Listing** | Enumerate detected devices | Plain text table | Immediate exit |
| **Snapshot** | Capture full device state | JSON | Immediate exit or on TUI quit |

These commands are mutually exclusive with the interactive TUI and with reset operations.

## Snapshot Commands

Snapshots capture comprehensive device state in structured JSON format using Pydantic models. Three command variants control output destination and behavior.

### Snapshot to STDOUT (`-s`)

The `-s` or `--snapshot` flag dumps a JSON snapshot to standard output and exits immediately.

### Snapshot to File (`-f`)

The `-f` or `--filename` flag writes the snapshot to a file. The filename is optional.

**Command-line syntax:**

**Implementation:**[tt_smi/tt_smi.py 608-616](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L608-L616) defines the argument with complex default handling:

**Argument parsing logic:**

| User Input | `args.filename` Value | Behavior |
| --- | --- | --- |
| (no `-f`) | `False` | No snapshot, continue to TUI |
| `-f` (no arg) | `None` | Use default filename |
| `-f path.json` | `"path.json"` | Use specified path |
| `-f -` | `"-"` | Write to stdout |

### TTY Detection and Override (`--snapshot_no_tty`)

TT-SMI automatically detects whether stdout is connected to a terminal (TTY) and adjusts output formatting accordingly. The `--snapshot_no_tty` flag forces non-TTY behavior.

**Command-line syntax:**

**Implementation:**[tt_smi/tt_smi.py 638-642](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L638-L642) defines the flag:

The `is_tty` value controls output formatting:

*   When `True`: Pretty-printed JSON with color codes (for interactive terminals)

## JSON Data Model Structure

All snapshot output uses structured Pydantic models defined in [tt_smi/log.py](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py) ensuring consistent schema for downstream processing and Elasticsearch compatibility.

**Sources:**[tt_smi/log.py 266-279](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py#L266-L279)

The log models use Pydantic `BaseModel` with Elasticsearch type annotations:

| Model Class | Purpose | Location | Fields |

| `HostInfo` | Host system metadata | [tt_smi/log.py 140-148](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py#L140-L148) | OS, Distro, Kernel, Hostname, Platform, Python, Memory, Driver |
| `HostSWVersions` | Software versions | [tt_smi/log.py 151-153](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py#L151-L153) | `tt_smi`, `pyluwen` |
| `TTSMIDeviceLog` | Per-device container | [tt_smi/log.py 257-263](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py#L257-L263) | `smbus_telem`, `board_info`, `telemetry`, `firmwares`, `limits` |

| `Telemetry` | Processed telemetry | [tt_smi/log.py 225-231](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/log.py#L225-L231) | `voltage`, `current`, `aiclk`, `power`, `asic_temperature` |
