## Bounty Program

The Tenstorrent Bounty Program is a discretionary initiative designed to reward community members for improving in-scope open-source projects. Participants can earn payouts (Bounties) by addressing GitHub issues tagged with `bounty` across various repositories including `tt-metal`, `tt-forge`, and `tt-kmd`.

*   **Eligibility:** Participants must be of legal age, not sanctioned, and not current Tenstorrent employees or contractors.
*   **Process:** Contributors must be assigned to an issue on GitHub and submit a Pull Request (PR) that is subsequently merged as an "Accepted Contribution".
*   **Payouts:** Rewards are categorized into tiers (e.g., Warmup, Easy, Medium, Hard) as defined in the program terms.

For details, see [Bounty Program](https://deepwiki.com/tenstorrent/tenstorrent.github.io/5.1-bounty-program).

## Community & Developer Resources

Tenstorrent maintains several platforms to assist developers in building and deploying models:

*   **Discord:** The primary hub for real-time community interaction and informal troubleshooting.
*   **Developer Hub:** A centralized portal for technical guides, API references, and model demos.
*   **Tenstorrent Console:** Provides cloud-based access to Tenstorrent hardware for developers who do not have local AIBs or systems.
*   **TT-Awesome:** A curated catalog of community projects, models, and tools optimized for Tenstorrent hardware.

For details, see [Community & Developer Resources](https://deepwiki.com/tenstorrent/tenstorrent.github.io/5.2-community-and-developer-resources).
