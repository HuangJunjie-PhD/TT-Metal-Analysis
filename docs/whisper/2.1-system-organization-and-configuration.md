## Purpose and Scope

This document describes the top-level system organization in Whisper, including the `System` class that manages multiple cores and harts, the `HartConfig` class that handles JSON-based configuration, and the initialization flow from program startup through configured system ready to execute. This page focuses on structural organization and configuration mechanics. For details on individual hart execution, see [Hart Execution Unit](https://deepwiki.com/tenstorrent/whisper/2.2-hart-execution-unit). For memory subsystem details, see [Memory Subsystem](https://deepwiki.com/tenstorrent/whisper/4-memory-subsystem).

## System Class

The `System<URV>` class is the top-level container that manages all system resources. It is templated on `URV` (Unsigned Register Value), which is either `uint32_t` for RV32 or `uint64_t` for RV64.

### Core Responsibilities

| Responsibility | Description |
| --- | --- |
| **Core Management** | Creates and maintains `Core` instances, each containing multiple harts |
| **Hart Indexing** | Maps hart IDs (MHARTID CSR values) to system hart indices |
| **Memory Management** | Owns the shared `Memory` instance and optional `SparseMem` for callback-based memory |
| **Device Configuration** | Configures interrupt controllers (IMSIC, APLIC), IOMMU, PCI, and I/O devices |
| **MCM Coordination** | Manages the Memory Consistency Model (`Mcm`) when enabled |
| **System Call Emulation** | Coordinates the `Syscall` subsystem for Linux/newlib emulation |

### Key Data Members

```
System<URV>
├── cores_: vector<shared_ptr<Core<URV>>>           // All cores in the system
├── sysHarts_: vector<shared_ptr<Hart<URV>>>        // Flat list of all harts
├── hartIdToIndex_: unordered_map<URV, unsigned>    // MHARTID → system index
├── memory_: shared_ptr<Memory>                     // Shared physical memory
├── syscall_: shared_ptr<Syscall<URV>>              // System call emulator
├── sparseMem_: unique_ptr<SparseMem>               // Sparse memory model (optional)
├── mcm_: shared_ptr<Mcm<URV>>                      // Memory consistency model (optional)
├── imsicMgr_: ImsicMgr                            // IMSIC manager
├── aplic_: shared_ptr<Aplic>                      // APLIC interrupt controller
├── iommu_: shared_ptr<Iommu>                      // IOMMU
├── pci_: shared_ptr<Pci>                          // PCI bus
└── ioDevs_: vector<shared_ptr<IoDevice>>          // I/O devices (UART, etc.)
```

## Core and Hart Organization

### Hierarchical Structure

Whisper organizes harts hierarchically into cores:

### Hart Indexing Scheme

Whisper uses two indexing schemes for harts:

| Index Type | Description | Usage |
| --- | --- | --- |
| **Hart ID** | Value of MHARTID CSR | Architectural hart identifier, may have gaps |
| **System Index** | Sequential 0 to N-1 | Internal indexing for arrays/vectors |

The mapping formula:

*   **Core Hart ID**: `coreIx * hartIdOffset`
*   **Hart ID**: `coreHartId + hartInCore`
*   **System Index**: `coreIx * hartsPerCore + hartInCore`

Example: For a system with 2 cores, 4 harts per core, and hartIdOffset=16:

*   Core 0: Hart IDs 0-3, System Indices 0-3
*   Core 1: Hart IDs 16-19, System Indices 4-7

### Configuration Application Methods

The `HartConfig` class provides specialized methods for different configuration categories:

#### CSR Configuration

Applies CSR definitions, reset values, write masks, and poke masks to each hart.

```
applyCsrConfig(Hart& hart, string_view name, const json& conf, bool verbose)
```

Key parameters configurable per CSR:

*   `reset`: Reset value
*   `mask`: Write mask (bits writable via CSR instructions)
*   `poke_mask`: Poke mask (bits writable via debugger/server)
*   `exists`: Whether CSR is implemented
*   `shared`: Whether CSR is shared across harts
*   `number`: CSR number (for custom CSRs)
*   `privilege_mode`: Privilege level (M/S/U)

**Sources:**[HartConfig.cpp 228-488](https://github.com/tenstorrent/whisper/blob/733fd921/HartConfig.cpp#L228-L488)

#### Vector Configuration

Configures the vector extension parameters:

```
applyVectorConfig(Hart& hart, const json& config)
```

Key parameters:

*   `bytes_per_vec`: VLEN in bytes (vector register length)
*   `min_bytes_per_elem` / `max_bytes_per_elem`: Element width bounds
*   `min_bytes_per_lmul` / `max_bytes_per_lmul`: LMUL-specific constraints
*   `mask_agnostic_policy` / `tail_agnostic_policy`: "ones" or "undisturb"
*   `trap_non_zero_vstart` / `trap_out_of_bounds_vstart`: Exception behavior

**Sources:**[HartConfig.cpp 791-1119](https://github.com/tenstorrent/whisper/blob/733fd921/HartConfig.cpp#L791-L1119)

#### Trigger Configuration

Configures debug triggers (breakpoints, watchpoints):

```
applyTriggerConfig(Hart& hart, const json& config)
```

Each trigger specifies arrays for:

*   `reset`: Reset values for tdata1/tdata2/tdata3/tinfo/tcontrol
*   `mask`: Write masks
*   `poke_mask`: Poke masks

**Sources:**[HartConfig.cpp 494-566](https://github.com/tenstorrent/whisper/blob/733fd921/HartConfig.cpp#L494-L566)

#### Performance Counter Configuration

Configures hardware performance monitoring:

```
applyPerfEvents(Hart& hart, const json& config, bool userMode, bool cof)
```

Key parameters:

*   `num_mmode_perf_regs`: Number of machine-mode performance counters
*   `max_mmode_perf_event`: Maximum event ID
*   `mmode_perf_events`: Array of event IDs
*   `mmode_perf_event_map`: Mapping of event names to IDs

**Sources:**[HartConfig.cpp 622-687](https://github.com/tenstorrent/whisper/blob/733fd921/HartConfig.cpp#L622-L687)

## Memory System Setup

The memory system is initialized during `System` construction:

### Memory Callbacks

When `MEM_CALLBACKS` is defined at compile time, Whisper uses a sparse memory model where memory operations are routed through callbacks:

| Callback | Purpose |
| --- | --- |
| `readf` | Read from sparse memory on memory read operations |
| `writef` | Write to sparse memory on memory write operations |
| `initf` | Initialize pages when loading ELF/hex/binary files |

This allows simulating larger address spaces without allocating physical memory for unused regions.

### UART Configuration

UART provides serial I/O:

```
defineUart(type, addr, size, iid, channel, regShift)
```

Supported UART types:

*   `uartsf`: Simple UART (no interrupt support)
*   `uart8250`: 16550-compatible UART with interrupts

Channel types for uart8250:

*   `stdio`: Standard input/output
*   `pty`: Pseudo-terminal
*   `unix:<path>`: Unix domain socket
*   Semicolon-separated for fork (e.g., `stdio;pty`)

## Configuration File Format

The JSON configuration file has a hierarchical structure:

`{  "cores": 1,  "harts": 1,  "isa": "rv64imafdc",  "memsize": "0x80000000",  "pagesize": "0x1000",    "csr": {    "mhartid": { "reset": "0x0" },    "misa": { "reset": "0x800000000014112d", "mask": "0x0" }  },    "vector": {    "bytes_per_vec": 128,    "min_bytes_per_elem": 1,    "max_bytes_per_elem": 8  },    "triggers": [    {      "reset": ["0x0", "0x0", "0x0"],      "mask": ["0xffffffff", "0xffffffff", "0xffffffff"],      "poke_mask": ["0xffffffff", "0xffffffff", "0xffffffff"]    }  ],    "memmap": {    "pma": [      {        "low": "0x80000000",        "high": "0xffffffff",        "attribs": ["read", "write", "exec"]      }    ]  },    "aclint": {    "mswi": { "base": "0x2000000", "size": "0x4000" },    "mtimer": { "base": "0x2004000", "size": "0x8000" }  },    "imsic": {    "mbase": "0x28000000",    "mstride": "0x1000",    "sbase": "0x29000000",     "sstride": "0x1000",    "guests": 4,    "interrupt-ids": [255, 255, 255]  },    "aplic": {    "num_sources": 1023,    "domains": {      "machine": {        "parent": null,        "base": "0xc000000",        "size": "0x8000",        "is_machine": true,        "harts": [0]      }    }  },    "iommu": {    "base": "0x30000000",    "size": "0x10000",    "capabilities": "0x1f",    "aplic_source": 1  },    "pci": {    "config_base": "0x40000000",    "mmio_base": "0x50000000",    "mmio_size": "0x10000000",    "buses": 1,    "slots": 8  }}`
### Configuration Value Formats

The JSON parser supports multiple formats for numeric values:

| Format | Example | Description |
| --- | --- | --- |
| Decimal | `"reset": 255` | Plain integer |
| Hexadecimal | `"reset": "0xff"` | String with 0x prefix |
| Boolean | `"exists": true` | true/false or 0/1 or "True"/"False" |
| Array | `"mask": [1, 2, 3]` | Array of values |
