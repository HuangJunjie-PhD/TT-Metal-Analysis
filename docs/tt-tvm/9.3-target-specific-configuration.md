This document explains how to configure TVM for different hardware targets (CPU, GPU, embedded, accelerators) through the CMake build system. It covers the configuration options, CMake modules, and workflow for enabling target-specific features.

For general CMake configuration and build options, see [CMake Configuration](/tenstorrent/tt-tvm/9.1-cmake-configuration). For a comprehensive list of all build flags, see [Build Options Reference](/tenstorrent/tt-tvm/9.2-build-options-reference). For information about the generated library outputs, see [Library Outputs](/tenstorrent/tt-tvm/9.4-library-outputs).

## Configuration System Overview

TVM uses CMake with a template-based configuration system. The main configuration file is `cmake/config.cmake`, which is copied to the build directory and modified with target-specific options. Each hardware target has a corresponding CMake module in `cmake/modules/` that is included by the main [CMakeLists.txt518-561](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L518-L561)

**Sources:** [CMakeLists.txt1-898](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L1-L898) [cmake/config.cmake1-441](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L1-L441) [cmake/modules/LibInfo.cmake1-142](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake#L1-L142)

## Configuration Workflow

The standard workflow for configuring a target-specific build:

| Step | Command | Description |
| --- | --- | --- |
| 1 | `mkdir build && cd build` | Create build directory |
| 2 | `cp ../cmake/config.cmake .` | Copy template configuration |
| 3 | Edit `config.cmake` | Set target-specific options |
| 4 | `cmake ..` | Generate build files |
| 5 | `make -j8` | Build with 8 threads |

Options can be set in three ways:

* **Template modification**: Edit `build/config.cmake` directly
* **Command-line override**: `cmake -DUSE_CUDA=ON ..`
* **Script configuration**: Use shell scripts to append options

**Sources:** [cmake/config.cmake18-37](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L18-L37) [CMakeLists.txt17-23](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L23)

## GPU Target Configuration

### CUDA Configuration

The CUDA module provides comprehensive GPU support including NVPTX code generation and multiple CUDA libraries.

**CUDA Configuration Example:**

The `find_cuda()` macro in [cmake/utils/FindCUDA.cmake41-143](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/utils/FindCUDA.cmake#L41-L143) handles three configuration modes:

* **Auto-search**: `USE_CUDA=ON` uses CMake's `find_package(CUDA)`
* **Custom path**: `USE_CUDA=/path/to/cuda` specifies CUDA toolkit location
* **Library detection**: Automatically finds NVRTC, cuBLAS, cuDNN in toolkit paths

**Sources:** [cmake/config.cmake43-56](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L43-L56) [cmake/modules/CUDA.cmake1-98](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/CUDA.cmake#L1-L98) [cmake/utils/FindCUDA.cmake41-143](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/utils/FindCUDA.cmake#L41-L143) [tests/scripts/task\_config\_build\_gpu.sh26-54](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/scripts/task_config_build_gpu.sh#L26-L54)

### CUDA Library Integration

The CUDA module conditionally includes libraries based on configuration:

| Library | Runtime Sources | Linker Flags |
| --- | --- | --- |
| **NVRTC** | Always included with CUDA | `CUDA_NVRTC_LIBRARY` |
| **cuDNN** | `src/runtime/contrib/cudnn/*.cc` | `CUDA_CUDNN_LIBRARY` |
| **cuBLAS** | `src/runtime/contrib/cublas/*.cc` | `CUDA_CUBLAS_LIBRARY`, `CUDA_CUBLASLT_LIBRARY` |
| **Thrust** | `src/runtime/contrib/thrust/*.cu` | (header-only, requires CUDA) |
| **cuRAND** | `src/runtime/contrib/curand/*.{cc,cu}` | `CUDA_CURAND_LIBRARY` |
| **NCCL** | `src/runtime/disco/nccl/*.cc` | `nccl` target |

**Sources:** [cmake/modules/CUDA.cmake32-94](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/CUDA.cmake#L32-L94) [CMakeLists.txt431-443](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L431-L443)

### Hexagon DSP Configuration

Qualcomm Hexagon DSP requires special configuration for both host-side compilation and target execution:

**Hexagon Configuration Example:**

The build system detects Hexagon target compilation through a compile test [CMakeLists.txt237-255](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L237-L255) that checks for the `__hexagon__` preprocessor macro. When building for Hexagon:
