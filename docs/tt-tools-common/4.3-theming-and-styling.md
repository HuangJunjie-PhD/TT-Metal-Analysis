# Theming and Styling

Relevant source files
*   [tt_tools_common/ui_common/common_style.css](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css)

## Purpose and Scope

This page documents the CSS-based theming system used by tt-tools-common's Textual UI components. It explains the color variables, global style rules, and widget-specific styling classes defined in the shared stylesheet. For documentation of the actual widget components and their functionality, see [Widget Library Reference](https://deepwiki.com/tenstorrent/tt-tools-common/4.1-widget-library-reference). For information on modal screen components, see [Modal Screens and Dialogs](https://deepwiki.com/tenstorrent/tt-tools-common/4.2-modal-screens-and-dialogs).

The theming system provides a consistent visual identity across all TT-Tools applications through centralized CSS definitions that leverage Textual's CSS engine.

* * *

## CSS Theming Architecture

The theming system uses a single centralized CSS file that applications load to apply consistent styling across all widgets. The architecture separates presentation (CSS) from widget logic (Python classes).

**Diagram: CSS Theming System Architecture**

Applications load the CSS file through Textual's `CSS_PATH` or `THEME` class variables. The CSS engine processes variable definitions, selectors, and rules to apply styles to widget instances at runtime.

**Sources:**[tt_tools_common/ui_common/common_style.css 1-226](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L1-L226)

## Color Variable Definitions

The theme defines six primary color variables that establish the Tenstorrent visual identity. These variables are referenced throughout the stylesheet using CSS variable syntax.

| Variable | Hex Value | Usage Context |
| --- | --- | --- |
| `$red` | `#f04f5e` | Error states, warnings, hover effects |
| `$green` | `#1eb57d` | Primary accent color, borders, success states |
| `$yellow` | `#ffd10a` | Text color, highlights, scrollbars, key labels |
| `$purple` | `#786bb0` | Reserved for specialized use cases |
| `$orange` | `darkorange` | Warning messages, compatibility notes |
| `$gray` | `ansi_bright_black` | Placeholder text, disabled states |

## Global Style Rules

The universal selector (`*`) defines default properties inherited by all widgets unless explicitly overridden. These establish baseline theming across the entire application.

### Global Properties

`* {  color: $yellow;  scrollbar-color: $yellow;  scrollbar-color-active: white;  scrollbar-color-hover: white;  scrollbar-background: $green;  scrollbar-background-active: $green;  scrollbar-background-hover: $green;  border-title-align: left;  border-title-color: $yellow;  border-title-style: bold;}`
| Property | Value | Effect |
| --- | --- | --- |
| `color` | `$yellow` | Default text color for all widgets |
| `scrollbar-color` | `$yellow` | Default scrollbar thumb color |
| `scrollbar-color-active` | `white` | Scrollbar thumb when clicked |
| `scrollbar-color-hover` | `white` | Scrollbar thumb on hover |
| `scrollbar-background` | `$green` | Scrollbar track background |
| `border-title-align` | `left` | Border title positioning |
| `border-title-color` | `$yellow` | Color for all border titles |
| `border-title-style` | `bold` | Bold styling for border titles |

**Sources:**[tt_tools_common/ui_common/common_style.css 15-26](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L15-L26)

## Header and Footer Styling

The `TTHeader` and `TTFooter` widgets share common styling with slight positioning differences. Both widgets have fixed height and bold text styling with green outlines.

**Diagram: Header and Footer Style Application**

### CSS Rules

Both widgets use shared rules defined at [tt_tools_common/ui_common/common_style.css 30-43](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L30-L43):

```
TTHeader,
TTFooter {
  height: 3;
  text-style: bold;
  outline: solid $green;
}

TTHeader {
  dock: top;
}

TTFooter {
  dock: bottom;
}
```

The `dock` property determines vertical positioning, while `height: 3` allocates three terminal rows for content display.

**Sources:**[tt_tools_common/ui_common/common_style.css 28-43](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L28-L43)[tt_tools_common/ui_common/widgets.py 22-62](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L22-L62)

## Menu Widget Styling

Three menu variants (`TTMenu`, `TTCompatibilityMenu`, `TTHostCompatibilityMenu`) share a common base style defined collectively in the CSS.

### Shared Menu Properties

```

TTCompatibilityMenu,
TTHostCompatibilityMenu {
  padding: 1 1;
  border: solid $green;
  color: $green;
}
```

These widgets display key-value information with consistent padding and border styling. Individual menu widgets apply programmatic styling for specific use cases:

| Widget Class | Programmatic Style Features | Location |

### TTSettingsMenu Styling

The settings menu uses `Switch` widgets with custom slider styling:

```
Switch>.switch--slider {
  color: $red;
  background: red 12%;
}

Switch.-on>.switch--slider {
  color: $green;
  background: $green 15%;
}
```

The `.switch--slider` element changes color based on the switch state:

*   **Off state**: Red color with 12% red background
*   **On state** (with `-on` modifier): Green color with 15% green background

This provides clear visual feedback for boolean settings like dark mode toggles.

### Button States

Buttons use a three-state color system:

| State | Background | Text Color | Text Style | Trigger |
| --- | --- | --- | --- | --- |
| Default | `$yellow` | `white` | normal | Base state |
| Hover | `$red` | `white` | `bold` | Mouse over |

The hover and focus states share styling to provide consistent feedback regardless of input method. Defined at [tt_tools_common/ui_common/common_style.css 127-138](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L127-L138)

## Style Application Flow

The following diagram shows how styles are resolved and applied to widget instances at runtime:

**Diagram: Style Resolution and Application Process**

### Precedence Rules

Styles are applied in order of increasing specificity:

1.   **Global styles** (`*` selector) - Base defaults
2.   **Widget type styles** (e.g., `TTHeader`, `DataTable`) - Widget-specific rules
3.   **ID-based styles** (e.g., `#dialog`, `#question`) - Instance-specific rules
4.   **Programmatic styles** (e.g., `self.styles.border_title_color = "red"`) - Runtime overrides

Programmatic styles set via `self.styles` in widget code have highest precedence and override CSS definitions. For example, `TTHostCompatibilityMenu` sets `self.styles.border_title_color = "red"` at [widgets.py 173](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/widgets.py#L173-L173) to override the default yellow from the global styles.

**Sources:**[tt_tools_common/ui_common/common_style.css 1-226](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L1-L226)[tt_tools_common/ui_common/widgets.py 1-365](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L1-L365)

## Customization Guide

### Loading Custom CSS

Applications can extend or override the default theme by loading additional CSS files:

`from textual.app import App class MyApp(App):    CSS_PATH = [        "tt_tools_common/ui_common/common_style.css",        "my_custom_styles.css"    ]`
Later files in the `CSS_PATH` list override earlier definitions due to CSS cascade rules.

### Overriding Color Variables

Define new color variable values in a custom CSS file:

`/* my_custom_styles.css */$red: #ff0000;$green: #00ff00;`
This redefines the color palette while preserving all existing style rules that reference these variables.

### Widget-Specific Overrides

Target specific widget instances by ID or add new widget type rules:

`/* Custom TTHeader styling */TTHeader {  height: 5;  outline: solid $purple;} /* Specific widget instance */#my_custom_table {  border: solid $red;  color: $red;}`
### Programmatic Style Modification

Widgets can apply dynamic styles in their code:

`class MyWidget(Container):    def on_mount(self):        if self.some_condition:            self.styles.border = ("solid", "$red")            self.border_title = "Error State"            self.styles.border_title_color = "red"`
This approach is useful for state-dependent styling, as seen in `TTHostCompatibilityMenu` at [widgets.py 168-176](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/widgets.py#L168-L176) where border colors change based on compatibility status.

**Sources:**[tt_tools_common/ui_common/widgets.py 168-176](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L168-L176)
