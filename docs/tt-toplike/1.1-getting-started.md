# Getting Started

The `tt-toplike` utility is a high-performance telemetry visualizer for Tenstorrent hardware, providing real-time insights into power, temperature, and NoC (Network-on-Chip) traffic through both Terminal (TUI) and Graphical (GUI) interfaces. This page covers the installation process, backend selection logic, and core usage patterns for new users.

### System Requirements

*   **Rust Toolchain**: Version 1.75 or later is required, though 1.93.0 is used in the official toolchain configuration [rust-toolchain.toml 1-4](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/rust-toolchain.toml#L1-L4)
*   **Operating System**: Linux (required for `sysfs` and `linux-procfs` features).
*   **Hardware**: Tenstorrent Grayskull or Wormhole-based devices (optional if using the Mock backend).
*   **Dependencies**: 
    *   `tt-smi`: Required if using the `json` backend [QUICK_START.md 64-69](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L64-L69)
    *   `ttkmd`: Tenstorrent kernel driver, required for `sysfs` and `luwen` backends [CONTRIBUTING.md 57-59](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/CONTRIBUTING.md?plain=1#L57-L59)

## Installation

The project provides multiple binaries targeting different environments. You can install via the provided script or manually via `cargo`.

### Quick Install Script

The `install.sh` script automates the build and installation of both the TUI and the `egui`-based GUI to `~/.local/bin/`.

`./install.sh`
**Sources**: [install.sh 1-31](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/install.sh#L1-L31)

### Manual Cargo Installation

If you prefer fine-grained control over features, use `cargo install`. Note that `--all-features` is not recommended due to hardware-specific dependencies [CONTRIBUTING.md 65-67](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/CONTRIBUTING.md?plain=1#L65-L67)

| Target Binary | Recommended Command | Key Features |
| --- | --- | --- |
| **TUI** | `cargo install --path . --bin tt-toplike-tui --features tui,json-backend,linux-procfs` | Standard terminal interface [CONTRIBUTING.md 52](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/CONTRIBUTING.md?plain=1#L52-L52) |
| **GUI** | `cargo install --path . --bin tt-toplike-egui --features egui` | Lightweight dashboard [install.sh 30](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/install.sh#L30-L30) |
| **Native App** | `cargo install --path . --bin tt-toplike-app --features app,json-backend` | PTY-hosted TUI in a native window [QUICK_START.md 146-148](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L146-L148) |

## Backend Selection

`tt-toplike` abstracts hardware access through several backends. By default, the application uses an **Auto-detect** sequence: **Sysfs → JSON → Mock**[QUICK_START.md 54-55](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L54-L55)

### Backend Types

| Type | Command Flag | Characteristics |
| --- | --- | --- |
| **Sysfs** | `--backend sysfs` | **Safe/Non-invasive.** Reads from `/sys/class/hwmon/`. Ideal for use during active workloads (LLM inference) [QUICK_START.md 57-62](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L57-L62) |
| **JSON** | `--backend json` | **Subprocess-based.** Spawns `tt-smi -s` and parses output. Requires `tt-smi` to be in `$PATH`[QUICK_START.md 64-69](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L64-L69) |
| **Luwen** | `--backend luwen` | **Direct PCI.** High-speed BAR0 access. **Warning:** May disrupt running workloads; never auto-detected [QUICK_START.md 77-82](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L77-L82) |
| **Mock** | `--mock` | **Simulation.** Generates synthetic data. Use `--mock-devices [N]` to simulate multi-chip systems [QUICK_START.md 71-75](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L71-L75) |

**Sources**: [QUICK_START.md 52-83](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L52-L83)

## Quick Start Commands

Launch into specific visualization modes directly from the CLI:

`# Unified view with Starfield, Castle, and Flowtt-toplike-tui --mode arcade # Monitor 4 simulated devices (useful for testing UI scaling)tt-toplike-tui --mock --mock-devices 4 # High-frequency updates (50ms) using the safe sysfs backendtt-toplike-tui --backend sysfs --interval 50 # Monitor specific physical devices onlytt-toplike-tui --devices 0,2`
**Sources**: [QUICK_START.md 88-95](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L88-L95)

## TUI Keyboard Shortcuts

Once the TUI is running, use the following keys to interact with the display:

| Key | Action |
| --- | --- |
| `v` | **Cycle Visualizations**: Switch between Normal, Arcade, Starfield, Castle, and Flow [QUICK_START.md 103](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L103-L103) |
| `b` | **Cycle Backends**: Hot-swap the telemetry source (Sysfs → JSON → Luwen → Mock) [QUICK_START.md 104](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L104-L104) |
| `r` | **Force Refresh**: Clear and redraw the UI [QUICK_START.md 106](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L106-L106) |
| `q` / `ESC` | **Quit**: Gracefully exit the application [QUICK_START.md 105](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L105-L105) |

## Troubleshooting

### Terminal Color Issues

If you see grey boxes or incorrect colors while using SSH or `tmux`:

*   `tt-toplike` uses `Color::Reset` for backgrounds to prevent "grey box" artifacts [QUICK_START.md 128](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L128-L128)
*   The TUI includes a 256-color fallback specifically for `tmux` on macOS Terminal.app [install.sh 60-63](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/install.sh#L60-L63)
*   Ensure your `COLORTERM` environment variable is set to `truecolor` for the best experience.

### Device Discovery

If no hardware is detected:

1.   Verify the kernel driver is exposing metrics: `ls /sys/class/hwmon/`[QUICK_START.md 123](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L123-L123)
2.   Check if `tt-smi` can see the devices.
3.   Run with `--mock --mock-devices 2` to confirm the binary and your terminal's rendering capabilities are functional [QUICK_START.md 124](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/QUICK_START.md?plain=1#L124-L124)
