# Command Line Options

Relevant source files
*   [README.md](https://github.com/tenstorrent/cosim-arch-checker/blob/157c6637/README.md?plain=1)
*   [bridge/bridge.cc](https://github.com/tenstorrent/cosim-arch-checker/blob/157c6637/bridge/bridge.cc)

This page provides a comprehensive reference for all command-line options, build flags, and runtime parameters used in the cosim-arch-checker framework. These options control build configuration, Whisper ISS integration, architectural checking behavior, and debug output.

For configuration of DUT-specific parameters like hart count and register widths, see [DUT Parameters](https://deepwiki.com/tenstorrent/cosim-arch-checker/4.2-dut-parameters). For Whisper ISS configuration files, see [Whisper Configuration](https://deepwiki.com/tenstorrent/cosim-arch-checker/4.1-whisper-configuration).

## Option Categories

The cosim-arch-checker accepts options at three different stages:

1.   **Build-time options** - Bazel build flags and preprocessor definitions
2.   **Runtime simulation flags** - Plusargs passed to the SystemVerilog simulator
3.   **Environment variables** - Used during build and runtime

## Build-time Options

### Bazel Build Flags

These options are passed to the `bazel build :dpi` command to configure the build:

| Flag | Purpose | Example |
| --- | --- | --- |
| `--copt=-DCONFIG=<config>` | Sets DUT configuration | `--copt=-DCONFIG=MediumBoomVecConfig` |
| `WHISPER_PATH=<path>` | Path to Whisper executable | `WHISPER_PATH=/opt/whisper/bin/whisper` |
| `WHISPER_JSON=<path>` | Path to Whisper config JSON | `WHISPER_JSON=bridge/whisper/config/ocelot.json` |
| `TESTFILE=<path>` | Path to test ELF binary | `TESTFILE=/path/to/test.elf` |
| `BOOTCODE=<path>` | Path to boot ROM binary | `BOOTCODE=bootrom/bootrom` |

## Runtime Simulation Flags

These plusargs are passed to the SystemVerilog simulator and parsed by the bridge system:

### Core Control Options

| Flag | Type | Default | Description |
| --- | --- | --- | --- |
| `+cosim` | Boolean | OFF | Enable co-simulation mode |
| `+testfile=<path>` | String | - | Override test ELF file path |
| `+bootcode=<path>` | String | - | Override boot ROM path |
| `+whisper_path=<path>` | String | - | Override Whisper executable path |
| `+whisper_json_path=<path>` | String | - | Override Whisper config JSON path |

### Debugging and Tracing Options

| Flag | Type | Default | Description |
| --- | --- | --- | --- |
| `+bridge_tracer` | Boolean | ON | Enable bridge debug output |
| `+harness_tracer` | Boolean | OFF | Enable harness debug tracing |
| `+dutmon_tracer` | Boolean | OFF | Enable DUT monitor tracing |

### Architecture Checking Control

| Flag | Type | Default | Description |
| --- | --- | --- | --- |
| `+arch_checks_exclude=<list>` | String | - | Exclude specific architectural checks |
| `+arch_checks_instr_exclude=<list>` | String | - | Skip checks for specific instructions |

## Implementation Details

Command line parsing is implemented in the `cBridge` constructor and `parseArgs()` method:

*   **VPI Interface**: Uses `vpi_get_vlog_info()` to access SystemVerilog simulator arguments
*   **String Parsing**: Simple substring matching with `std::string::find()`
*   **Default Values**: Built using preprocessor definitions from build-time options
*   **Error Handling**: Missing required options may cause Whisper connection failures

The parsed options are stored as private member variables in the `cBridge` class and used throughout the co-simulation process.
