# TUI Navigation and Layout

Relevant source files
*   [tt_smi/constants.py](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/constants.py)
*   [tt_smi/tt_smi.py](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py)

This document describes the interactive Text User Interface (TUI) layout structure, navigation mechanisms, and keyboard controls implemented in the `TTSMI` Textual application. For details about the data displayed in each tab, see [Device Information Tab](https://deepwiki.com/tenstorrent/tt-smi/3.2-device-information-tab), [Telemetry Tab](https://deepwiki.com/tenstorrent/tt-smi/3.3-telemetry-tab), and [Firmware Version Tab](https://deepwiki.com/tenstorrent/tt-smi/3.4-firmware-version-tab). For the underlying data collection architecture, see [Application Layer Architecture](https://deepwiki.com/tenstorrent/tt-smi/6.1-application-layer-architecture).

## Purpose and Scope

The TT-SMI TUI provides an interactive terminal interface for monitoring Tenstorrent devices using the Textual framework. This page documents the structural composition of the interface, including the three-column grid layout, sidebar visibility controls, tabbed content navigation, and the complete keyboard shortcut system.

* * *

## Application Layout Architecture

The `TTSMI` class extends `textual.app.App` and implements a grid-based layout with three primary regions: a collapsible left sidebar, a central tabbed content area, and a footer.

**Sources:**[tt_smi/tt_smi.py 114-146](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L114-L146)[tt_smi/tt_smi_style.css 8-21](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_style.css#L8-L21)

The layout is defined in the `compose()` method, which yields widgets in a hierarchical structure. The grid container uses CSS grid layout with gutter spacing defined in `tt_smi_style.css`.

| Component | Widget Type | ID | Purpose |
| --- | --- | --- | --- |
| Header | `TTHeader` | (none) | Displays app name and version |
| Sidebar | `Vertical` | `left_col` | Contains host compatibility information |
| Host Info | `TTHostCompatibilityMenu` | `host_info` | Shows driver version, OS, kernel info |
| Tab Container | `TabbedContent` | `tab_container` | Houses three data tables |
| Device Info | `TTDataTable` | `tt_smi_device_info` | Static device information |
| Telemetry | `TTDataTable` | `tt_smi_telem` | Real-time telemetry data |
| Firmware | `TTDataTable` | `tt_smi_firmware` | Firmware version information |
| Footer | `Footer` | (none) | Displays available keyboard shortcuts |

**Sources:**[tt_smi/tt_smi.py 114-146](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L114-L146)

## Grid Layout Configuration

The application uses CSS Grid with specific column sizing and docking behavior:

**Sources:**[tt_smi/tt_smi_style.css 8-21](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_style.css#L8-L21)

The left column has a fixed width of 40 characters and is docked to the left side. The tab container takes the remaining space and enables automatic scrolling in both directions. The grid gutter of `1 1` provides spacing between columns.

## Tab Container and Navigation

The `TabbedContent` widget contains three tabs, each with a `TTDataTable` displaying different categories of device information.

### Tab Structure

| Tab Index | Label | Tab ID | Table ID | Data Source Method |
| --- | --- | --- | --- | --- |
| 1 | "Information (1)" | `tab-1` | `tt_smi_device_info` | `format_device_info_rows()` |
| 2 | "Telemetry (2)" | `tab-2` | `tt_smi_telem` | `format_telemetry_rows()` |
| 3 | "FW Version (3)" | `tab-3` | `tt_smi_firmware` | `format_firmware_rows()` |

**Sources:**[tt_smi/tt_smi.py 125-145](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L125-L145)

### Tab Configuration

Each `TTDataTable` is initialized with:

*   **title**: Display title for the table
*   **id**: Unique identifier for widget queries
*   **header**: Column headers from `constants.py`
*   **header_height**: Set to 2 rows for all tables

The headers are defined as:

*   `INFO_TABLE_HEADER`: 9 columns including Bus ID, Board Type, DRAM status, PCIe properties
*   `TELEMETRY_TABLE_HEADER`: 8 columns including voltage, current, power, temperature
*   `FIRMWARES_TABLE_HEADER`: 8 columns for various firmware versions

**Sources:**[tt_smi/tt_smi.py 128-145](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L128-L145)[tt_smi/constants.py 166-198](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/constants.py#L166-L198)

### Tab Population

**Sources:**[tt_smi/tt_smi.py 148-164](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L148-L164)

All tables have their cursor type set to `"none"` to disable row selection, as the interface is read-only. The `DataTable` widgets are populated during the `on_mount()` lifecycle event.

## Action Handler Implementation

### Tab Switching Actions

Tab switching is implemented by setting the `active` property on the `TabbedContent` widget:

**Sources:**[tt_smi/tt_smi.py 528-541](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L528-L541)[tt_smi/tt_smi.py 568-574](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L568-L574)

The action methods at [tt_smi/tt_smi.py 528-541](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L528-L541) set the active tab by ID. Textual automatically generates tab IDs as `tab-1`, `tab-2`, `tab-3` for the three tabs in the `TabbedContent` widget.

### Telemetry Thread Dispatch

When the Telemetry tab (`tab-2`) is activated, a background thread is automatically spawned to update telemetry data:

**Sources:**[tt_smi/tt_smi.py 550-574](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L550-L574)

The thread update interval is defined in `constants.GUI_INTERVAL_TIME` as `0.1` seconds (100ms), providing near-real-time telemetry updates.

**Sources:**[tt_smi/constants.py 159](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/constants.py#L159-L159)

### Quit Action with Cleanup

The `action_quit()` method at [tt_smi/tt_smi.py 514-526](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L514-L526) performs proper cleanup:

1.   Sets the global `INTERRUPT_RECEIVED` flag to `True`
2.   Waits for all telemetry threads to complete (0.1s timeout per thread)
3.   Optionally saves snapshot logs if `self.snapshot` is enabled
4.   Exits the application with a formatted exit message

**Sources:**[tt_smi/tt_smi.py 514-526](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L514-L526)

### Help Menu Display

The `action_help()` method at [tt_smi/tt_smi.py 543-548](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L543-L548) creates a `TTHelperMenuBox` widget and pushes it as a modal screen overlay. The help content is defined in `constants.HELP_MENU_MARKDOWN`, which includes a table of all keyboard shortcuts.

## CSS Styling and Layout Rules

The TUI appearance is controlled by two CSS files loaded in order:

1.   `common_style.css` from `tt_tools_common.ui_common` package
2.   `tt_smi_style.css` from the TT-SMI package

**Sources:**[tt_smi/tt_smi.py 80-89](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L80-L89)

### Key CSS Rules

| Selector | Properties | Purpose |
| --- | --- | --- |
| `#app_grid` | `layout: grid; grid-gutter: 1 1;` | Defines grid container with spacing |
| `#left_col` | `width: 40; column-span: 1; dock: left` | Fixed-width sidebar docked left |
| `#tab_container` | `overflow: auto auto` | Enables scrolling in tab area |
| `TabPane` | `width: 100%; overflow: auto hidden` | Full-width tabs with horizontal scroll |
| `DataTable` | `text-align: center` | Centers all table cell content |
| `DataTable > .datatable--header-hover` | `background: 0%; color: white 0%` | Disables header hover effects |

**Sources:**[tt_smi/tt_smi_style.css 1-113](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_style.css#L1-L113)

The CSS disables header hover effects on the `DataTable` widgets to maintain a static appearance, as the tables are read-only displays.

## Global State Management

The TUI uses several global variables to manage application state across threads:

| Variable | Type | Purpose | Modified By |
| --- | --- | --- | --- |
| `INTERRUPT_RECEIVED` | `bool` | Signals threads to stop | `interrupt_handler()`, `action_quit()` |
| `TELEM_THREADS` | `List` | Stores telemetry thread references | `dispatch_telem_thread()` |
| `RUNNING_TELEM_THREAD` | `bool` | Prevents duplicate telemetry threads | `dispatch_telem_thread()` |

**Sources:**[tt_smi/tt_smi.py 54-58](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L54-L58)[tt_smi/tt_smi.py 61-64](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L61-L64)[tt_smi/tt_smi.py 550-566](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L550-L566)

Signal handlers are registered at [tt_smi/tt_smi.py 697-698](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L697-L698) to handle `SIGINT` (Ctrl+C) and `SIGTERM`, ensuring graceful shutdown by setting `INTERRUPT_RECEIVED = True`.

## Summary

The TT-SMI TUI implements a three-region layout using Textual's grid system:

*   **Left sidebar** (`#left_col`): 40-character width, toggleable via `c` key
*   **Central tab container** (`#tab_container`): Three tabs switchable via `1`, `2`, `3` keys
*   **Footer**: Displays available keyboard shortcuts

Navigation is controlled through seven keyboard bindings mapped to action methods, with special handling for the Telemetry tab which spawns a background update thread. The layout is styled via CSS with fixed widths, automatic overflow, and centered table content.
