# System Management Interface (tt-smi)

## Purpose and Overview

The System Management Interface (tt-smi) is a command-line utility designed to monitor and manage Tenstorrent hardware. It provides a straightforward way to check device status, view hardware information, and monitor performance metrics of Tenstorrent accelerators. This tool is essential for confirming successful installation of the Tenstorrent software stack and for ongoing health monitoring of your hardware.

For information about updating firmware, see [Firmware and tt-flash](https://deepwiki.com/tenstorrent/tt-installer/5.2-firmware-and-tt-flash), and for details about the programming environment, see [tt-metalium Container](https://deepwiki.com/tenstorrent/tt-installer/5.5-tt-metalium-container).

## Installation

The tt-smi utility is automatically installed during the Tenstorrent software stack installation process. It's installed as a Python package directly from the GitHub repository:

```
${PYTHON_INSTALL_CMD} git+https://github.com/tenstorrent/tt-smi
```

The installation command varies based on the Python environment configuration chosen during the installation process:

*   In a virtual environment (new or existing): `pip install git+https://github.com/tenstorrent/tt-smi`
*   Using system Python: `pip install --break-system-packages git+https://github.com/tenstorrent/tt-smi`
*   Using pipx: `pipx install git+https://github.com/tenstorrent/tt-smi`

After installation, the `tt-smi` command becomes available (either directly or within the configured Python environment).

## System Integration

tt-smi connects directly to the Tenstorrent hardware through the kernel-mode driver (KMD). It serves as one of the primary system tools for interacting with the hardware.

## Basic Usage

After installation and system reboot, you can run the tt-smi command to verify that your Tenstorrent hardware is properly recognized and functioning:

```
tt-smi
```

If you installed tt-smi in a Python virtual environment, you'll need to activate that environment first:

```
source /path/to/venv/bin/activate
tt-smi
```

The tt-smi interface typically displays information such as:

*   Device enumeration and identification
*   Hardware status
*   Temperature and power metrics
*   Memory utilization
*   Device health information

## Usage Scenarios

The System Management Interface serves several key purposes:

| Scenario | Description |
| --- | --- |
| Installation Verification | Confirm that Tenstorrent hardware is recognized and functional after installing the software stack |
| Health Monitoring | Check device temperature, power usage, and other health metrics during operation |
| System Diagnostics | Troubleshoot hardware issues and verify device configuration |
| Performance Tracking | Monitor resource utilization during workload execution |
| System Administration | Manage multiple Tenstorrent devices in a single system |
