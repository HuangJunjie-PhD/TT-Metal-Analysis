# Local Documentation Server

Relevant source files
*   [docs/README.md](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/README.md?plain=1)
*   [docs/build.py](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py)

This document describes the local HTTP server functionality provided by the `docs/build.py` script for viewing documentation during development. The local server allows developers to preview generated HTML documentation in a web browser without deploying to GitHub Pages.

For information about the `DocBuilder` class architecture and build options, see [DocBuilder Class](https://deepwiki.com/tenstorrent/riescue/6.1.1-docbuilder-class). For general documentation structure and contribution guidelines, see [Writing Documentation](https://deepwiki.com/tenstorrent/riescue/6.2.2-writing-documentation).

## Overview

The local documentation server provides two modes for viewing built documentation:

| Mode | Flag | Description |
| --- | --- | --- |
| Basic | `--local_host` | Serves documentation once; requires CTRL+C to stop |
| Interactive | `--interactive` or `-i` | Allows STOP and RELOAD commands for iterative development |

Both modes automatically clean the build directory before building (equivalent to `--clean`) and start a Python HTTP server to serve the generated HTML files.

**Sources:**[docs/build.py 46-52](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L46-L52)[docs/build.py 70-73](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L70-L73)[docs/README.md 27-46](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/README.md?plain=1#L27-L46)

## Interactive Mode

### Usage

`./docs/build.py --interactive# or./docs/build.py -i`
Interactive mode provides commands for managing the server without restarting the script:

| Command | Alias | Action |
| --- | --- | --- |
| `stop` | `s` | Stop the server and exit |
| `reload` | `r` | Rebuild documentation and restart server |
| CTRL+C | - | Stop the server and exit |

**Sources:**[docs/build.py 71](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L71-L71)[docs/build.py 124-177](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L124-L177)

### Interactive Mode Workflow

**Diagram: Interactive Mode Control Flow**

The interactive mode uses a `threading.Event` object [docs/build.py 136](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L136-L136) to coordinate between the main thread (which reads user input) and the server thread (which serves HTTP requests). The server thread is marked as `daemon=True`[docs/build.py 140](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L140-L140) so it automatically terminates when the main thread exits.

## Configuration Options

### Port Configuration

Default port is `8888`. Override with `--port`:

`./docs/build.py --local_host --port 9000`
The port number is stored in `self.port`[docs/build.py 47](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L47-L47) and passed to `socketserver.TCPServer`[docs/build.py 204](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L204-L204)

**Sources:**[docs/build.py 72](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L72-L72)[docs/build.py 47](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L47-L47)[docs/build.py 204](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L204-L204)

### Host Configuration

Default host is `0.0.0.0` (binds to all interfaces). Override with `--host`:

`./docs/build.py --local_host --host 127.0.0.1`
The host configuration [docs/build.py 48](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L48-L48) determines which network interface the server listens on:

| Host Value | Behavior |
| --- | --- |
| `0.0.0.0` | Listen on all interfaces (default) |
| `127.0.0.1` | Listen on localhost only |
| Specific IP | Listen on specific interface |

When displaying the URL, `0.0.0.0` is converted to `localhost` for user convenience [docs/build.py 127-128](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L127-L128)[docs/build.py 182-183](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L182-L183)

## Error Handling

### Address Reuse

The custom `ReusableTCPServer` class [docs/build.py 193-194](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L193-L194) sets `allow_reuse_address = True` to prevent "Address already in use" errors. This is based on a Stack Overflow solution for allowing quick restarts of the server without waiting for the OS to release the port.

**Sources:**[docs/build.py 192-195](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L192-L195)

### Keyboard Interrupt Handling

The server distinguishes between interactive and non-interactive modes when handling `KeyboardInterrupt`[docs/build.py 209-211](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L209-L211):

*   **Non-interactive mode**: Prints "Local host server stopped" and exits cleanly
*   **Interactive mode**: Silently stops the server (message handled by main thread)

**Sources:**[docs/build.py 209-216](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L209-L216)

### Exception Propagation

Exceptions other than `KeyboardInterrupt` are handled differently based on mode [docs/build.py 212-216](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L212-L216):

*   **Interactive mode with stop_event**: Print error message if event not set
*   **Non-interactive mode**: Re-raise the exception

**Sources:**[docs/build.py 212-216](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/build.py#L212-L216)
