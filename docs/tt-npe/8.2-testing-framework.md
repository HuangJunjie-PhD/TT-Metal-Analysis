## Purpose and Scope

This document covers the testing infrastructure for tt-npe, including unit tests, integration tests, test execution scripts, and test data management. The testing framework ensures code quality and prevents regressions across both C++ core components and Python bindings.

For information about the build system that compiles the tests, see [Build System](https://deepwiki.com/tenstorrent/tt-npe/5.1-device-model-interface). For details about CI/CD workflows that execute tests automatically, see [CI/CD and Workflows](https://deepwiki.com/tenstorrent/tt-npe/5.4-multi-chip-device-models).

## Adding New Tests

### C++ Test Addition

To add new C++ tests:

1.   **Create Test File**: Follow naming convention `test_<component>.cpp`
2.   **Include Headers**: Include `gtest/gtest.h` and component headers
3.   **Define Test Class**: Use namespace `tt_npe` and descriptive class names
4.   **Test Structure**: Use `TEST(TestClass, TestName)` macro format

Example test structure:

`#include "gtest/gtest.h"#include "your_component.hpp" namespace tt_npe {    TEST(YourComponentTest, CanDoSomething) {        // Test implementation        EXPECT_TRUE(condition);    }}`
### Python Test Addition

For Python tests:

1.   **File Naming**: Use `test_*.py` pattern in `py/pytest/` directory
2.   **Import Bindings**: Import `tt_npe_pybind as npe`
3.   **Function Naming**: Use `test_*` function prefix
4.   **Assertions**: Use standard pytest assertions

Example Python test:

`import tt_npe_pybind as npe def test_new_functionality():    config = npe.Config()    assert config is not None`
