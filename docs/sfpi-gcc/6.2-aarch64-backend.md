This document covers the AArch64 backend implementation in GCC, which provides compilation support for ARM's 64-bit architecture. The backend handles instruction selection, register allocation, optimization passes, and intrinsic functions specific to AArch64 processors including support for Advanced SIMD (NEON), Scalable Vector Extensions (SVE), and various architectural features.

For information about RISC-V SFPU extensions, see [RISC-V SFPU Core](https://deepwiki.com/tenstorrent/sfpi-gcc/2.1-risc-v-sfpu-core). For general GCC optimization systems, see [Analysis and Optimization](https://deepwiki.com/tenstorrent/sfpi-gcc/5-analysis-and-optimization).

## Core Backend Implementation

The core backend functionality is implemented primarily in `aarch64.cc`, which contains the main target hooks and backend logic.

### Key Backend Components

Sources: [gcc/config/aarch64/aarch64.cc 297-308](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64.cc#L297-L308)[gcc/config/aarch64/aarch64-protos.h 1-50](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-protos.h#L1-L50)

The backend defines register classes, addressing modes, and instruction costs specific to AArch64 processors. Key data structures include `aarch64_frame` for stack frame layout and `simd_immediate_info` for SIMD constant handling.

## NEON/SIMD Support System

The AArch64 backend provides comprehensive support for Advanced SIMD (NEON) instructions through intrinsics and machine descriptions.

### NEON Intrinsics Architecture

Sources: [gcc/config/aarch64/arm_neon.h 40-80](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/arm_neon.h#L40-L80)[gcc/config/aarch64/aarch64-simd-builtins.def 46-60](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-simd-builtins.def#L46-L60)[gcc/config/aarch64/aarch64-simd.md 21-100](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-simd.md?plain=1#L21-L100)

The NEON system provides vector types like `int8x8_t`, `float32x4_t` and hundreds of intrinsic functions for vector operations. The `arm_neon.h` header defines these intrinsics, which map to built-in functions that are expanded using RTL patterns in `aarch64-simd.md`.

### Key NEON Instruction Categories

| Category | Examples | Pattern Prefix |
| --- | --- | --- |
| Vector Arithmetic | `vadd_s32`, `vmul_f32` | `add<mode>3`, `mul<mode>3` |
| Vector Logic | `vand_s32`, `vorr_u16` | `and<mode>3`, `ior<mode>3` |
| Vector Shifts | `vshl_n_s32`, `vshr_n_u16` | `ashl<mode>3`, `lshr<mode>3` |
| Vector Loads/Stores | `vld1_s32`, `vst1_f32` | `aarch64_ld1<mode>`, `aarch64_st1<mode>` |
| Vector Permute | `vext_s32`, `vrev32_s16` | `aarch64_ext<mode>`, `aarch64_rev<mode>` |

Sources: [gcc/config/aarch64/aarch64-simd.md 383-408](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-simd.md?plain=1#L383-L408)[gcc/config/aarch64/arm_neon.h 235-375](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/arm_neon.h#L235-L375)

## SVE Support System

The Scalable Vector Extension (SVE) provides variable-length vector processing capabilities. The backend includes comprehensive SVE support through dedicated machine descriptions and built-ins.

### SVE Implementation Architecture

Sources: [gcc/config/aarch64/aarch64-sve.md 21-83](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-sve.md?plain=1#L21-L83)[gcc/config/aarch64/iterators.md 400-450](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/iterators.md?plain=1#L400-L450)

SVE introduces predicate registers (`p0`-`p15`) and scalable vector registers (`z0`-`z31`) with variable vector lengths. The backend supports predicated operations, gather/scatter memory access patterns, and first-faulting loads for efficient vector processing.

## Built-in Functions System

The AArch64 backend implements a comprehensive built-in function system that provides access to architecture-specific instructions and intrinsics.

### Built-in Function Architecture

Sources: [gcc/config/aarch64/aarch64-builtins.cc 51-100](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-builtins.cc#L51-L100)[gcc/config/aarch64/aarch64-simd-builtins.def 46-100](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-simd-builtins.def#L46-L100)

The built-in system uses macro-based definitions to generate hundreds of intrinsic functions. Each built-in has a type signature defined by iterator macros like `BUILTIN_VDQ_I` (vector double/quad integer) and operation flags like `BINOP` (binary operation).

### Built-in Function Processing Pipeline

| Stage | Function | Purpose |
| --- | --- | --- |
| Initialization | `aarch64_init_builtins` | Register built-in functions with GCC |
| Expansion | `aarch64_expand_builtin` | Convert built-in calls to RTL |
| Folding | `aarch64_fold_builtin` | Constant folding optimization |
| GIMPLE Folding | `aarch64_gimple_fold_builtin` | GIMPLE-level optimizations |
| Vectorization | `aarch64_builtin_vectorized_function` | Auto-vectorization support |

Sources: [gcc/config/aarch64/aarch64-builtins.cc 1200-1300](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-builtins.cc#L1200-L1300)

## Machine Description System

The machine description files define instruction patterns, constraints, and iterators that control instruction selection and RTL generation.

### Machine Description File Organization

Sources: [gcc/config/aarch64/aarch64.md 354-370](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64.md?plain=1#L354-L370)[gcc/config/aarch64/iterators.md 29-50](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/iterators.md?plain=1#L29-L50)[gcc/config/aarch64/constraints.md 21-50](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/constraints.md?plain=1#L21-L50)

The machine descriptions use mode iterators to generate multiple instruction patterns from single templates. For example, `GPI` (General Purpose Integer) covers both 32-bit and 64-bit operations, while `VDQ_I` covers various vector integer modes.

### Key Constraint Categories

| Constraint | Register Class | Usage |
| --- | --- | --- |
| `w` | `FP_REGS` | Floating point and SIMD registers |
| `Upl` | `PR_LO_REGS` | SVE predicate registers p0-p7 |
| `Upa` | `PR_REGS` | SVE predicate registers p0-p15 |
| `I` | Immediate | 12-bit unsigned immediate with optional shift |
| `m` | Memory | General memory operand |

Sources: [gcc/config/aarch64/constraints.md 36-50](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/constraints.md?plain=1#L36-L50)

## Configuration and Tuning System

The AArch64 backend includes extensive configuration files that define supported architectures, CPU cores, and optimization parameters.

### Configuration File Structure

Sources: [gcc/config/aarch64/aarch64-arches.def 20-25](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-arches.def#L20-L25)[gcc/config/aarch64/aarch64-option-extensions.def 1-50](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-option-extensions.def#L1-L50)[gcc/config/aarch64/aarch64-tuning-flags.def 16-25](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-tuning-flags.def#L16-L25)

The configuration system allows fine-grained control over which architectural features are enabled, how instruction costs are calculated, and how the compiler optimizes for specific CPU cores. The tuning system includes parameters for instruction scheduling, cost modeling, and pipeline-specific optimizations.

### Key Architectural Features

| Feature | Extension | Description |
| --- | --- | --- |
| Advanced SIMD | `+simd` | NEON vector instructions |
| Crypto | `+crypto` | AES, SHA, PMULL instructions |
| Floating Point 16 | `+fp16` | Half-precision floating point |
| SVE | `+sve` | Scalable Vector Extension |
| SVE2 | `+sve2` | SVE version 2 |
| Large System Extensions | `+lse` | Atomic instructions |
| Dot Product | `+dotprod` | 8-bit integer dot product |

Sources: [gcc/config/aarch64/aarch64-option-extensions.def 50-100](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/aarch64/aarch64-option-extensions.def#L50-L100)
