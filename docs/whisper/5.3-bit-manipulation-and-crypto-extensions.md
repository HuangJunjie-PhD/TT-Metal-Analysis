## Purpose and Scope

This page documents the RISC-V bit manipulation (Zb*) and cryptographic (Zk*) extensions as implemented in Whisper. These extensions provide specialized instructions for efficient manipulation of bits and cryptographic operations. Both scalar and vector variants are covered here. For general vector extension architecture, see [Vector Extension (V)](https://deepwiki.com/tenstorrent/whisper/5.1-vector-extension-(v)). For overall ISA configuration, see [Instruction Set Architecture](https://deepwiki.com/tenstorrent/whisper/2.3-instruction-set-architecture).

* * *

## Extension Overview

Whisper supports multiple ratified and draft bit manipulation and cryptographic extensions. These extensions add specialized instructions for operations such as bit counting, rotation, carry-less multiplication, AES encryption, and SHA hashing.

### Bit Manipulation Extensions

| Extension | Description | Instruction Count |
| --- | --- | --- |
| **Zba** | Address generation (shifted add) | 8 instructions |
| **Zbb** | Basic bit manipulation | 30+ instructions |
| **Zbc** | Carry-less multiplication | 3 instructions |
| **Zbs** | Single-bit operations | 8 instructions |
| **Zbkb** | Bit manipulation for cryptography | Subset of Zbb + pack/brev8 |
| **Zbkx** | Crossbar permutation | 2 instructions (xperm.n, xperm.b) |

### Cryptographic Extensions

| Extension | Description | Algorithm Support |
| --- | --- | --- |
| **Zknd** | AES decryption | AES-128/192/256 |
| **Zkne** | AES encryption | AES-128/192/256 |
| **Zknh** | SHA-2 hashing | SHA-256, SHA-512 |
| **Zksed** | SM4 block cipher | SM4 (Chinese standard) |
| **Zksh** | SM3 hashing | SM3 (Chinese standard) |
| **Zkr** | Entropy source | Random number generation |

### Vector Crypto Extensions

| Extension | Description | Base Requirement |
| --- | --- | --- |
| **Zvbb** | Vector basic bit manipulation | Requires V extension |
| **Zvbc** | Vector carry-less multiply | Requires V extension |
| **Zvkg** | Vector GCM/GMAC | Requires V extension |
| **Zvkned** | Vector AES encryption/decryption | Requires V extension |
| **Zvknha** | Vector SHA-256 (RV32) | Requires V extension |
| **Zvknhb** | Vector SHA-256/512 (RV64) | Requires V extension |
| **Zvksed** | Vector SM4 | Requires V extension |
| **Zvksh** | Vector SM3 | Requires V extension |
| **Zvkb** | Vector bit manipulation for crypto | Requires V extension |

## Extension Configuration Flow

**Extension Enablement Flow**

Extensions are enabled during hart initialization through `Hart::processExtensions()`. The process checks the MISA CSR bits and configuration, then calls `enableExtension()` for each supported extension.

**Sources:**[Hart.cpp 469-647](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L469-L647)

## Instruction Identification

The simulator marks bit manipulation instructions with a flag for efficient identification during decode and execution.

**Bit Manipulation Instruction Marking**

The `InstEntry` constructor checks if an instruction belongs to the bit manipulation extension family (Zba through Zbs) and sets the `isBitManip_` flag accordingly. This allows the decoder and execution logic to quickly identify bit manipulation instructions.

**Sources:**[InstEntry.cpp 40-41](https://github.com/tenstorrent/whisper/blob/733fd921/InstEntry.cpp#L40-L41)

## Scalar Bit Manipulation Instructions

### Zba: Address Generation

Zba provides shifted-add instructions commonly used in address computation for array indexing.

| Instruction | Encoding | Operation | Purpose |
| --- | --- | --- | --- |
| `sh1add` | RV32/64 | rd = rs1 << 1 + rs2 | Add shifted-left by 1 |
| `sh2add` | RV32/64 | rd = rs1 << 2 + rs2 | Add shifted-left by 2 |
| `sh3add` | RV32/64 | rd = rs1 << 3 + rs2 | Add shifted-left by 3 |
| `sh1add.uw` | RV64 | rd = (rs1[31:0] << 1) + rs2 | Unsigned word shift-add |
| `sh2add.uw` | RV64 | rd = (rs1[31:0] << 2) + rs2 | Unsigned word shift-add |
| `sh3add.uw` | RV64 | rd = (rs1[31:0] << 3) + rs2 | Unsigned word shift-add |
| `add.uw` | RV64 | rd = rs1[31:0] + rs2 | Zero-extend word add |
| `slli.uw` | RV64 | rd = rs1[31:0] << shamt | Zero-extend word shift |

**Sources:**[InstId.hpp 359-367](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L359-L367)

### Zbb: Basic Bit Manipulation

Zbb provides a comprehensive set of bit manipulation operations including counting, rotation, sign extension, and logical operations.

| Category | Instructions | Description |
| --- | --- | --- |
| **Count** | `clz`, `clzw`, `ctz`, `ctzw`, `cpop`, `cpopw` | Count leading/trailing zeros, population count |
| **Rotation** | `rol`, `rolw`, `ror`, `rori`, `roriw`, `rorw` | Rotate left/right |
| **Min/Max** | `min`, `minu`, `max`, `maxu` | Signed/unsigned minimum/maximum |
| **Sign Extend** | `sext.b`, `sext.h` | Sign extend byte/halfword |
| **Logic** | `andn`, `orn`, `xnor` | AND-NOT, OR-NOT, XOR-NOT |
| **Other** | `orc.b`, `rev8_32`, `rev8_64` | OR-combine bytes, byte reverse |

**Sources:**[InstId.hpp 308-330](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L308-L330)

### Zbc: Carry-less Multiplication

Zbc provides carry-less multiplication operations used in cryptography (e.g., GCM mode).

| Instruction | Operation | Usage |
| --- | --- | --- |
| `clmul` | Carry-less multiply (lower half) | CRC, GCM |
| `clmulh` | Carry-less multiply (upper half) | CRC, GCM |
| `clmulr` | Carry-less multiply (reversed) | CRC variants |

**Sources:**[InstId.hpp 354-357](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L354-L357)

### Zbs: Single-bit Operations

Zbs provides instructions for setting, clearing, inverting, and extracting individual bits.

| Instruction Type | Instructions | Operation |
| --- | --- | --- |
| **Register-Register** | `bset`, `bclr`, `binv`, `bext` | Bit indexed by register |
| **Immediate** | `bseti`, `bclri`, `binvi`, `bexti` | Bit indexed by immediate |

**Sources:**[InstId.hpp 344-352](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L344-L352)

### Zbkb/Zbkx: Crypto Bit Manipulation

These extensions provide bit operations specifically useful for cryptographic algorithms.

| Extension | Instructions | Purpose |
| --- | --- | --- |
| **Zbkb** | `pack`, `packh`, `packw`, `brev8`, `rev8_*`, `zip`, `unzip` | Packing and reversing operations |
| **Zbkx** | `xperm.n`, `xperm.b` | Crossbar permutation for substitution boxes |

**Sources:**[InstId.hpp 333-342](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L333-L342)

## Scalar Crypto Instructions

### AES Encryption and Decryption (Zkne, Zknd)

**AES Instruction Organization**

The Zkne (encryption) and Zknd (decryption) extensions provide different instruction sets for RV32 and RV64. RV32 operates on 32-bit words and requires multiple instructions per AES column, while RV64 operates on 64-bit values and processes two columns at once.

**Sources:**[InstId.hpp 968-978](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L968-L978)

### SHA-2 Hashing (Zknh)

The Zknh extension provides instructions for SHA-256 and SHA-512 hash computation.

| XLEN | Instructions | Hash Functions |
| --- | --- | --- |
| **RV32** | `sha256sig0`, `sha256sig1`, `sha256sum0`, `sha256sum1` | SHA-256 |
| **RV32** | `sha512sig0h`, `sha512sig0l`, `sha512sig1h`, `sha512sig1l` | SHA-512 |
| **RV32** | `sha512sum0r`, `sha512sum1r` | SHA-512 |
| **RV64** | `sha256sig0`, `sha256sig1`, `sha256sum0`, `sha256sum1` | SHA-256 |
| **RV64** | `sha512sig0`, `sha512sig1`, `sha512sum0`, `sha512sum1` | SHA-512 |

Each instruction implements one of the Sigma (Σ) or sigma (σ) functions used in SHA-2 compression. RV32 splits 64-bit SHA-512 operations across multiple 32-bit instructions.

**Sources:**[InstId.hpp 979-992](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L979-L992)

### SM3 and SM4 Chinese Cryptographic Standards (Zksed, Zksh)

| Extension | Standard | Instructions | Purpose |
| --- | --- | --- | --- |
| **Zksed** | SM4 | `sm4ed`, `sm4ks` | Block cipher encryption/key schedule |
| **Zksh** | SM3 | `sm3p0`, `sm3p1` | Hash function permutations |

**Sources:**[InstId.hpp 993-996](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L993-L996)

## Vector Bit Manipulation and Crypto

### Vector Basic Bit Manipulation (Zvbb)

Zvbb extends Zbb operations to vector form, operating on all elements in parallel.

**Vector Bit Manipulation Architecture**

Vector bit manipulation instructions operate element-wise across the entire vector register, with masking support. Each element undergoes the same operation independently.

**Sources:**[InstId.hpp 904-920](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L904-L920)

### Vector Carry-less Multiplication (Zvbc)

Zvbc provides vector carry-less multiply operations for parallel GCM/CRC computation.

| Instruction | Operation | Use Case |
| --- | --- | --- |
| `vclmul.vv`, `vclmul.vx` | Carry-less multiply (lower) | Parallel GCM authentication |
| `vclmulh.vv`, `vclmulh.vx` | Carry-less multiply (upper) | Parallel GCM authentication |

**Sources:**[InstId.hpp 930-935](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L930-L935)

### Vector AES (Zvkned)

**Vector AES Architecture**

Zvkned processes multiple AES blocks in parallel. The `.vv` variants operate vector-on-vector, while `.vs` variants use a scalar round key for all vector elements. This enables efficient AES-CTR and AES-GCM modes.

**Sources:**[InstId.hpp 940-951](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L940-L951)

### Vector SHA-2 (Zvknha, Zvknhb)

The vector SHA extensions differ based on XLEN:

*   **Zvknha**: RV32 only, SHA-256 support
*   **Zvknhb**: RV64 only, SHA-256 and SHA-512 support

| Instructions | Hash Operation | Extension |
| --- | --- | --- |
| `vsha2ms.vv` | Message schedule | Both |
| `vsha2ch.vv` | Compression high | Both |
| `vsha2cl.vv` | Compression low | Both |

The instructions process SHA-2 message blocks in parallel, with element width determining whether SHA-256 (EEW=32) or SHA-512 (EEW=64) is performed.

**Sources:**[InstId.hpp 953-956](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L953-L956)[Hart.cpp 623-637](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L623-L637)

### Vector SM4 and SM3 (Zvksed, Zvksh)

Vector versions of Chinese cryptographic standards:

| Extension | Instructions | Purpose |
| --- | --- | --- |
| **Zvksed** | `vsm4r.vv`, `vsm4r.vs`, `vsm4k.vi` | SM4 rounds and key expansion |
| **Zvksh** | `vsm3me.vv`, `vsm3c.vi` | SM3 message expansion and compression |

**Sources:**[InstId.hpp 958-965](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L958-L965)

### Vector GHASH (Zvkg)

Zvkg provides vector instructions for GHASH (used in AES-GCM):

| Instruction | Operation |
| --- | --- |
| `vghsh.vv` | GHASH multiply-accumulate |
| `vgmul.vv` | GCM multiply |

These instructions accelerate authenticated encryption by processing multiple GHASH operations in parallel.

**Sources:**[InstId.hpp 937-938](https://github.com/tenstorrent/whisper/blob/733fd921/InstId.hpp#L937-L938)

## Extension Compatibility Matrix

**Extension Dependencies**

Scalar bit manipulation and crypto extensions depend only on the base integer ISA. Vector crypto extensions require the V extension. Zbkb is a subset of Zbb plus additional crypto-specific operations.

**Sources:**[Hart.cpp 550-643](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L550-L643)

## Implementation Details

### Extension State in Hart

The `Hart` class maintains extension state through the `isa_` member and various enable flags:

| Member | Type | Purpose |
| --- | --- | --- |
| `isa_` | `Isa` | Tracks enabled ISA extensions |
| `decoder_` | `Decoder` | Decodes instructions based on enabled extensions |
| `extensionIsEnabled()` | Method | Queries whether an extension is enabled |

Extensions are enabled during `reset()` via `processExtensions()`, which reads the MISA CSR and applies configuration settings.

**Sources:**[Hart.cpp 469-647](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.cpp#L469-L647)

### Instruction Execution

Bit manipulation and crypto instructions follow the standard execution pipeline:

1.   **Fetch**: Instruction fetched from memory
2.   **Decode**: `Decoder` identifies instruction based on opcode and enabled extensions
3.   **Execute**: Instruction-specific execution function called
4.   **Writeback**: Result written to destination register

The `isBitManip_` flag in `InstEntry` allows quick identification during profiling and analysis.

**Sources:**[InstEntry.cpp 32-41](https://github.com/tenstorrent/whisper/blob/733fd921/InstEntry.cpp#L32-L41)
