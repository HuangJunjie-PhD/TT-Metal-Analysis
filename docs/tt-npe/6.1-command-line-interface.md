This document covers the `tt_npe.py` command line interface tool, which provides the primary user interface for running NoC performance simulations. The CLI tool accepts workload definitions, device configurations, and simulation parameters, then executes the tt-npe simulation engine and outputs performance results.

For programmatic usage of the same functionality, see [Python API](https://deepwiki.com/tenstorrent/tt-npe/3.2-creating-workloads-from-json). For details on workload file formats and structure, see [Workload Definition](https://deepwiki.com/tenstorrent/tt-npe/3.3-programmatic-workload-construction). For processing raw NoC traces from tt-metal profiler, see [NoC Trace Analysis](https://deepwiki.com/tenstorrent/tt-npe/3.4-converting-noc-traces-to-workloads).

## Overview

The `tt_npe.py` CLI tool serves as the main entry point for running network-on-chip performance simulations. It wraps the underlying C++ simulation engine with a user-friendly command line interface that handles configuration, workload loading, simulation execution, and result reporting.

## Command Line Options

### Required Parameters

| Option | Description | Type | Required |
| --- | --- | --- | --- |
| `-w, --workload` | Path to workload JSON file or NoC trace directory | string | Yes |

### Device and Simulation Configuration

| Option | Description | Default | Choices |
| --- | --- | --- | --- |
| `-d, --device` | Target device architecture | `wormhole_b0` | `wormhole_b0`, `N150`, `N300`, `T3K`, `blackhole`, `P100`, `P150` |
| `-c, --cycles-per-timestep` | Cycles per simulation timestep | `32` | Integer ≥ 1 |
| `--cong-model` | Congestion modeling algorithm | `fast` | `none`, `fast` |
| `--no-injection-rate-inference` | Disable automatic injection rate inference | `false` | Flag |

The device parameter selects both the NoC topology and performance characteristics. Multi-chip configurations (`N150`, `N300`, `T3K`, `P100`, `P150`) require additional cluster coordinate configuration.

Sources: [tt_npe/py/pycli/tt_npe.py 18-33](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/tt_npe/py/pycli/tt_npe.py#L18-L33)[tt_npe/py/pycli/tt_npe.py 36-51](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/tt_npe/py/pycli/tt_npe.py#L36-L51)

### Timeline and Visualization Output

| Option | Description | Default | Type |
| --- | --- | --- | --- |
| `-e, --emit-timeline-file` | Generate timeline JSON for visualization | `false` | Flag |
| `--timeline-json-filepath` | Output path for timeline file | `npe_timeline.json` | String |
| `--compress-timeline-output-file` | Compress timeline output with zstd | `false` | Flag |
| `--use-legacy-timeline-format` | Use legacy timeline format | `false` | Flag |

Timeline files can be imported into ttnn-visualizer for graphical analysis of transfer scheduling and congestion patterns.

Sources: [tt_npe/py/pycli/tt_npe.py 54-92](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/tt_npe/py/pycli/tt_npe.py#L54-L92)

### Workload Processing Options

| Option | Description | Default | Type |
| --- | --- | --- | --- |
| `-t, --workload-is-noc-trace` | Parse workload as tt-metal NoC trace format | `false` | Flag |
| `--scale-workload-schedule` | Scale workload timing by factor | `0.0` | Float |
| `--cluster-coordinates-json` | Path to cluster coordinate mapping file | `""` | String |

The NoC trace format option enables processing raw trace files from tt-metal profiler runs.

Sources: [tt_npe/py/pycli/tt_npe.py 62-66](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/tt_npe/py/pycli/tt_npe.py#L62-L66)[tt_npe/py/pycli/tt_npe.py 74-79](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/tt_npe/py/pycli/tt_npe.py#L74-L79)[tt_npe/py/pycli/tt_npe.py 105-107](https://github.com/tenstorrent/tt-npe/blob/d1c8b91a/tt_npe/py/pycli/tt_npe.py#L105-L107)

## Output Format

### Successful Execution

The CLI outputs execution time and detailed performance statistics:

```
Loaded workload successfully, starting tt-npe ... 
tt-npe simulation finished successfully in 1234 us!
--- stats ----------------------------------
Stats {
  completed                         = true
  estimated_cycles                  = 12345
  estimated_cong_free_cycles        = 11000  
  golden_cycles                     = 0
  cycle_prediction_error           = 0.123
  num_timesteps                    = 456
  wallclock_runtime_us             = 1234
  overall_avg_link_demand          = 0.45
  overall_max_link_demand          = 0.78
  overall_avg_link_util            = 0.34
  overall_max_link_util            = 0.67
  dram_bw_util                     = 0.23
}
```

### Error Conditions

Error messages are displayed in red bold text to stderr:

*   **Missing workload file**: `E: Must provide a tt-npe workload JSON file with option -w,--workload`
*   **Invalid workload**: `E: Could not create tt-npe workload from file 'filename'; aborting ...`
*   **API initialization failure**: `E: tt-npe could not be initialized, check that config is sound?`
*   **Simulation error**: `E: tt-npe exited unsuccessfully: [error details]`
