## Purpose and Scope

The Configuration Management system provides a YAML-based framework for defining and managing simulation parameters, workload specifications, and architecture mappings in the Polaris simulation framework. This system enables users to specify how neural network operations map to hardware resources, define data types, configure operation fusion patterns, and manage device architectures without modifying code.

For detailed information about specific workload-to-architecture mapping configurations, see [Workload to Architecture Mapping](https://deepwiki.com/tenstorrent/polaris/2.3.1-workload-to-architecture-mapping). For information about the configuration parsing implementation, see [Configuration Parsers](https://deepwiki.com/tenstorrent/polaris/2.3.2-configuration-parsers).

## Core Configuration Files

### Workload to Architecture Mapping Configuration

The `wl2archmapping.yaml` file serves as the central configuration for mapping neural network operations to hardware resources. It contains four main specification sections:

| Configuration Section | Purpose | Example |
| --- | --- | --- |
| `op_data_type_spec` | Defines data types for operations | `global_type: int8` |
| `op_removal_spec` | Lists operations to remove from graphs | `[Dropout, Identity, Constant]` |
| `op_fusion_spec` | Defines operation fusion patterns | `[Matmul, Add, Gelu]` |
| `op_rsrc_spec` | Maps operations to compute resources | `matrix: [Matmul, Conv]` |

**Sources:**[config/wl2archmapping.yaml 3-49](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/wl2archmapping.yaml#L3-L49)

## Configuration Validation and Models

### Pydantic Validation Models

The system uses Pydantic models for configuration validation:

| Validator Class | Purpose | Configuration Type |
| --- | --- | --- |
| `PYDWorkloadListValidator` | Validates workload specifications | Workload definitions |
| `IPBlocksModel` | Validates IP block configurations | Architecture IP blocks |
| `PackageInstanceModel` | Validates package instances | Device packages |
| `PYDWlMapSpecValidator` | Validates mapping specifications | Operation mappings |

**Sources:**[ttsim/config/__init__.py 10-13](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/config/__init__.py#L10-L13)
