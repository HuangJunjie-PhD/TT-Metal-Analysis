## Purpose and Scope

This document describes the MLIR backend compilation pipeline, the most critical component in the tt-forge-onnx system (importance: 443.92). The MLIR backend is responsible for:

*   Lowering Forge's graph representation (`ForgeGraphModule`) to MLIR's TTIR (Tenstorrent Intermediate Representation) dialect
*   Managing attribute conversions and type mappings between Forge and MLIR representations
*   Applying optimization passes through the TTIR â†’ TTNN pipeline
*   Generating executable outputs (Flatbuffer binaries, C++ source, or shared objects)

For information about the preceding compilation stages, see [Compilation Pipeline Overview](https://deepwiki.com/tenstorrent/tt-forge-onnx/3.1-compilation-pipeline-overview). For runtime execution of compiled binaries, see [Runtime Execution Model](https://deepwiki.com/tenstorrent/tt-forge-onnx/3.6-runtime-execution-model). For MLIR configuration options available during compilation, see [Compilation Configuration](https://deepwiki.com/tenstorrent/tt-forge-onnx/8.2-compilation-configuration).

The `MLIRGenerator` class orchestrates the conversion from Forge graphs to MLIR operations:

| Component | Type | Purpose |
| --- | --- | --- |
| `graphModule_` | `mlir::ModuleOp` | Top-level MLIR module containing all functions |
| `builder_` | `mlir::OpBuilder` | Stateful builder for creating MLIR operations |
| `symbolTable_` | `std::map<std::string, std::pair<mlir::Value, graphlib::Node*>>` | Maps Forge node names to MLIR values |
| `lowering_handler_map` | `std::map<std::string, HandlerType>` | Maps operation types to lowering functions |
| `attr_mapper_` | `AttributeMapper` | Handles attribute name/type conversions |

| Forge DataFormat | MLIR Type | Method |
| --- | --- | --- |
| `Float32` | `F32Type` | `builder_.getF32Type()` |
| `Float16` | `F16Type` | `builder_.getF16Type()` |

## Output Generation

The MLIR backend supports three output formats, selected by the `MLIROutputKind` enum:

### Output Format Comparison

| Format | Enum Value | Function | Use Case |
| --- | --- | --- | --- |
| **Flatbuffer Binary** | `MLIROutputKind::Flatbuffer` | `ttnnToFlatbuffer()` | Runtime execution on device |
| **C++ Source** | `MLIROutputKind::Cpp` | `translateToCpp()` | Debug, inspection, manual compilation |
| **Shared Object** | `MLIROutputKind::SharedObject` | `compileCppToSo()` | Dynamic loading, standalone execution |
