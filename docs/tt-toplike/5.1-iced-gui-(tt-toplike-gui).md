The `tt-toplike-gui` is a native desktop application built using the **iced 0.14** framework [src/bin/gui.rs 16-19](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L16-L19) It provides a high-performance, GPU-accelerated interface for monitoring Tenstorrent hardware across Wayland and X11 environments [src/bin/gui.rs 6-7](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L6-L7) The GUI shares the same robust telemetry backend as the TUI but extends it with historical circular buffers, line charts, and specialized Canvas-based visualizations [docs/GUI_FEATURES.md 96-101](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L96-L101)

## System Architecture

The GUI follows the **Model-View-Update (MVU)** pattern inherent to `iced`. It maintains a `TTTopGUI` state struct that manages a `TelemetryBackend`, a `HistoryManager` for time-series data, and per-device visualization engines [src/bin/gui.rs 52-85](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L52-L85)

## Core Components

### 1. History Management

The GUI maintains a `HistoryManager` which stores `TelemetryHistory` for every detected device [src/ui/gui/history.rs 133-140](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/gui/history.rs#L133-L140)

*   **Circular Buffer**: Uses `VecDeque<f32>` to store exactly 300 samples (representing 30 seconds at a 100ms polling interval) [src/ui/gui/history.rs 14-39](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/gui/history.rs#L14-L39)
*   **O(1) Operations**: Metrics like `power`, `temperature`, and `aiclk` are pushed to the back and popped from the front to maintain a constant memory footprint [src/ui/gui/history.rs 56-74](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/gui/history.rs#L56-L74)

### 2. View Modes

### 3. Canvas Rendering and GPU Acceleration

The GUI utilizes the `iced::widget::canvas` API for custom drawing.

*   **HardwareStarfield**: Each star represents a physical Tensix core. Its brightness is mapped to power consumption, color to temperature, and twinkle rate to current draw [docs/GUI_FEATURES.md 64-67](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L64-L67)
*   **TerminalCanvas**: A specialized widget that renders a `TerminalGrid` (character-based buffer) using monospace fonts, allowing the GUI to replicate the TUI's aesthetic for complex ASCII-art portraits [src/ui/gui/terminal_canvas.rs 15-22](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/gui/terminal_canvas.rs#L15-L22)
*   **Backend**: Defaults to **wgpu** with Vulkan or OpenGL ES fallback [docs/GUI_FEATURES.md 217-221](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L217-L221)

## Implementation Details

### Telemetry Update Loop

The GUI uses an `iced` subscription or periodic `Message::Tick` to drive the update cycle [src/bin/gui.rs 89-91](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L89-L91)

1.   **Backend Update**: `self.backend.update()` is called to fetch fresh hardware data [src/bin/gui.rs 173](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L173-L173)
2.   **History Push**: New samples are added to the `HistoryManager`[src/bin/gui.rs 182](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L182-L182)
3.   **Visual Sync**: Visualizations (Starfield/Dashboard) call `update_from_telemetry(&self.backend)` to synchronize their internal animation states with the new hardware metrics [src/bin/gui.rs 188-194](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L188-L194)
4.   **Cache Invalidation**: `self.cache.clear()` is called on Canvas widgets to trigger a redraw [src/ui/gui/visualization.rs 131](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/gui/visualization.rs#L131-L131)

### Multi-Device Handling

The GUI implements a tabbed interface for multi-device systems [docs/GUI_FEATURES.md 73-79](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L73-L79)

*   **Independent State**: Each device index maintains its own `TelemetryHistory`, `HardwareStarfield`, and `DashboardVisualization` instances [src/bin/gui.rs 77-85](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L77-L85)
*   **Selection**: The `Message::SelectDevice(usize)` message updates `self.selected_device`, which changes the context for all view modes [src/bin/gui.rs 197-201](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/gui.rs#L197-L201)

## Technical Requirements

*   **Update Rate**: Configurable via `--interval`, defaulting to 100ms (10 FPS) [docs/GUI_FEATURES.md 148](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L148-L148)
*   **Memory Footprint**: Typically 15-20MB, optimized via circular buffers [docs/GUI_FEATURES.md 149-157](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L149-L157)
*   **Graphics Backend**: Managed by `iced`'s `wgpu` integration, supporting Vulkan, Metal, or DX12 depending on the host OS [docs/GUI_FEATURES.md 59](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/docs/GUI_FEATURES.md?plain=1#L59-L59)
