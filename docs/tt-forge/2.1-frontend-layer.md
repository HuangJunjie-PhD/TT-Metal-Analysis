## Purpose and Scope

The Frontend Layer of the TT-Forge software stack serves as the entry point for AI/ML models from various frameworks into the Tenstorrent compilation pipeline. This layer ingests models from PyTorch, JAX, ONNX, TensorFlow, and other frameworks, translating them into intermediate representations (IR) such as StableHLO or TTIR that the [TT-MLIR Compiler Layer](https://deepwiki.com/tenstorrent/tt-forge/2.2-tt-mlir-compiler-layer) can process.

The frontend systems abstract hardware-specific details, allowing developers to run models on Tenstorrent hardware using standard framework APIs like `torch.compile` or JAX JIT [README.md 15-17](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L15-L17)[README.md 27-29](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L29)

## Frontend Systems Overview

The TT-Forge ecosystem provides three primary frontend systems, each targeting different use cases and ML framework ecosystems:

| Frontend | Repository | Multi-chip Support | Primary Frameworks | Status | Output IR |
| --- | --- | --- | --- | --- | --- |
| **TT-XLA** | `tenstorrent/tt-xla` | Yes | PyTorch, JAX | Active | StableHLO |
| **TT-Forge-ONNX** | `tenstorrent/tt-forge-onnx` | No | ONNX, TensorFlow, PaddlePaddle | Active | TTIR |
| **TT-Torch** | `tenstorrent/tt-torch` | No | PyTorch | Deprecated | StableHLO |

## TT-XLA Frontend

TT-XLA is the primary frontend for PyTorch and JAX models. It leverages the OpenXLA PJRT (Plugin JAX Runtime) interface to compile models into StableHLO graphs for the TT-MLIR compiler [README.md 27](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L27)

### Key Components and Code Entities

*   **PJRT Interface**: The hardware abstraction layer that allows JAX and PyTorch to communicate with Tenstorrent devices [README.md 27](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L27)
*   **`torch_xla.runtime`**: Used to set the device type and initialize runtime parameters [README.md 51-56](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L51-L56)
*   **`torch_xla.core.xla_model`**: Provides the `xla_device()` function to acquire a Tenstorrent device handle [README.md 50-57](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L50-L57)

### PyTorch Integration (torch.compile)

TT-XLA integrates with PyTorch 2.x via the `torch.compile` backend. When `backend="tt"` is specified, the model is lowered through XLA into StableHLO [README.md 61](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L61-L61)

Sources: [README.md 49-62](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L49-L62)

### JAX Integration (jax.jit)

For JAX, TT-XLA captures the graph during `jax.jit` compilation. It supports multi-chip configurations via JAX `Mesh`[demos/tt-xla/nlp/jax/gpt_demo.py 27-41](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/tt-xla/nlp/jax/gpt_demo.py#L27-L41)

Sources: [demos/tt-xla/nlp/jax/gpt_demo.py 26-45](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/tt-xla/nlp/jax/gpt_demo.py#L26-L45)[README.md 93](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L93-L93)

### Execution Environment

TT-XLA behavior is often controlled via environment variables to force the use of the Tenstorrent PJRT plugin and StableHLO compilation:

*   `PJRT_DEVICE="TT"`[README.md 93](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L93-L93)
*   `XLA_STABLEHLO_COMPILE="1"`[README.md 94](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L94-L94)

## TT-Forge-ONNX Frontend

TT-Forge-ONNX (previously referred to as the TVM-based frontend) is designed for ONNX, TensorFlow, and PaddlePaddle models. It utilizes `tt-tvm` to ingest these formats and lower them to the TTIR dialect [README.md 28](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L28-L28)[demos/README.md 11-13](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/README.md?plain=1#L11-L13)

### Characteristics

*   **TVM-based**: Uses Apache TVM's ingestion capabilities to handle diverse formats [demos/README.md 12](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/README.md?plain=1#L12-L12)
*   **Single-chip only**: Currently does not support multi-chip configurations [demos/README.md 12](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/README.md?plain=1#L12-L12)
*   **Graph Transformation**: Designed to optimize and transform computational graphs before passing them to the TT-MLIR compiler [demos/README.md 12](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/README.md?plain=1#L12-L12)

## Comparison of Frontend Capabilities

The choice of frontend depends on the source framework and the requirement for multi-chip execution.

| Feature | TT-XLA | TT-Forge-ONNX |
| --- | --- | --- |
| **Primary Use Case** | PyTorch, JAX (Inference & Training) | ONNX, TF, Paddle (Inference) |
| **Multi-chip** | Supported [README.md 27](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L27) | Not Supported [demos/README.md 12](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/demos/README.md?plain=1#L12-L12) |
| **IR Generation** | StableHLO [README.md 27](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L27) | TTIR [README.md 28-29](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L28-L29) |
| **Optimization Engine** | XLA / MLIR | TVM / MLIR |
| **Ease of Use** | High (Native Framework APIs) | High (Standard model formats) |
