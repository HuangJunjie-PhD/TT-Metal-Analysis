This page provides a complete reference for the `tt_npe_pybind` module, which exposes the C++ simulation engine to Python. The module provides classes for configuration (`Config`), workload construction (`Workload`, `Phase`, `Transfer`), simulation execution (`API`), and result analysis (`Stats`). All functionality is accessed by importing the module:

## Module Contents

The `tt_npe_pybind` module exposes the following classes and functions:

| Name | Type | Purpose |
| --- | --- | --- |
| `Config` | class | Simulation configuration parameters |
| `InitAPI()` | function | Initialize simulation engine with config |
| `API` | class | Simulation runner with `runNPE()` method |
| `Workload` | class | Container for simulation workload |
| `Phase` | class | Group of transfers with no mutual dependencies |
| `Transfer` | class | Individual NoC transfer definition |
| `Coord` | class | 3D coordinate (device_id, row, col) |
| `MulticastCoordSet` | class | Multicast destination specification |
| `NocType` | enum | NOC_0 or NOC_1 selection |
| `createWorkloadFromJSON()` | function | Load workload from JSON file |
| `Stats` | class | Simulation results and metrics |
| `DeviceStats` | class | Per-device statistics (nested in Stats) |
| `Exception` | class | Error information from failed simulation |

## npe.InitAPI()

**Function Signature**

Initializes the simulation engine with the provided configuration. Returns an `npe.API` handle if successful, or `None` if initialization fails (e.g., invalid device name, malformed configuration).

**Parameters**

*   `cfg`: `npe.Config` object containing simulation parameters

**Returns**

*   `npe.API` object on success
*   `None` on failure

## npe.API

Handle for running simulations. Obtained via `npe.InitAPI()`.

### Methods

**`runNPE(workload: npe.Workload) -> Union[npe.Stats, npe.Exception]`**

Executes performance estimation simulation for the provided workload. Returns `npe.Stats` on success or `npe.Exception` on failure.

**Parameters**

*   `workload`: `npe.Workload` object containing transfers to simulate

**Returns**

*   `npe.Stats` containing performance metrics if simulation succeeds
*   `npe.Exception` containing error information if simulation fails

**Example**

## npe.Coord

Represents a 3D coordinate on the device mesh.

### Constructor

**Important:** Coordinates use `(device_id, row, col)` ordering, NOT `(x, y)` pairs. The coordinate system follows the device grid layout where `row` and `col` correspond to the physical core grid positions.

**Properties**

*   `device_id` (int): Device identifier in multi-chip systems
*   `row` (int): Row coordinate in device grid
*   `col` (int): Column coordinate in device grid

## npe.Transfer

Represents a contiguous series of packets from a single source to one or more destinations. Equivalent to a single `noc_async_read()` or `noc_async_write()` call in tt-metal.

### Constructor

**Parameters**

1.   `packet_size` (int): Size of each packet in bytes (must be > 0)
2.   `num_packets` (int): Number of packets in transfer (must be ≥ 1)
3.   `src` (npe.Coord): Source coordinate
4.   `dst` (npe.Coord or npe.MulticastCoordSet): Destination coordinate(s)
5.   `injection_rate` (float): Injection rate in GB/s (0 = auto-infer from source core type)
6.   `phase_cycle_offset` (int): Earliest cycle transfer can start relative to phase start
7.   `noc_type` (npe.NocType): Which NoC to use

## npe.createWorkloadFromJSON()

**Function Signature**

Loads a workload from a JSON file. Supports both tt-npe native workload format and tt-metal NoC trace format.

**Parameters**

*   `json_wl_filename`: Path to JSON file
*   `device_name`: Target device architecture
*   `is_noc_trace_format`: If True, parse as tt-metal NoC trace; if False, parse as tt-npe workload
*   `verbose`: Enable verbose parsing output

**Returns**

*   `npe.Workload` object on success

## npe.DeviceStats

Per-device performance statistics. Accessed via `npe.Stats.per_device_stats[device_id]`.

### Core Metrics

| Property | Type | Description |
| --- | --- | --- |
| `completed` | bool | Whether simulation completed successfully |
| `estimated_cycles` | int | Total estimated cycles including congestion |
| `estimated_cong_free_cycles` | int | Estimated cycles without congestion |
| `golden_cycles` | int | Actual measured cycles (if provided) |
| `cycle_prediction_error` | float | Prediction error vs golden (if available) |
| `wallclock_runtime_us` | int | Simulation wall-clock time in microseconds |

### NoC Utilization Metrics

| Property | Type | Description |
| --- | --- | --- |
| `overall_avg_link_demand` | float | Average link demand across all links |
| `overall_max_link_demand` | float | Peak link demand |
| `overall_avg_niu_demand` | float | Average NIU demand |
| `overall_max_niu_demand` | float | Peak NIU demand |
| `overall_avg_link_util` | float | Average link utilization [0.0-1.0] |
| `overall_max_link_util` | float | Peak link utilization [0.0-1.0] |

### NoC-Specific Metrics

| Property | Type | Description |
| --- | --- | --- |
| `overall_avg_noc0_link_demand` | float | Average NoC 0 link demand |
| `overall_avg_noc0_link_util` | float | Average NoC 0 link utilization |
| `overall_max_noc0_link_demand` | float | Peak NoC 0 link demand |
| `overall_avg_noc1_link_demand` | float | Average NoC 1 link demand |
| `overall_avg_noc1_link_util` | float | Average NoC 1 link utilization |
| `overall_max_noc1_link_demand` | float | Peak NoC 1 link demand |

### DRAM and Ethernet Metrics

| Property | Type | Description |
| --- | --- | --- |
| `dram_bw_util` | float | DRAM bandwidth utilization from workload data |
| `dram_bw_util_sim` | float | DRAM bandwidth utilization from simulation |
| `eth_bw_util_per_core` | dict[Coord, float] | Per-core Ethernet bandwidth utilization |

### Methods

**`getCongestionImpact() -> float`**

Returns congestion impact as the ratio of cycles added by congestion:

```
(estimated_cycles - estimated_cong_free_cycles) / estimated_cong_free_cycles
```

**`getEthBwUtilPerCoreStr() -> str`**

Returns formatted string representation of per-core Ethernet bandwidth utilization.

**`getAggregateEthBwUtil() -> float`**

Returns average Ethernet bandwidth utilization across all Ethernet cores.

## npe.Exception

Error object returned when simulation fails. Contains error code and descriptive message.

### Methods

**`err() -> str`**

Returns error message describing what went wrong.

**Example**

**Common Error Scenarios**

| Error Type | Description |
| --- | --- |
| `INVALID_CONFIG` | Configuration validation failed |
| `DEVICE_MODEL_INIT_FAILED` | Unknown device name or device model initialization error |
| `WORKLOAD_VALIDATION_FAILED` | Malformed workload (invalid coordinates, negative values, etc.) |
| `EXCEEDED_SIM_CYCLE_LIMIT` | Simulation exceeded maximum cycle limit (potential hang) |
| `TRACE_INGEST_FAILED` | Failed to parse JSON workload file |
| `DEPENDENCY_GEN_FAILED` | Failed to generate transfer dependencies |
