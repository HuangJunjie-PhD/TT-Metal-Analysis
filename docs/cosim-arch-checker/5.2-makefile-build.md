## Purpose and Scope

The Makefile build system provides an alternative approach to building the cosim-arch-checker framework as a shared library (`libcosim.so`). This build method compiles all C++ source files in the repository into a single shared library that can be linked against other applications or loaded dynamically.

For information about the Bazel-based build approach that creates the `dpi` binary target, see [Bazel Build](https://deepwiki.com/tenstorrent/cosim-arch-checker/5.1-bazel-build). For details on integrating the built artifacts with hardware simulation environments, see [Integration Guide](https://deepwiki.com/tenstorrent/cosim-arch-checker/6-integration-guide).

## Build Process Overview

The Makefile implements a straightforward compilation workflow that discovers all C++ sources, compiles them with dependency tracking, and links them into a shared library. The build system automatically handles directory creation and maintains separate build and output directories.

### Compiler Flags

The build system uses specific compiler flags to ensure compatibility with the cosim-arch-checker framework:

| Flag | Purpose |
| --- | --- |
| `-std=c++17` | Enable C++17 language features |
| `-Wall -Werror` | Enable all warnings and treat as errors |
| `-Wno-unused-private-field` | Suppress specific warning for private fields |
| `-fpic` | Generate position-independent code for shared library |

## Dependency Management

The Makefile implements automatic dependency tracking using GCC's dependency generation features:

*   **`-MMD`**: Generate dependency files alongside object files
*   **`-MP`**: Add phony targets for header files to avoid errors when headers are deleted
*   **`-include $(DEPS)`**: Include generated dependency files to track header changes

This ensures that object files are automatically rebuilt when their source files or included headers change.

## Output Artifacts

The Makefile build produces a single primary artifact:

*   **`lib/libcosim.so`**: Shared library containing all cosim-arch-checker functionality

This shared library can be:

*   Linked against other C++ applications
*   Loaded dynamically at runtime
*   Used with SystemVerilog DPI through appropriate wrapper functions

The shared library includes compiled code from all framework components: DUT monitoring interface, bridge orchestration, Whisper client integration, and Core Arch Checker interfaces.
