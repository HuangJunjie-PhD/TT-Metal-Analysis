This page explains the `TTLAssignDST` pass, which allocates DST (destination) registers for tile-level operations within `ttl.compute` operations. The pass implements interval-based linear scan register allocation with in-place operation merging, assigns `dst_idx` attributes to tile operations, and calculates the `ttl.unroll_factor` for subblocking. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 8-32](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L8-L32)

For information about how the `ttl.unroll_factor` is used to partition compute operations into DST-sized subblocks, see **3.3.3 Subblocking and Tiling Strategy**. For details on the elementwise fusion that precedes DST assignment, see **3.3.1 Elementwise Fusion**.

## Purpose and Scope

The DST register file is a limited hardware resource (4-16 tiles depending on data type and sync mode) that temporarily holds tile values during computation. The `TTLAssignDST` pass must:

1.   **Allocate DST registers** to tile values such that no more than the hardware capacity are live simultaneously. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 25-28](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L25-L28)
2.   **Respect hardware constraints** such as in-place operations and FPU accumulation. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 149-151](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L149-L151)
3.   **Insert copy operations** (`ttl.copy_dst` or `ttl.copy_tile`) where needed to avoid conflicts or clobbering. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 14-17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L14-L17)
4.   **Assign dst_idx attributes** that downstream passes use for code generation. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 31-32](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L31-L32)
5.   **Calculate ttl.unroll_factor** to inform the subblocking pass. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 932-961](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L932-L961)

**Sources:**[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 8-38](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L8-L38)

## DST Register File Architecture

The DST register file has a fixed physical capacity of 16 tiles, but the logical capacity available to user operations varies based on data type and synchronization mode. This is managed by the `getDstCapacity` function which adjusts the limit based on whether double-buffering or f32 precision is active. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 891-918](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L891-L918)

| Data Type | Sync Mode | Physical Tiles | Logical Capacity | Notes |
| --- | --- | --- | --- | --- |
| bf16/f16 | Double-buffered (default) | 16 | 8 | Half reserved for ping-pong buffering [test/ttlang/Dialect/TTL/Transforms/AssignDST/dst_capacity_combinations.mlir 13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/AssignDST/dst_capacity_combinations.mlir#L13-L13) |

## Four-Phase Allocation Algorithm

The pass implements a multi-phase algorithm to ensure optimal register usage while respecting hardware semantics. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 14-32](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L14-L32)

### Phase 0: FPU Binary Detection

Marks binary operations (`ttl.tile_add`, `ttl.tile_sub`, `ttl.tile_mul`) as FPU-eligible when both operands are input block arguments. FPU binary operations read directly from Circular Buffers (CB) and write to DST, requiring zero DST input slots. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 645-694](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L645-L694)

**Sources:**[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 645-694](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L645-L694)[test/ttlang/Dialect/TTL/Transforms/AssignDST/separate_output_region_merged_set.mlir 18-20](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/AssignDST/separate_output_region_merged_set.mlir#L18-L20)

### Phase 1: Copy Insertion

Inserts `ttl.copy_dst` operations for values with multiple consumers where any consumer is an in-place operation. Since in-place operations (like unary SFPU ops) overwrite their input DST register, any other operation needing that same value must receive a copy first to prevent data corruption. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 135-188](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L135-L188)

**Sources:**[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 135-188](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L135-L188)[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 877-885](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L877-L885)

### Phase 2: Build Live Intervals with In-Place Merging

Constructs live intervals `[start, end]` for each tile value. It uses `llvm::EquivalenceClasses<Value>` to merge intervals for in-place operations, ensuring the input and output share the same DST register. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 352-519](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L352-L519)[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 74-86](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L74-L86)

**Sources:**[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 352-519](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L352-L519)[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 74-86](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L74-L86)[include/ttlang/Dialect/TTL/Transforms/LiveIntervalUtils.h 41-58](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/Transforms/LiveIntervalUtils.h#L41-L58)

### Phase 3 & 4: Linear Scan Allocation

Implements interval-based linear scan allocation. If the `separate-output-region` flag is enabled, the algorithm splits allocation into two regions to prevent computation intermediates from overwriting persistent outputs (essential for correct reduction behavior). Phase 3 handles intermediates, while Phase 4 handles yielded values and their merged partners. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 521-619](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L521-L619)[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 973-1025](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L973-L1025)[test/ttlang/Dialect/TTL/Transforms/AssignDST/separate_output_region_merged_set.mlir 1-16](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/AssignDST/separate_output_region_merged_set.mlir#L1-L16)

## In-Place Operation Merging

Operations are classified by their DST usage patterns. The `TTLInPlaceOpTrait` is used to identify operations that overwrite their inputs. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 149-151](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L149-L151)

| Category | Input Source | DST Input Slots | DST Output Slots | In-Place |
| --- | --- | --- | --- | --- |
| SFPU binary | DST (both) | 2 | 1 (fresh) | No |
| FPU binary | CB (both) | 0 | 1 (fresh) | No |
| Unary | DST | 0 | 0 (overwrites) | Yes |
| Broadcast | CB | 0 | 1 (fresh) | No |

## Copy Operations

The pass manages two specific copy operations to facilitate register allocation:

1.   **ttl.copy_tile**: Transfers data from a Circular Buffer to a DST register. Inserted for block arguments that are not consumed by FPU-eligible operations. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 98-132](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L98-L132)
2.   **ttl.copy_dst**: Performs a DST-to-DST copy. Inserted during Phase 1 to preserve values consumed by in-place operations. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 170-188](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L170-L188)

## Unroll Factor Calculation

After register assignment, the pass calculates the `ttl.unroll_factor`. This factor determines how many tiles can be processed in a single DST synchronization cycle (`tile_regs_acquire`/`tile_regs_release`) based on DST pressure. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 932-961](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L932-L961)

$$D = \text{maxDstUsed} + 1$$ $$N = \min\left(\left\lfloor \frac{\text{Capacity}}{D} \right\rfloor, \text{totalTiles}\right)$$

Where $N$ is the `unroll_factor`. For example, a simple FPU binary add with capacity=8 and a 2x2 tensor (4 tiles) results in an `unroll_factor=8` if pressure $D=1$. [test/ttlang/Dialect/TTL/Transforms/AssignDST/dst_capacity_combinations.mlir 21-24](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/AssignDST/dst_capacity_combinations.mlir#L21-L24)

**Sources:**[lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 932-961](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L932-L961)[test/ttlang/Dialect/TTL/Transforms/AssignDST/dst_capacity_combinations.mlir 11-16](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/AssignDST/dst_capacity_combinations.mlir#L11-L16)

## Pass Options

*   **dst-capacity**: Manually override the DST capacity (default is hardware-specific). [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 1062](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L1062-L1062)
*   **separate-output-region**: If true, ensures output tiles are allocated to a distinct set of DST registers that do not overlap with input/intermediate registers. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 1065](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L1065-L1065)
*   **enable-fpu-binary-ops**: Enables the Phase 0 detection of FPU-eligible operations to reduce DST pressure. [lib/Dialect/TTL/Transforms/TTLAssignDST.cpp 1068](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLAssignDST.cpp#L1068-L1068)
