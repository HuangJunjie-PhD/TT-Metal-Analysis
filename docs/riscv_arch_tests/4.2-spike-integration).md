# Spike Integration

Relevant source files
*   [.gitmodules](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/.gitmodules)

## Purpose and Scope

This document covers the integration of the Spike RISC-V Instruction Set Simulator (ISS) within the RISC-V architectural test framework. Spike serves as one of the primary simulation backends for executing RISC-V assembly tests and validating architectural compliance. The integration encompasses Git submodule management, container build processes, and test execution workflows.

For information about Whisper ISS integration, see [Whisper Configuration](https://deepwiki.com/tenstorrent/riscv_arch_tests/4.1-whisper-configuration). For general test execution mechanisms, see [Test Execution](https://deepwiki.com/tenstorrent/riscv_arch_tests/3.1-test-execution).

## Submodule Architecture

The Spike ISS is integrated as a Git submodule, providing version-controlled access to the Tenstorrent fork of the Spike simulator. This approach ensures reproducible builds and controlled dependency management across the testing infrastructure.

### Submodule Configuration

**Spike Submodule Integration Architecture**

The submodule configuration establishes the relationship between the main repository and the Spike ISS dependency, enabling automated builds and consistent simulator versions across different environments.

Spike is built and integrated into the containerized testing environment through the container build system. The build process compiles Spike from source within the container image, making it available for test execution.

The test infrastructure integrates with Spike through the `quals.py` test runner, which orchestrates test execution against the Spike simulator. This integration supports both individual test execution and batch processing workflows.

## Simulator Backend Selection

The testing framework supports multiple ISS backends, with Spike serving as an alternative to Whisper for different validation scenarios. The choice between simulators can be configured through test runner parameters and environment settings.

| Component | Spike Integration | Whisper Integration |
| --- | --- | --- |
| Repository | Git submodule | Git submodule |
| Configuration | Command-line parameters | JSON configuration files |
| Build Process | Container compilation | Container compilation |
| Test Runner | `quals.py` integration | `quals.py` integration |
