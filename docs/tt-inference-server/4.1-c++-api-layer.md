The C++ API Layer is the high-performance entry point for all HTTP-based inference requests in the `tt_media_server_cpp` backend. It utilizes the **Drogon** web framework to provide asynchronous handling of OpenAI-compatible endpoints. This layer is responsible for request validation, session resolution (including prefix caching), mapping JSON payloads to internal domain objects, and dispatching tasks to the underlying inference services.

## Architecture and Controller Hierarchy

The API is organized into specialized controllers that inherit from `drogon::HttpController`. Each controller is responsible for a specific domain of the API.

| Controller | Purpose | Key Endpoints |
| --- | --- | --- |
| `LLMController` | Handles Chat Completions, Prefix Caching, and Session Management. | `/v1/chat/completions`, `/v1/sessions` |
| `EmbeddingController` | Handles vector embedding generation via worker processes. | `/v1/embeddings` |
| `HealthController` | Provides liveness and readiness checks for services and sockets. | `/health`, `/tt-liveness` |
| `OpenAPIController` | Serves API documentation and the OpenAPI schema. | `/docs`, `/openapi.json` |

## Request Lifecycle: LLM Chat Completions

The `LLMController::chatCompletions` method handles the complex lifecycle of an LLM request, including a multi-layered session resolution strategy.

1.   **Parsing**: The JSON body is parsed into a `domain::ChatCompletionRequest`[tt-media-server/cpp_server/src/api/llm_controller.cpp 189-196](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L189-L196) This includes handling `tool_choice`, `response_format` for structured output, and `enable_reasoning` for models like DeepSeek-R1 [tt-media-server/cpp_server/include/domain/chat_completion_request.hpp 80-92](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/domain/chat_completion_request.hpp#L80-L92)
2.   **Session Resolution**: The `resolveSession` function implements a three-layer lookup [tt-media-server/cpp_server/src/api/llm_controller.cpp 59-182](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L59-L182): 
    *   **Layer 1 (Legacy)**: If `sessionId` is provided in the JSON, attempt to `acquireInFlight` that specific session [tt-media-server/cpp_server/src/api/llm_controller.cpp 84-101](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L84-L101)
    *   **Layer 2 (Prefix Cache)**: Compute a `lookupHash` from chat messages. If a match is found in `SessionManager::tryAcquireByPrefixHash`, reuse that session's KV cache [tt-media-server/cpp_server/src/api/llm_controller.cpp 123-144](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L123-L144)
    *   **Layer 3 (New Allocation)**: If no match is found, call `SessionManager::createSession` to allocate a new slot via the session manager [tt-media-server/cpp_server/src/api/llm_controller.cpp 159-181](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L159-L181)

3.   **Dispatch**: 
    *   **Streaming**: Calls `handleStreaming`, which uses `SSEStreamWriter` to push `ChatCompletionStreamChunk` objects to the client [tt-media-server/cpp_server/src/api/llm_controller.cpp 184-187](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L184-L187)
    *   **Blocking**: Submits to `LLMService` via `submitRequest` and returns a full `ChatCompletionResponse`.

## Session and Slot Management

The `SessionManager` coordinates the lifecycle of sessions and their association with hardware KV cache slots.

*   **Prefix Indexing**: Maintains a `prefixIndex` to enable rapid KV cache reuse based on conversation history [tt-media-server/cpp_server/src/api/llm_controller.cpp 73-81](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L73-L81)
*   **In-Flight Tracking**: The `acquireInFlight` method marks a session as busy and registers a `cancelFn` to handle request abortion [tt-media-server/cpp_server/src/api/llm_controller.cpp 87-92](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L87-L92)

## Embedding Service Integration

The `EmbeddingController` handles vector embedding generation. Unlike the LLM service's internal scheduling, it utilizes a multi-process worker model.

*   **Worker Process Management**: The `EmbeddingService` spawns `WorkerProcess` instances using `fork()`. Each worker is assigned specific hardware via `TT_VISIBLE_DEVICES`[tt-media-server/cpp_server/src/services/embedding_service.cpp 75-133](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/embedding_service.cpp#L75-L133)
*   **Pipe Communication**: Parent and child communicate via length-prefixed pipes (`pipeWrite`/`pipeReadBinary`) [tt-media-server/cpp_server/src/services/embedding_service.cpp 34-71](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/services/embedding_service.cpp#L34-L71)
*   **Asynchronous Callbacks**: To prevent blocking Drogon's event loops, the controller offloads the blocking wait for worker responses to a dedicated `ThreadPool` (the "Callback Pool") [tt-media-server/cpp_server/src/api/embedding_controller.cpp 19-25](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/embedding_controller.cpp#L19-L25)[tt-media-server/cpp_server/src/api/embedding_controller.cpp 83-131](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/embedding_controller.cpp#L83-L131)

## Error Mapping

The API layer translates C++ exceptions into OpenAI-compatible HTTP error responses using the `errorResponse` helper [tt-media-server/cpp_server/src/api/llm_controller.cpp 13-14](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/api/llm_controller.cpp#L13-L14)

| Exception / Error | HTTP Status | OpenAI Error Type |
| --- | --- | --- |
| `SessionInFlightException` | 429 Too Many Requests | `rate_limit_exceeded` |
| `QueueFullException` | 429 Too Many Requests | `rate_limit_exceeded` |
| Invalid JSON / Missing `input` | 400 Bad Request | `invalid_request_error` |
| Session Allocation Failure | 503 Service Unavailable | `server_error` |
| No Workers Ready | 503 Service Unavailable | `unhealthy` |
