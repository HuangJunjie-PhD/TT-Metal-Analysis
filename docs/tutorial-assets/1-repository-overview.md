## Purpose and Scope

The `tutorial-assets` repository serves as a centralized storage location for image assets used across TT-NN and TT-Metalium documentation and tutorials. This repository provides visual content that supports the broader Tenstorrent documentation ecosystem, specifically targeting materials published on `docs.tenstorrent.com`.

This document covers the repository's role within the Tenstorrent ecosystem, its organizational structure, and asset management workflows. For detailed information about the TT-NN Visualizer interface screenshots stored in this repository, see [TT-NN Visualizer Documentation](https://deepwiki.com/tenstorrent/tutorial-assets/2-tt-nn-visualizer-documentation). For technical infrastructure details including Git LFS configuration, see [Repository Infrastructure](https://deepwiki.com/tenstorrent/tutorial-assets/3-repository-infrastructure). For licensing terms and proper usage guidelines, see [Licensing and Usage](https://deepwiki.com/tenstorrent/tutorial-assets/4-licensing-and-usage).

## Integration Points

The `tutorial-assets` repository serves as a critical integration point within the Tenstorrent documentation pipeline:

*   **Content Separation**: Technical content originates from `tt-metal` repository while visual content is centralized here
*   **Documentation Targets**: Supports `TTNNDocs`, `TTMetaliumDocs`, and `TTNNTutorials` content streams
*   **Publishing Pipeline**: Assets flow through markdown documentation to `docs.tenstorrent.com`
*   **Version Control**: Git LFS ensures efficient handling of large binary assets while maintaining version history

The repository's architecture enables independent management of visual assets while supporting integration across multiple documentation products within the Tenstorrent ecosystem.
