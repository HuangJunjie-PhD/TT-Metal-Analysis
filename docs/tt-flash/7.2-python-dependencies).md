# Python Dependencies

Relevant source files
*   [pyproject.toml](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml)
*   [setup.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py)

This page documents the standard Python library dependencies used by tt-flash, excluding the Tenstorrent-specific ecosystem dependencies. For Tenstorrent ecosystem dependencies (pyluwen, tt_tools_common, tt-kmd, tt-smi), see [Tenstorrent Ecosystem Dependencies](https://deepwiki.com/tenstorrent/tt-flash/7.1-tenstorrent-ecosystem-dependencies).

The dependencies fall into three categories: core runtime libraries for YAML/TOML parsing and terminal output formatting, Python version compatibility shims for older Python versions, and development tools for code quality enforcement.

## Core Runtime Dependencies

tt-flash relies on three primary Python libraries for its core functionality:

### pyyaml

**Version:**`6.0.2` (exact pin)

**Purpose:** YAML parsing for firmware metadata and configuration files

**Usage:**

*   Parsing `fw_defines.yaml` firmware message type definitions [tt_flash/chip.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/chip.py)
*   Loading manifest data from firmware bundles [tt_flash/flash.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py)
*   Processing system configuration files

The version is strictly pinned to ensure consistent parsing behavior across deployments. The library provides the `yaml.safe_load()` function used throughout the codebase for secure YAML deserialization.

### tabulate

**Version:**`0.9.0` (exact pin)

**Purpose:** Formatted table output for terminal display

**Usage:**

*   Formatting device enumeration tables in verification output [tt_flash/main.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py)
*   Creating aligned columns for firmware version comparisons
*   Pretty-printing chip status information

The `tabulate()` function generates ASCII tables with automatic column width calculation and alignment, used extensively in the `verify` command output.

### tomli

**Version:**`2.0.1` (exact pin)

**Python Version Constraint:**`python_version < '3.11'`

**Purpose:** TOML parsing for older Python versions

**Usage:**

*   Reading `pyproject.toml` configuration in `setup.py`[setup.py 8-12](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L8-L12)
*   Provides backport of Python 3.11's native `tomllib` module

This dependency is conditionally installed only on Python versions before 3.11, which introduced native TOML parsing in the standard library. Once installed, the codebase imports it as `tomli` for compatibility.

**Sources:**[pyproject.toml 24-28](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L24-L28)[setup.py 8-12](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L8-L12)

## Python Version Compatibility Shims

tt-flash maintains compatibility with Python 3.10+ officially, with legacy support mechanisms for older versions. Two compatibility shims handle missing standard library features:

### importlib-metadata

**Version:**`>= 1.4.0`

**Python Version Constraint:**`python_version < '3.8'`

**Purpose:** Backport of package metadata introspection APIs

This dependency provides `importlib.metadata` functionality for Python versions before 3.8. The module enables reading package version information and entry points at runtime. While tt-flash officially requires Python 3.10+, this shim exists for legacy compatibility layers.

### importlib-resources

**Version:**`>= 1.3.0`

**Python Version Constraint:**`python_version < '3.9'`

**Purpose:** Backport of resource file access APIs

This dependency provides the `files()` function from `importlib.resources`, which allows accessing package data files (such as `fw_defines.yaml`) in a filesystem-agnostic way. Python 3.9+ includes this functionality in the standard library.

**Sources:**[pyproject.toml 30-34](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L30-L34)

## Development Dependencies

Development dependencies are specified in the `[project.optional-dependencies]` section and are not required for runtime usage:

### black

**Purpose:** Code formatting enforcement

**Version Constraints (Python-version specific):**

| Python Version | Black Version |
| --- | --- |
| 3.7 | `23.3.0` |
| 3.8 | `24.8.0` |
| 3.9+ | `24.10.0` |

Black is the opinionated Python code formatter that enforces consistent style across the codebase. Different versions are specified for different Python versions due to black's own Python compatibility requirements. Installation: `pip install tt-flash[dev]`

### basedpyright

**Purpose:** Static type checking

**Configuration:**[pyproject.toml 67-69](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L67-L69)

Basedpyright is configured to target Python 3.7 compatibility with "standard" type checking mode. This ensures type annotations and hints remain compatible with older Python versions even though tt-flash officially requires 3.10+.

`[tool.basedpyright]pythonVersion = "3.7"typeCheckingMode = "standard"`
**Sources:**[pyproject.toml 37-42](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L37-L42)[pyproject.toml 67-69](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L67-L69)

## Dependency Version Strategy

**Diagram: Version Constraint Strategy by Dependency Type**

The project uses three version constraint strategies:

1.   **Exact Pins (`==`)**: Applied to stable third-party libraries (pyyaml, tabulate, tomli) to ensure reproducible builds and prevent unexpected breaking changes from minor version updates.

2.   **Compatible Release (`~=`)**: Applied to Tenstorrent ecosystem libraries (covered in [7.1](https://deepwiki.com/tenstorrent/tt-flash/7.1-tenstorrent-ecosystem-dependencies)) using the `~=` operator, which allows patch version updates (e.g., `~=1.5` allows `>=1.5, <2.0`).

3.   **Minimum Version (`>=`)**: Applied to compatibility shims where any version meeting the minimum requirement is acceptable since they're only needed for specific Python versions.

**Sources:**[pyproject.toml 23-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L35)

## Python Version Support Matrix

**Diagram: Python Version Support and Dependency Activation**

Python 3.10 requires the `tomli` backport for TOML parsing, while Python 3.11+ uses the native `tomllib` standard library module. The legacy shims (`importlib-metadata`, `importlib-resources`) are only activated for Python versions below the officially supported range, indicating historical compatibility concerns.

**Sources:**[pyproject.toml 6](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L6-L6)[pyproject.toml 28](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L28-L28)[pyproject.toml 31-34](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L31-L34)

## Installation and Requirements

### Runtime Installation

The standard installation includes all necessary runtime dependencies:

`pip install tt-flash`
This installs pyyaml, tabulate, tt_tools_common, pyluwen, and conditionally installs tomli (Python <3.11) and compatibility shims (Python <3.9).

### Development Installation

For development work with code formatting and type checking:

`pip install tt-flash[dev]`
This adds black (version appropriate for your Python version) for code formatting.

### Build Requirements

The `[build-system]` section specifies dependencies needed only during package building:

| Dependency | Version | Purpose |
| --- | --- | --- |
| `setuptools` | `>=43.0.0` | PEP 517/518 build backend |
| `wheel` | (any) | Wheel package format support |

These are only required when building from source using `python -m build` or similar tools, not for end-user installations.

**Sources:**[pyproject.toml 74-81](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L74-L81)

## Dependency Declaration Location

Dependencies are declared in two files for compatibility with different packaging ecosystems:

### pyproject.toml (Primary)

[pyproject.toml 23-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L35) contains the authoritative dependency declarations using the PEP 621 `[project]` table format. This is the modern Python packaging standard (PEP 517/518/621).

### setup.py (Legacy Compatibility)

[setup.py 1-25](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L1-L25) exists solely for Ubuntu 22.04 packaging tool compatibility. It reads `pyproject.toml` using `tomli` and extracts dependency information, ensuring older `setuptools` versions can still process the package.

The `setup.py` file explicitly notes:

`"""Compatibility setup.py for Ubuntu 22.04 packaging tools.Uses setuptools_scm for dynamic versioning from git tags."""`
This dual-declaration approach ensures the package builds correctly in both modern Python environments (using `pyproject.toml` directly) and legacy Debian/Ubuntu packaging workflows (requiring `setup.py`).

**Sources:**[pyproject.toml 1-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L1-L35)[setup.py 1-25](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L1-L25)
