## Types of Contributions

### Bug Reports

Bug reports must be filed using the issue board template. The template requires:

*   Example test case and command demonstrating the issue
*   Environment information (OS, version, container status)
*   Minimum reproduction steps
*   Expected behavior description

## Pull Request Standards

### PR Requirements Checklist

| Requirement | Description | Enforced By |
| --- | --- | --- |
| **Issue Linkage** | PR must link to existing issue | Manual review |
| **Clear Description** | Accurate description of changes | PR template |
| **Descriptive Commits** | Commit messages explain changes | Manual review |
| **CI Checks Pass** | All automated checks green | GitHub Actions |
| **Tests Included** | New features have unit tests | Manual review + CI |
| **Single Logical Change** | Focused, not mixing concerns | Manual review |
| **Rebased on main** | Current with latest main branch | Git check |

**Sources:**[.github/CONTRIBUTING.md 38-46](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CONTRIBUTING.md?plain=1#L38-L46)[.github/ISSUE_TEMPLATE/pull_request_template.md 1-19](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/ISSUE_TEMPLATE/pull_request_template.md?plain=1#L1-L19)

### PR Description Structure

The PR template [.github/ISSUE_TEMPLATE/pull_request_template.md 1-19](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/ISSUE_TEMPLATE/pull_request_template.md?plain=1#L1-L19) requires:

*   **Related Issues:** Link using `#issue_number`
*   **Type of Change:** Bug fix, new feature, documentation, refactoring, performance, tests

## Code Requirements

### Python Code Standards

### Requirements Summary

1.   **Python Version:** Use Python 3.9 features
2.   **Type Hints:** Apply type hints wherever possible for better IDE support and type checking
3.   **Import Style:** Use absolute imports in `riescue/` source code
4.   **Error Handling:** Handle errors gracefully with appropriate exceptions
5.   **Global Variables:** Avoid using global variables
6.   **Documentation:** Follow [Sphinx Docstring Format](https://sphinx-rtd-tutorial.readthedocs.io/en/latest/docstrings.html)

**Sources:**[.github/CONTRIBUTING.md 54-58](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CONTRIBUTING.md?plain=1#L54-L58)

### Code Quality Tools

RiESCUE enforces uniform code style using three tools:

*   **`flake8`:** Python linter for style and error checking
*   **`pyright`:** Static type checker
*   **`black`:** Code formatter for consistent formatting

All contributions must pass these checks. Detailed configuration and usage: [5.2](https://deepwiki.com/tenstorrent/riescue/5.2-code-quality-and-linting)

**Sources:**[.github/CONTRIBUTING.md 60-63](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/CONTRIBUTING.md?plain=1#L60-L63)

## Code of Conduct

All contributors must follow the Contributor Covenant Code of Conduct v2.0. Key principles:

*   Harassment-free experience for everyone
*   Respectful of differing opinions and experiences
*   Graceful acceptance of constructive feedback
*   Focus on community benefit

Violations can be reported to community leaders. Full details and enforcement guidelines: [5.1.2](https://deepwiki.com/tenstorrent/riescue/5.1.2-code-of-conduct)
