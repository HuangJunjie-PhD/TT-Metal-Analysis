## Purpose and Scope

This document describes the compiler optimization test suite, which validates the SFPI-enhanced GCC compiler's optimization passes, particularly the **move pass** that minimizes register-to-register moves in SFPU code. These tests ensure that the compiler efficiently allocates registers and avoids unnecessary move instructions that would degrade performance.

For general SFPU instruction-level testing, see [SFPU Kernel Tests](https://deepwiki.com/tenstorrent/sfpi/6.2-sfpu-kernel-tests). For information about the overall test infrastructure, see [Test Infrastructure Overview](https://deepwiki.com/tenstorrent/sfpi/6.1-test-infrastructure-overview).

## Move Pass Optimization Overview

The SFPU move pass is a compiler optimization that operates on two-operand instructions to minimize register moves. Because many SFPU instructions overwrite their destination operand, the compiler must manage register allocation carefully to preserve values that are still needed.

### Optimization Strategies

The move pass employs three primary strategies:

| Strategy | Condition | Action | Example |
| --- | --- | --- | --- |
| **Operand Swap** | Operation is commutative and dst is not live after | Swap operands to avoid move | `z = x + y` → use y's register for z |
| **No-op** | Operands already in optimal order | Do nothing | `z = y + x` where x will be dst |
| **Move Injection** | Cannot swap or reorder | Inject explicit SFPMOV | `z = x - y` → SFPMOV then operate |

### Non-Commutative Operation Tests

Tests for operations where operand order matters and swapping is not allowed:

| Test Function | Operation | Expected Assembly | Reason |
| --- | --- | --- | --- |
| `replace_sub1()` | `z = y - x` | `SFPMOV L0, L1` + `SFPIADD L0, L2, 0, 6` | Subtraction not commutative |
| `replace_shft()` | `y = x << 1` | `SFPMOV L0, L1` + `SFPSHFT L0, L0, 1, 1` | Shift requires specific operand |

### Example: Remove vs Replace Optimization

The following table shows how the same logical operation produces different assembly depending on liveness:

| Test Case | C++ Code | Expected Assembly | Optimization Applied |
| --- | --- | --- | --- |
| `remove_add1()` | `vInt z = x + y;` `dst_reg[0] = x;` `dst_reg[2] = z;` | `SFPLOADI L1, 1, 4` `SFPLOADI L0, 2, 4` `SFPIADD L0, L1, 0, 4` | Operand swap, no move |
| `replace_add3()` | `vInt z = x + y;` `dst_reg[0] = x;` `dst_reg[1] = y;` `dst_reg[2] = z;` | `SFPLOADI L2, 1, 4` `SFPLOADI L1, 2, 4` `SFPMOV L0, L1, 0` `SFPIADD L0, L2, 0, 4` | Move injection required |

**Sources:**[tests/wormhole/sfpi/regmov.cc 19-53](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/regmov.cc#L19-L53)[tests/wormhole/sfpi/gold/regmov.S 10-41](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/regmov.S#L10-L41)

### Key Assembly Patterns

The gold standard validates these specific optimization patterns:

| Pattern | Suboptimal | Optimized | Test Functions |
| --- | --- | --- | --- |
| **Direct operation** | `SFPMOV + SFPIADD` | `SFPIADD` (swapped) | `remove_add1`, `remove_and1` |
| **Live preservation** | Long GCC move | `SFPMOV + operation` | `replace_add3`, `replace_or3` |
| **Constant registers** | Extra load | Direct use of Ln register | `swap_const_reg_or`, `move_const_reg_or` |
| **Shift operations** | GCC-generated move | `SFPMOV + SFPSHFT` | `replace_shft` |
| **Loop invariants** | Move in loop | Move outside loop | `bb_loop1`, `bb_loop2` |

**Sources:**[tests/wormhole/sfpi/gold/regmov.S 10-455](https://github.com/tenstorrent/sfpi/blob/ed989b9d/tests/wormhole/sfpi/gold/regmov.S#L10-L455)

The `unnecessarymov.cc` file contains tests for identifying and eliminating compiler-generated internal moves that serve no optimization purpose:

These tests help identify cases where the compiler generates unnecessary register moves that could be eliminated through better register allocation or code generation strategies.

### Compiler Warning Validation

The `warn.txt` and `warn_spill.txt` files validate that the compiler produces appropriate errors and warnings:

| Error Type | File | Example | Purpose |
| --- | --- | --- | --- |
| Memory access | `warn.txt:6-8` | `error: cannot write sfpu vector to memory` | Prevent illegal memory operations |
| Stack passing | `warn.txt:33` | `error: invalid declaration for function 'argcall'` | Enforce register-only parameter passing |
| Uninitialized use | `warn.txt:23-31` | `warning: '<anonymous>' is used uninitialized` | Detect potential bugs |
| Register spill | `warn_spill.txt:7-9` | `fatal error: cannot store sfpu register` | Prevent impossible register allocations |
| Malformed code | `warn.txt:42-43` | `error: malformed program, sfpsetcc_v outside of pushc/popc` | Enforce instruction constraints |
