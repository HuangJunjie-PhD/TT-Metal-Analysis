## Purpose and Scope

The LLK Reference Implementation provides a standardized, reference implementation of the Low-Level Kernel (LLK) API. It serves as a baseline implementation against which other LLK implementations can be compared and tested. This implementation prioritizes correctness, completeness, and clarity over performance optimizations.

For information about the LLK API itself, see [LLK API](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.1-llk-api). For other implementations such as the basic implementation or compute-optimized implementation, see [LLK Basic Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.2-llk-basic-implementation) and [LLK Compute Implementation](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3.4-llk-compute-implementation) respectively.

## Architecture Overview

The Reference Implementation serves as a bridge between the LLK API interface and the underlying hardware functionality provided by the core support libraries. It implements the complete set of features defined in the LLK API specification, providing a reliable and well-documented implementation that serves as the standard against which other implementations are measured.

Sources: [tensix/whb0/prj/build_llk_ref.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_llk_ref.sh)[tensix/whb0/prj/build_demo_llk_ref.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_ref.sh)

## Implementation Structure

The Reference Implementation is organized as a collection of components that implement the various subsystems defined by the LLK API. The implementation is designed to be complete and correct, following the API specification precisely.

Sources: [tensix/whb0/prj/build_llk_ref.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_llk_ref.sh)

## Key Components

The Reference Implementation includes implementations for several key subsystems that together form a complete LLK implementation:

| Component | Description | Primary Responsibilities |
| --- | --- | --- |
| Memory Management | Handles allocation and deallocation of memory resources | Buffer allocation, memory mapping, address translation |
| Execution Control | Manages the execution of tasks and kernels | Task scheduling, dispatch, context switching |
| Synchronization | Provides primitives for synchronization between tasks | Semaphores, mutexes, barriers |
| I/O Operations | Manages input/output operations | Device I/O, data transfer, peripherals access |
| Error Handling | Manages error detection and reporting | Error codes, exception handling, logging |

## Build System

The Reference Implementation is built using a dedicated build script that compiles the source files and creates a static library (`llk_ref.a`). The build process includes the following steps:

1.   Compilation of source files from the LLK API directory
2.   Creation of the static library
3.   Cleanup of temporary object files

## Building the Reference Implementation

To build the Reference Implementation, run the `build_llk_ref.sh` script:

```
$ ./build_llk_ref.sh
```

This script compiles the necessary source files and creates the `llk_ref.a` library in the `../lib` directory.

To build the demo application, run the `build_demo_llk_ref.sh` script:

```
$ ./build_demo_llk_ref.sh
```

This script compiles the demo application and creates the `demo_llk_ref` executable in the `../bin` directory.

Sources: [tensix/whb0/prj/build_llk_ref.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_llk_ref.sh)[tensix/whb0/prj/build_demo_llk_ref.sh](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_ref.sh)

## Usage in Applications

Applications that want to use the LLK Reference Implementation should:

1.   Include the necessary header files from the LLK API and Reference Implementation directories
2.   Link against the `llk_api.a` library that contains the API definitions
3.   Ensure that the supporting libraries (`core.a`, `schedule.a`, `sfpi.a`) are also linked

The demo application in `$SRC/demo/llk_ref/` provides a practical example of how to use the Reference Implementation in a real application.
