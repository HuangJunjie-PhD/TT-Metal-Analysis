## Purpose and Scope

This document provides detailed documentation of the demo applications that showcase the different LLK (Low-Level Kernel) implementations within the Tensix ISA Simulator. These demo applications serve as practical examples of how to use the LLK API and its various implementations.

For information about the LLK system itself, see [LLK System](https://deepwiki.com/tenstorrent/tensix-isa-simulator/3-llk-system). For information about r37-specific demo applications, see [r37 Demo Applications](https://deepwiki.com/tenstorrent/tensix-isa-simulator/6.2-r37-demo-applications).

## Overview of LLK Demo Applications

The Tensix ISA Simulator includes multiple demo applications that demonstrate the usage and capabilities of different LLK implementations. These demos serve as educational tools and reference implementations for developers working with the LLK system.

### Available Demo Applications

| Demo Application | Purpose | Related LLK Implementation |
| --- | --- | --- |
| demo_llk_basic | Demonstrates the basic functionality of the LLK system | LLK Basic Implementation |
| demo_llk_ref | Demonstrates the reference implementation of the LLK system | LLK Reference Implementation |

## Demo LLK Basic Application

### Purpose and Functionality

The `demo_llk_basic` application demonstrates the usage of the basic LLK implementation. It serves as a simple introduction to the LLK system's core functionality, showcasing basic operations and workflows.

### Build Process

The `demo_llk_basic` application is built using the following dependencies:

The build script compiles all source files in the `demo/llk_basic` directory and links them with the `llk_basic.a`, `core.a`, and `schedule.a` libraries.

Sources: [tensix/whb0/prj/build_demo_llk_basic.sh 12-17](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_basic.sh#L12-L17)

## Demo LLK Reference Application

### Purpose and Functionality

The `demo_llk_ref` application demonstrates the usage of the reference LLK implementation. It provides a more comprehensive example of the LLK system's capabilities, serving as a reference for developers implementing their own LLK-based applications.

The build script compiles all source files in the `demo/llk_ref` directory and links them with the `llk_api.a`, `core.a`, `schedule.a`, and `sfpi.a` libraries.

## Usage and Execution

To build and run the LLK demo applications:

1.   Navigate to the project directory
2.   Run the appropriate build script: 
    *   `./build_demo_llk_basic.sh` for the basic demo
    *   `./build_demo_llk_ref.sh` for the reference demo

3.   Execute the compiled binary from the bin directory: 
    *   `../bin/demo_llk_basic` for the basic demo
    *   `../bin/demo_llk_ref` for the reference demo

Sources: [tensix/whb0/prj/build_demo_llk_basic.sh 1-20](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_basic.sh#L1-L20)[tensix/whb0/prj/build_demo_llk_ref.sh 1-25](https://github.com/tenstorrent/tensix-isa-simulator/blob/afb83529/tensix/whb0/prj/build_demo_llk_ref.sh#L1-L25)

## Conclusion

The LLK demo applications provide practical examples of how to use the different LLK implementations in the Tensix ISA Simulator. They serve as valuable learning resources for developers working with the LLK system, demonstrating both basic and more advanced usage patterns.

The demo applications follow a consistent structure and build process, making them easy to understand and modify for specific needs. By examining these demos, developers can gain insights into the LLK system's capabilities and how to leverage them effectively in their own applications.
