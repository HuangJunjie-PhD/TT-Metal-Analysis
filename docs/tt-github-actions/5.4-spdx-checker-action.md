The `spdx-checker` is a composite GitHub Action designed to enforce standardized SPDX license headers across Tenstorrent repositories. It leverages the Espressif `check-copyright` tool to validate that source files contain approved license identifiers and correct copyright holder information. The action is designed to ensure consistency across the organization by locking down license configurations while allowing individual repositories to define their own file exclusion patterns.

## System Architecture and Data Flow

The action operates by merging a hardcoded set of corporate compliance rules with repository-specific ignore patterns to generate a transient configuration file for the underlying validation engine.

### Execution Logic

The action follows a linear execution path within a composite runner:

1.   **Environment Setup**: Installs `pyyaml` and the `check-copyright` tool from the Espressif GitHub repository [[.github/actions/spdx-checker/action.yml:34-39]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).
2.   **Configuration Generation**: Executes `merge_config.py` to produce `spdx_check_config.yaml`[[.github/actions/spdx-checker/action.yml:41-53]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).
3.   **Validation**: Invokes `python -m check_copyright` using the generated config and user-provided arguments like `dry_run` and `target_directory`[[.github/actions/spdx-checker/action.yml:58-73]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).

## Implementation Details

### Configuration Management (`merge_config.py`)

The `merge_config.py` script acts as a gatekeeper for corporate policy. It defines a `DEFAULT_CONFIG` dictionary that contains the authoritative list of allowed licenses and the required notice format for new files.

*   **Allowed Licenses**: The action currently approves `Apache-2.0`, `Apache-2.0 WITH LLVM-exception`, `MIT`, `BSD-2-Clause`, and `BSD-3-Clause`[[.github/actions/spdx-checker/merge_config.py:21-21]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).
*   **Copyright Holder**: The standard holder is defined as "Tenstorrent USA, Inc." [[.github/actions/spdx-checker/merge_config.py:24-30]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action). Note that the checker also accepts "Tenstorrent AI ULC" for legacy files [[.github/actions/spdx-checker/README.md:21-21]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).
*   **Restricted Overrides**: The `merge_configs` function specifically only extracts the `ignore` key from the user-provided YAML [[.github/actions/spdx-checker/merge_config.py:49-55]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action). Any attempts by a repository to modify `allowed_licenses` or `new_notice_c` in their local config file are ignored during the merge process [[.github/actions/spdx-checker/README.md:124-125]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).

### Action Interface

The action is configured via `action.yml` with the following key inputs:

| Input | Description | Default |
| --- | --- | --- |
| `ignore_config_file` | Path to a YAML file containing repository-specific ignore patterns [[.github/actions/spdx-checker/action.yml:5-8]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action). | `''` |
| `target_directory` | The root directory to begin scanning for source files [[.github/actions/spdx-checker/action.yml:9-12]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action). | `.` |
| `dry_run` | If `true`, the action reports violations but does not return a non-zero exit code [[.github/actions/spdx-checker/action.yml:21-24]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action). | `false` |

## Usage and Configuration

### Repository-Specific Ignores

Repositories can skip directories such as `third_party/`, `build/`, or specific file extensions by providing a YAML file. An example template is provided in the action directory [[.github/actions/spdx-checker/check_copyright_config.yaml:13-28]](https://deepwiki.com/tenstorrent/tt-github-actions/5.4-spdx-checker-action).
