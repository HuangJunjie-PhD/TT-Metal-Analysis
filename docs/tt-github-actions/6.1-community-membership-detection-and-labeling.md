# Community Membership Detection & Labeling

Relevant source files
*   [.github/actions/collect_data/download_workflow_data.sh](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/collect_data/download_workflow_data.sh)

This section provides a deep dive into the automated workflows used to distinguish between internal Tenstorrent organization members and external community contributors. These workflows facilitate the automatic labeling of issues and pull requests, ensuring that community-driven contributions are easily identifiable for triage and the bounty program.

## Overview

The community detection logic is implemented across two primary workflows:

1.   `on-community-issue.yml`: A lightweight workflow focused on membership detection and debugging.
2.   `issues-community-autotag.yml`: A robust, production-ready workflow that performs deep membership checks (including collaborator status) and manages the lifecycle of the `community` label.

These workflows rely on the GitHub REST API to verify if a user belongs to the `tenstorrent` organization.

## Membership Detection Logic

The core mechanism for detection involves querying the GitHub API's membership endpoint. Both workflows use `curl` to send a `GET` request to the organization members endpoint.

### API Interaction Flow

The detection logic interprets HTTP status codes from the GitHub API:

*   **204 No Content**: The user is a member of the organization [.github/workflows/on-community-issue.yml 46-48](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L46-L48)
*   **404 Not Found**: The user is not a member of the organization [.github/workflows/on-community-issue.yml 49-54](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L49-L54)

In `issues-community-autotag.yml`, if a 404 is received, the workflow performs a secondary check to see if the user is an external collaborator on the specific repository [.github/workflows/issues-community-autotag.yml 99-103](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L99-L103)

## Workflow Implementations

### on-community-issue.yml

This workflow is designed to be called externally or triggered by local issue/PR events [.github/workflows/on-community-issue.yml 5-15](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L5-L15) It sets environment variables based on the event type (`issues`, `pull_request`, or `push`) to capture the actor's username and the item number [.github/workflows/on-community-issue.yml 25-38](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L25-L38)

**Key Variables:**

*   `is_member`: An output of the `is_organization_member` step, set to `true` or `false`[.github/workflows/on-community-issue.yml 48-53](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L48-L53)
*   `TT_BOT_ORG_MEMBERS_READ_TOKEN`: A Personal Access Token (PAT) required to read private organization membership [.github/workflows/on-community-issue.yml 43](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L43-L43)

Sources: [.github/workflows/on-community-issue.yml 1-82](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L1-L82)

### issues-community-autotag.yml

This is a more comprehensive implementation that includes label management. It ensures that the `community` label (Color: `2DEB44`) exists in the repository before attempting to apply it [.github/workflows/issues-community-autotag.yml 11-182](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L11-L182)

**Detection Logic Steps:**

1.   **Initialize**: Dynamically extracts the organization name from `github.repository_owner`[.github/workflows/issues-community-autotag.yml 35](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L35-L35)
2.   **Org Check**: Queries `https://api.github.com/orgs/${ORG_NAME}/members/${USERNAME}`[.github/workflows/issues-community-autotag.yml 80](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L80-L80)
3.   **Collaborator Check**: If the Org Check fails, it queries `https://api.github.com/repos/${{ github.repository }}/collaborators/${USERNAME}`[.github/workflows/issues-community-autotag.yml 103](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L103-L103)
4.   **Label Management**: Uses `POST` requests to create the label if missing and `POST` to `/issues/${NUMBER}/labels` to apply it [.github/workflows/issues-community-autotag.yml 163-173](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L163-L173)

## Token & Permission Requirements

The membership detection process requires specific permissions because organization membership (especially private membership) is not visible to the default `GITHUB_TOKEN`.

### Required Secrets

*   **`TT_BOT_ORG_MEMBERS_READ_TOKEN`**: Used in `on-community-issue.yml`. Must have `read:org` permissions [.github/workflows/on-community-issue.yml 43](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L43-L43)
*   **`AUTOLABEL_COMMUNITY_ISSUES`**: Used in `issues-community-autotag.yml`. Requires `read:org` to check membership and `repo` permissions to manage labels on the target repository [.github/workflows/issues-community-autotag.yml 5-67](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L5-L67)

### Workflow Permissions

The jobs themselves require write permissions for issues and pull requests to apply labels:

`permissions:  issues: write  pull-requests: write`
Sources: [.github/workflows/on-community-issue.yml 21-23](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/on-community-issue.yml#L21-L23)[.github/workflows/issues-community-autotag.yml 18-21](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/%20.github/workflows/issues-community-autotag.yml#L18-L21)
