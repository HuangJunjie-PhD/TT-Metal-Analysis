This document provides a comprehensive guide for setting up a local development environment for tt-tools-common. It covers installing the package in editable mode, managing development dependencies, and configuring the environment for both software-only and hardware-based testing.

For information about running tests after setup, see [Testing Guide](https://deepwiki.com/tenstorrent/tt-tools-common/6.1-testing-guide). For code quality tools configuration, see [Code Style and Quality Tools](https://deepwiki.com/tenstorrent/tt-tools-common/6.2-code-style-and-quality-tools). ### Issue: Hardware Tests Skip or Fail

**Symptom:**

```
ERROR: No hardware detected
```

**Solutions:**

1.   Check driver is loaded: `lsmod | grep tenstorrent`
2.   Verify driver version: `modinfo tenstorrent | grep version` (should be ≥1.26.0)
3.   Check device nodes: `ls /dev/tenstorrent/`
4.   Test pyluwen directly: `python3 -c "from pyluwen import detect_chips; print(detect_chips())"`

### Issue: Package Data Files Not Found

**Symptom:**

```
FileNotFoundError: data/chip_data/fw_defines.yaml not found
```
