This guide provides step-by-step instructions for installing and configuring Tenstorrent hardware and software. It covers physical setup, the automated `tt-installer` path, manual installation of the driver stack, firmware updates, and system verification.

## Prerequisites

Before beginning installation, ensure your host machine meets the following requirements:

| Requirement | Specification |
| --- | --- |
| **Operating System** | Ubuntu 22.04 LTS (Jammy Jellyfish) recommended [core/getting-started/README.md 27-29](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L27-L29) |
| **Privileges** | Administrator (sudo) access [core/getting-started/README.md 24](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L24-L24) |

### Quick Installation (Recommended)

Tenstorrent recommends using the `tt-installer` script to automate the setup of drivers, firmware, and system tools.

1.   **Execute the installer:**

`/bin/bash -c "$(curl -fsSL https://github.com/tenstorrent/tt-installer/releases/latest/download/install.sh)"`
[core/getting-started/README.md 54-58](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L54-L58)

2.   **Select Components:**

    *   **TT-Metalium Slim Container:** Appropriate for users only needing `tt-nn`[core/getting-started/README.md 94-102](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L94-L102)
    *   **TT-Metalium Model Demos Container:** A ~10GB container including full source code and prebuilt models [core/getting-started/README.md 109-115](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L109-L115)

3.   **Python Environment:** The installer defaults to creating a virtual environment at `/home/$USER/.tenstorrent-venv`[core/getting-started/README.md 125-132](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/README.md?plain=1#L125-L132)

### Manual Installation

If you prefer to install components individually, follow the sequence below.

#### 1. Install System Dependencies

`sudo apt update && sudo apt install -y wget git python3-pip dkms cargo`
Sources: [core/getting-started/manual-software-install.md 12-16](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L12-L16)

#### 2. Install Kernel-Mode Driver (TT-KMD)

The driver is managed via DKMS to ensure it persists across kernel updates.

`git clone https://github.com/tenstorrent/tt-kmd.gitcd tt-kmdsudo dkms add .sudo dkms install tenstorrent/1.34sudo modprobe tenstorrent`
Sources: [core/getting-started/manual-software-install.md 20-31](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L20-L31)[syseng/kmd.version 1](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/kmd.version#L1-L1)

#### 3. Update Device Firmware

Use `tt-flash` to apply the latest firmware bundle.

`pip install git+https://github.com/tenstorrent/tt-flash.gitwget https://github.com/tenstorrent/tt-system-firmware/releases/download/v18.4.0/fw_pack-18.4.0.fwbundlett-flash --fw-tar fw_pack-18.4.0.fwbundle`
_Note: If the existing firmware is too old, use the `--force` flag_[core/getting-started/manual-software-install.md 63-65](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L63-L65) Sources: [core/getting-started/manual-software-install.md 33-67](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L33-L67)[syseng/firmware.version 1](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/firmware.version#L1-L1)

#### 4. Configure HugePages

HugePages are required for optimized memory performance during device communication.

`wget https://github.com/tenstorrent/tt-system-tools/releases/download/v1.3.1/tenstorrent-tools_1.3.1_all.debsudo dpkg -i tenstorrent-tools_1.3.1_all.debsudo systemctl enable --now tenstorrent-hugepages.servicesudo systemctl enable --now 'dev-hugepages\x2d1G.mount'sudo reboot`
Sources: [core/getting-started/manual-software-install.md 69-85](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L69-L85)[syseng/sys_tools.version 1](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/sys_tools.version#L1-L1)

## Multi-Card Configuration (Wormhole™ Only)

For systems with multiple Wormhole processors (e.g., TT-LoudBox, TT-QuietBox), a mesh topology must be configured for inter-card communication.

`pip install git+https://github.com/tenstorrent/tt-topology.gittt-topology -l mesh`
_Note: QuietBox and LoudBox typically ship pre-configured. Only run this if the topology has been modified_[core/getting-started/manual-software-install.md 89-91](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L89-L91)

## Verification

After installation and reboot, verify the system status using `tt-smi`.

`tt-smi`
The output should display:

*   Detected Tenstorrent devices (e.g., n150, p150).
*   Current firmware versions.
*   Telemetry (Temperature, Power, Voltage).
*   Any host compatibility warnings.

Sources: [core/getting-started/manual-software-install.md 106-114](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/manual-software-install.md?plain=1#L106-L114)[syseng/softwaresetup.md 75-83](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L75-L83)

## Running Model Demos

To verify the software stack is functional, run a sample operation inside the demos container.

1.   **Enter the container:**`tt-metalium-models`
2.   **Run a test program:**`python ttnn/ttnn/examples/usage/run_op_on_device.py`

Successful execution confirms that the `tt-kmd` driver, firmware, and `ttnn` library are communicating correctly with the hardware.

Sources: [core/getting-started/model-demos.md 34-54](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/model-demos.md?plain=1#L34-L54)
