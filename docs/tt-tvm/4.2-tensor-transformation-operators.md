This document describes the tensor transformation operators in Relay, which manipulate tensor shapes, layouts, and data organization without performing arithmetic computation. These operators handle tasks like reshaping tensors, selecting elements, concatenating data, and converting data types. For neural network-specific operators like convolution and pooling, see [Neural Network Operators](https://deepwiki.com/tenstorrent/tt-tvm/4.1-neural-network-operators). For the type system and layout transformations, see [Layout Transformation System](https://deepwiki.com/tenstorrent/tt-tvm/4.4-layout-transformation-system).

## Overview

Tensor transformation operators form a core category in Relay's operator library, providing over 40 operators for manipulating tensors. These operators are defined in [src/relay/op/tensor/transform.cc](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc) and exposed through Python APIs in [python/tvm/relay/op/transform.py](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/transform.py) Unlike computational operators that perform arithmetic operations, transformation operators primarily reorganize data or metadata.

### Type Relation Functions

Type relations infer output tensor types from input types and operator attributes. Each operator implements a type relation function that validates inputs and computes output shapes.

**Example: ReshapeRel**

The `ReshapeRel` function [src/relay/op/tensor/transform.cc 799-855](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L799-L855) implements sophisticated shape inference with special values:

*   `0`: Copy dimension from input
*   `-1`: Infer dimension to maintain total size
*   `-2`: Copy all remaining dimensions
*   `-3`: Merge two consecutive dimensions
*   `-4`: Split one dimension into two

## Shape Manipulation Operators

Shape manipulation operators change tensor dimensions without modifying underlying data values.

### Core Shape Operators

| Operator | Function | Attributes | Example |
| --- | --- | --- | --- |
| `reshape` | Change tensor shape | `newshape`, `allowzero` | `(2,3,4) → (6,4)` |
| `expand_dims` | Add singleton dimensions | `axis`, `num_newaxis` | `(2,3) → (2,1,3)` |
| `squeeze` | Remove singleton dimensions | `axis` | `(2,1,3,1) → (2,3)` |
| `transpose` | Permute dimensions | `axes` | `(2,3,4) → (4,3,2)` |
| `reshape_like` | Reshape to match another tensor | `lhs_begin`, `lhs_end`, `rhs_begin`, `rhs_end` | Match shapes partially |

## Data Combination Operators

Combination operators merge multiple tensors into a single tensor.

### Concatenate vs Stack

| Operator | Behavior | Example Input | Example Output |
| --- | --- | --- | --- |
| `concatenate` | Join along existing axis | `[(2,3), (2,4)]` axis=1 | `(2,7)` |
| `stack` | Create new axis | `[(2,3), (2,3)]` axis=0 | `(2,2,3)` |

**Concatenate Type Relation**[src/relay/op/tensor/transform.h 47-159](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.h#L47-L159):

*   Validates tuple of tensors as input
*   Checks axis bounds and dimension compatibility
*   Accumulates sizes along concatenation axis
*   All non-concat dimensions must match

**Stack Type Relation**[src/relay/op/tensor/transform.cc 357-415](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L357-L415):

*   Creates new dimension at specified axis
*   All input tensors must have identical shapes
*   Output shape inserts stack dimension

## Type Conversion Operators

Type conversion operators change the data type interpretation of tensors.

### Cast vs Reinterpret

| Operator | Operation | Example |
| --- | --- | --- |
| `cast` | Convert values to new type | `int32(5) → float32(5.0)` |
| `cast_like` | Cast to match another tensor's dtype | Cast data to dtype_like.dtype |
| `reinterpret` | Reinterpret bit pattern | `float32(5.0) → int32(1084227584)` |

**Cast Implementation**[src/relay/op/tensor/transform.cc 133-176](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L133-L176):

*   Changes dtype while converting values
*   Uses `topi::cast` for compute
*   Marked as `kElemWise` pattern

## Special Transformation Operators

### Sliding Window

The `sliding_window` operator [src/relay/op/tensor/transform.cc 59-129](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L59-L129) extracts overlapping windows from a tensor, useful for convolution-like operations:

**Parameters:**

*   `axis`: Starting dimension for window
*   `window_shape`: Size of window in each dimension
*   `strides`: Step size for sliding in each dimension

**Output Shape:**

```
Input: (batch, d1, d2, ..., dn)
Window over dims [axis:], window=(w1, w2, ..., wk), strides=(s1, s2, ..., sk)
Output: (batch, ..., num_windows_d1, num_windows_d2, ..., w1, w2, ..., wk)
```

Where `num_windows_di = (di - wi + si) // si`
