## Purpose and Scope

This document describes the technical details of how Tenstorrent's documentation is built and deployed. It covers the build infrastructure, automation process, and the orchestration script used to generate and publish the documentation website at `docs.tenstorrent.com`. For information about the documentation structure and organization, see [4.2 Documentation Structure](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/4.2%20Documentation%20Structure) For technical details about Sphinx configuration, see [4.3 Sphinx Configuration](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/4.3%20Sphinx%20Configuration)

## System Overview

The Tenstorrent documentation system uses Sphinx to generate HTML documentation from source files. The build process is automated using GitHub Actions, which builds and deploys the documentation when changes are pushed to the `main` branch.

### Build Architecture

The system bridges the "Natural Language Space" of documentation into the "Code Entity Space" via a Python orchestration script `build_docs.py` and a GitHub Actions workflow.

## Build Process Workflow

The build process is a multi-stage pipeline that handles environment setup, version resolution, per-project Sphinx builds, and artifact consolidation.

### Workflow Execution Sequence

Sources: [.github/workflows/pages_deploy.yml 14-65](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/.github/workflows/pages_deploy.yml#L14-L65)[build_docs.py 8-98](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/build_docs.py#L8-L98)

### Version Resolution in CI

The GitHub Actions workflow dynamically fetches the latest release tags from external Tenstorrent repositories to ensure the documentation reflects current software versions.

1.   **TT-KMD**: Fetches from `tenstorrent/tt-kmd` and writes to `syseng/kmd.version`[[.github/workflows/pages_deploy.yml:35-38]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
2.   **Firmware**: Fetches from `tenstorrent/tt-system-firmware` and writes to `syseng/firmware.version`[[.github/workflows/pages_deploy.yml:40-43]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
3.   **System Tools**: Fetches from `tenstorrent/tt-system-tools` and writes to `syseng/sys_tools.version`[[.github/workflows/pages_deploy.yml:45-48]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).

### Orchestration Script Implementation

The `build_docs.py` script performs the heavy lifting of organizing the multi-project output:

*   **`build_doc(project)`**: Executes the `make html` command within the project's subdirectory [[build_docs.py:8-14]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
*   **`move_dir(src, dst)`**: A helper using `subprocess` to move built HTML files from the Sphinx `_build` directory to the final `output/` structure [[build_docs.py:17-19]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
*   **`copy_pdfs(project, output_dir)`**: Uses `glob.glob` to find all PDF files within a project and copies them to the output directory, preserving relative paths [[build_docs.py:49-76]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
*   **`write_sitemap_index(output_dir, projects)`**: Generates a root `sitemap.xml` that indexes individual project sitemaps (e.g., `sitemap_core.xml` or `project/latest/sitemap.xml`) [[build_docs.py:25-46]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).

## Output Structure

The final `output/` directory, which is deployed to GitHub Pages, follows a specific hierarchy to support multi-project documentation under a single domain.

| Path | Source Project | Description |
| --- | --- | --- |
| `/` | `core` | The landing page and core documentation (Systems, AIBs). |
| `/ttnn/latest/` | `ttnn` | Documentation for the TT-NN library. |
| `/tt-metalium/latest/` | `tt-metalium` | Documentation for the TT-Metalium SDK. |
| `/syseng/latest/` | `syseng` | Documentation for System Engineering and Utilities. |
| `/sitemap.xml` | N/A | Global sitemap index. |

## Local Development

Developers can simulate the build process locally to verify changes before pushing.

### Full Build

To build the entire site structure locally:

1.   `pip install -r requirements.txt`[[README.md:15]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process)
2.   `python build_docs.py`[[README.md:21]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process)
3.   Open `output/index.html` in a browser.

### Live Preview (Core)

For rapid iteration on the core documentation:

1.   `cd core`
2.   `make watch`[[core/Makefile:27-35]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process)
3.   This uses `sphinx-autobuild` to serve the site at `http://127.0.0.1:3000/` and watches for changes in the `core/`, `shared/`, and `syseng/` directories [[core/Makefile:32-35]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).

Sources: [README.md 28-45](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/README.md?plain=1#L28-L45)[core/Makefile 27-44](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/Makefile#L27-L44)

## Adding a New Documentation Project

To integrate a new project into the automated build process:

1.   **Directory**: Create a new folder (e.g., `forge/`) at the repository root [[README.md:89]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
2.   **Configuration**: Create a `conf.py` in that folder, ideally based on `shared/example-conf.py`[[README.md:90]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
3.   **Registry**: Add the project name to `versions.yml`[[README.md:91]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
4.   **Content**: Add `.md` or `.rst` files to the project folder [[README.md:92]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).

The `build_docs.py` script will automatically detect the new key in `versions.yml` and include it in the next build cycle [[build_docs.py:84-85]](https://deepwiki.com/tenstorrent/tenstorrent.github.io/4.1-documentation-build-process).
