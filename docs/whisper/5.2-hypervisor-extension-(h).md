## Purpose and Scope

This page documents the implementation of the RISC-V Hypervisor Extension (H) in Whisper. The hypervisor extension enables hardware-assisted virtualization by providing:

*   **Two-stage address translation** (VS-stage and G-stage) for guest virtual machines
*   **Virtual Supervisor (VS) mode** for running guest operating systems
*   **Hypervisor CSRs** for managing virtual machine state and configuration
*   **Hypervisor-privileged instructions** (HLV/HSV/HFENCE) for accessing guest memory and managing TLBs

For information about virtual memory address translation without hypervisor extensions, see [Virtual Memory and Address Translation](https://deepwiki.com/tenstorrent/whisper/4.2-virtual-memory-and-address-translation). For two-stage translation specifics, see [Hypervisor Virtual Memory](https://deepwiki.com/tenstorrent/whisper/4.4-hypervisor-virtual-memory).

* * *

## CSR Organization and Hierarchy

The hypervisor extension introduces three categories of CSRs:

1.   **Hypervisor CSRs** (H-level): Accessible only in HS-mode (supervisor with hypervisor privilege)
2.   **Virtual Supervisor CSRs** (VS-level): Shadow copies of supervisor CSRs for guest VMs
3.   **AIA Hypervisor CSRs**: Advanced Interrupt Architecture extensions for virtual interrupt management

**Sources:**[CsRegs.hpp 256-285](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.hpp#L256-L285)[CsRegs.cpp 860-989](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L860-L989)

The hypervisor extension is enabled through `enableHypervisorMode()`, which configures CSRs, interrupt handling, and triggers:

**Key Configuration Steps:**

1.   **CSR Enabling**: [CsRegs.cpp 872-887](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L872-L887)

    *   Enables hypervisor CSRs (HSTATUS, HEDELEG, HIE, etc.)
    *   Enables VS CSRs if supervisor mode is also enabled
    *   Handles RV32-specific high-half CSRs (HENVCFGH, HTIMEDELTAH)

2.   **MSTATUS Configuration**: [CsRegs.cpp 890-918](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L890-L918)

    *   Makes MPV (Machine Previous Virtual) and GVA (Guest Virtual Address) bits writable
    *   Controls these bits through write/poke/read masks

3.   **MIDELEG Configuration**: [CsRegs.cpp 924-945](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L924-L945)

    *   Makes VSEIP, VSTIP, VSSIP, SGEIP bits read-only-one when enabled
    *   Makes them read-only-zero when disabled
    *   Updates VSIE/VSIP masks via `updateVsieVsipMasks()`

4.   **MEDELEG Configuration**: [CsRegs.cpp 948-955](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L948-L955)

    *   Controls delegation of hypervisor-specific exceptions (bits 23:20, 10)

5.   **MIP/MIE Configuration**: [CsRegs.cpp 958-979](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L958-L979)

    *   Makes MIP.VSSIP writable when hypervisor is enabled
    *   Controls VS interrupt enable bits in MIE

## Two-Stage Address Translation

The hypervisor extension implements two-stage address translation for guest VMs:

**Translation Modes:**

| Mode | Virtual Mode | VSATP.MODE | HGATP.MODE | Translation |
| --- | --- | --- | --- | --- |
| **Bare** | false | Any | Any | No translation |
| **VS-stage only** | true | Sv39/48/57 | Bare | GVA → GPA (no G-stage) |
| **Two-stage** | true | Sv39/48/57 | Sv39x4/48x4/57x4 | GVA → GPA → HPA |

**HGATP Register (Guest Address Translation):**

For RV64, `HGATP` has the following structure (defined in `HgatpFields`):

| Field | Bits | Description |
| --- | --- | --- |
| **PPN** | 43:0 | Physical page number of G-stage root page table |
| **VMID** | 57:44 | Virtual machine identifier (14 bits in RV64, 7 bits in RV32) |
| **Reserved** | 59:58 | Reserved |
| **MODE** | 63:60 | Translation mode (0=Bare, 8=Sv39x4, 9=Sv48x4, 10=Sv57x4) |

**Sources:**[VirtMem.cpp - two stage translation logic](https://github.com/tenstorrent/whisper/blob/733fd921/VirtMem.cpp%20-%20two%20stage%20translation%20logic)[CsrFields.hpp 656-691](https://github.com/tenstorrent/whisper/blob/733fd921/CsrFields.hpp#L656-L691)[Diagram 3 from high-level architecture](https://github.com/tenstorrent/whisper/blob/733fd921/Diagram%203%20from%20high-level%20architecture)

## Virtual Mode State Machine

The hart maintains a `virtMode_` flag that controls whether it's executing in virtual supervisor/user mode:

**State Transitions:**

1.   **Entering Virtual Mode:**

    *   Execute `MRET` with `MSTATUS.MPV=1` sets `virtMode_=true`
    *   Execute `SRET` in HS-mode with `HSTATUS.SPV=1` sets `virtMode_=true`

2.   **Exiting Virtual Mode:**

    *   Trap to HS-mode: Save state in `HSTATUS.SPV/SPVP`, set `virtMode_=false`
    *   Trap to M-mode: Save state in `MSTATUS.MPV`, set `virtMode_=false`

3.   **Remaining in Virtual Mode:**

    *   Traps delegated via HEDELEG/HIDELEG remain in VS-mode
    *   VU-mode to VS-mode transitions via delegation

**Sources:**[Hart.hpp - virtMode_ member](https://github.com/tenstorrent/whisper/blob/733fd921/Hart.hpp%20-%20virtMode_%20member)[trap.cpp - trap handling logic](https://github.com/tenstorrent/whisper/blob/733fd921/trap.cpp%20-%20trap%20handling%20logic)

## HFENCE Instructions

The hypervisor extension provides TLB management instructions for virtual machines:

### HFENCE.VVMA (VS-stage TLB invalidation)

**Instruction Format:**`HFENCE.VVMA rs1, rs2`

**Operation:**

*   `rs1=0, rs2=0`: Invalidate all VS-stage and G-stage TLB entries for current VMID
*   `rs1=0, rs2≠0`: Invalidate entries matching ASID (from `x[rs2]`) and current VMID
*   `rs1≠0, rs2=0`: Invalidate entries matching VPN (from `x[rs1]`) and current VMID
*   `rs1≠0, rs2≠0`: Invalidate entries matching VPN and ASID

**Sources:**[hypervisor.cpp 28-82](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L28-L82)

### HFENCE.GVMA (G-stage TLB invalidation)

**Instruction Format:**`HFENCE.GVMA rs1, rs2`

**Operation:**

*   `rs1=0, rs2=0`: Invalidate all G-stage TLB entries
*   `rs1=0, rs2≠0`: Invalidate entries for VMID from `x[rs2]`
*   `rs1≠0, rs2=0`: Invalidate entries for GPA from `x[rs1]` (shifted left by 2)
*   `rs1≠0, rs2≠0`: Invalidate entries matching both GPA and VMID

**Note:** The implementation supports a configuration option `hfenceGvmaIgnoresGpa_` which, when set, causes HFENCE.GVMA to over-invalidate (ignore GPA parameter) to match certain hardware implementations.

**Sources:**[hypervisor.cpp 87-153](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L87-L153)

## HLV/HSV Instructions (Hypervisor Load/Store)

The hypervisor extension provides explicit load/store instructions that access guest virtual memory from HS-mode:

### HLV (Hypervisor Load Virtual)

**Available HLV Instructions:**

| Instruction | Type | Description |
| --- | --- | --- |
| `HLV.B` | int8_t | Load signed byte from guest virtual address |
| `HLV.BU` | uint8_t | Load unsigned byte from guest virtual address |
| `HLV.H` | int16_t | Load signed halfword from guest virtual address |
| `HLV.HU` | uint16_t | Load unsigned halfword from guest virtual address |
| `HLV.W` | int32_t | Load signed word from guest virtual address |
| `HLV.WU` | uint32_t | Load unsigned word from guest virtual address (RV64) |
| `HLV.D` | uint64_t | Load doubleword from guest virtual address (RV64) |
| `HLVX.HU` | uint16_t | Load halfword for execute (uses exec permission) |
| `HLVX.WU` | uint32_t | Load word for execute (uses exec permission) |

**Key Features:**

1.   **VS-mode Translation Settings**: HLV instructions use VSSTATUS.MXR (make executable readable), VSSTATUS.SUM (permit supervisor user memory), and HSTATUS.VSBE (big-endian) for address translation.

2.   **Pointer Masking**: In U-mode with hypervisor enabled, uses HSTATUS.HUPMM for pointer masking mode.

3.   **HLVX Variants**: Set `useExecForRead(true)` to enable reading from execute-only pages [hypervisor.cpp 263-271](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L263-L271)[hypervisor.cpp 275-285](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L275-L285)

**Sources:**[hypervisor.cpp 158-294](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L158-L294)

### HSV (Hypervisor Store Virtual)

| Instruction | Type | Description |

## Exception and Interrupt Handling

The hypervisor extension modifies exception and trap handling to support virtual machines:

### Exception Delegation Chain

**Key CSRs Involved:**

1.   **HEDELEG**: Determines which exceptions delegate from HS to VS
2.   **MEDELEG**: Determines which exceptions delegate from M to HS
3.   **HSTATUS.SPV**: Saves whether previous mode was virtual
4.   **HSTATUS.SPVP**: Saves previous nominal privilege
5.   **MSTATUS.MPV**: Saves whether previous mode was virtual (M-mode)
6.   **MSTATUS.GVA**: Indicates if trap was due to guest virtual address
7.   **MTVAL2**: Holds guest physical address on trap to M-mode
8.   **MTINST**: Holds transformed instruction on trap to M-mode

## Interrupt Aliasing and Injection

The hypervisor extension implements complex interrupt aliasing between HS-mode and VS-mode:

### SIP/VSIP Aliasing

**SIP Calculation** (implemented in `readSip()`):

```
SIP = (MIP & MIDELEG) | (MVIP & MVIEN & ~MIDELEG)
```

The bits are masked to exclude SGEIP, VSEIP, VSTIP, VSSIP (bits 12, 10, 6, 2).

**VSIP Calculation** (implemented in `readVsip()`):

```
VSIP = ((HIP & HIDELEG) >> 1)  // Bits 1,5,9 from HIP bits 2,6,10
     | (SIP & HIDELEG)           // Bits 13-64 from SIP
     | (HVIP & HVIEN & ~HIDELEG) // From HVIP where not delegated
```

**Sources:**[CsRegs.cpp 175-223](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L175-L223)[CsRegs.cpp 227-265](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L227-L265)

### HIP Special Bits

The HIP register has special aliasing behavior:

1.   **SGEIP (bit 12)**: Set if any bit in `(HGEIP & HGEIE)` is set
2.   **VSEIP (bit 10)**: OR of HVIP[10] and HGEIP bit selected by HSTATUS.VGEIN
3.   **VSTIP (bit 6)**: Set if `time + htimedelta >= vstimecmp`, OR'd with HVIP[6]

**Sources:**[CsRegs.cpp 268-311](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L268-L311)

## Implementation Notes

### TLB Structure

The hypervisor extension uses separate TLBs:

*   **vsTlb_**: VS-stage TLB (guest virtual to guest physical)
*   **stage2Tlb_**: G-stage TLB (guest physical to host physical)

Both TLBs are tagged with VMID and optionally with secure world ID (WID) when STEE is enabled.

### CSR Read/Write Behavior

Many hypervisor CSRs have special read/write behavior:

1.   **HIDELEG**: Read mask depends on MVIEN and MIDELEG [CsRegs.cpp 395-419](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L395-L419)
2.   **MVIP**: Bits alias to MIP based on MVIEN [CsRegs.cpp 316-390](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L316-L390)
3.   **HIP**: Computed value based on HGEIP, HGEIE, HVIP, timer [CsRegs.cpp 268-311](https://github.com/tenstorrent/whisper/blob/733fd921/CsRegs.cpp#L268-L311)

### Virtual Instruction Traps

Instructions that are legal in HS-mode but illegal in VS-mode cause virtual instruction exceptions rather than illegal instruction exceptions. This includes:

*   HFENCE.VVMA/GVMA
*   HLV/HSV instructions
*   WFI (if HSTATUS.VTW=1)
*   SRET (if HSTATUS.VTSR=1)
*   SFENCE.VMA (if HSTATUS.VTVM=1)

**Sources:**[hypervisor.cpp 39-41](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L39-L41)[hypervisor.cpp 98-101](https://github.com/tenstorrent/whisper/blob/733fd921/hypervisor.cpp#L98-L101)

## Summary

The Hypervisor Extension (H) implementation in Whisper provides complete support for hardware-assisted virtualization:

*   **26 hypervisor-specific CSRs** managing virtual machine state and configuration
*   **8 virtual supervisor CSRs** shadowing standard supervisor registers
*   **Two-stage address translation** with separate VS-stage and G-stage TLBs
*   **9 hypervisor-privileged instructions** (HLV/HSV/HFENCE variants)
*   **Virtual mode state management** with proper exception/interrupt delegation
*   **AIA integration** for virtual interrupt management
*   **Configurable behavior** matching RISC-V Privileged Architecture Specification
