# Installation Process Flow

Relevant source files
*   [install.sh](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh)

This document details the sequential steps in the Tenstorrent software stack installation process, focusing on how the installation components are organized and executed. For information about specific installation options and configuration, see [Configuration Options](https://deepwiki.com/tenstorrent/tt-installer/4-configuration-options).

## Overview

The Tenstorrent installer implements a modular installation process that installs and configures various components needed to use Tenstorrent hardware. The process is orchestrated by the `install.sh` script, which handles component installation, configuration, and verification.

The installation flow follows a logical progression from low-level drivers to high-level tools, with built-in decision points that allow customization of the installation based on user preferences and environment variables.

## Installation Phases

The installation process can be divided into several phases, each handling specific aspects of the Tenstorrent software stack setup.

### 1. Initialization Phase

The initialization phase prepares the system for installation by:

1.   Displaying welcome information
2.   Setting up a working directory (`WORKDIR`)
3.   Initializing logging
4.   Checking sudo permissions
5.   Detecting the Linux distribution

This phase ensures all prerequisites are in place before proceeding with the actual installation.

Sources: [install.sh 391-436](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L391-L436)

### 2. Environment Setup Phase

This phase prepares the Python environment for installing Tenstorrent tools:

1.   Determines Python installation preference (existing venv, new venv, system Python, or pipx)
2.   Sets up the chosen Python environment
3.   Configures the appropriate installation command

The available Python environment options are:

*   Option 1: Use active virtual environment
*   Option 2: Create new virtual environment
*   Option 3: Use system Python
*   Option 4: Use pipx for isolated package installation

The finalization phase completes the installation process:

1.   Displays success message and provides usage instructions
2.   Handles system reboot based on configuration

    *   Option 2: Never reboot
    *   Option 3: Always reboot

## Installation Modes

The installation process supports different modes that affect the flow:

### 1. Interactive Mode (Default)

In interactive mode, the installer prompts the user for decisions at key points in the installation process.

### 2. Non-Interactive Mode

When `TT_MODE_NON_INTERACTIVE=0`, the installer runs without requiring user input, using default values or environment variables for configuration.

### 3. Container Mode

When `TT_MODE_CONTAINER=0`, the installer skips components that must be installed on the host system (KMD and HugePages) and disables reboot.

## Environment Variable Control Flow

The installation process can be controlled through environment variables that affect different aspects of the flow:

| Environment Variable | Purpose | Effect on Flow |
| --- | --- | --- |
| `TT_SKIP_INSTALL_KMD` | Skip KMD installation | Bypasses KMD installation step |
| `TT_SKIP_INSTALL_HUGEPAGES` | Skip HugePages setup | Bypasses HugePages configuration step |
| `TT_SKIP_UPDATE_FIRMWARE` | Skip firmware update | Bypasses tt-flash installation and firmware update |
| `TT_SKIP_INSTALL_PODMAN` | Skip Podman installation | Bypasses Podman installation step |
| `TT_SKIP_INSTALL_METALIUM_CONTAINER` | Skip Metalium installation | Bypasses Metalium container installation step |
| `TT_PYTHON_CHOICE` | Python installation method | Determines Python environment setup approach |
| `TT_REBOOT_OPTION` | Reboot behavior | Controls system reboot after installation |
| `TT_MODE_CONTAINER` | Container mode | Skips host-only components (KMD, HugePages) |
| `TT_MODE_NON_INTERACTIVE` | Non-interactive mode | Skips user prompts and uses defaults |

Sources: [install.sh 54-113](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L54-L113)

## Conclusion

The Tenstorrent installation process follows a well-defined flow that installs components in a logical sequence, with various decision points that allow customization based on user preferences and environment variables. The modular design enables users to tailor the installation to their specific needs, from a full installation with all components to a minimal setup with only selected components.
