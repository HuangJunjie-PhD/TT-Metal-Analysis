This document covers the installation, basic usage, and command-line interface of the tt-burnin utility. It provides the essential information needed to run burn-in testing on Tenstorrent AI hardware. For detailed information about the system architecture, see [System Architecture](https://deepwiki.com/tenstorrent/tt-burnin/1.2-system-architecture). For comprehensive hardware support details, see [Supported Chip Architectures](https://deepwiki.com/tenstorrent/tt-burnin/4.1-supported-chip-architectures).

## Purpose and Scope

TT-Burnin is a command-line utility designed to run high-power consumption workloads on Tenstorrent AI devices for stress testing and validation purposes. The utility automatically detects connected hardware, loads appropriate power virus workloads, provides real-time telemetry monitoring, and manages device reset operations.

## Command Line Interface

### Basic Usage

The utility is invoked using the `tt-burnin` command, which is registered as a console script entry point:

`tt-burnin [OPTIONS]`
### Available Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `-h, --help` | flag | - | Show help message and exit |
| `-v, --version` | flag | - | Show program version number |
| `--reset_file` | file | auto-detect | Custom reset configuration JSON file |
| `--no-reset` | flag | false | Skip device reset operations (WARNING: May cause issues) |
| `--no-check` | flag | false | Skip tensix firmware verification (WARNING: May affect load) |
| `--idle` | flag | false | Run tensix idle instead of power virus workload |

## TTX Workload Loading

### Architecture-Specific Workload Files

Each supported chip architecture has a corresponding TTX workload file that contains the power virus program:

| Architecture | TTX File | Load Function | Chip Class |
| --- | --- | --- | --- |
| Grayskull | `gspv.ttx` | `start_burnin_gs()` | `GsChip` |
| Wormhole | `whpv.ttx` | `start_burnin_wh()` | `WhChip`/`RemoteWhChip` |
| Blackhole | `bhpv.ttx` | `start_burnin_bh()` | `BhChip` |

## Real-time Monitoring

The utility provides live telemetry monitoring during burn-in execution using the Rich library for terminal UI:

### Telemetry Display Components

| Component | Function | Update Rate |
| --- | --- | --- |
| Device Table | `generate_table(devices)` | 10 Hz |
| Status Text | Press Enter prompt | Static |
| Live Widget | `rich.live.Live` | 10 Hz refresh |

### Monitored Parameters

*   Core Voltage (V)
*   Core Current (A)
*   AICLK frequency (MHz)
*   Power consumption (W)
*   Core Temperature (°C)
