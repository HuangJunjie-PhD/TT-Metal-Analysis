# License and Usage Restrictions

Relevant source files
*   [LICENSE_understanding.txt](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/LICENSE_understanding.txt)

## Purpose and Scope

This document explains the licensing model and usage restrictions that apply to the tt-firmware repository. It covers the binary release model, specific licensing terms, third-party intellectual property considerations, and clarifies what users can and cannot do with the firmware. For information about installing and using firmware bundles, see [Firmware Bundles](https://deepwiki.com/tenstorrent/tt-firmware/3-firmware-bundles).

Sources: [README.md 66-71](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L66-L71)

## Binary Release Model

The tt-firmware is distributed as binary releases rather than source code. This distribution model is necessitated by licensing agreements with third parties whose intellectual property is incorporated into the firmware.

The firmware code implements critical functionality for Tenstorrent devices, including:

*   Hardware initialization
*   External I/O connectivity maintenance (PCIe, Ethernet, DRAM)
*   Power and thermal envelope management

Tenstorrent has chosen to focus resources on making future firmware components open-source rather than rewriting existing code to isolate proprietary components.

Sources: [README.md 15-19](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L15-L19)[README.md 7-13](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L7-L13)

## Third-Party IP Considerations

The firmware contains intellectual property from third parties, which impacts how users can interact with and use the firmware.

It's important to understand that:

1.   Tenstorrent has agreements with third parties that prevent distribution of source code related to their intellectual property
2.   Making, using, or selling hardware, models, or IP based on or derived from this firmware may require licensing of additional rights from Tenstorrent or other parties

Sources: [README.md 15-19](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L15-L19)[LICENSE_understanding.txt 1-3](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/LICENSE_understanding.txt#L1-L3)

## Usage Permissions and Limitations

The table below summarizes what users are permitted and not permitted to do with the tt-firmware:

| Permitted | Limitations/Restrictions |
| --- | --- |
| Using the firmware to program Tenstorrent products | Making hardware based on the firmware may require additional licensing |
| Redistributing the firmware under the terms specified in the LICENSE file | Selling hardware, models, or IP may require licensing from Tenstorrent or others |
| Using experimental firmware bundles to address specific issues | Source code access is not provided due to third-party IP restrictions |

Users interested in understanding how applications interact with this firmware should consult the [User Mode Driver (UMD)](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/User%20Mode%20Driver%20(UMD)) repository, which provides the interface between applications and the firmware.

## Integration with Tenstorrent Ecosystem

The tt-firmware is part of a larger ecosystem of Tenstorrent software and hardware components, each with its own licensing considerations.

The licensing of tt-firmware specifically covers the binary firmware bundles that manage Tenstorrent hardware. The User Mode Driver (UMD) provides the interface through which applications interact with the firmware, and may have different licensing terms.

Sources: [README.md 15-21](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L15-L21)

## Future Direction

Tenstorrent has indicated that while current firmware is distributed as binary-only due to third-party IP constraints, they are working toward making future firmware components open-source where possible. This reflects a commitment to openness while respecting intellectual property obligations.

Sources: [README.md 17-19](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L17-L19)
