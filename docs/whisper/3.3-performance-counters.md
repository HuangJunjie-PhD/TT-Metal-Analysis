## Purpose and Scope

This document describes the hardware performance monitoring counter subsystem in Whisper, including the basic counters (MCYCLE and MINSTRET), programmable performance counters (MHPMCOUNTER3-31), event configuration, counter overflow handling, and privilege-based access control. Performance counters enable tracking of microarchitectural events such as instruction execution, cache misses, branch mispredictions, and other implementation-specific events.

For information about other CSRs, see [Control and Status Registers (CSRs)](https://deepwiki.com/tenstorrent/whisper/3.1-control-and-status-registers-(csrs)). For information about privilege modes, see [System Organization and Configuration](https://deepwiki.com/tenstorrent/whisper/2.1-system-organization-and-configuration).

## Counter Architecture Overview

The RISC-V architecture defines three types of performance counters:

1.   **MCYCLE/CYCLE**: A 64-bit counter that increments on each clock cycle
2.   **MINSTRET/INSTRET**: A 64-bit counter that tracks retired instructions
3.   **MHPMCOUNTER3-31**: Up to 29 programmable hardware performance monitoring (HPM) counters

Each counter has machine-mode (M-prefix) and unprivileged (no prefix) views. In RV32, each counter is split into low (counter) and high (counterH) halves.

## Counter Control and Inhibition

### MCOUNTINHIBIT Register

The `MCOUNTINHIBIT` CSR (0x320) controls which counters are active. When a bit is set, the corresponding counter stops incrementing:

| Bit | Counter | Description |
| --- | --- | --- |
| 0 | MCYCLE | Inhibit cycle counter |
| 2 | MINSTRET | Inhibit retired instruction counter |
| 3-31 | MHPMCOUNTER3-31 | Inhibit corresponding HPM counter |

## Privilege-Based Access Control

Three CSRs control which privilege modes can access performance counters:

### Access Control CSRs

| CSR | Address | Purpose |
| --- | --- | --- |
| MCOUNTEREN | 0x306 | Controls S/U mode access when in M-mode is executing |
| SCOUNTEREN | 0x106 | Controls U mode access when S-mode exists |
| HCOUNTEREN | 0x606 | Controls VS/VU mode access when H-mode exists |

Each CSR has bits corresponding to counters:

*   Bit 0: CYCLE counter
*   Bit 1: TIME counter
*   Bit 2: INSTRET counter
*   Bits 3-31: HPMCOUNTER3-31

## Counter Overflow (Sscofpmf Extension)

The Sscofpmf (Supervisor-mode Counter Overflow and Privilege Mode Filtering) extension adds overflow detection and interrupt capability to performance counters.

### Overflow Detection

When a counter overflows (transitions from 0xFFFF...FFFF to 0x0000...0000), the OF bit (bit 63) in the corresponding MHPMEVENT register is set, and a local counter overflow interrupt (LCOF) is signaled.

## Summary

The performance counter subsystem in Whisper provides:

1.   **Basic Counters**: MCYCLE and MINSTRET for fundamental metrics
2.   **Programmable Counters**: Up to 29 MHPMCOUNTER registers with configurable event selection
3.   **Access Control**: Hierarchical privilege-based access through MCOUNTEREN, SCOUNTEREN, and HCOUNTEREN
4.   **Inhibition Control**: MCOUNTINHIBIT for selective counter enable/disable
5.   **Event Configuration**: MHPMEVENT registers with event ID mapping and privilege filtering
6.   **Overflow Support**: Sscofpmf extension providing overflow detection, LCOF interrupts, and SCOUNTOVF visibility
7.   **Flexible Configuration**: APIs for customizing counter behavior, event mapping, and access control

The implementation efficiently caches critical values (like `perfControl_`) in the `Hart` class and ties CSR objects directly to counter storage for optimal performance during simulation.
