# maximize_space Action

The `maximize_space` action is a utility designed to reclaim disk space on GitHub-hosted runners by purging pre-installed software packages and toolchains that are typically unnecessary for Tenstorrent build and test workloads. By removing these assets, the action can free up approximately 6-7 GB of additional storage [.github/actions/maximize_space/Readme.md 19](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/Readme.md?plain=1#L19-L19)

## Purpose and Target Environment

GitHub-hosted runners come pre-configured with a large variety of toolsets (e.g., Go, Ruby, CodeQL, Android SDKs). In workflows that utilize the `container` option, the filesystem layout often results in high initial disk usage on the overlay filesystem [.github/actions/maximize_space/Readme.md 5-17](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/Readme.md?plain=1#L5-L17) This action targets specific directories known to house these large, optional toolchains to prevent "disk full" errors during heavy compilation or artifact generation steps.

### Target Filesystem Layouts

The action identifies and cleans two primary locations used by GitHub runners to store tool caches:

1.   `/__t`: Common in containerized runner environments [.github/actions/maximize_space/action.yml 21-27](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/action.yml#L21-L27)
2.   `/opt/hostedtoolcache`: The standard location for toolsets on Linux runners [.github/actions/maximize_space/action.yml 28-37](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/action.yml#L28-L37)

## Usage

The action is invoked as a step within a job. It is recommended to run this early in the workflow before dependencies are downloaded or builds are initiated.

### Basic Example

`- name: Maximize space  uses: tenstorrent/tt-github-actions/.github/actions/maximize_space@main`
[.github/actions/maximize_space/Readme.md 35-38](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/actions/maximize_space/Readme.md?plain=1#L35-L38)

### Comparison of Space Reclamation

Based on the project documentation, the impact on a typical runner is as follows:

| Metric | Before Cleanup | After Cleanup |
| --- | --- | --- |
| **Used Space** | ~63 GB | ~57 GB |
| **Available Space** | ~8.7 GB | ~16 GB |
| **Reclaimed** | - | **~6-7 GB** |

## Test Workflow

The repository includes a test workflow, `test_maximize_space.yml`, which validates the action's effectiveness across different runner configurations.

### Test Scenarios

The workflow executes two parallel jobs to ensure the action handles both standard and containerized environments correctly:

1.   **`run-with-container`**: Executes the action inside a `tt-mlir-ci-ubuntu-22-04` container image [.github/workflows/test_maximize_space.yml 12-24](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_maximize_space.yml#L12-L24)
2.   **`run-without-container`**: Executes the action directly on the `ubuntu-latest` runner host [.github/workflows/test_maximize_space.yml 26-36](https://github.com/tenstorrent/tt-github-actions/blob/3ecf8e49/.github/workflows/test_maximize_space.yml#L26-L36)
