This page documents the tensor operations system in the TTSIM framework, specifically focusing on `SimTensor` operations and functional interfaces for tensor manipulation. The tensor operations layer provides a PyTorch-like API for building computational graphs using `SimOp` instances wrapped in functional handles.

For information about the underlying SimOp implementation and graph construction, see [Operations and Graph Construction](https://deepwiki.com/tenstorrent/polaris/2.2.1-operations-and-graph-construction). For hardware-specific modeling details, see [Hardware Modeling](https://deepwiki.com/tenstorrent/polaris/2.2.3-hardware-modeling).

## Core Tensor Infrastructure

### SimTensor Class

`SimTensor` is the fundamental tensor representation in the TTSIM framework. It stores tensor metadata including shape, dtype, and connectivity information for graph construction.

Sources: [ttsim/front/functional/op.py 14-36](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/op.py#L14-L36)

### SimOpHandle Framework

`SimOpHandle` provides a functional interface that wraps `SimOp` instances, enabling PyTorch-like operation chaining while maintaining graph construction capabilities.

Sources: [ttsim/front/functional/op.py 68-129](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/op.py#L68-L129)[ttsim/front/functional/op.py 132-206](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/op.py#L132-L206)[ttsim/front/functional/op.py 211-256](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/op.py#L211-L256)

## Functional Operation Categories

### Unary Operations

Unary operations take a single tensor input and produce a single tensor output.

| Operation | Function | OpType | Description |
| --- | --- | --- | --- |
| `Identity` | `F.Identity(name)` | `Identity` | Pass-through operation |
| `Tanh` | `F.Tanh(name)` | `Tanh` | Hyperbolic tangent |
| `Gelu` | `F.Gelu(name)` | `Gelu` | Gaussian Error Linear Unit |
| `Relu` | `F.Relu(name)` | `Relu` | Rectified Linear Unit |
| `Sigmoid` | `F.Sigmoid(name)` | `Sigmoid` | Sigmoid activation |
| `Softmax` | `F.Softmax(name)` | `Softmax` | Softmax normalization |
| `Transpose` | `F.Transpose(name, perm=[...])` | `Transpose` | Tensor transpose |

Sources: [ttsim/front/functional/op.py 488-502](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/op.py#L488-L502)

The framework supports operation fusion to optimize performance by combining multiple operations into single execution units.

`op_fusion_spec:  - [Add, LayerNormalization, Matmul, Add, Matmul, Add, Gelu]  - [Matmul, Add, Gelu]  - [Conv, BatchNormalization, Relu]  - [Transpose, Transpose]`

## Usage Examples

### Building a Neural Network Layer

`# Create a linear layer with functional interfacelinear = F.Linear('fc1', in_features=512, out_features=256)bias = F.Bias('bias1', [256])activation = F.Gelu('gelu1') # Chain operationsx = linear(input_tensor)x = bias(x)  output = activation(x)`
### Tensor Manipulation Pipeline

`# Shape manipulation chainx = input_tensor.view(batch_size, -1)  # Flattenx = x.transpose(0, 1)                  # Transposex = x.unsqueeze(0)                     # Add batch dimensionx = x[:, :, 0]                         # Slice operation`
