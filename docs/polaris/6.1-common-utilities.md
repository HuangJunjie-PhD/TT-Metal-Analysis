## Data Format Parsing

The parsing system provides unified interfaces for reading various data formats commonly used in configuration files and data exchange:

### CSV and Excel Parsing

The `parse_csv()` function handles CSV file parsing with comment line filtering, while `parse_xlsx()` provides Excel worksheet parsing with automatic sheet selection and header mapping.

Key features:

*   Comment line filtering (lines starting with `#`)
*   Automatic column header detection
*   Support for sheet selection in Excel files using `@` syntax
*   Unified interface through `parse_worksheet()` function

### YAML Processing

YAML parsing supports both single-document and multi-document formats:

*   `parse_yaml()` for standard YAML files
*   `parse_multidoc_yaml()` for files containing multiple YAML documents
*   Integration with `locator_handle()` for URL/file path resolution

### Object Conversion

The `dict2obj` class converts dictionary structures to objects with attribute access:

*   Recursive conversion of nested dictionaries
*   Attribute-style access to configuration values
*   Error handling for missing attributes

## Dynamic Class Loading

The dynamic class loading system enables runtime loading of Python classes, essential for workload instantiation and plugin-style architecture:

### Class Loading Mechanism

The `get_ttsim_python_class()` function provides cached dynamic class loading with special syntax support:

*   Supports `class_name@file.py` syntax for importing specific classes
*   Uses `@lru_cache(128)` for performance optimization
*   Validates module paths and class existence
*   Returns callable class objects

### Instance Creation

The `get_ttsim_functional_instance()` function creates instances of dynamically loaded classes:

*   Takes class path, instance name, and configuration
*   Leverages the cached class loading mechanism
*   Returns configured instance objects

Sources: [ttsim/utils/common.py 134-162](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/utils/common.py#L134-L162)

## File and URL Handling

The file handling system provides transparent access to both local files and remote URLs with integrated caching:

### FileLocator Class

The `FileLocator` class provides unified access to files and URLs:

*   Automatic detection of local paths vs. URLs
*   Support for `http` and `https` schemes
*   Transparent caching for remote content
*   Path normalization and validation

### Caching System

The `CacheManager` class handles content caching:

*   Home directory-based cache storage (`.cache/urlcontent/`)
*   Key-based content retrieval and storage
*   Automatic cache directory creation
*   File-based persistence

## Type Conversion and Validation

The type conversion system provides robust data transformation and validation utilities:

| Function | Purpose | Supported Types |
| --- | --- | --- |
| `str_to_bool()` | String to boolean conversion | `true/false`, `yes/no`, `on/off`, `1/0` |
| `convert_units()` | Unit conversion | `B`, `KB`, `MB`, `GB`, `TB`, `HZ`, `FLOPS`, `OPS` |
| `make_tuple()` | Value to tuple conversion | `int`, `list`, `tuple` |

### Boolean Conversion

The `str_to_bool()` function handles multiple boolean representations:

*   Case-insensitive parsing
*   Support for common boolean strings
*   Numeric value handling (0 = false, non-zero = true)
*   Type preservation for existing booleans

### Unit Conversion

The `convert_units()` function performs metric unit conversions:

*   Supports data sizes (B, KB, MB, GB, TB)
*   Supports frequencies (HZ, KHZ, MHZ, GHZ, THZ)
*   Supports operation rates (OPS, KOPS, MOPS, GOPS, TOPS)
*   Supports floating point rates (FLOPS)
*   Binary vs. decimal conversion (1024 vs. 1000 base)

### Tuple Creation

The `make_tuple()` function normalizes values to tuples:

*   Scalar to tuple expansion
*   Length validation for existing sequences
*   Type checking and error handling

## Argument Validation

The argument validation system ensures proper function parameter handling:

### Argument Checking

The `check_known_args()` function validates function arguments:

*   Compares provided arguments against known valid arguments
*   Raises `AssertionError` for unknown arguments
*   Includes operation name in error messages

### Default Value Handling

The `get_kwargs_with_defaults()` function merges arguments with defaults:

*   Performs argument validation first
*   Creates deep copy of defaults to avoid mutations
*   Updates with provided arguments
*   Returns complete argument dictionary

Sources: [ttsim/utils/common.py 217-234](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/utils/common.py#L217-L234)

## Output Generation

The output system provides consistent data serialization across multiple formats:

### Format-Specific Output

Each output function handles format-specific requirements:

*   `print_yaml()` uses standard YAML serialization
*   `print_csv()` uses `DictWriter` with proper newline handling
*   `print_json()` provides compact JSON serialization

### CSV Output Features

The CSV output function provides:

*   Automatic header row generation
*   Proper newline handling for cross-platform compatibility
*   Support for both `KeysView` and `list` column specifications
*   Dictionary-based row writing

## Utility Functions

Additional utility functions provide specialized functionality:

### Warning Management

The `warnonce()` function provides cached warning messages:

*   Uses `@lru_cache(128)` to prevent duplicate warnings
*   Maintains standard logging interface
*   Reduces log noise for repeated conditions

### Import Management

The system includes lazy import handling:

*   Optional dependency loading (e.g., `openpyxl`)
*   Runtime module importing for dynamic class loading
*   Graceful handling of missing dependencies
