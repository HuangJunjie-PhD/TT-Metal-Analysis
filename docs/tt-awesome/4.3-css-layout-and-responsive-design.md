The `tt-awesome` website uses a three-pane architecture designed for high-density information display on desktop and a stacked, state-driven experience on mobile. The layout is managed through a combination of CSS custom properties for theming, a state machine defined by CSS classes on a parent container, and a mobile drawer pattern for navigation.

## The Three-Pane Architecture

The core layout is defined in `src/assets/style.css` using a flexbox container with the class `.panes`. On desktop viewports, this container displays three distinct vertical sections simultaneously: the Sidebar, the List Pane, and the Detail Pane.

### CSS Custom Properties & Theming

The design system relies on CSS variables defined in the `:root` selector. These variables control the color palette (primarily dark teal and slate) and dynamic pane widths that scale with the viewport size.

| Variable | Default Value | Purpose |
| --- | --- | --- |
| `--bg0` | `#0F2A35` | Main application background |
| `--teal` | `#4FD1C5` | Primary accent color for active states and logos |
| `--sidebar-w` | `192px` | Base width of the navigation sidebar |
| `--list-w` | `340px` | Base width of the middle entry list |

The pane widths are responsive and increase at specific breakpoints to utilize extra screen real estate:

*   **1400px+**: Sidebar `200px`, List `380px`[src/assets/style.css 13-15](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L13-L15)
*   **1600px+**: Sidebar `224px`, List `420px`[src/assets/style.css 16-18](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L16-L18)
*   **1920px+**: Sidebar `248px`, List `460px`[src/assets/style.css 19-21](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L19-L21)

### Pane State Machine

The visibility and layout of the application are controlled by a state machine implemented as CSS classes on the `#panes` element. This allows the UI to switch between the "Home" view (category grid) and the "Browser" view (list + detail) without page reloads.

*   **`.panes.home-active`**: Displays the `.home-view` (the category grid) and hides the `.list-pane` and `.detail-pane`[src/assets/style.css 68-71](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L68-L71)
*   **`.panes:not(.home-active)`**: Hides the home grid and displays the functional panes for browsing entries.

## Mobile Responsive Design

The mobile implementation (viewports ≤ 767px) transforms the multi-pane desktop layout into a single-pane stacked navigation system. This transition is managed via a dedicated `@media` block and supplemental logic in `main.js`.

### The Mobile State Machine

On mobile, the `.panes` element uses additional classes to determine which single pane occupies the full viewport:

*   `.list-active`: Shows the middle entry list.
*   `.detail-active`: Shows the project detail view.
*   `.releases-active`: Shows the recent releases feed.

### Mobile Drawer & Navigation

The sidebar navigation is converted into a "hamburger" drawer on mobile devices.

*   **Drawer Pattern**: The sidebar uses `position: fixed` with a `transform: translateX(-100%)` to hide off-screen. Applying the `.drawer-open` class to the body triggers a transition to `translateX(0)`[docs/superpowers/specs/2026-05-29-mobile-responsive-design.md 65-70](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-29-mobile-responsive-design.md?plain=1#L65-L70)
*   **Backdrop**: A `.drawer-backdrop` element appears behind the drawer to dim the background and provide a "click-to-close" target [docs/superpowers/plans/2026-05-29-mobile-responsive.md 158-167](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/plans/2026-05-29-mobile-responsive.md?plain=1#L158-L167)

## UI Components & Styles

### Chip & Badge System

The UI uses a standardized system for filtering and categorization.

*   **Filter Chips**: Located in the `.topbar` (and duplicated in `.list-head` for mobile), these buttons use data attributes (e.g., `data-filter="community"`) to toggle project visibility [src/assets/style.css 52-59](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L52-L59)
*   **Project Badges**: Small indicators used in the list and detail panes to denote affiliation. 
    *   `.badge--community`: Green background [src/assets/style.css 126](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L126-L126)
    *   `.badge--affiliated`: Pink background [src/assets/style.css 127](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L127-L127)
    *   `.badge--official`: Muted background [src/assets/style.css 128](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L128-L128)

### Package Badges (PyPI/Apt/Cargo)

In the project detail view, external package links are styled as distinctive badges. These are often rendered using external "Shields.io" style badges but are contained within layout-aware wrappers to ensure they wrap correctly on smaller screens.

### Release History Callouts

The release history (displayed in the detail pane or the dedicated releases pane) uses specific callout styles to highlight new versions:

*   **Summaries**: Summaries generated by LLMs are displayed in `.row-desc` with a slightly smaller font and muted color to distinguish them from primary project descriptions [src/assets/style.css 117](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L117-L117)
*   **Star Counts**: GitHub star counts are displayed using the `.row-stars` class with the `--teal-lt` color [src/assets/style.css 116](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/style.css#L116-L116)

## Planet Tenstorrent Styles

The `/planet/` sub-site uses a separate CSS file (`src/assets/planet.css`) because it follows a standard document scrolling model rather than the SPA-style fixed viewport of the main application.

*   **Viewport Override**: It explicitly overrides the `overflow: hidden` on `html, body` set by the main stylesheet [src/assets/planet.css 13-16](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/planet.css#L13-L16)
*   **Content Cards**: Items in the planet feed use the `.planet-item` class, featuring a subtle hover transition that includes a `translateY(-1px)` and a teal box-shadow [src/assets/planet.css 235-247](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/planet.css#L235-L247)
*   **Type Badges**: Feed items are tagged with type-specific badges (e.g., `.planet-type-badge--paper`, `.planet-type-badge--article`) using varied background opacities of the theme colors [src/assets/planet.css 257-269](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/src/assets/planet.css#L257-L269)
