# Adaptive Baseline and Color System

Relevant source files
*   [src/animation/baseline.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs)
*   [src/animation/common.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs)

The `tt-toplike` visualization engine relies on a sophisticated normalization and color mapping system to ensure that hardware activity is visually perceptible regardless of the specific system's idle characteristics. This is achieved through the `AdaptiveBaseline` system, which learns the "noise floor" of a device, and a multi-dimensional HSV color model that maps telemetry metrics to visual properties like hue, brightness, and twinkle.

## Adaptive Baseline Learning

The fundamental challenge in hardware monitoring is that different systems (and different architectures like Greyskull or Wormhole) have varying idle power, current, and temperature profiles [src/animation/baseline.rs 11-12](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L11-L12) A fixed threshold for "high activity" would fail to show subtle fluctuations on a low-power system or would be permanently triggered on a high-power server.

### The Learning Phase

The `DeviceBaseline` struct manages the state for an individual device [src/animation/baseline.rs 21-48](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L21-L48) It operates in two phases:

1.   **Accumulation**: For the first 20 samples, the system sums power (Watts), current (Amps), temperature (Celsius), and AICLK (MHz) [src/animation/baseline.rs 76-82](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L76-L82)
2.   **Establishment**: Once exactly 20 samples are collected, the system computes the mean for each metric, establishing the `baseline`[src/animation/baseline.rs 85-90](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L85-L90)

### Relative Change Calculation

Once the baseline is established (`is_established()`), the system calculates activity as a ratio relative to the learned idle state [src/animation/baseline.rs 120-125](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L120-L125):

$$ \text{relative_change} = \frac{\text{current_value} - \text{baseline_value}}{\text{baseline_value}} $$

This allows the UI to respond to a 10% increase in power (e.g., 20W to 22W) with the same visual intensity as a 10% increase on a larger board (e.g., 50W to 55W) [src/animation/baseline.rs 12-13](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L12-L13)

### Sensitivity Tuning

The `AdaptiveBaseline` manager includes a `sensitivity` multiplier [src/animation/baseline.rs 185-186](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L185-L186) This is applied at output time, allowing users to tune how "reactive" the visualizations are without losing the learned baseline data [src/animation/baseline.rs 183-184](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L183-L184)

## Color Mapping Logic

The visualization engine uses the **HSV (Hue, Saturation, Value)** color space to represent multi-dimensional telemetry data simultaneously. This is converted to RGB for terminal rendering via `hsv_to_rgb`[src/animation/common.rs 34-59](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L34-L59)

### Temperature to Hue Mapping

Temperature is mapped to Hue to provide an intuitive "heat" visualization. The `temp_to_hue` function maps a range of 0°C–100°C to a hue range of 180° (Cyan) down to 0° (Red) [src/animation/common.rs 105-108](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L105-L108)

| Temperature | Hue | Color |
| --- | --- | --- |
| 0°C | 180° | Cyan |
| 50°C | 90° | Green/Yellow |
| 100°C | 0° | Red |

### Multi-Dimensional Mapping

In visualizations like the **Starfield**, multiple telemetry points are combined to produce a single color output:

| Visual Property | Telemetry Metric | Logic |
| --- | --- | --- |
| **Hue** | Temperature | Determined by `temp_to_hue(temp_c)`[src/animation/common.rs 105](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L105-L105) |
| **Brightness (Value)** | Power Change | Scaled by `power_change()` relative to baseline [src/animation/baseline.rs 128](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L128-L128) |
| **Twinkle/Saturation** | Current Change | Scaled by `current_change()`[src/animation/baseline.rs 135](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/baseline.rs#L135-L135) |

## Character Gradients and Intensity

Beyond color, the system uses character-based intensity mapping to represent data density in the TUI. The `value_to_char_intensity` function maps a normalized `0.0-1.0` value to an array of characters [src/animation/common.rs 119-123](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L119-L123)

### Standard Gradients

The engine defines several constant character sets for different architectural themes:

*   **Block Gradients**: `· ░ ▒ ▓ █` (General intensity) [src/animation/common.rs 126](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L126-L126)
*   **Particle Gradients**: `· ○ ◎ ◉ ● ✦` (Data flow) [src/animation/common.rs 132](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L132-L132)
*   **Architectural Symbols**: 
    *   **Greyskull**: Uses `WINDOW_CHARS` (`□ ▫ ▪ ▪ ■`) for Tensix cores [src/animation/common.rs 147-172](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L147-L172)
    *   **Wormhole**: Uses `PORTAL_CHARS` (`◯ ◎ ◉ ⊚ ⊛ ◉`) for the portal nexus theme [src/animation/common.rs 153-182](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L153-L182)
    *   **Blackhole**: Uses `SINGULARITY_CHARS` (`· ∘ ○ ● ◉`) for the event horizon theme [src/animation/common.rs 160-177](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/common.rs#L160-L177)
