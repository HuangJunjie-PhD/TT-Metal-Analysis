The LLM Engine is the core high-performance C++ component responsible for orchestrating the lifecycle of Large Language Model requests. It manages the transition of requests from initial prompt encoding through paged KV cache allocation, iterative token generation (prefill and decode phases), and final termination or abort handling.

## LLMRunner and Request Lifecycle

The `LLMRunner` is the primary execution orchestrator within a worker process. It integrates the scheduler, the hardware-specific model runner, memory management, and guided decoding [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp23-48](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L23-L48)

### Execution Flow

1. **Step Loop**: The runner continuously calls `step()`, which first checks the `ICancelQueue` for any task IDs that need to be aborted [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp157-164](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L157-L164)
2. **Scheduling**: It invokes `scheduler->schedule()` to obtain a batch of `Sequence` objects and a boolean flag indicating if the batch is for `prefill` or `decode` [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp166](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L166-L166)
3. **Guided Decoding**: If enabled, bitmasks are generated and applied to the sampling parameters of each sequence to constrain output tokens based on a schema or grammar [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp135-154](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L135-L154)
4. **Model Inference**: The `IModelRunner` executes the forward pass on the Tenstorrent device [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp171](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L171-L171)
5. **Result Processing**: A callback handles the resulting tokens, updating sequence state and pushing results back to the `IResultQueue` [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp50-97](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L50-L97)

## Sequence Management

A `Sequence` represents a single active request. It tracks the token history, sampling parameters, and the `block_table` used for paged attention [tt-media-server/cpp\_server/include/domain/sequence.hpp33-40](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/domain/sequence.hpp#L33-L40)

## Scheduler and Scheduling Policies

The `Scheduler` determines which sequences are processed in each iteration. It manages two primary queues: `waiting` (for prefill) and `running` (for decode) [tt-media-server/cpp\_server/tests/scheduler\_test.cpp80-126](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L80-L126)

### Scheduling Policies

The system selects a scheduler implementation based on the `SchedulingPolicy` configuration [tt-media-server/cpp\_server/tests/scheduler\_test.cpp53-60](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L53-L60)

| Policy | Description | Key Behavior |
| --- | --- | --- |
| `PREFILL_FIRST` | Prioritizes new requests to minimize Time-To-First-Token (TTFT). | Prefills any waiting requests before continuing decodes [tt-media-server/cpp\_server/tests/scheduler\_test.cpp80-91](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L80-L91) |
| `MAX_OCCUPANCY` | Maximizes hardware utilization by filling batches to `max_in_flight_count`. | Attempts to pack as many sequences as possible into every batch. |

### Preemption

When KV cache memory is exhausted, the scheduler can preempt running sequences, moving them back to the waiting queue and freeing their allocated blocks [tt-media-server/cpp\_server/tests/scheduler\_test.cpp207-215](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L207-L215)

## Memory and KV Cache Management

Memory is managed using a paged approach (Paged Attention). The `BlockManager` tracks physical blocks of memory, while the `PagedMemoryManager` handles asynchronous memory operations [tt-media-server/cpp\_server/src/runners/llm\_runner.cpp30-33](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L30-L33)

### Block Allocation

* **Prefill Allocation**: Blocks are allocated based on the prompt length when a sequence enters the engine [tt-media-server/cpp\_server/tests/cancellation\_test.cpp113-130](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/cancellation_test.cpp#L113-L130)
* **Decode Expansion**: As a sequence generates tokens, new blocks are allocated from the free pool when the current block capacity (e.g., 8 tokens) is exceeded [tt-media-server/cpp\_server/tests/scheduler\_test.cpp31-37](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L31-L37)
* **Deallocation**: Blocks are immediately returned to the free pool when a sequence is finished or aborted [tt-media-server/cpp\_server/tests/cancellation\_test.cpp131-136](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/cancellation_test.cpp#L131-L136)

## Abort Handling

The engine supports robust request cancellation through a dedicated `ICancelQueue`. This allows the server to stop processing and free resources as soon as a client disconnects or explicitly cancels a request [tt-media-server/cpp\_server/tests/cancellation\_test.cpp105-111](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/cancellation_test.cpp#L105-L111)
