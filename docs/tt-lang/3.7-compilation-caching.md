## Purpose and Scope

Compilation caching enables tt-lang kernels to skip redundant MLIR compilation passes by reusing previously compiled artifacts when the same kernel is invoked with tensors that have compatible properties. This significantly improves development iteration speed and runtime performance, as the MLIR pass pipeline (Python AST → TTL → TTKernel → C++) can be expensive for complex kernels.

The caching system is managed by the `@ttl.kernel` (and its alias `@ttl.operation`) decorator and the `CompiledTTNNKernel` class, which together handle cache key generation, lookup, and artifact storage.

## Cache Key Generation

The compilation cache uses a composite key based on tensor properties and runtime configuration parameters to determine whether a previously compiled kernel can be reused. Cache keys are constructed by `_make_cache_key()` which combines:

1.   **Tensor properties** - Extracted via `_get_tensor_cache_info()` for each `ttnn.Tensor` argument: [python/ttl/ttl_api.py 119-128](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L119-L128)

    *   **Shape**: Tuple of dimensions. [python/ttl/ttl_api.py 121](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L121-L121)
    *   **Dtype**: Data type string (e.g., "bfloat16"). [python/ttl/ttl_api.py 122](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L122-L122)
    *   **Memory space**: Buffer type (L1, DRAM, or System Memory). [python/ttl/ttl_api.py 123-126](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L123-L126)
    *   **Layout**: Tensor layout configuration (e.g., TILE_LAYOUT). [python/ttl/ttl_api.py 127](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L127-L127)

2.   **Compute configuration** - Runtime parameters that affect code generation: [python/ttl/ttl_api.py 131-156](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L131-L156)

    *   `fp32_dest_acc_en`: Float32 accumulation in DST registers. [python/ttl/ttl_api.py 152](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L152-L152)
    *   `dst_full_sync_en`: Full synchronization of DST registers. [python/ttl/ttl_api.py 153](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L153-L153)
    *   `target_arch`: Target hardware architecture (e.g., "wormhole_b0"). [python/ttl/ttl_api.py 154](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L154-L154)
    *   `compiler_options`: Instance of `CompilerOptions` containing flags like subblocking or unrolling strategies. [python/ttl/ttl_api.py 155](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L155-L155)

3.   **Mesh Configuration** - For multi-device execution, the mesh shape is included to prevent collisions between single-device and multi-device compilations where shard shapes might differ. [python/ttl/ttl_api.py 142-148](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L142-L148)

The kernel function object and the cache key tuple together form the cache lookup key, allowing different kernels with the same signature to maintain separate cache entries.

**Sources:**[python/ttl/ttl_api.py 119-156](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L119-L156)

## CompiledTTNNKernel: The Cached Artifact

The `CompiledTTNNKernel` class encapsulates all artifacts needed to execute a kernel without recompiling. It stores compilation results in a reusable form that can be invoked with different tensor instances as long as their properties match the cache key.

## Cache Mechanism

The compilation cache is implemented as a global dictionary `_kernel_cache` that maps `(kernel_function, cache_key)` tuples to `CompiledTTNNKernel` objects. [python/ttl/ttl_api.py 1038](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1038-L1038)

### Cache Lookup Flow

When a decorated kernel function is invoked: [python/ttl/ttl_api.py 1040-1206](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1040-L1206)

1.   **Resolve dynamic parameters**: If the grid is callable or 'auto', evaluate it with current arguments. [python/ttl/ttl_api.py 1050-1070](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1050-L1070)
2.   **Generate cache key**: Call `_make_cache_key()` with tensor arguments. [python/ttl/ttl_api.py 1100-1110](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1100-L1110)
3.   **Check cache**: Look up `(f, cache_key)` in `_kernel_cache`. [python/ttl/ttl_api.py 1115-1120](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1115-L1120)
4.   **Cache hit**: Return cached `CompiledTTNNKernel` and execute or return based on `_should_execute()`. [python/ttl/ttl_api.py 1122-1130](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1122-L1130)
5.   **Cache miss**: Proceed with full compilation pipeline (AST traversal and MLIR lowering). [python/ttl/ttl_api.py 1135-1150](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1135-L1150)

### Cache Storage

After compilation completes:

1.   **Compile kernel**: Call `_compile_kernel()` which orchestrates thread compilation and MLIR generation via `TTLGenericCompiler`. [python/ttl/ttl_api.py 922-1036](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L922-L1036)[python/ttl/_src/ttl_ast.py 129-130](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/ttl_ast.py#L129-L130)
2.   **Build CompiledTTNNKernel**: Call `_compile_ttnn_kernel()` to extract paths, configs, and argument specs from the compiled module. [python/ttl/ttl_api.py 724-919](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L724-L919)
3.   **Store in cache**: Insert `_kernel_cache[(f, cache_key)] = compiled_kernel`. [python/ttl/ttl_api.py 1190-1195](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1190-L1195)

**Sources:**[python/ttl/ttl_api.py 1036-1206](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L1036-L1206)[python/ttl/ttl_api.py 922-1036](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L922-L1036)[python/ttl/ttl_api.py 724-919](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L724-L919)

## Compile-Only Mode

The `TTLANG_COMPILE_ONLY=1` environment variable enables compilation without kernel execution. This is useful for testing the compilation pipeline or generating C++ artifacts without hardware access. [python/ttl/ttl_api.py 161-163](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L161-L163)

## Cache Invalidation and Management

The compilation cache persists for the lifetime of the Python process and has no automatic invalidation mechanism. Cache entries remain valid as long as the cache key matches.

### When Cache is Valid

Cache reuse is safe when:

*   **Tensor shapes** remain identical. [python/ttl/ttl_api.py 121](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L121-L121)
*   **Data types (dtype)** are unchanged. [python/ttl/ttl_api.py 122](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L122-L122)
*   **Memory configuration** (L1/DRAM) is consistent. [python/ttl/ttl_api.py 123-126](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L123-L126)
*   **Layout** (tile layout) matches. [python/ttl/ttl_api.py 127](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L127-L127)
*   **Device Mesh** shape is the same. [python/ttl/ttl_api.py 147](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L147-L147)
*   **Target Architecture** matches the compiled artifact. [python/ttl/ttl_api.py 154](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L154-L154)

### When Cache Must Be Cleared

Manual cache clearing (restarting the process) is required when:

*   Kernel function implementation (Python source) changes.
*   Grid dimensions change for a given kernel (if not using 'auto' grid).
*   MLIR passes or compilation flags are modified during development.
