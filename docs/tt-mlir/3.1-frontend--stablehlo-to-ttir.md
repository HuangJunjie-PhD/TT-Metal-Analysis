## Purpose and Scope

This page documents the frontend conversion stage from StableHLO and Shardy operations to the Tenstorrent Intermediate Representation (TTIR). This conversion is the entry point of the `tt-mlir` compilation pipeline, normalizing operations from ML frameworks (PyTorch, JAX, etc.) into a hardware-agnostic intermediate representation. Key responsibilities include implicit broadcasting handling, operation decomposition, sharding propagation via Shardy, complex data type conversion, and composite flattening.

**Related pages:**

*   For the TTIR dialect specification, see [2.1 TTIR Dialect](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/2.1%20TTIR%20Dialect)
*   For the next compilation stage (TTIR to TTNN), see [3.2 TTIR to TTNN Backend Pipeline](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/3.2%20TTIR%20to%20TTNN%20Backend%20Pipeline)

## StableHLO and Shardy Integration

StableHLO serves as the primary input dialect. To support multi-device execution, `tt-mlir` integrates with the **Shardy (SDY)** dialect for tensor sharding propagation.

### Shardy Utilities and Mesh Management

The frontend utilizes Shardy to define device meshes and propagate sharding constraints across the graph. Key utilities in `ShardyUtils.cpp` handle the lifecycle of these annotations:

*   `getMeshOps`: Extracts all `mlir::sdy::MeshOp` definitions from the module [[lib/Dialect/StableHLO/Utils/ShardyUtils.cpp:24-34]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   `normalizeMeshTo2D`: Ensures meshes are in a 2D format compatible with Tenstorrent hardware grids by prepending an axis of size 1 if necessary [[lib/Dialect/StableHLO/Utils/ShardyUtils.cpp:105-141]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   `createTTMeshAttrFromSdyMeshOp`: Converts Shardy mesh operations into `ttcore::MeshAttr` for use in Tenstorrent-specific dialects [[lib/Dialect/StableHLO/Utils/ShardyUtils.cpp:144-150]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   `sdyAnnotationsExist`: Scans the module for `sdy.sharding` attributes on arguments, results, or operations to determine if the graph requires sharding logic [[lib/Dialect/StableHLO/Utils/ShardyUtils.cpp:153-209]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).

The `createStableHLOPipeline` function defines the sequence of transformations applied to the StableHLO/Shardy IR before it is lowered to TTIR.

### Key Pipeline Passes

*   **`FlattenOrConvertCompositesPass`**: Composites with custom sharding rules are converted to `stablehlo.custom_call` ops so Shardy can propagate through them; others are flattened (inlined) [[lib/Dialect/StableHLO/Pipelines/StableHLOPipelines.cpp:70-73]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   **`AnalyzeMeshPass`**: Analyzes the mesh of the graph and updates shardings or annotations to match the target device [[lib/Dialect/StableHLO/Pipelines/StableHLOPipelines.cpp:49-52]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   **`RegisterCustomShardingRulePass`**: Attaches `ShardingRuleOpInterface` to ops like `stablehlo.CustomCallOp` so Shardy can propagate shardings through Tenstorrent-specific primitives [[lib/Dialect/StableHLO/Pipelines/StableHLOPipelines.cpp:75-76]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   **`StableHLOComplexDataTypeConversionPass`**: Converts `complex<fN>` types to float pairs (`tensor<...x2xfN>`). It handles `stablehlo.complex`, `real`, `imag`, and constants.

## Lowering to TTIR

The final stage of the frontend is the actual dialect conversion from StableHLO to TTIR.

### Conversion Infrastructure

The conversion relies on patterns that map StableHLO ops to TTIR equivalents while performing type conversion. The infrastructure handles implicit broadcasting and constant value extraction.

**Sources:**[lib/Conversion/StableHLOToTTIR/StableHLOToTTIRPatterns.cpp 1-192](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Conversion/StableHLOToTTIR/StableHLOToTTIRPatterns.cpp#L1-L192)[lib/Dialect/StableHLO/Pipelines/StableHLOPipelines.cpp 17-134](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/StableHLO/Pipelines/StableHLOPipelines.cpp#L17-L134)

### Constant Value Extraction

The conversion logic includes helpers to identify and extract constant values for reduction initializations and other operations.

*   `checkInitValue`: Verifies if a `stablehlo.ConstantOp` matches typical reduction identity values like `0` or `-inf`[[lib/Conversion/StableHLOToTTIR/StableHLOToTTIRPatterns.cpp:119-187]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   `getConstantValue`: Traverses through operations that preserve constant semantics (Reshape, Broadcast, Typecast) to find the defining `ttir.constant`[[lib/Conversion/StableHLOToTTIR/StableHLOToTTIRPatterns.cpp:189-192]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).

### Implicit Broadcasting and Attributes

StableHLO convolution and windowed operations often have optional attributes that default to specific values (e.g., strides of 1, padding of 0).

*   `getI64ArrayOrDefault`: Extracts array attributes or provides defaults for convolution parameters like `window_strides` or `lhs_dilation`[[lib/Conversion/StableHLOToTTIR/StableHLOToTTIRPatterns.cpp:94-101]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).
*   `getPaddingOrDefault`: Specifically handles padding attributes represented as `DenseIntElementsAttr`[[lib/Conversion/StableHLOToTTIR/StableHLOToTTIRPatterns.cpp:104-110]](https://deepwiki.com/tenstorrent/tt-mlir/3.1-frontend:-stablehlo-to-ttir).

## Summary of Operation Mapping

| StableHLO / Shardy Op | TTIR Op | Key Logic |
| --- | --- | --- |
| `stablehlo.add/sub/mul` | `ttir.add/sub/multiply` | Direct mapping via pattern rewrites. |
| `stablehlo.constant` | `ttir.constant` | Handled by `ConstantOpConversionPattern`. |
| `stablehlo.composite` | `ttir.topk`, etc. | Legalized to Tenstorrent-specific TTIR ops. |
| `stablehlo.complex` | Decomposed | Converted to float-pair representations (`...x2xfN`). |
| `sdy.manual_computation` | `ttir.mesh_shard` | SPMD regions are converted to local-view TTIR operations with explicit shard ops. |
