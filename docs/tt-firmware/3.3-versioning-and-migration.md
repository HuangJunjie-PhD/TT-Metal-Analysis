# Versioning and Migration

Relevant source files
*   [README.md](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1)
*   [fw_pack-18.2.0.0.fwbundle](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/fw_pack-18.2.0.0.fwbundle)

## Purpose and Scope

This document outlines the versioning scheme used for tt-firmware releases and provides guidance on migrating between firmware versions. It covers version numbering conventions, where to find migration guides, and key considerations when upgrading firmware across Tenstorrent hardware platforms. For information about specific firmware bundles and their contents, see [Firmware Bundles](https://deepwiki.com/tenstorrent/tt-firmware/3-firmware-bundles) and [Official Releases](https://deepwiki.com/tenstorrent/tt-firmware/3.1-official-releases).

## Versioning Scheme

The tt-firmware repository uses a versioning scheme that follows a pattern of `X.Y.Z.W` for official releases:

Sources: [README.md 27](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L27-L27)

A notable change occurred in the versioning scheme between the previous release (80.18.1.0) and the current release (18.2.0.0), representing a shift in the versioning approach.

### Version Identification

Firmware bundles are named using their version numbers, making it easy to identify the version from the filename:

*   Current version: `fw_pack-18.2.0.0.fwbundle`
*   Previous version: `fw_pack-80.18.1.0.fwbundle`

A symbolic link named `latest.fwbundle` typically points to the most recent stable release.

Sources: [README.md 25-27](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L25-L27)

## Version History

The repository maintains a chronological history of firmware releases. Each release includes detailed release notes documenting new features, improvements, and bug fixes.

### Latest Release (18.2.0)

Key improvements in version 18.2.0 include:

*   Updated Blackhole ERISC FW to v1.4.0
*   Added ETH mailbox with new messages
*   Virtual UART enabled by default for Blackhole firmware
*   Various stability improvements

Sources: [README.md 31-49](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L31-L49)

## Migration Process

When upgrading between firmware versions, users should follow a structured migration process to ensure system stability and compatibility.

Sources: [README.md 51-53](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L51-L53)

### Migration Guides

Migration guides are provided for transitions between major releases. These guides contain:

*   Required configuration changes
*   API compatibility considerations
*   Deprecated features
*   New feature enablement instructions

The migration guide from version 80.18.1 to 18.2.0 is available in the tt-zephyr-platforms repository: `https://github.com/tenstorrent/tt-zephyr-platforms/tree/main/doc/release/migration-guide-18.2.0.md`

Sources: [README.md 51-53](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L51-L53)

## Board-Specific Migration Considerations

Different Tenstorrent boards (Grayskull, Wormhole, Blackhole) may have specific migration requirements. The following table outlines key considerations for each board type when migrating to version 18.2.0:

| Board Type | Migration Considerations |
| --- | --- |
| Blackhole | - ERISC FW updated to v1.4.0 - Virtual UART now enabled by default - New ETH mailbox functionality - Enhanced SMC I2C recovery |
| Wormhole | - PCIe Maximum Payload Size now set by TT-KMD |
| Grayskull | - PCIe Maximum Payload Size now set by TT-KMD |

## Experimental Bundles and Versioning

Alongside official releases, the repository contains experimental firmware bundles that address specific issues. These experimental bundles are based on the latest official firmware with targeted modifications.

Experimental bundles follow their own versioning scheme while referencing the official release they're derived from. Details about specific changes can be found in the experiments README file.

Sources: [README.md 61-65](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L61-L65)

## Migration Best Practices

1.   **Review Release Notes**: Always read the release notes for the target version before upgrading
2.   **Consult Migration Guides**: Follow the migration guide specific to your version transition
3.   **Test in Non-Production**: When possible, test firmware upgrades in a non-production environment
4.   **Backup Configuration**: Save your current configuration before upgrading
5.   **Verify Compatibility**: Ensure compatibility with your TT-UMD version

Sources: [README.md 51-53](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/README.md?plain=1#L51-L53)

## Accessing Previous Versions

Previous versions of the firmware can be accessed through git tags in the repository. To access a specific version:

1.   Navigate to the tt-firmware repository
2.   Access the tag corresponding to the desired version (e.g., v80.18.1.0)
3.   Download the firmware bundle for that version
