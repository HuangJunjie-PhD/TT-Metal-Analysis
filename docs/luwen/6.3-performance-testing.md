This page documents the performance testing capabilities provided by the Luwen project. It covers benchmark tools, test methodologies, and procedures for measuring and validating data transfer performance between host systems and Tenstorrent AI accelerator chips. For telemetry-specific monitoring, see [Telemetry](https://deepwiki.com/tenstorrent/luwen/4.4-telemetry), and for the Prometheus exporter that can expose performance metrics, see [Prometheus Exporter](https://deepwiki.com/tenstorrent/luwen/6.2-prometheus-exporter).

## Performance Testing Tools

Luwen includes several tools for measuring and testing performance across different communication interfaces and chip architectures.

### Ethernet Benchmark Utility

The primary performance benchmarking tool is the Ethernet benchmark utility, which measures data transfer speeds across the network-on-chip (NoC) interface.

Sources: [src/bin/ethernet_benchmark.rs 8-79](https://github.com/tenstorrent/luwen/blob/55e35616/src/bin/ethernet_benchmark.rs#L8-L79)[src/bin/ethernet_benchmark.rs 81-108](https://github.com/tenstorrent/luwen/blob/55e35616/src/bin/ethernet_benchmark.rs#L81-L108)

The benchmark utility supports different chip architectures and provides the following key features:

*   Random test pattern generation for data integrity validation
*   Support for Direct Memory Access (DMA) to accelerate transfers
*   Architecture-specific testing parameters for Wormhole, Grayskull, and Blackhole chips
*   Performance reporting in MB/s for both read and write operations

#### Benchmark Implementation Details

The core of the benchmark utility is the `read_write_test` function which:

1.   Generates random test data
2.   Optionally configures DMA for accelerated transfers
3.   Measures write performance using `noc_write()`
4.   Measures read performance using `noc_read()`
5.   Verifies data integrity by comparing written and read data

Sources: [src/bin/ethernet_benchmark.rs 8-79](https://github.com/tenstorrent/luwen/blob/55e35616/src/bin/ethernet_benchmark.rs#L8-L79)

## Performance Test Methodologies

Luwen implements several methodologies for testing different aspects of chip performance.

### DMA vs. Regular I/O Performance Testing

The benchmark tools can measure the performance difference between DMA-accelerated transfers and regular I/O operations. DMA transfers typically provide higher throughput for large data sizes by offloading transfer management from the CPU.

The Ethernet benchmark utility supports configuring DMA with parameters such as:

*   Read/write thresholds that determine when to use DMA
*   MSI (Message Signaled Interrupts) configuration
*   64-bit DMA addressing support

Sources: [src/bin/ethernet_benchmark.rs 17-41](https://github.com/tenstorrent/luwen/blob/55e35616/src/bin/ethernet_benchmark.rs#L17-L41)

### Register Access Testing

The test suite includes comprehensive register access testing that validates both correctness and performance:

1.   **Aligned Register Access**: Tests basic 32-bit aligned read/write operations
2.   **Unaligned Register Access**: Tests operations with 1-byte and 3-byte offsets
3.   **Sequential Pattern Tests**: Verifies data integrity with recognizable patterns
4.   **Block Transfer Tests**: Measures performance of bulk transfers from 8 bytes to 1KB

Sources: [tests/read_write_test.rs 94-465](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L94-L465)

### Block Transfer Performance

Block transfer testing is critical for validating the performance of data-intensive operations. The test suite includes:

*   **Small Block Tests**: Testing transfers of 8-16 bytes, important for register operations
*   **Medium Block Tests**: Testing transfers of 64-256 bytes, common for command packets
*   **Large Block Tests**: Testing transfers up to 1KB to measure sustained throughput

For each test size, both aligned and unaligned transfers are tested to measure the impact of memory alignment on performance.

Sources: [tests/read_write_test.rs 237-465](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L237-L465)

## Hardware Requirements

All performance tests require physical Tenstorrent hardware. The test suite is designed to automatically detect compatible hardware and will skip tests if appropriate hardware isn't available.

The tests support:

*   Wormhole chips
*   Grayskull chips
*   Blackhole chips

The test utilities include helper functions to detect available hardware and determine chip types:

Sources: [tests/test_utils.rs 7-68](https://github.com/tenstorrent/luwen/blob/55e35616/tests/test_utils.rs#L7-L68)[tests/read_write_test.rs 30-79](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L30-L79)

## Running Performance Tests

### Ethernet Benchmark

To run the Ethernet benchmark utility:

```
cargo run --bin ethernet_benchmark
```

This will test data transfer performance across all detected compatible chips.

Sources: [src/bin/ethernet_benchmark.rs 81-108](https://github.com/tenstorrent/luwen/blob/55e35616/src/bin/ethernet_benchmark.rs#L81-L108)

### Register and Block Transfer Tests

The register access tests are implemented as part of the test suite and can be run with:

```
cargo test --test read_write_test -- --ignored
```

These tests are marked with `#[ignore = "Requires hardware"]` to prevent them from running in environments without physical hardware.

Sources: [tests/read_write_test.rs 94-465](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L94-L465)

## Interpreting Test Results

The performance test utilities provide output in the following formats:

### Ethernet Benchmark Results

```
Chip 0 write: 1234.5 MB/s, read: 987.6 MB/s
```

This output indicates the write and read speeds in MB/s for each detected chip.

Sources: [src/bin/ethernet_benchmark.rs 99-104](https://github.com/tenstorrent/luwen/blob/55e35616/src/bin/ethernet_benchmark.rs#L99-L104)

### Register Access Test Results

The register access tests use assertions to validate correctness. If a test fails, it will indicate which operation failed and provide the expected and actual values.

For example:

```
thread 'tests::wormhole_test_aligned_register_access' panicked at 'Aligned read/write of 0xfaca failed, got 0x0'
```

Sources: [tests/read_write_test.rs 105-108](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L105-L108)[tests/read_write_test.rs 66-75](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L66-L75)

## Performance Validation Testing

In addition to benchmarking, Luwen includes validation tests that ensure performance-related functionality works correctly across all supported chip architectures.

### Telemetry Performance Testing

The telemetry tests verify that chip status information can be collected efficiently:

*   Testing telemetry collection on Wormhole, Grayskull, and Blackhole chips
*   Ensuring consistent and reliable telemetry data retrieval
*   Validating chip status reporting mechanisms

Sources: [tests/telem_test.rs 24-171](https://github.com/tenstorrent/luwen/blob/55e35616/tests/telem_test.rs#L24-L171)

### Boot File System Performance

For Blackhole chips, tests validate the performance of the boot file system interface:

*   Reading board configuration data from SPI flash
*   Accessing flash information and firmware configuration
*   Testing read operations for various filesystem tags

Sources: [tests/boot_fs_test.rs 21-97](https://github.com/tenstorrent/luwen/blob/55e35616/tests/boot_fs_test.rs#L21-L97)

## Summary

Luwen provides a comprehensive set of performance testing tools and methodologies for measuring and validating the performance of data transfer operations between host systems and Tenstorrent AI accelerator chips. These tools support testing different chip architectures, communication interfaces, and transfer methods, allowing developers to identify performance bottlenecks and optimize their applications accordingly.
