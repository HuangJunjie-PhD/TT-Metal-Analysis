## Overview

The Device Management Controller (DMC) is a firmware application running on a RISC-V processor with Zephyr RTOS. It is responsible for board-level device management operations, including chip reset coordination, JTAG-based bootrom loading, peripheral control (fans, power monitoring), and bidirectional communication with the Chip Management Firmware (CMFW) running on ARC processors within Blackhole ASICs.

For information about the System Management Controller that coordinates with DMC, see [SMC](https://deepwiki.com/tenstorrent/tt-system-firmware/2.1-system-management-controller-(smc)). For details on the CM2DM communication protocol specifically, see [CM2DM Communication Protocol](https://deepwiki.com/tenstorrent/tt-system-firmware/2.2.2-cm2dm-communication-protocol). For JTAG and chip reset operations, see [JTAG and Chip Reset Management](https://deepwiki.com/tenstorrent/tt-system-firmware/2.2.3-jtag-and-chip-reset-management).

**Key Responsibilities:**

*   Perform JTAG-based chip resets and bootrom loading
*   Process messages from CMFW (CM2DM protocol)
*   Aggregate fan control across multiple chips
*   Monitor board-level power consumption
*   Handle critical events (thermal trips, watchdog timeouts, power failures)
*   Forward chip telemetry and logs to SMC

**Version:** 0.23.99.0 [app/dmc/VERSION 1-6](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/app/dmc/VERSION#L1-L6)

## Hardware Interfaces

### Chip Control Interfaces

**DMC Hardware Control Interfaces**

| Interface | Purpose | Implementation |
| --- | --- | --- |
| **JTAG** | Chip reset, bootrom loading, register access | `chip->config.jtag` device, jtag_bootrom library |
| **SMBus** | Bidirectional communication with CMFW on ARC | `chip->config.arc.smbus` device, CM2DM protocol |
| **GPIO - ASIC Reset** | Assert/deassert chip reset | `chip->config.asic_reset` GPIO |
| **GPIO - SPI Reset** | Reset SPI peripherals | `chip->config.spi_reset` GPIO |
| **GPIO - SPI Mux** | Control SPI bus routing | `chip->config.spi_mux` GPIO |
| **GPIO - Thermal Trip** | Detect thermal trip events (input) | `chip->config.therm_trip` GPIO interrupt |
| **GPIO - PGOOD** | Monitor power rail status (input) | `chip->config.pgood` GPIO interrupt |

### JTAG Reset Implementation

The function `jtag_bootrom_reset_sequence()`[lib/tenstorrent/jtag_bootrom/reset.c 32-84](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/reset.c#L32-L84) performs a complete chip reset:

**Step-by-step JTAG Reset Flow:**

1.   **Cancel SMBus Transfers**[lib/tenstorrent/jtag_bootrom/reset.c 46](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/reset.c#L46-L46)

    *   Clear bus cancel flag to allow I2C for strap setting

2.   **Assert Reset Signals**[lib/tenstorrent/jtag_bootrom/jtag_bootrom.c 97-98](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/jtag_bootrom.c#L97-L98)

    *   `bh_chip_assert_asic_reset(chip)` - Assert ASIC_RESET GPIO
    *   `bh_chip_assert_spi_reset(chip)` - Assert SPI_RESET GPIO
    *   `jtag_setup()` - Initialize JTAG interface

3.   **Wait 1ms**[lib/tenstorrent/jtag_bootrom/jtag_bootrom.c 107](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/jtag_bootrom.c#L107-L107)

    *   Allow reset to settle

4.   **Configure Straps**[lib/tenstorrent/jtag_bootrom/jtag_bootrom.c 109](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/jtag_bootrom.c#L109-L109)

    *   `bh_chip_set_straps(chip)` - Drive GPIO strapping pins
    *   Straps configure boot mode, PCIe settings, etc.

5.   **Deassert Reset**[lib/tenstorrent/jtag_bootrom/jtag_bootrom.c 111-112](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/jtag_bootrom.c#L111-L112)

    *   Release ASIC_RESET and SPI_RESET
    *   Wait 2ms for chip to initialize

6.   **Wait for JTAG Ready**[lib/tenstorrent/jtag_bootrom/jtag_bootrom.c 117-123](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/jtag_bootrom.c#L117-L123)

    *   Poll JTAG until ID code readable (0x138A5)
    *   Wait for AXI interface to become accessible

7.   **Load Bootrom**[lib/tenstorrent/jtag_bootrom/reset.c 57](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/reset.c#L57-L57)

    *   `jtag_bootrom_patch_offset()` - Write bootcode via JTAG
    *   Target address: 0x80 (ARC ICCM)
    *   Bootcode size: ~250 words

8.   **Verify Bootrom** (optional) [lib/tenstorrent/jtag_bootrom/reset.c 63-65](https://github.com/tenstorrent/tt-system-firmware/blob/ab59b09c/lib/tenstorrent/jtag_bootrom/reset.c#L63-L65)
