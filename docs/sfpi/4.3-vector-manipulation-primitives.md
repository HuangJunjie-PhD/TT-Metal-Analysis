# Vector Manipulation Primitives

This page documents vector data manipulation primitives in the SFPI library that perform data movement, rearrangement, and element reordering operations on vector registers. These functions enable efficient reorganization of data within the SFPU's 64-element vector registers.

For mathematical operations on vectors (e.g., `lut`, `exexp`, `setexp`), see [Functional Math Operations](https://deepwiki.com/tenstorrent/sfpi/4.1-functional-math-operations). For type conversion between different numeric formats, see [Type Conversions and Casting](https://deepwiki.com/tenstorrent/sfpi/4.2-type-conversions-and-casting).

**Scope**: This page covers functions that rearrange or exchange vector data without performing mathematical transformations:

*   Subvector transpose operations
*   Subvector shuffle operations
*   Vector swap and min/max operations

## Overview of Vector Organization

The SFPU operates on 64-element vectors organized as four 16-element subvectors. Vector manipulation primitives exploit this structure to perform efficient data reorganization.

Sources: [include/wormhole/sfpi_lib.h 274-299](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/wormhole/sfpi_lib.h#L274-L299)[include/blackhole/sfpi_lib.h 284-309](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L284-L309)

## Subvector Transpose Operation

The `subvec_transp` function performs a 4x4 transpose across four local registers, treating each register as containing four 16-element subvectors and transposing the subvector layout.

### Function Signature

### Parameters

| Parameter | Type | Direction | Description |
| --- | --- | --- | --- |
| `l0` | `__vBase&` | In/Out | First vector register (modified in place) |
| `l1` | `__vBase&` | In/Out | Second vector register (modified in place) |
| `l2` | `__vBase&` | In/Out | Third vector register (modified in place) |
| `l3` | `__vBase&` | In/Out | Fourth vector register (modified in place) |

### Operation Description

The transpose operation reorganizes data across four 64-element vectors. Before the operation, each vector contains four consecutive subvectors. After the operation, subvector boundaries are reorganized so that corresponding elements from each input vector are grouped together.

### Hardware Intrinsic

The function directly invokes the `__builtin_rvtt_sfptransp` intrinsic, which maps to the SFPU transpose instruction.

## Subvector Shuffle Operations

Shuffle operations perform rotations or shifts within the subvector structure of a single 64-element vector.

### Subvector Shuffle Rotate Right by 1

Rotates each 16-element subvector right by one position, with wraparound. Element 15 of each subvector moves to position 0.

| Parameter | Type | Description |
| --- | --- | --- |
| `src` | `const __vBase&` | Source vector to shuffle |
| **Returns** | `__rvtt_vec_t` | Shuffled vector (can be assigned to any vector type) |

### Subvector Shuffle Shift Right by 1

Shifts each 16-element subvector right by one position, introducing zeros at position 0. Element 15 of each subvector is discarded.

| Parameter | Type | Description |
| --- | --- | --- |
| `src` | `const __vBase&` | Source vector to shuffle |
| **Returns** | `__rvtt_vec_t` | Shuffled vector (can be assigned to any vector type) |

## Vector Swap and Min/Max Operations

These operations work on pairs of complete 64-element vectors, either exchanging their contents or computing element-wise comparisons.

### Vector Swap

Exchanges the complete contents of two vector registers. After the operation, `dst` contains the original value of `src`, and `src` contains the original value of `dst`.

| Parameter | Type | Direction | Description |
| --- | --- | --- | --- |
| `dst` | `__vBase&` | In/Out | First vector (receives `src` value) |
| `src` | `__vBase&` | In/Out | Second vector (receives `dst` value) |

### Vector Min/Max

Performs element-wise comparison between two vectors, storing the minimum values in `dst` and maximum values in `src`. For each element position `i`:

*   `dst[i] = min(dst[i], src[i])`
*   `src[i] = max(dst[i], src[i])`

| Parameter | Type | Direction | Description |
| --- | --- | --- | --- |
| `dst` | `__vBase&` | In/Out | Vector containing minimums after operation |
| `src` | `__vBase&` | In/Out | Vector containing maximums after operation |

## Platform Availability

All vector manipulation primitives are available on both Wormhole and Blackhole architectures with identical interfaces and behavior. The implementations use architecture-specific hardware intrinsics but expose a unified API.

| Operation | Wormhole | Blackhole | Underlying Intrinsic |
| --- | --- | --- | --- |
| `subvec_transp` | ✓ | ✓ | `__builtin_rvtt_sfptransp` |
| `subvec_shflror1` | ✓ | ✓ | `__builtin_rvtt_sfpshft2_e` |
| `subvec_shflshr1` | ✓ | ✓ | `__builtin_rvtt_sfpshft2_e` |
| `vec_swap` | ✓ | ✓ | `__builtin_rvtt_sfpswap` |
| `vec_min_max` | ✓ | ✓ | `__builtin_rvtt_sfpswap` |
