The TTX Workload System handles the loading and execution of burn-in workloads on Tenstorrent AI hardware. This system manages TTX files (packaged workloads), loads them onto chip cores using NOC operations, and provides mechanisms to verify workload completion. The system supports different chip architectures (Grayskull, Wormhole, Blackhole) and handles both broadcast and per-core workload deployment.

For information about hardware abstraction and chip-specific implementations, see [Hardware Abstraction Layer](https://deepwiki.com/tenstorrent/tt-burnin/2.2-hardware-abstraction-layer). For details about hardware management utilities used by this system, see [Hardware Management Utilities](https://deepwiki.com/tenstorrent/tt-burnin/2.4-hardware-management-utilities).

## TTX File Format

TTX files are ZIP archives containing workload binaries and metadata for Tenstorrent chips. The file format supports both binary and hex representations of workload data organized by core coordinates.

### Core File Organization

Each core directory follows the naming convention `x-y` where `x` and `y` are the core coordinates. The system prioritizes binary files over hex files when both formats are present.

| File Type | Purpose | Format |
| --- | --- | --- |
| `image.bin` | Main workload binary | Binary with chunk headers |
| `ckernels.bin` | Compute kernel binaries | Binary with chunk headers |
| `image.hex` | Main workload (legacy) | Hex format with address lines |
| `ckernels.hex` | Compute kernels (legacy) | Hex format with address lines |
