## Purpose and Scope

This document describes the architecture of the TT-Studio frontend application. It outlines the technologies used, component structure, state management patterns, routing system, and UI design patterns. The frontend is a React-based Single Page Application (SPA) designed to manage AI model lifecycles and provide specialized interfaces for interacting with Tenstorrent hardware.

## Technology Stack

The TT-Studio frontend is built using modern JavaScript technologies centered around React 18 with TypeScript.

| Category | Technologies | Version |
| --- | --- | --- |
| Framework | React with React DOM | ^18.3.1 |
| Language | TypeScript | ^5.6.3 |
| Build Tool | Vite with SWC plugin | ^6.3.5 |
| Routing | React Router DOM | ^6.28.0 |
| Styling | Tailwind CSS, Shadcn/UI | ^4.1.12 |
| UI Components | Radix UI primitives, Lucide React, Tabler Icons | Various |
| State Management | React Query (TanStack), React Context | ^5.28.4 |
| Animation | Framer Motion | ^11.18.2 |
| Form Handling | React Hook Form, Zod validation | ^7.53.2, ^3.23.8 |
| API Communication | Axios | ^1.7.7 |

The build system uses `@vitejs/plugin-react-swc` for fast compilation and hot module replacement during development.

### Deployment Wizard (Stepper)

The deployment process is managed by a multi-step wizard implemented in `SelectionSteps.tsx`. It dynamically adjusts based on the detected hardware (e.g., QB2/P300Cx2 boards).

### Deployment Monitoring

Once a model is deploying, `DeployModelStep.tsx` handles asynchronous job monitoring.

## API Communication

### Vite Proxy Configuration

The frontend communicates with the backend services through a Vite proxy that maps specific API paths to the Django backend container.

| Proxy Path | Actual Backend Path | Purpose |
| --- | --- | --- |
| `/docker-api` | `/docker` | Container and deployment management |
| `/models-api` | `/models` | Model metadata and status |
| `/logs-api` | `/logs` | Streaming container logs |
| `/board-api` | `/board` | Hardware/Chip status |

### Server-Sent Events (SSE) Support

The proxy configuration in `vite.config.ts` specifically handles `text/event-stream` to support real-time log streaming from the backend.

## State Management Patterns

The application uses a decentralized state pattern combining React Context for UI state and TanStack Query for server state.

1.   **Server State**: Managed by `QueryClient` in `App.tsx`. Handles caching and synchronization of model lists and hardware status.
2.   **Hardware State**: `DeviceStateProvider` tracks the current state of Tenstorrent chips across the application.
3.   **UI Refresh Logic**: `RefreshProvider` allows components to trigger a global data re-fetch (e.g., after a successful model deletion).
4.   **Deployment Context**: `ModelsProvider` stores the list of currently available and deployed models.

## Build and Environment

The build process is managed by Vite and utilizes environment variables for feature toggling and versioning.

*   **Versioning**: The application version is injected from `package.json` into `import.meta.env.VITE_PACKAGE_VERSION`.
*   **Development Tools**: The `vite.config.ts` includes polling for the HMR (Hot Module Replacement) server to ensure stability in Docker environments.
*   **Strict Linting**: The `package.json` defines strict linting and type-checking scripts (`npm run type-check`, `npm run lint`) used in CI/CD.
