## Purpose and Scope

This page describes the overall physical organization of Tenstorrent ASICs, focusing on the tile-based architecture, Network-on-Chip (NoC) topology, and the distinct types of tiles present in the chip. This provides the foundational understanding of how computational resources, memory, and I/O are organized at the chip level.

## Overall Chip Organization

Tenstorrent ASICs employ a **tile-based architecture** where each tile is a semi-autonomous processing or I/O element connected via a high-bandwidth interconnect fabric. The chip is organized as a 2D grid of tiles, with each tile type serving a specific purpose in the overall system.

### Wormhole B0 Tile Inventory

**Key specifications:**

*   **Total tiles in grid:** 10 columns × 12 rows = 120 tile positions
*   **Tensix tiles:** 80 total (some may be fused off for yield)
*   **Ethernet tiles:** 16 total
*   **DRAM tiles:** 18 total (each 2 GiB GDDR6 bank is accessible via 3 tile positions)
*   **PCIe tile:** 1
*   **ARC tile:** 1
*   **Empty tiles:** Remaining positions contain only routing infrastructure

Sources: [WormholeB0/README.md 1-10](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/README.md?plain=1#L1-L10)[Diagrams/Src/NoC.lua 12-44](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Diagrams/Src/NoC.lua#L12-L44)

## Network-on-Chip (NoC) Topology

The chip features a **dual NoC architecture** providing redundant, high-bandwidth communication paths between all tiles. Each NoC operates independently with its own routing discipline.

### Dual NoC Architecture

**NoC characteristics:**

| Feature | NoC #0 | NoC #1 |
| --- | --- | --- |
| **Base address** | `0xFFB2_0000` | `0xFFB3_0000` |
| **Routing order** | X-major (horizontal first) | Y-major (vertical first) |
| **Data width** | 256 bits per flit | 256 bits per flit |
| **Topology** | 2D torus (10×12) | 2D torus (10×12) |
| **Purpose** | Primary data movement | Redundant paths, load balancing |

**Topology details:**

*   **2D torus structure:** The 10×12 grid wraps around at edges (left connects to right, top connects to bottom)
*   **Each tile contains 2 NIUs:** One for each NoC, providing dual independent data paths
*   **Router per tile:** Every tile position includes a NoC router for packet forwarding
*   **Virtual channels:** Multiple virtual channels prevent deadlock and enable QoS

Sources: [WormholeB0/NoC/README.md 1-11](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/README.md?plain=1#L1-L11)[WormholeB0/NoC/MemoryMap.md 1-29](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/MemoryMap.md?plain=1#L1-L29)

### NoC Coordinate System

**Coordinate encoding in NoC transactions:**

*   **Physical (X, Y) coordinates:** 6 bits each, range [0, 9] for X, [0, 11] for Y
*   **Stored in:**`NOC_TARG_ADDR_MID` and `NOC_RET_ADDR_MID` registers
*   **Field layout:** Bits [36:41] = X coordinate, bits [42:47] = Y coordinate
*   **Coordinate translation:** Optional logical-to-physical mapping via `NOC_X_ID_TRANSLATE_TABLE_0` and `NOC_Y_ID_TRANSLATE_TABLE_0`

Sources: [WormholeB0/NoC/MemoryMap.md 93-114](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/MemoryMap.md?plain=1#L93-L114)[Diagrams/Src/NoC.lua 1-120](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Diagrams/Src/NoC.lua#L1-L120)

## Tile Types

### Tensix Tiles

**Purpose:** Primary compute engines for tensor and matrix operations.

**Key components per tile:**

**Specifications:**

| Component | Specification |
| --- | --- |
| **Count** | 80 tiles (64-80 active) |
| **L1 memory** | 1464 KiB per tile, 16-way banked |
| **Control processors** | 5× RV32IM cores @ 1 GHz |
| **Compute units** | Matrix Unit (FPU), Vector Unit (SFPU), Scalar Unit (ThCon) |
| **Peak throughput** | 4.096 TFLOP/s (low-precision) + 0.064 TFLOP/s (FP32) per tile |
| **Memory bandwidth** | 256 bits/cycle read + 256 bits/cycle write per NoC |

**Programming model:**

*   **RISCV B + NC:** Orchestrate data movement between L1 and NoC
*   **RISCV T0/T1/T2:** Orchestrate Tensix coprocessor pipeline (UNPACK/MATH/PACK phases)
*   **Tensix coprocessor:** High-throughput tensor operations via instruction streams

Sources: [WormholeB0/TensixTile/README.md 1-16](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/README.md?plain=1#L1-L16)[WormholeB0/TensixTile/BabyRISCV/README.md 1-12](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/BabyRISCV/README.md?plain=1#L1-L12)

### Ethernet Tiles

**Purpose:** Inter-chip networking via 100 Gigabit Ethernet links.

**Key components per tile:**

**Specifications:**

| Component | Specification |
| --- | --- |
| **Count** | 16 tiles |
| **L1 memory** | 256 KiB per tile |
| **Control processor** | 1× RV32IM core @ 1 GHz |
| **Network interface** | 100 Gigabit Ethernet (100GBASE-R) |
| **Link type** | QSFP-DD connectors |

**Firmware model:**

*   **Cooperative execution:** RISCV E runs both Tenstorrent firmware and customer code
*   **Tenstorrent firmware:** Link training, baseline data movement service
*   **Customer code:** Called by firmware for custom packet processing

Sources: [WormholeB0/EthernetTile/BabyRISCV/README.md 1-14](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/EthernetTile/BabyRISCV/README.md?plain=1#L1-L14)

### DRAM Tiles

**Purpose:** Off-chip memory interface to GDDR6 DRAM.

**Key components per tile:**

**Specifications:**

| Component | Specification |
| --- | --- |
| **Count** | 18 tile positions (6 physical DRAM banks) |
| **Memory per bank** | 2 GiB GDDR6 |
| **Total capacity** | 12 GiB (6 banks × 2 GiB) |
| **NIUs per tile** | 6 (3 per NoC) |
| **Redundancy** | Each 2 GiB bank accessible via 3 tile positions |

**Access model:**

*   **No control processors:** DRAM tiles only respond to NoC transactions
*   **Multiple NIUs:** Each 2 GiB bank has 6 NIUs (3 per NoC) for bandwidth
*   **NoC transactions:** Read/write requests from other tiles

Sources: [WormholeB0/NoC/README.md 15-16](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/README.md?plain=1#L15-L16)[WormholeB0/README.md 5-6](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/README.md?plain=1#L5-L6)

### PCIe Tile

**Purpose:** Host interface for communication with x86 CPU.

**Transaction flow (Host-to-Device):**

**Specifications:**

| Component | Specification |
| --- | --- |
| **Count** | 1 tile |
| **Interface** | PCIe 4.0 x16 |
| **Bandwidth** | ~25 GB/s bidirectional |
| **Address translation** | Inbound/Outbound iATU + Configurable TLBs |
| **DMA engines** | Multiple DMA engines for bulk transfers |

**Programming interfaces:**

*   **Host writes to device:** PCIe → AXI → iATU → TLB → NoC transaction
*   **Device writes to host:** NoC transaction → Outbound iATU → AXI → PCIe
*   **Ordering control:** TLB configuration bits control ordering semantics

Sources: [WormholeB0/NoC/README.md 17-18](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/README.md?plain=1#L17-L18)[WormholeB0/NoC/Ordering.md 35-46](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/Ordering.md?plain=1#L35-L46)[Diagrams/Src/EdgeTile.lua 131-229](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/Diagrams/Src/EdgeTile.lua#L131-L229)

### ARC Tile

**Purpose:** System management and control.

**Specifications:**

| Component | Specification |
| --- | --- |
| **Count** | 1 tile |
| **Processors** | 4× ARC CPUs |
| **Memory** | 512 KiB CSM (Closely-coupled Shared Memory) |
| **Purpose** | Chip initialization, power management, monitoring |

**Note:** Customer software typically does not directly interact with the ARC tile. It is used by Tenstorrent firmware for chip-level management tasks.

Sources: [WormholeB0/README.md 7-8](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/README.md?plain=1#L7-L8)

### Empty Tiles

**Purpose:** NoC routing infrastructure without compute or memory resources.

**Contents:**

*   **NoC router:** Packet forwarding logic
*   **2× NIUs:** Provide access to NIU configuration registers only
*   **No compute:** No processors or functional units
*   **Minimal memory:** Only NIU control/status registers

Empty tiles exist at positions in the 10×12 grid not occupied by other tile types. They maintain NoC connectivity across the entire grid.

Sources: [WormholeB0/NoC/README.md 21-22](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/README.md?plain=1#L21-L22)

## Memory Addressing Across Tiles

### NoC Transaction Addressing

Each NoC transaction specifies both **physical coordinates** and **memory address**:

**Address structure in `NOC_TARG_ADDR_MID` / `NOC_RET_ADDR_MID`:**

| Bits | Field | Description |
| --- | --- | --- |
| [0:35] | Memory address | 36-bit byte address within target tile |
| [36:41] | X coordinate | 6-bit X position (0-9) |
| [42:47] | Y coordinate | 6-bit Y position (0-11) |
| [48:63] | Reserved | Not used |

**Address space per tile type:**

| Tile Type | L1/Memory Base | L1/Memory Size | MMIO Base | Notes |
| --- | --- | --- | --- | --- |
| Tensix | `0x0000_0000` | 1464 KiB | `0xFFB0_0000` | L1 + device registers |
| Ethernet | `0x0000_0000` | 256 KiB | `0xFFB0_0000` | L1 + device registers |
| DRAM | `0x0000_0000` | 2 GiB | N/A | Pure memory (no MMIO) |
| PCIe | N/A | N/A | `0xF000_0000` | Configuration registers only |

**Transaction types:**

*   **Read:** Fetch data from `(X, Y, Address)` → return to initiator
*   **Write:** Send data to `(X, Y, Address)`
*   **Atomic:** Read-modify-write at `(X, Y, Address)` in L1
*   **Broadcast:** Send to rectangle of (X, Y) coordinates

Sources: [WormholeB0/NoC/MemoryMap.md 73-114](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/MemoryMap.md?plain=1#L73-L114)[WormholeB0/TensixTile/BabyRISCV/README.md 80-106](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/TensixTile/BabyRISCV/README.md?plain=1#L80-L106)

### NIU Configuration Registers

Each tile's NIUs expose control/status registers for initiating and monitoring transactions:

**Key register groups:**

| Register Base | Purpose | Size |
| --- | --- | --- |
| `NIU_BASE + 0x000` | Request initiator #0 | 44 bytes |
| `NIU_BASE + 0x02C` | NIU identification | 8 bytes |
| `NIU_BASE + 0x054` | Combined initiator status | 4 bytes |
| `NIU_BASE + 0x100` | NIU/router configuration | 60 bytes |
| `NIU_BASE + 0x200` | Performance counters | 256 bytes |
| `NIU_BASE + 0x400` | Request initiator #1 | 44 bytes |
| `NIU_BASE + 0x800` | Request initiator #2 | 44 bytes |
| `NIU_BASE + 0xC00` | Request initiator #3 | 44 bytes |

**Programming sequence for NoC transaction:**

1.   Configure `NOC_TARG_ADDR_LO/MID` with destination (X, Y, address)
2.   Configure `NOC_RET_ADDR_LO/MID` with return address (for reads)
3.   Configure `NOC_AT_LEN_BE` with length or byte-enable mask
4.   Configure `NOC_CTRL` with transaction type and flags
5.   Write `1` to `NOC_CMD_CTRL` to initiate transaction
6.   Poll `NOC_CMD_CTRL` or counters to check completion

Sources: [WormholeB0/NoC/MemoryMap.md 11-45](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/MemoryMap.md?plain=1#L11-L45)[WormholeB0/NoC/Counters.md 1-40](https://github.com/tenstorrent/tt-isa-documentation/blob/9600a914/WormholeB0/NoC/Counters.md?plain=1#L1-L40)

## Summary

The Tenstorrent chip architecture provides a highly parallel, tile-based compute platform with:

*   **80 Tensix tiles** for tensor/matrix computation
*   **Dual NoC fabric** for redundant, high-bandwidth communication
*   **Distributed memory hierarchy** (L1 per tile + shared GDDR6)
*   **Flexible I/O** (Ethernet for inter-chip, PCIe for host)
*   **Heterogeneous tiles** optimized for specific functions

This architecture enables massive parallelism (327+ TFLOP/s aggregate) while maintaining programmability through standard RISC-V processors and well-defined memory/communication interfaces.
