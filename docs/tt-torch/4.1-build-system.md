This document describes the build system architecture for tt-torch, covering dependency management, CMake configuration, Python packaging, and containerized builds. The build system orchestrates multiple external projects and integrates C++ libraries with Python packages using scikit-build.

For information about testing frameworks, see [Testing Framework](https://deepwiki.com/tenstorrent/tt-torch/4.2-testing-framework). For CI/CD pipeline details, see [CI/CD Pipeline](https://deepwiki.com/tenstorrent/tt-torch/4.3-cicd-pipeline).

**Build Command Options**

| --- | --- | --- |

## Environment Setup

The build system requires specific environment configuration managed through [env/activate](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/env/activate):

**Library Path Configuration**

The `LD_LIBRARY_PATH` includes multiple critical paths:

*   `$TT_TORCH_HOME/env/venv/lib/python3.11/site-packages/torch/lib` - PyTorch libraries
*   `$TTMLIR_TOOLCHAIN_DIR/lib` - MLIR toolchain libraries
*   `$TT_TORCH_HOME/install/lib/` - tt-torch installation libraries

Sources: [env/activate 6-52](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/env/activate#L6-L52)

The complete build process follows a structured sequence from environment setup to final package installation:
