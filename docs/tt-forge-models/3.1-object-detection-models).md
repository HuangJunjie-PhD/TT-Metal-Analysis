This page covers the object detection model implementations in the tt-forge-models repository, focusing on the YOLO family of models and CenterNet. These models detect and localize objects within images, returning bounding boxes, confidence scores, and class predictions.

For information about image classification models, see [Image Classification Models](/tenstorrent/tt-forge-models/3.2-image-classification-models). For general computer vision model architecture, see [Computer Vision Models](/tenstorrent/tt-forge-models/3-computer-vision-models).

## Common Interface Methods

All object detection models implement the standard `ForgeModel` interface with consistent method signatures:

* **`load_model(dtype_override=None)`**: Returns the model instance with optional dtype conversion
* **`load_inputs(dtype_override=None)`**: Returns preprocessed sample inputs
* **`_get_model_info(variant=None)`**: Returns `ModelInfo` with task and metadata
* **`post_processing()`** (YOLOv4 only): Performs NMS and visualization

The models support the standard dtype override pattern for hardware compatibility, particularly for bfloat16 conversion on TT hardware.
