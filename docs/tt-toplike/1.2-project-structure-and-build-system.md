The `tt-toplike` project is organized as a multi-binary Rust workspace designed for high-performance telemetry visualization and hardware monitoring. It prioritizes zero-cost abstractions in the render path, modular backend support, and reproducible Debian packaging for deployment on Tenstorrent-equipped systems.

## Workspace Layout and Binary Targets

The codebase is structured around a central library crate (`src/lib.rs`) that provides core traits and data models, with specialized binary entry points for different user interfaces.

| Binary Target | Path | Feature Flag | Description |
| --- | --- | --- | --- |
| `tt-toplike-tui` | `src/bin/tui.rs` | `tui` | The primary terminal-based monitoring tool using `ratatui`. [Cargo.toml 11-15](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L11-L15) |
| `tt-toplike-gui` | `src/bin/gui.rs` | `gui` | Legacy native GUI using the `iced` framework. [Cargo.toml 16-19](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L16-L19) |
| `tt-toplike-egui` | `src/bin/egui.rs` | `egui` | Lightweight dashboard using `eframe` and `egui_plot`. [Cargo.toml 21-24](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L21-L24) |
| `tt-toplike-app` | `src/bin/app.rs` | `app` | Native window host that wraps the TUI inside a PTY (Portable PTY). [Cargo.toml 26-29](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L26-L29) |

## Cargo Feature Flags

The build system uses feature flags to manage dependencies, particularly for frontends and invasive hardware access.

### Frontend Features

*   `tui`: Enables `ratatui` and `crossterm` for terminal rendering. [Cargo.toml 119](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L119-L119)
*   `gui`: Enables the `iced` framework for the legacy graphical interface. [Cargo.toml 120](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L120-L120)
*   `egui`: Enables `eframe` and `egui_plot` for the lightweight dashboard. [Cargo.toml 121](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L121-L121)
*   `app`: Enables `portable-pty` and `vte` to host the TUI in a native window. [Cargo.toml 122](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L122-L122)

### Backend Features

*   `json-backend`: Enables subprocess-based polling of `tt-smi` to parse JSON telemetry. This is considered a "safe" backend. [Cargo.toml 129-130](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L129-L130)
*   `linux-procfs`: Enables the `procfs` crate for enhanced process detection (scanning `/proc/` for open Tenstorrent device descriptors). [Cargo.toml 130](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L130-L130)
*   `luwen-backend`: **Optional/Invasive.** Enables direct PCI BAR0 access via `all-smi-luwen`. It is excluded from default builds to prevent accidental hardware disruption during active workloads. [Cargo.toml 125-128](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/Cargo.toml#L125-L128)

## Debian Packaging

The project includes a robust Debian packaging system located in the `debian/` directory, supporting `noble` and `jammy` suites.

### Build Strategy: `build-deb.sh`

The `build-deb.sh` script automates the creation of `.deb` files. It follows a strict "offline build" strategy required by standard Debian build daemons.

1.   **Vendoring**: It runs `cargo vendor vendor/` to download all dependencies into the local tree. [build-deb.sh 60-77](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/build-deb.sh#L60-L77)
2.   **Config Generation**: It writes a `.cargo/config.toml` that redirects Cargo to use the `vendored-sources` instead of `crates.io`. [build-deb.sh 84-93](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/build-deb.sh#L84-L93)
3.   **Build Execution**: It invokes `dpkg-buildpackage` with `-us -uc -b` flags for a binary-only, unsigned build. [build-deb.sh 118](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/build-deb.sh#L118-L118)

### Package Definitions

*   `tt-toplike`: Contains the TUI binary (renamed from `tt-toplike-tui` to `tt-toplike` for the user) and documentation. [debian/rules 64-72](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/debian/rules#L64-L72)
*   `tt-toplike-app`: Contains the native window host. It depends on `tt-toplike` and requires a graphical environment. [debian/control 36-52](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/debian/control#L36-L52)

## Offline Build Strategy

To ensure reproducibility and compliance with network-restricted build environments, `tt-toplike` utilizes a vendoring strategy.

**Build and Vendor Implementation**

### Reproducibility Notes

*   **`.orig` files**: The `debian/rules` file includes a custom `override_dh_clean` to prevent `dh_clean` from deleting `*.orig` files inside the `vendor/` directory, which are required by Cargo for checksum verification during `--frozen` builds. [debian/rules 35-45](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/debian/rules#L35-L45)
*   **`CARGO_HOME` Isolation**: During the Debian build, `CARGO_HOME` is set to `$(CURDIR)/debian/.cargo` to prevent pollution from the developer's local environment. [debian/rules 25](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/debian/rules#L25-L25)
