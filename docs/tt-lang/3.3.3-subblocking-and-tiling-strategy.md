## Purpose and Scope

This page explains how `ttl.compute` operations are partitioned into DST-sized subblocks to fit within the limited DST register file capacity. Subblocking is a critical optimization that enables processing large tile blocks by dividing the iteration space into smaller chunks that can be executed within a single DST synchronization region (acquire to release). [lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp 5-13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L5-L13)[docs/development/DST_Utilization.md 5-11](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/development/DST_Utilization.md?plain=1#L5-L11)

This document covers the `unroll_factor` calculation, the subblock partitioning pass, and the multi-dimensional tiling logic. For DST register assignment and allocation, see **3.3.2 DST Register Assignment**. For loop lowering that generates the final `scf.for` constructs, see **3.3.4 Loop Lowering to SCF**.

## Overview of Subblocking

The DST (destination) register file has limited capacity—typically 4-16 tiles. When a `ttl.compute` operation processes more tiles than fit in DST, it must be partitioned into subblocks that execute sequentially. [docs/development/DST_Utilization.md 23-24](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/development/DST_Utilization.md?plain=1#L23-L24)[lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp 9-13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L9-L13)

The subblocking process has two phases:

1.   **Unroll Factor Calculation** (`ttl-assign-dst`): Determines how many tiles can be processed per DST sync region based on capacity and operation register pressure. [docs/development/DST_Utilization.md 100-109](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/development/DST_Utilization.md?plain=1#L100-L109)
2.   **Iteration Space Partitioning** (`ttl-subblock-compute-for-dst`): Uses `TilingInterface` to split the compute into outer loops and inner subblocked computes. [lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp 9-11](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L9-L11)[docs/development/DST_Utilization.md 68](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/development/DST_Utilization.md?plain=1#L68-L68)

## Multi-Dimensional Subblock Size Calculation

When the required tiles exceed capacity, the `ttl-subblock-compute-for-dst` pass partitions the iteration space. The utility `computeMultiDimSubblockSizes` finds subblock sizes `[t0, t1, ...]` such that:

1.  Each `ti` divides `dimSizes[i]` evenly. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp#L18-L20" min=18 max=20 file-path="lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp">Hii</FileRef>
2.  `product(ti) <= maxTiles`. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp#L55-L57" min=55 max=57 file-path="lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp">Hii</FileRef>
3.  `product(ti)` is maximized, preferring larger inner dimensions to minimize outer loop overhead. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp#L31-L38" min=31 max=38 file-path="lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp">Hii</FileRef>

## TilingInterface and MLIR Integration

The `ComputeOp` implements `TilingInterface`. The `TTLSubblockComputeForDSTPass` invokes this interface to generate the tiled structure. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L30-L31" min=30 max=31 file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>

The transformation steps:
1.  **Identify Ops**: Walk the function and collect `ComputeOp` instances that exceed the `unroll_factor`. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L56-L80" min=56 max=80 file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>
2.  **Calculate Strides**: Compute row-major strides for CB linearization. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L143-L143" min=143  file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>
3.  **Generate Loops**: Create `scf.for` loops for dimensions where `ti < dimSizes[i]`. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/subblock_compute_edge_cases.mlir#L18-L22" min=18 max=22 file-path="test/ttlang/Dialect/TTL/Transforms/subblock_compute_edge_cases.mlir">Hii</FileRef>
4.  **Slice Tensors**: Insert `tensor.extract_slice` for all input/output operands based on the loop induction variables. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/subblock_compute_for_dst.mlir#L122-L123" min=122 max=123 file-path="test/ttlang/Dialect/TTL/Transforms/subblock_compute_for_dst.mlir">Hii</FileRef>
5.  **Annotate**: Attach `ttl.full_linearization_strides` to the inner compute to preserve global indexing context. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L189-L191" min=189 max=191 file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>

## Matmul-Specific Strategy

Matmul operations receive special treatment because the reduction dimension ($K$) accumulates in-place in the DST register file. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L103-L108" min=103 max=108 file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>

*   **Effective Budget**: Only the parallel dimensions ($M \times N$) consume DST slots. $K$ is considered "free". <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L147-L155" min=147 max=155 file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>
*   **Partitioning**: Subblocking partitions the $M \times N$ output space while keeping the $K$ loop whole within each subblock compute. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/subblock_matmul.mlir#L10-L12" min=10 max=12 file-path="test/ttlang/Dialect/TTL/Transforms/subblock_matmul.mlir">Hii</FileRef>
*   **Validation**: If a single output tile ($1 \times 1 \times K$) requires more slots than available (e.g., due to scaled accumulation), the pass signals failure. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp#L131-L136" min=131 max=136 file-path="lib/Dialect/TTL/Transforms/TTLSubblockComputeForDST.cpp">Hii</FileRef>

## Subblocking Transformation Example: 6x6 Tensor

For a 6x6 tensor with DST capacity 4 and unary operations (1 slot/iter), the algorithm finds a 2x2 subblock. <FileRef file-url="https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/ttlang/Dialect/TTL/Transforms/subblock_compute_edge_cases.mlir#L10-L13" min=10 max=13 file-path="test/ttlang/Dialect/TTL/Transforms/subblock_compute_edge_cases.mlir">Hii</FileRef>

```mlir
// Before: ttl.compute on tensor<6x6>

// After:
scf.for %i = 0 to 6 step 2 {
  scf.for %j = 0 to 6 step 2 {
    %in_slice = tensor.extract_slice %arg0[%i, %j] [2, 2] [1, 1]
    %out_slice = tensor.extract_slice %init[%i, %j] [2, 2] [1, 1]
    %res = ttl.compute ins(%in_slice) outs(%out_slice) {
       // Inner compute now fits in 4 DST slots (2x2)
    }
  }
}
```
