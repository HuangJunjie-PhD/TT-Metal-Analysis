The Operator System is a foundational component of the PyBuda framework that defines the arithmetic and computational primitives used to construct neural network models. This system provides a comprehensive set of operators ranging from basic element-wise operations to complex matrix multiplications and convolutions. The Operator System also handles the evaluation, shape calculation, lowering to hardware, and gradient computation for these operators.

For information about how operators are placed on hardware, see [Balancer and Placement System](https://deepwiki.com/tenstorrent/tt-buda/2.2-balancer-and-placement-system).

## Operator System Architecture

The PyBuda Operator System is divided into two main evaluation paths: the PyBuda path for high-level Python evaluation, and the Buda path for hardware-specific evaluation.

Sources: [pybuda/pybuda/op/eval/pybuda/__init__.py 1-271](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/pybuda/op/eval/pybuda/__init__.py#L1-L271)[pybuda/pybuda/op/eval/buda/__init__.py 1-180](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/pybuda/op/eval/buda/__init__.py#L1-L180)

### Core Components

1.   **Operator Definitions**: PyBuda provides implementations for various operator types, including elementwise unary/binary operations, matrix multiplications, convolutions, and tensor manipulations.

2.   **Evaluation Interfaces**: Each operator implements methods for shape calculation, forward evaluation, backward pass, and hardware lowering.

3.   **Operator Registry**: Operators are registered in mappings that link operation names to their implementation modules or classes.

## Operator Types

The PyBuda framework supports a rich variety of operators, categorized into different types based on their functionality and implementation characteristics:

### 1. Elementwise Binary Operators

These operators apply an element-wise binary operation to two input tensors:

*   Add, Subtract, Multiply, Divide
*   Maximum, Minimum
*   Greater, GreaterEqual, Less, LessEqual
*   Equal, NotEqual
*   LogicalAnd, Power
*   Heaviside, BinaryStack

### 2. Elementwise Unary Operators

These operators apply an element-wise unary operation to a single input tensor:

*   Abs, Exp, Log, Sqrt
*   Relu, LeakyRelu, Gelu
*   Sigmoid, Tanh, Sine, Cosine
*   Reciprocal, Identity, Buffer
*   Clip, Dropout, Pow

### 3. Matrix Multiplication Operators

*   Matmul (dense matrix multiplication)
*   SparseMatmul (for sparse matrices)

### 4. Tensor Manipulation Operators

*   Transpose, Reshape, Broadcast
*   Squeeze, Unsqueeze
*   Gather, Select, Index
*   Pad, Narrow

### 5. Reduction Operators

*   ReduceAvg, ReduceSum, ReduceMax
*   GroupedReduceAvg

### 6. N-ary Operators

*   Concatenate (join tensors along a dimension)
*   Stack (stack tensors along a new dimension)
*   Where (conditional selection)
*   IndexCopy (copy values to specific indices)

### 7. Neural Network Operators

*   Convolution (Conv2d, Conv3d)
*   Pooling (MaxPool, AvgPool)
*   Softmax, LogSoftmax
*   LayerNorm, BatchNorm

### 8. Quantization Operators

*   Quantize, Dequantize, Requantize

The following table shows how operators are mapped to their implementation modules:

| Operator Category | Example Operators | Implementation Module |
| --- | --- | --- |
| Elementwise Binary | Add, Multiply, Subtract | `eltwise_binary` |
| Elementwise Unary | Relu, Sigmoid, Tanh | `eltwise_unary` |
| Matrix Multiplication | Matmul, SparseMatmul | `matmul` |
| Tensor Manipulation | Transpose, Reshape, Broadcast | `tm` |
| Reduction | ReduceSum, ReduceAvg | `reduce` |
| N-ary | Concatenate, Stack, Where | `eltwise_nary` |
| Neural Network | Softmax, LayerNorm | `nn` |

| Quantization | Quantize, Dequantize | `quantize` |

## Operator Registration and Dispatch

The PyBuda framework uses a registry-based approach to map operator names to their implementations. This allows for flexible extension of the operator system.

The system supports two styles of operator implementations:

1.   **Module-based implementation**: Where the operator functions are defined in a module
2.   **Class-based implementation**: Where the operator methods are defined in a class

The dispatch system determines which implementation to use at runtime.

## Input Sources and Configurations

The PyBuda framework supports various input sources and configurations for operators, which affects how they are compiled and executed.

### Input Sources

Operators can receive inputs from different sources:

1.   **From another operator**: The output of one operator becomes input to another
2.   **From tensor manipulation**: After tensor transformations
3.   **From DRAM queue**: Data loaded from device DRAM
4.   **From constants**: Predefined constant values
5.   **From host**: Data sent directly from the host

### Input Shapes

The system supports a wide variety of input shapes:

1.   **Full tensors**: Tensors with all dimensions populated
2.   **Reduced tensors**: Tensors with some dimensions of size 1
3.   **Vectors**: Special case of tensors with one dimension
4.   **Various dimension sizes**: From small (3x3) to large (10000x1)

### Data Formats and Math Fidelity

Operations support different data formats and math fidelity settings:

| Data Format | Description |
| --- | --- |
| Float16_b | 16-bit floating point (bfloat16) |
| Float32 | 32-bit floating point |
| Bfp2/4/8 | Block floating point formats |
| Int8 | 8-bit integer |
| UInt16 | 16-bit unsigned integer |

| Math Fidelity | Description |
| --- | --- |
| LoFi | Lower precision, higher performance |
| HiFi2 | Medium precision |
| HiFi3 | Higher precision |
| HiFi4 | Highest precision, lower performance |

## Testing and Validation

The PyBuda framework includes a comprehensive testing system for operators, covering different scenarios and configurations:

### Test Models

The testing framework uses different model types to test various input source scenarios:

1.   **ModelFromAnotherOp**: Tests operators receiving inputs from other operators
2.   **ModelFromHost**: Tests operators receiving inputs directly from the host
3.   **ModelFromDramQueue**: Tests operators receiving inputs from DRAM queues
4.   **ModelFromDramQueuePrologued**: Tests operators with prologued inputs from DRAM
5.   **ModelConstEvalPass**: Tests constant evaluation optimization
6.   **ModelOpSrcFromTmEdge**: Tests operators receiving inputs after tensor manipulations

### Test Coverage

The tests cover:

1.   Various input shapes (2D, 3D, 4D tensors)
2.   Different microbatch sizes
3.   Various data formats (Float16, Float32, BFP, etc.)
4.   Different math fidelity settings (LoFi, HiFi2, HiFi3, HiFi4)
5.   Special operator attributes (when applicable)

### Verification Utilities

The testing framework includes utilities for:

1.   **Verifying operator correctness**: Comparing outputs with golden references
2.   **Netlist validation**: Checking that the compiled model has the expected structure
3.   **Input source configuration**: Setting up different input sources for testing
4.   **Shape utilities**: Helper functions for manipulating tensor shapes

## Operator Implementation Examples

To illustrate how operators are implemented in PyBuda, here are examples of typical operator implementations:

### Elementwise Unary Operator: Clip

The Clip operator constrains tensor values between specified minimum and maximum values:

1.   **Shape Calculation**: Returns the same shape as the input
2.   **Evaluation**: Applies torch.clip to input tensor
3.   **Backward Pass**: Computes gradient mask based on where input values fall within bounds
4.   **Lowering**: Decomposes into a sequence of more primitive operations

### N-ary Operator: Stack

The Stack operator stacks multiple tensors along a new dimension:

1.   **Shape Calculation**: Computes output shape based on input shapes and stack axis
2.   **Evaluation**: Applies torch.stack to input tensors
3.   **Implementation Constraints**: Only works with axis=1 in some cases
4.   **Validation**: Requires special handling for different axis values

### Tensor Manipulation Operator: Transpose

The Transpose operator swaps dimensions of a tensor:

1.   **Shape Calculation**: Swaps the specified dimensions in the shape
2.   **Evaluation**: Applies torch.transpose to the input tensor
3.   **Is TM Operation**: Categorized as a tensor manipulation operation
4.   **Lowering**: Maps directly to hardware transpose operations

## Conclusion

The PyBuda Operator System provides a comprehensive framework for defining, evaluating, and executing neural network operations. The system supports a wide range of operators, from basic elementwise operations to complex neural network primitives, with flexible evaluation paths for both Python-based reference computation and hardware-specific execution.

The architecture allows for easy extension with new operators and provides rich functionality for shape calculation, forward and backward evaluation, and hardware lowering. The extensive testing framework ensures that operators work correctly across different input sources, shapes, data formats, and math fidelity settings.

Understanding the Operator System is essential for effectively using PyBuda to build and run neural network models on Tenstorrent hardware.
