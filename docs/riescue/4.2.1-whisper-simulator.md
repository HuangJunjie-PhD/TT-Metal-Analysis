## Purpose and Scope

This page documents the Whisper Instruction Set Simulator (ISS) integration in RiESCUE, including its configuration files, usage patterns, and integration with the RiescueD framework. Whisper is Tenstorrent's RISC-V ISS used to execute generated test ELF files and validate their behavior.

For information about the alternative Spike simulator, see [Spike Simulator](/tenstorrent/riescue/4.2.2-spike-simulator). For details on the toolchain abstraction layer that wraps Whisper, see [Toolchain Abstraction Layer](/tenstorrent/riescue/7.3-toolchain-abstraction-layer).

---

## Overview

Whisper is an external RISC-V instruction set simulator developed by Tenstorrent. Within RiESCUE, it serves as the primary simulation backend for validating generated test cases. The simulator is configured via JSON configuration files that specify ISA features, CSR behavior, memory maps, and hardware characteristics.

## Configuration File Schema

All three Whisper configuration files follow the same JSON schema with different values. The schema defines:

### Top-Level Fields

| Field | Type | Purpose | Example |
| --- | --- | --- | --- |
| `xlen` | integer | Register width (32 or 64) | `64` |
| `isa` | string | ISA string with extensions | `"rv64imafdchvus_svinval_zba..."` |
| `reset_vec` | string (hex) | Reset vector address | `"0x80000000"` |
| `enable_performance_counters` | boolean | Enable perf counters | `true` |
| `print_load_store_address` | boolean | Print memory addresses in trace | `true` |
| `num_mmode_perf_regs` | integer | Number of M-mode perf registers | `8` |

## Configuration File Differences

### whisper\_config.json (Standard)

* Reset vector: `0x80000000`
* 8 M-mode performance registers
* No PMP registers (all set to `exists: false`)
* 16 PMA configuration registers (`pmacfg0-15`)
* PMA regions preconfigured for IO (0-2GB) and DRAM (2GB-52bit)

### whisper\_basic\_config.json

* Reset vector: `0x0`
* 8 M-mode performance registers
* 16 PMP address registers enabled (`pmpaddr0-15`)
* 2 PMP configuration registers (`pmpcfg0`, `pmpcfg2`)
* Simplified PMA configuration
* Cycle and instret counters disabled (`exists: false`)

### whisper\_secure\_config.json

* Reset vector: `0x80000000`
* 2 M-mode performance registers (reduced for secure mode)
* All 64 PMP address registers enabled
* 8 PMP configuration registers enabled
* Physical memory size: 52-bit (`0x10000000000000`)
* Detailed PMA region definitions for memory-mapped IO
* STEE (Secure Trusted Execution Environment) configuration:
* PMP address 0 pre-initialized: `reset: "0x0001ffffffffffff"`
* PMPcfg0 pre-initialized: `reset: "0x1f"`

Sources: [riescue/dtest\_framework/lib/whisper\_config.json1-350](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L1-L350) [riescue/dtest\_framework/lib/whisper\_basic\_config.json1-362](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json#L1-L362) [riescue/dtest\_framework/lib/whisper\_secure\_config.json1-395](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L1-L395)

## Command-Line Arguments

When `RiescueD.simulate()` executes Whisper, it constructs command-line arguments based on the test configuration:

### Standard Arguments

### Argument Reference Table

| Argument | Condition | Purpose |
| --- | --- | --- |
| `--quitany` | Multi-processor mode | Exit simulation when any hart terminates |
| `--harts` | Multi-processor mode | Number of hardware threads |
| `--deterministic` | Multi-processor mode | Deterministic execution with scheduling quantum |
| `--seed` | Multi-processor mode | Random seed for deterministic multi-hart scheduling |

## Multi-Processor Mode

Whisper supports multi-processor simulation when `featmgr.mp_mode_on()` returns `True`:

The `--deterministic 16` argument ensures reproducible multi-hart execution by specifying a scheduling quantum of 16 instructions. The `--quitany` flag causes the simulation to terminate as soon as any hart exits, which is the expected behavior for most test cases.

## WYSIWYG Mode Support

WYSIWYG (What You See Is What You Get) mode uses a special end-of-test mechanism. In this mode, tests signal completion by writing `0xc001c0de` to register `x31`. Since Whisper cannot directly detect this pattern, RiescueD:

1. Searches the disassembly for the  label
2. Calculates the PC address 4 instructions beyond  (0x10 bytes)
3. Passes this as `--endpc` to Whisper
4. Post-processes the Whisper log to verify `x31` contains `0xc001c0de`

## External Dependencies

Whisper is an external dependency that must be installed before running RiESCUE. The build process is documented in the container build scripts:

1. **Source**: Tenstorrent's Whisper fork at `github.com/tenstorrent/whisper`
2. **Build**: Handled by `build_whisper.sh` in the container infrastructure
3. **Installation**: Typically installed to `/usr/bin/whisper`
4. **Version**: Pinned to commit `b24e30f2` as specified in the build scripts

For information about installing Whisper, see [Building Whisper](/tenstorrent/riescue/3.2.2-building-whisper).
