## Purpose and Scope

This page documents the `ttnn-standalone` example application and related development tools for building standalone C++ and Python applications that execute TTNN operations. The `ttnn-standalone` tool demonstrates how to compile MLIR code to C++ via the EmitC dialect and link against the TTNN runtime libraries. Additionally, this page covers **tt-alchemist**, a code generation utility that automates the creation of standalone projects from MLIR models, and the profiler integration.

For information about the EmitC conversion pipeline itself, see [3.5 Code Generation](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/3.5%20Code%20Generation#LNaN-LNaN) For runtime execution using flatbuffer binaries, see [8.2 ttrt](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/8.2%20ttrt#LNaN-LNaN)

## ttnn-standalone Application

### Overview

The `ttnn-standalone` application is a minimal example that demonstrates:

*   Compilation of TTNN MLIR to C++ code via the EmitC dialect.
*   Direct linking against TTNN runtime libraries including `tt_metal`, `tt_stl`, and `_ttnncpp.so`. [tools/ttnn-standalone/CMakeLists.txt 120-125](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttnn-standalone/CMakeLists.txt#L120-L125)
*   Execution of TTNN operations on Tenstorrent hardware.
*   Use of precompiled headers (`ttnn-precompiled.hpp`) for faster compilation. [tools/ttnn-standalone/ttnn-standalone.cpp 5](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttnn-standalone/ttnn-standalone.cpp#L5-L5)

The application is distributed as part of a standalone install component, which includes all necessary headers, libraries, and build files. [tools/ttnn-standalone/CMakeLists.txt 12-38](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttnn-standalone/CMakeLists.txt#L12-L38)

## EmitC and EmitPy Code Generation

### Conversion Pipelines

The TTNN dialect can be lowered to either C++ (EmitC) or Python (EmitPy). Both paths involve mapping TTNN operations to their respective runtime calls.

| Pipeline | Core Class / Pass | Role |
| --- | --- | --- |
| **EmitC** | `ConvertTTNNToEmitCPass` | Orchestrates conversion of TTNN ops to C++ calls and includes headers. [lib/Conversion/TTNNToEmitC/TTNNToEmitCPass.cpp 64-139](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Conversion/TTNNToEmitC/TTNNToEmitCPass.cpp#L64-L139) |
| **EmitPy** | `TTNNToEmitPyBaseOpConversionPattern` | Base pattern for converting TTNN ops to Python strings. |

### Type Mapping and Emitters

## tt-alchemist Code Generation Tool

`tt-alchemist` is a high-level tool designed to generate complete, buildable projects (C++ or Python) from MLIR models. It automates the process of running `ttmlir-opt`, `ttmlir-translate`, and populating build templates. [tools/tt-alchemist/csrc/CMakeLists.txt 148-160](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/tt-alchemist/csrc/CMakeLists.txt#L148-L160)

### Core Capabilities

`tt-alchemist` supports several modes of operation via its CLI and library:

| Component | Description |
| --- | --- |
| `model_to_cpp.cpp` | Generates a C++ source file from an MLIR model. [tools/tt-alchemist/csrc/CMakeLists.txt 18](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/tt-alchemist/csrc/CMakeLists.txt#L18-L18) |

## Development Utilities

### ttmlir-translate

The primary utility for generating code from MLIR. It can produce C++ code for use with `ttnn-standalone` or flatbuffer binaries for use with `ttrt`.

`# Generate C++ codettmlir-opt --ttir-to-emitc-pipeline model.mlir | ttmlir-translate --mlir-to-cpp > output.cpp # Generate Flatbufferttmlir-opt --ttir-to-ttnn-runtime-pipeline model.mlir | ttmlir-translate --ttnn-to-flatbuffer -o output.ttnn`
Sources: [docs/src/ttmlir-translate.md 1-27](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/docs/src/ttmlir-translate.md?plain=1#L1-L27)

The `ci_compile_dylib.py` script serves as a CLI wrapper for the `EmitCCompiler`. It can compile C++ files generated by the compiler into shared libraries or standalone binaries.
