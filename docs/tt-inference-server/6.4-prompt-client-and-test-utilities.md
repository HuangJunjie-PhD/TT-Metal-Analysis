The **Prompt Client and Test Utilities** provide a robust infrastructure for interacting with the inference server, generating synthetic test data, and executing specialized benchmarks for various modalities (LLM, Image, Audio, Video, etc.). These utilities are used extensively by the `run_benchmarks.py` and `run_evals.py` workflows to measure performance and accuracy across the supported model catalog.

## PromptClient and CLI

The `PromptClient` is the primary interface for sending requests to an OpenAI-compatible API server. It handles authentication, health monitoring, and response parsing.

### Implementation and Data Flow

The client supports two authentication methods: `VLLM_API_KEY` (standard Bearer token) and `JWT_SECRET` (used for internal development/container environments) [utils/prompt_client.py 108-125](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/prompt_client.py#L108-L125)

1.   **Initialization**: The client resolves the base URL and health endpoints using `EnvironmentConfig`[utils/prompt_client.py 127-147](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/prompt_client.py#L127-L147)
2.   **Health Synchronization**: Before starting tests, `wait_for_healthy` polls the `/health` endpoint. It integrates with `CacheMonitor` to dynamically extend timeouts if the server is currently generating hardware-specific tensor caches [utils/prompt_client.py 164-213](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/prompt_client.py#L164-L213)
3.   **Request Execution**: The client sends completion requests and calculates metrics like Time To First Token (TTFT) and Inter-Token Latency (ITL) [utils/README.md 130-144](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/README.md?plain=1#L130-L144)

### Trace Capture and Padding

For Tenstorrent hardware, models often require pre-compiled "traces" for specific sequence lengths. `PromptClient` includes `get_trace_context_lens` which filters a set of power-of-2 padded sequence lengths (`PADDED_SEQ_LENS`) against the model's `max_context`[utils/prompt_client.py 35-67](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/prompt_client.py#L35-L67)

## Media Client Strategies

For non-LLM tasks (Image, Audio, CNN, TTS, Video, Embedding), the system uses a **Strategy Pattern** implemented via `BaseMediaStrategy`. These clients bypass standard text completion logic to handle binary payloads and modality-specific metrics.

### System Architecture: Media Client Space

This diagram shows how high-level benchmark tasks are dispatched to modality-specific code entities.

**Sources:**[utils/media_clients/media_client_factory.py](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/media_client_factory.py)[utils/media_clients/base_strategy_interface.py 20-40](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/base_strategy_interface.py#L20-L40)[utils/media_clients/image_client.py 66-82](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/image_client.py#L66-L82)[utils/media_clients/video_client.py 28-140](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/video_client.py#L28-L140)[utils/media_clients/embedding_client.py 35-91](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/embedding_client.py#L35-L91)

### Concrete Strategy Implementations

#### 1. Image and Video Clients

*   **ImageClientStrategy**: Supports SDXL, SD3.5, FLUX, and Motif. It manages multi-step parameters like `GUIDANCE_SCALE` and calculates FID and CLIP scores for accuracy [utils/media_clients/image_client.py 37-170](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/image_client.py#L37-L170)
*   **VideoClientStrategy**: Handles Mochi and Wan2.2 models. It manages asynchronous job polling (e.g., `VIDEO_JOB_STATUS_COMPLETED`) and calculates FVD (Fréchet Video Distance) [utils/media_clients/video_client.py 20-95](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/video_client.py#L20-L95)

#### 2. Audio and TTS Clients

*   **AudioClientStrategy**: Specialized for Whisper. It calculates **RTR (Real-Time Ratio)** and **TSU (Tokens/Second/User)**[utils/media_clients/audio_client.py 38-93](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/audio_client.py#L38-L93)
*   **TTSClientStrategy**: Measures `ttft_ms` (latency to first audio chunk) and RTR for `SpeechT5`[utils/media_clients/tts_client.py 80-111](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/tts_client.py#L80-L111) It defaults to 10 calls for benchmarks and 5 for evals [utils/media_clients/tts_client.py 180](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/tts_client.py#L180-L180)

#### 3. Embedding and CNN Clients

*   **EmbeddingClientStrategy**: Uses the `vllm bench serve` tool from the `BENCHMARKS_VLLM` virtual environment to measure request throughput and Mean E2EL (End-to-End Latency) [utils/media_clients/embedding_client.py 102-178](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/embedding_client.py#L102-L178)
*   **CNNClientStrategy**: Validates classification accuracy for `tt-xla-mobilenetv2` using an ImageNet subset [utils/media_clients/cnn_client.py 21-81](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/cnn_client.py#L21-L81)

## Utilities and Infrastructure

### CacheMonitor

The `CacheMonitor` tracks the state of the hardware tensor cache directory. It detects if the server is stalled during startup by checking if the cache size has stopped growing for a configurable duration [utils/cache_monitor.py](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/cache_monitor.py) This is critical for preventing CI timeouts during the first run of a large model [utils/prompt_client.py 197-213](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/prompt_client.py#L197-L213)

### Logging

The system uses specialized logging configurations to display formatted output to the console and capture detailed traces for evaluation tests [server_tests/run_spec_tests.py 54-64](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/server_tests/run_spec_tests.py#L54-L64)

### Batch Processor

The `batch_processor.py` utility allows for the execution of multiple inference requests in parallel or sequential batches, often used by benchmarking tools to simulate concurrent user load [utils/batch_processor.py](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/batch_processor.py)
