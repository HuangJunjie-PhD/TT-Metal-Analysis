The Performance Charts module provides visualization tools for analyzing and comparing the runtime performance of Tenstorrent Neural Network (TT-NN) operations. These charts display metrics related to execution time, device kernel behavior, and operation distribution, helping users identify performance bottlenecks and optimization opportunities in their TT-NN models.

For tabular performance data analysis, see [Performance Reports](https://deepwiki.com/tenstorrent/ttnn-visualizer/5.1-performance-reports).

## Overview

Performance Charts allow users to visualize performance metrics in multiple ways, filtering by operation types and comparing between different reports. The charts provide insights into:

*   Operation count versus runtime distribution
*   Device kernel runtime across cores
*   Kernel duration statistics
*   Comparative performance between different model runs

## Data Sources and Processing

### Performance Data Retrieval

Performance chart data is retrieved using the `usePerformanceReport` and `useDeviceLog` hooks from the API layer. The performance data contains metrics about different operations including:

*   Operation codes and names
*   Runtime information
*   Core utilization
*   Device architecture information

## Chart Types

The Performance Charts module includes several visualization types, each focusing on different aspects of performance data.

### Operation Count vs. Runtime Chart

This chart plots the number of operations against their runtime, providing insights into how different operation types contribute to the overall execution time.

### Device Kernel Runtime Chart

This visualization shows runtime metrics distributed across device cores, helping identify potential load balancing issues and core utilization patterns.

### Device Kernel Duration Chart

This chart focuses on the duration of kernel executions, helping identify operations that take longer to complete and might be candidates for optimization.

## Comparison Feature

A key capability of the Performance Charts module is the ability to compare performance data between different runs or models. This feature helps identify performance improvements or regressions.

### Comparing Performance Reports

1.   Users select a comparison report using the folder picker component
2.   Both reports' data is fetched and processed
3.   Charts display data from both reports side by side

## Implementation Details

### Component Structure

The Performance Charts functionality is implemented through a hierarchy of React components:

| Component | Purpose |
| --- | --- |
| `Performance` | Main route component that handles data fetching and state |
| `PerfCharts` | Container for the three main chart types |
| `PerfChartFilter` | UI for selecting which operation codes to display |
| `PerfDeviceKernelDurationChart` | Visualizes kernel duration metrics |
| `PerfDeviceKernelRuntimeChart` | Visualizes runtime across device cores |
| `PerfOpCountVsRuntimeChart` | Visualizes operation count vs runtime |
| `NonFilterablePerfCharts` | Container for aggregate statistics charts |

Sources: [src/routes/Performance.tsx 23-212](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/routes/Performance.tsx#L23-L212)[src/components/performance/PerfCharts.tsx 19-37](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/components/performance/PerfCharts.tsx#L19-L37)

### Data Flow

The data flow in the Performance Charts module follows a unidirectional pattern:

1.   The `Performance` component fetches data using hooks from `useAPI.tsx`
2.   Raw data is filtered based on user selection of operation codes
3.   Filtered data is passed to chart components through props
4.   Chart components render visualizations based on this data

### Key Hooks and Functions

| Hook/Function | Purpose |
| --- | --- |
| `usePerformanceReport` | Fetches performance report data for the active report |
| `usePerformanceComparisonReport` | Fetches comparison report data |
| `useDeviceLog` | Retrieves device log information including architecture |
| `setFilteredPerfData` | Updates filtered performance data based on selected operation codes |
| `setSelectedOpCodes` | Updates the list of selected operation codes for filtering |

## Usage Example

1.   Navigate to the Performance analysis page
2.   Select the "Charts" tab to view performance visualizations
3.   Use the filter panel on the left to select specific operation codes to display
4.   To compare with another performance report, use the comparison selector at the top
5.   Charts will update to display filtered data and comparison data (if selected)

The Performance Charts provide a powerful visual tool for analyzing and optimizing TT-NN model performance, enabling users to identify bottlenecks and compare different implementations or configurations.
