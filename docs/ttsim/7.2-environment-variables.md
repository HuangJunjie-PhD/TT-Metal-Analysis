# Environment Variables

Relevant source files
*   [README.md](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1)

## Purpose and Scope

This document provides a complete reference for all environment variables used by ttsim. These variables control how the TT-Metalium software stack discovers and communicates with the simulator, enabling it to run workloads on simulated hardware instead of physical silicon.

For information about configuring ttsim for daily use, see [Configuration](https://deepwiki.com/tenstorrent/ttsim/4.1-configuration). For details about the integration between TT-Metalium and ttsim, see [TT-Metalium Integration](https://deepwiki.com/tenstorrent/ttsim/3.3-tt-metalium-integration).

**Sources:**[README.md 1-121](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L1-L121)

## Overview

ttsim integrates with TT-Metalium through a set of environment variables that must be configured before running workloads. These variables replace the hardware discovery and communication mechanisms that would normally detect and interact with physical Tenstorrent devices. The simulator requires proper configuration of three mandatory variables and expects specific file system arrangements to function correctly.

## Required Environment Variables

All three environment variables documented in this section must be set for ttsim to function. Missing or incorrectly configured variables will result in `ConfigurationError` messages or runtime failures.

### Variable Reference Table

| Variable Name | Required | Valid Values | Purpose |
| --- | --- | --- | --- |
| `TT_METAL_HOME` | Yes | Absolute path to directory | Locates TT-Metalium installation |
| `TT_METAL_SIMULATOR` | Yes | Absolute path to `.so` file | Specifies simulator library to load |
| `TT_METAL_SLOW_DISPATCH_MODE` | Yes | `1` | Enables slow dispatch mode (fast dispatch not supported) |

**Sources:**[README.md 23-64](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L23-L64)

### TT_METAL_HOME

**Purpose:** Specifies the root directory of the TT-Metalium installation. TT-Metalium uses this variable to locate SOC descriptor files, build artifacts, and other resources required during initialization.

**Type:** Required

**Valid Values:** Absolute path to a directory containing a valid TT-Metalium installation

**Example:**

`export TT_METAL_HOME=~/tt-metal`
**Common Errors:**

*   Variable not set: TT-Metalium cannot locate required resources
*   Path does not exist: Directory structure validation fails
*   Path points to incomplete installation: Missing SOC descriptors or build artifacts

**Sources:**[README.md 25](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L25-L25)

### TT_METAL_SIMULATOR

**Purpose:** Instructs TT-Metalium to use ttsim instead of physical hardware by specifying the path to the simulator shared library. When this variable is set, TT-Metalium's device detection logic loads the specified `.so` file and communicates with it through the ttsim API.

**Type:** Required (for simulator operation)

**Valid Values:** Absolute path to `libttsim_wh.so` (Wormhole) or `libttsim_bh.so` (Blackhole)

**Architecture-Specific Values:**

*   Wormhole: `~/sim/libttsim_wh.so`
*   Blackhole: `~/sim/libttsim_bh.so`

**Example:**

`# For Wormholeexport TT_METAL_SIMULATOR=~/sim/libttsim_wh.so # For Blackholeexport TT_METAL_SIMULATOR=~/sim/libttsim_bh.so`
**File System Requirements:** The directory containing the `.so` file must also contain a file named `soc_descriptor.yaml`. This file must be the appropriate SOC descriptor for the simulator architecture:

*   Wormhole: `wormhole_b0_80_arch.yaml`
*   Blackhole: `blackhole_140_arch.yaml`

**Common Errors:**

*   Variable not set: TT-Metalium attempts to use physical hardware
*   File does not exist: Simulator library cannot be loaded
*   Wrong architecture: Mismatched simulator and SOC descriptor
*   Missing `soc_descriptor.yaml`: Simulator cannot initialize device model

**Sources:**[README.md 41-61](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L41-L61)

### TT_METAL_SLOW_DISPATCH_MODE

**Purpose:** Controls the dispatch mode used by TT-Metalium. ttsim currently only supports slow dispatch mode; fast dispatch is not implemented. This variable must be set to `1` for every ttsim invocation.

**Type:** Required

**Valid Values:**`1` (enabled)

**Example:**

`export TT_METAL_SLOW_DISPATCH_MODE=1 # Or set per-command:TT_METAL_SLOW_DISPATCH_MODE=1 ./build/programming_examples/metal_example_add_2_integers_in_riscv`
**Implementation Status:** Fast dispatch mode is currently not functional in ttsim. Attempting to run without `TT_METAL_SLOW_DISPATCH_MODE=1` will result in errors or undefined behavior.

**Common Errors:**

*   Variable not set: Fast dispatch attempted, causing runtime errors
*   Value set to `0`: Explicitly disables slow dispatch, causing failure

**Sources:**[README.md 51-64](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L51-L64)

## Configuration Patterns

### Basic Setup Script

A typical setup script for Wormhole development:

`#!/bin/bash # Set TT-Metalium locationexport TT_METAL_HOME=~/tt-metal # Configure Wormhole simulatorexport TT_METAL_SIMULATOR=~/sim/libttsim_wh.so # Enable slow dispatch (required)export TT_METAL_SLOW_DISPATCH_MODE=1 # Verify configurationecho "TT_METAL_HOME: $TT_METAL_HOME"echo "TT_METAL_SIMULATOR: $TT_METAL_SIMULATOR"echo "TT_METAL_SLOW_DISPATCH_MODE: $TT_METAL_SLOW_DISPATCH_MODE"`
**Sources:**[README.md 46-52](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L46-L52)

### Architecture Switching

Switching between Wormhole and Blackhole simulators:

`# Function to configure Wormholesetup_wormhole() {    export TT_METAL_SIMULATOR=~/sim/libttsim_wh.so    cp $TT_METAL_HOME/tt_metal/soc_descriptors/wormhole_b0_80_arch.yaml ~/sim/soc_descriptor.yaml    echo "Configured for Wormhole"} # Function to configure Blackholesetup_blackhole() {    export TT_METAL_SIMULATOR=~/sim/libttsim_bh.so    cp $TT_METAL_HOME/tt_metal/soc_descriptors/blackhole_140_arch.yaml ~/sim/soc_descriptor.yaml    echo "Configured for Blackhole"} # Enable slow dispatch onceexport TT_METAL_SLOW_DISPATCH_MODE=1 # Switch to desired architecturesetup_wormhole# orsetup_blackhole`
**Sources:**[README.md 45-61](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L45-L61)

### Common Configuration Errors

| Error Type | Symptom | Cause | Solution |
| --- | --- | --- | --- |
| `ConfigurationError` | Cannot locate TT-Metalium | `TT_METAL_HOME` not set or invalid | Set to valid installation path |
| Library load failure | Cannot load simulator | `TT_METAL_SIMULATOR` path incorrect | Verify file exists and path is absolute |
| Missing SOC descriptor | Simulator initialization fails | No `soc_descriptor.yaml` in simulator directory | Copy appropriate YAML from `$TT_METAL_HOME/tt_metal/soc_descriptors/` |
| Fast dispatch error | Runtime failures or crashes | `TT_METAL_SLOW_DISPATCH_MODE` not set | Set to `1` before execution |
| Architecture mismatch | Unexpected behavior | Wrong simulator for SOC descriptor | Ensure Wormhole simulator uses Wormhole descriptor, etc. |

**Sources:**[README.md 63-77](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L63-L77)

## Advanced Topics

### Per-Command Configuration

Environment variables can be set for individual command invocations without affecting the shell session:

`# Run single command with specific configurationTT_METAL_SLOW_DISPATCH_MODE=1 \TT_METAL_SIMULATOR=~/sim/libttsim_wh.so \./build/test/tt_metal/test_kernels # TT_METAL_HOME typically remains as shell exportexport TT_METAL_HOME=~/tt-metal`
**Sources:**[README.md 51](https://github.com/tenstorrent/ttsim/blob/67b1733e/README.md?plain=1#L51-L51)

### Multiple Architecture Support

Systems with both simulators can use wrapper scripts to manage configuration:

`# wh-sim wrapper script#!/bin/bashexport TT_METAL_SIMULATOR=~/sim/libttsim_wh.soexport TT_METAL_SLOW_DISPATCH_MODE=1exec "$@" # bh-sim wrapper script#!/bin/bashexport TT_METAL_SIMULATOR=~/sim/libttsim_bh.soexport TT_METAL_SLOW_DISPATCH_MODE=1exec "$@" # Usage:# wh-sim ./build/test/tt_metal/test_kernels# bh-sim ./build/test/tt_metal/test_kernels`
