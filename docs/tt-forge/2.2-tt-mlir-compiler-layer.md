## Purpose and Scope

This document describes the TT-MLIR compiler layer, which serves as the central compilation infrastructure for transforming machine learning models into optimized code for Tenstorrent hardware. The compiler ingests high-level intermediate representations from various frontends and applies optimization passes before lowering to hardware-specific dialects.

TT-MLIR defines multiple dialects including **TTIR**, **TTNN**, and **TTKernel**, and applies optimization passes such as fusion, sharding, and layout transformation to lower workloads to **TT-Metalium**[README.md 29](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L29-L29)

## Compiler Architecture Overview

TT-MLIR is an MLIR-based compiler framework that bridges the gap between ML frameworks and Tenstorrent hardware. The compiler is organized into multiple MLIR dialects, each representing a different level of abstraction, with optimization passes that transform code between these representations.

**Diagram: TT-MLIR Compiler Architecture and Tooling**

The compiler follows a multi-stage lowering strategy where high-level operations are progressively transformed into hardware-specific implementations. The central TTIR dialect acts as a common representation that can be lowered to multiple backend targets depending on the desired abstraction level [CLAUDE.md 74-80](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L74-L80)

## Intermediate Representations

### StableHLO-IR

StableHLO serves as the input IR for models coming from the **TT-XLA** frontend. TT-XLA leverages a PJRT interface to compile models into StableHLO graphs for ingestion by the TT-MLIR compiler [README.md 27](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L27)

**Diagram: StableHLO-IR Input Path**

StableHLO provides a well-defined operation set that captures the semantics of ML computations without framework-specific quirks. It is the standard MLIR representation used by XLA-based frontends [CLAUDE.md 76](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L76-L76)

**Sources:**[README.md 27](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L27-L27)[CLAUDE.md 71-76](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L71-L76)

### TTIR (Tenstorrent Intermediate Representation)

TTIR is the common intermediate representation within TT-MLIR. It serves as the central hub where all frontend representations converge. TT-Forge-ONNX (via TVM) lowers models to TTIR [CLAUDE.md 72](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L72-L72) and StableHLO is also converted to TTIR [CLAUDE.md 75](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L75-L75)

**Characteristics:**

*   **Hardware-Agnostic**: Operations represent high-level intent before device-specific lowering.
*   **Graph-Level Abstractions**: Captures the full computational graph.
*   **Optimization Target**: Primary dialect for layout transformations, op fusing, and sharding [CLAUDE.md 77](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L77-L77)

**Sources:**[CLAUDE.md 72-77](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L72-L77)

### TTNN Dialect

The TTNN dialect provides the entry point into the TTNN library operations. It represents neural network ops that map to the high-level C++ TTNN library [CLAUDE.md 78](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L78-L78)

**Diagram: TTNN Dialect to Library Mapping**

TTNN operations are designed to have a straightforward mapping to TTNN library calls, which provide optimized implementations for Tenstorrent hardware [CLAUDE.md 81](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L81-L81)

**Sources:**[CLAUDE.md 78-81](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L78-L81)

### TTMetal Dialect

The TTMetal dialect provides direct access to **TT-Metalium** kernels. This offers lower-level control for performance-critical operations or specialized custom logic [CLAUDE.md 79](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L79-L79)

**Capabilities:**

*   Direct kernel API access.
*   Fine-grained control over core placement and memory layout.
*   Access to the low-level programming model of TT-Metalium [CLAUDE.md 81](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L81-L81)

**Sources:**[CLAUDE.md 79-81](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L79-L81)

### TTKernel Dialect

The TTKernel dialect enables the integration of custom kernels. It is closely related to **TT-Lang**, a Python DSL for authoring high-performance kernels [README.md 30](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L30-L30)

**Sources:**[README.md 29-30](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L29-L30)

## Optimization Passes

The TT-MLIR compiler employs a comprehensive suite of optimization passes that transform the IR at various levels. These passes operate primarily on TTIR before lowering to backend-specific dialects [CLAUDE.md 77](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L77-L77)

**Diagram: Optimization Pass Pipeline**

### Key Optimization Categories:

1.   **Layout Transformation**: Converts tensors between row-major and hardware-optimized tile layouts [README.md 29](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L29-L29)
2.   **Operation Fusion**: Combines multiple operations into a single kernel to reduce memory traffic and launch overhead [CLAUDE.md 77](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L77-L77)
3.   **Decomposition**: Breaks down complex operations into hardware-native primitives.
4.   **Sharding**: Determines how tensors are distributed across multiple cores or multiple chips (N150, N300, P150B) [README.md 29](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/README.md?plain=1#L29-L29)[CLAUDE.md 83](https://github.com/tenstorrent/tt-forge/blob/6f2d9645/CLAUDE.md?plain=1#L83-L83)

## Compiler Tools

TT-MLIR provides a suite of tools for developing, testing, and debugging compiled models.

| Tool | Purpose |
| --- | --- |
| `ttmlir-opt` | The primary tool for executing and testing individual compiler passes on MLIR files. |
| `ttmlir-translate` | Handles IR ingestion from external formats and code generation for backends. |
| `TTRT` | A standalone runtime for executing and profiling compiled model binaries. |
