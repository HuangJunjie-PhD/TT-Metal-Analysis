# Polaris Main Orchestrator

Relevant source files
*   [polaris.py](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/polaris.py)
*   [ttsim/config/validators.py](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/config/validators.py)

The Polaris Main Orchestrator is the central executable that coordinates AI workload performance simulation across different hardware architectures. It serves as the primary entry point for users to run simulations, handling command-line arguments, configuration processing, workload graph construction, simulation execution, and results generation.

This document covers the main orchestration logic implemented in `polaris.py`. For information about the underlying simulation framework, see [TTSIM Framework](https://deepwiki.com/tenstorrent/polaris/2.2-ttsim-framework). For configuration management details, see [Configuration Management](https://deepwiki.com/tenstorrent/polaris/2.3-configuration-management). For project-level orchestration using `polproj.py`, see [Project Orchestration](https://deepwiki.com/tenstorrent/polaris/2.4-project-orchestration).

## Integration Points

The Polaris Main Orchestrator integrates with several key components:

*   **TTSIM Framework**: Instantiates and executes TTSIM workloads via `get_ttsim_functional_instance()`
*   **ONNX Frontend**: Converts ONNX models to internal graph representation via `onnx2graph()`
*   **Configuration System**: Loads specifications via `get_arspec_from_yaml()`, `get_wlspec_from_yaml()`, and `get_wlmapspec_from_yaml()`
*   **Performance Models**: Utilizes device models and performance statistics from the TTSIM framework
