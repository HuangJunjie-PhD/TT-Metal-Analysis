## Purpose and Scope

This page documents the simulator test suite that validates the functional simulator's correctness. These tests ensure the simulator accurately reproduces hardware behavior including data movement, circular buffer operations, multi-core execution, deadlock detection, and error handling. For information about the simulator's architecture and execution model, see [Simulation Framework](https://deepwiki.com/tenstorrent/tt-lang/6-simulation-framework). For general testing infrastructure including pytest and lit tests, see [Testing Infrastructure](https://deepwiki.com/tenstorrent/tt-lang/8-testing-infrastructure).

The simulator tests cover four primary areas:

1.   **CLI example testing** - Validating example programs execute correctly via `ttlang-sim` using `test_examples.py`[[test/sim/test_examples.py:5-9]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
2.   **Program execution testing** - Verifying cooperative scheduling and multi-core behavior in `test_program.py`[[test/sim/test_program.py:7-12]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
3.   **Dataflow Buffer (DFB) & Math Validation** - Testing circular buffer state machines and tile math operations in `test_dfb.py` and `test_math.py`.
4.   **Copy and Pipe Handlers** - Verifying the registry-based copy system and inter-core communication in `test_copyhandlers.py`[[test/sim/test_copyhandlers.py:5-9]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

## Program Execution Tests (test_program.py)

These tests validate the `Program` class, which manages multi-node execution and thread coordination [[python/sim/program.py:97-106]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

### Node Context Building and Isolation

The simulator builds a unique context for every node in the grid using `_build_node_context`[[python/sim/program.py:153-197]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

*   **Fresh DFBs**: Each node gets a new `DataflowBuffer` instance with the same properties as the template [[python/sim/program.py:180-187]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Isolation**: Local variables are deep-copied using `copy.deepcopy` to prevent cross-node interference [[python/sim/program.py:189]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Module Shadowing**: Modules are skipped during deep-copying to maintain standard library and simulator library access [[python/sim/program.py:167-171]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Zero-Globals Architecture**: `test_no_mutable_globals.py` ensures that the simulator does not rely on mutable global state that could leak between nodes [[test/sim/test_no_mutable_globals.py:5-10]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

### Cooperative Scheduling and Deadlock Detection

The `GreenletScheduler` enables cooperative multitasking using greenlets [[python/sim/greenlet_scheduler.py:74-81]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

*   **Uniqueness Invariant**: Identity is `(linear_node, kind, func_name)`. The scheduler rejects duplicate registrations on the same node [[python/sim/greenlet_scheduler.py:122-130]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Deadlock Detection**: If all active greenlets are blocked on `wait()` or `reserve()` operations and no progress can be made, a `RuntimeError: Deadlock detected` is raised [[python/sim/greenlet_scheduler.py:183-193]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Closure Variable Extraction**: The `Program` class extracts closure variables from kernel functions to ensure variables defined outside the kernel (like DFBs) are available in the per-node context [[python/sim/program.py:120-140]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

## Dataflow Buffer and TTNN Simulation Tests

### DFB State Machine (test_dfb.py)

Tests in `test_dfb.py` ensure the circular buffer state machine is strictly enforced via the `BlockStateMachine`[[python/sim/dfb.py:124-129]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

*   **Context Manager Support**: `Block` implements `__enter__` and `__exit__`, allowing automatic `push()` or `pop()` when used in a `with` statement [[python/sim/dfb.py:151-175]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **BFP8B Capacity**: `test_ttnnsim.py` verifies that `DataflowBuffer` correctly calculates `capacity_bytes` for `bfloat8_b` including shared-exponent overhead (1 byte per 16 elements) [[test/sim/test_ttnnsim.py:55-101]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

### TTNN Compatibility (test_ttnnsim.py)

This test suite validates the thin `ttnnsim` layer that mirrors the `ttnn` API [[python/sim/ttnnsim.py:7-18]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

*   **Sharding Strategies**: Verifies `ShardSpec` and `MemoryConfig` resolution for `HEIGHT_SHARDED`, `WIDTH_SHARDED`, and `BLOCK_SHARDED` layouts [[python/sim/ttnnsim.py:189-212]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Device Simulation**: Tests `open_device()` and `compute_with_storage_grid_size()` (returning default 8x8) [[test/sim/test_ttnnsim.py:142-160]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Float32 Promotion**: Validates that `bfloat8_b` and `bfloat16` tensors are backed by `torch.float32` by default to ensure host-architecture compatibility [[test/sim/test_ttnnsim.py:103-140]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

## Copy and Pipe Handler Tests

### Copy Registry and Validation (test_copyhandlers.py)

The simulator uses a registry-based strategy pattern for data transfers [[test/sim/test_copyhandlers.py:29-35]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

*   **Handler Lookup**: `HANDLER_REGISTRY` maps `(src_type, dst_type)` to specific handlers like `TensorToBlockHandler` or `BlockToPipeHandler`[[test/sim/test_copyhandlers.py:56-69]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Shape Validation**: Handlers validate that source and destination shapes match. Mismatched tile counts raise `ValueError`[[test/sim/test_copyhandlers.py:74-109]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

### Inter-core Pipes and Deadlocks

Pipe operations simulate inter-core data movement [[test/sim/test_copyhandlers.py:112-113]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).

*   **Deadlock Simulation**: Tests verify that receiving from a pipe with no sender results in a detected deadlock within the `GreenletScheduler`[[test/sim/test_copyhandlers.py:114-147]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Multicast**: `Pipe` supports multicast to multiple receivers using `NodeRange` or `slice` notation [[test/sim/test_copyhandlers.py:188-216]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
*   **Length Mismatch**: Pipe transfers validate that the amount of data sent matches the destination block's capacity [[test/sim/test_copyhandlers.py:149-183]](https://deepwiki.com/tenstorrent/tt-lang/8.3-simulator-tests).
