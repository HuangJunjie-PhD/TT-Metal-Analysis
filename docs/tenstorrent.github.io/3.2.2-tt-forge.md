## Introduction

TT-Forge™ is Tenstorrent's end-to-end, MLIR-based compiler stack designed to bridge high-level machine learning frameworks with Tenstorrent hardware [core/forge/index.rst 1-5](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/index.rst#L1-L5) It provides an automated path to run diverse model architectures without requiring manual custom kernel development by lowering high-level graphs into TT-Metalium for hardware execution [core/getting-started/tt-software-stack.md 72-78](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/tt-software-stack.md?plain=1#L72-L78)

The stack consists of three primary components:

1.   **Frontends**: Framework-specific compilers (TT-Torch, TT-XLA, TT-Forge-ONNX) [core/forge/index.rst 5-6](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/index.rst#L5-L6)
2.   **Backend**: The TT-MLIR compiler [core/forge/index.rst 5-6](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/index.rst#L5-L6)
3.   **Cookbook**: TT-Blacksmith for fine-tuning and training experiments [core/forge/index.rst 6](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/index.rst#L6-L6)

## System Architecture

TT-Forge occupies the highest layer of the Tenstorrent software stack, providing the most automated path from model research to silicon execution.

## Backend Compiler: TT-MLIR

TT-MLIR is the core backend for the TT-Forge stack. It consumes the Tenstorrent-specific graph IR generated by the frontends and lowers it into hardware-specific code using the MLIR framework [core/forge/forge-backend-compiler/index.rst 3](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/forge-backend-compiler/index.rst#L3-L3)

## Tools and Validation

TT-Forge includes specialized tools for debugging and performance analysis:

*   **TT-NPE**: A tool used to validate compiled operations and simulate execution at the tile or kernel level. It is primarily used to debug mismatches between the compiler and the hardware [core/forge/tools/index.rst 3](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/tools/index.rst#L3-L3)
*   **TT-Forge-Models**: The authoritative repository for tracking model validation status across different hardware generations. It serves as the starting point for users looking to compile PyTorch, ONNX, or JAX models [core/forge/index.rst 8-9](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/index.rst#L8-L9)[core/getting-started/tt-software-stack.md 115](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/getting-started/tt-software-stack.md?plain=1#L115-L115)

## Model Bring-up Workflow

Bringing a model to Tenstorrent hardware via TT-Forge typically involves the following flow:

1.   **Selection**: Identify a model from the [TT-Forge-Models](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/TT-Forge-Models) catalog or provide a custom model in a supported framework [core/forge/index.rst 8-9](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/index.rst#L8-L9)
2.   **Frontend Compilation**: Use `TT-Torch`, `TT-XLA`, or `TT-Forge-ONNX` to generate the initial graph IR [core/forge/forge-frontend-compiler/index.rst 3](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/forge-frontend-compiler/index.rst#L3-L3)
3.   **Backend Lowering**: `TT-MLIR` processes the IR, performing optimizations and lowering to `TT-NN` or `TT-Metalium`[core/forge/forge-backend-compiler/index.rst 3](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/forge-backend-compiler/index.rst#L3-L3)
4.   **Execution & Monitoring**: Run the model on hardware and monitor telemetry using tools like `TT-SMI` or `TT-Toplike`[core/tools/index.rst 7-13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/tools/index.rst#L7-L13)
5.   **Validation**: Use `TT-NPE` to ensure the compiled ops match expected simulation results [core/forge/tools/index.rst 3](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/forge/tools/index.rst#L3-L3)
