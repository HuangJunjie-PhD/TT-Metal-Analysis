The **Insights** screen provides a dense, real-time visualization of Tenstorrent hardware state. It combines an auto-adapting column layout for multi-device monitoring with a high-fidelity "Chip Portrait" that renders the physical core topology (Tensix, ETH, DRAM, and PCIe) using character art and heatmaps.

## Architecture and Data Flow

The Insights screen is the primary monitoring interface in the TUI. It consumes data from the `TelemetryBackend` and transforms it into a grid of device panels. Each panel consists of a **Chip Portrait** (left) and a **Stats Sidebar** (right).

## Auto-Column Layout

To maximize screen real estate, the Insights screen uses an automatic column calculation. Unlike the default vertical stacking, it computes how many device panels can fit horizontally based on the current terminal width.

| Variable | Blackhole (BH) | Wormhole (WH) |
| --- | --- | --- |
| Portrait Width | 18 chars | 11 chars |
| Stats Sidebar | 31 chars | 31 chars |
| **Total Panel Width** | **50 chars** | **43 chars** |

The layout engine calculates `cols_per_row = max(1, terminal_width / panel_w)`. For example, a 200-column terminal can display 4 Blackhole devices in a single row, preventing unnecessary vertical scrolling.

## Chip Portrait Visualization

The Chip Portrait is a character-art grid representing the physical layout of the ASIC. It is implemented in `src/ui/tui/chip_portrait.rs`.

### Core Topology Mapping

The grid dimensions and cell types are architecture-specific, following authoritative definitions from the hardware specifications:

*   **Blackhole (17x12):**
    *   **ETH:** Columns 0 and 16.
    *   **PCIe:** Column 8 (rendered as `╋`).
    *   **DRAM:** Rows 0 and 11 (rendered as `▪`).
    *   **Tensix:** All other cells.

*   **Wormhole (10x12):**
    *   **ETH:** Rows 0 and 6.
    *   **DRAM:** Columns 0 and 5.
    *   **Tensix:** All other cells.

Sources: `src/ui/tui/chip_portrait.rs:16-23`(), `docs/superpowers/specs/2026-05-20-tt-smi-accuracy-chip-portrait-design.md:120-138`()

### Temperature Heatmap and Harvesting

The portrait uses a color interpolation system to represent temperatures. Tensix cores use a Teal-to-Red palette based on ASIC temperature, while DRAM cells use a Blue-to-Red palette based on per-channel GDDR temperatures.

| Core Type | Character | Logic |
| --- | --- | --- |
| **Tensix (Active)** | `░`, `▒`, `▓` | Density based on `power_change` relative to baseline. |
| **Tensix (Idle)** | `░` | Minimum floor ensures heatmap visibility. |
| **Tensix (Harvested)** | `·` | Determined by `ENABLED_TENSIX_COL` bitmask; rendered Dim Gray. |
| **ETH (Live)** | `●` | Determined by `ETH_LIVE_STATUS` bitmask; rendered Green. |
| **DRAM** | `▪` | Colored by `GddrTempPair` values. |

Sources: `docs/superpowers/specs/2026-05-21-insights-redesign.md:50-94`(), `src/models/telemetry.rs:107-112`()

## Particle Animation System

To visualize data movement, the portrait renders `Particle` structs that travel between DRAM and Tensix cores.

1.   **Read Particles:** Teal particles (`·`, `∘`, `○`) spawn at DRAM boundaries and travel toward the center.
2.   **Write Particles:** Pink particles retreat from the center toward the DRAM rows.
3.   **Dynamics:** The speed of particles is normalized by `aiclk` (AICLK MHz). Higher clock speeds result in faster particle movement (5 ticks to cross at max clock vs 12 at idle).
4.   **Heartbeat:** A minimum animation floor ensures the chip "feels alive" even at idle by spawning a read particle every ~20 ticks.

Sources: `src/ui/tui/chip_portrait.rs:25-52`(), `src/ui/tui/chip_portrait.rs:147-169`()

## Interaction: Process Management

The Insights screen includes a process monitor that identifies Linux PIDs using Tenstorrent hardware by scanning `/proc/[pid]/fd`.

*   **Kill Workflow:** Pressing `k` on a selected process opens a centered modal dialog (`KillDialogState`).
*   **Signal Support:** The dialog allows sending `SIGTERM` (Enter) or `SIGKILL` (K).
*   **Safety:** The modal uses `ratatui::widgets::Clear` to overlay the UI, preventing accidental keystrokes from affecting the background state.
