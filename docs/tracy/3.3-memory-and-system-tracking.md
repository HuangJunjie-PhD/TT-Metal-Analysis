This document covers Tracy's memory allocation tracking and system-level profiling capabilities. These features provide insight into memory usage patterns, system resource consumption, and low-level system events like context switches and power consumption.

For basic profiling zones and GPU profiling, see [Core Profiling API](https://deepwiki.com/tenstorrent/tracy/3.1-core-profiling-api) and [GPU Profiling](https://deepwiki.com/tenstorrent/tracy/3.2-gpu-profiling). For advanced features like callstack capture and lock profiling, see [Advanced Client Features](https://deepwiki.com/tenstorrent/tracy/3.4-advanced-client-features).

### Named and Secure Allocations

Tracy supports named allocations for categorizing memory usage and secure allocations for sensitive data tracking:

| Allocation Type | Purpose | Queue Type |
| --- | --- | --- |
| Named | Associates allocations with custom names | `MemAllocNamed`, `MemFreeNamed` |
| Secure | Tracks sensitive allocations with availability checks | Uses `secure` parameter |
| Named + Callstack | Combines naming with callstack capture | `MemAllocCallstackNamed` |

## Queue Integration and Data Flow

Memory and system tracking events flow through Tracy's queue system alongside other profiling data:

| Queue Type | Purpose | Data Structure |
| --- | --- | --- |
| `MemAlloc` | Basic allocation tracking | `QueueMemAlloc` |
| `MemAllocCallstack` | Allocation with callstack | `QueueMemAlloc` + callstack |
| `MemAllocNamed` | Named allocation | `QueueMemAlloc` + name |
| `SysTimeReport` | System time events | `QueueSysTime` |
| `SysPowerReport` | Power consumption | `QueueSysPower` |
| `ContextSwitch` | Thread context switches | `QueueContextSwitch` |
| `ThreadWakeup` | Thread wakeup events | `QueueThreadWakeup` |
