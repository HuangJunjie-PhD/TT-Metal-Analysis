This page provides a comprehensive reference of all CMake build options available in TVM. These options control which features, backends, and libraries are included in the compiled TVM binaries. For information about the general CMake configuration system, see [CMake Configuration](/tenstorrent/tt-tvm/9.1-cmake-configuration). For target-specific configuration examples, see [Target-Specific Configuration](/tenstorrent/tt-tvm/9.3-target-specific-configuration). For details on the resulting library outputs, see [Library Outputs](/tenstorrent/tt-tvm/9.4-library-outputs).

## Overview

TVM uses CMake's `tvm_option()` macro to define approximately 100+ build-time configuration options. These options control:

* **Backend runtimes**: GPU (CUDA, ROCm, Vulkan, Metal, OpenCL), embedded (Hexagon, Micro), and specialized accelerators
* **Code generation**: LLVM version, MLIR support, target-specific codegen
* **Execution models**: Graph executor, VM, AOT executor, profiling support
* **External libraries**: BLAS, DNNL, cuDNN, TensorRT, Arm Compute Library, and many others
* **Build configuration**: Debugging, symbol visibility, static vs dynamic linking, ccache usage

All options are defined with defaults in [CMakeLists.txt29-128](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L29-L128) and can be overridden in a user-provided `config.cmake` file.

## Configuration Workflow

The build option system follows a three-stage workflow: user configuration, compile-time injection, and runtime introspection.

**Sources**: [CMakeLists.txt17-23](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L23) [cmake/modules/LibInfo.cmake21-141](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake#L21-L141) [src/support/libinfo.cc267-368](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L267-L368) [python/tvm/support.py33-48](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/support.py#L33-L48)

### Configuration Process

1. **User creates config.cmake**: Copy the template from [cmake/config.cmake](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake) to your build directory
2. **User edits config.cmake**: Set desired options using CMake `set()` commands
3. **CMake processes configuration**: [CMakeLists.txt17-23](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L23) loads config.cmake and applies options
4. **Compile-time definitions injected**: [cmake/modules/LibInfo.cmake](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake) calls `add_lib_info()` to inject all option values as preprocessor definitions into [src/support/libinfo.cc](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc)
5. **Runtime introspection available**: Compiled options accessible via `tvm.support.libinfo()` in Python or `tvm::GetLibInfo()` in C++

**Sources**: [CMakeLists.txt17-28](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L28) [cmake/config.cmake18-37](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L18-L37)

## Runtime Introspection

All build options are embedded into the compiled libraries and can be queried at runtime. This enables applications to check which features are available without recompilation.

### Introspection Architecture

**Sources**: [cmake/modules/LibInfo.cmake21-141](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake#L21-L141) [src/support/libinfo.cc267-372](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L267-L372) [python/tvm/support.py33-68](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/support.py#L33-L68)

### Python Introspection API

The Python API provides dictionary access to all build options:

**Implementation**: The function `tvm.support.libinfo()` calls the packed function `support.GetLibInfo` registered at [src/support/libinfo.cc370](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L370-L370)

### C++ Introspection API

**Sources**: [src/support/libinfo.cc267-372](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L267-L372) [python/tvm/support.py33-68](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/support.py#L33-L68)

### Introspection Data Flow

Each option follows this transformation:

1. **CMake time**: `set(USE_CUDA ON)` in config.cmake
2. **Compile definition**: `add_lib_info()` adds `-DTVM_INFO_USE_CUDA="ON"` to compiler flags for libinfo.cc
3. **C++ constant**: `#ifndef TVM_INFO_USE_CUDA` / `#define TVM_INFO_USE_CUDA "NOT-FOUND"` in [src/support/libinfo.cc41-259](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L41-L259)
4. **Runtime map entry**: `{"USE_CUDA", TVM_INFO_USE_CUDA}` in GetLibInfo() at [src/support/libinfo.cc268-366](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L268-L366)
5. **User query**: `tvm.support.libinfo()["USE_CUDA"]` returns `"ON"`

**Sources**: [cmake/modules/LibInfo.cmake38-139](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake#L38-L139) [src/support/libinfo.cc25-372](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/support/libinfo.cc#L25-L372)

## Option Validation and Testing

TVM includes a lint check to ensure all build options are properly mirrored between the definition site (CMakeLists.txt), the compile-time injection (LibInfo.cmake), and the runtime introspection (libinfo.cc).

### Validation Script

The script [tests/lint/check\_cmake\_options.py](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/lint/check_cmake_options.py) validates:

1. **All `tvm_option()` calls in CMakeLists.txt** are extracted
2. **Each option appears in libinfo.cc** with entry `{"OPTION_NAME", TVM_INFO_OPTION_NAME}`
3. **Each option has compile definition in LibInfo.cmake** with `TVM_INFO_OPTION_NAME="${OPTION_NAME}"`

This ensures options remain synchronized across all three systems.

**Sources**: [tests/lint/check\_cmake\_options.py32-83](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/lint/check_cmake_options.py#L32-L83)

## Summary

TVM provides comprehensive build-time configuration through 100+ CMake options organized into:

* **Backend runtimes** (CUDA, ROCm, Vulkan, Metal, OpenCL, Hexagon, Micro)
* **Code generation** (LLVM, MLIR)
* **Execution models** (Graph executor, AOT, VM, profiling)
* **External libraries** (cuDNN, DNNL, TensorRT, Arm Compute Library, etc.)
* **Build configuration** (debugging, symbol visibility, linking)

All options are:

1. Defined with defaults in [CMakeLists.txt29-128](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L29-L128)
2. Overridden by users in `config.cmake`
3. Injected as compile definitions via [cmake/modules/LibInfo.cmake](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/modules/LibInfo.cmake)
4. Queryable at runtime via `tvm.support.libinfo()` or `tvm::GetLibInfo()`

For practical usage examples, see [Target-Specific Configuration](/tenstorrent/tt-tvm/9.3-target-specific-configuration). For details on the resulting library outputs, see [Library Outputs](/tenstorrent/tt-tvm/9.4-library-outputs). For runtime introspection details, see [Runtime Introspection](/tenstorrent/tt-tvm/9.5-runtime-introspection).
