## Purpose and Scope

This document explains the compiler configuration system in the PyBuda framework, which allows developers to control how models are compiled, optimized, and executed on Tenstorrent hardware. It covers the central configuration class, key configuration parameters, methods for setting configuration options, and how configuration affects the compilation pipeline. For information about the overall compilation process, see [Core Architecture](https://deepwiki.com/tenstorrent/tt-buda/2-core-architecture) and [TTDevice and Compilation Pipeline](https://deepwiki.com/tenstorrent/tt-buda/2.1-ttdevice-and-compilation-pipeline). For testing-specific configuration, see [Test Configuration](https://deepwiki.com/tenstorrent/tt-buda/5.2-test-configuration).

## Overview

The compiler configuration system provides fine-grained control over many aspects of the PyBuda compilation process, including training behavior, graph optimization, operation placement strategies, memory optimization, data formats, and precision. Proper configuration is essential for achieving optimal model performance on Tenstorrent hardware.

The central class for compiler configuration is `CompilerConfig`, defined in `pybuda/config.py`. This dataclass contains numerous fields that control different aspects of compilation, with reasonable defaults for most use cases.

## Setting Configuration Options

### Using the API

The recommended way to configure the compiler is through the `set_configuration_options()` function. This function provides a clean interface to modify the global compiler configuration (`g_compiler_config`).

`# Example of setting configuration optionsimport pybudafrom pybuda import DataFormat, MathFidelity pybuda.set_configuration_options(    balancer_policy="Ribbon",    enable_t_streaming=True,    default_df_override=DataFormat.Float16_b,    math_fidelity=MathFidelity.HiFi3,    amp_level=1,    backend_opt_level=3)`
Sources: [pybuda/pybuda/config.py 436-565](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/pybuda/config.py#L436-L565)

## Key Configuration Parameters

### Training and Learning Parameters

*   **enable_training**: Enable training mode, including autograd passes
*   **enable_recompute**: Optimize memory usage during training by recomputing activations instead of storing all intermediate values

### Graph Optimization Parameters

*   **match_subgraph_patterns**: Control pattern matching for optimizing subgraphs
*   **enable_consteval**: Enable constant evaluation for graph simplification
*   **enable_auto_fusing**: Enable automatic operation fusion for better performance
*   **op_names_dont_fuse**: List of operations that should not be fused
*   **op_names_manual_fuse**: List of operations to allow being fused when auto-fusing is disabled

### Placement Strategy Parameters

*   **balancer_policy**: Strategy for operation placement. Valid values include:

    *   "NLP": Optimized for NLP-like models
    *   "Ribbon": Optimized for CNN-like models
    *   "MinimizeGrid": Simple policy that always chooses smallest grid (for verification)
    *   "Random": Pick random valid grids (for verification)

*   **scheduler_policy**: Policy for determining operation order during placement

*   **op_names_to_epoch_break**: Operations that should be placed on new epochs

*   **op_names_to_chip_break**: Operations that should be placed on new chips

### Memory Optimization Parameters

*   **enable_t_streaming**: Enable t-streaming for reduced buffering and improved memory usage
*   **manual_t_streaming**: Only respect explicit overrides for t-streaming, otherwise no streaming

### Data Format and Precision Parameters

*   **default_df_override**: Default data format for all operations
*   **default_accumulate_df**: Format used for accumulation, when supported by the hardware
*   **default_math_fidelity**: Default math fidelity for all operations
*   **amp_level**: Automatic Mixed Precision optimization level (0-3)

### Backend Configuration Parameters

*   **backend_opt_level**: Backend optimization level (0-4)
*   **backend_output_dir**: Directory for backend compilation outputs and binaries
*   **backend_device_descriptor_path**: Path to YAML file for device descriptor
*   **backend_cluster_descriptor_path**: Path to YAML file for multi-device cluster descriptor
*   **backend_runtime_params_path**: Path to YAML file for backend database configurations

## Advanced Configuration Options

### Balancer and Placement Configuration

The balancer system determines how operations are mapped to the physical hardware. It uses a balancer policy to make decisions about grid shapes, block layouts, and memory allocation.

Configuration options for the balancer include:

*   **balancer_policy**: The high-level strategy for placement decisions
*   **balancer_op_overrides**: Override balancer settings for specific operations
*   **place_on_new_epoch(op_names)**: Place specified operations on new epochs
*   **place_on_new_chip(op_names)**: Place specified operations on new chips
*   **dont_fuse(op_names)**: Prevent specified operations from being fused
*   **manual_fuse(op_names)**: Allow only specified operations to be fused

Sources: [pybuda/csrc/balancer/legalizer/legalizer.cpp 13-133](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/csrc/balancer/legalizer/legalizer.cpp#L13-L133)[pybuda/csrc/balancer/python_bindings.cpp 39-241](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/csrc/balancer/python_bindings.cpp#L39-L241)

### Operation Model Configuration

The operation model (`OpModel`) represents how an operation is mapped to hardware. It includes information about grid shape, block shapes, data formats, and more.

Configuration options that affect operation models include:

*   **default_df_override**: Sets the default data format for all operation models
*   **default_math_fidelity**: Sets the default math precision for operations
*   **enable_t_streaming**: Allows operations to be streamed for reduced buffering
*   **op_overrides**: Provides per-operation overrides for grid shapes, streaming, etc.

### Sparse Matrix Multiplication Configuration

Sparse matrix multiplication has specific configuration options due to its special implementation on the hardware.

Important parameters include:

*   **u_kt**: Controls the ublock k-dimension tile size for sparse matmuls
*   **sparse_tile_ptr_bits**: Bits used for sparse tile pointers
*   **sparse_ublock_idx_bits**: Bits used for sparse ublock indices

### Performance and Evaluation Configuration

PyBuda provides configuration options for measuring and improving performance.

*   **performance_trace**: Controls performance tracing verbosity
*   **amp_level**: Automatic Mixed Precision optimization level
*   **enable_amp_light**: Enable a "light" version of mixed precision

## Common Configuration Scenarios

### Performance Optimization

`# Configure for performance optimizationimport pybudafrom pybuda import DataFormat, MathFidelity pybuda.set_configuration_options(    balancer_policy="Ribbon",  # Use Ribbon policy for CNN-like models    enable_t_streaming=True,   # Enable t-streaming for memory optimization    default_df_override=DataFormat.Bfp8_b,  # Use BFP8_b for reduced precision    math_fidelity=MathFidelity.HiFi2,      # Use HiFi2 for good accuracy/performance balance    amp_level=2,               # Use automatic mixed precision level 2    backend_opt_level=4        # Maximum backend optimization)`
### Accuracy-Focused Configuration

`# Configure for maximum accuracyimport pybudafrom pybuda import DataFormat, MathFidelity pybuda.set_configuration_options(    balancer_policy="NLP",     # Use NLP policy for transformer-like models    enable_t_streaming=False,  # Disable t-streaming for stability    default_df_override=DataFormat.Float16_b,  # Use higher precision format    math_fidelity=MathFidelity.HiFi4,        # Use maximum math fidelity    amp_level=0,               # Disable automatic mixed precision    backend_opt_level=2        # Moderate backend optimization)`
### Debugging Configuration

`# Configure for debuggingimport pybudafrom pybuda import DataFormat, MathFidelity pybuda.set_configuration_options(    enable_consteval=False,    # Disable constant evaluation for more detailed graph    compile_depth=pybuda.CompileDepth.BUDA_GRAPH_PRE_PLACER,  # Stop at pre-placer phase    performance_trace=pybuda.PerfTraceLevel.VERBOSE,  # Enable detailed performance trace    backend_output_dir="./debug_output"  # Set specific output directory) # Also set environment variables for even more debugging controlimport osos.environ["PYBUDA_BALANCER_PLACER_DATA"] = "1"  # Print balancer/placer infoos.environ["EVAL_DEBUG"] = "2"  # Print inputs/outputs during evaluation`

## Conclusion

The PyBuda compiler configuration system provides extensive control over how models are compiled and executed on Tenstorrent hardware. By properly configuring the compiler, you can optimize for performance, accuracy, memory usage, or debugging capabilities, depending on your specific needs.

The primary interface for configuration is the `set_configuration_options()` function, with additional control available through environment variables for advanced users and debugging scenarios.

Understanding the key configuration parameters and their effects is essential for achieving optimal results when deploying models on Tenstorrent hardware.
