# GitHub Workflows

Relevant source files
*   [.github/workflows/community-issue-tagging.yml](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/.github/workflows/community-issue-tagging.yml)

## Purpose and Scope

This document details the GitHub workflows implemented in the tt-firmware repository. GitHub workflows are automated processes defined in YAML configuration files that execute in response to specific repository events. In the tt-firmware repository, the primary workflow focuses on community issue and pull request management through automated tagging and labeling.

## Overview of GitHub Workflows

The tt-firmware repository utilizes GitHub Actions workflows to automate repository management tasks. These workflows are defined in YAML files stored in the `.github/workflows/` directory and execute on GitHub's infrastructure when triggered by specific events.

### Available Workflows

The repository currently implements one workflow:

*   **Community Issue/PR Labeling Workflow**: Automatically tags and categorizes issues and pull requests upon creation

## Community Issue/PR Labeling Workflow

The community issue and PR labeling workflow provides automated categorization of issues and pull requests when they are first opened in the repository.

### Workflow Configuration

The workflow is defined in [.github/workflows/community-issue-tagging.yml 1-12](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/.github/workflows/community-issue-tagging.yml#L1-L12) and uses the following configuration:

| Trigger | Event Type | Description |
| --- | --- | --- |
| Issues | opened | Workflow runs when a new issue is created |
| Pull Requests | opened | Workflow runs when a new PR is created |

Sources: [.github/workflows/community-issue-tagging.yml 3-7](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/.github/workflows/community-issue-tagging.yml#L3-L7)

### Workflow Implementation

The workflow does not contain direct implementation logic. Instead, it calls a centralized workflow defined in the `tenstorrent/tt-github-actions` repository:

`jobs:  call-central-workflow:    uses: tenstorrent/tt-github-actions/.github/workflows/on-community-issue.yml@main`
This centralized approach ensures consistent issue and PR management across multiple Tenstorrent repositories.

Sources: [.github/workflows/community-issue-tagging.yml 9-11](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/.github/workflows/community-issue-tagging.yml#L9-L11)

## Expected Behavior and Benefits

The community issue tagging workflow likely performs the following functions:

1.   Categorizes issues and PRs based on type and content
2.   Applies labels for prioritization and routing
3.   May assign appropriate teams or individuals for review
4.   Ensures consistent handling of community contributions

Benefits of this automated approach include:

*   Consistent labeling across all repository contributions
*   Faster initial processing of issues and PRs
*   Reduced manual categorization effort for maintainers
*   Improved visibility into contribution types and sources

The workflow's integration with a central repository also offers:

*   Simplified maintenance of common workflows
*   Consistent processes across multiple Tenstorrent repositories
*   Single source of truth for workflow definitions
*   Easier updates to workflow logic
