## Architecture and Target Options

The most critical configuration options determine the target architecture, ABI, and instruction set features.

Sources: [configure.ac 45-96](https://github.com/tenstorrent/sfpi/blob/ed989b9d/configure.ac#L45-L96)[Makefile.in 36-40](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L36-L40)[scripts/build.sh 25-26](https://github.com/tenstorrent/sfpi/blob/ed989b9d/scripts/build.sh#L25-L26)

### Key Architecture Options

These options determine the target triple `riscv-$MFC-elf` and the instruction set extensions.

| Option | Description | Default | SFPI Common Values |
| --- | --- | --- | --- |
| `--with-arch` | Sets the base RISC-V ISA | `rv64imafdc` | `rv32i` (bare minimum for SFPI) |
| `--with-abi` | Sets the base RISC-V ABI | Based on arch | `ilp32` (32-bit integer, 32-bit pointers) |
| `--with-tune` | Sets the CPU tuning | `rocket` | `rocket` |
| `--with-isa-spec` | Sets the ISA spec version | `2.2` | `2.2` |
| `--with-mfc` | Sets manufacturer in target triple | `unknown` | `tt` (creates `riscv-tt-elf`) |

The `--with-arch` and `--with-abi` options are passed to both GCC stages and the C library build.

**Architecture String Processing:**

```
WITH_ARCH=--with-arch=$with_arch
WITH_ABI=--with-abi=$with_abi
```

These are used in: `stamps/build-gcc-newlib-stage1` (line 301), `stamps/build-gcc-newlib-stage2` (line 403)

### Multilib Generator Syntax

The `--with-multilib-generator` option accepts a string specifying combinations:

```
--with-multilib-generator="rv32i-ilp32-- rv32im-ilp32--"
```

Format: `<arch>-<abi>-<extra-flags>`

The double dash (`--`) at the end indicates no extra flags.

## Compilation Pipeline Details

The build process uses a bootstrap approach where each stage depends on the previous one.

**Diagram: Build Stage Dependencies and Stamp Files**

The build system uses stamp files in the `stamps/` directory to track completion of each stage and avoid rebuilding completed stages.

Sources: [Makefile.in 235-410](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L235-L410)[scripts/build.sh 148](https://github.com/tenstorrent/sfpi/blob/ed989b9d/scripts/build.sh#L148-L148)

### Stage 1: Binutils (`build-binutils-newlib`)

Builds the assembler, linker, and binary utilities.

```
cd build-binutils-newlib && $BINUTILS_SRCDIR/configure \
  --target=riscv-tt-elf \
  --prefix=$INSTALL_DIR \
  --disable-werror
make -C build-binutils-newlib
make -C build-binutils-newlib install
```

Configuration variables: `BINUTILS_TARGET_FLAGS`, `BINUTILS_GDB_FLAGS`, `IDENT_OPTIONS`

Sources: [Makefile.in 235-251](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L235-L251)

### Stage 2: GCC Stage 1 (`build-gcc-newlib-stage1`)

Builds a minimal GCC that can compile freestanding code but lacks C library support.

```
cd build-gcc-newlib-stage1 && $GCC_SRCDIR/configure \
  --target=riscv-tt-elf \
  --prefix=$INSTALL_DIR \
  --disable-shared --disable-threads --disable-tls \
  --enable-languages=c,c++ \
  --with-newlib \
  --disable-libmudflap --disable-libssp \
  --disable-libquadmath --disable-libgomp
make -C build-gcc-newlib-stage1 all-gcc
make -C build-gcc-newlib-stage1 install-gcc
```

Key configuration options: `GCC_CHECKING_FLAGS`, `GCC_MULTILIB_FLAGS`, `WITH_ARCH`, `WITH_ABI`, `WITH_TUNE`

Sources: [Makefile.in 274-308](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L274-L308)

### Stage 3: Newlib (`build-newlib` and `build-newlib-nano`)

Builds the C library in two variants:

**Standard Newlib** - Full-featured C library:

```
cd build-newlib && $NEWLIB_SRCDIR/configure \
  --target=riscv-tt-elf \
  --enable-newlib-io-long-double \
  --enable-newlib-io-long-long \
  --enable-newlib-io-c99-formats \
  CFLAGS_FOR_TARGET="-O2 -D_POSIX_MODE"
```

**Newlib-nano** - Size-optimized variant:

```
cd build-newlib-nano && $NEWLIB_SRCDIR/configure \
  --target=riscv-tt-elf \
  --enable-newlib-reent-small \
  --enable-newlib-nano-malloc \
  --enable-newlib-nano-formatted-io \
  CFLAGS_FOR_TARGET="-Os -ffunction-sections"
```

The nano libraries are installed as `libc_nano.a`, `libm_nano.a`, etc., allowing users to select them with `-specs=nano.specs`.

Sources: [Makefile.in 310-349](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L310-L349)[Makefile.in 351-372](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L351-L372)

### Stage 4: GCC Stage 2 (`build-gcc-newlib-stage2`)

Builds the complete compiler with full C/C++ library support.

```
cd build-gcc-newlib-stage2 && $GCC_SRCDIR/configure \
  --target=riscv-tt-elf \
  --prefix=$INSTALL_DIR \
  --disable-shared --disable-threads \
  --enable-languages=c,c++ \
  --enable-tls \
  --with-newlib \
  --with-native-system-header-dir=/include
make -C build-gcc-newlib-stage2
make -C build-gcc-newlib-stage2 install
```

This stage produces the final, complete toolchain with support for both C and C++.

Sources: [Makefile.in 374-410](https://github.com/tenstorrent/sfpi/blob/ed989b9d/Makefile.in#L374-L410)
