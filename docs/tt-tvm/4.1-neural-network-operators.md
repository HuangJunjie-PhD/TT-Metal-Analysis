## Purpose and Scope

This document describes the neural network (NN) operators available in the Relay IR system. These operators form the high-level vocabulary for expressing deep learning models and include convolution, pooling, normalization, activation, and dense operations.

NN operators are part of the Relay IR's graph-level representation and provide the building blocks for constructing neural network computation graphs. For information about tensor transformation operators (reshape, transpose, etc.), see [Tensor Transformation Operators](https://deepwiki.com/tenstorrent/tt-tvm/4.2-tensor-transformation-operators). For details on the type system and shape inference, see [Type System and Shape Inference](https://deepwiki.com/tenstorrent/tt-tvm/4.3-type-system-and-shape-inference). For layout transformations, see [Layout Transformation System](https://deepwiki.com/tenstorrent/tt-tvm/4.4-layout-transformation-system).

## Operator Categories

The neural network operators are organized into several functional categories:

| Category | Operators | Purpose |
| --- | --- | --- |
| **Convolution** | `conv1d`, `conv2d`, `conv3d`, `conv2d_transpose`, `conv3d_transpose` | Spatial feature extraction with learnable filters |
| **Pooling** | `max_pool1d/2d/3d`, `avg_pool1d/2d/3d`, `global_max_pool2d`, `global_avg_pool2d`, `adaptive_max/avg_pool1d/2d/3d` | Downsampling and spatial invariance |
| **Normalization** | `batch_norm`, `layer_norm`, `group_norm`, `instance_norm`, `lrn`, `l2_normalize` | Feature normalization for training stability |
| **Activation** | `relu`, `leaky_relu`, `prelu`, `softmax`, `log_softmax`, `fast_softmax` | Non-linear transformations |
| **Dense** | `dense`, `matmul`, `batch_matmul`, `contrib_dense_pack` | Fully-connected layers and matrix multiplication |
| **Sparse** | `sparse_dense`, `sparse_conv2d`, `sparse_transpose` | Optimized operations for sparse tensors |
| **Specialized** | `bias_add`, `fifo_buffer`, `pad`, `mirror_pad`, `upsampling`, `upsampling3d` | Supporting operations |

## Operator Registration Architecture

The NN operators follow a consistent registration pattern that binds Python APIs to C++ implementations with multiple extension points.

### Registration Components

**Sources:**[src/relay/op/nn/convolution.cc 400-500](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.cc#L400-L500)[python/tvm/relay/op/nn/_nn.py 207-240](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L207-L240)

### Operator Registration Example

Each operator is registered with multiple components:

1.   **Operator Definition** - Declared in C++ with `RELAY_REGISTER_OP`:

    *   Operator name (e.g., `"nn.conv2d"`)
    *   Human-readable description
    *   Number of inputs
    *   Argument documentation
    *   Support level

2.   **Attributes** - Typed parameters defined via `TVM_DECLARE_ATTRS`:

    *   `strides`, `padding`, `dilation` for spatial operators
    *   `channels`, `kernel_size` for convolution
    *   `data_layout`, `kernel_layout` for layout specification

3.   **Type Relation** - Shape and type inference function:

    *   Validates input types
    *   Computes output shapes
    *   Handles dynamic dimensions

4.   **Compute/Strategy** - Target-dependent implementation:

    *   Generic fallback implementations
    *   Specialized schedules per target (CUDA, x86, ARM)

**Sources:**[src/relay/op/nn/nn.cc 223-240](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/nn.cc#L223-L240)[include/tvm/relay/attrs/nn.h 119-188](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/include/tvm/relay/attrs/nn.h#L119-L188)

## Convolution Operators

Convolution operators are the most complex NN operators with extensive support for different dimensionalities and specialized implementations.

### Convolution Family

**Sources:**[src/relay/op/nn/convolution.cc 57-800](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.cc#L57-L800)[python/tvm/relay/op/nn/nn.py 30-678](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/nn.py#L30-L678)

### Conv2D Attributes

The `Conv2DAttrs` structure defines convolution parameters:

```
strides: Array<IndexExpr>           // Stride in each spatial dimension
padding: Array<IndexExpr>           // Padding (supports asymmetric)
dilation: Array<IndexExpr>          // Dilation rate for dilated convolution
groups: int                         // Number of groups (depthwise when groups=in_channels)
channels: IndexExpr                 // Number of output channels
kernel_size: Array<IndexExpr>       // Kernel spatial dimensions
data_layout: String                 // Input layout (NCHW, NHWC, etc.)
kernel_layout: String               // Weight layout (OIHW, HWIO, etc.)
out_layout: String                  // Output layout (default: same as data_layout)
out_dtype: DataType                 // Output dtype for mixed precision
```

**Key Features:**

*   **Layout Flexibility**: Supports arbitrary layouts via `data_layout` and `kernel_layout`
*   **Grouped Convolution**: Implemented via `groups` parameter
*   **Mixed Precision**: Explicit `out_dtype` for int8→int32 accumulation
*   **Asymmetric Padding**: Supports 1/2/4-element padding specification

**Sources:**[include/tvm/relay/attrs/nn.h 119-188](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/include/tvm/relay/attrs/nn.h#L119-L188)[src/relay/op/nn/convolution.cc 400-600](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.cc#L400-L600)

### Type Relation and Shape Inference

The `Conv2DRel` type relation function performs shape inference:

The shape computation follows standard convolution formulas:

```
out_h = (in_h + pad_top + pad_bottom - dilated_kernel_h) / stride_h + 1
out_w = (in_w + pad_left + pad_right - dilated_kernel_w) / stride_w + 1
```

**Sources:**[src/relay/op/nn/convolution.cc 400-550](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.cc#L400-L550)[src/relay/op/nn/convolution.h 43-44](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.h#L43-L44)

## Pooling Operators

Pooling operators perform spatial downsampling through aggregation functions.

### Pooling Variants

| Operator | Aggregation | Dimensions | Adaptive |
| --- | --- | --- | --- |
| `max_pool1d/2d/3d` | Maximum | 1D/2D/3D | No |
| `avg_pool1d/2d/3d` | Average | 1D/2D/3D | No |
| `global_max_pool2d` | Maximum | 2D | N/A (pools to 1×1) |
| `global_avg_pool2d` | Average | 2D | N/A (pools to 1×1) |
| `adaptive_max_pool1d/2d/3d` | Maximum | 1D/2D/3D | Yes |
| `adaptive_avg_pool1d/2d/3d` | Average | 1D/2D/3D | Yes |

**Adaptive Pooling**: Output size is specified directly; stride is computed automatically to achieve target output dimensions.

**Sources:**[src/relay/op/nn/pooling.cc 43-350](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/pooling.cc#L43-L350)[python/tvm/relay/op/nn/nn.py 753-1250](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/nn.py#L753-L1250)

## Dense and Matrix Multiplication Operators

Dense layers and matrix multiplication form the backbone of fully-connected networks and attention mechanisms.

### Dense Operator Family

**Key Differences:**

*   **`dense`**: Matrix multiplication with implicit transpose of weight (`Y = XW^T`)
*   **`matmul`**: General matrix multiplication with optional transpose flags
*   **`batch_matmul`**: Batched matrix multiplication with broadcasting

**Sources:**[src/relay/op/nn/nn.cc 170-310](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/nn.cc#L170-L310)[python/tvm/relay/op/nn/nn.py 1847-2100](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/nn.py#L1847-L2100)

### Dense Type Relation

The `MatmulRel` template handles type inference for dense and matmul operators:

```
Input: data shape [*, input_dim], weight shape [units, input_dim]
Output: [*, units]

Where * represents arbitrary leading batch dimensions
```

For mixed precision, the `out_dtype` parameter specifies accumulation type:

```
data: int8, weight: int8 → output: int32
```

## Activation and Normalization Operators

### Activation Functions

| Operator | Formula | Attributes |
| --- | --- | --- |
| `relu` | `max(x, 0)` | None |
| `leaky_relu` | `x > 0 ? x : alpha * x` | `alpha` (slope) |
| `prelu` | `x > 0 ? x : alpha[c] * x` | `axis` (channel axis) |
| `softmax` | `exp(x_i) / sum(exp(x_j))` | `axis` |
| `log_softmax` | `log(softmax(x))` | `axis` |
| `fast_softmax` | Approximated softmax | `axis` |

**Sources:**[src/relay/op/nn/nn.cc 312-518](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/nn.cc#L312-L518)[python/tvm/relay/op/nn/nn.py 680-750](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/nn.py#L680-L750)

### Batch Normalization

`nn.batch_norm` performs normalization across the batch dimension with learnable affine parameters:

```
Inputs:
  - data: Input tensor
  - gamma: Scale parameter (optional if scale=False)
  - beta: Shift parameter (optional if center=False)
  - moving_mean: Running mean for inference
  - moving_var: Running variance for inference

Outputs: Tuple of (normalized_data, updated_mean, updated_var)

Attributes:
  - axis: Channel axis (default: 1)
  - epsilon: Small constant for numerical stability
  - center: Whether to use beta parameter
  - scale: Whether to use gamma parameter
  - momentum: Momentum for moving average updates
```

**Sources:**[python/tvm/relay/op/nn/_nn.py 150-152](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L150-L152)[python/tvm/relay/op/nn/nn.py 2600-2700](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/nn.py#L2600-L2700)

## Target-Dependent Strategy System

The strategy system allows operators to have multiple implementations optimized for different hardware targets.

### Strategy Registration Pattern

**Sources:**[python/tvm/relay/op/nn/_nn.py 207-212](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L207-L212)

### OpStrategy Structure

An `OpStrategy` contains multiple implementation choices:

1.   **Compute Function**: Generates TensorIR (TIR) from operator attributes and inputs
2.   **Schedule Function**: Applies target-specific optimizations to the TIR
3.   **Implementation Name**: Identifies the specific algorithm (e.g., "direct", "winograd")
4.   **Plevel**: Priority level when multiple implementations are available

The strategy system integrates with:

*   **AutoTVM**: Template-based autotuning selects optimal implementation
*   **AutoScheduler**: Template-free search generates schedules automatically
*   **Meta Schedule**: Advanced scheduling with learned cost models

## Layout Transformation Hooks

NN operators support three types of layout transformation hooks that enable layout optimization passes.

### Layout Hook Types

**Sources:**[python/tvm/relay/op/nn/_nn.py 106-316](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L106-L316)[src/relay/transforms/alter_op_layout.cc 1-200](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/transforms/alter_op_layout.cc#L1-L200)

### Layout Transformation Functions

1.   **`FTVMAlterOpLayout`**: Suggests alternative layouts with higher performance

    *   Called by `AlterOpLayout` pass
    *   Returns modified operator with new layout attributes
    *   Example: Suggest `NCHW16c` for vectorized convolution

2.   **`FTVMConvertOpLayout`**: Converts operator to specified layout

    *   Called by `ConvertLayout` pass
    *   Inserts `layout_transform` nodes as needed
    *   Handles default kernel layouts based on data layout

3.   **`FInferCorrectLayout`**: Declares preferred layouts for an operator

    *   Returns tuple of (input_layouts, output_layouts)
    *   Used by layout-aware passes for optimization

## Legalization System

Legalization transforms high-level operators into canonical forms suitable for code generation.

### Legalization Hook Registration

**Sources:**[python/tvm/relay/op/nn/_nn.py 56-74](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L56-L74)[src/relay/transforms/legalize.cc 1-150](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/transforms/legalize.cc#L1-L150)

### Common Legalization Patterns

1.   **Mixed Precision Handling**: Insert explicit cast operations for mixed precision ops

```
conv2d(int8, int8, out_dtype=int32) 
→ conv2d(cast(int8→int32), cast(int8→int32))
```
2.   **Layout-Specific Optimizations**: Transform to layout-optimized implementations

```
conv2d(NHWC) → conv2d_nhwc_optimized  # Target has specialized NHWC kernel
```
3.   **Decomposition**: Break complex operators into primitives

```
batch_norm → (subtract, multiply, add)  # For inference mode
```
4.   **Padding Insertion**: Extract padding from operator into explicit pad node

```
conv2d(padding=(1,1)) → pad(...) → conv2d(padding=(0,0))
```

## Specialized Operator Variants

### Winograd Convolution

Winograd convolution uses the Winograd minimal filtering algorithm for faster computation with small kernels:

*   **`nn.contrib_conv2d_winograd_weight_transform`**: Pre-transforms weights
*   **`nn.contrib_conv2d_winograd_without_weight_transform`**: Convolution with pre-transformed weights

**Tile Sizes**: `F(2×2, 3×3)`, `F(4×4, 3×3)` for 2×2 and 4×4 output tiles with 3×3 kernels

**Sources:**[src/relay/op/nn/convolution.cc 42-56](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.cc#L42-L56)[python/tvm/relay/op/nn/_nn.py 783-836](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L783-L836)

### NCHWc Layout Operators

Blocked layout operators for efficient vectorization:

*   **`nn.contrib_conv2d_NCHWc`**: Convolution with channel-blocked layout (e.g., NCHW16c)
*   **`nn.contrib_depthwise_conv2d_NCHWc`**: Depthwise convolution with blocked layout

These operators use tiled channel dimensions for better cache utilization and SIMD vectorization.

**Sources:**[python/tvm/relay/op/nn/_nn.py 838-842](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L838-L842)

### Sparse Operators

Sparse operators optimize for tensors with many zero elements:

*   **`nn.sparse_dense`**: Dense layer with sparse weight matrix (CSR format)
*   **`nn.sparse_conv2d`**: Convolution with sparse weights
*   **`nn.sparse_transpose`**: Transpose sparse matrix representation

Sparse formats reduce memory bandwidth and enable specialized kernels that skip zero multiplications.

**Sources:**[python/tvm/relay/op/nn/_nn.py 154-204](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/nn/_nn.py#L154-L204)

## Testing and Validation

NN operators are extensively tested across multiple dimensions:

### Test Categories

1.   **Type Inference Tests**: Verify shape propagation and type checking

 
2.   **Numerical Correctness Tests**: Compare against reference implementations

 
3.   **Layout Transformation Tests**: Verify layout conversion correctness
