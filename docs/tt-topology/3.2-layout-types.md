This document covers the different topology layout types supported by tt-topology and their corresponding coordinate generation algorithms. Layout types determine how Tenstorrent chips are logically arranged and connected in multi-chip configurations.

For information about the command-line interface options, see [Command Options](https://deepwiki.com/tenstorrent/tt-topology/3.1-command-options). For details about hardware-specific configuration, see [Hardware Support](https://deepwiki.com/tenstorrent/tt-topology/5-hardware-support).

## Overview

The tt-topology system supports four distinct layout types that define how chip coordinates are calculated and ethernet connections are configured:

| Layout Type | Description | Algorithm | Use Case |
| --- | --- | --- | --- |
| `mesh` | Trivalent graph with max 3 connections per node | BFS coordinate assignment | Fully connected topologies |
| `linear` | Single line connection of all chips | Cycle detection or longest path | Chain configurations |
| `torus` | Cyclic connection forming a closed loop | Cycle detection | Ring topologies |
| `isolated` | Individual chips with disabled ethernet ports | Default state assignment | Single chip or disconnected setups |

## Mesh Layout

The mesh layout implements a trivalent graph topology where each chip can have a maximum of 3 connections. This layout uses a breadth-first search (BFS) algorithm to assign coordinates while maintaining spatial consistency.

### Mesh Coordinate Generation Algorithm

The mesh algorithm follows specific rules defined in [tt_topology/backend.py 578-653](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/backend.py#L578-L653):

1.   **TFLY connections** (`"T"`) always increment in Y direction
2.   **Left-to-Right chip connections** always increment in X direction
3.   **Regular connections** (`"X"`) use the first unused direction from parent

Sources: [tt_topology/backend.py 578-653](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/backend.py#L578-L653)[tt_topology/tests/test_mesh.py 1-143](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/tests/test_mesh.py#L1-L143)

## Linear and Torus Layouts

Both linear and torus layouts share the same coordinate generation algorithm since they both represent sequential chip arrangements. The difference lies in whether the sequence forms a closed loop (torus) or remains open-ended (linear).

### Cycle Detection Algorithm

### Coordinate Assignment

For both linear and torus layouts, chips are assigned coordinates sequentially:

```
chip_0: (0, 0)
chip_1: (0, 1) 
chip_2: (0, 2)
...
chip_n: (0, n)
```

The key difference between linear and torus is in the port disable configuration during hardware flashing, where torus maintains all connections while linear disables end connections.

## Isolated Layout

The isolated layout disables ethernet connections and sets all chips to default coordinates. This mode is used for single-chip configurations or when chips should operate independently.

### Isolated Configuration Process

The isolated mode specifically handles port disabling in [tt_topology/backend.py 230-275](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/backend.py#L230-L275):

*   **Local chips**: Sets `ETH_PARAM_PORT_DISABLE` to `0xFFFC`
*   **Remote chips** (n300): Sets `ETH_PARAM_PORT_DISABLE` to `0xFCFF`
*   **Coordinates**: All chips default to `(0,0)` except n300 remote chips at `(1,0)`

Sources: [tt_topology/backend.py 218-312](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/backend.py#L218-L312)[tt_topology/backend.py 230-275](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/backend.py#L230-L275)
