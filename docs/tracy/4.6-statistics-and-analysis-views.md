This document covers Tracy's comprehensive statistics and analysis system, which provides various views and tools for analyzing profiling data. These views enable detailed performance analysis through statistical aggregation, zone searching, memory tracking, and data visualization.

For general profiler application overview, see [Application Overview](https://deepwiki.com/tenstorrent/tracy/4.1-application-overview). For source code viewing capabilities, see [Source Code Viewer](https://deepwiki.com/tenstorrent/tracy/4.4-source-code-viewer).

## Purpose and Scope

The statistics and analysis views provide multiple specialized windows for examining profiling data from different perspectives:

*   **Statistics Window**: Aggregated performance data across instrumentation, sampling, and GPU zones
*   **Find Zone Window**: Zone search and filtering with histogram visualization
*   **Zone Info Window**: Detailed analysis of individual zones including time distribution
*   **Memory Analysis**: Memory allocation tracking and visualization
*   **Sample Analysis**: Call stack sampling statistics and symbol analysis

## Statistics Window Modes

The main statistics window supports three distinct analysis modes controlled by `m_statMode`:

### Mode Selection and Configuration

### Instrumentation Mode (Mode 0)

Analyzes CPU zones with three accumulation modes:

*   **Self Only**: Excludes child zone execution time using `GetZoneChildTimeFast()`
*   **All Children**: Includes total zone execution time
*   **Non-Reentrant**: Filters out recursive calls using `IsZoneReentry()`

### Sampling Mode (Mode 1)

Processes call stack samples with symbol resolution and inline function analysis:

*   Symbol location mapping through `m_statSampleLocation`
*   Inline function separation via `m_statSeparateInlines`
*   Time-based vs count-based display using `m_statSampleTime`

### GPU Mode (Mode 2)

Analyzes GPU zone performance across different contexts and command queues.

## Zone Analysis System

The zone analysis system provides detailed investigation capabilities for individual zones and zone collections:

### Find Zone Window

The `DrawFindZone()` function implements a powerful zone search and analysis interface:

*   **Pattern Matching**: Uses `m_worker.GetMatchingSourceLocation()` for zone discovery
*   **Range Filtering**: Optional time range limitation via `m_findZone.range`
*   **Histogram Visualization**: Statistical distribution of zone execution times
*   **Grouping Options**: Multiple grouping strategies through `GetSelectionTarget()`

### Zone Information Window

Individual zone analysis through `DrawZoneInfoWindow()`:

*   **Time Distribution**: Child zone analysis via `CalcZoneTimeData()`
*   **Call Stack Traces**: Parent-child relationship visualization
*   **Thread Context**: Associated thread information and context switches
*   **Source Location**: Integration with source code viewing

## UI Components and Interaction

The analysis views utilize Tracy's ImGui integration for rich interactive interfaces:

### Common UI Patterns

*   **Filterable Tables**: `ImGuiTextFilter` integration for data filtering
*   **Sortable Columns**: Column-based sorting with persistent state
*   **Context Menus**: Right-click actions for detailed investigation
*   **Tooltips**: Hover information with detailed statistics
*   **Clipboard Integration**: Copy functionality via `ClipboardButton()`

### Navigation Integration

Analysis views integrate with Tracy's main timeline through:

*   **Zone Selection**: Click-to-select zone highlighting
*   **Timeline Synchronization**: Zoom-to-zone functionality
*   **Cross-Reference**: Jump between analysis views and source code
