# Experimental Bundles

Relevant source files

## Purpose and Scope

This document describes experimental firmware bundles within the tt-firmware repository - what they are, how they're used, and when they should be deployed. Experimental bundles are modified versions of official firmware releases containing targeted adjustments to address specific issues or add experimental features. For information about standard firmware releases, see [Official Releases](https://deepwiki.com/tenstorrent/tt-firmware/3.1-official-releases). For details on firmware versioning and migration procedures, see [Versioning and Migration](https://deepwiki.com/tenstorrent/tt-firmware/3.3-versioning-and-migration).

Sources: [experiments/README.md 1-2](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/experiments/README.md?plain=1#L1-L2)

## What Are Experimental Bundles?

Experimental bundles are firmware packages built from the latest available official firmware release with specific modifications to address particular issues. These bundles serve as interim solutions for known problems while maintaining compatibility with the baseline firmware functionality.

Sources: [experiments/README.md 1-2](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/experiments/README.md?plain=1#L1-L2)

## Differences from Official Releases

Experimental bundles differ from official firmware releases in several key aspects:

| Aspect | Official Releases | Experimental Bundles |
| --- | --- | --- |
| Purpose | General use across all supported hardware | Address specific issues or provide targeted fixes |
| Testing | Comprehensive validation | Limited testing focused on the specific issue |
| Stability | Thoroughly validated | May contain experimental features |
| Recommendation | Preferred for production environments | Use only when addressing a specific known issue |
| Release cycle | Regular scheduled releases | Released as needed to address specific problems |

## Available Experimental Firmware

The experimental firmware bundles are published in the experiments directory of the tt-firmware repository. Each bundle includes release notes that detail:

1.   The specific issue(s) addressed
2.   Base firmware version used
3.   Modifications applied
4.   Compatibility considerations
5.   Installation instructions

The current list of available experimental firmware bundles can be found in the table within the experiments README file. This table is regularly updated as new experimental bundles are released.

Sources: [experiments/README.md 4-9](https://github.com/tenstorrent/tt-firmware/blob/c9a325bb/experiments/README.md?plain=1#L4-L9)

## When to Use Experimental Bundles

Experimental bundles should be used in the following scenarios:

1.   When you encounter a specific issue that is known to be addressed by an experimental bundle
2.   For testing specific fixes before they are incorporated into official releases
3.   When the latest official release doesn't support a specific hardware configuration that an experimental bundle does

It is recommended to return to the official firmware release path once the issue has been incorporated into an official release.
