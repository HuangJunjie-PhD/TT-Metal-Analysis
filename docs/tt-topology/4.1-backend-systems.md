## Purpose and Scope

The Backend Systems are the core processing components of tt-topology that handle hardware configuration, coordinate generation, and chip management. This document covers the `TopoBackend` and `TopoBackend_Octopus` classes, which serve as the primary interfaces between the CLI and the underlying hardware. For information about the CLI interface, see [CLI Reference](https://deepwiki.com/tenstorrent/tt-topology/3-cli-reference). For details about hardware memory layout, see [Memory Layout](https://deepwiki.com/tenstorrent/tt-topology/5.2-memory-layout).

### Device Configuration Management

The backend manages hardware configuration through several key methods:

| Method | Purpose | Memory Regions |
| --- | --- | --- |
| `get_eth_config_state()` | Read current chip configuration | `ETH_FW_VERSION_ADDR`, `ETH_PARAM_CHIP_COORD` |
| `flash_to_default_state()` | Reset chips to default state | `ETH_PARAM_CHIP_COORD`, `ETH_PARAM_PORT_DISABLE` |
| `flash_to_specified_state()` | Configure chips with coordinates | Custom coordinate addresses |

## Memory Address Management

Both backend classes rely on memory address constants for hardware configuration:

| Constant | Address | Purpose |
| --- | --- | --- |
| `ETH_PARAM_CHIP_COORD` | `0x21100 + 0x0` | Chip X,Y coordinates |
| `ETH_PARAM_PORT_DISABLE` | `0x21100 + 0x8` | Port enable/disable mask |
| `ETH_PARAM_MOBO_ETH_EN` | `0x21100 + 0xD0` | Mobo ethernet enable |
| `ETH_PARAM_RACK_SHELF` | `0x21100 + 0xD4` | Rack and shelf ID |
| `ETH_PARAM_RIGHT_OFFSET` | `0x100` | Right chip parameter offset |
