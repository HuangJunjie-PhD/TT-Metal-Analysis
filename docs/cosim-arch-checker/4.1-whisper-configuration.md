# Whisper Configuration

This document covers the configuration of the Whisper RISC-V Instruction Set Simulator (ISS) used as the reference model in the cosim-arch-checker framework. The configuration files define processor parameters, ISA extensions, memory layout, and Control and Status Register (CSR) initialization values that allow Whisper to accurately model the target DUT architecture.

For information about configuring DUT-specific parameters, see [DUT Parameters](https://deepwiki.com/tenstorrent/cosim-arch-checker/4.2-dut-parameters). For build system configuration, see [Build System](https://deepwiki.com/tenstorrent/cosim-arch-checker/5-build-system).

## Configuration File Overview

The Whisper ISS is configured through JSON files located in the `bridge/whisper/config/` directory. Each configuration file corresponds to a specific processor implementation and defines the architectural parameters that Whisper uses to model that processor.

## Processor-Specific Configurations

### Boom Core Configuration

The Boom configuration targets a 64-bit RISC-V core with standard extensions:

| Parameter | Value | Description |
| --- | --- | --- |
| `xlen` | 64 | 64-bit address width |
| `isa` | "rv64gc" | RV64 with Integer, Multiply, Atomic, Compressed, and Floating-point |
| `reset_vec` | "0x10040" | Boot address |
| `clint` | "0x2000000" | Core Local Interruptor base address |

### Ocelot Vector Configuration

The Ocelot configuration extends Boom with vector processing capabilities:

| Parameter | Value | Description |
| --- | --- | --- |
| `xlen` | 64 | 64-bit address width |
| `isa` | "rv64imabfdvus_zfh" | RV64 with Vector, Supervisor, User, and Half-precision Float |
| `reset_vec` | "0x10040" | Boot address |
| `clint` | "0x2000000" | Core Local Interruptor base address |

### Ascalon Configuration

The Ascalon configuration provides a different processor variant:

| Parameter | Value | Description |
| --- | --- | --- |
| `xlen` | 64 | 64-bit address width |
| `isa` | "rv64imabfdvus" | RV64 with Vector, Supervisor, and User modes |
| `reset_vec` | "0x80000000" | Different boot address |

## Vector Configuration

For processors with vector extensions (Boom and Ocelot), the vector configuration section defines the vector unit parameters:

The vector configuration specifies:

*   **bytes_per_vec**: 32 bytes (256 bits) per vector register
*   **min_bytes_per_elem**: 1 byte minimum element width
*   **max_bytes_per_elem**: 8 bytes maximum element width

### MIP Register Configuration

The Machine Interrupt Pending register (`mip`) configures initial interrupt state:

| Configuration | Reset Value | Purpose |
| --- | --- | --- |
| Boom | 0x8 | Software interrupt enabled for boot |
| Ocelot | 0x8 | Software interrupt enabled for boot |
| Ascalon | 0x0 | No initial interrupts |

### MSTATUS Register Configuration

The Machine Status register (`mstatus`) defines processor state:

*   **Reset Value**: `0x8000000a00007e00` (Boom/Ocelot) or `0x8000000a00007800` (Ascalon)
*   **Write Mask**: `0xFFFFFFF0007FF9FF` - Controls which fields are writable
*   **Read-only Fields**: SXL[35:34], UXL[33:32], WPRI[31:23], VS[10:9]
