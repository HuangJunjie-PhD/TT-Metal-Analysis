## Purpose and Scope

The Testing Framework provides automated testing infrastructure for validating tt-topology functionality across different hardware configurations and topology algorithms. This framework ensures reliability of coordinate generation, hardware detection, and system configuration operations. For information about the broader development workflow, see [Contributing](https://deepwiki.com/tenstorrent/tt-topology/7.1-contributing).

The testing framework covers unit tests for topology algorithms, integration tests for hardware interfaces, and validation tests for configuration detection. It does not include performance benchmarking or stress testing, which are handled separately in the broader Tenstorrent ecosystem.

## Running Tests

### Test Execution Methods

Tests can be executed as standalone Python modules using the standard module execution pattern:

`# Run individual test modulespython -m tt_topology.tests.test_detect_current_configspython -m tt_topology.tests.test_new_meshpython -m tt_topology.tests.test_galaxy`
Each test module includes a `main()` function that can be executed directly through the `if __name__ == "__main__":` pattern implemented in all test files.

### Configuration Pattern Analysis

The test validates different topology configurations by analyzing coordinate patterns extracted from hardware:

*   **Unflashed Configuration**: All coordinates are `(0,0)` or `(0,1)`
*   **Linear/Torus Configuration**: x-coordinate is 0, y-coordinates are sequential
*   **Mesh Configuration**: Coordinates form a rectangular grid pattern

Pattern detection logic is implemented in [test_detect_current_configs.py 33-63](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/test_detect_current_configs.py#L33-L63) with specific conditional checks for each configuration type.
