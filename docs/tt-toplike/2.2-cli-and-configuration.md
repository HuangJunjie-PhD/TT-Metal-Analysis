# CLI and Configuration

Relevant source files
*   [src/cli.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs)
*   [src/config.rs](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs)

This page provides a technical reference for the command-line interface and the persistent configuration system used by `tt-toplike`. The application uses a multi-layered configuration approach: CLI arguments provide immediate execution control, while a TOML-based configuration file allows for persistent tuning of the animation engine's sensitivity and density.

## Command-Line Interface

The CLI is implemented using the `clap` crate in `src/cli.rs`. It defines the `Cli` struct which governs backend selection, telemetry polling rates, device filtering, and initial UI state [src/cli.rs 37-67](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L37-L67)

### Backend Selection

The `--backend` (or `-b`) flag determines which data source `tt-toplike` uses to retrieve hardware metrics [src/cli.rs 69-70](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L69-L70)

| Backend Type | CLI Option | Description |
| --- | --- | --- |
| **Auto** | `auto` | Default. Safely attempts `Sysfs` ->`Json` ->`Mock`. It never tries `Luwen` automatically [src/cli.rs 171-173](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L171-L173) |
| **Hybrid** | `hybrid` | Recommended for Linux. Uses `sysfs` for real-time core metrics and background `tt-smi` polls for SMBUS/DDR status [src/cli.rs 189-196](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L189-L196) |
| **Sysfs** | `sysfs` | Non-invasive Linux `hwmon` sensor reading [src/cli.rs 185-187](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L185-L187) |
| **JSON** | `json` | Spawns `tt-smi -s` as a subprocess and parses JSON output [src/cli.rs 178-179](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L178-L179) |
| **Luwen** | `luwen` | Direct PCI BAR0 access. Requires explicit selection due to invasiveness [src/cli.rs 181-184](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L181-L184) |
| **Mock** | `mock` | Simulated telemetry for development/CI [src/cli.rs 175-176](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L175-L176) |

### Key Arguments and Flags

*   **`--interval`, `-i`**: Polling interval in milliseconds (10ms to 1000ms). Controls the update frequency of the telemetry loop [src/cli.rs 89-95](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L89-L95)
*   **`--devices`, `-d`**: A comma-separated list of device indices (e.g., `--devices 0,2`) to monitor [src/cli.rs 97-102](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L97-L102)
*   **`--mock [N]`**: Shortcut to enable the mock backend. If a number is provided, it overrides `--mock-devices`[src/cli.rs 72-77](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L72-L77)
*   **`--mode`, `-m`**: Directly launches the TUI into a specific `VisualizationMode` (e.g., `starfield`, `castle`, `arcade`) [src/cli.rs 134-139](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L134-L139)
*   **`--profile`**: Selects an `AnimationProfile` to control visualization responsiveness [src/cli.rs 141-147](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/cli.rs#L141-L147)

## Animation Profiles

The `AnimationProfile` enum defines presets that scale the visual response to hardware activity. These profiles modify the `AnimConfig` struct, which is consumed by the visualization engine to determine particle counts and motion sensitivity [src/config.rs 14-25](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L14-L25)

| Profile | Sensitivity | Particle Scale | Use Case |
| --- | --- | --- | --- |
| `AnomaliesOnly` | 0.2 | 0.5 | Monitoring for extreme spikes only. |
| `Relaxed` | 0.5 | 0.75 | Subtle movement for light workloads. |
| `Normal` | 1.0 | 1.0 | Default balanced behavior [src/config.rs 27-29](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L27-L29) |
| `Hyper` | 2.0 | 2.0 | Vigorous animation for moderate activity. |
| `Paranoid` | 5.0 | 3.0 | Maximum sensitivity; visualizes every fluctuation. |

### Configuration Logic

The `AnimConfig` struct calculates specific limits for different visualizations based on the selected profile's `max_particles_scale`[src/config.rs 32-41](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L32-L41):

*   **Memory Castle**: Base 600 particles [src/config.rs 55-58](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L55-L58)
*   **Memory Flow**: Base 200 particles [src/config.rs 60-63](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L60-L63)
*   **Portrait**: Base 32 particles per device [src/config.rs 65-68](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L65-L68)

## Persistent Configuration (TOML)

`tt-toplike` supports a TOML override system located at `~/.config/tt-toplike/config.toml`. This allows users to fine-tune sensitivity and particle density beyond the standard CLI presets [src/config.rs 88-97](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L88-L97)

### TOML Schema

The configuration file uses the `AnimConfigOverrides` struct [src/config.rs 80-84](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L80-L84) All fields are optional.

`# ~/.config/tt-toplike/config.toml # Multiplier for AdaptiveBaseline::power_change()# 1.0 is default, 5.0 is extremely sensitivesensitivity = 1.5 # Scale factor for the number of particles rendered# 1.0 is default, 3.0 creates very dense visualizationsmax_particles_scale = 2.0`
### Configuration Merging

The system first loads the profile specified by the `--profile` CLI flag (defaulting to `Normal`), then merges any values found in the TOML file using the `AnimConfig::merge` method [src/config.rs 71-75](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/config.rs#L71-L75)
