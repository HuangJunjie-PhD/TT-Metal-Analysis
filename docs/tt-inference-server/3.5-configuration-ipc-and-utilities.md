This page describes the foundational infrastructure of the `tt-media-server`, including the centralized configuration system, Inter-Process Communication (IPC) mechanisms, hardware device management, and media processing utilities.

## Configuration System

The server utilizes a centralized settings management system based on environment variables and constants, primarily managed via `Pydantic`. The configuration is split into functional areas: model identification, runner selection, and service mapping.

### Key Constants and Enums

The system uses several Enums to maintain type safety across the codebase:

*   **`SupportedModels`**: Maps human-readable model names to their HuggingFace repository IDs [tt-media-server/config/constants.py 8-40](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/constants.py#L8-L40)
*   **`ModelNames`**: Internal unique identifiers for models used in environment variables and registry lookups [tt-media-server/config/constants.py 45-84](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/constants.py#L45-L84)
*   **`ModelRunners`**: Defines the backend implementation used to execute a model (e.g., `TT_SDXL_TRACE`, `VLLMForge`, `TT_WHISPER`, `LORA_SINGLE_CHIP`) [tt-media-server/config/constants.py 87-126](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/constants.py#L87-L126)
*   **`ModelServices`**: Categorizes models into functional domains like `IMAGE`, `LLM`, `AUDIO`, `VIDEO`, `CNN`, `TRAINING`, `TEXT_TO_SPEECH`, or `EMBEDDING`[tt-media-server/config/constants.py 128-137](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/constants.py#L128-L137)

### Service and Runner Mapping

The `MODEL_SERVICE_RUNNER_MAP` defines which runners are valid for specific API services [tt-media-server/config/constants.py 139-193](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/constants.py#L139-L193) For example, the `ModelServices.IMAGE` service can route to `TT_SDXL_TRACE`, `TT_SD3_5`, or `TT_FLUX_1_DEV`[tt-media-server/config/constants.py 140-152](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/constants.py#L140-L152)

### Runner Environment Setup

The `setup_runner_environment` function is the primary entry point for configuring a process before it interacts with Tenstorrent hardware [tt-media-server/utils/runner_utils.py 24-73](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/runner_utils.py#L24-L73) It performs the following:

1.   **CPU Threading**: Sets `OMP_NUM_THREADS` and `TORCH_NUM_THREADS` via `setup_cpu_threading_limits` to prevent CPU oversubscription [tt-media-server/utils/runner_utils.py 75-83](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/runner_utils.py#L75-L83)
2.   **Device Visibility**: Sets the `TT_VISIBLE_DEVICES` environment variable based on the assigned `device_id`[tt-media-server/utils/runner_utils.py 37-39](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/runner_utils.py#L37-L39)
3.   **JIT Cache**: Configures `TT_METAL_CACHE` with a unique path per device and MPI rank to avoid compilation conflicts [tt-media-server/utils/runner_utils.py 44-50](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/runner_utils.py#L44-L50)
4.   **Mesh Configuration**: Applies hardware-specific mesh graph descriptors for Blackhole (`_setup_blackhole_mesh_config`) or Galaxy (`_setup_galaxy_mesh_config`) systems [tt-media-server/utils/runner_utils.py 85-110](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/runner_utils.py#L85-L110)

## Device and Runner Management

The server uses a factory pattern to instantiate runners based on the current configuration.

### Runner Fabric

The `runner_fabric.py` module contains the `AVAILABLE_RUNNERS` registry, which uses lazy imports to instantiate concrete `BaseDeviceRunner` implementations [tt-media-server/tt_model_runners/runner_fabric.py 10-133](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/tt_model_runners/runner_fabric.py#L10-L133) The `get_device_runner` function resolves the `ModelRunners` enum from the `settings` and returns the appropriate instance [tt-media-server/tt_model_runners/runner_fabric.py 136-151](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/tt_model_runners/runner_fabric.py#L136-L151)

## Media Utilities

The server provides specialized managers for handling media byte-streams and file conversions.

### Video Manager

The `VideoManager` class handles the export of raw RGB frames into MP4 files using FFmpeg subprocess pipes [tt-media-server/utils/video_manager.py 29-32](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/video_manager.py#L29-L32)

*   **Frame Normalization**: Converts input arrays to `uint8` and ensures correct dimensions (N, H, W, C) [tt-media-server/utils/video_manager.py 199-207](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/video_manager.py#L199-L207)
*   **FFmpeg Integration**: Builds complex command strings to pipe raw video data into `libx264` with specific presets like `ultrafast` and `faststart` flags [tt-media-server/utils/video_manager.py 70-121](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/video_manager.py#L70-L121)
*   **Channel Handling**: Automatically expands grayscale (1 channel) to RGB (3 channels) or strips alpha channels (4 channels) [tt-media-server/utils/video_manager.py 210-222](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/video_manager.py#L210-L222)

### Image and Audio Managers

*   **Image Manager**: Used by runners like `TTYolov4Runner`, handling image loading and resizing [tt-media-server/tt_model_runners/yolov4_runner.py 68-70](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/tt_model_runners/yolov4_runner.py#L68-L70)
*   **Audio Processing**: Managed via `Settings`, which controls `max_audio_duration_seconds` (60s default) and `default_sample_rate` (16000Hz) [tt-media-server/config/settings.py 94-105](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/settings.py#L94-L105)

## Telemetry and Monitoring

The server integrates with Prometheus for real-time monitoring of inference performance and system health across both Python and C++ backends.

### Python Telemetry

The `telemetry_client` is initialized during runner setup if `enable_telemetry` is true [tt-media-server/utils/runner_utils.py 41-42](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/utils/runner_utils.py#L41-L42)

### C++ Server Metrics

The C++ backend implements a `ServerMetrics` singleton [tt-media-server/cpp_server/src/metrics/metrics.cpp 15-18](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/metrics/metrics.cpp#L15-L18) It uses a dedicated background thread to process events asynchronously from the critical inference path [tt-media-server/cpp_server/include/metrics/metrics.hpp 27-40](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/metrics/metrics.hpp#L27-L40)

*   **Metrics Tracked**: Prompt tokens, generation tokens, TTFT (Time to First Token), ITL (Inter-Token Latency), and queue depth [tt-media-server/cpp_server/src/metrics/metrics.cpp 43-174](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/metrics/metrics.cpp#L43-L174)

## Environment and Dependency Isolation

The codebase manages complex dependencies through specialized Dockerfiles and setup scripts.

### Docker Environments

*   **Standard Build**: The primary `Dockerfile` builds `tt-metal` from source, installs the Python inference server requirements, and builds the C++ backend (`cpp_server`) using Clang-20 [tt-media-server/Dockerfile 11-168](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/Dockerfile#L11-L168)
*   **TT-VLLM Plugin**: Integrated into the main image by copying and installing the `tt-vllm-plugin` directory [tt-media-server/Dockerfile 148-152](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/Dockerfile#L148-L152)

### Inter-Process Communication (IPC)

The server supports multiple queue implementations for multiprocessing via `QueueType`[tt-media-server/config/settings.py 67-68](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/config/settings.py#L67-L68)

*   **`TTQueue`**: The default implementation for high-performance IPC between the scheduler and device workers.
*   **`MemoryQueue`**: Used for localized shared memory transfers.
