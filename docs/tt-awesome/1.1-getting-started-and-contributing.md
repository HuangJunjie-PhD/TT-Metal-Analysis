This page details the mechanisms for contributing new projects and resources to the Awesome Tenstorrent ecosystem. It covers the technical implementation of the contribution pipelines, the data schema requirements, and the affiliation policies that govern how entries are displayed.

## Contribution Paths

There are three distinct technical paths for adding an entry to the `entries/` directory. All paths eventually result in a validated JSON file stored in a category-specific subdirectory [CONTRIBUTING.md 5-28](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L5-L28)

### 1. GitHub Issue Form (Automated)

This is the recommended path for most contributors. It uses a structured GitHub Issue Template to collect data without requiring the user to touch the codebase [CONTRIBUTING.md 7-11](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L7-L11)

**Data Flow:**

1.   **Submission**: User fills out the `submit-entry.yml` form [.github/ISSUE_TEMPLATE/submit-entry.yml 1-131](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.github/ISSUE_TEMPLATE/submit-entry.yml#L1-L131)
2.   **Approval**: A maintainer reviews the issue and adds the `approved` label [.github/workflows/entry-from-issue.yml 18-20](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.github/workflows/entry-from-issue.yml#L18-L20)
3.   **Automation**: The `entry-from-issue.yml` workflow triggers, executing `scripts/issue_to_entry.py` to parse the Markdown body [.github/workflows/entry-from-issue.yml 34-39](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.github/workflows/entry-from-issue.yml#L34-L39)
4.   **PR Creation**: The script generates the JSON, validates it, and the workflow opens a Pull Request automatically [.github/workflows/entry-from-issue.yml 50-76](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.github/workflows/entry-from-issue.yml#L50-L76)

### 2. Interactive CLI

For contributors with a local clone of the repository, the `scripts/add_entry.py` script provides a guided experience [scripts/add_entry.py 5-10](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L5-L10)

**Implementation Details:**

*   **Validation**: It uses a `while True` loop with the `ask()` function to ensure required fields are not empty [scripts/add_entry.py 26-37](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L26-L37)
*   **Selection**: The `pick()` function provides numbered menus for enums like `AFFILIATIONS`, `CATEGORIES`, and `LINK_TYPES`[scripts/add_entry.py 39-54](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L39-L54)
*   **Slugification**: It automatically generates a kebab-case `id` from the project name using `slugify()`[scripts/add_entry.py 77-78](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L77-L78)
*   **Persistence**: It determines the output path based on the primary category and writes the formatted JSON [scripts/add_entry.py 119-126](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L119-L126)

### 3. Direct JSON Manual Entry

Advanced contributors can manually create a JSON file following the schema in the `entries/{primary-category}/` directory [CONTRIBUTING.md 21-27](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L21-L27)

## Affiliation Policy

The directory uses an explicit `affiliation` field to categorize the relationship between the project and Tenstorrent.

| Value | Criteria |
| --- | --- |
| `community` | Projects built by individuals or organizations not employed by Tenstorrent [CONTRIBUTING.md 56](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L56-L56) |
| `affiliated` | Personal projects by Tenstorrent employees (hosted on personal GitHub accounts) [CONTRIBUTING.md 57](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L57-L57) |
| `official` | Projects residing within the `tenstorrent` GitHub organization [CONTRIBUTING.md 58](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L58-L58) |

**Sorting Logic:** To prioritize community growth, **community entries are rendered first** in all generated views, followed by affiliated, and then official entries [CONTRIBUTING.md 60](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L60-L60)

## Entry JSON Schema

All entries must conform to the following schema to pass the `scripts/validate.py` check [CONTRIBUTING.md 79-81](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L79-L81)

### Field Definitions

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | Kebab-case slug; must match the filename [CONTRIBUTING.md 35](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L35-L35) |
| `name` | string | Yes | The display name of the project [CONTRIBUTING.md 36](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L36-L36) |
| `description` | string | Yes | 1–3 sentences of plain text [CONTRIBUTING.md 37](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L37-L37) |
| `affiliation` | enum | Yes | `community`, `affiliated`, or `official`[CONTRIBUTING.md 38](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L38-L38) |
| `categories` | string[] | Yes | List of slugs. The first determines the subdirectory [CONTRIBUTING.md 39](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L39-L39) |
| `links` | object[] | Yes | Array of link objects containing `type` and `url`[CONTRIBUTING.md 41-43](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L41-L43) |
| `hardware` | string[] | No | Supported hardware: `grayskull`, `wormhole`, `blackhole`, etc [CONTRIBUTING.md 45](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L45-L45) |
| `featured` | boolean | No | If `true`, the entry is highlighted in its tier [CONTRIBUTING.md 46](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L46-L46) |

## Technical Data Flow

The contribution process is designed to ensure that the `entries/` directory remains the single source of truth.

### CLI Execution Logic (`add_entry.py`)

The script manages the lifecycle of a new entry from user input to file creation:

1.   **Collection**: Prompts for metadata using predefined constants like `AFFILIATIONS` and `CATEGORIES`[scripts/add_entry.py 15-20](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L15-L20)
2.   **Link Processing**: The `collect_links()` function ensures at least one repository link is provided and validates URL patterns via `URL_RE`[scripts/add_entry.py 56-74](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L56-L74)
3.   **Path Resolution**: The script calculates the destination using `ENTRIES_DIR / categories[0]`[scripts/add_entry.py 120](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L120-L120)
4.   **Collision Detection**: It checks if `dest.exists()` to prevent overwriting existing entries [scripts/add_entry.py 123-125](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/add_entry.py#L123-L125)
