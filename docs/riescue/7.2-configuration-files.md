This document provides a comprehensive reference for all configuration file formats used in the RiESCUE framework. Configuration files control simulator behavior, define memory layouts, specify CPU features, and configure the development environment.

For information about command-line arguments and test directives, see [Test Directives and Syntax](https://deepwiki.com/tenstorrent/riescue/2.1.2-test-directives-and-syntax). For details on simulator tool interfaces, see [Instruction Set Simulators](https://deepwiki.com/tenstorrent/riescue/4.2-instruction-set-simulators).

## Overview

RiESCUE uses JSON-formatted configuration files to control various aspects of test generation and simulation. These configuration files fall into three main categories:

| Configuration Type | Purpose | Primary Consumers |
| --- | --- | --- |
| CPU Configuration | Defines memory maps and hardware features for test generation | `FeatMgr`, `Generator`, `Parser` |
| Whisper Configuration | Configures the Whisper ISS simulator behavior and CSR state | `Whisper` tool class |
| Container Configuration | Specifies container registry and bind mount settings | `container.py` management scripts |

## Configuration File Types and Locations

### CPU Configuration Files

CPU configuration files are user-provided JSON files that define the memory map and feature settings for test generation. These files are passed via the `--cpuconfig` command-line argument to `riescued`.

**Location**: User-specified path (not included in repository)

**Usage**:

`riescued test.s --cpuconfig cpu_config.json`
**Key Sections**:

*   `memory_map`: Defines RAM regions with start addresses and sizes
*   `features`: Boolean flags for enabling/disabling ISA extensions

Sources: [riescue/dtest_framework/README.md 106-123](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/README.md?plain=1#L106-L123)

### Whisper Configuration Files

Whisper configuration files are JSON files that configure the Whisper instruction set simulator's behavior, CSR reset values, memory attributes, and performance counter mappings.

**Location**: [riescue/dtest_framework/lib/](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/)

Three variants are provided:

| File | Purpose | Key Differences |
| --- | --- | --- |
| `whisper_config.json` | Standard configuration | Default memory map, ACLINT at 0x042180000 |
| `whisper_basic_config.json` | Basic configuration | PMP support enabled, different pmacfg0 reset |
| `whisper_secure_config.json` | Secure mode configuration | STEE support, 52-bit memory, extensive PMA regions |

**Major Configuration Categories**:

*   ISA string and XLEN
*   Reset vector address
*   CSR existence, reset values, and write masks
*   Memory map and physical memory attributes (PMA)
*   Vector configuration
*   Performance counter mappings
*   ACLINT timer configuration

Sources: [riescue/dtest_framework/lib/whisper_config.json 1-350](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L1-L350)[riescue/dtest_framework/lib/whisper_basic_config.json 1-362](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json#L1-L362)[riescue/dtest_framework/lib/whisper_secure_config.json 1-395](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L1-L395)

### Container Configuration Files

Container configuration files control the container management system's behavior for pulling, pushing, and running Singularity containers.

**Location**: `.container_config` (user-created in repository root)

**Format**: JSON

**Usage**: Automatically loaded by `container.py` management scripts

## Common Configuration Patterns

### JSON Structure

All configuration files in RiESCUE follow standard JSON formatting with the following conventions:

1.   **Comments**: Unofficial `"Comment"` or `"<field>-comment"` keys are used for documentation
2.   **Hex Values**: Memory addresses and masks are represented as strings with `"0x"` prefix
3.   **Boolean Flags**: Features use JSON boolean `true`/`false` or string `"true"`/`"false"`
4.   **Nested Objects**: Complex configurations use nested JSON objects for logical grouping

Example pattern from Whisper configurations:

`{    "csr": {        "misa": {            "reset-comment": "imabfcvsu",            "reset": "0x403411af",            "mask-comment": "Misa is not writable by CSR instructions",            "mask": "0x0"        }    }}`
Sources: [riescue/dtest_framework/lib/whisper_config.json 22-28](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L22-L28)

### Address and Size Representation

Memory-related values consistently use hexadecimal string representation:

| Field Type | Format | Example |
| --- | --- | --- |
| Addresses | Hex string | `"0x80000000"` |
| Sizes | Hex string | `"0x10000000"` |
| Masks | Hex string | `"0xffffffffffffffff"` |

Sources: [riescue/dtest_framework/README.md 112-117](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/README.md?plain=1#L112-L117)

## Whisper Configuration Deep Dive

### ISA String Format

The `isa` field specifies the RISC-V ISA string with all supported extensions:

`{    "xlen": 64,    "isa": "rv64imafdchvus_svinval_zba_zbb_zbc_zbs_zfa_zfh_zfbfmin_zvfbfmin_zvfbfwma_zvkg_zvkned_zvknhb_zvfh_zvbb_zvbc_zbkb_zbkx_zknd_zkne_zksed_zksh_zicbom_zicboz_zicond_zihintpause_zifencei_zimop_zcmop_zawrs_zicntr_zihpm_zvqdot_zjpm_svpbmt_svadu_sstc"}`
**Components**:

*   Base ISA: `rv64imafdchvus`
*   Supervisor extensions: `svinval`, `svpbmt`, `svadu`, `sstc`
*   Zicsr extensions: `zba`, `zbb`, `zbc`, `zbs`, `zfa`, `zfh`, etc.
*   Vector extensions: `zvfbfmin`, `zvfbfwma`, `zvkg`, etc.
*   Security extensions: `zbkb`, `zbkx`, `zknd`, `zkne`, `zksed`, `zksh`

Sources: [riescue/dtest_framework/lib/whisper_config.json 2-3](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L2-L3)[riescue/dtest_framework/lib/whisper_basic_config.json 2-3](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json#L2-L3)[riescue/dtest_framework/lib/whisper_secure_config.json 2-3](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L2-L3)

### CSR Configuration Structure

Each CSR can be configured with the following fields:

| Field | Type | Purpose |
| --- | --- | --- |
| `exists` | Boolean | Whether the CSR is implemented |
| `number` | String (hex) | CSR number/address |
| `reset` | String (hex) | Reset value on processor reset |
| `mask` | String (hex) | Writeable bits mask |
| `poke_mask` | String (hex) | Mask for direct memory writes |

Example from [riescue/dtest_framework/lib/whisper_config.json 205-210](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L205-L210):

`{    "pmacfg0": {        "number": "0x7e0",        "reset": "0x0",        "mask": "0xffffffffffffffff",        "poke_mask": "0xffffffffffffffff"    }}`
**Hardwired CSRs**: Many CSRs are configured as read-only (mask = `"0x0"`) or non-existent:

`{    "tselect": {        "Comment": "Hardwired to 0",        "reset": "0x0",        "mask": "0x0"    }}`
Sources: [riescue/dtest_framework/lib/whisper_config.json 69-91](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L69-L91)

### Memory Map Configuration

Whisper's memory map is defined in two parts:

1.   **Global Memory Size**:

`{    "memmap": {        "Comment": "Set physical memory size to be 56-bits",        "size": "0x00fffffffffff000"    }}`
1.   **Physical Memory Attributes (PMA)** (only in secure config):

`{    "memmap": {        "size": "0x10000000000000",        "pma": [            {                "low": "0x30000000",                "high": "0x30000010",                "attribs": ["read", "write", "io", "mem_mapped"],                "masks": [["0x30000000", "0x30000010", "0xff00"]],                "register_size": 8            }        ]    }}`
**PMA Attributes**:

*   `read`, `write`: Access permissions
*   `io`: I/O region (affects caching)
*   `mem_mapped`: Memory-mapped region
*   `misal_acc_fault`: Generate fault on misaligned access

Sources: [riescue/dtest_framework/lib/whisper_config.json 17-20](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L17-L20)[riescue/dtest_framework/lib/whisper_secure_config.json 22-63](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L22-L63)

### Vector Configuration

Vector extension parameters are specified in the `vector` object:

`{    "vector": {        "bytes_per_vec": 32,        "min_bytes_per_elem": 1,        "Comment": "Hardware supports 32-bit ELEN plus some widening operations from 32 to 64-bit",        "max_bytes_per_elem": 8,        "mask_agnostic_policy": "ones",        "tail_agnostic_policy": "ones",        "update_whole_mask": true,        "trap_invalid_vtype": false,        "trap_non_zero_vstart": true,        "legalize_vsetvl_avl": true,        "legalize_vsetvli_avl": true,        "legalize_for_egs": true,        "max_bytes_per_lmul": {            "mf8": 1,            "mf4": 2,            "mf2": 4        },        "tt_fp_usum_tree_reduction": ["e16", "e32", "e64"],        "fp_usum_nan_canonicalize": ["e16", "e32", "e64"],        "partial_segment_update": false    }}`
**Key Parameters**:

*   `bytes_per_vec`: VLEN/8 (vector register length in bytes)
*   `max_bytes_per_elem`: ELEN/8 (maximum element size)
*   `mask_agnostic_policy`, `tail_agnostic_policy`: Agnostic element fill value
*   `trap_invalid_vtype`: Whether to trap on invalid vtype
*   `max_bytes_per_lmul`: LMUL fractional configuration limits

Sources: [riescue/dtest_framework/lib/whisper_config.json 332-349](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L332-L349)

### ACLINT Timer Configuration

The ACLINT (Advanced Core Local Interruptor) timer is configured with:

`{    "auto_increment_timer": true,    "time_down_sample": 2,    "aclint": {        "base": "0x042180000",        "size": "0xc000",        "timer_offset": "0x8000",        "time_offset": "0x0",        "deliver_interrupts": true,        "time_adjust": 0,        "timecmp_reset": "0xffffffff"    }}`
**Fields**:

*   `base`: Base address of ACLINT region
*   `timer_offset`: Offset to timer registers
*   `deliver_interrupts`: Whether to deliver timer interrupts
*   `timecmp_reset`: Initial value of timecmp register

Sources: [riescue/dtest_framework/lib/whisper_config.json 305-315](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L305-L315)

### Performance Counter Configuration

Performance counters are mapped to event types:

`{    "enable_performance_counters": "true",    "perf_count_atomic_load_store": "true",    "perf_count_fp_load_store": "true",    "enable_counter_overflow": "true",    "num_mmode_perf_regs": 8,    "mmode_perf_event_map": {        "None": "0x0",        "Mult": "0x00000e00",        "Div": "0x00000f00",        "Csr": "0x00001000",        "MultDiv": "0x00003000",        "Bitmanip": "0x00005000",        "FpSingle": "0x00006000",        "FpHalf": "0x00007000",        "InstCommited": "0x00800000",        "Load": "0x00c00000",        "Store": "0x00d00000"    }}`
Sources: [riescue/dtest_framework/lib/whisper_config.json 5-330](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L5-L330)

## Configuration Variants and Use Cases

| Configuration | Use Case | Memory Size | PMP Support | STEE Support |
| --- | --- | --- | --- | --- |
| `whisper_config.json` | Standard testing | 56-bit | Disabled | No |
| `whisper_basic_config.json` | Basic CPU model | 56-bit | Enabled (16 entries) | No |
| `whisper_secure_config.json` | Security testing | 52-bit | Enabled (64 entries) | Yes |

### Key Differences

**Basic vs. Standard**:

*   Basic enables PMP (Physical Memory Protection) with 16 pmpaddr registers
*   Basic has different pmacfg0 reset value
*   Basic disables `cycle` and `instret` CSRs

**Secure vs. Standard**:

*   Secure enables STEE (Secure Trusted Execution Environment)
*   Secure has 52-bit memory instead of 56-bit
*   Secure includes extensive PMA regions for I/O and memory-mapped devices
*   Secure enables all 64 pmpaddr registers
*   Secure has different default pmacfg configurations

Sources: [riescue/dtest_framework/lib/whisper_config.json 133-204](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_config.json#L133-L204)[riescue/dtest_framework/lib/whisper_basic_config.json 141-217](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_basic_config.json#L141-L217)[riescue/dtest_framework/lib/whisper_secure_config.json 17-248](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/lib/whisper_secure_config.json#L17-L248)

## Configuration Validation

Configuration files are validated at runtime when loaded by their respective consumers:

1.   **JSON Syntax**: Must be valid JSON format
2.   **Required Fields**: ISA string, xlen, and reset_vec are mandatory in Whisper configs
3.   **Value Ranges**: Addresses and masks must be valid hex strings
4.   **Feature Consistency**: Enabled features must be supported by the ISA string

Invalid configurations result in runtime errors during test generation or simulation execution.

## Best Practices

1.   **Version Control**: Keep CPU configuration files under version control alongside tests
2.   **Documentation**: Use `"Comment"` fields to document non-obvious configuration choices
3.   **Consistency**: Use consistent hex formatting (`"0x"` prefix, lowercase hex digits)
4.   **Validation**: Test configuration changes with known-good test cases before committing
5.   **Minimal Changes**: When creating new Whisper configs, start from an existing variant and modify only necessary fields
