# Tool Subclasses

Relevant source files
*   [riescue/lib/toolchain/tool.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py)

## Purpose and Scope

This page documents the concrete implementations of the `Tool` base class that abstract external RISC-V toolchain executables. Each subclass implements executable discovery, subprocess execution, and tool-specific error handling for a particular external tool. For the base class architecture including executable discovery and subprocess management, see [Tool Base Class](https://deepwiki.com/tenstorrent/riescue/7.3.1-tool-base-class). For integration of these tools into the broader toolchain system, see [Toolchain Abstraction Layer](https://deepwiki.com/tenstorrent/riescue/7.3-toolchain-abstraction-layer).

The four tool subclasses documented here are:

*   `Compiler` - wraps `riscv64-unknown-elf-gcc`
*   `Disassembler` - wraps `riscv64-unknown-elf-objdump`
*   `Objcopy` - wraps `riscv64-unknown-elf-objcopy`
*   `Spike` - wraps `spike` or `tt_spike` ISS

Sources: [riescue/lib/toolchain/tool.py 1-335](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L1-L335)

## Compiler Tool

The `Compiler` class wraps the RISC-V GCC compiler (`riscv64-unknown-elf-gcc`) with RiESCUE-specific defaults and configuration options.

### Constructor Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `compiler_path` | `Optional[Path]` | `None` | Explicit path to compiler executable |
| `compiler_opts` | `Optional[list[str]]` | `[]` | Additional GCC flags |
| `compiler_march` | `str` | `default_compiler_march` | Architecture string for `-march=` |
| `test_equates` | `Optional[list[str]]` | `[]` | Preprocessor defines in format `KEY=VALUE` |
| `abi` | `Optional[str]` | `None` | ABI for `-mabi=` flag |

Sources: [riescue/lib/toolchain/tool.py 135-166](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L135-L166)

### Default Architecture String

The `Compiler` class defines a comprehensive default `march` string that includes common RISC-V extensions:

This default is used when no explicit `compiler_march` is provided to the constructor.

Sources: [riescue/lib/toolchain/tool.py 133](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L133-L133)

### Standard Compiler Arguments

The `Compiler` constructor assembles the following standard GCC arguments:

| Flag | Purpose |
| --- | --- |
| `-static` | Produce statically linked executable |
| `-mcmodel=medany` | Use medium any code model for addressing |
| `-fvisibility=hidden` | Hide symbols by default |
| `-nostdlib` | Do not link standard library |
| `-nostartfiles` | Do not use standard startup files |
| `-march={compiler_march}` | Specify target architecture |
| `-mabi={abi}` | Specify ABI (only if `abi` parameter provided) |
| `-D{key}={value}` | Define preprocessor macros (for each `test_equates` entry) |

These are combined with any user-provided `compiler_opts`.

Sources: [riescue/lib/toolchain/tool.py 151-165](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L151-L165)

### Command-Line Integration

The `Compiler` class provides static methods for integration with `argparse`:

**Compiler Command-Line Integration**: The `add_arguments()` static method registers compiler-related arguments with an `ArgumentParser`, and `from_clargs()` constructs a `Compiler` instance from parsed arguments. The `march_override` parameter allows overriding the command-line march value.

Sources: [riescue/lib/toolchain/tool.py 169-190](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L169-L190)

### Error Classification

The `Compiler._classify()` method implements tool-specific error handling:

**Compiler Error Classification Logic**: Non-zero return codes are classified as `COMPILE_FAILURE` errors. The method extracts error text from either the output file or stderr, providing diagnostic information in the raised `ToolchainError`.

Sources: [riescue/lib/toolchain/tool.py 192-203](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L192-L203)

## Disassembler Tool

The `Disassembler` class wraps `riscv64-unknown-elf-objdump` for generating disassembly listings from ELF files.

### Constructor and Configuration

The constructor is simpler than `Compiler` because disassembly typically requires fewer configuration options. It searches for the executable using:

*   Provided `disassembler_path`
*   `RV_OBJDUMP` environment variable
*   `riscv64-unknown-elf-objdump` in PATH

Sources: [riescue/lib/toolchain/tool.py 207-214](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L207-L214)

### Command-Line Arguments

| Argument | Type | Default | Description |
| --- | --- | --- | --- |
| `--disassembler_path` | `Path` | `None` | Explicit path to objdump executable |
| `--disassembler_opts` | `list[str]` | `[]` | Additional objdump flags (currently ignored with warning) |

**Note**: The current implementation ignores `--disassembler_opts` and logs a warning. This is a known limitation documented in the code for backwards compatibility with legacy flows.

Sources: [riescue/lib/toolchain/tool.py 217-230](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L217-L230)

### Error Classification

**Disassembler Error Classification Logic**: Non-zero return codes are classified as `COMPILE_FAILURE` (reusing the same error type). The method attempts to extract diagnostic information from stderr first, then falls back to reading the output file or reporting no output.

Sources: [riescue/lib/toolchain/tool.py 232-242](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L232-L242)

## Objcopy Tool

The `Objcopy` class wraps `riscv64-unknown-elf-objcopy` for manipulating object file formats.

### Constructor

Similar to `Disassembler`, the constructor is straightforward. It searches for the executable using:

*   Provided `objcopy_path`
*   `RV_OBJCOPY` environment variable
*   `riscv64-unknown-elf-objcopy` in PATH

The `objcopy_opts` are passed directly to the base class constructor as the `args` parameter.

Sources: [riescue/lib/toolchain/tool.py 246-253](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L246-L253)

### Command-Line Arguments

| Argument | Type | Default | Description |
| --- | --- | --- | --- |
| `--objcopy_path` | `Path` | `None` | Explicit path to objcopy executable |
| `--objcopy_opts` | `list[str]` | `[]` | Additional objcopy flags |

Sources: [riescue/lib/toolchain/tool.py 256-261](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L256-L261)

The `Objcopy._classify()` method is the simplest of all tool subclasses:

## Spike Tool

The `Spike` class wraps both the standard `spike` ISS and Tenstorrent's custom `tt_spike` variant. This is the most complex tool subclass due to ISS-specific configuration.

### Constructor Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `spike_path` | `Optional[Path]` | `None` | Explicit path to spike executable |
| `spike_args` | `Optional[list[str]]` | `[]` | Additional spike flags |
| `spike_isa` | `str` | `""` | ISA string override (uses default if empty) |
| `third_party_spike` | `bool` | `False` | If True, use standard spike; if False, use tt_spike |
| `spike_max_instr` | `int` | `2000000` | Maximum instructions to simulate (tt_spike only) |

Sources: [riescue/lib/toolchain/tool.py 274-302](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L274-L302)

### ISA String Defaults

The `Spike` constructor provides different default ISA strings depending on whether standard spike or tt_spike is used:

**Spike ISA String Selection Logic**: The constructor selects an ISA string based on whether a custom value is provided, and which spike variant is being used. The tt_spike default includes additional extensions (`zvknhb`, `sstc`, `zicntr`) not present in the third-party spike default.

Sources: [riescue/lib/toolchain/tool.py 288-294](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L288-L294)

### Vector Architecture Configuration

For `tt_spike` (when `third_party_spike=False`), the constructor sets vector architecture parameters:

This is passed to spike via the `--varch=vlen:256,elen:64` command-line argument. Standard spike does not receive this configuration.

Sources: [riescue/lib/toolchain/tool.py 297-300](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L297-L300)

### Standard Spike Arguments

The constructor assembles these standard arguments for all spike variants:

| Flag | Description |
| --- | --- |
| `-l` | Enable logging |
| `--log-commits` | Log committed instructions |
| `--max-instrs={spike_max_instr}` | Maximum instructions (tt_spike only) |

Sources: [riescue/lib/toolchain/tool.py 285-298](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L285-L298)

### run_iss Method

Unlike the other tool subclasses, `Spike` overrides the `run()` method to raise `NotImplementedError` and provides a specialized `run_iss()` method instead:

This method:

1.   Accepts an `elf_file` parameter as the program to simulate
2.   Appends `--isa={spike_isa}` to arguments
3.   Appends `--varch=vlen:{vlen},elen:{elen}` if `varch` is set
4.   Appends the ELF file path
5.   Calls the base class `run()` method with assembled arguments

Sources: [riescue/lib/toolchain/tool.py 321-334](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L321-L334)

### Spike-Specific Execution Flow

**Spike Execution Flow**: The specialized `run_iss()` method assembles spike-specific arguments before delegating to the base class `run()` method. Direct calls to `run()` are prevented to ensure proper argument construction.

Sources: [riescue/lib/toolchain/tool.py 321-334](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/lib/toolchain/tool.py#L321-L334)

### Error Classification

The `Spike._classify()` method is the simplest:

Unlike compilation tools, spike failures are classified as `NONZERO_EXIT` rather than `COMPILE_FAILURE`, reflecting that simulation errors have different semantics than compilation errors.

## Tool Subclass Comparison

| Feature | Compiler | Disassembler | Objcopy | Spike |
| --- | --- | --- | --- | --- |
| **Executable** | `riscv64-unknown-elf-gcc` | `riscv64-unknown-elf-objdump` | `riscv64-unknown-elf-objcopy` | `spike` or `tt_spike` |
| **Environment Variable** | `RV_GCC` | `RV_OBJDUMP` | `RV_OBJCOPY` | `SPIKE_PATH` |
| **Default Args** | `-static`, `-mcmodel=medany`, `-march`, etc. | None | User-provided only | `-l`, `--log-commits`, `--max-instrs` |

| **Error Type** | `COMPILE_FAILURE` | `COMPILE_FAILURE` | `COMPILE_FAILURE` | `NONZERO_EXIT` |
| **Special Run Method** | No | No | No | Yes (`run_iss()`) |
| **Configuration Complexity** | High | Low | Low | Medium |
| **march_override Support** | Yes (`from_clargs`) | No | No | No |

## Argument Parser Integration Pattern

All tool subclasses follow a consistent pattern for command-line integration:

**Tool Argument Parser Integration Pattern**: Each tool provides two static/class methods that enable declarative argument registration and factory-based construction. This pattern is used throughout RiESCUE's command-line interfaces to consistently configure toolchain components.

Each tool subclass implements:

1.   **`add_arguments(parser)`** - Registers tool-specific arguments with an `ArgumentParser`
2.   **`from_clargs(args)`** - Constructs a tool instance from parsed command-line arguments

This pattern enables:

*   Consistent command-line interfaces across RiescueD, RiescueC, and CTK
*   Centralized documentation of tool configuration via argparse help
*   Easy extension of tool configuration without modifying client code
