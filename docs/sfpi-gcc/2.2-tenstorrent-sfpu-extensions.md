## Purpose and Scope

This document covers the Tenstorrent-specific SFPU (Special Function Processing Unit) extensions in sfpi-gcc, which provide compiler support for Tenstorrent's custom SFPU instruction sets on RISC-V architecture. These extensions target Tenstorrent's Wormhole and Blackhole architectures with specialized vector operations, mathematical functions, and memory access patterns optimized for AI/ML workloads.

For information about the base RISC-V SFPU core infrastructure, see [RISC-V SFPU Core](https://deepwiki.com/tenstorrent/sfpi-gcc/2.1-risc-v-sfpu-core). For broader GCC infrastructure topics, see [GCC Infrastructure](https://deepwiki.com/tenstorrent/sfpi-gcc/3-gcc-infrastructure).

## Instruction Set and Builtin System

### Instruction Definition Framework

The instruction system is built around a macro-based framework that defines instructions consistently across architectures while allowing for architecture-specific variations. The core instruction data structure `rvtt_insn_data` contains metadata for each instruction:

| Field | Purpose |
| --- | --- |
| `id` | Unique instruction identifier |
| `name` | Instruction name string |
| `decl` | GCC function declaration |
| `flags` | Instruction properties (CC setting, live operands, etc.) |
| `dst_arg_pos` | Destination argument position for src-as-dst instructions |
| `mod_pos` | Modifier argument position |
| `schedule` | Scheduling requirements |
| `nonimm_pos` | Non-immediate operand position |

## Machine Descriptions

### Wormhole Machine Description

The Wormhole machine description [gcc/config/riscv/tt/rvtt-wh.md](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt-wh.md?plain=1) defines over 120 instruction patterns using GCC's machine description language. Key patterns include:

#### Load/Store Operations

*   `rvtt_wh_sfpload` / `rvtt_wh_sfpload_lv`: Vector load operations with live value support
*   `rvtt_wh_sfpstore`: Vector store operations
*   `rvtt_wh_sfpxloadi` / `rvtt_wh_sfploadi_int`: Immediate load operations

#### Arithmetic Operations

*   `rvtt_wh_sfpmul` / `rvtt_wh_sfpadd`: Basic arithmetic with modifier support
*   `rvtt_wh_sfpmad`: Multiply-add operations
*   `rvtt_wh_sfpdivp2`: Division by power of 2

Sources: [gcc/config/riscv/tt/rvtt-wh.md 148-262](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt-wh.md?plain=1#L148-L262)[gcc/config/riscv/tt/rvtt-wh.md 424-436](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt-wh.md?plain=1#L424-L436)

### Condition Code Integration

SFPU instructions can set condition codes for subsequent conditional operations. The system tracks CC-setting instructions and optimizes their usage:

Sources: [gcc/config/riscv/tt/rvtt.cc 418-453](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt.cc#L418-L453)[gcc/config/riscv/tt/rvtt.h 171](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt.h#L171-L171)

### Register Allocation and Constraints

The system defines specialized register constraints for SFPU registers:

| Constraint | Purpose |
| --- | --- |
| `xr` | Any SFPU register |
| `x0`-`x7` | Specific SFPU registers L0-L7 |
| `xn` | Constant vector (unused inputs) |

Register allocation uses the `rvtt_sfpu_regno()` macro to map from GCC register numbers to SFPU register indices.

Sources: [gcc/config/riscv/tt/rvtt-constraints.md 23-49](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt-constraints.md?plain=1#L23-L49)[gcc/config/riscv/tt/rvtt-protos.h 26](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/config/riscv/tt/rvtt-protos.h#L26-L26)
