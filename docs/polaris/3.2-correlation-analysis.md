# Correlation Analysis

Relevant source files
*   [tools/run_metal_tensix_correlation.py](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/tools/run_metal_tensix_correlation.py)

## Purpose and Scope

The Correlation Analysis system provides automated benchmarking and validation capabilities for the Polaris simulation framework. It compares simulation results against reference performance data from Metal/Tensix hardware measurements to assess simulation accuracy. The system processes reference benchmark data, generates corresponding simulation workloads, executes simulations, and produces detailed correlation reports.

For information about performance metrics extraction from Metal/Tensix data, see [Performance Metrics Extraction](https://deepwiki.com/tenstorrent/polaris/3.1-performance-metrics-extraction). For details about projection comparison between different simulation runs, see [Projection Comparison](https://deepwiki.com/tenstorrent/polaris/3.3-projection-comparison).

## System Overview

The correlation analysis system bridges the gap between hardware benchmark results and simulation predictions. It automates the process of validating simulation accuracy by comparing ideal throughput values from polaris simulations against reference performance measurements from Metal/Tensix hardware.

## Output and Reporting

The correlation analysis produces a comprehensive CSV report containing performance comparisons between reference and simulation results.

### Report Structure

| Field | Description | Calculation |
| --- | --- | --- |
| `Arch` | Target architecture | From `ScoreTuple[0]` |
| `Workload` | Workload name | From `ScoreTuple[2]` |
| `Instance` | Instance identifier | From `ScoreTuple[3]` |
| `Api` | API type | From `ScoreTuple[1]` |
| `Tensix-Score` | Reference performance | From `metal_ref_scores` |
| `Actual-Score` | Simulation result | From `actual_scores` |
| `Diff` | Absolute difference | `ref_score - actual` |
| `Ratio` | Performance ratio | `ref_score / actual` |
