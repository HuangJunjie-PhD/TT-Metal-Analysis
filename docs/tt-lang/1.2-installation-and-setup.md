This page covers the installation and setup of `tt-lang`, including prerequisites, building from source, Docker configurations, and the lightweight simulator-only setup. `tt-lang` is designed to be flexible, supporting full hardware compilation via MLIR or pure Python functional simulation.

* * *

## Installation Methods Overview

The project supports three primary workflows depending on whether you have Tenstorrent hardware and whether you intend to modify the compiler itself.

| Method | Use Case | Requirements | Build Time |
| --- | --- | --- | --- |
| **PyPI Install** | Quick start for users | Python 3.10/3.12 | < 1 min |
| **Functional Simulator** | API Learning, logic validation | Python 3.10+ | ~2 min |
| **Docker (Pre-built)** | Running kernels on hardware | Docker, TT Device | 0 (Pull only) |
| **Build from Source** | Compiler development | Clang/GCC, CMake, Ninja | 30-90 min |

## Prerequisites

To build and run `tt-lang` locally, the following environment is required:

*   **OS**: Ubuntu 22.04 or 24.04 (Recommended) [[.github/workflows/call-build.yml 34](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[.github/workflows/call-build.yml#L34-L34)]
*   **Python**: 3.10+ (Python 3.12 recommended) [[docs/sphinx/build.md 16](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[docs/sphinx/build.md?plain=1#L16-L16)]
*   **Build Tools**: CMake 3.28+, Ninja [[docs/sphinx/build.md 13-14](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[docs/sphinx/build.md?plain=1#L13-L14)]
*   **Compiler**: Clang 17+ or GCC 12+ [[docs/sphinx/build.md 15](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[docs/sphinx/build.md?plain=1#L15-L15)]
*   **Git**: Submodules must be initialized: `git submodule update --init --recursive`[[docs/sphinx/build.md 17-18](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[docs/sphinx/build.md?plain=1#L17-L18)]

## Method 1: Install from PyPI

We provide two primary packages: `tt-lang` (full compiler + hardware support) and `tt-lang-sim` (simulator only).

### Setup Steps

1.   Create a virtual environment matching the wheel version (3.10 or 3.12) [[README.md 47-54](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[README.md?plain=1#L47-L54)].
2.   Install the desired package: `pip install tt-lang      # For hardware + simulator# ORpip install tt-lang-sim  # For simulator only`
3.   Run the setup script to download the `sfpi` runtime and tutorials: `tt-lang-setup`

**Implementation Detail**: `tt-lang-setup` is an idempotent command that extracts the `sfpi` compiler into the `ttnn` runtime directory and copies bundled tutorials to `./tutorials/`[[README.md 68-74](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[README.md?plain=1#L68-L74)].

**Sources:**[README.md 43-80](https://github.com/tenstorrent/tt-lang/blob/d76e6233/README.md?plain=1#L43-L80)[docs/sphinx/getting-started.md 9-52](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/getting-started.md?plain=1#L9-L52)

## Method 2: Functional Simulator (Build from Source)

The functional simulator runs kernels as pure Python using `greenlet` for cooperative multi-threading [[README.md 35](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[README.md?plain=1#L35-L35)]. It does not require a C++ compiler or Tenstorrent hardware.

### Setup Steps

1.   Clone the repository.
2.   Configure with `TTLANG_SIM_ONLY=ON`.
3.   Activate the generated environment.

`git clone https://github.com/tenstorrent/tt-lang.gitcd tt-langcmake -G Ninja -B build -DTTLANG_SIM_ONLY=ONsource build/env/activate`
**Implementation Detail**: In `TTLANG_SIM_ONLY` mode, the build system skips the LLVM and `tt-metal` builds, installs Python requirements (like `greenlet`), and generates an activation script that points to the local source tree [[CMakeLists.txt 79-131](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[CMakeLists.txt#L79-L131)].

**Sources:**[CMakeLists.txt 79-131](https://github.com/tenstorrent/tt-lang/blob/d76e6233/CMakeLists.txt#L79-L131)[docs/sphinx/getting-started.md 132-145](https://github.com/tenstorrent/tt-lang/blob/d76e6233/docs/sphinx/getting-started.md?plain=1#L132-L145)

## Method 3: Docker Setup

Tenstorrent provides pre-built Docker images for both distribution and development.

| Image Type | Name | Purpose |
| --- | --- | --- |
| **dist** | `tt-lang-dist-ubuntu-24-04` | End users: Pre-installed `tt-lang` and simulator. [[README.md 91](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[README.md?plain=1#L91-L91)] |
| **ird** | `tt-lang-ird-ubuntu-24-04` | Developers: Toolchain (LLVM/tt-metal) included; build `tt-lang` from source. [[README.md 92](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[README.md?plain=1#L92-L92)] |

### Running the Distribution Image

The **dist** image contains a fully built installation at `/opt/ttlang-toolchain`[[docs/sphinx/getting-started.md 157-158](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[docs/sphinx/getting-started.md?plain=1#L157-L158)].

`docker run -it --rm \  --device=/dev/tenstorrent/0 \  -v /dev/hugepages:/dev/hugepages \  ghcr.io/tenstorrent/tt-lang/tt-lang-dist-ubuntu-24-04:latest`
**Hardware Access**: Accessing Tenstorrent hardware requires mapping the device file `/dev/tenstorrent/0` and providing access to hugepages [[.github/workflows/call-test-hardware.yml 49-54](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[.github/workflows/call-test-hardware.yml#L49-L54)].

### Build Commands

To build against a pre-provided toolchain (like in the **ird** image):

`cmake -G Ninja -B build -DTTLANG_USE_TOOLCHAIN=ONsource build/env/activatecmake --build build`
**Toolchain Reuse**: Setting `-DTTLANG_USE_TOOLCHAIN=ON` skips the LLVM and `tt-metal` builds, using a pre-built toolchain at `$TTLANG_TOOLCHAIN_DIR` (default: `/opt/ttlang-toolchain`) [[docs/sphinx/build.md 109-120](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[docs/sphinx/build.md?plain=1#L109-L120)].

## Verification and Testing

Verify the installation using the built-in test targets [[test/TESTING.md 16-27](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[test/TESTING.md?plain=1#L16-L27)].

### Verification Commands

`# Activate environmentsource build/env/activate # Run MLIR transformation tests (FileCheck)cmake --build build --target check-ttlang-mlir # Run Python DSL lit testscmake --build build --target check-ttlang-python-lit # Run end-to-end hardware tests (requires device)python3 -m pytest test/python -v`
**Test Categories**:

*   `check-ttlang-mlir`: Verifies MLIR dialects and passes using `llvm-lit`[[test/TESTING.md 20](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[test/TESTING.md?plain=1#L20-L20)].
*   `check-ttlang-python-lit`: Verifies generated C++ and initial IR from Python DSL [[test/TESTING.md 21](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[test/TESTING.md?plain=1#L21-L21)].
*   `check-ttlang-pytest`: Parametric Python tests for functional correctness [[test/TESTING.md 24](https://github.com/tenstorrent/tt-lang/blob/d76e6233/[test/TESTING.md?plain=1#L24-L24)].
