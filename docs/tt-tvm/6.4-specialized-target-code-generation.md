## Purpose and Scope

This page documents code generation for specialized hardware targets beyond standard CPU and GPU architectures. The primary focus is on Hexagon DSP code generation, which demonstrates patterns for extending TVM to support domain-specific processors with custom instruction sets and libraries.

For general CPU code generation (x86, ARM), see [CPU Code Generation](https://deepwiki.com/tenstorrent/tt-tvm/6.2-cpu-code-generation). For GPU code generation (CUDA, ROCm), see [GPU Code Generation](https://deepwiki.com/tenstorrent/tt-tvm/6.3-gpu-code-generation). For the overall LLVM backend architecture, see [LLVM Backend Architecture](https://deepwiki.com/tenstorrent/tt-tvm/6.1-llvm-backend-architecture).

## Overview

Specialized target code generators inherit from `CodeGenCPU` or `CodeGenLLVM` and customize behavior for target-specific features such as:

*   Custom vector instruction sets (e.g., HVX on Hexagon)
*   Specialized libraries (e.g., QHL on Hexagon)
*   Target-specific memory hierarchies
*   Custom intrinsics and calling conventions

## Hexagon DSP Code Generator

### Class Structure

The `CodeGenHexagon` class is the primary implementation for Qualcomm Hexagon DSP targets, supporting the Hexagon Vector eXtensions (HVX) SIMD architecture.

| Method | Purpose | Key Features |
| --- | --- | --- |
| `Init` | Initialization | Calls parent `CodeGenCPU::Init` |
| `InitTarget` | Target setup | Detects HVX vector length (64 or 128 bytes) |
| `CreateCallExtern` | External calls | Routes QHL functions to split implementation |
| `CreateIntrinsic` | Intrinsic generation | Handles profiling intrinsics |
| `CreatePrintf` | Debug output | Uses HAP_debug_v2 for Hexagon |
| `CreateBufferPtr` | Memory access | Supports 2D physical buffers |
| `VectorLookupLoad` | Optimized lookup | Generates HVX lookup intrinsics |

**Sources:**[src/target/llvm/codegen_hexagon.cc 73-119](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_hexagon.cc#L73-L119)[src/target/llvm/codegen_hexagon.cc 121-146](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_hexagon.cc#L121-L146)

### HVX Vector Length Detection

Hexagon supports two HVX vector modes: 64-byte (512-bit) and 128-byte (1024-bit). The code generator detects this from target features at initialization.

**Sources:**[src/target/llvm/codegen_hexagon.cc 127-146](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_hexagon.cc#L127-L146)

**Implementation:**

`// src/target/llvm/codegen_hexagon.cc:127-146void CodeGenHexagon::InitTarget() {  native_vector_bits_ = 64;  // Assume scalar vectors initially  const auto hvx_length_feature = "+hvx-length";  // +hvx-length{64|128}b  for (const std::string& f : llvm_target_->GetTargetFeatures()) {    llvm::StringRef fs(f);    if (!fs.startswith(hvx_length_feature)) continue;        ICHECK(fs.endswith("b")) << "malformed target feature: " << f;    int hvx_bytes = 0;    size_t len_begin = std::strlen(hvx_length_feature);    ICHECK(!fs.substr(len_begin, fs.size() - len_begin - 1).getAsInteger(10, hvx_bytes))        << "invalid HVX length in feature string: " << f;    ICHECK(hvx_bytes == 64 || hvx_bytes == 128)        << "invalid HVX vector length: " << hvx_bytes << ", should be 64 or 128";    native_vector_bits_ = hvx_bytes * 8;    break;  }  CodeGenCPU::InitTarget();}`

### Vector Lookup Optimization

Hexagon provides specialized HVX intrinsics for efficient table lookups. The `VectorLookupLoad` method generates optimized code for small lookup tables (≤256 elements).

**Requirements:**

*   Table size: 1-256 elements
*   Element type: 8-bit values
*   Index: Vectorized

**Intrinsics Used:**

| Intrinsic | LLVM ID | Purpose |
| --- | --- | --- |
| `vshuffb` | `hexagon_V6_vshuffb_128B` | Shuffle bytes in table |
| `vsplatb` | `hexagon_V6_lvsplatb_128B` | Splat byte value |
| `vlut32` | `hexagon_V6_vlutvvbi_128B` | 32-byte lookup |
| `vlut32_or` | `hexagon_V6_vlutvvb_oracci_128B` | Accumulating lookup |

### Profiling Support

Hexagon code generator supports lightweight profiling using the `hexagon_instrprof_custom` intrinsic (LLVM 15+). This enables integration with LLVM's instrumentation profiling infrastructure.

Specialized code generators are registered via the TVM packed function registry. The factory pattern allows runtime selection based on target triple.

## Adding New Specialized Targets

To add support for a new specialized target:

1.   **Create code generator class** inheriting from `CodeGenCPU` or `CodeGenLLVM`
2.   **Override key methods:**
    *   `InitTarget()`: Set `native_vector_bits_`, configure target-specific settings
    *   `GetThreadIndex()`: For GPU-like targets with thread hierarchies
    *   `CreateStorageSync()`: For targets with explicit synchronization
    *   `CreateIntrinsic()`: For target-specific intrinsics
    *   `CreateCallExtern()`: For external library integration

3.   **Register factory function** with appropriate target name
4.   **Add target-specific optimizations** in `InitPassManagerBuilder()` (legacy) or custom passes

**Example skeleton:**

`class CodeGenMyTarget final : public CodeGenCPU { public:  void InitTarget() final {    native_vector_bits_ = 256;  // Target vector width    CodeGenCPU::InitTarget();  }    llvm::Value* CreateIntrinsic(const CallNode* op) override {    // Handle target-specific intrinsics    return CodeGenCPU::CreateIntrinsic(op);  }}; TVM_REGISTER_GLOBAL("tvm.codegen.llvm.target_mytarget").set_body([](TVMArgs args, TVMRetValue* rv) {    *rv = static_cast<void*>(new CodeGenMyTarget());});`
