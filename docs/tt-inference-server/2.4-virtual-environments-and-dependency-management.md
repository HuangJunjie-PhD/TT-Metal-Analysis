The `tt-inference-server` uses a sophisticated dependency management system to maintain isolation between the core server and various client-side tasks such as evaluations, benchmarking, and model setup. This system leverages `uv` for fast, reproducible virtual environment creation and management.

## Overview and Purpose

To ensure that the host-side orchestration layer does not conflict with the diverse requirements of evaluation frameworks (e.g., `lm-eval`, `lmms-eval`) or benchmarking tools (e.g., `genai-perf`), the codebase utilizes specialized virtual environments. These environments are managed via the `WorkflowVenvType` enum and `VenvConfig` class [workflows/workflow_types.py 26-50](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_types.py#L26-L50)

The system serves three primary purposes:

1.   **Isolation**: Prevents dependency conflict between different evaluation frameworks (e.g., Llama-cookbook vs. common lm-eval) [workflows/workflow_venvs.py 122-135](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L122-L135)
2.   **Automation**: Automatically bootstraps `uv` and creates required environments during workflow execution [workflows/workflow_venvs.py 67-85](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L67-L85)
3.   **Portability**: Ensures that benchmarks and evals run in consistent environments regardless of the user's local Python state [docs/workflows_user_guide.md 72-74](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/docs/workflows_user_guide.md?plain=1#L72-L74)

## Key Components

### WorkflowVenvType

An `IntEnum` that categorizes every environment used in the system. Key types include:

*   `EVALS_COMMON`: General LLM evaluations using a specific fork of `lm-evaluation-harness`[workflows/workflow_types.py 34](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_types.py#L34-L34)
*   `EVALS_META`: Specialized environment for Llama-specific "cookbook" evaluations [workflows/workflow_types.py 35](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_types.py#L35-L35)
*   `BENCHMARKS_VLLM`: Environment for running `vLLM` benchmarks [workflows/workflow_types.py 43](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_types.py#L43-L43)
*   `HF_SETUP`: Environment for downloading and preparing HuggingFace weights [workflows/workflow_types.py 46](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_types.py#L46-L46)
*   `EVALS_AUDIO` / `EVALS_VISION`: Environments for multi-modal evaluation tasks [workflows/workflow_types.py 36-37](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_types.py#L36-L37)

### VenvConfig

A dataclass that defines the blueprint for a virtual environment. It handles path inference and the execution of the setup logic [workflows/workflow_venvs.py 36-66](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L36-L66)

| Attribute | Description |
| --- | --- |
| `venv_type` | The `WorkflowVenvType` identifier. |
| `setup_function` | A callable that performs specific `pip install` or `git clone` logic [workflows/workflow_venvs.py 38](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L38-L38) |
| `venv_path` | Defaults to `.workflow_venvs/.venv_{name}`[workflows/workflow_venvs.py 58](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L58-L58) |
| `python_version` | Target Python version (defaults to 3.10) [workflows/workflow_venvs.py 40](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L40-L40) |
| `venv_python` | Path to the python executable within the venv [workflows/workflow_venvs.py 62](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L62-L62) |

### Specialized Setup Functions

These functions contain the logic for installing complex dependencies:

*   **`setup_evals_common`**: Installs `lm-eval` with specific extras (e.g., `ifeval`, `math`, `r1_evals`, `ruler`) and CPU-specific PyTorch [workflows/workflow_venvs.py 88-102](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L88-L102)
*   **`setup_evals_meta`**: Clones the `meta-llama/llama-cookbook` repository and installs it in editable mode for specialized Llama benchmarks [workflows/workflow_venvs.py 122-187](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L122-L187)
*   **`setup_venv`**: A generic setup function that ensures a `work_dir` exists for the virtual environment [workflows/workflow_venvs.py 105-119](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L105-L119)

## Implementation Details

### Bootstrapping with UV

The server avoids relying on the system's `pip` by using a standalone `uv` binary. The `bootstrap_uv.py` script ensures `UV_EXEC` is available [workflows/bootstrap_uv.py 15](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/bootstrap_uv.py#L15-L15)

When `VenvConfig.setup()` is called:

1.   It checks if the directory exists [workflows/workflow_venvs.py 70](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L70-L70)
2.   If not, it runs `uv venv --managed-python` to create a clean environment [workflows/workflow_venvs.py 76-80](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L76-L80)
3.   It then executes the assigned `setup_function` to populate dependencies [workflows/workflow_venvs.py 82](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/workflow_venvs.py#L82-L82)

### Subprocess Integration

Workflows do not import code from these virtual environments directly. Instead, they resolve the `venv_python` path and use it to launch external scripts as subprocesses via `run_command`[workflows/utils.py 20-21](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/utils.py#L20-L21)
