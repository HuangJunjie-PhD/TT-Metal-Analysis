## Overview

The Blackhole architecture represents an enhanced version of Tenstorrent's SFPU hardware, providing 54 hardware operations (compared to 51 in Wormhole) with additional capabilities for approximate reciprocal computation, 24-bit multiplication, random number generation, and enhanced shift operations. This page documents the Blackhole-specific hardware abstraction layer, implementation details, and enhanced library functions.

Sources: [include/blackhole/sfpi_hw.h 1-306](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L1-L306)[include/blackhole/sfpi_lib.h 1-327](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L1-L327)

## Key Architectural Differences from Wormhole

| Feature | Wormhole | Blackhole | Enhancement |
| --- | --- | --- | --- |
| **Total Operations** | 51 | 54 | +3 operations |
| **ADDR_MODE_NOINC** | 3 | 7 | Different address mode encoding |
| **Approximate Reciprocal** | Not available | `__builtin_rvtt_sfparecip` | Hardware-accelerated reciprocal/exp |
| **24-bit Multiply** | Not available | `__builtin_rvtt_sfpmul24` | Optimized 24-bit integer multiplication |
| **Config Move** | Not available | `__builtin_rvtt_sfpmov_config` | Direct config register access |
| **Shift Modes** | Limited | Arithmetic/Logical with LREG7 options | Enhanced shift capabilities |
| **SFPCAST Modes** | 2 modes | 4 modes | Sign-magnitude conversions added |
| **Random Generator** | Not available | `SFPCONFIG_SRC_RAND` | Hardware RNG access |

## Enhanced Hardware Intrinsics

### Blackhole-Specific Operations

Blackhole introduces three new hardware intrinsics not available in Wormhole:

#### 1. Approximate Reciprocal (`__builtin_rvtt_sfparecip`)

Provides hardware-accelerated approximate reciprocal and exponential operations with three modes:

| Mode | Constant | Function |
| --- | --- | --- |
| Reciprocal | `SFPARECIP_MOD1_RECIP` (0) | Computes 1/x approximation |
| Conditional Reciprocal | `SFPARECIP_MOD1_COND_RECIP` (1) | Computes 1/x if condition met |
| Exponential | `SFPARECIP_MOD1_EXP` (2) | Computes e^x approximation |

Sources: [include/blackhole/sfpi_hw.h 93](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L93-L93)[include/blackhole/sfpi_hw.h 146-148](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L146-L148)

#### 2. 24-bit Integer Multiply (`__builtin_rvtt_sfpmul24`)

Optimized 24-bit integer multiplication with flexible source/destination register options:

| Mode Flag | Constant | Purpose |
| --- | --- | --- |
| Lower result | `SFPMUL24_MOD1_LOWER` (0) | Returns lower 24 bits |
| Upper result | `SFPMUL24_MOD1_UPPER` (1) | Returns upper 24 bits |
| Source A from LREG7 | `SFPMUL24_MOD1_SRCA_LREG7` (4) | Use LREG7 as source A |
| Dest to LREG7 | `SFPMUL24_MOD1_DST_LREG7` (8) | Write result to LREG7 |

Sources: [include/blackhole/sfpi_hw.h 94](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L94-L94)[include/blackhole/sfpi_hw.h 150-154](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L150-L154)

#### 3. Config Register Move (`__builtin_rvtt_sfpmov_config`)

Direct read access to configuration registers, including the random number generator source:

`// Access random number generatorvInt rand_value = __builtin_rvtt_sfpmov_config(SFPCONFIG_SRC_RAND);`
Sources: [include/blackhole/sfpi_hw.h 92](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L92-L92)[include/blackhole/sfpi_hw.h 276](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L276-L276)

### Enhanced Shift Operations

Blackhole extends shift operations with arithmetic/logical mode control and LREG7 options:

| Mode Flag | Constant | Description |
| --- | --- | --- |
| Logical shift | `SFPSHFT_MOD1_LOGICAL` (0) | Zero-fill right shift |
| Arithmetic shift | `SFPSHFT_MOD1_ARITHMETIC` (2) | Sign-extend right shift |
| Source A from LREG7 | `SFPSHFT_MOD1_SRCA_LREG7` (4) | Use LREG7 as shift source |
| Dest to LREG7 | `SFPSHFT_MOD1_DST_LREG7` (8) | Write result to LREG7 |

This enables both the vector shift functions in `sfpi_lib.h` to properly handle signed vs unsigned shifts:

`// Arithmetic shift for signed integersvInt shft(const vInt v, const vInt amt);  // Uses SFPSHFT_MOD1_ARITHMETIC // Logical shift for unsigned integers  vUInt shft(const vUInt v, const vInt amt); // Uses SFPSHFT_MOD1_LOGICAL`
Sources: [include/blackhole/sfpi_hw.h 65-66](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L65-L66)[include/blackhole/sfpi_hw.h 253-256](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L253-L256)[include/blackhole/sfpi_lib.h 173-191](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L173-L191)

### Enhanced Type Casting

Blackhole adds sign-magnitude conversion modes to the `sfpcast` instruction:

| Mode | Constant | Function |
| --- | --- | --- |
| INT32 → FP32 (RNE) | `SFPCAST_MOD1_INT32_TO_FP32_RNE` (0) | Round to nearest even |
| INT32 → FP32 (RNS) | `SFPCAST_MOD1_INT32_TO_FP32_RNS` (1) | Round stochastic |
| Sign-Mag → INT32 | `SFPCAST_MOD1_SM32_TO_INT32` (2) | Convert from sign-magnitude |
| INT32 → Sign-Mag | `SFPCAST_MOD1_INT32_TO_SM32` (3) | Convert to sign-magnitude |

Sources: [include/blackhole/sfpi_hw.h 230-238](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_hw.h#L230-L238)

## Enhanced Math Library Functions

Blackhole extends the `sfpi_lib.h` functional math library with three new functions:

### Random Number Generation

`sfpi_inline vInt rand() {    return vInt(__builtin_rvtt_sfpmov_config(SFPCONFIG_SRC_RAND));}`
Provides access to the hardware random number generator via the config register move instruction. Returns a vector of random integer values.

Sources: [include/blackhole/sfpi_lib.h 311-313](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L311-L313)

### Approximate Reciprocal

`template<bool uncond = true>sfpi_inline vFloat approx_recip(const vFloat &src){    return vFloat(__builtin_rvtt_sfparecip(src.get(),         uncond ? SFPARECIP_MOD1_RECIP : SFPARECIP_MOD1_COND_RECIP));}`
Computes hardware-accelerated approximate reciprocal (1/x). The template parameter controls whether the operation is unconditional or predicated on the current condition code.

Sources: [include/blackhole/sfpi_lib.h 315-319](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/blackhole/sfpi_lib.h#L315-L319)

### Approximate Exponential

`sfpi_inline vFloat approx_exp(const vFloat &src){    return vFloat(__builtin_rvtt_sfparecip(src.get(), SFPARECIP_MOD1_EXP));}`
Computes hardware-accelerated approximate exponential (e^x). Uses the same hardware unit as `approx_recip` but in exponential mode.

The Blackhole implementation provides proper arithmetic vs logical shift handling through operator overloads:

## Address Mode and Memory Operations

One of the key differences between Blackhole and Wormhole is the `ADDR_MODE_NOINC` constant:

| Architecture | Value | Usage |
| --- | --- | --- |
| Wormhole | 3 | Address mode for non-incrementing load/store |
| Blackhole | 7 | Address mode for non-incrementing load/store |

This affects how `__builtin_rvtt_sfpload` and `__builtin_rvtt_sfpstore` operate when accessing destination registers without auto-increment:

`// Load from destination register without incrementing addressv = __builtin_rvtt_sfpload(SFPLOAD_MOD0_FMT_SRCB,                            SFPLOAD_ADDR_MODE_NOINC,  // = 7 on Blackhole                           dreg.get()); // Store to destination register without incrementing address  __builtin_rvtt_sfpstore(vec.get(),                         SFPSTORE_MOD0_FMT_SRCB,                        SFPSTORE_ADDR_MODE_NOINC,  // = 7 on Blackhole                        reg);`

### Destination Register File

The Blackhole architecture provides a destination register file with the following characteristics:

| Property | Value | Description |
| --- | --- | --- |
| `SFP_DESTREG_MAX_SIZE` | 1024 | Maximum rows in register file |
| `SFP_DESTREG_STRIDE` | 2 | Address stride (rows per 32-bit word) |
| `SFP_DESTREG_MAX_ADDR` | 512 | Maximum addressable locations (1024/2) |
| `SFP_LREG_COUNT` | 8 | Number of local registers (LReg0-7) |

The `__vDReg` class provides typed access to these destination registers with automatic format conversion:

`// Store float to destination registerdst_reg[10] = vFloat(1.5f);  // Uses SFPSTORE_MOD0_FMT_SRCB // Store integer to destination register (with sign-magnitude conversion)dst_reg[10] = vInt(42);  // Uses SFPSTORE_MOD0_FMT_INT32_TO_SM`

## Compilation and Architecture Detection

The Blackhole architecture is selected via preprocessor macro:

`// In sfpi.h, architecture-specific headers are conditionally included:#if defined(__riscv_xtttensixwh)#include "wormhole/sfpi_imp.h"#elif defined(__riscv_xtttensixbh)#include "blackhole/sfpi_imp.h"  // Blackhole selected#elif defined(__riscv_xtttensixgs)#include "grayskull/sfpi_imp.h"#endif`
The compiler defines `__riscv_xtttensixbh` when targeting Blackhole, which triggers inclusion of the Blackhole-specific implementation files.

## Usage Example

Here's a basic usage pattern for the Blackhole implementation:

`// Load vector valuesvFloat a = 1.0f;vFloat b = 2.0f; // Perform vector operationsvFloat c = a + b;vFloat d = a * b; // Conditional executionv_if (a > 0.0f) {    c += d;} v_else {    c -= d;} v_endif // Store result to destination registerdReg[0] = c;`
## Compilation and Build Integration

When compiling for the Blackhole architecture, you need to:

1.   Define the `ARCH_BLACKHOLE` preprocessor symbol
2.   Include the appropriate header: `#include <blackhole/sfpi_hw.h>`
3.   Use the provided build scripts with the correct architecture flag

Sources: [include/sfpi_fp16.h 15-19](https://github.com/tenstorrent/sfpi/blob/ed989b9d/include/sfpi_fp16.h#L15-L19)

## Summary

The Blackhole implementation of SFPI provides a comprehensive set of vector operations optimized for the Blackhole architecture. It implements the full SFPI interface with hardware-specific optimizations, supporting various data formats, conditional execution, and specialized instructions.

The implementation is designed to work seamlessly with the SFPI core interface while providing efficient access to the unique capabilities of the Blackhole hardware.
