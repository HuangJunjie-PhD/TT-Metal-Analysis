## Overview

This page documents the remaining PyTorch model fine-tuning experiments in TT-Blacksmith that are not covered in earlier sections (4.1.1â€“4.1.4):

*   **Phi-1 and Phi-1.5:** Causal LM fine-tuning using LoRA on SST2 (sentiment) and SQuAD V2 (QA)
*   **ALBERT:** Intent classification using a custom `AlbertWithMLP` model on Banking77
*   **ViT:** Vision Transformer fine-tuning using LoRA on the StanfordCars image dataset

Each experiment follows the same structural pattern: a `TrainingConfig` Pydantic model, one or more YAML presets, and a single training script that instantiates `DeviceManager`, `TrainingLogger`, `CheckpointManager`, and `ReproducibilityManager`. For the shared infrastructure details, see pages 3.3 and 3.5.1.

Sources: [blacksmith/experiments/torch/phi/test_phi_finetuning.py 1-207](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/phi/test_phi_finetuning.py#L1-L207)[blacksmith/experiments/torch/albert/test_albert_finetuning.py 1-188](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/albert/test_albert_finetuning.py#L1-L188)[blacksmith/experiments/torch/vit/README.md 1-78](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/vit/README.md?plain=1#L1-L78)

## Model Loading by Experiment

Phi and ViT experiments use the shared `get_model()` factory from `hf_models.py`, which loads a HuggingFace model and applies LoRA via the PEFT library. ALBERT diverges from this pattern and instantiates `AlbertWithMLP` directly before calling `torch.compile` with the TT backend.

**Model loading paths by experiment:**

| Experiment | Model Loader | PEFT Method | Compile Backend |
| --- | --- | --- | --- |
| Phi-1 / Phi-1.5 | `get_model()` in `hf_models.py` | LoRA via `peft.get_peft_model` | `torch_xla` (via `DeviceManager`) |
| ViT | `get_model()` in `hf_models.py` | LoRA via `peft.get_peft_model` | `torch_xla` (via `DeviceManager`) |
| ALBERT | `AlbertWithMLP(config)` in `albert.py` | MLP classification head | `torch.compile(backend="tt")` |

Sources: [blacksmith/experiments/torch/phi/test_phi_finetuning.py 86-88](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/phi/test_phi_finetuning.py#L86-L88)[blacksmith/experiments/torch/albert/test_albert_finetuning.py 66-74](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/albert/test_albert_finetuning.py#L66-L74)

## ALBERT with MLP Head Fine-tuning

### Overview

The ALBERT experiment fine-tunes `albert/albert-base-v2` on the Banking77 intent classification dataset (77 classes). Unlike the other experiments on this page, it does **not** use `get_model()` from `hf_models.py`. Instead, the training script directly instantiates `AlbertWithMLP`, a custom wrapper defined in `blacksmith/models/torch/huggingface/albert.py`.

**ALBERT Experiment Characteristics:**

| Attribute | Value |
| --- | --- |
| **Model class** | `AlbertWithMLP` |
| **HF backbone** | `albert/albert-base-v2` |
| **Dataset** | Banking77 (77 intent classes) |
| **Task** | Multi-class classification |
| **Default dtype** | `torch.bfloat16` |
| **Hardware** | N150 (single chip) |

| Parameter | Default | Description |
| --- | --- | --- |

## Phi Model Experiments

### Overview

Phi-1 (`microsoft/phi-1`) and Phi-1.5 (`microsoft/phi-1_5`) are fine-tuned using LoRA (`training_type: "lora"`) via the shared `get_model()` infrastructure. A single training script â€” `test_phi_finetuning.py` â€” handles all four dataset/model combinations, selected by passing the appropriate YAML file.

**Phi Model Variants:**

| YAML Config | Model | Dataset | `batch_size` | `max_length` |
| --- | --- | --- | --- | --- |
| `test_phi1_finetuning_sst2.yaml` | `microsoft/phi-1` | SST-2 | 8 | 32 |
| `test_phi1_finetuning_squadV2.yaml` | `microsoft/phi-1` | SQuAD V2 | 2 | 256 |
| `test_phi15_finetuning_sst2.yaml` | `microsoft/phi-1_5` | SST-2 | 8 | 32 |
| `test_phi15_finetuning_squadV2.yaml` | `microsoft/phi-1_5` | SQuAD V2 | 2 | 256 |

The SQuAD V2 variants use a smaller `batch_size` of 2 and a much longer `max_length` of 256 to accommodate the context+question format.

### Configuration

All Phi YAML files share the following LoRA settings:

| Parameter | Value | Description |
| --- | --- | --- |
| `training_type` | `"lora"` | Selects `_apply_lora()` path in `get_model()` |
| `lora_r` | `4` | LoRA rank |
| `lora_alpha` | `8` | LoRA scaling factor |
| `lora_target_modules` | `["q_proj", "v_proj"]` | Attention projection layers |
| `lora_task_type` | `"CAUSAL_LM"` | Task type passed to PEFT |
| `learning_rate` | `6e-5` | AdamW learning rate |
| `dtype` | `"torch.bfloat16"` | Training precision |
| `ignored_index` | `-100` | Padding token ignored in loss |
| `mesh_shape` | `null` | Single-device only |

The `TrainingConfig` for Phi is defined in [blacksmith/experiments/torch/phi/configs.py 11-79](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/phi/configs.py#L11-L79)

## ViT Fine-tuning

### Overview

The ViT experiment fine-tunes `google/vit-base-patch16-224` (Vision Transformer) on the StanfordCars dataset using LoRA. This is the only image-domain experiment in the PyTorch section.

**ViT Experiment Characteristics:**

| Attribute | Value |
| --- | --- |
| **Model** | `google/vit-base-patch16-224` |
| **Dataset** | StanfordCars (196 car model classes) |
| **PEFT method** | LoRA |
| **Task** | Fine-grained image classification |
| **Input size** | 224Ă—224, RGB |
| **Default dtype** | `torch.bfloat16` |
| **Hardware** | N150 (single chip) |

### Configuration

The `TrainingConfig` for ViT is defined in [blacksmith/experiments/torch/vit/configs.py 11-80](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/vit/configs.py#L11-L80) and the YAML preset at [blacksmith/experiments/torch/vit/test_vit_stanfordcars.yaml 1-59](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/vit/test_vit_stanfordcars.yaml#L1-L59)

**Config fields unique to ViT (not present in text-model configs):**

| Parameter | Default | Description |
| --- | --- | --- |
| `image_size` | `224` | Input image resolution |
| `image_mean` | `[0.5, 0.5, 0.5]` | Per-channel normalization mean |
| `image_std` | `[0.5, 0.5, 0.5]` | Per-channel normalization std |
| `loss_fn` | `"torch.nn.CrossEntropyLoss"` | Loss function (string-evaluated) |

**LoRA settings (from `test_vit_stanfordcars.yaml`):**

| Parameter | Value |
| --- | --- |
| `lora_r` | `16` |
| `lora_alpha` | `16` |
| `lora_target_modules` | `["query", "value"]` |
| `lora_dropout` | `0.1` |

Note the higher rank (`lora_r: 16`) compared to the text model experiments (which use `lora_r: 4`), and the ViT-specific layer names `"query"` and `"value"` instead of `"q_proj"` / `"v_proj"`.

**Other training hyperparameters:**

| Parameter | Value |
| --- | --- |
| `learning_rate` | `1e-3` |
| `batch_size` | `10` |
| `num_epochs` | `8` |
| `val_steps_freq` | `100` |
| `save_strategy` | `"step"` |

Sources: [blacksmith/experiments/torch/vit/configs.py 1-80](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/vit/configs.py#L1-L80)[blacksmith/experiments/torch/vit/test_vit_stanfordcars.yaml 1-59](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/vit/test_vit_stanfordcars.yaml#L1-L59)
