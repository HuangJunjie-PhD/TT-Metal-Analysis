This page details the technical architecture of the `tt-awesome` ecosystem directory. The project is designed as a **data-first, unidirectional pipeline** where project entries are stored in a structured format and transformed into multiple user-facing representations.

## System Overview

The system architecture follows a "Single Source of Truth" (SSOT) pattern. All project data resides in the `entries/` directory as individual JSON files. A suite of Python scripts and an Eleventy-based frontend consume this data to produce the repository's `README.md`, a machine-readable `data.json`, and a full-featured static website.

## Design Decisions

### 1. JSON as Single Source of Truth

Instead of maintaining a manual Markdown list, project data is stored in `entries/`. This allows for:

*   **Strict Validation:** CI can enforce required fields like `affiliation`, `categories`, and `links`[docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 98-115](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L98-L115)
*   **Multi-output Rendering:** The same data generates the skimmable `README.md` and the interactive website.
*   **Automated Metadata:** Star counts and update timestamps are fetched via `scripts/fetch_github_meta.py` and merged into the JSON at build time, preventing "commit noise" in the source entry files [docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 122-125](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L122-L125)

### 2. Community-First Sorting Philosophy

A core design goal is to feature community and affiliated projects above official Tenstorrent repositories [docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 22](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L22-L22) This is implemented via the `sort_entries` function in both the Python and JavaScript layers.

**Sorting Algorithm Implementation** The sorting logic uses a three-tier priority system:

1.   **Affiliation Tier:**`community` (0) → `affiliated` (1) → `official` (2). Lower values rank higher [scripts/generate_readme.py 31](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L31-L31)
2.   **Featured Flag:** Within a tier, entries with `"featured": true` are promoted [scripts/generate_readme.py 89](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L89-L89)
3.   **Star Count:** Finally, entries are sorted by star count in descending order [scripts/generate_readme.py 90](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L90-L90)

**Sources:**[scripts/generate_readme.py 80-91](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L80-L91)[scripts/generate_data_json.py 50-56](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L50-L56)

### 3. Decoupled Metadata

Star counts and last-updated timestamps are not stored in the `entries/*.json` files. Instead, `scripts/fetch_github_meta.py` populates a central `src/_data/github_meta.json` file nightly [docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 42](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L42-L42) This file is ignored by standard logic but used by the Eleventy data loader (`src/_data/entries.js`) to enrich the project objects during the site build [docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 45-47](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L45-L47)

## Build & Deployment Lifecycle

The project uses GitHub Actions to automate the entire lifecycle from entry submission to site deployment.

1.   **Validation (`validate.yml`):** Runs on every PR. It executes `scripts/validate.py` to ensure all JSON files in `entries/` conform to the schema (required fields, valid URLs, correct category slugs) [docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 63](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L63-L63)
2.   **Nightly Updates (`nightly.yml`):** Runs daily at midnight. It executes `scripts/fetch_github_meta.py` to refresh star counts and release data, committing changes to `src/_data/github_meta.json`[docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 65](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L65-L65)
3.   **Deployment (`deploy.yml`):** Triggered on merges to `main`. 
    *   Runs `scripts/generate_readme.py` to update the root `README.md`[scripts/generate_readme.py 1-5](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L1-L5)
    *   Runs `scripts/generate_data_json.py` to update the API export [scripts/generate_data_json.py 87-93](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_data_json.py#L87-L93)
    *   Executes `npm run build` (Eleventy) to generate the static site in `_site/`[package.json 6](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/package.json#L6-L6)
    *   Deploys `_site/` to the `gh-pages` branch [docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md 64](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-08-awesome-tenstorrent-design.md?plain=1#L64-L64)
