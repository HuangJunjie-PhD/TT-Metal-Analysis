## Purpose and Scope

This page provides comprehensive API reference documentation for all Textual widget classes, modal screens, and styling components in the `tt_tools_common/ui_common` module. Each widget class is documented with its constructor parameters, public methods, properties, CSS selectors, and usage patterns.

For higher-level guidance on using these widgets to build TUI applications, see [Widget Library Reference](https://deepwiki.com/tenstorrent/tt-tools-common/4.1-widget-library-reference). ## Core Display Widgets

### TTHeader

**Class Definition:**`TTHeader(Widget)`

A time-updating header widget that displays application name, version, and current timestamp.

#### Constructor Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `app_name` | `str` | Yes | Application name displayed in center column |
| `app_version` | `str` | Yes | Version string displayed in left column (prefixed with "Version ") |

#### Methods

| Method | Return Type | Description |
| --- | --- | --- |
| `on_mount()` | `None` | Sets 1-second interval timer for automatic refresh |
| `render()` | `RenderResult` | Renders 3-column Rich Panel with version, app name, and timestamp |

#### Layout Structure

The header uses a Rich `Table.grid` with three columns:

*   **Left (25%):** Version string with "Version " prefix
*   **Center (50%):** Application name
*   **Right (25%):** Current date/time in format "MMM DD YYYY HH:MM:SS AM/PM"

#### CSS Selectors

*   `TTHeader` - Main widget selector
*   Inherits global `*` styles

#### Example Usage

**Sources:**[tt_tools_common/ui_common/widgets.py 22-45](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L22-L45)[tt_tools_common/ui_common/common_style.css 30-39](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L30-L39)

### TTFooter

**Class Definition:**`TTFooter(Widget)`

A customizable footer widget for displaying multi-column content in a panel.

#### Constructor Parameters

| Parameter | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `content` | `List[str]` | Yes | - | List of strings, one per column |
| `justify` | `str` | No | `"center"` | Text justification: "left", "center", or "right" |

#### Methods

| Method | Return Type | Description |
| --- | --- | --- |
| `render()` | `RenderResult` | Renders Rich Panel with collapsible grid of content columns |

#### Layout Structure

Creates a Rich `Table.grid` with:

*   Number of columns equal to length of `content` list
*   Each column has equal ratio (1)
*   `collapse_padding=False` to maintain spacing

#### CSS Selectors

*   `TTFooter` - Main widget selector

### TTDataTable

**Class Definition:**`TTDataTable(ScrollableContainer)`

A scrollable container wrapping a Textual `DataTable` with additional configuration and update capabilities.

#### Constructor Parameters

| Parameter | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `title` | `str` | Yes | - | Border title for both container and inner DataTable |
| `header` | `List[str]` | No | `None` | Column headers (auto-styled with underline and center justification) |
| `id` | `str` | No | `None` | Widget ID (inner DataTable gets `id + "_table"`) |
| `**kwargs` | `Any` | No | - | Additional kwargs passed to DataTable constructor |

#### Properties

| Property | Type | Description |
| --- | --- | --- |
| `dt` | `DataTable` | Reference to inner DataTable widget |
| `header` | `List[str]` | Column headers (converted to Rich Text objects when styled) |
| `border_title` | `str` | Container border title |
| `_title` | `str` | Private copy of title for DataTable configuration |

#### Methods

| Method | Parameters | Return Type | Description |
| --- | --- | --- | --- |
| `style_header()` | None | `None` | Converts header strings to Rich Text with center justification and underline style |
| `config_dt(**kwargs)` | `**kwargs` | `DataTable` | Creates and configures inner DataTable with zebra_stripes=False, cursor_type="cell" |
| `update_data(rows)` | `rows: List[str]` | `None` | Updates DataTable cells individually; clears and re-adds rows if dimensions changed |
| `compose()` | None | `ComposeResult` | Yields nested ScrollableContainer with inner DataTable |

#### DataTable Configuration

The inner DataTable is configured with:

*   `zebra_stripes = False`
*   `cursor_type = "cell"`
*   `border_title` set to title parameter
*   `id` set to `{container_id}_table`

#### Update Behavior

The `update_data()` method attempts cell-by-cell updates using `update_cell_at()`. If `CellDoesNotExist` is raised (row count increased), it falls back to `clear()` followed by `add_rows()`.

#### CSS Selectors

*   `TTDataTable` - Container selector (inherits from ScrollableContainer)
*   `DataTable` - Inner DataTable styling
*   `.datatable--header` - Header row styling
*   `.datatable--cursor` - Cell cursor styling
*   `.datatable--hover` - Hover state styling

#### Example Usage

**Sources:**[tt_tools_common/ui_common/widgets.py 65-117](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L65-L117)[tt_tools_common/ui_common/common_style.css 45-85](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L45-L85)

## Menu Display Widgets

### TTMenu

**Class Definition:**`TTMenu(Container)`

A key-value list widget that displays data in a formatted menu style with bullet points.

#### Constructor Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | `str` | Yes | Widget identifier |
| `title` | `str` | Yes | Border title for the container |
| `data` | `OrderedDict[str, str]` | Yes | Key-value pairs to display |

#### Properties

| Property | Type | Description |
| --- | --- | --- |
| `data` | `OrderedDict[str, str]` | Key-value data to render |
| `justify_width` | `int` | Width for key column alignment (max key length + 1) |
| `border_title` | `str` | Container border title |

#### Methods

| Method | Return Type | Description |
| --- | --- | --- |
| `render()` | `RenderResult` | Renders Rich Text with formatted key-value pairs |

#### Rendering Format

Each key-value pair is rendered as:

```
* <key (left-justified to justify_width)>: <value>
```

**Special Handling:** If key is `"Failed to fetch"`, the value is displayed on a new line in dark orange without the colon separator.

#### Styling

*   Keys: `#ffd10a` (yellow), bold
*   Values: Default color
*   Failed fetch values: `dark_orange`
*   Bullet points: `*` prefix for normal entries

#### CSS Selectors

*   `TTMenu` - Main widget selector

#### Example Usage

**Sources:**[tt_tools_common/ui_common/widgets.py 120-146](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L120-L146)[tt_tools_common/ui_common/common_style.css 88-95](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L88-L95)

### TTHostCompatibilityMenu

**Class Definition:**`TTHostCompatibilityMenu(Container)`

A menu widget for displaying host information with compatibility warnings. Used primarily in tt-smi to show system configuration issues.

#### Constructor Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | `str` | Yes | Widget identifier |
| `title` | `str` | Yes | Base border title (appended with compatibility status) |
| `data` | `Dict[str, Union[str, Tuple]]` | Yes | Key-value pairs or key-(value, recommendation) tuples |

#### Data Format

The `data` dictionary accepts two value types:

1.   **String value:** Simple key-value pair (indicates compatible configuration)
2.   **Tuple value:**`(current_value, recommendation_note)` (indicates configuration warning)

#### Title and Styling Behavior

| Condition | Border Title | Border Color | Subtitle |
| --- | --- | --- | --- |
| All values are strings | `{title} (Fully Compatible)` | Default (yellow) | None |
| Any value is tuple | `{title} (Config Warning!)` | Red | `"* Recommended Config"` (red) |

#### Properties

| Property | Type | Description |
| --- | --- | --- |
| `data` | `Dict[str, Union[str, Tuple]]` | Key-value data with optional recommendations |
| `justify_width` | `int` | Width for key column alignment (max key length + 1) |
| `border_title` | `str` | Container border title with compatibility status |
| `border_subtitle` | `str` | Set to "* Recommended Config" if warnings exist |

#### Methods

| Method | Return Type | Description |
| --- | --- | --- |
| `render()` | `RenderResult` | Renders Rich Text with formatted entries and warnings |

#### Rendering Format

**For string values:**

```
* <key>: <value>
```

**For tuple values:**

```
* <key>: <current_value>
         * <recommendation>
```

#### Styling

*   Keys: `#ffd10a` (yellow), bold
*   String values: Default color
*   Tuple values (current): Default color
*   Recommendations: `dark_orange` with indentation
*   Border title: Red if warnings present

#### CSS Selectors

*   `TTHostCompatibilityMenu` - Main widget selector

#### Example Usage

**Sources:**[tt_tools_common/ui_common/widgets.py 149-210](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L149-L210)[tt_tools_common/ui_common/common_style.css 89-95](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L89-L95)

### TTConfirmBox

**Class Definition:**`TTConfirmBox(ModalScreen)`

A modal confirmation dialog with "Yes" and "No" buttons and optional callbacks.

#### Constructor Parameters

| Parameter | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `text` | `str` | Yes | - | Question or confirmation message |
| `on_yes` | `Callable` | No | `None` | Callback executed when "Yes" is pressed |
| `on_no` | `Callable` | No | `None` | Callback executed when "No" is pressed |

#### Methods

| Method | Parameters | Return Type | Description |
| --- | --- | --- | --- |
| `compose()` | None | `ComposeResult` | Yields Grid container with Label and two Buttons |
| `on_button_pressed(event)` | `event: Button.Pressed` | `None` | Handles button clicks and executes callbacks |

#### Composition Structure

```
Grid (id="dialog")
├── Label (id="question") - Displays confirmation text
├── Button (id="yes") - "Yes" button
└── Button (id="no") - "No" button
```

#### Behavior

1.   When "Yes" button is pressed:

    *   Executes `on_yes()` callback if provided
    *   Calls `app.pop_screen()` to close modal

2.   When "No" button is pressed:

    *   Executes `on_no()` callback if provided
    *   Calls `app.pop_screen()` to close modal

#### CSS Selectors

*   `TTConfirmBox` - Modal screen selector (aligned center middle)
*   `#dialog` - Grid container
*   `#question` - Label with confirmation text
*   `Button` - Both Yes/No buttons
*   `Button:hover`, `Button:focus` - Interactive states

#### Example Usage

**Sources:**[tt_tools_common/ui_common/widgets.py 246-271](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L246-L271)[tt_tools_common/ui_common/common_style.css 105-138](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L105-L138)

### TTSettingsMenu

**Class Definition:**`TTSettingsMenu(ModalScreen)`

A modal settings screen with switch controls for dark mode and software version checking.

#### Constructor Parameters

| Parameter | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `text` | `str` | Yes | - | Settings text (currently unused) |
| `theme` | `OrderedDict` | No | `{}` | Theme dictionary for text styling |

#### Key Bindings

| Key | Action | Description |
| --- | --- | --- |
| `q, Q, s, S` | `quit` | Close settings screen |
| `escape` | `esc_screen` | Close settings screen |

#### Methods

| Method | Parameters | Return Type | Description |
| --- | --- | --- | --- |
| `compose()` | None | `ComposeResult` | Yields Grid with heading, switches, and labels |
| `select_changed(event)` | `event: Switch.Changed` | `None` | Handles switch toggle events |
| `action_setting()` | None | `None` | Pops screen to return to app |
| `action_esc_screen()` | None | `None` | Pops screen to return to app |
| `action_quit()` | None | `None` | Pops screen to return to app (duplicate definitions) |
| `action_help()` | None | `None` | Pops screen to return to app |

#### Composition Structure

```
Grid (id="settings_menu_box")
├── Label (id="heading") - "TT-SMI Settings"
├── Label (id="instruction") - Navigation instructions
├── Label (id="dark_label") - "o Dark Mode:"
├── Switch (id="dark_switch") - Dark mode toggle
├── Label (id="sw_label") - "o Latest SW versions:"
└── Switch (id="sw_switch") - Software version checking toggle
```

#### Switch Behavior

**Dark Mode Switch (id="dark_switch"):**

*   Reads initial value from `app.dark`
*   Updates `app.dark` property on change
*   Controls application dark mode state

**Software Version Switch (id="sw_switch"):**

*   Reads initial value from `app.get_latest_sw_vers`
*   Updates `app.get_latest_sw_vers` property on change
*   Controls whether app fetches latest software versions

#### Switch Change Handler

The `@on(Switch.Changed)` decorator handles switch toggle events:

*   Event source identified by `event.switch.id`
*   Switch value read from `event.switch.value` (boolean)
*   Corresponding app property updated based on switch state

#### CSS Selectors

*   `TTSettingsMenu` - Modal screen selector
*   `#settings_menu_box` - Grid container
*   `#heading`, `#instruction` - Label selectors
*   `#dark_label`, `#sw_label` - Switch label selectors
*   `#dark_switch`, `#sw_switch` - Switch selectors
*   `Switch>.switch--slider` - Switch slider styling
*   `Switch.-on>.switch--slider` - Active switch styling
