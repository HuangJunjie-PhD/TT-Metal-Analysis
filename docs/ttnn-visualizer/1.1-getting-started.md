This guide provides instructions for installing, configuring, and using the TT-NN Visualizer - a tool designed to analyze and visualize Tenstorrent Neural Network (TT-NN) models. The visualizer provides detailed insights into model operations, memory usage, tensor allocations, and performance metrics.

For detailed information about specific visualization features, consult the dedicated pages for [Operation Analysis](https://deepwiki.com/tenstorrent/ttnn-visualizer/4-operation-analysis), [Memory Visualization](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.3-memory-visualization), and [Performance Analysis](https://deepwiki.com/tenstorrent/ttnn-visualizer/5-performance-analysis).

## Prerequisites

Before using TT-NN Visualizer, you need data generated from TT-NN for analysis. This data is generated when running TT-NN models with profiling enabled.

### Required Data

1.   **Memory Report**: A folder containing at minimum:

    *   `config.json` - Configuration information
    *   `db.sqlite` - SQLite database with operation and memory information

2.   **Performance Report** (optional): A folder containing:

    *   `profile_log_device.csv` - Core device logs
    *   Performance results CSV (e.g., `ops_perf_results_*.csv`) - Operation performance metrics
    *   `tracy_profile_log_host.tracy` - Tracy profiling data

3.   **NPE Data** (optional): Data generated with tt-npe for Network-on-chip performance estimation.

### Generating Data

To generate the necessary data, you'll need to set up and use [TT-Metalium](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/TT-Metalium) and [TT-NN](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/TT-NN)

Recommended PyTest configuration for generating profiling data:

`export TTNN_CONFIG_OVERRIDES='{    "enable_fast_runtime_mode": false,    "enable_logging": true,    "report_name": "YOUR_REPORT_NAME",    "enable_graph_report": false,    "enable_detailed_buffer_report": true,    "enable_detailed_tensor_report": false,    "enable_comparison_mode": false}'`
To run a test with custom input data:

`pytest --disable-warnings --input-path="path/to/input.json" path/to/test_file.py::test_function[param]`

## Running the Application

After installation, start the TT-NN Visualizer with:

`ttnn-visualizer`
The application will automatically open in your default web browser at `http://localhost:8000`.

### Command-line Options

The following command-line options are available:

| Option | Description |
| --- | --- |
| `--profiler-path` | Specify the local path to the folder containing profiler data |
| `--report-path` | Specify the local path to the folder containing the report |

Example usage:

`ttnn-visualizer --report-path ~/reports/ttnn/17274205533343344897 --profiler-path ~/reports/profiler/2025_02_24_23_17_27`
Sources: [docs/getting-started.md 86-127](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/docs/getting-started.md?plain=1#L86-L127)

### Environment Configuration

You can customize the application behavior by creating a `.env` file in the current directory with the following settings:

```
# Frontend
VITE_API_ROOT=http://localhost:8000/api

# Backend
HOST=localhost
PORT=8000
LAUNCH_BROWSER_ON_START=true
WEB_CONCURRENCY=1
FLASK_ENV=production
```

## Data Loading Methods

The visualizer can load data from your local machine or remotely via SSH. The diagram below illustrates the components involved in the data loading process:

### Loading Local Data

1.   Click the "Local Folder" button in the application
2.   Browse to the directory containing your report or profiler data
3.   Select the directory and click "Open"

The application expects profiler and performance data to be in their own separate directories, each containing all the required files.

### Loading Remote Data

1.   Click the "Remote Sync" button in the application
2.   Enter the SSH connection details: 
    *   Host: Remote server hostname or IP address
    *   Username: Your SSH username
    *   Password or SSH Key: Authentication credentials
    *   Remote Path: Path to the data directory on the remote server

3.   Click "Test Connection" to verify connectivity
4.   Click "Sync" to download the data from the remote server

You can have multiple sets of profiler data on the remote paths, but they must be organized in separate folders.

## Main Application Components

Once data is loaded, you can navigate between different views to analyze your model:

1.   **Operation List**:

    *   Displays all operations in the model
    *   Allows filtering by name, type, or attributes
    *   Enables sorting by various metrics
    *   Click on an operation to view detailed information

2.   **Operation Details**:

    *   Shows L1 and DRAM memory plots
    *   Displays tensor allocation details
    *   Presents device operations
    *   Visualizes input and output tensors

3.   **Performance Analysis**:

    *   Shows operation performance metrics
    *   Displays performance charts
    *   Provides resource utilization data

4.   **Buffer Summary**:

    *   Offers an overview of all buffers for the entire model run

5.   **NPE**:

    *   Network-on-chip Performance Estimator visualization
    *   Available on the `/npe` route

For more detailed information on specific features, refer to the other sections in this documentation:

## Troubleshooting

If you encounter any issues with the TT-NN Visualizer, check the following:

1.   **Python Version**: Ensure you're using Python 3.10 or higher

`python --version`
2.   **Package Installation**: Verify that the package is correctly installed

`pip list | grep ttnn-visualizer`
3.   **Browser Issues**: If the application doesn't open automatically, try navigating to `http://localhost:8000` manually

4.   **Version Conflicts**: If you've installed TT-NN Visualizer both globally and in a virtual environment, version conflicts may occur. Ensure you're using the correct environment.

5.   **Port Conflicts**: If port 8000 is already in use, you may need to configure a different port using the `.env` file
