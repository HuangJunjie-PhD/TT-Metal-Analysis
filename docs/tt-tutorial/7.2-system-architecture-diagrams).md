## Purpose and Scope

This page provides visual documentation of key system architectures, data flows, and operational concepts within the Tenstorrent software stack. The diagrams focus primarily on the dispatch system architecture, kernel execution patterns, and hardware abstraction layers that enable efficient operation on Tenstorrent hardware platforms.

For detailed analysis of dispatch optimization techniques and performance considerations, see [Dispatch System and Optimization](https://deepwiki.com/tenstorrent/tt-tutorial/7.1-dispatch-system-and-optimization). For lower-level programming concepts, see [TT-Metalium Programming Model](https://deepwiki.com/tenstorrent/tt-tutorial/4.1-tt-metalium-programming-model).

## Key Data Structures

| Component | Key Classes/Functions | Purpose |
| --- | --- | --- |
| Device Management | `tt::tt_metal::Device` | Hardware abstraction and resource management |
| Program Execution | `tt::tt_metal::Program` | Container for kernels and execution context |
| Memory Management | `tt::tt_metal::Buffer` | Memory allocation and data transfer |
| Kernel Definition | `tt::tt_metal::CreateKernel` | Kernel compilation and configuration |
| Command Queue | `tt::tt_metal::CommandQueue` | Asynchronous command submission |
| Circular Buffers | `tt::CBHandle` | L1 memory management for data streaming |
