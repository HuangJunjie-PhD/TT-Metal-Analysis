## Purpose and Scope

The Benchmarking System provides a standardized framework for measuring the performance and validating the functionality of neural network models on Tenstorrent hardware. It enables consistent evaluation of different model architectures, configurations, and optimization strategies across various hardware targets. This system is designed to be used by both internal developers and external users to understand relative performance characteristics and identify optimization opportunities.

For model-specific implementations, see [CNN Models](https://deepwiki.com/tenstorrent/tt-buda/4.1-cnn-models) and [NLP Models](https://deepwiki.com/tenstorrent/tt-buda/4.2-nlp-models). For testing functionality rather than performance, see the [Testing Framework](https://deepwiki.com/tenstorrent/tt-buda/3-testing-framework).

### Key Parameters

The benchmark model functions accept a standard set of parameters:

| Parameter | Description |
| --- | --- |
| `training` | Boolean indicating whether to benchmark in training or inference mode |
| `config` | String specifying the model configuration variant to benchmark |
| `microbatch` | Integer specifying the batch size for the benchmark |
| `devtype` | String specifying the device type (e.g., "grayskull", "wormhole") |
| `arch` | String specifying the architecture variant |
| `data_type` | String specifying the data format (e.g., "Fp16_b", "Bfp8_b") |
| `math_fidelity` | String specifying the computation precision (e.g., "HiFi3") |

### Environment Variables

The benchmarking system uses environment variables to fine-tune hardware behavior:

| Environment Variable | Purpose | Example |
| --- | --- | --- |
| `PYBUDA_ENABLE_DRAM_IO_BUFFER_SCALING` | Enable DRAM I/O buffer scaling | Used for Bfp8 on Wormhole B0 |
| `PYBUDA_ENABLE_INPUT_BUFFER_SCALING_FOR_NOC_READERS` | Enable buffer scaling for network-on-chip readers | Used for Bfp8 on Wormhole B0 |
| `PYBUDA_RIBBON2_CALCULATE_TARGET_CYCLES` | Enable target cycle calculation for Ribbon balancer | Common setting |
| `PYBUDA_FORK_JOIN_BUF_QUEUES` | Enable fork-join buffer queues | Used for certain models |
| `PYBUDA_ALLOW_MULTICOLUMN_SPARSE_MATMUL` | Allow multi-column sparse matrix multiplication | Used for certain models |
| `PYBUDA_SUPRESS_T_FACTOR_MM` | Suppress T-factor for matrix multiplication | Used with a numeric value |
| `PYBUDA_ENABLE_HOST_INPUT_NOP_BUFFERING` | Enable host input NOP buffering | Common setting |

## Model-Specific Optimizations

Different model architectures require specific optimizations to achieve optimal performance on Tenstorrent hardware.

### CNN Optimizations

For CNN models like ResNet, MobileNet, and YOLOv5:

*   Convolution operation fracturing: `PYBUDA_FORCE_CONV_MULTI_OP_FRACTURE`
*   Buffer scaling for DRAM I/O and NOC readers
*   T-factor suppression for matrix multiplications
*   Ribbon balancer policy with target cycle calculation

Sources:

*   [pybuda/test/benchmark/benchmark/models/resnet.py 76-78](https://github.com/tenstorrent/tt-buda/blob/74755d1f/pybuda/test/benchmark/benchmark/models/resnet.py#L76-L78)

### Transformer Optimizations

For transformer models like BERT, ViT, and T5:

*   Exponential approximation: `PYBUDA_EXP_APPROX`
*   Mixed precision for specific operations (add, subtract, reciprocal)
*   Fork-join buffer queues
*   Auto-transposing placement
*   Epoch breaks for layer-wise execution

## Hardware-Specific Configurations

The benchmarking system includes specific configurations for different Tenstorrent hardware:

### Grayskull

*   CPU fallback for certain tensor manipulations
*   Decomposition of complex operations (e.g., sigmoid)
*   Auto-fusing disabled for some models
*   Specific epoch breaks and buffer configurations

### Wormhole B0

*   DRAM I/O buffer scaling
*   Input buffer scaling for NOC readers
*   Bandwidth estimation for balancer
*   Fork-join buffer configurations
*   Device-specific mixed precision settings

## Adding New Models to the Benchmark System

To add a new model to the benchmarking system:

1.   Create a new Python file in the `pybuda/test/benchmark/benchmark/models/` directory
2.   Define a function decorated with `@benchmark_model` that accepts the standard parameters
3.   Implement model loading, configuration, and input generation
4.   Return a tuple containing the models dictionary, inputs, targets, and extra parameters
