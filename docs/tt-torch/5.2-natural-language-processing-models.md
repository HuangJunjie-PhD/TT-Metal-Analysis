This document details the Natural Language Processing (NLP) models tested in tt-torch, including their integration patterns, test configurations, and execution modes on Tenstorrent hardware. For information about Computer Vision models, see page 5.1. For specialized models, see page 5.3.

## Supported NLP Models Overview

tt-torch tests four primary NLP models focusing on causal language modeling and code generation tasks. All models integrate through `ModelLoader` classes from `third_party.tt_forge_models` or direct Hugging Face Transformers integration.

## Model-Specific Implementation Details

### CodeGen

CodeGen is a code generation model from Salesforce that generates code from natural language prompts. The model uses causal language modeling trained specifically on programming languages.

**CodeGen Test Implementation**

Key implementation details:

*   `ModelLoader` from `third_party.tt_forge_models.codegen.pytorch`
*   Loads model with `dtype_override=torch.bfloat16`
*   Batch size of 2 for input generation
*   Tokenizer accessed via `self.tokenizer = self.loader.tokenizer`
*   Extracts logits and decodes: `logits = results.logits if hasattr(results, "logits") else results[0]`
*   Uses `torch.argmax(logits, dim=-1)` to get token IDs
*   Decodes with `tokenizer.batch_decode(token_ids, skip_special_tokens=True)[0]`

Test configuration:

*   `run_generate=False` - disables text generation mode
*   `assert_atol=False` - skips absolute tolerance checking
*   `assert_pcc=True` - enables PCC verification
*   Supports all three execution modes: op-by-op TORCH, op-by-op STABLEHLO, full compilation

### OPT (Open Pre-trained Transformer)

OPT is Facebook's open-source causal language model. The test uses the 350M parameter variant with direct Hugging Face Transformers integration.

**OPT Test Flow**

Key implementation details:

*   Direct Hugging Face integration without `ModelLoader`
*   Model: `OPTForCausalLM.from_pretrained("facebook/opt-350m", torch_dtype=torch.bfloat16)`
*   Tokenizer: `GPT2Tokenizer.from_pretrained("facebook/opt-350m")`
*   Input prompt: `"A chat between a curious"`
*   Returns tokenized inputs: `self.tokenizer([prompt], return_tensors="pt")`
*   Calls `test_model(assert_eval_token_mismatch=False)` to disable token mismatch checking

Test configuration:

*   `relative_atol=0.015` - allows 1.5% relative tolerance
*   `assert_pcc=True` - enables PCC verification
*   `assert_atol=False` - skips absolute tolerance checking
*   Supports all three execution modes

Sources: [tests/models/opt/test_opt.py 1-59](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/opt/test_opt.py#L1-L59)

### Bloom

Bloom is a multilingual causal language model supporting 46 natural languages and 13 programming languages. Uses `ModelLoader` for flexible variant support.

**Bloom Test Architecture**

Key implementation details:

*   `ModelLoader` from `third_party.tt_forge_models.bloom.pytorch`
*   Both model and inputs loaded with `dtype_override=torch.bfloat16`
*   Uses `loader.decode_output(results)` for output decoding
*   Prints input/output pairs from `loader.test_input` and decoded output
*   Iterates through results: `for i, (inp, out) in enumerate(zip(loader.test_input, decoded_output))`

Test configuration:

*   `relative_atol=0.01` - allows 1% relative tolerance
*   `assert_pcc=True` - enables PCC verification
*   `assert_atol=False` - skips absolute tolerance checking
*   Supports all three execution modes: op-by-op TORCH, op-by-op STABLEHLO, full compilation

Sources: [tests/models/bloom/test_bloom.py 1-63](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tests/models/bloom/test_bloom.py#L1-L63)

### Mamba

Mamba is a state-space sequence model that provides an efficient alternative to Transformer architectures for causal language modeling. Supports multiple model size variants.

**Mamba Variant Support**

Key implementation details:

*   `ModelLoader` from `third_party.tt_forge_models.mamba.pytorch.loader`
*   Queries available variants: `ModelLoader.query_available_variants()` at module level
*   Prints variants: `print("Available variants: ", [str(k) for k in available_variants.keys()])`
*   Loads model with `dtype_override=torch.bfloat16`
*   Tokenizer stored: `self.tokenizer = self.loader.tokenizer`
*   Handles two output formats: `results.logits` attribute or `results[0]` tuple indexing
*   Generates text with argmax and batch decode

Test configuration:

*   `run_generate=False` - disables text generation mode
*   `required_pcc=0.95` - requires 95% PCC
*   `assert_atol=False` - skips absolute tolerance checking
*   Supports all three execution modes
*   Each variant tested separately via pytest parameterization

## Testing Framework Integration

All NLP models extend `ModelTester` base class and use `CompilerConfig` for execution control. Tests follow consistent patterns with `ModelLoader` integration or direct Hugging Face usage.

**Test Framework Class Structure**

### Test Execution Modes

All NLP model tests support three execution modes via pytest parameterization:

| Mode | CompilerConfig Settings | Usage |
| --- | --- | --- |
| **op_by_op_stablehlo** | `cc.compile_depth = CompileDepth.EXECUTE_OP_BY_OP` `cc.op_by_op_backend = OpByOpBackend.STABLEHLO` | Per-operation execution with StableHLO IR |
| **op_by_op_torch** | `cc.compile_depth = CompileDepth.EXECUTE_OP_BY_OP` `cc.op_by_op_backend = OpByOpBackend.TORCH` | Per-operation execution with PyTorch backend |
| **full** | No compile depth override | Full graph compilation and execution |

## Verification Methods

The NLP models in tt-torch employ specific verification methods to ensure correctness:

1.   **PCC (Pearson Correlation Coefficient)**: Measures the correlation between reference and tt-torch model outputs
2.   **ATOL (Absolute Tolerance)**: Verifies that differences between reference and tt-torch model outputs are within an acceptable tolerance
3.   **Model-specific verification**: Some models have additional verification steps, such as: 
    *   Text matching for Whisper and MGP-STR
    *   Token prediction validation for Perceiver IO
    *   Class prediction validation for SqueezeBERT

## Summary

tt-torch provides comprehensive support for a variety of NLP models, enabling their execution on Tenstorrent hardware. The testing framework ensures that these models produce accurate results compared to their reference implementations. While not all models pass strict verification checks yet (as indicated by TODOs in the test files), active development is ongoing to improve compatibility and performance across all supported models.

Future work includes:

*   Expanding support to more NLP model architectures
*   Improving verification accuracy for existing models
*   Enabling training mode for all supported models
*   Optimizing performance for large language models
