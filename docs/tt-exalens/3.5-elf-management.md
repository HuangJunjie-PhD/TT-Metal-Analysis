This page documents the ELF management API in the `ttexalens` Python library: the `parse_elf`, `load_elf`, and `run_elf` functions, and the `ElfLoader` class that implements ELF-to-device writing. It covers how ELF files are parsed, loaded into RISC-V cores, executed, and how multiple core locations can be targeted simultaneously.

For symbolic variable access on a parsed ELF (reading global variables, struct members, etc.), see [Symbolic Variable Access](https://deepwiki.com/tenstorrent/tt-exalens/3.8-symbolic-variable-access). For callstack unwinding using ELF debug symbols, see [Debugging and Callstacks](https://deepwiki.com/tenstorrent/tt-exalens/3.7-debugging-and-callstacks). For the lower-level ELF parsing and DWARF infrastructure, see [ELF and DWARF Parsing](https://deepwiki.com/tenstorrent/tt-exalens/7.3-elf-and-dwarf-parsing). For the ELF Loader implementation detail and memory segment routing, see [ELF Loader Implementation](https://deepwiki.com/tenstorrent/tt-exalens/6.6-elf-loader-implementation).

* * *

## API Overview

Three library functions form the public ELF management interface, all defined in [ttexalens/tt_exalens_lib.py](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py):

| Function | Purpose | Core state required |
| --- | --- | --- |
| `parse_elf` | Parse an ELF file from disk into a `ParsedElfFile` object | None |
| `load_elf` | Write ELF segments to a core's memory | Core **must** be in reset |
| `run_elf` | Write ELF segments and release reset to begin execution | Core may be in any state (reset is forced, then released) |

* * *

## `parse_elf`

[ttexalens/tt_exalens_lib.py 575-584](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L575-L584)

Reads an ELF file from disk and returns a `ParsedElfFile` object. This is the entry point for loading debug symbol information without writing anything to hardware.

| Parameter | Type | Description |
| --- | --- | --- |
| `elf_path` | `str` | Path to the ELF file on disk |
| `context` | `Context | None` | TTExaLens context; uses global if `None` |
| `require_debug_symbols` | `bool` | If `True`, raises if no DWARF debug info is present |

The returned `ParsedElfFile` has lazily-evaluated cached properties for `variables`, `types`, `enumerators`, `symbols`, `file_line`, `frame_info`, and `sections`. It can be passed directly to `load_elf`, `run_elf`, `callstack`, and other functions that accept an ELF.

**Sources:**[ttexalens/tt_exalens_lib.py 575-584](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L575-L584)[ttexalens/elf/parsed.py 145-250](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L145-L250)[ttexalens/elf/__init__.py](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/__init__.py)

## `load_elf`

[ttexalens/tt_exalens_lib.py 295-362](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L295-L362)

Writes the loadable ELF sections to the specified RISC-V core(s). **The core must already be in reset** before calling `load_elf`; the function asserts this at the `ElfLoader` level.

### Key Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| `elf_file` | `str | ParsedElfFile` | Path to ELF file or pre-parsed object |
| `location` | `str | OnChipCoordinate | list | "all"` | Target core location(s); see [Location Targeting](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/Location%20Targeting) |
| `risc_name` | `str` | Core name: `"brisc"`, `"trisc0"`, `"trisc1"`, `"trisc2"`, `"ncrisc"`, `"erisc"`, etc. |
| `neo_id` | `int | None` | NEO block ID for Quasar devices |
| `return_start_address` | `bool` | If `True`, returns the `.init` section address instead of `None` |
| `verify_write` | `bool` | If `True`, reads back each written block to confirm correctness |

### Return Value

*   If `return_start_address=False` (default): returns `None`.
*   If `return_start_address=True` and a single location: returns `int` (the `.init` section address).
*   If `return_start_address=True` and multiple locations: returns `list[int]`.

**Sources:**[ttexalens/tt_exalens_lib.py 295-362](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L295-L362)[ttexalens/elf_loader.py 201-217](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L201-L217)

## `run_elf`

[ttexalens/tt_exalens_lib.py 366-420](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L366-L420)

Combines ELF loading and execution start. Internally calls `ElfLoader.run_elf`, which:

1.   Puts the core in reset (if not already).
2.   Calls `load_elf` to write ELF segments.
3.   Releases reset via `set_reset_signal(False)`.
4.   Asserts the core is no longer in reset and not halted (unless an `ebreak` was hit).

`run_elf` accepts the same `location`, `risc_name`, and `verify_write` parameters as `load_elf`, but does **not** have a `return_start_address` option.

**Sources:**[ttexalens/tt_exalens_lib.py 366-420](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/tt_exalens_lib.py#L366-L420)[ttexalens/elf_loader.py 219-232](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L219-L232)

## Location Targeting

Both `load_elf` and `run_elf` accept several forms for the `location` argument, enabling single-core or multi-core targeting in a single call.

**Location forms:**

| Value | Behavior |
| --- | --- |
| `"X-Y"` (NOC0/translated string) | Single core |
| `"X,Y"` (logical string) | Single core |
| `OnChipCoordinate` object | Single core |
| `list[str | OnChipCoordinate]` | All listed cores, types may be mixed |
| `"all"` | All `functional_workers` blocks on the device |

When `"all"` is used, the function calls `device.get_block_locations(block_type="functional_workers")` to enumerate every functional worker core.

## The `ElfLoader` Class

[ttexalens/elf_loader.py 12-232](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L12-L232)

`ElfLoader` is the internal class that performs the actual memory write operations. It is instantiated per-core per-call by the library functions.

**Constructor:**

Takes a `RiscDebug` instance for the target core, from which it obtains the `OnChipCoordinate` (for NOC writes) and a `MemoryAccess` object (for debug-interface writes to private memory).

### Sections Written

`ElfLoader` writes only the sections listed in `SECTIONS_TO_LOAD`:

```
[".init", ".text", ".ldm_data", ".gcov_info"]
```

[ttexalens/elf_loader.py 23](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L23-L23)

Other ELF sections (e.g., `.debug_*`, `.bss`) are not written to the device.

### Address Remapping

`remap_address`[ttexalens/elf_loader.py 124-141](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L124-L141) handles private address relocation. If a section's address falls within private data or code memory and a loader section (`.loader_init` or `.loader_code`) is present in the ELF, the section is remapped to the loader's address rather than the private address. This allows ELFs compiled for private addresses to be loaded via the debug interface.

### `verify_write`

When `verify_write=True` (the default), after writing each section the loader reads it back and compares:

```
read_data = self.read_block(address, len(section.data))
if read_data != section.data:
    util.ERROR(...)
```

[ttexalens/elf_loader.py 185-192](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L185-L192)

Mismatches are logged but do not raise exceptions by default.

### Start Address and Code Execution

`load_elf` tracks the address of the `.init` section. If `return_start_address=False`, it calls `risc_debug.set_code_start_address(init_section_address)` to configure the program counter. If `return_start_address=True`, the address is returned instead for the caller to use.

**Sources:**[ttexalens/elf_loader.py 143-232](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf_loader.py#L143-L232)

## `ParsedElfFile`

[ttexalens/elf/parsed.py 145-250](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L145-L250)

`ParsedElfFile` is the data model for a parsed ELF. It is the return type of `parse_elf` and is accepted by `load_elf`, `run_elf`, `callstack`, and `top_callstack`. All properties are lazy (`@cached_property`) — parsing is deferred until first access.

**Key properties:**

| Property | Type | Description |
| --- | --- | --- |
| `sections` | `list[ParsedElfFileSection]` | All ELF sections with `.name`, `.address`, `.data` |
| `symbols` | `dict[str, ElfDwarfSymbol]` | Symbol table entries (name → value, size) |
| `variables` | `dict[str, ElfDie]` | DWARF global variables by name |
| `types` | `dict[str, ElfDie]` | DWARF type definitions by name |
| `enumerators` | `dict[str, ElfDie]` | DWARF enum values by name |
| `file_line` | `dict[int, tuple[str, int, int]]` | PC → (filename, line, column) mapping |
| `frame_info` | `FrameInfoProvider` | DWARF CFI for callstack unwinding |

`ParsedElfFile` wraps a `pyelftools``ELFFile` and a path string. Passing a pre-parsed `ParsedElfFile` to `load_elf` or `run_elf` avoids re-reading and re-parsing the file from disk on every call.

**Sources:**[ttexalens/elf/parsed.py 133-250](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L133-L250)[ttexalens/elf/__init__.py](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/__init__.py)

## Options Summary

| Option | Available on | Default | Effect |
| --- | --- | --- | --- |
| `verify_write` | `load_elf`, `run_elf` | `True` | Read back each written section and log mismatches |
| `return_start_address` | `load_elf` | `False` | Return `.init` section address instead of `None` |
| `require_debug_symbols` | `parse_elf` | `True` | Raise if the ELF lacks DWARF debug info |
| `neo_id` | `load_elf`, `run_elf` | `None` | NEO block index for Quasar multi-NEO cores |
