The `run_reports.py` pipeline is the final stage of the `tt-inference-server` workflow. It aggregates raw performance and accuracy data from benchmarks and evaluations, compares them against theoretical targets defined in the codebase, and generates human-readable Markdown and CSV reports. This system ensures that every model release meets "Functional", "Complete", or "Sellable" acceptance criteria.

## Reporting Pipeline Architecture

The reporting system acts as a post-processing layer that consumes JSON artifacts produced by `run_benchmarks.py` and `run_evals.py`. It also integrates with the `tt-inference-server-v2``report_module` for structured media reports.

## Performance Reference and Scaling

Performance targets are stored in `model_performance_reference.json`. These targets represent theoretical hardware capabilities for specific `DeviceTypes` (e.g., N150, T3K, Galaxy).

### Theoretical Target Loading

The function `read_performance_reference_json` loads the target database from `benchmarking/benchmark_targets/model_performance_reference.json`[workflows/model_spec.py 65-76](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/model_spec.py#L65-L76) The `get_perf_reference_map` function then maps these targets to `BenchmarkTaskParams` objects, applying scaling factors based on customer-defined percentages (e.g., functional vs sellable) [workflows/model_spec.py 82-129](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/model_spec.py#L82-L129)

### Data Parallel Scaling

For multi-device configurations (e.g., Galaxy), throughput targets must be scaled. The `scale_llm_perf_targets` function multiplies the `tput` (throughput) by the `data_parallel` factor while maintaining the same `ttft_ms` (Time to First Token) target [workflows/model_spec.py 132-161](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/model_spec.py#L132-L161)

## Acceptance Criteria Implementation

Acceptance is determined by comparing measured metrics against the `PerformanceTarget` dataclass.

| Metric | Description | Logic Site |
| --- | --- | --- |
| **TTFT** | Time to First Token (ms). Must be $\le$ target. | `acceptance_criteria.py` |
| **TPUT** | Total throughput (tokens/sec). Must be $\ge$ target. | `acceptance_criteria.py` |
| **TPUT_USER** | Per-user throughput (tokens/sec/user). Must be $\ge$ target. | `acceptance_criteria.py` |
| **Accuracy** | Model-specific score (e.g., MMLU, FID). Must be within tolerance. | `evals/eval_config.py` |
| **RTR** | Real-Time Ratio (for Audio/TTS). Must be $\le$ target. | `tts_client.py` |

### Criteria Levels

The system supports different acceptance thresholds defined by percentages of the theoretical maximum [workflows/run_reports.py 56-57](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/run_reports.py#L56-L57):

*   **Functional**: Basic operation, often 10% of theoretical.
*   **Complete**: Optimized performance, often 50% or higher.
*   **Sellable**: Production-ready performance.

## Report Generation Logic

The `run_reports.py` script orchestrates the generation of various report types based on the `ModelType`.

### Media Task Reporting

For non-LLM models, specific helper functions resolve the result file patterns and metrics:

*   **Audio**: `generate_audio_report_data` locates `*_results.json` from `lmms-eval`[workflows/run_reports.py 75-87](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/run_reports.py#L75-L87)
*   **TTS**: `TtsClientStrategy` calculates `ttft_value`, `rtr_value` (Real-Time Ratio), and tail latencies (`p90_ttft`, `p95_ttft`) [utils/media_clients/tts_client.py 81-91](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/tts_client.py#L81-L91)
*   **CNN**: `CnnClientStrategy` handles accuracy checks for models like MobileNetV2 and generates `results_*.json`[utils/media_clients/cnn_client.py 74-111](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/utils/media_clients/cnn_client.py#L74-L111)
*   **Image/Video**: Specific generators handle SDXL, Flux, and video models [workflows/run_reports.py 105-131](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/run_reports.py#L105-L131)

### Structured Media Reports (V2)

The `tt-inference-server-v2``report_module` provides a structured framework for media benchmarks:

*   **ReportSchema**: Defines the structure for performance and accuracy data.
*   **Block**: Represents individual sections of a report (e.g., a table or summary).
*   **ReportGenerator**: Orchestrates the assembly of blocks into final Markdown/CSV formats.

*   `main()`: Entry point that parses arguments, loads the `ModelSpec`, and calls the appropriate report generators [workflows/run_reports.py 221-230](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/run_reports.py#L221-L230)

### `summary_report.py`

*   `get_markdown_table()`: Converts a list of benchmark results into a formatted Markdown table [benchmarking/summary_report.py 25](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/benchmarking/summary_report.py#L25-L25)
