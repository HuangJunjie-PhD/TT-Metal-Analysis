# Elasticsearch Integration

Relevant source files
*   [pyproject.toml](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/pyproject.toml)
*   [tt_topology/log.py](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/log.py)

This document covers the Elasticsearch integration system in tt-topology, which provides structured logging and data storage capabilities for topology configuration operations. The integration enables centralized log storage, searchability, and analysis of hardware configuration events and diagnostic data.

For information about the general logging format and data structures, see [Log Format](https://deepwiki.com/tenstorrent/tt-topology/6.1-log-format). For troubleshooting log-related issues, see [Troubleshooting](https://deepwiki.com/tenstorrent/tt-topology/8-troubleshooting).

## Purpose and Architecture

The Elasticsearch integration serves as the backend storage system for structured logs generated during topology configuration operations. It transforms Python data models into Elasticsearch-compatible documents and provides type mapping for efficient storage and querying.

## Configuration and Setup

### Dependencies

The Elasticsearch integration requires specific package versions as defined in the project configuration:

```
elasticsearch==8.11.0
pydantic>=1.2
```

The system includes backward compatibility for both Pydantic v1 and v2 through conditional imports in [tt_topology/log.py 14-21](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/log.py#L14-L21)

### Pydantic Compatibility Layer

The integration handles Pydantic version differences through a try-catch import mechanism:

`try:    # Try the newer v2 pydantic and use that first    from pydantic.v1 import BaseModel    from pydantic.v1.fields import Fieldexcept:    # Assume we are on v1 and give that a go    from pydantic import BaseModel    from pydantic.fields import Field`
**Sources:**[pyproject.toml 28-29](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/pyproject.toml#L28-L29)[tt_topology/log.py 14-21](https://github.com/tenstorrent/tt-topology/blob/f86e46c6/tt_topology/log.py#L14-L21)

## Elasticsearch Field Types

### Custom Type Classes

The integration defines custom type classes that map to specific Elasticsearch field types:

| Custom Class | Base Type | Elasticsearch Mapping |
| --- | --- | --- |
| `Long` | `int` | `{"type": "long"}` |
| `Keyword` | `str` | `{"type": "keyword"}` |
| `Text` | `str` | `{"type": "text"}` |
| `Date` | `datetime.datetime` | `{"type": "date", "format": custom}` |

### Nested Object Support

The system supports nested objects through the `Nested` generic type, which converts object mappings to nested Elasticsearch documents:

`class Nested(list, Generic[T]):    ...`
