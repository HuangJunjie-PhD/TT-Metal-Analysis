This page documents the standardized Python virtual environment configuration used across TVM's CI infrastructure and recommended for local development. The setup ensures reproducible builds through pinned dependencies, SHA256-verified package installations, and isolated environments.

For information about the Docker images that use this Python environment, see [Docker CI Environments](https://deepwiki.com/tenstorrent/tt-tvm/10.2-docker-ci-environments). For details on how tests are executed within this environment, see [Testing Framework](https://deepwiki.com/tenstorrent/tt-tvm/10.4-testing-framework).

* * *

## Overview

TVM's CI infrastructure uses a consistent Python virtual environment configuration across all Docker images to ensure:

*   **Reproducibility**: Exact package versions with SHA256 hash verification
*   **Isolation**: Virtual environments prevent conflicts with system Python packages
*   **Consistency**: Same environment across CPU, GPU, ARM, and embedded target builds
*   **Security**: Hash-verified package downloads prevent supply chain attacks

The Python environment is established during Docker image builds and activated for all CI operations.

## Dependency Management with Lockfiles

TVM uses a constraint-based dependency management system with SHA256 hash verification to ensure reproducible builds.

### Lockfile Structure

**Sources:**[docker/install/ubuntu_install_python.sh 98-109](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L98-L109)[docker/Dockerfile.ci_cpu 38](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_cpu#L38-L38)

### Bootstrap Packages

The constraint files contain pinned versions with SHA256 hashes for core packages:

| Package | Purpose | Verified Installation |
| --- | --- | --- |
| `pip` | Package installer | `--require-hashes` with SHA256 |
| `setuptools` | Package build system | `--require-hashes` with SHA256 |
| Base dependencies | Build and runtime requirements | Installed from constraints file |

**Example from ubuntu_install_python.sh:**

```
pip_spec=$(cat /install/python/bootstrap/lockfiles/constraints-${PYTHON_VERSION}.txt | grep 'pip==')
pip3 install -U --require-hashes -r <(echo "${pip_spec}") \
     -c /install/python/bootstrap/lockfiles/constraints-${PYTHON_VERSION}.txt
```

**Sources:**[docker/install/ubuntu_install_python.sh 98-109](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L98-L109)

## Environment Variables

The Python environment is configured through key environment variables set in all Docker images:

### TVM_VENV

`ENV TVM_VENV /venv/apache-tvm-py3.8`
**Purpose:** Specifies the virtual environment location. Used by installation scripts and can be referenced by build tools.

**Sources:**[docker/Dockerfile.ci_cpu 37](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_cpu#L37-L37)[docker/Dockerfile.ci_gpu 44](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_gpu#L44-L44)[docker/Dockerfile.ci_arm 56](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_arm#L56-L56)

### PATH

`ENV PATH ${TVM_VENV}/bin:$PATH`
**Purpose:** Prepends the virtual environment's `bin` directory to PATH, ensuring that the venv Python and all installed scripts are used by default.

**Sources:**[docker/Dockerfile.ci_cpu 41](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_cpu#L41-L41)[docker/Dockerfile.ci_gpu 48](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_gpu#L48-L48)

### PYTHONNOUSERSITE

`ENV PYTHONNOUSERSITE 1`
**Purpose:** Disables Python's user site-packages directory (`~/.local/lib/pythonX.Y/site-packages`). This prevents locally installed packages from affecting CI builds and makes it easier to reproduce CI failures locally.

**Configuration in sudoers:**

`echo "Defaults env_keep += \"PYTHONNOUSERSITE\"" >/etc/sudoers.d/91-preserve-python-nousersite`
**Sources:**[docker/Dockerfile.ci_cpu 42](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_cpu#L42-L42)[docker/install/ubuntu_install_python.sh 80-82](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L80-L82)

## Security Considerations

### SHA256 Hash Verification

All pip packages in the constraint files are verified with SHA256 hashes:

`pip3 install -U --require-hashes -r <(echo "${pip_spec}") \     -c /install/python/bootstrap/lockfiles/constraints-${PYTHON_VERSION}.txt`
This prevents:

*   Supply chain attacks via compromised PyPI packages
*   Accidental installation of modified packages
*   Version confusion from registry updates

**Sources:**[docker/install/ubuntu_install_python.sh 99-100](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L99-L100)

### User Site-Packages Disabled

Setting `PYTHONNOUSERSITE=1` prevents Python from loading packages from:

*   `~/.local/lib/pythonX.Y/site-packages/`
*   User-specific installation directories

This ensures:

*   CI builds are not affected by developer's local packages
*   Reproducibility across different build environments
*   Easier debugging of CI failures locally

**Sources:**[docker/Dockerfile.ci_cpu 42](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_cpu#L42-L42)[docker/install/ubuntu_install_python.sh 80-82](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L80-L82)

### System Pip Removal

After creating the virtual environment, all system `pip` and `pip3` executables are removed:

`while [ "$(which pip3)" != "" ]; do    rm "$(which pip3)"donewhile [ "$(which pip)" != "" ]; do    rm "$(which pip)"done`
**Purpose:** Prevents accidental installation of packages to system Python, ensuring all package operations target the virtual environment.

**Sources:**[docker/install/ubuntu_install_python.sh 115-123](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L115-L123)

## Multi-User Support with ACL

The virtual environment is configured with POSIX Access Control Lists (ACLs) to support multi-user CI environments:

`addgroup tvm-venvchgrp -R tvm-venv "${TVM_VENV}"setfacl -R -d -m group:tvm-venv:rwx "${TVM_VENV}"setfacl -R -m group:tvm-venv:rwx "${TVM_VENV}"`
**Benefits:**

*   Multiple CI users can write to the venv
*   Shared cache directories work correctly
*   Enables parallel builds with different users

**Sources:**[docker/install/ubuntu_install_python.sh 110-113](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_python.sh#L110-L113)

1.   **Create virtual environment:**
