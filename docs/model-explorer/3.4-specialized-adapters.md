This document covers the specialized conversion utilities and dialect-specific adapters that extend Model Explorer's core conversion capabilities. These components handle advanced features like HLO computations, Shardy dialect attributes, and specialized printing utilities that work alongside the main model conversion adapters.

For general model conversion adapters (TFLite, TensorFlow, etc.), see [Model Conversion Adapters](https://deepwiki.com/tenstorrent/model-explorer/3.1-model-conversion-adapters). For the Python server that orchestrates these adapters, see [Python Server and APIs](https://deepwiki.com/tenstorrent/model-explorer/3.2-python-server-and-apis).

## HLO Direct Adapter

The HLO (High Level Operations) adapter provides specialized conversion from XLA HLO computations directly to Model Explorer's JSON graph format, bypassing the typical MLIR conversion path for performance and feature completeness.

### HLO Conversion Architecture

Sources: [src/builtin-adapter/hlo_adapter/direct_hlo_to_json_graph_convert.h 31-93](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/hlo_adapter/direct_hlo_to_json_graph_convert.h#L31-L93)[src/builtin-adapter/hlo_adapter/direct_hlo_to_json_graph_convert.cc 533-585](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/hlo_adapter/direct_hlo_to_json_graph_convert.cc#L533-L585)

### Key HLO Adapter Components

The HLO adapter consists of several specialized components:

| Component | Purpose | Key Functions |
| --- | --- | --- |
| `HloAdapterOption` | Configuration for conversion behavior | Controls GTE folding, async parameter hiding |
| `NodeFilter` | Lambda for instruction inclusion | `MakeInstructionRadiusAroundFilter()` |
| `ComputationExpand` | Lambda for computation expansion | Determines fusion computation handling |
| `GraphNodeBuilder` | Node construction utility | `BuildHloInstructionNode()` |

### HLO Specialized Features

**Get-Tuple-Element (GTE) Folding**: When enabled via `get_tuple_element_folding`, the adapter connects tuple producers directly to their consumers, annotating which tuple elements are accessed rather than showing intermediate GTE operations.

**Async Collective Fusion Parameter Hiding**: The adapter can detect and hide implementation detail parameters in async collective operations using the `IsAcfPrameter()` function, which identifies parameters that flow between `AsyncCollectiveStart` and `AsyncCollectiveDone` operations.

**Radius-Based Filtering**: For large graphs, the adapter supports neighborhood filtering around a specific instruction using `MakeInstructionRadiusAroundFilter()`, which includes instructions within a specified graph distance.

## Shardy Dialect Support

The Shardy utilities provide specialized support for the Shardy sharding dialect, including pretty printing of sharding attributes and extraction of propagation edge overlays for visualization.

### Shardy Processing Pipeline

Sources: [src/builtin-adapter/tools/shardy_utils.h 43-79](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/shardy_utils.h#L43-L79)[src/builtin-adapter/tools/shardy_utils.cc 148-181](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/shardy_utils.cc#L148-L181)[src/builtin-adapter/tools/shardy_utils.cc 318-359](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/shardy_utils.cc#L318-L359)

### Shardy Attribute Pretty Printing

The Shardy utilities provide simplified pretty printing for complex sharding attributes:

*   **TensorShardingAttr**: Displays sharding patterns like `<@mesh, [{}, {"c"}]>` instead of full MLIR syntax
*   **TensorShardingPerValueAttr**: Shows per-value shardings with indexed display
*   **ManualAxesAttr**: Displays manual axis specifications as `{"a", "b"}`
*   **PropagationEdgesAttr**: Shows propagation step information in readable format

The `PrintShardyAttribute()` function uses LLVM TypeSwitch to dispatch to specialized pretty printers for each Shardy attribute type.

Sources: [src/builtin-adapter/tools/shardy_utils.cc 64-146](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/shardy_utils.cc#L64-L146)

### Propagation Edge Extraction

Shardy propagation edges represent data flow constraints for sharding. The utilities extract these edges and convert them to Model Explorer's edge overlay format:

1.   `HasShardyPropagationEdges()` detects propagation edge attributes on operations
2.   `CollectEdgesForOperation()` extracts edge information and maps to node IDs
3.   `ExtractShardyPropagationEdges()` creates colored edge overlays grouped by axis
4.   `ColorMapper` assigns consistent colors to axes across the visualization

Sources: [src/builtin-adapter/tools/shardy_utils.cc 212-237](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/shardy_utils.cc#L212-L237)[src/builtin-adapter/tools/shardy_utils.cc 276-316](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/shardy_utils.cc#L276-L316)

### Color Management System

The `ColorMapper` class manages consistent color assignment for Shardy axes, with support for sub-axis color jittering to distinguish related axes visually.

## Attribute Printing Utilities

The attribute printer provides specialized rendering for MLIR attributes, with particular handling for dense tensors, strings, and Shardy dialect attributes.

### Attribute Printing Architecture

Sources: [src/builtin-adapter/tools/attribute_printer.cc 187-200](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/attribute_printer.cc#L187-L200)[src/builtin-adapter/tools/attribute_printer.cc 176-185](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/attribute_printer.cc#L176-L185)

### Dense Tensor Printing

The attribute printer includes sophisticated handling for dense tensor attributes with size limiting capabilities:

*   **Mixed-radix counter**: Iterates through tensor shapes to print nested bracket structures
*   **Size limiting**: Truncates large tensors based on `size_limit` parameter
*   **Type-specific formatting**: Boolean values as "true"/"false", appropriate numeric formatting
*   **Splat optimization**: Special handling for splat (repeated value) tensors

The `PrintDenseElementsAttrImpl()` function implements the core tensor printing logic with proper bracket nesting for multi-dimensional arrays.

Sources: [src/builtin-adapter/tools/attribute_printer.cc 50-97](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/attribute_printer.cc#L50-L97)[src/builtin-adapter/tools/attribute_printer.cc 151-174](https://github.com/tenstorrent/model-explorer/blob/70d3e3e4/src/builtin-adapter/tools/attribute_printer.cc#L151-L174)

## Build Integration

The specialized adapters are integrated into the build system as separate libraries with specific dependencies:

| Library | Purpose | Key Dependencies |
| --- | --- | --- |
| `shardy_utils` | Shardy dialect support | `@shardy//shardy/dialect/sdy/ir:dialect` |
| `attribute_printer` | MLIR attribute printing | `@llvm-project//mlir:IR` |
| `convert_type` | TFLite type conversion | `@org_tensorflow//tensorflow/compiler/mlir/lite/schema:schema_fbs` |
| `namespace_heuristics` | Namespace organization | String processing utilities |

These libraries are designed to be composable and are used by the main conversion adapters to provide specialized functionality without tight coupling.
