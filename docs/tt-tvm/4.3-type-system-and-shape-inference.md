## Purpose and Scope

This document describes Relay's type system and shape inference mechanisms. It covers how Relay infers tensor shapes and validates operator types during compilation, both for static shapes (known at compile time) and dynamic shapes (determined at runtime). The type system ensures that operators are correctly used and automatically deduces output shapes from input shapes and operator attributes.

For information about operator implementations and compute definitions, see [4.1](https://deepwiki.com/tenstorrent/tt-tvm/4.1-neural-network-operators) and [4.2](https://deepwiki.com/tenstorrent/tt-tvm/4.2-tensor-transformation-operators). For layout transformations, see [4.4](https://deepwiki.com/tenstorrent/tt-tvm/4.4-layout-transformation-system).

## Type System Architecture

Relay's type system is hierarchical and extensible, supporting tensor types, tuple types, function types, and user-defined types. The system performs bidirectional type inference to validate programs and deduce missing type information.

**Sources:**[src/relay/op/tensor/transform.cc 133-145](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L133-L145)[tests/python/relay/test_any.py 39-43](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/python/relay/test_any.py#L39-L43)[src/relay/op/nn/nn.cc 55-79](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/nn.cc#L55-L79)

## Core Type System Concepts

### TensorType

`TensorType` represents tensors with a shape (array of `IndexExpr`) and a data type. Each dimension can be:

*   **Concrete**: An integer literal (e.g., `10`, `224`)
*   **Symbolic**: A `te.size_var` or `te.var` representing an unknown value
*   **Dynamic**: `relay.Any()` representing a dimension determined at runtime

### Type Relations

Type relations are functions that define how output types are computed from input types and operator attributes. Each operator registers a type relation using the `add_type_rel` attribute.

**Sources:**[src/relay/op/tensor/transform.cc 133-145](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L133-L145)[src/relay/op/tensor/transform.cc 253-286](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L253-L286)[src/relay/op/tensor/transform.cc 799-855](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L799-L855)

### Type Relation Implementation Pattern

Type relations follow a common implementation pattern:

1.   **Extract input types** from the `types` array
2.   **Validate inputs** (check for `nullptr`, `IncompleteType`)
3.   **Extract attributes** from the `attrs` parameter
4.   **Compute output shape** based on input shapes and attributes
5.   **Assign output type** using `reporter->Assign()`
6.   **Return** true on success, false if type is incomplete

**Example: CastRel**

[src/relay/op/tensor/transform.cc 133-145](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L133-L145)

This relation:

*   Checks that input is a `TensorType`
*   Returns false if incomplete (allows retry later)
*   Assigns output type with same shape but new dtype from attributes

**Sources:**[src/relay/op/tensor/transform.cc 133-145](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L133-L145)

## Shape Inference Mechanisms

Relay provides two complementary shape inference mechanisms:

1.   **Static Shape Inference**: Performed at compile time via type relations
2.   **Dynamic Shape Inference**: Performed at runtime via shape functions

### Static Shape Inference

Static shape inference occurs during the `InferType` pass. The pass traverses the IR and invokes type relations for each operator.

**Example: ReshapeRel**

The reshape operator infers output shape using special symbolic values:

*   `0`: Copy dimension from input
*   `-1`: Infer dimension from other dimensions
*   `-2`: Copy all remaining dimensions
*   `-3`: Merge two consecutive dimensions
*   `-4`: Split one dimension into two

#### Transpose Operator

**Purpose:** Permute the dimensions of a tensor.

[src/relay/op/tensor/transform.cc 502-549](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L502-L549)

**Type Relation Logic:**

3.   Validate axes: no duplicates, in valid range
4.   Permute shape according to axes

*   Output: `(4, 2, 3)`

#### Reshape Operator

**Purpose:** Change tensor shape while preserving element count.

The reshape type relation [src/relay/op/tensor/transform.cc 799-855](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L799-L855) handles complex shape inference with special values:

**Special Values in newshape:**

| Value | Meaning | Example |
| --- | --- | --- |
| `> 0` | Use this dimension | `(2, 3, 4)` → `newshape=(6, 4)` → `(6, 4)` |
| `0` | Copy from input (if allowzero=False) | `(2, 3, 4)` → `newshape=(2, 0, 0)` → `(2, 3, 4)` |
| `-1` | Infer from other dimensions | `(2, 3, 4)` → `newshape=(-1, 12)` → `(2, 12)` |
| `-2` | Copy all remaining dimensions | `(2, 3, 4)` → `newshape=(2, -2)` → `(2, 3, 4)` |
| `-3` | Merge two consecutive dimensions | `(2, 3, 4)` → `newshape=(-3, 4)` → `(6, 4)` |
| `-4` | Split dimension into two | `(2, 3, 4)` → `newshape=(-4, 1, 2, -2)` → `(1, 2, 3, 4)` |

**Sources:**[src/relay/op/tensor/transform.cc 663-796](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/tensor/transform.cc#L663-L796)[python/tvm/relay/op/transform.py 246-298](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/op/transform.py#L246-L298)

### Neural Network Operators

#### Conv2D Operator

Convolution has a complex type relation that computes output spatial dimensions from input, kernel, padding, stride, and dilation.

[src/relay/op/nn/convolution.cc](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/relay/op/nn/convolution.cc) (exact lines not shown in excerpt, but referenced in includes)

**Output Shape Calculation:**

```
out_height = (in_height + pad_top + pad_bottom - dilated_kernel_height) / stride_height + 1
out_width = (in_width + pad_left + pad_right - dilated_kernel_width) / stride_width + 1
```

**Type Relation Validation:**

*   Checks layout compatibility (NCHW, NHWC, etc.)
*   Validates groups parameter for grouped/depthwise convolution
*   Ensures channel consistency
*   Supports mixed-precision (int8 input, int32 output)

## Summary

Relay's type system provides comprehensive static and dynamic shape inference through:

1.   **Type Relations**: Operator-specific functions that compute output types from inputs and attributes
2.   **TypeReporter**: API for type relations to assign inferred types and report errors
3.   **Dynamic Shapes**: `relay.Any()` support for runtime-determined dimensions
4.   **Shape Functions**: Runtime shape computation for dynamic shapes in the VM
5.   **Validation**: Comprehensive error checking and diagnostic reporting

This system enables Relay to validate programs at compile time while supporting flexible dynamic shapes for deployment scenarios where input dimensions vary at runtime.
