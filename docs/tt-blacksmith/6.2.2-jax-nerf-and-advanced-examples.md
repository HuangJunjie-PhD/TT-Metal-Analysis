## Purpose and Scope

This page documents advanced JAX experiments in TT-Blacksmith beyond the foundational MNIST examples. The primary focus is on Neural Radiance Fields (NeRF) training for 3D scene reconstruction, which demonstrates JAX's capabilities for complex differentiable rendering tasks on Tenstorrent hardware.

For basic JAX MNIST implementations including single-chip, data parallel, and tensor parallel training, see [JAX MNIST Implementations](https://deepwiki.com/tenstorrent/tt-blacksmith/6.2.1-jax-mnist-implementations). For PyTorch-based experiments, see [PyTorch Experiments](https://deepwiki.com/tenstorrent/tt-blacksmith/6.1-pytorch-experiments).

## NeRF Training Overview

Neural Radiance Fields (NeRF) represent 3D scenes as continuous volumetric functions that can be rendered from arbitrary viewpoints. The TT-Blacksmith implementation uses the Blender synthetic dataset format and follows volumetric rendering principles.

### Training Architecture

**Sources:**[blacksmith/datasets/torch/nerf/blender.py](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/torch/nerf/blender.py)

### Key Differences from MNIST

| Aspect | MNIST | NeRF |
| --- | --- | --- |
| **Input Dimensionality** | 784 (28×28 images) | 6 (ray origin + direction) |
| **Output** | 10 class logits | RGB color + density (4 values) |
| **Architecture** | Simple MLP (3 layers) | Deep MLP (8 layers + skip connections) |
| **Batch Structure** | Fixed images | Ray samples with variable depth |
| **Computation** | Single forward pass | Volume rendering integral approximation |
| **Training Time** | Minutes | Hours to days |

### Split-Specific Behavior

The dataset implements different behaviors for training, testing, and validation splits:

| **val** | Generate 1000 novel viewpoints on sphere | Use `pose_spherical()` for camera poses |

## Advanced JAX Patterns for NeRF

### Volumetric Rendering in JAX

NeRF training requires differentiable volume rendering, which integrates color and density along rays. This is naturally expressed in JAX using `jax.lax.scan` for sequential ray marching:

### Positional Encoding

NeRF uses high-frequency positional encoding to enable the MLP to represent high-frequency details:

**Encoding formula:**`γ(p) = [sin(2^0πp), cos(2^0πp), ..., sin(2^Lπp), cos(2^Lπp)]`

This is efficiently implemented in JAX using `jnp.sin` and `jnp.cos` with vectorized operations across frequency bands.

### Hierarchical Sampling Strategy

NeRF uses a coarse-to-fine sampling strategy with two networks:

1.   **Coarse network**: Samples uniformly along ray
2.   **Fine network**: Importance samples based on coarse density

This requires managing two sets of parameters and coordinating their training, similar to the multi-model patterns seen in GAN training.

### Expected Training Script Structure

4.   **Training Loop**:

    *   Forward pass through fine network

6.   **Logging**: Log loss curves and rendered validation images

## Other Advanced JAX Examples

While NeRF is the primary advanced example, the repository structure supports additional complex JAX experiments:

### Potential Future Examples

| Task | JAX Features | TT Hardware Benefits |
| --- | --- | --- |
| **Implicit Neural Representations** | Coordinate-based MLPs | Fast MLP inference |
| **Differentiable Physics** | `jax.grad` through simulators | Parallel trajectory evaluation |
| **Neural ODEs** | `jax.experimental.ode` | Efficient ODE solving |
| **Gaussian Processes** | Kernel computations | Matrix operations |
| **Transformer Models** | Attention mechanisms | Large batch inference |

### Common Advanced Patterns

Across advanced JAX examples, common patterns include:

1.   **Custom Gradient Functions**: Using `jax.custom_vjp` for specialized backward passes
2.   **Nested JIT Compilation**: Compiling inner loops within outer training loops
3.   **Checkpointing**: Using `jax.checkpoint` for memory-efficient backpropagation
4.   **Scan Operations**: `jax.lax.scan` for sequential processing with fixed memory
5.   **Mesh-Based Parallelism**: `jax.sharding.Mesh` for multi-device distribution

NeRF-specific validation metrics:

*   **PSNR**: Peak Signal-to-Noise Ratio for image quality
*   **SSIM**: Structural Similarity Index
*   **LPIPS**: Learned Perceptual Image Patch Similarity
*   **Rendering Speed**: Rays per second throughput
