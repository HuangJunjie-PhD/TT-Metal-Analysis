This document describes the Python wheel building and publication process for tt-flash. It covers the package configuration in `pyproject.toml`, the GitHub Actions workflow that builds wheels, and the sequential publication process to TestPyPI and PyPI using trusted publishing.

For information about the overall release orchestration and version management, see [Release Process](https://deepwiki.com/tenstorrent/tt-flash/9.1-release-process). For Debian package distribution, see [Debian Packaging](https://deepwiki.com/tenstorrent/tt-flash/9.2-debian-packaging).

### Entry Points and Command Registration

The CLI command is registered as a console script entry point at [pyproject.toml 49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L49-L50):

```
[project.scripts]
tt-flash = "tt_flash.main:main"
```

This creates the `tt-flash` command that invokes `main()` from `tt_flash.main` when the package is installed.

**Sources:**[pyproject.toml 49-50](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L49-L50)

### Dependency Specification

Dependencies are specified with precise version constraints at [pyproject.toml 23-35](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L35):

**Pinned versions** (exact match) are used for:

*   `pyyaml` (6.0.2)
*   `tabulate` (0.9.0)
*   `tomli` (2.0.1)

**Compatible releases** (tilde operator) allow patch updates:

*   `tt_tools_common ~= 1.5` (allows 1.5.x but not 1.6+)
*   `pyluwen ~= 0.8.0` (allows 0.8.x but not 0.9+)

**Conditional dependencies** use environment markers:

*   `tomli` only on Python < 3.11 (native `tomllib` available in 3.11+)
*   `importlib-metadata` only on Python < 3.8
*   `importlib-resources` only on Python < 3.9

**Sources:**[pyproject.toml 23-42](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L23-L42)

### Build System Configuration

The build system configuration at [pyproject.toml 74-81](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L74-L81) specifies PEP 517 build requirements:

```
[build-system]
requires = [
  "setuptools>=43.0.0",
  "wheel"
]
build-backend = "setuptools.build_meta"
```

This uses `setuptools` as the build backend with the modern `setuptools.build_meta` interface rather than legacy `setup.py` invocation.

**Sources:**[pyproject.toml 74-81](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L74-L81)

### Package Data Inclusion

The package includes non-Python files specified at [pyproject.toml 55-59](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L55-L59):

```
[tool.setuptools.package-data]
"*" = [
    ".ignored/version.txt",
    "data/*/*.yaml"

```

This ensures firmware definition YAML files and version information are bundled in the wheel.

This dual-configuration approach ensures compatibility across different packaging environments while maintaining `pyproject.toml` as the source of truth.

## Publication Pipeline

The publication process is orchestrated by `.github/workflows/release.yml` in two sequential stages: TestPyPI validation followed by PyPI publication.

### TestPyPI Publication

The TestPyPI publication job at [.github/workflows/release.yml 391-420](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L391-L420) provides pre-production validation:

**Job Configuration:**

*   **Dependencies:** Requires `build_all_depends` and `generate-release` completion
*   **Environment:**`testpypi` with URL `https://test.pypi.org/p/tt-flash`
*   **Permissions:**`id-token: write` for trusted publishing

**Steps:**

1.   **Download Artifacts** ([.github/workflows/release.yml 408-413](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L408-L413)): Downloads wheel artifacts matching pattern `tt-flash*.whl`
2.   **Publish** ([.github/workflows/release.yml 414-419](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L414-L419)): Uses `pypa/gh-action-pypi-publish@release/v1` with custom repository URL

The `repository-url` parameter at [.github/workflows/release.yml 418](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L418-L418) points to TestPyPI:

**Sources:**[.github/workflows/release.yml 391-420](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L391-L420)

### PyPI Publication

The PyPI publication job at [.github/workflows/release.yml 424-453](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L424-L453) publishes to production PyPI:

**Job Configuration:**

*   **Dependencies:** Requires `publish-to-testpypi` completion (sequential validation)
*   **Environment:**`pypi` with URL `https://pypi.org/p/tt-flash`
*   **Permissions:**`id-token: write` for trusted publishing

**Steps:**

1.   **Download Artifacts** ([.github/workflows/release.yml 442-447](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L442-L447)): Downloads same wheel artifacts
2.   **Publish** ([.github/workflows/release.yml 448-452](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L448-L452)): Uses default PyPI endpoint (no custom repository URL)

The sequential dependency chain ensures TestPyPI validation before production publication:

```
build_all_depends → generate-release → publish-to-testpypi → publish-to-pypi
```

**Sources:**[.github/workflows/release.yml 424-453](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L424-L453)

### Trusted Publishing with OIDC

Both publication jobs use OpenID Connect (OIDC) trusted publishing, eliminating the need for long-lived API tokens. This is configured through:

1.   **GitHub Environment Setup:** Each job declares an environment (`testpypi` or `pypi`)
2.   **Permission Grant:**`id-token: write` at [.github/workflows/release.yml 405](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L405-L405) and [.github/workflows/release.yml 439](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L439-L439)
3.   **PyPI Configuration:** Pre-configured trust relationship between GitHub repository and PyPI project

The `pypa/gh-action-pypi-publish` action automatically exchanges the GitHub OIDC token for PyPI upload credentials.

**Sources:**[.github/workflows/release.yml 400-406](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L400-L406)[.github/workflows/release.yml 434-440](https://github.com/tenstorrent/tt-flash/blob/60c06032/.github/workflows/release.yml#L434-L440)

## Version Synchronization

The version number is maintained in a single location at [pyproject.toml 3](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L3-L3) and propagated through the build system:

The version flows from `pyproject.toml` through:

1.   **setup.py:** Extracted at [setup.py 17](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L17-L17) using `tomli.load()`
2.   **Build backend:** Read directly by `setuptools.build_meta`
3.   **Wheel filename:** Embedded in artifact name (e.g., `tt_flash-3.5.0-py3-none-any.whl`)
4.   **Package metadata:** Written to `METADATA` file in wheel's `.dist-info` directory

This ensures version consistency across all distribution artifacts.

**Sources:**[pyproject.toml 3](https://github.com/tenstorrent/tt-flash/blob/60c06032/pyproject.toml#L3-L3)[setup.py 12-17](https://github.com/tenstorrent/tt-flash/blob/60c06032/setup.py#L12-L17)
