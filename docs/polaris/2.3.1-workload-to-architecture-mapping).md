# Workload to Architecture Mapping

Relevant source files
*   [config/wl2archmapping.yaml](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/wl2archmapping.yaml)
*   [ttsim/front/functional/op.py](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/ttsim/front/functional/op.py)

This document explains the workload to architecture mapping system in Polaris, which defines how neural network operations are mapped to computational resources and how operations are optimized through fusion patterns. The mapping configuration controls data types, operation removal, fusion patterns, and resource allocation for simulation execution.

For information about the configuration parsers that process these mappings, see [Configuration Parsers](https://deepwiki.com/tenstorrent/polaris/2.3.2-configuration-parsers). For details on the main simulation orchestrator that uses these mappings, see [Polaris Main Orchestrator](https://deepwiki.com/tenstorrent/polaris/2.1-polaris-main-orchestrator).

## Operation Resource Mapping

The `op_rsrc_spec` section defines how operations are categorized and mapped to computational resources. Operations are classified into two main compute types:

| Compute Type | Operations | Description |
| --- | --- | --- |
| `matrix` | `Matmul`, `Conv` | Matrix multiplication and convolution operations requiring high compute throughput |
| `vector` | `Gather`, `Reshape`, `Transpose`, `Gelu`, `Relu`, etc. | Element-wise and data movement operations |

**Sources:**[config/wl2archmapping.yaml 40-48](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/wl2archmapping.yaml#L40-L48)

## Data Type Specifications

The `op_data_type_spec` section controls data types used in operations:

| Setting | Value | Description |
| --- | --- | --- |
| `global_type` | `int8` | Default data type for all operations |
| `override.dropout` | `int32` | Override for dropout operations |

**Sources:**[config/wl2archmapping.yaml 3-6](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/wl2archmapping.yaml#L3-L6)

## Operation Removal

The `op_removal_spec` defines operations that should be removed from the computational graph during preprocessing:

The removed operations are:

*   `Dropout` - Training-only regularization
*   `Identity` - No-operation passthrough
*   `Constant` - Constant values that can be folded

**Sources:**[config/wl2archmapping.yaml 8-11](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/wl2archmapping.yaml#L8-L11)
