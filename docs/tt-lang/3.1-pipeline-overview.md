## Purpose and Scope

This page provides a high-level overview of the tt-lang compilation pipeline, which transforms Python kernel definitions into optimized C++ code for Tenstorrent hardware. The pipeline consists of four major phases: Python AST parsing, TTL dialect transformations, TTL-to-TTKernel lowering, and final code generation.

For details on individual compilation stages:

*   Python AST to TTL MLIR: see [Python AST to Initial TTL MLIR](https://deepwiki.com/tenstorrent/tt-lang/3.2-python-ast-to-initial-ttl-mlir)
*   TTL dialect transformations: see [TTL Dialect Transformations](https://deepwiki.com/tenstorrent/tt-lang/3.3-ttl-dialect-transformations)
*   TTL to TTKernel conversion: see [TTL to TTKernel Conversion](https://deepwiki.com/tenstorrent/tt-lang/3.4-ttl-to-ttkernel-conversion)
*   Final code generation: see [Code Generation and EmitC](https://deepwiki.com/tenstorrent/tt-lang/3.5-code-generation-and-emitc)

For information about executing compiled kernels, see [TTNN Integration](https://deepwiki.com/tenstorrent/tt-lang/5-ttnn-integration) and [Simulation Framework](https://deepwiki.com/tenstorrent/tt-lang/6-simulation-framework).

## Phase 1: Python AST to Initial TTL

The compilation process begins when a user-decorated kernel function is invoked. The `@ttl.kernel` decorator registers compute and data movement threads, which are then compiled into MLIR.

**Key Components:**

*   **Entry Point:** The compiler is triggered via the kernel runner logic when a decorated function is called [python/ttl/ttl_api.py 90-95](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L90-L95)
*   **Compiler:**`TTLGenericCompiler` in `python/ttl/_src/ttl_ast.py` handles the visitor patterns for Python AST [python/ttl/_src/ttl_ast.py 128-170](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/ttl_ast.py#L128-L170)
*   **Output:** Initial TTL MLIR with `ttl.bind_cb`, `ttl.copy`, `ttl.cb_wait`, `ttl.cb_reserve`, and high-level tile operations.

**Process:**

1.   Python AST is parsed using the standard `ast` module [python/ttl/ttl_api.py 9](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L9-L9)
2.   Captured variables (tensors, CBs, constants) are resolved from function closures and globals [python/ttl/_src/ttl_ast.py 136-153](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/ttl_ast.py#L136-L153)
3.   Function body is visited to emit TTL operations. `TTLGenericCompiler` supports `AsyncFunctionDef`, `With`, and standard assignments [python/ttl/_src/ttl_ast.py 138-140](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/ttl_ast.py#L138-L140)
4.   Tensor arguments become `func.FuncOp` arguments with `TTLLayoutAttr` encoding via `_build_tensor_type`[python/ttl/_src/ttl_ast.py 69-116](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/ttl_ast.py#L69-L116)

## Phase 2: TTL Dialect Transformations

Phase 2 applies a sequence of transformation passes to the initial TTL MLIR. These passes optimize the IR, allocate hardware resources (like DST registers), and prepare for lowering to TTKernel.

### Major Transformation Passes

| Pass | Purpose | Implementation |
| --- | --- | --- |
| `ConvertTTLToCompute` | Fuses elementwise operations into `ttl.compute` regions and handles broadcasting. | [lib/Dialect/TTL/Transforms/ConvertTTLToCompute.cpp 22-34](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLToCompute.cpp#L22-L34) |
| `TTLAssignDST` | DST register allocator using linear scan and in-place merging to minimize spills. | [lib/Dialect/TTL/Transforms/CMakeLists.txt 26](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/CMakeLists.txt#L26-L26) |
| `TTLSubblockComputeForDST` | Partitions `ttl.compute` into subblocks fitting in DST capacity. | [lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp 1-29](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/DstSubblockUtils.cpp#L1-L29) |
| `TTLLowerToLoops` | Lowers high-level `ttl.compute` to explicit `scf.for` loops with tile indexing. | [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 44-48](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L44-L48) |
| `TTLInsertIntermediateDFBs` | Inserts compiler-allocated Dataflow Buffers (DFBs) at fusion split points. | [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 21-25](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L21-L25) |

## Phase 3: TTL to TTKernel Lowering

Phase 3 converts TTL dialect operations to the TTKernel dialect, which directly models Tenstorrent hardware operations and RISC-specific instructions.

**Key Transformations:**

*   **Tile Operations:** TTL tile math ops (add, mul, SFPU) → TTKernel instructions. Handled by `ConvertTTLTileOpsToTTKernel`[lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp 8-17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp#L8-L17)
*   **CB Operations:**`ttl.bind_cb`, `ttl.copy`, and `ttl.wait` are lowered to TTKernel NOC and CB operations [lib/Dialect/TTL/Transforms/ConvertTTLToTTKernel.cpp 65-96](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLToTTKernel.cpp#L65-L96)
*   **L1 Accumulation:**`TTKernelInsertL1Accumulation` inserts `pack_reconfig_l1_acc` guards inside reduction loops to prevent overwriting [include/ttlang/Dialect/TTL/Passes.td 143-165](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/Passes.td#L143-L165)

**CB Index Annotation:** The `createTTLAnnotateCBAssociations` pass prepares for lowering by mapping tensors to hardware CB slots [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 53](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L53-L53)

**Init Insertion:** The `createTTKernelInsertInits` pass inserts hardware initialization operations (`init_sfpu`, `tile_init`) required before compute operations [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 63](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L63-L63)

**Sources:**[lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 53-64](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L53-L64)[include/ttlang/Dialect/TTL/Passes.td 38-90](https://github.com/tenstorrent/tt-lang/blob/d76e6233/include/ttlang/Dialect/TTL/Passes.td#L38-L90)[lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp 133-153](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/ConvertTTLTileOpsToTTKernel.cpp#L133-L153)

## Phase 4: Code Generation

The final phase lowers TTKernel MLIR to C++ through the EmitC dialect. Standard MLIR optimization passes run first, followed by dialect conversion to EmitC.

**Key Steps:**

1.   **Canonicalization & CSE:** Remove redundant operations and fold constants [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 68-69](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L68-L69)
2.   **LowerAffine:** Convert affine index expressions (e.g., linearized tile offsets) to standard arithmetic [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 71](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L71-L71)
3.   **ConvertTTKernelToEmitC:** Map TTKernel operations to the Tenstorrent C++ Kernel API [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 72](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L72-L72)
4.   **FormExpressions:** The `mlir::emitc::createFormExpressionsPass` combines C++ operations into complex expressions for readability [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 74](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L74-L74)

**Kernel Output:** The compiler generates C++ source files for each RISC processor (BRISC, NCRISC, TRISC). These are then compiled by the Tenstorrent toolchain into firmware binaries.

**Sources:**[lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 67-75](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L67-L75)[lib/Dialect/TTL/Transforms/CMakeLists.txt 9-13](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/CMakeLists.txt#L9-L13)

## Compilation Configuration and Debugging

### Compiler Options

Compilation behavior is controlled via `CompilerOptions`, which can be set through decorators or environment variables.

*   `reduce_full_fp32`: Enables FP32 accumulation for reduction operations [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 32](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L32-L32)
*   `maximize_dst`: Enables subblocking to fit DST capacity [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 37](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L37-L37)
*   `use_block_matmul`: Enables hardware-optimized block matmul paths [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 46](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L46-L46)
*   `compiler_dfbs`: Automatically inserts intermediate buffers for fused operations [lib/Dialect/TTL/Pipelines/TTLPipelines.cpp 23](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Pipelines/TTLPipelines.cpp#L23-L23)

### MLIR Inspection

During development, intermediate MLIR can be inspected using environment variables:

*   `TTLANG_COMPILE_ONLY`: Set to "1" to skip kernel execution and only run the compilation pipeline [python/ttl/ttl_api.py 161-163](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/ttl_api.py#L161-L163)
*   `TTLANG_INITIAL_MLIR`: Can be used to dump the IR immediately after AST parsing.
*   `TTLANG_FINAL_MLIR`: Can be used to dump the IR before EmitC lowering.
