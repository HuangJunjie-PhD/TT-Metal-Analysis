# Parser Implementation

Relevant source files
*   [riescue/compliance/tp.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/compliance/tp.py)
*   [riescue/dtest_framework/parser.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py)

## Purpose and Scope

This document provides a detailed technical explanation of the `Parser` class in [riescue/dtest_framework/parser.py](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py) which is responsible for processing custom RiESCUE directives embedded in assembly test files. The Parser extracts structured information from special comment-based syntax and populates dataclass structures that are later consumed by the Generator component.

For information about how the Generator uses parsed data to produce assembly code, see [Generator Implementation](https://deepwiki.com/tenstorrent/riescue/7.1.3-memory-management). For an overview of the complete test generation pipeline, see [Test Generation Internals](https://deepwiki.com/tenstorrent/riescue/7.1-test-generation-internals).

**Sources:**[riescue/dtest_framework/parser.py 1-910](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L1-L910)

## Parser Class Overview

The `Parser` class reads assembly files (`.rasm` or `.S` files) line by line and extracts RiESCUE-specific directives that begin with `;#`. These directives provide metadata and configuration that controls test generation, including test headers, random data generation, memory addressing, and page table setup.

### Key Responsibilities

| Responsibility | Description |
| --- | --- |
| **Directive Recognition** | Identify lines starting with `;#` as RiESCUE directives |
| **Pattern Matching** | Use regular expressions to parse directive syntax |
| **Data Extraction** | Extract parameters from directives and convert to appropriate types |
| **Dataclass Population** | Create and populate parsed data structures |
| **Pool Integration** | Register parsed data with the Pool for later use |
| **Validation** | Basic type conversion and format validation |

**Sources:**[riescue/dtest_framework/parser.py 20-38](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L20-L38)

## Initialization and Data Structures

### Parser Initialization

The Parser maintains internal dictionaries for organizing parsed data by name and uses counters (`parsed_csr_id`, `parsed_pte_id`) to generate unique identifiers for certain directive types.

**Sources:**[riescue/dtest_framework/parser.py 40-50](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L40-L50)

## Parsing Workflow

### Main Parse Method

The `parse()` method implements a single-pass line scanner that checks each line against multiple directive patterns using `startswith()` checks. The order of checks matters for performance but not correctness, as directives are mutually exclusive.

**Sources:**[riescue/dtest_framework/parser.py 52-98](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L52-L98)

## Directive Types and Parsing Methods

### Random Data Directive

The `;#random_data()` directive specifies random data generation with optional masking.

**Parsing Logic:**

1.   Match pattern `^;#(random_data)\((.+)\)`
2.   Split arguments on commas
3.   For each `var=val` pair: 
    *   Keep `name` and `type` as strings
    *   Convert numeric values using `int(val, 0)` (supports hex)

4.   Store in `self.random_data[name]` dictionary
5.   Register with Pool via `add_parsed_data()`

**Sources:**[riescue/dtest_framework/parser.py 116-130](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L116-L130)[riescue/dtest_framework/parser.py 702-707](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L702-L707)

### Random Address Directive

The `;#random_addr()` directive defines random address generation with type, size, alignment, and PMA constraints.

**Type Parsing:** The `type` parameter can be `linear`, `physical`, or include bit width like `linear32`, `physical64`. The parser uses regex `([a-z]+)(\d*)` to extract both components.

**PMA Integration:** When any `pma_*` parameter is present, a `PmaInfo` object is created and attached to the `ParsedRandomAddress`.

**Sources:**[riescue/dtest_framework/parser.py 132-166](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L132-L166)[riescue/dtest_framework/parser.py 720-734](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L720-L734)

### Page Mapping Directive

The `;#page_mapping()` directive is the most complex, supporting virtual memory configuration with extensive control over page table entries.

**Two-Phase Extraction:** The parser first extracts list parameters (like `pagesize=[...]`) using regex, removes them from the line, then processes the remaining key-value pairs. This prevents comma-separated lists from interfering with argument splitting.

**Boolean Conversion:** Permission flags are extracted with regex patterns like `v_nonleaf=(\d+)` and converted to boolean with `bool(int(extracted[0]))`.

**Security Propagation:** When `secure=1`, the parser updates all referenced addresses in the Pool to mark them as secure.

**Sources:**[riescue/dtest_framework/parser.py 168-572](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L168-L572)[riescue/dtest_framework/parser.py 737-845](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L737-L845)

### Test Header Directive

Test headers use a dot-notation syntax where the field name follows `.test.` and the value is everything after it. Multiple header directives accumulate into a single `ParsedTestHeader` instance.

**Sources:**[riescue/dtest_framework/parser.py 590-598](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L590-L598)[riescue/dtest_framework/parser.py 854-868](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L854-L868)

## Dataclass Structures

### Core Parsed Data Types

Each dataclass is defined with `@dataclass` decorator and includes default values. The Parser creates instances, populates their fields, and registers them with the Pool.

**Sources:**[riescue/dtest_framework/parser.py 701-910](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L701-L910)

### ParsedPageMapping Fields

The `ParsedPageMapping` dataclass is exceptionally large due to supporting multiple levels of page tables with guest-stage virtualization:

| Field Category | Example Fields | Purpose |
| --- | --- | --- |
| **Address References** | `lin_name`, `phys_name` | Link to address definitions |
| **Basic Permissions** | `v`, `r`, `w`, `x`, `u`, `g` | Standard page table flags |
| **Level-Specific V** | `v_level0` through `v_level4` | Valid bit per page table level |
| **Leaf/Non-Leaf** | `v_leaf`, `v_nonleaf`, `r_nonleaf` | Separate control for leaf vs non-leaf entries |
| **G-Stage Virtualization** | `v_leaf_gleaf`, `v_nonleaf_gnonleaf` | Guest-stage combinations |
| **Access/Dirty** | `a`, `d`, `a_nonleaf`, `d_leaf_gleaf` | Access and dirty bit control |
| **Page Sizes** | `pagesizes`, `gstage_vs_leaf_pagesizes` | Allowed page sizes (4KB, 2MB, etc.) |
| **Security** | `secure`, `pbmt`, `n` | Security and memory type attributes |

**Sources:**[riescue/dtest_framework/parser.py 737-845](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L737-L845)

## Pool Integration

### Registration Flow

The Parser maintains its own internal dictionaries (e.g., `self.random_data`, `self.random_addrs`) for organizing data during parsing, but the authoritative storage is in the Pool. The Generator later queries the Pool to retrieve parsed data.

**Sources:**[riescue/dtest_framework/parser.py 40-50](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L40-L50)[riescue/dtest_framework/parser.py 114-166](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L114-L166)

## Special Parsing Cases

### Vectored Interrupt Parsing

The parser supports both numeric indices and symbolic names for interrupt vectors. If the index is a string that matches a key in `interrupt_index`, it's converted to the corresponding numeric value.

**Sources:**[riescue/dtest_framework/parser.py 646-663](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L646-L663)[riescue/dtest_framework/parser.py 871-893](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L871-L893)

### CSR Access Parsing with Privilege Detection

The parser automatically infers privilege mode from the CSR name prefix, ensuring correct privilege level setup for CSR access instructions.

**Sources:**[riescue/dtest_framework/parser.py 665-680](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L665-L680)[riescue/dtest_framework/parser.py 896-902](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L896-L902)

## Error Handling and Validation

The Parser performs minimal validation during parsing:

| Validation Type | Implementation | Location |
| --- | --- | --- |
| **Pattern Matching** | Returns silently if regex doesn't match | All `parse_*()` methods |
| **Integer Conversion** | Uses `int(val, 0)` which raises `ValueError` on invalid input | Throughout |
| **Type Checking** | Checks field names with `re.match()` | `parse_random_addr()`, `parse_page_mappings()` |
| **Required Fields** | Not validated - FIXME comment indicates this limitation | [parser.py 36-38](https://github.com/tenstorrent/riescue/blob/07cefde3/parser.py#L36-L38) |

The Parser is designed for speed and assumes well-formed directives. Semantic validation (e.g., checking if referenced addresses exist) is deferred to later stages like FeatMgr.

**Sources:**[riescue/dtest_framework/parser.py 36-38](https://github.com/tenstorrent/riescue/blob/07cefde3/riescue/dtest_framework/parser.py#L36-L38)

## Summary: Parser to Pool to Generator

The Parser serves as the entry point for extracting structured test configuration from assembly directives. Its output—populated dataclass instances stored in the Pool—provides the raw material for feature management and code generation in subsequent pipeline stages.
