The `tt-toplike` TUI provides several high-fidelity visualization modes designed to translate raw telemetry (power, current, temperature, and memory bandwidth) into intuitive, real-time animations. These modes range from abstract starfields to architecturally-accurate memory flow diagrams.

## Overview of Display Modes

The TUI manages transitions between these modes via the `DisplayMode` enum [src/ui/tui/mod.rs 50-64](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/tui/mod.rs#L50-L64) Users can cycle through these modes using specific keyboard bindings.

| Mode | Key | Description | Implementation Class |
| --- | --- | --- | --- |
| **Insights** | `i` | Per-chip portraits with process monitoring. | `ChipPortrait` |
| **Grid** | `g` | Compact grid of all detected chips. | `ChipPortrait` |
| **Starfield** | `s` | Hardware-responsive stellar animation. | `HardwareStarfield` |
| **Memory Castle** | `c` | Vertical flow through the memory hierarchy. | `MemoryCastle` |
| **Memory Flow** | `f` | NoC-aware DRAM perimeter visualization. | `MemoryFlowVis` |
| **Arcade** | `a` | Unified compositor with a "Hero" character. | `ArcadeVisualization` |

## The Arcade Mode

Arcade Mode is the most complex visualization, acting as a compositor that combines `Starfield`, `Memory Castle`, and `Memory Flow` into a single split-screen view [src/animation/arcade.rs 7-13](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L7-L13)

### Layout Composition

The screen is divided into three distinct vertical regions [src/animation/arcade.rs 11-13](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L11-L13):

1.   **Top 30% (Starfield):** Stars represent Tensix cores; planets represent the memory hierarchy.
2.   **Middle 40% (Memory Castle):** Visualizes particles flowing from DDR channels through L2 cache to L1 SRAM.
3.   **Bottom 30% (Memory Flow):** Shows NoC particles and a heat map center.

### The Hero Character (`@`)

A unique feature of Arcade mode is the "Hero" character, which serves as a visual proxy for total system health [src/animation/arcade.rs 15-20](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L15-L20):

*   **Vertical Position:** Driven by total power consumption [src/animation/arcade.rs 16](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L16-L16)
*   **Horizontal Position:** Driven by current draw [src/animation/arcade.rs 17](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L17-L17)
*   **Color:** Mapped to temperature using `temp_to_hue`[src/animation/arcade.rs 18](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L18-L18)
*   **Smoothing:** The character uses 10-frame linear interpolation (`lerp`) to move smoothly between telemetry samples [src/animation/arcade.rs 300-310](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L300-L310)

## Memory Castle and Memory Flow

These modes focus on the data movement within the Tenstorrent architecture.

### Memory Castle

`MemoryCastle` uses a particle system to represent data hits and misses across the memory hierarchy [src/animation/castle.rs 5-15](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/castle.rs#L5-L15)

*   **Particles:** Up to 600 `MemoryParticle` instances [src/animation/arcade.rs 117](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L117-L117)
*   **Types:** Particles are categorized as `Read`, `Write`, `CacheHit`, or `CacheMiss`, each with distinct glyphs and colors [src/animation/castle.rs 25-35](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/castle.rs#L25-L35)
*   **Z-Order:** Implements a Z-order stack to ensure particles in the foreground overlap those in the background correctly [src/animation/castle.rs 150-160](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/castle.rs#L150-L160)

### Memory Flow

`MemoryFlowVis` focuses on the perimeter of the chip, where DDR controllers reside [src/animation/flow.rs 5-10](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/flow.rs#L5-L10)

*   **Topology Awareness:** Uses `BoardTopology` to map physical DDR locations to screen coordinates [src/animation/arcade.rs 158-170](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L158-L170)
*   **Flow Direction:** Particles stream from the DDR perimeter toward the center Tensix grid, representing memory bandwidth utilization [src/animation/flow.rs 40-50](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/flow.rs#L40-L50)

## Hardware Starfield

The `HardwareStarfield` mode treats the entire device cluster as a celestial map [src/animation/starfield.rs 5-12](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L5-L12)

*   **Tensix Stars:** Each core is a star. Its brightness is mapped to power, and its "twinkle" frequency is mapped to current fluctuations [src/animation/starfield.rs 20-30](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L20-L30)
*   **Memory Planets:** Memory controllers are rendered as orbiting bodies. The orbit speed and radius are influenced by memory bandwidth [src/animation/starfield.rs 45-55](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/starfield.rs#L45-L55)
*   **Sensitivity:** Responds to the `--profile` CLI flag (e.g., `paranoid`, `chill`) via the `set_sensitivity` function [src/animation/arcade.rs 142-146](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/animation/arcade.rs#L142-L146)
