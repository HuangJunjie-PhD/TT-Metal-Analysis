The accuracy evaluation system in `tt-inference-server` provides a standardized framework for verifying the functional correctness of models across different modalities (LLM, VLM, Image, Audio, etc.). It leverages industry-standard tools like `lm-evaluation-harness` and `lmms-eval` while providing custom utilities for specialized tasks like Stable Diffusion FID/CLIP scoring.

## Overview and Architecture

The evaluation lifecycle is orchestrated by `run_evals.py`, which acts as the primary entry point. It resolves model specifications, manages isolated virtual environments (venvs) for different evaluation backends, and dispatches tasks to either third-party harnesses or internal accuracy scripts.

## Task Configuration and Selection

Evaluations are defined using the `EvalTask` dataclass. Each task specifies its execution environment, the specific harness class to use, and how the resulting score should be calculated.

### EvalTask Specification

The `EvalTask` class contains parameters that control the behavior of the evaluation harness:

*   `task_name`: The name of the dataset/benchmark (e.g., `ifeval`, `mmlu_pro`). [evals/eval_config.py 31](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L31-L31)
*   `workflow_venv_type`: Defines which isolated environment to use (e.g., `EVALS_COMMON`, `EVALS_META`, `EVALS_AUDIO`). [evals/eval_config.py 33](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L33-L33)
*   `eval_class`: Specifies the harness interface, such as `local-completions` or `openai_compatible`. [evals/eval_config.py 34](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L34-L34)
*   `limit_samples_map`: Allows defining different sample sizes for `SMOKE_TEST` vs `CI_NIGHTLY` modes. [evals/eval_config.py 56-61](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L56-L61)

### EvalTask Selection Logic

The `_select_eval_config` function in `run_evals.py` determines which tasks to run based on the `RuntimeConfig`. If `SMOKE_TEST` mode is active, it restricts execution to the first task in the config to ensure fast turnaround. [evals/run_evals.py 74-86](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/run_evals.py#L74-L86)

## Harness Integration

The server integrates with external harnesses by dynamically building command-line strings that point to the `tt-media-server` or `vLLM` endpoints.

### Command Generation

The `build_eval_command` function templates the arguments for `lm_eval`. It handles the discrepancy between different harness requirements, such as whether the `base_url` should include the `/v1` suffix.

*   **LLM (lm-eval):** Uses `local-completions` and expects the full API route. [evals/run_evals.py 221-226](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/run_evals.py#L221-L226)
*   **VLM (lmms-eval):** Uses `eval_class` and expects only the host/port in `base_url`. [evals/run_evals.py 234-241](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/run_evals.py#L234-L241)
*   **Audio:** For audio models using `tt-media-server`, the base URL is adjusted to exclude the `/v1` prefix. [evals/run_evals.py 217-218](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/run_evals.py#L217-L218)

### Media Server Health Checks

Before running evaluations for media models (Audio, CNN, Image, Video), `run_evals.py` performs a liveness check using `_check_media_server_health`. This ensures the model runner is active and responsive before starting long-running accuracy tests. [evals/run_evals.py 100-138](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/run_evals.py#L100-L138)

## Image and Video Generation Accuracy

For generative models like SDXL, FLUX, and Mochi-1, accuracy is measured using **Fréchet Inception Distance (FID)** and **CLIP Score**.

### Accuracy Reference Registry

The `model_accuracy_reference.json` file serves as the ground truth for generation quality. It defines valid ranges for FID and CLIP based on the number of prompts used (e.g., 100, 500, or 5000). [evals/eval_targets/model_accuracy_reference.json 1-252](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_targets/model_accuracy_reference.json#L1-L252)

Sources: [evals/eval_targets/model_accuracy_reference.json 122-133](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_targets/model_accuracy_reference.json#L122-L133)

### Implementation Details

The `sdxl_accuracy_utils.py` module (and associated video utilities) provide the following key functions:

*   **Prompt Management**: Downloads and parses datasets like COCO 2014 captions.
*   **Metric Calculation**: Orchestrates the decoding of generated media, invokes the `CLIPEncoder`, and calculates FID scores.
*   **Validation**: Compares measured scores against reference ranges, typically allowing for small variances in hardware implementation.

## Reporting and Acceptance

After `run_evals.py` completes, `run_reports.py` aggregates the results into a final summary.

### Score Aggregation

Different tasks require different scoring functions, which are mapped in `EvalTaskScore`:

*   `score_task_single_key`: Extracts a specific metric from the harness output. [evals/eval_config.py 102](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L102-L102)
*   `score_multilevel_keys_mean`: Averages scores across multiple sub-tasks. [evals/eval_config.py 9](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L9-L9)

### Acceptance Criteria

The `acceptance_criteria_check` function compares the final scores against the `published_score` or `gpu_reference_score` defined in `eval_config.py`. [workflows/run_reports.py 33-36](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/workflows/run_reports.py#L33-L36) A default tolerance (usually 5%) is applied to determine if the model passes the accuracy gate. [evals/eval_config.py 26](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/evals/eval_config.py#L26-L26)
