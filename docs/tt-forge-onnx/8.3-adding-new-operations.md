This document provides a step-by-step guide for adding new operations to the Forge compiler. It covers the complete workflow from defining the Python API to implementing C++ evaluation logic, MLIR lowering, and testing.

For information about the overall operation system architecture, see [Operation System Architecture](https://deepwiki.com/tenstorrent/tt-forge-onnx/5.1-operation-architecture). For details on how operations are implemented internally, see [Operation Implementation Guide](https://deepwiki.com/tenstorrent/tt-forge-onnx/5.2-operation-implementation-guide).

## Overview of the Operation Addition Process

Adding a new operation to Forge requires changes across multiple layers of the compilation stack. The process involves:

1.   **Python API Definition** - Create user-facing Python function
2.   **OpType Registration** - Add operation to the C++ OpType enum
3.   **C++ Implementation** - Implement eval, shape, backward, and decompose interfaces
4.   **Build Configuration** - Add source files to CMakeLists.txt
5.   **Python Bindings** - Expose OpType to Python via pybind11
6.   **MLIR Lowering** - Add handler to translate operation to TTIR dialect
7.   **TVM Integration** - Map TVM/Relay operations to Forge operations (if applicable)
8.   **Testing** - Create operator tests with verification

## Common Patterns and Best Practices

### Attribute Naming Conventions

*   Use lowercase with underscores: `stride`, `padding`, `kernel_size`
*   Boolean flags: `channel_last`, `keep_dim`, `ceil_mode`
*   Match PyTorch naming where possible for consistency

### Shape Handling

*   Always use negative indexing for dimensions: `dim -= len(shape)`
*   Validate dimension bounds with assertions
*   Handle dynamic batch dimension (typically dimension 0 is `-1` or variable)

### Error Handling

Use assertions liberally to catch misuse early in development.

### Operation Categories

Choose the correct category for consistent organization:

*   **Eltwise Unary**: Single input, element-wise computation
*   **Eltwise Binary**: Two inputs, element-wise computation
*   **Eltwise N-ary**: Multiple inputs (concat, stack, where)
*   **Tensor Manipulation**: Shape changes without computation
*   **Neural Network**: Layers with learnable parameters
