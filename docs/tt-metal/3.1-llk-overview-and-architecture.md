## Purpose and Scope

This document describes the Low-Level Kernel (LLK) APIs that provide the programming interface for writing compute kernels that execute on Tenstorrent cores. The LLK layer sits between the Metalium runtime system and the hardware, exposing C++ APIs in header files like `llk_pack_api.h`, `llk_unpack_*.h`, and `llk_math_*.h`. These APIs are used by the TRISC (Tensix RISC) processors to perform data movement and math operations.

The LLK APIs are architecture-specific, with separate implementations in:

*   `tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/` - Wormhole B0 architecture [tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu_types.h 1-167](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu_types.h#L1-L167)
*   `tt_metal/hw/ckernels/blackhole/metal/llk_api/` - Blackhole architecture [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu_types.h 1-167](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu_types.h#L1-L167)

The LLK system interacts closely with the Hardware Abstraction Layer (HAL), which defines the memory maps and processor identifiers for each architecture.

**Sources:**[tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu_types.h 1-167](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu_types.h#L1-L167)[tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu_types.h 1-167](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu_types.h#L1-L167)

## LLK Layer in System Architecture

The LLK layer provides four categories of APIs:

1.   **Dataflow API**: NOC read/write and circular buffer operations [tt_metal/hw/inc/api/dataflow/dataflow_api.h 1-180](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/inc/api/dataflow/dataflow_api.h#L1-L180)
2.   **Unpack APIs** (`llk_unpack_*.h`): Load data from circular buffers to SRCA/SRCB registers [tt_metal/hw/ckernels/quasar/metal/llk_io/llk_io_unpack.h 1-20](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/quasar/metal/llk_io/llk_io_unpack.h#L1-L20)
3.   **Math/Compute APIs** (`llk_math_*.h`): Execute operations on FPU/SFPU [tt_metal/hw/inc/api/compute/compute_kernel_api.h 1-50](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/inc/api/compute/compute_kernel_api.h#L1-L50)
4.   **Pack APIs** (`llk_pack_*.h`): Write results from DEST register to circular buffers [tt_metal/hw/ckernels/quasar/metal/llk_io/llk_io_pack.h 1-20](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/quasar/metal/llk_io/llk_io_pack.h#L1-L20)

## SFPU Operations and Architecture

The SFPU (Special Function Processing Unit) provides high-precision transcendental and math functions. These are implemented using the SFPI toolchain, which allows for vector-style C++ programming on the Tenstorrent hardware. The SFPI toolchain is currently at version 7.55.0 [tt_metal/sfpi-version 4](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/sfpi-version#L4-L4)

### SFPU Implementation Characteristics

SFPU operations use specific kernels for complex math like trigonometry (sine, tangent) and exponential functions. The `SfpuType` enumeration defines the full suite of supported operations for each architecture [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu_types.h 7-167](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu_types.h#L7-L167)

## Dataflow and NOC Infrastructure

Kernels interact with the Network-on-Chip (NOC) through the `noc_nonblocking_api.h`. This API provides functions for issued reads/writes and tracking their completion via barrier counters [tt_metal/hw/inc/internal/tt-1xx/blackhole/noc_nonblocking_api.h 85-112](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/inc/internal/tt-1xx/blackhole/noc_nonblocking_api.h#L85-L112)

## Unary and Composite Operations

The LLK layer is the foundation for both simple unary operations and more complex composite operations.

### Unary Operations

Unary operations are typically mapped directly to a single `UnaryOpType` and executed via the `unary_impl` helper [ttnn/cpp/ttnn/operations/eltwise/unary/unary.cpp 14-49](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/unary.cpp#L14-L49) This function determines the necessary precision (e.g., `fp32_dest_acc_en`) and data formats for the kernel [ttnn/cpp/ttnn/operations/eltwise/unary/unary.cpp 28-31](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/unary.cpp#L28-L31)

### Composite Operations

Composite operations are implemented by chaining multiple unary or binary operations together. Examples include:

*   **Multigammaln**: Uses `ttnn::lgamma` and `ttnn::add`[ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_composite_op.cpp 111-130](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_composite_op.cpp#L111-L130)
*   **Normalize**: Combines `ttnn::mean`, `ttnn::bcast`, and `ttnn::reciprocal`[ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_composite_op.cpp 148-156](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_composite_op.cpp#L148-L156)
*   **GLU (Gated Linear Unit)**: Involves splitting tensors and applying activations [ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_composite_op.cpp 60-79](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_composite_op.cpp#L60-L79)

## Parameterized Operations

Many LLK operations require runtime parameters (e.g., the slope in `LeakyRelu` or the threshold value). The `EltwiseUnaryWithParam` struct is used to pass these parameters from the host to the compute kernels [ttnn/cpp/ttnn/operations/eltwise/unary/common/unary_op_types.hpp 250-252](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/common/unary_op_types.hpp#L250-L252)

The `get_op_init_and_func_parameterized` function generates the specific kernel initialization and function calls based on the input data type and operation parameters [ttnn/cpp/ttnn/operations/eltwise/unary/common/unary_op_utils.cpp 112-149](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/ttnn/cpp/ttnn/operations/eltwise/unary/common/unary_op_utils.cpp#L112-L149)
