## Purpose and Scope

The `flash` command is the primary subcommand of `tt-flash` that writes firmware bundles to Tenstorrent AI accelerator chips. This page documents the command-line interface, all available options, and the execution workflow.

For information about the firmware bundle format being flashed, see [Firmware Bundle Format](https://deepwiki.com/tenstorrent/tt-flash/5.1-firmware-bundle-format). For details on the underlying flash operations, see [Flash Stages](https://deepwiki.com/tenstorrent/tt-flash/5.2-flash-stages). For the `verify` subcommand, see [verify Command](https://deepwiki.com/tenstorrent/tt-flash/3.2-verify-command).

* * *

## Command Syntax

`tt-flash flash [OPTIONS] <fwbundle>`
**Positional Arguments:**

*   `fwbundle` - Path to the firmware bundle file (`.fwbundle` extension)

**Optional Arguments:**

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `--force` | flag | False | Force ROM update regardless of version checks |
| `--no-reset` | flag | False | Skip automatic device reset after flashing |
| `--skip-missing-fw` | flag | False | Continue flashing if bundle lacks firmware for detected boards |
| `--allow-major-downgrades` | flag | False | Permit downgrade across major firmware versions |
| `--fw-tar` | path | None | ⚠️ **Deprecated:** Use positional `fwbundle` argument instead |

**Sources:**[tt_flash/main.py 84-131](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py#L84-L131)

## Basic Usage Examples

### Standard Flash Operation

`# Flash all detected devices with firmware bundlett-flash flash ~/tt-firmware/latest.fwbundle`
### Force Flash (Skip Version Checks)

`# Update even if firmware appears currenttt-flash flash --force ~/tt-firmware/v18.12.0.fwbundle`
### Flash Without Automatic Reset

`# Flash firmware but defer device resettt-flash flash --no-reset ~/tt-firmware/latest.fwbundle`
### Downgrade to Older Firmware

`# Allow major version downgrade (e.g., v19.x.x → v18.x.x)tt-flash flash --allow-major-downgrades ~/tt-firmware/v18.10.0.fwbundle`

## Detailed Option Descriptions

### --force

**Purpose:** Bypass version comparison checks and force firmware update regardless of current state.

**Behavior:**

*   Skips all version comparison logic in `flash_chip_stage1()`[flash.py 296-298](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L296-L298)
*   Overrides "firmware already up-to-date" detection [flash.py 301-318](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L301-L318)
*   Allows flashing when running firmware version cannot be determined [flash.py 234-241](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L234-L241)
*   Does **not** bypass major version downgrade protection (use `--allow-major-downgrades` for that)

**Use Cases:**

*   Re-flashing same firmware version (e.g., after corruption)
*   Forcing SPI update when running version lags behind SPI version [flash.py 308-310](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L308-L310)
*   Recovering from unreadable firmware state

**Code Path:**

**Sources:**[tt_flash/flash.py 194-333](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L194-L333)[tt_flash/main.py 100-101](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py#L100-L101)

* * *

### --no-reset

**Purpose:** Prevent automatic device reset after successful firmware flash, leaving the chip in a state where new firmware is in SPI but not yet running.

**Behavior:**

*   Firmware is successfully written to SPI flash [flash.py 495-617](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L495-L617)
*   Reset stage is skipped [flash.py 902-910](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L902-L910)
*   User must manually trigger reset (cold boot or explicit reset command)
*   Post-flash validation is skipped (only runs after reset on BH v19+)

**When to Use:**

*   Testing firmware without immediately activating it
*   Coordinating reset timing across multiple systems
*   Debugging firmware update procedures
*   Deferring reset to maintenance window

**Code Path:**

**Sources:**[tt_flash/flash.py 890-951](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L890-L951)[tt_flash/main.py 103-107](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py#L103-L107)

* * *

### --skip-missing-fw

**Purpose:** Allow flash operation to continue when the firmware bundle lacks images for some detected boards.

**Behavior:**

*   During `flash_chip_stage1()`, if board-specific firmware files are missing [flash.py 336-366](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L336-L366)
*   Without flag: Raises `TTError` with message "Could not find flash data for {boardname}" [flash.py 356-358](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L356-L358)
*   With flag: Prints warning and returns `FlashStageResultState.NoFlash`[flash.py 348-354](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L348-L354)
*   Useful for heterogeneous systems or partial bundle updates

**Example Scenario:**

```
System has: NEBULA_X1, NEBULA_X2, P150a boards
Bundle contains: Only NEBULA_X1 and NEBULA_X2 firmware
Result with --skip-missing-fw: Flash N1, N2; skip P150a with warning
Result without flag: Error and abort entire operation
```

**Code Logic:**

**Sources:**[tt_flash/flash.py 335-366](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L335-L366)[tt_flash/main.py 93-98](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py#L93-L98)

* * *

### --allow-major-downgrades

**Purpose:** Permit firmware downgrade across major version boundaries (e.g., from v19.x.x to v18.x.x).

**Default Behavior:** Major version downgrades are blocked to prevent potential compatibility issues [flash.py 263-273](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L263-L273)

**Version Comparison Logic:**

*   Firmware versions use format `(major, minor, patch, debug)`[flash.py 127-139](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L127-L139)
*   Legacy format `(80, major, minor, patch)` is normalized to `(major, minor, patch, 0)`[flash.py 127-139](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L127-L139)
*   Major version is `version[0]`
*   Comparison: `running_version[0]` vs `manifest_version[0]`

**Allowed Without Flag:**

*   Minor/patch upgrades: v18.10.0 → v18.12.0
*   Major upgrade (single boundary): v18.x.x → v19.x.x [flash.py 274-279](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L274-L279)
*   Same major version

**Blocked Without Flag:**

*   Major downgrade: v19.x.x → v18.x.x [flash.py 263-273](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L263-L273)
*   Multi-major upgrade: v18.x.x → v20.x.x (unless `--force`) [flash.py 280-289](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L280-L289)

**Version Checking Flow:**

**Sources:**[tt_flash/flash.py 127-139](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L127-L139)[tt_flash/flash.py 263-289](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L263-L289)[tt_flash/main.py 128-130](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/main.py#L128-L130)

## Flash Stage Details

### Stage 1: Preparation (flash_chip_stage1)

**Function:**`flash_chip_stage1(chip, boardname, manifest, fw_package, force, allow_major_downgrades, skip_missing_fw)`[flash.py 194-492](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L194-L492)

**Responsibilities:**

1.   Version checking and comparison
2.   Firmware file extraction from bundle
3.   Parameter handling via `TAG_HANDLERS`[flash.py 142-148](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L142-L148)
4.   Write operation preparation (`FlashWrite` objects)
5.   Reset capability determination

**Return Value:**`FlashStageResult` dataclass [flash.py 187-191](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L187-L191)

*   `state`: `Ok`, `NoFlash`, or `Err`
*   `can_reset`: Boolean indicating if automatic reset is supported
*   `msg`: Error message (if applicable)
*   `data`: `FlashData` with write operations [flash.py 173-177](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L173-L177)

**Tag Handlers for Parameter Injection:**

| Handler | Function | Purpose |
| --- | --- | --- |
| `rmw` | `rmw_param()`[flash.py 33-42](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L33-L42) | Read-Modify-Write existing SPI data |
| `incr` | `incr_param()`[flash.py 45-62](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L45-L62) | Increment counter with overflow protection |
| `date` | `date_param()`[flash.py 65-74](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L65-L74) | Inject current date (0xYYYYMMDD format) |
| `flash_version` | `flash_version()`[flash.py 77-97](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L77-L97) | Inject tt-flash tool version |
| `bundle_version` | `bundle_version()`[flash.py 105-124](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L105-L124) | Inject firmware bundle version |

**Sources:**[tt_flash/flash.py 194-492](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L194-L492)[tt_flash/flash.py 142-148](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L142-L148)

### Stage 2: Write and Verify (flash_chip_stage2)

**Function:**`flash_chip_stage2(chip, data)`[flash.py 495-650](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L495-L650)

**Write/Verify Cycle:**

**SIGINT Protection:** Both `perform_write()` and `perform_verify()` install signal handlers to prevent Ctrl-C interruption [flash.py 500-501](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L500-L501)[flash.py 504-511](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L504-L511)[flash.py 514-531](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L514-L531)

**Retry Logic:**

*   First write → First verify
*   If verify fails: Second write → Second verify
*   If second verify fails: Return `None` (fatal error)

**Sources:**[tt_flash/flash.py 495-650](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L495-L650)

## Special Handling by Board Type

### NEBULA_X2 Boards

**Local-to-Remote SPI Copy:** NEBULA_X2 boards have dual SPI flash chips. After flashing the local chip, a copy operation transfers data to the remote chip [flash.py 619-649](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L619-L649)

**Copy Trigger:**`MSG_TRIGGER_SPI_COPY_LtoR` ARC message [flash.py 642](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L642-L642)

**Bootloop Mitigation:** M3 firmware version 5.8.0.1 has a bug that can trigger a boot loop during L→R copy. Mitigation: Disable M3 auto-reset before triggering copy [flash.py 625-639](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L625-L639)

**Copy Wait Time:** 15 seconds for remote copy completion [flash.py 884-888](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L884-L888)

* * *

### P300 Series Boards

**Location-Based Naming:** P300 boards have left and right ASICs that use different firmware images [flash.py 830-837](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L830-L837)

**ASIC Location Detection:**

`# 0 = Right, 1 = Leftif dev.get_asic_location() == 0:    boardname = f"{boardname}_right"elif dev.get_asic_location() == 1:    boardname = f"{boardname}_left"`
**Sources:**[tt_flash/flash.py 619-649](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L619-L649)[tt_flash/flash.py 830-837](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L830-L837)

## Reset Behavior by Platform

### Galaxy Systems (GALAXY-1, WH_UBB)

**Reset Method:** IPMI-based tray reset via `glx_6u_trays_reset()`[flash.py 738-789](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L738-L789)

**Procedure:**

1.   Send IPMI reset command: `run_wh_ubb_ipmi_reset()`[flash.py 760](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L760-L760)
2.   Wait 30 seconds for hardware reset [flash.py 761](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L761-L761)
3.   Wait for driver reload: `run_ubb_wait_for_driver_load()`[flash.py 762](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L762-L762)
4.   Re-enumerate chips with `detect_chips_with_callback(local_only=True, ignore_ethernet=True)`[flash.py 777](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L777-L777)
5.   Validate Ethernet link status (WH Galaxy only): `check_wh_galaxy_eth_link_status()`[flash.py 783](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L783-L783)

**Ethernet Link Validation:** Checks all 16 Ethernet links on 32 devices for `LINK_INACTIVE_FAIL_DUMMY_PACKET` errors [flash.py 700-735](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L700-L735)

**Sources:**[tt_flash/flash.py 738-789](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L738-L789)[tt_flash/flash.py 915-925](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L915-L925)

### Standard Blackhole Chips

**Reset Method:**`BHChipReset().full_lds_reset(pci_interfaces, reset_m3=True, m3_delay)`[flash.py 933-936](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L933-L936)

**Adaptive M3 Delay:**

*   Default: 20 seconds for M3 boot after reset [flash.py 893](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L893-L893)
*   Major version boundary crossing: 60 seconds [flash.py 895-900](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L895-L900)

**Post-Flash Validation (BH v19+):**

*   Sends `MSG_CONFIRM_FLASHED_SPI` with random challenge value [flash.py 941-951](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L941-L951)
*   Validates response matches challenge (cryptographic confirmation firmware is running)
*   Warning printed if validation fails

## Error Handling and Return Codes

**Return Value:** Integer return code from `flash_chips()`[flash.py 792-958](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L792-L958)

| Return Code | Meaning |
| --- | --- |
| `0` | All flash operations successful |
| `> 0` | Number of chips with flash failures |

**Fatal Errors (TTError exceptions):**

*   Invalid firmware bundle format [main.py 217](https://github.com/tenstorrent/tt-flash/blob/60c06032/main.py#L217-L217)
*   Unrecognized board type [flash.py 828](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L828-L828)
*   Major version downgrade without `--allow-major-downgrades`[flash.py 270-273](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L270-L273)
*   Major version jump without `--force`[flash.py 286-289](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L286-L289)
*   Missing firmware files without `--skip-missing-fw`[flash.py 356-366](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L356-L366)
*   Invalid mask.json format [flash.py 403-424](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L403-L424)

**Non-Fatal Errors:**

*   Second verification failure: Returns `None` from `flash_chip_stage2()`, increments error counter [flash.py 605-611](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L605-L611)
*   Automatic reset skipped if errors detected [flash.py 912-913](https://github.com/tenstorrent/tt-flash/blob/60c06032/flash.py#L912-L913)

**Sources:**[tt_flash/flash.py 792-958](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/flash.py#L792-L958)[tt_flash/error.py](https://github.com/tenstorrent/tt-flash/blob/60c06032/tt_flash/error.py)
