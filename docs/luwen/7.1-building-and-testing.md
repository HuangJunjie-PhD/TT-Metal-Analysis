## Building the Project

### Basic Build

To build the Luwen workspace with default settings:

`cargo build`
This builds all crates in debug mode. For better performance with debugging symbols:

`cargo build --release`
Sources: [.gitlab-ci.yml 58](https://github.com/tenstorrent/luwen/blob/55e35616/.gitlab-ci.yml#L58-L58)

### Build Configurations

Luwen supports multiple build configurations:

| Configuration | Command | Description |
| --- | --- | --- |
| Debug | `cargo build` | Includes debugging symbols, no optimizations |
| Release | `cargo build --release` | Optimized binary with minimal debug info |
| Specific target | `cargo build --target x86_64-unknown-linux-gnu` | Build for a specific architecture |
| All features | `cargo build --all-features` | Build with all optional features enabled |

Sources: [.gitlab-ci.yml 52-58](https://github.com/tenstorrent/luwen/blob/55e35616/.gitlab-ci.yml#L52-L58)

## Testing

### Test Organization

The Luwen codebase contains several types of tests, organized by functionality and hardware requirements:

Sources: [tests/telem_test.rs 1-22](https://github.com/tenstorrent/luwen/blob/55e35616/tests/telem_test.rs#L1-L22)[tests/boot_fs_test.rs 1-21](https://github.com/tenstorrent/luwen/blob/55e35616/tests/boot_fs_test.rs#L1-L21)[tests/read_write_test.rs 1-21](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L1-L21)

### Running Tests

#### Basic Test Execution

To run all non-hardware tests:

`cargo test`
#### Hardware-Dependent Tests

Hardware tests are marked with `#[ignore = "Requires hardware"]` and can be run with:

`cargo test -- --ignored`
To run a specific hardware test file:

`cargo test --test telem_test -- --ignored`
Sources: [tests/telem_test.rs 15-19](https://github.com/tenstorrent/luwen/blob/55e35616/tests/telem_test.rs#L15-L19)[tests/boot_fs_test.rs 15-19](https://github.com/tenstorrent/luwen/blob/55e35616/tests/boot_fs_test.rs#L15-L19)[tests/read_write_test.rs 15-19](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L15-L19)

### Chip-Specific Tests

The repository contains tests specific to each supported chip architecture:

| Chip Type | Test Command | Required Hardware Tag |
| --- | --- | --- |
| Wormhole | `cargo test --test telem_test -- --ignored wormhole` | wh-nebula_x2-card |
| Grayskull | `cargo test --test telem_test -- --ignored grayskull` | gs-1-card |
| Blackhole | `cargo test --test boot_fs_test -- --ignored blackhole` | N/A |

The tests automatically detect if compatible hardware is present; if not found, the test will be skipped.

Sources: [tests/telem_test.rs 27-72](https://github.com/tenstorrent/luwen/blob/55e35616/tests/telem_test.rs#L27-L72)[tests/boot_fs_test.rs 22-96](https://github.com/tenstorrent/luwen/blob/55e35616/tests/boot_fs_test.rs#L22-L96)[.gitlab-ci.yml 89-97](https://github.com/tenstorrent/luwen/blob/55e35616/.gitlab-ci.yml#L89-L97)

### Test Categories

#### 1. Telemetry Tests

Tests for chip telemetry collection, status reporting, and architecture detection.

`cargo test --test telem_test -- --ignored`
Sources: [tests/telem_test.rs 1-172](https://github.com/tenstorrent/luwen/blob/55e35616/tests/telem_test.rs#L1-L172)

#### 2. Boot File System Tests

Tests for reading boot FS tables from Blackhole chips' SPI ROM.

`cargo test --test boot_fs_test -- --ignored`
Sources: [tests/boot_fs_test.rs 1-97](https://github.com/tenstorrent/luwen/blob/55e35616/tests/boot_fs_test.rs#L1-L97)

#### 3. Read/Write Tests

Tests for PCI device register read/write operations, including aligned and unaligned access.

`cargo test --test read_write_test -- --ignored`
Sources: [tests/read_write_test.rs 1-467](https://github.com/tenstorrent/luwen/blob/55e35616/tests/read_write_test.rs#L1-L467)

### Test Utility Functions

The codebase provides utility functions for hardware detection in tests:

| Function | Purpose |
| --- | --- |
| `hardware_available()` | Checks if any compatible hardware is available |
| `has_chip_type(F)` | Checks if any chip of a specific type is available |

Example usage:

`if !test_utils::has_chip_type(|chip| chip.as_wh().is_some()) {    return; // Skip the test if no Wormhole chips are found}`
Sources: [tests/test_utils.rs 1-68](https://github.com/tenstorrent/luwen/blob/55e35616/tests/test_utils.rs#L1-L68)

### Creating Debian Packages

Luwen can be packaged as a Debian package using:

`cargo install --locked cargo-debmake deb`
This creates a `.deb` package for installation on Debian-based systems.

## Troubleshooting

### Common Build Issues

1.   **Missing protobuf-compiler**:

```
error: failed to run custom build command for `luwen-*`
```

Solution: Install protobuf-compiler (`apt install -y protobuf-compiler`)

2.   **Hardware tests failing**: Ensure you're running hardware-specific tests on compatible hardware or use the test utility functions to skip tests when hardware is not available.

3.   **Cross-compilation issues**: The default cross RISCV image doesn't work with SIMD instructions. Use the custom image from the CI configuration.

Sources: [.gitlab-ci.yml 120-136](https://github.com/tenstorrent/luwen/blob/55e35616/.gitlab-ci.yml#L120-L136)
