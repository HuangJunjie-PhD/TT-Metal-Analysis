## Purpose and Scope

This document describes the four core manager classes that provide reusable infrastructure for all training experiments in tt-blacksmith: `ReproducibilityManager`, `TrainingLogger`, `CheckpointManager`, and `DeviceManager`. These managers handle cross-cutting concerns including random seeding, experiment logging, checkpoint persistence, and device/parallelism configuration.

For information about the configuration system that drives these managers, see [Configuration System](https://deepwiki.com/tenstorrent/tt-blacksmith/5.1-manager-classes#3.2). For details about specific parallelism strategies, see [Multi-Chip Parallelism](https://deepwiki.com/tenstorrent/tt-blacksmith/5.1-manager-classes#3.6). For checkpoint retention policies and artifact logging, see [Checkpoint Management](https://deepwiki.com/tenstorrent/tt-blacksmith/5.1-manager-classes#3.5.3). For Weights & Biases integration details, see [Logging and Monitoring](https://deepwiki.com/tenstorrent/tt-blacksmith/5.1-manager-classes#3.5.2).

## Manager Overview

The manager classes form the operational backbone of every training experiment. Each manager is initialized from the `TrainingConfig` object and provides a focused set of capabilities:

| Manager Class | Primary Responsibilities | Key Methods |
| --- | --- | --- |
| `ReproducibilityManager` | Random seed control, deterministic behavior | `setup()` |
| `TrainingLogger` | Metrics logging, W&B integration, artifact tracking | `log_metrics()`, `log_artifact()`, `finish()` |
| `CheckpointManager` | Save/load checkpoints, retention policies | `save_checkpoint()`, `load_checkpoint()`, `should_save_checkpoint()` |
| `DeviceManager` | Device selection, mesh creation, model/batch sharding | `prepare_batch()`, `shard_model()`, `optimizer_step()` |

## ReproducibilityManager

The `ReproducibilityManager` configures random number generation and deterministic behavior to ensure reproducible training runs.

**Configuration Parameters:**

The manager reads two fields from `TrainingConfig`:

*   `seed`: Integer seed value (default: 23)
*   `deterministic`: Boolean flag for deterministic operations (default: False)

**Setup Operations:**

When `setup()` is called, the manager:

1.   Sets the Python random seed
2.   Sets the NumPy random seed
3.   Sets the PyTorch manual seed
4.   Configures PyTorch deterministic algorithms if `deterministic=True`

**Usage Pattern:**

The `setup()` method must be called before any model initialization, dataset loading, or training operations to ensure all random operations use the configured seed.

## TrainingLogger

The `TrainingLogger` provides unified logging infrastructure with Weights & Biases integration for experiment tracking and visualization.

**Initialization:**

The logger reads configuration from multiple fields in `TrainingConfig`:

*   `log_level`: Python logging level (default: "INFO")
*   `use_wandb`: Enable W&B logging (default: True)
*   `wandb_project`: W&B project name
*   `wandb_run_name`: W&B run name
*   `wandb_tags`: List of tags for run organization
*   `wandb_watch_mode`: Model watching mode ("all", "gradients", "parameters", "none")
*   `wandb_log_freq`: Frequency for gradient/parameter logging

**Key Methods:**

| Method | Purpose | Example |
| --- | --- | --- |
| `log_metrics(metrics_dict, step, commit)` | Log scalar metrics to W&B and console | `logger.log_metrics({"train/loss": 0.5}, step=100)` |
| `log_artifact(path, artifact_type, name)` | Upload files to W&B as artifacts | `logger.log_artifact("model.pth", "model", "final_model")` |
| `info(message)` | Log informational message | `logger.info("Starting training...")` |
| `error(message, traceback)` | Log error with traceback | `logger.error(f"Failed: {e}", tb)` |
| `finish()` | Close W&B run and flush logs | `logger.finish()` |

## CheckpointManager

The `CheckpointManager` handles checkpoint persistence, resume logic, and retention policies for both step-based and epoch-based checkpointing strategies.

**Initialization:**

The manager requires both the config and logger objects. The logger is used for artifact uploads to Weights & Biases.

**Configuration Parameters:**

| Parameter | Purpose | Default |
| --- | --- | --- |
| `resume_from_checkpoint` | Enable checkpoint resume | False |
| `resume_option` | Resume strategy: "last", "best", or "path" | "last" |
| `checkpoint_path` | Explicit checkpoint path (if resume_option="path") | "" |
| `checkpoint_metric` | Metric for "best" selection | "val/loss" |
| `checkpoint_metric_mode` | "min" or "max" for best selection | "min" |
| `keep_last_n` | Number of recent checkpoints to retain | 3 |
| `keep_best_n` | Number of best checkpoints to retain | 1 |
| `save_strategy` | "step" or "epoch" for save frequency | "epoch" |
| `steps_freq` | Step frequency (if save_strategy="step") | 100 |
| `epoch_freq` | Epoch frequency (if save_strategy="epoch") | 5 |
| `save_optim` | Include optimizer state in checkpoint | False |
| `project_dir` | Base directory for checkpoint storage | varies |

## DeviceManager

The `DeviceManager` is the most complex manager, handling device selection, mesh topology for multi-chip parallelism, model sharding for tensor parallelism, batch sharding for data parallelism, and optimizer synchronization.

**Initialization:**

**Device Selection and Environment Setup:**

The manager's `_setup()` method [blacksmith/tools/device_manager.py 28-36](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/tools/device_manager.py#L28-L36) determines the execution device:

1.   If `config.use_tt=False`: Use CPU or CUDA device
2.   If `config.use_tt=True`: Call `_setup_tt_environment()`[blacksmith/tools/device_manager.py 38-50](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/tools/device_manager.py#L38-L50)
    *   Set `PJRT_DEVICE=TT`
    *   Set `XLA_STABLEHLO_COMPILE=1`
    *   If multi-chip (mesh_shape provided): 
        *   Set `XLA_ALWAYS_ALLREDUCE=1`
        *   Set `MESH_SHAPE` environment variable
        *   Set `CONVERT_SHLO_TO_SHARDY=1`
        *   Set `DISABLE_NUMERIC_CC_TOKEN=1`
        *   Enable SPMD with `xr.use_spmd()`
