The `tt-perf-report` tool is a performance analysis utility designed to help developers optimize machine learning workloads running on Tenstorrent hardware. It analyzes performance traces generated during TT-Metal operations, providing detailed insights into computational throughput, memory bandwidth utilization, and potential bottlenecks. For installation instructions, see [Installation](/tenstorrent/tt-perf-report/2-installation), and for detailed usage information, see [Usage Guide](/tenstorrent/tt-perf-report/3-usage-guide).

## Purpose and Functionality

The tt-perf-report tool serves as a critical bridge between raw performance data and actionable insights by:

1. Analyzing CSV performance traces from TT-Metal operations
2. Identifying performance bottlenecks and classifying operations
3. Quantifying resource utilization (DRAM bandwidth, compute FLOPs)
4. Providing targeted optimization advice based on detected bottlenecks
5. Presenting results in an easily digestible tabular format

## System Architecture

The tt-perf-report tool consists of several interrelated components that form a data processing pipeline:

The system processes data through the following components:

1. **Trace Parser**: Reads CSV performance trace files generated by Tracy profiler, filtering data based on signposts or operation ID ranges.
2. **Performance Analyzer**: Calculates key metrics for each operation, including:

   * Device execution time
   * Operation gaps
   * DRAM bandwidth utilization
   * Computational throughput (FLOPs)
   * Core utilization
3. **Operation Classifier**: Categorizes operations as:

   * DRAM-bound (>65% of peak DRAM bandwidth)
   * FLOP-bound (>65% of peak computational capacity)
   * BOTH (both DRAM and FLOP bound)
   * SLOW (neither resource is effectively utilized)
   * HOST (operations running on the host CPU)
4. **Optimization Advisor**: Generates tailored suggestions based on operation classification and performance characteristics.
5. **Report Generator**: Formats results as a color-highlighted table, optionally exporting to CSV.

## Key Performance Metrics and Classification

The tt-perf-report tool focuses on several critical performance metrics:

| Metric | Description | Significance |
| --- | --- | --- |
| Device Time | Time spent executing on device (μs) | Core performance metric |
| Op-to-op Gap | Time between operations (μs) | Indicates host overhead |
| DRAM Bandwidth | Memory bandwidth (GB/s) | Memory subsystem utilization |
| DRAM % | Percentage of peak bandwidth | Identifies memory bottlenecks |
| FLOPs | Compute throughput (TFLOPs) | Computational efficiency |
| FLOPs % | Percentage of theoretical peak | Identifies compute bottlenecks |
| Cores | Number of cores used | Resource utilization |
| Math Fidelity | Precision configuration | Impacts peak FLOP capacity |

Based on these metrics, the tool classifies operations to identify bottlenecks:

This classification system provides the foundation for targeted optimization advice.

## Command-Line Interface

The tt-perf-report tool is accessed via a command-line interface, providing various options for filtering and analyzing trace data:

```
tt-perf-report [trace.csv] [options] 
```

Key options include:

* Signpost filtering (`--signpost`, `--ignore-signposts`)
* Operation ID range selection (`--id-range`)
* Output customization (`--min-percentage`, `--color/--no-color`, `--csv`)
* Analysis control (`--no-advice`)

For a comprehensive list of command-line options, see [Command Line Options](/tenstorrent/tt-perf-report/3.1-command-line-options).

## Performance Report Output

The tool generates a tabular report highlighting key performance metrics for each operation:

1. **Basic timing information**: Op ID, name, device time, op-to-op gap, total percentage
2. **Resource utilization**: DRAM bandwidth, FLOPs, core count
3. **Classification**: Performance bottleneck category
4. **Optimization advice**: Tailored suggestions based on operation classification

The report uses visual highlights to draw attention to areas needing optimization:

* Underutilized resources (low core count)
* High host overhead (large op-to-op gaps)
* Operations with poor resource utilization

For detailed information on the report format, see [Performance Report Format](/tenstorrent/tt-perf-report/4-performance-report-format).
