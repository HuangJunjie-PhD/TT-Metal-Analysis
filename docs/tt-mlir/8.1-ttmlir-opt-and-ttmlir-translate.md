## Purpose and Scope

This document describes the primary command-line tools in the `tt-mlir` compiler infrastructure: `ttmlir-opt`, `ttmlir-translate`, and the supporting `ttmlir-lsp-server`. These tools provide the interface for applying transformation passes, converting between different IR formats, and providing IDE support during the compilation process.

*   **ttmlir-opt** is the pass manager that applies transformation pipelines to MLIR modules, enabling progressive lowering through the dialect hierarchy (e.g., StableHLO → TTIR → TTNN). It is the primary tool for IR-to-IR transformations [tools/ttmlir-opt/ttmlir-opt.cpp 13-23](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-opt/ttmlir-opt.cpp#L13-L23)
*   **ttmlir-translate** handles format conversions between MLIR text representation and external formats, such as binary Flatbuffer files for runtime execution or target source code (C++ or Python) [tools/ttmlir-translate/ttmlir-translate.cpp 33-50](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L33-L50)
*   **ttmlir-lsp-server** provides Language Server Protocol support for Tenstorrent dialects, enabling features like syntax highlighting and diagnostics in editors [tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp 1-20](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp#L1-L20)

## Tool Architecture Overview

Both tools are built using standard MLIR tool headers and link against the `TTMLIRCompilerStatic` library [lib/CMakeLists.txt 71-82](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/CMakeLists.txt#L71-L82) They serve as drivers that register custom Tenstorrent dialects and passes into the MLIR context.

### Tool Registration and Entry Points

The tools initialize by registering all available MLIR and Tenstorrent-specific dialects and passes via centralized registration functions. `ttmlir-opt` specifically calls `mlir::tt::registerAllPasses()`, `mlir::tt::registerAllDialects()`, and `mlir::tt::registerAllExtensions()` to populate its registry [tools/ttmlir-opt/ttmlir-opt.cpp 14-19](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-opt/ttmlir-opt.cpp#L14-L19)

| Tool | Entry Point | Primary Responsibility |
| --- | --- | --- |
| `ttmlir-opt` | `tools/ttmlir-opt/ttmlir-opt.cpp` | Running optimization and lowering passes [tools/ttmlir-opt/ttmlir-opt.cpp 13-23](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-opt/ttmlir-opt.cpp#L13-L23) |
| `ttmlir-translate` | `tools/ttmlir-translate/ttmlir-translate.cpp` | Exporting IR to Flatbuffer, C++, or Python [tools/ttmlir-translate/ttmlir-translate.cpp 45-50](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L45-L50) |
| `ttmlir-lsp-server` | `tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp` | Providing IDE backend for MLIR source [tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp 1-20](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp#L1-L20) |

## ttmlir-opt: Pass Management Tool

### Overview

`ttmlir-opt` is the specialized MLIR pass manager for Tenstorrent hardware. It loads a registry containing all project dialects (TTIR, TTNN, TTKernel, D2M, etc.) and makes them available for command-line pipeline construction [tools/ttmlir-opt/ttmlir-opt.cpp 17-19](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-opt/ttmlir-opt.cpp#L17-L19) It is a thin wrapper around `mlir::MlirOptMain`[tools/ttmlir-opt/ttmlir-opt.cpp 21-22](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-opt/ttmlir-opt.cpp#L21-L22)

### Pass Registration

Passes are registered via `mlir::tt::registerAllPasses()`, which internally calls `mlir::tt::registerTTMLIRConversionPasses()`[lib/RegisterAll.cpp 142-144](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/RegisterAll.cpp#L142-L144) This includes conversions like:

*   `ConvertStableHLOToTTIR`[include/ttmlir/Conversion/Passes.td 11-20](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/include/ttmlir/Conversion/Passes.td#L11-L20)
*   `TTIRToTTNN`[include/ttmlir/Conversion/Passes.td 240-244](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/include/ttmlir/Conversion/Passes.td#L240-L244) (Note: Reference from full pass list)
*   `TTIRToD2M`[include/ttmlir/Conversion/Passes.td 65-124](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/include/ttmlir/Conversion/Passes.td#L65-L124)
*   `TTIRToTTIRDecomposition`[include/ttmlir/Conversion/Passes.td 45-62](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/include/ttmlir/Conversion/Passes.td#L45-L62)
*   `ConvertTosaToTTIR`[include/ttmlir/Conversion/Passes.td 39-43](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/include/ttmlir/Conversion/Passes.td#L39-L43)

### Usage in Testing

`ttmlir-opt` is the core driver for Lit tests. It validates transformations by running specific pipelines and checking the output against expected IR patterns. The `TTMLIR_TEST_DEPENDS` variable in the build system explicitly lists `ttmlir-opt` as a requirement for the test suite [test/CMakeLists.txt 35-40](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/CMakeLists.txt#L35-L40)

## ttmlir-translate: Format Conversion Tool

### Overview

`ttmlir-translate` is responsible for converting MLIR modules to non-MLIR formats. It acts as the final stage of the compilation process, producing artifacts that can be executed on hardware or integrated into other software stacks. It registers custom Tenstorrent-specific translations via `registerCustomTranslations()`[tools/ttmlir-translate/ttmlir-translate.cpp 45-48](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L45-L48)

### Key Translation Targets

The tool supports several custom translations:

1.   **TTNN Flatbuffer**: Registered via `mlir::tt::ttnn::registerTTNNToFlatbuffer()`[tools/ttmlir-translate/ttmlir-translate.cpp 35](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L35-L35)
2.   **TTMetal Flatbuffer**: Registered via `mlir::tt::ttmetal::registerTTMetalToFlatbuffer()`[tools/ttmlir-translate/ttmlir-translate.cpp 36](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L36-L36)
3.   **TTKernel to C++**: Registered via `mlir::tt::ttkernel::registerTTKernelToCpp()`[tools/ttmlir-translate/ttmlir-translate.cpp 38](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L38-L38) This handles low-level kernel generation.
4.   **EmitPy**: Registered via `mlir::tt::emitpy::registerToPythonTranslation()`[tools/ttmlir-translate/ttmlir-translate.cpp 39](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L39-L39)
5.   **LLVM to Dynamic Library**: Registered via `mlir::tt::llvm_to_cpu::registerLLVMToDynamicLibrary()`[tools/ttmlir-translate/ttmlir-translate.cpp 37](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-translate/ttmlir-translate.cpp#L37-L37)

## Related Infrastructure

### ttmlir-lsp-server

The LSP server enables IDE integration (syntax highlighting, diagnostics). It registers the Tenstorrent dialect registry to ensure consistency across the development environment [tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp 1-20](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/tools/ttmlir-lsp-server/ttmlir-lsp-server.cpp#L1-L20) It uses the same `registerAllDialects` logic as the other tools to support custom Tenstorrent types and attributes.

### Build System and Environment

The tools are built as part of the core toolset and depend on the `TTMLIRCompilerStatic` library [lib/CMakeLists.txt 71-82](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/CMakeLists.txt#L71-L82) Build options like `TTMLIR_ENABLE_TOOLS` control whether these are generated [CMakeLists.txt 46](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/CMakeLists.txt#L46-L46)
