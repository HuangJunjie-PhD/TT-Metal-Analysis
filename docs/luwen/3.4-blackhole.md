## Overview

This document describes the Blackhole chip implementation in the Luwen project. Blackhole is one of the three Tenstorrent AI accelerator chip architectures supported by Luwen, alongside Wormhole ([Wormhole](https://deepwiki.com/tenstorrent/luwen/3.2-wormhole)) and Grayskull ([Grayskull](https://deepwiki.com/tenstorrent/luwen/3.3-grayskull)). The Blackhole implementation provides interfaces for chip initialization, communication, telemetry data collection, SPI flash access, and boot file system operations.

Sources: [crates/luwen-if/src/chip/blackhole.rs 1-52](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L1-L52)

## Architecture

Blackhole is implemented as a struct that provides the `ChipImpl` trait, which is the common interface for all chip implementations in Luwen. The architecture consists of core components that facilitate communication with the hardware, manage message queues, and provide access to various chip features.

Sources: [crates/luwen-if/src/chip/blackhole.rs 52-64](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L52-L64)[crates/luwen-if/src/chip/blackhole.rs 78-93](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L78-L93)[crates/luwen-if/src/chip/blackhole/message.rs 16-27](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole/message.rs#L16-L27)

### Core Components

The Blackhole implementation includes several key components:

1.   **Chip Interface (`chip_if`)**: An implementation of `ChipInterface` that provides the base hardware interaction capabilities.

2.   **Communication Interface (`arc_if`)**: An implementation of `ChipComms` that provides communication methods for interacting with the chip.

3.   **Message Queue**: A queue system for sending and receiving messages to and from the chip's ARC processor.

4.   **Ethernet Configuration**:

    *   `eth_locations`: An array of Ethernet core locations on the chip
    *   `eth_addrs`: Ethernet addresses for remote communication

5.   **Memory Addresses**:

    *   `spi_buffer_addr`: Address for SPI buffer operations
    *   `telemetry_struct_addr`: Address for telemetry data structure
    *   `scratch_ram_base`: Base address for scratch RAM

Sources: [crates/luwen-if/src/chip/blackhole.rs 52-64](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L52-L64)

## Initialization

The Blackhole chip initialization process involves setting up the communication interfaces, configuring the memory addresses, and preparing the Ethernet locations.

Sources: [crates/luwen-if/src/chip/blackhole.rs 100-201](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L100-L201)[crates/luwen-if/src/chip/blackhole.rs 494-563](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L494-L563)

The `update_init_state` method checks the initialization status of various components:

*   ARC processor status
*   DRAM status
*   Ethernet status
*   CPU status

Each component's status is tracked using a `ComponentStatusInfo` structure that maintains wait status, start time, and timeout information.

Sources: [crates/luwen-if/src/chip/blackhole.rs 462-492](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L462-L492)[crates/luwen-if/src/chip/blackhole.rs 497-562](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L497-L562)

## Communication

### ARC Messaging

Blackhole implements ARC messaging through a message queue system. The `bh_arc_msg` method sends commands to the chip's ARC processor and receives responses.

Sources: [crates/luwen-if/src/chip/blackhole.rs 237-315](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L237-L315)[crates/luwen-if/src/chip/blackhole.rs 569-583](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L569-L583)[crates/luwen-if/src/chip/blackhole/message.rs 157-175](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole/message.rs#L157-L175)

The message queue system works as follows:

1.   Messages are sent to the chip using `push_request`
2.   The chip processes the messages and puts responses in a response queue
3.   Responses are retrieved using `pop_response`
4.   The `trigger_int` method is used to signal the chip that a new message is available

Sources: [crates/luwen-if/src/chip/blackhole/message.rs 75-175](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole/message.rs#L75-L175)

### SPI Flash Access

Blackhole provides methods for reading from and writing to SPI flash memory:

Sources: [crates/luwen-if/src/chip/blackhole.rs 317-375](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L317-L375)

The SPI access works by:

1.   Getting a buffer in memory using `get_spi_buffer`
2.   For write operations: 
    *   Copying data to the buffer using `axi_write`
    *   Sending a SPI write command to the ARC processor

3.   For read operations: 
    *   Sending a SPI read command to the ARC processor
    *   Reading data from the buffer using `axi_read`

The SPI flash access is used for various operations, including accessing boot file system tables.

Sources: [crates/luwen-if/src/chip/blackhole.rs 317-375](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L317-L375)[crates/luwen-if/src/chip/blackhole.rs 386-399](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L386-L399)

## Telemetry

Blackhole provides telemetry data through the `get_telemetry` method. This method reads telemetry data from memory and populates a `Telemetry` structure with various metrics.

Sources: [crates/luwen-if/src/chip/blackhole.rs 593-696](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L593-L696)[crates/luwen-if/src/chip/blackhole/telemetry_tags.rs 1-39](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole/telemetry_tags.rs#L1-L39)

Telemetry data includes:

*   Board and ASIC identification
*   Temperature readings (ASIC, VREG, board)
*   Clock frequencies (AI, AXI, ARC, L2CPU)
*   Firmware versions
*   Power and thermal information
*   Ethernet and DRAM status

The telemetry data is particularly useful for monitoring and diagnostics, as demonstrated by its use in the Prometheus exporter application.

Sources: [crates/luwen-if/src/chip/blackhole/telemetry_tags.rs 1-39](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole/telemetry_tags.rs#L1-L39)[app/prometheus-exporter/src/main.rs 84-154](https://github.com/tenstorrent/luwen/blob/55e35616/app/prometheus-exporter/src/main.rs#L84-L154)

### Telemetry Tags

Blackhole uses a set of predefined telemetry tags to identify different metrics in the telemetry data. These tags are defined in the `TelemetryTags` enum.

| Tag | Description |
| --- | --- |
| BoardIdHigh | High bits of board ID |
| BoardIdLow | Low bits of board ID |
| AsicId | ASIC identifier |
| AsicTemperature | ASIC temperature reading |
| VregTemperature | Voltage regulator temperature |
| BoardTemperature | Board temperature |
| AiClk | AI clock frequency |
| AxiClk | AXI bus clock frequency |
| ArcClk | ARC processor clock frequency |
| EthFwVersion | Ethernet firmware version |
| DdrFwVersion | DDR firmware version |
| FlashBundleVersion | Flash bundle version |

Sources: [crates/luwen-if/src/chip/blackhole/telemetry_tags.rs 1-39](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole/telemetry_tags.rs#L1-L39)

## Boot File System

Blackhole provides access to the boot file system through the `get_boot_fs_tables_spi_read` and `decode_boot_fs_table` methods. These methods allow reading and decoding configuration data stored in the SPI flash.

Sources: [crates/luwen-if/src/chip/blackhole.rs 386-459](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L386-L459)

The boot file system provides access to various configuration tables:

*   `cmfwcfg` / `origcfg`: Firmware configuration
*   `boardcfg`: Board configuration
*   `flshinfo`: Flash information

These tables are stored as protocol buffers in the SPI flash and are padded to align with SPI sector boundaries.

Sources: [crates/luwen-if/src/chip/blackhole.rs 401-415](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L401-L415)[crates/luwen-if/src/chip/blackhole.rs 425-458](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L425-L458)

## Integration with Luwen

Blackhole integrates with the Luwen system by implementing the `ChipImpl` trait, which provides a common interface for all chip types. This allows the rest of the Luwen system to interact with Blackhole chips without needing to know the specific details of the Blackhole architecture.

Sources: [crates/luwen-if/src/chip/blackhole.rs 494-697](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L494-L697)

### Communication Interface

Blackhole implements the `HlComms` trait, which provides high-level communication methods like AXI and NOC read/write operations. This is done by leveraging the underlying `ChipComms` and `ChipInterface` implementations.

Sources: [crates/luwen-if/src/chip/blackhole.rs 66-76](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/blackhole.rs#L66-L76)[crates/luwen-if/src/chip/hl_comms.rs 9-98](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/hl_comms.rs#L9-L98)

The `HlCommsInterface` trait provides additional higher-level methods built on top of the base `HlComms` methods, such as:

*   `axi_translate`: Translates a string representation of an address to an `AxiData` structure
*   `axi_read_field` / `axi_write_field`: Read/write specific bit fields in memory
*   `axi_sread` / `axi_swrite`: Read/write memory using string address representations

This interface makes it easier to work with memory-mapped registers and hardware-specific address spaces.

Sources: [crates/luwen-if/src/chip/hl_comms.rs 219-334](https://github.com/tenstorrent/luwen/blob/55e35616/crates/luwen-if/src/chip/hl_comms.rs#L219-L334)

## Applications

Blackhole chip support enables various applications built on the Luwen framework, such as the Prometheus exporter for monitoring chip telemetry.

The Prometheus exporter collects telemetry data from all supported chips (including Blackhole) and exposes it as metrics that can be collected by Prometheus for monitoring and alerting purposes.

Key telemetry metrics exposed include:

*   Clock frequencies (AI, AXI, ARC)
*   Temperature readings (ASIC, VREG, board)
*   Voltage and power information
*   Firmware versions
*   PCIe connection details

Sources: [app/prometheus-exporter/src/main.rs 156-244](https://github.com/tenstorrent/luwen/blob/55e35616/app/prometheus-exporter/src/main.rs#L156-L244)
