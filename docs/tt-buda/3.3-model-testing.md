This page documents the model testing framework in PyBuda, which allows developers to verify that neural network models can be correctly compiled and executed on Tenstorrent hardware. The framework focuses on ensuring models produce accurate results compared to reference implementations across different hardware configurations and model types.

## Overview of Model Testing

The PyBuda model testing system provides a structured approach to validate that various models work correctly on Tenstorrent devices. It supports testing models from multiple frameworks (PyTorch, ONNX, TensorFlow) and different model architectures (CNNs, Transformers, etc.).

The primary goals of the model testing framework are:

1.   Ensure models compile successfully for target hardware
2.   Verify correct execution of models on Tenstorrent devices
3.   Validate output accuracy against reference implementations
4.   Support testing across different hardware configurations
5.   Enable both inference and training tests

## Core Components

### verify_module Function

The `verify_module` function is the cornerstone of the model testing framework. It takes a PyBuda module, input shapes, and a verification configuration, then validates the module by:

1.   Compiling the model for the target hardware
2.   Executing it on the specified device (or simulation)
3.   Comparing the results with the reference implementation
4.   Determining if the results meet the accuracy criteria

### VerifyConfig

The `VerifyConfig` class configures the verification process with several important parameters:

| Parameter | Description | Example Values |
| --- | --- | --- |
| `arch` | Target hardware architecture | `BackendDevice.Grayskull`, `BackendDevice.Wormhole_B0`, `BackendDevice.Blackhole` |
| `devtype` | Device type | `BackendType.Silicon`, `BackendType.Golden` |
| `devmode` | Device mode | `DeviceMode.CompileAndRun` |
| `test_kind` | Type of test | `TestKind.INFERENCE`, `TestKind.TRAINING` |
| `pcc` | Pearson Correlation Coefficient threshold | `0.99`, `0.98` |
| `verify_pybuda_codegen_vs_framework` | Compare PyBuda codegen vs framework | `True`, `False` |
| `verify_post_placer` | Verify after placement | `True`, `False` |

The `TestKind` enumeration specifies the type of test being performed:

| Test Kind | Description |
| --- | --- |
| `INFERENCE` | Tests only forward pass execution |

## Best Practices

When writing model tests, follow these best practices:

1.   **Separate configuration from model logic**: Keep hardware-specific configurations separate from the model definition

2.   **Use appropriate PCC thresholds**: Set Pearson Correlation Coefficient thresholds based on model precision requirements

 
3.   **Handle architecture-specific requirements**: Use conditional logic for architecture-specific settings

 
4.   **Use pytest parametrization**: Parametrize tests to run the same model with different configurations

 
5.   **Preprocess inputs appropriately**: Ensure inputs match the model's expected format
