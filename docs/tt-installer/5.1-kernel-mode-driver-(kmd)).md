## Purpose and Overview

The Kernel-Mode Driver (KMD) is a critical low-level component in the Tenstorrent software stack. It serves as the primary interface between the operating system kernel and Tenstorrent hardware accelerators. This Linux kernel module enables direct hardware access, memory management, and device control operations necessary for all higher-level Tenstorrent tools and applications to function.

This document explains the KMD's purpose, installation process, and integration with other Tenstorrent components. For information about firmware updates that work with the KMD, see [Firmware and tt-flash](https://deepwiki.com/tenstorrent/tt-installer/5.2-firmware-and-tt-flash).

## KMD Architecture

The KMD is implemented as a Linux kernel module named `tenstorrent` that interfaces directly with Tenstorrent hardware devices through the PCIe bus. It provides the necessary abstractions for higher-level software components to interact with the hardware.

## Installation Process

The KMD is installed using the Dynamic Kernel Module Support (DKMS) framework, which automates building and installing kernel modules. The installation process includes cloning the KMD repository, registering it with DKMS, building it for the current kernel, and loading the module.

## Integration With Other Components

The KMD serves as a foundation for other Tenstorrent software components. It creates device nodes (typically `/dev/tenstorrent`) that allow user-space applications to communicate with the hardware.

| Component | Relationship with KMD |
| --- | --- |
| tt-smi | Uses KMD to monitor device status, performance, and health metrics |
| tt-flash | Relies on KMD to access and update device firmware |
| HugePages | Works alongside KMD to enable efficient memory management for the hardware |
| tt-metalium | Depends on KMD for hardware acceleration of AI workloads |

## Configuration Options

The installer provides options to customize the KMD installation process:

| Option | Environment Variable | Default | Description |
| --- | --- | --- | --- |
| Skip KMD Installation | TT_SKIP_INSTALL_KMD | 1 (install) | Set to 0 to skip KMD installation |
| KMD Version | TT_KMD_VERSION | latest | Specify a particular KMD version |
| Container Mode | TT_MODE_CONTAINER | 1 (disabled) | Set to 0 to skip KMD (container mode) |

When running in container mode, KMD installation is automatically skipped since the container would use the host system's KMD.

## Verification

After installation, you can verify that the KMD is properly installed and loaded using the following commands:

1.   Check module information:

```
modinfo tenstorrent
```
2.   Check if the module is loaded:

```
lsmod | grep tenstorrent
```
3.   Check device nodes:

```
ls -l /dev/tenstorrent*
```
4.   Use tt-smi to verify that the hardware is accessible:

```
tt-smi
```

Sources: [install.sh 535](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L535-L535)[install.sh 646-647](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L646-L647)

## Troubleshooting

If you encounter issues with the KMD installation:

1.   Check kernel logs for error messages:

```
dmesg | grep tenstorrent
```
2.   Check DKMS status:

```
dkms status
```
3.   Reinstall the KMD using the installer with forced reinstallation.

4.   Ensure your system has the required build tools for kernel modules (typically provided by packages like `build-essential` and `linux-headers`).

Sources: [install.sh 538-543](https://github.com/tenstorrent/tt-installer/blob/19fdd6a2/install.sh#L538-L543)

## Summary

The Kernel-Mode Driver (KMD) is an essential component that bridges Tenstorrent hardware with the operating system. It's installed using DKMS to ensure compatibility with the current kernel and proper integration with the Linux module system. The KMD enables all other Tenstorrent tools to communicate with the hardware and is a prerequisite for most Tenstorrent software stack functionality.
