# Whisper Integration

This document covers the client-side integration with the Whisper RISC-V Instruction Set Simulator (ISS), which serves as the golden reference model for architectural verification. The Whisper integration provides a TCP socket-based communication interface that allows the cosim-arch-checker framework to step the reference model in lockstep with the DUT and query architectural state for comparison.

For configuration of Whisper ISS parameters and initialization, see [Whisper Configuration](https://deepwiki.com/tenstorrent/cosim-arch-checker/4.1-whisper-configuration). For details on how the Bridge orchestrates DUT-Whisper synchronization, see [Bridge System](https://deepwiki.com/tenstorrent/cosim-arch-checker/3.1-bridge-system).

## Architecture Overview

The Whisper integration consists of a client library that communicates with a separate Whisper ISS process via TCP sockets. The client provides a C API for stepping the simulator, querying state, and managing the connection lifecycle.

## Core Operations

The Whisper client provides several key operations for instruction-level simulation and state management.

### Instruction Stepping

The primary operation is `whisperStep()`, which executes a single instruction in the Whisper ISS and returns execution results along with the count of architectural state changes.

### State Access Operations

| Operation | Purpose | Request Fields | Response Fields |
| --- | --- | --- | --- |
| `whisperPeek()` | Read register/CSR | `hart`, `resource`, `addr` | `value`, `valid` |
| `whisperPoke()` | Write register/CSR | `hart`, `resource`, `addr`, `value` | `valid` |
| `whisperChange()` | Get state change | `hart` | `resource`, `addr`, `value`, `valid` |
| `whisperReset()` | Reset hart | `hart` | `valid` |

## Memory Coherence Support

The client includes specialized functions for memory coherence model (MCM) operations, supporting advanced memory consistency verification.

### MCM Operations

| Function | Purpose | Usage |
| --- | --- | --- |
| `whisperMcmRead()` | Memory read operation | Load instruction verification |
| `whisperMcmWrite()` | Memory write operation | Store instruction verification |
| `whisperMcmInsert()` | Insert memory operation | Memory ordering constraints |

These operations support the verification of memory consistency models by tracking memory operations with precise timing and ordering information.

## Message Transport Layer

The socket communication layer handles reliable message transmission with proper error handling and connection management.

### Error Handling

The transport layer implements robust error handling for network operations:

*   **Connection errors**: Detection and reporting of socket connection failures
*   **Transmission errors**: Retry logic for interrupted system calls (`EINTR`)
*   **Protocol errors**: Validation of message integrity and proper deserialization
*   **Termination handling**: Graceful shutdown via `whisperQuit()` command

## SystemVerilog DPI Integration

The Whisper client integrates with SystemVerilog through the DPI (Direct Programming Interface), enabling seamless integration with hardware simulation environments.

### DPI Interface Points

The client uses `svdpi.h` for SystemVerilog integration, particularly for:

*   **Array handling**: `svOpenArrayHandle` for memory write operations
*   **Data type compatibility**: Proper mapping between SystemVerilog and C data types
*   **Function export**: All client functions are declared with `extern "C"` linkage

This integration allows the Whisper client to be called directly from SystemVerilog testbenches or through the DPI layer of the cosim-arch-checker framework.
