The Planet Feeds Aggregator is an automated pipeline responsible for polling external content sources—including YouTube, arXiv, Reddit, and community blogs—to populate the unified stream of Tenstorrent ecosystem activity. It complements the GitHub-specific metadata and release summarization pipelines by capturing multimedia and research-oriented content.

## Overview and Purpose

The aggregator is driven by `scripts/fetch_planet_feeds.py`[[scripts/fetch_planet_feeds.py:5-15]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator). Its primary role is to monitor trusted external feeds and append new items to `src/_data/planet_feeds.json`[[scripts/fetch_planet_feeds.py:30]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator). This data is then consumed by the Eleventy frontend to render the `/planet/` sub-site [[src/planet/index.njk:9-12]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).

### Key Features

*   **Multi-Source Polling**: Aggregates from YouTube (Official and affiliated channels), arXiv (research papers), Reddit (r/tenstorrent), and specific community Atom/RSS feeds [[scripts/fetch_planet_feeds.py:32-45]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
*   **Auto-Approval Logic**: Items from trusted sources (like official YouTube channels) are marked as `approved: true` automatically to appear on the feed without manual intervention [[scripts/fetch_planet_feeds.py:13-15]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
*   **Content Sanitization**: Includes robust HTML stripping and Unicode entity decoding to ensure third-party feed noise does not break the site layout [[scripts/fetch_planet_feeds.py:60-76]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
*   **ArXiv Deduplication**: Cross-references arXiv IDs against existing curated entries in the `entries/` directory to prevent duplicate listings of papers already featured in the main directory [[scripts/fetch_planet_feeds.py:104-117]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).

## Implementation Details

### Polling Mechanisms

The script uses standard Python `urllib.request` to perform HTTP GET requests with a custom `User-Agent`[[scripts/fetch_planet_feeds.py:47-57]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator). It parses XML/Atom responses using `xml.etree.ElementTree`[[scripts/fetch_planet_feeds.py:25]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).

| Source | Identifier / URL | Type |
| --- | --- | --- |
| **YouTube** | `UC7041p6DlAh0r4_Fnlk10pQ` (Official) | `video` |
| **arXiv** | `all:tenstorrent` (Search Query) | `paper` |
| **Reddit** | `r/tenstorrent` | `article` |
| **Blogs** | `clehaxze.tw`, `anuraagw.me`, etc. | `article` |

### Data Schema

Every item added to `planet_feeds.json` follows a strict schema to ensure compatibility with the Eleventy `planetItems` filter.

`{  "type": "video",  "source": "youtube",  "approved": true,  "title": "Tenstorrent Grayskull Architecture Overview",  "url": "https://www.youtube.com/watch?v=...",  "description": "A deep dive into the Tensix core...",  "date": "2026-06-13",  "dateISO": "2026-06-13T17:31:51Z",  "label": "Tenstorrent — YouTube",  "projectName": "Tenstorrent",  "projectId": null,  "affiliation": "official"}`
*   **`type`**: Determines the badge displayed (e.g., `paper`, `video`, `article`) [[src/planet/index.njk:139-148]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
*   **`approved`**: Boolean flag. Only `true` items are rendered on the live site [[src/_data/planet_feeds.json:5]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
*   **`affiliation`**: Mapped from the source configuration (e.g., `official`, `community`) [[scripts/fetch_planet_feeds.py:33-44]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).

## Automation and Approval Workflow

### GitHub Actions Schedule

The aggregator is triggered by `.github/workflows/planet-feeds.yml` on a cron schedule (twice daily) [[.github/workflows/planet-feeds.yml:3-4]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).

1.   **Fetch**: Executes `scripts/fetch_planet_feeds.py`[[.github/workflows/planet-feeds.yml:17]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
2.   **Commit**: If changes are detected in `src/_data/planet_feeds.json`, the action uses `peter-evans/create-pull-request` to open a PR [[.github/workflows/planet-feeds.yml:19-25]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
3.   **Review**: Maintainers review the PR. While most sources are auto-approved in the current implementation [[scripts/fetch_planet_feeds.py:13-15]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator), the workflow is designed to support a `approved: false` state for manual vetting if new, less-trusted feeds are added [[.github/workflows/planet-feeds.yml:29]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).

## Testing and Validation

The logic for feed item generation and sorting is validated via the JavaScript test suite to ensure the frontend correctly handles the aggregated data.

*   **Filter Validation**: `tests/test_eleventy_filters.js` contains unit tests for `articleFeedItems` and `jsonFeedItems`, ensuring that different link types (video, paper, article) are correctly categorized and sorted by date [[tests/test_eleventy_filters.js:71-100]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
*   **Deduplication Testing**: Verifies that duplicate URLs across different entries or feeds are collapsed into a single item [[tests/test_eleventy_filters.js:102-109]](https://deepwiki.com/tenstorrent/tt-awesome/3.3-planet-feeds-aggregator).
