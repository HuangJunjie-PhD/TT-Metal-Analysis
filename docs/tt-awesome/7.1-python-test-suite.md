The `tt-awesome` project utilizes a `pytest`-based test suite to ensure the integrity of its data-driven architecture. These tests validate the JSON schema of project entries, verify the community-first sorting logic of the README generator, and ensure the reliability of the automated metadata and release summarization pipelines.

## Test Suite Overview

The Python tests are located in the `tests/` directory and target the core scripts responsible for data processing and content generation.

| Test File | Target Script | Primary Focus |
| --- | --- | --- |
| `test_validate.py` | `scripts/validate.py` | Schema enforcement, URL protocols, and ID matching. |
| `test_generate_readme.py` | `scripts/generate_readme.py` | Community-first sorting and Markdown rendering. |
| `test_summarize_releases.py` | `scripts/summarize_releases.py` | LLM integration, release filtering, and sparse note detection. |
| `test_fetch_github_meta.py` | `scripts/fetch_github_meta.py` | GitHub API interaction, fork detection, and release fetching. |

## Schema Validation (`test_validate.py`)

This suite tests the `validate_entry` function, which acts as the primary gatekeeper for the `entries/` data layer. It uses a `valid()` factory function to generate a base valid entry and then applies overrides to test edge cases [tests/test_validate.py 12-23](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_validate.py#L12-L23)

### Key Validations

*   **ID Consistency**: Verifies that the `id` field inside the JSON matches the filename stem [tests/test_validate.py 34-37](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_validate.py#L34-L37)
*   **Protocol Enforcement**: Ensures that all URLs use `https://`[tests/test_validate.py 76-79](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_validate.py#L76-L79)
*   **Enum Constraints**: Validates that `affiliation`, `categories`, and `hardware` fields contain only allowed values defined in the schema [tests/test_validate.py 51-64](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_validate.py#L51-L64)[tests/test_validate.py 81-89](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_validate.py#L81-L89)
*   **Structure**: Checks for non-empty lists in `categories` and `links`[tests/test_validate.py 56-70](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_validate.py#L56-L70)

## README Generation (`test_generate_readme.py`)

These tests verify the implementation of the community-first philosophy and the correct rendering of the `README.md` file.

### Community-First Sorting Logic

The `sort_entries` function is tested to ensure that the primary sort key is `affiliation` (Community < Affiliated < Official), followed by `featured` status, and finally `stars` count [scripts/generate_readme.py 80-91](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/generate_readme.py#L80-L91)

*   **Tier Ranking**: `test_community_always_before_official` ensures community projects appear at the top [tests/test_generate_readme.py 22-25](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_generate_readme.py#L22-L25)
*   **Secondary Ranking**: `test_featured_first_within_tier` checks that featured flags override star counts within the same tier [tests/test_generate_readme.py 27-31](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_generate_readme.py#L27-L31)

### Rendering Accuracy

*   **Link Icons**: `test_render_entry_includes_all_links` verifies that link types (e.g., `talk`) are rendered with the correct emoji (e.g., `🎤`) [tests/test_generate_readme.py 53-61](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_generate_readme.py#L53-L61)
*   **Badges**: Ensures that the correct Shields.io badge is applied based on affiliation [tests/test_generate_readme.py 46-51](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_generate_readme.py#L46-L51)

## Release Summarization (`test_summarize_releases.py`)

This suite heavily utilizes `unittest.mock` to simulate GitHub API responses and LLM (Anthropic) completions without making network calls.

### Sparse and Pre-release Filtering

The pipeline is designed to ignore "noise" in the release feed:

*   **Sparse Notes**: `test_is_sparse_true_for_short_body` confirms that release notes shorter than the `SPARSE_LIMIT` (150 characters) are skipped [scripts/summarize_releases.py 33](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L33-L33)[tests/test_summarize_releases.py 96-108](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_summarize_releases.py#L96-L108)
*   **Pre-releases**: `test_main_skips_prerelease_tags` ensures that tags containing `dev`, `rc`, or `qa` are filtered out [tests/test_summarize_releases.py 110-146](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_summarize_releases.py#L110-L146)

### Mocking Strategy

The tests use a helper `_mock_urlopen` to simulate the `urllib.request.urlopen` context manager, allowing for the inspection of JSON payloads sent to the Anthropic API [tests/test_summarize_releases.py 68-74](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_summarize_releases.py#L68-L74)

## Metadata Fetching (`test_fetch_github_meta.py`)

This suite validates the interaction with the GitHub REST API.

*   **Fork Detection**: `test_fetch_repo_fork_captures_parent` verifies that if a project is a fork, the script correctly identifies and stores the upstream parent's name and URL [tests/test_fetch_github_meta.py 40-56](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_fetch_github_meta.py#L40-L56)
*   **Release Sanitization**: `test_fetch_releases_name_falls_back_to_tag` ensures that if a GitHub release lacks a title, the script falls back to using the `tag_name`[tests/test_fetch_github_meta.py 110-120](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_fetch_github_meta.py#L110-L120)
*   **Draft Exclusion**: `test_fetch_releases_returns_non_draft_releases` confirms that draft releases are never included in the metadata [tests/test_fetch_github_meta.py 74-102](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_fetch_github_meta.py#L74-L102)
