tt-torch requires Tenstorrent hardware to execute compiled models:

*   **Wormhole B0** (`wormhole_b0`) - Single chip

## Docker Installation

### Using Pre-built Docker Images

Docker images are automatically built and published for each commit. This is the fastest installation method.

`# Pull the latest CI imagedocker pull ghcr.io/tenstorrent/tt-torch/tt-torch-ci-ubuntu-22-04:latest # Run container with hardware accessdocker run -it \  --device /dev/tenstorrent/0 \  --shm-size=2gb \  -v /dev/hugepages:/dev/hugepages \  -v /dev/hugepages-1G:/dev/hugepages-1G \  ghcr.io/tenstorrent/tt-torch/tt-torch-ci-ubuntu-22-04:latest`
**Device mounting options:**

*   `--device /dev/tenstorrent/0` - Single Tenstorrent device
*   `--shm-size=2gb` - Shared memory for device communication
*   Volume mounts for hugepages required for performance

## External Dependencies

### Build-Time Dependencies Managed by CMake

The build system fetches and builds several large external projects:

| Dependency | Repository | Version Management | Purpose |
| --- | --- | --- | --- |
| **tt-xla** | github.com/tenstorrent/tt-xla | `TT_XLA_VERSION` in CMakeLists.txt | XLA integration, PJRT plugin |
| **tt-mlir** | github.com/tenstorrent/tt-mlir | Nested in tt-xla/third_party | MLIR compiler stack, TTIR/TTNN dialects |
| **torch-mlir** | github.com/llvm/torch-mlir | `TORCH_MLIR_VERSION` in CMakeLists.txt | PyTorch → MLIR lowering |
| **tt-forge-models** | github.com/tenstorrent/tt-forge | Optional (`--include-models`) | Model test suite |
| **tt-metal** | Nested in tt-mlir | Transitive dependency | Low-level hardware runtime |

**Version Pinning Example** from CMakeLists.txt (conceptual):

`set(TT_XLA_VERSION "commit-hash-here")set(TORCH_MLIR_VERSION "commit-hash-here")`

## Environment Configuration

### Environment Variables

The `env/activate` script sets critical environment variables:

| Variable | Purpose | Default/Example Value |
| --- | --- | --- |
| `TTMLIR_TOOLCHAIN_DIR` | LLVM/MLIR toolchain location | `/opt/ttmlir-toolchain/` |
| `TT_TORCH_HOME` | Repository root | `$(pwd)` |
| `TT_METAL_HOME` | tt-metal runtime path | `third_party/.../tt-metal/src/tt-metal` |
| `LD_LIBRARY_PATH` | Shared library search path | Includes torch, toolchain, install libs |
| `PYTHONPATH` | Python module search path | Includes install dir, tt-metal, tools |
| `PATH` | Executable search path | Includes ttmlir-opt, toolchain bins |
| `ARCH_NAME` | Target architecture | `wormhole_b0` (default) |
| `TOKENIZERS_PARALLELISM` | Disable for fork safety | `false` |

**Library Path Configuration:**

`export LD_LIBRARY_PATH=\  $TT_TORCH_HOME/env/venv/lib/python3.11/site-packages/torch/lib:\  $TTMLIR_TOOLCHAIN_DIR/lib:\  $TT_TORCH_HOME/install/lib/:\  $LD_LIBRARY_PATH`
**Sources:**[env/activate 22-52](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/env/activate#L22-L52)

### Runtime Configuration via tt_torch/**init**.py

The package initialization sets up critical runtime behavior:

**Key Runtime Initialization:**

*   **Multiprocessing:** Forces `forkserver` mode to prevent device descriptor issues in forked processes
*   **TT_METAL_HOME:** Automatically located from installed wheel or source tree
*   **Backend Registration:**`tt-legacy` and `tt-experimental` backends registered with torch.compile
*   **PJRT Plugin:** XLA plugin located and registered for tt-experimental backend

**Sources:**[tt_torch/__init__.py 1-75](https://github.com/tenstorrent/tt-torch/blob/f5e3955f/tt_torch/__init__.py#L1-L75)

## Troubleshooting

### Common Installation Issues

| Issue | Symptoms | Solution |
| --- | --- | --- |
| Missing `TTMLIR_TOOLCHAIN_DIR` | Build fails with missing headers | Set `export TTMLIR_TOOLCHAIN_DIR=/opt/ttmlir-toolchain/` before building |
| Python version mismatch | Import errors, incompatible wheels | Ensure Python 3.11 is used: `python3.11 -m venv env/venv` |
| Missing device | `/dev/tenstorrent/` not found | Verify hardware installed and drivers loaded |
| Library not found at runtime | `ImportError: cannot open shared object file` | Re-run `source env/activate` to set `LD_LIBRARY_PATH` |
| Setuptools version conflict | Wheel install fails | Pin `setuptools==59.6.0` as in requirements.txt |
| ONNX IR version error | `Unsupported model IR version: 11` | Use `onnx==1.17.0`, not 1.18.0 |

### Build Performance Tips

*   **Use ccache:** Speeds up rebuilds significantly `export PATH=/usr/lib/ccache:$PATH`
*   **Parallel builds:** Ninja automatically uses multiple cores
*   **Docker layer caching:** Reuse base image to avoid rebuilding system dependencies
