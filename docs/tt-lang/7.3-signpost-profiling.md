This page describes the signpost profiling system in tt-lang, which provides fine-grained performance instrumentation for kernel code. Signpost profiling enables cycle-accurate measurement of kernel operations by inserting profiling markers that interface with the device's hardware profiler.

For automated line-level profiling with pre-configured signposts, see **Auto-Profiling System (7.1)**. For viewing profiling results in a timeline UI, see **Performance Trace Visualization (7.2)**.

* * *

## Overview

The signpost profiling system allows developers to insert custom profiling regions into their kernels using the `ttl.signpost()` context manager. These profiling markers are compiled into Tracy profiler calls (`DeviceZoneScopedN`) that interface with Tenstorrent's device profiler to capture cycle counts.

The system consists of three main components:

1.   **Python DSL Support**: `with ttl.signpost("name"):` syntax for marking regions [python/ttl/_src/signpost_profile.py 9-10](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/signpost_profile.py#L9-L10)
2.   **MLIR Representation**: `ttl.signpost` operations with begin/end pairs [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 12-17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L12-L17)
3.   **Code Generation**: Lowering to `DeviceZoneScopedN()` C++ calls via the `LowerSignpostToEmitC` pass [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 125-126](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L125-L126)

## Signpost Types and Usage

### User-Defined Signposts

Users can insert custom profiling regions using the `with` statement within a kernel thread. These are typically used to wrap complex math or broadcast blocks to measure their specific contribution to kernel latency. These user-defined zones are internally prefixed with `ttl_` to distinguish them from low-level hardware zones [python/ttl/_src/signpost_profile.py 21](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/signpost_profile.py#L21-L21)

**Signpost Naming Rules**:

*   Must be a string literal.
*   Does not produce a value (`as` clause not supported).
*   Nested signposts are supported and correctly handled by the compiler [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 46-52](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L46-L52)

### Auto-Generated Signposts

When `TTLANG_AUTO_PROFILE=1` is enabled, the compiler automatically generates signposts for every operation to provide a line-level cycle breakdown [python/ttl/_src/auto_profile.py 8-11](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/auto_profile.py#L8-L11)

**Auto-profiling Signpost Formats**[python/ttl/_src/auto_profile.py 78-100](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/auto_profile.py#L78-L100):

*   **Line boundaries**: `{kernel}_L{lineno}` (e.g., `compute_L52`).
*   **CB operations**: `{kernel}_L{lineno}_{op}` (e.g., `compute_L41_cb_wait`) [test/python/auto_profile_signposts.py 68](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/auto_profile_signposts.py#L68-L68)
*   **Implicit operations**: `{kernel}_L{lineno}_implicit_{op}` (e.g., `compute_L41_implicit_cb_pop`) [test/python/auto_profile_signposts.py 80](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/auto_profile_signposts.py#L80-L80)

## MLIR Signpost Operations

### SignpostOp Structure

The `SignpostOp` represents a profiling marker in the MLIR representation. It is defined in the TTL dialect and later lowered to `EmitC` verbatim calls [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 14-17](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L14-L17)

**Attributes**:

*   `name: StringAttr` - The signpost identifier (e.g., "compute_L42_cb_wait").
*   `is_end: BoolAttr` - A boolean flag where `false` indicates the start of a scope and `true` indicates the end [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 113-135](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L113-L135)

**Signpost Pairs**: Each profiling region consists of two operations:

1.   **Begin marker**: `ttl.signpost "name" {is_end = false}`
2.   **End marker**: `ttl.signpost "name" {is_end = true}`

Sources: [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 113-135](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L113-L135)[test/python/auto_profile_signposts.py 67-81](https://github.com/tenstorrent/tt-lang/blob/d76e6233/test/python/auto_profile_signposts.py#L67-L81)

## LowerSignpostToEmitC Pass

### Pass Architecture

The `TTLLowerSignpostToEmitCPass` manages the conversion of `SignpostOp` pairs into C++ scoping braces and Tracy macros.

Title: "LowerSignpostToEmitC Logic"

### Optimization: Skipping Empty or Unsafe Regions

The pass performs several checks to avoid emitting profiling overhead for regions that contain no meaningful work or could cause C++ compilation errors [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 29-59](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L29-L59):

1.   **Skipped Operations**: Certain `ttkernel` ops are considered too "cheap" to profile and do not trigger a profiling scope if they are the only ops present [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 29-33](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L29-L33): 
    *   `ttkernel.my_x`, `ttkernel.my_y`
    *   `ttkernel.my_logical_x_`, `ttkernel.my_logical_y_`

2.   **Nested Signposts**: Operations already covered by a nested `SignpostOp` scope are not counted toward the "interesting ops" of the outer scope [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 46-52](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L46-L52)
3.   **Escaping Values**: If a signpost scope defines values (MLIR results) that are used outside that scope, the pass skips emitting the `{}` braces and the profiler macro to avoid C++ scoping errors [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 63-76](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L63-L76) This triggers a warning: "value defined in scope is used after scope exits. PLEASE FILE A BUG." [lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp 119-122](https://github.com/tenstorrent/tt-lang/blob/d76e6233/lib/Dialect/TTL/Transforms/LowerSignpostToEmitC.cpp#L119-L122)

## Profile Data Collection

### Device Profiler Integration

The signpost profiling system requires the Tenstorrent device profiler to be enabled during execution.

**Required Environment Variables**:

*   `TT_METAL_DEVICE_PROFILER=1`: Enables hardware cycle counters.
*   `TTLANG_AUTO_PROFILE=1`: Enables automatic signpost generation and report printing [python/ttl/_src/auto_profile.py 52-54](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/auto_profile.py#L52-L54)
*   `TTLANG_SIGNPOST_PROFILE=1`: Enables the custom user signpost summary report [python/ttl/_src/signpost_profile.py 24-25](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/signpost_profile.py#L24-L25)

**Data Flow**:

1.   Kernel executes on-device with `DeviceZoneScopedN` markers.
2.   `tt-metal` device profiler captures `ZONE_START` and `ZONE_END` timestamps [python/ttl/_src/auto_profile.py 166-169](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/auto_profile.py#L166-L169)
3.   The `parse_device_profile_csv` (for auto-profiling) or `parse_signpost_zones` (for user signposts) reads `profile_log_device.csv`[python/ttl/_src/auto_profile.py 126-141](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/auto_profile.py#L126-L141)[python/ttl/_src/signpost_profile.py 28-35](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/signpost_profile.py#L28-L35)
4.   A summary report is generated showing cycle counts (Total, Avg, Min, Max) organized by thread [python/ttl/_src/signpost_profile.py 73-119](https://github.com/tenstorrent/tt-lang/blob/d76e6233/python/ttl/_src/signpost_profile.py#L73-L119)

## Environment Variables Reference

| Variable | Values | Description |
| --- | --- | --- |
| `TTLANG_AUTO_PROFILE` | `0`, `1` | Enable per-line cycle count reporting and automated signpost insertion. |
| `TTLANG_SIGNPOST_PROFILE` | `0`, `1` | Enable summary report for user-defined `ttl.signpost` zones. |
| `TT_METAL_DEVICE_PROFILER` | `0`, `1` | Enable the low-level hardware profiler (required for cycle counts). |
| `TTLANG_COMPILE_ONLY` | `0`, `1` | Skip execution; useful for inspecting generated C++ with signposts. |
