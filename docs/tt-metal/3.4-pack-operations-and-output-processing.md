**Purpose**: This page documents the Pack operations in the Low-Level Kernel (LLK) API, which handle writing computational results from the DEST register to L1 circular buffers (output tensors). Pack is the final stage in the three-stage compute pipeline on Tensix cores.

**Scope**: Covers pack initialization, standard tiled packing, untilize operations, data format configuration, and architectural differences between Wormhole and Blackhole. For information about the preceding stages, see [Unpack Operations and Input Processing](https://deepwiki.com/tenstorrent/tt-metal/3.3-unpack-operations-and-input-processing) and [Math and Compute Operations](https://deepwiki.com/tenstorrent/tt-metal/3.5-math-and-compute-operations).

## Compute Pipeline Overview

Pack operations are the third and final stage in the Tensix compute pipeline. Data flows from input circular buffers through unpacking, computation in the MATH unit, and finally packing results from the DEST register back to the output circular buffers.

**Sources:**[tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_A_api.h 48-68](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_A_api.h#L48-L68)[tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_A_api.h 52-72](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_A_api.h#L52-L72)

## Pack Operation Types

Pack operations are categorized by how they transform the data layout from the DEST register to memory. While standard packing maintains the tile layout, untilization transforms it to row-major.

| Operation Type | Purpose | Key Functions | Output Layout |
| --- | --- | --- | --- |
| **Standard Pack** | Write tiled data from DEST to output CB | `llk_pack_init`, `llk_pack_tile` | Tiled (e.g., 32x32) |
| **Pack Untilize** | Convert tiled data in DEST to row-major format | `llk_unpack_tilize_init`, `llk_unpack_tilize` | Row-major |
| **Tilize** | Convert row-major input to tiled output | `llk_unpack_tilize_init`, `llk_unpack_tilize_block` | Tiled |

**Sources:**[tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h 21-28](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h#L21-L28)[tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_tilize_api.h 21-29](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_tilize_api.h#L21-L29)[tests/tt_metal/tt_metal/llk/test_untilize_tilize.cpp 61-66](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tests/tt_metal/tt_metal/llk/test_untilize_tilize.cpp#L61-L66)

## Pack and Tilize Operations

The `tilize` operation is critical for converting linear row-major formats into the hardware-native tile format required for Tensix compute.

### Initialization and Configuration

Initialization configures the unpacker/packer hardware by deriving face geometry and data formats from the Circular Buffer (CB) metadata.

*   **`llk_unpack_tilize_init`**: Programs the unpacker for tilize operations. It extracts `face_r_dim`, `narrow_tile` flag, and `num_faces` from the operand's CB metadata. [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h 21-28](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h#L21-L28)
*   **`llk_unpack_tilizeA_B_init`**: A specialized initialization for operations involving two source operands where Operand A is tilized and Operand B is unpacked normally. It validates that both unpackers are configured consistently using `are_unpackers_AB_configured_correctly`. [tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_tilize_api.h 134-161](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_tilize_api.h#L134-L161)

### Block Processing

Tilize typically works on contiguous blocks of tiles.

*   **`llk_unpack_tilize_block`**: Repeatedly calls `llk_unpack_tilize` to process a row or block of column tiles. It handles tile index calculations to move across rows of files correctly. [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h 78-85](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h#L78-L85)

**Sources:**[tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h 50-69](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h#L50-L69)[tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_tilize_api.h 81-90](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_tilize_api.h#L81-L90)

## Data Format and Precision Handling

The packer and output processing units handle various data formats and precision modes.

*   **FP32 Accumulation**: When `fp32_dest_acc_en` is enabled, the DEST register maintains higher precision. For 8-bit formats (Int8, UInt8, Fp8_e4m3), the hardware keeps integers as-is in DEST. [tests/tt_metal/tt_metal/llk/test_untilize_tilize.cpp 110-116](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tests/tt_metal/tt_metal/llk/test_untilize_tilize.cpp#L110-L116)
*   **Output Gasket**: For 8-bit floats, the packer gasket converts the internal Float32 DEST representation back to 8-bit at the output stage. [tests/tt_metal/tt_metal/llk/test_untilize_tilize.cpp 110-116](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tests/tt_metal/tt_metal/llk/test_untilize_tilize.cpp#L110-L116)
*   **Fidelity Masks**: During compute, fidelity masks (e.g., `srca_fid_mask`, `srcb_fid_mask`) are used to simulate lower precision (HiFi2, LoFi) by masking bits before the packer writes to L1. [tests/tt_metal/tt_metal/llk/test_reduce.cpp 167-189](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tests/tt_metal/tt_metal/llk/test_reduce.cpp#L167-L189)

## Architectural Variations

### Blackhole

Blackhole introduces specific checks and initialization flows for 32-bit inputs.

*   **32-bit Input Handling**: In `llk_unpack_A_init`, if `unpack_to_dest` is used with 32-bit formats, the debug feature is explicitly disabled to ensure compatibility. [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_A_api.h 28-31](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_A_api.h#L28-L31)
*   **Narrow Tile Support**: Blackhole supports `narrow_tile` configurations derived from operand metadata. [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h 24-27](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_unpack_tilize_api.h#L24-L27)

### Wormhole B0

Wormhole supports complex broadcast and reduction operations during the pack/unpack phase.

*   **Broadcast Modes**: Supports `BroadcastType::ROW`, `COL`, and `SCALAR`. In row broadcast, the unpacker can be indexed by `bcast_row_idx`. [tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_AB_api.h 38-43](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_unpack_AB_api.h#L38-L43)
*   **Matmul Unpack**: Specialized `llk_unpack_AB_matmul` handles partial face dimensions and kt-dimensions for matrix multiplication.
