## Purpose and Scope

This page documents the comprehensive test coverage for RISC-V cache management extensions: **ZICBOM** (Cache Block Management Operations), **ZICBOZ** (Cache Block Zero), and **ZICBOP** (Cache Block Prefetch). The test plan contains 47+ test scenarios verifying cache operation instructions, CSR-based control mechanisms, privilege mode interactions, virtual address aliasing, multi-processor synchronization, and integration with atomic operations.

For information about the test step primitives used to construct these scenarios, see [Test Steps Reference](https://deepwiki.com/tenstorrent/riscv-coretp/5-test-steps-reference). For details on the test environment configuration system, see [Test Environment Configuration](https://deepwiki.com/tenstorrent/riscv-coretp/3.2-test-environment-configuration).

## Extension Overview

The cache management extensions provide instructions for explicit cache control, prefetching, and cache block zeroing. Each extension serves a distinct purpose in the cache hierarchy management:

| Extension | Purpose | Instructions | CSR Control |
| --- | --- | --- | --- |
| **ZICBOM** | Cache Block Management | `cbo.clean`, `cbo.flush`, `cbo.inval` | `menvcfg.CBIE`, `menvcfg.CBCFE`, `senvcfg.CBIE`, `senvcfg.CBCFE`, `henvcfg.CBIE`, `henvcfg.CBCFE` |
| **ZICBOZ** | Cache Block Zero | `cbo.zero` | `menvcfg.CBZE`, `senvcfg.CBZE`, `henvcfg.CBZE` |
| **ZICBOP** | Cache Block Prefetch | `prefetch.i`, `prefetch.r`, `prefetch.w` | None (hint instructions) |

## Test Plan Registration

The test plan is registered using the `@zicbom_zicboz_zicbop_scenario` decorator, which is created by calling `new_test_plan()` with metadata describing the test plan coverage.

Each test scenario function is decorated with `@zicbom_zicboz_zicbop_scenario` and returns a `TestScenario` object constructed using `TestScenario.from_steps()`.

## CSR Control Mechanisms

Cache management operations are controlled by fields in the `menvcfg`, `senvcfg`, and `henvcfg` CSRs. These fields enable hierarchical privilege-based access control.

### CSR Control Hierarchy

### Control Bit Semantics

| CSR Field | Bit Position | Controlled Instructions | Value 0 Behavior |
| --- | --- | --- | --- |
| `CBIE` | `[5:4]` | `cbo.inval` | Illegal instruction exception |
| `CBCFE` | `[6]` | `cbo.clean`, `cbo.flush` | Illegal instruction exception |
| `CBZE` | `[7]` | `cbo.zero` | Illegal instruction exception |

**Sources:**[coretp/plans/zicbom_zicboz_zicbop/zicbom_zicboz_zicbop_scenarios.py 24-26](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/zicbom_zicboz_zicbop/zicbom_zicboz_zicbop_scenarios.py#L24-L26)[coretp/plans/zicbom_zicboz_zicbop/zicbom_zicboz_zicbop_scenarios.py 58-60](https://github.com/tenstorrent/riscv-coretp/blob/e4fb9eea/coretp/plans/zicbom_zicboz_zicbop/zicbom_zicboz_zicbop_scenarios.py#L58-L60)

## Prefetch Instructions (ZICBOP)

The ZICBOP extension provides prefetch hint instructions that do not cause exceptions and allow implementations to ignore them without side effects.

### Prefetch Instruction Semantics

| Instruction | Purpose | Immediate Field | Trap Behavior |
| --- | --- | --- | --- |
| `prefetch.i` | Instruction prefetch | `imm[11:5]` used, `imm[4:0]` must be 0 | Never traps |
| `prefetch.r` | Data read prefetch | `imm[11:0]` hint only | Never traps |
| `prefetch.w` | Data write prefetch | `imm[11:0]` hint only | Never traps |

### SID_ZICBO_013: Prefetch No-Trap Verification

## Summary of Test Coverage

The cache management extension test plan provides comprehensive coverage across multiple dimensions:

| Dimension | Coverage |
| --- | --- |
| **Instructions** | All ZICBOM (`cbo.clean`, `cbo.flush`, `cbo.inval`), ZICBOZ (`cbo.zero`), ZICBOP (`prefetch.i/r/w`) instructions |
| **CSR Control** | All `menvcfg`, `senvcfg`, `henvcfg` control fields and hierarchical interactions |
| **Privilege Modes** | M-mode, S-mode, U-mode, VS-mode, VU-mode, HS-mode |
| **Virtualization** | Non-virtualized and virtualized guest modes |
| **Exceptions** | Illegal instruction, store page fault exceptions |
| **Multi-Processor** | Hart synchronization, cache coherence, LR/SC interaction |
| **Virtual Memory** | Virtual address aliasing, address alignment, paging modes |

The test scenarios leverage the coretp framework's composable test step architecture to construct precise, reusable verification tests that can be executed across different hardware configurations and privilege levels.
