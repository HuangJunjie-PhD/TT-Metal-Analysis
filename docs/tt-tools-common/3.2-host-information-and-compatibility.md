Relevant source files

## Purpose and Scope

This document describes the host information collection and compatibility checking subsystem within tt-tools-common. This subsystem provides functions for gathering system details (OS, distribution, memory, platform architecture), validating compatibility against Tenstorrent requirements, and querying software version information from remote services.

For driver version management and validation, see [Driver Version Management](https://deepwiki.com/tenstorrent/tt-tools-common/3.1-driver-version-management). For chip detection and hardware communication, see [Chip Detection and Communication](https://deepwiki.com/tenstorrent/tt-tools-common/3.3-chip-detection-and-communication). For UI widgets that display this information, see [Widget Library Reference](https://deepwiki.com/tenstorrent/tt-tools-common/4.1-widget-library-reference).

## Host Information Collection

### `get_host_info()` Function

The `get_host_info()` function collects comprehensive system information by querying multiple Python standard library modules. It returns a dictionary containing eight key system attributes.

**Data Sources and Collection:**

| Information | Source Module | Collection Method |
| --- | --- | --- |
| OS | `platform.uname()` | `uname.system` |
| Distro | `distro` | `distro.name(pretty=True)` |
| Kernel | `platform.uname()` | `uname.release` |
| Hostname | `platform.uname()` | `uname.node` |
| Platform | `platform.uname()` | `uname.machine` (e.g., "x86_64", "aarch64") |
| Python | `platform` | `platform.python_version()` |
| Memory | `psutil` | `psutil.virtual_memory().total` (formatted via `get_size()`) |
| Driver | Custom | `get_driver_version()` reads `/sys/module/tenstorrent/version` |

**Return Format:**

`{    "OS": str,           # e.g., "Linux"    "Distro": str,       # e.g., "Ubuntu 22.04.3 LTS"    "Kernel": str,       # e.g., "5.15.0-89-generic"    "Hostname": str,     # e.g., "tenstorrent-dev"    "Platform": str,     # e.g., "x86_64" or "aarch64"    "Python": str,       # e.g., "3.10.12"    "Memory": str,       # e.g., "62.68 GB" (human-readable format)    "Driver": str        # e.g., "TT-KMD 1.27.0" (prefixed with "TT-KMD ")}`
**Memory Formatting:**

The `get_size()` helper function [tt_tools_common/utils_common/system_utils.py 20-32](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L20-L32) scales bytes to human-readable format using 1024-based units (KB, MB, GB, TB, PB). For example, `1253656678` bytes becomes `"1.17 GB"`.

**Sources:**[tt_tools_common/utils_common/system_utils.py 145-168](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L145-L168)[tt_tools_common/utils_common/system_utils.py 35-45](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L35-L45)[tt_tools_common/utils_common/system_utils.py 20-32](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L20-L32)

## Compatibility Checking

### `get_host_compatibility_info()` Function

This function extends `get_host_info()` by adding compatibility validation against Tenstorrent requirements. The return format uses a special convention to indicate compatibility status.

**Sources:**[tt_tools_common/utils_common/system_utils.py 170-211](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L170-L211)

### Return Value Convention

The function returns `Dict[str, Union[str, Tuple[str, str]]]` where:

*   **Compatible items** return a single string: the current value
*   **Incompatible items** return a tuple: `(current_value, recommended_value)`

This convention enables UI components to distinguish compatible from incompatible items without additional metadata.

### Compatibility Requirements

| Item | Requirement | Check Logic |
| --- | --- | --- |
| **OS** | Linux (x86_64) | Compares `uname.system` and `uname.machine`. Non-Linux or ARM platforms return tuple. |
| **Distro** | Ubuntu 22.04 or 24.04 | Checks `distro.id() == "ubuntu"` and version >= 22. Other distros return tuple. |
| **Kernel** | N/A | Always returns string (informational only). |
| **Hostname** | N/A | Always returns string (informational only). |
| **Python** | N/A | Always returns string (informational only). |
| **Memory** | 32GB+ | Checks `psutil.virtual_memory().total >= 32e9`. Less than 32GB returns tuple. |
| **Driver** | TT-KMD present | Returns tuple if driver version is `None`, otherwise returns string. |

**Implementation Details:**

The Ubuntu distro version check [tt_tools_common/utils_common/system_utils.py 188-195](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L188-L195) parses the first two version parts and converts to float:

`distro_version = float(".".join(distro.version_parts()[:2]))if distro_version >= 22:    checklist["Distro"] = host_info["Distro"]  # Compatibleelse:    checklist["Distro"] = (host_info["Distro"], "22.04 or 24.04")  # Incompatible`
**Sources:**[tt_tools_common/utils_common/system_utils.py 170-211](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L170-L211)

## Software Version Reporting

### Version Query System

The software version reporting system queries a remote service to retrieve recommended software versions for Tenstorrent boards. This system respects privacy flags configured in the reset configuration file.

**Sources:**[tt_tools_common/utils_common/system_utils.py 213-295](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L213-L295)

### Configuration Flags

The system reads two privacy-related flags from `~/.config/tenstorrent/reset_config.json`:

*   **`disable_serial_report`**: If true, queries without board serial numbers (empty string)
*   **`disable_sw_version_report`**: If true, returns default N/A values without querying

**Configuration File Format:**

`{  "disable_serial_report": false,  "disable_sw_version_report": false}`
**Flag Parsing:**[tt_tools_common/utils_common/system_utils.py 213-228](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L213-L228)

### Remote API Integration

**Endpoint:**`https://cereal.tenstorrent.com?SerialNumber=<board_id>`

**Query Process:**

1.   Construct URL with board serial number (or empty if `disable_serial_report` is true)
2.   Issue HTTP GET request via `requests` library
3.   Parse JSON response
4.   Transform keys to user-facing names (e.g., "Metallium" → "TT-Metalium")
5.   Add hardcoded Buda/Metallium versions

**Response Transformation:**

The function [tt_tools_common/utils_common/system_utils.py 242-251](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L242-L251) performs key renaming:

`version["TT-Metalium"] = version["Metallium"]del version["Metallium"]version["TT-Buda"] = version["Buda"]del version["Buda"]`
**Error Handling:**

| Exception Type | User-Facing Message |
| --- | --- |
| `HTTPError` | "We encountered an HTTP error." |
| `ConnectionError` | "There was an error connecting to the server. Please check your internet connection." |
| `Timeout` | "Timeout error. It seems the server is taking too long to respond." |
| `RequestException` | "Something unexpected happened." |
| `JSONDecodeError` | "No response from server" |

**Sources:**[tt_tools_common/utils_common/system_utils.py 230-267](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L230-L267)

### Default Return Format

When software version reporting is disabled or fails, the function returns:

`{    "Firmware Bundle": "N/A",    "tt-smi": "N/A",    "tt-flash": "N/A",    "tt-kmd": "N/A",    "TT-Buda": "N/A",    "TT-Metalium": "N/A",}`
**Sources:**[tt_tools_common/utils_common/system_utils.py 276-288](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L276-L288)

## TTHostCompatibilityMenu Widget

### Widget Overview

The `TTHostCompatibilityMenu` is a Textual widget that displays host information with visual compatibility indicators. It integrates with the compatibility checking system to provide real-time feedback on system requirements.

**Widget Features:**

*   Displays results from `get_host_compatibility_info()`
*   Color-codes compatible (green checkmark) vs incompatible (red X) items
*   Shows recommended values for incompatible items
*   Auto-updates on system changes
*   Integrates with Textual's reactive system

**Changelog Reference:**

Version 1.4.12 [CHANGELOG.md 28-31](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/CHANGELOG.md?plain=1#L28-L31) introduced `TTHostCompatibilityMenu` to replace separate "Host Info" and "Compatibility" boxes. Version 1.4.13 [CHANGELOG.md 23-26](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/CHANGELOG.md?plain=1#L23-L26) removed hugepages counting from the widget due to allocation complexity.

### Display Format

The widget renders compatibility information in a table-like format:

**Compatible Item Display:**

```
✓ OS: Linux (x86_64)
```

**Incompatible Item Display:**

```
✗ Memory: 16.00 GB (Recommended: 32GB+)
```

### Integration Example

Applications using this widget should:

1.   Import the widget class from `tt_tools_common.ui_common.widgets`
2.   Call `get_host_compatibility_info()` to retrieve data
3.   Pass data to widget constructor or update method
4.   Mount widget in Textual application layout

**Sources:**[CHANGELOG.md 28-31](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/CHANGELOG.md?plain=1#L28-L31)

## Usage Patterns

### Collecting Basic Host Information

`from tt_tools_common.utils_common.system_utils import get_host_info host_info = get_host_info()print(f"Running on {host_info['Distro']} with {host_info['Memory']} memory")print(f"Driver: {host_info['Driver']}")`
### Checking Compatibility

`from tt_tools_common.utils_common.system_utils import get_host_compatibility_info compat_info = get_host_compatibility_info() for key, value in compat_info.items():    if isinstance(value, tuple):        current, recommended = value        print(f"✗ {key}: {current} (Recommended: {recommended})")    else:        print(f"✓ {key}: {value}")`
### Querying Software Versions

`from tt_tools_common.utils_common.system_utils import get_sw_ver_info # Query with board IDsboard_ids = ["12345ABC", "67890DEF"]versions = get_sw_ver_info(show_sw_ver=True, board_ids=board_ids) for package, version in versions.items():    print(f"{package}: {version}")`

## Relationship to Other Subsystems

### Driver Version Management

Host information collection includes driver version detection via `get_driver_version()`, but driver version comparison and validation is handled by the Driver Version Management subsystem (see [Driver Version Management](https://deepwiki.com/tenstorrent/tt-tools-common/3.1-driver-version-management)). The `get_host_info()` function calls `get_driver_version()` but does not perform version validation.

### Reset Configuration

The software version reporting system reads flags from the reset configuration file generated by the reset logging subsystem (see [Reset Configuration and Logging](https://deepwiki.com/tenstorrent/tt-tools-common/2.6-reset-configuration-and-logging)). These flags control privacy-related behavior for board serial number reporting.

### UI Integration

The `TTHostCompatibilityMenu` widget (see [Widget Library Reference](https://deepwiki.com/tenstorrent/tt-tools-common/4.1-widget-library-reference)) consumes the data produced by `get_host_compatibility_info()` to provide visual compatibility feedback in Textual-based applications like tt-smi.

**Sources:**[tt_tools_common/utils_common/system_utils.py 1-296](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/utils_common/system_utils.py#L1-L296)
