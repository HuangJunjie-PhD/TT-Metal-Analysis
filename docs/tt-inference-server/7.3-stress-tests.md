The Stress Tests framework is a comprehensive model validation system designed to verify model readiness and performance across diverse parameter configurations. It provides targeted single-parameter testing, exhaustive multi-parameter matrix testing, and long-running endurance testing to ensure models meet performance specifications before deployment [stress_tests/README.md 1-16](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L1-L16)

## Parameter Generation Modes

The `StressTests` class in `stress_tests_core.py` generates test matrices based on the selected `run_mode` or detected custom parameters [stress_tests/stress_tests_core.py 102-125](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_core.py#L102-L125)

### 1. Single Mode

Tests a specific combination of parameters. If not explicitly provided via `input_size` or `output_size`, it defaults to `max_context_length=8192`, `max_concurrent=1`, and `num_prompts=1`[stress_tests/README.md 73-77](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L73-L77)

### 2. Multiple Mode

Generates a cross-product of all valid parameter combinations defined in the model's `DeviceModelSpec`[stress_tests/stress_tests_config.py 120-121](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_config.py#L120-L121) It includes:

*   **Context Sizes**: Max, ~50%, and ~10% of the model's limit [stress_tests/README.md 84](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L84-L84)
*   **Output Sizes**: Typically 128 and 2048 tokens [stress_tests/README.md 85](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L85-L85)
*   **Concurrency**: 1, 2, and 50% of `max_concurrency`[stress_tests/README.md 86](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L86-L86)

### 3. Custom Mode

Allows users to specify comma-separated lists for ISL, OSL, and concurrency via `workflow-args` (e.g., `custom-isl-values=1024,2048,4096`). The system generates a full cross-product of these values [stress_tests/stress_tests_core.py 10-27](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_core.py#L10-L27) It supports a `custom_num_prompts_strategy` such as `match_concurrency` or `fixed:N`[stress_tests/stress_tests_core.py 21-23](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_core.py#L21-L23)

### Context Limit Enforcement

To ensure hardware stability, the `enforce_context_limit` function ensures that `ISL + OSL <= max_context`. It supports different policies:

*   `preserve_isl`: Keeps the input length and shrinks OSL to fit [stress_tests/stress_tests_config.py 45-48](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_config.py#L45-L48)
*   `preserve_osl`: Keeps the output length and shrinks ISL to fit [stress_tests/stress_tests_config.py 49-52](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_config.py#L49-L52)
*   `neutral`: Scales both proportionally using fractional scaling and residual distribution to maintain the exact sum [stress_tests/stress_tests_config.py 53-84](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_config.py#L53-L84)

## Endurance Testing

Endurance mode is a specialized configuration designed for 24-hour stability validation [stress_tests/README.md 40-44](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L40-L44) When enabled via `endurance_mode=true` in `workflow_args`, the `StressTests` class:

1.   Forces `run_mode` to `single`[stress_tests/stress_tests_core.py 98](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_core.py#L98-L98)
2.   Overrides `max_context_length` to 8640 [stress_tests/stress_tests_core.py 99](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_core.py#L99-L99)
3.   Executes continuous high-concurrency load to detect memory leaks or hardware hangs over extended periods [stress_tests/README.md 43-44](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L43-L44)

## Reporting and Comparison

After execution, `stress_tests_summary_report.py` processes the resulting JSON files to generate a unified view of performance.

### Performance Target Comparison

The reporting tool extracts `performance_targets` from the `ModelSpec` (via `perf_reference`) and compares actual measured values against these targets [stress_tests/stress_tests_config.py 175-177](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_config.py#L175-L177) It calculates `tps_decode_throughput` and `tps_prefill_throughput` based on the measured `mean_tpot_ms` and `mean_ttft_ms`[stress_tests/stress_tests_summary_report.py 179-182](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_summary_report.py#L179-L182)

### Report Generation

*   **CSV Export**: Consolidates all test runs into a single spreadsheet for historical tracking [stress_tests/stress_tests_summary_report.py 37-41](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_summary_report.py#L37-L41)
*   **Markdown Tables**: Formats metrics (P50, P99, Mean) into human-readable tables [stress_tests/README.md 69-70](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/README.md?plain=1#L69-L70)
*   **Filename Parsing**: Automatically extracts metadata (model, device, ISL, OSL, concurrency) from result filenames using complex regex patterns for both text and image tasks [stress_tests/stress_tests_summary_report.py 45-110](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/stress_tests/stress_tests_summary_report.py#L45-L110)
