The ONNX Frontend is TVM's converter for ONNX (Open Neural Network Exchange) format models into Relay IR. It provides comprehensive support for 100+ ONNX operators across multiple opset versions through a pattern-matching based conversion system with sophisticated type inference capabilities.

For information about other frontend converters, see [Frontend Converters](https://deepwiki.com/tenstorrent/tt-tvm/3-frontend-converters). For details on the common infrastructure shared across all frontends, see [Common Frontend Infrastructure](https://deepwiki.com/tenstorrent/tt-tvm/3.1-common-frontend-infrastructure).

## Overview and Architecture

The ONNX frontend serves as the bridge between ONNX models and TVM's Relay IR. It handles model ingestion, operator mapping, type inference, and parameter extraction to produce a unified IRModule representation suitable for optimization and compilation.

The primary interface to the ONNX frontend is the `from_onnx()` function, which orchestrates the entire conversion process.

### Opset Version Dispatch Mechanism

The `get_converter()` class method implements intelligent version selection:

**Version Selection Algorithm:**

**Example:** For `opset=13` and a class with `_impl_v1`, `_impl_v7`, `_impl_v11`:

*   Sorted list: `[1, 7, 11, 13]`
*   Position of 13: index 3
*   Selected version: `11` (index 3-1=2)
*   Returns: `_impl_v11`

**Sources:**[python/tvm/relay/frontend/onnx.py 566-585](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/onnx.py#L566-L585)

## Advanced Features

### Span Tracking for Debugging

The ONNX frontend preserves source information through TVM's span system:

**Usage Pattern:**

*   Each node name is tracked through conversion
*   Parameters get span names like `"conv2d.weight"` or `"conv2d.bias"`
*   Enables debugging by tracing back to original ONNX operators

**Sources:**[python/tvm/relay/frontend/onnx.py 531-560](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/onnx.py#L531-L560)[python/tvm/relay/frontend/common.py 800-850](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/python/tvm/relay/frontend/common.py#L800-L850)
