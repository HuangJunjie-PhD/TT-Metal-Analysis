## Purpose and Scope

This document describes the Alpine Kernel Module Support (AKMS) configuration for the Tenstorrent kernel driver (`tt-kmd`) on Alpine Linux systems. AKMS provides automatic kernel module building and installation when the kernel is upgraded, functioning as the Alpine-specific alternative to DKMS.

For DKMS configuration on Debian/Ubuntu/Fedora/RHEL systems, see [DKMS Configuration](https://deepwiki.com/tenstorrent/tt-kmd/7.1-dkms-configuration). For manual building without a module management system, see [Manual Build with Make](https://deepwiki.com/tenstorrent/tt-kmd/7.3-manual-build-with-make). For runtime module parameters, see [Module Parameters](https://deepwiki.com/tenstorrent/tt-kmd/7.4-module-parameters).

## AKMS Overview

AKMS (Alpine Kernel Module Support) is the framework used by Alpine Linux to manage out-of-tree kernel modules. It is designed to work within Alpine's environment, which typically utilizes `musl` libc, BusyBox, and the `apk` package manager.

### Key Characteristics

| Feature | Description |
| --- | --- |
| **Package Manager** | Integrates with Alpine's `apk` package system [README.md 28](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/README.md?plain=1#L28-L28) |
| **Privilege Escalation** | Uses `doas` instead of `sudo`[Makefile 50](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L50-L50) |
| **Module Location** | Installs to `/lib/modules/<kernel-version>/extra/` |
| **Configuration File** | `AKMBUILD`[AKMBUILD 1-4](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L1-L4) |
| **Supported Hardware** | Wormhole and Blackhole [README.md 7-9](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/README.md?plain=1#L7-L9) |

**Sources:**[README.md 28-36](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/README.md?plain=1#L28-L36)[Makefile 49-56](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L49-L56)[AKMBUILD 1-4](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L1-L4)

## AKMBUILD Configuration File

The `AKMBUILD` file is a minimalist configuration file that defines the module identity and the resulting binary objects for the AKMS system.

### Configuration Variables

The `AKMBUILD` file contains three essential variables:

| Variable | Current Value | Purpose |
| --- | --- | --- |
| `modname` | `tenstorrent` | The name of the kernel module [AKMBUILD 1](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L1-L1) |
| `modver` | `2.9.1-pre` | The module version string [AKMBUILD 2](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L2-L2) |
| `built_modules` | `'tenstorrent.ko'` | The compiled kernel module file to be installed [AKMBUILD 3](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L3-L3) |

The `modver` in this file must be synchronized with `TENSTORRENT_DRIVER_VERSION_MAJOR`, `MINOR`, and `PATCH` in `module.h`[module.h 19-21](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/module.h#L19-L21) and the version in `dkms.conf`[VERSION_UPDATE.md 6-9](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/VERSION_UPDATE.md?plain=1#L6-L9)

## Installation Process

### Prerequisites

On Alpine Linux, the `akms` package must be installed prior to driver setup:

`apk install akms`
**Sources:**[README.md 28](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/README.md?plain=1#L28-L28)

### Installation Flow

The installation is performed via the `akms` target in the `Makefile`.

**Alpine Installation Logic**

**Sources:**[Makefile 49-52](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L49-L52)[AKMBUILD 1-4](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L1-L4)

### Step-by-Step Breakdown

1.   **Makefile Invocation**: The user runs `make akms`[Makefile 49](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L49-L49)
2.   **AKMS Installation**: The command `doas akms install .`[Makefile 50](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L50-L50) registers the current directory with AKMS.
3.   **Module Compilation**: AKMS uses the `Makefile` to compile the source files defined in `tenstorrent-y`[Makefile 5](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L5-L5) which include core logic like `wormhole.o`, `blackhole.o`, and `pcie.o`.
4.   **Module Loading**: Once installed, the driver is loaded into the kernel using `doas modprobe tenstorrent`[Makefile 51](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L51-L51)

**Sources:**[Makefile 5](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L5-L5)[Makefile 49-52](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L49-L52)

## Removal and Cleanup

To uninstall the driver from an Alpine system, the `akms-remove` target is used.

`make akms-remove`
### Removal Steps

1.   **Unload**: The module is removed from the running kernel using `doas modprobe -r tenstorrent`[Makefile 54](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L54-L54)
2.   **Unregister**: AKMS is instructed to remove the module from its database and delete the installed `.ko` files via `doas akms remove tenstorrent`[Makefile 55](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L55-L55)

**Sources:**[Makefile 53-56](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L53-L56)[README.md 62-65](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/README.md?plain=1#L62-L65)

## Version Management

Version consistency is critical across the build systems. While `dkms.conf` is the primary source of truth [VERSION_UPDATE.md 6](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/VERSION_UPDATE.md?plain=1#L6-L6) the `AKMBUILD` file must be manually updated during a release.

### Version Locations

| Component | File | Code Entity |
| --- | --- | --- |
| **AKMS** | `AKMBUILD` | `modver`[AKMBUILD 2](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L2-L2) |
| **Kernel Module** | `module.h` | `TENSTORRENT_DRIVER_VERSION_MAJOR/MINOR/PATCH`[module.h 19-21](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/module.h#L19-L21) |

## Comparison with DKMS

The AKMS implementation mirrors the DKMS implementation but uses Alpine-native tools.

| Feature | DKMS Implementation | AKMS Implementation |
| --- | --- | --- |
| **Command** | `sudo dkms add/install`[Makefile 36-37](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L36-L37) | `doas akms install`[Makefile 50](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L50-L50) |
| **Config** | `dkms.conf`[VERSION_UPDATE.md 6](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/VERSION_UPDATE.md?plain=1#L6-L6) | `AKMBUILD`[AKMBUILD 1](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/AKMBUILD#L1-L1) |
| **Cleanup** | `dkms remove`[Makefile 45](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L45-L45) | `akms remove`[Makefile 55](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/Makefile#L55-L55) |
| **Privilege** | `sudo` | `doas` |
