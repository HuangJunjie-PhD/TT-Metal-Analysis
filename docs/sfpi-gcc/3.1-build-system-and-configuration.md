This document covers the build system infrastructure and configuration mechanisms in sfpi-gcc, focusing on how the compiler is built, configured, and how the SFPU extensions are integrated into the overall build process. For information about command-line interface options available to users, see [Command Line Interface](https://deepwiki.com/tenstorrent/sfpi-gcc/3.2-command-line-interface). For details about the optimization pass pipeline execution, see [Optimization Pipeline](https://deepwiki.com/tenstorrent/sfpi-gcc/3.3-pass-management-and-optimization-pipeline).

## Build System Architecture

The sfpi-gcc build system is based on the standard GCC autotools infrastructure, with extensions to support SFPU-specific components. The build system handles multiple targets, cross-compilation, and the integration of specialized SFPU backends.

## Main Build Configuration

The primary build configuration is managed through `Makefile.in`, which defines the complete build process for sfpi-gcc. This makefile handles cross-compilation, multiple targets, and the integration of all compiler components.

## Configuration Variables and Debugging

The build system provides extensive configuration variables for controlling compilation behavior, debugging output, and optimization levels. These variables are essential for SFPU development and debugging.

### Key Configuration Categories

| Category | Variables | Purpose |
| --- | --- | --- |
| Optimization | `optimize`, `optimize_size`, `optimize_debug` | Control optimization levels |
| Debugging | `write_symbols`, `debug_info_level` | Debug information generation |
| LTO | `flag_generate_lto`, `in_lto_p` | Link-time optimization |
| Target | `target_flags`, `flag_shlib` | Target-specific configuration |
