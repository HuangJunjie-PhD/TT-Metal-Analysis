The Special Function Processing Unit (SFPU) emulation in `tt-emule` provides software-defined implementations of non-linear activation functions, complex math operations, and binary elementwise functions. While real Tenstorrent hardware uses specialized SIMD hardware to execute these operations, `tt-emule` implements them as scalar C++ loops that iterate over the [DST register file](https://github.com/tenstorrent/tt-emule/blob/a57cf072/DST%20register%20file)

## Implementation Overview

All SFPU operations follow a consistent implementation pattern:

1.   **Validation**: The `idst` index is verified against the DST register file bounds using `__emule_dst_check`.
2.   **Iteration**: A `for` loop iterates over all `__EMULE_TILE_ELEMS` (1024 elements for a 32x32 tile) [include/jit_hw/api/compute/compute_kernel_api.h 37-40](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/compute_kernel_api.h#L37-L40)
3.   **Scalar Math**: Operations use standard C++ library functions (`std::exp`, `std::log`, `std::pow`, etc.) or custom scalar logic.
4.   **In-place Update**: The result is written back to the same DST register (for unary) or a specified destination (for binary).

## Math & Trigonometry Operations

Mathematical operations often involve bit-casting parameters (like exponents or scales) from `uint32_t` back to `float` to match the hardware's register-passing convention.

### Power and Logarithms

*   **`power_tile`**: Computes $x^n$ where $n$ is passed as a packed float bit-pattern [include/jit_hw/api/compute/compute_kernel_api.h 71-83](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/compute_kernel_api.h#L71-L83)
*   **`log_tile`**: Implements natural logarithm using `std::log`[include/jit_hw/api/compute/compute_kernel_api.h 62-67](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/compute_kernel_api.h#L62-L67)
*   **`exp_tile`**: Implements $e^x$. Includes `InputClamping` options to match hardware behavior for large negatives [include/jit_hw/api/compute/eltwise_unary/exp.h 37-42](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/eltwise_unary/exp.h#L37-L42)

### Clamping and Rounding

*   **`clamp_tile`**: Restricts values between `min_val` and `max_val`. Supports both `float` and `int32_t` variants [include/jit_hw/api/compute/eltwise_unary/clamp.h 21-41](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/eltwise_unary/clamp.h#L21-L41)
*   **`typecast_tile`**: Handles conversion between data formats (e.g., `Float32` to `Int32`). In `tt-emule`, this involves rounding floats and storing them as integer bit-patterns within the `float` DST array [include/jit_hw/api/compute/eltwise_unary/typecast.h 21-55](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/eltwise_unary/typecast.h#L21-L55)

## Binary SFPU Operations

Binary SFPU operations take two source DST registers and write to a destination DST register.

### Remainder (Modulo)

The emulator implements `remainder_binary_tile` and `remainder_int32_tile` following `torch.remainder` semantics, where the sign of the result follows the divisor. This differs from the standard C++ `std::fmod`[include/jit_hw/api/compute/binary_remainder.h 9-11](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/binary_remainder.h#L9-L11)

Sources: [include/jit_hw/api/compute/binary_remainder.h 41-61](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/binary_remainder.h#L41-L61)

### Atan2

The `atan2_binary_tile` function computes `std::atan2(y, x)`. It preserves the hardware ordering where `idst0` is $y$ and `idst1` is $x$ [include/jit_hw/api/compute/atan2.h 8-25](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/atan2.h#L8-L25)

## Experimental Operations

Certain operations are injected into the compute pipeline to support specialized D2M (Direct-to-Metal) flows.

### Arange and Masking

`fill_arange_tile` is used to initialize a tile with sequential indices $[0, 1, \dots, 1023]$. Because the hardware expects tiles in `nfaces` layout, the emulator uses a lookup table `rowmajor_to_nfaces` to scatter the sequential values into the correct L1 memory locations [include/jit_hw/api/compute/experimental/fill_arange.h 37-61](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/experimental/fill_arange.h#L37-L61)

## Int32 SFPU Support

While the DST register file is fundamentally a `float` array, `tt-emule` supports `Int32` SFPU operations by type-punning the 32-bit values.

*   **`__emule_dst_load_i32(idst, i)`**: Reinterprets the float bits as `int32_t`.
*   **`__emule_dst_store_i32(idst, i, val)`**: Reinterprets the `int32_t` value as `float` bits for storage.

This allows operations like `abs_tile_int32`, `negative_tile_int32`, and `clamp_tile_int32` to function correctly on integer data stored in DST [include/jit_hw/api/compute/eltwise_unary/activations.h 25-32](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/eltwise_unary/activations.h#L25-L32)[include/jit_hw/api/compute/eltwise_unary/negative.h 21-27](https://github.com/tenstorrent/tt-emule/blob/a57cf072/include/jit_hw/api/compute/eltwise_unary/negative.h#L21-L27)
