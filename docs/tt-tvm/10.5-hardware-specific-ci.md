## Purpose and Scope

This page documents the specialized CI environments configured for testing TVM across diverse hardware targets. Each Docker image provides a complete toolchain, runtime libraries, and hardware-specific dependencies for a particular platform. These environments enable continuous integration testing for CPUs, GPUs, embedded microcontrollers, DSPs, and alternative architectures.

For general CI infrastructure and pipeline configuration, see [CI/CD Infrastructure](https://deepwiki.com/tenstorrent/tt-tvm/10-cicd-infrastructure). For Docker environment setup and Python dependencies, see [Docker CI Environments](https://deepwiki.com/tenstorrent/tt-tvm/10.2-docker-ci-environments) and [Python Environment Setup](https://deepwiki.com/tenstorrent/tt-tvm/10.3-python-environment-setup).

#### Zephyr Project Setup

Zephyr initialization involves multiple steps to configure the RTOS and SDK:

1.   **Install dependencies**: `libsdl2-dev`, `device-tree-compiler`, `dfu-util`, `west` tool
2.   **Initialize Zephyr project**: `west init --mr v3.2-branch /opt/zephyrproject`
3.   **Update modules**: `west update && west zephyr-export`
4.   **Install SDK**: Download and extract `zephyr-sdk-0.15.2_linux-x86_64.tar.gz`
5.   **Configure world-writable permissions**: For Docker container user switching

**Sources:**[docker/install/ubuntu_install_zephyr.sh 24-30](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_zephyr.sh#L24-L30)[docker/install/ubuntu_install_zephyr.sh 64-73](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_zephyr.sh#L64-L73)[docker/install/ubuntu_install_zephyr_sdk.sh 45-56](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_zephyr_sdk.sh#L45-L56)

## Testing Hardware-Specific Features

### GPU Testing Workflow

GPU CI tests CUDA, ROCm, and Vulkan code paths:

1.   **CUDA/cuDNN Tests**: Convolution kernels, tensor cores, cuBLAS operations
2.   **ROCm Tests**: AMD GPU kernels, MIOpen integration
3.   **Vulkan Tests**: Portable GPU compute shaders
4.   **OpenCL Tests**: Cross-vendor compute kernels

The environment variables ensure proper library resolution for each stack.

**Sources:**[docker/Dockerfile.ci_gpu 158-172](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/Dockerfile.ci_gpu#L158-L172)

### Embedded Target Testing

Embedded CI environments test cross-compilation and simulation:

1.   **Cortex-M**: Compile to ARM Cortex-M, run on FVP simulator or Zephyr QEMU
2.   **RISC-V**: Cross-compile, execute on QEMU or Spike simulator
3.   **Hexagon**: Compile with Hexagon toolchain, test on Android device or simulator

## Hardware CI Image Maintenance

### Version Pinning Strategy

Hardware CI images pin critical dependency versions to ensure reproducibility:

*   **Zephyr SDK**: `0.15.2` (SHA256 verified)
*   **Arduino-CLI**: `0.21.1`
*   **Emscripten**: `3.1.30`
*   **CUDA**: `11.8.0`
*   **Python packages**: SHA256-hashed in `constraints-3.8.txt`

**Sources:**[docker/install/ubuntu_install_zephyr_sdk.sh 45-48](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_zephyr_sdk.sh#L45-L48)[docker/install/ubuntu_install_arduino.sh 25](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_arduino.sh#L25-L25)[docker/install/ubuntu_install_emscripten.sh 26](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/docker/install/ubuntu_install_emscripten.sh#L26-L26)

### Image Tags

Each Docker image is tagged with a version number that increments when the image changes:

*   `ci_cpu`: Frequently updated (v0.60+)
*   `ci_gpu`: v0.60
*   `ci_arm`: v0.02
*   `ci_hexagon`: v0.02
*   `ci_cortexm`: v0.62
*   `ci_riscv`: v0.62
*   `ci_wasm`: Not tagged (follows base)
*   `ci_i386`: v0.53
