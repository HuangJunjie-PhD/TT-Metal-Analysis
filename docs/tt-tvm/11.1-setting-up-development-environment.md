## Purpose and Scope

This guide covers the process of setting up a local development environment for building and testing TVM from source. It explains the CMake-based build system, configuration options, dependency installation, and common development workflows.

For information about the CI/CD infrastructure used for automated testing, see [CI/CD Infrastructure](https://deepwiki.com/tenstorrent/tt-tvm/10-cicd-infrastructure). For build system internals and advanced configuration, see [Build System](https://deepwiki.com/tenstorrent/tt-tvm/9-build-system).

## Build System Architecture

The TVM build system is based on CMake and uses a hierarchical configuration approach.

**Sources:**[CMakeLists.txt 17-23](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L23)[CMakeLists.txt 288-350](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L288-L350)[CMakeLists.txt 581-612](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L581-L612)

### Configuration Precedence

The build system reads configuration in this order:

1.   Default values from `tvm_option()` calls in [CMakeLists.txt 29-128](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L29-L128)
2.   Template suggestions in [cmake/config.cmake 1-441](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L1-L441)
3.   User overrides in `build/config.cmake` (highest priority)

**Sources:**[CMakeLists.txt 17-28](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L17-L28)

### Step 3: Configure Build Options

#### Key CMake Options

The build system defines options using the `tvm_option()` macro. Here are the most important ones:

| Option | Values | Default | Description |
| --- | --- | --- | --- |
| `USE_LLVM` | ON/OFF/path | OFF | LLVM backend for CPU |
| `USE_CUDA` | ON/OFF/path | OFF | NVIDIA CUDA support |
| `USE_ROCM` | ON/OFF/path | OFF | AMD ROCm support |
| `USE_OPENCL` | ON/OFF/path | OFF | OpenCL support |
| `USE_VULKAN` | ON/OFF/path | OFF | Vulkan support |
| `USE_MICRO` | ON/OFF | OFF | MicroTVM for embedded |
| `USE_RPC` | ON/OFF | ON | RPC runtime |
| `USE_GRAPH_EXECUTOR` | ON/OFF | ON | Graph executor |
| `USE_PROFILER` | ON/OFF | ON | Profiling support |
| `USE_RELAY_DEBUG` | ON/OFF | OFF | Relay debug mode |
| `BUILD_STATIC_RUNTIME` | ON/OFF | OFF | Static runtime library |
| `HIDE_PRIVATE_SYMBOLS` | ON/OFF | OFF | Hide private symbols |

**Sources:**[CMakeLists.txt 29-128](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/CMakeLists.txt#L29-L128)[cmake/config.cmake 43-160](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/cmake/config.cmake#L43-L160)

### Step 4: Run CMake Configuration

CMake will detect available dependencies and configure the build. Check the output for warnings about missing optional dependencies.

**Sources:**[tests/scripts/task_config_build_cpu.sh 31-32](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/tests/scripts/task_config_build_cpu.sh#L31-L32)

### Step 5: Build TVM

This produces the following libraries in the `build` directory:

*   `libtvm.so` - Full compiler with all frontends and backends
*   `libtvm_runtime.so` - Runtime-only library (lighter weight)
