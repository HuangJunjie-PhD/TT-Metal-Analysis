## Environment Setup

Two scripts manage dependency installation. Use whichever fits your workflow:

| Script | Purpose |
| --- | --- |
| `scripts/create-venv.sh` | Creates a new virtual environment and installs all dependencies |
| `scripts/install-deps.sh` | Installs dependencies into an existing environment |

### Using `create-venv.sh`

`./scripts/create-venv.sh.venv/bin/activate`
### Using `install-deps.sh`

`./scripts/install-deps.sh`
`install-deps.sh` detects whether `uv` is available and prefers it over `pip` for package management. It installs from three requirement files:

| Requirement file | Contents |
| --- | --- |
| `ttexalens/requirements.txt` | Runtime dependencies |
| `ttexalens/dev-requirements.txt` | Development dependencies (mypy, pre-commit, etc.) |
| `test/test_requirements.txt` | Test runner dependencies (`pytest`, `coverage`, `parameterized`, `unittest-xml-reporting`) |

Sources: [scripts/install-deps.sh 1-39](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/scripts/install-deps.sh#L1-L39)[README.md 56-73](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/README.md?plain=1#L56-L73)

## Static Checks

### Pre-commit Hooks

Pre-commit hooks enforce code formatting and license header checks before each commit. The configuration is stored in `.pre-commit-config.yaml` at the repository root.

**Pre-commit setup and execution flow**

**Installation steps:**

`pip install pre-commitpre-commit install`
After installation, hooks execute automatically on every `git commit`. To run them manually on all files (useful when you have prior commits without hooks installed):

`pre-commit run --all-files`
The CI pipeline enforces this as well — `build-ttexalens.yml` runs `pre-commit run --all-files --show-diff-on-failure` as an explicit build step before compiling.

Sources: [README.md 128-152](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/README.md?plain=1#L128-L152)[.github/workflows/build-ttexalens.yml 40-43](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/.github/workflows/build-ttexalens.yml#L40-L43)

### mypy Static Type Checking

TTExaLens uses [mypy](https://mypy.readthedocs.io/) for static type analysis. Configuration is stored in a `mypy.ini` or `setup.cfg` at the repository root.

Run via the `Makefile` wrapper:

`make mypy`
Or directly:

`mypy`
The `make mypy` target invokes `python3 -m mypy`[Makefile 34-37](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/Makefile#L34-L37)

mypy is also enforced in CI via the `build-ttexalens.yml` workflow:

`- name: Run mypy  run: |    make mypy`

## Testing

### Running All Tests

`make test`
This command:

1.   Installs all dependencies (runtime, dev, test)
2.   Runs all test suites via `test/run_all.sh`

### Running Tests Individually

`# Install runtime dependenciespip install -r ttexalens/requirements.txt # Install test dependenciespip install -r test/test_requirements.txt # Run library testspytest test/ttexalens # Run CLI application testspytest test/app`
VSCode users can use the `Python` extension's Testing tab to enumerate and run individual test cases.

For a full description of test categories and hardware requirements, see [8.3](https://deepwiki.com/tenstorrent/tt-exalens/8.3-test-categories-and-coverage).

## Pull Request Process

**PR checks enforced by CI**

The `on-pr.yml` workflow is triggered on every pull request targeting `main`. It runs the full `build-and-test.yml` matrix for Ubuntu 22.04 and 24.04. The build job explicitly enforces:

| Check | Command |
| --- | --- |
| Pre-commit hooks | `pre-commit run --all-files --show-diff-on-failure` |
| Static typing | `make mypy` |
| Kernel compilation | `make` |
| Wheel build | `make wheel` |
| Library tests (NOC0) | `python3 -m xmlrunner discover -s test/ttexalens` |
| CLI tests (NOC0) | `python3 -m unittest discover -s test/app` |
| Library tests (NOC1) | `TTEXALENS_TESTS_USE_NOC1=1 python3 -m unittest discover -s test/ttexalens` |
| Wheel smoke tests | `test/wheel/run-wheel.sh` |

NOC1 tests are skipped on `n300` boards due to a known issue (tracked as issue #862 in the repository) [.github/workflows/build-and-test.yml 147-158](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/.github/workflows/build-and-test.yml#L147-L158)

Sources: [.github/workflows/on-pr.yml 1-19](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/.github/workflows/on-pr.yml#L1-L19)[.github/workflows/build-and-test.yml 83-167](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/.github/workflows/build-and-test.yml#L83-L167)[.github/workflows/build-ttexalens.yml 38-47](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/.github/workflows/build-ttexalens.yml#L38-L47)

## Checklist Summary

| Task | Command | When |
| --- | --- | --- |
| Create virtual environment | `./scripts/create-venv.sh` | Once, on setup |
| Install pre-commit hooks | `pre-commit install` | Once, on setup |
| Run pre-commit on all files | `pre-commit run --all-files` | After cloning with prior history |
| Run static type analysis | `make mypy` | Before committing |
| Build kernels | `make` | After modifying C/C++ or kernel code |
| Run all tests | `make test` | Before submitting PR |
| Regenerate documentation | `make docs` | After modifying docstrings |
| Build wheel | `make wheel` | Before submitting PR |
