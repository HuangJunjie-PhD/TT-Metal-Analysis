This page documents the evolution of tt-flash through its version releases, tracking major feature additions, platform support changes, dependency updates, and architectural improvements. tt-flash follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html) with major version boundaries reflecting significant platform generation changes.

## Version Numbering Conventions

### Semantic Versioning Interpretation

tt-flash follows semantic versioning with specific interpretations:

| Component | Meaning | Example Triggers |
| --- | --- | --- |
| Major (X.0.0) | Platform generation change | 1.x→2.x (GS→WH), 2.x→3.x (WH→BH) |
| Minor (x.Y.0) | Feature additions | boot-fs support, manifest v2.0.0 |
| Patch (x.y.Z) | Bug fixes, dependency updates | pyluwen bumps, error handling fixes |

**Note**: Version 3.0.0 was explicitly marked as having "NO BREAKING CHANGES" despite the major version bump, which was done for marketing purposes to signify the new Blackhole generation.

### Special Version Formats

**Debian Package Versions**: Follow the format `X.Y.Z` or `X.Y.Z-N` where `-N` indicates Debian packaging revisions:

*   `3.1.1-1`: Debian revision 1 of tt-flash 3.1.1
*   `3.5.0`: Direct upstream version

**Firmware Version 80.X Handling**: Starting in v3.4.8, firmware versions formatted as `80.X.Y.Z` treat the `X` component as the major version rather than `80`, accommodating special firmware numbering schemes.
