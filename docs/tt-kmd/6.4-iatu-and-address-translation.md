# IATU and Address Translation

Relevant source files
*   [enumerate.c](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/enumerate.c)
*   [memory.c](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c)
*   [memory.h](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h)

## Purpose and Scope

This document describes the IATU (Internal Address Translation Unit) subsystem and its role in translating between host physical addresses and device NOC (Network-on-Chip) addresses. The IATU enables outbound DMA operations where host memory must be made accessible to the device's internal network.

For inbound address translation (device-to-host), see [TLB Management](https://deepwiki.com/tenstorrent/tt-kmd/6.3-tlb-management). For the higher-level memory operations that utilize IATU, see [DMA Buffer Allocation](https://deepwiki.com/tenstorrent/tt-kmd/6.1-dma-buffer-allocation) and [Page Pinning](https://deepwiki.com/tenstorrent/tt-kmd/6.2-page-pinning).

## Overview

The IATU is a PCIe hardware feature that translates addresses in the outbound direction—from the host's perspective to the device's NOC address space. When host memory needs to be accessible to the device for DMA operations, the driver allocates an IATU region that maps a contiguous range in the NOC address space to the corresponding host physical addresses.

The driver manages up to 16 outbound IATU regions, allocating them on-demand when DMA buffers are allocated or user pages are pinned with the `NOC_DMA` flag. Each region is tracked per-file-descriptor and automatically released when resources are freed.

**Key Properties:**

*   **16 outbound IATU regions** available per device [memory.h 60](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L60-L60)
*   **Protected by `iatu_mutex`** for concurrent access control [memory.c 170](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L170-L170)
*   **Device-specific configuration** through the device class interface [memory.c 145](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L145-L145)
*   **Automatic allocation** during DMA operations [memory.c 159-187](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L159-L187)
*   **Top-down or bottom-up** allocation strategies [memory.c 54-120](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L54-L120)

Sources: [memory.h 58-68](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L58-L68)[memory.c 54-187](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L54-L187)

## Address Spaces and Translation

### Address Space Mapping

The IATU bridges three distinct address spaces:

**Address Space Components:**

| Address Type | Description | Range |
| --- | --- | --- |
| **Host Physical Address** | Physical address in host memory (IOMMU IOVA or real physical) | System-dependent |
| **NOC Address** | Address in device's NOC address space | `0` to `noc_dma_limit` |
| **Device DMA Address** | Actual address device uses for DMA | `noc_pcie_offset + NOC address` |

Sources: [memory.c 159-187](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L159-L187)

### IATU Region Structure

Each IATU region (defined by `struct tenstorrent_outbound_iatu_region`) contains:

*   **`priv`**: Pointer to the owning `chardev_private` context (NULL if free) [memory.h 62](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L62-L62)
*   **`base`**: Starting NOC address of this region [memory.h 63](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L63-L63)
*   **`limit`**: Ending NOC address of this region (inclusive) [memory.h 64](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L64-L64)
*   **`target`**: Host physical address this region translates to [memory.h 65](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L65-L65)

Sources: [memory.h 59-66](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L59-L66)

## IATU Region Allocation

### Allocation Strategies

The driver implements two allocation strategies for finding free space in the NOC address space by sorting existing regions using `get_sorted_iatu_region_indices`[memory.c 26-51](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L26-L51)

#### Top-Down Allocation

Used by default for DMA buffers and when explicitly requested for page pinning.

**Implementation:**`find_iatu_region_top_down`[memory.c 54-86](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L54-L86)

#### Bottom-Up Allocation

Used when the `top_down` parameter in `setup_noc_dma` is false [memory.c 174](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L174-L174)

**Implementation:**`find_iatu_region_bottom_up`[memory.c 88-120](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L88-L120)

Sources: [memory.c 54-120](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L54-L120)

### Region Allocation Process

The function `setup_noc_dma`[memory.c 159-187](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L159-L187) coordinates the allocation:

1.   Acquires `tt_dev->iatu_mutex`[memory.c 170](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L170-L170)
2.   Invokes the selected search strategy (`find_iatu_region_top_down` or `find_iatu_region_bottom_up`) [memory.c 172-174](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L172-L174)
3.   Calls `configure_outbound_iatu` to update the hardware and tracking structure [memory.c 182](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L182-L182)
4.   Calculates the `noc_address` using `tt_dev->dev_class->noc_pcie_offset`[memory.c 183](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L183-L183)

Sources: [memory.c 123-187](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L123-L187)

## Hardware Configuration

### Device Class Interface

IATU configuration is hardware-specific and implemented through the device class abstraction:

`// Function pointer in tenstorrent_device_classint (*configure_outbound_atu)(    struct tenstorrent_device *ttdev,    u32 region,    // Region index (0-15)    u64 base,      // NOC base address    u64 limit,     // NOC limit address    u64 target     // Host physical address);`
The driver iterates through `TENSTORRENT_MAX_OUTBOUND_IATU_REGIONS` to find a free slot where `priv == NULL`[memory.c 134-140](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L134-L140)

Sources: [memory.c 123-156](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L123-L156)[memory.h 60](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.h#L60-L60)

## Integration with Memory Operations

### Peer-to-Peer BAR Mapping

The `MAP_PEER_BAR` ioctl allows mapping a BAR from one Tenstorrent device (the peer) into the address space of another (the local device) using an outbound iATU region.

Implementation: `ioctl_map_peer_bar`[memory.c 1218-1316](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L1218-L1316)

### Diagnostics and Visibility

The `mappings` debugfs file (implemented by `mappings_seq_show`) provides visibility into active iATU regions [enumerate.c 84-212](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/enumerate.c#L84-L212) It displays the translation path: `VA -> IOVA/PA -> NOC Address`[enumerate.c 148-150](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/enumerate.c#L148-L150)

## Synchronization

### Mutex Protection

The `iatu_mutex` serializes all IATU operations [memory.c 170](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L170-L170)

| Operation | Lock Scope |
| --- | --- |
| **Allocation** | `mutex_lock(&tt_dev->iatu_mutex)` in `setup_noc_dma`[memory.c 170](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L170-L170) |
| **Hardware Configuration** | Protected by `iatu_mutex` inside `setup_noc_dma` |
| **Diagnostics** | `mutex_trylock(&priv->device->iatu_mutex)` in `mappings_seq_show`[enumerate.c 117](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/enumerate.c#L117-L117) |

Sources: [memory.c 170](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L170-L170)[enumerate.c 117](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/enumerate.c#L117-L117)

## Error Handling

### Common Error Conditions

| Error | Condition | Return Value |
| --- | --- | --- |
| **No free regions** | All 16 IATU regions allocated | `-ENOSPC`[memory.c 142](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L142-L142) |
| **No free NOC space** | Cannot find gap in NOC address space | `-ENOMEM`[memory.c 178](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L178-L178) |
| **Invalid parameters** | `base > limit` or `size == 0` | `-EINVAL`[memory.c 131-168](https://github.com/tenstorrent/tt-kmd/blob/0ab170d7/memory.c#L131-L168) |
