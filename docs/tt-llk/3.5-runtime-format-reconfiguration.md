## Purpose and Scope

Runtime format reconfiguration enables dynamic modification of data formats for the unpacker, math unit, and packer during kernel execution without requiring full reinitialization of the hardware pipeline. This allows kernels to process tiles with different data formats within a single operation, supporting mixed-precision computations and format conversion workflows.

This page covers the mechanisms for reconfiguring data formats at runtime. For initial hardware configuration during kernel setup, see [Core Kernel API (ckernel)](https://deepwiki.com/tenstorrent/tt-llk/2-core-kernel-api-(ckernel)). For the complete set of supported data formats and their characteristics, see [DataFormat Enumeration and Support](https://deepwiki.com/tenstorrent/tt-llk/3.1-dataformat-enumeration-and-support).

## Overview

The Tensix hardware pipeline (unpack→math→pack) is configured at initialization with specific data formats for each stage. However, many operations require processing tiles with varying formats or performing format conversions. Runtime format reconfiguration provides a lightweight mechanism to change these formats without resetting the entire pipeline state.

Key capabilities:

*   **Per-stage reconfiguration**: Each pipeline stage (unpacker A/B, math, packer) can be reconfigured independently
*   **Partial updates**: Only format-related registers are modified, preserving other configuration state
*   **Read-modify-write protocol**: Uses atomic register updates to avoid race conditions
*   **Stall synchronization**: Ensures hardware quiescence before configuration changes

## Configuration Structures and Registers

Runtime format reconfiguration modifies specific fields within hardware configuration structures. The following table summarizes the key structures and their format-related fields:

| Structure | Location | Format Fields | Purpose |
| --- | --- | --- | --- |
| `unpack_tile_descriptor_t` | THCON_SEC0/1_REG0 | `in_data_format` (4 bits) | L1 input format specification |
| `unpack_config_t` | THCON_SEC0/1_REG2 | `out_data_format` (4 bits) | Unpacker output to SrcA/SrcB registers |
| `alu_config_t` | ALU_FORMAT_SPEC_REG0 | `SrcA` (4 bits), `SrcB` (4 bits) | Math unit input format specification |
| `pack_config_t` | THCON_SEC0_REG1 | `in_data_format` (4 bits), `out_data_format` (4 bits) | Packer input from Dest and output to L1 |
| `dest_rd_ctrl_t` | PCK_DEST_RD_CTRL | `Read_32b_data`, `Read_int8`, `Read_unsigned` | Destination register read mode control |

## Unpacker Reconfiguration Protocol

The unpacker supports independent reconfiguration of its two channels (A and B) through dedicated functions. The protocol ensures hardware synchronization and proper format conversion setup.

### Unpacker A Reconfiguration

The `_llk_unpack_reconfig_data_format_srca_impl_()` function modifies unpacker A's format configuration:

**Reconfiguration Steps:**

1.   **Stall unpacker**: `TTI_STALLWAIT(p_stall::STALL_CFG, p_stall::UNPACK0)` ensures unpacker A is idle [tt_llk_blackhole/llk_lib/llk_unpack_common.h 88](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L88-L88)

2.   **Update tile descriptor**: Modify `THCON_SEC0_REG0_TileDescriptor` with new `in_data_format`[tt_llk_blackhole/llk_lib/llk_unpack_common.h 95](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L95-L95)

3.   **Update output format**: Set `THCON_SEC0_REG2_Out_data_format` to destination format [tt_llk_blackhole/llk_lib/llk_unpack_common.h 99](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L99-L99)

4.   **Update tile size**: Store new tile size in GPR `p_gpr_unpack::TILE_SIZE_A`[tt_llk_blackhole/llk_lib/llk_unpack_common.h 100](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L100-L100)

5.   **Special format handling**:

    *   FP8 E4M3: Set `THCON_SEC0_REG1_Unp_LF8_4b_exp` flag [tt_llk_blackhole/llk_lib/llk_unpack_common.h 97](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L97-L97)
    *   UInt8: Set `ALU_FORMAT_SPEC_REG0_SrcAUnsigned` flag [tt_llk_blackhole/llk_lib/llk_unpack_common.h 92](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L92-L92)

6.   **Dimension/stride reconfiguration** (if `dim_stride_target == FACE_ROW_MAJOR`):

    *   Recalculate Z-stride based on new format [tt_llk_blackhole/llk_lib/llk_unpack_common.h 109-110](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L109-L110)
    *   Update `THCON_SEC0_REG5_Tile_x_dim_cntx0` with face dimensions [tt_llk_blackhole/llk_lib/llk_unpack_common.h 115](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L115-L115)
    *   Update `THCON_SEC0_REG0_TileDescriptor` Z-dim with `unpack_num_faces`[tt_llk_blackhole/llk_lib/llk_unpack_common.h 118](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L118-L118)
    *   Set unpacker X-end counter via `TT_SETADCXX`[tt_llk_blackhole/llk_lib/llk_unpack_common.h 120](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L120-L120)

**Template Parameters:**

*   `is_fp32_dest_acc_en`: Enable FP32 destination accumulation mode
*   `to_from_int8`: Enable Int8 format transitions (requires FP32 dest mode)
*   `dim_stride_target`: Control dimension/stride updates (`IGNORE` or `FACE_ROW_MAJOR`)

Sources: [tt_llk_blackhole/llk_lib/llk_unpack_common.h 79-122](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_unpack_common.h#L79-L122)[tt_llk_wormhole_b0/llk_lib/llk_unpack_common.h 80-120](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/llk_lib/llk_unpack_common.h#L80-L120)

## Math Unit Reconfiguration

The math unit (FPU/ALU) requires format reconfiguration when the unpacker output formats change. The `llk_math_common.h` provides three reconfiguration functions:

### Source A Format Reconfiguration

`_llk_math_reconfig_data_format_srca_(srca_data_format)` updates the ALU format specification for source A:

**Wormhole B0 Implementation**[tt_llk_wormhole_b0/llk_lib/llk_math_common.h 143-161](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/llk_lib/llk_math_common.h#L143-L161):

1.   Stall math unit: `TTI_STALLWAIT(p_stall::STALL_CFG, p_stall::MATH | p_stall::WAIT_SFPU)`
2.   For `to_from_int8` conversions: 
    *   Calculate `int8_math_enabled` flag based on format
    *   Update both `ALU_FORMAT_SPEC_REG0_SrcA` and `ALU_ACC_CTRL_INT8_math_enabled` atomically

3.   Otherwise: Update only `ALU_FORMAT_SPEC_REG0_SrcA` field

**Blackhole Implementation**[tt_llk_blackhole/llk_lib/llk_math_common.h 156-167](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/llk_lib/llk_math_common.h#L156-L167):

*   Simplified: Only handles `int8_math_enabled` flag for Int8/Int32 formats
*   ALU format is inferred from unpacker configuration (no explicit SrcA/SrcB programming)

### Source B Format Reconfiguration

`_llk_math_reconfig_data_format_srcb_(srcb_data_format)` mirrors the Source A protocol for source B, updating `ALU_FORMAT_SPEC_REG1_SrcB` on Wormhole B0 [tt_llk_wormhole_b0/llk_lib/llk_math_common.h 163-181](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_wormhole_b0/llk_lib/llk_math_common.h#L163-L181)

All configuration updates use the `cfg_reg_rmw_tensix` template function to perform atomic read-modify-write operations on configuration registers. This prevents race conditions when multiple configuration fields share the same register word.

### RMW Execution Sequence

1.   **Mutex Acquisition** (when needed): `t6_mutex_acquire(mutex::REG_RMW)`[tt_llk_blackhole/common/inc/cpack_common.h 430](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L430-L430)
2.   **Read Current Value**: From configuration register at `ADDR32`
3.   **Modify Field**: `new_value = (old_value & ~MASK) | ((data << SHAMT) & MASK)`
4.   **Write Back**: To same configuration register
5.   **Mutex Release**: `t6_mutex_release(mutex::REG_RMW)`[tt_llk_blackhole/common/inc/cpack_common.h 448](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tt_llk_blackhole/common/inc/cpack_common.h#L448-L448)
