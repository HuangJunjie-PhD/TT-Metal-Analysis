This document covers the automated testing infrastructure in the Polaris repository, focusing on the `checkin_tests.py` framework and its integration with CI/CD workflows. This system provides comprehensive test automation for code quality, static analysis, and workload validation.

For information about the development environment setup, see [Development Environment](https://deepwiki.com/tenstorrent/polaris/5.1-development-environment). For details about code quality tools and SPDX compliance, see [Code Quality and Compliance](https://deepwiki.com/tenstorrent/polaris/5.3-code-quality-and-compliance).

## Test Automation Framework Overview

The test automation system is built around the `checkin_tests.py` script, which orchestrates different types of tests and integrates with GitHub Actions for continuous integration. The framework supports multiple test categories and provides flexible execution options.

## Test Categories and Command Preparation

The test automation framework supports three primary test categories, each with its own command preparation function and execution strategy.

### Coverage Testing

The coverage testing system uses the `coverage` tool to measure code coverage during test execution:

The coverage testing leverages configuration from `pyproject.toml` and excludes tests marked with `tools_secondary` markers [checkin_tests.py 19](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/checkin_tests.py#L19-L19)

### Static Analysis

Static analysis is performed using the `mypy` type checker:

The static analysis configuration is centralized in `pyproject.toml` to avoid configuration duplication [checkin_tests.py 30-36](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/checkin_tests.py#L30-L36)

### Workload Testing

Workload testing validates the core simulation functionality by running `polaris.py` with various parameter combinations:

The workload testing system generates test combinations using `itertools.product()` to create comprehensive parameter sweeps [checkin_tests.py 39-63](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/checkin_tests.py#L39-L63)

## CI/CD Integration

The test automation system integrates with GitHub Actions to provide continuous integration capabilities.

### GitHub Actions Workflow

The workflow uses specific conda channels and configurations to ensure reproducible builds [.github/workflows/checkin_tests.yml 26-35](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/.github/workflows/checkin_tests.yml#L26-L35)

### Environment Configuration

The CI/CD system uses conda environment management with specific channel configurations:

| Configuration | Value | Purpose |
| --- | --- | --- |
| `miniforge-version` | `latest` | Conda distribution |
| `channels` | `conda-forge` | Package source |
| `conda-remove-defaults` | `"true"` | Avoid default channels |
| `activate-environment` | `poldevenv` | Target environment |
| `environment-file` | `envdev.yaml` | Environment specification |
| `condarc-file` | `etc/condarc` | Conda configuration |

Sources: [.github/workflows/checkin_tests.yml 1-69](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/.github/workflows/checkin_tests.yml#L1-L69)

## Configuration and Customization

The test automation system provides extensive configuration options for customizing test execution.

### Command Line Interface

The CLI system supports flexible test selection and execution control [checkin_tests.py 98-125](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/checkin_tests.py#L98-L125)

### Test Configuration Constants

| Constant | Value | Purpose |
| --- | --- | --- |
| `ODIR` | `'__RUN_TESTS'` | Output directory |
| `LOGD` | `'__RUN_TESTS/logs'` | Log directory |
| `PYTEST_KNOBS` | `'--durations=0 -m "not tools_secondary"'` | pytest options |

### Workload Test Parameters

The workload testing system uses predefined parameter sets for comprehensive testing:

| Parameter | Options | Purpose |
| --- | --- | --- |
| `filterarch` | `'A100,Q1_A1'` | Architecture filtering |
| `filterwli` | `'gpt_nano'` | Workload filtering |
| `option1` | `["", ' --instr_profile']` | Instruction profiling |
| `option2` | `["", ' --dump_ttsim_onnx']` | ONNX dumping |
| `option3` | `["", ' --frequency 1000 1200 100']` | Frequency sweeps |
| `option4` | `["", ' --batchsize 1 8 2']` | Batch size sweeps |

Sources: [checkin_tests.py 12-49](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/checkin_tests.py#L12-L49)
