This document covers the local development build system for the tt-burnin project, including the `Makefile`, `setup.py` compatibility layer, and Debian build configuration. This system enables developers to set up their development environment and build the package locally.

For information about the automated CI/CD build workflows and release management, see [CI/CD Workflows](https://deepwiki.com/tenstorrent/tt-burnin/5.2-cicd-workflows). For details about Debian packaging structure and distribution, see [Debian Packaging](https://deepwiki.com/tenstorrent/tt-burnin/5.1-debian-packaging).

## Local Development Build System

The primary build system for local development is managed through a `Makefile` that provides convenient targets for setting up virtual environments and installing the package in development mode.

### Build Targets and Workflow

**Local Development Build Workflow**

The `Makefile` provides three primary targets:

| Target | Purpose | Virtual Environment | Installation Mode |
| --- | --- | --- | --- |
| `build` | Development setup | `.env/` | Editable mode with dev dependencies |
| `release` | Release testing | `my-env/` | Production mode from requirements.txt |
| `clean` | Environment cleanup | N/A | Removes `.env/` directory |

**Sources:**[Makefile 1-24](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L1-L24)

### Build Target Implementation

The `build` target creates a development environment with editable installation:

*   Creates virtual environment in `.env/` directory [Makefile 9](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L9-L9)
*   Upgrades pip to latest version [Makefile 10](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L10-L10)
*   Installs package in editable mode with development dependencies [Makefile 11](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L11-L11)

The installation command uses several important flags:

*   `--upgrade`: Updates existing packages
*   `--ignore-installed`: Reinstalls even if already present
*   `-v`: Verbose output
*   `-e`: Editable/development mode installation
*   `.[dev]`: Installs package with dev extra dependencies

**Sources:**[Makefile 7-12](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L7-L12)

### Release Target Implementation

The `release` target simulates a production environment:

*   Creates separate virtual environment in `my-env/`[Makefile 15](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L15-L15)
*   Installs from `requirements.txt` first [Makefile 17](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L17-L17)
*   Performs standard (non-editable) package installation [Makefile 18](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L18-L18)

**Sources:**[Makefile 13-19](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L13-L19)

## Package Setup Configuration

The project uses a compatibility `setup.py` file alongside the primary `pyproject.toml` configuration to support older packaging tools, particularly on Ubuntu 22.04 systems.

### Dynamic Versioning System

**Dynamic Versioning from Git Tags**

The `setup.py` file implements dynamic versioning using `setuptools_scm`:

*   Uses `use_scm_version=True` to derive version from git tags [setup.py 12](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L12-L12)
*   Requires `setuptools_scm` as a build dependency [setup.py 15](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L15-L15)
*   Provides fallback package metadata for older tools [setup.py 11-14](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L11-L14)

**Sources:**[setup.py 1-17](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L1-L17)

### Package Discovery and Requirements

The setup configuration specifies:

*   Package name as `"tt-burnin"`[setup.py 11](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L11-L11)
*   Automatic package discovery with `find_packages()`[setup.py 13](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L13-L13)
*   Python version requirement of `>=3.10`[setup.py 14](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L14-L14)

**Sources:**[setup.py 9-16](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L9-L16)

## Debian Build Rules

The Debian packaging system uses `pybuild` to integrate with the Python build system through standardized rules.

### Build System Integration

**Debian Build System Integration**

The `debian/rules` file configures the build process:

*   Uses `dh` (debhelper) with `pybuild` buildsystem [debian/rules 4-6](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/rules#L4-L6)
*   Specifies Python 3 support explicitly [debian/rules 6](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/rules#L6-L6)
*   Overrides test execution to skip during packaging [debian/rules 9-10](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/rules#L9-L10)

**Sources:**[debian/rules 1-12](https://github.com/tenstorrent/tt-burnin/blob/809f293d/debian/rules#L1-L12)

## Build Environment Configuration

The build system maintains clean separation between different environment types and manages ignored files appropriately.

### Environment Isolation

| Environment | Directory | Purpose | Lifecycle |
| --- | --- | --- | --- |
| Development | `.env/` | Editable installation, dev dependencies | Created by `make build`, removed by `make clean` |
| Release Testing | `my-env/` | Production-like environment | Created by `make release`, manually managed |
| Build Artifacts | `build/`, `*.egg-info` | Temporary build files | Ignored by git, cleaned automatically |

**Sources:**[.gitignore 1-9](https://github.com/tenstorrent/tt-burnin/blob/809f293d/.gitignore#L1-L9)[Makefile 9](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L9-L9)[Makefile 15](https://github.com/tenstorrent/tt-burnin/blob/809f293d/Makefile#L15-L15)

### Ignored Build Artifacts

The `.gitignore` configuration excludes common Python build artifacts:

*   Egg directories and info: `.eggs/`, `*.egg-info`[.gitignore 1-2](https://github.com/tenstorrent/tt-burnin/blob/809f293d/.gitignore#L1-L2)
*   Virtual environments: `my-env/`, `.env/`[.gitignore 3-4](https://github.com/tenstorrent/tt-burnin/blob/809f293d/.gitignore#L3-L4)
*   Python cache: `__pycache__/`[.gitignore 6](https://github.com/tenstorrent/tt-burnin/blob/809f293d/.gitignore#L6-L6)
*   Build directories: `build/`[.gitignore 8](https://github.com/tenstorrent/tt-burnin/blob/809f293d/.gitignore#L8-L8)

**Sources:**[.gitignore 1-9](https://github.com/tenstorrent/tt-burnin/blob/809f293d/.gitignore#L1-L9)

## Integration with Release Pipeline

The local build system connects to the broader release pipeline through consistent configuration and tooling choices.

### Build System Coordination

**Build System Integration with Release Pipeline**

The local build system supports the same package configuration used in automated builds:

*   `setup.py` provides compatibility for Debian packaging tools [setup.py 3-4](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L3-L4)
*   `setuptools_scm` ensures version consistency across local and CI builds [setup.py 4](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L4-L4)[setup.py 12](https://github.com/tenstorrent/tt-burnin/blob/809f293d/setup.py#L12-L12)
*   Virtual environment patterns match CI environment isolation strategies
