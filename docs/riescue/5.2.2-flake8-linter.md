## Purpose and Scope

This document describes the Flake8 linter configuration and usage within the RiESCUE project. Flake8 is a style guide enforcement tool that checks Python code for PEP 8 compliance, programming errors, and code complexity issues. This page covers the configuration file [.flake8](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8) the excluded directories, ignored error codes, and how Flake8 is invoked both locally and in the CI/CD pipeline.

For information about code formatting with Black, see [Black Code Formatter](https://deepwiki.com/tenstorrent/riescue/5.2.1-black-code-formatter). For type checking, see [Pyright Type Checker](https://deepwiki.com/tenstorrent/riescue/5.2.3-pyright-type-checker). For information about pre-commit hooks that automate linting, see [Pre-commit Hooks](https://deepwiki.com/tenstorrent/riescue/5.2.4-pre-commit-hooks). For the complete CI/CD pipeline, see [CI/CD Pipeline](https://deepwiki.com/tenstorrent/riescue/5.3-cicd-pipeline).

## Configuration Overview

Flake8 is configured through the [.flake8](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8) file in the repository root. This configuration file uses INI format with a `[flake8]` section that defines behavior, output options, excluded paths, and ignored error codes.

Sources: [.flake8 1-26](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L1-L26)

## Behavior Configuration

The following table describes the behavior settings configured in [.flake8 3-7](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L3-L7):

| Configuration Key | Value | Description |
| --- | --- | --- |
| `max-line-length` | `200` | Maximum line length before Flake8 reports an error. Set higher than PEP 8's 79 characters to accommodate complex expressions |
| `statistics` | `True` | Display count of each error/warning type at the end of the run |
| `show-source` | `True` | Display the source code line that triggered each violation |
| `tee` | `True` | Write output to both stdout and the output file simultaneously |
| `output_file` | `flake8.log` | Path where Flake8 writes its output log |

Sources: [.flake8 3-7](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L3-L7)

## Excluded Paths

Flake8 skips checking files in the following directories, configured in [.flake8 8-19](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L8-L19):

| Path Pattern | Reason for Exclusion |
| --- | --- |
| `riscv-dv` | External dependency: Google's RISC-V DV framework |
| `riscv-isa-sim` | External dependency: Spike ISS source code |
| `whisper` | External dependency: Whisper ISS source code |
| `fpgen` | External or generated code |
| `riescue_config` | Configuration data, not source code |
| `*_clone` | Cloned repositories (external code) |
| `*-scripts` | Build and utility scripts (external) |
| `infra` | Infrastructure scripts (excluded from linting) |
| `.git` | Git version control metadata |
| `opensrc*` | Open source dependencies |
| `.venv` | Python virtual environments |

These exclusions prevent Flake8 from checking external dependencies and generated files that the project does not control.

Sources: [.flake8 8-19](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L8-L19)

## Ignored Error Codes

The following error codes are explicitly ignored using `extend-ignore` in [.flake8 21-25](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L21-L25):

| Error Code | Category | Description | Reason for Ignoring |
| --- | --- | --- | --- |
| `E203` | Whitespace | Whitespace before ':' | Conflicts with Black formatter's handling of slicing syntax |
| `F401` | Import | Module imported but unused | Common in `__init__.py` files for re-exporting symbols |
| `F841` | Variable | Local variable assigned but never used | Occurs in test scaffolding and debugging code |
| `E731` | Lambda | Do not assign a lambda expression, use a def | Project style allows lambda assignments for brevity |

The `extend-ignore` directive adds these codes to Flake8's default ignore list rather than replacing it entirely.

Sources: [.flake8 21-25](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8#L21-L25)[docs/lint.md 38-48](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/lint.md?plain=1#L38-L48)

## CI/CD Integration

In the CI/CD pipeline, Flake8 is executed as part of the lint phase. The workflow defined in [.github/workflows/smoke.yml](https://github.com/tenstorrent/riescue/blob/07cefde3/.github/workflows/smoke.yml) invokes Flake8 using the container system:

This command:

1.   Executes `flake8` inside the Singularity container environment
2.   Loads configuration from [.flake8](https://github.com/tenstorrent/riescue/blob/07cefde3/.flake8)
3.   Checks all Python files in the repository root (`.`)
4.   Applies exclusions and ignore rules from the configuration
5.   Returns exit code 0 if no violations found, non-zero otherwise

The CI pipeline fails if Flake8 reports any violations, ensuring all code adheres to the configured style guidelines before merging.

Sources: [docs/lint.md 7-10](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/lint.md?plain=1#L7-L10)[docs/lint.md 44-48](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/lint.md?plain=1#L44-L48)

## Local Usage

Developers can run Flake8 locally to check their code before committing:

This replicates the exact CI flow locally, allowing developers to catch and fix violations before pushing code. The output displays:

*   Each violation with file path, line number, column, and error code
*   The source code line causing the violation (due to `show-source = True`)
*   Statistics summary of violation types (due to `statistics = True`)

The output is written to both stdout and `flake8.log` (due to `tee = True`).

Sources: [docs/lint.md 44-48](https://github.com/tenstorrent/riescue/blob/07cefde3/docs/lint.md?plain=1#L44-L48)

## Integration with Other Linting Tools

Flake8 is one component of a multi-tool linting strategy:

**Diagram: Multi-Tool Linting Integration**

*   **Black** ([#5.2.1](https://deepwiki.com/tenstorrent/riescue/5.2.1-black-code-formatter)) handles formatting (whitespace, indentation)
*   **Flake8** ([#5.2.2](https://deepwiki.com/tenstorrent/riescue/5.2.2-flake8-linter)) enforces style guidelines and catches programming errors
*   **Pyright** ([#5.2.3](https://deepwiki.com/tenstorrent/riescue/5.2.3-pyright-type-checker)) performs static type checking

The `extend-ignore` of `E203` in Flake8 prevents conflicts with Black's whitespace handling.
