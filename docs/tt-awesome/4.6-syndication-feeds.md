# Syndication Feeds

The `tt-awesome` platform provides several automated syndication feeds to keep the community updated on ecosystem developments. These feeds are generated during the Eleventy build process and cover new project entries, software releases, and aggregated resources like articles and videos.

## Overview of Feed Templates

The project implements four distinct feeds located in `src/feeds/`. These templates leverage custom Nunjucks filters defined in `.eleventy.js` to transform the internal data model into standard XML (Atom) and JSON formats.

| Feed Type | File Path | Content Description |
| --- | --- | --- |
| **Recent Releases** | `src/feeds/releases.njk` | Atom feed of the latest GitHub releases summarized by LLM. |
| **New Entries** | `src/feeds/new-entries.njk` | Atom feed of the 50 most recently added projects. |
| **Articles & Resources** | `src/feeds/articles.njk` | Atom feed for external links (videos, papers, blog posts). |
| **JSON Feed 1.1** | `src/feeds/feed.njk` | A unified export following the JSON Feed 1.1 specification. |

## Data Flow and Logic

The generation of feeds relies on the transformation of the global `entries` collection (provided by `src/_data/entries.js`) and the `recentReleases` collection (provided by `src/_data/recentReleases.js`).

### The jsonFeedItems Filter

The primary engine for the unified JSON feed is the `jsonFeedItems` filter. This function constructs a standardized schema by iterating through three data sources:

1.   **Release Items**: Maps `recentReleases` to feed items, using the summarized release notes as the summary field [.eleventy.js 149-158](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L149-L158)
2.   **Entry Items**: Takes the 50 most recent entries (sorted by `added_at`) and resolves a canonical URL—prioritizing repository links over general links [.eleventy.js 169-180](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L169-L180)
3.   **Article Items**: Extracts links of specific types (`article`, `lesson`, `paper`, `talk`, `video`, `demo`) from every entry to provide a feed of educational and research content [.eleventy.js 183-195](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L183-L195)

### Article Extraction Logic

The `articleFeedItems` filter is used specifically for the `articles.xml` Atom feed. It deduplicates URLs across all entries to ensure that if a resource is linked in multiple projects, it only appears once in the feed [.eleventy.js 107-114](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L107-L114)

## Feed Discovery

Feeds are made discoverable to browsers and RSS readers via `<link>` tags in the `base.njk` layout. This ensures that when a user visits the site, their feed reader can automatically detect the available syndication options.

| Link Relation | Type | URL Path |
| --- | --- | --- |
| `alternate` | `application/atom+xml` | `/feeds/releases.xml` |
| `alternate` | `application/atom+xml` | `/feeds/new-entries.xml` |
| `alternate` | `application/atom+xml` | `/feeds/articles.xml` |
| `alternate` | `application/feed+json` | `/feeds/feed.json` |

## Implementation Details

### Sorting and Limits

To maintain feed performance and relevance, the following constraints are applied:

*   **Recency**: Entries are sorted by the `added_at` property in descending order [.eleventy.js 100-105](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L100-L105)
*   **Truncation**: The `articleFeedItems` filter defaults to a limit of 50 items [.eleventy.js 98](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L98-L98)
*   **Deduplication**: The system uses a `Set` named `seen` or `seenLinks` to ensure unique URLs within a single feed generation run [.eleventy.js 107-114](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L107-L114)[.eleventy.js 167-185](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L167-L185)

### Date Formatting

Feeds require specific date formats (ISO 8601 for JSON Feed, RFC 3339 for Atom). The system utilizes the `added_at` field from entry JSON files, defaulting to `"1970-01-01"` if the field is missing to prevent build failures during sorting [.eleventy.js 101-102](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.eleventy.js#L101-L102)
