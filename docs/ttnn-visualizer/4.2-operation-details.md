The Operation Details view provides in-depth information about individual operations in a Tenstorrent Neural Network (TT-NN) model. It visualizes memory allocations, tensor information, and device operations for a selected operation. This page explains the structure, features, and usage of the Operation Details component.

For information about navigating between operations, see [Operation List](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.1-operation-list). For memory-specific visualization features, see [Memory Visualization](https://deepwiki.com/tenstorrent/ttnn-visualizer/4.3-memory-visualization).

## Overview

The Operation Details view is a comprehensive interface for analyzing a single operation's memory usage, tensor allocations, and device-level operations. It allows users to visualize memory layouts in both L1 and DRAM, examine tensor properties, and understand device operations and their memory impacts.

### Key Features

*   L1 and DRAM memory visualizations with interactive plots
*   Tensor details with memory allocation information
*   Device operations graph and detailed execution breakdown
*   Stack trace information with source code access
*   Memory allocation analysis including peak memory tracking
*   Tensor memory layout visualization per core
*   Navigation controls for moving between operations

## Memory Visualization

The Operation Details view provides detailed visualization of memory allocations in both L1 and DRAM memory. Users can toggle between these views and customize their display options.

### Memory Sections

Memory visualization is divided into two main sections:

1.   **L1 Memory**: Shows allocations in on-chip memory (tensor cores)
2.   **DRAM Memory**: Shows allocations in off-chip memory

Users can toggle each section independently using the buffer type controls.

### Display Options

The following display options are available for memory visualizations:

| Option | Description |
| --- | --- |
| Buffer zoom | Zooms in on relevant memory regions with allocations |
| Show Circular Buffers Details | Displays detailed information about circular buffers |
| Tensor memory layout overlay | Shows tensor memory layout patterns in memory plots |
| Show L1 Small | Displays L1 Small memory region (if available) |
| Hex axis labels | Shows memory addresses in hexadecimal format |
| Memory regions | Displays memory region boundaries |

### Tensor Memory Layout Visualization

For L1 memory, two additional visualization options are available:

*   **All allocations per core**: Shows how tensor memory is allocated across all cores
*   **Input/Output allocations per core**: Shows specifically how inputs and outputs are allocated across cores

## Device Operations

The Device Operations section provides a detailed view of low-level device operations performed during the execution of the current operation.

### Device Operations Graph

The Device Operations Graph button opens a graphical representation of device operations and their relationships. This graph shows the flow of execution and data dependencies between device operations.

### Device Operations Detailed View

The detailed view shows a hierarchical breakdown of device operations, including:

*   Function starts and ends
*   Buffer allocations and deallocations
*   Circular buffer allocations and deallocations
*   Tensor operations
*   Memory usage tracking

For each device operation, the following information is displayed:

*   Operation name and parameters
*   Memory allocations and their sizes
*   Memory layout information
*   Peak memory usage tracking

The Operation Details view includes navigation controls that allow users to move between operations and access related views.

## Implementation Details

The Operation Details view is implemented by the `OperationDetailsComponent` class, which uses several hooks and helper functions to retrieve and process operation data.

### Key Data Processing Classes

The `OperationDetails` class (in [src/model/OperationDetails.ts](https://github.com/tenstorrent/ttnn-visualizer/blob/b191aae4/src/model/OperationDetails.ts)) is responsible for processing raw operation data into a format suitable for visualization. It provides methods for:

*   Retrieving memory data for different buffer types
*   Calculating peak memory usage
*   Processing device operations
*   Mapping tensors to memory addresses
*   Generating chart data for memory plots

### Memory Data Processing

Memory data processing involves several steps:

1.   Filtering buffers by buffer type (L1, DRAM, etc.)
2.   Calculating continuous memory ranges
3.   Identifying fragmentation in memory
4.   Generating chart data for visualization
5.   Processing circular buffers and device-level buffers

### Device Operations Processing

Device operations processing involves:

1.   Preprocessing operation connections to establish relationships
2.   Merging operations across devices (if applicable)
3.   Sorting operations by buffer deallocation
4.   Tracking memory allocations during operation execution
5.   Calculating peak memory usage
