# Modal Screens and Dialogs

Relevant source files
*   [tt_tools_common/ui_common/common_style.css](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css)

## Purpose and Scope

This document covers the modal screen components provided by tt-tools-common for building interactive dialogs in Textual-based TUI applications. Modal screens are overlay widgets that block interaction with the underlying application until dismissed. The library provides three pre-built modal components: `TTConfirmBox` for yes/no confirmations, `TTHelperMenuBox` for displaying help documentation, and `TTSettingsMenu` for application settings with toggle switches.

For information about non-modal widgets like headers, footers, and data tables, see [Widget Library Reference](https://deepwiki.com/tenstorrent/tt-tools-common/4.1-widget-library-reference). For theming and CSS customization of these modals, see [Theming and Styling](https://deepwiki.com/tenstorrent/tt-tools-common/4.3-theming-and-styling).

* * *

## Modal Screen Architecture

All modal components in tt-tools-common inherit from Textual's `ModalScreen` base class, which provides the fundamental overlay behavior. Modal screens suspend interaction with the base application until explicitly dismissed via `app.pop_screen()`.

**Sources:**[tt_tools_common/ui_common/widgets.py 10](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L10-L10)[tt_tools_common/ui_common/widgets.py 246-365](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L246-L365)

## TTConfirmBox: Confirmation Dialogs

### Overview

`TTConfirmBox` provides a simple yes/no confirmation dialog with customizable text and callback functions. It renders as a centered modal with two buttons and automatically dismisses itself after user selection.

### Class Definition

The class is defined at [tt_tools_common/ui_common/widgets.py 246-272](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L246-L272) with the following signature:

`class TTConfirmBox(ModalScreen):    def __init__(        self, text: str, on_yes: Callable = None, on_no: Callable = None    ) -> None`
**Parameters:**

| Parameter | Type | Description |
| --- | --- | --- |
| `text` | `str` | The question or confirmation message to display |
| `on_yes` | `Callable` (optional) | Callback function executed when "Yes" is pressed |
| `on_no` | `Callable` (optional) | Callback function executed when "No" is pressed |

### Component Structure

The dialog is composed using a `Grid` container with three child widgets:

The grid layout is configured at [tt_tools_common/ui_common/widgets.py 259-264](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L259-L264) with a 2-column structure where the label spans both columns.

### Event Handling

The button press handler at [tt_tools_common/ui_common/widgets.py 266-271](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L266-L271) implements the following logic:

1.   Check which button was pressed via `event.button.id`
2.   If "yes" and `on_yes` callback exists, execute it
3.   If "no" and `on_no` callback exists, execute it
4.   Always pop the screen to dismiss the dialog

The modal styling is defined at [tt_tools_common/ui_common/common_style.css 105-138](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L105-L138):

### Key Bindings

Three key bindings provide identical dismiss functionality:

## TTSettingsMenu: Application Settings Dialog

### Overview

`TTSettingsMenu` provides an interactive settings dialog with toggle switches for application configuration. It dynamically reads and updates application state using Switch widgets.

### Class Definition

The class is defined at [tt_tools_common/ui_common/widgets.py 308-365](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L308-L365):

`class TTSettingsMenu(ModalScreen):    BINDINGS = [        ("q, Q, s, S", "quit", "Quit"),        ("escape", "esc_screen", "Escape the open screen"),    ]        def __init__(self, text: str, theme: OrderedDict = {}) -> None`
**Parameters:**

| Parameter | Type | Description |
| --- | --- | --- |
| `text` | `str` | Text parameter (currently unused in implementation) |
| `theme` | `OrderedDict` | Theme dictionary for styling text labels |

### Settings Configuration

The settings menu is specifically designed for tt-smi with two configurable options:

The composition at [tt_tools_common/ui_common/widgets.py 321-332](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L321-L332) creates a grid with paired labels and switches for each setting.

### Switch Event Handling

The event handler at [tt_tools_common/ui_common/widgets.py 334-348](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L334-L348) uses Textual's `@on` decorator to handle `Switch.Changed` events:

`@on(Switch.Changed)def select_changed(self, event: Switch.Changed) -> None:    if event.switch.id == "dark_switch":        self.app.dark = event.switch.value    if event.switch.id == "sw_switch":        self.app.get_latest_sw_vers = event.switch.value`
**Event Flow:**

1.   User toggles a switch via keyboard (Tab/Enter) or mouse click
2.   `Switch.Changed` event is emitted
3.   Handler checks `event.switch.id` to identify which switch changed
4.   Corresponding app property is updated with `event.switch.value` (boolean)
5.   Application state persists across screen dismissals

### Initial Switch State

Switches are initialized with current application state at [tt_tools_common/ui_common/widgets.py 328-330](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L328-L330):

`Switch(value=self.app.dark, id="dark_switch")Switch(value=self.app.get_latest_sw_vers, id="sw_switch")`
This ensures the UI reflects the current configuration when the modal opens.

### Key Bindings

Multiple keys dismiss the settings menu:

| Keys | Action | Description |
| --- | --- | --- |
| `q`, `Q`, `s`, `S` | `action_quit` | Return to main app |
| `escape` | `action_esc_screen` | Escape the modal |

All dismissal actions at [tt_tools_common/ui_common/widgets.py 350-364](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L350-L364) call `self.app.pop_screen()`. Note that settings changes are applied immediately on switch toggle, not on modal dismiss.

### CSS Styling

Switch styling is defined at [tt_tools_common/ui_common/common_style.css 208-217](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/common_style.css#L208-L217):

| CSS Selector | Properties | Purpose |
| --- | --- | --- |
| `Switch>.switch--slider` | `color: $red`, `background: red 12%` | Off state (red) |
| `Switch.-on>.switch--slider` | `color: $green`, `background: $green 15%` | On state (green) |

The grid layout styling (not explicitly defined) inherits from default Grid container styles.

## Design Considerations and Limitations

### Current Implementation Notes

1.   **Theme Dictionary Usage**: Both `TTHelperMenuBox` and `TTSettingsMenu` accept a `theme` parameter, but:

    *   `TTHelperMenuBox` does not use it [tt_tools_common/ui_common/widgets.py 283-289](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L283-L289)
    *   `TTSettingsMenu` uses it only for label styling [tt_tools_common/ui_common/widgets.py 325-326](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L325-L326)

2.   **Duplicate Method Definitions**: `TTHelperMenuBox.action_quit` is defined twice [tt_tools_common/ui_common/widgets.py 291-301](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L291-L301) which may indicate a copy-paste artifact.

3.   **Hard-Coded Settings**: `TTSettingsMenu` is specifically designed for tt-smi with fixed settings options. To use in other applications, you would need to modify the compose method or create a subclass.

4.   **No Border Titles**: `TTConfirmBox` has a TODO comment indicating border titles are not yet implemented [tt_tools_common/ui_common/widgets.py 256](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L256-L256)

### CSS Customization Points

All modal screens can be styled through CSS by targeting:

*   The class name (e.g., `TTConfirmBox`)
*   Component IDs (e.g., `#dialog`, `#help_menu_box`, `#settings_menu_box`)
*   Child widget selectors (e.g., `Button`, `Switch`, `MarkdownH1`)

For comprehensive styling information, see [Theming and Styling](https://deepwiki.com/tenstorrent/tt-tools-common/4.3-theming-and-styling).

### Extensibility

To create custom modal dialogs:

1.   Inherit from `ModalScreen`
2.   Override `compose()` to define layout
3.   Implement action methods for key bindings
4.   Call `self.app.pop_screen()` to dismiss
5.   Access parent app via `self.app` for state management
