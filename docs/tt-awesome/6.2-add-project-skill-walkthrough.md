The `add-project` skill is a specialized Claude Code plugin designed to streamline the contribution process for the Awesome Tenstorrent ecosystem. It provides a conversational interface that automates metadata gathering, validates entry data against the project's schema, and offers multiple submission paths ranging from low-friction GitHub issues to automated Pull Requests.

## Phase 1: Field Gathering & Automation

The skill follows a strict "one question at a time" protocol to prevent overwhelming the contributor `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L24-L26" min=24 max=26 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### Argument Handling

The skill accepts an optional argument hint: `[project name or URL]``<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L4-L4" min=4  file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

*   If a URL is provided, it is treated as the `repo_url`, and the skill skips the manual URL question `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L17-L18" min=17 max=18 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.
*   If non-URL text is provided, it pre-fills the `name` field `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L18-L19" min=18 max=19 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### GitHub API Auto-Fetch

Once a GitHub URL is obtained, the skill uses the `gh` CLI to fetch repository metadata `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L78-L82" min=78 max=82 file-path="skills/add-project/SKILL.md">Hii</FileRef>`:

`gh api repos/{owner}/{repo} --jq '{description:.description, language:.language, license:.license.spdx_id}'`
This data is used to pre-fill the `description`, `language`, and `license` fields, which the user then confirms or overrides `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L38-L40" min=38 max=40 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### ID Generation Logic

To ensure consistency with the automated `scripts/issue_to_entry.py` utility, the skill derives the entry ID from the **Project Name** using a specific slugification algorithm `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L88-L97" min=88 max=97 file-path="skills/add-project/SKILL.md">Hii</FileRef>`:

1.   Convert to lowercase.
2.   Strip non-word characters (except hyphens).
3.   Replace whitespace/underscores with `-`.
4.   Collapse consecutive hyphens.

**Sources:**`<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L22-L97" min=22 max=97 file-path="skills/add-project/SKILL.md">Hii</FileRef>`, `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-05-27-add-project-skill-design.md?plain=1#L75-L95" min=75 max=95 file-path="docs/superpowers/specs/2026-05-27-add-project-skill-design.md">Hii</FileRef>`.

The skill enters a loop, asking: _"Does this look right? Any fields to change?"_`<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L128-L130" min=128 max=130 file-path="skills/add-project/SKILL.md">Hii</FileRef>`. ## Phase 3: Submission Paths

The skill supports three distinct "exit paths" based on the user's environment and permissions `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L134-L151" min=134 max=151 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### Path A: GitHub Issue (No local repo needed)

This is the recommended path for most contributors `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/CONTRIBUTING.md?plain=1#L7-L11" min=7 max=11 file-path="CONTRIBUTING.md">Hii</FileRef>`. The skill constructs a body using the `### Heading` format expected by the project's issue template `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/.github/ISSUE_TEMPLATE/submit-entry.yml#L1-L131" min=1 max=131 file-path=".github/ISSUE_TEMPLATE/submit-entry.yml">Hii</FileRef>`.

The skill executes:

`gh issue create --repo tenstorrent/tt-awesome --title "[Entry] {name}" --label submission --body "{body}"`
This triggers the `issue_to_entry.py` automation once a maintainer adds the `approved` label `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L203-L210" min=203 max=210 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### Path B: Local JSON (Local checkout required)

The skill writes the JSON to the appropriate subdirectory: `entries/{primary_category}/{id}.json``<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L216-L216" min=216  file-path="skills/add-project/SKILL.md">Hii</FileRef>`. It then runs `python3 scripts/validate.py` to ensure schema compliance `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L217-L217" min=217  file-path="skills/add-project/SKILL.md">Hii</FileRef>`. If validation passes, it provides the user with the exact `git` commands needed to branch, commit, and push `<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L221-L230" min=221 max=230 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### Path C: JSON + Auto PR (Maintainer/Push access)

This path performs all actions in Path B but automates the Git workflow and Pull Request creation using `gh pr create``<FileRef file-url="https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/skills/add-project/SKILL.md?plain=1#L146-L147" min=146 max=147 file-path="skills/add-project/SKILL.md">Hii</FileRef>`.

### Allowed Tools

The skill is restricted to the following Claude Code tools to ensure security and focused operation:

| Tool | Purpose |
| --- | --- |
| `Bash` | Running `gh` CLI and `scripts/validate.py` |
| `Read` | Reading existing entries or category definitions |
| `Write` | Writing the local `.json` file in Path B/C |
