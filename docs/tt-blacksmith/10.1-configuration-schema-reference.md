This page provides complete reference documentation for the `TrainingConfig` Pydantic models used throughout TT-Blacksmith. These configuration models define all parameters for training experiments, including dataset selection, model architecture, training hyperparameters, logging, checkpointing, device management, and parallelization strategies.

For information about how configuration files are loaded and merged at runtime, see [Configuration System](https://deepwiki.com/tenstorrent/tt-blacksmith/4-configuration-system). For guidance on creating experiment configurations, see [Experiments](https://deepwiki.com/tenstorrent/tt-blacksmith/6-experiments). For device-specific configuration details, see [Device Management and Hardware Abstraction](https://deepwiki.com/tenstorrent/tt-blacksmith/5.2-device-management-and-hardware-abstraction).

TT-Blacksmith uses Pydantic `BaseModel` classes to define strongly-typed configuration schemas. Each experiment framework has its own `TrainingConfig` class that extends this pattern with framework-specific fields.

## Configuration Validation and Type Safety

TT-Blacksmith leverages Pydantic's validation system to ensure type safety and constraint enforcement at configuration load time.

**Configuration Validation Pipeline**

**Common Validation Constraints:**

*   `gt=0`: Greater than zero (e.g., `batch_size`, `learning_rate`)
*   `ge=0`: Greater than or equal to zero (e.g., `weight_decay`, `keep_last_n`)
*   `lt=1`: Less than one (e.g., `train_ratio`)
*   `le=1`: Less than or equal to one (e.g., `dropout_rate`)

**Example Validation Error:**

`# Invalid configurationconfig = TrainingConfig(    batch_size=-1,  # Fails gt=0 constraint    learning_rate=0  # Fails gt=0 constraint)# Raises: ValidationError: 2 validation errors for TrainingConfig#   batch_size: ensure this value is greater than 0#   learning_rate: ensure this value is greater than 0`
Sources: [blacksmith/experiments/torch/llama/configs.py 11-93](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/llama/configs.py#L11-L93)[blacksmith/experiments/torch/mnist/configs.py 12-93](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/torch/mnist/configs.py#L12-L93)

## TestConfig Schema

The optional `TestConfig` nested model enables test-specific configuration overrides for pytest integration:

`class TestConfig(BaseModel):    max_steps_per_epoch: Optional[int] = None    test_mode: bool = True    golden_file_path: Optional[str] = None    loss_tolerance: float = 0.1`
When present in the main configuration, `TestConfig` allows test scripts to:

*   Limit training steps for faster test execution
*   Enable test-specific behaviors
*   Specify golden file paths for validation
*   Configure loss comparison tolerances

**Usage Example:**

`# In test configuration override filetest_config:  max_steps_per_epoch: 10  test_mode: true  golden_file_path: "tests/golden/mnist_loss.csv"  loss_tolerance: 0.05`

## Configuration File Loading

Configuration files are loaded and merged using the `generate_config()` function:

`from blacksmith.tools.cli import generate_config, parse_cli_options # Parse CLI argumentsargs = parse_cli_options(default_config="path/to/default.yaml") # Generate and validate configurationconfig: TrainingConfig = generate_config(    TrainingConfig,    args.config,         # Main config path    args.test_config     # Optional test override path)`
**Merging Priority:**

1.   Default values defined in Pydantic model
2.   Main YAML configuration file (`--config`)
3.   Test override YAML file (`--test-config`)
4.   CLI argument overrides (if implemented)
