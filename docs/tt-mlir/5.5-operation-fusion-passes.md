This page documents the operation fusion passes in `tt-mlir`: `TTIRFusing`, `TTNNFusing`, and specialized patterns for high-performance kernels like SDPA, RoPE, and TopK. These passes recognize sequences of operations that can be collapsed into single hardware-optimized ops or attributes, reducing intermediate tensor materialization and improving data locality.

* * *

## Where Fusion Fits in the Pipeline

Fusion and canonicalization happen at multiple stages in the compilation pipeline:

1.   **TTIR Fusing**: Early in the pipeline, high-level ops are fused to simplify the graph before decomposition. This pass is often run multiple times to catch opportunities exposed by other transformations [lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 27-28](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L27-L28)
2.   **TTIR Erase Inverse Ops**: A specialized pass that commutes tensor manipulation (TM) ops (like `Reshape` or `Permute`) to find and erase inverse pairs.
3.   **TTNN Fusion**: Applied after conversion to the TTNN dialect. This stage leverages hardware-specific "fused" attributes (e.g., embedding activations into Matmul or Conv ops) and complex pattern matching for optimized kernels like SDPA, RoPE, and TopK [lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 12-21](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L12-L21)

## TTIRFusing Pass

**Pass name:**`ttir-fusing`**Implementation:**[lib/Dialect/TTIR/Transforms/TTIRFusing.cpp](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp)

This pass uses the `GreedyPatternRewriteDriver`[lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 19](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L19-L19) to apply a suite of patterns. It recognizes high-level mathematical identities and hardware-friendly combinations before the graph is lowered to device-specific dialects.

### Key Fusion Patterns

#### ConvAddBias

**File:**[lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 38-128](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L38-L128) Anchors on `AddOp`. It detects if one operand is a convolution (`Conv2dOp`, `ConvTranspose2dOp`, `Conv3dOp`) and the other is a bias tensor.

*   **Logic**: Supports adding bias to a conv that has no bias, or accumulating into an existing bias [lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 55-59](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L55-L59)
*   **Constraints**: Validates that the bias shape matches the convolution's output feature dimension [lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 92-109](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L92-L109)

#### ReductionWithReshapePattern

**File:**[lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 135-160](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L135-L160) Detects when a reduction (e.g., `SumOp`, `MaxOp`) is followed by a `ReshapeOp` that simply adds back dimensions that were reduced (size-1).

*   **Optimization**: Fuses the reshape into the reduction by setting the `keep_dim` attribute to `true`[lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 153-155](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L153-L155)

#### ConvMultiplyCommute

**File:**[test/ttmlir/Dialect/TTIR/fusing/conv2d_multiply_commute.mlir 4-14](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/ttmlir/Dialect/TTIR/fusing/conv2d_multiply_commute.mlir#L4-L14) Matches a `Conv2dOp` followed by a `MultiplyOp` by a constant scale. It commutes the multiply before the convolution by scaling the weights (and bias if present), allowing for further fusion or constant folding [test/ttmlir/Dialect/TTIR/fusing/conv2d_multiply_commute.mlir 31-36](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/ttmlir/Dialect/TTIR/fusing/conv2d_multiply_commute.mlir#L31-L36)

Sources: [lib/Dialect/TTIR/Transforms/TTIRFusing.cpp 38-160](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTIR/Transforms/TTIRFusing.cpp#L38-L160)[test/ttmlir/Dialect/TTIR/fusing/conv2d_multiply_commute.mlir 4-48](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/ttmlir/Dialect/TTIR/fusing/conv2d_multiply_commute.mlir#L4-L48)

## TTNNFusing Pass

**Pass name:**`ttnn-fusing`**Implementation:**[lib/Dialect/TTNN/Transforms/TTNNFusing.cpp](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp)

At the TTNN level, fusion utilizes hardware-specific attributes and optimized kernels.

### Activation Fusion

Matches `Conv2dOp`, `MatmulOp`, or `LinearOp` followed by an activation (e.g., `ReluOp`).

*   **Conv2d Fusion**: Detects `ActivationOp` following a convolution and updates the `Conv2dConfigAttr` with the activation type [lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 48-56](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L48-L56) It also handles cases where a `ReshapeOp` exists between the conv and activation [lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 111-121](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L111-L121)
*   **Matmul/Linear Fusion**: Sets the `activation` attribute on the op (e.g., via `setActivationAttr`) and removes the standalone activation [lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 142-146](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L142-L146)

#### SDPA Fusing

*   **Logic**: It performs backtracking matching to identify `Query`, `Key`, `Value`, and optional `Mask` inputs [lib/Dialect/TTNN/Transforms/Fusing/SDPAFusingPattern.cpp 34-41](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/Fusing/SDPAFusingPattern.cpp#L34-L41)
*   **Layouts**: Handles permutations required for `SDPA` decode kernels (e.g., `[S, B, H, D]` format) [lib/Dialect/TTNN/Transforms/Fusing/SDPAFusingPattern.cpp 26-28](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/Fusing/SDPAFusingPattern.cpp#L26-L28)

## Specialized Decomposition and Workarounds

Fusion often interacts with decomposition workarounds to handle hardware constraints.

### NLP Concat Heads Decode

**File:**[lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 197-214](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L197-L214) Matches the decode-phase concat-heads pattern in LLMs.

*   **Pattern**: Matches a `Permute([1, 2, 0, 3])` (converting `[S, B, H, D] -> [B, H, S, D]`) followed by a `Reshape` that collapses the head and head-dim dimensions [lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 187-188](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L187-L188)
*   **Replacement**: Replaces the sequence with the optimized hardware op `nlp_concat_heads_decode`[lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 191-196](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L191-L196)

### SDPA Padding Workaround

**File:**[test/ttmlir/Dialect/TTNN/Transforms/Workarounds/sdpa_pad_sequence_dim_workaround.mlir 71-103](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/ttmlir/Dialect/TTNN/Transforms/Workarounds/sdpa_pad_sequence_dim_workaround.mlir#L71-L103) Hardware SDPA kernels may have alignment requirements for the `head_dim`.

*   **Workaround**: If the `head_dim` is unaligned (e.g., 48), the pass inserts `ttnn.pad` for Query, Key, and Value tensors to a supported size (e.g., 64) and a `ttnn.slice_static` on the result to restore the original shape [test/ttmlir/Dialect/TTNN/Transforms/Workarounds/sdpa_pad_sequence_dim_workaround.mlir 79-94](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/ttmlir/Dialect/TTNN/Transforms/Workarounds/sdpa_pad_sequence_dim_workaround.mlir#L79-L94)

Sources: [lib/Dialect/TTNN/Transforms/TTNNFusing.cpp 180-214](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/lib/Dialect/TTNN/Transforms/TTNNFusing.cpp#L180-L214)[test/ttmlir/Dialect/TTNN/Transforms/Workarounds/sdpa_pad_sequence_dim_workaround.mlir 71-103](https://github.com/tenstorrent/tt-mlir/blob/c7d92e92/test/ttmlir/Dialect/TTNN/Transforms/Workarounds/sdpa_pad_sequence_dim_workaround.mlir#L71-L103)
