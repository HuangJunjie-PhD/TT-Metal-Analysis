The Telemetry Tab provides real-time monitoring of hardware telemetry metrics for all detected Tenstorrent devices. This tab displays voltage, current, power, temperature, clock frequency, fan speed, and heartbeat status with automatic updates every 100 milliseconds. For information about static device properties, see [Device Information Tab](https://deepwiki.com/tenstorrent/tt-smi/3.2-device-information-tab). For firmware version information, see [Firmware Version Tab](https://deepwiki.com/tenstorrent/tt-smi/3.4-firmware-version-tab).

## Overview

The Telemetry Tab is the second tab in the TT-SMI TUI, accessible via the "2" keyboard shortcut or by clicking "Telemetry (2)" in the tab bar. It presents a data table where each row represents a device and each column displays a specific telemetry metric. Values are color-coded to indicate whether they are within safe operating limits, with current readings displayed alongside maximum thresholds where applicable.

## Real-Time Update Mechanism

The Telemetry Tab implements a background thread-based update system that activates only when the tab is visible. This design prevents unnecessary hardware polling when users are viewing other tabs.

### Thread Lifecycle

The telemetry update thread is managed through the `dispatch_telem_thread()` method and controlled by global flags:

| Global Variable | Type | Purpose |
| --- | --- | --- |
| `INTERRUPT_RECEIVED` | bool | Signals thread termination when user quits |
| `TELEM_THREADS` | List[Thread] | Tracks active telemetry threads |
| `RUNNING_TELEM_THREAD` | bool | Prevents duplicate thread creation |

## Platform-Specific Telemetry Collection

### Wormhole Telemetry Extraction

Wormhole chips use SMBus-based telemetry with specific bit field extractions:

| Metric | Source Field | Extraction Logic | Units |
| --- | --- | --- | --- |
| Voltage | `VCORE` | `int(hex_value, 16) / 1000` | Volts (V) |
| Current | `TDC` | `int(hex_value, 16) & 0xFFFF` | Amperes (A) |
| Power | `TDP` | `int(hex_value, 16) & 0xFFFF` | Watts (W) |
| Temperature | `ASIC_TEMPERATURE` | `(int(hex_value, 16) & 0xFFFF) / 16` | Celsius (Â°C) |
| AICLK | `AICLK` | `int(hex_value, 16) & 0xFFFF` | MHz |
| Fan Speed | `FAN_SPEED` | `int(hex_value, 16)` | Percentage (%) |
| Heartbeat | `ARC3_HEALTH` | `int(hex_value, 16) // 5` | Counter (Ă·5) |

### Blackhole Telemetry Extraction

Blackhole chips use TAG-based telemetry with different data formats:

| Metric | Source Field | Extraction Logic | Units |
| --- | --- | --- | --- |
| Voltage | `VCORE` | `int(hex_value, 16) / 1000` | Volts (V) |
| Current | `TDC` | `int(hex_value, 16) & 0xFFFF` | Amperes (A) |
| Power | `TDP` | `int(hex_value, 16) & 0xFFFF` | Watts (W) |
| Temperature | `ASIC_TEMPERATURE` | `convert_signed_16_16_to_float(int(hex_value, 16))` | Celsius (Â°C) |
| AICLK | `AICLK` | `int(hex_value, 16) & 0xFFFF` | MHz |
| Fan Speed | `FAN_RPM` | `min(int(hex_value, 16), 5000) / 50` | Percentage (%) |
| Heartbeat | `TIMER_HEARTBEAT` | `int(hex_value, 16) // 6` | Counter (Ă·6) |

## Color Coding and Alert System

The Telemetry Tab uses color coding to provide visual feedback on whether metrics are within safe operating ranges. The `format_telemetry_rows()` method applies styling based on comparisons with chip limits.

### Color Scheme

| Style | Rich Theme Key | Meaning | Applied When |
| --- | --- | --- | --- |

| Yellow | `yellow_bold` | Limit indicator | Always for max values |
| Attention | `attention` | Warning/Over limit | Value >= Limit |
| Gray | `gray` | Not available/Invalid | Value is "N/A" or out of valid range |

## Chip Limits Extraction

Chip limits define the maximum safe operating values for each telemetry metric. These are read from device telemetry structures during initialization.

### Wormhole Limits

Wormhole limits are extracted from packed bitfields in the telemetry structure:

| Limit Field | Source | Bit Extraction | Format |
| --- | --- | --- | --- |
| `vdd_min` | `VDD_LIMITS` | `(value & 0xFFFF) / 1000` | `{value:4.2f}` |
| `vdd_max` | `VDD_LIMITS` | `(value >> 16) / 1000` | `{value:4.2f}` |
| `tdp_limit` | `TDP` | `value >> 16` | `{value:3.0f}` |
| `tdc_limit` | `TDC` | `value >> 16` | `{value:3.0f}` |
| `asic_fmax` | `AICLK` | `value >> 16` | `{value:4.0f}` |
| `therm_trip_l1_limit` | `THM_LIMITS` | `value >> 16` | `{value:2.0f}` |
| `thm_limit` | `THM_LIMITS` | `value & 0xFFFF` | `{value:2.0f}` |

**Sources:**[tt_smi/tt_smi_backend.py 528-556](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L528-L556)

### Blackhole Limits

Blackhole limits are stored in dedicated telemetry fields rather than packed bitfields:

| Limit Field | Source | Extraction | Format |
| --- | --- | --- | --- |
| `vdd_min` | `VDD_LIMITS` | `(value & 0xFFFF) / 1000` | `{value:4.2f}` |
| `vdd_max` | `VDD_LIMITS` | `(value >> 16) / 1000` | `{value:4.2f}` |
| `tdp_limit` | `TDP_LIMIT_MAX` | `int(value, 16)` | `{value:3.0f}` |
| `tdc_limit` | `TDC_LIMIT_MAX` | `int(value, 16)` | `{value:3.0f}` |
| `asic_fmax` | `AICLK_LIMIT_MAX` | `int(value, 16)` | `{value:4.0f}` |
| `therm_trip_l1_limit` | `THM_LIMIT_THROTTLE` | `int(value, 16)` | `{value:2.0f}` |
| `thm_limit` | `THM_LIMITS` | `int(value, 16)` | `{value:2.0f}` |

**Sources:**[tt_smi/tt_smi_backend.py 558-586](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L558-L586)

## Tab Activation and Deactivation

The telemetry thread is only active when the Telemetry Tab is visible. Switching to other tabs does not stop the thread, but the `update_telem_table()` method catches `NoMatches` exceptions when the table widget is not in the current view:

This design ensures that the thread continues running even when switching tabs or opening the help menu, preventing the need to restart the thread each time the user returns to the Telemetry Tab.

**Sources:**[tt_smi/tt_smi.py 166-176](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi.py#L166-L176)

## Keyboard Shortcuts

The Telemetry Tab can be accessed using the following keyboard shortcuts:

| Action | Key | Description |
| --- | --- | --- |
| Navigate to Telemetry Tab | `2` | Switches to the Telemetry tab (action_tab_two) |

When the user presses "2" or clicks the tab, the `action_tab_two()` method sets the active tab to "tab-2", which triggers the `on_tabbed_content_tab_activated()` event handler to start the telemetry thread if not already running.
