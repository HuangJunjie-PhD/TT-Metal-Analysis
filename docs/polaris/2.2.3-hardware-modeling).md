## Purpose and Scope

This document covers the hardware modeling subsystem within the TTSIM Framework, which defines and manages hardware specifications for simulation. The hardware modeling system provides a structured approach to specify IP blocks, memory subsystems, and complete hardware packages that can be used in performance simulations.

For information about how these hardware models are used in workload simulation, see [Operations and Graph Construction](https://deepwiki.com/tenstorrent/polaris/2.2.1-operations-and-graph-construction). For configuration management of these hardware specifications, see [Configuration Management](https://deepwiki.com/tenstorrent/polaris/2.3-configuration-management).

## IP Block Architecture

The hardware modeling system is built around the concept of IP blocks, which are fundamental hardware components that can be either compute or memory units. Each IP block is defined with specific performance characteristics and capabilities.

## Package Configuration

Hardware packages combine multiple IP blocks into complete system configurations. Each package contains multiple instances representing different SKUs or configurations.

### Package Structure

**Sources:**[config/tt_wh.yaml 44-69](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/tt_wh.yaml#L44-L69)

### IP Group Configuration

Within each package instance, IP blocks are organized into groups that specify the number of units and any overrides:

| Property | Description | Example |
| --- | --- | --- |
| `ipname` | Reference to IP block definition | `tensix` |
| `iptype` | Type of IP block | `compute`, `memory` |
| `num_units` | Number of instances | `72`, `128` |
| `freq_MHz` | Operating frequency override | `1200` |
| `ramp_penalty` | Performance ramp penalty | `50` |
| `ip_overrides` | Specific property overrides | `pipes.matrix.freq_MHz: 1000` |

**Sources:**[config/all_archs.yaml 95-127](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/all_archs.yaml#L95-L127)[config/tt_wh.yaml 46-69](https://github.com/tenstorrent/polaris/blob/ac6ed4a3/config/tt_wh.yaml#L46-L69)

## Unit Conversion Support

The hardware modeling system includes utilities for converting between different units commonly used in performance specifications:

| Unit Category | Supported Units | Conversion Support |
| --- | --- | --- |
| Frequency | `HZ`, `KHZ`, `MHZ`, `GHZ`, `THZ` | Decimal scaling (1000x) |
| Memory | `B`, `KB`, `MB`, `GB`, `TB` | Binary scaling (1024x) |
| Operations | `OPS`, `KOPS`, `MOPS`, `GOPS`, `TOPS` | Decimal scaling (1000x) |
| Floating Point | `FLOPS`, `KFLOPS`, `MFLOPS`, `GFLOPS`, `TFLOPS` | Decimal scaling (1000x) |
