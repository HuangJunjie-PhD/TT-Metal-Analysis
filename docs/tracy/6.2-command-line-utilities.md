This document covers Tracy's command-line utilities for capturing, processing, and managing profiling data outside of the main GUI profiler application. These tools enable automated profiling workflows, continuous integration scenarios, and batch processing of trace data.

For information about the main Tracy profiler GUI application, see [Profiler Application](https://deepwiki.com/tenstorrent/tracy/4-profiler-application). For details on file formats and data import/export, see [Import & Export Tools](https://deepwiki.com/tenstorrent/tracy/5.2-import-and-export-tools).

## Overview

Tracy provides several command-line utilities that complement the main profiler application:

*   **Capture utility** - Connects to running applications and saves profiling data to `.tracy` files
*   **Update utility** - Processes and converts Tracy dump files between versions
*   **Import utilities** - Convert external trace formats (Chrome, Fuchsia) to Tracy format
*   **Export utilities** - Extract data from Tracy files to other formats (CSV, etc.)

These utilities enable headless profiling scenarios, automated testing, and integration with build systems and CI/CD pipelines.

## Capture Utility

The capture utility (`capture`) is the primary command-line tool for collecting profiling data from instrumented applications. It connects to Tracy clients over the network and saves the profiling session to a `.tracy` file.

### Architecture

Sources: [capture/src/capture.cpp 1-354](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp#L1-L354)

The capture utility acts as a lightweight Tracy server that connects to instrumented applications, processes the profiling data stream, and writes compressed trace files to disk.

### Command-Line Interface

Sources: [capture/src/capture.cpp 97-140](https://github.com/tenstorrent/tracy/blob/68c5cbd6/capture/src/capture.cpp#L97-L140)

### Usage and Options

The capture utility supports the following command-line options:

| Option | Description | Default |
| --- | --- | --- |
| `-a address` | Target application IP address | `127.0.0.1` |
| `-o output.tracy` | Output file path | Required |
| `-p port` | Network port for Tracy protocol | `8086` |
| `-f` | Force overwrite existing output file | Off |
| `-s seconds` | Time limit for capture session | Unlimited |

**Example usage:**

`# Capture from local applicationcapture -o profile.tracy # Capture from remote application with time limitcapture -a 192.168.1.100 -o remote_profile.tracy -s 30 # Force overwrite existing filecapture -o existing_profile.tracy -f`

### Error Handling

The utility handles various connection and instrumentation failures:

*   **Protocol mismatch** - Version incompatibility between client and capture tool
*   **Client unavailable** - Target application already connected to another server
*   **Connection dropped** - Network issues during handshake
*   **Instrumentation failures** - Runtime errors in the target application

## Other Command-Line Utilities

Based on the Tracy ecosystem architecture, additional command-line utilities include:

### Update Utility

*   Converts Tracy files between different format versions
*   Handles legacy file format migration
*   Supports batch processing of multiple files

### Import Utilities

*   **Chrome trace import** - Converts Chrome DevTools traces to Tracy format
*   **Fuchsia trace import** - Processes Fuchsia system traces
*   **External format converters** - Handle various profiling data formats

### Export Utilities

*   **CSV export** - Extracts timing and statistical data
*   **Statistics export** - Generates performance reports
*   **Custom format export** - Supports integration with external analysis tools
