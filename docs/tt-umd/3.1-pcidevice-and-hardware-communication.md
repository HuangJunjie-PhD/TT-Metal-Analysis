## Purpose and Scope

This document covers the `PCIDevice` class and its role as the lowest-level hardware communication layer in the `tt-umd` stack. `PCIDevice` provides direct access to PCIe-attached Tenstorrent devices through memory-mapped I/O (MMIO), DMA operations, and kernel driver ioctls. This class handles device enumeration, BAR (Base Address Register) mapping, memory pinning, and IOMMU management.

For higher-level device abstraction and architecture-specific operations, see [TTDevice Interface](https://deepwiki.com/tenstorrent/tt-umd/2.4-ttdevice-interface). For TLB-based memory access patterns, see [Memory Management Systems](https://deepwiki.com/tenstorrent/tt-umd/3.2-memory-management-systems).

### PciDeviceInfo Structure

The `PciDeviceInfo` structure captures essential device metadata [device/api/umd/device/pcie/pci_device.hpp 33-52](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/device/api/umd/device/pcie/pci_device.hpp#L33-L52):

*   `vendor_id`: PCI vendor ID (Tenstorrent).
*   `device_id`: `0x401e` (WH) or `0xb140` (BH).

*   `pci_bdf`: String representation `"{domain}:{bus}:{device}.{function}"`.
*   `physical_slot`: Physical slot number derived from sysfs [device/pcie/pci_device.cpp 130-171](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/device/pcie/pci_device.cpp#L130-L171)

## Warm Reset Lifecycle

Warm reset is a critical lifecycle operation that coordinates between the User Mode Driver, Kernel Driver, and Hardware (ARC processor).

### Reset Communication (IPC)

The `WarmResetCommunication` system uses Unix Domain Sockets in `/tmp/tt_umd_listeners` to synchronize reset events across multiple processes [device/api/umd/device/warm_reset.hpp 77-100](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/device/api/umd/device/warm_reset.hpp#L77-L100)

## DMA and Hugepage Allocation

`PCIDevice` manages memory pinning for high-performance DMA transfers.

*   **Hugepage Allocation**: Handled via `map_for_hugepage()`. Requires the memory to be page-aligned [device/pcie/pci_device.cpp 465-492](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/device/pcie/pci_device.cpp#L465-L492)
*   **DMA Buffers**: `allocate_pcie_dma_buffer()` creates a contiguous buffer and a completion page for status tracking [device/pcie/pci_device.cpp 717-778](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/device/pcie/pci_device.cpp#L717-L778)
*   **IOMMU**: If IOMMU is enabled, the driver uses I/O Virtual Addresses (IOVA). If disabled, `TENSTORRENT_PIN_PAGES_CONTIGUOUS` is mandatory for multi-page buffers [device/pcie/pci_device.cpp 508-510](https://github.com/tenstorrent/tt-umd/blob/9bd70c7a/device/pcie/pci_device.cpp#L508-L510)
