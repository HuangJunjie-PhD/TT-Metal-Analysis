This page provides regulatory compliance statements and technical standards for Tenstorrent hardware products. These statements cover electromagnetic compatibility (EMC), radio frequency interference, and hazardous substance restrictions across major global markets including the United States, Canada, Europe, the United Kingdom, and South Korea.

The compliance framework applies to all current and legacy Tenstorrent Add-In Board (AIB) models, including the Grayskull™, Wormhole™, and Blackhole™ processor families [core/aibs/compliance.md 9-12](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/aibs/compliance.md?plain=1#L9-L12)

## Detailed Compliance Statements

### United States (FCC)

This equipment has been tested and found to comply with the limits for a **Class B** digital device, pursuant to Part 15 of the FCC Rules [core/aibs/compliance.md 29-35](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/aibs/compliance.md?plain=1#L29-L35) These limits are designed to provide reasonable protection against harmful interference in a residential installation.

*   **Interference:** The device may not cause harmful interference and must accept any interference received [core/aibs/compliance.md 29-33](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/aibs/compliance.md?plain=1#L29-L33)
*   **Modifications:** Changes or modifications not expressly approved by Tenstorrent could void the user's authority to operate the equipment [core/aibs/compliance.md 47](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/aibs/compliance.md?plain=1#L47-L47)

### Canada (ISED)

Tenstorrent products comply with **ICES-003 Issue 7, CAN ICES-003(B)/NMB-003(B)**[core/aibs/compliance.md 55](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/aibs/compliance.md?plain=1#L55-L55) Operation is subject to the condition that the device does not cause interference and must accept any interference, including that which may cause undesired operation [core/aibs/compliance.md 57](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/core/aibs/compliance.md?plain=1#L57-L57)

## Software-Hardware Interface for Compliance Monitoring

While regulatory compliance is a hardware characteristic, the software stack provides utilities to monitor the physical state of the board (e.g., power consumption and thermal levels) to ensure the device operates within its certified envelopes.

**Compliance Monitoring Data Flow**

### Key Software Entities

*   `tt-kmd`: The Linux Kernel-Mode Driver responsible for low-level communication with the hardware [syseng/devdocs.md 10](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/devdocs.md?plain=1#L10-L10)
*   `tt-smi`: The System Management Interface utility used to verify system configuration and monitor device health [syseng/softwaresetup.md 65-70](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L65-L70)[syseng/devdocs.md 13](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/devdocs.md?plain=1#L13-L13)
*   `tt-flash`: Utility used to update `TT-System-Firmware`, ensuring the hardware runs validated and compliant control logic [syseng/softwaresetup.md 34-39](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/softwaresetup.md?plain=1#L34-L39)[syseng/devdocs.md 12](https://github.com/tenstorrent/tenstorrent.github.io/blob/eda54884/syseng/devdocs.md?plain=1#L12-L12)
