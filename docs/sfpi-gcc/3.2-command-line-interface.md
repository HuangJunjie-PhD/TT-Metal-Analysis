This document describes the command line interface for sfpi-gcc, covering the organization of command-line options, their processing mechanisms, and the major option categories available to users. The CLI system provides comprehensive control over compilation stages, optimization levels, target-specific features (including SFPU extensions), and debugging capabilities.

For information about the build system configuration, see [Build System and Configuration](https://deepwiki.com/tenstorrent/sfpi-gcc/3.1-build-system-and-configuration). For details about the optimization pipeline that these options control, see [Optimization Pipeline](https://deepwiki.com/tenstorrent/sfpi-gcc/3.3-pass-management-and-optimization-pipeline).

## Option Processing Architecture

The sfpi-gcc command line interface is built on GCC's option processing framework defined in [gcc/common.opt](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/common.opt) and related `.opt` files. Options are processed through a multi-stage pipeline involving AWK scripts, generated C code, and runtime option handlers.

### Option Definition Framework

**Option File Format and Processing Pipeline**

The `.opt` file format uses a structured syntax where each option definition includes:

*   Option flags: `Common`, `Driver`, `Joined`, `RejectNegative`, `Var(variable_name)`
*   Help text for `--help` output
*   Variable bindings (e.g., `Var(warn_unused)` in [gcc/common.opt 851-853](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/common.opt#L851-L853))
*   Integer ranges (e.g., `IntegerRange(0, 100)` in [gcc/params.opt 106-107](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/params.opt#L106-L107))

**Key Data Structures**

The `cl_decoded_option` structure (generated in options.h) represents a parsed command-line option:

*   `opt_index`: enum value identifying the option
*   `arg`: string argument (for joined or separate args)
*   `value`: integer value for options with numeric parameters
*   `errors`: validation error flags

The `gcc_options` structure (generated from Var() declarations) contains all option state variables, including those defined in [gcc/common.opt 25-230](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/common.opt#L25-L230) such as `target_flags`, `optimize`, `optimize_size`, and `flag_complex_method`.

## Major Option Categories

### Overall Options

Overall options control the fundamental behavior of the compilation process, determining which stages to execute and the type of output to produce.

| Option | Purpose | Default Behavior |
| --- | --- | --- |
| `-c` | Compile only, don't link | Produces object files |
| `-S` | Compile to assembly only | Produces `.s` files |
| `-E` | Preprocess only | Outputs to stdout |
| `-o file` | Specify output file | Default varies by stage |
| `-v` | Verbose output | Shows command execution |
| `-###` | Show commands without execution | For debugging build |

Sources: [gcc/doc/invoke.texi 70-100](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/doc/invoke.texi#L70-L100)[gcc/common.opt 456-500](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/common.opt#L456-L500)

### Optimization Options and Pass Control

**Optimization Levels and Pass Execution**

Optimization levels control which passes execute via the `gate()` function in each pass. The `optimize` variable (set by `-O` flags in [gcc/common.opt 470-488](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/common.opt#L470-L488)) is checked in pass gates.

**Individual Optimization Flags**

Individual `-f` flags control specific optimizations and are typically checked in pass `gate()` functions:

*   Inlining: `flag_inline_functions`, `flag_inline_small_functions`
*   Vectorization: `flag_tree_vectorize`, `flag_tree_slp_vectorize`
*   Loop optimization: `flag_unroll_loops`, `flag_loop_optimize`
*   LTO: `flag_lto` (controls link-time optimization)

Example from pass structure:

```
virtual bool gate (function *) {
  return optimize >= 2 && flag_tree_vectorize;
}
```

Sources: [gcc/common.opt 470-488](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/common.opt#L470-L488)[gcc/passes.def 30-469](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/passes.def#L30-L469)[gcc/tree-pass.h 30-204](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/tree-pass.h#L30-L204)

### Parameter Options

**Parameter Definition and Access**

Parameters are defined in [gcc/params.opt](https://github.com/tenstorrent/sfpi-gcc/blob/763d6834/gcc/params.opt) using the `Joined UInteger Var()` syntax. Each parameter has:

*   C variable binding (e.g., `Var(param_align_loop_iterations)`)
*   Default value (`Init(4)`)
*   Optional range constraint (`IntegerRange(1, 65536)`)
*   Optimization flag indicating if it affects optimization

**Parameter Usage in Passes**

Passes access parameters directly via the generated `param_*` global variables. For example:

*   Inlining checks `param_inline_unit_growth` and `param_early_inlining_insns`
*   Vectorizer checks `param_vect_max_*` parameters
*   IPA passes check `param_ipa_*` parameters

The parameters can be queried and modified at runtime, affecting optimization heuristics without recompilation.
