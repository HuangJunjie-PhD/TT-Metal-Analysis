# Backend System

Relevant source files
*   [tt_smi/tt_smi_backend.py](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py)

The Backend System serves as the core component of the TT-SMI architecture, providing a bridge between the user interfaces (CLI and TUI) and the Tenstorrent hardware devices. It handles device discovery, information retrieval, telemetry collection, and reset operations. This document describes the architecture and functionality of the TTSMIBackend component, which is the main class implementing the backend system functionality.

For information about the Data Models and Logging system that the backend uses, see [Data Models and Logging](https://deepwiki.com/tenstorrent/tt-smi/5.2-standard-pci-device-resets).

## Backend System Architecture

The TTSMIBackend class is designed to interact with multiple Tenstorrent devices simultaneously, providing a unified interface for the frontend components. It initializes communication with the hardware and maintains state information about all detected devices.

## Core Data Structures

The TTSMIBackend maintains several key data structures to organize device information:

| Data Structure | Purpose | Content |
| --- | --- | --- |
| `devices` | List of PciChip objects representing hardware | Direct chip interfaces |
| `device_infos` | Basic device information | Board ID, type, PCIe specs, DRAM status |
| `device_telemetrys` | Real-time operational metrics | Voltage, current, power, temperature |
| `firmware_infos` | Firmware version information | CM firmware, Ethernet firmware, bootloader versions |
| `chip_limits` | Operational constraints | Voltage limits, TDP/TDC limits, thermal limits |
| `pci_properties` | PCIe connection details | Link speed, link width, max capabilities |
| `smbus_telem_info` | Raw SMBus telemetry data | Register values from hardware |

## Telemetry Collection

One of the primary functions of the Backend System is to collect and process telemetry data from Tenstorrent devices. The system handles different data formats for each device type:

### Device-Specific Telemetry

For all device types, the backend collects:

*   Current consumption
*   Power usage
*   ASIC temperature

## Reset Functionality

The Backend System provides specialized reset operations for different device types:

### Grayskull Tensix Reset

For Grayskull devices, the backend provides a Tensix core reset operation that resets the computation cores while preserving the PCIe connection.

### Wormhole and Blackhole Board Reset

For Wormhole and Blackhole devices, the backend supports board-level resets that can reinitialize the entire device.

### UBB Reset

For Wormhole devices in UBB (Universal Baseboard) configurations, the backend provides a special reset procedure that coordinates reset across multiple boards.

## Code Structure and Implementation Details

The Backend System is primarily implemented in two files:

1.   `tt_smi_backend.py` - Contains the TTSMIBackend class and related reset functions
2.   `tt_smi.py` - Contains the frontend implementation that uses the backend

### TTSMIBackend Class

The TTSMIBackend class is the central component of the Backend System. It maintains the state of all detected devices and provides methods to interact with them.

Key methods include:

| Method | Purpose |
| --- | --- |
| `__init__` | Initializes the backend with a list of devices and gathers information |
| `get_device_info` | Retrieves basic device information |
| `get_chip_telemetry` | Collects telemetry data from devices |
| `get_firmware_versions` | Retrieves firmware version information |
| `get_chip_limits` | Collects operational limits for devices |
| `update_telem` | Updates telemetry data for all devices |
| `save_logs_to_file` | Saves collected data to a file |
| `print_logs_to_stdout` | Prints collected data to the console |
| `gs_tensix_reset` | Performs a Tensix reset on Grayskull devices |

Sources: [tt_smi/tt_smi_backend.py 49-580](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L49-L580)

### Reset Functions

The Backend System includes several standalone functions for reset operations:

| Function | Purpose |
| --- | --- |
| `pci_indices_from_json` | Parses device indices from a configuration file |
| `mobo_reset_from_json` | Performs motherboard-level reset based on configuration |
| `pci_board_reset` | Resets boards at specific PCI indices |
| `wh_ubb_reset` | Performs a UBB reset for Wormhole devices |

These functions coordinate with the TTSMIBackend class to execute complex reset operations.

Sources: [tt_smi/tt_smi_backend.py 603-791](https://github.com/tenstorrent/tt-smi/blob/a2791bfe/tt_smi/tt_smi_backend.py#L603-L791)

## Summary

The Backend System of TT-SMI provides a comprehensive interface between the user-facing components and the Tenstorrent hardware. Its modular design handles different hardware types with specialized code paths, while presenting a unified interface to the frontend. The system's capabilities include device discovery, information retrieval, telemetry collection, and reset operations, making it the central component in the TT-SMI architecture.
