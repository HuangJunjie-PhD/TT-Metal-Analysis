This page documents how data flows through the TT-NN Visualizer system and how application state is managed. The document covers both backend and frontend state management patterns, specifically focusing on how data is requested, transferred, and maintained throughout the application lifecycle.

## Overview

The TT-NN Visualizer implements a state management system that spans both frontend and backend components:

1.   **Backend State**: Managed through Flask sessions, SQLite database, and file system
2.   **Frontend State**: Managed through Jotai atoms and React Query
3.   **Data Flow**: Connects backend and frontend through RESTful API endpoints

## Session Management

The TT-NN Visualizer uses a session management system to track the current visualization context and maintain state between requests.

### Backend Session Management

Sessions are managed through the `InstanceTable` model which stores:

*   Active report references (profiler, performance, NPE data)
*   File system paths
*   Remote connection information

Key session management functions include:

*   `get_or_create_instance()`: Retrieves or initializes a session
*   `update_instance()`: Updates session data including active reports and paths
*   `update_paths()`: Sets file paths based on active reports

The `with_session` decorator provides session context to API routes:

## Data Flow Patterns

### From Backend to Frontend

The key patterns for data flow from backend to frontend are:

1.   React components use API hooks to request data
2.   API hooks use React Query for caching and fetching
3.   Hooks use Jotai atoms to determine what to request
4.   Backend processes requests and returns data
5.   Data flows back to components through hooks

## API Hooks and Data Request Flow

The frontend uses specialized hooks to request data from the backend:

The data request flow follows this pattern:

1.   Component calls an API hook (e.g., `useOperationsList()`)
2.   Hook uses React Query to fetch and cache data
3.   Query depends on Jotai atoms for parameters
4.   Backend processes the request and returns data
5.   Data is cached and returned to the component

## Conclusion

The TT-NN Visualizer implements a sophisticated data flow and state management system that combines:

1.   **Backend Persistence**: Session management and database storage
2.   **Frontend State**: Jotai atoms for global state and React Query for data fetching
3.   **Data Synchronization**: Between remote sources, local cache, and user interface

This design enables a responsive user experience while maintaining data consistency across the application.
