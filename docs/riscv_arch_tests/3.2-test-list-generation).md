# Test List Generation

 Relevant source files

* [infra/gen\_testlist.py](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/infra/gen_testlist.py)

## Purpose and Scope

The test list generation system automatically creates standardized `.list` files from the RISC-V test directory structure. The `gen_testlist.py` script scans for ISA extension directories and generates test manifest files that specify assembly tests, execution tools, and options. These generated list files serve as input to the test execution infrastructure described in section 3.1.

## Overview

The test list generation system scans the RISC-V test directory structure and generates `.list` files for each ISA extension directory. These files contain entries for each assembly test file, specifying test names, execution tools, and options.

## Directory Structure and List Files

The test list generation system traverses a specific directory structure, focusing on ISA extension directories (those starting with `rv_`). For each such directory, it creates a corresponding `.list` file in the parent directory.

Sources: [infra/gen\_testlist.py26-33](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/infra/gen_testlist.py#L26-L33) [infra/gen\_testlist.py70-72](https://github.com/tenstorrent/riscv_arch_tests/blob/e54223b0/infra/gen_testlist.py#L70-L72)

## List File Format

Each `.list` file contains entries for assembly test files found in the corresponding directory. The entries follow a standardized format:

```
name=test_name tool=whisper test=relative/path/to/test opts=default 
```

Where:

* `name`: The test name (derived from the assembly file name without the `.S` extension)
* `tool`: The simulator to use (defaulted to "whisper")
* `test`: The relative path to the test file (without the `.S` extension)
* `opts`: Test options (defaulted to "default")

For example, a test file at `riscv_tests/bare_metal/machine/paging_bare/rv_a/amoadd_w.S` would generate an entry:

```
name=amoadd_w tool=whisper test=riscv_tests/bare_metal/machine/paging_bare/rv_a/amoadd_w opts=default 
```

## Implementation Details

The test list generation system is implemented in Python in the `gen_testlist.py` script. The script uses a set of directory and file traversal functions to locate test files and generate the appropriate list files.

### Key Functions

1. **extension\_generator**: Recursively traverses directories to find those starting with "rv\_" (RISC-V extensions).
2. **gen\_test\_list**: Generates the list file for a given extension directory.
3. **main**: Finds all extension directories and generates a test list for each.

## Integration with Test Execution

The generated `.list` files serve as inputs to the test execution infrastructure. The test runner scripts (`quals.py` and `smoke.py`) can read these files to determine which tests to run and how to run them.
