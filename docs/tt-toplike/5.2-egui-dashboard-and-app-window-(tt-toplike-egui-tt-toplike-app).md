This section covers the graphical frontends based on the `egui` framework. These include `tt-toplike-egui`, a specialized real-time monitoring dashboard, and `tt-toplike-app`, a native window wrapper that hosts the standard TUI inside a portable PTY.

## tt-toplike-egui: The K-RAD Dashboard

The `tt-toplike-egui` binary provides a lightweight, high-performance graphical dashboard. Unlike the TUI, it focuses on long-term telemetry history, multi-device comparison via plots, and a "cyberpunk" visual aesthetic.

### Architecture and Data Flow

The dashboard follows a standard `eframe` application loop. It initializes a `TelemetryBackend` via the `factory` pattern and maintains a local history buffer for plotting.

| Component | Description |
| --- | --- |
| `DashboardApp` | The main application state, holding the backend, device history, and UI theme [src/ui/egui/mod.rs 222-240](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L222-L240) |
| `DeviceHistory` | A circular buffer (using `VecDeque`) storing up to 120 samples (~2 minutes) of power, temperature, current, and voltage [src/ui/egui/mod.rs 139-155](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L139-L155) |
| `Particle` | A background animation system that simulates a "psychedelic" field with hue-cycling and velocity [src/ui/egui/mod.rs 33-41](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L33-L41) |
| `CyberpunkTheme` | Defines the neon color palette (cyan, magenta, yellow, green) used across the UI [src/ui/egui/mod.rs 114-122](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L114-L122) |

### Key Implementation Details

*   **Backend Integration**: The app uses `factory::create_backend` to instantiate the telemetry source based on CLI arguments [src/bin/egui.rs 35](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/egui.rs#L35-L35)
*   **Plotting**: Telemetry is visualized using `egui_plot`. The `DeviceHistory` struct provides `PlotPoints` helpers to convert `VecDeque` data into a format suitable for `Line` widgets [src/ui/egui/mod.rs 188-218](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L188-L218)
*   **Color Conversion**: A custom `hsv_to_rgb` function is used to generate the vibrant neon colors for particles and UI accents [src/ui/egui/mod.rs 87-111](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L87-L111)

## tt-toplike-app: Native TUI Host

The `tt-toplike-app` binary is a specialized wrapper that hosts the `tt-toplike-tui` inside a native window. This allows users to run the TUI without a pre-existing terminal emulator, using `portable-pty` for process management and a custom VT100 state machine for rendering.

### PTY and VT100 Architecture

The application acts as a terminal emulator. It spawns the TUI as a child process and communicates via a Pseudo-Terminal (PTY).

1.   **PTY Master**: The app creates a `PtyPair` using `native_pty_system()`[src/bin/app.rs 18](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L18-L18)
2.   **VT Parser**: Output from the TUI is piped into a `vte::Parser`. This parser implements the `Perform` trait to update a local `Screen` grid [src/bin/app.rs 22](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L22-L22)
3.   **Screen Grid**: A 2D `Vec` of `Cell` objects representing characters, colors (FG/BG), and attributes (bold/underline) [src/bin/app.rs 27-33](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L27-L33)
4.   **Egui Rendering**: Every frame, the `Screen` grid is rendered using `egui::Painter`. For performance, it uses `LayoutJob` to batch text rendering by row [src/bin/app.rs 13-15](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L13-L15)

### Visual State Machine (VT100)

The `Screen` struct manages the terminal state, including cursor position and "pending wrap" logic to handle edge-of-screen character placement correctly [src/bin/app.rs 68-82](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L68-L82)

## Implementation Comparison

The two GUI approaches serve different technical requirements within the workspace.

| Feature | tt-toplike-egui | tt-toplike-app |
| --- | --- | --- |
| **Primary Goal** | Real-time Charts & Dashboard | Native Window for TUI |
| **Backend** | Direct `TelemetryBackend` trait | Indirect via PTY pipe to TUI |
| **Rendering** | `egui` widgets &`egui_plot` | `vte` parser to `egui` painter |
| **History** | 120-sample `VecDeque`[src/ui/egui/mod.rs 26](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/ui/egui/mod.rs#L26-L26) | Managed by TUI process |
| **Input** | Standard GUI interaction | ANSI escape sequence encoding [src/bin/app.rs 14-15](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L14-L15) |

### Technical Highlights

*   **ANSI 256 Color Support**: `tt-toplike-app` implements a full 256-color palette, including the standard 16 ANSI colors, the 6x6x6 color cube, and the grayscale ramp [src/bin/app.rs 155-192](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L155-L192)
*   **SGR Parsing**: The app handles `Select Graphic Rendition` codes for bold, underline, and extended 24-bit (RGB) colors [src/bin/app.rs 196-220](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L196-L220)
*   **Deferred Wrapping**: To prevent visual artifacts, the VT state machine implements "pending wrap," where the cursor only moves to the next line when the _next_ printable character arrives after hitting the column limit [src/bin/app.rs 77-81](https://github.com/tenstorrent/tt-toplike/blob/487bfe5a/src/bin/app.rs#L77-L81)
