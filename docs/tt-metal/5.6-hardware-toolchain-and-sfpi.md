## Purpose and Scope

This page documents the SFPI (Scalar Floating Point Instructions) compiler toolchain, the custom RISC-V toolchain used for Tenstorrent hardware, and the associated linker scripts and firmware compilation processes. It explains how the build system integrates the toolchain to generate binaries for Tensix and Ethernet cores across different hardware architectures like Wormhole and Blackhole.

**Scope of this page:**

*   SFPI toolchain architecture, installation, and versioning.
*   Linker script generation and memory mapping for Wormhole and Blackhole architectures.
*   ELF image processing and XIP (Execute In Place) transformations.
*   SFPI math operation implementation in compute kernels.
*   Integration with the CI/CD build pipeline and Docker environments.

## SFPI Toolchain Overview

The SFPI toolchain is a customized GCC-based RISC-V cross-compiler optimized for Tenstorrent's Tensix architecture. It targets various processor classes defined in the Hardware Abstraction Layer (HAL) to support data movement and compute operations.

**Key Components:**

*   **Compiler**: `riscv-tt-elf-g++`[tt_metal/hw/CMakeLists.txt 109](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/CMakeLists.txt#L109-L109)
*   **Linker**: Custom linker scripts that manage distinct memory regions (L1, IRAM, LDM) [tt_metal/hw/toolchain/erisc-b0-kernel.ld 9-15](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/toolchain/erisc-b0-kernel.ld#L9-L15)
*   **Processors**: Targets include BRISC (Base RISC), NCRISC (Non-compute RISC), TRISC (Tensix RISC), ERISC (Ethernet RISC), and BH-specific DRISC [tt_metal/hw/CMakeLists.txt 6-18](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/CMakeLists.txt#L6-L18)

### Supported Architectures and Processors

The toolchain handles multiple hardware generations, mapping logical processor types to specific RISC-V CPU configurations. Blackhole architecture introduces additional complexity with processors like DRISC.

## SFPI Version Management

### Version Detection and Configuration

The SFPI version is strictly controlled via `tt_metal/sfpi-version`[tt_metal/sfpi-version 1-13](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/sfpi-version#L1-L13) This file contains the release version (e.g., `7.55.0`), build number, and SHA256 hashes for different host architectures (x86_64, aarch64) and distributions (Debian, Fedora) [tt_metal/sfpi-version 4-13](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/sfpi-version#L4-L13)

The script `sfpi-info.sh` parses this file to generate system-specific info for downloading or building the toolchain [tt_metal/sfpi-info.sh 3-6](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/sfpi-info.sh#L3-L6) In the CMake build, `sfpi-info.sh` is executed to generate `sfpi-version.cmake`, which is then included to set variables like `SFPI_version` and `SFPI_hash`[tt_metal/hw/CMakeLists.txt 46-61](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/CMakeLists.txt#L46-L61)

## Linker Scripts and Memory Layout

### Memory Region Definitions

Linker scripts define the memory layout for firmware and kernels. For Ethernet cores on Wormhole, the script `erisc-b0-kernel.ld` manages code and data placement. CMake pre-processes these linker scripts using the C preprocessor to handle conditional defines like `ENABLE_IRAM`[tt_metal/hw/CMakeLists.txt 194-207](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/CMakeLists.txt#L194-L207)

**Key Sections:**

*   **.text**: Placed at `__firmware_start` in `REGION_APP_KERNEL_CODE`[tt_metal/hw/toolchain/erisc-b0-kernel.ld 52-62](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/toolchain/erisc-b0-kernel.ld#L52-L62)
*   **.data**: Contains read-only and initialized data, placed in `REGION_APP_KERNEL_DATA`[tt_metal/hw/toolchain/erisc-b0-kernel.ld 70-86](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/toolchain/erisc-b0-kernel.ld#L70-L86)
*   **.bss**: Uninitialized data, bounded by `__ldm_bss_start` and `__ldm_bss_end`[tt_metal/hw/toolchain/erisc-b0-kernel.ld 101-112](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/toolchain/erisc-b0-kernel.ld#L101-L112)
*   **.device_print_strings**: Specialized section for `DPRINT` at VMA `0x6400000`[tt_metal/hw/toolchain/erisc-b0-kernel.ld 35-41](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/toolchain/erisc-b0-kernel.ld#L35-L41)

| Processor | Region Alias | Target Memory |
| --- | --- | --- |
| ERISC | `REGION_APP_KERNEL_CODE` | `ERISC_APP_KERNEL_CODE` or `ERISC_APP_IRAM_CODE` |
| ERISC | `REGION_APP_KERNEL_DATA` | `ERISC_APP_KERNEL_DATA` |
| ERISC | `REGION_LDM` | `LOCAL_DATA_MEM` |

## SFPI Math Operations

SFPI allows writing high-performance math in C++ that compiles to optimized Tensix instructions. These implementations are found in `ckernel_sfpu` headers and utilize the `sfpi` namespace.

### Implementation: Trigonometry (Tan/Sine)

The SFPU implementation utilizes Cody-Waite reduction to bring arguments into the `[-PI/2, PI/2]` or `[-PI/4, PI/4]` range [tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h 142-149](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h#L142-L149)

*   **Polynomial Approximation**: Uses high-precision coefficients for `sin` and `tan`[tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h 36-45](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h#L36-L45)
*   **Reciprocal Refinement**: Uses Newton-Raphson iterations to refine approximate reciprocals near poles. Blackhole uses the `approx_recip` intrinsic [tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h 53-65](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/blackhole/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h#L53-L65) while Wormhole implements it manually [tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h 54-73](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_trigonometry.h#L54-L73)

## Build System Integration

The hardware toolchain is integrated into the main CMake build system via `add_dependencies(tt_metal hw_toolchain)`[tt_metal/CMakeLists.txt 138](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/CMakeLists.txt#L138-L138) The `hw` subdirectory manages the retrieval and verification of the SFPI toolchain [tt_metal/hw/CMakeLists.txt 41-171](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/hw/CMakeLists.txt#L41-L171)

### JIT API Headers

The build system defines a `jitapi` interface library that collects all headers required for on-device kernel compilation, including `FABRIC_HW_INC`, `COMPUTE_KERNEL_API`, and `TT_LLK_HEADERS`[tt_metal/CMakeLists.txt 51-76](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/CMakeLists.txt#L51-L76) These are installed to `tt-metalium/tt_metal` to support runtime JIT compilation [tt_metal/CMakeLists.txt 192-200](https://github.com/tenstorrent/tt-metal/blob/f30f8df0/tt_metal/CMakeLists.txt#L192-L200)
