## Purpose and Scope

This document covers TT-Forge and TT-Torch, which provide PyTorch integration for running models on Tenstorrent hardware. TT-Torch serves as a `torch.compile` backend that enables seamless compilation of PyTorch models to execute on Tenstorrent accelerators. This material focuses on practical integration patterns, configuration options, and workflow examples for converting existing PyTorch/HuggingFace models.

For information about the underlying neural network operations library, see [TT-NN Neural Network Library](https://deepwiki.com/tenstorrent/tt-tutorial/3.2-tt-nn-neural-network-library). For low-level hardware programming concepts, see [TT-Metalium Programming Model](https://deepwiki.com/tenstorrent/tt-tutorial/4.1-tt-metalium-programming-model).

## Integration Patterns

### Stable Diffusion Integration

The Stable Diffusion integration demonstrates selective compilation of the transformer component:

**Key Integration Points:**

*   Authentication via `HF_TOKEN` environment variable
*   Selective compilation: `pipe.transformer.forward` only
*   Memory optimization: `enable_attention_slicing()`
*   Error handling for compilation and generation phases

Sources: [tt-forge/tt-torch/stable_diffusion_tutorial.md 115-124](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-forge/tt-torch/stable_diffusion_tutorial.md?plain=1#L115-L124)

### CodeGen Integration

The CodeGen integration shows full model compilation:

**Key Integration Points:**

*   Tokenizer padding configuration: `tokenizer.pad_token_id = tokenizer.eos_token_id`
*   Data type conversion to `torch.bfloat16`
*   Full model compilation: `torch.compile(model, backend="tt", options=options)`
*   Dynamo state management: `torch._dynamo.reset()`

Sources: [tt-forge/tt-torch/codegen_hf_tutorial.md 67-92](https://github.com/tenstorrent/tt-tutorial/blob/389c542c/tt-forge/tt-torch/codegen_hf_tutorial.md?plain=1#L67-L92)

## Error Handling and Debugging

### Common Error Patterns

**Compilation Error Handling:**

*   Wrap `torch.compile` calls in try-catch blocks
*   Provide fallback to eager execution
*   Reset dynamo state between compilations

**Runtime Error Handling:**

*   Handle generation failures gracefully
*   Implement timeout mechanisms for long-running operations
*   Validate input tensor shapes and types
