## Purpose and Scope

This document describes the JAX-based MNIST training implementations in the TT-Blacksmith repository. These implementations demonstrate different JAX frameworks (pure JAX, Flax, NNX) and parallelization strategies (single chip, data parallel, tensor parallel) for training multi-layer perceptron (MLP) models on MNIST digit classification.

For PyTorch-based MNIST examples, see [MNIST Training Examples](https://deepwiki.com/tenstorrent/tt-blacksmith/6.1.1-mnist-training-examples). For advanced JAX experiments like NeRF, see [JAX NeRF and Advanced Examples](https://deepwiki.com/tenstorrent/tt-blacksmith/6.2.2-jax-nerf-and-advanced-examples).

### Data Parallel Implementation

The data parallel implementation demonstrates distributed training across multiple TT devices using JAX's sharding primitives.

#### Sharding Configuration

The `ShardingConfig` class defines the mesh topology and sharding specifications:

The sharding configuration is defined at [blacksmith/experiments/jax/mnist/multi_chip/data_parallel/test_pure_jax_mnist.py 23-29](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/multi_chip/data_parallel/test_pure_jax_mnist.py#L23-L29):

*   **Mesh creation**: `Mesh(np.array(jax.devices("tt")), axis_names=("dp",))` creates a 1D mesh along the data parallel dimension
*   **Data sharding**: `PartitionSpec("dp")` shards batch along first dimension
*   **Parameter sharding**: `PartitionSpec()` replicates parameters across all devices
*   **Scalar sharding**: `PartitionSpec()` replicates scalars (loss, learning rate)

**Sources:**

*   [blacksmith/experiments/jax/mnist/multi_chip/data_parallel/test_pure_jax_mnist.py 23-29](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/multi_chip/data_parallel/test_pure_jax_mnist.py#L23-L29)

#### Gradient Synchronization

Data parallel training requires gradient synchronization across devices:

The synchronization logic is implemented in `compute_loss_grads_logits` at [blacksmith/experiments/jax/mnist/multi_chip/data_parallel/test_pure_jax_mnist.py 75-93](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/multi_chip/data_parallel/test_pure_jax_mnist.py#L75-L93):

1.   Each device computes local loss and gradients on its data shard
2.   `lax.all_gather` collects gradients from all devices along the `"dp"` axis
3.   `jnp.mean` averages the gathered gradients
4.   `tree_util.tree_map` applies averaging to all parameter gradients in the pytree

**Sources:**

### Tensor Parallel Implementation

The tensor parallel implementation shards model parameters across devices using a column-row-column sharding strategy.

#### Column-Row-Column Sharding Strategy

The model implementation at [blacksmith/experiments/jax/mnist/multi_chip/tensor_parallel/test_pure_jax_mnist.py 52-66](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/multi_chip/tensor_parallel/test_pure_jax_mnist.py#L52-L66) follows a specific sharding pattern to minimize communication:

1.   **Layer 1 (Column)**: Weight matrix sharded along output dimension, producing sharded activations
2.   **Layer 2 (Row)**: Weight matrix sharded along input dimension, requires `lax.psum` for partial sum reduction
3.   **Layer 3 (Column)**: Weight matrix sharded along output dimension, produces sharded logits

This strategy is motivated by a metal limitation (GitHub issue #21987) that prevents all-reduce operations on dimensions not divisible by 32.

**Sources:**

*   [blacksmith/experiments/jax/mnist/multi_chip/tensor_parallel/test_pure_jax_mnist.py 47-66](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/experiments/jax/mnist/multi_chip/tensor_parallel/test_pure_jax_mnist.py#L47-L66)

## Data Loading

The `load_mnist_jax` function provides a unified data loading interface for all JAX MNIST implementations.

### Data Loading Pipeline

The data loader at [blacksmith/datasets/jax/mnist/dataloader.py 18-76](https://github.com/tenstorrent/tt-blacksmith/blob/fe25eba0/blacksmith/datasets/jax/mnist/dataloader.py#L18-L76) performs these transformations:

1.   **Download**: Uses `torchvision.datasets.MNIST` to download MNIST data
2.   **Normalization**: Scales pixel values from [0, 255] to [0.0, 1.0]
3.   **Reshaping**: Flattens 28x28 images to 784-dimensional vectors
4.   **One-hot encoding**: Converts integer labels to 10-dimensional one-hot vectors
5.   **Shuffling**: Randomizes training examples with fixed seed 42
6.   **Splitting**: Creates 80% train / 20% validation split
7.   **Truncation**: Optionally limits dataset size for testing via `test_config.max_steps_per_epoch`

### Performance Characteristics

The implementations exhibit different performance trade-offs:

**Pure JAX Single Chip:**

*   Simplest implementation with lowest overhead
*   Direct control over all operations
*   Best for small models and debugging

**Pure JAX Data Parallel:**

*   Scales linearly with number of devices
*   Communication overhead from gradient synchronization
*   Best for large batch sizes

**Pure JAX Tensor Parallel:**

*   Enables training models larger than single device memory
*   Communication overhead from partial sums
*   Requires careful sharding strategy design
*   Best for large models with small batches

**Flax:**

*   Higher-level abstractions reduce boilerplate
*   Built-in checkpointing and state management
*   Good balance of simplicity and features

**NNX:**

*   Most PyTorch-like API
*   Mutable module design
*   Built-in metrics tracking
*   Best for users familiar with PyTorch
