## Element-wise Unary SFPU Test Example

The `test_eltwise_unary_sfpu.py` file demonstrates the canonical test structure used throughout the suite. It validates 60+ SFPU operations across multiple data formats.

### Test Function Structure

**Test Parametrization via `FLOAT_TEST_PARAMS`:**

[tests/python_tests/test_eltwise_unary_sfpu.py 89-111](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/test_eltwise_unary_sfpu.py#L89-L111) constructs `FLOAT_TEST_PARAMS` using `itertools.chain()` and `product()`:

*   **Formats:**`input_output_formats([Float32, Float16, Float16_b, Bfp8_b])` generates `InputOutputFormat` pairs
*   **Approximation Modes:**`ApproximationMode.No`, `ApproximationMode.Yes`
*   **Math Operations:**`SUPPORTED_FAST_MODE_OPS` (Log1p, Exp, Rsqrt, Sqrt) with `FastMode.No`/`FastMode.Yes`, plus 19 other operations with `FastMode.No` only
*   **Dest Accumulation:**`DestAccumulation.No`, `DestAccumulation.Yes`

The `@pytest.mark.parametrize` decorator [tests/python_tests/test_eltwise_unary_sfpu.py 115-118](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/test_eltwise_unary_sfpu.py#L115-L118) applies these combinations, creating approximately **~500+ test variants**.

**C++ Kernel Implementation:**

[tests/sources/eltwise_unary_sfpu_test.cpp 21-138](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/sources/eltwise_unary_sfpu_test.cpp#L21-L138) implements the three-stage pipeline:

1.   **UNPACK stage**[tests/sources/eltwise_unary_sfpu_test.cpp 26-42](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/sources/eltwise_unary_sfpu_test.cpp#L26-L42):

    *   `_llk_unpack_hw_configure_<is_fp32_dest_acc_en>()` sets up unpacker formats
    *   `_llk_unpack_A_init_<BroadcastType::NONE, false, EltwiseBinaryReuseDestType::NONE, unpack_to_dest>()` initializes unpacker
    *   Loop iterates `TILE_CNT` times, calling `_llk_unpack_A_<>()` to read from `L1_ADDRESS(params->buffer_A[i])`

2.   **MATH stage**[tests/sources/eltwise_unary_sfpu_test.cpp 59-99](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/sources/eltwise_unary_sfpu_test.cpp#L59-L99):

    *   `_llk_math_eltwise_unary_datacopy_init_<DataCopyType::A2D>()` prepares datacopy
    *   Loop processes tiles in blocks of `MAX_TILES_DEST` (calculated based on `is_fp32_dest_acc_en`)
    *   `_llk_math_wait_for_dest_available_<DST_SYNC>()` waits for destination capacity
    *   `_llk_math_eltwise_unary_datacopy_<DataCopyType::A2D>()` copies SrcA to DEST
    *   `_llk_math_eltwise_unary_sfpu_init_<SFPU_UNARY_OPERATION>()` initializes SFPU operation
    *   `_llk_math_eltwise_unary_sfpu_start_<DST_SYNC>(block_tile)` begins SFPU on specific tile
    *   `test_utils::call_sfpu_operation<APPROX_MODE, is_fp32_dest_acc_en, iterations, FAST_MODE>()` dispatches to specific SFPU function
    *   `_llk_math_dest_section_done_<DST_SYNC, is_fp32_dest_acc_en>()` signals completion

3.   **PACK stage**[tests/sources/eltwise_unary_sfpu_test.cpp 108-136](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/sources/eltwise_unary_sfpu_test.cpp#L108-L136):

    *   `_llk_pack_hw_configure_<is_fp32_dest_acc_en>()` configures packer
    *   `_llk_pack_init_<>()` initializes with output format
    *   Loop packs tiles in blocks: `_llk_pack_<DST_SYNC, is_fp32_dest_acc_en, false>(block_tile, L1_ADDRESS(params->buffer_Res[block_start + block_tile]))`
    *   `_llk_pack_dest_section_done_<DST_SYNC, is_fp32_dest_acc_en>()` signals section complete

**Golden Model Computation:**

The `UnarySFPUGolden` class in [tests/python_tests/helpers/golden_generators.py](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/helpers/golden_generators.py) implements the golden model:

*   Retrieves PyTorch operation from `self.operation_map` dictionary
*   For MX formats, calls `quantize_mx_tensor_chunked()` to simulate hardware quantization
*   Applies PyTorch operation to compute expected output
*   Converts result back to target format

## Pack Untilize Test Example

The `test_pack_untilize.py` file validates the packer's ability to convert tiled data back to row-major format.

### Pack Untilize Configuration

**Key Parameters:**

[tests/python_tests/test_pack_untilize.py 38-53](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/test_pack_untilize.py#L38-L53) defines the test matrix:

*   **Input dimensions:**`[64, 512]`, `[64, 64]`, `[32, 128]`, `[128, 128]`, `[32, 64]`
*   **Dest sync modes:**`DestSync.Half`
*   **Block column dimension:** Calculated dynamically based on `dest_acc`

**Block CT Dimension Calculation:**

[tests/python_tests/test_pack_untilize.py 118-125](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/test_pack_untilize.py#L118-L125) implements `block_ct_dim` calculation:

This logic:

*   Sets `max_bct_dim=8` for 16-bit DEST mode (8 tiles capacity), `max_bct_dim=4` for 32-bit DEST mode (4 tiles capacity)
*   Calculates `full_ct_dim` as number of 32-element column tiles in input matrix
*   Uses generator expression with `next()` to find largest `bct` in range `[max_bct_dim, ..., 1]` that evenly divides `full_ct_dim`
*   Falls back to `bct=1` if no divisor found

The calculation ensures untilize processes multiple tiles simultaneously while respecting DEST capacity. The C++ kernel's `_llk_pack_untilize_init_<>()` contains `static_assert(block_ct_dim <= 8)` to enforce this limit.

**Format Constraints:**

[tests/python_tests/test_pack_untilize.py 57-99](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/test_pack_untilize.py#L57-L99) implements several skip conditions:

*   Skip large dimension test `[64, 512]` in coverage mode due to issue #1063
*   Skip `Bfp8_b` output format (hardware limitation: pack untilize does not support block float formats)
*   Skip mixing `Int32` with other formats (XOR constraint: both input and output must be Int32 or neither)
*   Skip `Float16` input with 32-bit intermediate format and `dest_acc=No` (gasket cannot convert 8-bit exponent datums to Float16 without 32-bit intermediate DEST storage)

**Sources:**[tests/python_tests/test_pack_untilize.py 38-165](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/test_pack_untilize.py#L38-L165)[tests/python_tests/helpers/golden_generators.py](https://github.com/tenstorrent/tt-llk/blob/366f58e2/tests/python_tests/helpers/golden_generators.py)

## Multi-Format Coverage Strategy

Tests validate format combinations using the `input_output_formats()` helper:

**Format Outliers:**

Certain format combinations require specific configurations:

*   `Float16_b → Float16`: Requires `dest_acc=Yes` (cannot convert 8-bit exponent to Float16 in 16-bit DEST)
*   `Bfp8_b → Float16`: Requires `dest_acc=Yes` (same constraint)
*   `Float16 + dest_acc=No` on Blackhole: Not supported for SFPU operations [test_sfpu_binary.py 54-61](https://github.com/tenstorrent/tt-llk/blob/366f58e2/test_sfpu_binary.py#L54-L61)
