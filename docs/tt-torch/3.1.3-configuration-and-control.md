## CompilerConfig Class

The `CompilerConfig` class serves as the central configuration hub, managing compilation depth, profiling options, optimization settings, and verification controls.

### Core Configuration Options

**Compilation Depth Control**

The `compile_depth` property determines how far through the compilation pipeline to proceed:

| Depth | Description | Use Case |
| --- | --- | --- |
| `TORCH_FX` | Stop after FX graph creation | Graph analysis |
| `STABLEHLO` | Stop after StableHLO conversion | MLIR debugging |
| `TTNN_IR` | Stop after TTNN compilation | Full compilation test |
| `COMPILE_OP_BY_OP` | Compile operations individually | Op debugging |
| `EXECUTE_OP_BY_OP` | Execute operations individually | Op validation |
| `EXECUTE` | Full end-to-end execution | Production use |

## Summary

The configuration and control system in tt-torch provides flexible control over compilation behavior through:

*   **CompilerConfig**: Central configuration with environment variable overrides
*   **BackendOptions**: Execution-specific options with caching support
*   **Property validation**: Ensures configuration consistency
*   **Integration points**: Seamless integration with `torch.compile` and testing frameworks

This system enables everything from quick debugging with op-by-op execution to optimized production deployments with full compilation pipelines.
