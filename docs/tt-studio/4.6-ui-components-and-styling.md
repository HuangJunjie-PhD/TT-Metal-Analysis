## Theme Management System

TT-Studio implements a comprehensive theme system supporting light, dark, and system-based themes through the `ThemeProvider` component.

### ThemeProvider Architecture

The theme system is built around a React context that manages theme state and persistence:

The `ThemeProvider` supports three theme modes defined by the `Theme` type, with the default theme set to "dark". Theme preference is persisted using the `storageKey` parameter.

### CSS Variables System

The application uses CSS variables (HSL format) to handle dynamic theming. These are defined within `@layer utilities` in the main stylesheet [app/frontend/src/index.css 170-195](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/index.css#L170-L195)

| Variable Category | Light Mode Variables | Dark Mode Variables |
| --- | --- | --- |
| Background | `--background: 220 20% 96%` | `--background: 20 14.3% 4.1%` |
| Foreground | `--foreground: 220 15% 20%` | `--foreground: 60 9.1% 97.8%` |
| Primary | `--primary: 24 9.8% 10%` | `--primary: 60 9.1% 97.8%` |
| Secondary | `--secondary: 60 4.8% 95.9%` | `--secondary: 12 6.5% 15.1%` |
| Sidebar | `--sidebar-background: 0 0% 98%` | `--sidebar-background: 240 5.9% 10%` |

## Reusable UI Components

TT-Studio leverages `shadcn/ui` primitives built on top of Radix UI. These components are customized to match the Tenstorrent design language.

The following diagram maps high-level UI requirements to specific code entities and their Radix/Shadcn origins:

### Key UI Components

#### 1. Button System

The `Button` component uses `class-variance-authority` (CVA) to manage multiple variants, including a specialized `navbar` variant [app/frontend/src/components/ui/button.tsx 12-43](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/button.tsx#L12-L43)

*   **Default**: Stone-based colors with subtle hover transforms [app/frontend/src/components/ui/button.tsx 17-18](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/button.tsx#L17-L18)
*   **Navbar**: Specific styling for top-level navigation with shadow effects [app/frontend/src/components/ui/button.tsx 23-24](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/button.tsx#L23-L24)

#### 2. Status Badges

The `Badge` component has been modified from standard `shadcn` to serve specifically as a status indicator. It includes a `statusColor` prop that renders a colored dot alongside the text [app/frontend/src/components/ui/badge.tsx 37-50](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/badge.tsx#L37-L50)

*   **Variants**: `default` (Green), `destructive` (Red), `warning` (Yellow), and `outline`[app/frontend/src/components/ui/badge.tsx 20-29](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/badge.tsx#L20-L29)

#### 3. Form and Input

The form system integrates `react-hook-form` with custom UI components to provide accessible, validated input fields [app/frontend/src/components/ui/form.tsx 7-22](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/form.tsx#L7-L22) It utilizes a context-based approach via `FormFieldContext` to sync error states between labels and inputs [app/frontend/src/components/ui/form.tsx 31-46](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/components/ui/form.tsx#L31-L46)

2.   **Roboto Mono**: Monospace font for code and technical data [app/frontend/src/index.css 3](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/index.css#L3-L3)
3.   **Bricolage Grotesque**: Stylized font used for specific display elements [app/frontend/src/index.css 5](https://github.com/tenstorrent/tt-studio/blob/a6d347af/app/frontend/src/index.css#L5-L5)
