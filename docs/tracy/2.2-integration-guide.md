This document covers how to integrate the Tracy client library into applications to enable profiling. It focuses on the instrumentation API and configuration options needed to collect profiling data from your application.

For information about building Tracy itself, see [Build System](https://deepwiki.com/tenstorrent/tracy/2.1-build-system). For detailed API reference of specific profiling features, see [Core Profiling API](https://deepwiki.com/tenstorrent/tracy/3.1-core-profiling-api).

## Overview

Tracy integration involves embedding the Tracy client library into your application and instrumenting your code with profiling macros. The client library collects profiling events and transmits them to a Tracy server application for visualization and analysis.

### 2. Define TRACY_ENABLE

Enable Tracy profiling by defining `TRACY_ENABLE` for your entire project. This macro controls whether Tracy instrumentation is active or compiled out.

| Build Configuration | TRACY_ENABLE | Behavior |
| --- | --- | --- |
| Debug/Profiling | Defined | Full profiling enabled |
| Release/Production | Not defined | All Tracy macros become no-ops |

## Core Instrumentation API

### Zone Profiling Macros

Tracy provides hierarchical zone profiling through scoped macros:

**Zone Instrumentation API Mapping**

| Macro | Purpose | Example |
| --- | --- | --- |
| `ZoneScoped` | Profile current function | `void func() { ZoneScoped; }` |
| `ZoneScopedN(name)` | Named zone | `ZoneScopedN("render");` |
| `ZoneScopedC(color)` | Colored zone | `ZoneScopedC(0xFF0000);` |
| `ZoneText(txt, size)` | Add text to zone | `ZoneText("vertices", 8);` |
| `ZoneName(txt, size)` | Set zone name | `ZoneName("physics", 7);` |

## Multi-DLL Integration

For applications with multiple DLLs/shared objects:

**Multi-DLL Integration Pattern**

Required configuration:

*   Create dedicated profiler DLL containing [public/TracyClient.cpp](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/TracyClient.cpp)
*   Link all executables and DLLs to profiler DLL
*   Define `TRACY_IMPORTS` for Windows/MinGW builds
*   Consider `TRACY_DELAYED_INIT` and `TRACY_MANUAL_LIFETIME` for dynamic loading

## Platform-Specific Considerations

### Timer Resolution

Tracy achieves nanosecond resolution through platform-specific timers:

| Platform | Timer Method | Resolution |
| --- | --- | --- |
| x86/x64 | RDTSC instruction | ~1 nanosecond |
| ARM/ARM64 | Hardware timer registers | ~40 nanoseconds |
| iOS | `mach_absolute_time()` | Platform optimal |
| Fallback | `std::chrono::high_resolution_clock` | ~250-1000 nanoseconds |

Sources: [public/client/TracyProfiler.hpp 195-253](https://github.com/tenstorrent/tracy/blob/68c5cbd6/public/client/TracyProfiler.hpp#L195-L253)[manual/tracy.tex 166-170](https://github.com/tenstorrent/tracy/blob/68c5cbd6/manual/tracy.tex#L166-L170)
