## Purpose and Scope

This document provides a complete reference for all custom Textual widgets implemented in tt-tools-common. These widgets extend the [textual](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/textual) TUI framework with Tenstorrent-specific components for building consistent terminal-based interfaces across TT-Tools applications.

This page covers the core widget classes (`TTHeader`, `TTFooter`, `TTDataTable`, `TTMenu`, `TTHostCompatibilityMenu`, `TTCompatibilityMenu`). For modal screen components (`TTConfirmBox`, `TTHelperMenuBox`, `TTSettingsMenu`), see [Modal Screens and Dialogs](https://deepwiki.com/tenstorrent/tt-tools-common/4.2-modal-screens-and-dialogs). For styling and theming information, see [Theming and Styling](https://deepwiki.com/tenstorrent/tt-tools-common/4.3-theming-and-styling). For practical integration examples, see [Building Custom TUI Applications](https://deepwiki.com/tenstorrent/tt-tools-common/4.4-building-custom-tui-applications).

## Widget Library Architecture

The widget library is organized into three functional categories: layout widgets that structure the application frame, data display widgets that present information, and modal screens that handle user interactions.

## Layout Widgets

### TTHeader

The `TTHeader` widget provides a standardized application header displaying the application name, version, and a real-time clock. The header automatically refreshes every second to update the timestamp.

**Class Definition:**

```
TTHeader(Widget)
```

**Constructor Parameters:**

| Parameter | Type | Description |
| --- | --- | --- |
| `app_name` | `str` | Application name displayed in center column |
| `app_version` | `str` | Version string displayed in left column with "Version" prefix |

**Implementation Details:**

The widget uses a Rich `Table.grid` with three columns (ratio 0.25:0.5:0.25) for left-justified version, centered app name, and right-justified timestamp. The render method is called automatically every second via `set_interval(1, callback=self.refresh)` configured in `on_mount()`.

**Rendering Layout:**

```
[Version X.Y.Z]     [Application Name]     [MMM DD YYYY HH:MM:SS AM/PM]
```

**CSS Styling:**

The header receives the following default styles from `common_style.css`:

*   Height: 3 lines
*   Dock position: top
*   Outline: solid green border
*   Text style: bold

**Code Reference:**[tt_tools_common/ui_common/widgets.py 22-45](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L22-L45)

### TTFooter

The `TTFooter` widget provides a customizable footer for displaying static or dynamic content across the bottom of the application. Unlike the standard Textual `Footer` which shows keybindings, this widget supports arbitrary content.

**Class Definition:**

```
TTFooter(Widget)
```

**Constructor Parameters:**

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `content` | `List[str]` | required | List of strings to display as columns |
| `justify` | `str` | `"center"` | Text justification: "left", "center", or "right" |

**Implementation Details:**

The widget creates a Rich `Table.grid` with equal-ratio columns based on the length of the `content` list. Each string in the list occupies one column with the specified justification.

**CSS Styling:**

The footer receives the following default styles:

*   Height: 3 lines
*   Dock position: bottom
*   Outline: solid green border
*   Text style: bold

**Code Reference:**[tt_tools_common/ui_common/widgets.py 48-62](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L48-L62)

## Data Display Widgets

### TTDataTable

The `TTDataTable` widget wraps Textual's `DataTable` in a `ScrollableContainer` with standardized styling and data management methods. This widget is the primary component for displaying tabular data in TT-Tools applications.

**Class Definition:**

```
TTDataTable(ScrollableContainer)
```

**Constructor Parameters:**

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `title` | `str` | required | Border title for the table |
| `header` | `List[str]` | `None` | Column header labels (auto-styled with underline) |
| `id` | `str` | `None` | Widget identifier |
| `**kwargs` |  |  | Additional arguments passed to `DataTable` constructor |

**Key Attributes:**

| Attribute | Type | Description |
| --- | --- | --- |
| `dt` | `DataTable` | Internal DataTable widget instance (ID: `{id}_table`) |
| `border_title` | `str` | Title displayed in container border |

**Key Methods:**

**`update_data(rows: List[str]) -> None`**

Updates table data efficiently. Attempts cell-by-cell updates first; if row count changed, clears and re-populates the entire table. This method enables dynamic data refresh without recreating the table.

**`style_header() -> None`**

Internal method that applies Rich `Text` styling to header cells (centered, underlined). Called automatically during initialization if `header` is provided.

**DataTable Configuration:**

The internal `DataTable` is configured with:

*   `zebra_stripes = False`: Disabled for consistent coloring
*   `cursor_type = "cell"`: Cell-level selection rather than row
*   Border title matches the container title

**Usage Pattern:**

`# In compose():yield TTDataTable(title="Results", header=["ID", "Status", "Value"], id="results_dt") # In on_mount() or update handler:tt_dt = self.get_widget_by_id("results_dt")tt_dt.dt.add_rows([["0", "Pass", "100"], ["1", "Fail", "0"]]) # For dynamic updates:tt_dt.update_data([["0", "Pass", "150"], ["1", "Pass", "200"]])`
**CSS Styling:**

DataTable receives specific cursor and hover styles:

*   Cursor background: yellow (`$yellow`)
*   Cursor text: black
*   Hover background: yellow at 60% opacity
*   Header: bold, centered, transparent background

**Code Reference:**[tt_tools_common/ui_common/widgets.py 65-117](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L65-L117)

### TTMenu

The `TTMenu` widget displays a list of key-value pairs with consistent formatting. Keys are bold and yellow-colored, while values use default styling. This widget is suitable for displaying system information or configuration details.

**Class Definition:**

```
TTMenu(Container)
```

**Constructor Parameters:**

| Parameter | Type | Description |
| --- | --- | --- |
| `id` | `str` | Widget identifier |
| `title` | `str` | Border title |
| `data` | `OrderedDict[str, str]` | Ordered dictionary of key-value pairs to display |

**Display Format:**

Each entry is rendered as:

```
* Key         : Value
```

Keys are left-justified based on the longest key length plus 1. The asterisk bullet point is prepended to each line.

**Special Handling:**

If a key equals `"Failed to fetch"`, the value is displayed in dark orange color on a new line without the colon separator, used for error messages.

**Implementation Details:**

The widget calculates `justify_width` as the maximum key length plus 1 to ensure consistent alignment. Rich `Text` objects are composed with styled segments for keys (yellow, bold) and plain text for values.

**Code Reference:**[tt_tools_common/ui_common/widgets.py 120-146](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L120-L146)

### TTHostCompatibilityMenu

The `TTHostCompatibilityMenu` widget extends `TTMenu` to display host configuration information with compatibility warnings. Values can be simple strings (compatible) or tuples containing the current value and a recommended value (incompatible).

**Class Definition:**

```
TTHostCompatibilityMenu(Container)
```

**Constructor Parameters:**

| Parameter | Type | Description |
| --- | --- | --- |
| `id` | `str` | Widget identifier |
| `title` | `str` | Base border title |
| `data` | `Dict[str, Union[str, Tuple]]` | Dictionary with string values (compatible) or tuple values (current, recommended) |

**Value Type Handling:**

| Value Type | Display Behavior | Example |
| --- | --- | --- |
| `str` | Single line with key-value pair | `* OS: Ubuntu 22.04` |
| `Tuple[str, str]` | Two lines: current value + recommendation in orange | `* Kernel: 5.15.0` `* Kernel >= 6.1.0` |

**Border Title Behavior:**

The constructor automatically modifies the border title based on compatibility:

*   All values are strings → `"{title} (Fully Compatible)"`
*   Any tuple values present → `"{title} (Config Warning!)"` with red border title and subtitle `"* Recommended Config"`

**CSS Styling:**

Border title and subtitle colors are dynamically set via `self.styles.border_title_color` and `self.styles.border_subtitle_color` when warnings are present.

**Code Reference:**[tt_tools_common/ui_common/widgets.py 149-210](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L149-L210)

### TTCompatibilityMenu

The `TTCompatibilityMenu` widget displays boolean compatibility checks with pass/fail indication. Each item in the data dictionary contains a tuple of `(bool, str)` representing the check result and associated message.

**Class Definition:**

```
TTCompatibilityMenu(Container)
```

**Constructor Parameters:**

| Parameter | Type | Description |
| --- | --- | --- |
| `id` | `str` | Widget identifier |
| `title` | `str` | Border title |
| `data` | `OrderedDict[str, Tuple[bool, str]]` | Ordered dictionary mapping check names to `(result, message)` tuples |

**Display Format:**

Each entry renders as:

```
* Check Name  : Message
```

The message text color depends on the boolean result:

*   `True`: Default green color (normal text)
*   `False`: Dark orange color (warning)

**Implementation Details:**

The widget is structurally similar to `TTMenu` but expects tuple values where `value[0]` is the boolean check result and `value[1]` is the display message. Keys are justified based on the longest key length.

**Code Reference:**[tt_tools_common/ui_common/widgets.py 213-243](https://github.com/tenstorrent/tt-tools-common/blob/892885f4/tt_tools_common/ui_common/widgets.py#L213-L243)

## Styling Integration

All widgets inherit global styles from `common_style.css` and can be further customized through CSS selectors. The styling system uses CSS variables for consistent color theming.

### Color Variables

| Variable | Value | Usage |
| --- | --- | --- |
| `$red` | `#f04f5e` | Errors, warnings, hover states |
| `$green` | `#1eb57d` | Borders, primary text, success states |
| `$yellow` | `#ffd10a` | Headers, keys, scrollbars, cursors |
| `$purple` | `#786bb0` | (Reserved for future use) |
| `$orange` | `darkorange` | Warning text, incompatibility notes |
| `$gray` | `ansi_bright_black` | Secondary text |

### Global Widget Styles

All widgets (`*` selector) receive:

*   Text color: `$yellow`
*   Scrollbar colors: `$yellow` (thumb), `$green` (track)
*   Border title: left-aligned, yellow, bold

### Widget-Specific Selectors

| CSS Selector | Target Widgets |
| --- | --- |
| `TTHeader, TTFooter` | Layout widgets |
| `DataTable` | Internal tables in TTDataTable |
| `TTMenu, TTCompatibilityMenu, TTHostCompatibilityMenu` | All menu-style widgets |
| `TTConfirmBox` | Modal confirmation dialogs |
| `TTHelperMenuBox` | Help screen modals |

For complete styling documentation, see [Theming and Styling](https://deepwiki.com/tenstorrent/tt-tools-common/4.3-theming-and-styling).

## Usage Example: Test Widget Application

The test application demonstrates how to compose multiple widgets into a functional TUI. This example is used for both manual testing and automated validation.

**Key Implementation Points:**

1.   **CSS Loading**: The app specifies `CSS_PATH = "../ui_common/common_style.css"` to load the theme
2.   **Widget Composition**: Widgets are yielded in `compose()` using context managers for container hierarchy
3.   **Dynamic Data**: Table rows are populated in `on_mount()` after widget tree is mounted
4.   **Widget Access**: `self.get_widget_by_id()` retrieves widget instances for manipulation
5.   **Modal Integration**: Modal screens are pushed using `self.push_screen(TTConfirmBox(...))`

## Widget Lifecycle and Event Handling

Understanding the Textual widget lifecycle is essential for proper widget usage:

**Lifecycle Methods by Widget:**

| Widget | `__init__` Parameters | `on_mount` Actions | `render` Behavior |
| --- | --- | --- | --- |
| TTHeader | Store app_name, app_version | Set 1-second refresh interval | Build 3-column grid with timestamp |
| TTFooter | Store content list, justify | None | Build N-column grid from content |
| TTDataTable | Initialize DataTable, add header | User populates data | Container renders DataTable |
| TTMenu | Store data dict, calculate justify_width | None | Build styled Text from dict |
| TTHostCompatibilityMenu | Check compatibility, set border styling | None | Build Text with conditional orange styling |
| TTCompatibilityMenu | Store data dict | None | Build Text with pass/fail coloring |
