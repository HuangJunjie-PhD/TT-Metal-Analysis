## Purpose and Scope

This document covers the test suite for Natural Language Processing (NLP) models in the pytorch2.0_ttnn repository. These tests validate that transformer-based language models can be successfully compiled and executed using the torch_ttnn backend, ensuring correctness through accuracy comparisons.

The NLP test suite includes:

*   **Question Answering**: BERT-based models for extractive QA
*   **Sequence Classification**: GPT-2 sentiment analysis models
*   **Causal Language Models**: Llama variants, Falcon, Bloom for text generation

For computer vision model tests, see [Computer Vision Model Tests](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/5.1-computer-vision-model-tests). For multi-modal models that combine vision and language, see [Multi-Modal and Speech Model Tests](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/5.3-multi-modal-and-speech-model-tests). For the underlying testing framework, see [ModelTester Framework](https://deepwiki.com/tenstorrent/pytorch2.0_ttnn/4.2-modeltester-framework).

## Common Testing Pattern

All NLP model tests inherit from `ModelTester` and implement two key methods:

| Method | Purpose | Returns |
| --- | --- | --- |
| `_load_model()` | Downloads and initializes the HuggingFace model, creates tokenizer | PyTorch model in bfloat16 |
| `_load_inputs()` | Tokenizes input text, creates input tensors | Dictionary of input tensors |

## Causal Language Model Tests

Causal language models (GPT-2, Llama, Falcon, Bloom) predict the next token given a sequence of input tokens. These tests validate autoregressive text generation.

### GPT-2 Sequence Classification

GPT-2 is tested with a sentiment classification fine-tune that outputs class logits rather than token logits [tests/models/gpt2/test_gpt2.py 12-25](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/gpt2/test_gpt2.py#L12-L25):

**Sources**: [tests/models/gpt2/test_gpt2.py](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/gpt2/test_gpt2.py)

### Llama Model Tests

Llama tests support multiple model variants and use a wrapper module to extract logits directly [tests/models/llama/test_llama.py 11-41](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py#L11-L41):

**Key Features**:

*   **Model Wrapper**: `LlamaModelModule` wraps the HuggingFace model and returns only logits [tests/models/llama/test_llama.py 11-17](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py#L11-L17)
*   **Parametrization**: Tests run across 4 different Llama variants [tests/models/llama/test_llama.py 52-55](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py#L52-L55)
*   **Batch Size**: Default batch size of 32 [tests/models/llama/test_llama.py 50](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py#L50-L50)
*   **Tokenizer Setup**: Sets `pad_token = eos_token` for proper padding [tests/models/llama/test_llama.py 23](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py#L23-L23)
*   **Disabled Optimization**: Uses `disable_load_params_once` fixture [tests/models/llama/test_llama.py 56](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py#L56-L56)

**Sources**: [tests/models/llama/test_llama.py](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/llama/test_llama.py)

### Falcon-7B Tests

Falcon tests use a 7B parameter instruction-tuned model with specific tokenization [tests/models/falcon/test_falcon.py 12-25](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/models/falcon/test_falcon.py#L12-L25):

## Integration with Testing Framework

This enables:

*   Metrics collection by [tests/conftest.py](https://github.com/tenstorrent/pytorch2.0_ttnn/blob/7ba2867e/tests/conftest.py)
