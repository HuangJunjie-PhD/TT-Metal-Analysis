This page documents the symbolic variable system in TTExaLens: the `ElfVariable` class and the `ParsedElfFile` entry points that construct instances of it. These components allow Python code to read and write firmware global variables from device memory using the same field names and types defined in the original C/C++ source, without manual offset calculation.

For background on how ELF and DWARF files are parsed to produce the type information that `ElfVariable` consumes, see [ELF and DWARF Parsing](https://deepwiki.com/tenstorrent/tt-exalens/7.3-elf-and-dwarf-parsing). For callstack unwinding, which also uses `ElfVariable` to read local variables, see [Callstack Unwinding](https://deepwiki.com/tenstorrent/tt-exalens/7.5-callstack-unwinding). For the high-level library API that wraps these operations (`read_global`, `get_global`, etc.), see [Symbolic Variable Access](https://deepwiki.com/tenstorrent/tt-exalens/3.8-symbolic-variable-access).

* * *

## Overview

The symbolic variable system bridges the DWARF debug information embedded in a firmware ELF file with live device memory. Given a parsed ELF and a memory access interface, you can obtain an `ElfVariable` that behaves like a Python object mirroring the C/C++ type — accessing struct fields with dot notation, indexing arrays, dereferencing pointers, and using arithmetic operators — all of which transparently read (or write) bytes from device memory.

## MemoryAccess Interface

`ElfVariable` does not know how memory is physically accessed. All reads and writes are delegated to a `MemoryAccess` object passed at construction time. The interface has two methods:

| Method | Signature | Description |
| --- | --- | --- |
| `read` | `read(address: int, size_bytes: int) -> bytes` | Read `size_bytes` from `address` |
| `write` | `write(address: int, data: bytes) -> None` | Write `data` starting at `address` |

Concrete implementations used in the codebase:

| Class | Backing store | When used |
| --- | --- | --- |
| `MemoryAccess` (live) | `RiscDebug.read_memory` / `write_memory` | Default; live hardware access via `MemoryAccess.create(risc_debug)` |
| `CachedReadMemoryAccess` | In-memory `bytes` snapshot | Returned by `ElfVariable.read()` — all reads served from a single snapshot |
| `FixedMemoryAccess` | Fixed `bytes` buffer | Used for compile-time constants resolved from DWARF |
| `RestrictedMemoryAccessError` | — | Exception raised when accessing memory outside safe regions |

## ElfVariable

`ElfVariable` is defined in [ttexalens/elf/variable.py 20-617](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L20-L617)

### Construction

*   `type_die` — an `ElfDie` with `tag_is("base_type")`, `tag_is("struct_type")`, `tag_is("array_type")`, `tag_is("pointer_type")`, etc. Determines layout.
*   `address` — the memory address of the variable's data.
*   `mem_access` — the `MemoryAccess` implementation to use for all reads and writes.

You rarely construct `ElfVariable` directly; instead use `ParsedElfFile.get_global` or `ParsedElfFile.read_global` (see [ParsedElfFile Entry Points](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ParsedElfFile%20Entry%20Points) below).

### Member Access

**How member access resolves from Python attribute lookup to memory**

`__getattr__` is only called when normal Python attribute lookup fails, so it will not shadow actual methods of `ElfVariable`. If a struct field has the same name as a Python method (e.g. `bytes`, `read`), use `get_member('bytes')` explicitly.

**Member resolution precedence:**

1.   Direct child by name via `ElfDie.get_child_by_name`
2.   Unnamed struct/union members (`_resolve_unnamed_struct_union_member`) — handles anonymous `union { ... };` inside a struct
3.   Inherited base class members (`_resolve_inheritance_member`) — handles C++ `struct Derived : Base`

Both `__getattr__` and `__getitem__` with a string key delegate to `get_member`.

### Pointer Operations

Applies to variables whose `type_die.tag_is("pointer_type")`.

| Expression | Method | Behavior |
| --- | --- | --- |
| `var.dereference()` | `dereference` | Reads the address stored at `var`'s memory location; returns `ElfVariable` at that address with the pointed-to type |
| `var[i]` | `__getitem__(int)` | Dereferences then computes `address + i * element_size` |
| `var.field` | `__getattr__` → `get_member` | Auto-dereferences before member lookup |

Pointer size is assumed to be 4 bytes (32-bit RISC-V targets).

### Value Reading and Writing

#### `read_value() -> int | float | bool`

Reads the raw bytes from device memory and converts them to a Python scalar. Applies to `base_type`, `pointer_type`, and `enumeration_type` DIEs.

| C type | Returned Python type | Conversion |
| --- | --- | --- |
| `uint8_t`, `uint32_t`, `uint64_t`, integer types | `int` | `int.from_bytes(..., byteorder="little")` |
| `float` | `float` | `struct.unpack("f", ...)` |
| `double` | `float` | `struct.unpack("d", ...)` |
| `bool` | `bool` | `bool(int.from_bytes(...))` |
| pointer | `int` | address of pointee |
| `enum class` / `enum` | `int` | underlying integer value |

Calling `read_value()` on a struct, array, or non-base type raises an exception.

#### `write_value(value, check_data_loss=True)`

Writes a Python scalar back to device memory. Validates for data loss (e.g. writing a float value that doesn't round-trip). Only applies to `base_type` and `enumeration_type`.

Sources: [ttexalens/elf/variable.py 169-236](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L169-L236)

### Cached Memory Snapshot: `read()`

`ElfVariable.read()` reads all bytes for the variable's type in a single call and returns a **new**`ElfVariable` backed by `CachedReadMemoryAccess`. Subsequent field and element accesses on the returned variable are served from the in-memory snapshot with no additional device reads.

This is the key difference between `ParsedElfFile.get_global` (lazy, each field access hits device memory) and `ParsedElfFile.read_global` (eager, one read captures the whole struct).

```
# get_global: each field access is a separate device read
var = parsed_elf.get_global("g_global_struct", mem_access)
x = var.a   # device read
y = var.b   # device read

# read_global: one bulk read, then all accesses from cache
var = parsed_elf.read_global("g_global_struct", mem_access)
x = var.a   # from cache
y = var.b   # from cache
```

### Operator Overloads

`ElfVariable` implements a comprehensive set of Python operator overloads. All operators call `read_value()` internally and return plain Python scalars (not `ElfVariable`).

**Arithmetic Operators**

| Operator | Forward | Reverse |
| --- | --- | --- |
| `+` | `__add__` | `__radd__` |
| `-` | `__sub__` | `__rsub__` |
| `*` | `__mul__` | `__rmul__` |
| `/` | `__truediv__` | `__rtruediv__` |
| `//` | `__floordiv__` | `__rfloordiv__` |
| `%` | `__mod__` | `__rmod__` |
| `**` | `__pow__` | `__rpow__` |

**Bitwise Operators** (integers and booleans only; floats raise `TypeError`)

| Operator | Forward | Reverse |
| --- | --- | --- |
| `&` | `__and__` | `__rand__` |
| `|` | `__or__` | `__ror__` |
| `^` | `__xor__` | `__rxor__` |
| `<<` | `__lshift__` | `__rlshift__` |
| `>>` | `__rshift__` | `__rrshift__` |

**Comparison Operators**

| Operator | Method | Notes |
| --- | --- | --- |
| `==` | `__eq__` | For arrays: element-by-element; for base types: value comparison |
| `<` | `__lt__` | Base types only |
| `<=` | `__le__` | `< or ==` |
| `>` | `__gt__` | Base types only |
| `>=` | `__ge__` | `> or ==` |

**Unary Operators**

| Operator | Method |
| --- | --- |
| `-var` | `__neg__` |
| `+var` | `__pos__` |
| `abs(var)` | `__abs__` |
| `~var` | `__invert__` (integers/booleans only) |

**Protocol methods**

| Method | Purpose |
| --- | --- |
| `__index__` | Allows `ElfVariable` to be used as a list/tuple index: `my_list[var]` |
| `__str__` | Returns `str(read_value())`; for `enumeration_type`, returns the enumerator path (e.g. `"EnumClass::VALUE_C"`) |
| `__repr__` | Returns diagnostic string including type name, tag, size, address, and value |
| `__hash__` | `hash(read_value())` for base types |
| `__format__` | Allows `format(var, 'x')` and f-string formatting |

Sources: [ttexalens/elf/variable.py 238-600](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L238-L600)

## ParsedElfFile Entry Points

`ParsedElfFile` (in [ttexalens/elf/parsed.py 145-298](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L145-L298)) provides factory methods that look up variables in the DWARF info and construct `ElfVariable` instances.

**How ParsedElfFile entry points map to ElfVariable**

### `get_global(name, mem_access, allow_fallback=False) -> ElfVariable`

Returns a live `ElfVariable`; each field/element access triggers a device memory read. If the DWARF entry for `name` is a `const` pointer variable (i.e. `g_global_const_struct_ptr` pattern), automatically dereferences and returns the pointee type at the stored address.

### `read_global(name, mem_access, allow_fallback=False) -> ElfVariable`

Equivalent to `get_global(...).read()`. Reads all bytes for the named variable in one call and returns a snapshot-backed `ElfVariable`. Subsequent accesses do not touch device memory.

### `get_constant(name, allow_fallback=False) -> int | float | bool`

Looks up a compile-time constant (`constexpr` or `const`) in the DWARF info and returns its value directly as a Python scalar without any device memory access. The constant's value is stored in the DWARF `DW_AT_const_value` attribute.

### `get_enum_value(name, allow_fallback=False) -> int | None`

Returns the integer value of a named enumerator (e.g. `"EnumClass::VALUE_C"` → `2`). Used together with `ElfVariable.write_value()` to write enum fields by symbolic name.

Sources: [ttexalens/elf/parsed.py 206-298](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/parsed.py#L206-L298)

## Type System Mapping

The following table shows how C/C++ types map to `ElfDie` tags and how `ElfVariable` handles them.

| C/C++ construct | `ElfDie.tag` | `ElfVariable` behavior |
| --- | --- | --- |
| `uint8_t`, `int`, `float`, `double`, `bool` | `DW_TAG_base_type` | `read_value()` / `write_value()` |
| `struct Foo { ... }` | `DW_TAG_structure_type` | Member access via `__getattr__` / `get_member` |
| `union Bar { ... }` | `DW_TAG_union_type` | Member access; all offsets are 0 |
| `uint8_t arr[16]` | `DW_TAG_array_type` | `__len__`, `__iter__`, `__getitem__(int)` |
| `uint32_t*` | `DW_TAG_pointer_type` | `dereference()`, `__getitem__(int)` |
| `enum class Foo : uint32_t` | `DW_TAG_enumeration_type` | `read_value()` returns `int`; `__str__` returns enumerator path |
| `struct Derived : Base` | `DW_TAG_inheritance` child | `_resolve_inheritance_member()` walks base classes |
| anonymous `union { uint32_t packed; struct { ... }; }` | unnamed `DW_TAG_member` | `_resolve_unnamed_struct_union_member()` |
| `typedef` / `const` / `volatile` qualifiers | `DW_TAG_typedef`, `DW_TAG_const_type`, `DW_TAG_volatile_type` | Transparently resolved by `ElfDie.resolved_type` |

Sources: [ttexalens/elf/variable.py 92-193](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L92-L193)[ttexalens/elf/die.py 137-204](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/die.py#L137-L204)[riscv-src/globals_test.cc 1-141](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/riscv-src/globals_test.cc#L1-L141)

## Error Handling

All operator overloads and `read_value()` propagate the following exceptions without swallowing them:

| Exception | Source | When raised |
| --- | --- | --- |
| `TimeoutDeviceRegisterError` | `ttexalens/umd_device.py` | Hardware register read timed out |
| `RiscHaltError` | `ttexalens/hardware/risc_debug.py` | RISC-V core is not halted when a halt is required |
| `RestrictedMemoryAccessError` | `ttexalens/memory_access.py` | Address is outside a safe-to-read memory region |

`__str__`, `__repr__`, `__hash__`, and `__format__` catch general exceptions and fall back gracefully, but re-raise `TimeoutDeviceRegisterError` so that hardware timeouts are never silently swallowed.

Sources: [ttexalens/elf/variable.py 511-600](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/ttexalens/elf/variable.py#L511-L600)[test/ttexalens/unit_tests/test_debug_symbols.py 432-507](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_debug_symbols.py#L432-L507)

## Example Usage

The test file [test/ttexalens/unit_tests/test_debug_symbols.py](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/test/ttexalens/unit_tests/test_debug_symbols.py) and the corresponding RISC-V firmware [riscv-src/globals_test.cc](https://github.com/tenstorrent/tt-exalens/blob/046c35eb/riscv-src/globals_test.cc) together demonstrate the full range of supported operations.

**Struct field access (operator overloads make comparison natural)**

```
# Direct field access using dot notation
g_global_struct.a == 0x11223344          # ElfVariable.__eq__
g_global_struct.u.f32 == 0.5             # nested struct, then base type
str(g_global_struct.enum_class_field)    # -> "EnumClass::VALUE_C"
```

**Array iteration**

```
# Iterate over c[16]
for element in g_global_struct.c:
    print(element.read_value())

# Or as a value list
g_global_struct.c.as_value_list()    # -> [0, 1, 2, ..., 15]
```

**Pointer dereference and pointer arithmetic**

```
g_global_struct.uint_pointer.dereference()    # -> ElfVariable at uint_array[0]
g_global_struct.uint_pointer[1]               # -> ElfVariable at uint_array[1]
```

**Write a field by enum name**

```
enum_val = parsed_elf.get_enum_value("EnumClass::VALUE_D")  # -> 3
g_global_struct.enum_class_field.write_value(enum_val)
```

**Snapshot-based bulk read**

```
var = parsed_elf.read_global("g_global_struct", mem_access)
# All subsequent accesses come from the in-memory snapshot; read_count stays at 1
x = var.a
y = var.b
z = var.c[5]
```
