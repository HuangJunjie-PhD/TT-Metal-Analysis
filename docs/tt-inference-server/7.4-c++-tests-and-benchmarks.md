This page details the C++ testing infrastructure for the `tt-media-server` backend. The test suite is managed via CTest and covers unit testing of core LLM engine components, integration testing of IPC mechanisms, and performance benchmarking of critical utilities like tokenizers.

## Overview

The C++ testing suite is designed to validate the high-performance components of the inference server without requiring a full hardware stack for every test. It utilizes a `MockModelRunner` to simulate hardware execution while exercising the complex logic of the `Scheduler`, `BlockManager`, and `Sequence` lifecycle.

### Key Components Under Test

*   **Engine Logic**: Validating `Sequence` state transitions, `BlockManager` Paged Attention allocation, and `Scheduler` batching policies.
*   **IPC & Synchronization**: Smoke tests for shared memory queues and multi-threaded consumer loops.
*   **Performance**: Benchmarks for tokenization speed and scheduler overhead.
*   **Correctness**: Round-trip validation for tokenizers (Llama, DeepSeek) and guided decoding.

* * *

## Engine Unit Tests

These tests focus on the `tt::runners::llm_engine` namespace, ensuring that the orchestration of requests into hardware-ready batches is robust.

### Sequence and Block Management

The `sequence_test` and `block_manager_test` validate how memory is managed during inference.

*   **Sequence Lifecycle**: Tests serialization/deserialization, token appending, and status tracking (`WAITING`, `RUNNING`, `FINISHED`, `ABORTED`) [tt-media-server/cpp_server/include/runners/llm_runner.cpp 20-21](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/runners/llm_runner.cpp#L20-L21)
*   **Paged Attention**: The `BlockManager` is tested for its ability to allocate blocks, handle cache hits via hashing, and correctly deallocate blocks when a sequence finishes [tt-media-server/cpp_server/include/runners/llm_runner.hpp 32](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/include/runners/llm_runner.hpp#L32-L32)

### Scheduler Policies

The `scheduler_test` validates the two primary scheduling strategies [tt-media-server/cpp_server/tests/scheduler_test.cpp 53-60](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L53-L60):

1.   **Prefill-First**: Always prefills new requests when the `prefill_queue` has waiting requests. Decode is only attempted when nothing can be prefilled [tt-media-server/cpp_server/README.md 113-117](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/README.md?plain=1#L113-L117)
2.   **Max-Occupancy**: Keeps the device at full occupancy (`max_in_flight_count`) whenever possible, immediately refilling capacity when slots free up [tt-media-server/cpp_server/README.md 131-134](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/README.md?plain=1#L131-L134)

The scheduler is tested for its ability to handle preemption when KV cache limits are reached and its handling of `abortRequest` signals to clean up resources [tt-media-server/cpp_server/tests/scheduler_test.cpp 207-210](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/scheduler_test.cpp#L207-L210)

## Tokenizer Tests and Benchmarks

Correctness of tokenization is critical for model accuracy. The C++ backend includes specific tests for Llama and DeepSeek tokenizers.

### Correctness Testing

`test_tokenizer.cpp` performs validation for the `tt::utils::tokenizers::Tokenizer` hierarchy:

*   **Round-trip**: Ensures `decode(encode(text)) == text`.
*   **Active Tokenizer**: Tests the selection of tokenizer strategy (DeepSeek vs Llama) based on `LLM_DEVICE_BACKEND`[tt-media-server/cpp_server/CLAUDE.md 97-100](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/CLAUDE.md?plain=1#L97-L100)

### Performance Benchmarking

The `tokenizer_benchmark` measures:

*   **Encoding Latency**: Time to convert raw text to token IDs [tt-media-server/cpp_server/CLAUDE.md 63-65](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/CLAUDE.md?plain=1#L63-L65)
*   **Vocab Initialization**: Measures the overhead of loading `tokenizer.json` and `tokenizer_config.json` for supported models [tt-media-server/cpp_server/build.sh 175-183](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/build.sh#L175-L183)

## IPC and Integration Smoke Tests

The server uses a multi-process architecture where the main process communicates with `SingleProcessWorker` instances via shared memory queues.

### IPC Smoke Tests

*   **`ipc_scheduler_smoke_test`**: A multi-process test where a parent process pushes sequences to a `BoostIpcTaskQueue` and a child process deserializes them into a `Scheduler` to verify batch integrity [tt-media-server/cpp_server/tests/ipc_scheduler_smoke_test.cpp 39-68](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/tests/ipc_scheduler_smoke_test.cpp#L39-L68)
*   **Cancellation Test**: Validates that `cancelQueue` signals correctly propagate to the `LLMRunner` to abort active sequences [tt-media-server/cpp_server/src/runners/llm_runner.cpp 157-164](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/runners/llm_runner.cpp#L157-L164)
*   **Warmup Signaling**: Tests the `BoostIpcWarmupSignalQueue` used by workers to notify the main process when model initialization is complete [tt-media-server/cpp_server/src/worker/single_process_worker.cpp 105-112](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/src/worker/single_process_worker.cpp#L105-L112)

## CI Performance Thresholds and Nightly Benchmarks

The project uses GitHub Actions to enforce performance stability.

### Test Gate (PR Level)

The `test-gate.yml` workflow triggers on every pull request to ensure no regressions in the C++ server.

*   **`cpp_server` filter**: Detects changes in the C++ backend code [.github/workflows/test-gate.yml 27-28](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/%20.github/workflows/test-gate.yml#L27-L28)
*   **Unit Tests**: Executes the CTest suite, including `scheduler_test`, `llm_runner_test`, and `sequence_test`[tt-media-server/cpp_server/README.md 149-158](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/README.md?plain=1#L149-L158)
*   **Sanitizers**: PRs can be manually tested with `--asan` (AddressSanitizer) or `--tsan` (ThreadSanitizer) to detect memory leaks or data races [tt-media-server/cpp_server/build.sh 26-35](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/build.sh#L26-L35)

### Nightly Benchmarks

A separate `nightly-cpp-benchmarks.yml` workflow runs extended performance tests:

*   **`run_benchmarks.sh`**: A wrapper around `vllm bench serve` that sweeps through Input Sequence Length (ISL), Output Sequence Length (OSL), and Concurrency [tt-media-server/cpp_server/benchmarks/run_benchmarks.sh 43-72](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/benchmarks/run_benchmarks.sh#L43-L72)
*   **Automated Plotting**: Uses `plot_benchmarks.py` to generate latency and throughput graphs from the saved JSON results [tt-media-server/cpp_server/benchmarks/run_benchmarks.sh 76](https://github.com/tenstorrent/tt-inference-server/blob/8ab207f9/tt-media-server/cpp_server/benchmarks/run_benchmarks.sh#L76-L76)
