The **Release Summarization Pipeline** is an automated backend component that monitors the Tenstorrent ecosystem for new software releases, generates human-readable summaries using Large Language Models (LLMs), and surfaces them on the **Planet Tenstorrent** feed. This pipeline transforms raw technical release notes into engaging, one-paragraph summaries suitable for developers and researchers [docs/superpowers/specs/2026-06-05-release-summarization-design.md 6-12](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-06-05-release-summarization-design.md?plain=1#L6-L12)

## Pipeline Overview

The pipeline is implemented in `scripts/summarize_releases.py` and runs as part of the nightly automation suite [scripts/summarize_releases.py 5-15](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L5-L15) It operates by comparing the latest metadata from the GitHub API against the existing feed to identify new, substantial releases that warrant a summary.

### Key Logic Flow

1.   **Detection**: Loads `github_meta.json` to find all current releases across watched repositories [scripts/summarize_releases.py 167-174](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L167-L174)
2.   **Deduplication**: Filters out releases already present in `planet_feeds.json`[scripts/summarize_releases.py 176-185](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L176-L185)
3.   **Filtering**: Skips pre-releases (dev, RC, QA tags) and "sparse" releases where the body is empty or shorter than the `SPARSE_LIMIT` (150 characters) [scripts/summarize_releases.py 33](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L33-L33)[scripts/summarize_releases.py 98-100](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L98-L100)
4.   **Summarization**: Fetches the full markdown body from the GitHub REST API and sends it to the Anthropic Claude API [scripts/summarize_releases.py 113-147](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L113-L147)
5.   **Persistence**: Appends the generated summary as a `type: "release"` item to `planet_feeds.json`[scripts/summarize_releases.py 11](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L11-L11)

## LLM Prompt Design

The pipeline uses the `claude-haiku-4-5-20251001` model to ensure fast and cost-effective generation [scripts/summarize_releases.py 32](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L32-L32) The prompt is designed to produce a "colleague-to-colleague" tone rather than marketing hype.

### System Prompt

The `SYSTEM_PROMPT` instructs the model to write 2–4 sentences focusing on technical impact while explicitly forbidding "hype words" (e.g., "exciting", "powerful") and bullet points [scripts/summarize_releases.py 103-110](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L103-L110)

### User Message Construction

The `call_summarization_model` function constructs a message containing:

*   **Project Name & Affiliation**: Resolved via `build_affiliation_map()`[scripts/summarize_releases.py 53-67](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L53-L67)
*   **Release Tag**: The specific version being summarized.
*   **Raw Notes**: The full markdown content fetched from GitHub [scripts/summarize_releases.py 115-120](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L115-L120)

## Implementation Details

### Sparse Check and Filtering

To maintain high quality in the Planet feed, the script applies strict filters:

*   **SPARSE_LIMIT**: Any release body under 150 characters (after stripping whitespace) is ignored to avoid summarizing trivial "bug fix" or "placeholder" notes [scripts/summarize_releases.py 33](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L33-L33)[scripts/summarize_releases.py 98-100](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L98-L100)
*   **Pre-release Filtering**: The script uses regex to detect and skip tags containing `.dev`, `-rc`, or `-qa`[tests/test_summarize_releases.py 110-121](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/tests/test_summarize_releases.py#L110-L121)

### Affiliation Resolution

The pipeline ensures that each release item is correctly labeled with its affiliation (`official`, `community`, or `affiliated`). It pre-scans all JSON files in the `entries/` directory to build a mapping between repository URLs and their affiliation status [scripts/summarize_releases.py 53-67](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L53-L67)

## Operational Modes

### Dry-Run Mode

The script supports a `--dry-run` flag [scripts/summarize_releases.py 160](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L160-L160) In this mode:

*   The script performs all lookups and API calls (including LLM summarization).
*   The generated summaries are printed to `stdout` for inspection.
*   No changes are written to `planet_feeds.json`[scripts/summarize_releases.py 14-15](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/scripts/summarize_releases.py#L14-L15)

### GitHub Actions Integration

The pipeline is integrated into `nightly.yml`. It is gated on the presence of the `ANTHROPIC_API_KEY` secret; if the key is missing, the summarization step is skipped entirely to prevent workflow failures [docs/superpowers/specs/2026-06-05-release-summarization-design.md 134-139](https://github.com/tenstorrent/tt-awesome/blob/1d86dbe5/docs/superpowers/specs/2026-06-05-release-summarization-design.md?plain=1#L134-L139)

| Feature | Configuration |
| --- | --- |
| **Schedule** | Nightly (after `fetch_github_meta.py`) |
| **Secrets Required** | `ANTHROPIC_API_KEY`, `GITHUB_TOKEN` |
| **Output Path** | `src/_data/planet_feeds.json` |
| **Approval Policy** | `approved: false` (Requires manual PR review) |
