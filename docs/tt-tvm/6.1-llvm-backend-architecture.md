## Purpose and Scope

This document describes the architecture of TVM's LLVM-based code generation backend, which transforms TensorIR (TIR) programs into executable machine code for CPUs and GPUs. The LLVM backend supports multiple hardware targets including x86-64, ARM, NVPTX (NVIDIA GPUs), AMDGPU (AMD GPUs), and Hexagon DSP.

For information about the TensorIR representation that feeds into this backend, see [TensorIR (TIR) System](https://deepwiki.com/tenstorrent/tt-tvm/5-tensorir-(tir)-system). For details on the runtime systems that execute the compiled code, see [Runtime and Execution](https://deepwiki.com/tenstorrent/tt-tvm/7-runtime-and-execution).

The LLVM backend uses a hierarchical code generator architecture where `CodeGenLLVM` serves as the base class, and target-specific generators inherit and customize behavior for their respective platforms.

The code generation system uses inheritance to share common functionality while allowing platform-specific customization.

### Expression and Statement Visitors

The `CodeGenLLVM` class implements visitor methods for all TIR expression and statement types:

**Expression Visitors** (return `llvm::Value*`):

*   `VisitExpr_(const VarNode*)` - Load variable from `var_map_`
*   `VisitExpr_(const IntImmNode*)` - Create integer constant
*   `VisitExpr_(const AddNode*)`, `VisitExpr_(const MulNode*)`, etc. - Binary operations
*   `VisitExpr_(const BufferLoadNode*)` - Memory load with TBAA metadata
*   `VisitExpr_(const CallNode*)` - Function calls and intrinsics

**Statement Visitors** (return `void`):

*   `VisitStmt_(const BufferStoreNode*)` - Memory store with TBAA metadata
*   `VisitStmt_(const ForNode*)` - Loop generation
*   `VisitStmt_(const IfThenElseNode*)` - Conditional branches
*   `VisitStmt_(const AllocateNode*)` - Memory allocation
*   `VisitStmt_(const AttrStmtNode*)` - Attribute annotations

## Target-Specific Implementations

### CodeGenCPU

`CodeGenCPU` extends `CodeGenLLVM` for CPU targets, adding support for TVM's packed function calling convention and parallel execution.

**Packed Function Call Generation:**

The `CreateCallPacked()` method generates code to call TVM packed functions through the `TVMFuncCall` interface:

1.   Pack arguments into `TVMValue` array
2.   Pack type codes into `int` array
3.   Call `TVMFuncCall` with packed arguments
4.   Check return code and unpack result

Sources: [src/target/llvm/codegen_cpu.cc 77-189](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_cpu.cc#L77-L189)[src/target/llvm/codegen_cpu.h 95-123](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_cpu.h#L95-L123)

### CodeGenAMDGPU

`CodeGenAMDGPU` generates code for AMD GPUs using ROCm, with similar structure to NVPTX but different intrinsics and address spaces.

**Key Differences from NVPTX:**

*   Uses `amdgcn_workitem_id_*` and `amdgcn_workgroup_id_*` intrinsics
*   Sets calling convention to `AMDGPU_KERNEL`
*   Adds `amdgpu-flat-work-group-size` attribute
*   Uses address space 1 for global memory (vs. 0 for NVPTX)
*   Different atomic operation intrinsics

Sources: [src/target/llvm/codegen_amdgpu.cc 93-100](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_amdgpu.cc#L93-L100)[src/target/llvm/codegen_amdgpu.cc 159-195](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_amdgpu.cc#L159-L195)[src/target/llvm/codegen_amdgpu.cc 217](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_amdgpu.cc#L217-L217)

### CodeGenHexagon

`CodeGenHexagon` generates code for Qualcomm Hexagon DSP, with special support for HVX (Hexagon Vector Extensions) and the QHL (Qualcomm Hexagon Library).

**Hexagon-Specific Features:**

*   Configurable HVX vector length (64 or 128 bytes)
*   Vector lookup table intrinsics via `vlutvvbi` instructions
*   Integration with QHL math functions for trigonometric operations
*   Support for 2D buffer indexing (pointer-of-pointers)

Sources: [src/target/llvm/codegen_hexagon.cc 127-146](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_hexagon.cc#L127-L146)[src/target/llvm/codegen_hexagon.cc 396-485](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_hexagon.cc#L396-L485)[src/target/llvm/codegen_hexagon.cc 113-118](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_hexagon.cc#L113-L118)

## Optimization Pipeline

After generating LLVM IR, the code goes through LLVM's optimization passes. TVM supports both the legacy pass manager (LLVM < 16) and the new pass manager (LLVM >= 16).

### New Pass Manager (LLVM >= 16)

Sources: [src/target/llvm/codegen_llvm.cc 409-485](https://github.com/tenstorrent/tt-tvm/blob/02ad9c28/src/target/llvm/codegen_llvm.cc#L409-L485)

### Legacy Pass Manager (LLVM < 16)

**Key Optimizations:**

*   **Inlining**: Function inlining with `OptLevel`-based thresholds
*   **Loop Vectorization**: Enabled via `LoopVectorize = true`
*   **SLP Vectorization**: Superword-level parallelism via `SLPVectorize = true`
*   **Target-specific**: Via `adjustPassManager()` hook

## Memory Management

### Address Spaces

Different address spaces are used for different memory regions, especially on GPU targets:

| Target | Address Space | Purpose |
| --- | --- | --- |
| CPU | 0 | Default address space |
| NVPTX | 0 | Global memory |
| NVPTX | 3 | Shared memory |
| NVPTX | 5 | Local memory (conceptual) |
| AMDGPU | 1 | Global memory |
| AMDGPU | 3 | Shared (LDS) memory |
| AMDGPU | 5 | Local memory |
